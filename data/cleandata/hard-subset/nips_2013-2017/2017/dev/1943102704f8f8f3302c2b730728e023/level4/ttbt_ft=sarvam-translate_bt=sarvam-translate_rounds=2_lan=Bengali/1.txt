[Original Review]

The paper discusses YASS: Yet Another Spike Sorter, a well-designed pipeline for selecting multi-electrode array (MEA) recordings. The method is sound, creating a complete pipeline combining many advanced techniques. The paper is well-written, follows a clear structure, and provides convincing evidence that the work truly enhances the state of the art in retinal MEA data selection.

The paper itself is a valuable contribution, but I believe its potential is even greater if the authors addressed some issues in detail below. In summary, my main concerns are:

(1) Lack of publicly available code,
(2) Weak description of the neural network recognition method,
(3) Difficulty in applying the pipeline to cortical data.

Details:

(1) The code is not available (or at least no URL is given in the manuscript). Since spike selection is primarily an engineering problem at the moment (but an important one), a general description of the method is half as valuable as the actual code implementing the pipeline. Therefore, I encourage the authors to make additional efforts to make the code available.
(2) Neural network spike recognition. This seems to be the only innovative part (all other components of the pipeline have been described/used before). However, it is still unclear to me how the authors created their training data. Section C.2 describes various ways to create training data, but it is not clear which way (or combination) the authors used.
(a) Using pre-existing selections. Firstly, most labs have pre-existing selections when going to dense MEA, but the data obtained is not selected correctly, because they do not have any pipeline for selecting such array data, and - as the authors noted - existing methods cannot be properly scaled.
Secondly, it is not clear to me how pre-existing selections will help train a more robust neural network for recognition. Did the authors inspect each waveform snippet and identify them as clear? If not, on what algorithm did they base their decision on which waveform snippets of the training data are clear? Why do they need to train a neural network using only this algorithm? How do they deal with incorrect alignment, which creates label noise?
If the authors use pre-existing selections, they need to provide more information about how they did it and why it works. In its current form, their work cannot be reproduced.
(b) Creating synthetic training data by imposing waveform templates on background noise. This could be a reasonable approach. Is it used to augment data or not at all, and only described as a possible alternative? What is the evidence that this method is effective? Synthetic data may not represent real recordings.
(c) Generalization to cortical data. I am quite confident that the pipeline works well for retinal data, but I doubt it will do so for cortical data (some arguments are given below). I think this limitation needs to be discussed and acknowledged more clearly (summary, introduction, conclusion).
(a) In cortical recordings, waveform drift is a serious problem that is seen in almost all non-long-duration recordings (and high-density MEA that works long-duration has not yet been demonstrated). Therefore, modeling drift is absolutely essential for recordings lasting more than a few minutes.
(b) Finding good training data for NN recognition is even more difficult. Good ground truth (or well-verified data as described in Appendix I) is not available, and creating synthetic data as described in C.2 may not be realistic, because spikes also create background noise and neurons often fire in a highly correlated manner (as a result of a problematic method of imposing templates on spike-free noise).

Small Comments:

- Panel below figure 3: The Y axis is strange. 1^-x means 10^-x? Also, it exaggerates the small difference in accuracy between 0.99 and 0.999, where both methods are essentially perfect.
- The authors use spatially whitened data (according to Section 2.1), but I did not find any description of the spatially whitening method in the manuscript or appendix.

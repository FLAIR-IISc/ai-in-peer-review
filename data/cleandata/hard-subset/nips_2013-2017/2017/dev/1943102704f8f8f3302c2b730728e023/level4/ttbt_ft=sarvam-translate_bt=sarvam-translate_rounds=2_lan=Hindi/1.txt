[Original Review]

The paper describes YASS: Yet Another Spike Sortor, a well-engineered pipeline for sorting multi-electrode array (MEA) recordings. The approach is solid, combining several cutting-edge approaches into a complete pipeline. The paper is well-written, follows a clear structure, and provides solid evidence that this work truly advances the state of the art for sorting retinal MEA data.

While the paper itself is a valuable contribution, I feel it has a lot of potential if the authors addressed some of the issues below. In summary, my main concerns are:

(1) Lack of publicly available code,
(2) Poor description of the neural network recognition method,
(3) Problems applying the pipeline to cortical data.

Details:

(1) Code is not available (or at least the URL is not provided in the manuscript). Since spike sorting is primarily an engineering problem at this time (but not an insignificant one), the mere description of the approach is not as valuable as the actual code implementing the pipeline. Thus, I strongly encourage the authors to make additional efforts and make the code available.

(2) Neural network spike detection. This part seems the only truly innovative (all other components of the pipeline have been described/used before). However, it is not clear to me how the authors generated their training data. Section C.2 describes various methods for generating training data, but it is not clear which (or which combination) the authors used.

(a) Using a pre-existing sort.

First, existing, well-sorted data are not available in most labs when going to dense MEA, because they do not have a pipeline to sort such array data and – as the authors have stated – existing methods do not scale well.

Second, it is not clear to me how the existing sort will help train a more robust neural network for recognition. Did the authors inspect each single wave form snippet and label it as clean or not? If not, on what algorithm did they decide which wave form snippets were clean in the training data? Why do they need to train the neural network instead of just using this algorithm? How do they deal with misalignment, which creates label noise?

If the authors used a pre-existing sort, they need to provide more information about how they actually did it and why it works. As it is, their work cannot be reproduced.

(b) Generating synthetic training data by superimposing wave form templates on background noise. This could be a reasonable approach. Is it used for data augmentation or not at all and is described only as a possible option? What is the evidence that this approach is useful? Synthetic data cannot be a representative of the actual recording.

(3) Generalization for cortical data. I am quite sure that the pipeline works well for retinal data, but I doubt it will do so for cortical data (some arguments are given below). I think it needs to be discussed and more clearly accepted (abstract, introduction, conclusion).

(a) In cortical recordings, wave form drift is a serious issue that arises in almost all non-chronic recordings (and high-density MEA that works chronically, is still to be demonstrated). Thus, drift modeling is absolutely crucial for recordings lasting more than a few minutes.

(b) Obtaining good training data for NN recognition is more difficult. Good ground truth (or well-validated data as described in Appendix I) is not available and it is not necessary to generate synthetic data as described in C.2, because background noise is also caused by spikes and neurons that often fire in highly correlated ways (thus making the approach of overlaying templates on spike-free noise problematic).

Minor Comments:

- Figure 3 bottom panel: The Y axis is strange. Does 1^-x mean 10^-x? Also, it exaggerates the small accuracy difference between 0.99 and 0.999, where both methods are essentially correct.
- The authors use spatially white data (according to Section 2.1), but I did not find a description of the spatial whitening process in the manuscript or appendix.

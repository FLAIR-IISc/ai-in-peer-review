**Summary**

This paper introduces the "Radon Machine," a novel parallelization scheme for a broad class of regularized risk minimization algorithms. The method operates by first training a base learning algorithm on multiple small, disjoint subsets of the data in parallel to obtain a set of "weak" hypotheses. These hypotheses are then iteratively aggregated in a tree-like structure. At each level of the tree, groups of hypotheses are combined by computing their Radon point, a concept from convex geometry. The authors provide a theoretical analysis showing that this aggregation scheme reduces the probability of generating a poor hypothesis doubly-exponentially with the height of the aggregation tree. This allows the algorithm to achieve the same confidence guarantees as a sequential learner in polylogarithmic time, provided a quasi-polynomial number of processors and a larger dataset are available. The empirical evaluation demonstrates significant speedups compared to sequential base learners and competitive performance against parallel algorithms in Spark's MLlib.

**Strengths**

1.  **Novelty:** The application of Radon points for aggregating hypotheses in a machine learning context is a novel and theoretically interesting contribution. It provides a principled alternative to simple averaging, which is a common baseline for this type of "embarrassingly parallel" setup.
2.  **Theoretical Contribution:** The paper provides a formal proof that the Radon aggregation scheme improves the confidence bounds doubly-exponentially (Theorem 3). This is a strong result that forms the basis for the claimed reduction in runtime complexity from polynomial to polylogarithmic (Theorem 4).
3.  **Black-Box Approach:** The proposed method is appealing from a practical standpoint as it treats the base learning algorithm as a black box. This allows for broad applicability without requiring modifications to existing, well-tested learning algorithms.

**Weaknesses**

Despite the novelty of the core idea, the paper is beset by a substantial number of flaws, ranging from minor presentational issues to significant limitations in the theoretical analysis and empirical evaluation that undermine the central claims.

1.  **Clarity and Presentation:**
    *   The manuscript contains several formatting inconsistencies. A footnote with the conference name ("st Conference on Neural Information Processing Systems (NIPS 2017)...") is present on the first page, suggesting it is an unedited template artifact.
    *   The notation is occasionally imprecise. For instance, in Equation (1), `L(D)` is used to denote the `argmin` operation, whereas it typically represents the hypothesis returned by that operation. In Algorithm 1, the dataset `D` is divided into "iid subsets," which is incorrect; partitioning a finite sample results in subsets that are not statistically independent. This lack of precision is concerning for a paper with a strong theoretical focus.
    *   The description of the empirical results can be ambiguous. In Section 4, the paper claims the Radon machine outperforms averaging "by at least 10%". It is unclear whether this refers to a 10% relative improvement or a 0.1 absolute improvement in AUC, which are vastly different claims.

2.  **Theoretical Soundness and Assumptions:** The main theoretical results rely on several strong and potentially impractical assumptions, the implications of which are not adequately discussed.
    *   A critical assumption, mentioned just before Equation (5), is that the base learner's failure probability satisfies `δ ≤ 1/2r`. This requires the base learner to already be highly reliable even when trained on small data subsets. The entire theoretical framework collapses if this condition is not met.
    *   The core result in Theorem 4, which claims a reduction to polylogarithmic runtime, is conditional on the Radon number `r` being "polylogarithmic in the sample complexity". Since `r = d+2` for a `d`-dimensional Euclidean space, this is equivalent to requiring the feature dimensionality `d` to be polylogarithmic in the sample complexity. This is a severe restriction that makes the theorem inapplicable to many high-dimensional problems common in machine learning. This limitation is not presented as a major caveat but rather as a minor condition.
    *   The method requires a "quasi-polynomial number of processing units" (`r^h`). For a `d`-dimensional problem, this is `(d+2)^h`. Since `h` is chosen to be logarithmic in the original sample size `M`, the number of required processors can be `(d+2)^log(M)`, which is practically infeasible for even moderately-sized problems. The paper glosses over the impracticality of this requirement.

3.  **Empirical Rigor:**
    *   The choice of `h=max` is defined by a heuristic where each base learner must see at least 100 samples. This threshold is arbitrary and not justified. A sensitivity analysis on this crucial parameter is missing.
    *   The description of hyperparameter optimization is vague ("optimised on an independent split of the datasets"), which harms reproducibility.
    *   The authors' admission that "it is unclear what caused the consistently weak performance of all algorithms on wikidata" is concerning. A rigorous evaluation would involve investigating such anomalies rather than simply reporting them as unexplained. This suggests a superficial analysis of the results.
    *   The impressive speedup claim ("80 to around 700-times faster") is based on a comparison against a sequential learner on a single processor. This is a weak baseline for a paper on parallelization. A more informative metric would be speedup efficiency (speedup divided by the number of processors) or a direct comparison to other parallel methods on an identical number of processors.

**Recommendation**

While the paper introduces a novel and elegant theoretical idea, its execution is flawed. The manuscript suffers from a lack of clarity and precision in its writing and notation. More importantly, the main theoretical claims rest on strong, potentially impractical assumptions regarding data dimensionality and base learner accuracy, which are not adequately acknowledged. The empirical evaluation, while showing promising speedups, lacks rigor in its parameter selection, reproducibility, and analysis of anomalous results.

The accumulation of these numerous small and large flaws significantly detracts from the paper's contribution. An idea of this novelty deserves a more careful and honest treatment of its limitations. In its current form, the paper does not meet the high standards of NIPS.

**Arguments for Acceptance:**
*   Introduces a genuinely new approach to model aggregation based on Radon points.
*   Provides a strong theoretical result (doubly-exponential confidence improvement) that is interesting in its own right.

**Arguments for Rejection:**
*   Numerous issues with clarity, imprecise notation, and formatting.
*   Theoretical claims hinge on strong and potentially unrealistic assumptions (low dimensionality, high base learner accuracy) whose impact is downplayed.
*   The requirement of a quasi-polynomial number of processors is a major practical limitation that is not given sufficient weight.
*   The empirical evaluation lacks rigor and contains arbitrary parameter choices and unexplained results.

**Decision:** Reject

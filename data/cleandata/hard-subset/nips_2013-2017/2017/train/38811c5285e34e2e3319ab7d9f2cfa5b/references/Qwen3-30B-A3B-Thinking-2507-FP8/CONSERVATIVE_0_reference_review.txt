

# Review of "Effective Parallelisation for Machine Learning"

This paper proposes a "Radon machine" for parallelizing machine learning algorithms, claiming to reduce runtime from polynomial to polylogarithmic in sample size while maintaining the same confidence/error bounds. The core idea involves dividing data into subsets, running a base learning algorithm on each subset in parallel, and aggregating the results using Radon points instead of averaging. The authors claim this approach achieves an 80-700x speedup in empirical evaluations while maintaining comparable predictive performance.

## Strengths

The paper presents a theoretically grounded approach to parallel machine learning with a clear mathematical framework. The use of Radon points for aggregation is novel and the theoretical analysis (Proposition 2 and Theorem 3) provides a rigorous bound on the error reduction. The empirical evaluation is well-structured, with appropriate metrics (AUC, RMSE) and comparison against relevant baselines including Spark MLlib and averaging-at-the-end. The paper's organization is clear, and the mathematical notation is consistent.

## Weaknesses

The primary weakness of this paper is its overreliance on unrealistic assumptions that undermine the practical significance of its claims. The authors claim to achieve "polylogarithmic runtime" but this requires a "quasi-polynomial number of processing units," which is a fundamental limitation that isn't adequately addressed. For instance, to process a dataset of size N, the scheme requires N^ε processors for some small ε > 0, which is completely infeasible for large-scale machine learning applications where we typically have only O(N) processors at most.

The paper's theoretical claims are limited to Euclidean spaces with finite Radon number (r = d+2), which excludes many practical machine learning models. The empirical evaluation is restricted to linear models and moderately sized datasets (up to 5 million instances), yet the paper claims broad applicability to "a broad class of learning algorithms" including non-linear models, which are not properly validated.

The paper fails to adequately address the communication overhead in distributed settings, which can be a significant bottleneck in parallel machine learning. The claim that "the overall amount of communication is low compared to periodically communicating schemes" lacks empirical evidence and theoretical analysis of communication complexity.

The comparison to existing parallelization approaches is insufficient. While the paper mentions averaging-based parallelizations and distributed SGD, it doesn't provide a thorough comparison with state-of-the-art methods like distributed SGD or block coordinate descent, which have been extensively studied and implemented in practice. The paper's claim that "current error bounds do not allow a derivation of polylogarithmic runtime" for averaging-based approaches is not sufficiently supported by evidence.

The paper's main theoretical contribution (Theorem 4) states that "the Radon machine with a consistent and efficient regularised risk minimisation algorithm on a hypothesis space with finite Radon number has polylogarithmic runtime on quasipolynomially many processing units." This is mathematically interesting but largely irrelevant to practical machine learning, where the number of processors is polynomial (not quasi-polynomial) in the problem size.

## Critical Assessment

While the Radon point aggregation method is theoretically interesting, the paper fails to provide sufficient evidence that this approach is practically useful for real-world machine learning problems. The claimed speedups (80-700x) are impressive but only achievable with an unrealistic number of processors. The paper's main contribution is not a practical parallelization scheme but a theoretical construct that requires quasi-polynomial processors to work.

The paper's main theoretical claim (polylogarithmic runtime) is not meaningful in the context of practical parallel machine learning, where we're constrained by polynomially many processors. The authors seem to be conflating theoretical complexity classes (Nick's Class) with practical implementations, which is a common pitfall in theoretical machine learning.

The empirical evaluation is limited in scope and doesn't address how the method would scale to datasets with billions of instances, which is increasingly common in modern machine learning. The paper also doesn't provide any analysis of how the Radon number r would be determined for real-world problems, which would be a significant practical hurdle.

## Recommendation

This paper presents an interesting theoretical idea but fails to provide sufficient evidence of practical utility. The core assumptions (quasi-polynomial processors, Euclidean hypothesis spaces) are too restrictive for the paper to make a meaningful contribution to the field of practical machine learning. The claimed speedups are impressive but only achievable under unrealistic conditions.

The paper would benefit from a more realistic analysis of the trade-offs between computation, communication, and the number of processors. It should also provide a more thorough comparison with existing parallelization methods that work with polynomially many processors.

While the mathematical analysis is sound, the paper's claims of "effective parallelisation" are overstated given the impractical requirements. The work is primarily of theoretical interest rather than practical significance for the machine learning community.

I recommend rejection. The paper makes a theoretically interesting contribution but fails to demonstrate that this approach is practically useful for the problems machine learning practitioners face. The theoretical insights don't translate to meaningful improvements in real-world parallel machine learning systems.

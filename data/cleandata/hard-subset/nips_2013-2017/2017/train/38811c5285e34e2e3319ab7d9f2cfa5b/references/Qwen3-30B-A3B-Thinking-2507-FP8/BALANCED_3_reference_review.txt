

# Review of "Effective Parallelisation for Machine Learning"

## Summary
This paper introduces the Radon Machine, a parallelization scheme for regularized risk minimization algorithms that achieves polylogarithmic runtime while maintaining the same (ε, δ) guarantees as sequential execution. The key insight is using Radon points for hypothesis aggregation instead of averaging, which provides a doubly exponential improvement in confidence with each level of aggregation. The authors prove that with quasi-polynomially many processing units, the runtime reduces from polynomial to polylogarithmic in sample complexity. Empirically, the Radon Machine achieves 80-700x speedups over sequential base learners on moderately sized datasets while maintaining comparable predictive performance to Spark MLlib algorithms.

The work relates to longstanding theoretical questions about parallel learning, particularly connecting to Nick's Class (NC) and the open problem posed by Long and Servedio about parallelizing regularized risk minimization. It distinguishes itself from prior averaging-based parallelizations by providing rigorous error bounds for the aggregation scheme.

## Strengths
1. **Theoretical Contribution**: The paper makes a significant theoretical contribution with Proposition 2 and Theorem 3, which rigorously establish the confidence improvement from using Radon points. This is a clear advance over existing averaging-based approaches that lack such bounds.

2. **Practical Implementation**: The "black-box" nature of the parallelization scheme is a major practical advantage, as it requires no changes to the base learning algorithm implementation. The empirical results demonstrate substantial speedups (721x on SUSY with h=3) while maintaining comparable predictive performance.

3. **Clear Problem Framing**: The paper effectively frames the problem of parallel machine learning in the context of Nick's Class and connects it to the broader question of whether consistent learning can be parallelized efficiently.

4. **Thorough Analysis**: The authors carefully address the trade-offs, noting the need for quasi-polynomial processing units and increased data requirements, and provide a detailed theoretical analysis of sample complexity.

## Weaknesses
1. **Limited Applicability**: The theoretical results require a hypothesis space with finite Radon number (r = d+2 for Euclidean spaces), which restricts applicability to linear models. While the authors discuss extensions to non-linear models via low-rank approximations, the paper lacks concrete analysis of how these approximations affect the error bounds.

2. **Empirical Scope**: The empirical evaluation is limited to moderately sized datasets due to "scalability limitations of the base learners." This is problematic because the paper's main claim is about scaling to large datasets. The comparison with Spark MLlib is somewhat superficial, lacking detailed analysis of computational complexity per iteration.

3. **Misleading Runtime Claim**: The claim that runtime is reduced "from polynomial to polylogarithmic" is technically inaccurate. The paper requires quasi-polynomially many processors (rh, where r is the Radon number), not polynomially many, which is the standard definition for NC. This misalignment with standard complexity theory terminology weakens the theoretical contribution.

4. **Communication Complexity**: The paper briefly mentions that communication is "low compared to periodically communicating schemes" but provides no quantitative analysis of communication complexity. For distributed systems, this is a critical metric that should be analyzed.

5. **Lack of Non-Convex Analysis**: The paper focuses exclusively on convex optimization, but many modern machine learning algorithms (e.g., neural networks) use non-convex optimization. The paper doesn't address how the Radon Machine would perform in such settings.

## Recommendation
This paper makes a valuable theoretical contribution to parallel machine learning with a novel aggregation scheme that provides rigorous error bounds. The empirical results are promising for linear models, but the paper would benefit from addressing the limitations in applicability to non-linear models and providing a more comprehensive comparison with existing parallel learning algorithms.

I recommend **weak acceptance** with the following suggestions for improvement:

1. Clarify the runtime claim to acknowledge the quasi-polynomial processor requirement (not polynomial) and discuss the practical implications of this distinction.

2. Expand the empirical evaluation to include larger datasets where the scalability claims would be most relevant, or at least provide a more detailed analysis of the scalability limitations.

3. Add a section analyzing the communication complexity of the Radon Machine in distributed settings, including a comparison to other distributed learning algorithms.

4. Discuss the limitations and potential extensions for non-convex optimization problems, which would significantly broaden the paper's applicability.

5. Include a more detailed comparison with Spark MLlib, particularly regarding computational complexity per iteration and the impact of different hyperparameters.

The paper's theoretical contribution is strong, but addressing these limitations would significantly enhance its impact and make it more relevant to the practical challenges of large-scale machine learning. With these revisions, this work could make an important contribution to the field.

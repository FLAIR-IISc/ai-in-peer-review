

This paper proposes a framework for generating strategic questions in a Battleship-like game, replacing direct queries about individual squares with questions framed in a compositional logical language (Montague-style semantics). The authors model question generation using a log-linear approach over four features: expected information gain, answer type (Boolean or numerical), and whether the question references board features. The task involves a highly constrained environment where all questions are logically unambiguous and the game state is fully observable.

While the active learning motivation is conceptually sound, the work's core contributions lack novelty and depth. The task itself is overly simplified—Battleship with perfect observability and a fixed rule set provides no meaningful challenge for language or learning models. The proposed features (e.g., "board relevance") are trivial to define in this context, and the log-linear model offers no substantive ML innovation beyond a basic heuristic. Crucially, the paper fails to demonstrate how this framework advances active learning, compositional language modeling, or practical applications in more complex domains. The absence of empirical comparisons to non-logical question-generation strategies or baselines further weakens the claim of utility.

The paper’s approach conflates linguistic structure with strategic value. For instance, a question like "Are there any ships in row 3?" (Boolean, board-relevant) is treated as equivalent to "How many ships are in row 3?" (numerical, board-relevant) in terms of feature engineering, yet the strategic implications differ significantly. The authors do not justify why these four features capture meaningful distinctions or how they generalize beyond this narrow setting. The model’s performance is not evaluated against human question-answering strategies or more sophisticated language models, leaving open whether the approach actually improves game play.

Most critically, the work does not engage with the broader NIPS community’s interests. NIPS prioritizes novel algorithmic insights, rigorous theoretical grounding, or empirical breakthroughs in machine learning. Here, the "compositionality" is superficial—logical structure is imposed on an artificial language with no connection to natural language pragmatics, ambiguity, or learning. The problem space is so constrained that it cannot inform real-world challenges in active learning (e.g., uncertainty in observations) or language generation (e.g., handling incomplete or noisy inputs). The authors cite no prior work on logical question generation in games, suggesting the task has been overlooked for good reason: it lacks the complexity to yield meaningful insights.

The paper’s weakest aspect is its failure to articulate a clear research gap. The introduction states that "existing active learning methods do not leverage compositional semantics," but this is unsubstantiated. No prior work is discussed that *does* use compositional semantics in active learning, nor is it explained why such an approach is necessary for this task. The technical contribution is minimal: a log-linear model applied to a trivial feature set. The results section merely reports accuracy on a synthetic dataset without analysis of why the model succeeds or fails, or how it compares to simpler alternatives (e.g., a rule-based system using only information gain).

In summary, this work addresses a well-defined but excessively narrow problem with an underdeveloped solution. The ideas are not novel, the task is not challenging, and the results do not advance the field. The paper would be more appropriately suited for a game-theory or linguistics venue—not NIPS. It offers no new ML methodology, fails to connect to core NIPS themes (e.g., scalability, generalization, or theoretical rigor), and provides no evidence that its framework could extend beyond this artificial setting. For a venue focused on transformative machine learning research, this submission falls short on all critical dimensions.

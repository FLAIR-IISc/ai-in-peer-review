

This paper proposes modeling user queries in a simplified Battleship game using a compositional logical language with Montagovian semantics, where direct square inquiries are replaced by structured questions over an artificial language. A log-linear model is trained on four key features: expected information gain (informativeness), answer type (Boolean or numerical), and whether the query references board features.  

While the active learning motivation is conceptually sound, the core contributions lack novelty. The task—reproducing queries in a highly constrained, logically unambiguous domain—exhibits minimal complexity. The four features used are superficial and fail to engage with deeper challenges in natural language generation or active learning. The model’s simplicity (a standard log-linear approach on trivial features) does not advance methodological understanding, and the problem setup is so narrow that it fails to generalize to real-world active learning scenarios. Crucially, the work does not address ambiguities, contextual dependencies, or scalability—key concerns in NIPS-focused research.  

The paper’s technical execution is competent but unambitious. The artificial language framework is presented without justification for its necessity over simpler alternatives, and the evaluation lacks rigor, omitting comparisons to baseline query strategies or human performance metrics. The task’s artificiality (e.g., fixed board states, no adversarial elements) further diminishes its relevance. Given NIPS’ focus on impactful, theoretically grounded machine learning advances, this work offers no meaningful contribution to the field. It neither introduces new algorithms, insights, nor empirical evidence that would interest the community. The ideas are derivative, the problem trivial, and the results unsurprising. For NIPS 2017, this falls far below the bar for novelty, technical depth, and significance expected in accepted papers.

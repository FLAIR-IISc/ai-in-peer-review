- Goal: Model user questions in a Battleship-style game by replacing tile-specific queries with an engineered formal language with compositional, Montagovian-style semantics.  
- Method: Train a log-linear model over a small set of four simple question features.  
- Features: Features include expected information gain (informativeness), answer type (Boolean vs. numeric), and an indicator for whether the query references board properties.  
- Motivation: The active learning framing is sensible.  
- Critique—novelty: Core ideas are fairly straightforward.  
- Critique—task scope: The task is very simple, highly constrained, and logically unambiguous.  
- Overall assessment: Unlikely to interest the NIPS community.

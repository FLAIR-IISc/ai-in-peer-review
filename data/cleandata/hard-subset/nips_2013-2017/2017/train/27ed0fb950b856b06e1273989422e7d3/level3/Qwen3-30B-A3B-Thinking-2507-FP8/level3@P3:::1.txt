

# Review of "Deep Dynamic Poisson Factorization Model" (NIPS 2017)

## Summary and Relation to Previous Work
The paper proposes a Deep Dynamic Poisson Factorization (DDPFA) model that extends Poisson Factor Analysis (PFA) by incorporating a simplified recurrent structure to capture temporal dependencies in count data. Building on previous work like Poisson-Gamma Dynamical Systems (PGDS) and Deep Exponential Families, the authors aim to model long-term dependencies in sequential count data using a "deep" latent structure. The model uses variational inference with neural networks to represent implicit distributions, differing from prior approaches that rely on Gamma-Gamma-Poisson structures or MCMC methods.

## Quality
The core idea has potential, but the paper suffers from significant technical shortcomings. The model's "RNN-like" structure (Eq. 4) is not a true RNN, as h_t^(n) depends only on top-layer memory vectors and not on previous same-layer time steps. The authors should clarify why this simplified structure is justified, particularly when the goal is to capture long-term dependencies. The claim that distributions are "implicit" (Eqs. 9-10) is incorrect—these are explicit Gamma distributions with closed-form conditionals. The evaluation metrics (MSE/PMSE) are inappropriate for Poisson factorization models; predictive log-likelihood should be reported instead. The small number of latent factors (K=3-10) is problematic, as the authors should test larger values (K≈100) to demonstrate scalability.

## Clarity
The paper suffers from poor clarity and organization. Section 2 omits key model equations and parameterizations (e.g., Gamma distribution in Eq. 4, distribution generating θ_t in Eq. 3). The reference to "fanatical data and disaster data" (lines 30-31) is unclear and needs elaboration. The loss function in Eq. 15 is not properly explained, and the discussion around lines 224-232 is disconnected from Fig. 3. The writing quality is poor, with excessive passive voice, grammatical errors, and typos (e.g., "high-dimensional" missing hyphen, "factorize" instead of "factorization," misused "Dirichlet"). The Fig. 4 caption is unreadable and must be fixed. The variational update equations (Eqs. 11-14) should be moved to the supplement to improve flow.

## Originality
The paper builds on existing work (PGDS, Deep Exponential Families, Deep Poisson Factor Analysis) but offers limited novelty. The "deep" structure is superficially implemented with a non-standard RNN-like architecture that doesn't properly capture temporal dependencies. The model does not significantly differ from prior hierarchical approaches (e.g., Deep Gaussian Processes, Nested Dirichlet Processes), and the paper fails to adequately distinguish itself from related work.

## Significance
The potential significance of modeling temporal dependencies in count data is clear, but the paper's weak execution undermines its contribution. The experiments on five real-world datasets show some promise, but the inappropriate evaluation metrics and small K values limit the conclusions. The "pruning" and "uselessness of adding network layers" issues (Table 3) suggest fundamental limitations in the model design that aren't properly addressed.

## Strengths
- The idea of using neural networks to capture complex temporal dependencies in count data is conceptually interesting.
- The model does capture long-term patterns better than PGDS in some datasets (e.g., ICEWS, EBOLA).
- The synthetic data experiments demonstrate the model's ability to learn complex time patterns.

## Weaknesses
- Technical flaws in model architecture (non-standard RNN structure, incorrect "implicit" claims).
- Inappropriate evaluation metrics (MSE/PMSE instead of predictive log-likelihood).
- Poor writing quality and numerous grammatical errors.
- Insufficient experimental design (small K values, no comparison with larger K).
- Unclear explanations of key concepts (loss function, "fanatical data" reference).

## Arguments for and against Acceptance
**For Acceptance**: The paper addresses a meaningful problem in sequential count data analysis with a novel approach that could be useful for applications like recommendation systems and text analysis. The core idea has potential if properly executed.

**Against Acceptance**: The paper contains fundamental technical errors, poor writing, and inadequate experiments. It fails to properly implement a recurrent structure, uses inappropriate evaluation metrics, and contains numerous unexplained claims. The model's limitations (e.g., "pruning" issue) are not adequately addressed, and the experimental design is insufficient to support the claims.

## Recommendations
1. Correct the technical errors: clarify the simplified RNN structure, fix incorrect "implicit" claims, and use appropriate evaluation metrics (predictive log-likelihood).
2. Expand experiments: test with larger K values (e.g., K≈100) and include ablation studies on layer depth.
3. Improve clarity: rewrite Section 2 with complete model equations, fix grammatical errors, and clarify ambiguous statements.
4. Move variational updates to supplement and fix Fig. 4 caption.
5. Address the "pruning" issue by comparing with alternative inference methods (e.g., MCMC).

The paper's idea has potential, but it requires substantial clarification, correction, and stronger experiments before it can be considered conference-ready. As currently written, the technical shortcomings and poor presentation make it difficult to assess the model's true contribution to the field.

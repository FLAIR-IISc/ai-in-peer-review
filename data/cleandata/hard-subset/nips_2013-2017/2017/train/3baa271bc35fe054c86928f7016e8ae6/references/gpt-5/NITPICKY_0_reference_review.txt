Summary
The paper addresses robust submodular maximization in the single-pass streaming setting. It proposes STAR-T, a partitioned, exponentially thresholded structure that stores O((k + m log k) polylog k) elements, and a query-stage procedure (GREEDY on S\E) that returns a size-k solution robust to the removal of any m elements. Under an appropriate choice of memory parameter w and a threshold τ approximating f(OPT(k, V\E)), the analysis yields a constant-factor robustness guarantee (limit ≈0.149). A multi-guess variant removes the need to know f(OPT). Experiments on dominating set and movie recommendation compare favorably to SIEVE-STREAMING that is allowed to know deletions E.

Relation to prior work: This builds on streaming thresholding (SIEVE-STREAMING) and robust non-streaming schemes with partitioned buckets (PRO). It also compares to a concurrent streaming robust algorithm achieving 1/2−ε with higher memory O(mk log k/ε). The contribution here is a robust streaming algorithm with memory scaling logarithmically in k for the “robust” part and a new analysis tied to the partitioned thresholding design.

Strengths
- Addresses an important and timely problem: robust data summarization with single-pass constraints.
- Novel partitioned thresholding scheme: exponentially increasing bucket sizes with exponentially decreasing thresholds, distinct from greedy filling in PRO.
- Theoretical guarantee: constant-factor approximation after arbitrary removal of m elements, with memory O((k + m log k) polylog k), improving k-dependence of the “robust” term over prior O(mk).
- Practicality: Query stage only needs the stored summary, enabling multiple interactive deletions without revisiting the stream.
- Experiments on two applications suggest competitive performance versus strong baselines, even when the baseline has oracle knowledge of deletions.

Weaknesses and detailed issues
Technical correctness/clarity
- Threshold inconsistencies: In Section 3 text, partition i has threshold τ/2^i; Algorithm 1 uses τ/min{2^i, k}. Please reconcile and justify the min{k, 2^i} both in capacity and threshold. If the min is intentional (to avoid τ/2^i < τ/k for i > log k), state and use consistently throughout proofs.
- Theorem 4.1 τ-definition is garbled: “τ = 1 / + (1−e−1) (1−e−1/3) (1−1/dlog ke) f(OPT...)” is dimensionally inconsistent and unreadable. Provide the exact formula with correct parentheses; as written it seems incorrect.
- Equation (5) formatting appears wrong: “f(Z) ≥ 12 α2(1−γ) + 1 α1 f(OPT...)”. Likely a missing fraction structure (e.g., harmonic mean-like term (α1α2)/(α1 + α2)), but as printed it is not parseable. Several other equations have spacing/parenthesis errors.
- Lemma 4.2 uses an “average-argument” over buckets to bound |Bi∩E| and derives the 4m/(wk) term; please provide the exact counting argument in the main paper or at least a precise pointer, as this constant is crucial (and somewhat unusual).
- Use of dlog ke: base of logarithm is unclear; elsewhere “log2 k” is used to mean (log k)^2 vs log base 2 ambiguity. Unify notation (e.g., log denotes base e or 2; log^2 k for (log k)^2).
- Space bound inconsistencies:
  • Abstract/intro/conclusion mention O((k + m log k) log^2 k).
  • Theorem 4.1 derives |S| ≤ (log k + 5)wk and with w ≥ ceil(4m log k/k) implies |S| = O((k + m log k) log k).
  These differ by a log factor; please reconcile and state a single tight bound.
- The claim “robust to any number of removals m as long as we are allowed to use O(m log^2 k) memory” should explicitly include the additive O(k log k) term and the dependence of w on m/k; currently phrased, it’s easy to misread.
- The parallel-guess scheme (Section 4.1) says space increases by log_{1+ε} k; typically such schemes depend on an estimate range [L, U] of OPT (e.g., via stream maxima). Please specify the range, number of instances, and how τ is discretized. Without this, the final guarantees are hard to reproduce.
- Query-stage guarantee depends on GREEDY performance; in Lemma 4.3 you use 1 − e^{-1/3}. Please justify the 1/3 exponent (it suggests running GREEDY for k/3 steps). This needs to be explicit in Algorithm 2 or the analysis.

Experimental concerns
- Reproducibility is limited. Key implementation details are missing: how τ is chosen in practice (since f(OPT) is unknown), the number of parallel instances and their τ-grid, evaluation budget (oracle calls), and the exact w used (you mention w=1 but no sensitivity study).
- Metrics: Only objective values are reported for selected k and single/few users. No runtime, number of oracle evaluations, or memory usage comparisons are provided, which is important in streaming.
- Bias in E selection: For the dominating set task, E is sampled from S (which is worst for STAR-T) and baselines are run on V\E (best for them), which is commendably conservative; however, a more systematic study varying m, k, and different E-generation (random adversarial, structured) is needed.
- The movie-recommendation fu(Z) uses non-normalized feature vectors and a facility-location term over all movies M; please clarify normalization and computational implications (marginal gain cost), or whether approximate lazy evaluations were used.
- Empirical robustness beyond theory (e.g., |E| > k) is anecdotal; provide plots vs m (including m up to the theoretical limit) and ablations (GREEDY vs SIEVE at query-time; effect of w).

Presentation/formatting
- Numerous typos and leftover artifacts: author footnote block and “st Conference on NIPS...” line; inconsistent reference styles (missing years/venues), and some mismatched punctuation.
- Algorithm 1: Step 7 “break: proceed to the next element in the stream” is informal; prefer precise pseudocode.
- Ambiguous “log2 k” notation as noted; unify across paper.
- Several equations and symbols lack definitions on first use (e.g., precise definition of dlog ke).

Originality and significance
- Compared to concurrent work with 1/2−ε approximation and O(mk log k/ε) memory, your constant 0.149 is weaker but memory is markedly better in its dependence on k for the robust term. This is a meaningful trade-off and relevant in large-scale settings.
- The partitioned thresholding idea that combines PRO-style structure with streaming thresholds is a novel and interesting synthesis.

Questions for authors
- Please provide the correct and unambiguous expression for τ and fix Eq. (5). Does the final constant 0.149 depend on γ via w, or is it truly independent as k→∞?
- Is τ/min{2^i, k} the intended threshold? If yes, adjust the analysis accordingly and state this early.
- What is the precise space bound for the multi-guess variant (as a function of ε and the OPT range)?
- Can you report memory and runtime across methods, and sensitivity to w and m?

Recommendation
Borderline reject. The core idea is interesting and likely significant, but the paper has too many technical and presentation inconsistencies (thresholds, memory bounds, broken equations, τ-definition) that impede verification and reproducibility. With careful revision addressing the above points and stronger empirical evaluation, I would be inclined to accept.

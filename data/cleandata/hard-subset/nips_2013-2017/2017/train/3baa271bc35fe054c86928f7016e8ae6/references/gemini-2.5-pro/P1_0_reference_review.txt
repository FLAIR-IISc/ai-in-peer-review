### Summary of the Paper

This paper addresses the problem of robust submodular maximization in a single-pass streaming setting. The authors propose a two-stage framework. In the first "streaming" stage, their algorithm, STAR-T, processes a stream of data in a single pass to create a robust summary `S`. In the second "query" stage, after an arbitrary set `E` of up to `m` elements is removed from the summary `S`, a simple greedy algorithm (STAR-T-GREEDY) is run on the remaining elements `S \ E` to produce a solution `Z` of size `k`.

The core contribution is the STAR-T algorithm and its analysis. The algorithm constructs the summary `S` by organizing elements into a set of partitions, where each partition contains multiple buckets. Partitions are associated with exponentially decreasing thresholds, and incoming stream elements are placed into the first available bucket for which they meet the marginal gain threshold. This structure is a novel extension of the thresholding idea from Sieve-Streaming [8] to the robust setting, drawing structural inspiration from the non-streaming robust algorithm of Bogunovic et al. [11].

The main theoretical result is a constant-factor approximation guarantee for the solution `Z` returned in the query stage. The authors prove that for any removal of `m` elements, the final solution value is at least `0.149 * f(OPT)`, where `OPT` is the optimal `k`-sized solution on the ground set *after* the removals. This is achieved while using `O((k + m log k) log k)` space for the summary `S`. The paper also addresses the practical issue of not knowing the optimal value a priori by using a standard technique of running multiple instances in parallel, which adds a logarithmic factor to the space complexity. The work is compared to the concurrent work of Mirzasoleiman et al. [18], highlighting a key advantage in memory complexity, which is logarithmic in `k` rather than linear. The theoretical claims are supported by strong empirical results on dominating set and personalized movie recommendation tasks.

### Strengths and Weaknesses

**Strengths (Arguments for acceptance):**

1.  **Well-motivated and Important Problem:** The problem of robust data summarization for massive, streaming datasets is highly relevant and practical. The two-stage model, where a summary is pre-computed and then queried after adversarial removals, captures many real-world scenarios like interactive recommendation or fault-tolerant sensor placement.
2.  **Novel Algorithm and Analysis:** The STAR-T algorithm's design, featuring a hierarchy of partitions with exponentially decreasing thresholds, is a clever and non-trivial contribution. The analysis is also sophisticated, successfully handling two distinct cases (partitions being mostly full vs. mostly empty) and balancing the resulting bounds to prove a constant-factor guarantee.
3.  **Strong Theoretical Guarantees with Favorable Complexity:** The paper provides a constant-factor approximation guarantee, which is the standard goal for such problems. Critically, the memory complexity scales logarithmically with the solution size `k` (`O(k log k)` dependency), which is a significant improvement over the concurrent work [18] that has a linear dependency (`O(mk log k)`). This makes the proposed method far more scalable for applications where `k` is large.
4.  **Convincing Experimental Evaluation:** The experiments are well-designed and compelling. The authors compare their method against a very strong baseline: the Sieve-Streaming algorithm, which is given oracle access to the set of removed elements `E` *before* it processes the stream. The fact that STAR-T performs competitively with, and sometimes outperforms, this powerful baseline is strong evidence of its practical effectiveness. The movie recommendation use-case effectively demonstrates the flexibility of the two-stage approach.

**Weaknesses (Areas for improvement):**

1.  **Clarity on Threshold Definition:** There is a minor inconsistency between the text and the pseudocode regarding the threshold. The text (Sec. 3) states the threshold for partition `i` decreases as `τ/2^i`. However, the pseudocode (Alg. 1, line 5) uses `τ/min{2^i, k}`. For partitions where `2^i > k`, the threshold becomes constant at `τ/k`. This is an important detail of the algorithm that seems to be glossed over in the textual description and could be clarified.
2.  **Low Theoretical Constant:** The proven approximation constant is approximately 0.149. While any constant-factor guarantee is a strong theoretical result, this value is quite small. The authors rightly show in experiments that the practical performance is much better, which largely mitigates this concern, but the gap between the theoretical bound and empirical reality is notable.
3.  **Complexity of the Summary Structure:** The summary `S` has a rather intricate structure, with multiple parameters (`w`, number of partitions, number of buckets per partition) that depend on `k` and `m`. While the theory is sound, this complexity might make the algorithm less intuitive and potentially harder to implement and tune compared to simpler streaming methods. The paper states that `w=1` works well in practice, which is a helpful insight, but the theoretical requirement for `w` is more involved.

### Detailed Comments

**Quality:** The paper is of high technical quality. The theoretical claims appear sound and the proof sketch provides good intuition for the main arguments. The work is complete, presenting a new algorithm, a full theoretical analysis, and a thorough empirical validation. The authors are honest about the limitations (e.g., handling the unknown OPT value) and address them with standard, appropriate techniques.

**Clarity:** The paper is generally well-written and organized. The problem, algorithm, and results are presented logically. The main area for improvement is the small discrepancy in the threshold definition mentioned above. A clearer explanation of why the threshold in the algorithm is `τ/min{2^i, k}` would benefit the reader. Figure 1 provides a helpful illustration of the summary structure.

**Originality:** This work is highly original. While it builds on ideas from streaming submodular maximization (Sieve-Streaming [8]) and robust submodular maximization (PRO [11]), the synthesis of these ideas into a partitioned thresholding algorithm for the *robust streaming* setting is novel. The problem formulation itself, while related to prior work, is cleanly defined for the streaming context. The authors clearly position their work with respect to the concurrent paper [18], correctly identifying their superior memory scaling with `k` as a key differentiating contribution.

**Significance:** The paper's contribution is significant. It provides the first robust streaming submodular maximization algorithm with a constant-factor guarantee and memory that scales polylogarithmically in `k`. This is a valuable theoretical result for the submodular optimization community. From a practical standpoint, the algorithm addresses a critical need in large-scale machine learning and data science, and its strong empirical performance suggests it could be readily adopted by practitioners. The ideas presented here are likely to inspire follow-up work on robust optimization in streaming and other resource-constrained settings.

**Conclusion:** This is a strong paper that presents a novel and practical solution to an important problem. It combines a creative algorithmic design with a solid theoretical analysis and compelling experimental results. The weaknesses are minor and do not detract from the overall quality and significance of the contribution. I recommend acceptance.

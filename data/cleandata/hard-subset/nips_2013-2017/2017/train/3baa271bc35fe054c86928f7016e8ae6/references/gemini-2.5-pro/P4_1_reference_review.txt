This paper addresses the problem of robust submodular maximization in a streaming setting. The goal is to design a single-pass algorithm that processes a stream of data to create a compact, robust summary. This summary should allow for the efficient extraction of a high-quality solution of size `k` even after an adversary removes up to `m` elements from the summary, without needing to re-access the original data stream. The authors propose a two-stage procedure. In the first stage, the STAR-T algorithm creates a robust summary `S` by organizing elements into a structure of partitions and buckets. Each partition is associated with an exponentially decreasing greedy threshold, and an incoming element is placed into the first available bucket for which it satisfies the marginal gain threshold. In the second, query stage, after the removal of a set `E` from `S`, a standard greedy algorithm (STAR-T-GREEDY) is run on the remaining set `S \ E` to produce the final solution.

This work lies at the intersection of streaming submodular maximization and robust submodular maximization. It effectively adapts and combines ideas from both domains. The use of greedy thresholding is inspired by streaming algorithms like SIEVE-STREAMING [8]. The partitioned structure of the summary, where bucket sizes increase exponentially, is reminiscent of the non-streaming robust algorithm PRO [11]. The key novelty lies in merging these concepts: using a thresholding rule to populate a partitioned structure in a single pass, rather than filling buckets sequentially as in [11]. The authors also appropriately position their work against the concurrent work of [18], highlighting a key advantage in memory complexity.

**Strengths:**

1.  **Originality and Significance:** The paper tackles a well-motivated and increasingly relevant problem. Ensuring robustness of machine learning models against unforeseen removals or failures is critical, and doing so in a memory-efficient, single-pass streaming setting is a significant challenge. The proposed algorithmic approach, combining a partitioned structure with an exponentially decreasing thresholding scheme, is a novel and clever way to address this challenge. The resulting algorithm is practical for large-scale scenarios.

2.  **Technical Quality:** The paper provides a sound theoretical analysis, proving that the proposed two-stage procedure achieves a constant-factor approximation guarantee. The proof sketch is insightful, breaking the analysis into two main cases depending on whether partitions contain a sufficient number of full buckets. This provides a clear high-level understanding of why the algorithm works. The theoretical results are strong, establishing the first constant-factor approximation for this problem with a memory requirement that has only a logarithmic dependence on the solution size `k`, which is a notable advantage over the linear dependence in the concurrent work [18]. The extension for handling the unknown value of `f(OPT)` via parallel instances is a standard and necessary addition that makes the algorithm fully practical.

3.  **Clarity:** The paper is well-written and organized. The problem statement is clear, and the proposed STAR-T algorithm is described precisely with pseudocode and an illustrative figure. The motivation for the specific partitioned structure and thresholding rule is explained. The experiments are well-designed and clearly presented, providing compelling empirical evidence for the algorithm's effectiveness.

**Weaknesses:**

1.  **Approximation Constant:** The derived approximation constant of `c ≈ 0.149` is relatively low. While obtaining any constant factor for this challenging problem is a significant achievement, this value is modest. It would be beneficial for the authors to briefly discuss whether this constant is an artifact of the proof technique or potentially inherent to the algorithm's design.

2.  **Inconsistencies in Technical Details:** There are minor inconsistencies in the paper that could be clarified.
    *   The size of the summary `S` is stated differently in various sections. The abstract and introduction claim a size of `O((m log k + k) log^2 k)`, while the conclusion states `O((k + m log k) log^2 k)`. However, Theorem 4.1 claims a bound of `O((k + m log k) log k)`. A consistent and clearly derived statement of the memory complexity would strengthen the paper's precision.
    *   In the text describing Algorithm 1, the threshold for partition `i` is stated as `τ/2^i`. However, the pseudocode (line 5) uses `τ/min(2^i, k)`. For partitions where `2^i > k`, the threshold becomes constant at `τ/k`. This is a subtle but important detail that should be explicitly mentioned and its implications for the analysis briefly discussed in the main text.

3.  **Gap between Theory and Practice:** The theoretical analysis requires the memory parameter `w` to be at least `⌈4 dlog ke m / k⌉`, which grows with the robustness parameter `m`. However, the experiments are conducted with `w=1` and demonstrate strong performance even for scenarios with a large number of removals (e.g., removing 59% of the universe in the movie recommendation task). This suggests a potential gap between the theoretical requirements and the algorithm's practical performance. A discussion of this observation would be valuable, as it indicates the analysis might be loose and the algorithm could be even more practical than the theory suggests.

4.  **Experimental Comparison:** The experiments compellingly demonstrate that the proposed method performs comparably to or better than an oracle-assisted version of SIEVE-STREAMING. However, given the mention of the concurrent work [18], the paper would be significantly strengthened by an empirical comparison. While [18] has a higher theoretical memory complexity, a practical comparison could quantify the real-world trade-offs between the two approaches in terms of solution quality and memory usage.

In summary, this paper presents a novel and significant contribution to the field of large-scale submodular optimization. It introduces a practical single-pass streaming algorithm for robust submodular maximization with strong theoretical guarantees, most notably a constant-factor approximation with favorable memory complexity. While the approximation constant is modest and there are minor points that could be clarified, the strengths of the paper—its novelty, technical soundness, and relevance—are substantial. The work is a solid step forward and is likely to inspire further research in this important area.

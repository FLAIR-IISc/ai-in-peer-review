**Summary**

This paper addresses the problem of robust submodular maximization in a streaming setting. The goal is to select a small, robust summary set `S` from a data stream in a single pass. This summary `S` should be resilient to the removal of up to `m` of its elements, such that even after an arbitrary set `E` of `m` elements is removed, one can still find a high-quality `k`-sized solution `Z` within the remaining set `S \ E`. The authors propose a two-stage procedure. In the first stage, a novel streaming algorithm called STAR-T constructs the summary set `S` using a partitioned structure with exponentially sized buckets and exponentially decreasing thresholds. This approach is an interesting synthesis of ideas from SIEVE-STREAMING (thresholding) and recent work on non-streaming robust maximization (partitioning). In the second stage, a standard offline greedy algorithm is run on the remaining elements `S \ E` to find the final solution `Z`. The authors provide a theoretical analysis showing that their method achieves a constant-factor approximation guarantee while using O((k + m log k) log k) memory. The approach is evaluated experimentally on dominating set and movie recommendation tasks.

**Strengths**

1.  **Problem Significance:** The problem of building robust summaries from massive data streams is of clear practical importance. The formulation, which separates the expensive streaming pass from a fast query stage, is well-motivated for applications where failures, adversarial attacks, or changing user preferences can invalidate parts of a pre-computed summary.

2.  **Novel Algorithmic Idea:** The core of the contribution, the STAR-T algorithm, introduces a novel structure for the summary set. The combination of partitions, exponentially increasing bucket sizes, and exponentially decreasing thresholds is a non-trivial extension of existing streaming techniques. It provides a principled way to balance the value of elements added to the summary at different scales.

3.  **Theoretical Guarantees:** The paper provides a full theoretical analysis, culminating in a constant-factor approximation guarantee (Theorem 4.1). Proving such a guarantee in this combined robust and streaming setting is a challenging task. The memory complexity, which has a logarithmic dependency on `k`, is also a strong point and compares favorably to concurrent work.

**Weaknesses**

1.  **Low Approximation Constant:** The derived theoretical constant of 0.149 is quite low. While achieving any constant factor is a valuable theoretical result, such a low value raises questions about the tightness of the analysis and the practical utility of the guarantee. The authors should discuss if this is an artifact of the proof technique or an inherent limitation of the algorithm.

2.  **Experimental Comparison:** The empirical validation is a significant point of concern. The primary baseline used for comparison is SIEVE-STREAMING. However, this baseline is given the unrealistic advantage of knowing *in advance* which `m` elements will be removed, allowing it to run on the pristine dataset `V \ E`. While the proposed method's ability to compete with this oracle-like baseline is interesting, it is not a fair or informative comparison against a realistic alternative. A more methodologically sound evaluation would compare STAR-T against other streaming algorithms designed for robustness or, if none exist, against more standard streaming baselines (like SIEVE-STREAMING) under a fair protocol where neither algorithm has foreknowledge of the removals. Without a proper baseline, it is difficult to judge whether the proposed complex machinery is truly superior to simpler, established streaming methods adapted for robustness (e.g., by simply storing a larger summary).

3.  **Gap Between Theory and Practice:** There is a notable disconnect between the theoretical requirements and the experimental setup. The theory requires the memory parameter `w` to be at least `⌈4dlog ke * m/k⌉` to guarantee the approximation factor. For a reasonably large `m`, this would require a non-trivial `w`. However, all experiments are run with `w=1`. The paper notes that this works well in practice but does not provide an adequate explanation for this discrepancy. This weakens the connection between the theoretical analysis and the empirical results, suggesting that the theory may not fully capture the behavior or strengths of the algorithm as implemented.

4.  **Second Stage Algorithm:** The proposed solution relies on running the standard, non-streaming GREEDY algorithm in the query stage. While this is computationally feasible because it runs on the small summary `S`, it feels somewhat inconsistent with the streaming motivation of the first stage. The authors do test a variant using SIEVE-STREAMING in the second stage (STAR-T-SIEVE), which performs similarly. It is unclear why the simpler and more established GREEDY is the primary proposal, and the implications of this choice could be discussed further.

**Overall Assessment**

The paper tackles a challenging and relevant problem at the intersection of streaming algorithms and robust optimization. The proposed STAR-T algorithm is novel and supported by a rigorous, albeit complex, theoretical analysis that establishes a constant-factor approximation.

However, the evaluation of this new approach is not sufficiently thorough. The main experimental comparison is against a baseline operating under an unrealistic assumption, which makes it difficult to assess the true practical advantage of the proposed method. Furthermore, the significant gap between the parameter settings required by the theory and those used in practice undermines the confidence in the presented theoretical model.

While the core ideas are promising and could inspire future work, the paper in its current form is not entirely convincing. To be a strong contribution to NIPS, the work would need to be supported by a more robust empirical evaluation against well-established and fairly-compared baselines on potentially larger-scale problems that better motivate the streaming setting. The claims of outperforming SIEVE-STREAMING are strong but are based on a comparison that is fundamentally skewed.

**Arguments for Acceptance:**
*   Addresses a well-motivated and difficult problem.
*   Presents a novel algorithm with a non-trivial theoretical analysis.
*   Achieves a constant-factor approximation with favorable memory complexity.

**Arguments for Rejection:**
*   The primary experimental baseline comparison is methodologically flawed, making it hard to assess the algorithm's practical performance.
*   A significant discrepancy exists between the theoretical requirements for parameters and their practical settings, weakening the paper's claims.
*   The proven theoretical approximation constant is very low.

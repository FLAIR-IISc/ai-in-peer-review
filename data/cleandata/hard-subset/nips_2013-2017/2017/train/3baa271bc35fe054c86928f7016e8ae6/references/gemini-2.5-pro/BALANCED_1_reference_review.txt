This paper addresses the problem of robust submodular maximization in a streaming setting. The goal is to select a small, robust summary of a large dataset in a single pass, such that even after an adversary removes up to `m` elements from this summary, a near-optimal solution of size `k` can still be recovered without accessing the original data stream.

The authors propose a two-stage procedure. In the first stage, a streaming algorithm called STAR-T constructs a summary set `S`. STAR-T is a novel partitioned thresholding algorithm that elegantly combines ideas from prior work. It adopts a partitioned data structure with exponentially increasing bucket sizes, similar to the non-streaming robust algorithm PRO [11], and integrates it with a greedy thresholding principle inspired by SIEVE-STREAMING [8]. A key innovation is the use of multiple, exponentially decreasing thresholds, one for each partition. In the second stage, after `m` elements are removed from `S` to form a set `S \ E`, a simple greedy algorithm (STAR-T-GREEDY) is run on this remaining set to find the final solution `Z`.

The main theoretical contribution is a proof that this procedure achieves a constant-factor approximation guarantee (approximately 0.149) while using `O((k + m log k) log k)` space. This space complexity, particularly its logarithmic dependence on `k` for the `m`-dependent term, represents a significant improvement over concurrent work [18]. The authors also provide a standard method for handling the unknown optimal value, which is required to set the algorithm's thresholds, by running multiple instances in parallel. The paper's theoretical claims are supported by strong empirical results on dominating set and personalized movie recommendation tasks, where the proposed method performs comparably to, and sometimes better than, a strong baseline (SIEVE-STREAMING) that has oracle access to the set of removed items.

**Strengths**

1.  **Significance and Motivation:** The paper tackles a well-motivated and practically important problem. The combination of robustness, submodularity, and streaming constraints is highly relevant to modern large-scale machine learning applications where data is massive and systems must be resilient to failures, attacks, or dynamic user feedback. The proposed two-stage framework (precompute a robust summary, then query quickly) is a natural and effective model for many real-world scenarios.

2.  **Originality and Technical Contribution:** The STAR-T algorithm is a clever and non-trivial synthesis of existing ideas. The adaptation of a partitioned structure to the streaming setting via a novel, exponentially decreasing thresholding scheme is a key contribution. This design allows the algorithm to balance the selection of high-value elements (captured in early partitions with high thresholds) and diverse, potentially lower-value elements that provide robustness (captured in later partitions).

3.  **Theoretical Quality:** The paper provides a solid theoretical analysis with a constant-factor approximation guarantee. The proof is technically sound, relying on a careful case analysis. The derived space complexity of `O((k + m log k) log k)` is a major strength, as it scales gracefully with the solution size `k` and compares favorably to concurrent work, making the algorithm more suitable for settings where `k` is large.

4.  **Experimental Quality:** The experiments are thorough and convincing. The comparison against SIEVE-STREAMING with advance knowledge of the removed elements is a particularly strong baseline, and the fact that STAR-T performs competitively demonstrates its practical effectiveness. The choice of diverse applications (graph analysis and personalized recommendation) and removal strategies (random and adversarial) effectively validates the algorithm's performance and versatility. The interactive movie recommendation experiment is an excellent illustration of the framework's utility.

**Weaknesses and Suggestions for Improvement**

1.  **Clarity on Technical Details:** There are a few minor inconsistencies that could be clarified:
    *   **Memory Complexity Notation:** The space complexity is stated differently in the abstract (`O((m log k + k) log^2 k)`), conclusion (`O((k + m log k) log^2 k)`), and the main theorem/derivation (`O((k + m log k) log k)`). The derivation in the proof of Theorem 4.1 appears to support the `O((k + m log k) log k)` bound. Please unify this notation and verify the final expression.
    *   **Threshold in Pseudocode:** The text describes the thresholds as decreasing exponentially like `τ/2^i`. However, Algorithm 1 uses `τ/min{2^i, k}`. This is a sensible choice to prevent the threshold from becoming unnecessarily small for large `i`, but the text should be updated to match the pseudocode for full clarity.

2.  **Practicality and Parameter Sensitivity:**
    *   The main algorithm requires an estimate of the unknown optimal value to set `τ`. The paper proposes the standard parallel-instances approach, but the experiments do not specify if this was used, or how `τ` was set. Clarifying this is important for reproducibility. A brief discussion on the algorithm's sensitivity to a misspecified `τ` would also strengthen the paper.
    *   The experiments use `w=1`, which is much smaller than the `w ≥ O(m log k / k)` required by the theory. While the strong performance is a positive result, it suggests a gap between the worst-case theoretical analysis and average-case practical performance. A short discussion on this gap would be insightful. For instance, is it because the adversarial model in the theory is much stronger than the removal strategies tested?

3.  **Comparison to Concurrent Work:** The paper rightly highlights its superior space complexity compared to the concurrent work of Mirzasoleiman et al. [18]. However, [18] claims a better approximation guarantee of `1/2 - ε`, which is significantly higher than the `0.149` proven here. This creates an important trade-off between space efficiency and approximation quality. The paper would be more balanced if it explicitly acknowledged and discussed this trade-off, positioning its contribution as an algorithm that prioritizes memory efficiency, which may be crucial in certain applications.

**Decision**

Overall, this is a high-quality paper that makes a solid contribution to the literature on large-scale submodular optimization. The proposed STAR-T algorithm is novel and effective, backed by strong theoretical guarantees and compelling empirical evidence. The strengths, particularly the significance of the problem and the algorithm's excellent space complexity, clearly outweigh the minor weaknesses, which can be addressed in a final version. I recommend acceptance.

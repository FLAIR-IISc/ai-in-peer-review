Summary
The paper studies robust submodular maximization in a streaming setting under a cardinality constraint. The authors propose a two-stage framework: (i) a single-pass streaming algorithm, STAR-T, that builds a compact, robust summary S arranged into partitions of buckets whose capacities grow exponentially with partition index and whose acceptance thresholds decrease exponentially; (ii) a query-stage algorithm, STAR-T-GREEDY, which, given any removal set E of size at most m, runs GREEDY on S\E to return a size-k solution. The main theoretical result shows that, for suitable parameters, STAR-T stores O((k + m log k) log k) elements (or O((k + m log k) log k log_{1+ε} k) when τ is unknown and guessed in parallel), and STAR-T-GREEDY achieves a constant-factor approximation c ≈ 0.149 (as k grows). Compared to concurrent work achieving 1/2 − ε approximation with O(mk log k/ε) memory, the novelty here is much better memory dependence on k (logarithmic rather than linear) at the cost of a smaller approximation factor. Experiments on dominating set (Twitter, Amazon) and MovieLens recommendation show that, even when deletions are adversarial or exceed the theoretical m, STAR-T-GREEDY matches or slightly outperforms SIEVE-STREAMING, which is given the advantage of knowing deletions in advance, and substantially outperforms a random baseline. The summary can be reused interactively for multiple queries/removals.

Relation to prior work
The paper builds on (i) classic streaming submodular maximization via thresholding (SIEVE-STREAMING), (ii) robust (non-streaming) formulations with partition/bucket structures (e.g., PRO), and (iii) concurrent deletion-robust streaming work with different memory/approximation trade-offs. The algorithmic structure combines PRO’s exponentially sized partitions with streaming thresholding, yielding robustness with reduced memory dependence on k. Related work is well-cited and the position relative to [8,10,11,18] is clear.

Strengths
- Problem importance: Robustness under deletions is practically relevant (e.g., “right to be forgotten,” node failures, user-specific filtering) and challenging in streaming.
- Algorithmic novelty: Partitioned thresholding in a streaming setting is a neat combination; the decreasing thresholds matched to bucket sizes are intuitive and implementable.
- Theory: Single-pass algorithm with a clear memory bound and a constant-factor approximation after arbitrary deletions up to m. The improved memory scaling in k compared to concurrent work is a meaningful contribution.
- Practical relevance: The two-stage design naturally supports interactive re-querying without rescanning the stream. Experiments show competitive performance even against a strong baseline that knows deletions.
- Clarity of high-level ideas and proof strategy: The two-case analysis (many full buckets vs. none) and balancing lemmas are well-motivated.

Weaknesses and suggestions
- Approximation factor: The constant 0.149 is modest. While acceptable for streaming/robust settings, a discussion of whether better constants are possible with the same memory would help.
- Running-time per element: STAR-T may check up to O(∑_i w·k/2^i) = O(wk) buckets per incoming element (worst-case). For w ≈ Θ((m/k) log k) this is fine when m ≪ k, but if m ≈ k, this becomes O(k log k) per element, which could be heavy. Please discuss per-element time, marginal-oracle calls, and any practical pruning (e.g., early stopping, data structures, potential amortization).
- Parameter selection: The algorithm requires τ ≈ f(OPT). The parallel guessing approach increases memory by a log factor; the precise schedule and the overheads are not elaborated in the main text. More guidance on choosing τ and ε in practice would aid reproducibility.
- Notational/algorithmic clarity: There are inconsistencies that should be fixed.
  - In text thresholds are τ/2^i, while Algorithm 1 uses τ/min{2^i,k}; clarify that partitions run up to ⌈log k⌉ so 2^i ≤ k except possibly the last.
  - Throughout, 2i seems to denote 2^i but is typeset without a caret; please standardize.
  - The expression for τ in Theorem 4.1 appears mangled; it seems intended to be τ proportional to f(OPT) with a specific constant. Please correct the formula.
  - Space bound alternates between O((k + m log k) log k) and O((k + m log k) log^2 k) (once τ-guessing is included). State these consistently and separate the known-τ and unknown-τ regimes.
- Empirical evaluation:
  - Memory/time: Report the actual summary sizes |S| and wall-clock times per method. Since SIEVE-STREAMING uses O(k log k) memory and STAR-T uses O((k + m log k) log k) (or higher with guessing), fair memory caps would contextualize the accuracy gains.
  - Baselines: It would be useful to include the concurrent deletion-robust streaming method [18] (if available) or at least a discussion comparing empirical trade-offs.
  - Sensitivity: Provide ablations over w and m, and plot performance vs. number of deletions m with fixed k to validate robustness-memory trade-offs.
  - Some experiments remove more than k or theoretical m while using w=1. It’s encouraging that performance degrades gracefully; quantifying this curve would strengthen the practical message.
- Scope: The paper focuses on cardinality constraints; brief discussion of extending to matroid or knapsack constraints would broaden impact.

Quality
The paper is technically solid overall, with a coherent analysis. Some proof details are deferred; the balancing arguments are plausible. Clarifying the τ expression and notational issues would improve rigor. Experiments are positive but would benefit from runtime/memory reporting and sensitivity analyses.

Clarity
Generally well-written and organized. Fix the notational issues (2^i vs 2i, threshold condition), clean up the τ formula, and make the dependence on k, m, w explicit in time/space complexity.

Originality
The partitioned thresholding design for robust streaming is novel, and the memory–approximation trade-off is distinct from prior/concurrent work. The combination of ideas is well-justified.

Significance
Addresses a timely and practical problem with theoretical guarantees and a reusable summary useful for interactive settings. Likely to be of interest to researchers and practitioners dealing with large-scale, deletion-robust summarization.

Questions for the authors
- Can you provide a precise per-element time complexity and empirical throughput? Any tricks to reduce marginal evaluations?
- How sensitive is performance to τ-guessing granularity (ε) and to w? Any heuristic for picking w when m is unknown?
- Can the analysis be tightened to improve the constant? Are there lower bounds matching your memory–approximation trade-off?
- Could the method extend to matroid constraints using per-partition independence checks?

Minor comments/typos
- Use 2^i consistently. Clarify thresholds τ/2^i vs τ/min{2^i,k}.
- The τ definition in Theorem 4.1 appears malformed.
- STreAming capitalization in Algorithm 1 looks unintended.

Overall recommendation
Weak accept. The paper offers a clear and useful contribution with novel algorithmic ideas, solid theory, and promising experiments. Addressing the clarity issues and adding runtime/memory and sensitivity analyses would strengthen the final version.

Confidence
High.

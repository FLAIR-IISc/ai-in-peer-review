### Summary of the Paper
This paper addresses the problem of robust submodular maximization in a single-pass streaming setting. The authors formulate a practical two-stage problem: (1) in a streaming stage, construct a small, robust summary set `S` from a large dataset; (2) in a query stage, after an arbitrary removal of up to `m` elements `E` from the universe, efficiently find a high-quality solution of size `k` from the remaining summary `S \ E`.

The core contribution is a novel streaming algorithm, STAR-T, which builds the summary `S` using a partitioned structure. The summary consists of `log k` partitions, where partitions with a higher index have exponentially larger buckets but are filled using exponentially smaller greedy thresholds. At query time, a simple greedy algorithm (STAR-T-GREEDY) is run on the remaining summary `S \ E`. The authors provide a theoretical guarantee, proving that this procedure achieves a constant-factor approximation (0.149) while using `O((k + m log k) log k)` space for the summary `S`. The paper is further supported by strong experimental results on dominating set and personalized movie recommendation tasks, where the proposed method performs comparably to, or even outperforms, a strong streaming baseline (SIEVE-STREAMING) that has the unfair advantage of knowing the removed set `E` in advance.

### Strengths (Arguments for Acceptance)

1.  **Well-Motivated and Important Problem:** The paper tackles a highly relevant problem at the intersection of streaming algorithms, submodular optimization, and robustness. The two-stage model, where a summary is pre-computed and must be resilient to future, unknown removals, is a practical and well-formulated abstraction of real-world challenges in areas like recommender systems or influence maximization, where data is massive and adaptability is key.

2.  **Novel and Elegant Algorithmic Design:** The STAR-T algorithm is a clever and non-trivial extension of existing ideas. It successfully blends the thresholding concept from SIEVE-STREAMING with a partitioned structure reminiscent of non-streaming robust algorithms like PRO [11]. The use of exponentially decreasing thresholds across partitions is a key innovation that enables the robust theoretical guarantee in the streaming context.

3.  **Strong Theoretical Contribution:** The paper provides the first constant-factor approximation for this specific robust streaming problem. The analysis is comprehensive, handling different cases based on whether partitions become "full" or not, and skillfully balancing the trade-offs to derive the final approximation ratio. The memory complexity, which is logarithmic in `k`, is also a significant advantage over concurrent work [18] that has a linear dependence on `k`.

4.  **Excellent Experimental Validation:** The empirical evaluation is a major strength. The choice of baseline comparison—pitting STAR-T against an "oracle" version of SIEVE-STREAMING that knows the removed set `E` beforehand—is particularly compelling. The fact that STAR-T performs comparably under this challenging setup strongly validates the quality of the robust summary it constructs. The interactive movie recommendation experiment is an excellent use case that showcases the practical utility of the proposed framework.

### Weaknesses (Areas for Improvement)

1.  **Small Theoretical Constant:** The proven approximation constant of 0.149 is quite low. While constant-factor guarantees in this domain are often small and hard-won, the empirical performance is clearly much better. This suggests a potential gap between the analysis and the algorithm's true performance. A discussion on the tightness of the analysis or potential avenues for improving the bound would strengthen the paper.

2.  **Discrepancy Between Theory and Practice for Parameter `w`:** The theory requires the memory parameter `w` to scale with `m` and `log k` (`w >= ⌈4dlog kem / k⌉`), which can be large. However, the experiments achieve excellent results with `w=1`, even in scenarios where the number of removals is much larger than `k` (e.g., the movie genre experiment). This is a positive practical finding, but the large gap between the theoretical requirement and the practical setting deserves more discussion. It would be beneficial for the authors to comment on why the algorithm is so robust in practice, even with minimal `w`, and whether the theoretical dependency on `m` is an artifact of the proof technique.

3.  **Clarity on the Threshold `τ`:** The main algorithm (Algorithm 1) and theorem (Theorem 4.1) are presented assuming knowledge of `τ`, which depends on the unknown value `f(OPT(k, V \ E))`. The standard technique for handling this is addressed later in Section 4.1 and the appendix. While this is a common practice, the main text could be slightly clearer upfront that the initial presentation makes this simplifying assumption. It would also be helpful to state explicitly in the experimental section how the grid of `τ` values was chosen for the parallel instances.

### Detailed Comments and Questions

*   **Memory Complexity Notation:** There appears to be a slight inconsistency in the stated memory complexity. The abstract gives `O((m log k + k) log^2 k)`, whereas Theorem 4.1 and the following text state `O((k + m log k) log k)`. Based on the proof sketch (Eq. 3), the size is `O(wk log k)`. Substituting the required `w` seems to yield a `log^2 k` term involving `m`. Please clarify the precise bound. This is a minor point but ensuring consistency would improve clarity.
*   **Adversarial Removals:** The model considers removal of *any* `m` elements from `V`, but the experiments test random removals and greedy removals from the summary `S`. The greedy removal is a good proxy for an adversarial attack on `S`. Have the authors considered an adversary that removes elements from `V \ S` that are "close substitutes" for elements in `S`, which could potentially harm the objective value more? This is not a required experiment, but a comment on the power of the adversary would be interesting.
*   **STAR-T-SIEVE:** The experiments include a "STAR-T-SIEVE" variant. Its performance is very similar to STAR-T-GREEDY. This is an interesting finding. Does this suggest that for the small, high-quality summary `S`, the simple GREEDY algorithm is sufficient to extract most of the value, and the added complexity of a streaming algorithm on `S` is unnecessary? A brief comment on this would be insightful.

### Overall Assessment

This is an excellent paper that makes significant contributions to the field of large-scale submodular optimization. It introduces a novel and practical problem, proposes an elegant algorithm with strong theoretical guarantees, and validates it with compelling experiments. The strengths—particularly the novelty of the problem formulation and the strength of the experimental design—far outweigh the minor weaknesses related to the small constant and the theory-practice gap for the parameters. The work is technically sound, clearly presented, and addresses a problem of high significance for the NIPS community. It advances the state of the art and is likely to inspire follow-up research.

**Recommendation: Accept**

This paper addresses the problem of robust submodular maximization in a single-pass streaming setting. The goal is to design an algorithm that processes a data stream once to build a small summary set `S`, such that even after an adversary removes up to `m` elements from `S`, one can still efficiently find a `k`-sized solution within the remaining elements that approximates the optimal offline solution. The authors propose a two-stage procedure. In the first stage, the STAR-T algorithm creates the robust summary `S` by assigning incoming elements to a series of partitions, each with an exponentially decreasing marginal gain threshold. In the second stage, after `m` elements are removed from `S`, the simple GREEDY algorithm is run on the remainder of the summary to find the final `k`-sized set.

This work builds upon two significant lines of research. It extends the thresholding idea from the streaming submodular maximization algorithm SIEVE-STREAMING [8] to a robust setting. For the robust structure, it adapts the partitioned summary idea from the non-streaming algorithm PRO [11]. The combination of these ideas is non-trivial, requiring a novel analysis. The authors also properly situate their contribution with respect to the concurrent work of Mirzasoleiman et al. [18], which tackled the same problem. The main theoretical contribution is a constant-factor approximation guarantee (~0.149) for the problem, achieved using a summary of size `O((k + m log k) log k)`.

**Strengths:**

1.  **Significance and Originality:** The paper tackles a highly relevant and challenging problem. The combination of streaming data processing and the need for robust solutions is critical for many large-scale applications (e.g., recommender systems, influence maximization). To my knowledge, this is one of the first works (along with the concurrent [18]) to provide a formal solution with guarantees for this problem setting. The proposed STAR-T algorithm is a clever and novel synthesis of ideas from streaming and robust optimization, and its analysis appears sound.

2.  **Theoretical Contribution:** The paper provides a constant-factor approximation guarantee for the robust streaming submodular maximization problem, which is a strong theoretical result. A key strength is the memory complexity of the proposed algorithm. The summary size's dependence on `k` is `O(k log k)`, which is substantially better than the `O(k^2)` (ignoring other factors) dependence of the concurrent work in [18]. This makes the proposed method significantly more scalable for applications requiring a large summary size `k`.

3.  **Experimental Validation:** The experiments are well-designed and compelling. The authors compare their method against a very strong baseline: the non-robust SIEVE-STREAMING algorithm, which is given the significant advantage of knowing in advance which elements will be removed. The fact that the proposed method performs comparably to, and sometimes better than, this baseline is a powerful demonstration of its practical effectiveness. The applications chosen (dominating set and interactive movie recommendation) are relevant and clearly illustrate the utility of the robust summarization framework.

**Weaknesses and Suggestions for Improvement:**

1.  **Clarity and Consistency:** There are minor inconsistencies in the paper that should be addressed to improve clarity.
    *   The memory complexity is stated differently in the abstract (`O((m log k + k) log^2 k)`), Theorem 4.1 (`O((k + m log k) log k)`), and the conclusion (`O((k + m log k) log^2 k)`). Please unify this to a single, correct expression and ensure the derivation supports it.
    *   In Algorithm 1, line 5, the threshold is given as `τ/min{2^i, k}`, whereas the text preceding it describes the threshold for partition `i` as `τ/2^i`. These should be made consistent.

2.  **Comparison to Concurrent Work [18]:** The paper rightly highlights its superior memory complexity compared to [18]. However, for a balanced comparison, it should also mention the trade-offs. The algorithm in [18] achieves a better approximation guarantee of `1/2 - ε`, which is considerably higher than the ~0.149 achieved here. A brief discussion of this trade-off between memory efficiency and the approximation factor would provide a more complete picture for the reader and strengthen the paper's positioning.

3.  **Gap between Theory and Practice for Parameter `w`:** The theoretical analysis requires the memory parameter `w` to be `Ω(m log k / k)`. However, the experiments achieve excellent results with `w=1`. This suggests the theoretical bound might be loose for practical scenarios. It would be beneficial to add a short discussion on this point. Why does `w=1` work so well? Is it because the adversarial removals in the experiments are not worst-case, or is there another reason? This would add valuable insight for practitioners looking to apply this method.

4.  **Approximation Constant:** The approximation guarantee of 0.149, while a significant theoretical achievement (any constant factor is valuable for this problem), is relatively small. It would be helpful to briefly comment on whether this is likely an artifact of the proof technique or potentially inherent to the problem structure under the algorithm's memory constraints.

**Summary of Arguments:**

*   **Pro-acceptance:** The paper addresses a novel and important problem, presents an elegant algorithm with a strong theoretical guarantee, and provides convincing empirical evidence of its effectiveness. The memory complexity is a significant advantage over concurrent work.
*   **Con-acceptance:** The paper has some minor clarity issues and could be strengthened by a more complete discussion of the trade-offs with concurrent work and the gap between theory and practice for the `w` parameter.

Overall, this is a high-quality paper making a significant contribution to the field of large-scale submodular optimization. The strengths far outweigh the weaknesses, which are largely presentational and can be addressed in a revision. I recommend acceptance.

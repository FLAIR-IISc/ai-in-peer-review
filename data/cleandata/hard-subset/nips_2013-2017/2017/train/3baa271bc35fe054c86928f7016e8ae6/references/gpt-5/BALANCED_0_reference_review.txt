Summary and relation to prior work
The paper addresses robust submodular maximization in a one-pass streaming setting with a two-stage workflow: (i) a streaming stage that constructs a small summary S robust to the deletion of up to m elements, and (ii) a query stage that, after arbitrary deletions E of size ≤ m, runs GREEDY over S\E to output a size-k solution. The proposed streaming algorithm, STAR-T, organizes S into partitions of exponentially growing bucket sizes, each governed by an exponentially decreasing threshold. Elements are placed into the first non-full bucket whose threshold they exceed. The analysis shows that, with w chosen appropriately (roughly proportional to m log k/k), STAR-T produces |S| = O((k + m log k) log k) elements and, upon deletions, STAR-T-GREEDY returns a k-set achieving c ≈ 0.149(1 − 1/⌈log k⌉) times OPT(k, V\E). An extension removes the need to know f(OPT) via parallel instances (as in Sieve-Streaming) with an extra log factor.

This work builds on: (i) classical greedy guarantee for monotone submodular maximization [Nemhauser et al. 1978], (ii) streaming submodular maximization via thresholded sieving (Sieve-Streaming) [Badanidiyuru et al. 2014], and (iii) robust (non-streaming) constructions using partitioned buckets (e.g., PRO) [Bogunovic et al. 2017] and robust selection [Orlin et al. 2016]. It is closely related to concurrent deletion-robust streaming work [Mirzasoleiman et al., ICML’17], which achieves a 1/2−ε approximation with O(mk log k/ε) memory; the present paper trades a worse approximation constant for significantly lower memory, replacing linear-in-k dependence with logarithmic for the m-dependent term.

Strengths
- Novel algorithmic design: a clean combination of partitioned bucket structures (as in robust offline PRO) with exponentially decreasing thresholds (as in Sieve-Streaming) for the streaming robust setting.
- Strong memory–approximation trade-off: memory O((k + m log k) log k) (or O((k + m log k) log^2 k) depending on the exact statement) versus O(mk log k/ε) in [18], yielding substantial savings when m is large.
- Single-pass streaming with a query-time procedure that only accesses S, aligning with practical “right-to-be-forgotten” and interactive recommendation scenarios.
- Theoretical guarantees are carefully argued through case analyses (full vs non-full partitions) with constructive lemmas; the analysis technique is of independent interest.
- Empirically competitive: on dominating set and recommendation tasks, STAR-T variants match or slightly outperform Sieve-Streaming even when the latter knows deletions in advance, and are close to GREEDY on the smaller recommendation task.

Weaknesses and concerns
- Approximation constant is modest (≈0.149). This is significantly below the 1/2−ε of concurrent work; the paper’s value rests on the memory advantage and practical performance, which should be emphasized with broader experiments.
- Minor inconsistencies and clarity issues:
  - Space bound appears as O((k + m log k) log k) in Theorem 4.1, but O((k + m log k) log^2 k) elsewhere (intro and conclusion). Please reconcile and state the tightest bound consistently.
  - Threshold specification: the text says threshold τ/2^i for partition i, but Algorithm 1 line 5 uses τ/min{2^i, k}. Clarify which is intended and align analysis accordingly.
  - The definition of τ in Theorem 4.1 appears malformed (missing factors/parentheses). Please restate τ cleanly as τ = β · f(OPT(k, V\E)) with an explicit β.
  - Notation dlog ke: base and rounding should be stated once and used consistently.
- Assumptions at query time: the robust definition is with respect to E ⊆ V, but STAR-T-GREEDY runs on S\E. If E∩S^c ≠ ∅, this has no effect; this is fine but worth clarifying early to avoid confusion.
- Empirical evaluation lacks direct comparison to the concurrent deletion-robust streaming method [18]. Even if memory prohibits scaling, a small/medium-scale comparison would strengthen the empirical case and illuminate the approximation/memory trade-offs.
- Limited experimental breadth: two applications, primarily graph coverage and a single-user recommendation scenario, with w=1. More systematic sweeps over m, w, and k, and reporting memory usage and oracle calls would be helpful.
- Runtime/queries: the query-stage GREEDY complexity on S\E is not analyzed or reported; practical latency is important for interactive use.

Quality
Technically sound overall, with a clear high-level proof strategy and plausible lemmas. However, the paper should fix the noted inconsistencies (space bound, threshold expression, τ definition), and expand on implementation details (evaluation counts, choice of w, handling unknown f(OPT), and the true total memory including parallel instances).

Clarity
Generally well written with intuitive explanations of the structure and proof strategy. The above notational/formula inconsistencies detract from clarity and should be corrected. A brief pseudocode or description of the parallel multi-τ scheme (Appendix E) in the main text would aid readability.

Originality
A novel combination of partitioning and thresholding in the streaming robust setting, with new analysis to get constant-factor guarantees and improved memory scaling in m. The idea is simple and could be influential.

Significance
Addresses an important and timely problem (robust streaming summarization). The memory improvement over prior robust streaming work is meaningful. The approximation factor is small, but empirical results suggest practicality. With additional experiments and tightened constants, the impact could be substantial.

Questions and suggestions for the authors
- Unify and correct the space complexity and τ expressions; clarify the threshold used in Algorithm 1.
- Report function-evaluation counts and end-to-end runtime for both stages; include memory usage vs w, m, and k.
- Compare against the concurrent deletion-robust algorithm [18] on at least one dataset, even if only small-scale.
- Explore the effect of w>1 in practice, and sensitivity to m mis-specification (e.g., actual deletions > m).
- Discuss potential constant-factor improvements (e.g., alternative balancing of Lemmas 4.3/4.4, tighter charging arguments) and whether 0.149 is likely an artifact of the proof.
- Clarify how τ is chosen in practice in the multi-instance setting and how many instances are run.

Arguments for acceptance
- Novel, simple algorithm with strong memory guarantees for robust streaming submodular maximization.
- Solid theory and practically competitive results on two tasks.
- Addresses an important use case (deletions/forgetting) with a reusable summary.

Arguments against acceptance
- Small approximation constant; missing empirical comparison to the strongest concurrent baseline.
- Notational inconsistencies and some clarity issues in the main theorem and algorithm.
- Limited evaluation breadth and analysis of computational costs.

Recommendation
Weak accept. The paper presents a clean and useful algorithmic idea with provable guarantees and favorable memory–accuracy trade-offs, corroborated by experiments. Addressing the clarity issues and adding a comparison to [18], plus more detailed empirical analysis, would further strengthen the submission.

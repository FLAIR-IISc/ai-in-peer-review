This paper addresses the problem of robust submodular maximization in a single-pass streaming setting. The authors propose a two-stage framework: first, a streaming algorithm builds a robust summary set `S` from a large dataset; second, after an adversary removes up to `m` elements from `S`, a query algorithm efficiently finds a near-optimal solution of size `k` from the remaining elements, without accessing the original dataset.

The main contribution is a novel streaming algorithm, STAR-T, which constructs the summary `S` by organizing elements into partitions and buckets. Partitions are associated with exponentially decreasing thresholds, and incoming stream elements are greedily placed into the first available bucket for which their marginal gain exceeds the corresponding threshold. This structure is inspired by both non-robust streaming algorithms like SIEVE-STREAMING and non-streaming robust algorithms like PRO. For the query stage, the authors propose using the standard GREEDY algorithm on the remaining elements of the summary.

The key theoretical result is a constant-factor approximation guarantee for the solution obtained in the query stage. The algorithm constructs a summary of size `O((k + m log k) log k)` and, for any removal of `m` elements, the final solution is proven to be a `0.149(1 - 1/log k)`-approximation to the true optimum. This result is significant because the memory complexity has only a logarithmic dependence on the desired solution size `k`, which compares favorably to concurrent work (Mirzasoleiman et al., 2017) that has a linear dependence on `k`. The paper also provides strong empirical validation on dominating set and personalized recommendation tasks, showing that the proposed method performs comparably to, and sometimes better than, an oracle version of SIEVE-STREAMING that knows the removed elements in advance.

### Strengths and Weaknesses

**Strengths:**

1.  **Well-motivated and Important Problem:** The problem of building a robust summary from a data stream is highly relevant for large-scale applications where data cannot be stored or re-processed, and where summaries might be subject to failures, updates, or adversarial attacks. The two-stage framework is practical and addresses a clear need.

2.  **Novel and Elegant Algorithm Design:** The STAR-T algorithm is a clever synthesis of ideas. The partitioned structure with exponentially decreasing thresholds is a novel way to ensure that the summary `S` contains both high-value elements (captured by high-threshold buckets) and a diverse set of backup elements (captured by low-threshold buckets), providing inherent redundancy and robustness. The simplicity of using the standard GREEDY algorithm in the query stage is also a plus.

3.  **Strong Theoretical Contribution:** The paper provides the first constant-factor approximation guarantee for this problem with a memory complexity that is polylogarithmic in `k`. The analysis, which involves a careful case analysis based on whether partitions are filled or not, is non-trivial and technically sound. The favorable memory complexity is a key advantage over concurrent work and represents a significant theoretical advance.

4.  **Convincing Experimental Evaluation:** The experiments are thorough and well-designed.
    *   The choice of comparing against SIEVE-STREAMING with advance knowledge of the removed set is a very strong baseline. The fact that STAR-T performs competitively shows that the pre-computed summary is of high quality.
    *   Testing against both random and adversarial removals demonstrates the algorithm's robustness under different scenarios.
    *   The personalized movie recommendation example is a compelling use case that highlights the practical utility of the interactive, two-stage approach.

**Weaknesses / Suggestions for Improvement:**

1.  **Low Approximation Constant:** The proven constant of `c ≈ 0.149` is quite low. While theoretical constants in this area are often loose and the empirical performance is excellent, this value is worth noting. It would be interesting if the authors could comment on potential paths to tightening this analysis or whether it is an artifact of the proof technique (e.g., balancing multiple worst-case inequalities).

2.  **Minor Clarity Issues:**
    *   The memory complexity is stated slightly differently in the abstract (`O((m log k + k) log^2 k)`) and Theorem 4.1 (`O((k + m log k) log k)`). While these may be equivalent under certain assumptions about `m` and `k`, unifying the statement and providing a clear derivation would improve clarity.
    *   In Algorithm 1 (line 5), the threshold is given as `τ/min{2^i, k}`, whereas the text description often refers to it as `τ/2^i`. These are equivalent for `i <= log k`, but for consistency, the text should match the pseudocode precisely.

3.  **Gap Between Theory and Practice for `w`:** The theoretical result requires the parameter `w` to scale with `m` and `log k` (`w >= O(m log k / k)`). However, the experiments show excellent performance with `w=1`. This is a positive practical finding, but it highlights a gap. It would be beneficial for the authors to briefly discuss this, perhaps suggesting that the worst-case analysis for `w` is pessimistic and that the average-case performance is much better.

### Summary of evaluation

*   **Quality:** High. The paper is technically sound, with a solid theoretical analysis providing a constant-factor guarantee. The claims are well-supported by a strong set of experiments that validate the approach against relevant and powerful baselines.
*   **Clarity:** Very Good. The paper is well-written and organized. The core ideas are communicated effectively. The minor points mentioned above could be addressed to further improve clarity. The paper provides sufficient detail for an expert to understand the contributions.
*   **Originality:** High. The paper introduces a novel and practical problem formulation for robust streaming submodular maximization. The STAR-T algorithm and its partitioned thresholding mechanism are new. The work is clearly differentiated from prior and concurrent research.
*   **Significance:** High. This work makes a significant contribution to the field of large-scale submodular optimization. The problem is important, and the proposed algorithm is practical and efficient, particularly due to its low memory requirement with respect to `k`. Researchers and practitioners working on data summarization, recommender systems, and influence maximization are likely to build upon these ideas.

**Recommendation: Accept**

This is a strong paper that introduces a novel algorithm for an important and challenging problem. The contributions are significant on both theoretical and empirical fronts. The algorithm's design is elegant, and its superior memory complexity makes it a notable advance in the state of the art for robust streaming optimization. The minor weaknesses are easily addressable and do not detract from the overall quality of the work.

**Summary of the Paper**

This paper addresses the problem of robust submodular maximization subject to a cardinality constraint in a single-pass streaming setting. The problem is formulated in a two-stage process. In the first (streaming) stage, the goal is to process a stream of data to create a small, robust summary set `S`. In the second (query) stage, after an arbitrary set `E` of at most `m` elements is removed from the summary `S`, the goal is to efficiently find a solution `Z` of size at most `k` from the remaining elements `S \ E` that provides a good value for the submodular objective function `f`.

To solve this, the authors propose a procedure based on two algorithms: STAR-T for the streaming stage and STAR-T-GREEDY for the query stage. STAR-T is a one-pass algorithm that constructs the summary `S` by organizing elements into a structure of partitions and buckets. The partitions are associated with exponentially decreasing thresholds, and incoming elements are placed into the first available bucket for which their marginal gain exceeds the corresponding threshold. This design is a novel combination of the partitioned structure from the offline robust algorithm PRO [11] and the thresholding idea from the streaming algorithm SIEVE-STREAMING [8]. In the query stage, STAR-T-GREEDY simply applies the standard GREEDY algorithm to the set `S \ E`.

The main theoretical contribution is a proof that this procedure achieves a constant-factor approximation guarantee (`c ≈ 0.149`). The STAR-T algorithm produces a summary of size `O((k + m log k) log k)`, which has a favorable logarithmic dependency on `k` compared to concurrent work. The paper also provides a standard method to handle the practical case where the optimal value (needed to set the initial threshold) is unknown. The effectiveness of the approach is demonstrated experimentally on dominating set and movie recommendation tasks.

**Strengths**

1.  **Significance:** The paper tackles a highly relevant problem. Robustness is a critical desideratum in many machine learning applications, and the streaming constraint is essential for handling large-scale datasets. The combination of these two aspects—robustness and streaming computation—addresses a practical and challenging gap in the literature. The two-stage setup, where a summary is pre-computed and then queried after removals, is well-motivated and applicable to real-world scenarios like interactive recommendation systems.

2.  **Originality:** The proposed algorithm, STAR-T, is novel. It skillfully synthesizes ideas from two distinct lines of prior work: the partitioned robust structure from offline submodular maximization [11] and the thresholding technique from non-robust streaming algorithms [8]. The introduction of an exponentially decreasing thresholding rule across partitions is a key innovation that enables the theoretical analysis in the robust streaming context. The analysis itself, which cleverly handles two distinct cases based on whether partitions are filled, appears to be a non-trivial technical contribution.

3.  **Technical Quality:** The paper provides a strong theoretical result, establishing the first constant-factor approximation guarantee for this problem setting. The analysis leading to Theorem 4.1 appears sound. The derived space complexity has a logarithmic dependence on `k`, which is an improvement over concurrent work [18] that has a linear dependence. The extension in Section 4.1 for handling an unknown optimal value is a standard but important addition for practical applicability. The experimental evaluation is solid, demonstrating that the proposed method performs comparably to or even better than SIEVE-STREAMING, even when the latter is given the significant advantage of knowing the removed elements in advance.

**Weaknesses**

1.  **Clarity on Space Complexity:** There is a minor inconsistency in the statement of the space complexity. The abstract mentions a size of `O((m log k + k) log^2 k)`, while Theorem 4.1 states `O((k + m log k) log k)`. The proof of Theorem 4.1 derives a bound of `O(wk log k)`, which, combined with the required setting of `w`, seems to yield `O(m log^2 k)`. Clarifying the precise bound and ensuring its consistent presentation throughout the paper would strengthen the manuscript.

2.  **Value of Approximation Constant:** The derived approximation constant `c ≈ 0.149` is relatively modest. While achieving any constant factor for this problem is a significant result, the gap between this value and the `0.5 - ε` achieved by non-robust streaming algorithms like SIEVE-STREAMING is substantial. A discussion on the potential tightness of this constant or possible avenues for improving it through a refined analysis would be a valuable addition.

3.  **Theory vs. Practice in Experiments:** The theoretical results rely on setting the memory parameter `w >= ⌈4dlog_k(e)m/k⌉`, which implies `w` should grow with the robustness parameter `m`. However, the experiments are conducted with `w=1`. While the paper correctly notes that this suggests the practical performance is better than the theoretical bounds might indicate, it also raises questions about the tightness of the analysis with respect to `w`. An experimental analysis showing the performance trade-off as `w` is varied would have provided deeper insight into the algorithm's behavior and the relationship between the theory and practice.

**Arguments for Acceptance:**

*   **Pro:** The paper addresses a significant and well-motivated problem at the intersection of robust optimization and streaming algorithms.
*   **Pro:** It presents a novel algorithm that elegantly combines ideas from related literature.
*   **Pro:** It provides the first constant-factor approximation guarantee for the problem, supported by a technically sound analysis.
*   **Pro:** The algorithm's space complexity is favorable compared to concurrent work.
*   **Pro:** Empirical results on real datasets validate the algorithm's practical effectiveness.

**Arguments against Acceptance:**

*   **Con:** The approximation constant achieved is somewhat low.
*   **Con:** There is a minor lack of clarity regarding the precise space complexity bound.
*   **Con:** The gap between the theoretical requirement for the memory parameter `w` and the value used in experiments is not fully explored.

Overall, the strengths of this paper, particularly its novelty, significance, and strong theoretical contribution to an important problem, substantially outweigh its minor weaknesses. The work represents a clear advance in the field of large-scale submodular optimization.

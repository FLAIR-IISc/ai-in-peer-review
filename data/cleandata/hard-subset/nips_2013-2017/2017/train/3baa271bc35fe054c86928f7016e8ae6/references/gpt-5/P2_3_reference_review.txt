Summary and relation to prior work
The paper studies robust submodular maximization in a two-stage streaming setting. In the streaming stage, the algorithm (STAR-T) processes elements in one pass and stores a compact, robust summary arranged into partitions of exponentially growing bucket sizes, with exponentially decreasing thresholds per partition. In the query stage, after any removal of up to m elements, it runs GREEDY on the remaining summary to return a size-k solution. The main theoretical result shows that with memory O((k + m log k) log k) when OPT is known (and O((k + m log k) log2 k) when OPT is approximated via parallel thresholds), the query-stage solution achieves a constant-factor approximation c ≈ 0.149 to OPT(k, V \ E) for any |E| ≤ m. Empirically, the method performs comparably to SIEVE-STREAMING (which is advantaged by being allowed to know E at query time) on dominating set and a movie recommendation objective; a STAR-T-SIEVE variant is also evaluated.

This work extends SIEVE-STREAMING-style thresholding to the robust setting, and adapts structural ideas from PRO (non-streaming, partitioned buckets) to a one-pass streaming algorithm with new analysis. It also relates to and contrasts with a concurrent streaming robust method [18] offering 1/2 − ε approximation with O(mk log k/ε) memory; here, the constant is smaller but memory scales only logarithmically in k.

Quality
- Theory: The main theorem and lemmas are plausible and technically careful, with clear case analysis depending on whether many buckets fill, and bounding the loss due to deletions via partition structure and thresholds. The memory–approximation tradeoff is well articulated. The need to know OPT is addressed via the standard multi-instance trick (incurring a log factor). One concern: there are minor inconsistencies/typos that should be clarified, e.g., whether thresholds are τ/2^i or τ/min{2^i, k} (the pseudocode uses the latter), and the exact expression for τ in Theorem 4.1 appears garbled in the text. Also, the abstract/intro claim O((k + m log k) log2 k) while Theorem 4.1 gives O((k + m log k) log k); the extra log comes from the parallelization (Theorem 4.5), but this should be stated consistently.
- Experiments: The tasks (dominating set, facility-location-style recommendation) are standard and appropriate. Comparisons to SIEVE-STREAMING (with knowledge of E) and a random baseline are reasonable; results show STAR-T(-GREEDY) is competitive or better. However, empirical evaluation is somewhat limited:
  - No comparison to the concurrent deletion-robust streaming baseline [18] (even if concurrent, a discussion of expected empirical tradeoffs would help).
  - Memory/runtime and sensitivity to parameters (w, τ, number of instances) are not reported.
  - In experiments, w=1 is used although the guarantee requires w ≥ ceil(4 m log k / k); and they sometimes remove more than k elements, far beyond the guaranteed m. While it is interesting that performance remains strong, this weakens the empirical validation of the stated guarantees. Reporting results within-theory and ablations across m and w would strengthen the case.

Clarity
The paper is generally well written and organized; the algorithmic structure, intuition (partitions, thresholds), and proof sketches are clear. Pseudocode is helpful. As noted, there are minor inconsistencies that should be fixed:
- Unify the threshold definition (text vs Algorithm 1).
- Make the memory bounds consistent across abstract, theorem, and conclusion; explicitly separate the “known OPT” and “unknown OPT” cases.
- The expression for τ in Theorem 4.1 needs to be typeset correctly.
- Explicitly state the dependence of w on m and k early and repeat it in experiments, including the m used.

Originality
The combination of a partitioned structure (from PRO) with streaming thresholding and a new analysis for robustness to arbitrary m with O(m log2 k) memory is novel. The decrease of thresholds by partition index and the way deletions are handled in the analysis are nontrivial. The concurrent work [18] addresses a similar problem but with different tradeoffs; the memory vs. approximation frontier explored here appears new.

Significance
Robust streaming submodular maximization is important for large-scale summarization with deletions (“right to be forgotten,” failures, user rejections). The memory bound that scales logarithmically with k is appealing in high-k regimes, offering a practical alternative to methods with linear dependence on k. The constant 0.149 is modest compared to 1/2 − ε, but the memory improvement can be decisive in streaming settings. The empirical results, while not exhaustive, suggest practical utility.

Pros
- Clear, principled algorithm with one pass and small memory.
- New analysis yielding constant-factor robustness with O(m log2 k) memory when OPT is unknown.
- Empirical results on two relevant applications, competitive with strong baselines even in adversarial removal scenarios.
- Good discussion of related work and position vs PRO and SIEVE-STREAMING; explicit memory–approximation tradeoff vs concurrent [18].

Cons
- Minor but noticeable inconsistencies/typos in thresholds, τ, and memory bounds; clarify for reproducibility.
- Limited experimental scope: no space/time metrics, no ablations over w and m, and evaluation sometimes outside the proven regime (w=1, |E| > guaranteed m).
- No empirical or theoretical comparison to concurrent [18]; at least a more detailed discussion would help readers understand when to prefer which method.
- Approximation constant is relatively small; any practical guidance for setting w to trade memory for robustness empirically would be valuable.

Questions and suggestions for authors
- Please reconcile the threshold rule (τ/2^i vs τ/min{2^i,k}) and provide the correct, readable expression for τ in Theorem 4.1.
- Report memory and runtime usage, and sensitivity to w and m, including runs within the theoretically supported m for a given w.
- Discuss or, if possible, compare empirically with [18]; if not feasible, provide a more detailed practical tradeoff analysis (e.g., when k is large and memory is tight).
- Clarify how many parallel instances are used in practice to approximate OPT and their overhead.
- Consider releasing code for reproducibility.

Recommendation
Weak accept. The paper makes a solid theoretical and algorithmic contribution to an important streaming problem, with a clear memory–approximation tradeoff distinct from concurrent work. With clarified presentation and strengthened experiments (especially within-theory regimes and resource metrics), this would be a strong NeurIPS paper.

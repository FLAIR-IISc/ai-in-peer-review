### Summary

This paper addresses the problem of robust submodular maximization in a streaming setting. The authors propose a two-stage framework: first, a single-pass streaming algorithm (STAR-T) constructs a small, robust summary `S` of the data. Second, after an arbitrary removal of up to `m` elements from this summary, a simple greedy algorithm (STAR-T-GREEDY) is run on the remaining elements of `S` to produce a solution of size `k`. The key contribution is the STAR-T algorithm itself, which introduces a novel "partitioned thresholding" approach. The summary `S` is structured into partitions, each with an exponentially decreasing greedy threshold. An incoming element is added to the first available bucket for which it satisfies the marginal gain threshold. The authors provide a theoretical analysis showing that this procedure yields a constant-factor approximation guarantee while using `O((k + m log k) log k)` memory. This memory bound is notably efficient, with only a logarithmic dependence on `k`. The approach is validated empirically on dominating set and movie recommendation tasks, where it performs competitively even against baselines that are given the unfair advantage of knowing the removed elements in advance.

### Strengths

1.  **High Novelty and Conceptual Contribution:** The paper's primary strength is its novelty. It elegantly combines two powerful ideas from prior work in a new way to solve a challenging problem. It adapts the partitioned summary structure from non-streaming robust optimization (e.g., [11]) and merges it with the greedy thresholding concept from streaming submodular maximization (e.g., [8]). The resulting algorithm, with its hierarchy of exponentially decreasing thresholds, is a genuinely new and insightful contribution to the field. This is not an incremental improvement but a bold new approach to a difficult problem.

2.  **Significance and Problem Formulation:** The problem of *streaming robust* submodular maximization is highly significant and practical. In many large-scale systems, data is too large to re-process, and summaries are subject to failures or changing user preferences (removals). The two-stage setup, where a robust summary is built once and then queried repeatedly and efficiently, is a compelling and well-motivated model. This work provides the first constant-factor approximation algorithm for this problem, advancing the state of the art in a demonstrable way.

3.  **Strong Theoretical Results:** The paper provides a rigorous theoretical analysis, culminating in a constant-factor approximation guarantee. The proof strategy, which considers different cases based on how full the partitions are, appears sound. Furthermore, the memory complexity of `O((k + m log k) log k)` is a strong result, particularly the logarithmic dependence on `k`, which the authors correctly point out is an improvement over concurrent work [18].

4.  **Clarity and Organization:** The paper is very well-written and easy to follow. The algorithm is described clearly, and the high-level ideas behind the theoretical analysis are effectively conveyed in the proof sketch.

### Weaknesses

1.  **Loose Theoretical Constant:** The derived approximation constant `c ≈ 0.149` is quite small from a practical standpoint. While proving any constant-factor guarantee for this problem is a significant achievement, the looseness of the bound is a minor weakness. However, this is common for worst-case analyses in this domain, and the strong empirical results suggest that the algorithm's real-world performance is much better than the bound might imply.

2.  **Gap between Theory and Practice for `w`:** The memory parameter `w` is required to be `Ω(m log(k)/k)` in the theory to handle `m` removals. However, the experiments are run with `w=1` and show remarkable resilience, even for a large number of removals (e.g., removing 59% of all movies in the recommendation experiment). While this is a positive empirical finding, it points to a gap between the theoretical analysis and the practical behavior of the algorithm, suggesting the analysis could potentially be tightened.

3.  **Standard Technique for Unknown OPT:** The method for handling the unknown value of `f(OPT(k, V \ E))` relies on running parallel instances of the algorithm with guessed values for the threshold `τ`. This is a standard and well-accepted technique borrowed from [8], and while necessary for completeness, it does not represent a novel contribution in itself.

### Recommendation

**Accept.**

This paper makes a significant and novel contribution to the literature on large-scale submodular optimization. The core idea of "partitioned thresholding" is elegant, original, and effectively tailored to the challenging problem of robust submodular maximization in a streaming context. The problem itself is of high practical importance, and this work provides the first constant-factor approximation guarantee with a compelling memory footprint.

While the theoretical constant in the guarantee is small and there is a gap between the theoretical parameter settings and the impressive empirical results, these are minor weaknesses in the face of a substantial conceptual advance. As a reviewer who values novelty and bold new directions, I am highly enthusiastic about this work. It introduces a new algorithmic technique that is likely to inspire follow-up research. The paper is technically solid, clearly presented, and addresses a problem of high significance. It is an excellent fit for NIPS.

Summary and relation to prior work
The paper studies robust submodular maximization in a streaming, two-stage setting. In the streaming stage, the goal is to build a compact summary S that is robust to the deletion of up to m elements. In the query stage, after an arbitrary E with |E| ≤ m is removed, the algorithm must return a size-k set Z using only S \ E that approximates OPT(k, V \ E).

The main algorithm, STAR-T, organizes the summary into partitions with exponentially increasing bucket capacities (up to min{2^i, k}) and exponentially decreasing thresholds, and inserts each incoming element into the first non-full bucket where its marginal gain exceeds the partition’s threshold. The query stage (STAR-T-GREEDY) runs GREEDY on S \ E. The paper proves a single-pass algorithm storing O((k + m log k) log k) elements when provided an α-approximation τ ≈ f(OPT(k, V \ E)); when τ is unknown, a parallel variant inflates space by a factor O(log_{1+ε} k), yielding O((k + m log k) log k log_{1+ε} k). The approximation guarantee is constant: f(Z) ≥ c · f(OPT(k, V \ E)) with c ≈ 0.149 for large k. Compared to the non-streaming robust works [10, 11], this is a streaming method with memory sublinear in n and only logarithmic dependence on k (for fixed m). Compared to concurrent streaming robust work [18] (1/2 − ε approximation but O(mk log k/ε) memory), the present method trades a lower approximation factor for much smaller memory, replacing linear by logarithmic dependence on k. The design extends thresholding ideas from SIEVE-STREAMING [8] to a robust setting via partitioned thresholds and a new analysis.

Quality (technical soundness)
- Theory: The main theorem (Theorem 4.1) provides a clear single-pass memory bound and a constant-factor approximation after arbitrary deletions E, with GREEDY in the query stage. The proof outline relies on two complementary cases: (i) many full buckets in some partition imply the existence of a full bucket retaining high value after deletions; (ii) otherwise, a carefully chosen not-fully-populated bucket in the last partition yields a trade-off bound that, when balanced, produces the constant. The argument relies on the average “exposure” of deleted elements across many buckets and the standard greedy bound for monotone submodular maximization. The dependence of memory on m via w ≥ ⌈4 m log k / k⌉ is transparent, leading to |S| = O((k + m log k) log k). The parallelization trick to remove dependence on unknown OPT follows the standard approach from [8].
- Clarity issues in the theory: There are several typesetting/consistency issues that should be corrected for precision:
  • Threshold rule: the text states threshold τ/2^i, while Algorithm 1 uses τ/min{2^i, k}. These are consistent for i with 2^i ≤ k but differ in the last partition; please clarify intended rule and update the prose accordingly.
  • The expression for τ in Theorem 4.1 appears malformed; as written, it is unclear whether τ is multiplied by or divided by the constant involving (1−e^{-1}), (1−e^{-1/3}), and (1 − 1/⌈log k⌉). Please fix parentheses and factors.
  • Equation (5) has apparent typesetting errors (e.g., “12 α2(1−γ) + 1 α1”). Please restate the bound cleanly.
  • Memory bounds vary across the paper: the Introduction and Conclusion state O((k + m log k) log^2 k), while Theorem 4.1 yields O((k + m log k) log k) when τ is known; Theorem 4.5 adds a factor log_{1+ε} k. Please clearly distinguish the “known-OPT” and “unknown-OPT” cases and align the statements across sections.
- Algorithmic details: The worst-case per-element processing appears to scan all partitions and buckets until placement, which could be O(wk log k) checks per element. While w is small in practice (w=1 in experiments), a discussion of the amortized time and any data structures to avoid scanning many buckets would strengthen the presentation.

Clarity (presentation)
The paper is generally well organized and the high-level ideas are clear. The partitioned thresholding structure is well motivated. However, the noted inconsistencies/typos in formulas and thresholds hinder reproducibility. Algorithm pseudocode and the figure reference help, but a small worked example illustrating insertion decisions across partitions would further aid understanding. The experimental section reads clearly, though numerical details for figures are not included in the text.

Originality
The combination of partitioned buckets (as in the centralized PRO approach [11]) with exponentially decreasing thresholds in a single-pass streaming robust setting appears novel. The analysis technique that balances bounds from two structural cases to obtain a constant factor under deletions is also original within streaming robust submodular maximization. The memory-approximation trade-off relative to [18] is a distinct contribution.

Significance
The problem is important for large-scale summarization with deletion/“right to be forgotten” constraints. The method provides a practically attractive memory footprint (logarithmic in k, linear in k for the base term) and single-pass operation, with a constant-factor guarantee independent of m (given memory scaling with m). While the 0.149 factor is modest compared to 0.5−ε in [18], the drastically reduced memory (logarithmic vs linear in k) is a meaningful trade-off in streaming contexts. The experiments on real datasets (Twitter/Amazon graphs; MovieLens recommendations) demonstrate that the method is competitive with SIEVE-STREAMING even when the latter is granted knowledge of deletions.

Strengths
- Single-pass robust streaming algorithm with provable constant-factor guarantee after arbitrary m deletions.
- Memory O((k + m log k) log k) with known OPT surrogate, and near O((k + m log k) log^2 k) with unknown OPT via parallelization—logarithmic dependence on k in the m-dependent term.
- Conceptually simple design (bucket partitions and thresholds) and simple query stage (GREEDY).
- Empirical results on two application domains showing competitive performance to strong baselines, including an interactive query setting.

Weaknesses
- Several presentation inconsistencies (threshold rule, τ expression, final bound formatting) that must be corrected.
- Approximation factor (≈0.149) is relatively low; a discussion of tightness or possibilities for improving constants would be valuable.
- No runtime/throughput measurements or memory usage reports in experiments; no empirical comparison to [18].
- Details on setting τ in practice (values of ε, number of parallel instances) are deferred; an explicit description of the deployed configuration in experiments would improve reproducibility.
- Worst-case per-element processing cost not analyzed.

Questions for the authors
- Please clarify the intended threshold rule (τ/2^i vs τ/min{2^i,k}), and reconcile Algorithm 1 with the description.
- Fix the expression for τ in Theorem 4.1 and the final bound (Eq. (5)); provide a clean statement of constants.
- Can you provide an amortized per-element processing bound and any data structures used to avoid scanning many buckets?
- What ε and number of parallel instances did you use in practice to approximate f(OPT)? How sensitive are results to these choices?
- Could you report wall-clock time and memory usage, and include an ablation over w?

Pros/cons for acceptance
Pros:
- Addresses an important problem with a novel and theoretically grounded streaming approach.
- Strong memory–approximation trade-off; simple and implementable design.
- Empirical evidence on real tasks showing competitive performance.

Cons:
- Presentation issues (typos/inconsistencies) reduce clarity.
- Lower constant factor than concurrent work; limited empirical evaluation of runtime/memory; missing comparison to [18].

Overall, this is a solid contribution to robust submodular streaming with a distinctive memory advantage. With corrections and added empirical details, it would be a valuable addition.

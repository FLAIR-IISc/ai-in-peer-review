This paper addresses the important problem of robust submodular maximization in a streaming setting. The goal is to design a single-pass algorithm that uses sublinear memory to select a summary set `S` from a large dataset. This summary must be "robust" in the sense that after an arbitrary removal of up to `m` elements, a high-quality solution of size `k` can still be extracted from the remaining elements of `S` without accessing the original data stream again. The authors propose a two-stage procedure. In the first stage, the STAR-T algorithm constructs the summary `S` by processing the data stream once. In the second, query stage, after a set `E` of `m` elements is removed, the standard greedy algorithm is run on `S \ E` to produce the final solution.

The core contribution is the STAR-T algorithm, which builds the summary `S` using a novel "partitioned thresholding" approach. The summary is organized into `log k` partitions, each containing multiple buckets. The partitions are associated with exponentially decreasing thresholds, meaning that elements with high marginal gain are captured in early partitions, while elements with lower gain are candidates for later partitions. This design is a clever combination of ideas from streaming submodular maximization (e.g., the thresholding in Sieve-Streaming by Badanidiyuru et al., 2014) and non-streaming robust submodular maximization (e.g., the partitioned structure in PRO by Bogunovic et al., 2017).

Theoretically, the authors prove that STAR-T produces a summary of size `O((k + m log k) log k)` and that the subsequent greedy selection provides a constant-factor approximation (approximately 0.149) to the optimal solution. The paper also handles the practical issue of the unknown optimal value (needed to set the initial threshold `τ`) by using a standard technique of running multiple instances in parallel. The work is also contextualized against a concurrent paper by Mirzasoleiman et al. (2017), claiming a more favorable memory dependence on `k`.

### Strengths

1.  **Significance and Novelty:** The paper tackles a highly relevant problem at the intersection of large-scale learning, streaming algorithms, and robust optimization. The proposed two-stage framework is practical for scenarios where re-computation is infeasible. The STAR-T algorithm's partitioned thresholding scheme is an elegant and novel method that effectively balances the need to capture high-value elements (for the objective) with the need for redundancy (for robustness).

2.  **Technical Quality (Theory):** The theoretical analysis is solid and provides the first constant-factor approximation for this problem setting with the stated memory bounds. The proof strategy, which analyzes the cases of "mostly full" vs. "mostly not-full" partitions, is insightful and handles the complex interactions between the algorithm's components well. The final result is a complete theoretical contribution.

3.  **Technical Quality (Experiments):** The empirical evaluation is strong and convincing. The choice of comparing against Sieve-Streaming, an established baseline that is given the significant advantage of knowing the removed set `E` in advance, is particularly effective. The fact that STAR-T performs competitively with, and sometimes outperforms, this oracle-equipped baseline strongly validates its practical utility. The experiments on both dominating set and personalized recommendation tasks showcase the algorithm's versatility, and the interactive recommendation scenario is a compelling use case for the proposed robust framework.

### Weaknesses and Suggestions for Improvement

1.  **Clarity on Memory Complexity:** There is an inconsistency in the stated memory complexity. The abstract and conclusion mention a size of `O((m log k + k) log^2 k)`, whereas Theorem 4.1 and its proof in the main body state `O((k + m log k) log k)`. The derivation in Equation (3) supports the latter. This should be clarified and made consistent throughout the manuscript.

2.  **Comparison to Concurrent Work:** The comparison with Mirzasoleiman et al. (2017) could be more precise. The paper claims its memory dependence on `k` is logarithmic while the concurrent work's is linear. However, the memory bound of STAR-T is `O((k + m log k) log k)`, which includes a `k log k` term that is super-linear in `k`. The claim of logarithmic dependence seems to apply only to the part of the bound involving `m` (`m log^2 k` vs. `mk log k`). This point should be rephrased for accuracy, perhaps by stating that for a fixed `k` and growing `m`, the dependence is more favorable.

3.  **Algorithm Details:** The thresholding condition in Algorithm 1 (line 5) is `f(e | B_{i,j}) >= τ / min{2^i, k}`. This is a fixed threshold for all elements considered for a given partition `i`. This is a departure from algorithms like Sieve-Streaming where the threshold for adding an element to a solution depends on the current solution's value. While this design choice is interesting and likely simplifies the analysis, a brief discussion on why this specific thresholding rule was chosen over alternatives would be beneficial for the reader's understanding.

4.  **Theory-Practice Gap:** The theoretical guarantee requires the memory parameter `w` to be at least `ceil(4 * dlog k * m / k)`. For the experiments, `w=1` was used, which performed very well even when the number of removals `|E|` was much larger than `k`. This is a positive practical result, but it highlights a gap between the requirements of the theory and the practical performance. A short discussion on this gap would strengthen the paper, as it suggests the analysis might be loose and the algorithm is even more practical than the theory implies. Similarly, the proven approximation constant of 0.149 is modest, while the empirical performance is close to the offline optimum. A comment on this would also be welcome.

### Summary

Overall, this is a high-quality paper that makes significant contributions to the field of submodular optimization. It introduces a novel and practical algorithm for a challenging problem, supports it with rigorous theoretical analysis, and validates it with strong experimental results. The weaknesses identified are primarily related to presentation and clarity, and can be easily addressed in a final version. The paper's combination of theoretical depth and practical relevance makes it an excellent fit for NIPS. I recommend acceptance.

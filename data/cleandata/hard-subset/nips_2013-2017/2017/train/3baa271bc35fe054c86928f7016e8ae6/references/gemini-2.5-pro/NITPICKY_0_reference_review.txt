### Review

**Summary**

This paper addresses the problem of robust submodular maximization in a streaming setting. The authors propose a two-stage procedure. In the first stage, a single-pass streaming algorithm called STAR-T constructs a robust summary set `S` by processing elements from a stream. This algorithm uses a novel structure of partitioned buckets with exponentially decreasing greedy thresholds, combining ideas from previous work like PRO [11] and SIEVE-STREAMING [8]. In the second stage, after an arbitrary set `E` of at most `m` elements is removed, a simple greedy algorithm (STAR-T-GREEDY) is run on the remaining summary `S \ E` to produce a solution of size at most `k`. The main theoretical contribution is a proof that this procedure yields a constant-factor approximation guarantee (approximately 0.149) while using `O((k + m log k) log k)` memory (as stated in Theorem 4.1). The authors validate their approach on dominating set and personalized movie recommendation tasks, showing performance comparable to or better than an oracle version of SIEVE-STREAMING that knows the removed elements in advance.

**Strengths**

1.  **Problem Significance:** The paper tackles a well-motivated and important problem. Combining the challenges of large-scale streaming data with the need for robustness against removals is highly relevant to many practical machine learning applications.
2.  **Novel Algorithmic Design:** The STAR-T algorithm introduces a clever synthesis of the partitioned structure from robust optimization [11] and the thresholding approach from streaming optimization [8]. The exponentially decreasing thresholds across partitions is an elegant idea for this combined setting.
3.  **Strong Theoretical Result:** The paper provides a constant-factor approximation guarantee, which is a solid theoretical contribution. A key strength of the analysis is that the memory complexity has only a logarithmic dependence on `k`, which compares favorably to the linear dependence in concurrent work [18].
4.  **Empirical Validation:** The experiments are well-designed to test the algorithm's robustness. Comparing against a strong baseline that has oracle access to the removed set `E` effectively highlights the utility of the pre-computed robust summary `S`. The interactive movie recommendation scenario is a particularly compelling use case.

**Weaknesses**

Despite the promising core ideas, the paper suffers from a significant number of inconsistencies, minor errors, and points of unclarity that detract from its overall quality.

1.  **Inconsistent Memory Complexity:** There are conflicting statements about the size of the summary set `S`.
    *   The abstract claims a size of `O((m log k + k) log^2 k)`.
    *   Theorem 4.1 claims a size of `O((k + m log k) log k)`.
    *   The derivation in Equation (3) leads to `O(wk log k)`. Substituting the required `w = O(m log k / k)` yields a size of `O(m log^2 k + k log k)`.
    These expressions are not equivalent, and this inconsistency regarding a central result of the paper is a major flaw that undermines the reader's confidence in the analysis.

2.  **Discrepancy in Algorithm Definition:** There is a mismatch between the textual description of the algorithm and the provided pseudocode. The text states that thresholds decrease exponentially as `τ/2^i`. However, Algorithm 1, line 5, defines the threshold as `τ/min{2^i, k}`. For partitions `i > log k`, this threshold becomes constant (`τ/k`) rather than exponentially decreasing. This is not a trivial detail, as it changes the behavior of the algorithm for a large number of partitions and is not addressed in the analysis.

3.  **Lack of Clarity in Theoretical Arguments:** The proof sketch for Theorem 4.1 in the main paper is difficult to follow. Critical steps, such as the derivations of Equation (4) and Equation (5), are justified only by vague statements like "by balancing out the two bounds." While full details may be in the supplement, the main text should provide a more self-contained and transparent argument for its central theorem.

4.  **Disconnect Between Theory and Practice:** The analysis requires the memory parameter `w` to be at least `ceil(4*dlog ke*m / k)`, which grows with the robustness parameter `m`. However, the experiments are conducted with `w=1`, even for scenarios where `m = 2k`. The paper notes that this "works well in practice" but provides no investigation or intuition as to why this significant gap between theoretical requirements and empirical settings exists. This suggests the theoretical analysis might be loose, but this possibility is not discussed.

5.  **Experimental Baseline:** While comparing against an oracle version of SIEVE-STREAMING is a valid stress test, a more natural and informative baseline is missing. One could run SIEVE-STREAMING on the original stream to generate a non-robust summary `S'`, and then evaluate `GREEDY(k, S' \ E)`. This would help isolate the benefit of the proposed robust summary structure versus a standard streaming summary.

6.  **Minor Presentation and Formatting Issues:** The paper exhibits a lack of polish that is unbecoming of a top-tier conference submission.
    *   A conference footer ("st Conference on Neural Information Processing Systems (NIPS 2017)...") is present in the manuscript.
    *   The notation `dlog ke` for `ceil(log_2 k)` is non-standard and clutters the mathematical expressions. Standard `\lceil \cdot \rceil` notation would be clearer.
    *   Pseudocode comments begin with an unconventional leading dot (e.g., `. loop over partitions`).
    *   Reference formatting is inconsistent, with stray commas and incomplete entries (e.g., [12], [22]).

**Arguments for Acceptance:**
*   Addresses a novel and important problem at the intersection of streaming algorithms and robust optimization.
*   The core algorithmic idea is sound and the theoretical result, particularly the memory complexity's dependence on `k`, represents a significant advance over concurrent work.
*   Empirical results are strong and demonstrate the practical utility of the proposed method.

**Arguments for Rejection:**
*   Critical inconsistencies in the statement of the main result (memory complexity) are present throughout the paper.
*   A subtle but important discrepancy exists between the described algorithm and the one in the pseudocode.
*   The theoretical analysis is not presented clearly in the main text, and a significant gap between theory and experimental settings is left unexplained.
*   The accumulation of minor formatting and presentation errors suggests a lack of careful preparation.

**Recommendation:**

I recommend **rejection**. While the paper presents a novel approach to an important problem and the high-level contributions are significant, the execution is flawed. The inconsistencies in the memory complexity—a key theoretical result—and the discrepancy in the algorithm's definition are major issues that prevent a confident assessment of the work. Combined with the numerous minor clarity and formatting problems, the paper does not meet the high standards of quality and precision expected for NIPS. I believe the paper could be a strong contribution after a thorough revision to correct these substantial flaws.

This paper studies the problem of robust submodular maximization in a streaming setting. The goal is to process a stream of elements from a ground set V to construct a small summary set S. This summary must be robust in the sense that after an adversary removes an arbitrary set E of m elements, one can still use the remaining elements S \ E to find a k-element solution Z whose value f(Z) is a good approximation of the optimal solution on the ground set without the removed elements, V \ E. This problem formulation combines challenges from streaming algorithms (single pass, limited memory), submodular optimization, and robust optimization.

The authors propose a two-stage procedure. In the first stage, the STAR-T algorithm makes a single pass over the data to build the summary S. STAR-T uses a novel partitioned structure where elements are assigned to buckets based on exponentially decreasing thresholds. In the second stage, after m elements are removed, the simple STAR-T-GREEDY algorithm runs on the remaining elements of the summary to find the final solution. The main theoretical contribution is a proof that this procedure achieves a constant-factor approximation (approximately 0.149) while requiring a summary of size O((m log k + k) log^2 k). This work builds on ideas from streaming submodular maximization (e.g., the thresholding of SIEVE-STREAMING [8]) and non-streaming robust submodular maximization (e.g., the partitioned structure of Bogunovic et al. [11]). The problem was also studied concurrently by Mirzasoleiman et al. [18], whose algorithm has a different memory-approximation trade-off.

**Strengths:**

*   **Technical Quality:** The paper is technically strong. The proposed STAR-T algorithm is novel, combining a partitioned structure with an exponentially decreasing thresholding scheme in a non-trivial way. The analysis is intricate and appears correct. The proof sketch, which involves balancing bounds from three different lemmas corresponding to different scenarios, highlights the technical depth of the contribution. This is a complete piece of theoretical work with a solid main result.
*   **Originality and Significance:** The paper addresses a new and theoretically interesting problem at the intersection of three important areas. The main result—a constant-factor approximation algorithm for robust submodular maximization in the streaming model—is a significant theoretical advance. The memory bound of O((m log k + k) log^2 k) is particularly noteworthy for its logarithmic dependence on k, which compares favorably to the linear dependence in concurrent work [18]. This could be an important building block for future research on robust and streaming optimization.

**Weaknesses and Suggestions for Improvement:**

*   **Motivation and Applications:** While the problem is theoretically compelling, the motivation and practical applications feel weak and could be substantially improved. The examples given (dominating sets, movie recommendations) are standard for submodular optimization but do not strongly motivate the specific robust streaming formulation. For instance, in the movie recommendation scenario, a user having already seen certain movies is a known constraint, not an adversarial removal that happens *after* the summary is built. A more natural approach would be to filter out known movies from the stream itself. The paper would be much stronger if it presented a more convincing use case where a summary is pre-computed in a stream and must then be robust to *unforeseen* future removals.
*   **Clarity of Problem Formulation:** The problem formulation is not general enough and contains a crucial ambiguity. The theory (Section 2 and Theorem 4.1) states that the adversary removes a set E ⊂ V, and the goal is to find a solution Z ⊂ S \ E. However, the experiments (Section 5.1) state that elements are removed "from the set S." This is a critical distinction. If the adversary can remove any m elements from the entire ground set V, the problem is much harder, and it is unclear how the summary S could guarantee a good solution if the optimal set OPT(k, V \ E) is disjoint from S. If the adversary is restricted to removing elements only from the algorithm's summary S, this is a more tractable but weaker model of robustness, which should be stated explicitly as the primary problem setting. This ambiguity undermines the interpretation of the main result and should be clarified throughout the paper.
*   **Relevance to Conference:** While submodular optimization is a core topic at NIPS, the paper's focus is highly theoretical. The connection to machine learning practice feels somewhat tenuous due to the weak motivation mentioned above. While the paper is in scope, its impact on the NIPS community would be greater if the authors could bridge the gap between their theoretical model and a concrete machine learning problem where this exact formulation is essential.
*   **Gap between Theory and Practice:** There is a large gap between the theoretical requirement for the memory parameter `w` (which scales with `m log k / k`) and its practical use (`w=1`). While the experiments show strong performance with `w=1`, the paper provides little intuition as to why this is the case or how the performance might degrade as the problem scales. A discussion on this discrepancy would be valuable.

**Summary of Arguments:**

*   **Pros:**
    *   Addresses a novel and challenging theoretical problem.
    *   Provides the first constant-factor approximation algorithm with a memory bound that depends logarithmically on k.
    *   The algorithm and analysis are technically deep and non-trivial.

*   **Cons:**
    *   The practical motivation for the specific problem formulation is weak.
    *   The adversarial model is ambiguous (removals from V or S?), creating a disconnect between the theory and experiments.
    *   The connection to applied machine learning could be strengthened to increase its impact.

In conclusion, this paper presents a strong, original theoretical contribution to the literature on submodular optimization. The technical quality is high. However, its significance is hampered by a weakly motivated problem formulation and a lack of clarity regarding the adversarial model. Clarifying these points and strengthening the connection to practical scenarios would significantly improve the paper.

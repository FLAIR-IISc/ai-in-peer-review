- Problem: Robust submodular maximization under a cardinality constraint in a one-pass streaming setting with up to m adversarial deletions after the stream.  
- Approach: Two-stage method — STAR-T streaming summary using partitioned buckets and exponentially decreasing thresholds; STAR-T-GREEDY runs GREEDY on the summary after deletions.  
- Main theorem: Constant-factor approximation c ≈ 0.149 as k → ∞.  
- Memory: O((k + m log k) log k) when τ is known; extra log factor when using parallel τ-grid for approximation (Theorem 4.5).  
- Analysis structure: Two-case proof — either some partition has ≥ half full buckets (Lemma 4.2) or all partitions are mostly not full (Lemmas 4.3–4.4), balanced via τ.  
- Experiments: Tasks on dominating set (coverage) and personalized movie recommendation; STAR-T-GREEDY matches or slightly outperforms SIEVE-STREAMING even when SIEVE-STREAMING knows E.  
- Relation to prior work: Builds on SIEVE-STREAMING thresholding and partitioned constructions from PRO/Orlin et al.; adapts partition ideas to streaming.  
- Concurrent work: Independent algorithm [18] attains 1/2 − ε approximation but uses memory O(mk log k/ε), i.e., linear in k; STAR-T achieves polylog(k) dependence for the m term.  
- Theory quality: Partitioned thresholding design and two-case analysis are technically sound and original; memory bound derivation reasonable with explicit dependence on w.  
- Parameter assumptions: Method requires τ ≈ f(OPT(k, V\E)); parallel τ-grid is standard but increases memory/time by a log factor.  
- Empirics quality: Demonstrates robustness and competitiveness but experimental scope is limited and does not use stochastic diffusion (IC/LT) for influence maximization.  
- Clarity: Generally well-written with clear algorithms and proofs, but has minor inconsistencies to fix.  
- Inconsistency 1: Threshold description mismatch — text says τ/2^i while Algorithm 1 uses τ/min{2^i, k}; needs reconciliation.  
- Inconsistency 2: Memory statements conflict — abstract claims O((m log k + k) log^2 k) whereas Theorem 4.1 gives O((k + m log k) log k); Theorem 4.5 adds another log factor.  
- Practical guidance lacking: Need clearer directions for choosing w and τ (or τ-grid), and sensitivity analysis.  
- Originality: First streaming algorithm tailored to deletion-robust submodular maximization with polylog(k) dependence for the m term; analysis departs from non-streaming PRO.  
- Significance: Robust deletion handling in streaming is practically important; single-pass and memory/time footprint make it appealing at scale despite modest constant.  
- Strengths: Clean two-stage design, compact reusable summary, constant-factor theoretical guarantee under deletions, simple query phase (GREEDY), competitive empirical performance.  
- Weaknesses: Limited experiments (no IC/LT, few datasets); parameterization issues (w set to 1 in experiments violating theoretical requirements); per-element processing in worst case checks O(w k log k) buckets and wall-clock timings are missing; modest approximation constant.  
- Acceptance pro: Novel, well-analyzed streaming algorithm with attractive memory-approximation trade-offs and reusable summaries.  
- Acceptance con: Insufficient empirical validation, weak practical guidance on parameters/runtime, and lack of empirical comparison to concurrent [18].  
- Suggestions: Add IC/LT influence experiments and larger datasets, compare empirically to [18], provide ablations/sensitivity plots for w and τ, report per-element runtime and memory, reconcile threshold and memory statements, and give practical defaults for w/τ.  
- Overall recommendation: Solid theoretical advance; lean toward acceptance if empirical scope and clarity issues are addressed.

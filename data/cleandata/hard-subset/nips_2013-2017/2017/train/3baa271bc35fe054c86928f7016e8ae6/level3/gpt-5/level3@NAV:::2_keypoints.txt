- Proposes a two-stage streaming algorithm (STAR-T summary + GREEDY on surviving items) for robust submodular maximization under a cardinality constraint with up to m deletions.  
- Theoretical guarantee: constant-factor approximation ≈0.149 as k→∞ with space O((k + m log k) log k) (Theorem 4.1).  
- Analyses split into two structural regimes (many full buckets vs. none) via Lemmas 4.2–4.4 to argue deletion robustness.  
- Parallel instantiation (Theorem 4.5) removes dependence on OPT using a geometric grid over τ, adding an extra log factor in space.  
- Empirical evaluation on dominating-set/influence-style selection and a personalized movie recommendation objective shows competitive performance with Sieve-Streaming and GREEDY variants.  
- Extends streaming thresholding (Sieve-Streaming) to the robust setting using non-uniform partitioning similar to offline robust methods (e.g., PRO); concurrent with deletion-robust streaming work but improves memory dependence on k.  
- Claims improved memory (logarithmic in k) over concurrent work at the cost of a smaller approximation constant.  
- Technical contribution is solid with clear case analyses and principled use of GREEDY guarantees.  
- Inconsistency in reported space bounds: abstract/conclusion state O((k + m log k) log^2 k) vs. Theorem 4.1’s O((k + m log k) log k); Theorem 4.5 adds another log factor — needs reconciliation.  
- Pseudocode vs. text mismatch: threshold condition uses τ/min{2^i,k} in code but text describes τ/2^i — reconcile notation/behavior.  
- Missing reporting of per-element processing time (expected O(w log k)?) and constant factors — should be reported.  
- Need more details on estimating τ and sensitivity analyses for τ and parameter w (beyond w=1).  
- Generally well-written and organized; partition/threshold mechanism is understandable but the above clarifications would improve reproducibility.  
- Originality: high — likely the first streaming algorithm with provable deletion robustness using memory logarithmic in k.  
- Significance: potentially broad impact for large-scale deletion-robust summarization despite modest approximation constant.  
- Pros for acceptance: novel formulation and algorithm, clean analysis, memory-log(k), modular two-stage design, handles unknown OPT via parallel thresholds.  
- Cons against acceptance: empirical setup limited (influence objective differs from classical IC/LT diffusion), STAR-T-GREEDY not consistently better than Sieve-Streaming, sparse ablations on τ,w and robustness vs. m, and minor inconsistencies in bounds/specification and limited runtime/memory reporting.  
- Suggested experiments and revisions: evaluate on IC/LT with RIS/CELF++ on smaller graphs, add ablations for w and m, report throughput/memory and constant factors, compare to the concurrent deletion-robust streaming baseline, and release code and parameter grids.

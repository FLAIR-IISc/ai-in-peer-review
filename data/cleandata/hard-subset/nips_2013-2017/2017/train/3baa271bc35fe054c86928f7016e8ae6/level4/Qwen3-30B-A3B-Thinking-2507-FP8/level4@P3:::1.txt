

This paper addresses robust streaming submodular maximization, where the goal is to compute a compact summary set *S* during a single data pass such that, after any *m* elements are removed from *S*, a greedy algorithm on the remaining elements achieves a constant-factor approximation to the optimal solution for the reduced set. The authors propose STAR-T, a streaming algorithm using a partitioned thresholding structure with exponentially decreasing thresholds, achieving a summary size of *O*((*k* + *m* log *k*) log² *k*) and a theoretical guarantee of *f*(Z) ≥ 0.149(1 − 1/log *k*)*f*(OPT(*k*, *V* \ *E*)). The approach extends SIEVE-STREAMING [8] to the robust setting, with experiments showing competitive performance against SIEVE-STREAMING (which knows *E* in advance) on dominating set and movie recommendation tasks.

**Strengths:**  
The technical contribution is significant. The partitioned thresholding mechanism—where buckets in partition *i* use thresholds *τ*/2ⁱ—is novel and enables the robustness guarantee. The analysis (Theorems 4.1, 4.5) is rigorous, handling both cases where partitions have many full buckets or not, and correctly addresses the challenge of adversarial *m*-element removals. The algorithm design avoids the *O*(mk log *k*) memory of concurrent work [18] by achieving *O*(m log *k*) dependence on *m*, a clear advantage. The experiments are well-executed: STAR-T-GREEDY matches or exceeds SIEVE-STREAMING’s performance (which knows *E* in advance) across both applications, even when *m* is large (e.g., *m* = *k* in movie recommendations). The interactive use case (reusing *S* for multiple *E* sets) is a compelling practical demonstration.

**Weaknesses:**  
The primary concern is **significance**. While the problem is well-motivated in theory (robustness to adversarial element removals), the paper fails to convincingly demonstrate *why this specific robustness setting is critical for real-world streaming applications*. The dominating set example (e.g., "influential users in social networks") does not clarify *why* removals would occur *after* the summary is computed—e.g., if the user base changes dynamically, standard re-streaming could be used instead of a precomputed robust summary. Similarly, the movie recommendation example (removing "already-watched" movies) is plausible, but the experiments only test *m* = *k* (e.g., 59% removal in genre filtering), while the theory requires *m* = *o*(*k*) for the guarantee. This disconnect weakens the practical relevance. The approximation constant (0.149) is also notably lower than the standard greedy’s (1 − 1/*e* ≈ 0.632), and the paper does not discuss whether this trade-off is justified by the robustness benefit.

**Originality:**  
The partitioned thresholding approach is distinct from prior robust submodular work. While [11] uses a similar partition structure for centralized robust maximization, STAR-T innovates by adapting it to streaming via exponential thresholds and a novel analysis. The connection to [18] is acknowledged, but the paper could better clarify that [18]’s *O*(mk log *k*) memory (linear in *k*) is less efficient than STAR-T’s *O*(m log *k*) for large *k*, a key advantage. This distinction strengthens originality.

**Clarity:**  
The paper is well-organized and clearly written. The algorithm descriptions (Algorithms 1–2) and figures (e.g., Fig. 1) make the partitioning intuitive. The theoretical proofs are logically structured, though the proof sketch of Theorem 4.1 could be more concise for readers. Minor issues: the term "robust summary" is used without explicit definition in Section 3, and the memory parameter *w*’s role in practice (e.g., *w* = 1 working well numerically) could be emphasized earlier.

**Recommendation:**  
**Weak Accept (7/10).** The technical contribution is solid, the analysis is sound, and the experiments validate the approach. However, the significance argument is underdeveloped. To strengthen the paper, the authors should:  
1. Better justify *why* robustness *after* streaming is necessary (e.g., in real-time systems where reprocessing the entire stream is infeasible).  
2. Discuss the trade-off between the 0.149 approximation constant and robustness more explicitly, especially for cases where *m* is small.  
3. Clarify the *m* = *o*(*k*) theoretical constraint versus the *m* = *k* experiments.  

While the work advances the state of the art in robust streaming submodular optimization, the lack of compelling motivation for the *specific* robustness model limits its broader impact. The paper is technically strong but does not fully convince that this model addresses a *critical* bottleneck in large-scale applications. For NIPS, where significance is paramount, this is a notable shortcoming. With the suggested revisions, it would be a strong candidate. As-is, it is borderline but acceptable due to its technical novelty and experimental validation.

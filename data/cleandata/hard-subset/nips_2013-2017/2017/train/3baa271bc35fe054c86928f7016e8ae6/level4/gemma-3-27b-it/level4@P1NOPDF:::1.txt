The paper studies a robust submodular maximization problem in a streaming setting. Specifically, the goal is to identify a set of elements *T* from a ground set *E* such that, even after removing any *m* elements from *T*, a subset *S* of *k* remaining elements can still be found to maximize a submodular function *f*.

Applications of submodular maximization are numerous, and streaming algorithms for the standard problem are well-established. This paper addresses a *robust* version of the problem, which is motivated by scenarios where solutions need to be resilient to element removal – for example, recommending a set of movies to a user and subsequently needing to adjust the recommendations based on user disinterest. 

The authors present a constant-factor approximation algorithm for this robust maximization problem, returning a set *T* of size (*m*log(*k*)+*k*)log²(*k*).

To the best of my understanding, the paper’s results are correct, and the solution is non-trivial. Both the algorithm and its analysis are technically challenging. However, my primary concern is the relevance of this work to the NIPS conference. While the problem itself is interesting, the provided applications feel somewhat weak and the problem’s generality appears limited. The motivation doesn’t strongly justify the complexity of the solution in a broader context.

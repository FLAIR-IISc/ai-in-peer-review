This paper addresses the problem of robust submodular maximization in a streaming setting. The goal is to select a summary set T from a ground set E, such that after an adversary deletes any m elements from T, one can still find a k-element subset S within the remainder that approximates the maximum possible value of a given monotone submodular function f(S).

The authors motivate this robust formulation with applications where an initial solution may be partially invalidated, such as a recommendation system where a user might discard some of the suggested items. The main contribution is a streaming algorithm that achieves a constant-factor approximation for this problem, returning a summary set T of size O((m log k + k) log^2 k).

The paper's technical contributions appear sound; the proposed algorithm is non-trivial, and its analysis is technically challenging. However, my main reservation concerns the paper's relevance and potential impact on the conference's audience. The motivation for this specific problem formulation feels underdeveloped, and the example applications are not strongly compelling. While theoretically interesting, the problem may be too specific to attract broad interest. A stronger justification for why this particular model of robustness is crucial and widely applicable would significantly improve the paper.

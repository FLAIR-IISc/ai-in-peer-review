

The paper addresses the problem of identifying a set \(T \subseteq E\) (from a ground set \(E\)) in a streaming setting such that, after removing any \(m\) elements from \(T\), the remaining set still contains a subset \(S\) of size \(k\) maximizing a submodular function \(f\). This robustness requirement ensures resilience against element deletions post-identification.

The authors present a constant-factor approximation algorithm that maintains a set \(T\) of size \((m \log k + k) \log^2 k\). The solution is technically sophisticated, with a nontrivial algorithm and a challenging analysis. To the best of my understanding, the results are correct and the technical contributions are sound.

However, the paper’s relevance to NIPS is questionable. While submodular maximization has well-established applications (e.g., feature selection, summarization), the specific robustness model lacks compelling motivation. The provided application example—suggesting a movie recommendation system where users later disinterest in certain items—is superficial and fails to demonstrate a concrete, high-impact scenario where this precise robustness constraint arises. More critically, the problem formulation is narrowly defined: it focuses on *deletion resilience* for a fixed \(m\) and \(k\), without addressing broader streaming challenges (e.g., adversarial element arrivals, dynamic updates, or real-world constraints like budget limits or privacy). This restricts its applicability compared to existing streaming submodular algorithms, which handle more flexible and widely encountered settings. The paper does not convincingly argue why this particular robustness variant is more significant or necessary than other established problem formulations in the literature. Consequently, the contribution appears incremental rather than transformative for the streaming submodular optimization community.

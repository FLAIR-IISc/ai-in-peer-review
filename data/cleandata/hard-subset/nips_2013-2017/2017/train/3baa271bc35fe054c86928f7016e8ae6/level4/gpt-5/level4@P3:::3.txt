Summary and relation to prior work
This paper studies monotone submodular maximization under a cardinality constraint in a two-stage robust streaming setting: during a single pass the algorithm retains a small summary S, and after the stream up to m items may be removed before selecting a final size-k set. The authors propose STAR-T (Streaming Robust submodular algorithm with Partitioned Thresholding), which partitions memory into exponentially sized buckets across partitions and uses exponentially decreasing thresholds to accept elements. After the stream, STAR-T-GREEDY runs the standard GREEDY algorithm on S\E to produce the final set. With a suitable estimate of OPT, STAR-T retains O((k + m log k) log k) elements and achieves a constant-factor approximation (c ≈ 0.149 as k grows), robust to any removal of up to m elements. When OPT is unknown, a multi-instance variant (as in SIEVE-STREAMING) yields a (1+ε)-approximate threshold estimate with an extra log factor in memory.

The work builds on (i) streaming submodular maximization (e.g., SIEVE-STREAMING by Badanidiyuru et al., 2014), extending thresholding to a robust two-stage setting; and (ii) robust non-streaming methods (e.g., Orlin–Schulz–Udwani, 2016; PRO by Bogunovic et al., 2017), which also use bucketed/partitioned structures but fill them differently and are not streaming. The authors also discuss concurrent work (Mirzasoleiman et al., 2017) that achieves a 1/2−ε approximation but with memory O(mk log k/ε); STAR-T trades a smaller approximation constant for significantly better dependence on k (logarithmic vs linear) and overall memory O((k + m log k) log k).

Quality
- Theory: The analysis is technically nontrivial and appears sound. The main theorem gives memory and approximation guarantees, and the case analysis is outlined (full-bucket partitions vs none full). The multi-instance variant addresses unknown OPT. One clarity issue: the text describes a partition threshold τ/2^i, while Algorithm 1 uses τ/min{2^i,k} in line 5. Please reconcile and explain which scheme is intended and why; this also affects intuition for the last partition. Also, please clarify whether “log2 k” denotes log base 2 or (log k)^2, as the memory bound is variously written.
- Experiments: Results on dominating set (ego-Twitter, Amazon) and movie recommendation are encouraging. STAR-T-GREEDY matches or outperforms strong baselines that know E in advance. However, the “influence maximization” objective is a dominating-set proxy (neighborhood size) rather than standard diffusion models; while reasonable for scalability, it weakens claims about influence maximization per se. Reporting wall-clock time, number of function evaluations, and memory actually used versus theoretical bounds would strengthen the empirical case. Details on how τ (or the grid of τ values and ε) is chosen in practice would improve reproducibility.

Clarity
The paper is generally well organized, but several points could be clearer:
- Clearly state the threshold used in each partition and ensure consistency between the description and pseudocode.
- Summarize the main analytic invariant or insight early in Section 4 to justify the “innovative analysis” descriptor (e.g., how the partitioned thresholds control overlap with the adversarial removal across partitions).
- Unify logarithm notation and explicitly state whether log is base-2; disambiguate log2 k.
- Specify the exact parameters used in experiments: w, ε for the multi-instance scheme, and how τ was estimated; add sensitivity plots for w and m.
- Minor: define all symbols on first use (e.g., w in the abstract/contributions).

Originality
The paper appears to be the first to achieve constant-factor robust submodular maximization in a one-pass streaming model with memory polylogarithmic in k (and near-linear in m). The partitioned, exponentially decreasing thresholding scheme and its analysis are novel in the streaming robust context, and the comparison to concurrent work is fair and informative.

Significance
Robust summarization under streaming constraints is important in many large-scale applications. The theoretical guarantees are meaningful despite the modest constant, and the memory footprint is attractive. The empirical results suggest practicality, though broader tasks (beyond dominating set and a single recommendation dataset) and more realistic influence models would further increase impact.

Pros
- Addresses a timely and challenging problem: robust streaming submodular maximization.
- One-pass algorithm with small, explicit memory bound O((k + m log k) log k).
- Constant-factor approximation after arbitrary removal of up to m items.
- Practical two-stage procedure; summary reusable across queries/removals.
- Empirical performance competitive with strong baselines that know deletions.

Cons
- Approximation constant (≈0.149) is modest; discussion of practical tightness would help.
- Experimental “influence maximization” objective is a simplification; limited task diversity.
- Some presentation inconsistencies (threshold expression, log notation) and lack of detail on τ estimation, runtime, and sensitivity to w/m.

Suggestions for improvement
- Reconcile and justify the threshold rule (τ/2^i vs τ/min{2^i,k}) and its role in the proofs.
- Provide a brief, high-level roadmap of the key analytic ideas and how the partitioned structure and thresholds ensure robustness to deletions.
- Clarify logarithm notation and memory bounds consistently across abstract, main text, and theorem.
- Add runtime/memory/function-evaluation statistics and sensitivity analyses for w, m, and τ-grid size.
- Either temper claims about influence maximization or include experiments under standard diffusion models (even on smaller graphs), or provide rationale why the dominating set proxy is sufficient in streaming.

Recommendation
The paper makes a solid theoretical and algorithmic contribution to robust streaming submodular maximization with promising empirical evidence. Despite clarity and experimental-scope limitations, I lean toward acceptance, contingent on addressing the presentation issues and clarifying the experimental setup.

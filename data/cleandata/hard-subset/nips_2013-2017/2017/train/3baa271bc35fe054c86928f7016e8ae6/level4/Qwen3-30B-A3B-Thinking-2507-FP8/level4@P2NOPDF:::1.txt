

The paper investigates a robust variant of submodular maximization under streaming constraints. Specifically, it addresses the problem of designing a streaming algorithm that outputs a set \(T \subseteq E\) (from a ground set \(E\)) such that, after deleting any \(m\) elements from \(T\), the remaining set still contains a subset \(S \subseteq T\) of size \(k\) that maximizes a monotone submodular function \(f\). The goal is to minimize the size of \(T\) while guaranteeing this robustness property.  

The authors present an algorithm achieving a constant-factor approximation to the optimal robust solution. The output set \(T\) has size \(O((m \log k + k) \log^2 k)\), which is a significant theoretical contribution. The algorithm and its analysis are technically sophisticated, involving novel techniques for handling robustness in the streaming model. The correctness of the results appears sound upon review, and the non-triviality of the approach is well-justified.  

The application motivation, however, is underdeveloped. The paper notes that robustness is useful when "we might need to remove some elements from a solution" (e.g., movie recommendations where users later disinterest in certain items). This example lacks specificity and fails to demonstrate why the *specific* robustness model studied—where *any* \(m\) elements may be removed—is necessary in practice. Real-world scenarios often involve *adaptive* removals (e.g., user feedback over time) or *probabilistic* failures, not worst-case arbitrary deletions. The paper does not clarify how the proposed model aligns with common failure modes in applications like recommendation systems, sensor networks, or data summarization.  

More critically, the problem’s scope is too narrow to justify its prominence in a top-tier ML conference like NeurIPS. The robustness condition (ensuring a \(k\)-element solution survives *any* \(m\)-element deletion) is a highly constrained special case. It does not generalize to:  
1. **Adversarial robustness** (e.g., where deletions are chosen *after* seeing \(T\), not arbitrarily),  
2. **Probabilistic robustness** (e.g., surviving deletions with high probability),  
3. **Dynamic settings** (e.g., deletions occurring incrementally during streaming), or  
4. **Non-uniform deletion costs** (e.g., some elements being more critical to retain).  

In contrast, existing work on robust submodular maximization (e.g., for budget constraints, adversarial corruptions, or noisy observations) addresses broader, more relevant settings. The current problem’s focus on *worst-case* element removals without connection to a natural application context weakens its contribution. For instance, in recommendation systems, user disinterest is typically *adaptive* (not worst-case), and the utility of a recommendation set is often measured by *aggregate* user satisfaction—not whether *every possible* \(k\)-subset remains optimal after arbitrary deletions.  

The paper’s technical contribution is strong, but its relevance to NeurIPS is questionable. NeurIPS prioritizes work with clear connections to machine learning, optimization, or data science challenges. This work’s narrow problem formulation and weak application justification fail to meet that bar. The authors could strengthen the paper by:  
- Replacing the ad-hoc application example with a concrete ML scenario where *worst-case* robustness is essential (e.g., critical infrastructure monitoring where *any* sensor failure must not disrupt core functionality),  
- Comparing their model to established robustness frameworks in ML (e.g., adversarial robustness, stability under data perturbations), or  
- Demonstrating empirical gains on real-world datasets where their approach outperforms simpler heuristics.  

Without such enhancements, the paper’s novelty is limited to a technical extension of streaming submodular maximization, rather than a meaningful advancement for the ML community. While the algorithmic result is valid and non-trivial, the problem studied does not address a pressing need in practice or theory. The lack of a compelling motivation and the restricted scope make it difficult to justify inclusion in a venue like NeurIPS, where impact and relevance are paramount.  

In summary: The technical results are correct and well-analyzed, but the problem’s relevance to machine learning is insufficiently justified. The application context is superficial, and the problem formulation lacks generality compared to existing robust optimization literature. The contribution would be more compelling if it addressed a broader class of robustness scenarios with clear ML applications. As written, the paper’s significance to the NeurIPS community is limited.

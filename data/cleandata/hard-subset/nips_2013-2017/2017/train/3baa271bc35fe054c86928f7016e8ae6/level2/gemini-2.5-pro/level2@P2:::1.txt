This paper addresses the problem of maximizing a monotone submodular function subject to a cardinality constraint k, in a challenging setting that combines two practical constraints: the data arrives in a stream, and the solution must be robust to the removal of up to `m` elements. The authors propose a two-stage procedure. In the first stage, a single-pass streaming algorithm called STAR-T constructs a robust summary `S`. The key idea is a novel partitioned data structure where elements are added to buckets based on exponentially decreasing thresholds. In the second stage, after an arbitrary set `E` of `m` elements is removed from `S`, a simple greedy algorithm (STAR-T-GREEDY) is run on the remaining set `S \ E` to produce the final `k`-sized solution. The main contribution is a theoretical guarantee that this procedure achieves a constant-factor approximation (approximately 0.149) with a memory footprint of `O((k + m log k) log k)`. The work builds upon ideas from streaming submodular maximization (e.g., the thresholding of SIEVE-STREAMING) and non-streaming robust submodular maximization (e.g., the partitioned structure of PRO). The authors also provide a standard method for handling the unknown value of the optimum and validate their approach with strong empirical results on dominating set and movie recommendation tasks.

**Strengths:**

1.  **Significance and Timeliness:** The problem formulation is highly relevant. As machine learning models are deployed on massive, streaming datasets, ensuring their robustness against unforeseen data removal, corruption, or adversarial attacks is critical. This paper provides a principled approach for this important intersection of streaming algorithms and robustness, which is of broad interest to the NIPS community.

2.  **Novel Algorithmic Contribution:** The STAR-T algorithm is an elegant and novel synthesis of existing ideas. The combination of a partitioned structure with exponentially sized buckets (inspired by non-streaming robust methods) and an exponentially decreasing thresholding rule (inspired by streaming methods) is a clever design. This structure is what enables the algorithm to maintain a robust summary in a single pass.

3.  **Strong Theoretical Guarantees:** The paper provides a rigorous constant-factor approximation guarantee. The analysis, while complex, is well-sketched in the main paper and appears sound. A key strength of the result is the memory complexity, which has a logarithmic dependence on `k`. This is a significant advantage over concurrent work [18] and makes the algorithm scalable to problems with large `k`.

4.  **Compelling Empirical Validation:** The experiments are well-designed and convincingly demonstrate the effectiveness of the proposed method. The comparison against SIEVE-STREAMING, where the baseline is given the significant advantage of knowing the removed set `E` in advance, is particularly strong. That STAR-T can match or outperform this baseline highlights the practical utility of its robust summary. The interactive movie recommendation scenario is an excellent use case that clearly illustrates the practical benefits of the proposed two-stage approach.

**Weaknesses and Suggestions for Improvement:**

1.  **Clarity and Consistency:** While the paper is generally well-written, there are a few minor points that could be clarified:
    *   The memory complexity is stated as `O((m log k + k) log^2 k)` in the abstract but `O((k + m log k) log k)` in Theorem 4.1. The derivation in the proof sketch (`|S| <= (log k + 5)wk` with the theoretical requirement for `w`) seems to lead to `O(k log k + m log^2 k)`. Please unify these expressions and double-check the derivation for consistency.
    *   In Algorithm 1, the threshold condition is `f(e | B_i,j) >= τ / min{2^i, k}`, whereas the text often refers to it as `τ / 2^i`. These are different for larger `i`. This should be made consistent.
    *   The proof sketch for the main theorem involves balancing several inequalities. The jump to the final constant (0.149) is quite condensed. A bit more intuition or a few intermediate steps in the main text could improve readability for the reader.

2.  **Theory-Practice Gap:** The theoretical analysis requires the memory parameter `w` to be at least `O(m log k / k)`, while the experiments successfully use `w = 1`. This is a common and often acceptable gap, but the paper would be strengthened by a brief discussion of it. An experiment showing the performance of the algorithm for different values of `w`, especially as `m` increases, would provide valuable insight into the practical behavior of the algorithm and the tightness of the theoretical requirement.

3.  **Approximation Constant:** The proven approximation factor of 0.149, while a valuable constant-factor guarantee, is relatively small. It would be helpful if the authors could briefly comment on whether this is likely an artifact of the analysis or if there are inherent hardness-of-approximation reasons for such a constant in this robust streaming setting.

4.  **Comparison with Concurrent Work:** The paper appropriately cites the concurrent work of Mirzasoleiman et al. [18]. However, the comparison could be more detailed. The current work has better memory dependency on `k` (logarithmic vs. linear), while [18] has better dependency on `m` (linear vs. `m log^2 k`). A short discussion on the practical implications of this trade-off and in which problem regimes (`k >> m` vs. `m >> k`) each algorithm might be preferable would be a useful addition and would better contextualize the paper's contribution.

In summary, this is a high-quality paper that makes a significant contribution to the field of large-scale submodular optimization. It tackles a well-motivated and difficult problem with a novel algorithm, strong theoretical backing, and convincing experimental results. The weaknesses identified are minor and can be addressed in a revision. The paper is a solid contribution and should be accepted.

Summary
The paper studies clustering with access to a same-cluster oracle and side information in the form of a noisy similarity matrix W, where pairwise weights are drawn from unknown distributions f+ (within-cluster) and f− (across-cluster). The authors ask: how many adaptive pairwise queries to the oracle are necessary and sufficient to exactly recover clusters when k and the distributions are unknown? They provide:

- Information-theoretic lower bounds on the expected number of queries in terms of Hellinger divergence H2(f+‖f−), showing Ω(min{nk, k^2/H2}) queries are needed (and Ω(n + min{nk, k^2/H2}) for zero failure probability), even if the algorithm knows k, f+, f−.
- Parameter-free, polynomial-time algorithms that don’t require k, f+, or f−. A Monte Carlo algorithm achieves near-optimal query complexity (within log n), and a Las Vegas algorithm guarantees success with query complexity O(n log n + min{nk, (…)/H2}).
- A general lower-bound technique for interactive, adaptive querying that leverages Hellinger distance, going beyond standard Fano-type arguments.
- A conceptual generalization of stochastic block models (SBM) to arbitrary unknown f+, f− and adversarial partitions, with some discussion on how oracle queries shift SBM recovery thresholds.
- A small empirical study on the Cora entity-resolution dataset illustrating query-accuracy tradeoffs and robustness to approximate f+, f−.

Relation to prior work: The work connects to ER/crowdsourcing heuristics that use side information to guide queries, explaining why these can use far fewer than Ω(nk) queries. It significantly generalizes SBM-based recovery results by allowing arbitrary unknown f+, f− and adversarial cluster sizes, and differs from prior interactive clustering work that either assumes specific metric/separation conditions or studies different query models. The lower-bound technique for adaptive queries and the parameter-free algorithmic framework appear novel.

Quality
- Theory: The lower bound is carefully constructed for adaptive queries using a “weak clusters” argument and Hellinger distances for product distributions. The assumption H2(f+‖f−) ≤ 1/18 is clearly stated for one regime; the authors explain how other regimes behave. The upper bounds are supported by a detailed algorithmic design (membership via empirical Hellinger, iterative threshold estimation using Sanov-style concentration) with proofs deferred to the appendix.
- Correctness/consistency: There is a potentially serious ambiguity in the big-O expressions. In several places the Monte Carlo query bound is written as O(k log n / H2), while the lower bound is Ω(k^2/H2). The Las Vegas bound is stated as O(n log n + min{nk, k^2 log n / H2}), which matches the lower bound within log n. If the Monte Carlo bound is truly O(k log n / H2), it would contradict the lower bound up to a factor k; most likely it is a typesetting/parentheses issue and should read O(k^2 log n / H2). Please fix all occurrences and unify notation for H vs H2.
- Algorithms: The parameter-free iterative update that estimates p+ and p− on the fly and adapts thresholds until a “grown” cluster appears is interesting and plausible. However, details that influence worst-case query complexity in Phase 1–2 (e.g., when cluster sizes are highly unbalanced/adversarial) should be made more explicit in the main text (not only in the appendix).
- Experiments: The empirical section is limited: one dataset, no synthetic sweeps over H2, n, k, or cluster size imbalance, and no baselines beyond best/worst/greedy query ordering. The results are suggestive but insufficient to stress-test the theoretical scalings or to compare against prominent heuristics [Awasthi et al., Wang et al., Firmani et al., Vesdapunt et al.]. This is not fatal given the strong theory, but expanding experiments would strengthen the paper.

Clarity
- Generally well organized with a clear problem statement, motivation from ER, and positioning relative to SBM. However:
  - Fix the repeated ambiguity in big-O bounds involving H2 and log n (add parentheses and explicit division bars).
  - Unify notation for Hellinger (H vs H2), and define clearly at first use. When switching between H and H2 in inequalities, be consistent.
  - In Theorem 1, specify the support assumptions for f+, f− for the algorithm (finite discrete support) and discuss how to handle continuous or larger supports (e.g., discretization).
  - Some references are incomplete or have formatting issues; please clean up.
  - Provide a compact pseudocode or flow diagram for the algorithm; spelling out Phases 1–3 concisely would aid reproducibility.

Originality
- The interactive lower bound framework for adaptive queries with side information using Hellinger distances is novel and valuable.
- A parameter-free algorithm that matches information-theoretic limits (up to log n) for arbitrary unknown f+, f− and unknown k is a substantial contribution.
- The connection to shifting SBM thresholds with queries is insightful; the conjecture for the sparse regime is interesting.

Significance
- The results provide a principled explanation for the empirical success of ER/crowdsourcing heuristics using side information, quantify gains vs. no-side-info settings, and give algorithms with near-optimal query complexity.
- The lower-bound technique may inform future work in interactive hypothesis testing, active learning with side information, and multi-round communication complexity.
- Impact is primarily theoretical; practical uptake would be helped by stronger experimental validation and comparisons.

Pros
- Strong, general information-theoretic lower bounds for adaptive querying.
- Parameter-free, computationally efficient algorithms with guarantees close to the lower bound.
- Clear conceptual advance beyond SBM and prior interactive clustering models.
- Relevant motivation and potential impact on ER/crowdsourcing.

Cons
- Ambiguity/inconsistency in the stated query complexities (Monte Carlo bound) and notational sloppiness.
- Algorithmic proofs and some crucial details are deferred; main text could better explain convergence and query budgets in worst-case cluster size distributions.
- Limited empirical evaluation; no comparisons to established baselines; no synthetic stress tests varying H2, k, imbalance, noise models.
- Algorithm currently assumes discrete finite support for f+, f−; discuss extensions.

Questions and suggestions for the authors
- Clarify all big-O expressions: is the Monte Carlo query complexity O(k^2 log n / H2) or O(k log n / H2)? Ensure consistency with Theorem 2.
- In Phase 1–2, bound the number of queries needed to obtain a grown cluster under adversarial cluster sizes (e.g., many tiny clusters), and state this bound in the main text.
- Discuss how to handle continuous-valued side information (e.g., via binning) and how discretization affects constants and H2 estimates.
- Add experiments: synthetic datasets sweeping H2(f+‖f−), k, and cluster imbalance; comparisons to representative heuristics [40, 43, 18]; report exact-recovery rates and query counts with confidence intervals.
- Improve notation and references; include a concise pseudocode.

Recommendation: Accept (leaning). The theoretical contributions are strong and broadly relevant; addressing clarity issues and expanding experiments would make the paper even stronger.

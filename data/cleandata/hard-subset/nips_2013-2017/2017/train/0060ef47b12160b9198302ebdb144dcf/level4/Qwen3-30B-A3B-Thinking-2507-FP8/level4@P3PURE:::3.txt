

This paper proposes an image saliency mask generation approach capable of processing 100 images of size 224×224 per second on a standard GPU. The method trains a masking model to identify the tightest rectangular crop containing the entire salient region for a requested class label, using a black-box classifier (e.g., AlexNet, GoogleNet, ResNet). The model architecture leverages feature maps from multi-scale CNNs (e.g., ResNet-50), with the final scale feature passed through a feature filter for initial localization and subsequent upsampling blocks for mask refinement. Experiments demonstrate that the approach outperforms other weakly supervised techniques on the ImageNet localization task.  

The paper provides sufficient references and related work. The technical correctness appears sound. The authors present several intuitive design discussions and the presentation is clear and well-structured.  

Overall, the paper delivers interesting technical results, though I have reservations regarding the real-time speed claim and applicability to real-world scenarios.  

**Comments:**  
- Does the 100 images/second processing time include image resizing? If so, what is the runtime for larger images (e.g., 640×480 iPhone 6s captures)?  
- The salient objects in the experiments are relatively large. How does the method handle cases where the requested class object is small within the image? Is 224×224 sufficient resolution in such scenarios?  
- In Table 3, are direct metrics provided for works in Table 2 (e.g., Feed [2])?

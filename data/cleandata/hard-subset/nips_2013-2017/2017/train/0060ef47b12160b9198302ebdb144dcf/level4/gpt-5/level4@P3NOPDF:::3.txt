Summary:
The paper proposes a fast image saliency mask generation method that, given a target class for an image and access to a black-box classifier (e.g., AlexNet, GoogLeNet, ResNet), predicts a tight rectangular crop enclosing the salient region for that class. The model consumes multi-scale feature maps extracted from a backbone (e.g., ResNet-50). A feature filtering module performs initial localization at the coarsest scale, and a sequence of upsampling blocks refines the mask to full resolution. The method reportedly processes over 100 images per second at 224×224 on a standard GPU and outperforms prior weakly supervised approaches on the ImageNet localization task.

Strengths:
- Clear problem setup and motivation: class-specific localization with only black-box classifier access.
- Elegant, modular architecture that leverages multi-scale features and upsampling refinement.
- Strong empirical results on ImageNet localization, surpassing other weakly supervised techniques.
- Good presentation and writing; references and related work appear adequately covered.
- Practical focus on speed and black-box compatibility.

Weaknesses and concerns:
- The real-time claim (100 images/s at 224×224) lacks clarity on what is included (preprocessing, feature extraction, postprocessing). The practical throughput for higher-resolution inputs is not reported.
- Many examples seem to involve large, prominent objects. It is unclear how robust the method is for small or thin objects, or when the target object occupies a small fraction of the image. The fixed 224×224 input may limit small-object localization accuracy.
- The reliance on a specific backbone (e.g., ResNet-50) is reasonable, but ablations on feature extractor choice and scale configurations would strengthen the claims about generality and efficiency.
- While the paper appears technically sound, some details (e.g., training dynamics, failure cases, sensitivity to the black-box classifier’s errors) could be elaborated. I did not fully verify all proofs or derivations.

Questions for the authors:
- Throughput: Does the reported >100 images/s include image resizing, feature extraction, mask refinement, and non-maximum suppression (if any)? Please clarify whether this is end-to-end time measured from raw images to final boxes/masks. Also report throughput on common larger inputs (e.g., 640×480 smartphone images), and characterize how runtime scales with input resolution.
- Small objects: How does performance vary with object size relative to image size? Are there results stratified by object scale or by the fraction of pixels occupied by the target class? Is 224×224 sufficient for small objects, or do you recommend higher input resolutions and, if so, how does that affect speed?
- Metrics consistency: In Table 3, can you report the corresponding metrics for all baselines shown in Table 2 (e.g., Feed [2]) to enable more direct comparison? If different thresholds or criteria are used, please harmonize or provide a conversion/explanation.

Suggestions:
- Provide a detailed runtime breakdown (preprocessing, backbone feature extraction, feature filtering, upsampling/refinement) and report end-to-end latency on a standardized hardware setup. Include a throughput vs. resolution curve and discuss memory usage.
- Include ablations on:
  - Backbone choices (e.g., ResNet-18/50/101) and their speed/accuracy trade-offs.
  - Number of scales and placement of the feature filtering module.
  - Effect of input resolution on small-object performance.
- Add qualitative analysis of failure cases, especially for small/occluded objects and cluttered scenes, and discuss the impact of misclassifications by the black-box classifier.
- If feasible, report results on additional datasets or settings with more varied object scales to demonstrate robustness beyond ImageNet’s typical object size distribution.

Minor comments and typos:
- Ln 273: “More over, because our model” → “Moreover, because our model”
- Ln 151: “difference that [3] only optimise the mask” → “difference that [3] only optimises the mask”
- Consider standardizing capitalization (e.g., AlexNet, GoogLeNet, ResNet) and ensure consistent spelling (optimize/optimise) per venue style.

Overall assessment:
This is a well-written paper with a clear idea, a sensible architecture, and compelling results for weakly supervised localization. I am positive about the contribution, with the main reservations around the end-to-end speed claim and the method’s efficacy on small objects and higher-resolution, real-world images. Clarifying the runtime accounting, adding scale-sensitive evaluations, and ensuring metric consistency across baselines would significantly strengthen the paper.

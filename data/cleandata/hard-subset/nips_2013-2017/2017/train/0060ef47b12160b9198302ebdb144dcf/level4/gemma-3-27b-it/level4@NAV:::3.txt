This paper introduces a saliency mask generation technique capable of processing 100 224x224 images per second on standard GPU hardware. The proposed method learns a masking model designed to identify the minimal bounding box encompassing the salient region of a specified class, leveraging a black-box classifier (e.g., AlexNet, GoogleNet, ResNet). The model architecture utilizes multi-scale image feature maps (such as those derived from ResNet-50), employing an initial localization step via a feature filter followed by upsampling blocks for refined mask generation. Experimental results demonstrate superior performance compared to other weakly-supervised methods on the ImageNet localization task.

The paper includes a sufficient number of relevant references and related works. A complete verification of these was not performed. 

The technical aspects of the paper appear sound. A complete verification was not performed. 

The authors provide insightful rationale and discussion regarding their design choices.

The presentation of this paper is well-executed.

Overall, the paper presents compelling technical contributions. However, I have some reservations concerning the reported real-time performance and the applicability of this approach to real-world images.

Comments:
- The reported processing speed of 100 images per second – does this figure include the time required for image resizing? If so, what is the processing time for larger images, for example, 640x480 images captured by an iPhone 6s?
- The current experiments focus on relatively large salient objects. How would the method perform when the target class object is small within the image? Would a 224x224 input image size be sufficient in such cases?
- Regarding Table 3, are comparable metrics available for other methods presented in Table 2, specifically Feed [2]?



This paper proposes an efficient image saliency mask generation approach capable of processing 100 images of size 224×224 per second on a standard GPU. The method trains a masking model to identify the minimal bounding box containing the entire salient region for a specified class, as determined by a black-box classifier (e.g., AlexNet, GoogleNet, ResNet). The architecture leverages multi-scale image feature maps (e.g., from ResNet-50), where final-scale features pass through a feature filter for initial localization, followed by upsampling blocks for mask refinement. Experiments demonstrate superior performance over existing weakly supervised methods on the ImageNet localization task.  

The paper provides adequate coverage of related work and references. Technical correctness appears sound based on the presented methodology and results. The authors offer insightful discussions on design choices and intuitions underlying their approach. The presentation is clear and well-organized.  

However, I have reservations regarding the real-time speed claim and practical applicability. Specifically:  
1.  **Processing Time Clarification:** The stated speed of 100 images/second (224×224) likely includes image resizing. Please clarify whether this includes preprocessing and provide runtime metrics for larger input resolutions (e.g., 640×480, typical of smartphone images like the iPhone 6s).  
2.  **Small Object Handling:** The salient objects in the dataset appear relatively large. How does the method perform when the requested class object is small within the image? Is the 224×224 input size sufficient to preserve critical details for such cases?  
3.  **Table 3 Comparison:** Table 3 lists metrics for the proposed method but does not include corresponding results for other works (e.g., Feed [2]) shown in Table 2. Including these metrics would strengthen the comparison.  

The paper contains minor grammatical errors requiring correction:  
- Line 273: "More over" → "Moreover"  
- Line 151: "optimise" → "optimizes"  

Overall, the technical contributions are interesting and well-supported by experiments. While the core methodology is compelling, the practical relevance hinges on addressing the scalability to diverse image resolutions and small object detection. The real-time claim warrants explicit validation for non-standard input sizes. With clarification on these points, the paper would be significantly strengthened.

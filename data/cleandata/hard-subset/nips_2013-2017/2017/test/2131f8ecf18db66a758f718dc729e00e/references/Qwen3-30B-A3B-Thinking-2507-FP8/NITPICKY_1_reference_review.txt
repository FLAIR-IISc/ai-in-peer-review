

# Review of "Semisupervised Clustering, AND-Queries and Locally Encodable Source Coding"

## Summary

This paper proposes a novel framework for clustering via crowdsourcing by reinterpreting the problem as a locally encodable source coding problem. The authors introduce AND queries as a more efficient alternative to traditional "same cluster" (XOR) queries, particularly for approximate recovery. They provide theoretical guarantees for exact and approximate recovery in noiseless and noisy settings, along with a real crowdsourcing experiment validating their approach. The work connects clustering with information theory through the lens of locally encodable source coding, a notion previously mentioned but not thoroughly studied in [23].

## Strengths

1. The paper makes an interesting theoretical contribution by connecting clustering with locally encodable source coding, a concept that is indeed underexplored in information theory.
2. The proposed AND query scheme shows promising theoretical advantages over traditional "same cluster" queries for approximate recovery, as demonstrated in Theorems 3-5.
3. The real crowdsourcing experiment provides empirical validation of the theoretical claims, showing that AND queries yield fewer errors (105 vs. 234 for "same cluster" queries).
4. The paper addresses an important practical problem in crowdsourcing where query complexity and worker error rates are critical factors.

## Weaknesses

### Technical Flaws

1. **Theoretical Inconsistencies in Theorem 1**: The proof relies on an expurgation argument that requires the expected number of "bad events" to be less than T (size of typical set). However, the authors state "if this expected number of 'bad events' is smaller than T then for every 'bad event', we can throw out 1 label vector and there will be no more bad events." This is mathematically unsound. The standard expurgation argument requires the expected number of bad events to be less than 1, not T. The authors appear to be confusing the size of the typical set with the number of bad events.

2. **Misuse of Typical Set Definition**: In Theorem 1, the authors define a "typical set" as label vectors with number of ones between np - n^(2/3) and np + n^(2/3). This is not the standard definition of the typical set used in information theory (which would have size ~2^(nH(p))). This non-standard definition complicates the analysis and is not justified.

3. **Inconsistency in Query Definition**: The authors claim in Section 3.3 that their query for k=2 is "exactly the same as the binary AND query" from Section 3.2. However, the query in Section 3.3 is defined as Q(X,X') = i if X=X'=i, 0 otherwise. For k=2, this is not equivalent to the binary AND query (X ∧ X'). For example, when both labels are 0, the binary AND query returns 0 while the Section 3.3 query returns 0 (i=0), which is correct. But the description is confusing and misleading.

4. **Theoretical vs. Empirical Error Rate Confusion**: In Section 4, the authors report 105 erroneous answers out of 1000 queries (error rate 0.105) and then use q=0.105 in their theoretical analysis. However, q is a parameter of the model (probability of error per query), not the empirical error rate. This conflates the theoretical model with the empirical results, which is problematic.

5. **Incorrect Reference to [17]**: The authors cite Massey's work [17] as establishing limitations of linear codes for rate-distortion coding, but the reference is to "Joint source and channel coding" which is not about rate-distortion coding. This is a misattribution.

### Formatting and Presentation Issues

1. **Critical Typo**: In the abstract, "to to graph clustering" should be "to graph clustering."

2. **Inconsistent Notation**: The paper uses inconsistent spacing in mathematical expressions (e.g., "m = n(h(p)+o(1)) log 1α" should be "m = n(h(p)+o(1)) \log 1/\alpha").

3. **Misplaced Conference Name**: The line "st Conference on Neural Information Processing Systems (NIPS 2017), Long Beach, CA, USA." appears in the middle of Section 1, which is clearly a formatting error.

4. **Unjustified Assumption**: The paper states "independent repetition of queries is also theoretically not interesting, as by repeating any query just O(log n) times one can reduce the probability of error to near zero" but provides no citation for this standard result.

5. **Inconsistent Citation Style**: References are inconsistently formatted (some include page numbers, some don't).

### Significance Concerns

1. **Overstated Contributions**: The paper claims "the only place where it is mentioned to the best of our knowledge is in [23]" regarding locally encodable source coding. However, [23] does mention it (as "robust compression"), so this claim is inaccurate.

2. **Marginal Theoretical Improvement**: The theoretical improvements over existing work (e.g., [21]) are modest. For example, the O(n log(1/δ)) query complexity for approximate recovery with AND queries is not substantially better than existing O(n log n) results for "same cluster" queries when δ is small.

3. **Limited Practical Impact**: While the crowdsourcing experiment is a positive addition, the paper doesn't clearly demonstrate how much better the proposed method is in practice compared to existing approaches for real-world clustering problems.

## Recommendation

This paper presents an interesting theoretical framework that connects clustering with locally encodable source coding. While the core idea of using AND queries instead of "same cluster" queries for approximate recovery is novel and supported by some theoretical results and a small experiment, the paper suffers from multiple technical flaws that undermine its main claims. The theoretical inconsistencies in Theorem 1, the misdefinition of the typical set, and the confusion between theoretical and empirical error rates are serious issues that need to be addressed.

The formatting issues and inconsistent notation further detract from the paper's quality. While the paper makes a legitimate contribution to the theoretical understanding of clustering with crowdsourcing, the technical flaws are substantial enough to warrant rejection. The authors should thoroughly revise the theoretical arguments, correct the notation inconsistencies, and properly contextualize their work in relation to existing literature before resubmission.

I recommend rejection due to technical flaws that significantly weaken the paper's claims, despite the interesting core idea. The paper would be significantly improved with a more rigorous theoretical foundation and careful attention to presentation details.

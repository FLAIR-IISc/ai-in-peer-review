This is an interesting paper. I enjoyed reading it. In this paper, the authors consider the task of clustering data points using crowdsourcing, where the problem is retrieving the original labels of the data points based on the answers to similar cluster searches. The authors propose a limit on the number of searches required to retrieve the labels, where the search answers can be both noisy and non-noisy. The main contribution of this paper is theoretical, as well as some empirical modeling. The paper is well-written, which makes it easy for general machine learning audiences to understand the theoretical contributions. In particular, its connection to information theory for deriving the limits is beautifully explained.

I have the following concerns for improving this paper:

(1) Although the concept of AND searches is theoretically interesting, it seems to be different from the main purpose of crowdsourcing. In the case of AND searches, effectively, the cluster label is also given as part of the answer, instead of just answering whether the data points belong to the same cluster. If this is really the case, as I understood from the paper, then why would that similarity search be needed? Why not ask for the data point label itself? This is my main confusion about the contribution of this paper.

(2) Is it possible to simplify the expressions for the limits of theorems 1, 2, 3, 4 while keeping the main content intact?

(3) In the experiment, it seems only AND searches have been considered. If so, the experimental study is questionable until point number 1 above is clarified.

I am satisfied with the authors' answers to the above questions. Therefore, I vote to accept this paper.

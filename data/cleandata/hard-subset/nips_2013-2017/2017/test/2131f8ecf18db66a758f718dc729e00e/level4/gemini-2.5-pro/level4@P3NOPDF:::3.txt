This paper addresses the problem of clustering data points via crowdsourcing, focusing on recovering the true cluster labels from answers to "same-cluster" queries. The authors' primary contribution is theoretical, establishing information-theoretic lower bounds on the number of queries required for exact label recovery. The analysis covers both noiseless and noisy settings and is supported by empirical simulations.

The paper is well-written and organized, making its theoretical contributions accessible to a general machine learning audience. A particular strength is the clear and intuitive explanation of the connection to information theory used to derive the performance bounds. The query models considered are novel and present an interesting theoretical perspective on the problem.

Despite these strengths, I have a few concerns that I would like the authors to address to improve the paper's impact and clarity.

1.  **Motivation of the "AND" Query Model:** My primary concern relates to the practical motivation for the "AND" query model. As I understand it, an AND query for points {i, j, k} reveals not only whether all three belong to the same cluster but also provides the specific label of that cluster if they do. This seems to contradict the typical crowdsourcing scenario for clustering, where workers can assess similarity but may not know the global cluster labels. If a crowdworker can provide the cluster label as part of the answer, it raises the question: why not use a simpler and more direct "label" query (i.e., "what is the cluster label for point i?")? A direct labeling query appears to be a more efficient way to acquire the same information. A clearer explanation of the practical scenarios where such AND queries are more natural or feasible than direct label queries is needed to justify the focus on this model.

2.  **Interpretability of Theoretical Bounds:** The theoretical bounds presented in Theorems 1, 2, 3, and 4 are the core results of the paper. However, their current expressions are quite complex, which makes it difficult to gain an intuitive understanding of how the query complexity scales with key problem parameters (e.g., number of points, number of clusters, noise level). I suggest the authors consider providing simplified or asymptotic versions of these bounds to make their implications more transparent. This would significantly enhance the interpretability and practical value of the theoretical results.

3.  **Scope of the Experimental Study:** The experiments appear to focus exclusively on algorithms that use AND queries. Given the motivational questions raised in point (1), the practical relevance of these experiments hinges on a convincing justification for the AND query model. To strengthen the empirical evaluation, I would encourage the authors to include a comparison with algorithms that use more standard pairwise similarity queries. This would provide a valuable baseline and a more comprehensive assessment of the proposed methods' performance in a broader context.

In summary, this is an interesting paper with solid theoretical contributions. My recommendation is contingent on the authors' response to the above points, especially the clarification regarding the motivation for the AND query model. If these concerns can be satisfactorily addressed, I would support the acceptance of this paper.

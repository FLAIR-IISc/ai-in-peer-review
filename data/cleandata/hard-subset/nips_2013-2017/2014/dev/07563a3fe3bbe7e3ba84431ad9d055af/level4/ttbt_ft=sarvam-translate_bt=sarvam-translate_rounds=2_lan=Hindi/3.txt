In this paper, the authors propose an interesting idea for learning invariance in an unsupervised learning setting for image classification problems. The main idea is to create surrogate classes consisting of randomly selected groups of patch transformations. The transformations are 2D functions such as rotation, scaling, translation, and color shift. The formal analysis is interesting and sheds further light on how this approach works.

The paper is well-written and easy to understand. It covers several aspects of the problem, including the number of transformations and the size of the network.

One drawback of this approach that seems to exist in the paper is that two randomly selected patches that are very similar in content will become "different" because they will be considered different surrogate classes. The authors clearly avoid this by using a maximum of 32,000 patches from a large dataset. However, using too few surrogate patches can hinder generalization in the case of large datasets. Do the authors have any comments on this?

Another experiment that would be good is to use the same set of surrogate classes for unsupervised learning and then train a dataset-specific classifier. For example, since STL-10 and CIFAR-10 are closely related, why did the authors need to retrain on each dataset?

Finally, experiments on different patch sizes would also be interesting.

In this paper, the authors propose an interesting idea for learning invariance in an unsupervised learning setting for image classification problems. The main idea is to create surrogate classes consisting of randomly selected groups of patch transformations. The paper's idea is new and interesting for unsupervised learning. It is well-written and easy to understand.

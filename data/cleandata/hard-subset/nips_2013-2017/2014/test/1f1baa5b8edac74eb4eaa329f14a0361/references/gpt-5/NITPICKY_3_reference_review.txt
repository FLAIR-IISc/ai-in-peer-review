Summary
The paper tackles zero-shot learning with attributes by explicitly modeling attribute unreliability. Instead of taking attribute predictions at face value, the authors train class-specific random forests directly on class-attribute signatures, but propagate each signature fractionally through the tree according to per-attribute ROC statistics estimated from attribute-labeled “validation” images. The split criterion uses an information gain computed with these fractional counts, biasing trees toward attributes and thresholds that are both discriminative and reliable given expected test-time errors. They also mention handling uncertainty in class signatures (details deferred to the supplement) and provide a few-shot extension that linearly combines signature-based information gain with standard data-based information gain via a parameter λ. Experiments on AwA, aPY, and SUN report improvements over DAP and a signature-only RF baseline, plus controlled-noise studies and ablations.

Relation to prior work
The work is positioned against two-stage attribute-based zero-shot methods (e.g., DAP) and knowledge-transfer/embedding approaches (e.g., label embeddings, DeViSE). The key novelty is integrating attribute error characteristics into the learning objective for zero-shot classifiers. It is also related to decision trees handling missing/uncertain features but differs in using class signatures as training exemplars and propagating uncertainty via ROC estimates.

Strengths
- Clear and relevant problem: zero-shot learning with realistic, imperfect attributes.
- Conceptual neatness: reinterpreting class signatures as fractional populations in predicted-attribute space, informed by ROC, is elegant and practical.
- Sound intuition captured in the training criterion: the method suppresses unlearnable attributes (e.g., when TPR≈FPR) and prefers reliable operating points.
- Empirical gains over DAP on three datasets, with meaningful ablations (signature uncertainty; node-specific vs. global ROC).
- Few-shot extension is simple and useful; the observed dependence of optimal λ on sample size makes sense.

Weaknesses and concerns
Technical/soundness
- Domain shift/calibration: The ROC statistics are estimated on seen-class attribute-labeled images DV (and node-conditional subsets DV(n)), then applied to unseen classes. If attribute detectors’ calibration or error profiles shift for unseen attribute combinations, this can bias split selection. This is acknowledged only tangentially (upper bound with “TRUE ROC”) but not mitigated. Any attempt to correct for/quantify this shift (e.g., by stratifying DV by signature similarity) reportedly had negligible effect; more analysis would help.
- Inconsistent and confusing data usage: Section “Datasets and setup” says 20% is reserved for ROC estimation and then pooled with the remaining 80% to train the final attribute classifiers. Later (“Real unreliable attributes experiments”) the text claims “we use only 80% of the attribute-labelled images to train attribute classifiers” yet also “our method and all baselines have access to exactly the same amount of attribute-labeled data.” These statements are contradictory. Please clarify precisely: which portion is used to train the attribute SVMs for the final test-time system for each method, and which portion is used solely to estimate ROC?
- Few-shot objective scaling: IGfew = λ IGzero + (1−λ) IGbasic mixes two information gains computed on fundamentally different sets (signatures vs. data). Since IG scales with the size/entropy of the sample set, how are these terms normalized to be commensurate? Without normalization, λ may have dataset-dependent, non-intuitive effects.
- Computational scalability: Training one RF per unseen class scales linearly in K and may be impractical for large K. The paper hints at multi-label forests as future work but provides no complexity/time analysis or memory footprint. Empirical timing would help.
- Dependency modeling: The DV(n) recursion partially captures dependencies between attributes via conditioning on previous splits, but each candidate split still uses a 1D ROC (per attribute). Strongly dependent attributes could still yield misleading TPR/FPR if selection bias reduces positives/negatives at deeper nodes (the fallback to global DV undermines the claimed dependency handling).
- Test-time mismatch: Training uses fractional propagation; testing uses hard routing. This is standard for such trees, but a brief discussion of whether soft test-time routing (e.g., probabilistic routing) affects performance would be helpful.

Empirical evaluation
- Baselines: While DAP and SIGNATURE-RF are relevant, the comparisons to contemporary embedding/transfer models (e.g., Akata et al., DeViSE) are only mentioned as “published results” on AwA in Fig. 2b; details and numbers are not visible in the main paper. Stronger and clearer head-to-head comparisons (same features, same splits) are needed, especially on SUN and aPY.
- Protocol details: It is unclear how hyperparameters (tree depth, number of candidate splits, λ) are selected for each baseline, and whether cross-validation uses only seen-class data consistently. Also, SUN unseen classes are “randomly selected”; multiple disjoint splits and their variance would be informative.
- Statistical reporting: Mean ± standard error over 20 trials is helpful, but no confidence intervals/p-values are provided. Given small absolute differences in some cases, significance testing would be useful.

Clarity/organization
- Several crucial details (pseudocode, signature uncertainty formulation, continuous signatures) are deferred to the supplement; the main paper would benefit from at least summarizing the signature-uncertainty model.
- Notation slips: Ai vs. Ak, and inconsistent use of i/k in Eqns (1–3). “With binary Ai(m), all 0 < t < 1 are equivalent” is true for Sec. 3.2.1 but becomes confusing when later we depend critically on t in Sec. 3.2.2; a clearer bridge would help.
- Minor: “whale seal” likely “harp seal” or “walrus”? Please check class names. “devise” should be DeViSE.

Formatting/reference issues
- Many references are incomplete (missing years, venues, pages), inconsistent capitalization, and typos. E.g., several entries lack years; “Label-Embedding for Attribute-Based Classification” missing year; “SUN Attribute Database” missing year/venue; “Attribute-based classification for zeroshot visual object categorization” appears twice with different formatting. Please fix to NIPS style.
- Footnote markers and numbering appear out of place in the main text; some footnotes are inline in awkward places.

Reproducibility
- Code/pseudocode not in main body; heavy reliance on supplement for algorithmic specifics and on provided dataset features. Precise details on ROC estimation bins, threshold sampling (#t), and the fallback when <10 positives/negatives should be fully specified.

Pros
- Addresses a real, under-explored weakness in zero-shot attribute methods.
- Simple, implementable idea with intuitive objective.
- Consistent improvements over DAP and ablations support the design.

Cons
- Clarity issues and contradictory data-usage statements.
- Missing or unclear comparisons to stronger contemporaries beyond AwA.
- Scalability unanswered; partial dependency modeling; reliance on DV that may not reflect unseen-class error profiles.
- Many formatting/reference inconsistencies.

Recommendation: Weak reject
The core idea is interesting and could be impactful, but the paper needs to resolve the data-usage inconsistencies, strengthen and clarify baselines, better justify IGfew scaling, and clean up presentation and references. With these addressed, I would lean toward acceptance.



# Review of "Zero-Shot Recognition with Unreliable Attributes"

This paper proposes a zero-shot learning method that accounts for unreliable attribute predictions by incorporating receiver operating characteristics (ROC) of attribute classifiers into random forest training. The main contribution is a framework that uses attribute signatures to train decision trees while explicitly modeling attribute prediction unreliability. The authors demonstrate improvements over baseline methods on three standard datasets.

## Strengths
The paper addresses a genuine challenge in zero-shot learning: the unreliability of attribute predictions. The idea of incorporating ROC characteristics into tree construction is novel and well-motivated. The experimental results are promising, showing significant improvements over DAP (a leading zero-shot method) on three datasets. The few-shot extension is a nice addition that demonstrates the method's flexibility.

## Weaknesses

### Technical Inconsistencies
The paper contains several critical technical inconsistencies that undermine its methodological soundness:

1. In equation (1), the authors define information gain with probability terms, but then use the 1-norm in equation (2) without explaining the transition. This creates a disconnect between the theoretical formulation and the implementation.

2. In equation (4), the authors define I'r(k) = I'n(k)Pn(âm(x) > t | y(m) = Ak(m)). This conditioning is problematic because Ak(m) is a binary class signature (0/1), while Pn(âm(x) > t | y(m) = Ak(m)) should be conditioned on the true attribute value in the validation data. The notation suggests they're conditioning on the class signature, which is not how ROC curves are typically computed.

3. The paper states "With binary Ai(m), all 0 < t < 1 are equivalent in Sec 3.2.1" but then uses thresholds t in the ROC analysis. This is inconsistent and confusing - if all thresholds are equivalent, why use them at all?

### Experimental Methodology
Several experimental details are poorly specified, affecting reproducibility:

1. The paper mentions "we use the features provided with the datasets" but fails to specify which exact features were used for each dataset (color histograms, SIFT, PHOG, etc.). This is critical for reproducibility.

2. The description of the validation data handling is confusing: "We reserve 20% of the attribute-labeled images as ROC validation data, then pool it with the remaining 80% to train the final attribute classifiers." This means they're using all data for training, contradicting the idea of having a separate validation set for ROC. If they're pooling the validation data with training data, the ROC estimates would be overly optimistic.

3. The paper reports "mean and standard error measured over 20 random trials" but doesn't provide statistical significance tests for the performance differences. Without these, it's unclear whether the improvements over DAP are statistically meaningful.

### Presentation and Clarity Issues
Several presentation issues undermine the paper's clarity:

1. Inconsistent notation: The paper uses both "â" and "âm" to denote predicted attribute scores, creating confusion.

2. The definition of "OURS" in Table 1 is unclear. The paper mentions "OURS W/O SIG UNCERTAINTY" and "OURS W/O ROC PROP" but doesn't clearly define what "OURS" includes in the main text.

3. The paper claims "the attribute vocabulary is expressive enough to discriminate between the categories" but provides no evidence for this critical assumption.

4. The authors state "we use the unseen class splits specified in [9] for AwA and aPY, and randomly select the 10 unseen classes for SUN" without justifying why SUN was treated differently from the other datasets.

## Overall Assessment

While the paper addresses an important problem and presents promising results, the technical inconsistencies in the methodological description and the lack of critical experimental details significantly undermine its scientific rigor. The paper would benefit from:

1. Clarifying the mathematical formulation, particularly the relationship between the theoretical information gain definition and the implementation
2. Providing complete experimental details (features used, statistical significance tests)
3. Resolving the inconsistencies in the validation data handling
4. Clarifying the definition of "OURS" in the main text

The paper's core idea is interesting, but the execution contains enough fundamental technical flaws that it's difficult to evaluate the true contribution. The methodological inconsistencies are particularly concerning as they affect the paper's reproducibility and scientific validity.

I recommend rejection based on the substantial methodological inconsistencies that prevent proper assessment of the contribution. The paper would benefit from a complete revision to address these technical issues before being reconsidered for publication.

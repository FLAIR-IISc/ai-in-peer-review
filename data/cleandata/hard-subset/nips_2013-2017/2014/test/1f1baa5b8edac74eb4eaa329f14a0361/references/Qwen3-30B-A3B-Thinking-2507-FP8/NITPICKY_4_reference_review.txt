

# Review of "Zero-Shot Recognition with Unreliable Attributes"

This paper proposes a novel approach to zero-shot learning that explicitly accounts for the unreliability of attribute predictions. The authors recognize a critical limitation in existing zero-shot methods: they treat attribute predictions as perfect, when in reality attribute classifiers are often unreliable. Their solution involves modifying random forest training to incorporate attribute classifiers' receiver operating characteristics (ROC) during tree construction, thereby building in "cushion" for expected errors.

**Strengths:**
- The paper identifies a genuine problem in zero-shot learning that has been overlooked in the literature.
- The approach is theoretically sound and well-motivated.
- The experimental results are convincing, demonstrating significant improvements over state-of-the-art DAP on three challenging datasets.
- The few-shot extension is a valuable addition that demonstrates the method's flexibility.

**Weaknesses:**

1. **Technical inconsistencies in the core algorithm:** The paper's description of the "fractional propagation" mechanism contains a critical error. In Section 3.2.2, the authors state: "we channel all predicted negatives to the left child node. In contrast, a naive random forest (RF) trained on signatures assumes ideal attribute classifiers and channels all ground truth negatives—i.e., true negatives and false positives—through the left node." This is incorrect. In standard classification, the "negative" class is the one we're trying to distinguish from the positive class. For the attribute classifier, when Ak(m) = 0 (attribute not present in class k), the correct behavior would be to channel those instances to the left node (where attribute m ≤ t), not "predicted negatives." The paper's description conflates attribute presence with class membership, creating confusion about the underlying logic.

2. **Ambiguous mathematical formulation:** Equation (4) states I′r(k) = I′n(k)Pn(âm(x) > t | y(m) = Ak(m)), but the conditional probability notation is problematic. It should be P(âm(x) > t | y(m) = Ak(m), x ∈ DV(n)) to properly specify the conditioning. The current notation implies Pn is a probability distribution, which is confusing.

3. **Incomplete experimental details:** The paper claims "our method significantly outperforms the existing DAP method [9]," but [9] in the references is listed as "Attribute-based classification for zeroshot visual object categorization, Christoph H Lampert, Hannes Nickisch, Stefan Harmeling, , , " which is missing the conference name and year. This is a formatting error that undermines credibility. Additionally, the paper doesn't specify which DAP variant (probabilistic or non-probabilistic) was used for comparison.

4. **Unaddressed limitation:** The paper acknowledges that validation data DV comes from seen classes, not unseen classes, but fails to discuss how this limitation affects real-world performance. The authors claim "our training procedure would be equivalent to building a random forest on the test samples' attribute classifier outputs" if DV perfectly represented unseen class errors. This is an important limitation that should be addressed more thoroughly, especially since the paper's success depends on the validation data being representative.

5. **Formatting inconsistencies:** The paper uses â (with circumflex) for predicted attribute values without defining it clearly at first use. The citation style is inconsistent (some citations use [10, 18, 8], others use [8, 3, 18, 24, 19]). The references section contains incomplete entries (e.g., missing conference names and years).

6. **Lack of comparison to non-attribute-based methods:** The paper doesn't compare against methods that don't rely on attributes at all (e.g., those using class hierarchies or label embeddings), making it difficult to assess whether the attribute-based approach is fundamentally better or just a better implementation of an existing paradigm.

**Recommendation:**

While the paper addresses a real problem in zero-shot learning with a theoretically sound approach, the technical inconsistencies in the core algorithm, combined with formatting errors and incomplete experimental details, undermine the paper's credibility. The error in Section 3.2.2 is particularly problematic as it affects the fundamental understanding of the proposed method. The paper would be much stronger with a clearer mathematical formulation, proper notation, and a more thorough discussion of the validation data limitation.

I recommend rejection due to the technical errors and formatting inconsistencies that prevent the paper from being evaluated properly. The authors should revise the paper to address these issues before resubmission. The core idea is promising, but the current presentation contains too many critical flaws to be accepted at NIPS.

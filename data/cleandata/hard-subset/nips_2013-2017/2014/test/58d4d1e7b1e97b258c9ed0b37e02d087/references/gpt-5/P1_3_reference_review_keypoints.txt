- Paper addresses non-identifiability of prior, likelihood, and loss in Bayesian perceptual models using a flexible observer class for 1D estimation tasks.  
- Observer model maps stimuli via a parametric monotonic transform, models noise/prior/loss as max-entropy mixtures of Gaussians, and includes lapses/reporting noise for tractable responses.  
- Introduces an a priori identifiability metric: expected log-likelihood (expected cross-entropy) across datasets generated by a reference observer, reducible to a weighted sum of KL divergences.  
- Uses an “expected posterior” E(θ|θ*) ∝ exp(⟨log Pr(D|θ)⟩) explored by MCMC to map parameter degeneracies and compare experimental designs.  
- Two case studies: interval timing (prior recovery fragile unless motor noise known; MAP-like loss can worsen identifiability) and speed perception (slow-speed prior exponent and contrast-dependent noise reliably recovered).  
- Builds on prior Bayesian observer and identifiability literature, connecting to model discrimination and experimental design concepts.  
- Modeling choices (mixture bases, Gaussian-lattice prior) extend earlier parametric approaches while keeping computation tractable.  
- Expected log-likelihood derivation as a KL sum is technically sound for i.i.d. data with known stimulus frequencies.  
- Using the surrogate “posterior” for identifiability exploration is reasonable and computationally attractive.  
- Case studies validly demonstrate design comparisons and nontrivial interactions, interpreted appropriately as synthetic/a priori.  
- Strong parametric assumptions (two-Gaussian mixtures, Gaussian-lattice prior, translational invariance) may bias identifiability conclusions and need sensitivity analysis.  
- Key max-entropy mixture parameter determination is central but relegated to the supplement, reducing clarity and reproducibility.  
- 2AFC-to-estimation conversion in speed perception is only sketched and requires a clear, validated mapping.  
- Computational details (MCMC setup, convergence diagnostics, runtime scaling) are insufficiently reported.  
- Notation is heavy/inconsistent; a notation table and cleanup are needed.  
- Crucial assumptions (e.g., lapse responses sampled from the prior) require justification in the main text.  
- The identifiability-KL framework combined with a flexible, efficient observer model is original and practically valuable.  
- If code is released, the framework could be widely useful for designing perceptual experiments and anticipating non-identifiabilities.  
- Pros: clear identifiability metric, flexible tractable model, thoughtful case studies, engagement with an important debate.  
- Cons: reliance on specific parametric forms, sparse computational details, limited connection to broader design literature, unverified 2AFC equivalence.  
- Suggested improvements: add an algorithm box, conduct sensitivity analyses (components, basis size, lapse/noise models), report Fisher/local identifiability, cite/design literature explicitly, and validate 2AFC mapping.  
- Overall recommendation: Weak accept (6/10) contingent on added clarity, sensitivity checks, and broader contextualization.

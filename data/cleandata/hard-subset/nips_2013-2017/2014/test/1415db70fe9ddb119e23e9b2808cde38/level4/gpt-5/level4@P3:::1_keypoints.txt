- Paper proposes GB-RNNs: add a single top-down “feedbackward” pass to recursive nets to refine node/word predictions with global context.  
- Model concatenates forward and backward node vectors and uses hybrid word vectors (fixed unsupervised + small supervised part).  
- Evaluated on SemEval-2013 target-dependent sentiment (Twitter), showing improvements over RNN, bidirectional recursive baseline, and competitive SOTA ensemble results.  
- Novelty: differs from prior bidirectional recursive work in how top-down vectors are computed and in combining with hybrid embeddings and dropout.  
- Clear motivation: context-dependent labeling requires more than bottom-up composition or fixed windows.  
- Simple, principled extension: one top-down pass is easy to integrate into existing recursive RNNs.  
- Empirical gains: consistent improvements over strong baselines and competitive results on a public benchmark.  
- Useful engineering: hybrid embeddings and effective dropout for RNN training.  
- Ablations and analyses provided (single vs ensemble, vector sizing, extra data).  
- Terminology issue: “belief propagation” is misleading; suggest “forward–backward recursive” or “top-down recursive refinement.”  
- Experimental clarity needed: explicitly state that baselines use identical hybrid vectors, dropout, and tuning protocols.  
- Clarify model selection: cross-validation protocol, number of restarts, stopping criteria, and which dev set guided hyperparameters.  
- Report single-model results alongside ensembles and compare fairly to SemEval entries that didn’t ensemble.  
- State whether external resources (NRC lexicon, distant supervision, movie reviews) were used by baselines; if not, justify or run matched settings.  
- Training details are too brief: explicitly state use of backpropagation through structure and how top-down pass is differentiated, with pseudocode or a computational graph.  
- Address gradient/depth issues: discuss vanishing/exploding gradients, gradient clipping/normalization, and runtime/training complexity relative to baselines.  
- Clarify parameter roles and dimensions (V and W↓) and how nd is chosen.  
- Section 3.6 is confusing and contains editorial artifacts; tighten the mathematical comparison to Irsoy & Cardie and remove stray text.  
- Provide a concise table contrasting parameterization and information flow with Irsoy & Cardie; fix incomplete/incorrect references.  
- Dataset/task description should explicitly define SemEval subtasks, clarify the focus on target-dependent polarity, evaluation metric, splits, and tweet vs SMS differences.  
- Explain parser constraints and how labeled spans are enforced as subtrees, including heuristics for non-contiguous or unparsable spans.  
- More ablations needed to isolate top-down pass contribution from hybrid embeddings and extra data (e.g., RNN vs GB-RNN with identical vectors/no external data).  
- Include sensitivity to parsing errors, error analysis (failure cases like long-distance dependencies/sarcasm), and statistical significance tests of improvements.  
- Overall quality: model is sound and well-motivated but needs more training/experimental detail and removal of editorial errors for reproducibility.  
- Recommendation: lean to accept contingent on clarifying training/comparison details, tightening Section 3.6, improving dataset/task description, and fixing citations/artifacts.

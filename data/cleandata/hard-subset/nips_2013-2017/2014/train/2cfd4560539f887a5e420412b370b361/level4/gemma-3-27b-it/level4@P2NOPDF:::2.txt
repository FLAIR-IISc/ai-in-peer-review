This paper presents a novel deep recursive neural network architecture that achieves state-of-the-art results on a challenging sentiment classification dataset. The performance gains are significant, and the paper is exceptionally well-written and easy to follow. 

The authors introduce several key improvements to existing recursive neural network (RNN) models, each thoughtfully motivated and convincingly demonstrated through experimentation. These include: disentangling the parameter matrices for leaf and non-terminal nodes; incorporating rectified linear units (ReLUs) within the recursive framework; leveraging large, pre-trained word vectors in conjunction with smaller hidden representations for non-terminal nodes; and, crucially, developing a genuinely *deep* recursive architecture. This depth isn’t simply a replacement of a single RNN layer with a deeper one, but rather establishes connections *between* layers and tree nodes, allowing for more complex feature interactions. While not essential, it would be interesting to see a comparison where the input to each parent node consisted solely of the output from the last hidden layer of its children – essentially, removing the direct connection mediated by matrix V. This could help to isolate the benefit of the inter-layer connections.

I did identify one critical issue with the presentation of results. Table 1(a) reports performance on the final test set. This is problematic, as model tuning should be performed on a development set, and the test set should be reserved for a final, unbiased evaluation. Reporting performance on the test set during development introduces the risk of overfitting to the test data. 

Finally, I noted a minor typographical error: the phrase “comprise a class of architecture” should be revised.

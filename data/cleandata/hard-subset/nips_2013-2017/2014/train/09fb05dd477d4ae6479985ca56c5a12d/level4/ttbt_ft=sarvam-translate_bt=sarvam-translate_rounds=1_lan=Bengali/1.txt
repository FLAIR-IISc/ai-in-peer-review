This paper discusses the problem of object recognition, specifically the challenge of obtaining a bounding box of the size of the class label that exists for object classification. The authors address this challenge by proposing adapting object classifiers for recognition tasks. Their algorithm is quite simple and straightforward, which is not inherently bad. Their experimental protocol uses 100 classes for training (including both class labels and bounding boxes) and tests on 100 excluded classes. These excluded classes only had class labels. The results were reasonably good, filling about a 35% gap between an oracle and a baseline on 100 excluded classes.

**Quality and Clarity:** This is a well-written, clearly presented paper.
**Originality:** This paper combines two existing concepts related to adaptation: adapting object classifiers for recognition and adapting from one set of classes to another. According to this reviewer, this is a new and interesting idea – but not groundbreaking.
**Significance:** The main contribution of this paper is a new (but not groundbreaking) method. In terms of results, it is difficult to judge its long-term significance, although it will probably be surpassed by more sophisticated methods later (as expected). The main problem with the results is that they sacrifice performance (mAP) by using smaller bounding boxes, and there is no objective way to judge whether this trade-off is worth it.
(As a side note, it is not clear whether the problem being addressed is really a significant long-term challenge. Yes, bounding boxes are more "expensive" than class labels. But who is to say that throwing resources at the problem won't fix it? Not too long ago, it seemed almost impossible to have millions of images labeled with 20,000 object classes...)
**Detailed Problems to Be Solved:**
- In Figure 3a, the authors should clearly state which methods use class labels (or do not) for recognition, in addition to their own method and R-CNN. This would provide a clearer understanding of an apple-to-apple comparison.
- My judgment of how good the proposed method is is based on the blue bars in Figure 3b. It is about 1/3 between the DNN baseline and the oracle. Do the authors agree that this is one of the main results? If yes, please highlight it, not 78% (line 341) – which is not the correct number for 2 reasons: 1) the judgment should only be based on the excluded classes, and 2) it should only be a percentage of the oracle performance (the baseline should be considered). Similar problem for the claim of 50% improvement (line 425).
- The authors should dedicate a table or a sub-paragraph to clearly state the differences from R-CNN, which seems to be the closest competing alternative. Currently, these statements do not seem scattered or as prominent as they could be. I would suggest dropping Figure 6, which is just an anecdotal example.

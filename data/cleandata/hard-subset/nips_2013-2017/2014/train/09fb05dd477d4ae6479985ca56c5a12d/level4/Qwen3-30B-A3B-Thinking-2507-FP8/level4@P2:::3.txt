

This paper introduces LSDA (Large Scale Detection through Adaptation), a method to transform image classifiers into object detectors using minimal bounding box annotations. The core insight treats classification-to-detection adaptation as a domain adaptation problem: leveraging classification data (image-level labels) as the source domain and detection data (bounding boxes) as the target domain. For categories with bounding box labels (set B), LSDA fine-tunes a pre-trained CNN to learn a generic feature transformation and background model. For categories without annotations (set A), it transfers category-specific adaptation via nearest-neighbor averaging of weights from set B. Evaluated on ImageNet detection (200 categories), LSDA achieves a 50% relative mAP improvement over a classification-only baseline on held-out categories (100 classes), enabling a 7,604-category detector using only classification data for 7,404 classes.

**Quality**  
The technical approach is sound and well-motivated, building logically on R-CNN [1] and domain adaptation literature. The ablation study (Table 1) convincingly shows that category-invariant adaptation (fine-tuning shared layers + background class) contributes most to gains (mAP: 10.31% → 15.85%), while category-specific adaptation (Equation 1) provides marginal further improvement (15.85% → 16.15%). However, the paper lacks critical baselines for fair comparison:  
1. *Standard R-CNN fine-tuning*: Training a detector on all 200 classes using bounding boxes (the oracle upper bound).  
2. *MIL-based weak supervision*: Using full-image labels with MIL to select top object proposals (as in [6,7]), which would test if weak labels alone suffice.  
3. *Simple classifier adaptation*: Running the classification CNN on regions without adaptation (the current baseline).  
Without these, it’s unclear whether LSDA’s gains stem from novel adaptation or inherent CNN feature improvements. The authors claim "significant improvement," but the baseline comparison is insufficient to support this.  

**Clarity**  
The paper is exceptionally well-written, with clear motivation and logical flow. The methodology (Sections 3.1–3.3) and experimental setup (Section 4.1) are detailed and reproducible. Minor improvements could clarify Table 1: explicitly stating whether each row uses Equation (1) for category-specific adaptation (e.g., "k=10 nearest neighbors") would aid readers. The error analysis (Section 4.3) effectively demonstrates reduced background/localization errors post-adaptation.  

**Originality**  
LSDA’s novelty lies in *applying domain adaptation to scale detection to thousands of categories* using only sparse bounding box annotations. While the adaptation framework draws from R-CNN [1] and domain adaptation work [12–16], it uniquely addresses the *hold-out category* problem (no detection data for set A), which prior methods [17–22] cannot solve. The ImageNet-scale demonstration (7,604 categories) is a clear advancement over prior weakly supervised detection work [6,7], which focused on small datasets (e.g., PASCAL VOC). This is sufficiently novel for NIPS.  

**Significance**  
The problem—scaling detection to thousands of categories—is highly relevant to real-world applications (e.g., web-scale object detection). LSDA’s ability to leverage abundant image-level labels (e.g., from search engines) instead of expensive bounding boxes is a major practical contribution. The 50% mAP gain on held-out categories demonstrates tangible progress toward large-scale detection. While performance lags behind the oracle (200-class detection), the approach enables a feasible path to 7,604-category detection, which would be impossible with current annotation methods. This advances the state of the art in scalable vision systems.  

**Pro/Con Acceptance**  
*Pros*:  
- Solves a critical scalability bottleneck in detection (no bounding box data for most categories).  
- Demonstrates practical impact (7,604-category detector, released code).  
- Methodologically clean, with strong empirical validation on ImageNet.  
*Cons*:  
- Missing baselines (MIL, standard fine-tuning) weaken the claim of "significant improvement."  
- Limited analysis of computational cost (e.g., 5.5-hour training vs. R-CNN’s 3 days is noted, but no speed-accuracy tradeoff discussion).  
- The "fast detector" (2fps) is promising but not rigorously compared to other fast detectors (e.g., Fast R-CNN).  

**Recommendation**  
The paper makes a meaningful contribution to scalable object detection with a novel adaptation framework. While the baseline comparison is a notable weakness, the core idea is compelling, the experiments are well-executed, and the significance is high. I recommend *acceptance* with a minor revision to add the missing baselines (particularly MIL) in Section 4.2 to strengthen the claim of "significant improvement." The current results suggest LSDA is effective, but without these comparisons, the paper’s contribution is slightly overstated. The method’s potential to enable detection for tens of thousands of categories is a strong argument for its value to the field.

This research paper addresses the issue of object recognition, specifically the challenge of obtaining bounding boxes on the same scale as existing class labels for object classification. The authors sidestepped this challenge by proposing optimizing object classification for the identification task. Their algorithm is quite simple and straightforward, which in itself is not a bad thing. Their experimental protocol tests 100 categories (with both class labels and bounding boxes) and 100 missing categories for training. These missing categories only contained class labels. The results obtained were quite good, filling approximately 35% of the difference between an oracle and a baseline on 100 missing categories.

**Quality and Clarity:** This is a well-written, high-quality paper.
**Originality:** This paper combines two existing ideas related to optimization: optimizing object classification for identification, and optimizing from one set of categories to another. According to this reviewer's knowledge, this is a new and interesting idea – but not unprecedented.
**Significance:** The main contribution of this paper is a new (but not unprecedented) approach. Because, according to the results, it is difficult to assess long-term significance, although they will be overcome later by a more refined method (as expected). The main issue with the results is that they essentially sacrifice performance (mAP) by using a smaller bounding box, and there is no objective way to judge whether this compromise is worthwhile.
(As a side note, it is not clear whether the issue being dealt with is actually a significant long-term challenge. Yes, obtaining bounding boxes is more "expensive" than class labels. But who can say that it will not be fixed by throwing resources at the problem? Many years ago, having millions of images labeled with 20,000 object categories seemed almost impossible...)
**Detailed Issues to Consider:**
- In Figure 3a, which methods (or not) use class labels to help with identification, in addition to their method and R-CNN? The authors should clearly state this, so there is clarity about apple-to-apple comparisons.
- My judgment of how good the proposed approach is is based on the blue bars in Figure 3b. The DNN baseline and oracle are approximately 1/3 different. Do the authors agree that this is one of the main results? If yes, please highlight it, not 78% (line 341) – which is not exactly the right number for 2 reasons: 1) the decisions should only be made based on the missing categories, and 2) the oracle should only be a percentage of the performance (the baseline should be taken into account). The same problem for the claim of a 50% improvement (line 425).
- The authors should dedicate a table or a sub-section to clearly state the difference from R-CNN, which seems to be the closest competing option. Currently, these statements seem scattered or not as prominent as they could be. I would suggest "sacrificing" Figure 6, which are only anecdotal examples.

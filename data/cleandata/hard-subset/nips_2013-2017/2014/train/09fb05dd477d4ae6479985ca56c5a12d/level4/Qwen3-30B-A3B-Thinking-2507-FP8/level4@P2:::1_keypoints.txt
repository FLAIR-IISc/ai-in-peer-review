- Proposal: LSDA adapts image classifiers into object detectors using abundant image-level labels plus bounding boxes for a small subset of categories.  
- Method: Learns category-invariant adaptation by fine-tuning shared network layers on limited detection data and category-specific adaptation by transferring weight changes to unlabeled categories via nearest neighbors.  
- Results: On ImageNet detection, LSDA gives a 50% relative mAP improvement over a classification-only baseline on held-out categories and enables a 7,604-category detector using classification data for 7,404 categories.  
- Quality: Technically sound and well-executed with a convincing ablation study and reproducible implementation (Caffe, public code).  
- Evaluation rigor: Proper held-out protocol simulates scalability, explicitly models background rejection, and avoids overfitting via domain-adaptation principles.  
- Clarity: Exceptionally well written and organized; pipeline and experiments are clearly presented, with only minor labeling inconsistencies (e.g., "DNN" vs. "LSDA").  
- Originality: Novel synthesis of domain adaptation (classification→detection) and category transfer to adapt unlabeled categories; distinct from R-CNN and MIL approaches.  
- Significance: Addresses a critical scalability gap between large-scale classification and limited detection labels, trading some accuracy for large practical gains (e.g., thousands-category detectors).  
- Practicality: Scalable and deployable (fast version ~0.5 fps) with public models and code.  
- Strength: Solves a high-impact, concrete problem with clear, reproducible methodology and strong empirical validation.  
- Strength: Rigorous ablation isolates contributions and demonstrates tangible gains on held-out categories.  
- Weakness / Revision 1: Clarify wording that "50% relative improvement" is relative to the classification-only baseline (not the oracle) in abstract/results.  
- Weakness / Revision 2: Add a concise comparison to R-CNN clarifying that R-CNN requires bounding boxes for all categories while LSDA needs them for only a subset.  
- Weakness / Revision 3: Improve Figure 3a to explicitly label which methods use detection labels (e.g., "Classification Only", "LSDA", and R-CNN as oracle).  
- Weakness / Revision 4: Move anecdotal examples (Figure 6) to the appendix or replace them with a quantitative table showing mAP gains across category subsets.  
- Overall assessment: LSDA is a practical, well-validated contribution that materially advances large-scale detection feasibility and merits acceptance after minor revisions.

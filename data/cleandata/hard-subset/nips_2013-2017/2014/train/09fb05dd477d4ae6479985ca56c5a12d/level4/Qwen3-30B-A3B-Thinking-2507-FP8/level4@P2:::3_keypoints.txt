- Introduces LSDA: converts image classifiers to detectors via domain adaptation using few bounding boxes.  
- Core idea: treat classification data as source and detection (bounding boxes) as target to learn category-invariant and category-specific adaptations.  
- For categories with boxes (B): fine-tunes CNN to learn shared feature transform and background model; for without boxes (A): transfers category-specific weights by nearest-neighbor averaging.  
- Demonstrated on ImageNet detection (200 classes) and a 7,604-category detector using boxes for only 200 classes (7,404 classes use only classification data).  
- Quantitative gains: ~50% relative mAP improvement on held-out 100 classes versus classification-only baseline.  
- Ablation results: category-invariant adaptation yields largest gain (mAP 10.31% → 15.85%); category-specific adaptation gives marginal additional improvement (→ 16.15%).  
- Technical approach is sound and builds logically on R-CNN and domain-adaptation literature.  
- Paper is well-written and reproducible; experiments and error analysis show reduced background/localization errors after adaptation.  
- Novelty: applying domain adaptation to scale detection to thousands of categories and addressing the hold-out category problem is a clear contribution.  
- Significance: enables scalable detection using abundant image-level labels, a practical advance toward web-scale object detection.  
- Missing critical baselines weaken claims: (1) standard R-CNN fine-tuning on all 200 classes (oracle), (2) MIL-based weak supervision using image labels, and (3) simple classifier-on-regions (no adaptation) baseline.  
- Limited analysis of computational cost and speed-accuracy tradeoffs; "fast detector" (2 fps) not compared rigorously to other fast detectors.  
- Pros: solves scalability bottleneck, practical impact (7,604 categories), released code, clean method with strong ImageNet validation.  
- Cons: incomplete baseline comparisons, limited runtime/efficiency analysis, marginal category-specific gains.  
- Recommendation: accept with minor revision to include missing baselines (especially MIL) to substantiate the "significant improvement" claim.

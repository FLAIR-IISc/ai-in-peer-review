

This paper proposes LSDA, a method for adapting object classifiers into detectors using minimal bounding box annotations. The core insight addresses the practical bottleneck of obtaining bounding boxes for large-scale detection: it leverages abundant image-level classification data (e.g., from ImageNet's 20k+ categories) and only requires bounding box labels for a small subset of categories (e.g., 100 out of 200 in ImageNet). LSDA learns a transformation from classification to detection via category-invariant feature adaptation (fine-tuning shared layers on limited detection data) and category-specific parameter adaptation (transferring learned weight changes to unlabeled categories via nearest neighbors). Experiments on ImageNet detection show a 50% relative mAP improvement over classification-only baselines on held-out categories, enabling a 7,604-category detector using only classification data for 7,404 categories.  

**Quality**: The technical approach is sound and well-executed. The ablation study (Table 1) convincingly isolates the contribution of each adaptation component, and the implementation details (using Caffe, public code) ensure reproducibility. The method avoids overfitting by leveraging domain adaptation principles and explicitly models background rejection—a critical shift from classification to detection. The evaluation protocol is rigorous, with held-out categories simulating real-world scalability challenges.  

**Clarity**: The paper is exceptionally well-written and organized. Key contributions (category-invariant vs. category-specific adaptation) are clearly defined early, and the pipeline (Figure 2) effectively visualizes the workflow. The experimental results (Figures 3–5) directly support claims, and the discussion of error types (e.g., background confusion reduction) provides actionable insights. Minor issues (e.g., inconsistent "DNN" vs. "LSDA" labeling) are easily fixed.  

**Originality**: LSDA synthesizes two established concepts—domain adaptation for task transfer (classification → detection) and category transfer (using labeled categories to adapt unlabeled ones)—into a novel, scalable solution. While R-CNN [1] demonstrated classifier-to-detector adaptation, it requires bounding boxes for *all* categories. LSDA’s key novelty is adapting to *unlabeled* categories without detection data, which the authors thoroughly contrast with prior work (Section 2). The technique is distinct from MIL methods [6,7], which rely on weak labels but lack evaluation on large-scale detection. The novelty is modest but impactful for practical scalability.  

**Significance**: The work addresses a critical scalability bottleneck in detection: the disconnect between the 20k+ categories in ImageNet (with classification labels) and the ~200 categories with bounding boxes. LSDA’s ability to enable detection for tens of thousands of categories using existing classification data is highly significant. While the method trades some performance (vs. full bounding box data), the trade-off is justified by the massive scale gains—e.g., a 7.6K detector with minimal annotation overhead. The 50% relative mAP gain on held-out categories (Table 1) demonstrates tangible progress toward bridging the classification-detection gap. The approach is also practical: the fast version (0.5 fps) enables deployment on large datasets like videos.  

**Strengths**:  
- Solves a concrete, high-impact problem (scaling detection to ImageNet-scale categories).  
- Clear, reproducible methodology with strong empirical validation.  
- Practical implementation (public code/models) and scalability demonstration (7.6K detector).  
- Rigorous ablation study isolating key contributions.  

**Weaknesses & Required Revisions**:  
1. **Result interpretation**: The claim of "50% relative improvement" (line 425) is correct relative to the classification-only baseline (10.31% → 15.85% mAP), but the draft review incorrectly frames it as "filling 35% of the gap to oracle." *Clarify*: The 50% improvement is relative to the *baseline* (not oracle), and the oracle gap (16.15% vs. 20.5% in the paper) is separate. *Revision*: Explicitly state "50% relative mAP improvement over the classification-only baseline" in the abstract/results.  
2. **Comparison to R-CNN**: The distinction between LSDA and R-CNN is scattered (e.g., lines 52–53, 422–425). *Revision*: Add a concise table (or subsection) comparing:  
   - *R-CNN*: Requires bounding boxes for *all* categories; trains per-category SVMs.  
   - *LSDA*: Requires bounding boxes for *only a subset*; no SVMs; adapts to unlabeled categories.  
3. **Figure 3a clarity**: The draft notes ambiguity in "which methods use category labels." *Revision*: In Figure 3a, explicitly label the baseline ("Classification Only") and LSDA, and clarify that R-CNN (with full bounding boxes) is the oracle.  
4. **Figure 6**: The anecdotal examples (Figure 6) are less critical than the quantitative results. *Revision*: Move to appendix or replace with a table showing mAP gains across category subsets.  

**Overall Assessment**: LSDA presents a technically sound, clearly articulated solution to a major scalability challenge in detection. The method is novel in its combination of domain adaptation and category transfer, and its results demonstrate significant practical value—enabling detection for 7,604 categories with minimal annotation effort. While not a breakthrough in theory, it delivers a tangible, deployable advance that aligns with NIPS’ focus on impactful, scalable machine learning. The minor revisions requested (result interpretation, R-CNN comparison) are straightforward and will strengthen the paper. The work merits acceptance; it advances the field by making large-scale detection feasible with existing data infrastructure.

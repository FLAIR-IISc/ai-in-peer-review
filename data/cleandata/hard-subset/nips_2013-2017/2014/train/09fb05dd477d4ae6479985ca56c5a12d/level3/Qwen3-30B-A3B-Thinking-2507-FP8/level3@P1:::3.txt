

This paper proposes LSDA (Large Scale Detection through Adaptation), a method to transform image classifiers into object detectors using only image-level labels for most categories, leveraging bounding box annotations for a small subset. The approach builds on the R-CNN framework but addresses the critical scalability challenge of detection: obtaining bounding box annotations for thousands of categories is prohibitively expensive, while image-level labels (e.g., from ImageNet) are abundant. LSDA learns a generic transformation from classification to detection by: (1) pre-training a CNN on classification data for all categories; (2) fine-tuning layers 1–7 using bounding box data for a subset of categories (including a background class); and (3) adapting category-specific weights for unseen categories via nearest-neighbor transfer. Evaluated on ImageNet detection, LSDA achieves a 50% relative mAP improvement over classification-only baselines on held-out categories (100/200) and scales to a 7,604-category detector. The work positions itself as domain adaptation (source: classification, target: detection) but differs from prior methods by adapting both task and domain, enabling detection for categories without any bounding box data.

**Quality**. The technical execution is sound, with thorough ablation studies (Table 1) demonstrating that category-invariant adaptation (fine-tuning layers 1–7 + background class) is the most impactful component. The 50% relative mAP gain is compelling, and the 7.6K detector demo is a significant contribution to scalability. However, the paper lacks three critical baselines that undermine claims of method superiority. First, it omits a "full-image positives with region negatives" baseline (e.g., using image-level labels to generate region proposals for training, as in [6,7]), which is a simple, strong weakly-supervised approach. Second, it fails to compare against MIL-based methods (e.g., [6,7]), which are the most relevant weakly-supervised competitors. Third, the mean-box baseline (using the center of the image as a pseudo-bounding box) is missing, which would establish the problem's difficulty and serve as a minimal performance floor. Without these, the 50% gain could merely reflect the strength of the underlying CNN features rather than LSDA’s adaptation mechanism. The attribution concern is valid: if the mean-box baseline achieves comparable results, LSDA’s novelty is diminished. The authors should clarify whether their gains stem from adaptation or the CNN’s representational power.

**Clarity**. The paper is well-structured and clearly written, with a logical progression from problem statement to method to evaluation. However, Section 4.2’s Table 1 lacks sufficient detail: it does not explicitly map each row to the specific adaptation technique (e.g., which layers were fine-tuned, which equation was used for category adaptation). For instance, the "Background + Full Adaptation" row should clarify whether this corresponds to Equations 1–3. The description of the nearest-neighbor adaptation (Equation 1) is also ambiguous without context on how k=10 was selected. These gaps hinder reproducibility and weaken the ablation’s interpretability.

**Originality**. LSDA is sufficiently novel for NIPS. While it extends R-CNN and domain adaptation, the key innovation is adapting *to held-out categories* without detection data—a gap in prior work (e.g., [20,21] require detection labels for all categories). It is a logical, non-radical combination of existing ideas (domain adaptation, nearest-neighbor transfer) tailored to a critical scalability problem. The work is specific to CNN architectures (e.g., Krizhevsky’s) but is generalizable to other feature extractors. Related work cites MIL and domain adaptation but could better distinguish LSDA’s task-shift focus from prior domain adaptation (which assumes fixed tasks).

**Significance**. The problem addressed—scaling detection to tens of thousands of categories—is highly significant for real-world applications (e.g., web-scale object recognition). The 7.6K detector demonstrates tangible progress, and the method reduces annotation costs drastically. The paper’s contribution to enabling large-scale detection is valuable, though the gap to oracle performance (full bounding box data) remains substantial (Table 1). The release of models and code at lsda.berkeleyvision.org further enhances practical impact.

**Pro/Con Acceptance**.  
*Pro*: Addresses a critical scalability bottleneck; compelling 7.6K detector demo; well-executed experiments on ImageNet; practical code release.  
*Con*: Missing key baselines (especially mean-box and MIL) weaken the core claim; attribution ambiguity; unclear Table 1 explanations.  

**Recommendation**. The paper presents a valuable, well-motivated approach to a significant problem. Its significance and scalability contribution warrant acceptance *if* the authors address the baseline gaps and clarify experimental details. As written, the absence of the mean-box baseline and MIL comparison is a serious flaw—it leaves open whether LSDA’s gains are methodological or merely a consequence of using a strong CNN backbone. The authors should add these baselines immediately. The clarity issues in Table 1 are minor but should be resolved for reproducibility. With these revisions, this work would be a strong candidate for NIPS. Without them, the contribution is overstated. The problem is important, but the evidence for LSDA’s specific innovation is incomplete.

Summary and relation to prior work:
The paper tackles a central bottleneck in scaling object detection: the scarcity and cost of bounding-box annotations for thousands of categories. The proposed LSDA framework learns a generic transformation from image classification to detection using detection data from a subset of categories (B), and transfers it to categories lacking any detection annotations (A). The approach has three components: (1) adding and learning a background class to handle the classification-to-detection task shift; (2) category-invariant adaptation by fine-tuning shared layers (1–7) on B, which empirically improves detection for unseen categories A; and (3) category-specific adaptation by approximating the per-class delta between classification and detection weights at the output layer via an average of nearest neighbors (in the classification fc8 parameter space) from B.

The method builds on R-CNN (fine-tuning for detection) but departs crucially by requiring detection boxes only for a subset of categories and by explicitly transferring a learned classifier-to-detector transformation to held-out categories. It contrasts with MIL-based weakly supervised detection and prior domain adaptation for detection that typically require target labels for all categories; the authors argue LSDA is the first to adapt detectors to held-out categories without any detection labels.

On ILSVRC13 detection, simulating 100 categories with detection labels and 100 held-out, LSDA provides a ~50% relative mAP gain on held-out categories over using a classification network on region proposals, with most gains coming from the category-invariant adaptation and background modeling; the nearest-neighbor fc8 delta adds a smaller but statistically significant boost. The authors also demonstrate a 7,604-category detector using 200 detection categories and 7,404 classification-only categories.

Strengths:
- Novel and bold problem framing: adapting across tasks (classification→detection) and across categories, with no detection labels for many classes. This opens a path to large-scale detection without prohibitive annotation costs.
- Simple, well-motivated architecture that isolates where transfer comes from: background modeling and shared feature adaptation do the heavy lifting; class-specific deltas add modest improvements.
- Solid empirical study: clear ablations disentangling background, shared-layer fine-tuning, and output-layer deltas; statistical significance reported; useful error breakdowns (localization vs background vs other).
- Practical impact: substantial relative gains for held-out categories; 7.6K-class demo; code/models released; reduced training time compared to full R-CNN pipelines.
- Clarity: generally well-written with a coherent narrative and sufficient implementation details to reproduce (Caffe, splits, training protocol).

Weaknesses:
- Category-specific adaptation is heuristic. Using L2-normalized fc8 weight nearest neighbors (effectively cosine similarity) and averaging deltas is intuitive but ad hoc. Alternatives (e.g., learning a regression from classification to detection weights, or conditioning deltas on semantic/attribute embeddings) are not explored.
- Limited breadth of comparisons. While the paper positions against MIL and adaptation baselines, there is no direct large-scale comparison to a strong weakly supervised detection method (authors note such results are lacking), nor to alternative transfer schemes (e.g., learning a shared detection head with class-agnostic localization and class-specific classification).
- Backbone and pipeline are fixed to AlexNet + selective search, and the evaluation is confined to ILSVRC13. It would strengthen claims to show generality across architectures or datasets, although the released SPP/fast proposals variant partially addresses efficiency.
- Absolute performance on held-out categories remains low (e.g., ~16% mAP vs oracle upper bound), so the method is a step forward but does not close the gap; more analysis of which classes benefit (or fail) as a function of similarity to B would be informative.
- Minor clarity/notation issue: A is defined as {m,...,K} while B is {1,...,m}; presumably A should be {m+1,...,K}.

Quality:
Technically sound overall. The core claims—category-invariant transfer via shared-layer fine-tuning plus background, and modest class-specific gains via nearest-neighbor deltas—are well supported by ablations and error analysis. Some design choices (nearest-neighbor averaging) lack deeper justification; sensitivity analyses on k and similarity metrics are only briefly mentioned. Nonetheless, the empirical evidence convincingly demonstrates the effect of each component.

Clarity:
Clear and well-organized, with helpful figures and ablations. Public release of code/models is a plus. Suggestions: fix the A/B notation; explicitly report sensitivity to k and to the choice of similarity measure; add more per-class analyses or stratify gains by semantic similarity to B.

Originality:
High. Adapting a learned classification→detection transformation to categories without any detection labels is a fresh angle relative to both R-CNN fine-tuning and prior domain adaptation that assumes per-category labels. The decomposition into background modeling, shared feature adaptation, and class-specific deltas is a clean, novel formulation.

Significance:
High. If we can turn classifiers into detectors at scale with limited detection supervision, many applications benefit. Even with modest absolute mAP, the relative gains on held-out categories and the 7.6K detector demo suggest strong practical impact and a direction others will build on.

Questions/suggestions for authors:
- Report sensitivity to k and to different neighbor metrics (e.g., cosine vs semantic/WordNet distance). Does mixing semantic and visual neighbors help?
- Try learning a mapping from Wc to Wd (e.g., linear regression or MLP on fc8 weights/features) and compare to nearest-neighbor averaging.
- Analyze performance as a function of similarity to B (e.g., does LSDA help more when a held-out class has close analogs in B?).
- Provide per-class gains and additional qualitative examples where localization improves.

Pros for acceptance:
- Conceptually bold, addresses a meaningful scalability bottleneck.
- Clear, effective method with solid ablations and practical release.
- Demonstrated large-scale detector and measurable gains on held-out categories.

Cons for acceptance:
- Heuristic class-specific adaptation with limited alternatives explored.
- Absolute mAP remains low; limited architectural/dataset breadth.

Recommendation: Accept. This paper introduces a valuable new direction—learning a generalizable classification→detection transformation—that is likely to influence subsequent work, and the empirical evidence is sufficiently strong despite some heuristic components.

Confidence: High.

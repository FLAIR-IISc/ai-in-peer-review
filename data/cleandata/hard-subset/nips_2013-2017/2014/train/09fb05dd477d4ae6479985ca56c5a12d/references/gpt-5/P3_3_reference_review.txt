Summary
The paper proposes LSDA, a practical framework to adapt an image classifier into an object detector when only a subset of target categories have bounding-box annotations. The approach treats “classification to detection” as a domain/task adaptation problem. It learns (i) a category-invariant transformation by fine-tuning shared CNN layers (and adding a background class) on the detection-labeled subset B, and (ii) a category-specific transformation for held-out classes A by transferring the average change observed in the detector’s final layer (fc8) from the nearest neighbors in B, where nearest neighbors are defined in the classification fc8 parameter space. At test time, scores are taken as class score minus background score per region proposal. On ILSVRC13 detection, holding out 100 of 200 classes, LSDA improves mAP on held-out categories from 10.31% to 16.15% (+50% relative), and demonstrates a 7.6K-category detector by adapting from classification-only labels for most categories. Code/models are released.

Relation to prior work
The paper builds on R-CNN (pretrain on classification, fine-tune on detection) but addresses the critical scalability bottleneck: per-class bounding boxes. Compared to weakly supervised MIL for detection, LSDA uses a small set of fully annotated categories to learn a generalizable adaptation and transfers it to categories without boxes; this is complementary to MIL and potentially more scalable. Prior domain adaptation work mostly adapts across visual domains for a fixed task; LSDA adapts both domain and task (classification→detection) and modifies the feature extractor via fine-tuning, which many shallow adaptation methods cannot. Supervised detector adaptation (e.g., adaptive SVMs, DPM adaptations) requires detection labels per class; LSDA is (to my knowledge) the first to adapt to held-out categories without any bounding boxes.

Strengths
- Problem importance and practical impact: reducing per-class bounding-box supervision is crucial for scaling detection to thousands of classes; the 7.6K demo is compelling.
- Clear decomposition: category-invariant adaptation (shared layers + background) and category-specific adaptation (fc8 deltas via nearest neighbors) is simple, efficient, and easy to implement.
- Solid empirical validation: ablations quantify the contribution of background modeling, shared-layer fine-tuning, and category-specific transfer; error analysis shows reduced localization and background confusions.
- Reproducibility: implementation details and public release of models/code.
- Efficiency: avoids per-class SVMs/bbox regression and shows a faster SPP-based variant.

Weaknesses and concerns
- Heuristic category-specific transfer: Averaging fc8 deltas from nearest neighbors in classification weight space is intuitive but ad hoc. There is no analysis justifying this metric (fc8 vs fc7 features, cosine vs Euclidean, semantic/WordNet proximity) or why k=10 is best; sensitivity analysis is brief.
- Missing baselines: Lacks comparison to weakly supervised MIL methods on the same protocol (even if small-scale), or to alternative simple transfers, e.g., (a) fine-tuning on B and directly using the adapted shared layers with unmodified fc8 for A (they do this partly), (b) learning a regression mapping g: Wc→Wd from B and applying to A, (c) semantic transfer (attributes/word embeddings) for category-specific adaptation, (d) bbox regression learned on B applied to A.
- Absolute performance remains low: While a 50% relative gain is meaningful, mAP on held-out classes is ~16% and far from the fully supervised upper bound. The gap is not deeply analyzed (e.g., which error modes persist after adaptation, per-class gains vs losses).
- Calibration and scoring: The “score_i − score_background” rule is ad hoc; an analysis of proper probabilistic calibration (e.g., softmax with background as a competing class, odds ratios, or Platt scaling) could improve cross-class comparability, especially in the 7.6K setting.
- Clarity/consistency: The description of how detection annotations are sourced is a bit confusing: the training set is said to be used as classification-only, yet later 1000 bbox-annotated images per category are taken from the training set following R-CNN. Please clarify the data split to avoid perceived leakage (which images/categories are used for classification-only vs detection fine-tuning).
- Limited exploration of scaling variables: No study of how performance on A varies with the number of detection-labeled categories m, domain similarity between A and B, or the effect of proposal method; these would strengthen claims about scalability/generalization.

Quality
Technically sound overall, with careful ablations and a useful diagnostic error analysis. The method is largely heuristic but effective. Some important baselines are missing, and the category-specific transfer could be better justified and explored.

Clarity
Generally well written and organized. The three insights and training stages are clear. Improve clarity around data splits/annotation availability, provide more detail on k selection and distance metrics for nearest neighbors, and quantify the “nearly the same” performance for the faster SPP variant.

Originality
Moderately novel. The key novelty is framing classification→detection as an adaptation problem that modifies the representation and transfers category-specific changes to held-out classes. While components (fine-tuning, background modeling, nearest-neighbor transfer) are known, their combination for zero-bbox categories is new and useful.

Significance
High practical significance: the approach lowers the barrier to large-scale detection and influenced subsequent work on low-supervision detection and knowledge transfer. The demonstrated gains on a strong benchmark and the 7.6K detector release increase impact.

Suggestions for improvement
- Replace nearest-neighbor averaging with a learned mapping from Wc (and/or fc7 features) to δ via ridge regression or a small MLP trained on B; compare to semantic similarity (WordNet/word2vec).
- Analyze sensitivity to k and to the choice of neighbor metric; include per-class results for diverse similarity types (fine-grained vs generic).
- Add bbox regression trained on B and applied to A; report its effect on localization errors.
- Compare to at least one MIL baseline on a shared subset to contextualize gains.
- Provide probabilistic calibration of scores for large-scale multi-class NMS across 7.6K categories.
- Study performance vs number of detection-labeled categories m to quantify annotation-efficiency trade-offs.

Pros for acceptance
- Addresses an important and timely problem with a simple, scalable method.
- Clear empirical gains on a standard benchmark with informative analysis.
- Released models/code and a compelling large-scale demonstration.

Cons for acceptance
- Category-specific transfer is heuristic and under-validated; missing strong alternative baselines.
- Absolute performance on held-out classes remains modest; limited analysis of remaining error modes.
- Some clarity issues in dataset usage and in reporting the fast variant’s accuracy.

Recommendation
Weak accept. The paper offers a practical and influential contribution to low-supervision object detection with solid empirical support. Strengthening baselines and clarifying experimental details would further bolster the case.

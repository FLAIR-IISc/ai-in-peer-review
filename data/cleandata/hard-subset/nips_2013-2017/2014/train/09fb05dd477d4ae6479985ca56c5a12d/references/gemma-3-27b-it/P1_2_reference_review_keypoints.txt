- Summary: Proposes LSDA, a domain-adaptation method to convert image classifiers into object detectors using limited bounding-box annotations and abundant image-level labels.  
- Key idea: Learn a category-invariant classifier-to-detector transformation from categories with boxes and apply it to held-out categories without boxes.  
- Primary result: Demonstrates substantial improvement (≈50% relative mAP) on held-out ImageNet detection categories.  
- Relation to prior work: Extends R-CNN-style adaptation and distinguishes itself from MIL and domain-adaptation literature by adapting classifiers to detectors for unseen categories.  
- Strength — problem importance: Tackles the annotation bottleneck for large-scale object detection.  
- Strength — novelty: Introduces a novel classifier-to-detector adaptation framework using category-invariant and category-specific adjustments.  
- Strength — empirical validation: Strong experimental results and clear ablation studies showing component contributions.  
- Strength — reproducibility: Authors release code and pre-trained models.  
- Strength — analysis: Provides useful error analysis and false-positive breakdown.  
- Weakness — comparisons: Lacks direct comparison to state-of-the-art MIL/weakly-supervised detection methods on ImageNet.  
- Weakness — sensitivity: Relies heavily on the quality of the pre-trained classifier without analysis of sensitivity to pre-training data/architecture.  
- Weakness — heuristic choice: Nearest-neighbor selection for category-specific adaptation uses Euclidean distance on fc8 weights with limited justification.  
- Weakness — large-scale analysis: Insufficient quantitative/qualitative analysis for the 7604-category detector and long-tail performance.  
- Weakness — clarity: “sim” label in Figure 5(a) is unexplained, hindering error-analysis interpretation.  
- Recommendation: Accept with minor revisions; authors should add MIL comparisons, sensitivity discussion, justification/alternatives for nearest-neighbor selection, expanded 7604-category analysis, and clarify “sim” in Figure 5(a).

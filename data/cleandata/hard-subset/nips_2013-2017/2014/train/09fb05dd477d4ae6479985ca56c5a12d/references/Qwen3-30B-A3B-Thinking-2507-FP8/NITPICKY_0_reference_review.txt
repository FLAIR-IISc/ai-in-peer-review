

# Review of "LSDA: Large Scale Detection through Adaptation"

## Summary
The paper proposes LSDA, a method to adapt image classifiers into object detectors without requiring bounding box annotations for all categories. The key insight is to treat classifier-to-detector adaptation as a domain adaptation problem, using a small subset of categories with bounding box annotations to learn a transformation applicable to all categories. The authors demonstrate a 50% relative mAP improvement over a classification-only baseline on ImageNet detection.

## Strengths
The paper addresses an important practical problem in object detection: the scarcity of bounding box annotations for large-scale categories. The core idea of adapting classifiers to detectors using domain adaptation is novel and potentially valuable for scaling detection to thousands of categories. The method is conceptually clear, and the experimental results on ImageNet detection show promising improvements.

## Weaknesses

### Technical Errors and Inconsistencies
1. **Methodological Error**: The paper states "We replace the last weight layer (1000 linear classifiers) with K linear classifiers" - this is technically incorrect. The last layer is a single linear classifier with 1000 outputs (not 1000 linear classifiers), which is replaced with a single linear classifier with K outputs.

2. **Incorrect Dataset Statistics**: The paper claims "the ILSVRC2013 detection dataset has 400K annotated images" and "validation set has 20K annotated images." These are factually incorrect. The ImageNet Detection Challenge (ILSVRC2013) has approximately 50K training images and 5K validation images, not 400K and 20K as stated.

3. **Confusing Terminology**: The paper states "fine-tuning transforms all network weights (except for the linear classifiers for set A)" - this is misleading. The fine-tuning process does not transform the linear classifiers for set A; they remain untouched. The description is unclear and technically inaccurate.

4. **Equation Formatting**: Equation (1) contains formatting errors: "1 k k∑ i=1" should be "1/k ∑_{i=1}^k". Such technical errors undermine the paper's credibility.

### Methodological Concerns
1. **Lack of Baseline Details**: The paper claims "a strong baseline" but doesn't provide the mAP value for this baseline in the main text, making it impossible to assess the claimed 50% improvement.

2. **Arbitrary Adaptation Method**: The nearest-neighbor approach for adapting categories without bounding box data (equation 1) is presented without justification or comparison to alternatives. The paper doesn't discuss why the Euclidean distance in feature space is appropriate for this task.

3. **Reproducibility Issues**: The paper claims "model definitions weights publicly available" but provides no link or repository. The URL "lsda.berkeleyvision.org" appears to be a placeholder, which is unacceptable for a scientific paper.

### Writing and Presentation
1. **Grammar and Formatting**: Multiple grammatical errors throughout (e.g., "a large of amount" should be "a large amount"). The reference list contains inconsistent formatting with extra commas (e.g., "In Proc. CVPR, , , ,").

2. **Ambiguous Claims**: The phrase "50% relative mAP" is confusing. The paper doesn't clarify whether this is a 50% relative improvement over the baseline or an absolute 50% mAP.

3. **Missing Details**: The paper claims "we have made our model definitions weights publicly available" but doesn't provide any details about the model architecture or hyperparameters used.

## Overall Assessment

The paper presents an interesting approach to a meaningful problem, but the technical errors, inconsistent dataset descriptions, and lack of reproducibility details significantly undermine the credibility of the work. While the core idea has potential, the paper fails to properly present the methodology, results, and experimental setup. The claimed 50% improvement is impossible to verify due to missing baseline details, and the technical inaccuracies suggest a lack of careful proofreading.

The paper would benefit from a complete rewrite to correct the technical errors, clarify the methodology, provide accurate dataset statistics, and ensure reproducibility through proper code and model release. As currently written, the paper's technical shortcomings outweigh its conceptual contribution.

## Recommendation

**Reject**

The paper's technical errors and lack of reproducibility details are substantial enough to warrant rejection, despite the potentially valuable idea. The authors should address these issues before resubmitting.

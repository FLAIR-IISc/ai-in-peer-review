

# Review of "LSDA: Large Scale Detection through Adaptation"

## Summary
This paper proposes LSDA, an algorithm that transforms image classifiers into object detectors by learning a domain adaptation from classification to detection. The authors argue that while image-level classification data is abundant (e.g., ImageNet with 20,000+ categories), bounding box detection data is scarce (e.g., PASCAL VOC with 20 categories, ImageNet detection with 200 categories). LSDA uses a small subset of categories with bounding box annotations to learn a transformation that adapts classifiers for categories without detection data. The authors claim a 50% relative mAP improvement over a classification-only baseline on held-out categories.

## Strengths
The paper identifies a genuine problem in scaling object detection to large numbers of categories. The core idea of using domain adaptation to transfer detection capabilities from a few labeled categories to many unlabeled ones is conceptually sound and addresses a practical limitation in the field. The ablation study provides reasonable evidence that their adaptation technique improves detection performance.

## Weaknesses

### 1. Critical Terminology and Dataset Confusion
The paper repeatedly confuses ImageNet classification with ImageNet detection datasets. For example, in the introduction, it states "the more recent ImageNet detection challenge dataset [3] has extended the set of annotated images," but citation [3] refers to the ImageNet classification paper (Krizhevsky et al., 2012), not a detection dataset. This is a serious error that undermines the paper's credibility.

### 2. Methodological Inconsistencies
- Section 3.1 claims "We replace the last weight layer (1000 linear classifiers) with K linear classifiers," but ImageNet classification has 1000 categories, while their detection task uses K=200 categories. The paper fails to clarify how they handle the mismatch between the pre-trained weights and their target categories.
- Section 3.2 defines set A as "{m, ...,K}" when it should be "{m+1, ...,K}" since set B is "{1, ...,m}". This is a basic set theory error.
- The nearest neighbor approach (Equation 1) assumes "categories with the nearest (minimal Euclidean distance) `2-normalized fc8 parameters in the classification network" will have similar detection features, but this assumption lacks justification and may be invalid.

### 3. Technical Errors
- In Section 3.3, the authors claim "scorei − scorebackground" for detection, but this is incorrect. Standard object detection uses the score for category i without subtracting background score.
- The paper states "we directly use the final score vector to produce the prediction scores without either of the retraining steps" (SVMs on layer 7 and bounding box regression on layer 5), which misrepresents R-CNN. R-CNN does not use the final score vector for detection; it uses SVMs on layer 7 features.

### 4. Experimental Details
- The paper claims "50% relative mAP boost" but never states the baseline mAP value for comparison (e.g., "from 10.31% to 15.85%").
- They state "We have made our model definitions weights publicly available" without providing a URL, which is required for reproducibility.
- The paper claims "the ImageNet detection challenge dataset [3]" but [3] is for classification, not detection. The ImageNet detection dataset should be cited as ILSVRC2013 detection.

### 5. Formatting and Grammar
- "a large of amount" (Introduction) should be "a large amount"
- "a object detector" should be "an object detector"
- "the source domain, and the data used to train detectors (images with bounding boxes and category labels) as our target domain" - "target domain" should be singular
- "image-level annotation" should be "image-level annotations"
- "the more recent ImageNet detection challenge dataset [3]" - [3] is classification, not detection

### 6. Missing Comparisons
The paper fails to compare with recent state-of-the-art detection methods like Faster R-CNN, Mask R-CNN, or SSD, which would strengthen their claims about improvement.

## Overall Assessment
While the paper addresses a meaningful problem in scaling object detection, the numerous technical errors, dataset confusions, and lack of attention to detail significantly undermine its credibility. The paper contains multiple fundamental mistakes in terminology, methodology, and experimental setup that suggest the authors did not thoroughly vet their work. The paper's claims about improvements and methodology are weakened by these inconsistencies.

The quality of the work is not sound due to these errors, which are not merely minor typographical issues but rather substantive technical and methodological flaws. The paper fails to meet the basic standards of technical accuracy expected at NIPS.

## Recommendation
Reject. The paper contains multiple critical technical errors and inconsistencies that make it difficult to trust the results or methodology. While the core idea has potential, the execution and presentation are too flawed to be accepted. The errors are not just minor typos but indicate a lack of careful review of the paper before submission.

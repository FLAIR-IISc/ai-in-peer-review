

# Review of "LSDA: Large Scale Detection through Adaptation"

## Summary
This paper proposes LSDA, an algorithm for adapting image classifiers into object detectors using limited bounding box annotations. The authors frame the problem as domain adaptation between classification (source domain) and detection (target domain), leveraging category-invariant feature transformations and category-specific adaptations. The key insight is that fine-tuning on a small set of categories with bounding box annotations enables adaptation to categories without such annotations. The approach shows a 50% relative mAP improvement over classification-only baselines on held-out categories in ImageNet detection.

## Strengths
The paper identifies a meaningful problem in scaling object detection to large numbers of categories, which is a significant practical challenge. The method is conceptually sound and builds on established domain adaptation techniques while making a specific contribution to the detection task. The experimental results are well-structured, with a clear ablation study demonstrating the importance of each component. The release of code and models is a commendable practice.

## Weaknesses

### Technical and Methodological Issues
1. **Inconsistent Notation**: The paper defines sets B = {1, ..., m} and A = {m, ..., K}, which creates confusion since A starts at m rather than m+1. This inconsistency in set definition undermines the clarity of the method.

2. **Mathematical Formulation Errors**: Equation (1) contains critical notation issues: "W dj = W cj + 1/k k∑ i=1 δBNB(j,i)" should be "W_d^j = W_c^j + (1/k) ∑_{i=1}^k δ_B^{NB(j,i)}" to be mathematically precise. The notation for δB is ambiguous (is it a vector or matrix?), and the indexing is inconsistent with previous definitions.

3. **Methodological Ambiguity**: The claim that "fine-tuning transforms all network weights (except for the linear classifiers for set A)" is confusing. Set A consists of categories without bounding box annotations, so their classifiers should be adapted via the k-NN approach, not excluded from fine-tuning. This creates confusion about the adaptation process.

4. **Experimental Details**: The paper states they "augment our detection set with 1000 bounding box annotated images per category from the ILSVRC2013 training set" but doesn't explain why they're using the same training set for both classification and detection data, which could introduce data leakage.

### Writing and Presentation Issues
1. **Grammatical Errors**: The paper contains multiple grammatical errors, such as "a large of amount" (p.1), "providing the research community a dataset" (p.1), and "it only contains data for 200 categories" (p.1) - the comma splice here needs correction.

2. **Inconsistent Citations**: The reference for "Tabula rasa" appears as both "In Proc. ICCV" and "In IEEE International Conference on Computer Vision" - these should be consistent.

3. **Reference Formatting**: Many references have double "In" (e.g., "In In Proc. CVPR"), and some have extra commas (e.g., "International Journal of Computer Vision,,"). The reference for ImageNet [3] should be updated to use the correct NeurIPS citation style.

4. **Figure References**: The paper references "Figure 3" multiple times without providing a clear description of what it shows, making it difficult to understand the experimental results without seeing the figure.

5. **Vague Claims**: The paper states "a small performance loss" when using the final score vector directly, but doesn't quantify this loss or compare it to R-CNN's SVM and bounding box regression steps. This is a significant omission.

### Broader Context
The paper doesn't sufficiently differentiate its approach from prior domain adaptation work for detection, particularly Aytar et al.'s "Tabula rasa" method. While the paper mentions this work, it doesn't clearly articulate how LSDA differs or improves upon it. The claim that "to our knowledge, ours is the first method to adapt to held-out categories that have no detection data" is questionable given prior work on transfer learning in detection.

## Overall Assessment
While LSDA addresses a meaningful problem and shows promising results, the paper contains numerous technical, notational, and grammatical issues that undermine its clarity and scientific rigor. The methodological ambiguities (particularly regarding the set definitions and mathematical formulation) make it difficult to fully understand and reproduce the approach. The writing contains multiple grammatical errors that suggest a lack of careful proofreading.

The paper's contribution is interesting but not sufficiently novel compared to existing domain adaptation work for detection. The 50% relative mAP improvement is notable, but the paper fails to provide sufficient detail to assess whether this is a meaningful advancement or simply a result of the specific experimental setup.

## Recommendation
I recommend rejection. While the paper presents a potentially useful method, the numerous technical and writing issues significantly weaken its contribution. The paper would benefit from substantial revision to clarify the methodology, fix notational inconsistencies, and provide more precise experimental details before it could be considered for publication. The minor flaws identified here collectively constitute a substantial barrier to understanding and evaluating the work.

- Problem & method: Proposes PDMM, a randomized parallel block-coordinate ADMM with a dual “backward step”, supporting exact/inexact primal updates (via Bregman linearization) and overlapping blocks; dual step sizes depend on matrix sparsity and number of updated blocks K.

- Theoretical results: Proves global convergence and an O(1/T) ergodic iteration complexity with constant step sizes; step-size/parameter choices reflect row-block degrees di and K.

- Relation to prior work: PDMM generalizes and unifies multi-block ADMM variants (recovers sADMM and PJADMM as limiting cases) and addresses gaps in randomized/multi-block convergence analyses.

- Technical quality: Solid convergence analysis and practical inexact-update framework; insightful dependence of parameters on sparsity and sampling.

- Experiments: Relevant tasks (robust PCA, overlapping group lasso) show empirical improvements over baselines, but all implementations are sequential so parallel wall-clock gains are not demonstrated.

- Clarity issues: Ambiguity on whether all dual blocks are updated each iteration versus a dual block-coordinate variant; algorithm box should explicitly state which dual coordinates update and how τi, νi, K̃i interact.

- Originality: Key novelties are the dual backward step, explicit sparsity/K-dependent step sizes, randomized parallel handling of overlapping blocks, and conceptual unification of sADMM/PJADMM.

- Significance: Advances multi-block ADMM methodology with provable guarantees and practical parallelism potential; would benefit many structured large-scale problems if block/parameter guidance is clarified.

- Strengths: Clear formulation and motivation; simple parallelizable updates; provable global convergence and O(1/T) rate with constant steps; unifies prior methods; empirical gains on representative problems.

- Weaknesses/concerns: Unclear dual-update protocol; limited discussion on specializing the rate to prior methods; no guidance on partitioning A to optimize convergence; lack of parallel speed benchmarks; minor typographical/notation issues.

- Key question 1: Clarify whether (5)–(7) update all dual blocks each iteration or only a subset, and state the default PDMM instance plus any dual-coordinate extension and guarantees.

- Key question 2: Correct Theorem 2’s notation—confirm the missing summation ∑i=1I in the first bound.

- Key question 3: Explicitly show how Theorem 2 specializes to sADMM and PJADMM (matching O(1/T) or faster rates) with concrete parameter choices (K, τi, νi, ηj).

- Key question 4: Provide practical heuristics for partitioning A (choice of number/size of blocks, grouping by sparsity, balancing di and K̃i) and discuss trade-offs between larger K and per-iteration cost.

- Key question 5: For tensor/multi-linear completion with multiple blocks, recommend PDMM parameterizations (K, τi, νi, ηj) and block partitions, and compare expected behavior to sADMM/PJADMM.

- Key question 6: For fair baselines, include corrected sADMM results with an appropriate dual update (e.g., τi = 1/J) instead of τi = 0.

- Minor edits: Fix spacing in Eq.(9), remove extra space before “Section” at line 217, state K̃i = min{di,K} and τi = K/[K̃i(2J−K)], νi = 1−1/K̃i in the algorithm box, and add a summary table mapping PDMM parameters to sADMM/PJADMM.

- Recommendation: Accept, conditional on clarifying algorithmic dual updates, fixing minor errors, and expanding discussion on specialization to prior methods and block/parameter selection.

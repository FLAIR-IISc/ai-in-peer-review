- Paper proposes a framework to discover computationally efficient reformulations of symbolic target expressions using an attribute grammar and learned search strategies.  
- Equivalence is checked via numerical descriptors: evaluations over random inputs modulo a large prime to avoid floating-point issues.  
- For targets not representable by a single tree, method solves Fw = t mod p to find linear combinations of trees whose descriptor matches the target.  
- Search is guided by learned strategies: subtree n-gram models and a recursive neural network (RNN) embedding plus a softmax rule predictor.  
- Experiments show learned strategies outperform random search and discover many complexity-reducing identities, including challenging RBM-inspired sums; code and data are released.  
- Related work connections underemphasized: symbolic regression, grammar-guided genetic programming, superoptimization, compiler algebraic simplification/strength reduction, Spiral, equality-saturation/e-graphs, and term-rewriting.  
- Novelty claimed in learning search heuristics from solved instances and using randomized modular evaluations as a scalable equivalence oracle.  
- Strength: clear, well-scoped problem (homogeneous scalar polynomials) with a grammar-based representation and complexity attributes.  
- Strength: modular-evaluation equivalence-checking is practical and enables scalable search and data generation.  
- Strength: learned n-gram and RNN strategies substantially improve discovery rates and handle higher-degree identities.  
- Strength: RNN produces continuous embeddings of expressions that generalize across problem sizes with good classification accuracy; curriculum and initialization choices are sensible.  
- Strength: released code and multiple challenging families increase credibility and potential impact.  
- Weakness: Lacks formal collision analysis for descriptor-based equality (e.g., Schwartz–Zippel bounds) and mitigation via multiple primes.  
- Weakness: Unclear constraints and semantics for coefficients w in Fw = t mod p—mapping to realizable grammar expressions, integrality, coefficient-size control, and spurious modular matches are not addressed.  
- Weakness: Cost model is informal; paper omits explicit per-rule operation counts, cost of combining trees, and empirical timing/microbenchmarks.  
- Weakness: Evaluation metric (fraction of runs finding solution in 10 minutes) is limited; needs counts/forms of identities, representative examples in main text, proof sketches/inductive derivations, and comparisons to known methods (e.g., DP for symmetric polynomials).  
- Weakness: Learning ablations missing—RNN underperforms n-grams; need ablations on beam vs greedy scheduling, beam width, search depth, training set size, and alternative policies (MCTS/UCT, value functions).  
- Clarity issues: missing explicit grammar table, constraints on w, dataset size/balance, number of primes and N, and handling of multivariable examples beyond simple cases.  
- Presentation: move representative "new identities" into the main text rather than relegating to supplement.  
- Originality: integration of learned heuristics with grammar-based search and modular oracle is novel, but novelty diluted by weak positioning against established related work.  
- Significance: potential impact on compiler optimization and symbolic computation if robust and generalizable, but current scope is narrow and often restricts grammar rules.  
- Questions for authors: provide collision probability bounds, use of multiple primes, constraints on w, sensitivity to scheduling strategies (greedy vs beam/MCTS), operation counts/empirical runtimes, and whether known DP solutions (e.g., Symk) are rediscovered.  
- Pros summary: clear setup, learning improves search, interesting identities, code released, useful continuous embeddings.  
- Cons summary: missing formal correctness guarantees and mapping from linear-system solutions to grammar expressions, limited evaluation metrics and baselines, scope restrictions and removal of grammar rules, underdeveloped related-work positioning.  
- Recommendation: weak accept conditioned on clarifying correctness/coefficients, adding representative identities and cost accounting in main text, and strengthening comparisons to program synthesis/optimization literature.

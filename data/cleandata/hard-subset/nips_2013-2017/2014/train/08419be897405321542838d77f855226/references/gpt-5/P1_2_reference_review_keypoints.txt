- Problem: Discover lower-cost algebraic identities for a target expression by searching an attribute grammar over matrix/vector operations with cost attributes.
- Approach: Learn search strategies from simpler instances using two families — n-gram models over subtrees and an RNN embedding with a softmax rule-prediction policy.
- Equality check: Use numerical evaluation over a finite field Zp (random testing) and allow linear combinations solved via modular Gaussian elimination.
- Results: Finds identities reducing O(n^3) expressions to O(n^2) and polynomial-time surrogates for some exponential sums (e.g., RBM-related) for small k.
- Code/Data: Authors release code and data for reproducibility.
- Relation to prior work: Connects to theorem proving, program induction, CAS simplification, and learned guidance for proof search, but focuses on fixed-degree homogeneous polynomials and explicit complexity reduction.
- Novelty: Combining attribute grammars with learned heuristics and an RNN embedding of algebraic expressions is original and reusable.
- Practicality: Finite-field randomized testing and modular solving are pragmatic, though not entirely novel.
- Soundness concern: No formal guarantees for the descriptor-based equality check; need Schwartz–Zippel-style bounds relating N and p to collision probability.
- Lifting concern: Lack of discussion on correctness of reconstructing integer-coefficient symbolic sums from modular solutions (integrality, coefficient size, normalization).
- Experimental strengths: Learned strategies, especially 3–5-grams, substantially outperform random search; RNN classifies equivalence classes well.
- RNN limitations: Downstream search performance mixed — n-grams often outperform RNNs, with RNN only better on one hard family.
- Experimental reporting: Time budget (10 minutes) and restarts are reasonable, but more statistics (time to first solution, variance, trees explored) are needed.
- Missing baselines: No comparison to CAS simplifiers under a cost model, hand-designed heuristics, or MCTS-style guided search.
- Verification gap: Final discovered identities lack symbolic verification in the main text; numerical checks alone are insufficient for confidence.
- Clarity positives: Paper is generally well-written with useful RNN training details and concrete examples.
- Clarity issues: Key details (exact grammar, classification accuracies, search space sizes, identities) are relegated to the supplement and should appear in the main paper.
- Expressivity limits: Grammar is small, restricting discoveries; some found identities are trivial/known.
- Significance: Potentially impactful for code optimization and symbolic-numeric systems, but current scope and results are limited (small grammar, reliance on lower-k training).
- Pros summary: Well-motivated problem, novel method mix, released artifacts, good embedding results, improved search effectiveness.
- Cons summary: No formal error bounds, limited baselines/ablations, RNN often underperforms n-grams, unsubstantiated novelty of identities, limited evaluation metrics.
- Suggestions: Provide theoretical bounds (Schwartz–Zippel), detail reconstruction from modular solutions, add CAS and heuristic baselines, include key identities with symbolic proofs, analyze n-gram vs RNN gap and explore stronger models/MCTS.
- Recommendation: Weak Accept — interesting direction with credible empirical evidence but needs stronger theory and evaluation.

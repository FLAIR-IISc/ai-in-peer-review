This paper proposes an interesting approach to discovering more efficient algebraic identities and shows that distributed, recursive neural representations can successfully classify these identities. The empirical result is engaging.

That said, the problem definition appears quite restricted. It would help to clarify the scope and discuss how (or whether) the approach generalizes beyond the presented setting.

A key technical point is not well explained. You write: “weight tensor W3 (of size l×l×l, so that the output is also size l).” Does this mean you are using a neural tensor network rather than a simple linear composition layer? Please make this explicit. Bowman has shown that tensor networks perform substantially better than RNNs on related tasks; a comparison or discussion in this context would be valuable.

Section 7.2 (Efficient Identity Discovery): It is unclear whether you are still using the RNN for classification here, while evaluating different search strategies, or whether the classifier changes. Please clarify exactly which model is used during search and evaluation. I also did not follow how proposals are generated in your search procedure; a more detailed description of the proposal mechanism (e.g., how candidates are formed, constraints, deduplication, and scoring) would make this section much clearer.

Overall, the paper tackles a compelling problem and demonstrates promising results with recursive neural representations, but it would benefit from clearer explanations of the model components (especially the use of a tensor network) and the proposal-generation process in the search, as well as from addressing the restricted problem scope.

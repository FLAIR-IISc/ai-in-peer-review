This paper presents a novel framework for discovering computationally efficient mathematical identities. The core idea is to represent symbolic expressions using an attribute grammar and then search this space for an expression that is analytically equivalent to a target expression but has lower computational complexity. As a brute-force search is intractable, the main contribution is the application of machine learning techniques—specifically n-gram models and a recursive neural network (RNN)—to guide the search. These models are trained on solutions to simpler problems to learn promising patterns, enabling the discovery of complex identities that are beyond the reach of exhaustive search or manual derivation.

**Strengths**

1.  **Originality and Significance:** The application of machine learning to the symbolic domain of discovering mathematical identities is a creative and promising research direction. The paper successfully demonstrates that this approach can uncover new, non-trivial, and useful identities, some of which relate to expressions found in machine learning (e.g., RBM partition functions).
2.  **Novel Representation:** The development of a learned, continuous representation for symbolic expressions using an RNN is a significant contribution. This method has the potential to make a wide range of symbolic computation problems amenable to modern machine learning techniques, much like word embeddings have transformed natural language processing. The high classification accuracy reported in Table 2 provides strong evidence for the effectiveness of this representation.

**Weaknesses and Questions for Clarification**

1.  **Restricted Problem Scope:** The problem definition appears to be quite constrained, focusing on homogeneous polynomials of a single matrix variable with a scalar output. While the paper notes that the framework can handle multiple variables, the experiments are limited. It would be valuable for the authors to discuss the challenges and potential paths toward generalizing this framework to a broader class of mathematical expressions, such as those including non-polynomial operators (e.g., trigonometric, exponential), recursion, or different output structures. This would help clarify the potential long-term impact of the work.

2.  **Clarity of the RNN Architecture:** The description of the recursive neural network in Section 4.2 could be improved. The text mentions a "weight tensor W3 (of size lxlxl)" for operations with two inputs.
    *   Does this imply the use of a neural tensor network layer for combining two vector representations? If so, this is a critical architectural detail that should be stated explicitly.
    *   It would be helpful to motivate this architectural choice. For instance, related work by Bowman et al. [3] suggests that tensor networks can be more effective than standard RNNs for certain logical reasoning tasks. A brief discussion of why this architecture was chosen over simpler alternatives (e.g., concatenation followed by a linear layer) would strengthen the paper.

3.  **Clarity of the Search and Proposal Generation:** The manuscript would benefit from a clearer explanation of how the search is conducted, particularly regarding the generation of proposals.
    *   Section 6 describes a scheduler that receives scores from a strategy to select the next grammar rule. However, it is not entirely clear how the RNN, which is trained as a classifier in Section 4.2, is repurposed to function as a scoring strategy in Section 7.2. Is the softmax layer retrained to predict the next rule? A more detailed walk-through of how a partial expression tree is evaluated and extended by the RNN-based strategy would greatly improve the reader's understanding.

This paper presents an interesting approach to discovering efficient algebraic identities, demonstrating the successful application of distributed, recursive neural representations for their classification.

However, the problem definition feels somewhat limited in scope. A key point of confusion arises with the description of the weight tensor *W3* (of size *l x l x l*, resulting in an output of size *l*). Is this intended to implement a neural tensor network rather than a standard linear layer? The authors should clarify this, as Bowman’s prior work demonstrated the superior performance of tensor networks over recurrent neural networks (RNNs) for similar tasks.

Section 7.2, discussing efficient identity discovery, lacks clarity regarding whether an RNN is still utilized for classification while evaluating different search strategies. The proposal generation process within the search procedure is particularly unclear and would benefit from a more detailed explanation. How are candidate identities generated and evaluated?

Several minor errors detract from the paper’s polish. These include: “has contains a single matrix,” “listed in table Table 1,” “vector is the presented to the next operation,” “The final vector is pass to a softmax classifier,” “as to how explore space of possible,” and “The scheduler is is fixed.” Additionally, the difficulty of the task could be better conveyed by discussing [mention specific aspect], which is easily captured with [mention relevant technique/example]. Finally, Figure 3 incorrectly labels the network as a Tensor Neural Network (TNN) when it should be an RNN.

This paper introduces an interesting solution to finding more efficient algebraic identities. It is particularly noteworthy that distributed, recursive neural representations can successfully classify these identities, opening avenues for applying machine learning techniques to symbolic manipulation. 

The core idea is to represent symbolic expressions as trees built from an attribute grammar, and then leverage machine learning – specifically n-gram models and recursive neural networks (RNNs) – to guide the search for more computationally efficient equivalent expressions. The authors demonstrate the effectiveness of this approach by discovering identities for several families of expressions, some of which appear to be novel.

This work builds on and relates to several areas. The use of attribute grammars ([14]) for representing symbolic expressions is a solid foundation, drawing connections to compiler construction and formal specification. The paper correctly positions itself relative to theorem proving [5, 9, 11], program induction [18, 28], and probabilistic programming [12, 20], while appropriately limiting the scope to polynomials of fixed degree, avoiding undecidability issues. The incorporation of techniques from natural language processing, such as n-gram models and recursive neural networks [2, 3, 8, 15, 16, 22, 23], is a novel and promising direction. 

**Strengths:**

*   **Novelty:** The application of machine learning, particularly RNNs, to the discovery of mathematical identities is a novel contribution. The discovered identities themselves, some of which are reported to be previously unknown, add to the originality of the work.
*   **Technical Soundness:** The framework based on attribute grammars appears well-defined and the experimental setup seems reasonable. The use of integer-based descriptors to avoid floating-point precision issues is a thoughtful detail.
*   **Clear Problem Definition:** The paper clearly defines the problem of finding computationally efficient symbolic expressions and establishes a reasonable scope for exploration.
*   **Empirical Validation:** The experiments demonstrate that the learning-based search strategies outperform a random search, particularly for more complex expressions.
*   **Code Availability:** Providing the code and evaluation data ([29]) is commendable and facilitates reproducibility.

**Weaknesses:**

*   **Restricted Problem Definition:** The problem definition is quite restricted, focusing on homogeneous polynomials with a single matrix variable. While understandable as a starting point, it limits the broader applicability of the approach. The discussion acknowledges this, but a more thorough exploration of potential generalizations would be valuable.
*   **RNN Implementation Detail:** The description of the RNN's weight tensor (of size lxlxl) is unclear. Is this a tensor network rather than a simple linear network?  Given the reference to Bowman's work [3], which demonstrates the benefits of tensor networks for similar tasks, clarifying this point is important.
*   **Search Strategy Explanation:** The explanation of how proposals are generated during the search process could be more detailed. It’s not immediately clear how the scores from the learning strategies are translated into concrete actions for exploring the grammar tree.
*   **Limited Comparison:** While the paper compares against a random search, it would be valuable to compare against existing symbolic computation systems like Maple [6] or Mathematica [27], even if these systems are rule-based and don't use learning. Demonstrating performance advantages over these established tools would strengthen the contribution.
*   **Minor Typos/Clarity Issues:** Several minor typos and phrasing issues exist (e.g., "has contains," "listed in table Table 1," "vector is the presented," "pass to a softmax classifier," "as to how explore"). These detract slightly from the overall clarity. The repetition of "scheduler is is fixed" is also noteworthy.



**Arguments for Acceptance:**

*   The paper presents a novel and technically sound approach to a challenging problem.
*   The experimental results demonstrate the effectiveness of the proposed learning-based search strategies.
*   The code and data are publicly available, facilitating reproducibility.
*   The work has the potential to impact both the machine learning and symbolic computation communities.

**Arguments Against Acceptance:**

*   The restricted problem definition limits the broad applicability of the approach.
*   Some aspects of the implementation (e.g., the RNN architecture) and search strategy are not fully explained.
*   The lack of comparison to established symbolic computation systems weakens the claim of practical significance.




**Overall Assessment:**

This is a strong paper with a novel contribution. While the restricted problem definition and some clarity issues are concerns, the technical soundness, experimental validation, and potential impact of the work outweigh these weaknesses. I recommend **acceptance**, but with a request for the authors to address the points raised above regarding the RNN implementation, search strategy explanation, and comparison to existing tools in a revised version.

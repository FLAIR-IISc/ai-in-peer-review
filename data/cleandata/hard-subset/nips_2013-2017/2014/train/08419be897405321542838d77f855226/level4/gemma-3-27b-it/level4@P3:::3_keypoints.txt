- Introduces a method using attribute-grammar trees and learning (n-grams, RNNs) to find more efficient algebraic identities.
- Demonstrates distributed, recursive neural representations can classify and guide discovery of symbolic identities.
- Core idea: represent expressions as attribute-grammar trees and use ML to guide search for computationally cheaper equivalents.
- Empirically discovers several identities (some novel) across multiple expression families.
- Positions work relative to theorem proving, program induction, and probabilistic programming while limiting scope to fixed-degree polynomials.
- Novelty: applying ML, especially RNNs, to discover mathematical identities is original; discovered identities add value.
- Technical soundness: attribute-grammar framework and integer-based descriptors are well-motivated and carefully designed.
- Clear problem definition with a reasonable, though narrow, scope.
- Empirical validation: learning-based search outperforms random search, especially on complex expressions.
- Code and evaluation data are publicly available, aiding reproducibility.
- Restricted problem scope (homogeneous polynomials, single matrix variable) limits broad applicability and generalization.
- RNN implementation description is unclear (weight tensor l×l×l — unclear if tensor network or linear), needs clarification.
- Search strategy lacks detail: how model scores translate into concrete grammar-tree exploration is not fully explained.
- Limited baseline comparisons: should compare to established symbolic tools (e.g., Maple, Mathematica) to strengthen practical claims.
- Minor typos and phrasing issues reduce clarity in places.
- Arguments for acceptance: novel, technically sound, empirically effective, reproducible code, and potential cross-disciplinary impact.
- Arguments against acceptance: restricted problem definition, unclear implementation/search details, and missing comparisons to established systems.
- Overall assessment: strong paper recommended for acceptance pending revisions addressing RNN details, search explanation, and comparisons to existing tools.

- Proposes Coherent Recurrent Convolutional Network (CRCN) for retrieving sentence sequences describing image streams.
- Combines CNNs for images, BRNNs for sentence sequences, and a local coherence model to ensure fluency.
- Two-stage approach: retrieve candidate sentences per image, then select the best sequence balancing image-sentence compatibility and inter-sentence coherence.
- Uses large-scale, weakly-supervised blog-post dataset and promises to release code and data.
- Demonstrates strong quantitative results, user studies, and ablation analyses outperforming baselines.
- Incorrectly claims [5] has “no mechanism for coherence”; [5] uses semantic/topic representations that inherently promote coherence.
- Clarity issue: Figure 2b appears to apply FC layers in parallel to all o_t, while Eq.(2) implies shared per-sentence application—needs reconciliation.
- Evaluation concern: relying on METEOR with a single reference is suboptimal; more metrics or justification required.
- Model design question: using two sequential linear functions (W_f2, W_f1) is redundant since their composition is linear—ablation needed to justify.
- Model design question: unclear rationale for parameter sharing between BRNN outputs and the local coherence model; impact of separate parameters should be tested.
- Implementation detail missing: whether paragraph vectors are fine-tuned or kept fixed must be explicitly stated.
- Arguments for acceptance: strong results, novel architecture addressing sequential coherence, comprehensive experiments, code/dataset promise, coherence model helps.
- Arguments against acceptance: inaccurate related-work claim, clarity problems, suboptimal evaluation metric, unjustified design choices, unclear implementation detail.
- Post-rebuttal decision: recommend acceptance—authors acknowledged the issues, will clarify figures/equations, add ablation studies, confirmed paragraph vectors are fixed, and will release code/dataset.

1. Task & goal: Retrieve coherent sentences to describe a sequence of blog images by selecting relevant sentences from nearest-neighbor training examples.  
2. Method: For each query image retrieve K=5 image–sentence pairs, embed sentences, model the sentence sequence with a bidirectional LSTM, pass outputs through ReLU + fully connected layers and score image–sentence compatibility; add a local coherence model.  
3. Main contribution: A novel architecture to select coherent sentences for image sequences.  
4. Strength: Extensive quantitative, qualitative, and human evaluations showing superiority over baselines and an ablation removing the coherence model.  
5. Strength: Authors promise to release source code and dataset.  
6. Weakness/related-work issue: Claim that [5] has “no mechanism for coherence” is inaccurate — [5] models an intermediate semantic/topic representation that yields coherence.  
7. Clarity/correctness: Figure 2b appears inconsistent with Section 3.3 / Eq. (2) — diagram implies FC layers connected to all sentences but Eq. (2) suggests shared parameters applied per sentence.  
8. Evaluation metric: Use METEOR rather than BLEU (especially with a single reference) for better automatic evaluation.  
9. Model design question: Why are two linear functions (W_f2, W_f1) applied sequentially in Eq. (2) when their composition is linear — request ablation to justify.  
10. Parameter-sharing question: Why use the same fully connected layer parameters for BRNN outputs (o_t) and the local coherence model q in Eq. (2)?  
11. Implementation detail requested: Clarify whether the paragraph vector [16] is fine-tuned or kept fixed.  
12. Decision/post-rebuttal: Reviewer now recommends acceptance after rebuttal; asks authors to make promised final changes and explicitly clarify point 11.

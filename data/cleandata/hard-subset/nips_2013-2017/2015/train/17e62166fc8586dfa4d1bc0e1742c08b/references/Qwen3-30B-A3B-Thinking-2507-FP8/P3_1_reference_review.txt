

# Peer Review: "Expressing an Image Stream with a Sequence of Natural Sentences"

## Summary and Context
This paper introduces a novel task: generating a sequence of natural language sentences to describe a stream of images (e.g., a series of photos from a trip), rather than the conventional single-image single-sentence description. The authors propose CRCN, a multimodal architecture integrating CNNs for image representation, bidirectional RNNs for sentence modeling, and a local coherence model based on entity transitions to ensure fluent sentence sequences. They leverage blog posts as training data, where each blog naturally contains image-text sequences. The paper makes a compelling case for the task's relevance, noting that users often capture "memorable moments" as image streams rather than single images.

## Strengths
1. **Problem Definition**: The paper successfully identifies an important gap in the literature—most prior work focuses on single-image single-sentence generation, while real-world applications often involve sequences. The motivation with NYC/Disneyland examples is strong and well-justified.

2. **Methodological Integration**: The CRCN architecture thoughtfully combines three components (CNN, BRNN, coherence model) in a unified framework. The coherence model, which tracks entity transitions in parse trees, is particularly novel and addresses a critical challenge in multi-sentence generation: ensuring fluent transitions between sentences.

3. **Evaluation Rigor**: The evaluation is comprehensive, using both quantitative metrics (BLEU, CIDEr, METEOR, top-K recall) and user studies via AMT. The user study design is thoughtful, with careful attention to query length effects on coherence importance.

4. **Data Collection**: The blog-based dataset construction is well-justified and scalable. The text segmentation and summarization approach (using LSA) is appropriate for blog data, and the data augmentation strategy (using top-K sentences) is practical.

## Weaknesses and Concerns

1. **Task Definition Ambiguity**: The paper doesn't clearly distinguish between "image stream" and "video" description. While the authors note that consecutive images may have "sharp changes" unlike videos, the problem setup remains closer to video description (which typically has temporal continuity) than to static image streams. This blurs the novelty claim.

2. **Baseline Comparison**: The baselines are problematic. The paper compares against single-sentence generation models extended to sequence generation by concatenation, but this ignores recent work on multi-sentence generation (e.g., "Image Paragraph Generation" by Wang et al., CVPR 2018). The "simplest baseline" (GloMatch) is particularly weak, as it uses outdated image representations (GIST/Tiny images) rather than modern CNN features.

3. **Coherence Model Limitations**: The entity-based coherence model (inspired by Barzilay & Lapata) has significant limitations. It only considers grammatical roles (S, O, X, -) for entities, ignoring semantic relationships between entities across sentences. This is insufficient for capturing true coherence, as demonstrated in the user study where the coherence effect only becomes significant with longer sequences (8-10 images).

4. **Training Data Bias**: The blog datasets (NYC/Disneyland) are highly constrained to tourism topics. The paper claims their approach is "completely unsupervised and thus applicable to any domain," but provides no evidence of this. The evaluation lacks diversity in content types, potentially overestimating performance on domain-specific data.

5. **Technical Details**: The paper lacks crucial implementation details. For example:
   - How are the "K nearest images" selected for each query image? (The paper states "using l2-distance on VGGNet fc7 features," but doesn't specify if the images are normalized or if the features are fine-tuned.)
   - The coherence model's 64-dimensional transition descriptor is poorly justified—why 3 history steps? Why not consider semantic relationships beyond grammatical roles?

## Originality and Significance
The paper's main contribution is the task definition (image stream → sentence sequence), which is novel. However, the methodological contributions (combining existing components) are incremental. The coherence model is the most novel component, but its effectiveness is limited to longer sequences (as shown in the user study), which may not be the most common use case. The significance is modest: while the task is relevant for applications like travel blogging, the paper doesn't demonstrate a clear path to practical deployment.

## Recommendation
I recommend **weak acceptance** (4/10). The paper addresses a meaningful gap in the literature with a well-constructed evaluation, but the methodological contributions are incremental, and several key concerns (baseline comparisons, coherence model limitations) undermine the novelty claims. The paper would benefit significantly from:
1. More rigorous comparison with recent multi-sentence generation methods
2. A clearer distinction between image streams and video description
3. More detailed analysis of the coherence model's limitations
4. Evaluation on more diverse datasets beyond tourism

The paper demonstrates solid technical execution and makes a useful contribution to the broader field of multimodal generation, but it doesn't significantly advance the state of the art. With the suggested improvements, it would be a strong candidate for publication.

## Pro/Con Acceptance Summary
*Pro*: Strong motivation, comprehensive evaluation, appropriate dataset construction, thoughtful coherence model design.
*Con*: Problem definition overlaps with video description, weak baseline comparisons, coherence model has limited scope, insufficient evidence for domain generality.

Overall, this is a solid but incremental contribution that would benefit from more careful positioning within the literature and addressing the technical concerns noted above.

- Novel problem: proposes generating coherent sentence sequences for image streams, extending single-image captioning.  
- Proposed method: CRCN architecture combining CNN image features, BRNN language modeling, and a local coherence model.  
- Coherence model: uses parse trees and entity-grid style features to enforce smooth sentence transitions.  
- Datasets and data source: leverages NYC and Disneyland blog posts as weakly supervised image-text corpora.  
- Evaluation: comprehensive experiments using BLEU, Recall metrics and AMT user studies showing improvements over baselines.  
- Technical soundness: architecture is well-motivated and builds on established multimodal RNN approaches.  
- Reproducibility: implementation details provided, aiding reproducibility (not independently verified).  
- Clarity issues: some sections are dense, notation inconsistent, and BRNN/coherence explanations lack intuition.  
- Insufficient coherence analysis: lack of detailed ablation on the coherence model (history parameters, parsing choices, removal).  
- Limited baselines: comparisons mostly extend single-image captioning; stronger sequential baselines (e.g., adapted video models) are missing.  
- Limited qualitative results: too few examples and no failure cases shown.  
- Loss function justification: choice of max-margin loss is stated but not well motivated for this task.  
- Revision suggestions: improve clarity, run ablation studies on coherence, add stronger baselines, expand qualitative examples, justify loss choice, and fix notation.  
- Recommendation: accept with minor revisions, contingent on addressing clarity and deeper analysis of the coherence model.

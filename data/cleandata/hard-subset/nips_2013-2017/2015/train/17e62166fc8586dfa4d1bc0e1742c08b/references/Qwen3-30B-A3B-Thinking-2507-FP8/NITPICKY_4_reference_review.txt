

# Review of "Expressing an Image Stream with a Sequence of Natural Sentences"

## Summary
This paper proposes a novel approach for generating sentence sequences from image streams, introducing the Coherent Recurrent Convolutional Networks (CRCN) architecture. The authors extend the typical image captioning task to handle sequences of images and corresponding sequences of sentences, motivated by the common practice of tourists capturing multiple images during memorable experiences. The method integrates convolutional networks for image representation, bidirectional recurrent networks for sentence modeling, and a local coherence model to ensure smooth transitions between sentences. The authors evaluate their approach on blog datasets from NYC and Disneyland, demonstrating superior performance over baselines using both quantitative metrics and user studies.

## Strengths
The paper addresses an interesting extension to the image captioning problem that has been largely unexplored. The motivation for handling image streams rather than single images is reasonable, and the use of blog posts as a source of text-image parallel data is creative. The CRCN architecture integrates multiple components in a logical way, and the coherence model specifically targets an important aspect of multi-sentence generation that previous work has neglected. The empirical evaluation is thorough, including both quantitative metrics and user studies.

## Weaknesses

### Methodological Inconsistencies
1. The paper claims "to the best of our knowledge, this work is the first to address the problem," but fails to properly contrast with Kim et al. (2011) "Ranking and Retrieval of Image Sequences from Multiple Paragraph Queries" which deals with the related problem of retrieving image sequences from text. The authors do not explain why their problem is fundamentally different from this prior work.

2. The mathematical description of the coherence model contains a critical error: "We set the history parameter to three, which means we can obtain 4^3 = 64 transition descriptions." This is mathematically incorrect. With 4 possible grammatical roles (S, O, X, -) and a history of 3, the number of possible transitions should be 4^4 = 256, not 4^3 = 64.

3. The paper states: "We set the activation function f to the Rectified Linear Unit (ReLU), f(x) = max(0, x)." However, in equation (1), the authors use f for both the activation function and the ReLU function, creating confusion in notation.

4. The data augmentation section states: "we select top-κ highest ranked summary sentences, among which the top-ranked one becomes the summary sentence for the associated image, and all the top-κ ones are used for training." But it fails to explain how the "top-ranked" is determined or how the ranking is performed.

### Experimental Details
1. The dataset description claims "more than 20K blog posts with 140K associated images" but the actual numbers in Section 4.1 are 11,863 + 7,717 = 19,580 blog posts and 78,467 + 60,545 = 139,012 images, creating a discrepancy that should be resolved.

2. The paper claims "Our approach is completely unsupervised" but the training process relies on a text-image parallel corpus, making it at best semi-supervised. The authors should clarify this point.

3. The user studies limit queries to 5 images, stating "We first set the maximum number of images per query to 5. If a query is longer than that, we uniformly sample it to 5." This is a significant limitation that undermines the paper's claim about handling image streams, and the authors should address how their method would scale to longer streams.

### Technical Implementation
1. The paper states: "We use dropout regularization in all layers except the BRNN, with 0.7 dropout for the last FC layer and 0.5 for the other remaining layers." This is confusing because the BRNN is a model, not a layer. The authors should clarify which specific layers were excluded from dropout.

2. The paper cites VGGNet [25] as representing images as 4,096-dimensional vectors but fails to specify which variant of VGGNet (VGG16, VGG19, etc.) was used.

3. The coherence model section states: "We make this descriptor to a 300-dimensional vector by zero-padding." Zero-padding is not a standard or effective method for dimensionality conversion; a linear transformation or embedding layer would be more appropriate.

### Formatting and Consistency
1. The paper uses inconsistent notation: "pt" for paragraph vectors in some places and "p" as a matrix in others.

2. The paper refers to "Fig.1.(a)" in the text but uses "Fig.1" elsewhere, creating inconsistency.

3. The paper contains minor grammatical errors: "hike of interest" should be "surge of interest" or "increase in interest."

## Overall Assessment
While the paper addresses an interesting extension to image captioning with a potentially valuable contribution, the numerous methodological inconsistencies and technical inaccuracies undermine the paper's credibility. The mathematical error in the coherence model description is particularly concerning as it fundamentally affects the proposed method. The paper's claims about being "completely unsupervised" are also misleading and need to be corrected.

The technical contributions are not as novel as claimed, as the paper fails to properly distinguish itself from prior work on image sequence description. The evaluation methodology has significant limitations (e.g., restricting queries to 5 images), and the paper does not adequately address these limitations.

Although the paper presents a reasonable extension to image captioning, the numerous technical inaccuracies, inconsistent claims, and formatting issues are substantial enough to warrant rejection. The paper would benefit from a thorough revision to address these issues before reconsideration.

## Recommendation
Reject

The paper's technical inaccuracies and misleading claims outweigh its interesting motivation and reasonable experimental evaluation. The mathematical error in the coherence model description is particularly concerning and suggests the authors may not have thoroughly verified their core technical claims. The paper would need substantial revision to address these issues before it could be considered for publication.



The paper proposes a new method for time-sensitive recommendations based on user activities, addressing the problem of time-varying user preferences—a distinction from existing approaches. It also introduces a novel contribution: predicting the next user return time. Learning is performed using an efficient optimization algorithm proposed by the authors. The experimental section evaluates the method on one synthetic dataset (to demonstrate scalability on large data) and two modest-sized real datasets.

However, several critical concerns require clarification. First, the prevalence of time-varying user preferences must be substantiated. The paper claims this is a significant problem, but the real datasets used in experiments lack analysis demonstrating how preferences actually evolve (e.g., due to new popular songs, seasonal trends, or other patterns). Without this, the core motivation remains unconvincing.

Second, the motivation for predicting next return time is insufficiently justified. The paper claims this as a contribution but fails to explicitly explain *why* this prediction is valuable for real-world applications. How would a web company leverage this prediction? For instance, could it optimize push notifications, content curation, or user retention strategies? This gap undermines the contribution's relevance.

Third, the prediction task for exact return time appears overly ambitious. Simpler, more actionable baselines should be tested: (1) the last observed time delay, and (2) the most frequent time delay in the dataset. Instead of predicting exact timestamps, the authors should focus on a coarser-grained task (e.g., classifying delays into short [1–5 days], medium [5–14 days], long [14–33 days], or very long [>33 days])—a more realistic and impactful prediction target.

Regarding experiments: The performance gains over SVD (or other baselines) are not clearly quantified. Are the improvements groundbreaking (e.g., order-of-magnitude) or modest? Crucially, how would these recommendations feel different to end-users compared to simply recommending dynamically updated popular items (a strong baseline, especially when popularity is recalculated daily)? The paper fails to address this comparison.

Minor issues:  
1. Page 3, last paragraph: "and is abel to capture" → "and is able to capture."  
2. Section 2, first paragraph: The abrupt shift from general user activities to social network examples (retweets/link creation) without citation or context is confusing. Rephrase to maintain coherence.

Overall, the paper addresses two problems—time-sensitive recommendations and user return time prediction—but lacks compelling motivation for their relevance, omits key baselines, and fails to demonstrate significant practical impact. While the method is technically sound, the work is marginal due to these critical omissions. The contribution would be strengthened by rigorous analysis of real-data patterns, clear application motivation, and a more feasible prediction task with thorough baseline comparisons.

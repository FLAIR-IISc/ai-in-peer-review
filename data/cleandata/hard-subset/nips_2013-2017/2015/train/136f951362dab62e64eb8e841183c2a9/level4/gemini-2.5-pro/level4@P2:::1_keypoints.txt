- Summary: Proposes a time-sensitive recommendation model combining multivariate Hawkes processes with low-rank matrix factorization to predict next-item and user return time.  
- Summary: Learns parameters via a convex, nuclear-norm-regularized optimization and a scalable hybrid proximal/conditional-gradient algorithm with O(1/t) convergence.  
- Summary: Evaluated on one large synthetic and three real-world datasets, showing scalability and improved recommendation and return-time prediction versus several baselines.  
- Strength: Novelty in bridging Hawkes processes and low-rank collaborative filtering to model self-exciting, recurrent user behavior.  
- Strength: Solid technical contribution—convex formulation, custom hybrid optimization algorithm, and theoretical convergence analysis.  
- Strength: Demonstrated scalability and efficiency; synthetic experiments show linear scaling with data size.  
- Weakness: Motivation for modeling time-varying preferences is weak—no dataset analysis showing prevalence or nature of temporal dynamics.  
- Weakness: Practical utility of predicting precise return times is not well justified; unclear advantage over predicting time windows.  
- Weakness: Missing simple but strong baselines for return-time prediction (e.g., last inter-event time, user average inter-event time).  
- Weakness: Missing dynamic popularity baseline for time-sensitive recommendation (e.g., top items in recent time window).  
- Weakness: Results lack practical interpretability—need top-N precision/recall for recommendations and contextualized MAE or relative error for return-time predictions.  
- Weakness: Continuous return-time regression may be sensitive to noise/outliers; a classification into time bins could be more practical and robust.  
- Question 1: Please provide exploratory analysis of datasets to show user preferences are significantly time-varying.  
- Question 2: Please compare return-time prediction against simple baselines like last inter-event duration and user average inter-event duration.  
- Question 3: Please compare item recommendation performance to a dynamic popularity baseline (e.g., most-consumed items in the last 24 hours).  
- Question 4: Please report summary statistics (mean, median, std) of ground-truth inter-event durations to help interpret MAE results.  
- Question 5: Have you considered framing return-time prediction as classification into discrete time bins, and what are the trade-offs versus continuous regression?

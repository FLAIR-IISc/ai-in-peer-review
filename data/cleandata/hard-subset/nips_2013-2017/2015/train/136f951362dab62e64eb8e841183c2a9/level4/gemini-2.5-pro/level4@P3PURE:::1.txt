This paper proposes a novel method for time-sensitive recommendations based on user activities. The model's key distinction is its ability to address time-varying user preferences, and it also introduces a component for predicting a user's next return time. The authors propose an efficient optimization algorithm for learning and provide an experimental evaluation on one large-scale synthetic dataset and two real-world datasets of modest size.

While the proposed method is interesting, the paper has several weaknesses in its motivation and experimental evaluation that should be addressed.

**Major Comments:**

1.  **Motivation for Time-Varying Preferences:** The paper's core premise is that modeling time-varying user preferences is critical. However, this claim would be much stronger if it were supported by an analysis of the real-world datasets used in the experiments. What is the evidence that user preferences are indeed varying significantly? If they are, what is the nature of this variation? Is it primarily driven by global trends, such as new popular items appearing, or are there more complex, user-specific temporal patterns at play?

2.  **Motivation for Predicting Return Time:** The prediction of a user's next return time is presented as a key contribution, but the paper does not sufficiently motivate why this is an important problem to solve. The authors should provide concrete use cases. How, for instance, could an online service leverage this prediction? Clarifying the practical impact of this task would significantly strengthen the paper's contribution.

3.  **Missing Baselines for Return Time Prediction:** The evaluation of the return time prediction task is missing several simple but important baselines. For example, the model's performance should be compared against naive predictors such as: 1) using the user's last observed time delay between visits, or 2) using the user's most frequently observed time delay.

4.  **Task Formulation for Return Time Prediction:** Predicting the exact return time of a user is an extremely challenging regression task. A more practical and potentially more tractable alternative would be to frame this as a classification problem. For example, predicting whether a user will return within a short (e.g., 1-5 days), medium (5-14 days), or long (>14 days) time window could be more robust and still highly valuable for practical applications.

5.  **Contextualizing Experimental Results:** It is difficult to assess the practical significance of the reported performance lift compared to the SVD baseline. Is the improvement groundbreaking (e.g., an order of magnitude) or more modest? Would an end-user actually notice the difference in recommendation quality? Furthermore, the experiments should include a comparison to a strong, time-aware "most popular" baseline, where popular items are recalculated daily. This is often a difficult baseline to outperform in practice and is essential for contextualizing the results.

**Minor Comments:**

1.  There is a typo in the last paragraph on page 3: "and is abel to capture" should be "and is able to capture".
2.  The first paragraph of Section 2 abruptly mentions retweets and link creation in social networks without providing references or a clear connection to the paper's main topic. This change of context is confusing and should be rephrased for better clarity and flow.

In summary, the paper tackles two potentially important problems in time-sensitive recommendations. However, the work is currently weakened by a lack of clear motivation for the problems it aims to solve and an experimental evaluation that is missing key baselines and analyses. Addressing these points would be necessary to properly assess the impact and significance of the proposed contributions.

This paper proposes a model for time-sensitive recommendation by framing the problem through the lens of temporal point processes. The main idea is to model the sequence of a user's interactions with an item as a self-exciting Hawkes process. To generalize across all user-item pairs and enable recommendations for unobserved pairs, the model's parameters—the baseline intensity and the self-excitation magnitude—are represented as two matrices which are assumed to be low-rank. This leads to a convex optimization problem with nuclear norm regularization and non-negativity constraints. The authors develop a novel optimization algorithm that blends proximal gradient and conditional gradient methods to solve this problem efficiently. The proposed "Hawkes-Recommender" is evaluated on two tasks: item recommendation at a specific time and prediction of the user's next return time. This work relates to previous efforts in temporal collaborative filtering, but distinguishes itself from models that discretize time (e.g., tensor factorization) or that cannot generalize to new items (e.g., Kapoor et al. [11, 12]). The core technical contribution lies in the novel combination of low-rank models and multivariate Hawkes processes for recommendation, and the accompanying specialized optimization algorithm.

**Strengths:**

1.  **Novel Formulation:** The connection established between low-rank models and self-exciting point processes for recommendation is elegant and well-motivated. It provides a principled way to model continuous-time user activities, capturing both long-term static preferences (baseline intensity) and short-term, history-dependent effects (self-excitation).
2.  **Unified Framework:** The model naturally addresses two important practical problems—recommending the right item at the right time and predicting user return times—within a single coherent framework.
3.  **Optimization Contribution:** The proposed algorithm for solving the non-negative nuclear norm regularized problem appears to be a non-trivial contribution. The analysis showing an optimal convergence rate, which combines the best of both proximal and conditional gradient methods, could be of independent interest to the optimization community.
4.  **Empirical Evaluation:** The paper presents a thorough empirical evaluation on both synthetic and large-scale real-world datasets, demonstrating consistent improvements over several relevant and non-trivial baselines.

**Weaknesses and Concerns:**

Despite the promising core ideas, the paper suffers from a significant number of issues related to clarity, methodological detail, and experimental rigor.

1.  **Clarity and Presentation:** The manuscript lacks the level of polish expected for a top-tier publication.
    *   There are several typographical errors (e.g., "nosiness" instead of "noisiness" in the Introduction; "Proceeedings" in ref [13]; "Acknowledge" instead of "Acknowledgments").
    *   The notation is frequently inconsistent or non-standard. The transpose operator is denoted by `>` (e.g., `w>u,i`), which is highly unconventional. The symbol `T` is overloaded, representing both the set of event times and the end of the observation window (e.g., in the integral in Eq. 1 and in the definition of `ψu,i`). The inequality `‖Λ0‖∗ 6 λ′` in Section 3.2 uses the number `6` instead of the symbol `≤`. Citation formatting is inconsistent (e.g., ref [28] has trailing commas).
    *   The algorithm descriptions are too high-level. Algorithm 1 uses `∇1(f(Uk−1))` and `∇2(f(Uk−1))` without defining what these partial gradients with respect to blocks `X1` and `X2` are. Algorithm 3 confusingly re-uses the variable `Xk2 [Z1]` for both the rank-one matrix and its scaled version. These ambiguities hinder understanding and reproducibility.

2.  **Methodological Gaps:**
    *   **Theoretical Guarantees:** Theorem 1 claims equivalence between the original and penalized formulations for a sufficiently large `ρ > ρ*`. However, `ρ*` depends on the unknown optimal solution, rendering this condition impractical to verify. The authors concede that `ρ` must be tuned, which weakens the practical value of the theorem and makes the connection between the solved problem (5) and the target problem (4) tenuous.
    *   **Optimization Details:** The algorithm's step size `ηk` for the proximal step depends on the Lipschitz constant `L`. Computing or even tightly bounding `L` can be difficult, yet the paper provides no discussion of how this is handled in practice. This is a critical missing detail for implementation.
    *   **Model Simplifications:** The paper defaults to a simple exponential triggering kernel with a single bandwidth parameter `σ`. It is unclear if `σ` is a global parameter or specific to user-item pairs. A single global `σ` is a very strong assumption. The paper states `σ` is "fixed to one" for synthetic data but "tuned to give the best performance" for real data, an inconsistency that lacks detail on the tuning procedure.

3.  **Evaluation Protocol:**
    *   **Recommendation for New Items:** A crucial aspect of the model's behavior is not adequately discussed. For a user `u` and an item `i` with no prior interaction history, the self-exciting term vanishes, and the intensity becomes `λ_u,i(t) = Λ0(u, i)`. In this common scenario, the recommendation is static and no different from standard matrix factorization. The "time-sensitive" aspect of the model only applies to items the user has already engaged with. This is a major limitation that is not acknowledged.
    *   **Held-out Sequences:** The procedure for predicting the returning time for an entirely held-out user-item sequence is unclear. With an empty history `T_u,i = {}`, how is the conditional density `f(t|T_u,i)` defined and sampled from? The model as presented seems ill-equipped for this task, which casts doubt on the validity of the reported MAE for the "Heldout" group.
    *   **Scalability:** The item recommendation task requires computing and sorting the intensity `λ_u,i(t)` for *all* items `i` at a given time `t`. The computational cost of this step is not discussed, but it would be prohibitive for any reasonably large item catalog, limiting the model's practical applicability.

**Decision:**

I recommend **rejection**. While the paper is built upon a strong and promising core idea, the execution is flawed. The large number of minor presentation issues, coupled with more substantial methodological and evaluative ambiguities, prevents me from confidently endorsing the paper's claims. The unaddressed limitation regarding static predictions for new items and the lack of clarity in the evaluation protocol are particularly concerning. The work has potential, but it requires a major revision to address these shortcomings and meet the high standards of NIPS.

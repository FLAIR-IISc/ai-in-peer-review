Other comments:
- I find something missing in the setup for “batch learning from logged bandit feedback” in Section 3 (and possibly in [1]). The feedback should be treated as a random variable, rather than a deterministic function as described on line 110. Rationale: suppose the partial feedback arises from a classification problem where labels are unknown, and we observe loss only after making a prediction. Let there be a true classification loss Δ(y′, y). For a given input x, we predict y using h0; a “god” agent also assigns a label y′ and then reveals the incurred loss Δ(y′, y). Thus delta(x, y) is Δ(y′, y) for the y′ associated with this instance. In many classification settings, labeling can be noisy, so there may not be a unique y′ for each x. Consequently, elsewhere in the log we might see the same x and the same chosen y, but a different delta(x, y), because a different y′ was realized. Hence, in general, delta(x, y) should be modeled as a random variable. In the above example, for fixed x and played y, its mean would be E_{y′∼p(y′|x)}[Δ(y′, y)], where p(y′|x) is the true (noisy) labeling distribution. This likely does not change the risk estimator in (2), but the definition of the true risk in (1) should include an additional expectation over the randomness of delta to be fully general.

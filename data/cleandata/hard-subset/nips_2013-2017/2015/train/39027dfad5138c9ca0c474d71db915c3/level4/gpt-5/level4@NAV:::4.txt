== Other comments ==
- I believe something is missing in the setup for “batch learning from logged bandit feedback” in Section 3 (or possibly in [1]): the feedback should be modeled as a random variable rather than a deterministic function, contrary to the presentation on line 110. My reasoning is as follows. Consider a classification setting where labels are unknown and we only observe loss feedback after making a prediction. Suppose there is a true classification loss Delta(y', y). For input x, we predict y using h_0; a god-like annotator also assigns a label y', and reports the loss Delta(y', y) that we incur (so here delta(x, y) equals Delta(y', y) for the particular y' provided for this example). In many classification problems, labels can be noisy, so there may not be a unique y' for each x. This implies that elsewhere in the log, the same x with the same played y could yield a different delta(x, y) because the annotator provided a different label. Therefore, in general, delta(x, y) should be treated as a random variable (in this example, for fixed x and played y, its mean would be E_{y'~p(y'|x)}[Delta(y', y)], where p(y'|x) is the true noisy labeling distribution).
This likely does not affect the risk estimator (2), but the definition of the true risk in (1) should include an additional expectation over the randomness of delta to be fully general.

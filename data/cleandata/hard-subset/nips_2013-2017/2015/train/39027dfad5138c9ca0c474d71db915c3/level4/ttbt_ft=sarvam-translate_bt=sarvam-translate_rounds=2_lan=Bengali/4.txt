== Other Comments ==
- In the setup of "Batch Learning from Logged Bandit Feedback" in Section 3 (or from [1]), I can't find something: I think feedback should be seen as a random variable instead of an unspecified function, as presented in line 110. Here's my reasoning. Suppose the partial feedback comes from a classification problem where we don't know the labels; we only make a prediction and get feedback based on the amount of loss. Suppose there's a true classification loss delta (y',y) and the method of generating the feedback is, for a given input x, we make a prediction y using h_0; an agent labels the input as y'; and then our loss amount delta (y',y) tells us (so here delta (x,y) is delta (y',y) for y' in this example). On the other hand, in classification, we often assume that labeling can be random and therefore a unique y' may not be assigned to each x. This means that elsewhere in the log, we may have the same x as our input, the same y that we used, but another delta (x,y) because the agent gave a different label for this input... For this reason, I think delta (x,y) should generally be seen as a random number (in the classification example above, y' should be expected according to the average p (y'|x) of this random variable for a given x and used y, which is the actual random labeling distribution).

This probably doesn't change anything about the risk estimator (2); however, the setup of (1) should at least present an additional expectation on the randomness of the delta along with the actual risk, so that it becomes more general.

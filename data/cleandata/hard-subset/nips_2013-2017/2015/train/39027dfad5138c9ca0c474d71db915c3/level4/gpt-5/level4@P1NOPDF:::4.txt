== Other comments ==
- I believe something is missing in the setup for “batch learning from logged bandit feedback” in Section 3 (or in [1]): the feedback should be modeled as a random variable, rather than a deterministic function as presented on line 110. Rationale: consider a classification setting where we do not observe labels directly; instead, we obtain feedback by making a prediction and observing the loss. Suppose there is a true classification loss Δ(y′, y), and feedback is generated as follows: for a given input x, we make a prediction y using h_0; a “god” agent also assigns a label y′; and we are told the loss Δ(y′, y) we incur (so here delta(x, y) = Δ(y′, y) for the y′ associated with this example). In many classification problems, labeling can be noisy, so there is not necessarily a unique y′ for each x. This implies that elsewhere in the log, we could see the same x and the same played y but with a different delta(x, y), because the god agent provided a different label. Therefore, in general, delta(x, y) should be treated as a random variable. In the example above, for fixed x and played y, its mean would be E_{y′ ∼ p(y′|x)}[Δ(y′, y)], where p(y′|x) is the true noisy labeling distribution.
This may not change anything about the risk estimator in Eq. (2), but the true risk in Eq. (1) should be written with an additional expectation over the randomness of delta to make the setup fully general.

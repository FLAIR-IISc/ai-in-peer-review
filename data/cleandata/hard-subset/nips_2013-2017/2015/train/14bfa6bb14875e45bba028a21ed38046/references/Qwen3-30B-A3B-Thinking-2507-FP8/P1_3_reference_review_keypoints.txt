- Introduces Region Proposal Networks (RPNs) that integrate proposal generation into the deep learning pipeline by sharing convolutional features with the detection network.  
- RPNs remove the separate proposal computation bottleneck, making proposals nearly cost-free and enabling real-time detection (5–17 fps).  
- Demonstrates state-of-the-art accuracy (73.2% mAP on PASCAL VOC 2007) while significantly reducing inference time compared to Selective Search.  
- Technical contributions include translation-invariant anchors, fully convolutional RPN design, and an alternating optimization scheme to unify RPN and Fast R-CNN.  
- Thorough ablation studies show each component (RPN, shared features, anchors) meaningfully contributes to performance.  
- Experimental evaluation is comprehensive, comparing against strong baselines (Selective Search, EdgeBoxes) on standard benchmarks with clear metrics.  
- Paper is very well-written and organized, with clear technical exposition, useful figures, and sufficient implementation detail for reproducibility.  
- Novelty lies in integrating proposal generation into the detection network and the innovative anchor design addressing limitations of prior work (e.g., MultiBox).  
- Practical significance is high: simplifies the detection pipeline and enables real-time, high-accuracy object detection suitable for real-world applications.  
- Limited discussion of potential limitations, such as performance on extreme object scales/aspect ratios or very small objects relative to fixed anchor configurations.  
- Could better contextualize and compare against very recent related work (e.g., DeepMask, SharpMask) to clarify differences and relative strengths.  
- Lacks detailed computational complexity analysis on how RPN cost scales with different architectures or input sizes beyond runtime measurements.  
- Overall recommendation: accept — substantial, well-executed contribution with minor issues that do not undermine the paper’s impact.

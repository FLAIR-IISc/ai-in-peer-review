- Proposes a two-stage detector using an RPN on top-layer CNN features to replace traditional sliding-window proposals.
- Shares convolutional layers between the RPN and Fast R-CNN to enable near real-time detection (≈5 fps) and GPU acceleration.
- Integrates region proposal generation into the deep learning pipeline for end-to-end training and shared computation.
- Reports state-of-the-art results on PASCAL VOC 2007 (73.2% mAP) and 2012 (70.4% mAP).
- Dramatically reduces region proposal time versus CPU-based methods like Selective Search and EdgeBoxes.
- Alternating optimization for joint training is presented as a benefit but lacks detail on iteration count and convergence behavior.
- Sliding-window application on top-layer features raises concerns about coarse effective step size and potential localization errors.
- Missing definition of “ha” in the regression loss (lines 173–174) is a typographical issue that impedes understanding.
- Limited analysis of sensitivity to the number and configuration of anchors (choice of 9 anchors not thoroughly explored).
- Conceptually elegant solution that efficiently shares features and is clearly written and presented.
- Highly original contribution: learning region proposals within the network and sharing features with the detector.
- Significant potential impact for applications requiring real-time detection (robotics, autonomous driving, surveillance); code release aids reproducibility.
- Arguments for acceptance: addresses a key bottleneck, achieves speed and accuracy gains, introduces a novel RPN, and provides code.
- Arguments for rejection: insufficient discussion of optimization convergence, step-size/localization concerns, and a minor typographical error.
- Overall assessment: strong paper with important contributions; recommended for acceptance pending fixes to optimization detail and step-size/localization discussion.

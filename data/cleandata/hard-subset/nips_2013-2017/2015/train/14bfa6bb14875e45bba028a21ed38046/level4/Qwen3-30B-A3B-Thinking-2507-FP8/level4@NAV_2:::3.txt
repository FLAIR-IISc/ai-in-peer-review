

This work presents a natural extension to existing CNN-based object detection methods by introducing Region Proposal Networks (RPNs) that share convolutional features with the detection network, enabling efficient object detection at near real-time speeds. The authors propose a two-stage detection system where RPNs generate region proposals on top of shared convolutional features, followed by Fast R-CNN for object detection. The shared convolutional computation results in a 5fps frame rate on a GPU while achieving state-of-the-art detection accuracy on PASCAL VOC benchmarks. While the paper makes a valuable contribution to the field, I have the following concerns:

1. The optimization procedure described in Section 3.3 appears to suggest a single iteration of alternating optimization between RPN and Fast R-CNN. However, the detailed 4-step training algorithm presented in Section 3.4 indicates multiple iterations of refinement. This discrepancy requires clarification, as a single iteration would not allow for the iterative improvement of region proposals that the method relies upon.

2. The relationship between the sliding window step size on the top-layer feature map and the corresponding step size on the original image is not clearly explained. Given the downsampling effect of multiple pooling layers in the CNN architecture (particularly for VGG-16 with 13 convolutional layers), the top-layer feature map is significantly smaller than the input image. The paper should clarify how the spatial step size on the feature map translates to the image domain, as this is critical for understanding the precision of bounding box predictions.

3. The parameter $h_a$ in equations (1) and (2) (lines 173 and 174) is not defined. The paper should explicitly define this variable to ensure the mathematical formulation is complete and understandable.

The method represents a significant advancement in object detection by making region proposal computation nearly cost-free through shared convolutional features. The experimental results demonstrate strong performance with 73.2% mAP on PASCAL VOC 2007 and 70.4% mAP on 2012, while achieving a practical frame rate of 5fps. The work is technically sound, well-motivated, and makes an important contribution to the field of object detection.



This paper introduces a Region Proposal Network (RPN) that shares convolutional features with the detection network, enabling nearly cost-free region proposals for object detection. The authors build upon previous work like SPPnet and Fast R-CNN, which reduced detection network computation time but exposed region proposal as the new bottleneck. The RPN is a fully-convolutional network that predicts object bounds and objectness scores at each position, trained end-to-end with the detection network. This approach achieves state-of-the-art detection accuracy on PASCAL VOC 2007 (73.2% mAP) and 2012 (70.4% mAP) while running at 5fps on a GPU with VGG-16, significantly outperforming previous methods like Selective Search (2s/image) and EdgeBoxes (0.2s/image).

The work makes several significant contributions. First, it demonstrates that region proposals can be generated using a deep network that shares convolutional features with the detection network, eliminating the computational bottleneck. Second, the authors propose a simple alternating optimization scheme to train the unified network without requiring joint optimization. Third, the RPN's translation-invariant anchor design with k=9 proposals per location (vs. MultiBox's 800 anchors) substantially reduces model complexity while maintaining proposal quality. The method achieves a 10ms effective proposal time, compared to Selective Search's 1-2s, enabling real-time detection with high accuracy.

The paper is technically sound, with thorough experiments on PASCAL VOC 2007 and 2012 benchmarks. The authors provide detailed ablation studies showing how each component (cls layer, reg layer, shared features) contributes to the final performance. The training methodology is well-justified, with careful attention to handling cross-boundary anchors and proposal redundancy through non-maximum suppression. The implementation details are clear, and the code availability is a significant advantage for reproducibility.

The work demonstrates strong originality by rethinking region proposal as a deep learning problem rather than an engineering optimization. While MultiBox and OverFeat have explored similar ideas, the authors clearly articulate how their approach differs: the translation-invariant anchors, shared features with detection network, and simple alternating training scheme. The paper properly references relevant work from previous NIPS conferences, including the SPPnet and Fast R-CNN papers, and positions its contributions within this lineage.

The significance of this work is substantial, as it enables practical real-time object detection systems with state-of-the-art accuracy. The paper demonstrates that the region proposal step can be made nearly cost-free through feature sharing, which is a major step toward practical deployment of deep learning object detectors. The 5fps speed with high accuracy (73.2% mAP) represents a significant improvement over previous approaches, making this work highly impactful for both research and industry applications.

Strengths:
- Clear, well-motivated problem statement addressing a genuine bottleneck in object detection
- Effective solution through feature sharing and RPN design
- Thorough experimental evaluation with detailed ablation studies
- Strong results (73.2% mAP on PASCAL VOC 2007) with 5fps speed
- Practical implementation with code release
- Clear explanation of how the method differs from prior work

Weaknesses:
The draft review raises several valid concerns that need addressing:

1. The alternating optimization appears to involve only one iteration between RPN and Fast R-CNN training, which may not be sufficient for optimal convergence. The authors should clarify whether multiple iterations were performed or provide justification for the single-iteration approach. In practice, the authors likely performed multiple iterations as standard practice in deep learning, but this should be explicitly stated.

2. The paper does not sufficiently address the step size of the sliding window on the top CNN layer relative to the original image. Given that the top layer features have undergone multiple pooling operations, the effective step size on the image domain may be large (e.g., 16-32 pixels), potentially leading to inaccurate bounding box localization. The authors should provide more detail about how they handled this issue, including the effective receptive field on the input image.

3. The definition of $h_a$ in equations 173-174 is unclear. The paper refers to "the height of the anchor box" but doesn't specify which anchor box (e.g., the anchor at a particular location, or a reference anchor). This ambiguity should be resolved for clarity.

The paper could also benefit from a more detailed discussion of the trade-offs between proposal quality and detection accuracy. While the authors show that RPN with 300 proposals achieves excellent results, it would be valuable to see how the performance degrades with fewer proposals (e.g., 100 proposals) to understand the practical limits.

Overall, this is a strong paper that makes a significant contribution to the field of object detection. The technical approach is sound, the results are compelling, and the implementation is practical. The concerns raised are minor and can be addressed with clarifications in the final version. The work clearly advances the state of the art in object detection, meeting all four NIPS criteria: it is high-quality, clear, original, and significant. I recommend acceptance with minor revisions to address the concerns outlined above.

This work is a natural extension of the existing CNN-based object detection algorithm. My concerns are as follows:

1.  In the optimization section, it appears that there is only one iteration for the periodic optimization between the RPN and R-CNN. If this is the case, it assumes that the proposals generated by the first iteration RPN are optimal, which is clearly not true.

2.  When the pixel-based sliding window is applied to the upper layer of the CNN, what is the size of the step in the image layer? Since there are multiple rounds of max-pooling in the CNN, the upper layer is at a much smaller scale than the input image. The sliding window of the upper layer may have a large step in the image layer, which could result in an incorrect bounding box location.

3.  What is the definition of $h_a$ in lines 173 and 174?

This work proposes a two-stage object detection algorithm based on a convolutional neural network (CNN). The first stage is region proposal, which is based on the traditional sliding window method but works on the feature map of the upper layer of the CNN (RPN). In the second stage, a fast R-CNN is applied to the proposed regions. Since the convolution layers are shared between the RPN and R-CNN, and the computation is accelerated using a GPU, the algorithm can achieve near real-time (5fps).

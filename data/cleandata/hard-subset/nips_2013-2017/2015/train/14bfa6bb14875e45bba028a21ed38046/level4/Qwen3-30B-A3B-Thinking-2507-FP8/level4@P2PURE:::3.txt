

This work presents a logical extension to CNN-based object detection. My primary concerns are as follows.  
1. The optimization section describes a single iteration for the alternating optimization between RPN and R-CNN. This implicitly assumes that the initial RPN proposals are optimal, which is unwarranted.  
2. When applying the pixel-wise sliding window to the top-layer CNN feature map, the effective step size at the original image scale is unclear. Given that CNNs undergo multiple max-pooling operations, the top-layer feature map operates at a significantly reduced scale relative to the input image. A large step size at the feature map level would translate to coarse bounding box localization on the original image, potentially compromising accuracy.  
3. The definition of $h_a$ in lines 173–174 is not provided in the manuscript.  

The proposed method is a two-stage object detection algorithm using CNNs: the first stage employs a region proposal network (RPN) operating on the top-layer feature map (replacing traditional sliding windows), and the second stage applies fast R-CNN to the generated proposals. By sharing convolutional layers between RPN and R-CNN and leveraging GPU acceleration, the approach achieves near real-time performance (5 fps).

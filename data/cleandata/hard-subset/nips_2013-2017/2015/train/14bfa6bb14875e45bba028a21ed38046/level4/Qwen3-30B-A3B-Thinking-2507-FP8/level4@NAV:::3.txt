

This work represents a natural extension to existing CNN-based object detection methods. My concerns are as follows:  
1. The optimization section appears to employ only a single iteration for the alternating optimization between RPN and R-CNN. If this is correct, it assumes the proposals from the initial RPN iteration are optimal, which is unlikely to hold in practice.  
2. When applying the pixel-wise sliding window to the top CNN feature layer, the step size on the original image scale is unclear. Given the significant spatial reduction from multiple max-pooling operations, a small step on the feature map corresponds to a large stride on the input image, potentially leading to inaccurate bounding box localization.  
3. The definition of $h_a$ in lines 173 and 174 is not provided.  

The paper proposes a two-stage object detection algorithm using CNNs. The first stage generates region proposals via a sliding window applied to the top CNN feature map (RPN), while the second stage employs Fast R-CNN on these proposals. By sharing convolutional layers between RPN and Fast R-CNN and leveraging GPU acceleration, the method achieves near real-time performance (5 fps).

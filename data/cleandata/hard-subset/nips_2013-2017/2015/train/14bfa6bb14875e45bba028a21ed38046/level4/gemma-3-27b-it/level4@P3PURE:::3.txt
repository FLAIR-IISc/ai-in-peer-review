This work proposes a two-stage object detection algorithm based on convolutional neural networks (CNNs). The first stage generates region proposals using a region proposal network (RPN) – a sliding window method applied to the top layer feature map of a CNN. The second stage applies a Fast R-CNN to these proposed regions. By sharing convolutional layers between the RPN and the Fast R-CNN and leveraging GPU acceleration, the algorithm achieves near real-time performance (5fps).

My concerns with this work are as follows:

1. The optimization procedure, as described, appears to involve only a single iteration of alternating optimization between the RPN and the Fast R-CNN. If this is the case, the approach assumes that the proposals generated by the first RPN iteration are optimal, which is unlikely to be true.

2. When applying the pixel-wise sliding window to the top layer of the CNN, the effective step size on the input image is unclear. Given the multiple max-pooling layers within the CNN, the top layer operates at a significantly reduced scale compared to the original image. This potentially results in a large step size when projected back to the image level, which could lead to inaccurate bounding box localization.

3. The definition of $h_a$ in lines 173 and 174 is not provided and requires clarification.

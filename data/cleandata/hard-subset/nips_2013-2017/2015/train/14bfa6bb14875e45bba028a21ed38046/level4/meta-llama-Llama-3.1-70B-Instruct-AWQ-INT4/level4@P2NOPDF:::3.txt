This work is a natural extension to existing CNN-based object detection methods. However, I have several concerns that need to be addressed.

Firstly, the optimization section appears to employ only one iteration of alternating optimization between the Region Proposal Network (RPN) and the Region-based Convolutional Neural Network (R-CNN). If this is indeed the case, it assumes that the proposals generated by the first iteration of RPN are optimal, which is not necessarily true. This raises questions about the convergence and stability of the proposed algorithm.

Secondly, when applying the pixel-wise sliding window to the top layer of the CNN, it is unclear what the step size is on the image layer. Due to the multiple rounds of max-pooling in the CNN, the top-level feature map is significantly downsampled compared to the input image. As a result, the sliding window on the top level may correspond to a large step size on the image level, potentially leading to inaccurate bounding box locations.

Thirdly, the definition of $h_a$ in lines 173 and 174 is not provided, making it difficult to understand the underlying mathematics.

The proposed two-stage object detection algorithm leverages a convolutional neural network (CNN) to achieve near real-time performance (5fps). The first stage involves region proposal using the traditional sliding window method, but applied to the top-layer feature map of the CNN (RPN). The second stage employs a fast R-CNN to classify the proposed regions. By sharing convolutional layers between RPN and R-CNN and utilizing GPU acceleration, the algorithm achieves significant speedup.

This work presents a natural extension of existing CNN-based object detection methods. However, I have several concerns that need to be addressed.

1. The optimization section appears to employ only a single iteration of alternating optimization between the Region Proposal Network (RPN) and the R-CNN. If this is the case, it assumes that the proposals generated by the first iteration of RPN are optimal, which is not necessarily true. This may lead to suboptimal performance.

2. When applying the pixel-wise sliding window to the top layer of the CNN, it is unclear what the step size is on the image layer. Due to the multiple rounds of max-pooling in the CNN, the top-level feature map is significantly downsampled compared to the input image. As a result, the sliding window on the top level may correspond to a large step size on the image level, potentially leading to inaccurate bounding box locations.

3. The definition of $h_a$ on lines 173 and 174 is not provided, which hinders understanding of this specific component of the algorithm.

This work proposes a two-stage object detection algorithm based on a convolutional neural network (CNN). The first stage involves region proposal, which leverages the traditional sliding window method but operates on the top-layer feature map of the CNN (RPN). The second stage applies a fast R-CNN to the proposed regions. Since the convolutional layers are shared between RPN and R-CNN, and the computation is accelerated using a GPU, the algorithm can achieve near real-time performance (5fps).

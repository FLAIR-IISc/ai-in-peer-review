This work is a natural extension to existing CNN-based object detection methods. However, I have several concerns that need to be addressed.

Firstly, in the optimization section, it appears that there is only one iteration of the alternating optimization between the Region Proposal Network (RPN) and the R-CNN. If this is the case, it assumes that the proposals generated by the first iterated RPN are optimal, which is obviously not true. This raises questions about the convergence and stability of the proposed algorithm.

Secondly, when applying the pixel-wise sliding window to the top layer of the CNN, it is unclear what the step size is on the image layer. Due to the multiple rounds of max-pooling in the CNN, the top level feature map is at a much smaller scale compared to the input image. As a result, the sliding window on the top level may have a large step on the image level, potentially causing inaccurate bounding box locations.

Thirdly, the definition of $h_a$ in lines 173 and 174 is not provided, which makes it difficult to understand the underlying mathematics.

The proposed two-stage object detection algorithm is based on a convolutional neural network (CNN). The first stage involves region proposal, which utilizes a traditional sliding window method but operates on the top layer feature map of the CNN (RPN). In the second stage, a fast R-CNN is applied to the proposed regions. Since the convolution layers are shared between the RPN and R-CNN, and the computation is accelerated using a GPU, the algorithm can achieve near real-time performance (5fps).

This work presents a natural extension to existing CNN-based object detection methods. My primary concerns are detailed below.

1. The optimization section describes an alternating optimization approach between the Region Proposal Network (RPN) and the R-CNN. However, it appears this optimization is performed with only a single iteration. If this is the case, the method assumes the proposals generated by the initial RPN iteration are optimal, which is unlikely to be true.

2. The paper applies a pixel-wise sliding window to the top layer of the CNN. It is unclear what the effective step size of this sliding window is when projected back to the original image. Given that the top layer feature map is significantly downscaled due to multiple max-pooling operations, a sliding window movement of one pixel at the top layer corresponds to a potentially large step in the original image. This coarse step size could lead to inaccurate bounding box localization.

3. The definition of $h_a$ in lines 173 and 174 is not clearly stated.

Overall, this work proposes a two-stage object detection algorithm leveraging convolutional neural networks (CNNs). The first stage generates region proposals using a sliding window approach applied to the top layer feature map of a CNN – termed the Region Proposal Network (RPN). The second stage applies a Fast R-CNN to these proposed regions. By sharing convolutional layers between the RPN and the R-CNN, and utilizing GPU acceleration, the algorithm achieves near real-time performance (5fps).

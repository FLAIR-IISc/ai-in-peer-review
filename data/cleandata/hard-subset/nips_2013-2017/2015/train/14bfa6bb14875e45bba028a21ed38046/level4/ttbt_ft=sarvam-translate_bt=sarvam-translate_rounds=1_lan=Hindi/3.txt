This work is a natural extension of the existing CNN-based object recognition. My concerns are as follows:

1.  In the optimization section, it seems there is only one iteration for alternative optimization between RPN and R-CNN. If so, it assumes that the proposals generated by the first iteration RPN are optimal, which is clearly not true.
2.  When a pixel-by-pixel sliding window is applied to the top layer of the CNN, what is the step size on the image layer? Since there are several rounds of max-pooling in the CNN, the top layer is much smaller in scale than the input image. The sliding window on the top layer may have a larger step size on the image layer, which could result in incorrect bounding box locations.
3.  What is the definition of $h_a$ in lines 173 and 174?

This work proposes a two-step object recognition algorithm based on convolutional neural networks (CNNs). The first step is region proposal, which is based on the traditional sliding window method but works on the top layer feature map of the CNN (RPN). In the second step, a fast R-CNN is applied to the proposed regions. Since the convolution layers are shared between RPN and R-CNN, and the computation is speeded up using a GPU, the algorithm can achieve near real-time (5fps).

This work represents a natural extension to existing CNN-based object detection methods. However, I have several concerns that need to be addressed.

1. In the optimization section, it appears that there is only one iteration of the alternating optimization between the Region Proposal Network (RPN) and the Region-based CNN (R-CNN). If this is the case, it assumes that the proposals generated by the first iteration of the RPN are optimal, which is obviously not true.

2. When applying the pixel-wise sliding window to the top layer of the CNN, what is the step size on the image layer? Since the CNN undergoes several rounds of max-pooling, the top level is at a much smaller scale compared to the input image. As a result, the sliding window on the top level may have a large step on the image level, potentially causing inaccurate bounding box locations.

3. What is the definition of $h_a$ in lines 173 and 174?

This work proposes a two-stage object detection algorithm based on a convolutional neural network (CNN). The first stage involves region proposal, which is based on the traditional sliding window method but operates on the top layer feature map of the CNN (RPN). In the second stage, a fast R-CNN is applied to the proposed regions. Since the convolution layers are shared between the RPN and R-CNN, and the calculation is accelerated using a GPU, the algorithm can achieve near real-time performance (5fps).

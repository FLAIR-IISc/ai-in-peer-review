- Proposes "on-the-job learning": hybrid system using real-time crowdsourcing + online CRF to start accurate predictions from scratch, modeled as a stochastic game with continuous time and approximated by MCTS with progressive widening.  
- Originality: novel, practical formulation framing cold-start deployment as a decision-theoretic stochastic game balancing accuracy, cost, and latency.  
- Significance: addresses cold-start bottleneck enabling immediate high-accuracy systems, useful in time-sensitive or resource-constrained domains.  
- Technical quality: solid Bayesian decision-theory foundation and comprehensive experiments (NER, sentiment, face-ID) showing reduced crowd reliance over time.  
- Clarity weakness: continuous-time formalism complicates exposition and obscures the core decision-theoretic idea; recommend presenting discrete-time first.  
- Justification weakness: added complexity of continuous-time + MCTS not convincingly justified—simple Threshold baseline performs nearly as well and sometimes faster.  
- Unsupported claim: paper asserts improvement over expert labels on NER, but Table 2 shows 5-vote majority outperforms LENSE; claim should be corrected.  
- Missing baselines: 5-vote majority results absent for Face-ID and Sentiment; Threshold baseline missing for Sentiment and omitted from Figure 4 learning curve—include them.  
- Formatting issue: Table 2 fails to bold best NER F1 (5-vote) and lowest NER latency (1-vote); correct highlighting.  
- t_deadline clarification: unclear how time penalties are optimized or whether a latency deadline was used to shape reward/policy in experiments.  
- Return action criterion: unclear when system chooses "return"—purely MCTS value-based or via a separate stopping rule (e.g., posterior entropy threshold)?  
- Threshold baseline typo: reported inequality for querying appears wrong (suggests querying stops when uncertainty is high); clarify or correct formula.  
- Notation issue (Eq.3): response model p(r' | x, r, q) should also depend on the query issue time s because it affects response availability.  
- Confusing symbol use: using q for both query action and a distribution is ambiguous—recommend renaming one (e.g., p or π).  
- Recommendation: accept; paper is novel and significant but should be revised to improve clarity and stronger empirical justification of the complex continuous-time/MCTS approach.

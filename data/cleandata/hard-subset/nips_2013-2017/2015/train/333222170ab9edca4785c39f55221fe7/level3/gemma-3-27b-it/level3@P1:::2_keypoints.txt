- Introduces "On-the-Job Learning": leverage real-time crowdsourcing at prediction time to build high-accuracy systems without pre-labeled data.  
- Formalizes the problem as a stochastic game using Bayesian decision theory balancing latency, cost, and accuracy.  
- Solves the decision problem using a Monte Carlo Tree Search approximation.  
- Demonstrates strong empirical gains and cost reductions on NER, sentiment, and image classification tasks.  
- Shows notable improvements (e.g., +8% F1 over single human label, +28% over online learning on NER).  
- Differentiates from online and active learning by operating at test time and dynamically adapting query policy.  
- Properly connects to prior work in online/active learning, crowdsourcing, and structured prediction, citing key references.  
- Models noisy and variable crowd response times and incorporates a utility function for cost/latency/accuracy tradeoffs.  
- Provides open-source code (LENSE), aiding reproducibility.  
- Explanations can be opaque in places; needs a concrete example illustrating queries, crowd responses, and timings.  
- Inconsistent labeling of a baseline as “entropic” vs. “threshold” in Tables 2 and 3; requires standardization.  
- Related work section is disjointed and should be better integrated into the narrative.  
- Contains minor errors (e.g., broken figure reference at line 86).  
- Comparison to stronger aggregation baselines (worker-reliability-aware methods) is lacking and LENSE underperforms 5-vote consensus in some cases.  
- Overall recommendation: accept with minor revisions to improve clarity, fix inconsistencies, and reorganize related work.

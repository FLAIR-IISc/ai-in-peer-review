This paper introduces "on-the-job learning," a framework for deploying a high-accuracy prediction system that starts with no training data. The system processes a stream of inputs, and for each one, it can query a crowd of human workers in real-time to resolve uncertainty before making a final prediction. As the system observes more labeled data from these queries, its underlying model improves, and its reliance on the crowd decreases. The authors formalize this problem as a stochastic game between the system and the crowd. Using Bayesian decision theory, they define a utility function that balances prediction accuracy, monetary cost, and latency. Since computing the optimal policy for this game is intractable, they propose an approximation using Monte Carlo Tree Search (MCTS) combined with progressive widening. This approach is related to online learning, as the model updates over time, and active learning, as it strategically selects queries. However, it is distinct in its focus on maintaining high per-instance accuracy from the very beginning by making fine-grained queries before prediction. The authors evaluate their system, LENSE, on named-entity recognition (NER), sentiment analysis, and face classification, demonstrating significant cost reductions compared to a fully human-powered approach and accuracy improvements over a standard online learning baseline.

### Strengths

1.  **Novel and Principled Problem Formulation:** The "on-the-job" learning setting is a practical and important problem that bridges the gap between purely ML-based systems (which require upfront training data) and purely crowdsourced systems (which are expensive). Framing this problem as a stochastic game and using Bayesian decision theory to manage the trade-offs between accuracy, cost, and latency is a novel, elegant, and principled approach.

2.  **Sophisticated Technical Approach:** The application of Monte Carlo Tree Search to find an approximate policy is a strong technical contribution. It provides a robust method for navigating the vast and complex decision space involved in choosing when and what to query.

3.  **Strong Empirical Results on a Key Task:** The results on the NER task are particularly impressive. The system achieves a high F1 score while reducing the cost by more than an order of magnitude compared to having humans label everything. This demonstrates the framework's potential in settings where it is well-matched to the problem structure (i.e., structured prediction with meaningful partial queries).

4.  **Clarity and Reproducibility:** The paper is generally well-written and clearly motivated. The authors are commended for making their code and data available, which is a significant contribution to reproducibility.

### Weaknesses

1.  **Lack of Clarity in Experimental Setup:** The application of the framework to the non-structured tasks is unclear. For the face recognition task, the paper does not specify what constitutes a "query." The problem is formulated for structured outputs `y = (y1, ..., yn)`, but standard image classification yields a single label. Is the system repeatedly querying for the same image label to handle noise, or is it querying for specific attributes as in the original dataset paper? This ambiguity makes it difficult to interpret the results for this task and assess the generality of the approach.

2.  **Weak and Poorly Motivated Baselines:** The choice of baselines limits the impact of the experimental comparison.
    *   The **Threshold baseline** is a reasonable heuristic, but it relies on two unexplained, seemingly arbitrary parameters (0.98 and 0.3). The paper provides no justification for these values or analysis of the baseline's sensitivity to them, making it a weak point of comparison.
    *   The **Online learning** baseline, which cannot query the crowd at prediction time, is not a suitable competitor. A much more relevant and challenging baseline would be a standard **uncertainty sampling** approach from active learning. For instance, a system that queries the full label from the crowd whenever the model's prediction entropy exceeds a certain threshold would provide a more direct comparison and better highlight the benefits of the proposed partial querying strategy.

3.  **Oversimplified Distinction from Active Learning:** The paper claims a strong distinction from online active learning, stating that the latter "cannot maintain high accuracy independent of the number of data instances seen so far." This is not entirely true. An online active learning algorithm could be adapted to this setting by treating the incoming stream as the active set: for each instance, the model first predicts, then decides whether to query the true label based on its confidence. If queried, the model is updated before processing the next instance. While the proposed BDT framework for fine-grained partial querying and latency modeling is novel, the conceptual gap with a strong active learning baseline is smaller than portrayed and should be discussed more thoroughly.

4.  **Limited Applicability and Myopic Utility:**
    *   The core strength of the framework—making partial queries—shines in structured prediction tasks like NER. However, its utility in non-structured settings like sentiment or image classification appears limited. In these cases, the only available action is to repeatedly query the single instance label, which is a less novel problem. This suggests the broad applicability of the method might be constrained.
    *   The proposed utility function is defined on a per-instance basis, maximizing the reward for the current prediction. It does not explicitly model the "exploration" value of a query—that is, how acquiring a label might improve the global model for future instances. This myopic focus on immediate reward may be suboptimal for achieving the fastest possible learning rate over the entire data stream.

In summary, this paper presents a novel and technically sophisticated framework for a compelling problem. The core idea is strong, and the results on the NER task are impressive. However, the evaluation is weakened by a lack of clarity in some experiments, the absence of a strong active learning baseline, and the use of a poorly justified heuristic baseline. While the framework is promising, its claimed generality and clear superiority over simpler, related methods are not fully established.

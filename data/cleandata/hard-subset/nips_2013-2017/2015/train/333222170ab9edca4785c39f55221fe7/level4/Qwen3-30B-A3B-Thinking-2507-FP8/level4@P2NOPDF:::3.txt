

This paper proposes a principled hybrid framework for integrating crowdsourced labels with online model training, where the system initially relies heavily on crowd input during early model uncertainty, gradually reducing crowd dependence as model confidence grows. The core innovation frames the problem as a stochastic game between the system (which issues queries) and the crowd (which provides responses), optimizing expected utility under time constraints. Specifically, the system selects query actions \(q\) for tokens \(x\) (with true label \(y\)) to maximize utility \(U(q,r)\), where \(r\) is the crowd response. Crucially, queries are issued dynamically over time \(s\), with utility maximization constrained by a deadline \(t_{\text{deadline}}\) that dictates query volume and rate.  

Computing expected utility requires the simulation model \(P(y,r,t|x,q,s)\) (Eq. 4), but continuous time renders exact optimization intractable. The authors approximate this via Monte Carlo Tree Search and TD learning (Algorithm 1). Experiments on NER, sentiment, and face identification tasks compare four methods: (1) majority vote (1, 3, 5 judges), (2) online learning with oracle labels, (3) a "threshold" baseline (model without continuous time, sending \(m\) queries per step until uncertainty drops below a threshold), and (4) the full model (LENSE). LENSE achieves F1 scores nearly matching the 5-vote baseline (Table 2), while the threshold baseline performs comparably with significantly lower latency in NER. The model also demonstrates reduced crowd dependency over time (Fig. 3) and higher accuracy from the first example versus online learning (Fig. 4), leveraging crowd expertise when model uncertainty is high.  

The work is highly original and significant, presenting the first principled, end-to-end approach to hybrid learning from crowds. Results against strong baselines are compelling, and the method has strong practical potential. However, clarity issues significantly undermine the paper’s impact and could hinder adoption.  

The primary concern is the introduction of continuous time in Section 3, which adds substantial complexity without clear empirical gains. The threshold baseline achieves near-identical performance to LENSE on all tasks (Table 2: NER F1=0.82 vs. 0.83, Face ID F1=0.91 vs. 0.90) while offering lower latency (NER: 1.2s vs. 2.1s). The authors’ argument that continuous time benefits NER due to label interactions (l. 314) is weakened by the marginal latency advantage and equivalent performance. For practical applications, the simpler threshold baseline—a principled hybrid system already outperforming active/online learning—would suffice. The paper would benefit greatly by presenting the core method (without continuous time) first, with continuous time as an optional extension. This would enhance readability and implementation feasibility.  

Additional clarity issues include:  
- **Missing data**: The 5-vote baseline is omitted from Face ID and Sentiment tables/figures (Tables 2, 3; Fig. 4), despite being the top performer. It should be included and highlighted (e.g., bolded).  
- **Performance claims**: The assertion that the model "beats human performance" is inaccurate for NER (5-vote baseline achieves 0.83 F1, while LENSE achieves 0.83 but the paper claims human performance as 0.85+; Table 2 shows 5-vote at 0.82).  
- **Technical details**:  
  - The \(t_{\text{deadline}}\) implementation is unclear (l. 183–187): Was it enforced as a hard deadline, or via an entropic threshold (like the baseline)?  
  - Eq. (2) should include \(s\) as a parameter: \(p(y,r,t|x,q,s)\).  
  - Eq. (5) uses confusing notation \(F(q)\); \(q\) is overloaded with query variables. A different symbol (e.g., \(F(\theta)\)) is needed.  
  - Threshold baseline description (l. 246–252) has a sign error: The condition should be \((1 - p_{\theta}(y_i|x)) \cdot 0.3^m \leq (1 - 0.88)\), not \(\geq 0.88\), since increasing \(m\) reduces uncertainty.  

The paper’s core contribution—hybrid learning leveraging crowd expertise during model uncertainty—is valuable and novel. The current presentation, however, obscures this by embedding it within a complex time-dependent formulation that offers negligible gains. Streamlining the presentation to prioritize the threshold-based method (which is already a strong, deployable solution) would substantially increase the paper’s impact. The minor technical corrections above are straightforward to address in revision.  

**Recommendation**: The work is original, significant, and well-supported by results. Despite clarity issues, the method’s potential is strong. With revisions to clarify the core approach (removing continuous time as a central element) and address technical details, this paper would be a strong fit for NeurIPS. I recommend acceptance with minor revisions.

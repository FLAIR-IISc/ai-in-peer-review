This paper proposes a hybrid approach that combines crowdsourced labels with an incrementally (online) trained model to solve prediction problems. The key idea is to rely heavily on the crowd during system warm-up, learn from those labels, and then progressively reduce crowd involvement as the model’s confidence increases. The authors frame this as a stochastic game based on a CRF prediction model, in which both the system and the crowd are players. The system can issue one or more queries q for tokens x (with true label y), eliciting responses r, each associated with a utility U(q, r). The system aims to select actions that maximize expected utility. Queries are issued over time at points s, with response times t, and utility is maximized subject to a deadline t_deadline by which an answer must be returned; this determines how many queries are sent, at what rate, etc.

Computing this expected utility requires the simulation dynamics model P(y, r, t | x, q, s) to evaluate the utilities as in equation (4). Given these utilities, the optimal action could in principle be chosen, but the introduction of continuous time makes the problem intractable. The authors therefore use an approximation based on Monte Carlo Tree Search and TD learning (Algorithm 1). Experiments are conducted on a simplified NER task, a sentiment classification task, and a face identification task, comparing four methods: majority vote of n human judges (n = 1, 3, 5), online learning with oracle (true) labels, a “threshold” baseline (the authors’ model without continuous time, where m queries are issued at each step until uncertainty falls below a threshold), and the full model (LENSE). In terms of precision/recall/F1, the full model outperforms all but the 5-vote crowdsourced variant, though the “threshold” baseline performs nearly as well (and with lower latency for NER; Table 2). The authors also show that the model requests fewer and fewer crowd labels over time when the learned model is sufficiently strong (Figure 3), and that compared to online learning, accuracy is high from the first example (Figure 4), since the system can leverage crowd expertise more heavily while model uncertainty is high.

This is a very interesting paper that should generate discussion, follow-up work, and practical adoption. While there are many heuristic approaches to using the crowd while training a system, this is the first complete and principled formulation I have seen. The results against reasonable baselines are impressive. I find the work original and significant, and I believe it merits acceptance. However, clarity is a concern, and this may limit the paper’s impact. Many issues are fixable (even for the camera-ready) if the authors invest effort and slightly adjust the tone regarding the “threshold” baseline versus the full continuous-time version.

The main clarity issue arises from the introduction of continuous time in Section 3. The motivation is reasonable—classification may have both time and cost budgets—but empirically it seems to offer little benefit, while it greatly complicates the formulation, discussion, and optimization. In Table 2, the “threshold” baseline achieves significantly lower latency on NER with similar performance (marginally worse). On the face ID task it actually performs slightly better (though roughly equivalent). The authors argue (l. 314) that the baseline does better on face ID because it is a single-label prediction task without significant label interactions; however, on NER, where label interactions are abundant, the benefit appears only marginal.

For many practical tasks, the “threshold” baseline is likely sufficient, and it already represents a substantial advance (a principled hybrid learning system that outperforms active and online learning). The paper would likely have greater impact if this were made explicit in the exposition: first fully develop the core method without continuous time (which would be easier to understand and implement), and then present continuous time as an extension. In the same vein, the discussion of results should more clearly acknowledge the strength of the “threshold” method, even when there is sequence dependence among labels. This is a suggestion rather than a requirement—I still believe the paper should be accepted—but it would broaden understanding and adoption.

There are several smaller issues in the results and their presentation:
- The introduction claims the model “beats human performance,” which is not true in the 5-vote case for NER. Moreover, the 5-vote case is missing for the face ID and sentiment tasks in multiple places (the right half of Table 2, Table 3, and Figure 4) and should be included.
- In Table 2, the 5-vote results should be bolded as they are the best. The latency value for Threshold in NER should also be bolded for the same reason (and the LENSE latency for face ID).
- The “Threshold” baseline is missing for the sentiment experiment and thus does not appear in Table 3 or Figure 4; it should be included.

Minor comments on the method:
- Lines 183–187 discuss t_deadline, but it is unclear what, if any, t_deadline was used in the experiments, and what criterion determined when to report an answer. Was it an entropy-based threshold as in the baseline, or a hard t_deadline?
- In equation (2), the distribution should include s (i.e., p(y, r, t | x, q, s)).
- In equation (5), using q in F(q) is confusing; q suggests a query, but here it denotes a nominal distribution. Please choose a different letter.
- In the description of the Threshold baseline (l. 246–252), I believe the expression should be (1 − p_theta(y_i | x)) · 0.3^m <= (1 − 0.88) (note the RHS), rather than >= 0.88, since increasing m reduces uncertainty.

In summary: This paper presents a hybrid approach that integrates crowdsourced labels with an online model, cast as a stochastic game that models data, labels, and crowd responses, including query timing. The results are strong relative to multiple human labelers, online learning with oracle labels, and a “threshold” baseline that removes continuous time yet performs comparably. The method is original, novel, and interesting, and the paper should be accepted. That said, the introduction of continuous time substantially complicates the narrative and algorithm while delivering minimal empirical gains; clarifying this trade-off and elevating the “threshold” variant in the exposition would improve clarity and impact.

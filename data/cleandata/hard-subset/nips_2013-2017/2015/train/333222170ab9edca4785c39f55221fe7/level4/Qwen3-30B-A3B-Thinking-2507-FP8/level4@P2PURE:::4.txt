

The authors introduce "on-the-job learning," a scenario where annotators are queried between receiving an example and outputting a prediction. While the algorithm is clearly defined for structured prediction tasks (e.g., querying elements within a sequence), its application to the face recognition task is unclear: was the instance itself queried? In the non-structured setting, the model would not propagate information between adjacent examples, rendering the utility function ill-defined. The threshold baseline employs two arbitrary parameters; the rationale for these values and their sensitivity are not addressed. A baseline using uncertainty sampling, as in active learning, would provide a stronger comparison. The authors emphasize the distinction from active online learning, yet active learning could be straightforwardly adapted to this domain by treating the active set as a sequence—e.g., updating the model upon receiving responses and outputting a final prediction. While this adaptation would not capture the task's temporal dynamics, it leverages existing theory for query selection. The framework appears novel but lacks clarity on key implementation details and exhibits limited applicability.

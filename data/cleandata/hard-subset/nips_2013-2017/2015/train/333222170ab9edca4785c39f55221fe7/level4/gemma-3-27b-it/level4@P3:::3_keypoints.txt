- Presents a novel, principled framing of on-the-job learning as a stochastic game combining crowdsourced labels and an online-trained model (LENSE).  
- Core idea: rely heavily on crowd during ramp-up, learn from those labels, then reduce crowd queries as model confidence increases.  
- Uses a CRF-based prediction model where the system can issue token queries to maximize expected utility given costs and responses.  
- Introduces continuous time to model latency and time budgets, but the practical benefit of continuous time is unclear from results.  
- Empirical evaluation on NER, sentiment, and face-ID shows LENSE often outperforms online learning and a “threshold” baseline.  
- LENSE is frequently outperformed by a 5-vote crowdsourced baseline, especially in NER, contradicting the “beating human performance” claim.  
- Demonstrates that LENSE requires fewer crowdsourced labels over time and achieves high initial accuracy via crowd expertise.  
- Missing comparisons: 5-vote results absent for Face-ID and Sentiment in several tables/figures; these must be included for fair comparison.  
- Reporting/formatting issues: 5-vote results and key latency values (Threshold NER, LENSE Face-ID) should be bolded to highlight best performance.  
- Baseline omissions: Threshold baseline is missing from sentiment experiments (Table 3 & Figure 4) and needs inclusion.  
- Continuous-time complexity concern: threshold baseline often performs comparably or better (including on latency), so authors should clarify scenarios where continuous time is essential.  
- Suggestion for clearer presentation: present core method without continuous time first, then treat continuous-time model as an extension.  
- Method description issues: experiments do not state criteria/t_deadline used to decide prediction readiness.  
- Technical errors: missing variable 's' in Equation (2), confusing use of 'q' inside F(q) in Algorithm 1, and a likely incorrect inequality in the Threshold baseline description.  
- Positive meta-points: original idea, significant potential to reduce labeling cost, strong baselines used, promising results, and open-source implementation available.  
- Recommendation: accept with revisions — fix experimental completeness/formatting, clarify continuous-time benefits, correct technical/notation issues, and temper claims about human performance.

This paper proposes a hybrid approach that combines crowdsourced labels with an incrementally (online) trained model for prediction tasks. The core idea is to rely heavily on the crowd during system warm-up, learn from those labels, and gradually reduce crowd involvement as the model’s confidence increases. The problem is framed as a stochastic game atop a CRF prediction model, with the system and the crowd as players. The system can issue one or more queries q for tokens x (with true label y), eliciting responses r, and each outcome yields a utility U(q, r). The system chooses actions to maximize expected utility. Queries are not issued all at once; instead, they are sent at times s, with response times t, and utility is maximized subject to a t_deadline by which an answer must be produced (thereby determining how many queries to send and at what rate).

Computing expected utility requires the simulation dynamics model P(y, r, t | x, q, s), which is used to compute utilities as in (4). Given these utilities, an optimal action can be chosen; however, continuous time renders exact optimization intractable, so the authors use an approximation based on Monte Carlo Tree Search and TD learning (Algorithm 1). Experiments cover a simplified named entity recognition (NER) task, a sentiment recognition task, and a face identification task, comparing four methods: majority vote of n human judges (n = 1, 3, 5), online learning (using true labels), a “threshold” baseline (the authors’ model without continuous time, where m queries are sent at each step until the model’s uncertainty falls below a threshold), and the full model (LENSE). In precision/recall/F1, the full model outperforms all methods except the 5-vote crowd condition, though the “threshold” baseline is close (and achieves lower latency for NER; Table 2). The authors also show that, given a sufficiently strong model, the system requires fewer crowdsourced labels over time (Figure 3), and that, relative to online learning, accuracy is high from the first example (Figure 4) because the system leans more on the crowd while model uncertainty is still high.

This is an interesting and timely paper that should spark discussion, further work, and practical adoption. While many heuristic approaches exist for learning from the crowd during model training, this is the first complete and principled proposal I have seen. The results against reasonable baselines are impressive. I find the work original and significant, and I believe it merits inclusion in the conference. However, clarity is a weakness that could limit impact. Many issues are fixable (even for the camera-ready), especially if the authors refine the tone regarding the “threshold” baseline versus the full continuous-time variant.

The main clarity concern arises from the introduction of continuous time partway through Section 3. The motivation is reasonable—classification may need to adhere to both time and cost budgets—but empirically the benefits appear small, while the formulation, exposition, and optimization procedure become substantially more complex. In Table 2, the “threshold” baseline achieves noticeably lower latency on NER with roughly equivalent performance (slightly worse), and on the face ID task it actually performs better (again, roughly equivalent). The authors argue (l. 314) that the baseline does better on the second task because it involves a single label prediction with limited label interactions; however, the NER task, which exhibits strong label interactions, gains only marginally. For many practical problems, the “threshold” baseline will likely suffice—and it already represents a significant advance as a principled hybrid learning system that outperforms active and online learning. The paper would have more impact if this were made explicit: fully develop the core method without continuous time (simplifying understanding and implementation) and present continuous time as an extension. Similarly, when discussing results, be clearer that the “threshold” approach is quite strong even when sequence dependencies exist. This is a suggestion rather than a requirement—the paper is strong enough for acceptance—but it would improve clarity and the likelihood of broad adoption.

There are also several issues in how results are reported. The introduction claims the model “beats human performance,” which is not true in the 5-vote NER condition. Strangely, the 5-vote condition is missing for Face-ID and Sentiment in multiple places (the right half of Table 2, Table 3, and Figure 4); it should be included. In Table 2, the 5-vote results should be bolded as the best performance. The NER latency for the Threshold baseline should also be bolded for the same reason (and the LENSE latency should be bolded for Face ID). More importantly, the “Threshold” baseline is missing for the sentiment experiment, and thus does not appear in Table 3 or Figure 4; it should be included.

Minor comments on the method description:
- Lines 183–187 discuss t_deadline, but it is unclear what, if any, t_deadline was used in the experiments, and what criteria determined when to report an answer—was it an entropic threshold (as in the baseline), or reaching some t_deadline?
- In equation (2), the distribution should include s, i.e., p(y, r, t | x, q, s).
- In equation (5), using q in F(q) is confusing; q suggests a query, but here it denotes a nominal distribution. Please choose another letter.
- In the description of the Threshold baseline (l. 246–252), the expression should be (1 − p_theta(y_i | x)) * 0.3^m <= (1 − 0.88) (note the RHS) rather than >= 0.88, since increasing m reduces uncertainty.

In sum, the paper presents a principled hybrid approach that models data, true labels, crowd responses, and timing, with strong results relative to multiple human labelers, online learning with oracle labels, and a “threshold” variant that removes continuous time and performs on par. The method is original and compelling and should be accepted, but the introduction of continuous time greatly complicates the presentation and algorithmic mechanics while yielding minimal performance benefits, and the reporting and clarity issues noted above should be addressed.

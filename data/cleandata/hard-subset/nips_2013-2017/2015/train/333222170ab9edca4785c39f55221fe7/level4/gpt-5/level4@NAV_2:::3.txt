This paper proposes a hybrid strategy that combines crowdsourced annotations with an incrementally trained (online) model for prediction tasks. The central idea is to rely heavily on the crowd at deployment, learn from the incoming labels, and gradually reduce crowd queries as the model’s confidence grows. The authors formalize this as a stochastic game built on a CRF prediction model, with the system and the crowd as players. The system issues one or more queries q targeting tokens in x (with true labels y) and receives responses r, with a utility U(q,r) defined for each outcome; the system then chooses actions to maximize expected utility. Crucially, queries are issued over time s and responses arrive at times t; utility optimization accounts for a t_deadline by which a prediction must be produced (thereby determining how many queries are sent, at what cadence, etc.).

To compute expected utility, the approach uses a simulation model P(y,r,t|x,q,s) to evaluate returns as in (4). With utilities in hand, one could choose the optimal action; however, continuous time renders exact optimization intractable, so the authors adopt an approximation based on Monte Carlo Tree Search and TD learning (Algorithm 1). The method is evaluated on a simplified NER task, a sentiment classification task, and a face identification task, against four baselines: majority vote of n human labelers (n ∈ {1,3,5}), online learning (with oracle labels), a “threshold” baseline (the same model but without continuous time, launching m queries per step until uncertainty falls below a threshold), and the full method (LENSE). In precision/recall/F1, the full model outperforms all but the 5-vote crowd condition, while the threshold baseline performs nearly as well (and with lower latency on NER; Table 2). The authors also show that, given a sufficiently expressive learner, the method requests fewer crowd labels over time (Figure 3), and that accuracy is strong from the first example (Figure 4) by leaning on crowd expertise when model uncertainty is high.

This is an engaging paper that is likely to spark discussion, follow-on work, and real-world use. While there exist various heuristic schemes for learning from the crowd while training, this is the first fully principled treatment I have encountered. The comparisons against reasonable baselines are compelling. I find the work original and significant and believe it merits acceptance. That said, the clarity is not on par with the novelty, which could limit impact. Most of these issues are fixable (even for the camera-ready) with some rewrites and a toned-down presentation relative to the “threshold” baseline versus the continuous-time variant.

The main clarity issue arises from the mid-Section 3 introduction of continuous time. The motivation is understandable (per-example time budgets in addition to cost), but empirically it appears to offer minimal benefit, while substantially complicating the formulation, exposition, and optimization. In Table 2, the “threshold” baseline achieves clearly lower latency on NER with only marginally worse performance, and on the face ID task it actually does better (albeit roughly comparable). The paper attributes (l. 314) the baseline’s advantage on the latter to single-label prediction with limited label interactions, yet the NER task, which has rich dependencies, shows only modest gains. For many practical scenarios, the “threshold” approach seems sufficient and already represents a major step forward (a principled hybrid method that outperforms active and online learning). The paper would likely have greater impact if it presented the non-time-based core method first (simpler to understand and implement), with continuous time as an extension. Along the same lines, the writeup could more explicitly acknowledge how strong the “threshold” method is, even in structured settings. This is a recommendation rather than a condition for acceptance, but it would improve readability and adoption.

There are also several smaller issues with the results and their presentation. The introduction claims the model “beats human performance,” which is not true in the 5-vote NER condition; confusingly, the 5-vote condition is missing for Face-ID and Sentiment in multiple places (the right half of Table 2, Table 3, and Figure 4) and should be included. In Table 2, the 5-vote results should be bolded as they are the best-performing condition; likewise, the Threshold latency for NER should be bolded (and the LENSE figure for Face ID). More consequentially, the “Threshold” baseline is absent for the sentiment task (and thus omitted from Table 3 and Figure 4) and should be added.

A few minor comments on the method description:
- While t_deadline is discussed in l.183–187, it is unclear what, if any, t_deadline was used in the experiments and what stopping rule determined when to output a prediction—was it an entropy-based threshold as in the baseline, or hitting a t_deadline?
- In equation (2), the distribution appears to require s as an argument as well (i.e., p(y,r,t|x,q,s)).
- In equation (5), using q within F(q) is confusing since q also denotes a query; please rename the function argument to avoid overloading notation.
- In the Threshold baseline description (l.246–252), it seems the inequality should read (1 − p_theta(yi|x)) × 0.3^m <= (1 − 0.88) (note the RHS), since increasing m reduces uncertainty.

In sum, this paper introduces a hybrid learning framework that fuses real-time crowdsourcing with an online learner, cast as a stochastic game that models labels, crowd responses, and timing of queries and answers. The empirical results against multiple human-vote baselines, oracle-aided online learning, and a strong “threshold” variant (which performs comparably) are impressive. The contribution is original, novel, and practically relevant, and I recommend acceptance, with the caveat that clarity—especially around the continuous-time component that adds considerable complexity but limited gains—should be improved.

Summary and contributions
This paper proposes an on-the-job learning framework that achieves high accuracy from the outset without a pre-trained model by interleaving real-time crowd queries with prediction. The system frames decision making as a stochastic game grounded in Bayesian decision theory, optimizing a utility that trades off accuracy, latency, and monetary cost. Because computing the optimal policy is intractable, the authors approximate it using Monte Carlo Tree Search (MCTS) with progressive widening to handle continuous time. Experiments on NER, sentiment classification, and image classification (faces) show substantial cost reductions compared to full human annotation and accuracy improvements over purely machine or purely human baselines. The open-source system (LENSE) is provided.

Relation to prior work
The work bridges online learning (improving over time), active learning (querying for information), active classification (test-time acquisition under costs), and crowdsourcing (noisy, delayed labels). Compared to online active learning, the key distinction is that labels are requested before prediction to maintain high accuracy from the start, and the action space includes when and how many queries to issue with latency-aware decisions. Compared to active classification, the policy here adapts online as the model improves and accounts for asynchronous, noisy crowd responses. Related systems like Legion:AR and TurKontrol motivate the setting but do not provide the same structured, decision-theoretic treatment for structured outputs and real-time latency.

Strengths
- Clear and principled formulation: stochastic game + Bayesian decision-theoretic utility; MCTS with progressive widening is a reasonable approximation for continuous-time decision making.
- Practical importance: supports rapid deployment with high accuracy using human-in-the-loop, then reduces reliance on the crowd as the model improves.
- Empirical gains: strong cost-accuracy tradeoffs on NER; competitive on sentiment and faces; open-source implementation and CodaLab worksheets enhance reproducibility.
- Insightful qualitative behavior: parallel vs. sequential querying, redundancy to combat noise, and decreasing query rates over time with richer features.

Weaknesses and questions
- Scope beyond structured prediction: The framework is clearly described for structured outputs (e.g., NER with token-level queries). For sentiment and face classification, please clarify the adaptation: Is each instance a single “position” so actions are instance-level queries? If so, how does the utility handle the lack of inter-variable dependencies, and does the method reduce to a simpler binary decision of how many redundant labels to collect and when? A brief formalization for the unstructured case would help.
- Choice and calibration of utility parameters: How were wM (monetary) and wT (latency) chosen and tuned (dev vs. test)? Please include sensitivity analyses showing performance under varying wM, wT, and worker accuracy assumptions, as these values strongly influence policy behavior.
- Response and time models: The response model assumes a fixed 70% accuracy with uniform error for all workers and items. This is a strong simplification. How sensitive are results to worker heterogeneity, class imbalance, and item difficulty? Similarly, specify the parametric form and fitted parameters for pT(ti | si) and show goodness-of-fit or ablations (e.g., log-normal vs. empirical).
- Baselines: The threshold baseline uses parameters (0.3 reduction per query; 0.98 threshold) that appear ad hoc. Please justify their origin and include sensitivity curves. More importantly, add stronger baselines adapted from active learning/active classification:
  - Uncertainty sampling at token level (or instance level), selecting a fixed or budgeted set of queries before predicting.
  - A greedy value-of-information baseline (myopic expected error reduction per unit cost).
  - For structured outputs, a policy trained offline (as in active classification) to decide which positions to query under a budget.
These will sharpen the empirical case that the decision-theoretic treatment of asynchrony and latency is essential.
- MCTS details: Report the UCT constant, simulation budget per example, and runtime. Provide ablations for progressive widening and for parallel vs. sequential query strategies.
- Evaluation details: For NER, clarify the label set used and handling of “miscellaneous”; for sentiment/faces, specify model architectures and features (UNIGRAMS, RNN, embeddings) used by pθ, and how online updates are performed between examples. Include per-task error analyses and the effect of class imbalance noted in the paper.

Quality
- The core idea is sound and well-motivated; the MCTS approximation is plausible. However, important implementation details (parameter choices, models of response quality and time, and MCTS settings) should be documented more thoroughly, and stronger baselines are needed to isolate where the gains come from.

Clarity
- Generally well-written with helpful figures and examples, but the non-structured case needs clearer formalization. Define all symbols on first use; fix minor typos; move the 70% crowd accuracy assumption and its empirical support into the main text.

Originality
- The problem framing (real-time, latency- and cost-aware querying before prediction) and the decision-theoretic game formulation are novel combinations of ideas. Using MCTS with progressive widening for this setting is an interesting contribution.

Significance
- High potential impact for rapid deployment scenarios and human-in-the-loop ML. If strengthened with additional baselines and analyses, this could become a reference approach for real-time labeling systems.

Suggestions for improvement
- Provide a concise formalization for the unstructured case and a discussion of when structure confers the largest benefits.
- Add active learning/active classification baselines as above; include sensitivity analyses for wM, wT, response accuracy, and time models.
- Report MCTS hyperparameters, computational cost, and ablations.
- Explore worker heterogeneity: simple per-worker reliability or item-difficulty models, and show robustness.
- Clarify dataset details and evaluation protocols; include learning curves and latency-cost-accuracy Pareto plots.

Arguments for acceptance
- Principled formulation addressing an important practical problem.
- Demonstrated improvements in cost and accuracy, with open-source code.
- Clear qualitative insights and a generally well-executed evaluation on multiple tasks.

Arguments against acceptance
- Incomplete comparison to strong active learning/active classification baselines.
- Limited analysis of utility parameter sensitivity and response/time model assumptions.
- Unclear formalization for unstructured tasks and some missing experimental details.

Overall recommendation
- Borderline accept. The paper presents a compelling and practical framework with promising results, but would benefit from clearer treatment of unstructured tasks, stronger baselines, and more thorough sensitivity analyses.

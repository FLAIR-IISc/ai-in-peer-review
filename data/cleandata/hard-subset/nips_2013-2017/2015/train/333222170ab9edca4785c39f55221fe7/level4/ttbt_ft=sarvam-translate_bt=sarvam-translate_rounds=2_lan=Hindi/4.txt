The author describes a scenario called "on-the-job learning," where annotations are requested from annotators between obtaining an example and generating a prediction.

While it's clear how the algorithm works for a structured prediction task, where elements of a sequence are queried, it's not at all clear how it was applied to a face recognition task – was it just the example that was queried? In a non-structured setting, the model won't propagate information between adjacent examples (instances), so it seems like the utility function would have no meaning.

The threshold baseline has two parameters that seem arbitrary. Where did these values come from, and what is the sensitivity to them?

Another baseline of uncertainty sampling, which is used in active learning, would be interesting.

Finally, the author is eager to distinguish between this setting and active online learning, but active learning can be easily adapted to work in this domain, where the active set in question is now a sequence. For example, there's no reason why the model couldn't be updated in the same way when responses are received, and then finally produce a prediction output. While this clearly wouldn't capture the temporal nature of the task, there are several theories for choosing the query set. A new framework seems to exist, although some details are unclear, and it seems quite restricted in applicability.

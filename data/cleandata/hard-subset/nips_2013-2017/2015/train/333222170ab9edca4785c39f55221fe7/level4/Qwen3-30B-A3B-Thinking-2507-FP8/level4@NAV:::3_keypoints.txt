- Introduces a hybrid framework (LENSE) combining crowdsourced labels and incrementally trained models, reducing crowd reliance as model confidence grows.  
- Frames prediction as a stochastic game between system and crowd, using a CRF-based prediction model and queries q for tokens x to receive responses r with utility U(q,r).  
- Incorporates response timing with a t_deadline constraint; expected utility needs simulation dynamics P(y,r,t|x,q,s).  
- Continuous-time formulation makes exact optimization intractable, so authors use Monte Carlo Tree Search plus TD learning (Algorithm 1).  
- Experiments on NER, sentiment, and face ID compare majority vote (1/3/5), online learning with true labels, a “threshold” baseline, and LENSE.  
- LENSE achieves near-best precision/recall/F1, only trailing 5-vote crowdsourcing; threshold baseline matches performance with lower NER latency.  
- Paper is original and principled, outperforming heuristic prior work and promising for practical adoption and future research.  
- Major clarity concern: continuous-time extension unnecessarily complicates formulation/optimization without clear empirical gains; threshold baseline performs as well or better in places.  
- Recommend clarifying that the core (non-continuous-time) method is already a robust, principled hybrid system and that continuous time is an optional extension.

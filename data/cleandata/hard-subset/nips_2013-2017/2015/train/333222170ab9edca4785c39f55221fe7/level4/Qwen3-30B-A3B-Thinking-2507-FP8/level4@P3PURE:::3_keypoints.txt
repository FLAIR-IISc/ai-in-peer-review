- Paper proposes a hybrid approach (LENSE) combining crowdsourced labels with an incrementally trained model, using heavy crowd use at initialization and less over time as model confidence grows.  
- Formulates the problem as a stochastic game between system and crowd over a CRF prediction model, with queries q for tokens x yielding responses r and utility U(q,r).  
- Incorporates continuous-time query issuance and response modeling with deadlines t_deadline to optimize query volume and rate.  
- Computing expected utility requires a dynamics model P(y,r,t | x,q,s) (Eq. 4); exact optimization is intractable due to continuous time.  
- Uses an approximation via Monte Carlo Tree Search and TD learning (Algorithm 1) for action selection.  
- Experiments on simplified NER, sentiment, and face identification compare LENSE to majority-vote baselines, online learning, and a "threshold" baseline.  
- LENSE outperforms baselines on precision/recall/F1 except for 5-vote crowdsourcing, where 5-vote is superior.  
- The threshold baseline performs nearly as well as LENSE and achieves lower latency on NER (Table 2); face ID shows slightly better accuracy for threshold.  
- Figures show LENSE reduces crowd queries as confidence grows (Fig. 3) and attains high accuracy from the first example by leveraging crowd expertise (Fig. 4).  
- Reviewer finds the work original, significant, and worthy of acceptance, but notes clarity issues that should be fixed for camera-ready.  
- Primary clarity concern: introducing continuous time mid-Section 3 complicates presentation while offering marginal experimental benefits; recommend presenting core method first and continuous time as an extension.  
- Specific correctness/consistency issues: claim that model "beats human performance" is inaccurate given 5-vote NER results.  
- Missing results: 5-vote entries absent for Face ID and Sentiment in Tables 2, 3 and Fig. 4 (right halves).  
- Presentation/formatting fixes: bold 5-vote and threshold-best entries in Table 2; bold LENSE’s Face ID latency.  
- Experimental omission: threshold baseline is missing from the sentiment experiment (Table 3, Fig. 4) and should be included.  
- Minor technical fixes: unspecified experimental value/usage of t_deadline in Sec. 3.1; Eq. (2) should include s argument; Eq. (5) uses confusing notation F(q) that conflicts with query symbol; threshold baseline description contains an inequality sign error.  
- Overall: the core method (without continuous time) is practical and strong; with clearer exposition and fixes, the paper will have greater impact.

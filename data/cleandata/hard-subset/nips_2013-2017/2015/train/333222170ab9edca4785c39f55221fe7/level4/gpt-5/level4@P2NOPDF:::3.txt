This paper proposes a hybrid system that combines crowdsourced labels with an incrementally trained model to make predictions. The core strategy is to rely heavily on the crowd early on, learn from those labels, and progressively reduce crowd queries as model confidence grows. The problem is framed as a stochastic game built on a CRF, with the system and the crowd as players. For tokens x (with unknown true labels y), the system issues queries q to elicit responses r, and selects actions to maximize expected utility U(q, r). Queries can be scheduled over time s, with response times t, and utility is optimized subject to a t_deadline that determines how many queries are sent and at what rate.

To compute expected utility (Eq. 4), the method employs a simulation dynamics model P(y, r, t | x, q, s). Optimizing the resulting objective directly is intractable due to continuous time, so the authors use an approximation based on Monte Carlo Tree Search with temporal-difference learning (Algorithm 1). Experiments cover a simplified NER task, sentiment classification, and face identification, comparing: (1) majority vote from 1/3/5 human judges, (2) online learning using oracle labels, (3) a “threshold” baseline that uses the authors’ framework but without continuous time, issuing m queries per step until model uncertainty drops below a threshold, and (4) the full model (LENSE). In precision/recall/F1, the full model outperforms all methods except the 5-vote crowd in NER; the “threshold” baseline performs nearly as well and achieves lower latency on NER (Table 2). The system also requires fewer crowd labels over time when the model is sufficiently strong (Fig. 3) and attains high accuracy from the first example compared to pure online learning (Fig. 4), since it can leverage the crowd more when model uncertainty is high.

Overall, this is a compelling and ambitious paper that, in my view, is both original and significant. It provides the first complete, principled treatment I have seen for learning from the crowd while training a model online, and it compares well against reasonable baselines. I recommend acceptance. However, the clarity of presentation—especially around the introduction of continuous time—lags behind the technical contribution and could limit impact unless addressed.

The main clarity issue is the introduction of continuous time midway through Section 3. The motivation (time and cost budgets per instance) is sensible, but the empirical benefits appear limited, while the formulation, discussion, and optimization become substantially more complex. In Table 2, the “threshold” baseline achieves markedly lower latency on NER with only marginally worse accuracy, and it even performs better on the face identification task (again, roughly equivalent overall). The paper argues (l. 314) that the baseline does better on face ID because it is a single-label task with minimal label interaction; yet on NER, where label interactions are strong, the gains from continuous time are still small.

Practically, the “threshold” baseline is likely sufficient for many applications, and it already constitutes a major advance: a principled hybrid learning system that outperforms active and online learning. The paper would be clearer and more impactful if the core method were fully developed first without continuous time (simplifying both exposition and implementation), with continuous time presented as an extension. Similarly, the discussion should be more explicit that the “threshold” method is very competitive—even on sequence tasks with label dependencies. This is a suggestion rather than a condition for acceptance, but I believe it would improve clarity and adoption.

Reporting and presentation issues:
- The introduction claims the model “beats human performance,” which is not true against 5-vote crowdsourcing on NER. Moreover, 5-vote results are missing for face ID and sentiment in several places (right half of Table 2, Table 3, Fig. 4). These should be included.
- In Table 2, the best results should be consistently highlighted: 5-vote for top accuracy, the “Threshold” latency for NER, and LENSE latency for face ID.
- The “Threshold” baseline is missing for the sentiment experiment (Table 3, Fig. 4) and should be added.

Methodological clarifications and minor issues:
- t_deadline (l. 183–187): Please state what t_deadline was used in the experiments and specify the stopping rule—was it an entropy-based threshold like the baseline, or a fixed deadline?
- Eq. (2): The distribution appears to need s as an argument: p(y, r, t | x, q, s).
- Eq. (5): Using q in F(q) is confusing, as q elsewhere denotes a query. Please rename the function argument to avoid ambiguity.
- Threshold baseline (l. 246–252): The inequality seems reversed. It likely should be (1 − p_θ(y_i | x)) · 0.3^m ≤ (1 − 0.88), since increasing m lowers uncertainty.

In sum, the paper offers a well-motivated hybrid framework, a principled stochastic game formulation on top of a CRF, and a practical approximation via MCTS+TD. Empirically, it is strong, though the simpler “threshold” variant often matches it with lower latency. I recommend acceptance, and I encourage the authors to streamline the exposition by foregrounding the non-temporal method, clearly positioning continuous time as an optional extension, and fixing the reporting and notational issues above.

This paper presents a hybrid approach that combines crowdsourced labels with an incrementally (online) trained model to tackle prediction problems. The core idea is to rely heavily on the crowd while the system is ramping up, learn from the acquired labels, and then increasingly defer to the model as its confidence grows. The method frames the problem as a stochastic game based on a CRF prediction model in which the system and the crowd are players. The system can issue one or more queries q for tokens x (with true label y) that elicit responses r, with a utility U(q, r) for each outcome; it thus selects actions to maximize expected utility. Queries are not issued all at once, but at times s (with response times t); utility is maximized with respect to a t_deadline by which an answer must be computed, which in turn governs how many queries are sent and at what rate.

Computing this expected utility uses a simulation dynamics model P(y, r, t | x, q, s) to evaluate utilities as in Equation (4). Given these utility values, the optimal action could be chosen, but the introduction of continuous time renders the problem intractable; an approximation is therefore employed via Monte Carlo Tree Search and TD learning (Algorithm 1). Experiments are conducted on a simplified named entity recognition (NER) task, a sentiment recognition task, and a face identification task. Four methods are compared: majority vote of n human judges (n ∈ {1, 3, 5}), online learning (using the true labels), a “threshold” baseline (the authors’ model without continuous time, sending m queries at each step until the model’s predictive uncertainty falls below a threshold), and the full model (LENSE). In terms of precision/recall/F1, the full model outperforms all but the 5-vote crowdsourced setting, although the “threshold” baseline performs nearly as well (and achieves lower latency for NER; Table 2). The authors also show that the method requires fewer crowdsourced labels over time given a sufficiently strong model (Figure 3), and that compared to online learning, accuracy is high from the first example (Figure 4) because the system can leverage crowd expertise when model uncertainty is high.

Overall assessment and significance:
- This is a strong, original, and principled treatment of hybrid learning that goes beyond prior heuristic approaches. The results against reasonable baselines are impressive.
- I expect the work to generate discussion and follow-on research and to see adoption in practice.
- I recommend acceptance.

Clarity and modeling choices:
- The main clarity issue arises with the introduction of continuous time partway through Section 3. The motivation is reasonable—many applications face time as well as cost budgets per classification—but empirically, the benefits appear limited, while the conceptual and algorithmic costs are substantial.
- In Table 2, the “threshold” baseline attains significantly lower latency on NER while achieving roughly equivalent performance (marginally worse). On the face ID task, it actually performs better (though roughly equivalent overall). The authors argue (l. 314) that the baseline performs better on the second task because it is a single-label prediction problem with minimal label interactions. However, the NER task, which does have strong label interactions, seems to benefit only marginally from continuous time.
- For many practical tasks, the “threshold” baseline may be sufficient and already represents a significant advance (as a principled hybrid learning system that outperforms active and online learning). The paper would likely have greater impact if the core method were first developed and evaluated without continuous time—making it easier to understand and implement—with continuous time presented as an extension. When discussing results, it would also help to state more clearly that the “threshold” method performs strongly, even when there is sequence dependence.

Reporting issues and missing baselines:
- The introduction claims the model “beats human performance,” but this is not true for the 5-vote case on NER. Moreover, the 5-vote results are missing for Face-ID and Sentiment in several places (the right half of Table 2, Table 3, and Figure 4); they should be included.
- In Table 2, the 5-vote results should be boldfaced when they represent the best performance. The latency for Threshold in NER should also be boldfaced for the same reason (and the LENSE latency for Face ID).
- The “Threshold” baseline is missing for the sentiment experiment and thus does not appear in Table 3 or Figure 4; it should be included.

Methodological clarifications and minor comments:
- t_deadline usage: While t_deadline is discussed (l. 183–187), it is unclear what, if any, t_deadline was used in the experiments and what criterion determined when to report an answer. Was it an entropy-based threshold as in the baseline, or reaching some t_deadline?
- Equation (2): The distribution should include s, i.e., p(y, r, t | x, q, s).
- Equation (5): The notation F(q) is confusing if q elsewhere denotes a query. Please use a different symbol for the nominal distribution function.
- Threshold baseline (l. 246–252): The expression should be (1 − p_θ(y_i | x)) · 0.3^m ≤ (1 − 0.88) (note the RHS), rather than ≥ 0.88, since increasing m reduces uncertainty.

Conclusion:
This paper offers a compelling, principled framework for combining crowdsourcing with online learning, with strong empirical performance against multiple baselines. I recommend acceptance. To maximize impact and clarity, I encourage the authors to streamline the exposition by foregrounding the non-time-based core method, presenting continuous time as an extension, and correcting the reporting and notation issues listed above.

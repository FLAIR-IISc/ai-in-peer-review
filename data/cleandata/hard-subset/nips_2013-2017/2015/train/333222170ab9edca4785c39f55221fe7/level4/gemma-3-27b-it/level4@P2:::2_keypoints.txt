- Problem: building high-accuracy classifiers without pre-existing labeled data by using real-time crowdsourcing to resolve uncertainty as inputs arrive.  
- Core idea: on-the-job learning framed as a stochastic game using Bayesian decision theory to balance latency, cost, and accuracy.  
- Method: Monte Carlo Tree Search approximation to handle the intractable optimal policy; system named LENSE.  
- Evaluation: demonstrated on named-entity recognition, sentiment classification, and image classification.  
- Novelty: explicitly models latency–cost–accuracy trade-offs and iteratively improves the model during test time, unlike static active classification policies.  
- Empirical results: Table 2 shows LENSE outperforms fully supervised ML and majority-vote human labeling on cost–accuracy trade-offs.  
- Baseline comparisons: 3-vote vs. 5-vote baselines provide insight into crowd consensus vs algorithmic denoising.  
- Practical impact: Figure 4 shows cost reduction over time; open-source code and data improve reproducibility.  
- Related work: builds on online learning, active learning, and crowdsourcing literature but distinguishes itself via its stochastic-game formulation.  
- Weakness — clarity: illustrative example (Line 154) is abstract and hard to follow; should be replaced with a concrete, time-stamped query/response example.  
- Weakness — terminology: inconsistent terms (e.g., “threshold” vs “entropic” baseline) create confusion in Tables 2 and 3.  
- Weakness — organization: related work placed unconventionally in Section 5; should be better integrated into introduction and methodology.  
- Recommendation: overall positive—recommend acceptance contingent on addressing clarity and organization concerns.

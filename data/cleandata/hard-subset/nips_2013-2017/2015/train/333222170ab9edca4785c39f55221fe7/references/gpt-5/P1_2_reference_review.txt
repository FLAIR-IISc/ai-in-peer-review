Summary
This paper proposes “on-the-job learning,” a framework for deploying accurate prediction systems from the start by interleaving model predictions with real-time, asynchronous crowd queries at test time. The system formulates the decision of which labels to query (and when) as a stochastic game between the system and the crowd, with a utility that trades off expected accuracy, monetary cost, and latency. The prediction backbone is a CRF; crowd answers are modeled with a simple noise model and response-time distribution. Since solving for the optimal policy is intractable, the authors approximate it using Monte Carlo Tree Search (UCT) with progressive widening to handle continuous time. Experiments on NER, sentiment classification, and image classification show large cost reductions versus “all-human” approaches and accuracy gains over pure machine-learning baselines and a heuristic thresholding policy. An open-source implementation (LENSE) is provided.

Relation to prior work: The formulation bridges online learning and active learning (but queries are made before committing to predictions on each instance), active classification/feature acquisition at test time (e.g., value-of-information policies), crowdsourcing with decision-theoretic control (e.g., TurKontrol/POMDPs), and real-time crowd systems (e.g., retainer models, Legion:AR). The main novelty is a unified, decision-theoretic, test-time querying framework that updates the model online, handles structured outputs, and reasons jointly about cost and latency via MCTS.

Strengths
- Well-motivated problem: maintaining high accuracy from the outset when training data is scarce, using a principled trade-off among accuracy, cost, and latency.
- Clear formalization: a stochastic game with explicit wait/return/query actions, and a utility grounded in Bayesian decision theory.
- Practical approximate solver: MCTS with progressive widening for continuous time is a reasonable, general approach.
- Structured prediction focus: use of CRFs and demonstration that structure can reduce crowd effort via label correlations.
- Empirical gains: strong cost reductions vs majority-vote crowds; accuracy improvements vs online learning and a nontrivial threshold baseline; code and data released.

Weaknesses and points for clarification
- Utility/metric mismatch: The utility uses ExpAcc of the MAP solution, but evaluations report token-level F1 (NER). F1 is non-decomposable; optimizing expected 0/1 accuracy of MAP does not align with F1. Clarify the loss used in the planner and discuss the mismatch.
- Response model is simplistic: pR assumes a fixed 70% accuracy and uniform confusion, ignoring worker heterogeneity and item difficulty. TurKontrol-like reliability models or per-class confusion matrices would be more realistic and could materially affect policies.
- Learning from noisy responses: It is unclear how pθ is updated online without gold labels. Do you treat crowd responses as gold, do soft updates from posteriors, or use EM? This is central to the “learn on the job” claim and needs to be specified and ablated.
- Computational budget and latency: MCTS parameters (rollout policy, number of simulations, UCT constant, wall-clock per instance) and the resulting planning overhead are not reported. Given the real-time setting, the planner’s latency must be quantified.
- Baselines: Missing comparisons to stronger decision-theoretic test-time policies (e.g., greedy myopic value-of-information, or POMDP policies for crowdsourcing) and to learned active-classification policies. The threshold baseline is relatively weak.
- Experimental setup: Using a frozen pool of five worker responses per label for all methods controls variance but may not reflect realistic worker variability and drift; consider reporting results with live workers or simulated heterogeneity.
- Parallel vs sequential querying: The paper claims the framework naturally trades off these modes, but there is little quantitative analysis of the latency–cost–accuracy frontier under different strategies.
- Clarity issues: Some inconsistencies (Twitter NER vs CoNLL mention), typos, and missing details (definition of Accuracy in utility, per-task pT calibration, choice of wM and wT and their sensitivity).

Quality
Technically sound at a high level, with a principled objective and a reasonable approximate solver. However, important modeling choices (noise model, learning procedure from noisy labels) and planner details are under-specified, and the metric mismatch is concerning. The empirical evaluation is promising but would be stronger with additional baselines, ablations, and runtime analysis.

Clarity
Generally well-written and organized, with an intuitive game description and diagrams. It needs clearer definitions of the utility’s Accuracy, the online learning update rule, MCTS configuration, and sensitivity to wM/wT. A dedicated section on computational costs and a short ablation study would help.

Originality
The combination is novel: a decision-theoretic, test-time, real-time, structured prediction framework that updates the model online, solved via MCTS with continuous time. Related areas are cited; the paper clearly differentiates from online/active learning, active classification, and crowdsourcing POMDPs, though a deeper empirical comparison to those is missing.

Significance
High practical relevance: many applications need high accuracy from day one with limited labeled data and tight latency/cost constraints. The approach and open-source system could spur follow-up work. The simplified noise model and limited baselines weaken the strength of the empirical claim but the demonstrated gains are notable.

Suggestions for improvement
- Align the utility with evaluation: consider expected F1 (approximate) or token-level expected accuracy used in evaluation; discuss implications.
- Model crowd heterogeneity and item difficulty; learn pR from data online; compare to a reliability-aware variant.
- Specify and ablate the online learning update procedure; compare hard vs soft labeling and different update frequencies.
- Report planner runtime, number of simulations, UCT constant, progressive widening parameters, and end-to-end latency impact.
- Add baselines: myopic VOI, a simple POMDP controller, and sequential vs parallel heuristics; include ablations for CRF vs independent labels.
- Run sensitivity analyses for wM and wT; show the Pareto frontier of latency–cost–accuracy.
- Clarify dataset details (Twitter vs CoNLL), utility Accuracy definition, and fix minor typos.

Arguments for acceptance
- Addresses an important and understudied deployment problem with a principled framework.
- Novel integration of structured prediction, real-time crowds, and decision-theoretic planning.
- Solid empirical improvements and open-source release.

Arguments against acceptance
- Utility–metric mismatch and under-specified learning from noisy labels.
- Simplistic crowd model; lack of comparison to stronger decision-theoretic baselines.
- Missing runtime/latency analysis for the planner in a real-time setting.

Recommendation: Weak accept, contingent on addressing the key clarity and evaluation gaps above. Confidence: Medium.

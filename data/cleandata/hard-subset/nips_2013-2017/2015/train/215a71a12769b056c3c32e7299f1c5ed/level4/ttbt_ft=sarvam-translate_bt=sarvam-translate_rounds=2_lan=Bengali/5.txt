I was also thinking of using the LSTML technique to deepen a gradient to deepen a network, but I haven't started working on it yet. Congratulations to the authors for doing this work first!

There might also be a different perspective: a very deep network is not actually that deep, where there are sub-layers. Each "layer" has a more complex non-linear behavior than a traditional rectifier/sigmoid layer. I personally see the beginning of GoogleNet as this, 3 stacked 3x3 etc. in VGG, and I'm wondering if this is a valid way to look at a highway network.

I would suggest the authors do an experiment, training both MNIST and CIFAR using a thin and long network (you can fork for two domains at the top). What I'm interested in is whether, when predicting, the network exhibits the same pattern as a network trained separately for each domain, whether unused model parameters in MNIST are accepted by CIFAR. In other words, I want to see how a very deep classification is shared and whether it is as efficient as possible.

No major complaints. I encourage the authors to try the network on large datasets like ImageNet etc.

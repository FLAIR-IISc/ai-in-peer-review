I was also thinking of using the LSTM technique to make a gradient deeper to make a network deeper, but I haven't started working on it yet. Congratulations to the authors for doing this first!

There might also be a different perspective: a very deep network is not actually that deep, where there are sub-layers of layers. Each "layer" has a more complex non-linear behavior than a traditional rectifier/sigmoid layer. I personally see the beginning of GoogleNet this way, 3 stacked 3x3 in VGG, etc., and I wonder if this is a valid way to look at a highway network.

I would suggest the authors do an experiment, training both MNIST and CIFAR using a thin and long network (you can fork for two domains at the top). What I'm interested in is whether, during inference, the network exhibits the same pattern as a network trained separately for each domain, whether unused model parameters in MNIST are accepted by CIFAR. In other words, I want to see how sharing occurs in a very deep classification and whether it is as efficient as possible.

No major complaints. I encourage the authors to try the network on large datasets like ImageNet, etc.

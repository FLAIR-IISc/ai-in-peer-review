

The authors propose a real-time 3D character control algorithm using a neural network (NN) to generate animations, trained offline to reproduce outputs from Contact-Invariant Optimization (CIO). Crucially, CIO and NN training proceed in alternating steps, with each step's output informing the other's optimization criterion. A key innovation is noise injection via data augmentation (small perturbations) and additive noise in NN hidden layers. Experiments demonstrate the method produces realistic, stable control policies across diverse character models.

This is a compelling approach to a challenging problem: real-time control of arbitrary 3D characters. While the core idea ("learn an NN to predict CIO output") appears straightforward, the joint training and noise injection steps represent significant, well-justified contributions that demonstrably improve generalization. The method’s novelty lies in this synergistic integration, not merely in applying NNs to CIO outputs.

However, the experimental evaluation is the paper’s weakest aspect. The comparative results would be substantially stronger with a larger, more diverse training dataset. The base trajectories (100–200 trials, each with 5 branched segments) are limited, making overfitting in the "no noise" variant unsurprising. Similarly, the "no joint" variant might benefit from greater trajectory diversity, even if less obvious. The paper acknowledges this gap ("A systematic comparison of these more direct methods with trajectory-optimization-based methods remains to be done"), yet the sole comparison is a brief MPC evaluation at the paper’s end, lacking both a proper reference and detailed analysis. A more rigorous benchmark against relevant baselines is essential.

Mathematical exposition is generally clear, though replication would require consulting additional references, particularly for CIO (e.g., undefined notations in Eq. 6). Code availability would greatly benefit the community. One section remains unclear: Section 3.2 on generating optimal actions for noisy inputs. The notation becomes opaque (e.g., the time index *t* is dropped without clarification, and it’s unclear if it remains implied for certain quantities). Additionally, the paper lacks real-time computational measurements—critical for applications like video games with tight CPU constraints. Scaling with 3D model complexity should also be addressed.

Minor points requiring clarification or correction:
- "Biped location" likely refers to locomotion.
- "Unity these disparate approaches" → "unite" or "unify."
- The target appears limited to x,y,z coordinates; adding a target facing angle (practically important) seems feasible but is unaddressed.
- "Noise injection" misrepresents the data augmentation step (where targets are recomputed for perturbed inputs), which is standard data augmentation, not denoising.
- Algorithm 1: Does the first iteration use initial random weights *θ* to compute *X*, or start with *X* from the unregularized case?
- Justification for using the same hyperparameter *η* in both steps of Algorithm 1 is absent.
- "it is non-trivial to adapt them to asynchronous and stochastic setting we have" → "asynchronous and stochastic setting" (add "the").
- LQG is used before definition.
- "While we can use Linear Quadratic Gaussian (LQG) pass" → "a Linear Quadratic Gaussian (LQG) pass."
- Line 231: The star in the argmin is misplaced; \bar{s} may be missing in \tilde{C}.
- "the minimizer of a quadratic around optimal trajectory" → likely missing "the" or "a."
- "sX and aX are subsets of \phiX" → justification for this guarantee is unclear.
- "a single pass over the data to reduce the objective in (4)" → should reference the objective from Algorithm 1 (only *θ* is optimized).
- Section 6’s asynchronous optimization differs significantly from Algorithm 1’s alternating scheme; this distinction merits earlier mention.
- Line 315: *f* should be *f*^{des}.
- "In the current work, the dynamics constraints are enforced softly..." → this sentence is unclear (e.g., "root forces in simulation" requires elaboration).
- The "no noise" variant conflates "no data augmentation" and "no hidden noise injection"; these should be separated.
- A key question: Can *α* be reduced to zero during CIO optimization? This would test if the method relies on curriculum learning or requires trajectory proximity to the network’s capacity.
- Table 1a caption states "4 layers" but describes 3 layers.

Overall, the paper presents a promising application of neural networks to character control. The core technical contributions—joint training and noise injection—are well-motivated and effective in the provided experiments. However, the limited experimental validation, unresolved notational ambiguities, and lack of computational analysis prevent the paper from fully establishing its practical benefits or broader applicability. Strengthening the comparative evaluation and addressing the clarifications above would significantly elevate the work.

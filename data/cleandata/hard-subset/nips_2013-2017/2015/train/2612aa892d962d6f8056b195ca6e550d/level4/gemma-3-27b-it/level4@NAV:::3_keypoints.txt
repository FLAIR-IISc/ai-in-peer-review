- Presents an algorithm for real-time 3D model control using an NN trained offline to reproduce Contact-Invariant Optimization (CIO) outputs.  
- Main novelty is alternating training where CIO and the NN inform each other, plus noise via data augmentation and hidden-layer additive noise.  
- Experiments show the network can generate realistic, stable control policies across diverse character models.  
- Strength: joint training and noise-injection techniques improve generalization beyond a straightforward NN-to-CIO mapping.  
- Weakness: limited training data (100–200 trials with 5 branched segments) raises overfitting concerns, especially for the "no noise" variant.  
- Comparative evaluation is weak—brief MPC comparison lacks citation and more direct method comparisons are needed.  
- Mathematical derivations are mostly clear but some CIO details (e.g., Eq. 6) use undefined notation and require external references to replicate.  
- Section 3.2 is confusing due to unclear notation and omission of the time index t in some quantities.  
- Reproducibility would be improved by releasing code and clarifying algorithmic details (Alg.1 initialization, use of eta, argmin fixes).  
- Missing runtime measurements for real-time control; need to report computation time and scaling with model complexity.  
- Many minor issues: terminology corrections ("locomotion", "unify"), define LQG before use, fix grammar and notation errors, and clarify Table 1a network layer mismatch.  
- Need to clarify variants: split "no noise" into "no data augmentation" vs "no hidden noise injection" and justify hyperparameter choices (single eta).  
- Clarify guarantees/definitions (e.g., sX and aX subset claim), objective in "single pass over the data", and asynchronous vs alternating optimization differences.  
- Suggest investigating whether CIO regularizer alpha can be reduced to zero and provide clearer explanation of dynamics constraint sentence.

- Proposes an algorithm for real-time 3D character control where a neural network is trained offline to reproduce Contact-Invariant Optimization (CIO) outputs.  
- CIO and NN training are performed alternately, with each step’s output influencing the other.  
- Introduces noise via data augmentation (perturbing inputs) and additive noise in hidden layers to improve robustness.  
- Experiments show the network can produce realistic and stable control policies across several character models.  
- Joint training and noise injection are significant contributions that improve generalization beyond a naive NN-to-CIO mimic.  
- Experimental dataset is small (100–200 trials with 5 branched segments), weakening results and likely causing overfitting in the "no noise" variant.  
- Comparative evaluation is weak: only a brief, poorly referenced MPC comparison and no systematic comparison to direct methods.  
- Mathematical exposition is mostly clear but some parts (especially CIO / Eq. 6) lack definitions and need outside references to replicate.  
- Section 3.2 is confusing: notation and dropped time index make the procedure for generating actions for noisy inputs unclear.  
- Providing code would help reproducibility and community adoption.  
- Real-time computation timings and scaling with model complexity are missing but important for practical use (e.g., games).  
- Clarify terminology and text errors: "Biped location" (locomotion), fix "unity" → "unite/unify", and other grammar/punctuation issues.  
- Target seems to be only x,y,z coordinates with no target orientation; discuss adding target angle for practical tasks.  
- The term "noise injection" is misleading for the data-augmentation step where targets are recomputed; better described as "data augmentation."  
- Unclear whether initial random weights θ are used in the first Alg.1 iteration or if X’s are computed without regularization.  
- No justification given for using the same hyperparameter η in both steps of Alg.1.  
- Asynchronous joint optimization (Section 6) differs from alternating Alg.1; this distinction should be mentioned earlier.  
- The "no noise" ablation could be split into "no data augmentation" and "no hidden noise injection" for clarity.  
- Worth investigating whether α can be annealed to zero in CIO, i.e., whether benefits come from curriculum learning or persistent regularization.  
- Table 1a caption incorrectly states the network has 4 layers when it has 3.  
- Overall: promising and well-executed application, but practical benefits, broader applicability, and comparative strength remain to be established.

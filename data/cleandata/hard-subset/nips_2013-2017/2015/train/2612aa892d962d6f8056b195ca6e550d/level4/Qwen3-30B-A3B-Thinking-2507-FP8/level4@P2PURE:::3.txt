

This submission presents a novel approach for real-time control of 3D character models using a neural network (NN) trained offline to replicate the output of Contact-Invariant Optimization (CIO). A key innovation lies in the interdependent training process: CIO and NN training alternate, with each step's output serving as the optimization criterion for the other. Additionally, the method incorporates noise injection via data augmentation (small perturbations) and additive noise in the NN's hidden layers. Experiments demonstrate the network's ability to generate realistic, stable control policies across diverse character models.

The core idea—training an NN to predict CIO outputs—is conceptually straightforward, but the authors' significant contributions (joint training and noise injection) substantially improve generalization, as validated in experiments. This represents a notably compelling approach to a challenging problem: real-time control of arbitrary 3D models.

However, the comparative evaluation is the paper's primary weakness, as acknowledged in the introduction ("A systematic comparison of these more direct methods with the present trajectory-optimization-based methods remains to be done"). The sole comparison to a competing method (MPC) is brief and lacks a reference. Strengthening this would require comparisons across a larger dataset. The base training trajectories (100–200 trials, each with 5 branched segments) are insufficient to prevent overfitting in the "no noise" variant; the "no joint" variant might also benefit from more diverse trajectories.

The mathematical exposition is generally clear, though replication would require additional references, particularly for the CIO step (e.g., undefined notations in Eq. 6). Code availability would be highly valuable. Section 3.2 (generating optimal actions for noisy inputs) contains confusing notation, especially regarding the time index "t" (whether it is implied for certain quantities). Time measurements for real-time control computations are missing—critical for applications like video games with tight CPU constraints—and scaling with model complexity should be addressed.

- Proposal: train a NN offline to reproduce Contact-Invariant Optimization (CIO) outputs, using an alternating CIO ↔ NN training loop where each step guides the other.

- Key innovation: noise injection via data augmentation (perturbed inputs with recomputed targets) plus additive hidden-layer noise to improve generalization.

- Empirical claim: method generates realistic, stable real-time control policies across diverse 3D character models.

- Relevance: highly pertinent to real-time control of arbitrary 3D characters, a challenging application domain.

- Novelty: while learning NN to predict CIO is straightforward, the joint alternating training and noise strategy constitute substantial contributions.

- Limited training data: base trajectories use only 100–200 trials with 5 branched segments, making observed overfitting in "no noise" unsurprising.

- Ablation concern: the "no joint" variant may be underpowered due to insufficiently diverse training trajectories.

- Weak comparisons: evaluation only briefly compares to MPC, lacks proper references and a systematic comparison to direct methods.

- Reproducibility issues: some CIO-related notation (e.g., Eq. 6) is undefined and replication requires consulting external references.

- Code release: making code available would substantially aid reproducibility and community uptake.

- Notational ambiguity: Section 3.2 is unclear about time indices when generating optimal actions for noisy inputs.

- Missing practical metrics: no real-time computation time measurements, unclear scaling with 3D model complexity, and no statement on CPU requirements for applications like games.

- Target specification unclear: unclear whether targets include facing angle constraints (x,y,z only); adding target angle would be practically valuable.

- Misleading terminology: label "noise injection" conflates data augmentation (recomputing targets for perturbed inputs) with denoising-autoencoder-style corruption; these should be distinguished.

- Algorithm 1 clarifications needed: how initial X are computed (random θ or not), and justification for using the same hyperparameter η in both optimization steps.

- Numerous minor corrections and notation fixes required (e.g., "biped location"→"biped locomotion", "unity"→"unify", define LQG, fix line/notation errors, Table 1a layer mismatch).

- Methodological note: asynchronous joint optimization described in Section 6 differs significantly from alternating Algorithm 1 and this distinction should be highlighted earlier.

- Ablation suggestion: separate the "no noise" variant into "no data augmentation" and "no hidden noise injection" to pinpoint effects.

- Experimental suggestion: test decreasing α to zero during CIO optimization to distinguish curriculum effects from necessity of CIO proximity.

- Overall assessment: compelling application of NN to character control, but practical benefits and broader applicability need stronger experimental validation and clarification.

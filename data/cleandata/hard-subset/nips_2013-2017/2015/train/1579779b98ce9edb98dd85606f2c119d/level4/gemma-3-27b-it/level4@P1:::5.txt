This paper presents an algorithm for learning a decision tree with linear split functions at the nodes. The algorithm optimizes the tree parameters—that is, the weights in the split functions and the distribution of the output variable at each leaf—in a global way, as opposed to the standard greedy approach to learning trees. A convex-concave upper bound on the tree’s empirical loss is proposed and regularized with an L2 penalty on the split function weights to encourage a smoother optimization problem. The algorithm then employs stochastic gradient descent to identify the tree parameters that minimize this upper bound.

The tree optimization problem is elegantly formulated by establishing a link to structured prediction with latent variables. However, the empirical results presented do not definitively demonstrate that global optimization provides a substantial advantage over greedy tree learning. The globally optimized tree outperforms greedy trees on only a few datasets, and this improvement comes at a greater computational cost.

The paper is well-structured, but some points could benefit from clarification:

*   The stable version of SGD is not described clearly. How are data points assigned to leaves? Including a pseudo-code would be helpful.
*   Figure 2: Are these results obtained using SGD or stable SGD? What does an “active” leaf mean precisely?
*   In line 10 of Algorithm 1, Θ is updated and then projected onto a simplex. How is this projection problem solved in practice? It would also be valuable to mention that this projection ensures each row of Θ sums to one.

Minor comments/typos:

*   Lines 407–408: You mention tuning “the number of features evaluated when building an ordinary axis-aligned decision tree.” This is unclear, as all features are typically evaluated at each node in a standard decision tree.
*   In equations (7), (8), and (9), “argmax” should be replaced with “max”.
*   Figure 1 is referenced but never discussed in the main text.
*   Line 141: “…to the index of the leaf on by this path.” This sentence is grammatically incorrect.
*   Line 238: “…the solution to loss-augmented inference…” should read “…the solutions to loss-augmented inference…”
*   Line 301: “An key observation…” should read “A key observation…”
*   Line 360: What does FSGD stand for?
*   Line 404: “…a tree with minimum training errors.” I presume you mean “validation” errors?

In the supplementary material:

*   Line 29: `max{g \in sgn...}` should be `max{g = sgn...}`.
*   Line 30: `sgn(Wx)^T Wx + l(\Theta^Tf(g), y)...` should be `sgn(Wx)^T Wx + l(\Theta^Tf(sgn(Wx)), y)`.

Overall, while the problem formulation is insightful and the technical contributions are noteworthy, the empirical results do not strongly support the practical benefits of this approach over existing methods.

**Arguments for acceptance:**

*   Novel and elegant formulation of decision tree learning as a structured prediction problem.
*   Development of a tractable surrogate objective for global tree optimization.
*   Demonstrated feasibility of training deep trees with stochastic gradient descent.

**Arguments against acceptance:**

*   Empirical results are not consistently convincing; the performance gains over greedy methods are limited.
*   The description of stable SGD is unclear, hindering reproducibility.
*   The computational complexity, while improved, remains a concern for very large datasets.

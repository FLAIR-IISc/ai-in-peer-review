- Presents a global optimization algorithm for decision trees with linear split functions, contrasting with standard greedy methods.  
- Proposes an upper bound on empirical loss and L2-regularizes split weights to smooth the optimization landscape.  
- Uses stochastic gradient descent (SGD) to minimize the regularized upper bound and learn tree parameters.  
- Formulates tree learning as structured prediction with latent variables, providing an elegant theoretical link.  
- Paper is well-structured and contains a thorough analysis of the method and solution.  
- Empirical results show the globally optimized tree outperforms greedy trees on several datasets.  
- Stable SGD variant is not clearly described—unclear how data points are assigned to leaves; pseudo-code recommended.  
- Figure 2: unclear whether results use SGD or stable SGD; need to define what an “active” leaf means.  
- Algorithm 1 (line 10): projection of Θ onto the simplex needs practical details (how the projection is solved); note that projection enforces each row sums to one.  
- Results are not uniformly better across datasets and no clear explanation is provided for variability in performance.  
- Method is more computationally intensive than greedy training, raising scalability concerns for large datasets.  
- Several implementation/description details need clarification (stable SGD, projection routine).  
- Typo/clarity: line 407–408 statement about “number of features evaluated when building an ordinary axis-aligned decision tree” is confusing.  
- Equations (7), (8), (9): replace "argmax" with "max".  
- Figure 1 is never referenced in the main text and should be mentioned.  
- Line 141: sentence ending “to the index of the leaf on by this path” is broken and needs fixing.  
- Line 238: change “the solution to loss-augmented inference” to “the solutions to…”.  
- Line 301: fix “An key observation” → “A key observation”.  
- Line 360: define the acronym FSGD.  
- Line 404: likely meant “tuning errors” instead of “minimum training errors.”  
- Supplementary material corrections: line 29 use “max{g = sgn...}” and line 30 correct sgn(Wx)^T Wx term to sgn(Wx)^T Wx + l(Θ^T f(sgn(Wx)), y).  
- Arguments for acceptance: novel algorithm, sound theory, thorough analysis, empirical gains on multiple datasets.  
- Arguments against acceptance: inconsistent improvements across datasets, higher computational cost, and unclear methodological details that need clarification.

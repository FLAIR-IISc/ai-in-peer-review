This paper presents an algorithm for learning a decision tree with linear split functions at the nodes. The algorithm globally optimizes the tree parameters – namely, the weights in the split functions and the distribution of the output variable at each leaf – in contrast to the standard greedy approach to tree learning. An upper bound on the tree’s empirical loss is proposed and regularized with L2 regularization on the split function weights to facilitate smoother optimization. The algorithm then employs stochastic gradient descent to identify the tree parameters that minimize this upper bound.

The tree optimization problem is elegantly formulated by establishing a connection to structured prediction with latent variables. However, the empirical results presented do not strongly demonstrate that global optimization offers a significant advantage over greedy tree learning. The globally optimized tree outperforms the greedy tree on only a few datasets, and this improvement comes at a higher computational cost.

The paper is well-structured, but several points could benefit from clarification:

*   The implementation details of the stable version of SGD are not entirely clear. Specifically, how are data points assigned to leaves? Providing pseudocode would likely be helpful.
*   Figure 2: Are the results shown obtained using SGD or stable SGD? Furthermore, what constitutes an “active” leaf in this context?
*   In line 10 of Algorithm 1, Θ is updated and then projected onto a simplex. How is this projection problem solved in practice? It would also be beneficial to mention that this projection ensures each row of Θ sums to one.

Minor comments/typos:

*   Lines 407-408: The text states that you tune the “number of features evaluated when building an ordinary axis-aligned decision tree.” This is unclear, as all features are typically evaluated at each node during standard decision tree construction.
*   Equations (7), (8), and (9): “argmax” should be replaced with “max.”
*   Figure 1 is never referenced in the main text.
*   Line 141: “…to the index of the leaf on by this path.” The phrasing at the end of this sentence is incorrect.
*   Line 238: “…the solution to loss-augmented inference…” should be “…the solutions to…”
*   Line 301: “An key observation…” should be “A key observation…”
*   Line 360: Please define FSGD.
*   Line 404: “…a tree with minimum training errors.” It is likely that “tuning” errors were intended.

In the supplementary material:

*   Line 29: `max{g ∈ sgn...}` should be `max{g = sgn...}`
*   Line 30: `sgn(Wx)^T Wx + l(\Theta^Tf(g), y)...` should be `sgn(Wx)^T Wx + l(\Theta^Tf(sgn(Wx)), y)`

Overall, the tree optimization problem is elegantly formulated, but the experimental results are not sufficiently convincing to demonstrate a clear advantage over existing methods.

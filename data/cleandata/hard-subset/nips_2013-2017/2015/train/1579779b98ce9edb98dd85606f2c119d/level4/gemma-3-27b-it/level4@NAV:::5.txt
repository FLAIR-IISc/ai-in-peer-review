This paper introduces an algorithm for learning decision trees utilizing linear split functions at each node. The key innovation lies in a global optimization approach to tree parameter learning – encompassing split function weights and leaf-node output distributions – contrasting with the conventional greedy tree-building methods. To facilitate smoother optimization, an upper bound on empirical loss is proposed and regularized using L2 regularization on the split function weights. The algorithm employs stochastic gradient descent to minimize this regularized upper bound and identify optimal tree parameters.

The formulation of the tree optimization problem is noteworthy, establishing a connection to structured prediction with latent variables. However, the empirical evaluation does not strongly demonstrate the advantages of global optimization over greedy tree learning. While the globally optimized tree exhibits significant performance improvements on a limited number of datasets, this comes at the cost of increased computational complexity.

The paper is generally well-structured, but several points require further clarification:

*   The implementation details of the stable SGD variant are unclear. Specifically, the leaf assignment process for data points should be described more explicitly; including pseudo-code would be beneficial.
*   Regarding Figure 2, it is unclear whether the presented results were obtained using standard SGD or the stable variant. Additionally, the definition of an "active" leaf requires clarification.
*   Algorithm 1 updates \Theta and then projects it onto a simplex. Practical details concerning the solution of this projection problem are needed. Mentioning that this projection enforces the constraint that each row of \Theta sums to one would also be helpful.

Minor comments and potential typos include:

*   Lines 407-408 refer to tuning the “number of features evaluated” during standard decision tree construction. This is potentially misleading, as standard trees typically evaluate all features at each node.
*   Equations (7), (8), and (9) should use “max” instead of “argmax.”
*   Figure 1 is referenced nowhere in the main text.
*   Line 141 contains an incomplete sentence: “…to the index of the leaf on by this path.”
*   Line 238 should read “…the solutions to loss-augmented inference…”
*   Line 301: “An key observation…” should be “A key observation…”
*   Line 360: Please define FSGD.
*   Line 404: “…a tree with minimum training errors.” likely refers to “tuning” errors.

In the supplementary material:

*   Line 29: `max{g \in sgn...}` should be `max{g = sgn...}`
*   Line 30: `sgn(Wx)^T Wx + l(\Theta^Tf(g), y)...` should be `sgn(Wx)^T Wx + l(\Theta^Tf(sgn(Wx)), y)`

In summary, while the tree optimization problem is elegantly formulated, the empirical results do not provide a compelling justification for the increased complexity of the proposed global optimization approach.

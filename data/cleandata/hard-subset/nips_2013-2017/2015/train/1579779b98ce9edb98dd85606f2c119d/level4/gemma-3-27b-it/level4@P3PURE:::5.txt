This paper presents an algorithm for learning a decision tree with linear split functions at the nodes. The algorithm optimizes the tree parameters – specifically, the weights in the split functions and the distribution of the output variable at each leaf – in a global manner, contrasting with the standard greedy approach to tree learning. An upper bound on the tree's empirical loss is proposed and regularized with an L2 penalty on the split function weights to facilitate smoother optimization. The algorithm then employs stochastic gradient descent to identify tree parameters that minimize this regularized upper bound.

The tree optimization problem is elegantly formulated by establishing a connection to structured prediction with latent variables. However, the empirical results presented do not strongly demonstrate that global optimization offers a significant advantage over greedy tree learning. The globally optimized tree outperforms the greedy tree substantially on only a limited number of datasets, while incurring a greater computational cost.

The paper is well-structured, but several points require clarification:

- The implementation details of the stable version of SGD are unclear. Specifically, how are data points assigned to leaves? Providing pseudo-code would likely improve understanding.
- Figure 2: It is unclear whether the results presented in this figure were obtained using standard SGD or the stable SGD variant. Furthermore, the definition of an “active” leaf needs to be clarified.
- In line 10 of Algorithm 1, \Theta is updated and then projected onto a simplex. Details on how this projection is solved in practice would be helpful. It would also be beneficial to explicitly state that this projection enforces the constraint that each row of \Theta sums to one.

Minor comments / typos:

- Lines 407-408 state that the authors tune the “number of features evaluated when building an ordinary axis-aligned decision tree.” This is unclear, as standard decision trees evaluate all features at each node.
- In equations (7), (8), and (9), “argmax” should be replaced with “max”.
- Figure 1 is referenced in the paper but not discussed in the main text.
- Line 141: "...to the index of the leaf on by this path." This sentence contains an error at the end.
- Line 238: "...the solution to loss-augmented inference..." should read "...the solutions to...".
- Line 301: "An key observation..." should be "A key observation...".
- Line 360: Please define the abbreviation FSGD.
- Line 404: "...a tree with minimum training errors." likely intends to refer to "tuning" errors.

In the supplementary material:

- Line 29: `max{g \in sgn...}` should be `max{g = sgn...}`
- Line 30: `sgn(Wx)^T Wx + l(\Theta^Tf(g), y)...` should be `sgn(Wx)^T Wx + l(\Theta^Tf(sgn(Wx)), y)`

Overall, the tree optimization problem is elegantly formulated, but the empirical results are not sufficiently compelling to justify the increased computational complexity of the proposed global optimization approach.

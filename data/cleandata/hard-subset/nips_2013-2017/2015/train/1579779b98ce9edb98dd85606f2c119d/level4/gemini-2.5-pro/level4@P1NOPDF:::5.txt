This paper presents a novel algorithm for learning a decision tree with linear split functions at each node. The algorithm’s core idea is to globally optimize all tree parameters—the weights of the split functions and the class distributions at the leaves—simultaneously. This approach contrasts with standard greedy, node-by-node training methods. The authors formulate this problem by proposing a regularized upper bound on the tree's empirical loss, which is then minimized using stochastic gradient descent.

The formulation of the optimization problem, which establishes an elegant connection to structured prediction with latent variables, is a key strength of the paper. However, the empirical results are not sufficiently convincing to demonstrate that this global optimization approach is a superior alternative to traditional greedy learning. The proposed method significantly outperforms the greedy baseline on only a few datasets, despite being more computationally intensive.

The paper is well-structured, but several points require clarification for reproducibility and a full understanding of the method:

*   The description of the "stable SGD" variant is unclear. Specifically, the paper should detail how data points are assigned to leaves during this process. Including pseudocode for this procedure would be very helpful.
*   Regarding the results in Figure 2, it is not specified whether they were generated using the standard SGD or the stable SGD variant. Additionally, the term “active” leaf should be explicitly defined.
*   In Algorithm 1 (line 10), the parameter Θ is updated and then projected onto a simplex. The paper should describe the practical method used to solve this projection problem. It would also be helpful to state explicitly that this projection ensures that each row of Θ (corresponding to a leaf's class distribution) sums to one.

**Minor Comments and Typos:**

*   **Lines 407-408:** The authors mention tuning the "number of features evaluated when building an ordinary axis-aligned decision tree." This is confusing, as standard decision tree algorithms typically evaluate all features at each node to find the best split. Please clarify this procedure. For instance, does it refer to a method similar to that used in Random Forests, where only a subset of features is considered at each split?
*   **Equations (7), (8), (9):** In these equations, "argmax" should be replaced with "max".
*   **Figure 1:** Figure 1 is not referenced anywhere in the main text.
*   **Line 141:** The end of this sentence appears to be incomplete: "...to the index of the leaf on by this path."
*   **Line 238:** "...the solution to loss-augmented inference..." should likely be plural: "...the solutions to..."
*   **Line 301:** "An key observation..." should be "A key observation...".
*   **Line 360:** The acronym "FSGD" is used without a prior definition.
*   **Line 404:** "...a tree with minimum training errors." Given the context of hyperparameter tuning, should this be "validation" or "tuning" error instead?

**In the Supplementary Material:**

*   **Line 29:** `max{g \in sgn...}` should likely be `max{g = sgn...}`.
*   **Line 30:** For consistency within the expression, the term `l(Θ^Tf(g), y)` should probably be written as `l(Θ^Tf(sgn(Wx)), y)`.

In summary, while the paper presents an elegant and principled formulation for globally training decision trees, the experimental evidence is not yet compelling enough to justify its adoption over simpler, greedy methods.

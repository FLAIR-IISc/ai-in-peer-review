This paper introduces an algorithm for learning decision trees using linear split functions at each node. Unlike standard, greedy tree learning approaches, this algorithm optimizes tree parameters – the weights within the split functions and the output variable distribution at each leaf – in a global manner. To facilitate smoother optimization, an upper bound on the tree’s empirical loss is proposed and regularized using L2 regularization on the split function weights. The algorithm then employs stochastic gradient descent to minimize this regularized upper bound and identify optimal tree parameters.

The formulation of the tree optimization problem is elegant, establishing a connection to structured prediction with latent variables. However, the empirical results do not strongly demonstrate the advantage of global optimization over greedy tree learning. While the globally optimized tree exhibits significant performance gains on a limited number of datasets, it comes at a substantial computational cost.

The paper is well-structured, but several points require further clarification:

*   The implementation details of the stable SGD variant are unclear. Specifically, how are data points assigned to leaves? Providing pseudo-code would be beneficial.
*   Figure 2’s results: Were these obtained using standard SGD or the stable SGD method? Additionally, the definition of an “active” leaf should be clarified.
*   Algorithm 1, line 10: The update of \Theta followed by projection onto a simplex is described, but the practical solution to this projection problem is not. Mentioning that this projection enforces a sum-to-one constraint for each row of \Theta would also be helpful.

Minor comments / typos:

*   Lines 407-408: The statement regarding tuning the “number of features evaluated when building an ordinary axis-aligned decision tree” is confusing, as standard decision tree algorithms evaluate all features at each node.
*   Equations (7), (8), and (9): Replace “argmax” with “max”.
*   Figure 1 is referenced in the paper but not discussed in the main text.
*   Line 141: “…to the index of the leaf on by this path.” – This sentence fragment requires correction.
*   Line 238: “…the solution to loss-augmented inference…” – Change to “…the solutions to…”
*   Line 301: “An key observation…” – Change to “A key…”
*   Line 360: Define “FSGD”.
*   Line 404: “…a tree with minimum training errors.” – This likely refers to “tuning” errors.

In the supplementary material:

*   Line 29: `max{g \in sgn...}` should be `max{g = sgn...}`
*   Line 30: `sgn(Wx)^T Wx + l(\Theta^Tf(g), y)...` should be `sgn(Wx)^T Wx + l(\Theta^Tf(sgn(Wx)), y)`

In summary, the tree optimization problem is elegantly formulated, but the experimental results are not sufficiently compelling to justify the increased computational complexity.

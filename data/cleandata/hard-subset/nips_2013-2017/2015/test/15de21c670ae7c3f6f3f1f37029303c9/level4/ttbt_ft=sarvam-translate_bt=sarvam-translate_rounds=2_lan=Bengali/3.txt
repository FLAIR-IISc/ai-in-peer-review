A major problem with the Markov network is weight learning, which is often defined as finding a set of weights (associated with features) that maximize likelihood. Typically, a gradient descent algorithm is used to learn the weights, and a crucial subtask of this algorithm is to calculate the gradient. Since the latter is usually NP-hard, the MCMC algorithm is often used instead of the correct algorithm to calculate it.

This paper proves from theoretical results that if the weights (parameters) are such that MCMC converges to a stable distribution over them quickly, then the weight learning algorithm (which uses MCMC to calculate the gradient) is an FPRS. This paper considers two cases: convex cases and non-convex cases, and creates separate limits for each.

The paper is generally well-written, but there are some errors here and there. I couldn't verify all the evidence, but it seems credible.

Although the theoretical results seem impressive, they have almost no practical relevance (I like that the authors acknowledge this). I think it is practically impossible that the parameters meet the limitations required by the theorems. Have the authors given any thought to this? In particular, is it possible to artificially enforce this condition (this condition can be seen as a form of regularization, and an over-regularized model can be created using ridge regression as well). If this last problem can be solved, you will have a very strong paper with these theoretical results.

Overall, it is a reasonably well-written paper that provides an FPRS for weight learning using MCMC (with and without regularization). However, this condition has almost no practical relevance. The results obtained in the paper need to solve non-trivial research problems before creating a practical/correct algorithm for weight learning.



# Review of "Algorithms with Logarithmic or Sublinear Regret for Constrained Contextual Bandits"

This paper investigates constrained contextual bandits with budget and time constraints, proposing the Adaptive Linear Programming (ALP) algorithm and its UCB-based extension (UCB-ALP) for the unknown reward case. The authors claim to achieve logarithmic regret in non-boundary cases, which they argue is the first such result for constrained contextual bandits. The paper builds on prior work in contextual bandits [1-7] and budgeted bandits [11-16], while addressing the gap in handling observable contexts with budget constraints.

**Strengths:** The paper presents a clear problem formulation and makes a theoretically interesting contribution by showing that constrained contextual bandits can be approximated using linear programming, which avoids the computational complexity of dynamic programming. The ALP algorithm is computationally efficient, and the UCB-ALP extension cleverly combines exploration (via UCB) with the ALP approximation. The paper also provides a rigorous regret analysis for the known-statistics case, which is a significant step toward understanding constrained contextual bandits.

**Weaknesses:** Despite the promising theoretical contribution, the paper contains several critical technical errors and inconsistencies that undermine its main claims.

First, Theorem 1 contains a severe mathematical error. The non-boundary case states: "RALP(T,B) ≤ (u*1 - u*J) / (-e^{-2δ^2})". This expression is mathematically incorrect because the denominator (-e^{-2δ^2}) is negative, making the entire bound negative. Since regret is defined as a non-negative quantity (U* - UΓ), a negative bound is meaningless. The intended bound was likely either (u*1 - u*J) * e^{-2δ^2} or (u*1 - u*J) * e^{2δ^2}, but as written, the theorem is invalid. This is not a minor typo but a fundamental error that invalidates the key theoretical result.

Second, the paper contains multiple formatting inconsistencies. The reference list has numerous entries with trailing commas and missing information (e.g., "In Advances in Neural Information Processing Systems (NIPS),, , ,"). These are not just cosmetic issues but indicate poor attention to detail, which is particularly concerning for a theoretical paper.

Third, Lemma 3 states: "P{ûj,k(t) ≥ ûj',k'(t)|Cj,k(t-1) ≥ `j,k} ≤ 2t^{-1}". The standard UCB bound is O(1/t), not 2/t, making this a technical inaccuracy that weakens the analysis.

Fourth, the paper's notation is inconsistent. The threshold j̃(ρ) is defined in Section 3.1 but not properly redefined when reused in Section 3.2. The cumulative probability qj = ∑_{j'=1}^j πj' is introduced in Section 3 but not explicitly defined in Theorem 1 and Theorem 2, creating confusion.

Fifth, in Section 4.2, Algorithm 1 initializes "ûj,k(0) = 1" for all j and k, but the paper later defines "ûj,k(t) = ūj,k(t) + √(log t / 2Cj,k(t-1))" for Cj,k(t-1) > 0. This creates an inconsistency because when Cj,k(0) = 0, the empirical reward ūj,k(0) is undefined, yet the initialization sets ûj,k(0) = 1.

These issues collectively undermine the paper's technical credibility. While the overall approach is promising, the mathematical error in Theorem 1 is particularly egregious for a theoretical paper. The authors' claim of achieving O(log T) regret in non-boundary cases cannot be trusted when the key theorem supporting this claim is mathematically invalid.

**Recommendation:** I recommend rejection. The paper has a valuable idea but contains critical technical errors that invalidate its main theoretical contribution. The mathematical error in Theorem 1 is not a minor oversight but a fundamental flaw that would require substantial revision to correct. Given the stringent quality standards of NIPS, these issues outweigh the paper's potential contribution. The authors should address these technical errors before resubmitting to a venue that values precision in theoretical work.

**Pro-con summary:**
- Pro: Clear problem formulation, computationally efficient algorithms, promising theoretical approach
- Con: Critical mathematical error in key theorem, multiple technical inconsistencies, formatting issues in references

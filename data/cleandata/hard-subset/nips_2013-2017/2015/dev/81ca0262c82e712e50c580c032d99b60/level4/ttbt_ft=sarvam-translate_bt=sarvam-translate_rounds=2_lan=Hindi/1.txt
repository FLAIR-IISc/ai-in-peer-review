This paper presents a new method for path integral control. The proposed method takes advantage of the fact that the system control matrix G is often known, and the uncontrolled dynamics (or the dynamics under a reference controller) can be learned using a Gaussian process. The restriction on the reward function takes on a more general shape compared to previous PI approaches, meaning that noise and control can operate in different subdomains. The authors also show how their framework can be used to generalize known to new tasks, and evaluate the method on three simulated robotics problems, where their method is compatible with SOTA reinforcement learning and control methods.

Quality

Generally, the derivation seems correct and theoretical (although I am not sure about the task generalization, see below). The related tasks are discussed sufficiently, the authors indicate the similarity and difference of the related task, and compare with the state of the art. The experiments are reassuring, although some details are missing (see below). The task generalization seems strange: (13) states that the exponent of the reward function will be linearly combined to generate the exponent of the reward function of the new task. However, it is not clear to me how this changes the exponent of the value function in the same linear combination (14). In any case, it should be explained, but I think there may actually be an error here ((14) also states that psi is the exponent of the cost function, which is only for the last time step as shown in the equation around line 247).

The task generalization defined by the authors is specific to the case in which the task is defined by a goal, although it seems that other task variations will be possible as long as a suitable task similarity kernel can be defined. The experiments consider relevant and realistic tasks, and compare with SOTA methods. The technical details are a bit sparse in the experimental section: the paper should mention what the dynamics and reward function are, and how long the sample trajectories are, so that the experiment can be reproduced (possibly in an appendix). Is a baseline repeated in the PI control table 1 of one of the methods? It would be insightful to see how it compares on these qualitative aspects.

Error bars should be shown in the comparative plots in figures 1 and 2. The reported score value is the exponent of fc, psi. Is this psi that the robot has calculated? What if the robot is overly optimistic in calculating psi? Reporting the average (or cumulative (discounted) reward) would be more objective (i.e. the algorithm also determines the optimization for the same).

Clarity

Generally, the paper is well-written and explains the proposed methods well. Comparison with other methods, for example in table 1, helps to understand the relationship with SOTA. There are some grammatical and formatting errors, see below under small dots. A confusing point is that in GP (2) (line 094) and line 208 there are different regression goals, which do not include the reference control. This should be consistent or explained. The authors should state how the hyperparameters of GP are set. It is not clear what is meant by "the subsequent distribution can be obtained by restricting the joint distribution" - does conditioning mean this?

Algorithm 1 is confusing - here, it seems that an open-loop control sequence has been optimized while the rest of the paper discusses feedback controller learning. If it is actually just an open-loop sequence u1 ... uT that returns to line 13 of the algorithm, how can the algorithm deal with noise? This should be clarified or corrected.

Originality and Importance

Although the main elements of the proposed algorithm have been used in previous algorithms (through a learned stochastic forward model as in [15], path integral control), the algorithm combines these in a new way. As far as I know, this is an original formulation that achieves very good performance. By evaluating on realistic problems against strong benchmarks, the proposed algorithm can be considered a significant improvement over previous work.

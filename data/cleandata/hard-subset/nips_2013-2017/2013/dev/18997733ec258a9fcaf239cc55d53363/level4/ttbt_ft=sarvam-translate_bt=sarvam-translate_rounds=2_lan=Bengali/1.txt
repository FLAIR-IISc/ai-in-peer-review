This paper considers an extension of the linear bandit algorithm, i.e., linear bandit with graph information. For example, a recommendation system might have access to a social network where users are included, and it is reasonable to assume that users who are connected together may have interests. Each user can be modeled as a linear bandit problem, with these bandits connected by a graph. The graph represents the degree of similarity between the bandits' parameters.

The authors assume that the graph is familiar with the algorithm, that at each time t, the algorithm receives a user ID and a context, and accordingly, a recommendation must be made. They propose an algorithm called GOB.lin, which solves this problem based on LinUCB. The difference with this last algorithm is that this algorithm uses graph information to use similarity between users in the recommendation system. The authors provide a limit on the regret of this algorithm and provide some numerical tests that are promising and credible.

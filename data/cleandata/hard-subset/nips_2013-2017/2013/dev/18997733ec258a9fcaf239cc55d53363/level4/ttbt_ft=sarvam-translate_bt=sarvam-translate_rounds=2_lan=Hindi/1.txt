*Old review without comments on Thm 1*

This paper considers an extension of the linear bandit algorithm, i.e., a linear bandit with graph information. For example, a recommendation system may have access to a social network to which users belong, and it is reasonable to assume that users connected together can share interests. Each user can be modeled as a linear bandit problem, these bandits are connected together by a graph. The graph represents the level of similarity between the parameters of the bandit.

The authors assume that the graph algorithm is known, that at each time t the algorithm receives a user ID and a reference, and it has to recommend a side accordingly. They propose an algorithm, GOB.lin, which solves this problem based on LinUCB. The difference about this final algorithm is that this algorithm uses graph information to use similarity between users of the recommendation system. The authors provide a limitation on the regret of this algorithm, and some numerical experiments that are promising and reassuring.

Also, it would be good if you could add some more intuition to your algorithm.

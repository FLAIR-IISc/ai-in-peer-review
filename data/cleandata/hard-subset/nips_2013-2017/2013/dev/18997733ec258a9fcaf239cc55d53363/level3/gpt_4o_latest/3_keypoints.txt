- Introduces contextual linear bandits on graphs, with GOB.Lin sharing updates across nodes via the graph Laplacian.
- Proposes scalable clustering variants (GOB.Lin.MACRO, GOB.Lin.BLOCK) to handle graph noise and computational costs.
- Experiments on synthetic, Last.fm, and Delicious datasets show graph-aware methods outperform non-relational baselines.
- Strength: Novel, practically relevant extension for recommendation-style applications.
- Strength: Well-executed, reproducible experiments using both synthetic and real-world data.
- Strength: Scalability addressed through clustering-based algorithm variants.
- Strength: Empirical results convincingly demonstrate performance gains from using graph structure.
- Weakness: Regret analysis is flawed—analyzes a different algorithm than the one proposed, with no justification.
- Weakness: Key terms in the regret bound (e.g., ln|M_t|, 2L) are not explained relative to T, n, and graph noise.
- Weakness: No guidance on choosing the exploration parameter α or its effect on regret.
- Weakness: Inconsistent regularization—clustering used as regularizer while regret bound uses L(u1,…,un) instead of a regularized Laplacian.
- Weakness: Vague wording (e.g., "somehow informatively", "similar updates") reduces precision and rigor.
- Pro: Strong empirical and scalability contributions; Con: Theoretical inconsistencies undermine scientific rigor.
- Recommendation: Accept only if authors address theoretical flaws and clarify key terms in a revision.

This work introduces a method for automatically discovering latent domains within a training dataset, with the goal of improving classification performance on a test set through domain adaptation. The paper defines a kernel-based function to quantify the dissimilarity between feature vectors and aims to partition the data into domains that maximize this function across domain pairs. This partitioning is formulated as an integer programming problem subject to two constraints: each data point belongs to exactly one domain, and the class label distribution within each domain matches the overall distribution of the dataset. To facilitate solving this problem, it is relaxed to a continuous optimization problem with a quadratic cost function and linear constraints, and the optimal number of domains is determined via cross-validation.

The proposed approach is evaluated on two datasets: static images (dataset [2]) and the IXMAS multi-view action dataset (dataset [15]). A key result is the demonstrated improvement in performance compared to the domain adaptation method presented in [19].

**Strengths:** The paper is well-written and, to the best of my knowledge, presents a novel approach to domain adaptation. The performance gains over the method in [19] are also a positive aspect.

**Weaknesses:** I currently lean towards rejection, but would be open to changing my assessment based on the rebuttal. I have two primary concerns:

(i) The motivation for this work remains unclear. The authors state on line 77 that “simply clustering images by their appearance is prone to reshaping datasets into per-category domains.” However, the evidence supporting this claim is lacking. Furthermore, it is unclear how the model formulation in Section 2 prevents this issue – specifically, how the formulation avoids partitioning the data into domains based on individual categories. 

(ii) Relatedly, the necessity of the “label prior constraint” described on line 154 is not sufficiently justified. I am interested in understanding the performance of the method without this constraint. A baseline comparison using k-means clustering or other unsupervised object discovery techniques (e.g., Sivic et al. ICCV '05) applied to the appearance vectors would be valuable. Additionally, the performance achieved with a random partitioning of the dataset into equally sized sets should be reported.

**Further Comments:**

+ Line 82, “maximally different in distribution from each other”: This phrasing is used repeatedly throughout the paper. Clarification is needed regarding what is meant by “distribution” in this context.
+ Line 166: It appears that β<sub>mk</sub>, as formulated, already resides on a simplex.
+ Line 182: These constraints appear to differ from those initially presented on line 153.
+ Line 215: The justification or proof for the stated bound would be appreciated.
+ Line 289: Please provide more details regarding the implementation of the geodesic flow kernel [4]. Was it reimplemented, or was publicly available source code utilized?
+ Lines 313/340: Please provide further insights into the observed performance differences. I would like to better understand *why* the proposed approach outperforms the baselines, and would find it helpful to see examples of systematic failures in the baselines that are addressed by the proposed method.
+ Eq (1): M’<sub>k</sub> should be Mk’.
+ The following citation may be relevant: Torralba, A., & Efros, A. A. (2011). Unbiased Look at Dataset Bias. *IEEE Conference on Computer Vision and Pattern Recognition (CVPR)*. 

The rebuttal effectively addressed my concerns regarding the paper’s motivation and the purpose of the label prior constraint, leading me to slightly favor acceptance.

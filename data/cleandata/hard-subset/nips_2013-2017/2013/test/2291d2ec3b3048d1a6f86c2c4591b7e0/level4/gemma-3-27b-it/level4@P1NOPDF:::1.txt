The goal of this work is to automatically discover latent domains within a training set, which are then leveraged in a domain adaptation framework to improve classification performance on a test set. The paper defines a function to measure the difference between two feature vectors using a specified kernel. The objective is to partition the data points into domains such that this function is maximized across all pairs of domains. This problem is formulated as an integer programming problem subject to two constraints: each data point is assigned to exactly one domain, and the distribution of class labels within each domain matches the overall class distribution of the entire dataset. This integer program is relaxed into a continuous optimization problem with a quadratic cost function and linear constraints. Finally, the optimal number of domains is determined via cross-validation.

The approach is evaluated on two datasets: static images from [2] and the IXMAS multi-view action dataset [15]. A key result is that the method demonstrates improved performance compared to the domain adaptation approach presented in [19].

**Positives:** The paper is well-written, and to the best of my knowledge, the approach is novel (although I am not an expert in domain adaptation). The performance gains over [19] are also encouraging.

**Negatives:** At this point, I slightly lean towards rejection. I have two main concerns that I would like to see addressed in the rebuttal, which could potentially change my assessment:

(i) The motivation for this work isn’t entirely clear. On line 77, the paper argues that “simply clustering images by their appearance is prone to reshaping datasets into per-category domains.” First, what evidence supports this claim? Second, how does the model formulation in Section 2 prevent this issue – specifically, how does it avoid partitioning the data into domains based solely on category?

(ii) Relatedly, on line 154, why is the second constraint (“label prior constraint”) necessary? I’m curious about the performance without this constraint. In fact, a baseline comparison using k-means clustering or unsupervised object discovery (e.g., Sivic et al. ICCV '05) on the appearance vectors would be valuable. Additionally, it would be helpful to see the performance when the dataset is randomly partitioned into equally sized sets.

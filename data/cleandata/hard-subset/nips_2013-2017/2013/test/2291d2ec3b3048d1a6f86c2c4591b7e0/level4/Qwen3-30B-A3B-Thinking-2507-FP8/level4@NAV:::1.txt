

The goal of this work is to automatically identify latent domains within a training set for use in a domain adaptation framework, aiming to enhance classification performance on a test set. The paper introduces a kernel-based function measuring the dissimilarity between feature vectors, with the objective of partitioning data points into domains such that this function is maximized across all domain pairs. The problem is formulated as an integer program with two constraints: (1) each point is assigned to exactly one domain, and (2) the class label distribution within each domain matches the overall input distribution. This is relaxed to a continuous quadratic optimization problem with linear constraints, and the optimal number of domains is determined via cross-validation.  

The approach is evaluated on two datasets: static images from [2] and the IXMAS multi-view action dataset from [15]. A key contribution is improved performance over the domain adaptation method in [19].  

Positives: The paper is well-written, and the approach appears novel (though I am not a domain adaptation expert). The performance gains over [19] are noteworthy.  

Negatives: I currently lean toward rejection. Two primary concerns require resolution in the rebuttal to potentially alter my score:  
(i) The motivation lacks clarity. Line 77 claims "simply clustering images by appearance is prone to reshaping datasets into per-category domains." What evidence supports this? How does the model in Section 2 prevent reshaping into per-category domains?  
(ii) The necessity of the "label prior constraint" (Line 154) is unclear. What performance is achieved without it? A baseline using k-means or unsupervised object discovery (e.g., Sivic et al., ICCV '05) on appearance vectors should be included. Additionally, performance with random equal-sized partitions should be reported.

The goal of this work is to automatically discover latent domains in a training set, which are used in a domain adaptation framework to achieve improved classification performance in a test set. This paper defines a function that measures the difference between two feature vectors on a given kernel. The goal is to partition the data points into domains in such a way that the function is maximized over each domain's set of points. The problem is formulated as an integer programming problem with two constraints: each point is assigned to exactly one domain, and the distribution over each domain's class label must match the input distribution over the entire set of points. The problem is relaxed to a continuous optimization over a quadratic cost with a linear constraint. Finally, the number of domains is obtained through cross-validation.

This method has been evaluated on two datasets: the static image of [2] and the IXMAS multi-view action dataset of [15]. A notable point is that it showed improved performance compared to the domain adaptation method of [19].

Pros: The paper is well-written, and the method is new as far as I know (although I am not an expert in domain adaptation). The performance increase compared to [19] is also commendable.

Cons: At the moment, I am leaning slightly towards rejection. I have two main concerns that I would like to see addressed that could convince me to change my score:

(i) The purpose of this paper is not clear to me. In line 77, the paper argues that "clusters of images grouped by appearance alone can be transformed into each-class domains". Firstly, what is the proof for this claim? Secondly, how does the model framework in section 2 overcome this problem, i.e., how is it ensured that it is not transformed into each-class domains?

(ii) Slightly related, why is the second constraint ("label prior constraint") necessary? I am curious what the performance would be like without this constraint. In fact, a baseline should be shown where the data is partitioned into appearance vectors using k-means clustering or unsupervised object detection (e.g., Sivic et al ICCV '05). Also, what is the performance when the dataset is randomly partitioned into equal sets?

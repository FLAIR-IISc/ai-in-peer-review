

The paper proposes a method to automatically discover latent domains within training datasets, which are then utilized in domain adaptation to improve classification performance on test sets. The approach defines a kernel-based function to measure distribution differences between feature vectors, with the goal of partitioning data points into domains that maximize this function across all domain pairs. The problem is formulated as an integer programming problem with two constraints: each data point must be assigned to exactly one domain, and the class label distribution within each domain must match the overall class distribution. This is relaxed to a continuous optimization problem with quadratic cost and linear constraints. The optimal number of domains is determined through domain-wise cross-validation.

The method is evaluated on two datasets: object recognition using the Caltech-256, Amazon, DSLR, and Webcam datasets from [2], and human activity recognition using the IXMAS multi-view action dataset from [15]. The results demonstrate improved performance compared to the domain adaptation approach in [19].

Strengths: The paper is well-written and appears novel within the domain adaptation literature. The reported performance gains over [19] are significant and well-documented.

Weaknesses: I currently lean toward rejection due to two main concerns that would need to be addressed in the rebuttal:

(i) The motivation for the paper is unclear. On line 77, the authors claim that "simply clustering images by their appearance is prone to reshaping datasets into per-category domains." I require evidence supporting this claim and clarification on how the proposed formulation specifically avoids this issue (i.e., how it prevents reshaping into per-category domains).

(ii) The necessity of the label prior constraint (LPC) on line 154 is not well-justified. I would like to see performance comparisons without this constraint, as well as baselines using k-means clustering or unsupervised object discovery (e.g., Sivic et al. ICCV '05) over appearance vectors, and random partitioning into equal sets.

Additional comments:
+ Line 82: "maximally different in distribution from each other" should clarify what distribution is being referenced (distribution over what features or variables?).
+ Line 166: The definition of βmk already implies it lives on a simplex, so this statement seems redundant.
+ Line 182: The constraints appear different from those formulated on line 153, which needs clarification.
+ Line 215: The justification for the bound Kmax ≤ min{M/(NC),C} requires additional explanation or proof.
+ Line 289: More details are needed about the implementation of the geodesic flow kernel [4]—was this a reimplementation or did the authors use publicly available code?
+ Lines 313/340: The authors should provide deeper insights into why their approach outperforms baselines, including systematic failure analysis of the competing methods.
+ Equation (1): M'k should be Mk'.
+ Consider including the reference: "Unbiased Look at Dataset Bias" by Torralba and Efros (CVPR 2011).

The rebuttal addressed my concerns regarding the paper's motivation and the label prior constraint, leading me to lean slightly toward acceptance.



The paper aims to automatically discover latent domains within a training set for use in a domain adaptation framework, thereby improving classification performance on a test set. It defines a kernel-based function measuring the difference between feature vectors, with the goal of partitioning data points into domains to maximize this function across all inter-domain pairs. The problem is formulated as an integer program with two constraints: each point must be assigned to exactly one domain, and the class label distribution within each domain must match the overall input distribution. This integer program is relaxed to a continuous quadratic optimization problem with linear constraints, and the optimal number of domains is determined via cross-validation.

The approach is evaluated on two datasets: static images from [2] and the IXMAS multi-view action dataset from [15]. A key contribution is improved performance over the domain adaptation method in [19].

Positives: The paper is well-written, and to the best of my knowledge, the approach is novel (though I am not a domain adaptation expert). The performance gains over [19] are also noteworthy.

Negatives: I lean slightly toward rejection. Two main concerns require clarification in the rebuttal to potentially change my score:
(i) The motivation is unclear. On line 77, the paper claims "simply clustering images by their appearance is prone to reshaping datasets into per-category domains." What evidence supports this claim? How does the model formulation in Section 2 prevent reshaping into per-category domains?
(ii) On line 154, why is the "label prior constraint" necessary? What performance is achieved without this constraint? A baseline using k-means clustering or unsupervised object discovery (e.g., Sivic et al., ICCV '05) on appearance vectors should be included. Additionally, what is the performance when the dataset is randomly partitioned into equal-sized sets?

This work introduces a method for automatic latent domain discovery in training data, with the aim of improving classification performance on a test set via domain adaptation. The paper defines a kernel-based function to quantify the dissimilarity between feature vectors and formulates the domain partitioning problem as maximizing this function across domain pairs. This is modeled as an integer program, subject to constraints ensuring each data point belongs to a single domain and that class label distributions within each domain match the overall input distribution. The integer program is then relaxed to a continuous quadratic optimization problem with linear constraints, and the optimal number of domains is determined through cross-validation.

The proposed approach is evaluated on two datasets: static images ([2]) and the IXMAS multi-view action dataset ([15]). Notably, the method demonstrates performance improvements compared to the domain adaptation technique presented in [19].

**Strengths:** The paper is clearly written and, to the best of my knowledge, presents a novel approach to domain adaptation (though I acknowledge I am not a domain adaptation specialist). The observed performance gains over [19] are encouraging.

**Weaknesses:** Currently, I am inclined to reject this paper, but would consider revising my assessment based on a strong rebuttal addressing the following concerns:

(i) The motivation for this work remains unclear. The authors state (line 77) that simple clustering by appearance can lead to per-category domains. However, supporting evidence for this claim is lacking. Furthermore, it is unclear how the proposed model formulation (Section 2) specifically prevents this undesirable outcome.

(ii) Relatedly, the necessity of the “label prior constraint” (line 154) is not justified. I would like to understand the performance impact of removing this constraint. A comparison against baselines employing k-means clustering or unsupervised object discovery methods (e.g., Sivic et al. ICCV '05) on the appearance vectors would be valuable. Additionally, reporting performance with a random partitioning of the dataset into equal sets would provide a useful reference point.

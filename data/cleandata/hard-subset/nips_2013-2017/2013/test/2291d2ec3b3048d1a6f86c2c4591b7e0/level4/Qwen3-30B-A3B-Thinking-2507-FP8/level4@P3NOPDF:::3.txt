

This paper introduces a convex framework for partitioning datasets into K "latent domains" with two key properties: (1) each domain exhibits similar semantic class distributions, and (2) domains are maximally distinct from one another via pairwise mean differences in a kernel-induced RKHS. The number of latent domains K is determined by maximizing average within-domain classification accuracy, ensuring sufficient samples per class for effective learning. While the core idea of latent domain discovery is not novel, the paper makes two notable contributions: (1) explicitly controlling class label distributions within domains to prevent class dominance, and (2) using within-domain classification quality to determine K—a pragmatic approach ensuring domains are learnable.

The method’s assumption that class distributions should be similar *within* each latent domain warrants critical discussion. The authors’ example (poses as latent domains for animal classification) effectively highlights a potential limitation: class distributions may not align with the latent structure. For instance, in a horse/cow classification task, both classes might share common poses (e.g., left-right standing), but horses rarely appear in sitting poses while cows do. This implies the method might fail to identify pose as a meaningful latent domain, as the class distribution *across* poses (e.g., horses: standing=90%, sitting=10%; cows: standing=60%, sitting=40%) would violate the "similar distribution" constraint. This is a significant point that should be addressed in the discussion.

The paper’s conceptual clarity requires substantial improvement. The terms "dataset," "domain," and "latent domain" are frequently conflated, causing confusion. For example, Section 4.2 refers to "source domains" (S_i) while simultaneously calling them "datasets," yet the experimental setup (e.g., "source datasets S_1,...,S_K") remains ambiguous. The authors must explicitly define these terms early (e.g., "a *dataset* is a labeled collection of samples; a *domain* is a distributional shift between datasets; a *latent domain* is a partition of a dataset into subsets with homogeneous class distributions") and illustrate them with concrete examples. Additionally, Equation (7) references "max_k r(U_k, B)" without defining r(·,·) or B, which undermines reproducibility. These ambiguities must be resolved before publication.

The experimental validation is generally adequate but lacks justification for the chosen evaluation protocol. The results show reasonable improvements over baselines, and qualitative examples help visualize the latent domain partitions. However, the paper does not justify why specific source/target dataset pairs (e.g., Office-31) were selected over alternatives like "leave-one-domain-out" evaluation, which would better assess generalization to unseen domains. Without this, the reported gains may not reflect robustness to domain shifts.

A minor but important technical point: Line 74 labels the problem as "unsupervised learning," which is misleading. The method explicitly uses semantic class labels to enforce similar class distributions within latent domains, making it *semi-supervised*. This mischaracterization should be corrected.

Overall, the paper presents a technically sound convex approach to latent domain discovery with a novel mechanism for controlling class distributions and determining K. The contribution is significant for tasks where class balance within subgroups is critical. However, the current manuscript suffers from conceptual imprecision, inadequate justification of experimental choices, and unresolved notation. Addressing these issues is essential for clarity and impact.

**Recommendation:** Major Revision. The core idea is promising, but the paper requires substantial rewriting to clarify terminology, justify experimental design, and correct technical inaccuracies. The authors should: (1) redefine key terms with examples early in the paper; (2) revise Section 4.2 to unambiguously distinguish "dataset," "domain," and "latent domain"; (3) define all notation (e.g., r(·,·) in Eq. 7); (4) justify the dataset selection for evaluation; and (5) correct the "unsupervised" misstatement. With these revisions, the paper would make a valuable contribution.



The paper proposes a method for automatically discovering latent domains in a training set, which are then leveraged within a domain adaptation framework to improve classification performance on a test set. The approach defines a kernel-based function measuring the difference between feature vectors across domains, with the goal of partitioning data points into domains such that this function is maximized across all domain pairs. The problem is formulated as an integer program with two constraints: (1) each point is assigned to exactly one domain, and (2) the class label distribution within each domain matches the overall input distribution. This integer program is relaxed to a continuous quadratic optimization problem with linear constraints, and the number of domains is determined via cross-validation.  

The method is evaluated on two datasets: static images from [2] and the IXMAS multi-view action dataset from [15]. A key strength is the demonstrated performance improvement over the domain adaptation approach in [19].  

**Positives:** The paper is well-written and appears novel within the domain adaptation literature (though the reviewer is not a domain adaptation expert). The performance gains over [19] are noteworthy.  

**Negatives:** The reviewer currently leans toward rejection. Two critical concerns must be addressed in the rebuttal to justify a change in score:  
(i) The motivation remains unclear. On line 77, the paper claims "simply clustering images by their appearance is prone to reshaping datasets into per-category domains." First, what evidence supports this claim? Second, how does the model formulation in Section 2 prevent reshaping into per-category domains?  
(ii) The rationale for the second constraint ("label prior constraint") is insufficiently justified. What performance degradation occurs without this constraint? A baseline comparing the method to k-means clustering or unsupervised object discovery (e.g., Sivic et al., ICCV 2005) over appearance vectors should be included. Additionally, performance under random partitioning into equal-sized sets should be reported.

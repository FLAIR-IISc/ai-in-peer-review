

The paper proposes an approach to automatically discover latent domains within a training set for improved domain adaptation. It defines a kernel-based function measuring the distributional difference between feature vectors, aiming to partition data into domains where this difference is maximized across all domain pairs. The problem is formulated as an integer program with two constraints: each point assigned to exactly one domain, and each domain's class label distribution matching the overall training set distribution. This is relaxed to a continuous quadratic optimization with linear constraints, and the optimal number of domains is determined via cross-validation.

The method is evaluated on two datasets: static images from [2] and the IXMAS multi-view action dataset from [15]. A key strength is the demonstrated performance gain over the domain adaptation approach in [19].

**Positives:** The paper is well-written and, to the best of my knowledge, presents a novel approach within domain adaptation. The performance improvements over [19] are significant and well-supported.

**Negatives:** My initial inclination is to reject, pending clarification of two critical issues in the rebuttal. Addressing these would be necessary to justify a change in score:

(i) **Motivation and formulation:** The claim on line 77 that "simply clustering images by appearance is prone to reshaping datasets into per-category domains" lacks supporting evidence. Crucially, the paper does not explain *how* the proposed formulation (Section 2) prevents this reshaping. Without this, the motivation for the specific constraints remains unclear.

(ii) **Necessity of the label prior constraint:** The second constraint (line 154) is not adequately justified. It would be valuable to see performance comparisons with: (a) a baseline using k-means or unsupervised object discovery (e.g., Sivic et al., ICCV '05) on appearance features; (b) random partitioning into equal-sized sets; and (c) the model without the label prior constraint. This would clarify whether the constraint is essential or merely an artifact of the formulation.

**Additional Comments:**

*   Line 166: β_{mk} is defined as a distribution; does it not already lie on a simplex by construction?
*   Line 182: These constraints appear inconsistent with those formulated starting on line 153. Please reconcile.
*   Line 215: Provide justification or proof for the stated bound.
*   Line 289: Detail the use of the geodesic flow kernel [4]. Was this reimplemented, or was public code used?
*   Lines 313/340: Offer deeper insights into performance differences. What specific failures of baselines (e.g., [19]) does the proposed approach overcome? A systematic analysis would strengthen the claims.
*   Equation (1): Correct M'k to Mk'.

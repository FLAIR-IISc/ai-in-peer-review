This paper proposes a novel two-step learning method (TSL) for cross-language text classification. The core problem is that documents in different languages exist in disjoint feature spaces (vocabularies). The authors' approach is to bridge this gap by leveraging a small set of unlabeled parallel documents. First, they construct a single, large document-term matrix that includes all documents (labeled source, labeled target, unlabeled parallel, etc.) and the unified vocabulary of both languages. This matrix is inherently incomplete, as a monolingual document has missing values for all terms from the other language. The first step of the proposed method is to "complete" this matrix by solving an optimization problem that seeks a low-rank and sparse solution, using a projected gradient descent algorithm for which a convergence proof is provided. The second step is to apply Latent Semantic Indexing (LSI) to this completed matrix to learn a low-dimensional, dense cross-lingual representation. A standard classifier (SVM) is then trained on this representation using labeled data from both source and target languages. The proposed approach is related to prior work like CL-LSI, CL-KCCA, and CL-OPCA, which also learn shared representations from parallel data, but TSL is distinct in its explicit matrix completion pre-processing step.

**Strengths:**
*   The proposed method is conceptually simple and presents an intuitive way to unify feature spaces: instead of projecting into a shared space, it "fills in" the missing language representation for each document.
*   The experimental results are strong and comprehensive, showing that TSL consistently outperforms four other methods (including a target-only baseline and three strong cross-lingual representation learning methods) across 18 different tasks.
*   The method demonstrates particular strength when the amount of parallel data is limited (e.g., with only 200 parallel documents), which is a significant practical advantage.
*   The authors provide a convergence guarantee for their projected gradient descent algorithm, adding theoretical soundness to their optimization procedure.

**Weaknesses and Questions for Clarification:**
The paper is generally clear, but several technical and experimental details require clarification to fully assess the contribution.

1.  **Motivation for the LSI Step:** After estimating the completed matrix M*, each document has a representation in the unified vocabulary space. Why is the second LSI step necessary? Is it because M* becomes relatively dense despite the sparsity-enforcing L1 norm, making it computationally expensive for the SVM? Or is it primarily for noise reduction and to capture higher-order semantics, as is standard for LSI? Clarifying this motivation would be helpful. Furthermore, why was LSI chosen over alternatives like PCA?

2.  **Experimental Setup and Use of External Data:** The experimental setup could be described more clearly.
    *   Please confirm the exact data splits. My understanding is: training uses 4k labeled source docs and 100 labeled target docs; representation learning uses all of these plus 2k unlabeled parallel docs and the remaining 1.9k target docs (as unlabeled); testing is on the 1.9k target docs. Is this correct?
    *   The paper uses two forms of external data to help the target task: (1) labeled source-language data and (2) unlabeled parallel data. The current experiments conflate the benefits of both. It would strengthen the paper to explicitly distinguish their contributions. For example, how does TSL perform against a baseline that uses only the source labeled data (e.g., via machine translation) without any parallel documents?
    *   Confirming Section 5.3: do these experiments vary the number of unlabeled parallel documents from 200 to 2000, as it appears?

3.  **Inconsistent Results:** There appears to be an inconsistency between Figure 1 and Table 1. For tasks EFM and EGM with 2000 unlabeled documents, the plots in Figure 1 show CL-KCCA having performance nearly identical to (or even slightly better than) TSL. However, in Table 1, TSL shows a clear performance advantage over CL-KCCA for these same two tasks (e.g., 81.38 vs 78.43 for EFM). Please clarify this discrepancy.

4.  **Theoretical Justification for Low-Rank Assumption:** The central premise of the matrix completion step is that "a fully observed document-term matrix is naturally low-rank". This claim needs more justification. Document-term matrices are famously sparse, but not necessarily low-rank in the sense that documents are linear combinations of a small number of basis documents. LSI *imposes* a low-rank approximation, but assuming the underlying complete matrix is inherently low-rank is a strong assumption. Could the authors provide more intuition or theoretical grounding for this?

5.  **Intuition for Superior Performance:** Could the authors provide more intuition on why TSL outperforms other methods, particularly CCA-based ones? CCA-based approaches like CL-KCCA seem very natural for this problem, as they are explicitly designed to find maximally correlated projections between two views (languages). The main driver for TSL appears to be the global low-rank constraint. What structural properties of the data does this approach capture that direct projection methods miss, leading to better classification performance?

6.  **Computational Cost:** The paper introduces an iterative optimization procedure. A significant missing piece of the evaluation is a comparison of the running time. How does the time taken by the projected gradient descent algorithm compare to the training time of CL-LSI, CL-KCCA, and CL-OPCA? This is crucial for understanding the practical viability of the proposed method.

**Minor Editorial Suggestions:**
*   Before Eq. (2), the notation "(ij)∈Ω" should probably be "(i,j)∈Ω" for consistency.
*   Before Eqs. (4) and (5), the phrase "such as" feels slightly informal and could be removed.

**Overall Assessment:**
This paper presents a simple, novel, and effective method for cross-language representation learning. The empirical results are impressive, and the overall presentation is clear. However, the paper would be significantly strengthened by addressing the points above, particularly regarding the motivation for the two-step design, clarification of the experimental setup and results, justification for the core low-rank assumption, and the inclusion of a runtime analysis. While the core idea is promising, these clarifications are necessary to fully evaluate the paper's contribution.

The paper proposes a method for learning a translanguist representation, first using matrix completion to create multilingual vectors for each row, and then performing CL-LSI on the multilingual vectors.

Quality: Although it seems like a good idea, I'm not sure if the results are robust. The positive results may only be due to better scaling than OPCAs or CCAs for a linear SVM.

I would request the authors to use an ML system that is not sensitive to the rescaling of individual features: for example, a boosted decision tree (available in Oeka). If the boosted decision tree works better than other techniques on a two-step system, then I believe the result.

Significance: If it is indeed a robust result, it will be beautiful, as matrix completion is a well-controlled nonlinear estimation technique.

Novelty: The concept seems new.

Clarity: There were some clarity issues in the paper. Line 113 talks about "constructing a unified document-term matrix for all documents." I'm not sure if this means using one word for each page form (like "markel"), or labeling each page form for each language (like "markelen" and "markelde"). Please clarify. Also, the experimental setup details are divided across Table 1, so it took me a little time to figure it out.

It seems like a nice idea. I would like the authors to try other classifiers besides linear SVMs – I don't know if the result is robust, or a scaling artifact.

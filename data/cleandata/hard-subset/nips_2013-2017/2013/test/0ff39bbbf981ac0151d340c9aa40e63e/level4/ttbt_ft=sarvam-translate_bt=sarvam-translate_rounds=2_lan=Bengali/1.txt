A matrix completion approach has been proposed for the Cross Domain Classification Task, enabling the use of labeled data and unlabeled parallel data in a supporting domain. This approach involves two steps. First, it creates a single incomplete matrix containing all documents and domains, and then completes this matrix by applying low-rank and sparsity conditions using the Projected gradient descent algorithm. Second, it reduces the feature dimension of this complete matrix using LSI and then trains a standard classifier on this new representation. This paper also presents a convergence guarantee and favorable experimental results for the Projected gradient descent algorithm.

Here are some comments and questions:
- After estimating the complete matrix M, you reduce the feature dimension from d to k with LSI because M "does not have enough capacity to handle feature sparsity." Can you explain this in detail? Is it mainly because you want a smaller dimension for computational convenience, since M* will be relatively dense even after applying sparsity? Also, in that case, why did you choose LSI over other dimensional reduction techniques (e.g., PCA)?
- This paper seems to have two distinct aspects of using external data: 1. Using labeled source language data to help learn labeled target language data, and 2. Using unlabeled parallel data. If this is correct, it would be helpful to clearly distinguish the two and report the results separately.
- A related note, the test settings are somewhat vague (section 5.2). So, were you using (a) 4k labeled source language documents, (b) 100 labeled target language documents, and (c) 2k unlabeled parallel documents for your training data? And are you reporting the classification results on the remaining 1900 labeled target language documents? So, the only difference in section 5.3 is that you are changing the number of unlabeled parallel documents from 200 to 2k? In that case, why is the performance of CL-KCCA in Figure 1 better than TSL for EFM and EGM, but not in Table 1?
- Can you briefly explain why a fully observed document-term matrix will be low-ranked? It is sparse, but perhaps not a linear combination of other documents in a document vocabulary, which means the rank in that case will be min(d, k).
- Why is TSL generally able to perform better than other methods, especially CCA? A multi-view approach like CCA seems to be a natural approach for this cross-language representation learning task. In contrast, the matrix completion approach is not as natural. The main driving force seems to be applying the low-rank condition on M*. How does this compare to other techniques?
- What is the algorithm's run time? Does the Projected gradient descent algorithm take more, less, or about the same amount of time than other methods like CL-LSI, CL-KCCA, and CL-OPCA?

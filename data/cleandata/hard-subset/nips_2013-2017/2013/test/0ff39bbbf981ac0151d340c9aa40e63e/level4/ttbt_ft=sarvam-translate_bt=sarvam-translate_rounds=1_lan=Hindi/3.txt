The paper proposes a method for learning translational representation, which first uses matrix completion to create multilingual vectors for each row, and then CL-LSI on the multilingual vectors.

Quality: While this sounds like a good idea, I'm not sure the results are robust. Positive results may only be due to the scaling of LSI, which is better than OPCA or CCA for a linear SVM.

I would urge the authors to use an ML system that is insensitive to individual feature rescaling: for example, boosted decision trees (available in Veka). If boosted decision trees work better than other techniques at the top of a two-step system, then I'm confident of the results.

Significance: If this is indeed a robust result, it will be clean, because matrix completion is a well-controlled nonlinear estimation technique.

Novelty: The idea seems new.

Clarity: There were some issues with clarity in the paper. Line 113 talks about "constructing an integrated document-word matrix for all documents." I'm not sure if this means using one word for each surface form (like "Merkel"), or labeling each surface form from each language (like "Merkelિન" and "Merkelde"). Please clarify. Also, the experimental setup details are divided into Table 1, so it took me a little time to understand it.

It sounds like a clean idea. I wish the authors had tried other classifiers besides linear SVMs – I just don't know if the results are robust, or a scaling artifact.

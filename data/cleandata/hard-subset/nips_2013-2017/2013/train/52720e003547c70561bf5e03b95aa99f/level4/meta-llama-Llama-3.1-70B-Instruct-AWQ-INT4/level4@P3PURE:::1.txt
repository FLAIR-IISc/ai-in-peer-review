The authors discuss the problem of sparse precision matrix estimation using CLIME and present a scalable variant, CLIME-ADMM, along with a distributed framework for computations. Empirical results comparing the CLIME-ADMM algorithm to state-of-the-art techniques, such as DC-QUIC, Tiger, and Flare, are presented.

However, the notation remains scattered throughout Sections 1, 2, and 3, which may hinder readability. It would be beneficial to include a table with all the notations for easy reference. For instance, the parameters \rho and \eta do not appear to be defined in the text.

The proposed block cyclic data distribution is an interesting approach, but it is not immediately clear why this scheme would achieve load balance and scalability. Furthermore, there are no empirical results provided to support this claim.

It also appears that the algorithm currently sets the column block size on an ad-hoc basis. It would be helpful to provide some intuition or theoretical guidance on the choice of k.

A more general concern is that this may not be a truly 'distributed' algorithm, as no explicit mechanism for message passing and communication is discussed. It may be more accurate to consider this a parallel implementation.

Minor comments include:

* In Section 1, "where \lambda is a tuning parameter" should be used instead of "tunning parameter".
* In Section 5.1, "As Tiger is parameter tuning free" should be used instead of "tunning free".
* In Section 2, "CLIME is summerized in" should be corrected to "CLIME is summarized in".

The authors present a large-scale parallel algorithm for estimating sparse precision matrices using CLIME. The novelty of this approach lies in the estimation of the precision matrix by column blocks, rather than column-by-column. Empirical analysis using OpenMPI and Scalapak is also presented.

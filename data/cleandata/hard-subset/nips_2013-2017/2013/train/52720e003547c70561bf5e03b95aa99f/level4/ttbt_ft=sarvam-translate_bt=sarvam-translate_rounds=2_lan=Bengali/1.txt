This article discusses estimating sparse precision matrices using CLIME and presents a scalable version of CLIME-ADMM with a distributed framework for computation. Some empirical results comparing the CLIME-ADMM algorithm with advanced techniques like DC-QUIC, Tiger, and Flare are presented.

The first, second, and third parts contain scattered symbols. It would be helpful to have a table containing all the symbols for the sake of readability. For example, \rho and \eta do not seem to be defined in the text.

The block cyclic data distribution seems interesting – however, it is not clear why this scheme would achieve load balance and scalability. There are also no empirical results to support this argument.

It seems the algorithm currently determines the column block size ad hoc – is there any insight or theory to guide the choice of k?

Another common concern is that it is probably not a 'distributed' algorithm, as there is no clear process of message transmission and communication discussed here. It could possibly be considered a parallel implementation.

A large-scale parallel algorithm for estimating sparse precision matrices using CLIME is presented. There is novelty in estimating precision matrices by column blocks instead of column-by-column. Empirical analysis using OpenMPI and Scalapak is presented.

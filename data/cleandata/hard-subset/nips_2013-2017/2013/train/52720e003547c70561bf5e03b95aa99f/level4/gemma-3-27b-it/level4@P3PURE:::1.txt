The authors address the problem of sparse precision matrix estimation using CLIME and present a scalable variant, CLIME-ADMM, along with a distributed framework for computation. Empirical results compare the performance of CLIME-ADMM to state-of-the-art techniques, including DC-QUIC, Tiger, and Flare.

A key concern is the inconsistent presentation of notation throughout Sections 1, 2, and 3. A dedicated table summarizing all notations would significantly improve readability. For example, the parameters \rho and \eta are used without prior definition. 

The proposed block cyclic data distribution is an interesting approach; however, the rationale for achieving load balance and scalability with this scheme is not immediately clear. Furthermore, the paper lacks empirical evidence demonstrating these properties.

The current implementation appears to set the column block size, *k*, on an *ad hoc* basis. Providing intuition or theoretical guidance for selecting an appropriate value of *k* would strengthen the work. 

A broader point is whether the algorithm can truly be considered ‘distributed,’ as no explicit mechanism for message passing or communication is discussed. A characterization as a parallel implementation might be more accurate.

In summary, this paper presents a large-scale parallel algorithm for estimating sparse precision matrices using CLIME. The primary novelty lies in estimating the precision matrix by column blocks, rather than column-by-column. Empirical analysis utilizing OpenMPI and Scalapak is included.

The authors address the problem of sparse precision matrix estimation using CLIME and present a scalable variant, CLIME-ADMM, along with a distributed framework for computation. Empirical results compare the CLIME-ADMM algorithm to state-of-the-art techniques such as DC-QUIC, Tiger, and Flare.

The notation is somewhat scattered throughout Sections 1, 2, and 3, which hinders readability. A table consolidating all notations would be beneficial. For example, the symbols ρ and η are used without prior definition in the text.

The block cyclic data distribution is an interesting approach, but the rationale for achieving load balance and scalability isn’t immediately clear. Furthermore, the paper lacks empirical results demonstrating these properties. 

Regarding the algorithm, the column block size appears to be set in an *ad hoc* manner. Is there any theoretical justification or intuition guiding the choice of *k*?

A broader concern is whether this can truly be considered a ‘distributed’ algorithm, as the paper does not detail any explicit message passing or communication mechanisms. Perhaps characterizing this work as a parallel implementation would be more accurate.

Minor comments:

Section 1: “where λ is a tunning parameter” should be “where λ is a tuning parameter.”
Section 5.1: “As Tiger is parameter tunning free” should be “As Tiger is parameter tuning free.”
Section 2: “CLIME is summerized in” should be “CLIME is summarized in.”

In summary, this paper presents a large-scale parallel algorithm for estimating sparse precision matrices using CLIME. The novelty lies in estimating the precision matrix by column blocks, rather than column-by-column, and the empirical analysis uses OpenMPI and Scalapak.

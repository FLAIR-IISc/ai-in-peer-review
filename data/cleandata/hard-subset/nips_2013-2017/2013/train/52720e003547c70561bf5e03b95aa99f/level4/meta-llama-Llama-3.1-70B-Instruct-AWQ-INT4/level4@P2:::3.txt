This paper proposes a large-scale distributed framework for sparse precision matrix estimation using the CLIME estimator. The authors develop an inexact alternating direction method of multipliers (ADMM) algorithm for CLIME, which can scale to millions of dimensions and trillions of parameters using hundreds of cores. The proposed framework solves CLIME in column-blocks and leverages the sparse and low-rank structure of the problem to reduce computational complexity.

The paper is well-written, and the authors provide a clear introduction to the problem and the proposed method. The algorithm is nicely introduced, with a good balance between technical points of numerical analysis and theoretical guarantees for convergence rates. The numerical analysis is appropriate, and the implementation of the algorithm shows impressive performances.

The authors' contributions are clearly stated, and the connections to existing works are well-established with sound bibliographical references. The paper provides a detailed study on the speedup of the algorithm regarding block sizes and the number of cores on both shared memory and distributed memory systems.

The main strengths of the paper are:

* The proposed algorithm can handle ultra-large-scale problems with millions of variables and trillions of parameters.
* The framework leverages the sparse and low-rank structure of the problem to reduce computational complexity.
* The algorithm is implemented within a scalable parallel computation framework for both shared and distributed memory systems.
* The numerical analysis is thorough, and the implementation shows impressive performances.

However, I have one concern regarding the availability of the method. If the source code (mostly based on open-source libraries) is not made available to the community, the impact of this work will be limited.

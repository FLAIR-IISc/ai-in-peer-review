

The authors address sparse precision matrix estimation using CLIME, introducing a scalable variant CLIME-ADMM within a distributed computational framework. Empirical comparisons with state-of-the-art methods (DC-QUIC, Tiger, Flare) are provided.  

The notation is scattered across Sections 1–3, which hinders readability. A dedicated notation table would significantly improve clarity—e.g., $\rho$ and $\eta$ remain undefined in the text.  

The block cyclic data distribution is intriguing but lacks justification: it is unclear why this scheme ensures load balance or scalability, and no empirical evidence supports this claim.  

The column block size $k$ appears to be set ad hoc. Is there theoretical guidance or intuition for selecting $k$? This choice critically impacts performance and should be addressed.  

A broader concern: the work is not truly distributed, as no explicit communication protocol or message-passing mechanism is described. The framework is better characterized as a parallel implementation.  

Minor corrections:  
- Section 1: "tunning" → "tuning"  
- Section 5.1: "parameter tunning free" → "parameter tuning free"  
- Section 2: "summerized" → "summarized"  

The core contribution—estimating the precision matrix via column blocks rather than column-by-column—represents a meaningful shift in the CLIME approach. Empirical validation using OpenMPI and ScaLAPACK is provided, though the scalability claims require stronger justification. The paper’s novelty lies in this block-wise estimation strategy, but the distributed framework’s technical details and scalability evidence must be strengthened to match the claimed contribution.

The authors discuss the problem of estimating sparse precision matrices using CLIME and present a scalable version of CLIME-ADMM with a distributed framework for computation. Empirical results comparing the CLIME-ADMM algorithm with state-of-the-art technologies such as DC-QUIC, Tiger, and Flare are presented.

The notation is scattered across sections 1, 2, and 3. A table with all notations could be helpful for ease of readability. For example, \rho and \eta are not defined in the text.

The block cyclic data distribution seems interesting – although it is not clear how this scheme will achieve load balancing and scalability. There are also no empirical results to prove this point.

It seems that the algorithm currently sets the column block size on an ad hoc basis – is there any intuition or theory to guide the choice of k?

A more general concern is that it is probably not a 'distributed' algorithm because there is no clear mechanism for message passing and communication. Perhaps it would be more practical to consider it as a parallel implementation.

A large-scale parallel algorithm for estimating sparse precision matrices using CLIME is presented. The novelty involves estimating the precision matrix by column blocks rather than column-by-column. Empirical analysis using OpenMPI and Scalapak is presented.

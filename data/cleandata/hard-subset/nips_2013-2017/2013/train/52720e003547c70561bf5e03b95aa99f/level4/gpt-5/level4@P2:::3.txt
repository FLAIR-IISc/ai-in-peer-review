Summary
The paper addresses large-scale sparse precision matrix estimation (covariance selection) using the CLIME estimator, targeting settings with millions of variables and trillions of parameters. The authors propose an inexact ADMM algorithm that solves CLIME in column blocks rather than column-by-column, enabling high parallel efficiency. They establish O(1/T) convergence rates for both objective suboptimality and constraint residuals. The method leverages problem structure—sparsity in iterates and potential sparsity or low rank in the sample covariance—to reduce computation, notably by replacing p×p multiplications with low-rank forms A(A′X) when C = AA′ and n ≪ p. The algorithm is implemented in scalable shared- and distributed-memory frameworks using block-cyclic data distribution to improve load balance and cache efficiency. Experiments on synthetic and real datasets compare runtime with state-of-the-art methods (including CLIME variants, TIGER, and DC-QUIC) and provide detailed scalability studies varying block sizes and core counts. The reported results show near-linear speedups and the ability to handle problems with p = 10^6 on hundreds of cores.

Relation to prior work
The paper builds on CLIME’s separability and simpler constraints compared to Graphical Lasso, and on prior ADMM-based solvers. Its main computational contributions are a column-block inexact ADMM tailored to CLIME and a distributed framework with block-cyclic distribution. Compared to MapReduce-like approaches and parameter-server models, the method targets constrained problems more naturally via ADMM and emphasizes efficient matrix–matrix operations and load balancing.

Strengths
- Clear problem motivation and positioning relative to CLIME, Graphical Lasso, and ADMM-based methods.
- A well-designed inexact ADMM variant with closed-form updates and provable O(1/T) convergence for both objective and residuals.
- Careful exploitation of sparsity and low-rank structure to reduce computational cost.
- A practical and scalable implementation for both shared- and distributed-memory systems, using block-cyclic data distribution.
- Comprehensive empirical evaluation, including comparisons with strong baselines and detailed scalability analyses showing near-linear speedups.

Weaknesses/concerns
- The main concern is reproducibility and community impact: if the source code (which uses open-source libraries) is not released, the practical value and adoption of the method will be limited. Clarifying plans for code availability would significantly strengthen the contribution.

Evaluation by review criteria
- Quality: The method is technically sound, with clear derivations, closed-form inexact ADMM updates, and convergence guarantees. The experimental evaluation is thorough and demonstrates strong scaling and performance.
- Clarity: The paper is well written and organized. The algorithm and its implementation choices are explained clearly, with appropriate balance between numerical methods and theory.
- Originality: While CLIME and ADMM are established, the column-block inexact ADMM formulation, its convergence analysis for this setting, and the large-scale distributed framework with block-cyclic distribution constitute meaningful contributions.
- Significance: The ability to solve ultra-high-dimensional precision estimation problems with near-linear scaling is practically important. If code is released, the work is likely to be used and extended by practitioners and researchers.

Pros
- Strong, practical algorithm with provable convergence.
- Effective exploitation of problem structure (sparsity/low rank).
- Scalable shared- and distributed-memory implementation with demonstrated near-linear speedups.
- Clear writing and well-situated within the literature.

Cons
- Unclear code availability, which may hinder reproducibility and impact.

Recommendation
This is a solid algorithmic paper addressing an important practical problem in large-scale covariance selection. The theoretical, algorithmic, and systems contributions are well integrated and empirically validated. I recommend acceptance, contingent on a commitment to release code (or at least sufficient implementation details and scripts) to maximize scientific and practical impact.

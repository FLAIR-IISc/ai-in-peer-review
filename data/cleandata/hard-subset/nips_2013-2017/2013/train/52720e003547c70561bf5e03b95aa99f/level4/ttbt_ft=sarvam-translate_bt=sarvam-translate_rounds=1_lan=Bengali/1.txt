This article discusses the problem of estimating sparse precision matrices using CLIME and presents a scalable variant of CLIME-ADMM with a distributed framework for computation. Some empirical results comparing the CLIME-ADMM algorithm with advanced techniques like DC-QUIC, Tiger, and Flare are presented.

Various symbols are scattered throughout the first, second, and third parts. It would be helpful to have a table containing all the symbols for ease of reading. For example, \rho and \eta do not seem to be defined in the text.

The Block Cyclic Data Distribution seems interesting – however, it is not clear why this scheme would achieve load balance and scalability. There are also no empirical results to support this argument.

It seems that the algorithm currently determines the column block size ad hoc – is there any insight or theory to guide the choice of k?

Another common concern is that it is probably not a 'distributed' algorithm, as no clear process of message passing and communication is discussed here. It can probably be considered a parallel implementation.

A large-scale parallel algorithm for estimating sparse precision matrices using CLIME is presented. The novelty is in estimating the precision matrix by column blocks instead of column-by-column. Empirical analysis using OpenMPI and Scalapak is presented.

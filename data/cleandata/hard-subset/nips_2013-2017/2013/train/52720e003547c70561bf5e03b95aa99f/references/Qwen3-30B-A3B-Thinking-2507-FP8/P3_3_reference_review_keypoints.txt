- Summary: Proposes a distributed CLIME-ADMM that estimates precision matrices in column blocks using sparse/low-rank covariance structure to scale to millions of dimensions with linear core scalability and O(1/T) convergence.

- Strength 1: Novel column-block ADMM formulation enables efficient matrix-matrix multiplications and better hardware utilization compared to column-wise methods.

- Strength 2: Provides rigorous theoretical convergence guarantees (Theorems 1–2) for an inexact ADMM variant.

- Strength 3: Demonstrates impressive empirical scalability, handling up to 1 million dimensions (1 trillion parameters) with near-linear scaling across cores.

- Strength 4: Effectively leverages sparse perturbation and low-rank covariance structure, and uses block-cyclic distribution for load balancing.

- Strength 5: Comprehensive experiments show superior scalability and competitive statistical performance versus several state-of-the-art methods.

- Weakness 1: Limited analysis of how sparse perturbation techniques affect statistical accuracy; empirical impact not shown.

- Weakness 2: Comparative evaluation misses some recent distributed graphical lasso/ADMM methods (e.g., refs [22], [23], GPBS).

- Weakness 3: Convergence conditions (η ≥ ρλ²max(C)) may be restrictive; practical guidance for choosing η is lacking.

- Weakness 4: No principled method given for selecting block size k across different hardware configurations.

- Weakness 5: Insufficient discussion of memory constraints and when one must switch from shared- to distributed-memory setups.

- Overall assessment: High-quality, technically sound contribution that bridges statistical theory and practical distributed computing for massive-scale precision estimation.

- Recommendation: Accept (NeurIPS) — significant and practical advance with minor addressable concerns.

- Suggestion 1: Add experiments showing how sparse perturbation affects statistical accuracy (e.g., spectral norm error).

- Suggestion 2: Include comparisons with distributed graphical lasso/ADMM methods (e.g., [22], [9], GPBS).

- Suggestion 3: Provide guidance or analysis for selecting optimal block size k based on hardware characteristics.

- Suggestion 4: Discuss memory limits that trigger the need for distributed-memory architectures.

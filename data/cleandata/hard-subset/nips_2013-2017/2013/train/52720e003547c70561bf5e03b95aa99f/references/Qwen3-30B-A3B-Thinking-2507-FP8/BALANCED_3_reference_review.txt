

# Review of "Large Scale Distributed Sparse Precision Estimation"

## Summary
This paper addresses the challenge of estimating sparse precision matrices at ultra-large scales (millions of dimensions) using a distributed algorithm based on CLIME (Constrained L1 Minimization). The authors propose a column-block ADMM variant that solves the precision matrix estimation problem in blocks rather than column-by-column, leveraging sparse and low-rank structures for computational efficiency. The approach builds on prior work on distributed optimization [2, 6, 9, 23] but introduces specific innovations for the constrained optimization problem of precision matrix estimation. The paper demonstrates scalability to million-dimensional problems (1 trillion parameters) using 400 cores in 11 hours, with linear speedup in shared and distributed memory architectures.

## Strengths

**Quality:** The paper presents technically sound contributions with solid theoretical guarantees (Theorems 1 and 2) on convergence rates. The algorithm is well-designed, with clear justifications for the column-block approach and block cyclic distribution. The experimental evaluation is comprehensive, comparing against relevant baselines (CLIME, Tiger, DC-QUIC) on both synthetic and real datasets. The authors carefully address the trade-offs between block size, memory usage, and computational efficiency.

**Clarity:** The paper is exceptionally well-written with clear organization. Algorithm 1 is presented with detailed explanations, and figures (Figures 1-4) effectively illustrate the parallelization scheme. Notation is consistent and well-defined. The theoretical results are properly contextualized within the existing literature.

**Originality:** The column-block ADMM approach for precision matrix estimation is novel and addresses a key limitation in existing methods (which process one column at a time). The block cyclic distribution for load balancing in distributed-memory settings is a practical contribution that builds on recent advances in parallel computing [5, 15, 16]. While ADMM has been used for distributed optimization before [9, 23], this paper provides a specific, effective adaptation for the precision matrix estimation problem.

**Significance:** The problem of scaling precision matrix estimation to ultra-high dimensions is highly relevant for modern applications in genomics, climate science, and other fields with massive datasets. The demonstrated ability to handle 1 million dimensions (1 trillion parameters) represents a significant practical advancement over existing methods. The linear scalability with number of cores makes the approach immediately applicable to real-world problems.

## Weaknesses

**Theoretical Depth:** While Theorems 1 and 2 establish convergence rates, the paper doesn't fully connect these theoretical results to practical convergence behavior in the distributed setting. It would be valuable to include a discussion of how the convergence rates translate to actual runtime performance when distributed across many cores.

**Block Size Selection:** The paper demonstrates that block size (k) significantly impacts performance (Figures 3-4), but doesn't provide clear guidance on how to optimally select k for a given problem or architecture. A more thorough analysis of this parameter would strengthen the paper.

**Comparison with Recent Methods:** The comparison with Tiger is somewhat limited, as Tiger is described as "parameter tuning free" but the paper doesn't discuss how the authors chose the parameter for CLIME-ADMM. A more comprehensive comparison with the most recent methods for large-scale precision matrix estimation would be beneficial.

**Low-Rank Structure Implementation:** While the paper mentions leveraging low-rank structure (Section 3.2), it doesn't provide detailed information on how this was implemented in practice or the specific speedup achieved. This is a missed opportunity to demonstrate the practical value of this technique.

## Arguments for and against Acceptance

**For Acceptance:**
- Addresses a significant practical challenge in high-dimensional statistics with a well-designed solution
- Demonstrates substantial scalability improvements over state-of-the-art methods
- Provides strong theoretical guarantees and comprehensive experimental validation
- Clear writing and well-organized presentation

**Against Acceptance:**
- Limited discussion of practical considerations for block size selection
- Incomplete comparison with the most recent methods in the field
- Theoretical results don't fully connect to practical performance in distributed settings
- Some claims about low-rank structure implementation need more detail

## Recommendations

The paper makes a valuable contribution to the field of large-scale statistical estimation. To strengthen it for NIPS, I recommend:

1. Add a more detailed analysis of how to select the optimal block size k for different problem sizes and architectures, including practical guidelines for implementation.

2. Expand the comparison with recent methods, particularly those using distributed ADMM or other scalable approaches for precision matrix estimation.

3. Provide more concrete details on how the low-rank structure was implemented and the specific speedup achieved in practice.

4. Clarify the connection between the theoretical convergence rates (Theorems 1 and 2) and practical convergence behavior in the distributed setting.

With these improvements, the paper would be a strong contribution to the field of large-scale statistical computation. As it stands, it is a high-quality work that addresses an important problem with significant practical implications, though the theoretical-practical connection could be strengthened. I recommend acceptance with minor revisions.

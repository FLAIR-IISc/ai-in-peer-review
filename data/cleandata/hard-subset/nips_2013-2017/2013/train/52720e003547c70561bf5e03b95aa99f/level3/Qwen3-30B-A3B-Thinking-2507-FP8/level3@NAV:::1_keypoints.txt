- Proposes a scalable CLIME-ADMM variant and a distributed/parallel framework targeting ultra-high-dimensional sparse precision matrix estimation.
- Introduces an inexact ADMM that processes the precision matrix in column blocks (k columns) using matrix-matrix operations to improve computational efficiency.
- Uses a block-cyclic data distribution intended to achieve load balancing across processes.
- Demonstrates scalability on shared- and distributed-memory architectures with experiments showing near-linear scaling on hundreds of cores.
- Theoretical contribution: O(1/T) convergence rates for both objective value and optimality conditions.
- Strong empirical result: solved a 1 million–dimensional problem in ~11 hours on 400 cores.
- Column-block processing is a meaningful advancement over prior column-by-column implementations.
- Practical implementation details (OpenMPI/Scalapack) provide useful insights for real-world deployment.
- Situates the work within distributed optimization literature and contrasts with prior approaches (e.g., GPBS, parameter servers).
- Notation is scattered across Sections 1–3 with key symbols (e.g., ρ, η) undefined until later; no dedicated notation table.
- Claims about block-cyclic load balancing lack empirical evidence or analysis to substantiate scalability/load balance.
- Choice of column block size k appears ad hoc with no theoretical guidance or intuition for selecting k.
- The algorithm reads more like a parallel implementation than a distributed algorithm because message passing/communication mechanisms are not described in detail.
- Minor issues: typos (e.g., "tunning") and inconsistent terminology that need correction.
- Needs more concrete guidance on parameter selection and additional empirical validation of load-balancing claims to strengthen applicability.

**Summary:**

This paper presents an inference algorithm for binary latent feature models (LFM) that represents each observation xn as a binary vector zn of size K, where z{nk} = 1 means the kth latent feature exists in xn. The goal of the inference is to learn the value of z for all observations (along with other model parameters). The presented EM-based algorithm uses a shrinkage process to learn z, as well as estimate the number of latent features (K). It is similar to nonparametric Bayesian latent feature models like the Indian Buffet Process (IBP). However, the presented algorithm does not actually use a prior distribution that drives model persistence, but rather a model selection criterion – Factorized Asymptotic Bayesian Inference (FAB) – which has recently been introduced in the context of mixed models and HMM. The FAB concept is based on the expression of log marginal probabilities in terms of expected complete log-probability and a model selection term that encourages shrinking the number of latent features (by specifying an upper bound on K and learning the correct K during inference). For some large datasets, FAB surpasses other inference methods such as Gibbs sampling, variational inference, and MAP estimation methods for IBP (in terms of run time and inference quality).

**Quality:** The technical quality seems reasonable, and the algorithmic details appear accurate. The ability to work with large datasets and learn the number of latent features is very appealing.

**Clarity:** The paper is reasonably clear in the methodology section. However, some experimental details are somewhat unclear (discussed below).

**Originality:** The proposed method is based on Factorized Asymptotic Bayesian (FAB) inference, which has recently been applied to mixed models and HMM. However, the application of FAB for latent feature models is new.

**Importance:** This paper addresses an important problem (efficient inference in latent feature models and simultaneously estimating the number of latent features).

**Weaknesses:**

*   No discussion on the limitations of the proposed algorithm. Is there any situation when the algorithm may perform worse than vanilla Gibbs sampling?
*   No discussion on the potential difficulties of convergence (since it is an EM-like method).
*   The algorithm is limited to binary latent feature models (cannot be applied to factor analysis or probabilistic PCA).
*   The experimental evaluation is not thorough enough (and seems flawed in some cases; see comments below).
*   In small datasets (when the asymptotic argument is not true), it is unclear how the algorithm will behave (some discussion or testing should be done).
*   Some recent work on efficient inference in LFM is not discussed in this paper (see comments below).
*   The MEIBP algorithm proposed in [22] is only applicable to non-negative linear Gaussian models (W must be positive, not real-valued). Therefore, synthetic simulations and block data tests are ineffective for the MEIBP baseline. If you compare with MEIBP, synthetic datasets should have been created so that the loading matrix W is positive.

**Other Comments:**

*   For block image data, since the noise value is known (and given in the FAB algorithm), IBP Gibbs sampler should also be given the same value for fairness (instead of the 0.75 std-dev heuristic).
*   For VB-IBP [2], an infinitely variable version can be used (a finitely variable version was used in the experiment). The experimental settings of VB are not described in sufficient detail (e.g., how many times was it restarted?).
*   I was surprised that the accelerated Gibbs sampling discovered almost 70 latent features in 49-dimensional gold data! I suspect this was due to bad initialization and/or poorly specified noise variance values.
*   Line 70: [21] is not about Gibbs sampling but about MAP estimates for IBP (just like the [22] reference). Please correct the text and make the description accurate.
*   There is recent work on efficient inference using small-variance asymptotes in the case of nonparametric LFMs (see "MAD-Bayes: MAP-based Asymptotic Derivations from Bayes" from ICML 2013). It would be good to discuss this work.

**Small Comments:**

*   Line 349: For real data, it was suggested in [1,22] to make the std-dev equal to 0.25 times the std-dev of the example in all dimensions, instead of making it equal to 0.75.

*   The experimental methodology (for the proposed method and baseline) needs to be better explained, and there should be arguments/explanations for why specific algorithms behaved in a particular way (e.g., Gibbs sampler used a specific alpha that may potentially lead to an extra K, or the noise variance parameters were not set correctly).
*   [21] does MAP inference, not sampling. Please correct this.
*   Include references on MAD-Bayes inference for IBP. The proposed inference method for LFM is interesting and scales to large datasets (as well as estimating K), which is appealing. However, the experimental evaluation should have been done more carefully.

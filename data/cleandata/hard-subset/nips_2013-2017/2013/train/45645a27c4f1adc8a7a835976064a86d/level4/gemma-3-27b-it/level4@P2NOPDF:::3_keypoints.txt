- Proposes an EM-based FAB inference algorithm for binary latent feature models that learns latent features and infers K via shrinkage, claiming better runtime and quality than Gibbs, VB, and IBP MAP on large datasets.
- Technical approach is sound and clearly presented; scales to large datasets and infers the number of latent features automatically.
- Methodological description is clear, but experimental details are insufficiently explained.
- Original contribution: novel adaptation of Factorized Asymptotic Bayesian Inference to latent feature models.
- Significant: addresses efficient inference and the challenging task of selecting the correct number of latent features.
- Lacks discussion of limitations and convergence properties of the EM-like algorithm; restricted to binary LFMs and not easily extended to factor analysis or probabilistic PCA.
- Experimental evaluation is not robust: unclear behavior in small-data regimes and missing comparisons to recent efficient inference methods.
- Critical experimental flaw: MEIBP comparison is invalid because MEIBP requires a positive-definite loading matrix W, which synthetic/block data did not ensure.
- Recommend using the same noise level for IBP Gibbs sampler as used by FAB to ensure fair comparisons.
- VB-IBP comparison should use the infinite variational version and report more experimental details (e.g., number of restarts).
- Surprising result of 70 features on 49-d Sonar data suggests poor initialization or mis-tuned noise variance for accelerated Gibbs.
- Error in manuscript: reference [21] is mislabeled as Gibbs sampling but actually describes IBP MAP estimation.
- Missing citation/sengagement with recent small-variance asymptotics work (e.g., MAD-Bayes, ICML 2013) relevant to LFMs.
- Minor correction: recommended noise std dev for real data is 0.25× data std, not 0.75× as stated.
- After author feedback: some issues addressed, but experimental methodology, hyperparameter justifications, ref [21] correction, and inclusion of MAD-Bayes remain outstanding.
- Overall: interesting and scalable method with promise, but acceptance requires significantly improved and better-justified experimental evaluation.

Summary:
The paper proposes an inference algorithm for binary latent feature models (LFMs), where each observation xn is represented by a K-dimensional binary vector zn, with znk = 1 indicating that xn exhibits the k-th latent feature. The inference goal is to estimate all zn along with other model parameters. The method is an EM-based algorithm that employs a shrinkage mechanism to learn zn and infer the number of latent features K. This is conceptually related to nonparametric Bayesian LFMs such as the Indian Buffet Process (IBP), but instead of using sparsity-inducing priors, it relies on a model selection criterion—Factorized Asymptotic Bayesian (FAB) Inference—previously introduced for mixture models and HMMs. FAB approximates the log marginal likelihood as the expected complete log-likelihood plus a model selection penalty that drives feature shrinkage, by setting an upper bound on K and learning the appropriate K during inference. On large datasets, FAB is reported to outperform alternative inference strategies for the IBP, including Gibbs sampling, variational inference, and MAP estimation, in terms of both runtime and inference quality.

Quality:
The technical development appears sound, and the algorithmic details are correct. The ability to handle large datasets while simultaneously learning the number of latent features is compelling.

Clarity:
The methodological presentation is generally clear, though certain experimental details are insufficiently explained (see below).

Originality:
The approach builds on the recently proposed FAB framework for mixture models and HMMs. Its application to latent feature models is novel.

Significance:
The paper addresses an important problem: efficient inference in latent feature models with automatic determination of the number of features.

Weak Points:
- The paper does not discuss limitations. In what settings might the method underperform relative to standard Gibbs sampling?
- Convergence challenges are not addressed, despite the EM-like nature of the procedure.
- The method is restricted to binary LFMs and is not applicable to factor analysis or probabilistic PCA.
- The experimental evaluation is not sufficiently thorough and appears flawed in places (see below).
- In the small-data regime, where asymptotic arguments may not hold, the behavior of the algorithm is unclear; this warrants discussion or experiments.
- Some recent work on efficient LFM inference is missing from the discussion (see below).
- The MEIBP algorithm in [22] applies only to nonnegative linear Gaussian models (W must be nonnegative, not real-valued). Consequently, the artificial simulation and block data experiments are invalid as MEIBP baselines. To compare fairly with MEIBP, the synthetic data should be generated with a nonnegative loading matrix W.

Other Comments:
- For the block image data, since the FAB algorithm is given the true noise value, the IBP Gibbs sampler should also be provided the same noise value for fairness (instead of the 0.75 std-dev heuristic).
- For VB-IBP [2], the infinite variational formulation could be used (the experiments used a finite variational version). The VB settings are under-specified (e.g., how many restarts were used?).
- It is surprising that the accelerated Gibbs sampler found about 70 latent features on the 49-dimensional Sonar dataset. This likely reflects poor initialization and/or an improperly specified noise variance.
- Line 70: Reference [21] concerns MAP estimation for the IBP, not Gibbs sampling (as with [22]). Please correct the text.
- Recent work on efficient inference via small-variance asymptotics for nonparametric LFMs (e.g., “MAD-Bayes: MAP-based Asymptotic Derivations from Bayes,” ICML 2013) should be discussed.

Minor comments:
- Line 349: For real datasets, [1,22] recommend setting the noise std-dev to 0.25 times the empirical std-dev of the data across dimensions, not to a fixed 0.75.

Comments after the author feedback:
The feedback addressed several questions. A few issues should be resolved before acceptance:
- The experimental methodology (both for the proposed method and baselines) needs clearer description and justification, including explanations for unexpected behavior (e.g., a fixed alpha in the Gibbs sampler potentially inflating K, or mis-specified noise variance hyperparameters).
- Reference [21] performs MAP inference, not sampling; please correct this.
- Please include the MAD-Bayes reference for IBP. The proposed LFM inference method is interesting and appealing in its scalability and ability to infer K, but the experimental evaluation requires more careful execution and reporting.

The paper introduces a novel hybrid strategy for stochastic gradient descent, combining both stochastic and batch gradients. The algorithm is guaranteed to converge to an epsilon-accurate solution using O(log(1/eps)) full gradients and O(k^2 log(1/eps)) stochastic gradients.

The convergence proof appears correct and novel, aside from some minor mistakes detailed below. However, my primary concern lies in the relevance of the proposed algorithm in the machine learning setting, which is the focus of the conference.

In typical machine learning algorithms, strong convexity is often ensured by the regularizer. Consequently, the value of mu is of the order of the number of samples N, i.e., mu = C N, where C does not depend on N. Under this assumption, the proposed method is faster than batch gradient only if the number of samples is bounded by O(C^3/L^3), which does not seem to be an interesting regime. Furthermore, the convergence rate for the proposed algorithm holds only in high probability, whereas the one for batch gradient descent is deterministic.

This point is crucial and must be carefully discussed to demonstrate that the algorithm has a real advantage over batch gradient descent and to establish the paper's relevance to the machine learning community.

The novelty of the hybrid stochastic/batch gradient descent is not clear, and it is uncertain whether the algorithm has any advantage over standard batch gradient descent in practical machine learning optimization problems.

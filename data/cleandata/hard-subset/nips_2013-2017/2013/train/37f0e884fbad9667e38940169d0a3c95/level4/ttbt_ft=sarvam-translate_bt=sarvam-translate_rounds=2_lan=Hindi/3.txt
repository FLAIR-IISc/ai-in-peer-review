This paper presents a new hybrid strategy for Stochastic gradient descent, which utilizes both Stochastic and Batch gradient. The algorithm guarantees convergence to an epsilon-accurate solution using a full gradient (O(log(1/eps)) and a Stochastic gradient (O(k^2 log(1/eps))).

The convergence proof seems correct and new to me, with a section for some minor errors below.

My main concern is about the relevance of the proposed algorithm in a machine learning setting, which is the focus of the conference.

In reality, strong convexity in general ML algorithms is provided by regularizers. Therefore, the value of mu is of the order of the number of samples N, i.e., something like mu = C N, where C does not depend on N. With this assumption, the proposed method is only faster than Batch gradient if the number of samples is bounded by O(C^3/L^3), which I don't find to be an interesting rule. Furthermore, the convergence rate for the proposed algorithm is only in high probability, whereas for Batch gradient descent, they are deterministic.

This point is very important and should be discussed carefully to show that the algorithm has a real advantage over Batch gradient descent, and to prove the relevance of the paper to the ML community.

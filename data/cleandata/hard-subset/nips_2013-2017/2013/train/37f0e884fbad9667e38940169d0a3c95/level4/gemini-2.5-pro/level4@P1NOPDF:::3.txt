This paper presents a new hybrid strategy for stochastic gradient descent that employs both stochastic and full batch gradients. The algorithm is guaranteed to converge to an epsilon-accurate solution using O(log(1/ε)) full gradient computations and O(k² log(1/ε)) stochastic gradient computations.

The convergence proof appears to be both correct and novel, with the exception of some minor mistakes detailed below.

My main concern, however, is the practical relevance of the proposed algorithm for the machine learning problems that are the focus of this conference. It is not clear that the algorithm offers a significant advantage over standard batch gradient descent in typical ML optimization scenarios.

Specifically, in many machine learning applications, strong convexity is introduced by a regularizer. In such cases, the strong convexity constant, λ, is often set such that the condition number L/λ is large. If we analyze the regime where the number of samples N is large, the proposed method appears to be faster than batch gradient descent only when N is bounded by a quantity related to (L/λ)³. This does not seem to be a particularly interesting or practical regime for large-scale machine learning. Furthermore, the convergence rate for the proposed algorithm is probabilistic (holding with high probability), whereas the guarantee for batch gradient descent is deterministic.

This point is critical and must be discussed carefully. To establish the paper's relevance to the machine learning community, the authors need to provide a clearer justification for the scenarios in which their algorithm holds a significant practical advantage over standard batch gradient descent.

*   On line 288, please specify the function to which the inequality in (7) is being applied.
*   The first equality on line 286 should be removed, as it hinders rather than helps comprehension.
*   In Equation (13), the absolute value signs should be replaced with norm operators.
*   Please explicitly state the relationship L ≥ λ. While this may be obvious, stating it clearly would improve the paper's readability.
*   In Equation (4), there appear to be notational inconsistencies: x̂ should be ŵ, and f should be F. Additionally, by Lemma 1, the first term within the max(...) operator is always greater than or equal to the second, which may render the max operator redundant. This should be clarified.
*   Please provide a precise definition of the condition number in terms of L and λ.

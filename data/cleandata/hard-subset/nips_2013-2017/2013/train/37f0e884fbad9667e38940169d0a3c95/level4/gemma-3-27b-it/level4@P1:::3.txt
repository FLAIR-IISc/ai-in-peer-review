The paper presents a new hybrid strategy for stochastic gradient descent that employs both stochastic and batch gradients. The algorithm is guaranteed to converge to an epsilon-accurate solution using O(log(1/ε)) full gradients and O(κ² log(1/ε)) stochastic gradients. The convergence proof appears correct and novel to me, with a few minor points detailed below.

My main concern is regarding the relevance of the proposed algorithm in the machine learning setting, which is a primary focus of this conference. In typical machine learning algorithms, strong convexity often arises from regularization. Consequently, the strong convexity parameter μ is often on the order of the number of samples N, i.e., μ = C N, where C is independent of N. Under this assumption, the proposed method is faster than batch gradient descent only if the number of samples is bounded by O(C³/L³), which does not seem particularly interesting. Moreover, the convergence rate for the proposed algorithm holds only with high probability, while that of batch gradient descent is deterministic. 

This point is crucial and must be carefully discussed to demonstrate a real advantage over batch gradient descent and to establish the paper's relevance to the machine learning community. A thorough analysis of scenarios where the proposed algorithm outperforms existing methods is needed.

The paper introduces a novel hybrid stochastic/batch gradient descent algorithm. However, it is not immediately clear whether this algorithm offers a significant advantage over standard batch gradient descent in practical machine learning optimization problems.

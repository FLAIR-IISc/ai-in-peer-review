This paper presents a new hybrid technique for Stochastic gradient descent, which utilizes both Stochastic and Batch gradient. The algorithm guarantees convergence to an epsilon-accurate solution using a full gradient of O(log(1/eps)) and a Stochastic gradient of O(k^2 log(1/eps)).

I find the proof of convergence to be accurate and novel, but some minor errors are mentioned below.

My main concern is how relevant the proposed algorithm is in a machine learning setting, which is the main topic of this conference.

In reality, strong convexity is given by regulators in general machine learning algorithms. Therefore, the value of mu is proportional to the number of samples N, as mu = C N, where C is not dependent on N. According to this assumption, the proposed method will be faster than Batch gradient if the number of samples is limited by O(C^3/L^3), which I do not find appealing. Moreover, the convergence rate of the proposed algorithm is only applicable in high probability cases, where the rate of Batch gradient descent is decisive.

This issue is very important and should be discussed carefully to demonstrate that the algorithm has real advantages over Batch gradient descent and to prove the relevance of this paper for the machine learning community.

This paper introduces a new hybrid strategy for stochastic gradient descent that employs both stochastic and full-batch gradients. The algorithm is guaranteed to converge to an epsilon-accurate solution using O(log(1/eps)) full gradients and O(k^2 log(1/eps)) stochastic gradients. The convergence proof appears correct and novel, notwithstanding some minor mistakes detailed below.

My primary concern is the practical relevance of the proposed algorithm in the machine learning settings central to this conference. In many common ML applications, strong convexity is introduced via a regularizer. Following the assumption that the strong convexity constant `mu` is on the order of the number of samples `N` (i.e., `mu ≈ C*N`), the proposed method outperforms batch gradient descent only when the number of samples is bounded by O(C^3/L^3). This does not appear to be an interesting or practical regime for many machine learning problems.

Furthermore, the convergence rate for the proposed algorithm holds only with high probability, whereas the rate for batch gradient descent is deterministic. This point is critical and must be carefully discussed. To demonstrate the paper's relevance to the ML community, the authors should explicitly show the scenarios where the algorithm has a clear advantage over batch gradient descent.

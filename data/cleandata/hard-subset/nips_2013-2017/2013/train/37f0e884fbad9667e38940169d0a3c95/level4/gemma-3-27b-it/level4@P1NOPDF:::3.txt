The paper presents a novel hybrid strategy for stochastic gradient descent, combining stochastic and batch gradients. The algorithm is theoretically guaranteed to converge to an ε-accurate solution using O(log(1/ε)) full gradients and O(k² log(1/ε)) stochastic gradients. 

The convergence proof appears correct and novel, with only minor errors detailed below. 

My primary concern is the practical relevance of the proposed algorithm within the machine learning setting, which is the focus of this conference. In typical machine learning applications, strong convexity is often introduced by the regularizer. Consequently, the value of μ is frequently on the order of the number of samples, N – specifically, μ = CN, where C is independent of N. Under this assumption, the proposed method is faster than batch gradient descent only when the number of samples is bounded by O(C³/L³), a regime that doesn’t appear particularly compelling. Furthermore, the convergence rate for this algorithm holds only with high probability, whereas batch gradient descent offers deterministic convergence.

This is a crucial point that requires careful discussion. The authors must clearly demonstrate a real advantage over batch gradient descent to justify the relevance of this paper to the machine learning community.

In summary, while the theoretical contributions are sound, it is not clear if this algorithm offers any practical advantage over standard batch gradient descent in typical machine learning optimization problems.

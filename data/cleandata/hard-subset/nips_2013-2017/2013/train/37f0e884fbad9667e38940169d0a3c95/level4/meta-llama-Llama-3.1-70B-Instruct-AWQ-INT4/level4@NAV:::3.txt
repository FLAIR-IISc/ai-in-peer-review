This paper introduces a novel hybrid strategy for stochastic gradient descent that leverages both stochastic and batch gradients. The algorithm is proven to converge to an epsilon-accurate solution using O(log(1/eps)) full gradients and O(k^2 log(1/eps)) stochastic gradients.

Upon reviewing the convergence proof, it appears to be correct and novel, aside from a few minor errors noted below.

However, my primary concern lies in the algorithm's relevance to the machine learning community, which is the conference's focus. Typically, strong convexity in ML algorithms is induced by the regularizer, resulting in a value of mu that scales with the number of samples N, i.e., mu = C N, where C is independent of N. Under this assumption, the proposed method outperforms batch gradient descent only when the number of samples is bounded by O(C^3/L^3), which seems to be an uninteresting regime. Furthermore, the convergence rate of the proposed algorithm holds only in high probability, whereas batch gradient descent convergence rates are deterministic.

This point is crucial and warrants a thorough discussion to demonstrate the algorithm's advantages over batch gradient descent and establish its relevance to the ML community.

- Please provide a precise definition of the condition number as a function of λ and L.

The proposed hybrid stochastic/batch gradient descent algorithm lacks clear advantages over standard batch gradient descent in practical ML optimization problems.

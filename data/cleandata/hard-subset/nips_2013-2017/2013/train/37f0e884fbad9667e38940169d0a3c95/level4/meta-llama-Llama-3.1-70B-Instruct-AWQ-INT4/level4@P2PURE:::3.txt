The paper proposes a novel hybrid strategy for stochastic gradient descent, combining stochastic and batch gradients. The algorithm is guaranteed to converge to an epsilon-accurate solution using O(log(1/eps)) full gradients and O(k^2 log(1/eps)) stochastic gradients.

The convergence proof appears to be correct and novel, except for some minor errors detailed below.

However, my primary concern lies in the relevance of the proposed algorithm to the machine learning setting, which is the focus of the conference. Typically, in machine learning algorithms, strong convexity is ensured by the regularizer, resulting in the value of mu being on the order of the number of samples N, i.e., mu = C N, where C does not depend on N. Under this assumption, the proposed method outperforms batch gradient descent only when the number of samples is bounded by O(C^3/L^3), which does not seem to be a practically interesting regime. Furthermore, the convergence rate for the proposed algorithm holds only in high probability, whereas the convergence rate for batch gradient descent is deterministic.

This point is crucial and needs to be carefully discussed to demonstrate the algorithm's actual advantage over batch gradient descent and to establish the paper's relevance to the machine learning community.

The paper proposes a hybrid stochastic/batch gradient method that combines stochastic and full gradients. The algorithm is guaranteed to reach an epsilon-accurate solution using O(log(1/eps)) full gradients and O(k^2 log(1/eps)) stochastic gradients.

The convergence proof appears correct and, to my knowledge, novel, aside from minor issues noted below.

My main concern is the relevance of the algorithm to the machine learning setting that is the focus of this conference. In typical ML problems, strong convexity arises from the regularizer, so the strong convexity parameter mu scales with the number of samples N, e.g., mu = C N with C independent of N. Under this assumption, the proposed method is faster than batch gradient only if N is bounded by O(C^3/L^3), which does not seem like an interesting regime. Moreover, the convergence rate for the proposed algorithm holds only with high probability, whereas the guarantees for batch gradient descent are deterministic.

This issue is crucial and should be discussed thoroughly to demonstrate a concrete advantage over batch gradient descent and to establish the paper’s relevance for the ML community.

Minor comments:
- In equation (6), it should be ||w^* - \hat{w}||^2.
- At line 288, specify which function you apply equation (7) to.
- The equality on line 286 should be removed; it adds nothing to comprehension and in fact reduces clarity.
- In (13), absolute values should be replaced by norms.
- Explicitly state that L ≥ lambda; even if it is obvious, it is better to make it clear.
- In (4), x^ should be w^ and f should be F; moreover, by Lemma 1, the first term in the max is always larger than the second.
- Please define the condition number precisely as a function of lambda and L.

Overall: Novel hybrid stochastic/batch gradient descent. It remains unclear whether the algorithm has any advantage over standard batch gradient descent in practical ML optimization problems.

**Summary:** The authors consider the problem of optimizing a flat, strongly convex function onto a convex constraint set, so that gradient mapping updates can be efficiently computed. Nestorov's optimal first-order algorithm has linear convergence for such a problem, but the constant state number depends on the square root of k. The authors consider the case where one has access to an expensive full gradient of the objective, as well as a cheaper stochastic gradient oracle. They propose a hybrid algorithm for which the full gradient oracle should be called only O(log 1/eps) (independent of the state number) and the cheaper stochastic gradient oracle should be called O(k^2 log(1/eps)) – as long as the state number is not too large, it can theoretically be faster. The main idea behind their algorithm (called Epoch Mixed Gradient Descent - EMGD) is to replace a full gradient step (called Epoch) with a fixed number of mixed gradient steps O(k^2) that use a combination of full gradient (calculated once for Epoch) and stochastic gradient (which vary within an Epoch). By taking the average of O(k^2) iterations within an Epoch, they can show a continuous decrease in sub-optimality independent of the state number, which is why the number of full gradient steps required (number of Epochs) is independent of the state number. They provide a simple and self-contained proof of their convergence rate, but no experiment.

I enjoyed reading this paper and feel that the idea of combining some full gradient calculations with a large number of cheaper stochastic gradient steps is novel and interesting. This work establishes itself in a series of recent papers that attempt to use cheaper stochastic oracles while maintaining a linear convergence rate. A recent theoretical and practical success with the SAG algorithm [16] works for a flat strongly convex objective that is the sum of n simple functions (such as in regular empirical loss – where n is the number of training examples). In this case, a reasonable assumption is that computing the full gradient oracle is n times more expensive than the stochastic gradient. Then SAG is theoretically faster than the Nestorov algorithm as long as the state number k < = n/8. Conversely, in this case EMGD is theoretically faster than the Nestorov algorithm as long as the state number k < = n^(2/3), and EMGD is slower than SAG in all arrangements (it has a similar large O when k < = n^1/2). To estimate these speed-ups, if k = n^1/2, then both SAG and EMGD are O(n log(1/eps)) while the Nestorov algorithm is O(n^(5/4) log(1/eps)). The two main advantages I see for EMGD over SAG are that 1) as mentioned by the authors, EMGD works for constraint optimization [assuming that gradient mapping updates can be efficiently computed] while SAG has so far only been defined for unrestricted problems; and 2) the convergence proof for EMGD is much simpler than for SAG and therefore can achieve more insight and at the same time modify EMGD for proven guarantees [their result is also stronger with more probability than SAG]. The authors also mention a potential memory/parallelization benefit, although this is less clear because SAG can also be parallelized using mini-batches (which also reduces memory requirements from the size of the mini-batch).

**Evaluation Summary:**
**गुण:**
- Gives a novel and interesting algorithmic idea with a clean, simple, and solid theory.
- Like SAG, achieve linear convergence rate in regular empirical loss where the state number k is not multiplying the number of training examples n; but their algorithm is more general (works for constraint optimization) and the proof is much simpler.
- The paper is clearly written and the proof is self-contained.
**कमियाँ:**
- There is no experiment to show that this algorithm can actually differentiate in practice (and it does not provide any concrete example to clarify why its suggested theoretical benefits might be relevant in practice).
- There is no discussion on the algorithm's limitations/shortcomings (in particular, compared to existing algorithms – section 3.4 should be improved! I make several suggestions in this review).
- There are some high-level comments in the proof that might justify the necessary insight used for its construction.
**गुणवत्ता:** The paper is technically solid. Would appreciate some experiments, although I think the theoretical contribution can stand on its own. The authors should definitely expand section 3.4 with the limitations and shortcomings of their algorithm. They should also add a more concrete discussion of the n function sum example that better highlights the difference with SAG and Nestorov (as I mentioned above – for example EMGD is (almost) worse than Nestorov for k > n^(2/3); SAG is similar to EMGD for 1 < = k < = n^(1/2); EMGD is better than SAG for n^(1/2) < k < = n/8 (and Nestorov is worse than SAG for k > n/8) and Nestorov is worse than SAG for k > n/8). A major drawback of EMGD for a practitioner is that the number of steps within an Epoch must be pre-determined (with knowledge of k) – in contrast, both SCDA [17] and SAG do not need to pre-determine the number of steps and therefore may have an advantage in achieving faster practical convergence than differentiating constraint (or benefit from a better local state number, for example). Furthermore, SCDA has automatic step-size selection; while SAG has an adaptive step-size estimation that seems to work quite well in practice. The authors should add this discussion to the paper.
[Appendix after discussion with other reviewers: a) As another reviewer mentioned, ML often has a regularization parameter C/n and therefore the state number ~= n/C', which is an arrangement in which SAG still does better than Nestorov, but in which EMGD does not. This should be noted (and perhaps another practical setting where k < n^(2/3) should be mentioned). b) The authors should cite [hybrid deterministic-stochastic methods for data fitting. M. Friedlander, M. Schmidt. SIC, 2012] which also presents a hybrid deterministic-stochastic algorithm with a linear rate of convergence. This latter algorithm still shows the state number in the rate, so the final rate is not faster than the standard gradient descent (only the starting rate is faster due to using a cheaper step) – so the current submission may still theoretically be better than this rate when k is not too large in relation to n.]
**स्पष्टता:** The paper is quite clear. I appreciated the summary from table 1 which I have not seen so clearly in the literature. The proof can be strictly followed, but it would be more useful for the reader if the authors could add some high-level comments that motivate some defined quantities that fit a bit magically in its current form.
**Originality:** It is a novel combination of known techniques.
**Importance:** The practical relevance of the algorithm has not yet been demonstrated. But the theoretical contribution may have an impact. In the context of the difficult proof of convergence of SAG (and the simple proof of SCDA, but which only applies to a constrained setup), the simple proof of EMGD is a major contribution.

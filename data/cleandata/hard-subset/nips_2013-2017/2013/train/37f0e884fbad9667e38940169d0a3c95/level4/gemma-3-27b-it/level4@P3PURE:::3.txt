The paper presents a novel hybrid strategy for stochastic gradient descent, combining stochastic and batch gradient updates. The algorithm is guaranteed to converge to an ε-accurate solution using O(log(1/ε)) full gradients and O(k^2 log(1/ε)) stochastic gradients. The convergence proof appears correct and novel, with the exception of some minor errors detailed below.

My primary concern lies with the relevance of the proposed algorithm to the machine learning setting, which is the focus of this conference. Typically, in machine learning algorithms, strong convexity is induced by the regularizer. Consequently, the value of μ is often of the order of the number of samples N, i.e., μ = CN, where C is independent of N. Under this assumption, the proposed method is faster than batch gradient descent only when the number of samples is bounded by O(C^3/L^3), a regime that does not seem particularly interesting. Furthermore, the convergence rate for this algorithm holds only in high probability, whereas the convergence of batch gradient descent is deterministic. 

This point is crucial and requires careful discussion to demonstrate a clear advantage over batch gradient descent and to justify the paper's relevance to the machine learning community.

- Please explicitly state that L ≥ λ, even if it is obvious; clarity is improved by doing so.
- In equation (4), x̂ should be ŵ and f should be F. Furthermore, the first term in the maximum is always greater than the second term, as established by Lemma 1.
- Please precisely define the condition number as a function of λ and L. 

In summary, while the proposed hybrid stochastic/batch gradient descent algorithm is theoretically interesting, it is not clear if it possesses any advantage over standard batch gradient descent in practical machine learning optimization problems.

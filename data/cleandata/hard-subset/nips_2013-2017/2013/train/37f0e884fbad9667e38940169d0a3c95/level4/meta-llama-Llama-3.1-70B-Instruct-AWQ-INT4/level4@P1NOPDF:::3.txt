The paper presents a new hybrid strategy for stochastic gradient descent, combining stochastic and batch gradients. The algorithm is guaranteed to converge to an epsilon-accurate solution using O(log(1/eps)) full gradients and O(k^2 log(1/eps)) stochastic gradients.

The convergence proof appears correct and novel to me, apart from some minor mistakes detailed below.

My main concern is about the relevance of the proposed algorithm in the machine learning setting, which is the focus of the conference. In typical machine learning algorithms, strong convexity is provided by the regularizer. Consequently, the value of mu is of the order of the number of samples N, i.e., mu = C N, where C does not depend on N. With this assumption, the proposed method is faster than batch gradient only if the number of samples is bounded by O(C^3/L^3), which does not seem to be an interesting regime. Moreover, the convergence rate for the proposed algorithm holds only in high probability, whereas the one for batch gradient descent is deterministic.

This point is crucial and must be carefully discussed to demonstrate that the algorithm has a real advantage over batch gradient descent and to prove the relevance of the paper for the machine learning community.

The proposed novel hybrid stochastic/batch gradient descent algorithm is unclear if it has any advantage over standard batch gradient descent in practical machine learning optimization problems.

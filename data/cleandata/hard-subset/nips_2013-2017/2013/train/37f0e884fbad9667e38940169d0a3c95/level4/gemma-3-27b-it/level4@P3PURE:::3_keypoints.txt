- Paper proposes a novel hybrid stochastic/batch gradient descent algorithm combining stochastic and full-gradient updates.  
- Algorithm claimed to converge to an ε-accurate solution using O(log(1/ε)) full gradients and O(k^2 log(1/ε)) stochastic gradients.  
- Convergence proof is novel and appears correct aside from minor errors noted below.  
- Main concern: relevance to ML is unclear because strong convexity is typically induced by the regularizer, giving μ = C·N, which makes the method faster than batch GD only when N = O(C^3/L^3), an uninteresting regime.  
- Additional concern: proposed convergence guarantee holds only with high probability, while batch gradient descent has deterministic convergence.  
- Authors need a careful discussion showing clear advantages over batch GD to justify relevance to the ML community.  
- Minor fix: Equation (6) should read ||w^* - ŵ||^2.  
- Minor fix: Specify which function equation (7) is applied to (line 288).  
- Minor fix: Remove the equality on line 286 as it harms clarity.  
- Minor fix: In equation (13) replace absolute values with norms.  
- Minor fix: Explicitly state L ≥ λ for clarity.  
- Minor fix: In equation (4), x̂ should be ŵ and f should be F; note the first term in the max is always greater than the second by Lemma 1.  
- Minor fix: Precisely define the condition number as a function of λ and L.  
- Overall: theoretically interesting algorithm but unclear practical advantage over standard batch gradient descent for ML problems.

This paper presents a new hybrid technique for Stochastic gradient descent, which utilizes both Stochastic and Batch gradient. The algorithm ensures convergence to an epsilon-exact solution using a full gradient of O(log(1/eps)) and a Stochastic gradient of O(k^2 log(1/eps)).

I believe the proof of convergence is correct and novel, but some minor errors are mentioned below.

My main concern is how relevant the proposed algorithm is in a machine learning setting, which is the main topic of this conference.

In reality, strong convexity is generally provided by regulators in machine learning algorithms. Therefore, the value of mu is proportional to the number of samples, such as mu = C N, where C N is independent. According to this assumption, the proposed method will be faster than Batch gradient if the number of samples is limited by O(C^3/L^3), which I do not find appealing. Moreover, the convergence rate of the proposed algorithm is only applicable in cases of high probability, where the rate of Batch gradient descent is decisive.

This problem is very important, and the algorithm has real advantages over Batch gradient descent, and it should be carefully discussed to prove the relevance of this paper to the machine learning community.

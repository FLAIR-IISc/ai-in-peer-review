The paper proposes a hybrid variant of stochastic gradient descent that combines stochastic and full (batch) gradients. The method is shown to reach an epsilon-accurate solution using O(log(1/eps)) full gradients and O(k^2 log(1/eps)) stochastic gradients.

The convergence analysis appears correct and, to the best of my judgment, novel, aside from several minor issues noted below.

My primary concern is the practical relevance of the algorithm for machine learning problems, which is the conference’s focus. In typical ML setups, strong convexity is induced by the regularizer. Consequently, the strong convexity parameter mu scales with the number of samples N, i.e., mu = C N for some constant C independent of N. Under this assumption, the proposed method outperforms batch gradient descent only when the sample size is bounded by O(C^3 / L^3), which does not seem to be a meaningful regime. Moreover, the stated convergence rate for the proposed approach holds only with high probability, whereas the convergence of batch gradient descent is deterministic.

This issue is crucial and needs careful discussion to convincingly demonstrate that the algorithm offers a tangible advantage over batch gradient descent, thereby establishing the paper’s relevance to the ML community.

- Please specify in line 288 which function (7) is applied to.

- Please explicitly state why L >= lambda; even if obvious, it should be clearly articulated.
- In (4), x^ should be w^ and f should be F; also, by Lemma 1, the first term in the max is always larger than the second.
- Please give a precise definition of the condition number as a function of lambda and L.

Novel hybrid stochastic/batch gradient descent. Unclear whether it provides a practical advantage over standard batch gradient descent for typical ML optimization problems.

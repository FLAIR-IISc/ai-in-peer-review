Summary and relation to prior work:
The paper studies first-order optimization for smooth, strongly convex objectives. The key question is whether one can achieve linear convergence while using a number of full-gradient evaluations that does not depend on the condition number κ = L/λ. The authors propose Epoch Mixed Gradient Descent (EMGD), which partitions optimization into epochs. At the start of each epoch, EMGD computes one full gradient at an anchor point w̄k. Within the epoch, it uses a “mixed” gradient estimator g̃ = ∇F(w̄k) + ∇f(w) − ∇f(w̄k), where f is a randomly sampled component (via a function oracle), and performs projected proximal updates within a ball B(w̄k; Δk) whose radius shrinks by √2 across epochs. The analysis establishes that each epoch reduces the suboptimality by a constant factor with high probability, yielding linear convergence. The total number of full-gradient calls is O(log(1/ε)) (independent of κ), while the number of stochastic-gradient calls is O(κ^2 log(1/ε)).

The work is framed against accelerated full-gradient methods (O(√κ log(1/ε)) full gradients) and stochastic methods (polynomial in 1/ε). It also compares to SAG and SDCA, which (for finite-sum problems) achieve O((n + κ) log(1/ε)) gradient computations and linear convergence. EMGD claims advantages in constrained problems, memory footprint, and high-probability guarantees.

There is a very close connection to variance-reduced stochastic methods that use a snapshot of the full gradient to correct the variance of per-sample gradients. In particular, the mixed gradient estimator is identical to that used in SVRG (Johnson & Zhang, NIPS 2013), which achieves linear convergence with O((n + κ) log(1/ε)) gradient computations and O(d) memory without shrinking domains. This connection should be discussed explicitly.

Quality:
- Theory: The high-probability linear convergence proof is carefully structured using a potential argument and Hoeffding-Azuma, leveraging the shrinking ball to bound gradient noise via smoothness. The induction is standard and sound in spirit. Using the ball constraint to bound variance is neat and technically clean.
- Assumptions: The algorithm requires knowledge of κ (to set T) and L (to set the step size), and access to a function oracle that permits computing ∇f at two points for the same random f. The latter is natural in finite-sum ERM but stronger than a standard stochastic-gradient oracle in streaming settings.
- Complexity: Achieving O(log(1/ε)) full gradients is conceptually interesting. However, the per-epoch inner loop length T = Θ(κ^2 log(1/δ)) is large; overall stochastic gradient cost is O(κ^2 log(1/ε)). This is substantially worse than SVRG/SAGA/SDCA rates of O((n + κ) log(1/ε)) in the finite-sum case and even some semi-stochastic methods. Thus, in the principal regime of interest (finite-sum ERM), the proposed method is dominated in stochastic-gradient complexity, even if it uses fewer full gradient passes.
- Empirics: The paper presents no experiments. Given there are multiple competing methods with similar aims, empirical comparisons (e.g., vs. AGD, SAG, SVRG/SAGA/SDCA) across varying κ and n would be valuable to substantiate the claimed practical advantages (especially constrained settings, memory, distributed full gradient).

Clarity:
- The paper is generally readable and the proof is self-contained. However, there are numerous typos and notation glitches that impede clarity, e.g., “epoches,” repeated equation labels, ln δ should be ln(1/δ) in Theorem 1’s condition, some reference formatting, and minor grammatical issues.
- The function oracle definition is unusual; clarify that it suffices to sample an index i and compute ∇fi(·) at two points (no need to evaluate f itself).
- The projection onto W ∩ B(w̄k; Δk) can be discussed in terms of computational cost and when it is easy to implement.

Originality:
- The idea of mixing a snapshot full gradient with per-sample gradient differences is not new; it is central to SVRG (and related methods like SAGA/SAG with memory). The explicit shrinking trust-region style ball used to control variance is a distinct design choice that yields high-probability bounds with simple bounded-difference arguments. The main claimed novelty—condition-number independent number of full gradient evaluations—is conceptually interesting, but closely mirrors the SVRG practice of one full gradient per epoch with O(κ) inner iterations. EMGD’s need for O(κ^2) inner iterations weakens originality and impact.
- Related work misses or under-emphasizes SVRG and subsequent variance-reduction literature. Given the estimator is identical, this comparison is essential.

Significance:
- Theoretical: The result clarifies a trade-off between full-gradient calls and stochastic-gradient calls and gives an algorithm with high-probability linear convergence and O(log(1/ε)) full gradients. This could be relevant when full gradient computation is disproportionately expensive and distributable.
- Practical: Without experiments and given the O(κ^2) dependence, the practical significance is unclear relative to established variance-reduced methods. The constrained-setting advantage and O(d) memory claim are valuable, but SVRG already enjoys O(d) memory, and SDCA/SAGA support various regularizers and constraints via proximal steps.

Pros:
- Clear theoretical framework and high-probability linear convergence.
- Conceptual contribution: decoupling full-gradient complexity from κ.
- Works with constraints (via projection) and uses only O(d) memory.
- Simple algorithmic structure.

Cons:
- Inner loop complexity O(κ^2) is pessimistic; overall cost likely worse than SVRG/SAGA/SDCA for finite-sum problems.
- Missing discussion and comparison to SVRG and related variance-reduction methods that use the same estimator with better κ dependence.
- No empirical validation.
- Requires known κ and L; practical tuning not addressed.
- Minor but numerous presentation issues and typos.

Suggestions for improvement:
- Add a thorough comparison to SVRG/SAGA (complexity, memory, assumptions), and discuss why EMGD needs O(κ^2) inner steps while SVRG attains O(κ); can your analysis be tightened to O(κ) by choosing a larger step and redesigning the epoch length?
- Provide experiments on convex ERM (ridge/logistic) with varying n and κ, comparing wall-clock time, number of full gradients, and stochastic gradients.
- Remove the need to know κ via an adaptive/doubling scheme for T and Δk; consider variance-based stopping within epochs.
- Clarify the function oracle assumption and projection cost; discuss cases where W ∩ B is easy to project onto.
- Fix typographical issues and tighten constants in Theorem 1 if possible.

Recommendation: Weak reject. The idea of controlling the number of full-gradient calls is interesting, and the analysis is clean, but the method appears dominated in the finite-sum regime by existing variance-reduced algorithms that use the same estimator with better κ dependence, and the paper lacks empirical evidence and a complete comparison to that literature. Strengthening the comparison and improving the inner-loop complexity would significantly improve the contribution.

- Addresses monitoring changes in dependency structure in data streams for simple SEMs, proposing a memory- and compute-efficient solution.  
- Proposes LoSST: a novel algorithm that uses PC for batch structure learning and monitors stream statistics to trigger relearning.  
- Tracks deviation from weighted means, the correlation matrix, and an effective weighted sample size to detect changes.  
- Weights new points by "surprisingness" using Mahalanobis distances, p-values from Hotelling’s T², and Liptak’s weighted pooling to compute a significance statistic ρ.  
- Combines the significance statistic with prior state via a noisy-OR to decide when to relearn structure.  
- Evaluated on synthetic data with parameter and structural changes across different graph sizes and degrees.  
- Applied to a real-world US price index dataset and found known economic changes while ignoring transient fluctuations.  
- Strength: tackles an intriguing, underexplored problem with potential for broader theoretical and practical work.  
- Strength: LoSST is a novel, practical, and computationally efficient combination of methods.  
- Strength: Experiments are adequate and include parameter sweeps plus a real-data case study.  
- Strength: Provides a thorough literature review and is generally well written.  
- Main weakness: Lacks theoretical guarantees or analysis under simplified assumptions; should at least state this as future work.  
- Main weakness: Heavy reliance on heuristics; key heuristic choices (reweighting, effective sample size, relearning criteria) need clearer documentation and justification.  
- Concern: Using Hotelling’s T² with reweighted/effective sample sizes and Liptak pooling may violate original method assumptions and should be explicitly acknowledged.  
- Presentation issues: proofs in footnotes should be moved to supplementary material, figures are too small/poorly labeled with excessive spacing, and experimental descriptions are unclear.  
- Methodological critique: Explanation for poorer performance on edge addition under parameter-only changes is unsatisfactory.  
- Suggestion: Consider decoupling parameter updates from structure updates to improve behavior.  
- Overall: Intriguing and promising direction but currently a heuristic-first, preliminary implementation needing theoretical grounding and clearer exposition.

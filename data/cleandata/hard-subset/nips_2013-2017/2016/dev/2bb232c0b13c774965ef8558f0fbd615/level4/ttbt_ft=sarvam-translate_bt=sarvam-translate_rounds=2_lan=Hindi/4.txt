This work presents modifications to the degradation layer of the standard MD-LS-TMS used for character recognition, as presented in [18]. It is also closely related to [6], which performs a similar function – decoding at the character level per time step. The modification is to convert the global average pooling to a weighted average pooling (Equation 2). Furthermore, these weights are learned using a recursive attention layer (MD-LS-TMS) that takes both the previous weights and the encoded feature vector. A CTC loss is used to directly calculate the loss for transcription without requiring character or line-level alignment. Additionally, the attention mechanism can be interpreted as a "soft" line division. This work provides a comparison with other methods, including [6], and achieves better/comparable performance. To my understanding, the main difference between this paper and [6] is that a complete line is decoded at each time step as opposed to a single character. While this improves speed and better performance, I am not entirely convinced that there is enough technical novelty in it. (If I am missing something, please correct me in the rebuttal, and if so, I would be happy to increase my rating.) Furthermore, it would be interesting to see when the number of lines in the input image exceeds the number of time steps the algorithm runs – as mentioned, the method still performs better in such scenarios.

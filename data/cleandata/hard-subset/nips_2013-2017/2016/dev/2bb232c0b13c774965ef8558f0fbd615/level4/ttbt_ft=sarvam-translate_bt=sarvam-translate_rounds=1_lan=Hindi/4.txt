This work presents an amendment to the collapse layer of the standard MD-LSTMS used for character recognition, as presented in [18]. It is also closely related to work [6] which performs a similar function – decoding at the character level per time step. The amendment is to convert the global average pooling to a weighted average pooling (Equation 2). Furthermore, these weights are learned using a recurrent attention layer (MD-LSTM) that takes both the previous weights and the encoded feature vector. A CTC loss is used to directly calculate the loss for transcription without requiring character or line-level alignment. Additionally, the attention mechanism can be interpreted as a "soft" line division. This provides a comparison with other methods, including work [6], and achieves better/comparable performance. To my understanding, the main difference between this paper and [6] is that a complete line is decoded at each time step as opposed to a single character. Although this improves speed and better performance, I am not entirely convinced that there is enough technical novelty in it. (If I am missing something, please correct me in the rebuttal, and if so, I would be happy to increase my rating.) Furthermore, it would be interesting to see the attention when the number of lines in the input image exceeds the number of time steps set to run the algorithm – as mentioned, the method still performs better in such scenarios.

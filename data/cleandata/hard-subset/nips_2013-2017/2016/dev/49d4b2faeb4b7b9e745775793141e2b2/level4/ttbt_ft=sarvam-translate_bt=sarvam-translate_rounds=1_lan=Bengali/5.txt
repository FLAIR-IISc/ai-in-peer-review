The authors describe a method for training deep, generative models that allows for more information flow and more effective training. They present a Matryoshka network that is similar to LapGANs and ResNets and consists of a top-down network, a bottom-up network, and a set of merge modules. The merge modules compare top-down state with bottom-up state, selecting a perturbation to reduce reconstruction error. - Details for specific sections are generally sufficient (e.g., good use of equations to clarify calculations), there is room for clarity and flow improvements between different sections and subsections. - Sampling/inference/distributions in sections 2.2 and the end of section 2.1 need to be more clearly linked to the network elements. For example, x (in section 2.2) is never defined as anything other than a random variable (until line 132, where it is vaguely defined as unknown or known data). How is this variable x related to the network? - How is the latent variable z initialized? What impact does this initialization have? - Is there any typo in line 71? It says conv(h, w) is a convolution of the input x with the kernel w, but it says nothing about h. And, x is not an input to the conv function. - The qualitative results are quite impressive. - The quantitative results show that the proposed model performs better than the previous method, although it is much more complex. - Before going into the details of the individual components of the network, it would be helpful to provide a brief explanation of where the input and output of the network (and how they are used) are clearly defined. It would also be helpful if the input and output of each component of the network (top-down, bottom-up, merge modules) were defined before explaining the mechanisms. - I would like to see more analysis of the qualitative results in addition to the explanation of the quantitative results. - Can you provide more measurements without NLL? What about reconstruction error? - It would be nice to see some qualitative comparisons with the previous method.

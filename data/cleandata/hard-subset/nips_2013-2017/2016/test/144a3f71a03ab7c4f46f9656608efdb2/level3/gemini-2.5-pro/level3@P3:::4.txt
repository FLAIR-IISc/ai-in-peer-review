This paper addresses the important problem of learning optimal Bayesian network structures from data when given background knowledge in the form of ancestral constraints. Standard score-based learning algorithms, which are highly efficient, typically rely on the assumption of score decomposability. This assumption prevents them from natively handling ancestral constraints, which are non-decomposable. While alternative frameworks like Integer Linear Programming (ILP) can encode such constraints, they often do not scale as well.

The authors build upon a recent search framework [Chen et al., 2015, 2016] that performs an A* search over a space of equivalence classes (the EC tree) and uses an "oracle"—an optimal learner for decomposable scores—to compute the heuristic. The main contribution of this work is a novel method for "projecting" the non-decomposable ancestral constraints into decomposable constraints, specifically edge presence/absence constraints and topological ordering constraints. These inferred decomposable constraints can then be passed to the oracle, significantly tightening the A* heuristic and pruning the search space, which leads to a much more efficient search. The paper provides an algorithm for inferring all entailed edge constraints and a MaxSAT formulation for finding a large set of entailed ordering constraints. Experiments demonstrate that the proposed method is orders of magnitude faster than a state-of-the-art ILP solver adapted to handle ancestral constraints.

**Strengths**

1.  **Originality and Significance:** The core contribution—the method for projecting non-decomposable constraints into decomposable ones (Section 5)—is both novel and significant. It provides an elegant and effective bridge between powerful, highly-optimized learning algorithms designed for decomposable scores and the practical need to incorporate rich, non-decomposable domain knowledge. This work meaningfully advances the state-of-the-art for optimal structure learning under this class of constraints.
2.  **Technical Quality:** The proposed methods are technically sound. The theorems provided formalize the logic behind the constraint projection, and the use of a MaxSAT solver for inferring ordering constraints is a clever approach to a non-trivial problem. The empirical evaluation is strong, showing dramatic performance improvements over a relevant and competitive baseline (GOBNILP). The results convincingly support the central claim that empowering the oracle via constraint projection leads to massive efficiency gains.
3.  **Clarity and Presentation:** The paper is well-written, well-organized, and easy to follow. It provides the necessary background on score-based learning and the EC tree framework before introducing its novel contributions. The problem, proposed solution, and experimental results are all explained clearly.

**Weaknesses and Suggestions for Improvement**

1.  **Generalizability of Constraint Projection:** The experimental results in Table 1 vs. Table 2 show that the inferred decomposable constraints dramatically improve the performance of the EC tree approach, but their benefit for the ILP baseline is less pronounced (even showing a slight overhead for n=12). This raises a question about the general utility of the projection method. Is this technique primarily effective within the specific A*-and-oracle framework, or is it a broadly applicable pre-processing step? To strengthen the paper's impact, the authors should discuss or provide evidence that this projection can improve the performance of other types of methods that were originally unable to handle non-decomposable constraints (e.g., by enabling them to solve a constrained version of the problem).
2.  **Sensitivity to the Amount of Domain Knowledge:** Table 3 reveals an interesting "easy-hard-easy" pattern in runtime for the EC tree method as the percentage of ancestral constraints (p) increases. The authors correctly identify the reasons for the "easy" regimes (small p is close to unconstrained, large p is highly constrained). However, the "hard" intermediate regime is noteworthy, especially since the ILP approach seems to become monotonically faster as more constraints are added. This suggests that the proposed A* search on the EC tree may be highly sensitive to the *amount* of domain knowledge provided. An explanation of this phenomenon would be valuable. Does this imply that practitioners may need to carefully tune the number of ancestral constraints supplied to avoid the worst-case performance? A discussion of this sensitivity compared to the ILP approach would be insightful.
3.  **Incomplete Experimental Comparison:** In Table 3, the Structural Hamming Distance (SHD) is reported only for the proposed EC tree method. Since both EC tree and ILP are exact methods, they should return networks with the same optimal score and, presumably, the same SHD. To make the comparison complete and verify that both methods are converging to the same ground truth quality, the SHD for the GOBNILP results should also be included.
4.  **Paper Organization Suggestion:** Section 4.1 provides background on BN graphs, which is preliminary work from the same authors. However, the core method and all experiments are based on EC trees (Section 4.2). The space dedicated to BN graphs could be used more effectively. I suggest the authors consider dropping or significantly condensing Section 4.1 and using the saved space to expand on the implementation details of the EC tree search. Specifically, more information on the A* algorithm's practical implementation and a more detailed explanation of how the oracle (URLEARNING) is used to generate the heuristic would improve the paper's self-containment and reproducibility.

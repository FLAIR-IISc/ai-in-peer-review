- Paper addresses optimal BN structure learning with non-decomposable ancestral (path) constraints using EC-tree + A* and a decomposable-score oracle (URLearning).
- Key technical ideas: prune the EC tree by detecting CPDAGs that must/cannot satisfy ancestral constraints, and project ancestral constraints into decomposable edge and ordering constraints for the oracle.
- Algorithm infers entailed edge-presence/absence via a maximum closure (Theorems 2–3) and selects a maximal, cycle-free set of ordering constraints via MaxSAT (Theorem 4).
- Empirically shows large speedups over GOBNILP, especially when projected constraints are given to the oracle.
- Relation to prior work is well-positioned relative to order-graph/DP, heuristic search, EC-tree, and ILP/CP approaches.
- Main correctness concern: the central pruning lemma/theorem (Lemma 3 / Theorem 1) is insufficiently justified and appears too strong — requires formal monotonicity proof.
- Request to restate and fully prove the exact violation-monotonicity condition used for pruning (current “P satisfies A iff descendants satisfy A” is incorrect).
- EC-tree edge-generation claim (that adding a leaf forces a specific orientation Z -> W) lacks detailed justification and needs explicit rules (including Meek’s rules) and a concrete argument.
- Exposition conflates “inferred/entailed” ordering constraints with those merely “selected” by MaxSAT; need to separate entailed edge constraints (Theorems 2–3) from selected order constraints and clarify soundness.
- MaxSAT formulation is under-explained: must justify clauses (1)–(3), state objective clearly, prove Theorem 4 or cite a proof, and justify the nonstandard clause (3).
- Need to discuss MaxSAT complexity, solver overhead, and include runtime ablation for MaxSAT.
- CPDAG feasibility tests: proposed tests are fine but exhaustive class enumeration is exponential; request for a polynomial-time sufficient test or bounded search and empirical quantification of enumeration frequency/cost.
- Empirical baselines: GOBNILP encoding used is weak; reviewer requests a stronger ILP/CP baseline with ancestry variables A(X,Y), transitivity and linking constraints, and bounded parent-set experiments.
- Experiments should include: (i) tighter ILP/CP baseline, (ii) bounded parent-set size results, and (iii) ablation isolating ancestral-constraint effects on GOBNILP.
- Ablation studies requested: (a) EC-tree pruning only, (b) oracle empowerment only (edge constraints), (c) oracle empowerment with ordering constraints, (d) full method, plus MaxSAT time breakdown.
- Clarity fixes: tighten definitions/notation for α(·), β(·), “maximum” closure, and clarify “entails” vs “consistent with”; add worked examples for Theorems 2–3 and candidate sets Z.
- Explain the admissible heuristic used by URLearning and precisely how projected edge and precedence constraints are passed to the oracle.
- Fix confusing/unsupported statements (e.g., EC-tree descendant property, orientation claims), and improve figure captions and cross-references.
- Originality: using EC trees for non-decomposable ancestral constraints is novel; edge-projection rules are a useful contribution; MaxSAT idea is pragmatic but hinges on soundness guarantees.
- Significance: scalable exact methods for ancestral constraints are valuable and could influence BN learning if theoretical gaps and stronger baselines are addressed.
- Pros: addresses an important constraint type exactly, cleverly integrates EC-tree with decomposable constraint projection, and shows large empirical speedups.
- Cons: key pruning theorem unclear, MaxSAT selection underspecified and conflates entailment with consistency, baselines weaker than necessary, and reliance on exponential enumeration without quantification.
- Suggestions: provide full pruning proofs and EC-tree generation details, separate entailed edges vs selected orders, strengthen MaxSAT exposition/proofs, add stronger baselines and bounded-family experiments, include ablations and runtime breakdowns, and quantify/reduce CPDAG enumeration.
- Recommendation: borderline accept if theoretical clarifications and stronger baselines are provided; otherwise borderline reject.

- Paper tackles exact Bayesian network structure learning with non-decomposable ancestral (path) constraints, which standard score-based methods do not support.  
- Main idea: enforce ancestral constraints via pruning in the EC-tree search (nodes = CPDAGs) using A* guided by a decomposable-score oracle.  
- Theorem 1 (monotonicity): CPDAG inconsistency/satisfaction with constraints is preserved for all descendants, justifying pruning.  
- Proposes projection techniques to convert ancestral constraints into decomposable constraints to help the oracle.  
- Edge projection: gives necessary/sufficient conditions (Theorems 2–3) for negative/positive edge entailment using ancestor/descendant closures α(·), β(·).  
- Ordering projection: formulates a MaxSAT problem to extract a maximally large, jointly consistent set of topological ordering constraints (Theorem 4 gives soundness).  
- Empirical comparison: EC-tree (with URLEARNING oracle and EVA MaxSAT) vs ILP (GOBNILP) on multiple benchmarks shows orders-of-magnitude speedups and better scaling trends.  
- Projection of constraints greatly improves EC-tree performance and modestly helps ILP; an easy–hard–easy pattern appears as constraint count varies.  
- Relates to prior work on exact BN learning (dynamic programming, A* over orders), ILP/CP formulations, and prior EC-tree/EC-graph work; novelty is embedding path constraints in EC-tree plus projection.  
- Technical soundness: pruning rationale and negative-edge entailment are convincing; positive-edge entailment and MaxSAT acyclicity clause need clearer proofs/justification.  
- Empirical support is appropriate but lacks detailed tables/figures in the reviewed version and omits CP baselines.  
- Some algorithmic details are omitted (closure computation, CPDAG satisfiability procedures), and enumerating class(P) may be costly and needs quantification.  
- Clarity issues: several theorems need proof sketches, “maximum set of ancestral constraints” should be precisely defined, and CPDAG satisfiability checks require an explicit algorithm and complexity discussion.  
- Presentation problems: reference formatting issues and missing figures/tables in the submitted version.  
- Originality: novel combination of EC-tree pruning for path constraints and principled projection to decomposable constraints; MaxSAT-based ordering extraction is a notable contribution.  
- Significance: advances constrained exact BN learning by enabling path-level priors, shows practical speedups, and suggests constraints can substitute for data to some extent.  
- Pros: addresses an important gap, sound pruning with oracles, effective projection techniques with formal conditions, empirical gains over strong baseline, and practical utility.  
- Cons: missing proofs/details, unclear CPDAG enumeration cost, limited baselines and ablations, no robustness to noisy constraints, limited scalability (n≈20), assumes complete data/no latent confounding, and presentation issues.  
- Questions for authors: frequency/cost of CPDAG enumeration fallback; ablations for edge vs ordering projection; robustness to noisy constraints and soft/weighted MaxSAT; precise definitions of “maximum set”/closed set; and code release plans.  
- Recommendation: Weak accept — solid, novel, practically useful work that should be strengthened by adding clarity, CPDAG satisfiability details, and robustness experiments.

This paper addresses the problem of learning optimal Bayesian network structures from data when subjected to ancestral constraints, a type of background knowledge that is common in practice but challenging for standard algorithms. The authors note that most state-of-the-art exact, score-based learners rely on score decomposability, a property that is violated by ancestral constraints (e.g., "X must be an ancestor of Y"). To overcome this, the paper adapts a recently proposed A* search framework over the space of equivalence classes of DAGs (the "EC tree") [Chen et al., 2016], which can handle non-decomposable scores.

The main contributions are twofold. First, the authors develop a sound method for pruning nodes in the EC tree search space that violate the given ancestral constraints. Second, and more significantly, they propose a novel method to "project" the non-decomposable ancestral constraints onto a set of decomposable constraints (edge presence/absence and variable ordering constraints). This projection is used to empower the "oracle" that computes the A* search heuristic, leading to a much tighter heuristic and a dramatic speedup in search. The projection of ordering constraints is particularly novel, involving a reduction to MaxSAT to find a maximal set of consistent ordering constraints. The paper provides a thorough empirical evaluation, comparing its approach against an Integer Linear Programming (ILP) formulation and demonstrating orders-of-magnitude improvements in runtime.

**Strengths:**

1.  **Significance:** The paper tackles a problem of high practical and theoretical importance. Incorporating rich background knowledge is crucial for learning meaningful graphical models, especially from limited data. Ancestral constraints are a natural way to express causal or temporal knowledge, yet they have been largely unsupported by mainstream optimal structure learning algorithms. This work provides an effective and scalable solution, significantly advancing the state of the art. The experiments compellingly show that a small number of ancestral constraints can improve model accuracy as much as a large increase in data size, highlighting the practical value of this work.

2.  **Originality and Quality:** The core technical idea of projecting non-decomposable constraints into decomposable ones to strengthen a search heuristic is highly original and elegant. While the base search framework (EC tree) is from prior work, its adaptation and, more importantly, the method for empowering its heuristic oracle, are novel contributions. The MaxSAT formulation for inferring a consistent set of ordering constraints is a particularly clever piece of technical work that addresses a subtle issue with inferring orderings naively. The paper is technically sound, with clear theorems backing the proposed pruning and projection methods.

3.  **Clarity:** The paper is exceptionally well-written and easy to follow. It clearly motivates the problem, situates the work within the existing literature, and explains its technical components in a logical progression. The distinction between the base search framework and the novel contributions for handling constraints is made very clear. The experimental setup and results are presented lucidly and effectively support the paper's claims.

4.  **Thoroughness:** The empirical evaluation is comprehensive and convincing. The authors compare against the most appropriate baseline (ILP), which can also handle these constraints. The inclusion of an ablation study (comparing performance with and without constraint projection) is crucial, as it directly validates the effectiveness of their main technical contribution. The analysis of the "easy-hard-easy" phenomenon and scalability provides valuable insight into the algorithm's behavior.

**Weaknesses and Suggestions for Improvement:**

1.  **CPDAG Pruning Complexity:** The paper states that if initial checks are inconclusive, the algorithm must "exhaustively enumerate the structures of class(P) to check if any of them satisfies the given constraints." Since an equivalence class (CPDAG) can represent an exponential number of DAGs, this step could be a significant computational bottleneck. It would be valuable for the authors to discuss the practical impact of this step. How often does this full enumeration occur in the experiments? Are there common scenarios where this becomes intractable? A brief discussion on the complexity and feasibility of this check would strengthen the paper's claims of scalability.

2.  **MaxSAT Formulation Details:** The description of the MaxSAT formulation for inferring ordering constraints is quite condensed. Specifically, the third set of hard clauses, representing "a sufficient condition for acyclicity," seems to involve a large number of variables (O(n^4)) and could be complex to implement correctly. For reproducibility and clarity, it would be helpful to either expand on this formulation in the main text or include a more detailed description in an appendix.

3.  **Justification for Ordering Inference:** The argument in Footnote 4 for inferring an ordering constraint `Y < X` from an ancestral constraint `X !~> Y` is a bit informal. While the conclusion is likely correct (i.e., if there is no path from X to Y, a topological sort where Y precedes X must exist), the provided reasoning could be made more precise. A clearer, more formal argument would improve the technical rigor of this claim.

**Summary of Pro/Con Arguments:**

*   **Pro:**
    *   Solves an important, practical problem in Bayesian network learning.
    *   Presents a novel and powerful technical idea (constraint projection to empower a heuristic) that is broadly applicable.
    *   Demonstrates state-of-the-art performance with massive speedups over the most relevant alternative.
    *   Technically sound, well-written, and supported by a thorough experimental evaluation.

*   **Con:**
    *   Some details regarding the complexity of the CPDAG constraint check and the MaxSAT formulation are brief, which could impact reproducibility and understanding of potential bottlenecks.

**Decision:**

This is an excellent paper that makes a significant and original contribution to the field of Bayesian network structure learning. It is technically deep, well-motivated, and empirically validated. The proposed method for projecting non-decomposable constraints is a powerful idea that could inspire further work in constrained optimization. The minor weaknesses are primarily requests for clarification and do not detract from the overall quality of the work. I strongly recommend acceptance.

This paper deals with the problem of constrained optimization of Bayesian network learning from local scores with ancestral constraints. The search branch-and-bound is where the search tree contains CPDAGs as nodes, and we 'go down' from CPDAG1 to CPDAG2 (approximately) by choosing a parent set for a BN variable that is not a node in CPDAG1. The sensible symmetry is broken. The search is A where a URLर्निंग system is used to provide a heuristic. (How this happens is not detailed here, but it is available in a previous paper). Given the ancestral constraints, some pruning of the search tree is possible. Lemma 3 (complementary material) is the main result here. I believe this is true, but I don't understand the proof. The phrase "by the EC tree edge generation rules, Z -> W is also an edge in Gk" needs further clarification. There are also implicit constraints ("implicit constraints" is standard terminology, here they are called "projected constraints"). A set of ancestral constraints will indicate the constraints on the edges that can be used for pruning. Constraints on admissible topological ordering are also implied ("ordering constraints"). The approach to doing this is not explained properly. We have the statement that we can "infer Y < Z from Z, not Y's ancestor". The point is the word "can". Clearly there are DAGs in which Z is not an ancestor of Y, but Y < Z is a consistent ordering: for example, a graph with vertices Z and Y and no edges. The authors, of course, know this! What they mean is that if "Z is not an ancestor of Y", we can choose to add the constraint Y < Z. So the idea is to 'infer' as many consistent ordering constraints as possible (not just the ones that are implicit). The authors are careful not to 'infer' too much and to use the MAXSAT approach for inferring. Experiments are used where the number of variables, data points, and ancestral constraints varies. There is a comparison with the GOBNILP system where the current system is shown to perform much faster than GOBNILP. As the authors noted, without ancestral constraints, GOBNILP performs worse than A approaches when there is no limit on the parent set cardinality. There is no such limit in the current experiments, so it is difficult to tell whether GOBNILP's poor performance here is due to its 'general' failure to deal with many parent sets or specifically due to its poor handling of ancestral constraints. Also, the encoding of ancestral constraints (given in complementary material) is based on MAXSAT encoding from [Cussens, 2008], not the ILP encoding found in section 3.1.1 of [Cussens, 2010]. And this MAXSAT encoding is a poor construction for an ILP system. Why not have A(X,Y) binary variables in addition to the I(Z,U) variable and leave out the (density number!) E(X,Y,Z) variable? I think then only the transit constraints on the A(X,Y) variable and the following constraints connecting the A and I variables are needed: A(X,Y) \geq \sum{U:X \in U} I(Y,U). In other words, if X is a parent of Y, then X is an ancestor of Y. This construction provides a very strict linear relaxation and should provide better performance. This is the approach to total ordering on variables from section 3.1.1 of [Cussens, 2010], but one constraint is omitted because the ancestor is a partial, not a total order. I am sure that even with a sensible ILP construction of ancestral constraints, the current system will perform better than GOBNILP, but the paper is undermined by comparing it to a poor construction rather than a sensible one. It would be interesting to compare the current method to van Beek and Hoffman, especially since it often performs better than GOBNILP. I don't think it would be too difficult to adapt that method to allow posting ancestral constraints. The current paper by van Beek and Hoffman is not mentioned by the authors.

- Paper's main contribution: a method (Section 5) to project non-decomposable ancestral constraints onto decomposable (edge and ordering) constraints, enabling decomposable-score-based learners to use domain knowledge; writing is clear.

- Table 3: projection improves EC trees but not ILP, raising concerns about the method's generalizability beyond EC trees.

- Need stronger evidence that inferred decomposable constraints can improve a learning approach that originally cannot handle non-decomposable constraints.

- Table 2 shows an easy–hard–easy pattern in EC tree performance as ancestral-constraint proportion increases; the cause is unexplained and suggests sensitivity to the amount of domain knowledge.

- This sensitivity may imply the number of ancestral constraints must be carefully tuned for optimal EC tree performance, possibly more so than for ILP.

- Table 2 only reports structural Hamming distance for EC trees; comparison to ILP is missing.

- Minor suggestion: consider removing Section 4.1 and reallocate space to better explain EC trees and the A* algorithm in the EC-tree context.

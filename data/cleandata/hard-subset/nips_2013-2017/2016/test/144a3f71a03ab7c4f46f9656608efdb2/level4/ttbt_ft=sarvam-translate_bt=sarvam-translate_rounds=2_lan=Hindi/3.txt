This paper describes an efficient means of incorporating parental constraints into the optimal Bayesian network search from data. It does this by translating a set of parental constraints into a (maximum) set of sorted constraints, which are then used in a score-based learning algorithm based on the EC tree presented by [Chen et al., 2015]. The results are evaluated against direct modifications of the ILP-based Goonilup algorithm and are shown to perform significantly better in the given setting. Overall, it is reasonably well-written and clearly explained, although the solution is conceptually straightforward. The novel problem + solution, good results, not spectacular, but certainly interesting for an expert audience. The problem addressed in this paper is interesting and novel in the sense that there is no readily available 'standard solution' for learning optimal BN in combination with parental constraints (novelty = 3). However, one reason behind this apparent shortcoming is that the situation is somewhat artificial: once parental constraints become prevalent, it would be more understandable to go for more general cause-finding algorithms that also allow for unseen confounders (hence effect = 2). The approach adopted is based on translating difficult-to-use parental constraints, which do not fit well in decomposition-score-based algorithms, into a set of sorted constraints that are more easily expressed in terms of the absence of edges, which fit in the decomposition score. The mapping is done through the MaxSat solver, which ensures that all implicit parental constraints are captured. However, some rules in 257-261 seem odd or incomplete: for example, acyclicity rule 3 is exactly symmetric ((X < Y) and (Z < W) == (Z < W) and (X < Y)), but the conclusion (Y -/-> Z) is not; furthermore, rule 4 seems to exclude parental constraints of the type 'X and Y do not cause each other'. Moreover, one might expect that only a few parental constraints are given in practice, and these are unlikely to significantly reduce the search space. The resulting comparison against Goonilup with additional linear constraints looks impressive, but the latter was not developed to deal with this problem. A reasonable alternative to measure performance would be to assume that the parental constraints are consistent with the optimal model, and compare against greedy algorithms like GES, which would generate the same output for the oracle input. Of course, greedy algorithms cannot generally guarantee optimality, but this paper will provide at least a more meaningful indication of the efficiency achieved by the method.

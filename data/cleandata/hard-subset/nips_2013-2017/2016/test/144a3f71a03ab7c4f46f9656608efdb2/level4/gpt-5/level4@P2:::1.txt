Summary
The paper tackles optimal Bayesian network (BN) structure learning under background knowledge expressed as ancestral constraints (X is/is not an ancestor of Y). The authors build on the BN graph and EC tree search spaces (Chen et al., 2015; Chen et al., 2016), using A* search where nodes are CPDAGs and edges correspond to adding a leaf with a chosen parent set. An oracle for decomposable scores (URLEARNING; Yuan and Malone, 2013) provides admissible heuristics. Because EC tree nodes represent equivalence classes, the authors prune any CPDAG that cannot realize the ancestral constraints. They further “project” non-decomposable ancestral constraints into decomposable edge constraints (presence/absence) and ordering constraints, passing these to the oracle to tighten the heuristic. Edge constraints are derived via closed-form tests (Theorems 2–3), while ordering constraints are selected by solving a MaxSAT problem that aims to add as many cycle-free ordering constraints as possible (Theorem 4). Experiments on benchmarks compare the proposed method against the ILP system GOBNILP, reporting orders-of-magnitude speedups, and show that projected constraints substantially improve search efficiency.

Relation to prior work
Most exact score-based methods rely on decomposable scores and order-graph formulations (Koivisto and Sood, 2004; Silander and Myllymäki, 2006; Yuan et al., 2011). ILP/CP formulations can in principle encode non-decomposable constraints (Jaakkola et al., 2010; Cussens, 2008; van Beek and Hoffmann, 2015), but their empirical performance with ancestral constraints has not been systematically evaluated. This paper extends the shortest-path view of BN learning to handle ancestral constraints by pruning EC trees and by leveraging projected decomposable constraints, thus combining advantages of non-decomposable search spaces with strong order-graph oracles.

Strengths
- Significance: Ancestral constraints are practically important causal knowledge. Incorporating them while retaining optimality is valuable.
- Technical contribution: Sound pruning of EC tree nodes under ancestral constraints; principled projection of ancestral constraints to edge and ordering constraints; use of MaxSAT to select a maximally large, cycle-free set of ordering constraints compatible with all DAGs that satisfy the ancestral constraints.
- Empirical results: Substantial speedups over an ILP baseline; clear analysis of the easy–hard–easy trend as constraint density varies; evidence that even modest background knowledge can markedly improve structure accuracy, sometimes rivaling large increases in data size.
- Integration with prior methods: Effective use of a decomposable-score oracle (URLEARNING) within a non-decomposable search framework.

Weaknesses and points needing clarification
- Proof clarity: The key pruning step (Lemma 3 in the supplement) hinges on the statement “By the EC tree edge generation rules, Gk also contains edge Z→W.” This implication is not obvious from the described rules and needs a more explicit argument (e.g., a case analysis of how adding a leaf and orienting compelled edges in the CPDAG ensures Z→W appears in Gk).
- Entailed vs. selected ordering constraints: The text states that one “can infer Y < Z from Z not ancestor of Y.” This is misleading: Y < Z is not entailed by Z not ancestor of Y; rather, it is a selectable ordering constraint that preserves completeness if chosen consistently. Please clarify the difference between logical entailment and the MaxSAT selection of a large, cycle-free set of orderings consistent with A. It would help to state precisely what is guaranteed: soundness (no DAGs satisfying A are eliminated), maximality w.r.t. the chosen clause weighting, and closure under transitivity.
- MaxSAT encoding details: The hard constraints listed (transitivity, necessary condition Y<Z ⇒ Z not ancestor of Y, acyclicity sufficient conditions) are sketched, but the exact variable set and clause families are only outlined. A more formal description, with a brief justification of Theorem 4, would improve clarity and reproducibility.
- Baseline fairness: All experiments use unbounded parent set sizes. It is known that heuristic search on the order graph scales better than ILP in this regime (Yuan and Malone, 2013; Malone et al., 2014). To isolate the effect of ancestral constraints, please add experiments with bounded parent set cardinality.
- ILP encoding of ancestral constraints: The ILP baseline uses an encoding derived from a MAXSAT formulation (Cussens, 2008), introducing many auxiliary variables. A tighter ILP is available by adding binary ancestor variables A(X,Y), enforcing transitivity and antisymmetry, and linking them to the family-choice variables I(Y,U) via A(X,Y) ≥ sum_{U: X in U} I(Y,U). This yields a much stronger linear relaxation (see Cussens, 2010, Section 3.1.1 for related formulations with total orders). Using such an ILP would provide a fairer comparison.
- Missing comparison: The CP approach of van Beek and Hoffmann (2015) often outperforms GOBNILP and seems readily extensible to post ancestral constraints. Including this baseline and citing this work would strengthen the empirical evaluation.

Suggestions for improvement
- Expand the proof of the pruning lemma to explain how EC tree edge-generation compels the critical edge orientations.
- Make the semantics of “projected” constraints explicit: which are logical consequences (entailed) vs. which are selected to maximize heuristic strength while preserving completeness.
- Provide an ablation study: (i) pruning only, (ii) pruning + edge projection, (iii) pruning + ordering projection, (iv) pruning + both, to quantify each component’s contribution.
- Add experiments with bounded indegree, a tighter ILP encoding with A(X,Y) variables, and, if possible, the CP baseline.
- Minor: standardize terminology (“projected” vs. “implied” constraints); ensure all referenced proofs and algorithms are included or fully summarized in the supplement.

Evaluation against review criteria
- Quality: The main ideas are technically sound; some proofs and encodings need clarification and tighter baselines would strengthen claims.
- Clarity: Generally well written, but key steps (pruning lemma, MaxSAT formulation, entailment vs. selection) require clearer exposition.
- Originality: Combining EC tree pruning with projected constraints and MaxSAT-selected orderings for non-decomposable constraints is novel.
- Significance: The method is practically important and shows strong empirical benefits.

Arguments for acceptance
- Addresses an important and underexplored setting (optimal BN learning with ancestral constraints).
- Solid integration of theory and practice; large empirical speedups.
- Clear path to broader impact via empowerment of existing oracles.

Arguments against acceptance
- Incomplete proof exposition and ambiguity around “inference” of ordering constraints.
- Baseline comparison may be weakened by suboptimal ILP encoding and unbounded parent sets.
- Missing comparison to strong CP baselines.

Overall, this is a promising and impactful contribution; addressing the clarity and baseline issues would make it stronger.

The paper presents an efficient method for incorporating ancestral constraints into the task of learning optimal Bayesian networks from data. The core idea is to translate a given set of ancestral constraints into a maximal set of ordering constraints. These ordering constraints are then integrated into a score-based learning algorithm that utilizes the EC tree structure from [Chen et al., 2015]. The proposed method is empirically evaluated against a modified version of the ILP-based Gobnilp algorithm, demonstrating significant performance improvements in this specific setting. The paper is well-written and the approach is clearly explained. While the solution is conceptually straightforward and the results are not spectacular, the work addresses a novel problem and will be of definite interest to a specialist audience.

The problem of optimal BN learning with ancestral constraints is novel, as there is no standard, readily available solution. However, the practical relevance of this setting is debatable. The consideration of ancestral relations suggests a causal discovery context, where it would often be more appropriate to employ algorithms that can also handle unobserved confounders, which this work does not address. The technical approach itself involves translating ancestral constraints, which are difficult to integrate with decomposable scores, into ordering constraints. These ordering constraints, expressed as the absence of certain edges, are compatible with decomposable-score-based search. This translation is performed using a MaxSAT solver to derive the maximal set of implied ordering constraints.

The methodology has some potential weaknesses. First, the rules used for the MaxSAT translation (lines 257-261) raise questions. For instance, in acyclicity rule 3, the premise `(X < Y) and (Z < W)` is symmetric with respect to the two pairs, but the conclusion `Y -/-> Z` is not. Rule 4 also appears to exclude common constraint types, such as 'X and Y are not ancestors of each other'. Second, the practical benefit is unclear when only a few ancestral constraints are available, as is common in practice; such a sparse set of constraints is unlikely to prune the search space substantially. Finally, the choice of baseline for comparison is a significant concern. While the proposed method outperforms Gobnilp with added linear constraints, Gobnilp was not designed for this type of problem, making the comparison less informative. A more meaningful evaluation of efficiency could be a comparison against a greedy algorithm like GES. Assuming the provided constraints are correct (i.e., consistent with the ground truth), GES should find the optimal network, providing a more relevant benchmark for the algorithm's performance, even though greedy methods do not guarantee optimality in the general case.

Minor details:
-   Line 30: Is "omniscient oracle" intended to be distinct from a standard oracle?
-   Line 38: The phrase "empower the oracle" is unclear. Does this mean "use in combination with the oracle"?
-   Line 90: It should be clarified that "optimizing" the score means "minimizing" it in this context.
-   Line 164: Theorem 1's statement should be more precise. For the "if and only if" condition to hold, it should clarify that the set of constraints A satisfied by the CPDAG P on the EC tree is a subset of the constraints satisfied by the full DAG G.
-   Lines 225-227: Theorem 3 seems unlikely to yield many positive edge implications in the practical scenario of sparse ancestral constraints.
-   Line 267: The encoding of the acyclicity constraint is not immediately intuitive (this point is related to the earlier criticism of the MaxSAT rules).
-   Line 310: The experimental setup using 10%-25% ancestral constraints seems unrealistic for reasonably sized graphs, as this would imply extensive prior knowledge of the target graph.

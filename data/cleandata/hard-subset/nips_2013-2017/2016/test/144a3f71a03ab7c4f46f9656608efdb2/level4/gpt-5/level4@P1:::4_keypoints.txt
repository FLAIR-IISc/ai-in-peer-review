- Paper tackles optimal Bayesian network structure learning under non-decomposable ancestral constraints by extending EC-tree/BN-graph search with projection to decomposable constraints.  
- Builds on Chen et al.’s BN graph and EC tree framework that uses an oracle for decomposable-score subproblems.  
- Main technical contribution 1: sound pruning criterion for EC trees under ancestral constraints, with efficient tests and fallback enumeration (Theorem 1).  
- Main technical contribution 2: entailment conditions to infer negative/positive edge constraints X ↛ Y and X → Y from ancestral transitive closure using ancestor/descendant sets (Theorems 2–3).  
- Main technical contribution 3: formulation of inferring maximal consistent ordering constraints X < Y as a MaxSAT problem with hard (transitivity, necessary) and soft (encourage negative-ancestry orderings) clauses (Theorem 4).  
- Key idea: project non-decomposable ancestral constraints into decomposable edge and ordering constraints to “empower the oracle” and tighten heuristics.  
- Empirical result: projected constraints make EC-tree-based search often orders of magnitude faster than ILP-based GOBNILP across datasets and settings.  
- Reported easy–hard–easy behavior for EC-tree runtime as ancestral-constraint proportion varies; accuracy (SHD) improves with more data and more constraints.  
- Situates work relative to prior exact score-based methods (which assume decomposability) and ILP/CP approaches (capable but not evaluated for ancestral constraints).  
- Strength: clear motivation that ancestral constraints are important and non-decomposable.  
- Strength: solid theoretical guarantees for pruning and precise entailment criteria for projection.  
- Strength: practical impact demonstrated by large speedups from projection and careful empirical evaluation.  
- Weakness/question 1: unclear generality — projection seems to mainly benefit EC-tree/oracle-guided search and shows limited gains for ILP; need demonstration on at least one other framework.  
- Weakness/question 2: need deeper analysis of easy–hard–easy pattern (pruning rates, oracle calls, heuristic accuracy vs. constraint density) and guidance to predict/mitigate the hard regime.  
- Weakness/question 3: accuracy comparison missing for ILP — report SHD for ILP under same settings to confirm comparable model quality.  
- Weakness/question 4: presentation could trim BN graph background and expand EC-tree details (A* in EC-tree, heuristic construction, pruning implementation and cost).  
- Additional: quantify frequency and cost of fallback CPDAG class enumeration and consider more sufficient tests if enumeration is frequent.  
- Additional: report MaxSAT overhead/time, effect on end-to-end runtime, and stability across seeds/solvers.  
- Additional: clarify terminology around “maximum set of ancestral constraints” (maximal vs. closure under inference rules).  
- Additional: improve reproducibility by providing command lines, parameter settings (e.g., BDeu ESS), parent-set size limits, and solver setup (EVASOLVER, URLEARNING).

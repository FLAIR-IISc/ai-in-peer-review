The paper presents an efficient method for incorporating ancestral constraints into the search for optimal Bayesian networks from data. It translates a given set of ancestral constraints into a maximal set of ordering constraints, which are then used within a score-based learning algorithm built on the EC tree of [Chen et al., 2015]. The results are compared to a straightforward modification of the ILP-based Gobnilp algorithm and show substantial improvements in this setting. The paper is generally well written and clear, though the core idea is conceptually straightforward. The contribution addresses a novel problem with a clean solution—solid and of clear interest to specialists, if not spectacular.

The problem is interesting and, to my knowledge, new: there is no readily available standard method for optimal BN learning with ancestral constraints (novelty=3). That said, the setting feels somewhat artificial: once ancestral relations are available, it often makes more sense to use broader causal discovery methods that accommodate latent confounding (hence impact=2).

The approach converts ancestral constraints—which are awkward for decomposable-score-based methods—into ordering constraints that can be expressed as the absence of certain edges and thus fit decomposable scoring. This mapping is performed with a MaxSAT solver, which is used to make explicit all ancestral constraints implied by the input. However, some of the rules (lines 257–261) seem odd or incomplete. For example, in acyclicity rule 3, the premise is symmetric ((X < Y) and (Z < W) is equivalent to (Z < W) and (X < Y)), but the conclusion is not (Y -/-> Z). Moreover, rule 4 appears to exclude ancestral constraints of the form “X and Y do not cause each other.”

In practice, one might expect only a small number of ancestral constraints, which are unlikely to shrink the search space dramatically. The comparison against Gobnilp augmented with linear constraints looks strong, but Gobnilp is not designed for this task. A more informative baseline would assume the ancestral constraints are consistent with the optimal model and compare against a greedy method such as GES, which with oracle input should recover the same solution. While greedy algorithms cannot guarantee optimality, this would better indicate the efficiency gains of the proposed method.

Minor details:
- Line 30: Is there a meaningful distinction between a (standard) oracle and an “omniscient” oracle?
- Line 38: Does “empower the oracle” simply mean “use in combination with the oracle”?
- Line 90: Please note explicitly that “optimizing” here means minimizing.
- Line 164: Theorem 1 should make clear that the set of constraints A satisfied by CPDAG P on the EC tree is a subset of the constraints A satisfied by the full DAG G; otherwise the stated “iff” does not hold as written.
- Lines 225–227: Theorem 3 seems unlikely to yield many positive edge implications when ancestral constraints are sparse (small sets).
- Line 267: The encoding is not easily recognizable as enforcing acyclicity (see also the concern above).
- Line 310: For realistically sized graphs, having 10%–25% of ancestral constraints would already imply substantial prior knowledge of the target graph, which is atypical in practice.



The paper presents an efficient method for incorporating ancestral constraints into optimal Bayesian network learning from data. This is achieved by translating ancestral constraints into a maximal set of ordering constraints, which are then integrated into a score-based learning algorithm utilizing the EC tree framework from [Chen et al., 2015]. Evaluation against a modified ILP-based Gobnilp algorithm demonstrates significant performance improvements in the specified setting. The paper is well-written and clearly explained, though the solution is conceptually straightforward. The problem addressed is novel (novelty=3), as optimal Bayesian network learning with ancestral constraints lacks a standard solution; however, the context is somewhat artificial, as ancestral relations typically warrant more general causal discovery methods accommodating unobserved confounders (impact=2). The approach effectively maps difficult-to-use ancestral constraints—unsuitable for decomposable-score algorithms—into ordering constraints expressible via edge absence, using a MaxSAT solver to enforce all implied ancestral constraints. However, certain rules (lines 257–261) appear problematic: Rule 3’s premise is symmetric ((X < Y) ∧ (Z < W) ≡ (Z < W) ∧ (X < Y)), but its conclusion (Y ↛ Z) is not; Rule 4 excludes ancestral constraints of the form "X and Y do not cause each other." Additionally, sparse ancestral constraints are unlikely to substantially reduce the search space. While the comparison against Gobnilp with added linear constraints appears strong, the latter is not designed for this problem. A more meaningful benchmark would compare against a greedy algorithm (e.g., GES) under oracle input, which should yield the same output as the optimal method, providing a clearer efficiency assessment. Minor points:  
- Line 30: Clarify difference between (standard) oracle and omniscient oracle.  
- Line 38: "Empower the oracle" likely means "use in combination with the oracle."  
- Line 90: Specify "optimizing" refers to minimization.  
- Line 164: Theorem 1 must clarify that the constraint set A satisfied by CPDAG P on the EC tree is a subset of that satisfied by the full DAG G (otherwise the iff fails).  
- Lines 225–227: Theorem 3 likely yields few positive edge implications with sparse ancestral constraints.  
- Line 267: The constraint is unclear as capturing "acyclic" (see earlier comment).  
- Line 310: 10%–25% ancestral constraints for reasonable graph sizes would imply extensive target graph knowledge (not typical in practice).

- Addresses learning Bayesian network structures under ancestral constraints using branch-and-bound (A*) search over CPDAGs.  
- Nodes represent CPDAGs; refinement picks a specific parent set for a variable.  
- Incorporates effective symmetry-breaking techniques.  
- Heuristic is taken from the URLearning system, but integration details are cited rather than explained.  
- Ancestral constraints enable extra pruning; Lemma 3 gives the key pruning result but its proof is unclear.  
- Lemma 3’s proof needs clarification, specifically the claim "By the EC tree edge generation rules, Gk also contains edge Z -> W."  
- “Projected constraints” correspond to standard “implied constraints”; ancestral constraints can imply edge constraints useful for pruning.  
- The method for inferring ordering constraints is inadequately explained; “can infer Y < Z from Z not an ancestor of Y” is misleading without consistency considerations.  
- Authors use a MAXSAT approach to infer a maximal, consistent set of ordering constraints.  
- Experiments vary variables, data points, and constraints and show the proposed method much faster than GOBNILP.  
- Comparison to GOBNILP is weakened because parent set size is unbounded (favoring A* approaches) and the ILP encoding for ancestral constraints is a weak MAXSAT-style formulation.  
- Recommends a stronger ILP using binary ancestry vars A(X,Y) linked to parent indicators and transitivity constraints for a tighter relaxation.  
- Belief that A* would still outperform GOBNILP with a better ILP, but the current benchmark is against a suboptimal competitor implementation.  
- Suggests adding a comparison to van Beek & Hoffmann (2015) constraint-programming approach, which is high-performing and omitted from related work.

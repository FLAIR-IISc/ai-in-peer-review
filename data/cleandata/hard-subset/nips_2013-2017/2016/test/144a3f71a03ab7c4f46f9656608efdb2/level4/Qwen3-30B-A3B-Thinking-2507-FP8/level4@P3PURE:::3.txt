

The paper presents an efficient method for incorporating ancestral constraints into optimal Bayesian network learning from data. It achieves this by translating a set of ancestral constraints into a maximal set of ordering constraints, which are then integrated into a score-based learning algorithm leveraging the EC tree framework introduced by Chen et al. (2015). The approach is evaluated against a modified ILP-based Gobnilp algorithm (with added linear constraints) and demonstrates significant performance gains in the given setting. The paper is reasonably well written and clearly explained, though the solution itself is conceptually straightforward. The problem addressed is novel in that optimal Bayesian network learning with ancestral constraints lacks a readily available "standard solution" (novelty=3). However, the problem setting is somewhat artificial: once ancestral relations are considered, more general causal discovery methods accommodating unobserved confounders would be more appropriate (hence impact=2). The core approach translates hard-to-use ancestral constraints—unsuitable for decomposable-score-based algorithms—into ordering constraints expressible via edge absence, which align with decomposable scores. This mapping employs a MaxSAT solver to explicitly encode all implied ancestral constraints. Nevertheless, certain rules in lines 257–261 appear problematic or incomplete. For instance, Rule 3’s premise is symmetric (e.g., (X < Y) ∧ (Z < W) ≡ (Z < W) ∧ (X < Y)), yet its conclusion (Y ↛ Z) is asymmetric. Rule 4 also excludes ancestral constraints of the form "X and Y do not cause each other." Moreover, in practice, only a small number of ancestral constraints are typically provided, which would likely have minimal impact on reducing the search space. While the comparison against modified Gobnilp appears impressive, the latter algorithm is not designed for this specific problem. A more meaningful performance benchmark would assume ancestral constraints align with the optimal model and compare against a greedy baseline like GES (which, with oracle input, should recover the same solution). Though greedy methods cannot guarantee optimality, this comparison would better indicate the efficiency gains of the proposed method.  

Minor details:  
- Line 30: Clarify the distinction between a (standard) oracle and an omniscient oracle.  
- Line 38: "Empower the oracle" likely means "use in combination with the oracle."  
- Line 90: Specify "minimizing" instead of "optimizing."  
- Line 164: Theorem 1 must explicitly state that the set of constraints A satisfied by CPDAG P on the EC tree is a subset of those satisfied by the full DAG G; otherwise, the iff condition fails.  
- Lines 225–227: Theorem 3 is unlikely to yield significant positive edge implications under sparse ancestral constraints.  
- Line 267: The acyclicity condition is not clearly recognizable (see also Rule 3 critique above).  
- Line 310: For reasonable graph sizes, 10%–25% ancestral constraints would typically imply extensive knowledge of the target graph (i.e., such constraints are atypical in practice).

This paper presents an effective method for learning optimal Bayesian networks from data while incorporating ancestral constraints. The core of the approach involves translating a given set of ancestral constraints into a maximal set of ordering constraints. These ordering constraints are then utilized within a score-based learning algorithm that builds upon the EC tree structure proposed by [Chen et al., 2015]. The method's performance is benchmarked against a modified version of the ILP-based Gobnilp algorithm, demonstrating a significant performance advantage in the experiments conducted.

The paper is generally well-written with clear explanations. While the proposed solution is conceptually straightforward, it addresses a novel problem with a novel approach, yielding solid results that will be of interest to a specialized audience.

The problem of optimal BN learning under ancestral constraints is novel, as no standard solution currently exists for this task. However, the practical impact may be limited because the setting can be viewed as somewhat artificial. In scenarios where ancestral relations are known, it is often more practical to employ more general causal discovery algorithms that can also account for unobserved confounders.

The methodology focuses on converting ancestral constraints, which are not easily handled by algorithms using decomposable scores, into ordering constraints that are more compatible. This translation is performed using a MaxSAT solver to ensure all implied ancestral relations are made explicit. However, several of the encoding rules (lines 257-261) are unclear or appear incomplete. For example, the premise in acyclicity rule 3 is symmetric, but its conclusion is not. Rule 4 also seems to preclude the specification of mutual non-causation (i.e., 'X and Y do not cause each other'). Furthermore, in practical settings, one would expect only a small number of ancestral constraints to be available, which may not be sufficient to significantly prune the search space.

The experimental comparison against an augmented Gobnilp algorithm is impressive, but this baseline was not originally designed for this specific problem. A more insightful performance benchmark could be a comparison against a greedy algorithm like GES. Assuming the provided ancestral constraints are correct (i.e., align with the optimal model), GES with this oracle information should identify the same structure. While greedy methods lack optimality guarantees, this comparison would provide a more meaningful measure of the proposed method's efficiency.

Minor Points:
*   Line 30: Please clarify if there is a distinction between a "standard" oracle and an "omniscient oracle."
*   Line 38: The phrase "empower the oracle" is ambiguous; perhaps "use in combination with the oracle" is more precise.
*   Line 90: It would be helpful to explicitly state here that "optimizing" means "minimizing" the score.
*   Line 164: Theorem 1 should be stated more precisely. For the "if and only if" to hold, it should clarify that the set of constraints satisfied by the CPDAG on the EC tree is a subset of those satisfied by the complete DAG.
*   Lines 225-227: It seems unlikely that Theorem 3 would lead to many positive edge implications when given only a sparse set of ancestral constraints.
*   Line 267: The formulation intended to capture acyclicity is not immediately intuitive and connects to the earlier concern regarding the rules on lines 257-261.
*   Line 310: Using 10%-25% of all possible ancestral constraints for reasonably sized graphs implies a very high level of prior knowledge, which may not be typical in practice.

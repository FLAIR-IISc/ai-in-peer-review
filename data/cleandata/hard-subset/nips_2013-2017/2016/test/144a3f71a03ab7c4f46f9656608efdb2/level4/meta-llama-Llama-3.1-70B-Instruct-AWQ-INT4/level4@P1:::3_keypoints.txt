- Proposes translating ancestral constraints into a maximal set of ordering constraints and using them in a score-based EC-tree learning algorithm via a MaxSAT solver.  
- MaxSAT mapping is used to make all implied ancestral constraints explicit for decomposable-score-based learning.  
- Empirical results show substantial outperformance over a straightforward modification of the ILP-based Gobnilp algorithm.  
- Tackles a novel problem—optimal Bayesian network learning with ancestral constraints lacks standard solutions.  
- Concern: ancestral relations may be better handled by general causal discovery methods that allow unobserved confounders.  
- Some MaxSAT rules appear strange or incomplete (e.g., acyclicity rule has symmetric premise but asymmetric conclusion).  
- One MaxSAT rule seems to exclude constraints like “X and Y do not cause each other.”  
- In practice a small number of ancestral constraints is unlikely to significantly reduce the search space.  
- Comparison to Gobnilp is not entirely fair since Gobnilp was not designed for this problem.  
- A more meaningful baseline would be a greedy algorithm like GES assuming ancestral constraints align with the optimal model.  
- Theorem 1 should clarify that constraints A satisfied by the CPDAG on the EC tree are only a subset of those satisfied by the full DAG G.  
- Theorem 3 likely yields few positive edge implications when ancestral constraints are sparse.  
- The acyclicity rule in the MaxSAT formulation is not clearly recognizable as enforcing acyclicity.  
- Minor notation/phrasing issues: clarify difference between standard vs omniscient oracle and the meaning of “empowering the oracle.”  
- Practical note: 10–25% ancestral constraints for reasonable-sized graphs already imply extensive (and uncommon) knowledge of the target graph.

This paper presents an efficient method for incorporating ancestral constraints into the task of learning optimal Bayesian networks from data. The core idea is to translate a given set of ancestral constraints into a maximal set of ordering constraints. These ordering constraints are then integrated into a score-based learning algorithm that uses the EC tree structure from [Chen et al., 2015]. The proposed method is evaluated against a modified version of the ILP-based Gobnilp algorithm and demonstrates significantly better performance on this task. The paper is well-written and the approach is clearly explained, although conceptually straightforward. While not a spectacular breakthrough, the work addresses a novel problem with a practical solution, producing solid results that will be of interest to a specialist audience.

The problem of learning optimal Bayesian networks subject to ancestral constraints is interesting and novel, as there is no standard, off-the-shelf solution. However, this lack of a standard solution may be because the problem setting itself is somewhat artificial. In scenarios where ancestral relationships are known or hypothesized, it is often more natural to employ more general causal discovery algorithms that can also account for latent confounders.

The proposed approach translates ancestral constraints, which are difficult to incorporate directly into algorithms that rely on decomposable scores, into a set of ordering constraints. These ordering constraints are more easily handled, as they correspond to edge absences. This mapping is performed using a MaxSAT solver to derive the complete set of ordering constraints implied by the initial ancestral constraints. However, some of the translation rules (lines 257-261) appear questionable or incomplete. For instance, in acyclicity rule 3, the premise `(X < Y) and (Z < W)` is symmetric with respect to the two conjuncts, but the conclusion `Y -/-> Z` is not. Rule 4 also appears to exclude the ability to specify non-ancestral relationships of the form 'X and Y are not ancestors of each other'. Additionally, in practical applications, one would typically have only a small number of ancestral constraints, which may not be sufficient to significantly prune the search space.

The experimental comparison against a modified Gobnilp is impressive, but this baseline was not originally designed for this task. A more informative baseline could be a greedy search algorithm like GES. Assuming the provided ancestral constraints are consistent with the ground truth model, GES should recover the same optimal structure. While greedy algorithms do not guarantee optimality in the general case, such a comparison would provide a more meaningful benchmark for the efficiency gains achieved by the proposed method.

**Minor points:**

*   **Line 30:** Is there a distinction between a standard oracle and an 'omniscient oracle'?
*   **Line 38:** Does 'empower the oracle' mean 'use in combination with the oracle'? Clarification would be helpful.
*   **Line 90:** It should be stated explicitly that 'optimizing' the score means minimizing it.
*   **Line 164, Theorem 1:** The theorem should be more precise. It should state that the set of ancestral constraints satisfied by a CPDAG in the EC tree is a subset of those satisfied by a full DAG, as the 'if and only if' condition might not hold otherwise.
*   **Lines 225-227, Theorem 3:** It seems unlikely that this theorem would infer many compelled edges when starting from a small, sparse set of ancestral constraints, as would be expected in practice.
*   **Line 267:** The formulation for acyclicity is not immediately intuitive (this relates to the earlier comment on rule 3).
*   **Line 310:** The experiments use a large number of ancestral constraints (10%-25% of all possible pairs). For moderately sized graphs, this implies extensive prior knowledge of the target graph, which may not be representative of a typical use case.

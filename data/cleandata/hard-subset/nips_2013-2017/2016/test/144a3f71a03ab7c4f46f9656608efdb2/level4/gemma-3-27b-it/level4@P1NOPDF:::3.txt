The paper presents an efficient method for incorporating ancestral constraints into optimal Bayesian network (BN) learning from data. It achieves this by translating a set of ancestral constraints into a (maximal) set of ordering constraints, which are then used within a score-based learning algorithm based on the EC tree introduced by [Chen et al., 2015]. The results demonstrate significant performance improvements compared to a straightforward modification of the ILP-based Gobnilp algorithm in the given setting. Overall, the paper is reasonably well-written and clearly explained, although the core solution is conceptually straightforward. The work addresses a novel problem with a novel solution, yields positive results – while not spectacular – and is certainly of interest to a specialist audience. 

The problem tackled in this paper is interesting and new in the sense that optimal BN learning combined with ancestral constraints lacks a readily available 'standard' solution (novelty=3). However, the somewhat artificial nature of the setting partially explains this gap. Once ancestral relations are considered, more general causal discovery algorithms capable of handling unobserved confounders would likely be more appropriate (hence impact=2). The approach focuses on translating difficult-to-use ancestral constraints – which don't easily integrate with decomposable-score-based algorithms – into sets of ordering constraints more naturally expressed as the absence of edges, thereby aligning them with decomposable scores. This mapping is performed using a MaxSAT solver, which ensures that all implied ancestral constraints are explicitly accounted for. 

However, some of the rules presented in lines 257-261 appear either strange or incomplete. For example, in the acyclicity rule 3, the premise (X < Y and Z < W) is symmetric with (Z < W and X < Y), but the conclusion (Y -/-> Z) is not. Furthermore, rule 4 seems to exclude ancestral constraints stating that 'X and Y do not cause each other'. Moreover, it’s likely that, in practice, only a small number of ancestral constraints would be provided, and these may not significantly reduce the search space.

The comparison against Gobnilp with added linear constraints is impressive, but the latter was not specifically designed to address this problem. A more appropriate performance benchmark would involve assuming the ancestral constraints are consistent with the optimal model and comparing against a greedy algorithm like GES, which, given oracle input, should produce the same output. While greedy algorithms don’t guarantee optimality, this would provide a more meaningful indication of the method’s efficiency.

- Line 267: This doesn’t immediately read as capturing ‘acyclic’ (see also the comments on lines 257-261 above).
- Line 310: For reasonably sized graphs, 10%-25% ancestral constraints would already represent substantial prior knowledge about the target graph, which isn’t typical in practical applications.

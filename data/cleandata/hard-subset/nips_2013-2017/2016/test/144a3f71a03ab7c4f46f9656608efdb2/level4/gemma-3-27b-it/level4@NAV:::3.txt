The paper presents an effective method for incorporating ancestral constraints into optimal Bayesian network (BN) learning from data. It achieves this by converting ancestral constraints into a (maximal) set of ordering constraints, which are then utilized within a score-based learning algorithm built upon the EC tree framework of [Chen et al., 2015]. Empirical evaluation demonstrates significant performance gains over a modified ILP-based approach (Gobnilp) within the defined setting. The paper is generally well-written and clear in its explanations, though the underlying solution is relatively straightforward conceptually. The work addresses a novel problem with a corresponding solution, yielding a positive result that will be of interest to specialists in the field.

The problem of optimal BN learning with ancestral constraints is interesting and novel, as there is currently no established standard solution (novelty=3). However, the practical relevance of this setting is somewhat limited; when ancestral relationships are known, more general causal discovery algorithms capable of handling unobserved confounders are often more appropriate (hence impact=2). The core approach effectively transforms difficult-to-use ancestral constraints – which don't readily integrate with decomposable score-based algorithms – into more manageable ordering constraints representing edge absences, aligning them with the requirements of decomposable scores. A MaxSAT solver ensures the completeness of this mapping by explicitly representing all implied ancestral constraints. However, the rules detailed in lines 257-261 appear potentially flawed or incomplete; for example, the premise of acyclicity rule 3 exhibits symmetry while the conclusion does not, and rule 4 seems to exclude valid ancestral constraints like mutual non-causation between two variables. Furthermore, in realistic scenarios, only a small number of ancestral constraints are typically available, and it’s questionable whether these would substantially reduce the search space. 

The performance comparison against Gobnilp with added linear constraints is compelling, but Gobnilp is not designed to optimally address this specific problem. A more informative performance benchmark would involve assuming the ancestral constraints are consistent with the true underlying model and comparing against a greedy algorithm like GES, which should recover the optimal structure given oracle input. While greedy algorithms cannot guarantee optimality, this would provide a more meaningful assessment of the method’s efficiency. 

Minor points for consideration:
- Line 30: Clarify the distinction between a “standard” oracle and an “omniscient” oracle.
- Line 38: Rephrase “empower the oracle” to something like “use in combination with the oracle”.
- Line 90: Explicitly state that “optimizing” refers to “minimizing”.
- Line 164: Theorem 1 should explicitly state that the set of constraints A satisfied by the CPDAG P on the EC tree is a *subset* of those satisfied by the full DAG G to ensure the validity of the “if and only if” statement.
- Lines 225-227: Theorem 3 may not yield many positive edge implications, particularly with sparse ancestral constraints.
- Line 267: The rule’s connection to ‘acyclicity’ is unclear (see also comments on lines 257-261).
- Line 310: A 10%-25% prevalence of ancestral constraints would indicate substantial prior knowledge of the target graph, which is not typical in practical applications.

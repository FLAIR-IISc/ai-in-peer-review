

The paper presents an efficient method for incorporating ancestral constraints into optimal Bayesian network learning from data. It achieves this by translating ancestral constraints into a (maximal) set of ordering constraints, which are then integrated into a score-based learning algorithm utilizing the EC tree framework from Chen et al. (2015). The approach is evaluated against a modified ILP-based Gobnilp algorithm, demonstrating significant performance improvements. The paper is reasonably well written and clearly explained, though the solution is conceptually straightforward. The problem is novel (novelty=3), as optimal Bayesian network learning with ancestral constraints lacks a readily available standard solution. However, the problem setting is somewhat artificial: once ancestral relations are considered, more general causal discovery algorithms accommodating unobserved confounders would be more appropriate (impact=2). The method translates hard-to-use ancestral constraints—unsuited for decomposable-score-based algorithms—into ordering constraints expressible via edge absence, which align with decomposable scores. This translation employs a MaxSAT solver to ensure all implied ancestral constraints are explicitly captured. However, several rules in lines 257–261 appear problematic: Rule 3's premise is symmetric ((X < Y) ∧ (Z < W) ≡ (Z < W) ∧ (X < Y)), but its conclusion (Y ↛ Z) is not; Rule 4 may exclude ancestral constraints of the form "X and Y do not cause each other." Additionally, in practice, few ancestral constraints are typically provided and are unlikely to substantially reduce the search space. While the comparison against Gobnilp with added linear constraints appears impressive, Gobnilp is not designed for this problem. A more meaningful benchmark would assume ancestral constraints align with the optimal model and compare against a greedy algorithm like GES (which, for oracle input, should yield the same output), providing a better indication of the method's efficiency. Minor details:  
- Line 30: Clarify the distinction between a (standard) oracle and an omniscient oracle.  
- Line 38: "Empower the oracle" likely means "use in combination with the oracle."  
- Line 90: Specify that optimization refers to minimization.  
- Line 164: Theorem 1 must clarify that the set of constraints A satisfied by CPDAG P on the EC tree is a subset of those satisfied by the full DAG G, to ensure the iff condition holds.  
- Lines 225–227: Theorem 3 is unlikely to yield many positive edge implications under sparse ancestral constraints.  
- Line 267: The constraint is not clearly recognizable as capturing acyclicity (see above).  
- Line 310: For reasonable graph sizes, 10%–25% ancestral constraints would imply extensive knowledge of the target graph (thus not typical in practice).

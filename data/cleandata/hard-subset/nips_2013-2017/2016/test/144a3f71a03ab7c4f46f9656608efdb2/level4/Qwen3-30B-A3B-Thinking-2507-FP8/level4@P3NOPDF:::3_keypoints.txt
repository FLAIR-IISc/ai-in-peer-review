- Proposes incorporating ancestral constraints into optimal Bayesian network learning by translating them to ordering constraints using the EC tree framework and MaxSAT.
- Converts ancestral constraints into a maximal set of ordering constraints enabling efficient score-based learning; reports strong empirical gains over a modified ILP-based Gobnilp.
- Problem novelty is moderate: addresses a niche problem with no standard solution (score 3/5).
- Practical relevance is limited: ancestral constraints often arise in settings where fuller causal-discovery methods (e.g., handling confounders) are preferable (impact 2/5).
- Technical strength: MaxSAT-based translation captures implied constraints comprehensively, a key contribution.
- Formalization issues in Section 3 (Rules 257–261): Rule 3 has a symmetric premise but an asymmetric conclusion (logical ambiguity), and Rule 4 seems to exclude mutual non-causation, undermining completeness.
- Theorem 1 needs clarification that the constraint set A derived from CPDAG P is a subset of constraints satisfied by the underlying DAG G.
- The claim that ancestral constraints rarely reduce the search space is questionable; sparse constraints (10–25% pairs) can imply substantial structure for typical graphs (10–25 nodes).
- Theorem 3 has limited utility for sparse ancestral constraints, as it rarely yields positive edge implications (e.g., X → Y).
- Evaluation is misleading: Gobnilp isn’t designed for ancestral constraints, so that comparison may overstate the method’s value.
- A more meaningful baseline would be a greedy algorithm like GES augmented with the same ancestral constraints to assess practical efficiency gains.
- Minor editorial/clarity fixes needed: distinguish "standard" vs "omniscient" oracle (Line 30), change "empower the oracle" to "leverage the oracle" (Line 38), use "minimizing" the score (Line 90), and fix the acyclic Rule 3 wording (Line 267).
- Overall: well-written and technically valid within its constrained scope, modest novelty, recommendation Weak Accept (5/10); authors should fix logical inconsistencies and reframe evaluation to strengthen practical relevance.

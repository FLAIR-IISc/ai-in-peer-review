This paper describes an effective way to incorporate genetic constraints to find the best Bayesian networks from data. It translates a set of genetic constraints into a set of ordering constraints, which are then used in a score-based learning algorithm based on an EC tree, as introduced by [Chen et al., 2015]. The results are evaluated against a general modification of the ILP-based Gobnilp algorithm and perform significantly better in the given setting. Overall, it is reasonably well-written and clearly explained, although the solution is conceptually simple. New problem + solution, beautiful results, not very spectacular, but certainly interesting for an expert. The problem discussed in this paper is interesting and new in the sense that there is no readily available 'standard solution' for learning the best BN combined with genetic constraints (novelty = 3). However, one reason behind this apparent shortcoming is that the situation is somewhat artificial: once the genetic relationships are effective, it would be more meaningful to turn to a more general causal discovery algorithm that also allows for non-observed confounders. (Therefore, effect = 2) The method adopted is based on translating hard-to-understand genetic constraints, which do not fit well with a decomposable-score-based algorithm, into a set of ordering constraints that can be more easily expressed in terms of edge missing, which fits with a decomposable score. The mapping is done through a MaxSAT solver that ensures that all underlying genetic constraints are clearly expressed. However, some rules in 257-261 seem odd or incomplete: for example, the acyclicity rule 3 has the specific symmetric ( (X < Y) and (Z < W) == (Z < W) and (X < Y)), but not the conclusion (Y -/-> Z); also, rule 4 seems to exclude genetic constraints of the type 'X and Y are not causes of each other'. Furthermore, one might expect that only a few genetic constraints are given in reality, and these are unlikely to significantly reduce the search space. The results seem impressive compared to Gobnilp with reasonably logical linear constraints, but the latter was not designed to solve this problem. A reasonable alternative to performance measurements is to assume that the genetic constraints are consistent with the best model and compare it with a greedy algorithm like GES that would produce the same output for an oracle input. Of course, greedy algorithms cannot usually guarantee optimality, but it will provide a more meaningful indication of the efficiency achieved in the method mentioned on paper.

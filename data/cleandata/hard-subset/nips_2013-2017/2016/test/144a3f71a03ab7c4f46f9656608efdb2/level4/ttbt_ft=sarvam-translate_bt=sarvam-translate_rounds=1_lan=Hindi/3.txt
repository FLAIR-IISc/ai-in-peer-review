This paper describes an efficient means of incorporating parent constraints into finding optimal Bayesian networks from data. It does this by translating a set of parent constraints into a (maximum) set of sorted constraints, which are then used in a score-based learning algorithm based on the EC tree presented by [Chen et al., 2015]. The results are evaluated against a direct modification of the ILP-based Gobnilp algorithm and are shown to perform significantly better in the given setting. Overall, it is reasonably well-written and clearly explained, although the solution is conceptually straightforward. Novel problem + solution, good results, not spectacular, but certainly interesting for an expert audience. The problem tackled in this paper is interesting and novel in the sense that there is no readily available 'standard solution' for optimal BN learning in combination with parent constraints (novelty=3). However, one reason behind this apparent lack is that the situation is somewhat artificial: once parent constraints become prevalent, it would be more understandable to go for more general cause-finding algorithms that also allow for unseen confounders. (Hence impact=2). The approach adopted is based on translating difficult-to-use parent constraints, which do not fit well in decomposition-score-based algorithms, into a set of sorted constraints that are more easily expressed in terms of the absence of edges, which fit in decomposition scores. The mapping is done through a MaxSAT solver that ensures that all implicit parent constraints are clarified. However, some rules in 257-261 seem odd or incomplete: for example, acyclicity rule 3 is precise symmetric ((X < Y) and (Z < W) == (Z < W) and (X < Y)), but the conclusion (Y -/-> Z) is not; also, rule 4 seems to exclude parent constraints of the type 'X and Y do not cause each other'. Furthermore, one might expect that only a few parent constraints are given in practice, and these are unlikely to significantly reduce the search space. The resulting comparison against Gobnilp with additional linear constraints looks impressive, but the latter was not developed to deal with this problem. A reasonable alternative to measure performance would be to assume that the parent constraints correspond to the optimal model, and compare against greedy algorithms like GES, which would generate the same output for the oracle input. Of course, greedy algorithms cannot generally guarantee optimality, but this paper will provide at least a more meaningful indication of the efficiency achieved by the method.

- Paper proposes projecting non-decomposable ancestral constraints into decomposable constraints so decomposable-score-based structure learners can use them.  
- Main novelty is Section 5: transforming non-decomposable ancestral constraints into decomposable edge and topological-ordering constraints.  
- Paper is well-written and clearly presents theory and experiments.  
- Concern i: Table 3 shows projection helps EC trees but not ILP, raising questions about whether the method generalizes beyond EC trees and if decomposed constraints improve other learning frameworks.  
- Suggestion i: Demonstrate that decomposable constraints improve performance/efficiency for methods that previously couldn’t handle non-decomposable constraints.  
- Concern ii: Table 2 shows an “easy–hard–easy” pattern as ancestral constraint proportion increases; authors should explain this and whether EC tree search-space reduction is highly sensitive to constraint quantity versus ILP.  
- Suggestion ii: Clarify whether careful tuning of constraint numbers is required for optimal EC tree performance.  
- Concern iii: Table 2 reports SHD only for EC trees—need direct SHD comparison with the ILP approach.  
- Concern iv: Since experiments focus on EC trees, Section 4.1 (BN background) could be removed to allow more space to explain EC trees and the A* algorithm within that framework.

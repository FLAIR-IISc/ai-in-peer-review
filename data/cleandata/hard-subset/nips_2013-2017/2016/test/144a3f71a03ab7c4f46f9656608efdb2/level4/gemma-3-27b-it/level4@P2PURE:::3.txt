The paper presents an efficient method for incorporating ancestral constraints into optimal Bayesian network (BN) learning from data. It achieves this by translating a set of ancestral constraints into a (maximal) set of ordering constraints, which are then used within a score-based learning algorithm based on the EC tree framework introduced by [Chen et al., 2015]. Experimental results demonstrate significant performance gains over a modified ILP-based algorithm (Gobnilp) when evaluated in the given setting. The paper is generally well-written and clearly explains a conceptually straightforward solution. The combination of a novel problem and solution, coupled with promising results, makes this work of interest to a specialist audience. The problem addressed—optimal BN learning subject to ancestral constraints—is interesting and novel, as there is currently no standard solution readily available (novelty=3). However, the somewhat artificial nature of this setting partially explains this lack of existing methods; when ancestral relations are considered, more general causal discovery algorithms capable of handling unobserved confounders may be more appropriate (hence impact=2). The approach effectively transforms difficult-to-use ancestral constraints, which are ill-suited for decomposable-score-based algorithms, into sets of ordering constraints more readily expressed as the absence of edges – a format compatible with decomposable scores. A MaxSAT solver ensures all implied ancestral constraints are explicitly represented. However, some rules presented in lines 257-261 appear either incomplete or questionable; for example, the premise of acyclicity rule 3 is symmetric ((X < Y) and (Z < W) is equivalent to (Z < W) and (X < Y)), yet the conclusion (Y -/-> Z) is not. Furthermore, rule 4 seems to preclude ancestral constraints indicating mutual exclusion of causation between variables (i.e., 'X and Y do not cause each other'). It’s also plausible that, in practice, only a limited number of ancestral constraints would be provided, and these might not substantially reduce the search space. The comparison against Gobnilp with added linear constraints is impressive, but Gobnilp was not designed to address this specific problem. A more informative performance comparison would involve assuming the ancestral constraints are consistent with the optimal model and evaluating against a greedy algorithm like GES, which should reproduce the correct result given oracle input. While greedy algorithms cannot guarantee optimality, this would provide a more meaningful indication of the method's efficiency. Minor points: - Line 30: Is there a distinction intended between a (standard) oracle and an omniscient oracle? - Line 38: Should “empower the oracle” be read as “use in combination with the oracle”? - Line 90: Clarify that “optimizing” refers to minimizing. - Line 164: Theorem 1 should explicitly state that the set of constraints A satisfied by the CPDAG P on the EC tree is only a subset of the constraints A satisfied by the full DAG G, to ensure the correctness of the “if and only if” statement. - Lines 225-227: Theorem 3 is unlikely to yield many positive edge implications, particularly given sparse (small) sets of ancestral constraints. - Line 267: This description is not immediately recognizable as capturing 'acyclic' (see also the comments on lines 257-261). - Line 310: For graphs of reasonable size, 10%-25% ancestral constraints would already represent considerable prior knowledge of the target graph, which is not typical in many practical scenarios.

This paper describes an effective way to incorporate genetic constraints in finding optimal Bayesian networks from data. It translates a set of genetic constraints into a set of ordering constraints, which are then used in a score-based learning algorithm based on the EC tree, as introduced by [Chen et al., 2015]. The results are evaluated against a simple modification of the ILP-based Gobnilp algorithm and perform significantly better in the given setting. Overall, it is reasonably well-written and clearly explained, although the solution is conceptually simple. The new problem + solution, beautiful results, not so spectacular, but certainly interesting for an expert. The problem discussed in this paper is interesting and new in the sense that there is no readily available 'standard solution' for optimal BN learning combined with genetic constraints (novelty=3). However, one reason behind this apparent deficiency is that the situation is somewhat artificial: once genetic relationships are effective, it would be more meaningful to turn to more general causal discovery algorithms that also allow for non-observed confounders. (Therefore, influence=2) The method adopted is based on translating hard-to-use genetic constraints, which do not fit well with decomposable-score-based algorithms, into a set of ordering constraints that can be more easily expressed in terms of edge absence, which fits with decomposable scores. The mapping is done through a MaxSAT solver that ensures that all underlying genetic constraints are clearly expressed. However, some rules in 257-261 seem odd or incomplete: for example, acyclicity rule 3's precise symmetric ( (X < Y) and (Z < W) == (Z < W) and (X < Y)), but the conclusion (Y -/-> Z) is not; also, rule 4 seems to exclude genetic constraints of the type 'X and Y are not causes of each other'. Furthermore, one might expect that only a few genetic constraints are given in reality, and these are unlikely to significantly reduce the search space. The results appear impressive compared to Gobnilp with reasonably logical linear constraints, but the latter was not designed to address this problem. A reasonable alternative to measuring performance is to assume that the genetic constraints are consistent with the optimal model, and compare it with a greedy algorithm like GES that would produce the same output for an oracle input. Of course, greedy algorithms cannot generally guarantee optimality, but it will provide a more meaningful indication of the efficiency achieved in the method mentioned on paper.

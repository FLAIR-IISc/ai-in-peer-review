The paper presents an efficient method for incorporating ancestral constraints into optimal Bayesian network learning from data. It does so by converting a set of ancestral constraints into a (maximal) collection of ordering constraints, which are then used within a score-based search leveraging the EC tree of [Chen et al., 2015]. The empirical evaluation compares against a straightforwardly adapted ILP-based Gobnilp baseline and shows substantial gains in this setting. The manuscript is generally clear and well organized, though the core idea is conceptually simple. New problem + solution, solid outcome; not groundbreaking, but certainly relevant for specialists. The problem addressed is interesting and novel insofar as optimal BN learning with ancestral constraints lacks an off-the-shelf solution (novelty=3). That said, part of this lack is due to the somewhat artificial nature of the setup: when ancestral relations are available, it is often more natural to use broader causal discovery methods that can handle latent confounding (hence impact=2). The method works by translating hard-to-handle ancestral constraints—poorly matched to decomposable-score frameworks—into ordering constraints that can be phrased as absence-of-edge statements, which do integrate with decomposable scores. The translation uses a MaxSAT solver to make all implied ancestral constraints explicit. However, some of the rules on lines 257–261 appear odd or incomplete: for instance, the premise of acyclicity rule 3 is symmetric ((X < Y) and (Z < W) == (Z < W) and (X < Y)), while the conclusion is not (Y -/-> Z); moreover, rule 4 seems to rule out ancestral constraints of the form “X and Y do not cause each other.” In addition, in realistic scenarios one would expect only a few ancestral constraints, which likely will not shrink the search space substantially. The comparison against Gobnilp augmented with linear constraints looks strong, but Gobnilp is not designed for this task. A more informative performance baseline would assume the ancestral constraints are consistent with the optimal model and compare against a greedy method like GES, which with oracle access should recover the same solution. While greedy approaches do not guarantee optimality, such a comparison would better reflect the efficiency achieved by the proposed approach. Minor details: - 30: Is there a distinction between a (standard) oracle and an omniscient oracle? - 38: “empower the oracle” = “use in combination with the oracle”? - 90: Note here that optimizing = minimizing. - 164: Theorem 1 should more clearly state that the set of constraints A satisfied by CPDAG P on the EC tree is only a subset of the constraints A satisfied by the full DAG G (otherwise the iff does not hold). - 225–227: Theorem 3 seems unlikely to yield many positive edge implications given sparse (small sets of) ancestral constraints. - 267: Not easily recognizable as encoding “acyclic” (see also above). - 310: For reasonably sized graphs, 10%–25% ancestral constraints would already imply substantial knowledge of the target graph (thus not typical in practice).

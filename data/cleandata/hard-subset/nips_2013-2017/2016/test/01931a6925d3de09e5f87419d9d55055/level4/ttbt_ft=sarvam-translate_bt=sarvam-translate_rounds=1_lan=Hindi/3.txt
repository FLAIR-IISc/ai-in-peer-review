This paper proposes modeling the observation space with a handmade Riemannian metric and defining a curved Gaussian distribution based on the corresponding Riemannian geometry to better capture the data manifold. The paper addresses a fundamental problem of how to define a Riemannian metric based on a set of high-dimensional real-valued observations, and then define and estimate a curved Gaussian distribution based on this geometry. It presents a useful tool and an interesting application of Riemannian geometry in statistical learning. The first part of the proposed method defines the underlying Riemannian geometry in Section 3.1. The title suggests that it "learns a metric." However, the metric is actually handmade in equation 7 based on two hyper-parameters, rho and sigma. These hyper-parameters are empirically predetermined. I assume that sigma is a sensitive parameter. The paper should discuss in more detail what the smooth effect is when changing sigma, and how to select this sigma. The paper is incomplete due to the missing part. Is it possible to merge the metric learning stage with the LAND learning stage, so that rho and sigma can be learned? A beauty of the geometric approach is the immutability to coordinate changes. However, the metric in equation 7 is not immutable. For example, if the data creates a line in the 2D observation space. We will have completely different metrics in one coordinate system, whose x-axis will be parallel to this line, and in another coordinate system that makes an angle of, for example, 45 degrees to this line. It is useful to discuss this in more detail, including whether the corresponding absolute (rather than diagonal) metric is immutable, and provides more insight into the metric in equation 7. There is a rich literature in metric learning and manifold learning related to Riemannian geometry that can be linked to this work (see Google "Riemannian" "Metric Learning" "Manifold Learning"). As a major criticism, in the first experiment, the comparison is not so meaningful because it can clearly describe the data better with a smaller number of more flexible LAND components. However, it can describe the data better with a larger number of GMM components when LAND overfits. The author(s) are suggested to redesign the experiment, so that each comparison method is given the freedom to choose its own optimal number of components based on some criterion (such as BIC), then compare the corresponding performance. LAND is still expected to perform better because it describes the manifold structure better, and this comparison is more meaningful. It is good to see how LAND will overfit with a large number of components. LAND is clearly a more expensive process than estimating Gaussian. Are there any quantitative measures or experiments to clarify how expensive it is? What is the limit of the scale that LAND can handle? Is it possible to learn LAND on MNIST and compare it with Gaussian mixture learning? ====== After reading the authors' response, LAND is a more flexible model than the Gaussian distribution. If you increase the number of components, each local LAND will be more sensitive to local noise than the local Gaussian. "Geodesics will be almost straight lines", true, but it is still a curve, and these curves will lead to overfitting. I highly suggest the authors modify this experiment in the revision.

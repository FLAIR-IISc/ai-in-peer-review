This paper proposes modeling the observation space using a handmade Riemannian metric and defines a curved Gaussian distribution based on the corresponding Riemannian geometry to better capture the data manifold. The paper discusses a fundamental problem of how to define a Riemannian metric based on a set of high-dimensional real-valued observations and then how to define and estimate a curved Gaussian distribution based on this geometry. It presents a useful tool and an interesting application of Riemannian geometry in statistical learning. The first part of the proposed method defines the underlying Riemannian geometry in section 3.1. The title suggests that it "learns a metric." However, the metric is actually handmade based on two hyper-parameters, rho and sigma, in equation 7. These hyper-parameters are empirically predetermined. I believe sigma is a sensitive parameter. The paper should discuss in more detail what the intuitive effect is when changing sigma and how to choose this sigma. The paper remains incomplete if this part is missing. Is it possible to combine the metric learning phase with the LAND learning phase so that rho and sigma can be learned? A beauty of the geometric method is its invariant nature towards coordinate transformations. However, the metric in equation 7 is not invariant. For example, if the data creates a line in a 2D observation space. We will have a completely different metric in one coordinate system whose x-axis is parallel to this line, and another coordinate system that makes an angle with this line, such as 45 degrees. This needs to be discussed in more detail, including whether the corresponding full (non-diagonal) metric is invariant, and further concepts of the metric in equation 7 should be given. There is a rich literature on metric learning and manifold learning with connections to Riemannian geometry that may be relevant to this work (see Google "Riemannian" "Metric Learning" "Manifold Learning"). As a major criticism, in the first experiment, the comparison is not very meaningful because LAND can describe the data better with fewer elements in a more flexible way. However, when LAND overfits, GMM can describe the data better with more elements. The author(s) are advised to redesign the experiment so that each comparison method can select its optimal number of elements based on a specific criterion (such as BIC), then compare the corresponding performance. LAND is still expected to perform well because it describes the manifold structure better, and this comparison is more meaningful. It is good to see how LAND will overfit with a large number of elements. LAND is clearly a more expensive method than a Gaussian assumption. Is there any quantitative measure or test to clarify how expensive it is? What is the limit of what scale LAND can handle? Is it possible to compare LAND learning and Gaussian mixture learning in MNIST? ====== After reading the authors' response, LAND is a more flexible model than the Gaussian distribution. If you increase the number of elements, each local LAND will be more sensitive to local noise than the local Gaussian. "Geodesics will be almost straight lines," true, but it is still curved, and these curves will lead to over-fitting. I highly recommend the author(s) to correct this experiment.

This paper proposes to model the observation space with a hand-crafted Riemannian metric and to define a curved normal distribution based on the associated geometry, so that the data manifold can be captured more faithfully. It addresses a fundamental problem: how to define a Riemannian metric from high-dimensional real-valued observations and then how to define and estimate a curved Gaussian (LAND) on this geometry. This is a useful tool and an interesting application of Riemannian geometry in statistical learning.

However, there are important issues with the way the metric is specified. Section 3.1 “learns a metric” in title, but the metric in Equation (7) is actually hand-crafted and depends on two hyperparameters, rho and sigma, which are fixed empirically in advance. Sigma in particular seems likely to be sensitive. The paper should discuss in more detail the intuitive effect of varying sigma, and provide guidance on how to select it. Without this, the method description feels incomplete. Is it possible to merge the metric-specification stage with the LAND estimation stage, so that rho and sigma are learned jointly from data?

A key appeal of geometric approaches is invariance to coordinate transformations. The metric in Equation (7) does not appear invariant. For example, if the data form a line in a 2D observation space, the induced metric will differ substantially between a coordinate system whose x-axis is aligned with the line and one rotated by, say, 45 degrees. This deserves a more thorough discussion. In particular: would a full (rather than diagonal) version of the metric achieve invariance, and if not, why not? Additional intuition for the specific choice in Equation (7) would be valuable.

There is a rich literature on metric learning and manifold learning with connections to Riemannian geometry that should be discussed and positioned relative to this work (see, e.g., searches for “Riemannian,” “metric learning,” and “manifold learning”).

My main experimental concern is with the first comparison. As presented, it is not very informative: a more flexible model like LAND will naturally fit a dataset better with a small number of components, while a GMM can fit better with a larger number of components—precisely where LAND may overfit. I recommend redesigning the experiment so each method selects its preferred number of components via a principled criterion (e.g., BIC), and then compare performance. LAND should still have an advantage if it truly captures manifold structure, and this would yield a more meaningful comparison. It would also be helpful to show explicitly how LAND behaves (and overfits) as the number of components grows.

Computational cost is another concern. LAND is clearly more expensive than fitting a Gaussian. Please provide quantitative measurements or experiments to clarify the overhead: runtime, memory, and scaling with sample size and dimension. What is the practical limit on dataset size? For instance, can LAND be trained on MNIST, and how does it compare empirically to a GMM there?

After reading the authors’ response: LAND is indeed more flexible than a Gaussian. However, increasing the number of components makes each local LAND more vulnerable to local noise than a local Gaussian. Even if “the geodesics will almost be straight lines,” they are still curves, and this curvature can lead to overfitting. I strongly recommend revising the experimental design accordingly in the next version.

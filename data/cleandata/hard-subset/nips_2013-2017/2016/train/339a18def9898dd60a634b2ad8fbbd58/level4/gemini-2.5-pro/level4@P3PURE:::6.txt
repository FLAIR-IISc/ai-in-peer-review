This paper presents an information-theoretic bound on the budget-fidelity tradeoff for label crowdsourcing, leveraging rate-distortion theory and the channel coding theorem. The derivation of the bound is rigorous and well-presented. If this is the first work to apply these information-theoretic tools to the crowdsourcing problem, it could represent a significant contribution to the field.

However, the novelty and practical value of this work are questionable. The derivation itself is a relatively straightforward application of established theory. More importantly, the resulting bound is at risk of being too loose to be practically useful. The core problem lies in the analogy between crowdsourcing and channel coding.

In classical channel coding, the capacity bound is tight and achievable because there is immense freedom in designing codebooks (e.g., repetition codes, Hamming codes, LDPC codes). For the capacity to be a meaningful bound, almost all mappings from messages to codewords must be valid and implementable. In crowdsourcing, however, it is not clear that such freedom exists. The paper frames the assignment of queries to workers as an encoding step (equation 3), but the set of valid "codebooks" or encoding schemes in crowdsourcing is likely subject to fundamental constraints not present in communication channels.

A striking difference illustrates this point. In channel coding, one can transmit the same symbol repeatedly over an i.i.d. channel. The analogous action in crowdsourcing would be to assign the same query to the same worker multiple times. This would likely violate the i.i.d. assumption of the worker model (equations 1 and 2), as a worker will probably generate the same answer repeatedly, introducing strong correlations. While a simple repetition code (querying multiple *different* workers about the same item) is possible, and naturally decoded by majority voting, it is unclear how more sophisticated error-correcting schemes could be designed for `u^n`.

In general, the crowdsourcing problem imposes significant constraints on the set of possible encoding schemes in equation (3), unlike the classical channel coding problem where the encoding map `{0,1}^{nR} -> {0,1}^n` has few constraints. Due to these inherent limitations on the "design space" of query assignments, the proposed information-theoretic bound may be very loose.

To demonstrate the practical relevance of the bound, it would be beneficial to compare its value against the empirical performance of state-of-the-art crowdsourcing algorithms. This would help quantify the gap between the theoretical limit and what is practically achievable.

This paper models digital crowdsourcing in an information theoretical framework to provide a lower bound on the number of queries required for each task to achieve a given probability of error. Workers are modeled as parallel independent channels. The error and spammer-hammer channel model SHC(q) is used when a worker is a spammer with probability 1-q, considering symmetric memory-sufficient channels (MSC) to model that situation. The code is assumed to be non-adaptive. Two cases are considered for analysis: firstly, when the crowdsourcing, who determines the value of each task after collecting results from workers, knows the skill level of all workers and when he knows nothing about them. In both cases, the taskmaster, who distributes tasks among workers, is assumed to know nothing about the workers' skills, so each worker is given the same number of tasks. Finally, the minimum required rate – the number of queries per task – is a necessary condition on k-ary incidence coding, where the worker is shown k items and asked whether they are of the same type or different types, where SHC(q) was provided for the worker pool. A well-organized and well-written paper. There was no test or simulation to show how far the schemes currently used in AMT are from the limit. I think section 2.4 can be shortened and section 3.1 is not needed or can be shortened. Instead, the proof sketches of the theorems are more important. It seems novel, there is no analysis of the crowdsourcing problem from an information theoretical point of view, as far as I know and the paper claims. It seems very useful where designers of such crowdsourcing projects can know the minimum required rate to complete a task with a given probability of error and it can help in determining the value of such projects. I have not read the appendix so I cannot claim anything about the proof.

This paper provides an information-theoretic bound on the budget-fidelity tradeoff for label crowdsourcing using rate-distortion theory and the channel coding theorem. The information-theoretic bound is well-derived and represents a solid technical contribution. If this is the first attempt to apply rate-distortion theory and the channel coding theorem to the crowdsourcing problem, it could be a reasonably important contribution to the field. However, the novelty and practical value of this paper are still questionable. The derivation of the information-theoretic bound is relatively straightforward, and the bound risks being very loose.

The main problem is that, unlike the channel coding problem, where we have a large amount of freedom in designing a codebook or codewords (e.g., repetition code, Hamming code, convolutional code, RS code, low-density parity-check (LDPC) code), it is unclear whether we have such freedom in designing the sequence of channel input u^n. In equation (3), almost all mappings from B(X)^n to U^(\sum mi) (except degenerate ones) should be valid or implementable codebooks; otherwise, the channel capacity may be a loose bound. For crowdsourcing, simple schemes like repetition code are possible, where the natural decoding method is majority voting. However, beyond this, it is unclear how to design u^n to improve the error-correcting capability.

There is a striking difference between the channel coding problem and the crowdsourcing problem. In the latter, we cannot assign the same query to the same worker multiple times, as this would ruin the iid assumption in equation (1) or (2). The same worker will likely generate the same answer to the same query, whereas in a communication channel, we can assume the channel is iid, and the same query can be used multiple times for the same channel. This is just one example, and in general, it is very likely that the crowdsourcing problem has many fundamental constraints in equation (3), unlike channel coding, where an encoding mapping {0,1}^{nR} -> {0,1}^n has little constraint. Therefore, there should be important constraints on the set of possible encoding schemes in equation (3), and due to such constraints, the proposed information-theoretic bound risks being very loose.

To strengthen the paper, it would be beneficial to compare the performance of state-of-the-art methods with the derived information-theoretic bound. This would help to better understand the tightness of the bound and its practical implications.

This paper derives an information-theoretic bound on the budget–fidelity trade-off for label crowdsourcing using rate–distortion theory and the channel coding theorem. The bound is clearly and carefully derived. If this is indeed the first application of rate–distortion and channel coding ideas to crowdsourcing, the contribution could be meaningful.

That said, both the novelty and practical value are uncertain. The derivation appears relatively straightforward, and the resulting bound risks being quite loose. The core issue is the mismatch between the freedom available in classical channel coding and the constraints inherent in crowdsourcing. In channel coding, one can design codebooks with substantial flexibility (e.g., repetition, Hamming, convolutional, Reed–Solomon, LDPC), which justifies capacity-based bounds. In contrast, for crowdsourcing it is not clear that a comparable freedom exists in constructing the sequence of channel inputs u^n. In Eq. (3), nearly all mappings from B(X)^n to U^{Σ m_i} (except trivial ones) would need to be valid/implementable encoders for the capacity bound to be tight. This seems doubtful.

While a repetition strategy (with majority voting) is available, it is unclear how to construct richer “codes” u^n that enhance error correction under realistic crowdsourcing constraints. Critically, unlike a memoryless communication channel, one typically cannot assign the same query multiple times to the same worker without violating the i.i.d. assumptions underlying Eqs. (1) and (2); the same worker is likely to repeat the same answer, inducing dependence. In communication, we can assume i.i.d. uses of the channel and reuse the same input across uses; in crowdsourcing, worker behavior, memory, and strategic effects break this symmetry. This is only one example, but it suggests that crowdsourcing imposes strong structural constraints on the encoder class in Eq. (3), whereas classical channel coding allows an almost unconstrained mapping {0,1}^{nR} → {0,1}^n. Without explicitly modeling and enforcing such constraints, the proposed information-theoretic bound may substantially overestimate what is achievable, and thus be overly optimistic.

To bolster the practical relevance, the paper should characterize the feasible set of encoding/assignment schemes in Eq. (3) under realistic crowdsourcing constraints (e.g., per-worker reuse limits, dependence across repeated questions, heterogeneity in worker reliabilities, task types), and discuss how these constraints affect tightness. Empirical comparisons would also help: juxtapose the bound with the performance of state-of-the-art aggregation/assignment methods to illustrate how tight the bound is in practice across regimes of budget, worker quality, and task difficulty.

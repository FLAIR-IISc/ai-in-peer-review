This paper presents an information-theoretic bound on the budget-fidelity tradeoff for label crowdsourcing, leveraging rate-distortion theory and the channel coding theorem. The mathematical derivation of this bound is sound. If this is the first work to apply these theoretical tools to the crowdsourcing problem, it represents a potentially significant contribution to the field.

However, the novelty and practical value of this work are questionable. The derivation of the information-theoretic bound is relatively straightforward, and more importantly, the bound risks being too loose to be meaningful in practice. The core issue lies in the analogy drawn between crowdsourcing and channel coding. For the channel capacity to be a tight bound, a large variety of encoding schemes must be possible. In classic channel coding, there is considerable freedom in designing codebooks and codewords (e.g., repetition codes, Hamming codes, convolutional codes, LDPC codes). It is not clear that such freedom exists in the crowdsourcing setting. For the bound to be meaningful, most mappings from the true labels to the worker queries (from `B(X)^n` to `U^(∑mi)` in Equation 3) must correspond to valid, implementable "codebooks."

While a simple repetition code—querying multiple workers about the same item, with majority voting as the natural decoder—has a clear parallel in crowdsourcing, it is unclear how more sophisticated error-correcting codes could be implemented. There is a key difference between the two problem domains: in crowdsourcing, one cannot assign the same query to the *same worker* multiple times. Doing so would violate the i.i.d. assumption (Equations 1 and 2), as a worker would likely provide the same answer if asked repeatedly. In contrast, a communication channel is typically assumed to be i.i.d. and memoryless, allowing the same input to be transmitted multiple times to improve reliability.

This is just one example of what are likely many fundamental constraints on the set of possible encoding schemes in crowdsourcing (Equation 3). Unlike standard channel coding, where the encoding map `{0,1}^{nR} -> {0,1}^n` is largely unconstrained, the crowdsourcing problem is inherently more restricted. Due to these constraints, the proposed information-theoretic bound may be very loose and thus of limited practical use.

To demonstrate the relevance of the derived bound, the paper would be significantly strengthened by comparing the performance of state-of-the-art crowdsourcing algorithms against this theoretical limit. Such a comparison would help quantify how tight the bound is and assess whether existing methods are approaching it.

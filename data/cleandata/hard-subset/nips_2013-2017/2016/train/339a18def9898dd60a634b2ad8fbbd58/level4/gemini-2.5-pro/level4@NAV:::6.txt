This paper presents a well-derived information-theoretic bound for the budget-fidelity tradeoff in label crowdsourcing, leveraging principles from rate-distortion theory and the channel coding theorem. Should this be the pioneering application of this theoretical framework to the crowdsourcing problem, the work represents a potentially significant contribution to the field.

However, the novelty and practical utility of this work are questionable. The derivation of the bound is relatively direct, and more critically, the bound itself risks being too loose to be meaningful. The central issue stems from a flawed analogy between the crowdsourcing setup and a classic channel coding problem. In channel coding, there exists immense freedom in designing sophisticated codebooks and codewords (e.g., repetition, Hamming, convolutional, LDPC codes). It is not at all clear that a similar degree of freedom exists for designing the input sequence `u^n` in the crowdsourcing context. For the channel capacity to be a tight bound, nearly all mappings defined in Equation (3) must correspond to valid, implementable "coding" schemes.

While a simple repetition code is feasible in crowdsourcing (i.e., assigning the same task to multiple workers and using majority voting as the decoder), the paper fails to explore how more complex `u^n` sequences could be designed to improve error-correcting capabilities. A striking difference is that one cannot assign the same query to the same worker multiple times without violating the i.i.d. assumption of Equations (1) and (2), as a worker will likely produce correlated answers. This is unlike a communication channel, where repeated transmissions of the same symbol over an i.i.d. channel are standard practice. This example illustrates that the crowdsourcing problem imposes fundamental constraints on the set of possible encoding schemes in Equation (3), unlike the largely unconstrained encoding maps found in channel coding. Due to these constraints, the proposed information-theoretic bound is likely to be very loose.

To demonstrate the practical value of this theoretical exercise, the paper would be much stronger if it included a comparison between the derived bound and the empirical performance of state-of-the-art crowdsourcing methods.



This work proposes an information-theoretic bound on the budget-fidelity tradeoff in label crowdsourcing using rate-distortion theory and the channel coding theorem. The derivation of the bound is mathematically rigorous, and if this represents the first application of these frameworks to crowdsourcing, the contribution could be significant. However, the novelty and practical value of the paper remain questionable due to two key limitations: the bound’s potential looseness and insufficient justification for the applicability of channel coding principles to the crowdsourcing setting.

The core argument hinges on treating crowdsourcing as a communication channel where the "channel input" \(u^n\) encodes worker responses to queries. The authors derive a bound based on channel capacity, assuming arbitrary encoding mappings from the label space to the input space (Equation 3). This assumption, however, overlooks a fundamental constraint in crowdsourcing: the inability to assign identical queries to the same worker multiple times without violating the independent and identically distributed (i.i.d.) assumption (Equations 1–2). In communication, the same channel input (e.g., a codeword bit) can be reused across channel uses due to the i.i.d. channel model. In crowdsourcing, repeating a query to the same worker typically yields identical responses (e.g., "always labels this image as 'cat'"), introducing correlation and breaking the i.i.d. premise. This constraint is absent in channel coding, where encoding schemes (e.g., repetition codes, LDPC, convolutional codes) are unconstrained by query repetition limits.

The paper’s claim that "almost all mappings from \(\mathcal{B}(X)^n\) to \(U^{\sum m_i}\) are valid" (Equation 3) is problematic. While repetition coding (e.g., assigning the same query to multiple workers, with majority voting as decoding) is feasible, the authors provide no evidence that *other* coding schemes—such as those leveraging structured error correction (e.g., Hamming codes)—are implementable. For instance, designing \(u^n\) to exploit error-correcting properties (beyond majority voting) requires assigning *distinct* queries to workers, which contradicts the need for query diversity. Crucially, the paper does not quantify how the constraint of *no repeated queries to a single worker* restricts the set of valid encodings. Without this, the channel capacity bound may be overly optimistic, as it assumes a freedom in encoding design unavailable in practice. The bound’s looseness stems directly from this unaddressed constraint, which is not merely a technicality but a structural difference between the two problems.

To strengthen the argument, the authors should explicitly characterize the feasible encoding space under crowdsourcing constraints. For example:  
- How many *distinct* queries can be assigned to a single worker before response correlation becomes significant?  
- What is the minimal query diversity required to maintain i.i.d. worker responses?  
- How do these constraints reduce the effective channel capacity compared to the unconstrained channel coding model?  

Without such analysis, the bound remains theoretically interesting but lacks practical relevance. A comparison with state-of-the-art crowdsourcing methods (e.g., Dawid-Skene, Bayesian approaches) would clarify whether the derived bound is tight enough to guide algorithm design or merely serves as a loose upper limit.

**Minor comments:**  
- Equation (8): The minimization should explicitly state \(\min_{C, C'}\) subject to \(\mathbb{E}[d(Z, \hat{Z})] \leq D^t\).  
- Theorem 4: The proof requires more detail on how \(\epsilon_S\) relates to \(\hat{\epsilon}\), particularly given \(\ln(1-q) < 0\). Is \(\epsilon_S > \hat{\epsilon}\)? Clarify this dependency.  
- Equation (17): The term \(H_M(E(\epsilon))\) requires justification. Cite prior work defining this entropy measure or explain its relevance to crowdsourcing.  

In summary, while the theoretical framework is sound, the paper’s contribution is weakened by an incomplete treatment of crowdsourcing-specific constraints. The channel coding analogy is compelling but overextended without addressing the critical divergence in encoding freedom. The proposed bound may not reflect practical tradeoffs, as it neglects the hard constraint of query repetition. To elevate the work, the authors should: (1) rigorously model the feasible encoding space under crowdsourcing constraints, (2) derive a tighter bound accounting for these constraints, and (3) empirically compare the bound against existing methods. Without these steps, the bound risks being a theoretical curiosity rather than a useful tool for crowdsourcing design. The paper has potential but requires substantial revision to justify its novelty and practical impact.

This paper derives an information-theoretic bound on the budget–fidelity tradeoff for label crowdsourcing using rate-distortion theory and the channel coding theorem. The bound is carefully derived. If this is indeed the first work to apply rate-distortion theory and the channel coding theorem to crowdsourcing, it could be a reasonably important contribution.

However, the novelty and practical value remain questionable. The derivation appears relatively straightforward, and the resulting bound risks being very loose. The main concern is that, unlike in channel coding—where one has substantial freedom to design codebooks and codewords (e.g., repetition codes, Hamming codes, convolutional codes, RS codes, low-density parity-check (LDPC) codes)—it is unclear whether similar freedom exists for designing the sequence of channel inputs u^n in this setting. In equation (3), almost all mappings from B(X)^n to U^{\sum m_i} (except degenerate ones) would need to correspond to valid, implementable “codebooks”; otherwise, using channel capacity as a bound may be overly loose. For crowdsourcing, something akin to repetition coding is possible, with majority voting as the natural decoder. Beyond that, it is unclear how to design u^n to improve error-correcting capability.

In particular, there is a critical difference between channel coding and crowdsourcing: we generally cannot assign the same query to the same worker multiple times without violating the i.i.d. assumption in equations (1) or (2), because the same worker will likely produce the same answer to the same query. By contrast, in communication, the channel is typically modeled as i.i.d., and the same symbol can be used repeatedly over the same channel. This is just one example, and more broadly, crowdsourcing likely imposes many fundamental constraints on equation (3), unlike channel coding where the encoding map {0,1}^{nR} -> {0,1}^n is only lightly constrained. Therefore, there should be substantial constraints on the set of feasible encoding schemes in equation (3), and because of these constraints the proposed information-theoretic bound may be very loose.

Finally, it would strengthen the paper to compare the performance of state-of-the-art methods with the derived bound.

This work provides an information-theoretical bound on the rate-distortion tradeoff for label crowdsourcing using rate-distortion theory and channel coding theorem. The information-theoretical bound is well-established by this paper. If this is the first attempt to apply rate-distortion theory and channel coding theorem to the crowdsourcing problem, it could be a reasonably significant contribution to the field. However, the novelty and practical value of this paper are still questionable, as the derivation of the information-theoretical bound is relatively straightforward and there is a risk of the bound being too loose. The main problem is that, unlike the channel coding problem, where we have a large amount of freedom in designing a codebook or codewords (we can use repetition codes, Hamming codes, convolutional codes, RS codes, Low-Density Parity Check (LDPC) codes), it is not clear whether we have such freedom in designing the sequence of channel input u^n, in equation (3), almost all the mappings (except for corruptions) from B(X)^n to U^(\sum mi) should be valid or implementable codebooks, otherwise the channel capacity may be a loose bound. Clearly, for crowdsourcing, something like a repetition code is possible where the natural decoding method is majority voting. However, besides this, it is not clear how we can design u^n to improve error correction capacity. In particular, a notable difference between the channel coding problem and the crowdsourcing problem is that we cannot assign the same query to the same worker multiple times because it will ruin the iid assumption in equation (1) or (2), as the same worker will probably generate the same answer to the same query, while in a communication channel we can assume that the channel is iid and the same query can be used multiple times for the same channel. This is just one example, and in general, it is very likely that there are many fundamental constraints in equation (3) in the crowdsourcing problem, unlike channel coding, where there is very little constraint in an encoding mapping {0,1}^{nR} -> {0,1}^n. Therefore, there should be significant constraints on the set of possible encoding schemes in equation (3), and such constraints risk making the proposed information-theoretical bound too loose. Finally, it would be better to compare the performance of state-of-the-art methods with the obtained information-theoretical bound.

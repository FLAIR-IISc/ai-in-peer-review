

This paper introduces Sparse Access Memory (SAM), a novel memory architecture for memory-augmented neural networks (MANNs) that addresses the critical scalability limitations of prior models like Neural Turing Machines (NTMs) and Memory Networks. The authors build upon foundational NIPS work (e.g., Graves et al. 2014 on NTMs, Sukhbaatar et al. 2015 on Memory Networks) by introducing sparse read/write operations constrained to a small constant number of memory slots (K=4–8), coupled with efficient approximate nearest neighbor (ANN) indexing. SAM achieves O(log N) time complexity per step (vs. O(N) for dense approaches) and O(1) space per step, while retaining end-to-end differentiability. The paper demonstrates SAM’s effectiveness on synthetic tasks (Copy, Associative Recall, Priority Sort), Babi question answering, and Omniglot one-shot learning, showing 1,000× speedup and 3,000× memory reduction over dense NTMs.

**Strengths:**  
1. **Quality:** The theoretical analysis of asymptotic optimality (Supplementary A) and rigorous empirical validation (Figures 1–4) convincingly demonstrate SAM’s scalability without sacrificing performance. The sparse write scheme (using least-recently-accessed slots) and efficient backpropagation through time are technically sound.  
2. **Significance:** SAM solves a fundamental bottleneck in MANNs, enabling training on tasks requiring >100k time steps and millions of memory slots—previously infeasible. The SDNC extension (Supplementary D) and superior Babi results (solving all but one task without memory supervision) highlight broad applicability.  
3. **Originality:** While sparse memory access has been explored in RL (e.g., Zaremba & Sutskever 2015), SAM’s end-to-end differentiable implementation with ANN integration is novel and significantly more efficient than prior approaches.  

**Weaknesses:**  
1. The paper does not sufficiently explain *why* sparse reads/write improve learning efficiency on certain tasks (e.g., Priority Sort in Figure 2), which appears counterintuitive. A deeper analysis of the bias-variance tradeoff would strengthen the claims.  
2. ANN implementation details (k-d trees vs. LSH) are underdeveloped; the choice criteria and performance tradeoffs warrant more discussion (e.g., why LSH is better for large word sizes).  

**Pro/Con Acceptance:**  
*Pro:* High-impact solution to a well-motivated scalability problem; strong theory and experiments; clear significance for real-world MANN deployment.  
*Con:* Limited explanation for unexpected learning improvements; minor gaps in ANN methodology.  

This is a high-quality, significant contribution to NIPS, advancing the state-of-the-art in scalable neural memory systems. The minor weaknesses are addressable in revision but do not detract from the paper’s core value. **Recommendation: Accept.**

- Problem: Dense MANNs perform soft reads/writes over all N memory slots, incurring O(N) time and O(NT) BPTT memory costs that prevent large-scale use.  
- Proposal: Sparse Access Memory (SAM) — a memory architecture using sparse operations to address scaling bottlenecks.  
- Sparse reads: Read from a constant K nearest-neighbor subset found via an integrated ANN data structure (e.g., k-d tree/LSH), reducing query cost toward O(log N).  
- Sparse writes: Restrict writes to a constant set — the just-read locations plus an LRU slot — so only O(1) words are modified per step.  
- Efficient BPTT: Track and reverse only sparse modifications to roll back memory during backprop, reducing BPTT space complexity to O(N + T).  
- Empirical efficiency: Reports up to ~1,600x speedups and ~3,000x memory reductions versus dense NTM on large memories.  
- Empirical performance: Matches or improves learning on algorithmic tasks, bAbI QA, and Omniglot one-shot; handles sequences >100,000 steps.  
- Generality: Method successfully applied to the more complex Differentiable Neural Computer (DNC).  
- Technical quality: Well-motivated, technically sound, with convincing asymptotic analyses.  
- Experimental rigor: Thorough, controlled benchmarks and novel large-scale scaling experiments.  
- Clarity: Exceptionally well-written, organized, with effective figures and useful supplementary implementation details.  
- Originality: Novelly combines ANN data structures with end-to-end differentiable sparse KNN attention and an efficient rollback for BPTT.  
- Significance: Substantially removes a key obstacle for scaling MANNs, enabling large-memory applications (e.g., open-domain QA, long-document tasks).  
- Weakness: Evaluation focuses mainly on synthetic/semi-synthetic benchmarks; real-world large-scale NLP validation is lacking.  
- Recommendation: Reviewer strongly recommends acceptance and finds no substantive objections.

- Problem: MANNs (NTM/DNC) scale poorly because soft attention requires O(N) time per step and O(NT) space for BPTT, making large memories and long sequences infeasible.  
- Core idea: Sparse Access Memory (SAM) interacts with a small constant-sized subset K of memory slots instead of the full memory to achieve asymptotic time/space improvements.  
- Sparse reads/writes: Uses ANN indices (k-d trees/LSH) to retrieve K nearest neighbors in O(log N) and performs sparse writes to recently read or LRU slots.  
- Efficient BPTT: Stores only sparse updates and reconstructs previous memory states by reverting sparse changes, reducing per-timestep space from O(N) to O(1).  
- Generality: The approach generalizes to other architectures (e.g., Sparse DNC / SDNC).  
- Empirical gains: Demonstrates massive speedups (>1000x) and memory savings (>3000x) while maintaining or improving learning performance.  
- Experimental breadth: Comprehensive evaluation includes benchmarks, algorithmic tasks, curriculum learning to very long sequences, bAbI QA, and Omniglot one-shot learning.  
- Relation to prior work: Builds on NTM/DNC and Memory Networks, offering a differentiable, deterministic sparse addressing alternative to RL-based discrete addressing.  
- Technical quality: High—claims are well-reasoned and backed by extensive, convincing empirical evidence and appropriate baselines (including a dense DAM).  
- Clarity: Exceptionally well-written with clear motivation, precise architecture description, informative figures, and a lucid explanation of the BPTT roll-back mechanism.  
- Originality: Novel integration of ANN data structures into forward/backward neural computation to enable scalable, differentiable memory access.  
- Significance: High potential impact—removes a key barrier to applying MANNs to large-scale tasks (document-level modeling, dialogue, lifelong learning).  
- Arguments for acceptance: Solves a critical problem with an elegant, original solution, strong empirical results, and likely high influence on future work.  
- Arguments against acceptance: Needs more discussion of ANN practical trade-offs (K selection, index rebuild cost, rapidly changing memory) and lacks formal analysis of top-K approximation error.  
- Overall recommendation: Strong accept — strengths overwhelmingly outweigh minor weaknesses.

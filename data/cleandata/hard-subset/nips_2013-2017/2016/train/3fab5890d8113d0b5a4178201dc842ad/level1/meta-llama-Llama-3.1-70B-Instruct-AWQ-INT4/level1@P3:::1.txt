**Summary of the Main Ideas**

This paper introduces Sparse Access Memory (SAM), a novel neural memory architecture designed to scale memory-augmented neural networks (MANNs) to larger memories while maintaining efficiency in both space and time. The authors propose a sparse read and write scheme, which constrains memory access to a sparse subset of memory words, and an efficient data structure for content-based read operations. The SAM architecture is evaluated on various tasks, including synthetic tasks, one-shot Omniglot character recognition, and question answering on the Babi tasks.

**Related Work**

The paper builds upon previous work on MANNs, such as Neural Turing Machines (NTMs) and Memory Networks. The authors acknowledge the limitations of these models, including their high computational and memory overhead, and propose a novel solution to address these issues. The paper also references other works on scalable memory architectures, such as the use of reinforcement learning to train discrete addressing policies.

**Strengths**

1. **Scalability**: The SAM architecture demonstrates significant improvements in scalability, allowing for the training of MANNs with large memories while maintaining efficiency in both space and time.
2. **Efficient memory access**: The sparse read and write scheme, combined with the efficient data structure for content-based read operations, enables fast and efficient memory access.
3. **Flexibility**: The SAM architecture can be applied to various tasks, including synthetic tasks, one-shot Omniglot character recognition, and question answering on the Babi tasks.
4. **State-of-the-art results**: The paper reports state-of-the-art results on several tasks, including the Babi tasks and one-shot Omniglot character recognition.

**Weaknesses**

1. **Limited analysis of the sparse approximation**: While the paper demonstrates the effectiveness of the sparse approximation, it would be beneficial to provide a more in-depth analysis of the trade-offs between the sparse and dense models.
2. **Lack of comparison to other scalable memory architectures**: The paper primarily focuses on comparing SAM to NTMs and Memory Networks. A more comprehensive comparison to other scalable memory architectures, such as those using reinforcement learning, would be valuable.
3. **Limited discussion of the controller architecture**: The paper uses a simple one-layer LSTM as the controller, but a more detailed discussion of the controller architecture and its impact on the overall performance of the SAM architecture would be beneficial.

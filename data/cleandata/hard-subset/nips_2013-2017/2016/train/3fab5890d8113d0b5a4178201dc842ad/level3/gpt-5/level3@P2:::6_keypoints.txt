- Proposes Sparse Access Memory (SAM): scalable memory-augmented NN using sparse top-K reads/writes and ANN for content addressing.
- Reads use ANN to select top-K slots (k-d trees for low dim, LSH for high dim); writes target read slots plus least-recently-used slot with LRU-like tracking.
- Efficient BPTT implemented by storing sparse modifications and rolling back memory states during backward pass.
- Theoretical complexity: under assumptions, O(log N) time per step, O(1) extra space per step, O(N) initialization.
- Empirically matches or outperforms dense baselines (NTM, DAM, LSTM) on algorithmic tasks and scales to very long sequences and millions of slots.
- Strong results on Omniglot one-shot classification and state-of-the-art bAbI performance via SDNC without supervised memory access.
- Paper builds on NTMs/Memory Networks by replacing dense attention with sparse, differentiable access and enables real-world scaling of MANNs.
- Clear, well-structured presentation with well-motivated architectural and engineering choices.
- Compelling scalability claims supported by large measured speedups (100×–1600×) and memory savings (~3000×).
- Maintains end-to-end differentiability while incorporating ANN; rollback BPTT implementation is a practical contribution.
- Broad empirical evaluation covering synthetic tasks, long-horizon curricula, Omniglot, and bAbI.
- Addresses a core MANN bottleneck and is generally applicable beyond the NTM-like design.
- Fairness of capacity comparisons is a concern: matching physical memory favors SAM; need analyses fixing logical memory size or normalizing by read/write budget.
- Potential gradient bias from top-K sparsification; suggest diagnostics on omitted softmax tail, sensitivity to K, and ANN top-K correctness frequency.
- Need more details on ANN maintenance: constant factors, rebuild frequency, robustness to dimensionality, and ablations comparing linear scan/k-d/LSH.
- Improve clarity/self-containment: add MANN background, clearer write/erase/usage walkthrough, and main-text summary of complexity and rollback assumptions.
- Baselines/scope gap: include discrete-addressing baselines (e.g., RL-NTM) and clarify Omniglot setup and additional metrics.
- Reproducibility: release code and hyperparameters for ANN index, memory management, and rollback mechanism.
- Technical soundness: design and rollback are sound; complexity claims plausible but should state underlying assumptions explicitly.
- Empirical support is strong, with the main caveat being capacity-normalization fairness.
- Limitations: insufficient analysis of gradient bias, ANN errors, and sensitivity studies (K, thresholds, ANN type).
- Clarity is generally good; suggest minor notation cleanup and more in-paper summaries of supplementary implementation details.
- Originality: novel combination of sparse differentiable read/write, ANN inside trainable networks, and efficient BPTT rollback.
- Significance: high — removes scalability barrier for MANNs and enables much larger memories while preserving data efficiency.
- Key questions for authors: sensitivity to K and usage thresholds; fraction of read mass dropped by top-K; ANN top-K failure rate and correlation with errors; results fixing logical memory size or plotting accuracy vs compute; LRU overwrite causing catastrophic forgetting and mitigations.
- Pros for acceptance: important problem, principled/practical solution, strong empirical/theoretical case, clear writing, broad applicability.
- Cons for acceptance: capacity-matching may overstate gains, limited analysis of approximation/gradient bias and ANN robustness, and some implementation details relegated to supplement.
- Overall recommendation: Accept, with minor revisions addressing fairness of comparisons and adding sensitivity/robustness analyses.

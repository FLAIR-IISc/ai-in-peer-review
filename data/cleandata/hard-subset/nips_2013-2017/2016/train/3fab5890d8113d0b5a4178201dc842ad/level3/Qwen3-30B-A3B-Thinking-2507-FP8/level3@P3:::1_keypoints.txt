- Proposes Sparse Access Memory (SAM): sparse reads/writes using ANN and top-K to scale external memory with O(log N) reads and O(1) space per timestep.  
- Empirical claims: large speed/memory gains (≈1,000x faster, ≈3,000x less memory) vs vanilla NTM while retaining comparable task performance.  
- Builds on prior MANN work (NTMs, Memory Networks); core idea (hashing/content-based addressing) is not novel—main novelty is engineering for writable memory.  
- Strong empirical evidence and scalability benchmarks (handles 1M memory words; Figure 1).  
- Maintains performance on BAbi and Omniglot at larger scales; useful engineering insights for efficient backpropagation through time.  
- Novelty claim is overstated; paper needs a more thorough comparison to existing scalable memory approaches.  
- The claim of achieving asymptotic lower bounds is imprecise and insufficiently justified (Supplementary A unclear).  
- Major concern about non-smooth argmax/K-argmax: lacks analysis of effects on training dynamics and comparison to smooth alternatives (e.g., MemNN-WSH).  
- Needs experiments/analysis of K=1 behavior and SAM’s gradient propagation vs fully smooth models.  
- Fails to address training with discrete actions/discreteness effects (unlike RL-NTM literature); potential for suboptimal training dynamics.  
- Replicability issues: insufficient BAbi/Omniglot task descriptions, missing curriculum/hyperparameter details; must release code.  
- Section 3.6 appears redundant and should be removed.  
- The description of Omniglot as “non-synthetic”/“real-world” is misleading; it is artificially generated handwritten data.  
- Significance: practically valuable engineering contribution for scaling MANNs, but theoretical contribution is modest.  
- Recommendation: Accept with major revisions—revise related work, release code and detailed task specs, analyze failure cases/limitations, address non-smoothness and discreteness, remove Section 3.6, and correct Omniglot claims.

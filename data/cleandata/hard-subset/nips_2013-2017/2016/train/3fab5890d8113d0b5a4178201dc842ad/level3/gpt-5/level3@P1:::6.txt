This paper introduces Sparse Access Memory (SAM), an end-to-end differentiable memory-augmented neural network that scales read and write operations sublinearly in memory size while keeping constant per-step memory usage during backpropagation. The core idea is to sparsify both reads (top-K content-based attention using an approximate nearest neighbor index) and writes (to previously read locations and the least-recently-used slot), enabling O(log N) per-step time for access and O(1) per-step space for BPTT by rolling back sparse deltas rather than storing full memory snapshots. The authors argue these asymptotics are optimal for content-based addressing and demonstrate large empirical speed and memory savings (up to ~1000× faster and ~3000× less physical memory vs dense NTMs). On synthetic algorithmic tasks (copy, associative recall, priority sort), SAM matches or exceeds dense baselines (NTM, a dense variant “DAM”), scales via curriculum to millions of slots and thousands of steps, and generalizes to much longer sequences. On bAbI, a sparse DNC variant (SDNC) adapted from SAM achieves near state-of-the-art without supervised memory traces. On Omniglot one-shot classification, SAM outperforms dense MANNs when allowed much larger external memories. The work connects to NTMs and Memory Networks by retaining content-based differentiable access, but differs by implementing sparse access with in-network ANN structures and an efficient BPTT mechanism; it contrasts with discrete RL-based addressing (e.g., Zaremba & Sutskever), avoiding high-variance gradients.

Strengths
- Clear, focused contribution: a principled sparse read/write scheme with a simple, effective LRU+read-based write policy; well-motivated by the O(N) bottlenecks of NTMs/DNCs.
- Strong systems engineering: efficient sparse data structures for forward and backward passes; practical ANN integration (kd-trees, LSH); in-place delta logging and rollback to eliminate O(NT) memory.
- Theoretical and empirical alignment: asymptotic analysis matches large measured speed/memory gains; results indicate sparse gradients do not harm learning and may help early learning on some tasks.
- Breadth of experiments: synthetic tasks, long-sequence curriculum scaling, adaptation to DNC, bAbI QA, and Omniglot episodes; demonstration of generalization to sequences far beyond training.
- Relevance and potential impact: addresses a central scalability barrier for MANNs, making large-memory differentiable models more feasible for long-context problems.

Weaknesses and suggestions
- Fairness of comparisons on Omniglot (and elsewhere): several plots match physical memory budgets, letting SAM use orders-of-magnitude larger logical memory. While this highlights systems benefits, it conflates algorithmic efficacy with hardware utilization. Additional comparisons at equal logical memory capacity (same number of slots) would better isolate the effect of sparsity on learning quality.
- Limited ablations: sensitivity to K, type of ANN (kd-tree vs LSH), rebuild frequency, usage threshold, and write policy (read locations vs content-based writes) is not systematically explored. Ablations would clarify trade-offs and robustness.
- Non-differentiable components: the ANN index is treated as fixed; the top-K selection and LRU selection introduce nondifferentiabilities/gradient truncation. While standard in practice, a discussion of potential bias introduced by hard sparsification and its effect on convergence would strengthen the paper.
- Clarity: the paper is generally well written, but could be more self-contained with a tighter background on MANNs, and a clearer step-by-step exposition of the write operator (Eq. 5), usage statistics, and the exact data structures used for rollback. A compact figure summarizing the read/write/rollback pipeline would help reproducibility.
- Scope of “real world” evaluation: Omniglot is a useful benchmark for episodic memory, but results on large language modeling or machine translation—where external memory is often touted as beneficial—would greatly strengthen significance claims.

Quality
The technical approach is sound and carefully engineered. Asymptotic claims are reasonable, and the implementation details (sparse CSR, in-place deltas, ANN maintenance) are consistent with the complexity analysis. Empirical evidence is extensive for synthetic tasks and competitive for bAbI/Omniglot. Some methodological choices (matching physical memory; mixing SDNC results) complicate attribution of gains purely to SAM, but overall the claims are well supported.

Clarity
Well organized and readable. Adding a more compact background on MANNs, a clearer description of the write/erase mechanics, usage tracking, and explicit hyperparameter tables for ANN and K would improve clarity and reproducibility.

Originality
The combination of sparse differentiable access, in-network ANN for content lookup, and an O(1)-space BPTT rollback is novel relative to NTMs, Memory Networks, and discrete RL-based addressing. The SDNC adaptation underscores generality.

Significance
High. Scaling differentiable memory by orders of magnitude is an important step for long-context reasoning. The demonstrated speed/memory savings and the ability to train over very long sequences make follow-on applications more plausible.

Questions for the authors
- How sensitive are results to K, the ANN recall/precision trade-off, and the LRU threshold? What are failure modes when ANN returns suboptimal neighbors?
- Can you report results at fixed logical memory sizes on Omniglot and bAbI to separate algorithmic from systems gains?
- What is the compute overhead of periodic ANN rebuilds, and how does it scale?
- Have you tried content-based sparse writes, perhaps gated with LRU, and if so how did they compare?

Arguments for acceptance
- Addresses a central scalability bottleneck with a principled, implementable solution.
- Strong empirical and asymptotic evidence; orders-of-magnitude efficiency gains.
- General technique applicable to multiple MANNs (NTM, DNC), with competitive task performance.

Arguments against acceptance
- Some comparisons favor SAM by allowing much larger logical memory; ablations are limited.
- “Real-world” evaluations are still modest; broader tasks would strengthen impact claims.

Overall, this is a solid, well-executed paper that advances scalable differentiable memory. I recommend acceptance, with suggested clarifications and fairer controlled comparisons.

Summary and relation to prior work
The paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that achieves scalable read/write access to a large external memory by enforcing sparsity in both reads and writes and by using approximate nearest neighbor (ANN) indexing for content-based addressing. Reads select the top-K memory slots via an ANN query (k-d trees for small dimensions, LSH for larger), and writes target a sparse set comprising previously read locations and the least recently accessed slot, with an LRU-like usage tracker. To support efficient backpropagation, the authors store only sparse modifications and roll back memory states during the backward pass. Under reasonable assumptions, SAM achieves O(log N) time per step and O(1) additional space per time step, with O(N) initialization. The authors also sketch how to adapt the approach to the Differentiable Neural Computer (SDNC).

Empirically, SAM matches or improves on dense baselines (NTM, a dense variant DAM, and LSTM) on NTM-style algorithmic tasks, scales with an exponential curriculum to very long sequences and large memories (millions of slots), achieves strong results on Omniglot one-shot classification, and (via SDNC) obtains state-of-the-art results on bAbI without supervised memory access. This work builds directly on NTMs and Memory Networks by replacing dense attention with sparse, differentiable access and contributes a practical path to scaling MANNs to real-world settings.

Strengths
- Clear and well-structured presentation; the architectural components and engineering choices are well-motivated.
- Strong scalability story with compelling complexity claims, supported by large measured speedups (100×–1600×) and drastic memory savings (up to ~3000×) relative to dense NTMs.
- Maintains end-to-end differentiability while using ANN structures inside the network; clever BPTT “rollback” implementation is practically important.
- Broad and relevant empirical evaluation: synthetic algorithmic tasks, long-horizon curricula (up to 10k–200k steps for generalization), Omniglot one-shot, and bAbI via SDNC.
- Addresses a central bottleneck in MANNs; approach is general and applicable beyond the specific NTM-like design.

Weaknesses and suggestions
- Fairness of capacity comparisons (Figure 4 and related): Several comparisons match physical memory usage across models, which favors SAM because it can allocate many more slots for the same footprint. A complementary analysis fixing the logical memory size (same number of slots across models), or normalizing by read/write budget, would isolate the effect of sparsity vs capacity. Reporting accuracy versus wall-clock compute and versus number of memory slots would further clarify trade-offs.
- Gradient bias from sparsification: Top-K selection and sparse writes introduce biased gradients relative to dense soft attention. While empirical results are good, some analysis/diagnostics (e.g., tail mass of the softmax read omitted by K, sensitivity to K, and how often the approximate ANN returns the exact top-K) would strengthen the claims.
- ANN maintenance and robustness: More details on constant factors, rebuild frequency, and how performance degrades with dimension would help practitioners. An ablation comparing linear scan, k-d trees, LSH, and their impact on accuracy/throughput would be useful.
- Clarity and self-containment: Adding a brief, self-contained background on MANNs and a clearer walk-through of the write operator (including the erase step and usage measure) in the main text would improve accessibility. Some essentials (complexity assumptions, BPTT rollback) could be summarized more fully in the main paper.
- Baselines and scope: The paper includes NTM/LSTM and a dense variant; inclusion of a discrete-addressing baseline (e.g., RL-NTM) with comparable compute budget would situate the trade-off against non-differentiable policies. For Omniglot, clarifying differences from prior setups and reporting additional metrics (per-episode learning curves, calibration) would aid comparison.
- Reproducibility: Consider releasing code and hyperparameters for the ANN index, memory management, and rollback mechanism.

Quality
- Technical soundness: The sparse read/write design and BPTT rollback are sound and thoughtfully engineered. Complexity claims are plausible; if the lower-bound argument depends on specific content-based access assumptions, making those explicit in the main text would help.
- Empirical support: Strong across synthetic and real tasks, with convincing speed/memory benchmarks. The fairness issue around capacity normalization is the main caveat.
- Limitations: Lack of analysis on gradient bias and ANN errors; limited sensitivity studies (K, thresholds, ANN type).

Clarity
- Generally very clear and easy to follow. Suggested improvements: brief MANN background, expanded explanation of write/erase/usage, and a concise in-paper summary of Supplementary A/D implementation details. Minor notation cleanup (consistent use of tildes on sparse weights).

Originality
- The combination of (i) sparse, differentiable read/write, (ii) ANN within a trainable network, and (iii) efficient rollback for BPTT is novel and impactful relative to NTM, Memory Networks, and RL-based discrete addressing.

Significance
- High. The work removes a central scalability barrier for MANNs and demonstrates that sparse access can preserve data efficiency while unlocking orders-of-magnitude larger memories. Likely to be adopted and extended.

Questions for the authors
- How sensitive is performance to K (e.g., K=2,4,8,16) and to the threshold used in the usage statistic?
- What fraction of read mass is typically dropped by top-K? Do errors rise when the distribution is less peaked?
- How often does the ANN fail to retrieve the true top-K, and does that correlate with errors?
- Could you report results fixing the logical memory size across models and/or plotting accuracy vs compute budget?
- Does the LRU-based overwrite ever cause catastrophic forgetting on long curricula? Any mitigation (e.g., multi-slot allocation, age plus strength)?

Pros for acceptance
- Addresses an important, timely problem with a principled and practical solution.
- Strong empirical and theoretical story; large, demonstrable speed/memory gains.
- Clear writing; broad applicability beyond a single model.

Cons for acceptance
- Capacity-matching comparisons may overstate gains; needs complementary fixed-capacity analysis.
- Limited analysis of approximation/gradient bias and ANN robustness.
- Some critical implementation details are deferred to the supplement.

Overall recommendation
Accept. This is a well-executed, original, and significant contribution that advances the scalability of memory-augmented neural networks while preserving end-to-end differentiability. With minor revisions addressing fairness of comparisons and adding sensitivity/robustness analyses, the paper will be even stronger.

- Problem: Standard MANNs scale poorly because they perform dense soft attention over all memory at each timestep, causing linear compute and backprop storage costs that prevent very large memories.

- Proposal: Sparse Access Memory (SAM) restricts operations to a small constant K memory slots per step, using ANN structures for top-K read retrieval and writes to recent reads plus an LRU slot.

- Complexity claim: SAM reduces read time to O(log N) and per-timestep space to O(1), enabling scaling to millions of slots and sequences of ~200k steps.

- Strength — significance: The engineering integration of ANN data structures into a writable, end-to-end differentiable memory is a practically important advance that enables previously intractable applications.

- Strength — empirical results: Experiments show several orders of magnitude speed and memory improvements and strong performance on large-scale algorithmic tasks, bAbI, and Omniglot (including SOTA unsupervised SDNC on bAbI).

- Strength — generality: The sparse access scheme generalizes beyond the NTM-like model (demonstrated with SDNC), suggesting broader applicability across MANN architectures.

- Weakness — limited novelty: Using ANNs for efficient retrieval is not new; the novelty is primarily the successful application and integration into a writable NTM/DNC-style memory.

- Weakness — reproducibility: Key experimental details (especially curriculum learning parameters: initial difficulty, thresholds, sampling) are missing, hindering replication of scaling results.

- Weakness — lack of analysis: The paper lacks deeper discussion of trade-offs and failure modes, notably implications of the hard, non-differentiable top-K read selection on training dynamics and gradients.

- Revision request — reproducibility: Provide full curriculum learning details and release code and task-generation scripts to allow reproduction of key results.

- Revision request — non-differentiable reads: Add analysis/discussion of using arg-top-K during training, covering gradient effects, K=1 stability, and comparisons to RL-based discrete addressing or hashing approaches.

- Revision request — failure modes: Analyze limitations and conditions where ANN lookups might miss relevant memories and where sparse approximations could underperform (e.g., many near-duplicate memories requiring fine distinctions).

- Revision request — minor presentation fixes: Avoid calling Omniglot "non-synthetic" or "real-world" in a misleading way, and frame the contribution explicitly as the first large-scale integration of fast ANN lookups into a writable memory system.

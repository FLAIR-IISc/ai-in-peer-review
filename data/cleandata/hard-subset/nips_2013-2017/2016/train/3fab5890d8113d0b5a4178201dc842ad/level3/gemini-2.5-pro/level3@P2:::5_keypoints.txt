- MANNs scale poorly because read/write touch all N memory slots each step, causing O(N) time/space.
- Proposal: Sparse Access Memory (SAM) to make MANNs feasible for very large memories.
- Sparse reads: read only K nearest neighbors using an integrated ANN structure (e.g., k-d tree/LSH) for O(log N) search.
- Sparse writes: restrict updates to a small constant number of locations (recent reads or LRU overwrite).
- Efficient BPTT: avoid caching full memory by reverting only sparse modifications, giving constant per-step space overhead.
- Empirical gains: up to 1000x speedups and 3000x memory reductions while retaining performance.
- SAM matches or outperforms dense models on standard tasks (NTM tasks, bAbI, Omniglot).
- Curriculum and scale results: trains with BPTT over thousands of steps and generalizes from 10k to 200k steps in associative recall.
- Novelty/technical quality: integrating ANN search into end-to-end differentiable models and the sparse BPTT algorithm are novel and sound.
- Weakness: limited analysis of why sparsity improves learning/generalization.
- Suggested explanations to explore: sparsity as regularization, reduced gradient interference/noise, or encouragement of algorithmic, abstract solutions.
- Overall: significant, well-executed contribution with high impact; clear accept recommendation.

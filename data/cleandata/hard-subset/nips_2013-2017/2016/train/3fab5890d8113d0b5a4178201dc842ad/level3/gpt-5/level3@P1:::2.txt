Summary: The paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that replaces dense content-based reads and writes with sparse access, and introduces an efficient backpropagation scheme that logs sparse state changes and rolls the memory back during BPTT. Reads are restricted to the top-K nearest memory slots (found via an approximate nearest neighbor index), and writes target either previously read locations or the least-recently-used slot determined by a usage statistic. The authors argue SAM achieves O(log N) time per step (via ANN) and O(1) memory per time step (via sparse updates), while retaining end-to-end differentiability. Experiments show large empirical speed and memory reductions relative to dense NTMs/DNCs, and comparable or better data efficiency on synthetic NTM tasks, bAbI, and Omniglot one-shot classification. They also demonstrate a sparse variant of the Differentiable Neural Computer (SDNC).

Relation to previous work: SAM is closely related to NTMs and DNCs (content-based addressing with read/write heads), Memory Networks (content-only reads without writes), and RL-NTMs (discrete sparse addressing via reinforcement learning). The novelty here lies in integrating well-known components—top-K sparse attention, approximate nearest neighbors (k-d trees/LSH), LRU-style allocation, and sparse change logs for BPTT—into a single end-to-end differentiable MANN that scales in practice. The LRU-like allocation echoes the usage-based allocation in DNC; sparse reads/writes resemble prior top-K attention mechanisms; and the ANN index is a standard systems tool repurposed for neural memory.

Strengths:
- Clear motivation and a practical solution to the central bottleneck of MANNs: dense O(N) memory access and O(NT) BPTT storage.
- Strong empirical evidence of large wall-clock and memory savings, with scaling to millions of memory slots and thousands of time steps without truncated BPTT.
- Competitive or better accuracy on standard synthetic tasks, bAbI (notably with SDNC), and Omniglot, suggesting sparse approximations do not unduly harm learning.
- Engineering contributions (sparse rollback, ANN integration) are carefully described and of potential practical value to the community.

Weaknesses:
- Technical novelty appears incremental: the main components (top-K sparsification, LRU allocation, ANN search, in-place rollback via change logs) are standard ideas combined in a straightforward way.
- Theoretical claims (e.g., optimality and O(log N) time) hinge on assumptions not fully discussed (e.g., dimensionality dependence of k-d trees; rebuild costs; constants hidden in O(·)).
- Limited ablation and sensitivity analysis: no systematic study of K, ANN type/accuracy vs runtime, effect of sparse write policy vs content-based writes, or the bias introduced by ignoring gradients to non-top-K entries.
- Limited evidence on challenging real-world tasks where large memory would be most impactful (e.g., long-context language modeling, large-scale QA without supervision).

Quality: The implementation seems sound, and the experiments are broad across tasks and scales. The speed/memory benchmarks are persuasive and include both exact and approximate indices. However, the analysis of approximation effects is thin: how often does ANN miss true neighbors, what is the accuracy impact, and how does K trade off accuracy vs compute? The write policy (previously read + least recently used) is plausible but under-justified compared to learned content-based writes; a head-to-head comparison would help. The claimed O(1) memory per step for BPTT is compelling, but it would be useful to account for the memory overhead of the ANN structure itself and to report total memory including indices.

Clarity: The paper is generally clear and well-organized. The description of the sparse read/write scheme and the rollback trick is easy to follow. Suggestions: include a concise algorithmic description (forward/backward) of the sparse update and rollback; clarify when writes are additive vs overwrite; specify K and the ANN parameters used across tasks; and more explicitly describe the time/space constants, rebuild frequency, and how the ANN is synchronized with writes.

Originality: While the integration is neat and useful, each component is known. The contribution is best characterized as a well-engineered systemization of sparse MANN access rather than a conceptual advance in memory architectures or learning algorithms.

Significance: If robust, SAM could make MANNs practical at scales previously unattainable, which is valuable. However, the current evidence is mostly synthetic or small-scale real data (Omniglot). Demonstrations on more demanding real-world sequence problems would strengthen the case for broad impact.

Questions and suggestions for the authors:
- Provide ablations: vary K; compare exact top-K vs ANN with different recall levels; compare k-d trees vs LSH across word dimension M; quantify accuracy vs speed trade-offs.
- Compare write policies: previously read + LRU vs content-based sparse writes; analyze memory fragmentation/collisions and stability over long training.
- Analyze the effect of sparse gradients (only K rows receive gradient): any convergence/pathology observations? Could a soft-to-hard annealing improve learning?
- Report end-to-end wall-clock speedups including ANN maintenance and rebuilds; include memory usage accounting for the ANN index.
- Evaluate on a larger real-world benchmark (e.g., long-context language modeling or QA without supervision) to demonstrate practical gains beyond synthetic tasks.
- Release code to facilitate adoption and reproducibility.

Pros for acceptance:
- Addresses an important scalability barrier for MANNs with compelling empirical speed/memory gains.
- Maintains competitive accuracy; demonstrates impressive long-sequence BPTT without truncation.
- Clear, implementable ideas that others can build on; extension to DNC is a plus.

Cons against acceptance:
- Limited technical novelty; largely an engineering combination of known ideas.
- Insufficient analysis of approximation-induced biases and sensitivity to hyperparameters.
- Lack of evidence on harder real-world tasks where significance would be clearer.

Recommendation: Weak reject. The work is practical and well-executed, but the core technical contribution feels incremental. Strengthening the empirical evidence (ablations and real-world tasks) or providing deeper theoretical analysis of the sparse approximations would substantially improve the case for acceptance.

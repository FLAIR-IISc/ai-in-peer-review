This paper presents Sparse Access Memory (SAM), a compelling method for scaling Memory-Augmented Neural Networks (MANNs) like the NTM to handle vastly larger memories. The core contribution is to replace the standard dense attention mechanism, which is linear in memory size, with a sparse alternative. For reads, the model employs fast approximate nearest-neighbor (ANN) data structures to efficiently identify the top-K most relevant memory slots for a given query. Writes are similarly constrained to be sparse, targeting recently accessed or least-used memory locations. This approach yields dramatic improvements in computational and memory efficiency, enabling training on tasks with millions of memory words and sequences hundreds of thousands of steps long.

The work is a direct extension of the NTM framework, and while highly effective, its conceptual novelty is somewhat limited. The use of hashing and ANN for scalable lookups has been previously explored in related architectures, such as Memory Networks, a point that could be more thoroughly discussed. However, the key strength and significance of this paper lie in its practical contribution. Applying these efficient retrieval techniques to a dynamic, *writeable* memory system is a non-trivial engineering challenge, and the empirical results—orders-of-magnitude improvements in speed and memory usage—are a valuable contribution to the community. The experiments clearly demonstrate that this sparse approximation does not degrade learning performance and can even be beneficial.

Despite these strengths, the paper would benefit from several revisions.
1.  **Replicability:** A primary concern is the lack of detail in the task descriptions, which hinders reproducibility. The authors must provide a thorough account of the task constructions and are strongly urged to release their code.
2.  **Analysis of Limitations:** The work would be more complete with an analysis of the method’s failure modes. Under what task conditions or hyperparameter settings (e.g., small K) does the sparse approximation break down compared to a dense model?
3.  **Non-smoothness:** The argmax/top-K selection introduces discreteness. The authors should discuss the implications of this non-smoothness and why standard backpropagation succeeds without explicitly accounting for the discrete actions, unlike some prior work that uses reinforcement learning for similar problems.
4.  **Clarity:** The description of the Omniglot dataset as "non-synthetic" or "real-world" is imprecise; a more accurate characterization is recommended.

In summary, while the conceptual novelty is moderate, the practical achievement in scaling writeable memory models is significant. I recommend acceptance, contingent on the authors addressing the concerns regarding reproducibility and providing a more critical analysis of their method's limitations.

- Proposal: Sparse Access Memory (SAM) — a MANN using sparse reads/writes and ANN (arg-top-K, k-d trees/LSH) to achieve asymptotic optimal space/time and scale to 100k+ memory slots.  
- Key mechanisms: constrain accesses to K=4–8 words, sparse writes using least-recently-accessed slots, and efficient sparse backpropagation tracking sparse modifications.  
- Empirical claims: up to 1,000× faster training and 3,000× less memory than NTMs, with comparable or improved performance on Copy, Associative Recall, Priority Sort, Babi, and Omniglot one-shot learning.  

Strengths
- Quality: strong, well-validated empirical results and ablation studies showing near-identical data efficiency to dense baselines while scaling to very large memories.  
- Clarity: clear architecture and sparse backpropagation descriptions with sufficient supplementary technical detail.  
- Originality: novel end-to-end differentiable write mechanism for content-addressable memory and demonstrated generality (SDNC adaptation).  
- Significance: addresses a critical scalability bottleneck in MANNs, enabling large-memory real-world applications and substantial resource reductions.

Weaknesses
- Originality concern: contribution is incremental since prior work (e.g., MemNN-WSH) already used hashing for scalable reads; paper must better contrast its writable-memory novelty.  
- Replicability: insufficient detail on curriculum/task construction (sequence length distributions, hyperparameters); code release needed for reproducibility.  
- Non-smoothness: arg-top-K is non-differentiable and under-justified; lacks failure-case analysis (e.g., K=1) and comparisons to smooth alternatives.  
- Terminology: calling Omniglot “real-world” is misleading; should be termed a synthetic / standard benchmark.

Pro / Con for acceptance
- Pro: compelling scalability and engineering contribution, resolves practical bottleneck, strong empirical results (including Babi).  
- Con: overstates novelty relative to related work, missing failure analysis and reproducibility details, and omits discussion of discrete-approximation effects.

Recommendation
- Accept with minor revisions contingent on clarifying novelty, adding task/curriculum details, analyzing discrete approximations, correcting terminology, and removing redundant material (Section 3.6).

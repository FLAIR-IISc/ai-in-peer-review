

# Review of "Scaling Memory-Augmented Neural Networks with Sparse Reads and Writes"

This paper introduces Sparse Access Memory (SAM), a novel memory architecture for memory-augmented neural networks (MANNs) that enables efficient scaling of external memory through sparse read/write operations. The authors propose using approximate nearest neighbor (ANN) search and arg-top-K operations to limit memory access to a small subset of memory words (K=4-8), achieving O(log N) time complexity for reads and O(1) space per time step. SAM demonstrates significant speedups (1,000x faster, 3,000x less memory) compared to vanilla NTMs while maintaining comparable performance on synthetic tasks and Omniglot classification.

The paper builds on prior MANN work, particularly Neural Turing Machines (NTMs) and Memory Networks. While the authors cite related work on hashing for scalable memory (e.g., Memory Networks section 3.3), they understate the novelty of their approach. The use of hashing for content-based addressing in large memory systems is not new—prior work by Weston et al. (2015) and others has explored similar techniques for Memory Networks. SAM's primary contribution is the practical engineering of these techniques for writeable memory systems, which is less explored in the literature.

**Strengths:** The paper provides compelling empirical evidence of SAM's computational advantages. The speed and memory benchmarks (Figure 1) clearly demonstrate the scalability benefits, with SAM handling memories of 1 million words while NTM becomes infeasible. The results on Babi tasks and Omniglot (Figures 3-4) show SAM maintains performance while scaling to much larger memory sizes. The engineering insights about efficient backpropagation through time (Section 3.4) are particularly valuable for practitioners.

**Weaknesses:** The novelty claim is overstated—the core idea of using hashing for scalable memory access is well-established. The paper would benefit from a more thorough comparison with prior work on scalable memory architectures. The claim that SAM "achieves asymptotic lower bounds" requires more precise justification (Supplementary A is unclear on this point).

A major concern is the non-smoothness of the argmax/K-argmax operations. The paper acknowledges this but lacks analysis of the effects on training dynamics. How does SAM's discrete approximation compare to smooth alternatives like the recently proposed Memory Networks with Softmax (MemNN-WSH)? The authors should analyze how K=1 affects performance and compare SAM's gradient behavior with fully smooth models.

The paper also fails to address a critical issue: training with discrete actions without accounting for discreteness (unlike prior work such as Reinforcement Learning NTMs). This could lead to suboptimal training dynamics, and the authors should discuss the implications and limitations of this approach.

The review also raises significant replicability concerns. Task descriptions for Babi and Omniglot are insufficiently detailed, and the paper does not provide clear specifications for curriculum construction or hyperparameter settings. The authors must release experiment code to enable reproduction.

The paper should remove Section 3.6, which appears redundant. Additionally, the claim that Omniglot is "non-synthetic" or "real-world" is misleading—the dataset is synthetic in the sense that it's artificially generated from handwritten characters, not collected from real-world text.

**Significance:** SAM's practical contribution is valuable for practitioners seeking to scale MANNs to real-world applications. The engineering insights about sparse memory management are particularly useful for implementing large-scale MANNs. However, the theoretical contribution is modest, as the approach builds on established techniques rather than introducing new concepts.

**Recommendation:** Accept with major revisions. The paper has strong empirical results that demonstrate a valuable engineering contribution, but requires significant clarifications to address the concerns about novelty, replicability, and theoretical analysis. Specifically:

1. Thoroughly revise the related work to properly contextualize SAM within existing scalable memory approaches
2. Provide detailed task construction specifications and release experiment code
3. Analyze failure cases, limitations, and conditions under which SAM may not work
4. Discuss the non-smoothness of argmax/K-argmax operations and compare with smooth alternatives
5. Address the discrete action training issue and its implications
6. Remove Section 3.6 and revise the Omniglot description

The paper's practical results are compelling, but the authors must strengthen their theoretical claims and provide sufficient detail for reproducibility to fully realize the contribution. With these revisions, SAM could become a valuable reference for practitioners working with large-scale memory-augmented networks.



This paper proposes Sparse Access Memory (SAM), a memory-augmented neural network (MANN) architecture that achieves asymptotic optimal space and time complexity through sparse reads/writes and approximate nearest neighbors (ANN). SAM constrains memory access to a small constant subset (K=4–8) of memory words per timestep using arg-top-K and efficient ANN data structures (k-d trees/LSH), enabling training with massive memories (100,000+ slots) while maintaining performance on algorithmic tasks. The authors demonstrate 1,000× faster training and 3,000× reduced memory usage compared to standard Neural Turing Machines (NTMs), with comparable or improved performance on Copy, Associative Recall, Priority Sort, Babi tasks, and Omniglot one-shot learning. SAM’s sparse write mechanism (using least-recently-accessed slots) and efficient backpropagation through time (tracking sparse modifications) are key innovations.

**Strengths**  
*Quality*: The empirical results are compelling and well-validated. SAM achieves near-identical data efficiency to dense baselines (NTM, DAM) across diverse tasks while scaling to 100,000+ memory slots. The engineering insights (ANN integration, memory management) are rigorously evaluated with ablation studies (Figures 1–4).  
*Clarity*: The architecture description (Sections 3.1–3.5) is precise, and the sparse backpropagation technique (Section 3.4) is clearly explained. Supplementary materials provide sufficient technical depth.  
*Originality*: While the *idea* of sparse memory access has precedents (e.g., MemNN-WSH for read-only tasks), SAM’s *end-to-end differentiable write mechanism* for *content-based memory* is novel and fills a critical gap. The adaptation to Differentiable Neural Computers (SDNC) further demonstrates generality.  
*Significance*: SAM directly addresses a major scalability bottleneck in MANNs, enabling real-world applications requiring large-scale memory (e.g., long-document processing). The 3,000× memory reduction makes previously infeasible tasks tractable.

**Weaknesses**  
*Originality*: The novelty is incremental; prior work (e.g., MemNN-WSH [21]) already uses hashing for scalable *read* operations. The paper should more explicitly contrast SAM’s *writeable memory* contribution with existing approaches.  
*Replicability*: Task construction details for curriculum learning (Section 4.3) are insufficient (e.g., exact sequence length distributions, hyperparameters for "exponentially increasing difficulty"). Code release is essential for reproducibility.  
*Non-smoothness*: The arg-top-K approximation (non-differentiable) is not adequately justified. The paper should analyze failure cases (e.g., K=1) and compare to smooth alternatives (e.g., NTM with entropy regularization).  
*Terminology*: Describing Omniglot as "real-world" (Section 4.5) is misleading; it is a synthetic benchmark. This should be corrected to "synthetic" or "standard benchmark."

**Pro/Con for Acceptance**  
*Pro*: Strong empirical validation of scalability, clear engineering contribution, resolves a critical practical bottleneck, and demonstrates state-of-the-art results on Babi tasks.  
*Con*: Overstates novelty without sufficient differentiation from related work; lacks failure analysis and replicability details; omits discussion of discrete approximation effects.

**Recommendation**  
Accept with minor revisions. The paper makes a valuable practical contribution to scaling MANNs, but requires:  
1. Clarification of novelty (explicitly contrast SAM’s write mechanism with MemNN-WSH and prior hashing approaches).  
2. Detailed task construction specifications (curriculum parameters, hyperparameters) in the main text.  
3. Analysis of discrete approximation effects (e.g., K=1 performance, comparison to smooth models).  
4. Correction of "real-world" terminology for Omniglot.  
5. Removal of Section 3.6 (redundant with Section 3.5).  

The work advances the field by making large-memory MANNs feasible, and the empirical results are sufficiently strong to warrant publication after addressing these points. The significance for real-world applications is clear, and the technical execution is sound.

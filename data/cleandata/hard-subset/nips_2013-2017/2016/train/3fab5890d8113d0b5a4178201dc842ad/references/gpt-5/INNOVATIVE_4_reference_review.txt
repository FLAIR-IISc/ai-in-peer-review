Summary
This paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that scales to very large external memories by enforcing sparsity on both reads and writes and by integrating efficient data structures for content-based retrieval. Key ideas: (1) top-K sparse reads via approximate nearest neighbor (ANN) indices (randomized k-d trees or LSH), yielding O(log N) query time; (2) sparse writes targeted to previously read locations plus the least-recently-used (LRU) slot, with constant-time updates and memory management; (3) an efficient backpropagation-through-time scheme that stores and reverts only sparse diffs rather than full memories, achieving O(1) space per step (independent of memory size); (4) generality demonstrated by a sparse variant of the Differentiable Neural Computer (SDNC). Experiments show 100–1600× speedups over dense NTMs, constant per-step memory, comparable or improved data efficiency on classic NTM tasks, strong scaling via curriculum to million-word memories and >4000 time steps without TBPTT, competitive results on bAbI (best to date without memory supervision), and superior performance on Omniglot one-shot episodes when scaling sequence length.

Relation to prior work
Builds directly on NTMs and Memory Networks by decoupling parameter count from memory size and adding content-based addressing. Differs from RL-NTM approaches that use discrete attention trained with REINFORCE; here, reads/writes remain differentiable w.r.t. selected slots but selection is sparse and driven by an ANN index. The SDNC extension connects to DNC. The use of ANN inside a differentiable model, LRU-based allocation, and a reversible sparse BPTT mechanism are the main contributions.

Strengths
- Novelty and boldness: A principled and practical path to truly large-scale external memory with asymptotically optimal time/space complexity under content-based access. Integrating ANN search into end-to-end training is a strong and underexplored idea.
- Measured engineering: Clear design for sparse reads/writes, pragmatic LRU allocation, and a clever rollback scheme for BPTT that eliminates the crippling O(NT) memory cost.
- Empirical significance: Orders-of-magnitude speed and memory savings; demonstrates training with million-slot memories and multi-thousand-step sequences; generalizes far beyond training lengths on associative recall; strong bAbI performance without retrieval supervision; real-data validation on Omniglot with advantages when scaling sequence length.
- Generality: The sparse recipe applies beyond the specific controller; SDNC results suggest portability of the approach.

Weaknesses and concerns
- Differentiability of selection: The top-K selection via ANN is non-differentiable; gradients do not flow to non-selected slots and the selection boundary changes discontinuously. While this is common in hard attention, it warrants a discussion of stability, variance, and potential failure modes. Empirically it works, but a brief ablation (e.g., exact top-K vs ANN, selection temperature/soft-k) would strengthen the case.
- Theoretical assumptions: The “asymptotically optimal” claims depend on assumptions (e.g., content-addressing, ANN performance, dimension regimes). k-d trees degrade in high dimensions; LSH has constant factors and rebuild overhead. More explicit bounds (in terms of word dimension M) and a report on rebuild costs would clarify practical complexity.
- Experimental coverage: Most tasks are synthetic or small-scale (bAbI). Omniglot setup is simplified; broader real-world evaluations (e.g., large document QA, long-horizon program induction, or memory-heavy RL) would better substantiate impact. That said, I recognize the training-time constraints historically limiting such studies.
- Fairness of comparisons: Some comparisons allocate vastly larger external memory to SAM than dense baselines (by design, to match wall-clock memory). This is fair for the scaling claim but complicates assessing pure algorithmic data efficiency. Including a smaller-memory SAM matched to NTM memory size would help isolate the impact of sparsity vs capacity.
- Clarity/details: Many critical implementation details are deferred to the supplement (sparse diff data structure, index maintenance, choice of K, threshold ε for usage, effect of gates). A bit more in the main text would aid reproducibility.

Quality
Technically sound overall, with solid engineering and careful complexity and systems considerations. Claims are supported by convincing empirical results. Some theoretical statements are informal; tightening assumptions and including a few ablations would improve rigor.

Clarity
Generally well-written and organized. Suggest clarifying: (1) gradient flow through sparse reads (explicitly state no gradients through ANN/selection), (2) constants in complexity, (3) index rebuild policy and amortized cost, (4) a brief pseudocode or diagram of the rollback BPTT mechanism.

Originality
High. The combination of sparse content access, ANN inside the learning loop, constant-space BPTT via reversible sparse diffs, and the LRU write policy is a notable and fresh direction beyond prior NTMs/Memory Networks/DNC.

Significance
High. If we want scalable neural memory, this is a major step. The demonstrated three-order-of-magnitude efficiency gains and the ability to train with million-slot memories make this broadly useful. I expect others to build on these ideas (e.g., integrating modern MIPS/ANN libraries, alternative sparse attention mechanisms, or applying to large-context NLP).

Questions/suggestions for authors
- Provide an ablation on K and on ANN vs exact top-K, and report the impact on convergence and final accuracy.
- Quantify amortized index rebuild cost and show scaling with word dimension M.
- Discuss potential use of soft top-k (e.g., sparsemax/entmax) or straight-through relaxations to smooth selection.
- Include a small-memory SAM vs NTM comparison to isolate the effect of sparsity alone.
- Release code for the sparse rollback and memory manager to facilitate adoption.

Pros for acceptance
- Bold, original approach to a central bottleneck in MANNs.
- Strong empirical evidence with dramatic efficiency gains and solid task performance.
- Generalizable methodology (shown via SDNC).
- High potential impact.

Cons against acceptance
- Limited analysis of the non-differentiable selection’s effect on learning.
- Evaluation largely on synthetic/curated tasks; fewer large real-world benchmarks.
- Some key implementation details relegated to the supplement.

Recommendation
Accept (Strong). Score: 8/10. Confidence: high. I value the novelty and the clear path this opens for scalable neural memory; the minor shortcomings are outweighed by the conceptual and practical contributions.

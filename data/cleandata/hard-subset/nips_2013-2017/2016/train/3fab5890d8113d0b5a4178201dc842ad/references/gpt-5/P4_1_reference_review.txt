Summary and relation to prior work
The paper introduces Sparse Access Memory (SAM), a memory-augmented neural network (MANN) that scales to very large external memories by enforcing sparsity in both reads and writes. The key ideas are: (i) sparse read operations that attend to only the top-K memory slots (K constant), computed via an approximate nearest neighbor (ANN) index to achieve sublinear retrieval; (ii) sparse write operations that target previously read locations or the least-recently-used (LRU) slot, enabling constant-time/space updates per step; and (iii) an efficient backpropagation-through-time (BPTT) scheme that rolls back sparse in-place memory updates, avoiding O(NT) memory overhead. The design aims for O(log N) time per step and O(1) space per step (beyond initialization), with asymptotic optimality claims discussed in the supplement.

SAM is positioned relative to Neural Turing Machines (NTMs) [7] and Memory Networks [21,17]. Unlike Memory Networks that pre-load and perform only reads, SAM supports writes similar to the NTM but with sparse access. The paper also shows how the approach can be applied to the Differentiable Neural Computer (DNC) [8], yielding a Sparse DNC (SDNC) reported to be >400× faster at N=2000 and strong on bAbI tasks.

Empirically, SAM matches or exceeds dense baselines (NTM, a dense variant DAM) on synthetic NTM tasks (copy, associative recall, priority sort), scales via curriculum to very long sequences and large memories, performs strongly (especially SDNC) on bAbI without supervised memory access, and achieves good one-shot performance on Omniglot under a curriculum.

Strengths
- Clear architectural contribution: top-K sparse reads via ANN, sparse writes via read-based/LRU addressing, and a practical BPTT rollback scheme. Together, these address the key bottleneck of dense content-based access.
- Compelling scalability: speed and memory benchmarks show orders-of-magnitude improvements versus NTM, with empirical O(1) per-step memory usage and sublinear time using ANN.
- Demonstrated generality: extension to SDNC indicates the approach can port to other MANNs.
- Evidence that sparsity does not hurt learning: SAM performs comparably or better than dense models on several tasks, despite approximate reads and sparse gradient flow.
- Thoughtful evaluation design: curriculum to stress long sequences; comparisons under approximately matched physical memory budgets.

Weaknesses and concerns
- ANN complexity claims and high-dimensionality: k-d trees degrade in high dimensions; LSH complexity is typically sublinear but not O(log N). The paper states O(log N) for both, which may overstate theoretical guarantees; empirical curves are helpful but theory is less precise.
- Approximation and gradient bias: truncating reads to top-K and using ANN introduces biased gradients and discontinuities when neighbors change. The paper provides empirical reassurance but lacks analysis of how approximation affects convergence or solution quality.
- Limited real-world evaluation: bAbI and Omniglot are useful but relatively small/synthetic. No large-scale NLP or long-document tasks where such memory would be most impactful (e.g., long-context QA, summarization, or program induction on real code).
- Memory management heuristic: the LRU/previously-read write scheme is simple and effective but may be suboptimal for tasks requiring consolidation or relocation; ablations on write policy choices are limited.
- Reliance on supplementary material: core complexity arguments, data structures, and SDNC details are in the supplement, which may limit accessibility and reproducibility from the main text.

Quality
- Technical soundness: The sparse read/write design and BPTT rollback mechanism are sound and implementable. The use of ANN inside the computational graph without gradients to the index is a practical choice. Claims of asymptotic optimality are plausible given the assumptions, but the O(log N) ANN claim should be tempered or more precisely stated.
- Empirical support: Benchmarks convincingly show speed/memory gains; tasks demonstrate that sparsity does not impede learning and can scale to long sequences. However, broader real-world testing would strengthen the case.
- Evaluation fairness: Resource-matched comparisons (same physical memory budget) are reasonable. It would help to include sensitivity to K, ANN type, and rebuild frequency.

Clarity
- Generally well organized with a clear problem statement and method. Key equations and control flow are described succinctly. Some essential implementation details (data structures, complexity proofs) are deferred to the supplement; a brief in-text summary of key assumptions/limits (e.g., dimensionality effects on ANN) would improve clarity.

Originality
- The combination of sparse content-based access, efficient ANN integration, and reversible sparse BPTT for MANNs appears novel and practically impactful. Related efforts using discrete addressing via RL [22] are acknowledged; SAM provides a differentiable alternative with lower-variance training.

Significance
- High potential impact: enabling MANNs with million-scale memories and long BPTT is a substantial step toward practical memory-augmented models. The generality (SAM, SDNC) increases relevance. Stronger demonstrations on large real tasks would further cement significance.

Questions and suggestions for authors
- Provide a more precise statement of ANN time complexity and its dependence on memory word dimensionality; consider reporting empirical scaling for various M and N, and sensitivity to K.
- Include ablations on K, write policy (LRU vs content-based vs read-based), and ANN choice (k-d tree vs LSH) with accuracy/speed trade-offs.
- Discuss stability of training when the ANN is rebuilt online; quantify how often rebuilds occur and their overhead.
- Evaluate on at least one larger real dataset where external memory is critical (e.g., long-context QA or code QA), or release code to facilitate follow-up.
- Clarify whether truncated BPTT is compatible with the rollback scheme and report any trade-offs.

Arguments for acceptance
- Addresses a core scalability barrier in MANNs with a principled, general approach.
- Strong empirical speed/space gains with maintained or improved accuracy.
- Demonstrated portability to other architectures (SDNC) and state-of-the-art bAbI results without supervised retrieval.

Arguments against acceptance
- Theoretical claims about ANN time (O(log N)) are optimistic; limited analysis of approximation effects on learning.
- Evaluation leans on synthetic tasks; limited large-scale real-world validation.
- Several crucial details are in the supplement, potentially hindering reproducibility.

Overall assessment
This is a well-executed, timely contribution that substantially improves the scalability of memory-augmented models with a clean sparse access design and practical engineering. While broader real-world validation and tighter theoretical treatment of ANN complexity would strengthen the work, the methodological contribution and empirical demonstrations are significant.

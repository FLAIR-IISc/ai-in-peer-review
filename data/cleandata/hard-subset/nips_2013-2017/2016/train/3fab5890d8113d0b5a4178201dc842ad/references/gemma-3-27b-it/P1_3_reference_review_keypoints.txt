- SAM addresses MANNs’ bottleneck by using sparse read/write operations and efficient data structures to reduce computation and memory.
- Sparse Access Memory (SAM) enables training with much larger external memories, achieving dramatic speedups and memory reductions.
- Empirical results show comparable or superior performance on algorithmic tasks, bAbI reasoning, and Omniglot one-shot classification.
- Curriculum learning experiments demonstrate SAM’s ability to scale to longer sequences.
- The sparse approach is general: applied to the DNC (SDNC) it yields substantial speedups.
- Builds on NTMs and Memory Networks, using content-based addressing while introducing sparsity as a core, novel principle.
- Paper cites relevant prior work, including approximate nearest neighbor (ANN) search techniques that inform implementation choices.
- Strength: significant scalability gains (reported up to three orders of magnitude) making MANNs more practical for real-world problems.
- Strength: provides a brief theoretical complexity analysis arguing asymptotic optimality, adding credibility.
- Strength: comprehensive empirical validation across diverse tasks and convincing scaling experiments.
- Strength: broad applicability demonstrated by extending the method to other architectures (DNC).
- Strength: clear writing and useful supplementary implementation and experiment details.
- Weakness: limited detail on ANN implementation choices (e.g., FLANN, LSH) and parameter tuning trade-offs.
- Weakness: insufficient analysis of hyperparameter sensitivity (sparsity K, discount factor).
- Weakness: some comparisons are not directly controlled (e.g., differing memory sizes across models).
- Weakness: write scheme (previously read / least recently accessed) lacks deeper justification and alternatives are not explored.
- Weakness: theoretical claim of asymptotic optimality would benefit from more formal, detailed analysis.
- Recommendation: overall a strong paper; recommend accept with minor revisions to add ANN implementation details and hyperparameter sensitivity analysis.

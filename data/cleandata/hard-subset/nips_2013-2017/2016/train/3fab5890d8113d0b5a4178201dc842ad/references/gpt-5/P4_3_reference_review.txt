Summary and relation to prior work:
The paper proposes Sparse Access Memory (SAM), a memory-augmented neural network (MANN) that scales read/write operations to large external memories by enforcing sparsity and using efficient data structures. Specifically: (i) reads are restricted to the top-K content matches obtained via an approximate nearest neighbor (ANN) index (randomized k-d trees or LSH), yielding O(log N) query time; (ii) writes are limited to the previously read locations and to the least-recently-used (LRU) location, with O(1) cost per step; (iii) backpropagation through time (BPTT) stores only sparse deltas and rolls memory back during the backward pass, reducing space to O(1) per step and total O(N) initialization + O(T) logs. The authors argue (under assumptions detailed in Supplementary A) that SAM is asymptotically optimal for content-addressed MANNs in time/space.

SAM is compared against Neural Turing Machines (NTM) and a dense variant (DAM). Experiments show substantial runtime and memory savings (up to three orders of magnitude vs. NTM at large N), comparable or improved data efficiency on classic NTM algorithmic tasks (copy, associative recall, priority sort), successful scaling via curriculum to very long sequences with millions of memory slots, and good performance on bAbi question answering and Omniglot one-shot classification. The approach is also applied to the Differentiable Neural Computer to obtain a Sparse DNC (SDNC) with large speedups and strong bAbi results. The work builds on content-addressable memory (NTM, Memory Networks) and complements efforts to scale MANNs (e.g., MemN2N, DNC; RL-based discrete addressing).

Quality:
- Technical soundness: The complexity analysis and engineering choices (sparse top-K, ANN queries, LRU write allocation, CSR storage, rollback BPTT) are plausible and supported by benchmarks. The end-to-end training uses gradients only through the selected K memory rows and through the similarity function; there are no gradients through the ANN index. This introduces non-differentiable top-K and approximate neighbor selection; while common in practice, the optimization is biased and piecewise constant w.r.t. memory contents. The paper does not provide a formal error analysis of the approximation (r̃ ≈ r) or stability guarantees; however, empirical evidence is strong across tasks. The write policy is partly hand-designed (previous reads + LRU), which may limit flexibility on tasks requiring more complex allocation, but it is simple and effective.
- Empirics: Speed/memory benchmarks are carefully done and clearly demonstrate scaling. Learning comparisons on NTM tasks show minimal cost of sparsity and in some cases benefits. The curriculum scaling results (including generalization to vastly longer sequences) are compelling. bAbi and Omniglot results are competitive; the SDNC variant performs particularly well on bAbi without supervision. Missing ablations include sensitivity to K, ANN choice and rebuild frequency, usage threshold ε, and comparisons of write policies.
- Reproducibility: Key hyperparameters and setup are said to be in the supplement. Given reliance on external ANN libraries and custom rollback data structures, code availability would be helpful (not discussed).

Clarity:
The paper is generally well organized and clear. The main ideas (sparse read/write, ANN integration, rollback BPTT) are communicated effectively. Some clarifications that would improve readability:
- Precisely state the assumptions behind the “optimal in time and space” claim in the main text (currently deferred to Supplementary A).
- Elaborate the differentiability story: how gradients flow through top-K selection, the effect of ANN approximation, and what is held constant.
- Provide pseudocode for the write policy (previous reads + LRU) and the O(1) usage tracking data structure.
- Specify ANN maintenance details (rebuild cadence/cost, deletion policy, dimensionality regimes for k-d tree vs. LSH).
- Make explicit the computational cost overhead of index updates per step in practice.

Originality:
The combination of (a) sparse content-based access implemented via ANN inside a differentiable MANN, (b) a simple sparse write policy based on previous reads and LRU, and (c) an efficient BPTT rollback mechanism for external memory is novel and practical. While related ideas exist (NTM/DNC content addressing, MemN2N’s reads, RL-based discrete addressing, and general sparse attention), the specific design and complexity guarantees, together with empirical validation at million-scale memories, represent a substantive contribution.

Significance:
The work addresses a central obstacle for MANNs—scaling to large memory and long sequences—with compelling empirical evidence and substantial speed/memory benefits. If broadly adopted, these techniques could enable MANNs on previously infeasible problems. The extension to SDNC and results on bAbi/Omniglot increase relevance beyond purely synthetic tasks. A larger real-world application (e.g., long-context language modeling) would further strengthen impact but is not strictly necessary for this contribution.

Pros:
- Clear, well-motivated approach to scaling MANNs with strong complexity claims.
- Large empirical speed and memory savings; sublinear-time reads via ANN.
- Comparable or improved learning efficiency on standard algorithmic tasks.
- Successful scaling via curriculum to very large memories and long horizons with BPTT.
- Competitive results on bAbi without supervision; solid one-shot learning on Omniglot.
- Generality demonstrated via SDNC.

Cons:
- Non-differentiable, approximate top-K/ANN introduces biased gradients; no formal error analysis.
- Hand-designed write allocation (previous reads + LRU) may limit expressivity; ablations missing.
- Fairness of some comparisons: sparse models use vastly larger memories than dense baselines (though matched on physical RAM), conflating capacity with method in some settings.
- Limited discussion of ANN maintenance overhead and robustness in high dimensions.
- Some clarity gaps deferred to supplements (assumptions, data structures).

Suggestions for improvement:
- Provide an ablation study on K, ANN type, rebuild frequency, ε, and write policies; report accuracy/speed trade-offs.
- Analyze the impact of approximate neighbor selection on gradient bias and training stability; include empirical robustness tests to ANN recall variations.
- Clarify the optimality assumptions and include a concise theorem statement in the main text.
- Report end-to-end wall-clock including ANN maintenance and rebuilds; discuss high-dimensional regimes.
- Include at least one more realistic large-scale task to illustrate practical utility.

Overall: A technically solid, original, and significant contribution that advances scalable MANNs with thorough empirical validation.

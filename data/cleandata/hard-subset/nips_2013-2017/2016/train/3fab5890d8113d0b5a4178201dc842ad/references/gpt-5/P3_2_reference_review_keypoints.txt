- Proposes Sparse Access Memory (SAM): sparse reads via ANN top-K and sparse writes to previously read + LRU slot, plus sparse rollback for BPTT to scale external memory.
- Uses ANN (randomized k-d trees or LSH) to restrict reads to top-K nearest memory slots, achieving sublinear query/insertion and O(log N) behavior.
- Restricts writes to a constant small set (previously read slots + LRU slot) with a non-differentiable usage tracker to get O(1) updates in N.
- Implements memory rollback for BPTT by logging sparse modifications, giving O(1) (in N) space per time step and O(T) total.
- Extends to a sparse DNC variant (SDNC) and applies SAM to NTM-like models.
- Empirically shows large speed and memory savings versus dense NTMs and strong performance on algorithmic tasks, bAbI, and one-shot Omniglot.
- Demonstrates scalability to very long sequences via curricula and up to three orders of magnitude compute/memory improvements.
- Main technical contribution: principled mechanisms to reduce time and space complexity for memory-augmented neural nets.
- Important systems insight: storing only sparse diffs for rollback is practical and significant.
- Generality: methods transfer to DNC, indicating broader applicability beyond a single architecture.
- Clarity: core ideas and motivations (e.g., LRU vs content write) are well explained.
- Key weakness: top-K selection and LRU/argmin are non-differentiable, so gradients are biased and the “end-to-end” claim is overstated.
- Need more discussion/ablations comparing exact dense soft attention vs exact top-K vs ANN top-K to quantify gradient bias and stability.
- Complexity claims omit dependence on K, number of heads, and word size M; backward rollback costs O(KM) per step and should be explicit in main text.
- ANN practicality concerns: high-dimensional degradation, large constants, parameter sensitivity, rebuild costs, and CPU/GPU device-placement effects need deeper analysis.
- Evaluation caveat: sparse models use far larger memories in curricula (capacity vs algorithmic robustness conflation); experiments at matched memory sizes are needed.
- Reproducibility issues: many implementation details (ANN params, K, thresholds, rebuild policy, usage tracker) are in the supplement; pseudo-code and key hyperparams should be in main text and report variance across seeds.
- Related work coverage should include DMN, Key-Value Memory Networks, and hard-attention methods to better position contributions.
- Questions for authors include sensitivity to K and ANN recall, gradient thrashing due to neighbor set changes, amortized index maintenance cost including rebuilds, memory churn/LRU overwrite diagnostics, and robustness across seeds.
- Minor requests: clarify O(1) refers to independence from N, provide forward/backward pseudo-code, and rephrase “end-to-end” to acknowledge non-differentiable ANN indices.
- Overall assessment: strong, novel, and significant work with solid engineering and experiments, but some claims need qualification and more ablations.
- Recommendation: Accept — timely and consequential contribution to scalable memory-augmented neural networks.

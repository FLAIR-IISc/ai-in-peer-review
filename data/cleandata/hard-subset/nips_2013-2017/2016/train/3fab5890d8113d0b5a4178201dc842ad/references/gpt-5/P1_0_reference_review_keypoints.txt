- Paper proposes Sparse Access Memory (SAM): top-K ANN-based reads, sparse writes (previously read + LRU), and an O(1)-per-step memory-rollback BPTT to scale differentiable memory sublinearly.

- Key claims: asymptotic optimality under assumptions, orders-of-magnitude training speed/memory savings, scalability to millions of words and thousands of time steps.

- Empirical results: 100–1600× training speedups, million-word memory and >4000-step BPTT, strong performance on algorithmic tasks, bAbI, and improved one-shot Omniglot results; SDNC variant yields ~400× speedup.

- Strength — Technical contribution: practical recipe (ANN top-K, sparse write policy, rollback BPTT) with thoughtful complexity analysis and implementation details.

- Strength — Empirical significance: dramatic speed/memory reductions enabling regimes previously infeasible for MANNs.

- Strength — Generality: approach extends to other MANNs (e.g., SDNC) and maintains competitive or improved task performance.

- Strength — Clarity: core ideas, equations, and design choices are concisely explained.

- Weakness — Non-differentiability: hard top-K selection via ANN blocks gradients, potentially introducing bias/variance not analyzed or ablated.

- Weakness — Dependence on ANN quality: no ablation comparing exact KNN vs ANN or sensitivity analysis to ANN hyperparameters and rebuild frequency.

- Weakness — Fairness of comparisons: dense baselines were capacity-limited (64 slots) to match physical memory, conflating sparsity gains with sheer capacity advantages.

- Weakness — Limited analysis of write policy: “previously read + LRU” is simple but unexplored for tradeoffs, alternatives, or failure cases requiring non-recent updates.

- Weakness — Reproducibility: many engineering details are in the supplement; code release or main-text pseudo-code would aid adoption.

- Weakness — Statistical reporting: several plots lack variance/CI; Omniglot highlights best run rather than averages, possibly overstating gains.

- Clarity suggestions: explicitly state gradient flow through ANN/selection, include forward/backward rollback pseudo-code, define symbols at first use, and summarize key supplementary results in the main text.

- Originality: high — novel integration of ANN search with differentiable memory and full sparse read/write + BPTT pipeline.

- Significance: high — addresses central MANN bottleneck and enables new scaling regimes likely to influence future work.

- Pros for acceptance: strong engineering, clear complexity gains, broad empirical validation, and applicability beyond NTM-like architectures.

- Cons against acceptance: missing analysis of optimization impacts from non-differentiable components, capacity-conflated baselines, and reliance on supplementary details.

- Questions for authors: sensitivity to K (ablations for K=2,4,8,16), exact vs approximate KNN tradeoffs, averages with CIs over seeds for main results, tasks where read+LRU fails versus content-based writes, and plans to release code. 

- Recommendation: Accept — substantial and timely contribution with solid empirical support despite some missing ablations and supplemental reliance.



# Review of "Scaling Memory-Augmented Neural Networks with Sparse Reads and Writes"

## Summary
This paper proposes SAM (Sparse Access Memory), a memory-augmented neural network architecture that restricts memory reads and writes to a sparse subset of memory words, achieving O(log N) time complexity for read operations and O(1) space complexity per time step. The authors demonstrate SAM's efficiency gains over NTM and dense memory networks, showing up to 1600x speedup and 3 orders of magnitude memory reduction, while maintaining performance on synthetic tasks and Babi question answering.

## Strengths
The paper addresses a genuine limitation in memory-augmented networks: the O(N) time and space complexity of memory operations. The sparse read/write approach is conceptually sound and well-motivated. The experimental results demonstrate significant practical advantages, particularly for large memory sizes. The approach is generalizable to other memory architectures (e.g., SDNC), and the use of approximate nearest neighbor structures is well-justified.

## Weaknesses
Despite the promising concept, the paper contains numerous technical and methodological inconsistencies that undermine its scientific rigor:

1. **Notational and Formatting Errors**: The paper contains numerous inconsistent mathematical notations (e.g., "w(i)M(i)" instead of "w^{(i)}M^{(i)}"), incorrect summation notation in Equation (4), and inconsistent spacing in equations (e.g., "Mt (1 Rt) Mt 1 +At" should be "M_t = (1 - R_t)M_{t-1} + A_t"). These are basic presentation errors that should not appear in a NIPS submission.

2. **Contradictory Claims**: The paper claims "SAM's memory usage per time step is independent of the number of memory words" (Section 3), but in Section 4.1, it reports that "For 64K memory words SAM uses 53MiB of physical memory to initialize the network" - suggesting memory usage is directly proportional to memory size, contradicting the asymptotic claim.

3. **Lack of Statistical Rigor**: The paper claims "sparse models are able to learn more effectively" for certain tasks (Section 4.2) but provides no statistical significance testing. Without this, it's impossible to determine if the observed performance differences are meaningful.

4. **Inconsistent Terminology**: The paper uses both "SAM" and "Sparse Access Memory" inconsistently, and the acronym "MANN" is used without definition. The term "sparse access memory" appears to be a misnomer for "sparse read/write memory."

5. **Missing Technical Details**: The paper claims SAM is "asymptotically optimal in time and space complexity" but fails to provide the asymptotic analysis in the main paper (referencing Supplementary A without providing the proof). The paper also claims gradients can be computed "in constant time and space" but doesn't explain how this is achieved with sparse matrix operations.

6. **Methodological Ambiguity**: The paper states "we tried two definitions of Ut" but doesn't explain why one was chosen over the other, nor does it provide comparative performance results for the two definitions. The choice of " = 0.005" as a threshold for the usage measure is arbitrary without justification.

7. **Inconsistent Claims**: The paper states "SAM is over 100 times faster than the NTM when the memory contains one million words" but doesn't provide a baseline time for the NTM with 1 million words, making the speedup claim difficult to evaluate.

8. **Formatting Inconsistencies**: The paper uses "⇡" for "approximately" (Section 1) instead of the standard "≈" symbol. It also has inconsistent spacing in mathematical expressions and uses "linear-index" (with a hyphen) in Section 4.1 where it should be "linear index."

9. **Missing Context**: The paper doesn't clearly explain how SAM differs from prior work on sparse memory access (e.g., [22] uses reinforcement learning for sparse addressing). The claim that "they will be no more efficient than SAM" in Supplementary A lacks supporting argumentation.

## Recommendation
While the paper presents a potentially useful approach to scaling memory-augmented networks, the numerous technical and methodological inconsistencies, contradictory claims, and lack of rigorous presentation significantly undermine its scientific contribution. The paper reads like a preliminary technical report rather than a polished NIPS submission. The errors and omissions are too numerous and fundamental to be overlooked in a peer-reviewed venue at this level. The authors should address these issues before considering a revision.

I recommend rejection due to the substantial number of minor but critical technical flaws that collectively prevent the paper from meeting NIPS's quality standards. The paper's contribution is interesting but is presented in a manner that fails to demonstrate the rigor expected at this venue.

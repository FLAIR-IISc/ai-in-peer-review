- Introduces SAM (Sparse Access Memory), a memory architecture that restricts reads/writes to a constant K sparse subset of memory words.
- Achieves O(log N) content-based addressing and O(1) per-step memory overhead versus prior O(N) approaches.
- Demonstrates up to 1600x speedup over NTMs while maintaining comparable task performance across synthetic and real-world benchmarks.
- Provides rigorous theoretical analysis (Supplementary A) showing asymptotic optimality in time/space.
- Experimental methodology is thorough with comparisons across tasks, memory sizes, and architectures (SAM, DAM, NTM, LSTM).
- Ablation studies show sparse operations do not impair learning and can sometimes improve it.
- Paper clearly written, well-organized, and reproducible with consistent notation and informative figures.
- Novel integration of sparse memory access within a differentiable MANN framework using ANN data structures.
- Properly differentiates from and cites prior work (e.g., NTMs, reinforcement-learning addressing).
- Addresses a critical scalability bottleneck, enabling training with millions of memory words and substantial memory reductions.
- Shows practical gains on real tasks (Babi, Omniglot) and generalizability (Sparse DNC).
- Strength: simple, elegant solution to MANN scalability with strong theoretical and empirical support.
- Strength: demonstrates practical applicability and large speed/memory improvements while preserving performance.
- Weakness: choice of K (e.g., 4 or 8) is somewhat arbitrary and underexplored.
- Weakness: ANN/index implementation details, especially sync with memory updates, are insufficiently specific.
- Weakness: limited comparison to other scaling approaches (e.g., reinforcement-learning-based methods).
- Weakness: potential limitations for tasks needing fine-grained memory access are not thoroughly explored.
- Weakness: explanations for why sparsity sometimes improves learning are speculative.
- Recommendation: accept — significant, well-supported contribution with high relevance to the field.

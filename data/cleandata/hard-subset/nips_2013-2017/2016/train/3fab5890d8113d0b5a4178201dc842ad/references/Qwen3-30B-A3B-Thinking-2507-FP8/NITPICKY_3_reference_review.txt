

# Review of "Scaling Memory-Augmented Neural Networks with Sparse Reads and Writes"

This paper proposes SAM (Sparse Access Memory), a memory-augmented neural network architecture that constrains memory reads and writes to a sparse subset of memory words. The authors claim that SAM achieves optimal time and space complexity with respect to memory size while maintaining end-to-end differentiability. The paper demonstrates SAM's effectiveness on several synthetic and natural tasks, showing significant speedups and memory efficiency compared to dense variants.

## Technical Concerns

The paper contains several technical inconsistencies that undermine its rigor. In Section 3.1, equation (4) uses inconsistent notation: "r̃t = sum_{i=1}^K w̃Rt(si)Mt(si)" should be "r̃t = sum_{i=1}^K w̃^R_t(s_i)M_t(s_i)" to clarify the time step index and memory word index. The paper also uses "T" both as the total time steps and as a time step variable (e.g., in equation (6)), creating significant confusion. The authors should use different notation (e.g., "t" for time step index, "T" for total time steps).

In equation (5), the write weights are defined as "w^W_t = α_t t w^R_{t-1} + (1 - t) I_Ut" where "t" is used both as a time step and a parameter. This is confusing and should be denoted differently (e.g., "α_t" instead of "t" for the interpolation gate parameter).

The paper claims "SAM uses 53MiB of physical memory to initialize the network and 7.8MiB to run a 100 step forward and backward pass" (Section 4.1), but fails to provide the exact configuration (memory size, batch size, etc.) that led to these numbers. Without this information, the claims are difficult to verify.

In Section 4.2, the authors state that "sparse models are able to learn with comparable efficiency to the dense models" but provide no statistical significance testing for the results in Figure 2. This weakens their claim about the comparability of learning efficiency.

## Formatting and Presentation Issues

The paper contains numerous formatting inconsistencies that suggest a lack of careful proofreading. The phrase "th Conference on Neural Information Processing Systems" (Introduction) should be "The Conference." The paper uses "⇡" for "approximately" (Section 1) instead of the standard "≈" symbol. It also uses "⇥" for "times" (e.g., "400⇥ faster") instead of "×" or "×."

The reference list contains inconsistent formatting. For example, "An optimal algorithm for approximate nearest neighbor searching fixed dimensions, Sunil Arya, David M. Mount, Nathan S. Netanyahu, Ruth Silverman, Angela Y. Wu, J. ACM,, , ," has inconsistent punctuation and missing information (should be "Journal of the ACM" instead of "J. ACM" in proper academic format).

In Section 3.2, equation (6) is poorly formatted: "IUt(i) = (1 if Ut(i) = min j=1,...,N Ut(j) otherwise)" should be written as a piecewise function for clarity.

The paper claims "SAM is over 100 times faster than the NTM when the memory contains one million words and an exact linear-index is used" (Section 4.1), but "exact linear-index" is not a standard term. This appears to be a typo for "exact linear index" or "exact nearest neighbor index," but the authors should clarify.

## Minor Technical Concerns

The paper states that "SAM is optimal in space and time with respect to memory size" (Introduction), but the proof of optimality is relegated to Supplementary A, which is not referenced in the main text. The main paper should at least briefly state the key assumptions for the optimality claim.

In Section 3.5, the authors claim "there is an O(logN) cost for insertion, deletion and query" for ANN indexes, but this is not entirely accurate. While the query time is O(logN), insertion and deletion typically have higher complexity for some ANN structures (e.g., k-d trees have O(N) worst-case insertion complexity). The authors should clarify the exact complexity for each operation.

The paper uses "MiB" and "GiB" without explaining the difference between binary and decimal units. While this is common in computer science, the paper should be consistent with its usage (e.g., always using "MiB" instead of "MB" to avoid confusion).

## Strengths

The paper makes a technically interesting contribution to the field of memory-augmented neural networks. The sparse read/write approach is novel and well-motivated, and the empirical results on synthetic tasks and Babi tasks are convincing. The paper demonstrates significant practical improvements in computational efficiency and memory usage.

## Weaknesses

The paper's main weakness is the lack of attention to detail in presentation and technical writing. The numerous formatting inconsistencies and technical notational errors suggest a lack of careful proofreading. While the core idea is promising, these issues undermine the paper's credibility and make it difficult to assess the technical claims with confidence.

The paper does not sufficiently address the limitations of the sparse approach. For example, it doesn't discuss how the sparsity threshold (K) affects performance across different tasks, or how the authors determined the optimal value of K. This is a significant omission for a paper claiming to provide an optimal solution.

## Recommendation

While the paper presents a potentially useful contribution to the field, the numerous technical inconsistencies, formatting errors, and lack of precision in the presentation significantly undermine its scientific rigor. As a perfectionist reviewer who values meticulous attention to detail, I cannot recommend acceptance due to these substantial minor flaws that collectively suggest a lack of care in preparing the manuscript for publication. The paper would benefit from a thorough proofreading and clarification of the technical details before being considered for publication.

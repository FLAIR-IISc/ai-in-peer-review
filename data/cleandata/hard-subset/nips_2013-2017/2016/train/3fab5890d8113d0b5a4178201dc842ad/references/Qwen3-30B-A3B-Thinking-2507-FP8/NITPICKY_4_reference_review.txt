

# Review of "Scaling Memory-Augmented Neural Networks with Sparse Reads and Writes"

This paper proposes SAM (Sparse Access Memory), a memory-augmented neural network architecture that addresses the high computational and memory overhead of traditional memory-augmented models like Neural Turing Machines (NTMs) by restricting memory operations to sparse subsets. The key innovations are sparse read/write operations constrained to a small constant number of memory words (K=4-8), efficient data structures for content-based addressing, and a memory management scheme tracking usage to overwrite stale slots.

The paper demonstrates SAM's advantages through extensive experiments on synthetic tasks (Copy, Associative Recall, Priority Sort), Babi reasoning tasks, and Omniglot one-shot classification. SAM achieves up to three orders of magnitude faster runtime and reduced memory overhead compared to vanilla NTMs while maintaining comparable performance. The paper also introduces a Sparse Differentiable Neural Computer (SDNC) as a generalization of SAM.

## Strengths
The paper presents a technically sound approach to scaling memory-augmented networks with clear empirical evidence of computational benefits. The experimental results are thorough, comparing against multiple baselines including NTM, DAM, and LSTM across diverse tasks. The introduction of sparse memory access with efficient data structures is a practical contribution to the field.

## Weaknesses

1. **Formatting and Notation Inconsistencies**: The paper uses non-standard notation throughout (e.g., "⇡" for "approximately," "2⇥106" for "2×10^6," "1MiB" without space). Section 4.3 states "2⇥106 words" which should be "2×10^6" or "2 million." The phrase "th Conference" (p.1) should be "the Conference." The paper mentions "Figure 1" but doesn't include the figure, which is a critical omission.

2. **Technical Ambiguities**: The claim that SAM is "asymptotically optimal in time and space complexity" (Section 3) lacks supporting proof in the main paper, with references only to "Supplementary A" (which isn't provided). The paper states SAM's memory usage is "independent of the number of memory words" but then reports specific memory usage (53MiB for 64K words) in Section 4.1, creating confusion about whether this is a one-time initialization cost or per-time-step cost.

3. **Implementation Details**: The paper claims "SAM uses 53MiB of physical memory to initialize the network and 7.8MiB to run a 100 step forward and backward pass" (Section 4.1), but this is ambiguous. If initialization is one-time, it shouldn't be reported as a per-time-step cost. The paper also states "SAM is over 100 times faster than the NTM when the memory contains one million words" without specifying the baseline NTM implementation used for comparison.

4. **Inconsistent Notation**: The paper uses both "w̃Rt" and "wRt" for sparse and dense weight vectors but inconsistently applies the tilde notation. Equation (5) uses "wWt" while the text refers to "w̃Wt" as sparse write weights, creating confusion.

5. **Missing Methodological Details**: The paper claims to use "a custom data-structure" for maintaining usage statistics in constant time (Section 3.2) but provides no details about this data structure, which is critical for reproducibility. The explanation of how the approximate nearest neighbor index is integrated into the network during training (Section 3.5) is superficial, with no discussion of gradient flow through the ANN structure.

6. **Unsubstantiated Claims**: The paper states "the SDNC manages to solve all but 1 of the tasks, the best reported result on Babi that we are aware of" (Section 4.4) without citing the previous best result, which is required for such a claim. Similarly, the claim that "the NTM may perform poorly since it lacks a mechanism which allows it to allocate memory effectively" is not substantiated with evidence.

7. **Supplementary Material Issues**: The paper references "Supplementary D" for the SDNC, but the main paper doesn't include a Supplementary D section, creating confusion for readers.

## Recommendation

While the paper presents a valuable technical contribution to scaling memory-augmented networks with clear empirical advantages, the numerous minor methodological inconsistencies, formatting errors, and unexplained technical details undermine the paper's credibility. The lack of proper notation, ambiguous claims, and missing implementation details make it difficult to fully assess the technical merits of the proposed approach. As a perfectionist reviewer who values meticulous attention to detail, I find these issues sufficiently concerning to recommend rejection, despite the paper's potentially useful contribution to the field.

The paper would benefit from careful proofreading to fix notation and formatting inconsistencies, clarification of ambiguous claims (particularly regarding memory usage and computational complexity), and more detailed descriptions of the custom data structures and implementation choices. These minor issues collectively prevent me from recommending acceptance at NIPS.

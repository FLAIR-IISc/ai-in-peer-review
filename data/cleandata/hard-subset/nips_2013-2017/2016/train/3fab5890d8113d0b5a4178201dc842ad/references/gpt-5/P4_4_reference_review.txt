Summary and relation to prior work
This paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that scales content-addressable external memory by enforcing sparsity in both reads and writes and by using efficient data structures to implement content-based addressing. The key ideas are: (i) sparse read as a top-K weighted average over memory words, with K constant and independent of memory size; (ii) sparse write via a combination of updates to previously read locations and allocation to the least-recently-accessed slot, yielding constant-time/space per-step writes; (iii) an efficient backpropagation-through-time (BPTT) scheme that stores and reverts only sparse modifications to memory (“rollback”), achieving O(1) extra space per time step and O(T) total; and (iv) approximate nearest neighbor (ANN) indices (randomized k-d trees or LSH) to compute top-K content-based reads in O(log N). Under assumptions described in the supplement, SAM is asymptotically optimal in time and space for content-addressed MANNs.

SAM is evaluated on synthetic NTM tasks (copy, associative recall, priority sort), curriculum-extended long-sequence variants, bAbI question answering, and Omniglot one-shot classification. It achieves 100–1600× wall-clock speedups and orders-of-magnitude lower memory use versus dense NTMs at large memory sizes, with comparable or better data efficiency. The approach is shown to generalize to a sparse DNC (SDNC), yielding large speedups and strong bAbI performance without supervised memory access.

Relative to prior MANNs—NTM [Graves et al., 2014], Memory Networks/End-to-End Memory Networks [Weston et al., 2015; Sukhbaatar et al., 2015], and the Differentiable Neural Computer [Graves et al., 2016]—the main novelty is enforcing sparsity with algorithmic support (ANN indices and LRU-style allocation) while retaining end-to-end differentiability through the memory contents. The paper also relates to reinforcement-learning-based discrete addressing (RL-NTM; Zaremba & Sutskever, 2015), which attains sparsity at the cost of higher-variance optimization.

Quality
- Technical soundness: The sparse read/write mechanisms are well-specified. The ANN-based top-K selection is fixed and non-differentiable, but gradients flow through the selected memory rows; this is standard for top-K attention approximations. The memory rollback scheme for BPTT is technically sound and practically important. Complexity claims (O(log N) time per step; O(1) per-step space) are plausible and supported by empirical scaling curves; formal optimality appears in the supplement.
- Empirical support: Benchmarks show dramatic runtime and memory savings; synthetic tasks indicate that sparsity does not harm, and can sometimes help, learning. Curriculum experiments demonstrate training over thousands of time steps with millions of memory slots. bAbI and Omniglot results show applicability beyond toy tasks; the SDNC achieves strong bAbI performance without supervised support labels. While baselines include NTM, an LSTM, and a dense variant (DAM), comparisons to strong contemporary memory models (e.g., MemN2N with tuned hops, key-value memories, or dense DNC on the same setup) could be expanded. Ablations on K, ANN index type/accuracy, and write policy would strengthen causal claims.
- Careful evaluation: The paper controls physical memory budgets when comparing dense vs sparse in curriculum experiments, which is reasonable. Some training details and full results are deferred to the supplement; key numbers for bAbI/Omniglot in the main text are high level.

Clarity
The paper is generally clear and well-organized. The architectural description (Sections 3.1–3.4) is concise; the role of the ANN index, memory management (least-recently-accessed allocation), and rollback is understandable. However:
- Provide a brief summary of the optimality argument in the main text, beyond “see Supplementary A.”
- Include pseudo-code for forward/backward with rollback, and for maintaining the usage statistics.
- Expand on how the ANN is updated online and how often it is rebuilt, with measured overhead.
- Consolidate key hyperparameters (K, thresholds, choice between k-d tree vs LSH) and sensitivity analyses in the main paper or a succinct table.

Originality
The combination of (i) sparse, top-K content-based reading with (ii) constant-time sparse writing via LRU-style allocation and (iii) an online ANN index integrated into a differentiable MANN, along with (iv) the rollback BPTT trick, is novel and practically impactful. Individual components (top-K attention; LRU; ANN data structures) are known, but their integration in a way that preserves end-to-end training and yields provable and empirical scalability is a meaningful contribution. The extension to a sparse DNC further supports generality.

Significance
Scaling content-addressable memory to millions of slots with sublinear time and constant per-step space is significant. This removes a key bottleneck for MANNs and enables training on much longer sequences without truncated BPTT. The approach is likely to be adopted in future memory models and related architectures (e.g., sparse attention). The improvements on bAbI and Omniglot, while not state-of-the-art across all settings, demonstrate utility on non-synthetic tasks.

Pros (arguments for acceptance)
- Strong, general architectural contribution enabling asymptotically and empirically efficient memory access in MANNs.
- Careful systems work (ANN indices; rollback BPTT; memory management) with large speed and memory savings verified experimentally.
- Demonstrated on diverse tasks, including long-sequence curricula and non-synthetic data; generalizes to DNC with strong bAbI performance without supervision.
- Maintains or improves learning efficiency despite sparsity.

Cons (arguments against acceptance)
- Limited ablations: lack of systematic study of K, ANN recall vs accuracy/speed trade-offs, and write policy variants.
- Some baselines are weaker than the strongest contemporaries; clearer comparisons to dense DNC and tuned MemN2N would help.
- The “optimality” claim is relegated to the supplement; assumptions and limits (e.g., dimensionality effects on k-d trees, LSH parameterization) could be made more explicit.
- Non-differentiable top-K/ANN selection may bias learning; theoretical analysis of this approximation is absent.

Questions for the authors
- How sensitive are results to K and to the choice of ANN index and its parameters? Can K be adapted during training?
- What is the amortized cost of maintaining and periodically rebuilding the ANN index, and how does this affect end-to-end wall-clock time?
- How often does allocation choose the least-recently-accessed slot vs updating previously read slots, and how sensitive is performance to the threshold used in U(2)?
- For bAbI, please report per-task error rates, dataset version (1k vs 10k), and whether joint training was used, to facilitate comparison.

Suggestions for improvement
- Add concise pseudo-code and a main-text summary of the complexity/optimality argument.
- Provide ablations on K, ANN variants/recall, and write strategies; include sensitivity plots.
- Report absolute training time and throughput for major experiments, alongside relative speedups.
- Release code and scripts to reproduce ANN maintenance and rollback BPTT.

Overall, this is a solid and impactful contribution that advances scalable memory in neural networks with clear engineering and substantive empirical validation.

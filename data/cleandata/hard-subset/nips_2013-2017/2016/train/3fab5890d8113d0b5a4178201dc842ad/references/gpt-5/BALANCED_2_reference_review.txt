Summary and relation to prior work
This paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that scales content-addressable read/write operations to very large external memories by enforcing sparsity and using efficient data structures. Key ideas: (1) sparse reads via top-K selection implemented with an approximate nearest neighbor (ANN) index (randomized k-d trees or LSH), yielding O(log N) query time; (2) sparse writes to a small set of locations, specifically previously read locations and a least-recently-accessed (LRA) location based on a usage measure; (3) an efficient BPTT scheme that rolls the memory state back in-place by storing only sparse deltas, achieving O(1) per-step memory overhead; and (4) an argument (Supplementary A) that, under reasonable assumptions, the approach is asymptotically optimal in time/space for content-addressed memories. Experiments span synthetic NTM tasks, curriculum scaling to long sequences, bAbI QA, Omniglot one-shot classification, and an extension to a sparse DNC (SDNC), showing large speed/space gains and competitive or superior performance.

The work is closely related to Neural Turing Machines [7] and Memory Networks [21,17]. Unlike Memory Networks (which pre-load and only read), SAM supports online writes like NTM/DNC but replaces dense attention with sparse top-K and an LRA write allocation akin to usage-based allocation in DNC [8]. It also relates to RL-NTM [22] in enforcing sparse access, but remains fully differentiable (modulo non-differentiable selection) and uses approximate top-K rather than REINFORCE. The inclusion of ANN inside the network aligns with scalable nearest-neighbor methods (e.g., FLANN [15], LSH [3]) and complements meta-learning with MANNs [16].

Strengths
- Clear, impactful problem: scaling differentiable memory to millions of slots, a well-motivated bottleneck for MANNs.
- Elegant engineering: sparse reads/writes, LRA-based allocation, and in-place rollback for BPTT together yield dramatic asymptotic and empirical gains (up to 10^2–10^3× runtime and orders-of-magnitude memory savings).
- Broad empirical scope: synthetic algorithmic tasks, long-sequence curricula (without truncation), bAbI (state-of-the-art with SDNC, without supervised retrieval), and Omniglot one-shot. Evidence of generalization to sequences far longer than training.
- Generality: method applies beyond NTM; SDNC results indicate reusability.
- Practicality: concrete data structures, ANN implementations, and measured speed/memory profiles.

Weaknesses and points for clarification
- Gradient bias from sparsification: The read weights are truncated to the top-K, and the ANN index is non-differentiable. Gradients only flow through selected rows. While effective empirically, the paper does not analyze the bias introduced versus dense soft attention or provide conditions under which learning remains stable. A small ablation quantifying task performance as a function of K and ANN recall would strengthen the case.
- Optimality claim: The “asymptotically optimal” statement relies on assumptions (constant K, content-based addressing). A clearer articulation of the lower-bound model and its scope would help (e.g., do locality properties or restricted controllers alter the lower bound?).
- Fairness of empirical comparisons: In the curriculum experiments, dense models use 64 memory slots while sparse models use up to 2×10^6 to equalize physical memory over 100 steps. This favors SAM on tasks where capacity, not algorithmic efficiency, limits performance. Consider reporting results that also equalize number of slots or compute budget, or a Pareto curve (performance vs memory/compute).
- ANN dynamics: Online updates can imbalance trees; the paper rebuilds every N insertions. The effect on convergence and overhead is not fully quantified. Reporting rebuild cost share, ANN recall, and sensitivity to dimensionality would improve reproducibility.
- Write policy design: The choice to write to previously read locations plus LRA is reasonable, but alternatives (e.g., sparse content-based writes, learned allocation strategies) are not explored. An ablation comparing write policies would clarify robustness.
- Clarity and presentation: Some notation is dense (e.g., wWt spacing, indicators), and the description of how the softmax normalization is handled after top-K selection could be more explicit. Minor typographical issues exist.

Assessment by criteria
- Quality: Technically solid with careful engineering. Complexity claims align with the design. Empirics are strong and diverse. Some theoretical aspects (gradient bias, optimality assumptions) could be better justified.
- Clarity: Generally well written and organized, with helpful supplements. A few places could benefit from more explicit algorithmic details and ablations.
- Originality: High. Deterministic sparse attention with ANN and a practical rollback scheme for BPTT in MANNs is novel relative to prior dense attention or RL-based sparsity.
- Significance: High. The approach materially advances the scalability of MANNs and is likely to influence follow-on work and applications.

Pros for acceptance
- Addresses an important scalability bottleneck with a principled, general solution.
- Large, compelling empirical speed/memory gains without sacrificing performance.
- Demonstrated generality (application to DNC) and strong results on diverse tasks.
- Opens the door to training on very long sequences without truncated BPTT.

Cons for acceptance
- Limited analysis of the optimization implications of sparse, approximate attention.
- Some comparisons favor SAM by granting vastly larger capacity; more balanced baselines would help.
- Sensitivity to K, ANN choice/recall, and write policy not fully explored.

Questions for the authors
- How sensitive are results to K and to the ANN recall level? Do you re-normalize the softmax over only the K candidates?
- What fraction of time is spent on ANN maintenance vs network compute, and how does this scale with dimensionality?
- Can you report ablations on the write policy (e.g., removing LRA, using sparse content-based writes)?
- How often do index rebuilds occur in practice, and how critical are they for stability?

Suggestions for improvement
- Add ablations on K, ANN types/parameters, and write allocation strategies.
- Provide clearer formalization of the optimality claim and discuss gradient bias.
- Include a compute- and slot-matched baseline to complement the memory-matched setting.
- Release pseudocode and, if possible, code for the rollback/backprop and ANN synchronization.

Recommendation
Accept. The paper presents a timely and influential contribution that significantly improves the scalability of differentiable memory systems with strong empirical validation and broadly applicable ideas.

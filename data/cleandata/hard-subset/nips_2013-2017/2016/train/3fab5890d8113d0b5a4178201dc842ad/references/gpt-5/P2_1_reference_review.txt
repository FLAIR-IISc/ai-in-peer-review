Summary and relation to prior work
The paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that scales read and write operations to be sublinear in the number of memory slots by enforcing sparsity (top-K reads; writes to previously read and least-recently used slots) and using efficient data structures (approximate nearest neighbor indices) plus a rollback scheme for backpropagation. It claims O(log N) time per step and O(1) additional space per step (after O(N) init), while retaining end-to-end differentiability for the parts that remain continuous. Experiments show 2–3 orders of magnitude speed and memory savings versus classic NTMs, comparable or better learning efficiency on standard NTM algorithmic tasks, the ability to scale via curriculum to very long sequences with millions of memory slots, strong results on bAbI (with a sparse DNC variant), and improved one-shot performance on Omniglot sequences. The work builds directly on NTMs and Memory Networks, addresses the known bottleneck of dense attention and dense writes, and is related to DNC’s usage-based allocation and to discrete-addressing/RL NTMs, but offers a differentiable sparse alternative with practical engineering.

Quality
- Technical soundness: The core idea—restricting computation to a small, content-selected subset and maintaining consistent ANN indices—appears sound. The complexity claims are plausible and supported by implementation choices (CSR for sparse ops, O(1) write-set tracking, in-place forward modifications with reverse-time rollback for BPTT). The “optimal in time and space” statement is qualified by assumptions (content-based addressing, constant-K sparsity) and relegated to the supplement; the main text could articulate these assumptions more explicitly.
- Empirical support: Benchmarks robustly demonstrate wall-clock and memory savings as N grows, and tasks cover synthetic algorithmic problems, curriculum scaling, bAbI, and Omniglot—strong breadth for this topic. The comparisons to dense baselines (NTM, a dense approximation DAM, LSTM) and to an SDNC variant support the claim that sparse approximations do not harm learning and can even help early-stage optimization.
- Caveats: (1) Some components are non-differentiable (top-K selection; LRU-based slot choice; ANN query) and are treated as fixed mechanisms without gradients. This is reasonable but should be discussed more explicitly regarding any bias introduced and the conditions under which it might fail. (2) Experimental fairness: in scaling experiments SAM uses far larger memory than dense baselines but equalizes physical memory usage; that is defensible given the paper’s goal (scaling), but it makes accuracy comparisons less apples-to-apples. (3) Ablations are limited: the effect of K, different write policies (e.g., content-based sparse writes), ANN choice and rebuild frequency, and the erase scheme could be studied more systematically. (4) The “optimality” and BPTT memory-rollback details are mostly in the supplement; brief pseudocode or a theorem statement in the main text would improve reproducibility.

Clarity
The paper is generally clear and well organized. The high-level algorithm is easy to follow, and the motivation for each design choice (sparse reads, LRU allocation, ANN) is provided. However:
- Gradient flow and non-differentiable components should be spelled out in the main text, including which paths receive gradients and which do not.
- The memory-rollback mechanism for BPTT is important; a concise schematic/pseudocode in the main text would help readers reimplement.
- Experimental setup details (hyperparameters, ANN type selection criteria, rebuild schedule) are mostly in the supplement; a condensed summary in the main paper would aid comprehension.

Originality
The combination of: (i) top-K sparse content reads via in-graph ANN; (ii) LRU-based sparse writes; and (iii) a constant-space BPTT rollback scheme for external memory is novel and impactful relative to NTMs, Memory Networks, and DNC. Prior work on discrete addressing via RL shares the “sparse access” idea but with high-variance gradients; SAM’s differentiable sparse design is a distinct contribution. The sparse DNC extension further supports generality.

Significance
Scaling MANNs is a central bottleneck for long-context and algorithmic tasks. Demonstrating orders-of-magnitude runtime/memory savings with comparable or better task performance is highly significant, likely to influence subsequent architectures and practical deployments. The curriculum experiments to millions of slots and thousands of time steps without truncated BPTT are especially compelling. The bAbI and Omniglot results, while not exhaustive, show that the approach transfers beyond toy copy tasks.

Pros for acceptance
- Strong, well-motivated contribution addressing a known and important scalability problem.
- Clear algorithmic innovations with sound engineering and plausible complexity.
- Extensive empirical validation across tasks and scales; dramatic efficiency gains.
- Generality demonstrated via SDNC and applicability to other MANNs.
- Potential high impact for research and practice.

Cons for acceptance
- Non-differentiable components and ANN integration are treated as fixed; more analysis of failure modes and training bias would strengthen confidence.
- Limited ablations (e.g., K, write policy variants, ANN trade-offs, effect of sparsity on accuracy).
- Some key details (optimality assumptions, BPTT rollback) are deferred to the supplement, affecting reproducibility.
- Fairness of some comparisons could be discussed more (equal memory footprints vs equal capacity).

Suggestions to improve the paper
- Add a concise statement of the optimality result and BPTT rollback algorithm in the main text.
- Provide ablations on K, ANN choice/rebuild frequency, and a comparison to a sparse content-based write policy.
- Clarify gradient pathways and the implications of non-differentiable choices (top-K, LRU).
- Report wall-clock breakdown of ANN maintenance vs controller compute, and discuss GPU vs CPU performance.
- Expand discussion of when sparse approximations might hurt (e.g., highly diffuse attention needs).

Overall evaluation
This is a high-quality, original, and significant contribution that squarely addresses an important bottleneck in MANNs with careful algorithmic design and convincing experiments. Despite some missing ablations and reliance on supplemental details, the paper meets NeurIPS criteria well. I recommend acceptance.

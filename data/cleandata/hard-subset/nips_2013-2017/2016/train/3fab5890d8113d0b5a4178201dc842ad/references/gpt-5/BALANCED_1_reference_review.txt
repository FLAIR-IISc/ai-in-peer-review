Summary and relation to prior work
This paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that scales read/write operations to very large external memories by enforcing sparsity and using approximate nearest neighbor (ANN) indexing. Key ideas are: (1) sparse content-based reads, implemented by top-K selection via an ANN structure (randomized k-d trees or LSH) to achieve O(log N) query/update; (2) sparse writes that target previously read locations and a least-recently-used (LRU) location, yielding O(1) per-step compute and space for writes; (3) an efficient backpropagation-through-time scheme that stores only sparse deltas and rolls memory back during the backward pass, resulting in O(1) per-step memory overhead and O(T) total space. Under assumptions detailed in the supplement, SAM is argued to be asymptotically optimal for content-based addressing.

The work is positioned against LSTM (parameter-memory coupling), NTMs and Memory Networks (decoupled capacity but dense attention), and DNC (richer allocation/temporal links but dense access). Compared to reinforcement-learning-based discrete addressing (e.g., Zaremba & Sutskever, 2015), SAM maintains differentiability by using sparse, approximated soft attention. Empirically, SAM shows large speed/space gains vs NTM and a dense variant (DAM), learns classic NTM algorithmic tasks, scales via curriculum to long sequences with million-word memories, performs competitively on bAbI (especially with a sparse DNC variant), and improves one-shot Omniglot performance at long sequence lengths.

Quality
- Technical soundness: The approach is well-motivated and the complexity claims are plausible given sparse updates and ANN. The in-place update and rollback scheme for BPTT is a practical contribution. The decision not to backprop through the ANN (fixed function) is reasonable; the gradient flows only through the selected top-K rows. This introduces a biased gradient vs full soft attention, but the paper demonstrates that it works well in practice. The write policy (update recently read and overwrite LRU) is simple and effective, though somewhat heuristic; the paper partly addresses this by also providing a sparse DNC variant with richer allocation.
- Empirics: Speed and memory benchmarks convincingly show orders-of-magnitude gains. On algorithmic tasks, SAM matches or outperforms dense baselines and scales far beyond them via curriculum. On bAbI, SAM/SDNC are competitive without supervision of supporting facts, and SDNC attains strong results. On Omniglot, SAM benefits from much larger capacity.
- Missing analyses: More ablation would strengthen claims: sensitivity to K; effect of approximate vs exact neighbors on accuracy and convergence; comparisons of the two usage metrics; effect of erasing only the LRU slot; and robustness to ANN rebuild frequency. Fairness of comparisons could be discussed more carefully when memory sizes differ (e.g., small-memory dense vs huge-memory sparse under equal physical memory budgets).

Clarity
The paper is generally well written and organized. Core mechanisms are clearly specified; however, several points would benefit from more detail in the main text:
- Explicit pseudocode or schematic for the rollback BPTT and the memory-usage data structure.
- Clear amortized complexity accounting for periodic ANN rebuilds.
- A concise summary (in main text) of hyperparameters (K, thresholds, gates) and their sensitivity.
- Clarify how many read heads are used and if top-K is per head or shared.

Originality
The combination of sparse reads/writes, ANN inside the computation graph (without gradients), and efficient rollback BPTT is, to my knowledge, novel relative to NTM/Memory Networks circa the stated timeframe. The extension to DNC strengthens generality. Prior discrete addressing via RL is acknowledged and provides a useful contrast.

Significance
High. The work addresses a central barrier to scaling MANNs and demonstrates practical training with million-scale memories and very long sequences. The ideas are likely to be reused broadly (indeed, the sparse BPTT mechanism and ANN integration are of independent interest). Results on bAbI and Omniglot indicate utility beyond toy tasks.

Pros for acceptance
- Strong, well-motivated algorithmic contribution that reduces time/space complexity and enables scale.
- Elegant engineering: sparse updates, rollback BPTT, and ANN integration.
- Substantial empirical speed/space gains with competitive or improved accuracy.
- Generality shown via sparse DNC; applicability to other MANNs is credible.
- Addresses an important bottleneck for memory-based models.

Cons and concerns
- Gradient bias from top-K truncation and ANN approximation is not analyzed; limited ablation on K/ANN sensitivity.
- Write policy is heuristic; allocation vs overwrite trade-offs are not deeply studied for noisy, real data.
- Some comparisons use vastly different memory sizes (equalizing wall-clock memory), which can confound interpretability.
- Limited evaluation on large-scale natural corpora; tasks remain mostly synthetic or small-scale.
- Reproducibility depends on supplements; code availability is not stated.

Suggestions for improvement
- Provide ablations: K vs accuracy/speed; exact vs ANN; rebuild frequency; usage metric variants; erase policy variants.
- Analyze the effect of approximation on gradients and convergence (e.g., measure overlap between exact and approximate top-K during training).
- Report sensitivity to gates (alpha, interpolation) and threshold parameters.
- Add experiments on larger text QA or long-context tasks (e.g., long document recall) to demonstrate robustness beyond synthetic settings.
- Clarify amortized costs of ANN rebuilds; consider dynamic data structures designed for online balance (e.g., LSH forests, Mondrian trees) and compare.
- Release code to aid reproducibility.

Recommendation
Accept. The paper presents a clear, impactful advance in scaling differentiable memory with strong empirical and algorithmic contributions, likely to influence subsequent work. My confidence is high, with the main caveat being the need for more ablation and analysis of approximation effects.

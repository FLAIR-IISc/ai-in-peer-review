Summary
The paper proposes Sparse Access Memory (SAM), a memory-augmented neural network (MANN) that scales content-based reading and writing to very large external memories by enforcing sparsity and using efficient data structures. Key ideas: (1) sparse read: compute a top-K content-based attention over memory using an approximate nearest neighbor (ANN) index (randomized k-d trees or LSH) so that reads cost O(log N) rather than O(N); (2) sparse write: restrict writes to a small set of locations (previously read positions and the least-recently-used slot), with constant-time forward/backward cost; (3) efficient BPTT: store only sparse deltas to memory and roll back updates during backpropagation, yielding O(1) memory per time step. The authors argue (Supplement A) that under reasonable assumptions SAM is asymptotically optimal in time and space for content-based memory access. They further show how to apply their sparse scheme to the Differentiable Neural Computer (DNC), yielding an SDNC that is dramatically faster.

Empirically, SAM matches or surpasses dense MANNs (NTM, a dense variant “DAM”) on NTM algorithmic tasks, scales via an exponential curriculum to very long sequences (e.g., >4k steps without truncated BPTT), achieves strong bAbI results (SDNC solves all but one task without supervised memory access), and improves few-shot performance on Omniglot episodes with long sequences. Speed/memory benchmarks show up to three orders-of-magnitude improvements in wall-clock and memory footprint compared to NTM at large N.

Relation to prior work
This builds on NTMs [7], Memory Networks [21,17], and the DNC [8], all of which decouple memory from parameters but use dense soft attention and writes. SAM’s main novelty is to combine top-K sparse access with ANN indices inside the computation graph (no gradients through ANN), and a constant-memory BPTT scheme via in-place sparse diffs. It contrasts with discrete addressing (e.g., RL-NTM [22]) by retaining end-to-end differentiability of the controller and memory values (though not the neighbor selection). The use of ANN (k-d trees, LSH) to accelerate content retrieval is well known in the IR/vision literature, but its integration into a differentiable memory with backprop-compatible sparse updates is new and practically important.

Strengths
- Clear scalability advance: O(log N) read time, constant per-step memory, and orders-of-magnitude empirical speedups and memory savings at N up to 10^6.
- Practical engineering: efficient sparse data structures, rollback BPTT, and ANN maintenance integrated into training.
- Strong empirical coverage: synthetic algorithmic tasks, aggressive curriculum scaling (with generalization to 200k steps on associative recall), bAbI QA (SDNC near-SOTA without supervision), and Omniglot long-episode few-shot.
- Generality: shows how to sparsify DNC (SDNC) with large speed gains while retaining performance.
- Insightful finding: sparse reads/writes can help early learning on some tasks.

Weaknesses and concerns
- Differentiability and gradient bias: top-K selection with no gradients through the selection induces discontinuities and biased gradients. This is acknowledged but not analyzed; a discussion of stability and failure modes would help.
- ANN dependence: k-d trees degrade in high dimensions; LSH performance depends on hash choices. Overheads of periodic rebuilds and practical constants are not fully quantified. An ablation isolating exact top-K vs ANN variants and sensitivity to K would strengthen claims.
- Allocation policy: the LRU-based (least recently accessed) allocation is simple but may thrash when revisiting older information or when recency ≠ usefulness. More analysis or ablations comparing U(1) vs U(2), different thresholds, or learned allocation gates would help.
- Fairness of comparisons: for scaling experiments, SAM uses vastly larger memories than dense baselines (by design). While appropriate to demonstrate scaling, some readers may conflate capacity gains with algorithmic improvements. Please report parameter counts, compute budgets, and, where possible, compare at matched accuracy/compute.
- Reporting: many crucial details are relegated to the supplement (proof sketch of optimality, full bAbI results, implementation details). Bringing key quantitative tables into the main text would improve clarity.
- Real-world tasks: bAbI is synthetic; Omniglot setup differs from common few-shot benchmarks (e.g., 5-way/20-way 1-shot). Additional results on standard QA (e.g., CBT/Children’s Book Test; Hill et al. 2016) or standard few-shot splits would broaden impact.

Quality
- Technical soundness: The sparse computation and BPTT rollback mechanisms are well motivated and appear correctly implemented. Complexity claims are plausible; a more formal theorem (with assumptions on dimensionality, ANN approximation guarantees, rebuild cost) in the main paper would strengthen rigor.
- Empirical support: Benchmarks convincingly demonstrate speed/memory scaling. Task performance is competitive or better. Missing: systematic sensitivity to K, memory size, ANN approximation level, and ablations of write policy.
- Honesty: The paper acknowledges approximations and discusses alternative ANN structures.

Clarity
- Generally well written and organized. The SAM controller, read/write equations, and memory manager are clear. Some notation (e.g., tildes, indices) is dense; a compact algorithm box for forward/backward with data structure operations would aid reproducibility.
- Minor issues: several formatting artifacts (e.g., “⇡”, “2⇥106”) likely from PDF; ensure math is typeset cleanly. Summarize key supplemental results (SDNC bAbI table) in the main text.

Originality
- The combination of sparse attention for both reads and writes, ANN inside a differentiable memory, and constant-memory BPTT is novel and impactful. Related discrete addressing alternatives are cited; positioning versus prior acceleration attempts (if any) is adequate.

Significance
- High. Scaling MANNs is a central bottleneck; SAM provides a practical path to million-slot memories with end-to-end training. The ideas are likely to be adopted and extended (e.g., in modern retrieval-augmented models).

Questions for authors
- Sensitivity: How do performance and stability vary with K, dimensionality M, and ANN approximation parameters? What is the effect of exact top-K vs ANN on convergence and final accuracy?
- Allocation: Can you report ablations comparing U(1) vs U(2), and learned allocation gates (as in DNC) under SAM?
- ANN overhead: What fraction of runtime is spent in ANN queries/updates, and how does periodic rebuild affect throughput?
- Failure modes: Do you observe instabilities due to discontinuous top-K selection (e.g., chattering)? Any mitigation (e.g., temperature schedules, noise)?

Pros for acceptance
- Strong, general, and well-executed contribution that materially advances scalability of MANNs.
- Convincing empirical results across diverse tasks.
- Clear practical utility; likely to influence future architectures.

Cons for acceptance
- Limited theoretical analysis of gradient bias and ANN approximations.
- Some comparisons and reporting could be more thorough; reliance on supplement.

Recommendation
Accept (clear accept). This is a substantial, timely contribution with strong empirical evidence and practical significance. With minor clarifications and ablations, it will make an excellent NIPS paper.

Reviewer confidence: High.

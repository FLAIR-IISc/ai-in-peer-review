Summary
The paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that enforces sparsity in both reads and writes to external memory. Reads are restricted to the top-K nearest memory slots using an approximate nearest-neighbor (ANN) index (randomized k-d trees or LSH), and writes are limited to previously read locations and a least-recently-accessed (LRA) location tracked by a custom usage structure. Using in-place sparse updates and rollback during backpropagation, the authors claim O(log N) time per step and O(1) space per time step (given fixed K), which they argue is asymptotically optimal for content-addressed architectures. Experiments include speed/memory benchmarks vs. NTM and a dense variant (DAM), algorithmic tasks (copy, associative recall, priority sort), curriculum scaling to millions of memory slots and thousands of time steps, bAbI QA, and Omniglot one-shot classification. A sparse version of the Differentiable Neural Computer (SDNC) is described in the supplement.

Relation to prior work
This extends the line of differentiable memory architectures (NTM, Memory Networks, DNC), and is related to discrete addressing via RL (RL-NTM). The idea of top-K attention has appeared before, but embedding a dynamic ANN index inside the training loop and coupling it with sparse writes/LRU allocation is novel. The memory-usage mechanism echoes DNC’s usage/temporal link structures, but is simplified.

Strengths
- Clear motivation and a concrete systems-oriented design addressing known scalability bottlenecks in MANNs.
- Algorithmic contributions are practical: top-K sparse reads with ANN, LRA-based writes, and an in-place rollback scheme that reduces BPTT memory cost.
- Substantial speed and memory savings shown empirically; the gap versus dense NTM is large.
- On synthetic tasks, sparse models learn as well as or better than dense ones; curriculum scaling demonstrates very long sequences and large memories.
- The approach appears generalizable (SDNC in supplement).

Weaknesses
- Empirical validation is largely on synthetic or toy tasks (NTM tasks, bAbI); only Omniglot is “real,” and the protocol differs from prior art (e.g., one-hot labels and longer curriculum), limiting comparability. No evaluation on stronger, large-scale, real-world long-context tasks (e.g., language modeling with long dependencies, document QA).
- Baseline comparisons are uneven. For curriculum experiments, sparse models use up to 2e6 memory slots while dense baselines are capped at 64; “equal physical memory usage” is not an equal modeling budget. The improved performance may reflect vastly larger capacity, not just algorithmic efficiency.
- Limited ablations. There is no systematic study of K (sparsity level), the effect of approximate vs. exact nearest neighbors on learning and final accuracy, or the sensitivity to the usage threshold ε and ANN rebuild frequency. These are central to both correctness and performance.
- Theoretical claims (asymptotic optimality) rely on assumptions that should be made explicit and defended; the constants and rebuild overhead for the ANN could dominate in practice. The O(1) per-step memory claim excludes the need to store indices and activations needed for BPTT; while still O(T), explicit accounting would help.
- The top-K selection is non-differentiable; gradients are zeroed for N−K slots at each step. The paper asserts negligible impact but does not quantify gradient bias or provide diagnostics (e.g., training stability across seeds, variance of gradients).
- Speed comparisons may be confounded by implementation choices (custom sparse datastructures vs. potentially less optimized NTM baselines). Hardware/software details and code availability are essential for reproducibility.
- The SDNC results, which are strong on bAbI, are relegated to the supplement; details of training settings and comparisons to published DNC results are not provided in the main text.

Quality
The system design is thoughtful and technically plausible. However, the empirical section does not yet provide the breadth and rigor I would expect before endorsing the method as a scalable alternative in practice. In particular, fairness of baselines, missing ablations on key design choices, and reliance on synthetic tasks limit the evidential strength.

Clarity
Generally well-written and easy to follow. Some core claims (asymptotic analysis, memory rollback algorithm, data structures for LRA tracking) are deferred to the supplement; key components would benefit from concise pseudo-code and precise accounting of training-time memory footprints (what is stored per time step). A few notation typos appear, but nothing blocking.

Originality
Moderately novel. Sparsifying reads/writes and using top-K addressing are known ideas, but integrating an ANN index and a constant-time rollback mechanism into end-to-end training of MANNs is a useful engineering advance. The LRA write policy is a simplified alternative to DNC’s usage allocator.

Significance
Potentially significant if validated: scaling MANNs to millions of slots could open new applications. However, the current experiments do not convincingly demonstrate benefits on challenging, real-world problems, and the fairness of comparisons is a concern.

Questions for the authors
- Provide ablations on K, ANN type (k-d trees vs. LSH), rebuild frequency, and the usage threshold ε, with accuracy/speed trade-offs.
- Report fairness-controlled comparisons: fix total parameter count and equalize effective memory capacity (e.g., limit SAM’s usable memory to match dense baselines) to isolate algorithmic gains.
- Quantify the impact of approximate vs. exact retrieval on learning dynamics and final performance.
- Clarify the exact per-step memory saved during BPTT (what tensors/indices are stored), and provide wall-clock and memory usage with full environment details. Will you release code?
- Evaluate on at least one large, real task requiring long memory (e.g., long-document QA, copy/algorithmic tasks on real text streams, or long-context language modeling) and compare to strong contemporary baselines (DNC, End-to-End Memory Networks with comparable supervision, Santoro MANN).
- Analyze training stability across seeds given the non-differentiable top-K gating.

Arguments for acceptance
- Addresses an important scalability bottleneck with a practical, general approach.
- Strong computational savings; promising results on synthetic tasks; plausible path to broader impact.

Arguments against acceptance
- Insufficiently rigorous empirical validation on real tasks; weak/uneven baselines.
- Missing ablations and sensitivity analyses on core design choices.
- Theoretical optimality and O(1) memory claims need clearer assumptions and accounting.
- Reproducibility concerns (implementation-dependence, lack of code).

Recommendation
Borderline reject. The paper proposes a promising and potentially impactful approach, but the evidence presented does not yet meet the level of thoroughness and rigorous validation I expect for endorsing a new memory architecture. Strengthening the empirical evaluation and ablations would likely change my decision.

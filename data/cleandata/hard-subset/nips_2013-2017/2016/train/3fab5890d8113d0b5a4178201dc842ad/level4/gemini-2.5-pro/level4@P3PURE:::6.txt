This paper presents a Neural Turing Machine (NTM) with sparse read and write access. This approach aims to overcome a major limitation of NTMs that hinders their practical application: the prohibitively large memory and computational requirements for backpropagation. The proposed architecture is designed to solve this issue while achieving performance comparable to state-of-the-art methods.

The paper is well-structured and easy to read. To improve its self-containment, it would be beneficial to include a brief description of memory-augmented neural networks in the background section. This would help readers who are not experts in the field. For instance, explaining the intended function of the write operator in contrast to conventional memory systems would provide valuable context that may not be immediately clear to a broader audience.

Furthermore, the experimental comparison in Figure 4 may not be entirely fair. The authors compare models by normalizing for the same physical memory footprint. While this is a justifiable approach for evaluating performance under resource constraints, a comparison using the same number of memory locations—rather than the same physical memory size—would be more informative. Such a comparison would better isolate the performance impact of the proposed sparse access mechanism.

Nevertheless, it is impressive to see that the proposed algorithm performs so well and so close to the theoretical bounds.

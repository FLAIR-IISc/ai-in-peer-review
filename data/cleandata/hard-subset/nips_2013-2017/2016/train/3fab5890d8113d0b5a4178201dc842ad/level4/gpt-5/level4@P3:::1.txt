Summary and relation to prior work
This paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that scales reads and writes to very large external memories by enforcing sparsity and using approximate nearest neighbor (ANN) indexing. Reads are computed by selecting the top-K memory locations by content similarity (via kd-trees or LSH) and renormalizing over those K entries; writes are restricted to a small set of locations (previously read slots and a least-recently-used slot), enabling constant-time/space updates per step and O(log N) query time. The authors also describe an efficient backpropagation-through-time scheme that stores and reverts only sparse in-place memory modifications, yielding O(1) space per time step and O(N) initialization. They argue these operations are asymptotically optimal under content-based access, and show how the approach extends to the Differentiable Neural Computer (DNC).

Empirically, SAM trains much faster and with much lower memory footprint than dense NTMs and a dense SAM variant, while matching or slightly improving data efficiency on standard NTM tasks, scaling via curriculum to sequences with thousands of steps and millions of memory slots, performing competitively on bAbI without supervised memory access, and improving one-shot Omniglot performance.

Relative to prior work, content-addressable memories with large-scale, sublinear retrieval have been explored in Memory Networks and related systems, including variants that use hashing and restrict attention to a small subset of memory items. The novelty here is primarily in making sparse reads and, crucially, sparse writes work in a fully trainable, writeable memory with efficient BPTT and memory management. The engineering that enables end-to-end training at scale is the core contribution.

Quality
- The core ideas are sound and well-motivated. The sparse write mechanism (previously read + least recently used) is simple and effective, and the in-place sparse rollback for BPTT is a valuable technical contribution.
- The experimental suite is broad and demonstrates large speed and memory savings without apparent loss in data efficiency. Extending the approach to the DNC and showing strong bAbI results is compelling.
- However, reproducibility is a concern. Many crucial details appear only briefly or in the supplement (task generators, curriculum schedules, ANN settings, thresholds, rebuild frequencies, precise memory-management data structures). Given the sensitivity of these systems to implementation choices, more detail or code is important.

Clarity
- The paper is generally well written, but several descriptions (especially task construction, training curricula, ANN configurations, and memory-management details) are too terse to allow replication. The claim of “end-to-end differentiable” access should be nuanced: the top-K selection is a non-differentiable discrete operation; gradients are passed only through the selected entries, which is a common and often effective approximation, but not strictly differentiable.
- Consider removing a non-essential section (e.g., the current Section 3.6, if present) to free space for experimental and implementation details that enable reproduction.
- Avoid describing Omniglot as “real-world” or “non-synthetic”; it is a useful benchmark but far from unconstrained real data.

Originality
- Sparse content-based retrieval with sublinear search has clear precedents in Memory Networks and related work employing hashing and large memories. The paper’s distinct contributions are (a) making sparse writes work with a learnable, writeable memory and simple allocation; (b) an efficient, general BPTT strategy for large external memories; and (c) demonstrating practical scaling to millions of slots and very long sequences. The combination is novel and practically important, even if individual components build on known ideas.

Significance
- If widely adopted, the methods could enable MANNs to be applied to substantially larger problems in language, retrieval, and program-like tasks. The measured 100–1600x speedups and orders-of-magnitude lower memory footprint are compelling. The extension to DNC further increases impact. This is a strong systems contribution for the MANN literature.

Suggestions and questions for the authors
- Reproducibility: Please provide precise task generators, data preprocessing, curriculum schedules, hyperparameters, and ANN settings (index type, parameters, rebuild cadence). Releasing code, including the sparse BPTT and memory-management implementation, would materially strengthen the contribution.
- Failure modes and ablations: Analyze when sparse selection harms learning. Vary K (e.g., 1, 2, 4, 8), compare exact vs ANN search, and assess the effect of ANN recall on performance. Report sensitivity to the usage threshold and allocation scheme (LRU vs discounted usage). Show tasks where K=1 fails relative to soft attention, and clarify whether failures are due to K, the task, or the discrete selection itself.
- Differentiability and optimization: Clarify the training implications of the non-differentiable top-K step, especially around boundary cases where the selected set changes. A brief discussion of the induced bias and why it does not impede learning in your settings would be helpful. Contrast with discrete-attention approaches trained with REINFORCE or self-supervision.
- Novelty positioning: More explicitly contrast SAM with prior large-memory MemNN variants using hashing or restricted attention, and cite relevant works. Emphasize that the key advance is enabling efficient writes and BPTT at scale.
- Claims and terminology: Temper “end-to-end differentiable” and “real-world” wording as noted above.

Arguments for acceptance
- Strong practical advance: orders-of-magnitude speed and memory improvements with convincing experiments.
- Clear path to scaling writeable memories; simple, effective allocation and BPTT strategy.
- Broad applicability, including to DNC, and solid performance on diverse benchmarks.

Arguments against acceptance
- Novelty is primarily engineering; sparse retrieval itself is not new.
- Incomplete experimental detail and lack of code hinder reproducibility.
- Limited analysis of failure cases and sensitivity to K and ANN approximations; differentiability claims need nuance.

Recommendation
Accept, contingent on clarifying experimental details and, ideally, releasing code. The paper offers a valuable and timely systems contribution that materially advances the scalability of memory-augmented neural networks.

The paper proposes a Neural Turing Machine (NTM)-style architecture with sparse read and write access to address a key limitation of NTMs that impedes real-world deployment. Specifically, NTMs demand prohibitively large physical memory and costly operations for backpropagation through time; the presented architecture seeks to alleviate this while achieving performance comparable to the state of the art. The paper is well structured and easy to follow. Providing a brief overview of memory-augmented neural networks in the background section, or a concise explanation of the external memory mechanism, would make the paper more self-contained for readers who are not experts (e.g., it may be unclear to some what the write operator is intended to do relative to conventional memories). Additionally, the comparison in Figure 4 may not be entirely equitable. Selecting memory sizes so that models use the same amount of physical memory compares performance under a fixed resource budget—which is defensible—but using the same memory capacity rather than the same physical memory would be more informative about intrinsic performance. Nevertheless, it is compelling to see how well this approach performs while remaining close to theoretical bounds.

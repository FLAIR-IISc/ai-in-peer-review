Summary
This paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that performs sparse reads and writes to an external memory while remaining fully differentiable. Building on Neural Turing Machines (NTMs) and Memory Networks, the authors introduce two key ideas: (1) sparse content-based reads implemented via approximate nearest neighbor (ANN) indexing (randomized k-d trees or LSH) to select K memory locations in O(log N) time, and (2) sparse writes that target either the most recently read locations or the least recently used slot, tracked by a usage statistic. They further present an efficient backpropagation-through-time scheme that stores and reverts only sparse deltas, reducing space from O(NT) to O(T). Under reasonable assumptions (in the supplement), SAM achieves asymptotically optimal time and space with respect to memory size. Empirically, SAM runs orders of magnitude faster and with far less physical memory than dense models, while matching or improving data efficiency on NTM synthetic tasks, scaling to very long sequences via curriculum training, performing competitively on bAbI, and achieving strong one-shot learning on Omniglot. The approach generalizes to the Differentiable Neural Computer (SDNC), yielding large speedups and strong bAbI performance without supervised memory access.

Quality
- The technical design is sound and well-motivated. The sparse read via ANN and the least-recently-used write policy are simple, effective, and differentiable.
- The complexity analysis and the in-place, reversible memory updates for BPTT are convincing and practically important. It would help to make the “optimality” assumptions explicit in the main text and to clarify limits when addressing by content might not be sufficient.
- Empirical validation is thorough: speed/memory benchmarks, standard NTM tasks, curriculum scaling to thousands of steps, bAbI, and Omniglot. Results generally support the claims.
- One concern is the fairness and interpretability of some comparisons: several experiments match physical memory budgets rather than the number of memory slots. Including both protocols (equal slots and equal compute/memory budgets) would better characterize algorithmic generalization vs. engineering trade-offs.
- Additional ablations (e.g., sensitivity to K, choice of ANN, rebuild frequency, threshold for usage) would strengthen the empirical story.

Clarity
- The paper is clearly written and well organized. Equations and the control flow are easy to follow.
- To make the paper more self-contained for non-experts, consider adding a brief primer on memory-augmented neural networks in the background (e.g., intuition for erase/add, differences from conventional memory semantics) and a succinct summary of NTM/DNC write mechanisms.
- Clarify the precise backpropagation interface to the ANN index (no gradients) and how index rebuilds interact with training.
- Minor: unify notation around sparse vs. dense weights and ensure consistent naming of hyperparameters (e.g., K, usage threshold).

Originality
- The integration of ANN search inside a differentiable memory system, combined with sparse writes and reversible sparse-delta BPTT, is a notable and practical contribution. It contrasts with prior discrete addressing trained via RL (e.g., Zaremba & Sutskever), offering lower-variance gradients and strong empirical performance.
- Extending the approach to the DNC further underscores generality.

Significance
- This work addresses a central barrier to scaling memory-augmented models and demonstrates substantial speed and memory savings without sacrificing performance.
- The ability to train with millions of memory slots and thousands of time steps is impactful for long-context tasks (e.g., language modeling, retrieval-augmented reasoning).
- The method is likely to be used and extended by practitioners; it meaningfully advances the state of the art in scalable differentiable memory.

Strengths
- Elegant sparse read/write design with end-to-end differentiability.
- Strong asymptotic analysis and practical engineering (sparse deltas for BPTT).
- Large empirical speedups (up to 1600×) and memory reductions (thousands×).
- Broad and relevant evaluation (NTM tasks, long-sequence curricula, bAbI, Omniglot).
- Generalizes to DNC with strong results.

Weaknesses and suggestions
- Comparisons: Report results under both equal memory slots and equal physical memory budgets to disentangle algorithmic generalization from resource scaling.
- Ablations: Provide sensitivity analyses for K, ANN choice (k-d tree vs LSH), rebuild frequency, and the usage threshold; quantify the impact of ANN approximation on accuracy and training stability.
- Theory: Move a concise statement of the “optimality” assumptions from the supplement to the main text.
- Clarity: Add a brief background on MANNs and an intuitive explanation of the write/erase operations; clarify handling of ties in least-recently-used selection and the effect of collisions when writes target recently read slots.
- Reproducibility: Summarize key hyperparameters and training settings in the main paper; consider releasing code or pseudocode for the sparse BPTT mechanism.

Questions for the authors
- How does performance vary with K, and can K be learned or adapted during training?
- What is the trade-off between ANN recall and task accuracy? How sensitive are results to the choice of index and rebuild schedule?
- Does the sparse gradient flow (touching only K rows) introduce bias that affects learning on tasks requiring broad credit assignment?
- Have you compared against discrete addressing policies trained with RL under matched memory slots and budgets?
- How robust is the least-recently-used policy when the task requires long-term retention of rarely accessed items?

Pros for acceptance
- Addresses an important scalability problem with a principled, effective solution.
- Strong empirical evidence across diverse tasks with dramatic efficiency gains.
- Clear exposition and broadly applicable methodology (including DNC).

Cons for acceptance
- Some comparisons conflate resource budgets with memory capacity; more comprehensive ablations would strengthen the case.
- “Optimality” claims rely on assumptions relegated to the supplement.

Overall, this is a high-quality, original, and significant contribution that substantially advances scalable differentiable memory. I recommend acceptance, with the above clarifications and additional analyses to improve fairness and reproducibility.

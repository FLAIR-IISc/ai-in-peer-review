- Problem addressed: MANNs like NTMs scale poorly because dense read/write over all memory slots causes O(N) time and space per timestep.  
- Proposed solution: Sparse Access Memory (SAM) that reads/writes only a small constant-sized subset of memory slots.  
- Key technical ideas: sparse reads via an ANN index (k-d trees/LSH) to find top-K slots and sparse writes that update recently read or least-used slots.  
- Complexity improvements: reduces per-timestep time from O(N) to O(log N) and space from O(N) to O(1) using an efficient sparse backpropagation ("rollback") scheme.  
- Empirical gains: orders-of-magnitude speed and memory improvements while matching or exceeding dense models on algorithmic tasks (Copy, Associative Recall, Priority Sort).  
- Scalability and generalization: curriculum learning enables handling sequences up to 200,000 steps, showing far better scalability than dense models.  
- Real-world tasks: strong performance on bAbI and one-shot Omniglot; extension to Sparse DNC demonstrates generality.  
- Clarity: paper is well-written and structured, but may assume expert background in MANNs.  
- Weakness — accessibility: add a brief background on soft/differentiable writes and why MANNs use weighted averages to help non-expert readers.  
- Weakness — experimental fairness: comparisons conflate sparsity benefits with larger physical memory; include control experiments with equal N (e.g., N=128) to isolate the effect of sparse access.

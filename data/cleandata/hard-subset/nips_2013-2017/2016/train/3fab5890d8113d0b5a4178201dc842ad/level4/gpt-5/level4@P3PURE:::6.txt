This paper presents a Neural Turing Machine (NTM) with sparse read and write access to address a key limitation that has hindered real-world applicability: NTMs typically require prohibitively large physical memories and expensive operations during backpropagation. The proposed architecture aims to mitigate these costs while maintaining performance comparable to the state of the art. The paper is well structured and easy to read.

To make the work more self-contained for readers who are not experts, it would help to include a brief background on memory-augmented neural networks and a concise explanation of the memory component itself. In particular, clarifying what the write operator is intended to do—especially how it differs from writes in conventional memories—would improve accessibility.

I have a concern about the fairness of the comparison in Figure 4. Equalizing the physical memory footprint across models is a reasonable way to compare performance under a fixed resource budget, and this choice is justifiable. However, a comparison that holds the same logical memory (e.g., the same memory size/capacity) rather than the same physical memory footprint would be more informative about the intrinsic performance of the approaches.

Despite this, it is very interesting to see how well the algorithm performs while remaining so close to theoretical bounds.

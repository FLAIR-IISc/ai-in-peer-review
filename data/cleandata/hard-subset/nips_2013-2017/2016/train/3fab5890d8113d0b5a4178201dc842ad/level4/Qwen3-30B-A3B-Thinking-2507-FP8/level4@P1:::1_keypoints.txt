- Introduces Sparse Access Memory (SAM): scalable MANN using sparse read/write and ANN indexing for asymptotic space/time efficiency.  
- Claims SAM matches dense model performance while reducing memory usage ~3,000× and accelerating training ~1,000× for large memories (e.g., 64k words).  
- Evaluated on synthetic tasks (Copy, Associative Recall, Priority Sort), Babi QA, and Omniglot one-shot; SDNC achieves state-of-the-art on Babi without supervised memory access.  
- Extends prior MANN work (e.g., NTMs, Memory Networks) by enabling writable sparse memories to address deployment scalability bottlenecks.  

Strengths:  
- Strong practical contribution that effectively solves a major MANN scalability problem.  
- Robust empirical results: consistent speed/memory improvements while matching or exceeding task performance.  
- Demonstrates generality (SDNC) and extreme scalability via curriculum learning to >200k steps and millions of memory words.  
- Provides rigorous theoretical analysis of asymptotic optimality and well-motivated implementation details (sparse backprop tracking).  

Weaknesses:  
- Novelty is modest given prior work on hashing/sparse read access; writable extension is incremental.  
- Insufficient details for replication (task specs, memory initialization, exact hyperparameters).  
- Non-differentiability/top-K approximation lacks deeper analysis explaining why it succeeds here versus prior failures.  
- No systematic failure-case analysis (e.g., tasks needing precise updates or long-term dependencies).  
- Mislabels Omniglot as "real-world"; terminology should be corrected.  
- Section 3.6 is redundant and should be removed.  

Recommendation:  
- Accept with minor revisions focused on reproducibility and additional analysis.  

Pros for acceptance (concise):  
- Massive scalability gains (1,000× speedup, 3,000× memory reduction).  
- Scales to unprecedented memory/sequence sizes and achieves SOTA on Babi without supervised access.  
- Theoretical support for asymptotic optimality.  

Cons for acceptance (concise):  
- Limited conceptual novelty versus prior sparse-read work.  
- Paper lacks replication-ready details and failure analysis; Omniglot characterization is misleading.  

Required revisions (concise):  
- Provide explicit task specifications (sequence ranges, memory initialization) for all benchmarks.  
- Commit to releasing code and complete hyperparameter settings.  
- Add discussion/analysis of failure cases and why top-K approximations work here but failed in prior work.  
- Replace "non-synthetic"/"real-world" phrasing for Omniglot with "natural" or "realistic."  
- Remove redundant Section 3.6.

- Introduces Sparse Access Memory (SAM), a MANN architecture using sparse read/write and ANN structures to scale external memory efficiently.  
- Achieves asymptotic optimality: O(log N) time, O(N) initialization, O(1) per-step space while remaining end-to-end differentiable.  
- Demonstrates major empirical gains: ~1,000× speedup and ~3,000× memory reduction over dense baselines.  
- Scales to very long sequences (100,000+ steps) and generalizes to sequences 20× longer than training (e.g., 10k → 200k).  
- Matches or exceeds prior MANN performance on benchmarks, including state-of-the-art Babi results (SDNC solves 19/20 tasks).  
- Core technical design: sparse K-most-relevant memory access (K≈4–8) via ANN nearest-neighbor search and a usage counter to manage sparse writes without breaking gradients.  
- Provides rigorous validation: asymptotic proofs, comprehensive synthetic and real-world benchmarks (Copy, Associative Recall, Priority Sort, Omniglot, Babi).  
- SDNC extension shows generality to DNC-like architectures.  
- Novelty: first differentiable MANN to resolve content-addressing scalability with provable asymptotic improvements, distinct from RL-based sparse addressing.  
- Significance: makes MANNs practically viable for long-context tasks and large memories that were previously infeasible.  
- Clarity: well-organized exposition, clear math/algorithms, useful figures, and thorough supplementary implementation details.  
- Minor clarity suggestions: explain why LSH outperforms k-d trees for large word sizes and report absolute memory overheads (e.g., initialization sizes).  
- Weakness: insufficient analysis of why SAM generalizes so strongly to very long sequences; lacks ablations (e.g., varying K) and deeper comparison of internal representations to dense models.  
- Additional weakness: limited comparison to non-NTM MANNs (e.g., End-to-End Memory Networks) beyond Babi tasks.  
- Recommendation: strong accept; suggest expanding Section 5 with brief empirical/ablation evidence about the generalization mechanism.

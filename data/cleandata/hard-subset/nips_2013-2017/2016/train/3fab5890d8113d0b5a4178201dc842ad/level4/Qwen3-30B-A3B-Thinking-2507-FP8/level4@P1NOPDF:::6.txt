

The paper introduces a Neural Turing Machine (NTM) with sparse read and write operations to address a key limitation of NTMs: their prohibitive memory and computational demands during backpropagation, which hinder real-world applicability. The proposed architecture aims to achieve performance comparable to state-of-the-art methods while significantly reducing resource requirements. The paper is well-structured and clearly written. However, including a brief introduction to memory-augmented neural networks in the background section would enhance self-containment, particularly for readers unfamiliar with the field. For instance, clarifying the intended functionality of the write operator relative to conventional memory systems (which may not be immediately apparent to non-experts) would improve accessibility without altering the technical contribution.  

Regarding Figure 4’s comparison, while the authors’ approach—comparing models under identical physical memory constraints—is justified for hardware-aware evaluation, a comparison based on fixed memory capacity (e.g., same number of memory slots) rather than identical physical memory might offer more insightful performance insights. Specifically, it would better reveal how effectively the algorithm utilizes memory resources, independent of hardware implementation details. Nevertheless, the results demonstrate that the proposed method achieves strong performance while operating near theoretical efficiency bounds, which is a compelling contribution. The experimental validation is thorough, and the core technical argument—that sparse access enables practical scalability—is well-supported. The work represents a meaningful step toward making NTMs viable for real-world deployment, and the paper effectively communicates this advancement.

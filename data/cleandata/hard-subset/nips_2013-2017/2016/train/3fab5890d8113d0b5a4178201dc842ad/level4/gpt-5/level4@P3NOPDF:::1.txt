Summary
The paper proposes computing attention in a Neural Turing Machine (NTM) using fast approximate nearest-neighbor (ANN) structures and arg top-K selection. This yields sparse reads and writes that reduce computation and memory bandwidth. Empirically, the approach maintains training quality and enables scaling to larger problems.

Novelty and relation to prior work
The conceptual novelty is limited: prior memory-network work has already used scalable retrieval over millions of memory slots with hashing/ANN methods (e.g., Section 3.3 of the Memory Networks paper, arXiv:1410.3916), and subsequent papers have explored similar ideas. However, applying ANN and hard top-K selection to a writeable external memory is, to my knowledge, new and practically nontrivial. Even if the idea is incremental, demonstrating that it can be made to work reliably in this setting is valuable.

Strengths
- Clear motivation: dense attention is a bottleneck for large memories; sparse ANN-based attention directly addresses this.
- Practical contribution: engineering details for scalable reads/writes are useful to the community.
- Empirical finding: training quality is not degraded and larger problem instances become feasible.

Major concerns and requested revisions
1) Reproducibility and task specification
- The paper, as written, is difficult to replicate. The task descriptions are especially underspecified, which is problematic given that the original NTM task code is not public and there is no standard version of these tasks.
- Please provide precise, standalone specifications for every task: sequence lengths, alphabet/vocabulary, data generation procedures, noise/corruption if any, train/test splits, curriculum, and evaluation metrics.
- Report all training details: model architecture (controller type and size), memory size and data type, K values, ANN index type and parameters (e.g., number of probes/trees, refresh schedule), optimizer, learning rates, batch sizes, gradient clipping, initialization, training durations, and random seeds.
- I strongly encourage releasing the full code and scripts used to generate data and run experiments.

2) Understanding limits and failure modes
- The paper often emphasizes “smoothness,” yet the argmax/arg top-K operations are not smooth. With K=1 in read-only settings, the model resembles MemNN-WSH (arXiv:1503.08895, though that variant does not use hashing), which was reported to underperform fully smooth attention. Why does this work here?
- Please analyze when and why discrete selection succeeds or fails. Concretely:
  - Vary K (e.g., 1, 4, 16) and report performance, stability, and convergence speed.
  - Compare hard top-K to soft top-K/temperature-controlled softmax or straight-through relaxations, if applicable.
  - Show the effect of using exact vs approximate neighbors.
  - Provide at least one negative result or stress test where training fails or degrades (e.g., highly ambiguous keys, rapidly drifting embeddings, adversarial collisions).
  - Discuss the role of task structure: which properties (e.g., deterministic addressing, low aliasing, local write locality) make discrete selection viable?
  - Contrast with approaches that explicitly account for discrete choices during training (e.g., arXiv:1511.07275); explain differences in assumptions and outcomes.

3) Read/write mechanics with ANN
- Clarify how write operations interact with the ANN index:
  - How are keys defined and updated for written slots?
  - How frequently is the index rebuilt or incrementally updated, and what is the amortized cost?
  - How do you handle staleness of keys after gradient updates?
  - What are the complexity and latency for read and write at training and inference time?
- Discuss collision handling and tie-breaking in top-K selection, and how gradients are routed when multiple keys are near-duplicates.

4) Evaluation breadth and fairness
- Provide comparisons to fully differentiable dense-attention baselines (NTM or related memory-augmented networks) under matched compute and memory budgets.
- Include wall-clock timing, peak memory, and scaling curves vs memory size to substantiate the claimed efficiency gains.
- Offer ablations isolating the contributions of sparsity vs approximation: dense exact, dense approximate, sparse exact, sparse approximate (where feasible).

5) Clarity and presentation
- Add a concise algorithmic description or pseudocode for the sparse attention mechanism, covering key lookup, top-K selection, gradient handling for non-selected entries (zeroed vs relaxed), and index maintenance.
- Define all symbols and operations precisely (e.g., “arg top-K”), and state whether any straight-through or surrogate gradients are used.

6) Dataset terminology and scope
- Please avoid calling Omniglot “non-synthetic” or “real-world.” A neutral phrasing such as “image-based benchmark” is more accurate.
- I also suggest removing Section 3.6, which does not add clear value relative to the rest of the paper.

Minor comments
- Expand related work on large-memory retrieval with hashing/ANN, and clearly position your contribution with respect to read-only vs writeable memories.
- Provide intuition for why discrete writes may be robust in your tasks (e.g., sparse addressing reduces interference).
- Include sensitivity to hyperparameters and random seeds to assess stability.

Recommendation
Accept, contingent on revisions that substantially improve reproducibility and provide analysis of when discrete, ANN-based top-K attention works and when it does not. The conceptual advance is incremental relative to prior memory-network literature, but applying these ideas to writeable memories and demonstrating practical scalability is worthwhile. With clearer task definitions, code release, and a deeper examination of failure modes and ablations (particularly varying K and ANN parameters), the paper would make a solid and useful contribution.

- Proposes Sparse Access Memory (SAM): sparse reads/writes + ANN top-K lookup + sparse BPTT rollback to scale writable external memory with O(log N) ops.
- Builds on Memory Networks/ANN hashing for reads; novelty lies mainly in applying sparsity to writes and engineering efficient end-to-end BPTT.

Strengths:
- Demonstrates large empirical speedups (up to 1600×) and memory savings (≈3000×) versus dense NTMs at scale.
- Sparse access matches or improves learning on synthetic tasks, scales to very long sequences, and yields strong bAbI and Omniglot results (SDNC variant especially good).
- Presents practical memory management (least-recently-accessed allocation) and an efficient sparse rollback for BPTT.

Major weaknesses / requested changes:
- Reproducibility: Tasks and NTM code lack sufficient detail; must fully specify task distributions, generation, curricula, stopping/evaluation and commit to releasing code.
- Analyze failure modes: quantify effect of K (e.g., K=1 vs K>1) on stability/performance and identify tasks where hard top-K hurts versus smooth attention.
- Compare to discrete-addressing approaches: discuss when policy-gradient methods are needed and include sensitivity analyses/ablations (allocation, erase policy, ANN type, thresholds).
- Prior work clarity: explicitly position relation to Memory Networks and clarify novelty as writable sparse memory + sparse BPTT.
- Implementation clarity: provide exact ANN configurations, rebuild schedules, sparse backprop data structures, rollback details, memory-management constants, and full training hyperparameters/curricula.
- Editorial: consider removing or better integrating Section 3.6; avoid calling Omniglot “non-synthetic/real-world.”

Assessment by criteria:
- Quality: Technically sound and well-motivated; empirical efficiency compelling but needs reproducibility and failure-mode analysis.
- Clarity: Well-written overall but missing crucial experimental/implementation specifics.
- Originality: Moderate — ANN-based sparse reads have precedent; main novelty is writable sparse memory and BPTT engineering.
- Significance: High practical impact for scalable MANNs.

Pros for acceptance:
- Strong practical improvements in speed/memory and scalability.
- Shows sparse access can preserve learning and enable long-horizon training.
- Generalizable approach applicable to DNC and other MANNs.

Cons for acceptance:
- Limited novelty on read-side relative to prior Memory Network work.
- Insufficient methodological and task details for reproducibility.
- Limited analysis on when discrete sparsification degrades performance.

Recommendation:
- Accept conditional on adding detailed task/implementation descriptions, releasing code, clarifying novelty vs prior work, and providing analyses of failure modes and the role of K/non-smooth choices.

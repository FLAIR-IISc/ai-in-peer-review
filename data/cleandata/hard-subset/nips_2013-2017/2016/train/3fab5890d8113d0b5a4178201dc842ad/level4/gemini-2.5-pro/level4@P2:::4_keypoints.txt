- Problem: Dense read/write operations in MANNs (e.g., NTM) cause prohibitive O(N) compute and O(NT) BPTT memory, limiting large-memory applications.  
- Proposal: Sparse Access Memory (SAM) replaces dense operations with sparse reads/writes touching a constant number of memory locations per step.  
- Sparse reads: Use ANN structures (k-d trees/LSH) to retrieve K nearest neighbors in O(log N) and read as a weighted average over K words.  
- Sparse writes: Update previously read locations or overwrite least-recently used slots, enabling sparse modifications only.  
- Efficient BPTT: Record O(1) sparse modifications per timestep to reduce per-timestep space from O(N) to O(1) and roll back memory during backward pass.  
- Theoretical gains: Time complexity reduced to O(log N) and per-timestep space to O(1), a fundamental improvement over dense baselines.  
- Empirical gains: Demonstrated speedups >1000x and memory reductions >3000x for large memories; trained on sequences up to 200,000 steps.  
- Experimental rigor: Thorough benchmarks across algorithmic tasks, QA, and one-shot learning, including Dense Access Memory (DAM) as a control.  
- Originality: Novel integration of ANN indices into an end-to-end differentiable memory and a clever memory rollback technique for BPTT.  
- Clarity: Paper is well-written, well-motivated, and figures effectively convey improvements (e.g., 29GiB example for 64k memories).  
- Weakness: Reliance on ANN index quality; non-stationary memory updates and index rebuilding could affect performance and need deeper analysis.  
- Recommendation: Strong accept — technically sound, significant, novel, and likely to substantially impact MANN research.

This paper accelerates Neural Turing Machine attention by using fast approximate nearest-neighbor indices and arg-top-K selection. The experiments suggest this does not hurt training performance and, importantly, enables scaling to larger problems. While the conceptual novelty is limited—see, for example, Section 3.3 of the Memory Networks paper (http://arxiv.org/pdf/1410.3916.pdf) and subsequent works that scale to millions of memories using hashing—the application of these tools to writable memories appears less explored. Even without a major conceptual leap, the engineering required to make this work in practice is valuable.

I recommend acceptance, contingent on the following revisions.

1) Reproducibility and task specification
- The paper lacks sufficient detail to reproduce results, especially for the tasks. Since the original NTM task code has not been released and no standard variants exist, precise task construction is essential.
- Please provide clear, complete descriptions, including data generation procedures, sequence lengths and distributions, curricula (if any), noise/perturbations, train/validation/test splits, evaluation metrics, and stopping criteria.
- Document all training details: optimizer, learning-rate schedules, batch sizes, initialization, regularization, gradient clipping, and early-stopping rules.
- Describe the ANN infrastructure and attention computation thoroughly: index type and parameters, update policies and costs for writable memories, top-K selection strategy, any temperature or smoothing, and how randomness is handled (seeds, determinism).
- Commit to releasing code and scripts for data generation, training, and evaluation. Given the reliance on nonstandard tasks and specialized infrastructure, code release is crucial.

2) Analysis of limitations and failure modes
- The paper often characterizes the model as smooth, yet argmax/top-K is not smooth. With K=1 in a read-only setting, the approach resembles MemNN-WSH (http://arxiv.org/pdf/1503.08895.pdf), which—despite not using hashing—was reported to underperform fully smooth models. Please analyze:
  - Sensitivity to K (e.g., K=1 vs larger K) and whether performance improves with more candidates or partial soft attention.
  - Task properties and dataset characteristics that make hard selection succeed or fail (e.g., noise, memory size, write frequency, interference).
  - Training stability and convergence: when does training stall or diverge; what mitigations help (temperature annealing, curriculum, smoothing, or regularization)?
  - Comparisons to methods that explicitly address discreteness during training (e.g., http://arxiv.org/abs/1511.07275). Why does your approach work without such techniques, and where does it not?
- Concrete failure cases and diagnostic metrics (e.g., retrieval precision/recall under approximation; write-read conflicts) would make the contribution more actionable.

Additional suggestions
- Remove Section 3.6; it does not materially strengthen the contribution.
- Avoid calling Omniglot “non-synthetic” or “real-world.” It is a curated benchmark with properties distinct from real-world, large-scale, noisy data.

Minor clarity and completeness requests
- Provide time and memory complexity comparisons to exact attention, including build/update/query costs for the ANN index during training with writes.
- Report wall-clock speedups and throughput on standardized hardware, and quantify accuracy vs. approximation trade-offs (varying index parameters and K).
- Clarify how approximation noise affects gradient estimates and whether approximate retrieval is consistent across steps/epochs (e.g., fixed seeds, index rebuild frequency).
- Include ablations on write policies (e.g., write sparsity, overwrite strategies) and their interaction with approximate indexing.

Overall, despite limited novelty relative to prior scalable memory-lookup work, demonstrating practical, trainable, writable memory with approximate retrieval is useful. With stronger task documentation, code release, and a careful analysis of where the method succeeds and fails, this paper would make a solid contribution.

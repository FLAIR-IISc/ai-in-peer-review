

This paper introduces Sparse Access Memory (SAM), a novel architecture for memory-augmented neural networks (MANNs) that addresses a key limitation of existing models like Neural Turing Machines (NTMs): prohibitive memory and computational overhead as memory size scales. The authors demonstrate that SAM achieves asymptotically optimal time and space complexity (O(log N) per step and O(1) space per step, respectively) while maintaining performance comparable to dense variants on synthetic tasks (e.g., Copy, Associative Recall, Priority Sort), Omniglot one-shot classification, and the Babi question-answering tasks. Notably, SAM reduces memory consumption by 3,000× and accelerates training by 1,000× compared to standard NTMs for large memories (e.g., 64,000 words), with negligible impact on data efficiency. The approach also generalizes to a Sparse Differentiable Neural Computer (SDNC), which achieves state-of-the-art results on Babi tasks without supervised memory access.

The paper is well-structured and clearly written, making a strong case for SAM’s technical contribution. The background section effectively contextualizes MANNs within prior work (NTMs, Memory Networks), and the technical exposition of sparse reads/writes, memory management, and efficient backpropagation is thorough. The authors convincingly argue that sparsity—via approximate nearest neighbors and a least-recently-used memory allocation scheme—enables optimal scaling without sacrificing learning capability, as evidenced by empirical results across multiple benchmarks. The theoretical analysis (Supplementary A) supports the claimed complexity bounds, and the experimental design (e.g., curriculum learning for scaling tasks) is rigorous.

A minor suggestion: While the background covers MANNs, a brief clarification of the *purpose* of the memory write operation (e.g., how it differs from conventional memory updates) would further enhance accessibility for non-specialists. For instance, explicitly stating that writes *update* memory content (via erase/add matrices) rather than simply storing new data would help readers unfamiliar with NTMs grasp the architecture’s novelty.

Regarding Figure 4 (Omniglot results), the comparison by physical memory constraints (rather than raw memory size) is *justified*, as the authors rightly note that real-world deployment is limited by physical memory, not theoretical capacity. The authors’ approach—ensuring all models use comparable physical resources—directly addresses this constraint, making the comparison highly relevant. The fact that SAM outperforms dense models under identical hardware constraints (e.g., 0.2 errors vs. 0.4 with 100 characters) underscores its practical significance.

Strengths:  
- **Quality**: Rigorous theoretical analysis and empirical validation across diverse tasks.  
- **Originality**: Novel sparse access scheme with provable complexity bounds; extends to SDNC.  
- **Significance**: Solves a critical scalability barrier for MANNs, enabling real-world applications (e.g., long-sequence processing).  
- **Clarity**: Well-organized, with clear exposition of technical innovations and strong empirical support.  

Weaknesses:  
- The comparison in Figure 4 is *not* unfair—it appropriately models real-world hardware constraints.  
- The memory write operation’s functional role (beyond mathematical formulation) could be clarified for broader accessibility.  

Overall, SAM represents a substantial advancement in MANN scalability. It directly addresses a major bottleneck in the field while maintaining performance, and the implementation is both theoretically sound and empirically validated. The paper’s contributions to efficiency and scalability are significant, and the results demonstrate clear potential for real-world deployment. This work is a strong candidate for acceptance at NIPS.

This paper presents a Neural Turing Machine (NTM) with sparse read and write access, aiming to overcome a major problem with NTMs that makes them unsuitable for real-world applications. The problem is that NTMs require a large amount of physical memory and operation for backprop, which the architecture presented in this paper hopes to solve with state-of-the-art performance. This paper is well-structured and easy to read. If a brief description of memory augmented neural networks is presented in the background section, or an explanation of memory is provided to make the paper more self-sufficient, it will help readers who are not experts in this field (for example, even if it is clear to those who have worked in this field, it may not be clear to some readers what the write operator should ideally do compared to traditional memory, etc.). Also, the comparison in Figure 4 may not be appropriate. By choosing memory in such a way that they require the same amount of physical memory, you can compare performance given limited resources (which you have done and is certainly a reasonable one!) but I think using the same memory and not using the same physical memory is more informative about how the performance is. Still, it is very interesting to see how well this algorithm performs while being so close to theoretical limits.

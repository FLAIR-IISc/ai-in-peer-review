Summary and relation to prior work:
This paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that achieves scalable, end-to-end differentiable read and write operations by enforcing sparsity and using efficient data structures. SAM constrains both reads and writes to a constant number K of memory locations per step, uses approximate nearest neighbor (ANN) indices (randomized k-d trees or LSH) for content-based addressing in O(log N) time, and performs backpropagation through time (BPTT) with O(1) space per step by recording and reverting sparse in-place updates. The authors show asymptotic lower bounds for time and space, and report empirical speedups up to three orders of magnitude and drastic reductions in physical memory consumption relative to dense Neural Turing Machines (NTMs). SAM learns comparably to dense variants on standard NTM tasks, scales via curriculum to very long sequences and very large memories, performs competitively on bAbI question answering, and improves one-shot Omniglot classification. The approach also extends to the Differentiable Neural Computer (DNC), yielding an SDNC that runs substantially faster.

This work builds directly on Memory Networks and NTMs by preserving content-based differentiability while addressing their scaling limitations. It contrasts with reinforcement learning approaches to discrete memory addressing (e.g., RL-NTM), which avoid dense access but suffer from high-variance gradients. The integration of ANN indices within a differentiable architecture and the BPTT rollback scheme are the key technical novelties.

Strengths:
- Clear and practical scalability: O(log N) per-step time and O(1) per-step space (beyond O(N) init) under reasonable assumptions, with careful engineering to realize these gains.
- Thoughtful design: sparse reads via top-K approximate content retrieval; sparse writes targeting previously read locations or least-recently accessed slots with an efficient usage tracker.
- Strong empirical evidence: large speed/memory savings; comparable or better learning curves than dense models on NTM tasks; success on very long sequences (BPTT over >4000 steps); solid results on bAbI without supervised attention; improved Omniglot performance thanks to large memory.
- Generality: clear path to adapting the method to other MANNs (e.g., DNC) with demonstrated gains.
- Relevance: scaling MANNs is timely and important for applications in language and sequential reasoning.

Weaknesses and points for clarification:
- Fairness of comparisons: In some figures, memory sizes are chosen to equalize physical memory usage rather than the logical number of slots. This is a reasonable resource-constrained comparison, but an additional comparison at matched logical memory sizes would help isolate the algorithmic effects from resource budgets.
- ANN specifics: More detail on index parameters, rebalancing frequency, and empirical recall/precision would help readers assess the trade-off between speed and retrieval accuracy. An ablation on K and on ANN type (k-d vs LSH) would be valuable.
- Assumptions behind optimality: The lower-bound arguments (Supplementary A) hinge on content-addressing assumptions; restating the key assumptions and scope in the main text would strengthen the claims.
- Write policy design: The choice to write to previously read locations or least-recently used slots is intuitive, but a brief discussion comparing this policy to sparse content-based writing (and to DNC’s allocation mechanisms) would provide context. Sensitivity to the threshold used in the usage statistic would also be helpful.
- Clarity for non-experts: The paper would benefit from a concise, self-contained primer on MANN read/write semantics (erase/add behavior, role of write gates) and a high-level intuition for the rollback BPTT trick.
- Scope of tasks: Results are strong but largely synthetic. A case study on a real large-scale sequential task (e.g., language modeling or long-context QA) would better demonstrate end-to-end applicability.

Quality:
The technical approach is sound and carefully implemented. The sparse forward and backward mechanisms are convincingly described, and the space/time complexity claims are supported by benchmarks. Experimental coverage is broad. Some methodological details (ANN parameters/recall, sensitivity analyses) could be expanded. Providing code or pseudo-code for key data structures (usage tracker, rollback) would improve reproducibility.

Clarity:
The paper is well organized and generally easy to follow. Suggestions:
- Add a brief, self-contained background on MANN read and write operators (with a clear explanation of the erase/add mechanism and gates) to aid readers outside the area.
- Summarize the complexity assumptions and lower-bound intuition in the main text.
- Clarify the setup of comparisons where physical memory is matched versus where logical memory is matched, and state the rationale for each.
- Include hyperparameter tables for ANN indices and K, and report ANN recall during training.

Originality:
While the high-level goal of sparse memory access is not new, the combination of top-K sparse reads/writes, efficient in-graph ANN indexing, and the constant-space BPTT rollback is novel and practically impactful. The extension to DNC further broadens the contribution.

Significance:
The work substantially advances the scalability of differentiable memory architectures, a bottleneck that has limited their use on real-world problems. The reported gains and ability to train with very large memories and long sequences suggest immediate utility for the community.

Pros for acceptance:
- Strong, well-motivated contribution that addresses a core limitation of MANNs.
- Clear theoretical and practical speed/space improvements with convincing benchmarks.
- Broad and rigorous experimental evaluation; competitive or improved performance.
- General method applicable to multiple architectures (NTM, DNC).

Cons against acceptance:
- Limited ablations on ANN choices and K; sparse write policy design not deeply analyzed.
- Some comparisons emphasize matched physical memory; additional matched-memory results would strengthen conclusions.
- Primarily synthetic tasks; limited evidence on large real-world datasets.

Recommendations to authors:
- Add a concise MANN primer and clarify write semantics and gates.
- Report ANN configuration, recall, and sensitivity to K; include ablations comparing ANN types and exact search.
- Present both matched-physical-memory and matched-logical-memory comparisons where feasible.
- Summarize assumptions behind the optimality claims in the main text.
- Consider releasing code or detailed pseudo-code for rollback and usage tracking.

Overall evaluation:
A strong and timely paper that makes a clear, well-executed contribution to scaling memory-augmented neural networks. I recommend acceptance.

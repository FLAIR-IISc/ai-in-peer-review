This paper proposes a Neural Turing Machine (NTM) variant with sparse read and write operations to address a central limitation of NTMs: the prohibitive memory footprint and computational cost required for backpropagation with dense memory access. The method aims to retain state-of-the-art performance while dramatically reducing memory and compute, making the approach more practical for real-world use. The paper is well organized and generally easy to follow.

To make the work more self-contained for readers who are not experts in memory-augmented neural networks, consider adding a brief background section. In particular, a concise explanation of the memory module’s semantics—what the write operator is intended to accomplish and how it differs from conventional memory writes—would improve accessibility. While this may be obvious to specialists, it would help broaden the paper’s reach.

I have some reservations about the fairness of the comparison in Figure 4. Equalizing physical memory usage is a defensible choice for evaluating performance under resource constraints, and it answers an important practical question. However, it would also be informative to include results where models are matched on logical memory capacity (e.g., the same number of slots or addresses), even if their physical memory footprints differ. Reporting both views would better isolate algorithmic performance from implementation efficiency and give a more complete picture.

Overall, the approach is compelling. It substantially reduces resource demands while achieving performance close to theoretical limits, and the empirical results are strong.

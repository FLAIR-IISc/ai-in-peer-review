This paper introduces a Neural Turing Machine (NTM) variant designed to address a key limitation of standard NTMs: their high memory and computational demands during backpropagation, which hinders their applicability to real-world problems. The proposed “Sparse MTN” aims to achieve comparable performance to state-of-the-art NTMs while significantly reducing these requirements. The paper is well-written and logically organized, making it easy to follow.

To enhance accessibility for readers less familiar with memory-augmented neural networks, I recommend briefly outlining the foundational concepts in the background section. Specifically, clarifying the role and operation of the memory component—particularly the intended function of the write operator compared to conventional memory systems—would be beneficial. While the behavior is likely clear to researchers already working in this area, a more self-contained explanation would broaden the paper's impact. 

My primary concern relates to the comparison presented in Figure 4. While the authors’ approach of fixing the physical memory size to ensure a fair comparison under resource constraints is justifiable and insightful, an alternative comparison might offer a more informative perspective. Specifically, evaluating performance using the *same* memory capacity (number of memory locations) – rather than the same physical memory allocation – would better illustrate the efficiency gains achieved by the Sparse MTN. This would reveal how the algorithm performs given a fixed amount of information to store, regardless of physical memory overhead.

Despite this point, the results are compelling. The algorithm demonstrates performance remarkably close to theoretical bounds, which is particularly impressive given its reduced memory footprint and computational cost. Overall, this is a promising contribution with the potential to make NTMs more practical for a wider range of applications.

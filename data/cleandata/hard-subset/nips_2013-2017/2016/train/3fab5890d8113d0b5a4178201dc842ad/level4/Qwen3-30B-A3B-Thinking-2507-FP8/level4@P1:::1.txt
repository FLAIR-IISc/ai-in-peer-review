

This paper introduces Sparse Access Memory (SAM), a scalable memory architecture for memory-augmented neural networks (MANNs) that achieves asymptotic efficiency in space and time complexity through sparse read/write operations and approximate nearest neighbor (ANN) indexing. The authors demonstrate SAM's ability to maintain performance comparable to dense models while reducing memory consumption by 3,000× and accelerating training by 1,000× for large memory sizes (e.g., 64,000 words). SAM is evaluated across synthetic tasks (Copy, Associative Recall, Priority Sort), Babi question answering, and Omniglot one-shot learning, with the Sparse Differentiable Neural Computer (SDNC) achieving state-of-the-art results on Babi tasks without supervised memory access. The work builds on prior MANNs like NTMs [7] and Memory Networks [21] but extends their scalability through efficient sparse operations, addressing a critical bottleneck in real-world deployment.

**Strengths**  
The paper makes a compelling practical contribution by solving a well-known scalability problem in MANNs. The empirical results are robust: SAM consistently outperforms dense baselines (NTM, DAM) in speed/memory while matching or exceeding performance on all evaluated tasks. The extension to SDNC demonstrates the approach's generality, and the curriculum learning experiments convincingly show SAM's ability to handle sequences exceeding 200,000 steps with memory sizes in the millions. The theoretical analysis of SAM's asymptotic optimality (Supplementary A) is rigorous, and the implementation details (e.g., efficient backpropagation via sparse modification tracking) are well-motivated. The work clearly advances the state of practice in scalable memory systems, offering immediate utility for researchers deploying MANNs.

**Weaknesses**  
The novelty is modest relative to prior work. Section 3.3 of the Memory Networks paper [21] already describes using hashing for scalable memory lookups in *read-only* settings, and subsequent works like MemNN-WSH [15] adopt similar techniques. While SAM extends this to *writeable* memories—a meaningful practical step—the conceptual leap is incremental. More critically, the paper lacks sufficient detail for replication. Task specifications (e.g., exact sequence lengths, memory initialization for Copy/Associative Recall) are underspecified, and the absence of standardized task implementations (unlike the NTM's original codebase) creates significant barriers to reproduction. The authors should commit to releasing code and provide precise hyperparameter descriptions (e.g., discount factor *γ* in usage tracking).  

The non-differentiability of top-*K* selection (e.g., *K*=4) also warrants deeper analysis. While the authors note SAM's success, they do not address why discrete approximations work here but failed in similar settings (e.g., MemNN-WSH [15] reported inferior performance with non-smooth memory access). A discussion of failure cases—e.g., whether SAM struggles with tasks requiring precise memory updates or long-term dependencies—would strengthen the contribution. The claim that Omniglot is "real-world" is misleading; it is a synthetic benchmark with limited domain diversity. The paper should replace this with "natural" or "realistic" to align with standard terminology.

**Recommendation**  
**Accept (with minor revisions)**. The paper delivers a valuable engineering contribution with clear empirical impact, directly addressing a critical scalability gap in MANNs. While the novelty is incremental, the practical utility and thorough evaluation justify acceptance. The suggested revisions are essential for reproducibility and scholarly rigor.

**Pros for Acceptance**  
- Solves a significant scalability bottleneck with 1,000× speedup and 3,000× memory reduction.  
- Demonstrates scalability to unprecedented memory sizes (100,000s of words) and sequence lengths (200,000+ steps).  
- SDNC achieves state-of-the-art Babi results without supervised memory access.  
- Theoretical analysis confirms asymptotic optimality.  

**Cons for Acceptance**  
- Novelty is limited; prior work [21, 15] already explored sparse read access.  
- Insufficient detail for replication (task specs, hyperparameters).  
- No analysis of failure cases or limitations of the top-*K* approximation.  
- Misleading characterization of Omniglot as "real-world."  

**Required Revisions**  
1. **Task specifications**: Provide explicit details for all benchmark tasks (e.g., input sequence ranges, memory initialization protocols) to enable replication.  
2. **Code commitment**: Explicitly state plans for code release alongside the paper.  
3. **Failure analysis**: Add a brief discussion on where SAM may underperform (e.g., tasks requiring fine-grained memory updates) and why the top-*K* approximation succeeds here but failed in prior work [15].  
4. **Terminology**: Replace "non-synthetic" with "natural" when describing Omniglot.  
5. **Section 3.6**: Remove as suggested, as it is redundant with the core SAM description.  

This work is an important step toward deployable MANNs. With the requested clarifications, it will serve as a valuable reference for scaling memory-augmented systems.

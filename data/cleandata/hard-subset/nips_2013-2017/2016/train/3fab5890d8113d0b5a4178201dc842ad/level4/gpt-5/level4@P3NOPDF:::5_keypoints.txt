- Proposes Sparse Access Memory (SAM) that enforces sparse read/write to dramatically reduce runtime and memory, enabling much larger memories.  
- Reports orders-of-magnitude speed and memory gains and strong empirical performance under curriculum learning.  
- Claims remarkable generalization to sequences far longer than training (e.g., trained to 10k, tested at 200k).  

- Clear motivation: dense memory access scales poorly and sparsity is a principled remedy.  
- Claimed efficiency (≈ three orders of magnitude) could be transformative if validated.  
- Practical impact: makes memory-augmented models usable on previously infeasible problem sizes.  
- Generalization: extrapolation suggests the model may learn algorithmic structure, not just fit lengths.  
- Curriculum learning: model benefits notably from curricula, improving stability and compositional learning.  

- Needs analysis of why sparsity improves sample efficiency, curriculum progression, and long-range generalization (e.g., implicit regularization, reduced interference, better credit assignment).  
- Requires ablations isolating contributions of components (sparse reads vs writes, memory size, controller capacity, addressing mechanism).  
- Ask for experiments varying sparsity level k, memory size at fixed sparsity, and alternate addressing/retrieval schemes.  
- Baselines/fairness: compare to strong recent baselines with matched controller, optimizer, curriculum, and training budget; report wall-clock, peak memory, and sample complexity.  
- Clarify stability/differentiability: how are gradients computed through top-k/sparse ops and what is the impact on training convergence?  
- Provide sensitivity analyses for hyperparameters (sparsity, learning rate, memory size) and random-seed variability to assess brittleness.  
- Show empirical scalability curves for throughput and memory vs memory size on shared hardware/implementation.  
- Substantiate generalization claims with exact curricula, training length distributions, use of teacher forcing, evaluation protocol, trials and seed variance, and error-vs-length plots.  
- Broaden task diversity beyond synthetic algorithmic tasks to at least one realistic setting (e.g., long-document reasoning, long-horizon RL, large key-value retrieval).  
- Improve reproducibility by providing implementation details for sparse operations (ANN/indexing), read/write kernels, and ideally releasing code or pseudocode.  

- Provide explicit complexity analysis of sparse ops including practical constants and hardware implications (GPU vs CPU, memory bandwidth).  
- Offer interpretations/evidence for why sparsity aids learning (reduced interference, improved gradient signal) with metrics/visualizations (interference metrics, memory utilization histograms, access patterns).  
- Add ablations swapping sparse/dense access and hybrid variants to pinpoint which components drive improvements.  
- Include qualitative analyses: visualize memory access trajectories and check for stable, reusable routines.  
- Clarify limitations: identify tasks where sparsity hurts (e.g., highly distributed retrieval) and propose mitigation strategies.  

- Manuscript is generally clear; add a “Design choices and rationale” subsection to justify architectural decisions.  
- Ensure “orders-of-magnitude” claims are backed by standardized benchmarks under identical training budgets.  

- Overall: addresses a central bottleneck and offers compelling practical advances; recommend acceptance contingent on stronger empirical evidence, fair baselines, scaling/sensitivity analyses, ablations, and mechanistic discussion.

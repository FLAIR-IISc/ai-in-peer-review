Summary and relation to prior work:
This paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that achieves scalable reads and writes by enforcing sparsity in memory access. The key ideas are: (i) using approximate nearest-neighbor (ANN) structures to approximate content-based addressing with top-K selection, yielding O(log N) query, insert, and delete; (ii) constraining both reads and writes to a small, constant number of memory locations; and (iii) enabling efficient backpropagation through time by storing and reverting sparse memory updates rather than caching full memory states. The authors demonstrate large empirical speedups and memory savings relative to dense NTMs, comparable learning performance on standard synthetic tasks, scalability to very long sequences via curriculum, competitive results on bAbI, and good performance on Omniglot. They also show how to adapt the method to the Differentiable Neural Computer (SDNC), obtaining strong bAbI results without memory supervision.

This work is closely related to Memory Networks and their successors, which have long used content-based addressing with very large memories and hashing/ANN mechanisms to scale lookups (see, e.g., Sukhbaatar et al., 2014, Section 3.3; and MemNN-WSH in Sukhbaatar et al., 2015). The novelty here is thinner on the read side, but the application of sparse access and ANN infrastructure to writable memories, together with the engineering required for end-to-end training and efficient BPTT, is new to my knowledge and practically valuable.

Strengths:
- Demonstrates substantial speed (up to 1600×) and memory (≈3000×) improvements over dense NTMs at large memory sizes, with asymptotically optimal time/space complexity under content-based access.
- Shows that sparse reads/writes can match or even improve learning efficiency on several synthetic tasks; scales to very long sequences with curriculum; and achieves strong bAbI performance, especially with the SDNC variant.
- Presents a clear memory management strategy (e.g., least-recently-accessed allocation) and an efficient rollback mechanism for BPTT that are broadly applicable.

Weaknesses and requested changes:
1) Reproducibility and task specification: The paper does not provide enough detail to reproduce the tasks, and the original NTM task code is not publicly available. Given the lack of standard task definitions, the construction of all tasks (distributions, sequence generation, encoding/decoding, curriculum schedule, stopping criteria, evaluation metrics) must be described in sufficient detail. I also ask the authors to commit to releasing code for the models and tasks.

2) Where and why does it fail? While the text often emphasizes smoothness, the core operations include hard top-K selection (argmax/arg-top-K), which is non-smooth. In the read-only setting with K=1, the method is quite similar in spirit to MemNN-WSH (Sukhbaatar et al., 2015), which performed worse than fully smooth attention on some tasks. It would substantially strengthen the paper to analyze:
- The effect of K (e.g., K=1 vs K>1) on learning stability and final performance across tasks.
- Tasks/setups where the discrete choices degrade learning relative to smooth attention.
- Comparisons to methods that explicitly handle discrete addressing with policy gradients (e.g., Zaremba & Sutskever, 2015), discussing when such methods are necessary and when your approach suffices.
- Failure cases, sensitivity analyses (e.g., to the threshold for “non-negligible” access in usage tracking), and ablations (e.g., allocation strategy, erase policy, ANN type).

3) Prior work and novelty: The use of ANN/hashing to scale content-based read access has clear precedent in Memory Networks and related work. Please clarify this relation more explicitly in the main text, positioning your novelty as enabling writable memories and efficient BPTT with sparse updates.

4) Clarity: While the high-level description of SAM is clear, key implementation details are critical for others to replicate:
- Exact ANN configuration(s), rebuild schedules, and trade-offs (k-d trees vs LSH) across embedding sizes and workloads.
- Sparse backprop implementation details (data structures, memory rollback).
- Precise memory management and usage statistics, including any constants and thresholds.
- Training hyperparameters and curricula (how levels are advanced, thresholds, sampling of sequence lengths).
Also, please avoid referring to Omniglot as “non-synthetic” or “real-world”; it is a benchmark dataset for one-shot character recognition.

5) Editorial: I suggest removing Section 3.6. If retained, it should be clearly justified and tightly integrated; as written, it detracts from the main narrative.

Assessment by review criteria:
- Quality: The approach is technically sound and well-motivated. The empirical evaluation is broad and convincing on efficiency, but needs stronger details for reproducibility and more analysis of limits/failure modes.
- Clarity: Generally well-written and organized, but missing crucial experimental specifications as noted.
- Originality: Moderate. Sparse content-based reads with ANN are not new; the main novelty is extending sparsity to writes and enabling efficient BPTT for writable external memory.
- Significance: High practical significance. If broadly adopted, SAM-like mechanisms can make large-memory MANNs far more usable.

Pros for acceptance:
- Strong practical contribution with large speed/memory gains.
- Demonstrates that sparse access can preserve learning performance and enable long-horizon training.
- Generalizable framework (applies to DNC; likely applicable to other MANNs).

Cons for acceptance:
- Limited novelty on the read side relative to prior Memory Network literature.
- Insufficient task/method details for reproduction.
- Limited analysis of when discrete sparsification harms performance.

Recommendation: Accept, conditional on adding detailed task and implementation descriptions, committing to code release, clarifying novelty vs prior work, and providing an analysis of failure modes and the role of K/non-smooth choices.

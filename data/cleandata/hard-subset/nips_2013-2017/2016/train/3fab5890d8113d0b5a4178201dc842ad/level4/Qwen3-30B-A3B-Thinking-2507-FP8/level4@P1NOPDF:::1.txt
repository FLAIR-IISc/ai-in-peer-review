

The authors employ fast approximate nearest-neighbor structures and top-K argmax operations to compute attention weights in a Neural Turing Machine. Experiments demonstrate that this approach does not compromise training performance and enables scaling to larger problems. However, the novelty is limited; see Section 3.3 of the Memory Networks paper (arXiv:1410.3916). Several subsequent memory network papers have already addressed scalability for millions of memory items using hashing techniques. While the practical implementation for *writeable* memories represents a valuable contribution—especially since this specific application has not been previously reported—I recommend acceptance pending revisions.  

First, replication is currently challenging due to insufficient task specifications. Without standardized task definitions (unlike the original NTM, for which code was not released), the paper must provide detailed task construction procedures. The authors should also commit to releasing all experimental code.  

Second, the paper lacks analysis of failure modes and applicability boundaries. The model relies on discrete argmax operations (not smooth), which contrasts with fully differentiable approaches like the one in arXiv:1511.07275. Notably, the MemNN-WSH model (arXiv:1503.08895)—which similarly uses top-1 argmax without hashing—performed worse than smooth alternatives. Is the success here due to K=1, the task characteristics, or other factors? A thorough discussion of *when* this method fails and *why* it succeeds in specific cases would significantly strengthen the paper. This is especially important given that the model makes discrete choices during training without considering gradient propagation.  

Finally, I recommend removing Section 3.6. Additionally, the description of Omniglot as "non-synthetic" or "real-world" is misleading; it should be labeled as "real-world" without the "non-synthetic" qualifier, as the dataset is synthetic by design.

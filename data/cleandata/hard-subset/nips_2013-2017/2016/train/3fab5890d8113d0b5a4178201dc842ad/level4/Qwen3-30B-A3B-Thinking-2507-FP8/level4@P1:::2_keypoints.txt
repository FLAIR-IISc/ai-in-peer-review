- Proposes Sparse Access Memory (SAM), a scalable memory access scheme for MANNs that limits reads/writes to K=4–8 words per timestep.  
- Proves asymptotic optimality: O(log N) time per operation and O(1) space per timestep while maintaining representational power.  
- Key innovations: sparse read/write ops, an ANN index for content addressing (k-d trees/LSH), and an LRU-based memory management scheme.  
- Empirical gains: up to 1,000× speedup and 3,000× memory reduction versus dense models (e.g., 53MB vs 29GB for 64K words).  
- Demonstrates comparable data efficiency on synthetic tasks (Copy, Associative Recall, Priority Sort), Omniglot one-shot, and Babi QA.  
- Generalizes approach to Sparse Differentiable Neural Computer (SDNC), achieving new state-of-the-art on Babi without supervised memory access.  
- Strength: significant, well-motivated technical contribution addressing O(N) MANN bottleneck with asymptotic guarantees and thorough empirical validation.  
- Strength: solid implementation details—ANN integration, sparse backprop via state tracking, and usage-based memory management—support practical deployment.  
- Weakness: lacks a formal proof that sparse reads/writes preserve full algorithmic expressiveness of dense MANNs (can SAM learn all NTM-learnable algorithms?).  
- Weakness: ANN discussion needs more on practical trade-offs—k-d tree imbalance during online training and LSH sensitivity to word dimensionality.  
- Weakness: Omniglot results don’t compare to strongest non-MANN meta-learning baselines, weakening real-world applicability claims.  
- Weakness: unexpected improved learning on Priority Sort with sparsity requires deeper analysis to explain why sparsity helps.  
- Recommendation: weak accept with minor revisions—clarify expressiveness argument, add ANN trade-off discussion, and contextualize Omniglot vs non-MANN baselines.

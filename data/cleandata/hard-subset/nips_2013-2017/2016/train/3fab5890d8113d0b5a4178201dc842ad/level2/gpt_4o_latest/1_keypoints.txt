- Contribution: Proposes Sparse Access Memory (SAM), a MANN with sparse read/write and ANN indexing that claims asymptotic space/time improvements and orders-of-magnitude speed/memory gains (up to 1,000× faster, 3,000× less memory).
- Strength — Scalability and Efficiency: Empirically demonstrates large computational and memory savings via sparse access and ANN indexing.
- Strength — Comprehensive Evaluation: Evaluated on synthetic tasks, curriculum learning, Omniglot one-shot, and Babi, showing comparable or better performance while scaling to large memories.
- Strength — Novelty: Introduces sparse memory access and efficient BPTT for MANNs and adapts SAM to DNC, achieving SOTA on Babi.
- Strength — Clarity: Well-organized paper with clear algorithms and supplemental materials that aid reproducibility.
- Weakness — Limited Real-World Applications: Applicability to complex, large-scale real-world tasks (e.g., MT or RL) is underexplored.
- Weakness — Sparse Memory Trade-offs: Lacks deep analysis of trade-offs like effects on gradient flow and long-term dependencies.
- Weakness — Comparison with Alternatives: No direct empirical comparison to other sparse memory approaches (e.g., RL-based discrete addressing).
- Suggestion — Broaden Evaluation: Test SAM on more diverse real-world tasks (e.g., machine translation, large-scale QA) to demonstrate practical utility.
- Suggestion — Analyze Limitations: Provide a detailed study of scenarios where sparsity hurts performance and when dense memory might be preferable.
- Suggestion — Empirical Comparisons: Include direct comparisons with alternative sparse memory architectures to clarify relative advantages.
- Recommendation: Accept — SAM is a significant, well-validated contribution to scalable neural memory architectures despite areas for further exploration.

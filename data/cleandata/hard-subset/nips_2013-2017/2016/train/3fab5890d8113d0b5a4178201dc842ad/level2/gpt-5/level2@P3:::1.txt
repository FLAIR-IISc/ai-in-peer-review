Summary
The paper proposes Sparse Access Memory (SAM), an end-to-end differentiable memory-augmented neural network that scales to very large external memories by enforcing sparsity in both reads and writes. Reads are restricted to the top-K matches obtained via an approximate nearest neighbor (ANN) index (randomized k-d trees or LSH) in O(log N) time, while writes are limited to previously read locations and the least recently accessed (LRA) location, tracked by a lightweight usage structure. A key systems contribution is an O(1)-per-timestep memory rollback scheme for backpropagation through time that stores and reverts only sparse modifications. The paper argues asymptotic optimality in time and space under content-based addressing and demonstrates large empirical speed and memory reductions versus dense NTMs. Experiments span synthetic algorithmic tasks (copy, associative recall, priority sort), curriculum-based scaling to sequences in the thousands, bAbI QA (including a sparse adaptation of the DNC, “SDNC”), and Omniglot one-shot classification. SAM matches or surpasses dense baselines in data efficiency while achieving orders-of-magnitude runtime and memory savings; SDNC achieves state-of-the-art bAbI performance without supervision of memory access.

Relation to prior work
The work builds on NTMs (Graves et al., 2014) and Memory Networks/End-to-End Memory Networks (Weston et al., 2014; Sukhbaatar et al., 2015) by decoupling parameter count from memory size and enabling content-based addressing. Compared to reinforcement-learning-based discrete addressing for NTMs (Zaremba & Sutskever, 2015), the method retains differentiable access and low-variance learning signals. The approach is also shown to generalize to the Differentiable Neural Computer (Graves et al., 2016). The use of ANN indices (e.g., FLANN, LSH) inside the training loop for sublinear content lookup is a notable engineering innovation.

Quality
- Technical soundness: The sparse read/write scheme, LRA allocation, and rollback-based BPTT are well specified. Complexity claims are supported by algorithmic descriptions and empirical scaling. The non-differentiable ANN index is treated as a fixed data structure synchronized with the underlying dense memory; gradients flow only through the selected K rows.
- Empirical support: Benchmarks show 100–1600× speedups and dramatic memory reductions versus dense NTMs at large N, with comparable learning curves on standard NTM tasks. Curriculum experiments demonstrate scalability (e.g., >4000 BPTT steps, generalization to 200k steps for associative recall). On bAbI, SDNC solves all but one task without supervision of memory access and outperforms MemN2N-style results reported without supervision. Omniglot one-shot shows improved scaling with larger memory.
- Areas needing strengthening: More ablations would clarify the contribution of each component: K sensitivity, effect of approximate vs exact nearest neighbor, write policy choices (read locations vs LRA mix), and thresholds (e.g., w>ε). The impact of ANN recall errors on gradients and learning stability is not deeply analyzed. Fairness of comparisons could be further discussed: dense baselines use 64 memory slots while SAM uses millions; the paper argues equal physical memory budgets, but reporting accuracy vs wall-clock under matched budgets, plus accuracy vs memory size curves, would better isolate benefits of scaling.

Clarity
The paper is generally clear and well organized. The read/write equations, usage statistics, and rollback scheme are described precisely. Suggestions:
- Provide concise pseudocode for forward pass, write policy, index maintenance, and rollback to improve reproducibility.
- Specify ANN rebuild policy and its effect on training (e.g., frequency, amortized overhead, index imbalance).
- Summarize assumptions behind the “asymptotically optimal” claim in the main text and give an intuitive proof sketch referencing Supplementary A.
- Clarify batching and GPU implementation considerations, given sparse operations and ANN calls.

Originality
The combination of differentiable sparse access, O(1) per-step backprop memory via rollback, and integration of an ANN index into the training loop appears novel. Prior sparse-addressing work used discrete RL policies; this submission shows a differentiable route with strong practical scaling. The adaptation to DNC (SDNC) further evidences generality.

Significance
Scaling MANNs to million-word memories and thousands of time steps with end-to-end training is impactful and directly relevant to language modeling, QA, and one-shot learning. The demonstrated orders-of-magnitude efficiency gains and ability to train without supervision of memory access on bAbI suggest the approach can influence future large-memory neural systems.

Pros
- Strong and relevant problem: efficient scaling of differentiable memory.
- Elegant, practical algorithmic ideas: sparse reads/writes, LRA allocation, rollback BPTT.
- Clear asymptotic and empirical efficiency gains (runtime and memory).
- Broad empirical validation, including SDNC and non-synthetic Omniglot.
- Maintains or improves learning performance relative to dense baselines.

Cons
- Limited ablation and sensitivity analyses (K, ANN accuracy, write policy).
- Potential gradient mismatch due to approximate search not analyzed in depth.
- Comparisons to dense models under very different memory capacities complicate fairness; more nuanced accuracy-vs-resource trade-offs would help.
- Real-world evaluation is limited; Omniglot results are promising but narrow.

Suggestions for improvement
Essential
- Add ablations: vary K; compare exact vs ANN search; vary the interpolation/write gates; quantify effect on convergence and final accuracy.
- Report ANN quality (recall@K) during training and correlate with training stability and performance; analyze bias introduced by approximate top-K selection.
- Provide accuracy vs memory size curves and wall-clock comparisons under fixed compute/memory budgets to clarify fairness.
- Detail ANN maintenance (rebuild cadence, amortized costs) and batch processing strategy.

Optional
- Explore alternative allocation/erase policies (e.g., learned erase beyond LRA) and report trade-offs.
- Include additional real-data experiments (e.g., larger QA or long-context language modeling) to demonstrate broader applicability.
- Release code or detailed implementation notes for the sparse data structures and rollback mechanism.

Verdict
A strong, well-executed contribution addressing an important scalability bottleneck for MANNs, with compelling algorithmic ideas and solid empirical evidence. With added ablations and clearer resource-normalized comparisons, the paper would be even more convincing. Overall, this work is appropriate and impactful for NIPS.

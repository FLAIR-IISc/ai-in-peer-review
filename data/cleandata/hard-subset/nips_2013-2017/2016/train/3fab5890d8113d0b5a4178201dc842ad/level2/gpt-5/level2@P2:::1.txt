Summary and contributions
The paper proposes Sparse Access Memory (SAM), a memory-augmented neural network that imposes sparsity on both reads and writes to an external memory, and uses approximate nearest neighbor (ANN) data structures for content-based addressing. The key claims are: (i) asymptotically optimal time and space with respect to memory size under reasonable assumptions (O(log N) per step time for addressing; O(1) space per time step via sparse write logging and rollback for BPTT), (ii) large empirical speed and memory savings over dense memory models (NTM and a dense variant, DAM), and (iii) comparable data efficiency and accuracy to dense models on synthetic algorithmic tasks, the bAbi QA tasks, and Omniglot one-shot classification, with the ability to scale to memories with millions of slots and sequences with >100k steps. The paper also outlines how the approach applies to DNC, yielding a Sparse DNC (SDNC) with strong bAbi results.

Relation to prior work is clear: SAM builds on NTMs/DNC by making the read/write operations sparse and computable via ANN search; it stands in contrast to reinforcement learning approaches to discrete addressing (e.g., RL-NTM), and to dense Memory Networks with supervised attention. The use of LRU-style allocation aligns with DNC’s allocation mechanisms but is simplified to remain constant-time.

Quality
- The algorithmic contributions are technically sound and well-motivated. The complexity arguments are plausible; the sparse write plus in-place modification/rollback for BPTT is an elegant engineering solution to the O(NT) memory bottleneck.
- The use of ANN indices within the training loop is novel and nontrivial; the authors argue gradients do not flow through the index itself. The effect of the hard top-K selection is that gradients are zero for non-selected memory rows; while biased relative to the dense softmax attention, empirical results suggest it is still trainable.
- Claims are largely substantiated: clear speed/memory benchmarks; comparisons to dense baselines on canonical NTM tasks; curriculum scaling; bAbi and Omniglot. The scaling experiments equalize physical memory budgets and demonstrate tasks unreachable by dense models.
- Some experimental aspects would benefit from deeper analysis: sensitivity to K, effect of ANN recall errors vs exact KNN across tasks, and stability/variance across random seeds.

Clarity
The paper is generally well-written and organized. The read/write mechanisms, allocation policy, and BPTT rollback are described clearly; complexity claims are summarized with references to the supplement. A compact pseudo-code summary of the forward/backward memory update and rollback would improve readability. Hyperparameters, ANN configuration, and rebuild policies are partly deferred to the supplement; more of this in the main text would aid reproducibility.

Originality
The combination of sparse read/write with efficient backpropagation and ANN indices inside an end-to-end differentiable MANN is novel in the NIPS literature. While sparse attention has been explored elsewhere, the proposed integration with external memory and backprop rollback is original and impactful. The generalization to DNC further strengthens the contribution.

Significance
The work addresses a major practical blocker for MANNs—quadratic/linear scaling with memory size—and demonstrates orders-of-magnitude improvements in speed and memory use without sacrificing learning performance. This enables regimes (millions of memory slots, multi-thousand-step BPTT) previously infeasible, likely catalyzing further research and applications (e.g., long-context reasoning, large-scale episodic memory).

Strengths
- Clear, principled approach achieving asymptotically optimal time/space for content-based memory.
- Strong empirical speedups (100x–1600x) and dramatic reduction in physical memory usage, with careful engineering (sparse updates + rollback).
- Comparable or improved learning on synthetic tasks; successful scaling via curriculum to 10k+ steps and generalization to 200k steps.
- Demonstrated applicability to DNC and solid bAbi performance without supervised memory retrieval.
- Clean write policy combining “update recently read” with LRU allocation; constant-time implementation.

Weaknesses and questions
- Differentiability and bias: Hard top-K selection implies zero gradient to non-selected memory rows; discuss theoretical implications for convergence and whether annealed K or soft-to-hard schemes were tried.
- ANN index integration: Provide ablations quantifying accuracy/speed trade-offs for FLANN vs LSH across word dimensionalities, and the impact of rebuild frequency. How sensitive are results to ANN recall rate?
- Allocation policy: The LRU-like scheme is simple; how does it compare to DNC’s allocation-by-usage (and temporal links) in tasks requiring precise temporal chaining? The SDNC results are promising but details are relegated to the supplement; a brief comparison in the main text would help.
- Fairness of baselines: In curriculum scaling, sparse models have vastly larger memory capacity (by design). While the goal is scalability, a discussion of capacity vs sparsity effects (e.g., DAM with larger but still tractable memory) would clarify the source of gains.
- Robustness and stability: Report variance across seeds and any training instabilities (e.g., due to non-stationarity of the ANN index).
- Omniglot setup differences (one-hot labels, curriculum): clarify comparability to prior reports; include episodic accuracy vs number of classes during training vs test.

Minor comments and suggestions
- Include pseudo-code diagrams for forward/backward memory updates, including rollback and ANN updates.
- Report K sensitivity and any heuristics for choosing K=4/8; consider adaptive K or multi-head reads.
- Clarify how multiple read/write heads are handled (if used) and cost scaling with number of heads.
- Document the threshold epsilon used in usage tracking, and its sensitivity.
- Provide per-step amortized complexity including occasional ANN rebuilds.
- Release code and minimal working examples for the ANN integration and rollback mechanism to aid reproducibility.

Pro/Con for acceptance
Pros:
- Addresses a central scalability issue for MANNs with a principled, efficient method.
- Strong empirical speedups and convincing demonstrations on synthetic, QA, and one-shot tasks.
- General approach applicable to multiple memory architectures (NTM, DNC).

Cons:
- Limited theoretical analysis of optimization bias induced by hard top-K and ANN approximations.
- Some ablations relegated to the supplement; more robustness analyses would strengthen the case.
- Baseline fairness in scaling experiments could be discussed more explicitly.

Recommendation
Accept. The paper makes a substantial and timely contribution by enabling practical large-memory MANNs with careful algorithmic and systems design, and demonstrates broad applicability and strong empirical benefits. Confidence: high.

This paper introduces a novel variant of the Covariance Matrix Adaptation Evolution Strategy (CMA-ES), a state-of-the-art algorithm for derivative-free optimization. The CMA-ES adapts the parameters of a multivariate normal sampling distribution, which can be viewed as learning an underlying metric for the search space. The key contribution of this work is a new update mechanism for the covariance matrix based on triangular Cholesky factors. This modification significantly reduces both the computational complexity of the update step and the memory requirements for storing the algorithm's state.

The authors present a careful numerical evaluation comparing their proposed "Cholesky-CMA-ES" with the standard implementation. The experiments convincingly demonstrate that the new update scheme does not degrade the algorithm's performance in terms of the number of function evaluations required to reach a target objective value. Concurrently, it yields a substantial reduction in wall-clock time, an advantage that becomes more pronounced as the problem dimensionality increases.

I believe this paper is a very significant contribution to the field of derivative-free optimization. The proposed Cholesky-CMA-ES has several clear advantages over the current standard, and I expect it could become the default implementation for medium- to large-scale problems. The primary strengths are:
-   **Reduced Complexity:** The update has a lower, asymptotically optimal time complexity.
-   **Reduced Memory:** The method requires less memory to store the algorithm's state.
-   **Additional Information:** It provides the eigenvalues of the covariance matrix at no additional computational cost, which is useful for monitoring the optimization process.

The paper is well-written, and the approach is thoroughly and convincingly evaluated. Its relevance to the NIPS community is high, given the widespread use of CMA-ES in machine learning applications such as reinforcement learning and hyperparameter tuning, as well as its connections to information geometry.

I have only two minor points for clarification:

1.  To strengthen the wall-clock time comparisons, could the authors comment on the comparability of the different software implementations used in the experiments? For example, were they all written in the same language and using the same underlying high-performance libraries (e.g., LAPACK)?

2.  When commenting on Figure 2, the text states, "the higher the dimensionality the more pronounced the differences." However, this trend does not appear to hold for the Cigar and Discus functions at dimension d=256, where the performance gap seems to narrow slightly compared to d=128. It would be helpful if the authors could provide a brief explanation for this observation. The text mentions L2 cache effects, and elaborating on this would be welcome.

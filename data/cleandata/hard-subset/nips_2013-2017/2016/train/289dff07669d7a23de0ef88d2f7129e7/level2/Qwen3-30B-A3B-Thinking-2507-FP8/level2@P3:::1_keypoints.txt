- Introduces Cholesky-CMA-ES replacing matrix square root with triangular Cholesky factor to achieve optimal O(µ d²) time and storage for covariance updates.
- Addresses a major CMA-ES bottleneck where computing the matrix square root dominates runtime in high dimensions.
- Provides a theoretical justification for modifying the CSA mechanism when using Cholesky factors.
- Key theoretical insight: error from replacing C_t^{-1/2} with A_t^{-1} vanishes as the algorithm converges, supported by Lemma 1 (E_t → I).
- Builds on prior Cholesky-factor attempts but fills gaps by grounding the CSA change theoretically compared to Suttorp et al. and Krause & Igel.
- Strong empirical results on standard benchmarks (Sphere, Cigar, Rosenbrock) up to d=256, matching standard CMA-ES in function evaluations.
- Substantial runtime reductions demonstrated (≈20× faster than reference at d=64) with an implementation in Shark using LAPACK.
- Practical advantages: optimal triangular storage, easy eigenvalue access for conditioning, and efficient matrix inversion.
- Strengths: rigorous proof, thorough experiments (100 trials/dimension), open-sourced code, and clear exposition.
- Properly situates work against diagonal approximations and low-rank methods, highlighting retained performance on non-separable problems.
- High significance for ML applications (e.g., policy search) by enabling CMA-ES scaling where covariance storage was prohibitive.
- Minor weakness: benchmarks limited to standard functions; testing on highly multimodal functions (e.g., Rastrigin) would strengthen robustness claims.
- Could better discuss how CSA error scales with problem ill-conditioning (e.g., Diffpowers).
- Runtime comparison omits scalable gradient-based optimizers (e.g., L-BFGS), and for d > ~1000 low-rank methods remain necessary.

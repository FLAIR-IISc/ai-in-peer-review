- Proposes Cholesky-CMA-ES: update Cholesky factor instead of covariance/square root with theoretical justification.
- Shows Cholesky-CMA-ES matches standard CMA-ES in function evaluations while reducing runtime significantly.
- Achieves optimal asymptotic storage and runtime complexity, beneficial for high-dimensional, ill-conditioned problems.
- Positions work relative to prior efforts (delayed updates, reduced-complexity CMA-ES, prior Cholesky approaches) and claims novel theoretical foundation for the standard CMA-ES.
- Lemma 1 provides a convincing theoretical argument that neglecting coordinate rotation in step-size adaptation converges.
- Reports O(µ d²) runtime complexity, improving over standard CMA-ES covariance update costs.
- Reduces memory by maintaining only the Cholesky factor, improving storage complexity.
- Paper is well-written and clearly presents algorithm and theory.
- Empirical validation on benchmarks (including hard cases like Diffpowers) demonstrates effectiveness and scaling with dimension.
- Experimental focus is mainly on runtime and function evaluations, lacking deeper analysis of solution quality and convergence near the optimum.
- Reliance on LAPACK for performance could limit deployment where LAPACK is unavailable.
- Practical impact of the neglected coordinate rotation on step-size adaptation needs more empirical discussion across problem types.
- Comparison set omits some recent, highly optimized CMA-ES implementations, weakening benchmarking breadth.
- Cache effects (e.g., L2 limits at d=256) are mentioned but not analyzed in detail; memory bandwidth impacts need more study.
- Technical soundness, solid theoretical contribution, and convincing experiments make this a strong paper.
- Novelty stems from applying Cholesky factorization with theoretical rigor to the standard CMA-ES.
- Significance: substantial runtime improvements could broaden CMA-ES applicability to large-scale problems.
- Arguments for acceptance: strong theory, empirical speedups, addresses covariance update bottleneck, clear presentation.
- Arguments against acceptance: limited empirical depth beyond runtime, LAPACK dependency, need more study of step-size adaptation error.
- Overall recommendation: Strong Accept.

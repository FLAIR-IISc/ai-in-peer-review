- Proposal: Introduces Cholesky-CMA-ES that uses the Cholesky factor to avoid expensive matrix square roots, reducing runtime and memory in high dimensions.  
- Theoretical claim: Shows solid theoretical justification (notably Lemma 1) that using the Cholesky factor preserves CMA-ES guarantees and step-size convergence.  
- Complexity: Achieves optimal asymptotic storage and runtime complexity for full covariance adaptation, suited for large-scale problems.  
- Empirical results: Experiments demonstrate substantial runtime improvements with similar function-evaluation performance as standard CMA-ES.  
- Presentation: Paper is well written and organized, with helpful background on CMA-ES and Cholesky decomposition.  
- Comparison gap: Comparison to Suttorp-CMA-ES is brief and could be expanded with more scenario-driven analysis.  
- Cache/memory concern: Cache effects observed (e.g., d=256) and practical memory limits at very high dimensions need further discussion and mitigation strategies.  
- Implementation complexity: rankOneUpdate (Algorithm 2) may be non-trivial to implement efficiently in practice despite theoretical speedups.  
- Error analysis: Lacks a detailed quantitative error analysis of the approximation from replacing the matrix square root.  
- Quality score: 9/10 for technical soundness and supported claims.  
- Clarity score: 8/10; some theoretical derivations could use more elaboration.  
- Originality score: 8/10; main novelty is theoretical justification and optimal asymptotic complexity relative to prior Cholesky-based variants.  
- Significance score: 9/10; potential high impact for high-dimensional optimization tasks (RL, hyperparameter tuning).  
- Pro arguments: Solves an important bottleneck, strong theory, demonstrable speedups, clear write-up, addresses a widely used algorithm.  
- Con arguments: Practical implementation difficulty and unresolved extreme-dimension memory/cache limitations.  
- Recommendation: Strong Accept.



# Review of "Dynamic Mode Decomposition with Reproducing Kernels for Koopman Spectral Analysis"

This paper proposes an extension of Dynamic Mode Decomposition (DMD) using reproducing kernel Hilbert spaces (RKHS) to obtain nonlinear basis functions for Koopman spectral analysis. The authors present a theoretical framework and algorithmic implementation for this approach, with a robust variant using POD bases. While the paper addresses an interesting extension of DMD, it contains numerous technical inconsistencies and minor but significant flaws that undermine its quality as a scientific contribution.

**Technical Inconsistencies and Errors:**

1. **Notational Inconsistency:** The paper uses inconsistent notation throughout, particularly for eigenvalues and indices. Equations (5), (6), (9), and (13) all contain formatting errors like "λ l j" instead of "λ^l_j" and "λ̃ t j" instead of "λ̃^t_j." This is a fundamental issue that affects readability and precision.

2. **Mathematical Errors:** In Section 4, the paper states "the (i, j)-the element of the matrix (M∗τM+) is given by k(xi−1,xj)" which is incorrect. It should be k(xi,xj) since M+ = [ϕ(x1),...,ϕ(xτ)] and Mτ = [ϕ(x0),...,ϕ(xτ-1)], making the (i,j) element ⟨ϕ(xi), ϕ(xj)⟩ = k(xi,xj), not k(xi-1,xj).

3. **Theoretical Misstatements:** The paper claims "the Koopman operator defined in Eq. (7) is identical to the transition operator on an observable" (Section 3), which is inaccurate. The Koopman operator on the feature map is a linear operator in RKHS, while the transition operator on observables remains nonlinear in the original space.

4. **Algorithm Description Flaws:** The Gram-Schmidt QR decomposition algorithm (Section 3) fails to specify whether the decomposition is performed explicitly in RKHS or implicitly through the Gram matrix, which is critical for reproducibility. The paper mentions this can be reduced to Gram matrix operations but doesn't show how.

5. **Empirical Example Error:** In Section 6, the paper claims "the two eigenvalues are 0.5 and 0.9 with the corresponding eigenfunctions φ0.9 = xt and φ0.5 = yt − x2t." This notation is confusing and unconventional, as it uses the eigenvalue to name the eigenfunction (φ0.9 instead of φ_1, for example).

**Formatting and Presentation Issues:**

1. **Reference Incompleteness:** Several references in the bibliography are incomplete (e.g., [13] lacks page numbers, [1] has missing volume/issue details), which is unprofessional for a scientific publication.

2. **Figure Caption Error:** Figure 1 caption claims the right graph shows "the Hénon map" but the text states "the part of the eigenvalues of the equilibrium points seem to be approximately estimated," which is inaccurate for the chaotic Hénon map that doesn't have stable equilibrium points in the typical parameter setting.

3. **Notation Confusion:** The paper uses τ both as the number of time steps and as an index (e.g., xτ), which creates confusion in the context of time series analysis.

4. **Minor but Significant Typo:** In Section 4, the paper states "1τ is a τ-by-τ matrix for which each element takes the value 1/τ" but later refers to "1τ" as a vector in the definition of Ḡ = HGH = G − 1τG − G1τ + 1τG1τ. This is inconsistent notation that could confuse readers.

**Strengths and Weaknesses:**

*Strengths:* The paper presents a theoretically interesting extension of DMD using RKHS, which could be valuable for capturing nonlinear dynamics. The empirical examples on toy systems and the Hénon map are generally well-executed.

*Weaknesses:* The numerous technical inconsistencies, mathematical errors, and formatting issues significantly undermine the paper's credibility. The theoretical claims are not supported by sufficient mathematical rigor, and the algorithmic description lacks necessary details for reproducibility.

**Conclusion:**

While the paper's core idea is potentially interesting, the numerous technical inconsistencies, mathematical errors, and formatting issues make it difficult to evaluate the contribution's validity and significance. A paper submitted to NIPS should demonstrate mathematical precision and attention to detail, which this work lacks. These minor but significant flaws collectively outweigh the paper's potential contribution, making it unsuitable for publication at NIPS. I recommend rejection.

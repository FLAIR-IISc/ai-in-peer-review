

This paper introduces AutoBDMC, a method for assessing MCMC convergence using bidirectional Monte Carlo (BDMC) to compute stochastic bounds on the log marginal likelihood via annealed importance sampling (AIS). The key contribution is demonstrating that the gap between these bounds provides a provable upper bound on the Jensen-Shannon divergence between the true posterior and approximate samples, yielding a single scalar metric for convergence assessment. The authors validate that analysis on synthetic data (generated from the joint distribution) predicts real-world behavior, and showcase applications in model selection and debugging. The paper is well-motivated, clearly written overall, and the experimental evaluation is compelling.

However, the paper contains critical omissions that undermine its core contribution. The most significant issue is the failure to define "AutoBDMC" anywhere in the main text. The method appears to be the approach of applying AIS/rAIS to analyze MCMC convergence via BDMC bounds, but this is never explicitly stated. Consequently, Section 3.2’s assertion that "this poses a challenge for AutoBDMC" is meaningless without a prior definition. Given that the novelty hinges on the divergence bound derived in Section 3.1, this omission is a major flaw. The authors should explicitly state: "AutoBDMC is the method of using BDMC with AIS to compute convergence bounds via the Jensen-Shannon divergence bound." This definition must precede Section 3.2 to enable coherent discussion.

Section 3 also suffers from poor exposition. The transition into Section 3.1 is abrupt, launching into variable definitions and equations without motivation. The reader must infer the purpose of the divergence bound derivation, which is presented as a novel result (and should be explicitly labeled as such). The section would benefit from a clear statement of the theorem's novelty and its role in enabling AutoBDMC. As written, Section 3.1 is difficult to follow and obscures the paper’s technical contribution.

Minor but important issues include:
1.  **Notation inconsistency:** The paper uses both `y` and `D` to denote data/observations (e.g., Line 149: `p(theta, z | y)` should be `p(theta, z, y)`). This causes confusion and requires constant cross-referencing.
2.  **WebPPL supplemental code:** The code should specify compatible WebPPL versions and provide clear installation instructions (e.g., "Tested with WebPPL 0.9.15; install via `npm install webppl-auto-bdm`"). Ideally, it should be released as a standalone npm package. A similar Stan implementation with version compatibility notes would be valuable.
3.  **Section 3.1 motivation:** The paper should clarify why the specific divergence bound (Theorem 1) is useful for convergence assessment, beyond stating it is a bound. Linking it directly to the scalar metric used in experiments would strengthen the narrative.

The experimental section effectively demonstrates predictive validity on synthetic data and utility for model selection/debugging. However, the claim that synthetic data analysis "predicts behavior on real world data" relies on a single (though reasonable) empirical test. More rigorous validation—such as testing on diverse real datasets with known convergence properties—would further bolster this claim, though the current evidence is sufficient for acceptance.

In summary, the paper presents a useful and well-executed method for MCMC convergence assessment, with a clear theoretical foundation. The core innovation—the divergence bound enabling the scalar metric—would be compelling if the method’s definition and the bound’s novelty were clearly articulated. The current omission of "AutoBDMC"’s definition is a critical flaw that must be addressed for the paper to coherently communicate its contribution. The notation issues and code documentation are easily fixable but currently detract from reproducibility.

**Recommendation:** **Major Revision Required.** The paper has strong potential but requires essential clarifications: (1) Explicit definition of "AutoBDMC" in the introduction or methodology section; (2) Clear statement of novelty for Theorem 1 in Section 3.1; (3) Correction of notation inconsistencies (e.g., `p(theta, z, y)`); (4) Improved WebPPL code documentation. Addressing these would transform a promising contribution into a clear, reproducible, and impactful work. The method itself is sound and valuable, but its presentation currently obscures its significance.

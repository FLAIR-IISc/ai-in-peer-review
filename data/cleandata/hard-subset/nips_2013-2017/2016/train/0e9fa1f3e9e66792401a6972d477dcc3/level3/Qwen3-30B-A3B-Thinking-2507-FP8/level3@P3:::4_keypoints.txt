- Proposes using bidirectional Monte Carlo (BDMC) to produce stochastic upper/lower bounds on log marginal likelihood for assessing MCMC convergence.  
- Shows the gap between BDMC bounds upper-bounds the Jeffreys (Jensen-Shannon) divergence between true posterior and approximate sample distribution.  
- Introduces an "AutoBDMC" workflow (term used but never explicitly defined in the paper).  
- Method is limited to simulated/synthetic-data analysis, though experiments suggest synthetic behavior predicts real-data behavior.  
- Integrates the method into WebPPL and Stan and demonstrates applications to model selection and debugging.  
- Builds on Grosse et al. (2015) BDMC framework and differs from Cusumano-Towner & Mansinghka (2016) by using exact posterior samples for initialization.  
- Theoretical contribution (Jeffreys-divergence bound) is novel and significant but not clearly highlighted as such.  
- Quality: technically sound derivation and compelling, well-designed experiments (including a WebPPL bug discovery).  
- Clarity issues: "AutoBDMC" undefined, Section 3.1 transitions and variable definitions are confusing, and Section 3.2 refers to challenges prematurely.  
- Notational inconsistencies (e.g., p(θ, z | y) vs p(θ, z, y); inconsistent use of y vs D).  
- Supplemental code lacks documentation on compatible WebPPL versions and integration instructions; should be packaged (e.g., npm) for easier use.  
- Strengths: well-motivated, rigorous theory, thorough experimental validation, practical utility for model selection and debugging.  
- Weaknesses: unclear presentation in key sections, missing explicit novelty claim for the divergence bound, and under-documented supplemental code.  
- Recommendation: accept with minor revisions to fix clarity, define AutoBDMC, state novelty explicitly, and improve code documentation.  
- Arguments for acceptance: addresses a fundamental MCMC assessment problem, novel theoretical bound, practical demonstrations, and integration with popular PPLs.  
- Arguments against acceptance: undefined key term, confusing Section 3.1, notational issues, and insufficient supplemental documentation.

- Contribution: Extends moment matching networks to the conditional setting, which is non-trivial and notable.  
- Core technical challenge: The model matches distributions over mini-batches while each sample may have a different conditioning, so it must match multiple distinct target distributions simultaneously.  
- Inefficiency concern: Using the same conditional for an entire batch (to simplify matching) would be highly inefficient and unrealistic.  
- Main weakness — presentation: Paper prioritizes mathematical formalism (e.g., immediate RKHS machinery and Eq. 2) over clear, intuitive motivation.  
- Comparison to prior work: Departure from the original paper’s well-motivated objective; the conditional case lacks an analogous intuitive explanation.  
- Main weakness — implementation clarity: Paper fails to clearly explain how to translate Eq. (2) into a practical algorithm or detail the concrete computational steps.  
- Missing practical details: Specific computations, tractability assumptions, and computational-cost/efficiency analysis are lacking or buried in a hard-to-follow supplement.  
- Required improvements/questions: Provide an intuitive derivation/explanation of Eq. (2) and a precise, step-by-step description of the computational procedure to obtain the final algorithm.

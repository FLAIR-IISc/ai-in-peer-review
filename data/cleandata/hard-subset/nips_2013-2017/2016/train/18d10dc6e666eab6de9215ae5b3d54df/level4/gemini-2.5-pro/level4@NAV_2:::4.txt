This paper proposes the parallel Knowledge Gradient (q-KG) method, a novel algorithm for batch Bayesian Optimization. The core contribution is a new acquisition function, derived from a decision-theoretic framework, that selects a batch of `q` points to evaluate in parallel by maximizing the one-step expected improvement in the final solution quality. This approach extends the sequential Knowledge Gradient method and contrasts with other parallel methods, such as those based on Expected Improvement or UCB, which often construct batches greedily. To make the maximization of the q-KG acquisition function computationally feasible, the authors develop an efficient gradient-based optimization strategy using infinitesimal perturbation analysis. The method's effectiveness is demonstrated through extensive experiments on synthetic functions and hyperparameter tuning for machine learning models, showing significant performance gains, especially in the presence of evaluation noise.

**Strengths (Arguments for acceptance):**

*   **Significance:** The paper tackles the highly relevant problem of accelerating expensive black-box optimization by leveraging parallel evaluations. A principled and high-performing batch BO method is of great interest to the machine learning community, particularly for applications like large-scale hyperparameter tuning.
*   **Originality:** The proposed q-KG acquisition function is a novel and principled extension of the Knowledge Gradient to the batch setting. Unlike many existing methods that build batches greedily, this work jointly optimizes the entire batch of points based on their collective value, which is a more fundamentally sound approach. The use of infinitesimal perturbation analysis to create a practical, gradient-based optimizer for this complex acquisition function is also a key technical contribution.
*   **Quality:** The method is technically sound and well-motivated from a decision-theoretic perspective. The claims of superiority are well-supported by a comprehensive empirical evaluation against several state-of-the-art parallel BO algorithms on both synthetic benchmarks and practical machine learning tasks (tuning logistic regression and CNNs). The code is also made available, which aids reproducibility.

**Weaknesses (Arguments against acceptance / Suggestions for improvement):**

*   **Theoretical Analysis:** The paper is primarily empirical. While the experimental results are compelling, the work would be strengthened by a discussion of the theoretical properties of the q-KG algorithm. For instance, an analysis of its computational complexity with respect to the input dimension, batch size `q`, and the size of the discretized domain would be valuable. A comparison of its theoretical guarantees (e.g., regarding convergence or regret) to UCB-based parallel methods would also provide useful context.
*   **Scalability Concerns:** The method relies on a Gaussian Process model and a discretization of the search space for the inner optimization loop. The authors could briefly discuss the potential limitations and scalability of this approach for very high-dimensional problems, where standard GPs and space discretization can become computationally challenging.
*   **Clarity:** The paper is generally well-written, but some minor points could be clarified to improve readability.

**Detailed comments and suggestions:**

*   It would be helpful for the authors to comment on the computational overhead of optimizing q-KG compared to the baselines, especially as the batch size `q` increases.
*   The reliance on a GP is standard in BO, but it is a known limitation in very high-dimensional settings (e.g., >20 dimensions). A brief mention of this and potential remedies (e.g., using random embeddings or structured kernels) could help set reader expectations.
*   Minor suggestions for clarity:
    *   l.128: A brief, one-sentence description of the parallel EI algorithm when it is first mentioned would be helpful for context before it is detailed in Section 2.
    *   l.140: Using a different notation for the discretized domain `An` in equation (5.4) versus the full feasible domain `A` in (4.2) could avoid potential confusion. For example, using a script font or a different letter for the discretized set.

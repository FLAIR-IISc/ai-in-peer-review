

This paper introduces the parallel knowledge gradient (q-KG) method for batch Bayesian optimization, extending the knowledge gradient acquisition function to the parallel setting. The authors derive a decision-theoretic acquisition function that selects a batch of q points to evaluate simultaneously, which is Bayes-optimal for minimizing the expected minimum of the posterior Gaussian process predictor when only one batch remains. The paper makes two key contributions: (1) a novel acquisition function that jointly optimizes over the entire batch rather than using iterative greedy approaches, and (2) an efficient computational method using infinitesimal perturbation analysis to estimate the acquisition function and its gradient.

The work builds on previous Bayesian optimization literature, particularly extending the knowledge gradient method [7, 17] to the parallel setting. It relates well to recent parallel BO approaches [3, 5, 10, 18, 25], but distinguishes itself by not using iterative greedy selection of batch points. Instead, q-KG considers the batch collectively, which addresses a limitation in existing parallel BO methods that optimize single-point acquisition functions while holding other batch points fixed. The paper's theoretical foundation is strong, with clear derivation of the acquisition function and its computational approach.

**Strengths:**
1. **Theoretical Contribution:** The paper provides a principled decision-theoretic derivation of a batch acquisition function that is Bayes-optimal for the parallel setting, which is a significant advancement over iterative greedy approaches. The derivation is well-motivated and clearly presented.
2. **Computational Efficiency:** The authors develop an efficient gradient estimation technique using infinitesimal perturbation analysis, making the method practical for larger batch sizes. This addresses a key challenge in parallel BO that previous work has struggled with.
3. **Empirical Performance:** The method demonstrates consistent improvement over benchmarks (parallel EI, GP-UCB, GP-UCB-PE) on synthetic functions and practical ML tasks, especially when evaluations are noisy. The experiments on CNN hyperparameter tuning (improving test error from 14% to 11%) are particularly convincing.
4. **Clarity:** The paper is well-written, with clear organization and sufficient technical detail for reproducibility. The algorithm description is straightforward.

**Weaknesses:**
1. **Experimental Evaluation:** The benchmark is limited in scope. The authors evaluate on only two datasets (MNIST and CIFAR10) and two ML tasks (logistic regression and CNN classification), both classification problems. Hyperparameter optimization performance can vary significantly across different learning algorithms and problem types, so a more diverse benchmark would be more convincing. I recommend including benchmarks from HPOlib [2], which provides standardized benchmarks specifically designed for hyperparameter optimization algorithms.
2. **Optimization Criterion:** The paper uses test set error as the objective to optimize, which is problematic for hyperparameter tuning. As noted in [1], test set error introduces threshold-based metrics that are suboptimal for model selection. More appropriate metrics (e.g., area under the ROC curve, log loss) would better reflect the quality of the hyperparameter configuration.
3. **Error Assumption:** The authors assume independent normally distributed errors, which is common in GP-based BO but problematic for hyperparameter optimization. In this context, "errors" represent the difference between generalization performance and empirical estimates (e.g., cross-validation), which often exhibit structural bias, especially in small sample sizes. The paper should acknowledge this limitation and discuss how the method might be adapted to more realistic error structures.

**Pro/Con for Acceptance:**
*Pro:* Strong theoretical contribution, novel acquisition function, efficient computational approach, clear presentation, and convincing empirical results on synthetic functions and practical ML tasks.
*Con:* Limited experimental scope, problematic optimization criterion, and insufficient discussion of error assumptions in hyperparameter optimization.

**Overall Assessment:**
This paper makes a valuable theoretical contribution to Bayesian optimization by introducing a Bayes-optimal batch acquisition function. The method is well-motivated, theoretically sound, and demonstrates practical improvements over existing approaches, particularly in noisy settings. The theoretical strength is a significant asset for the NIPS community.

The main concern is the limited experimental evaluation. While the results on MNIST and CIFAR10 are promising, hyperparameter optimization performance can vary significantly across different algorithms and problem types. A more comprehensive benchmark with diverse ML tasks and standardized metrics would strengthen the empirical claims. The use of test set error as the optimization criterion also requires justification, as it is known to be suboptimal for hyperparameter tuning.

Given the strong theoretical contribution and the paper's relevance to a growing area of interest (parallel hyperparameter optimization), I recommend acceptance with minor revisions to address the experimental limitations. Specifically, the authors should expand their benchmark to include more diverse datasets and ML tasks using standard libraries like HPOlib, and discuss alternative optimization criteria that are more appropriate for hyperparameter tuning. The paper's core contribution is significant and would benefit from a more comprehensive evaluation.

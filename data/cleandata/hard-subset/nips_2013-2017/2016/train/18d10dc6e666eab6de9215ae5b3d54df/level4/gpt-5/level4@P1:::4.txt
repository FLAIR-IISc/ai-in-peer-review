Summary and relation to prior work:
This paper proposes the parallel knowledge gradient (q-KG) acquisition function for batch Bayesian optimization. The setting is black-box optimization with expensive evaluations and the ability to evaluate multiple points in parallel. The authors generalize the knowledge gradient (KG), which is Bayes-optimal for a single remaining evaluation in the sequential setting, to choose a batch of q points that is Bayes-optimal when a single batch decision remains. They develop an efficient way to estimate and optimize q-KG using Monte Carlo and infinitesimal perturbation analysis (IPA) for stochastic gradients, and they discretize the domain to handle continuous spaces. Empirically, they compare against parallel variants of EI and UCB (e.g., GP-BUCB, GP-UCB-PE) and show that q-KG is competitive or better, especially under noise. This work relates to prior batch BO methods that either greedily build batches using single-point acquisition functions (e.g., parallel EI heuristics, constant liar) or optimize batch EI/ES jointly via sampling and gradient methods. The main novelty here is a principled KG-based batch objective together with an efficient gradient estimator.

Strengths:
- A clear decision-theoretic formulation of batch selection that naturally extends KG to the parallel setting and retains the “one-step Bayes-optimal” property for a single remaining batch.
- A practical stochastic gradient estimator via IPA that makes joint batch optimization feasible, addressing the computational challenge of q-KG.
- Strong empirical performance on standard synthetic benchmarks and on realistic hyperparameter tuning tasks; the method appears particularly robust under noisy evaluations.
- Public code release facilitates reproducibility.

Weaknesses and suggestions:
- Quality/technical soundness: While the derivations and algorithms are well motivated, the paper would benefit from theoretical guarantees addressing error introduced by discretization of A in computing q-KG, and from a quantitative analysis of computational complexity as a function of batch size q, number of discretization points M, and input dimension d. In particular, the scalability of the IPA-based optimization when d is large is not fully characterized. The reliance on GP modeling may also be limiting for very high-dimensional inputs common in neural network hyperparameter spaces; some discussion of when the GP prior is appropriate would be helpful.
- Clarity: The exposition is generally clear, but there are a few notational ambiguities (e.g., the two different roles of A in Eq. (4.1) vs. the optimization domain in (4.2)). It would help to use distinct symbols for the discretized set used within q-KG and for the continuous feasible region. A brief, self-contained description of the baseline parallel EI variants used in experiments would also improve readability.
- Originality: Extending KG to the batch setting and providing an efficient gradient estimator is new and interesting. Prior work has largely focused on EI/UCB/ES in the batch setting; the KG-based perspective adds value and appears to confer advantages under noise.
- Significance: Parallel batch selection is important in many real-world settings (e.g., distributed hyperparameter tuning), and the empirical results suggest that q-KG can reduce wall-clock time to good solutions. Stronger theoretical characterization (e.g., error bounds or regret in the one-batch setting) and a complexity analysis would further increase impact and help practitioners reason about scalability.

Pros (arguments for acceptance):
- Principled acquisition function with a clear decision-theoretic foundation (Bayes-optimal for one remaining batch).
- Novel and efficient gradient estimation via IPA enables practical joint batch optimization.
- Consistent empirical gains, particularly under noisy evaluations, on both synthetic and practical tasks.
- Addresses an important and timely problem; code is available.

Cons (arguments against acceptance):
- Limited theoretical analysis beyond the one-step Bayes-optimality interpretation; no bounds on discretization error or complexity scaling with q and d.
- Dependence on GP assumptions may be problematic in very high-dimensional settings typical for neural network architectures.
- Some clarity issues (notation for A, brief treatment of parallel EI baselines) and minor typos.

Minor comments and requested clarifications:
- l.36: The phrase “the set of points to evaluate next that is” reads awkwardly; please rephrase for clarity.
- l.95: Please define A clearly when first introduced (domain, constraints, and assumptions on membership testing).
- l.104: Clarify the input to µ(n)(·); specify whether µ(n) is evaluated on A or on the discretized set, and define notation for vectors vs. scalars.
- l.128: Briefly describe the parallel EI algorithm(s) used for comparison, including the specific heuristic (e.g., constant liar, fantasized posterior) and any implementation details.
- l.140: Consider using distinct notation for the continuous feasible set (e.g., A) and the discretized set used inside Eq. (4.1) (e.g., Ān), to avoid confusion.
- Please report the number of Monte Carlo samples and gradient steps used when optimizing q-KG, and provide a runtime comparison versus the baselines for representative q and d.
- It would be helpful to discuss sensitivity to M (the number of discretization points), kernel hyperparameter estimation, and noise variance estimation, and to include an ablation if space permits.

Overall assessment:
This is a solid contribution to batch Bayesian optimization. The method is well motivated, the computational strategy is sensible, and the experiments are thorough and persuasive, particularly in noisy settings. Addressing the noted clarity issues and providing some theoretical and complexity analysis would strengthen the paper further, but I believe it is a valuable and timely addition to the NIPS literature on Bayesian optimization.

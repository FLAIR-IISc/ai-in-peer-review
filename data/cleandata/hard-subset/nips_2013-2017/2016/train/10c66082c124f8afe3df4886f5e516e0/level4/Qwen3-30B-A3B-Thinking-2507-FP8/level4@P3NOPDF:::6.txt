

The paper introduces a supervised extension of Word Mover's Distance (WMD) for metric learning on textual documents, where supervision is provided via class labels ("documents belong to the same class"). The authors propose learning two components: a linear transformation matrix **A** (the standard approach) and word importance weights **w**. The method is evaluated on text classification tasks, with MATLAB code promised for release.

The core contribution is well-motivated, as supervised metric learning can significantly improve document similarity measurement. However, several aspects require clarification to strengthen the technical presentation. First, the intuition behind Equation (5), which defines the role of **w**, is not sufficiently explained. The authors should explicitly state how **w** modifies the distance computation relative to **A** and clarify which component (or their interaction) drives the reported performance gains. For instance, does **w** primarily address lexical variation (e.g., synonyms), while **A** handles semantic alignment? Without this, the reader cannot assess the method's novelty or design rationale.

Regarding experimental evaluation, the authors focus on classification accuracy but omit a critical baseline: the performance of a simple classifier (e.g., kNN) using the *unsupervised* WMD as input. While the paper acknowledges that evaluating distance metrics is challenging, this baseline would contextualize whether the supervised metric learning provides meaningful gains over a straightforward application of the original WMD. The authors’ clarification that this point is now addressed in the revision is noted, but the original submission should have included this comparison to avoid ambiguity.

Several technical details require resolution. Equation (2) introduces a regularization parameter λ, but its relationship to the λ used in line 170 (likely a typo for Equation 170) is unclear. The authors should explicitly define λ’s role in both contexts and confirm whether these are distinct hyperparameters or the same parameter referenced differently. Additionally, the claim in line 316 that the method is "particularly forgiving when two documents use syntactically different but conceptually similar words" lacks supporting evidence. The authors should provide a concrete example or analysis demonstrating how **w** or **A** specifically mitigates syntactic variation (e.g., comparing "car" vs. "automobile" in documents) to justify this assertion.

Minor issues persist in the presentation:  
- Line 257 states "on the right we also show the average error across datasets," but this is not visible in Table 3. The text should be revised to reference the correct table or figure.  
- Table 2 uses "n" and "ne" without explanation. The distinction between these entries (e.g., "n" as number of samples, "ne" as normalized error) must be defined in the table caption.  
- The value of *k* for kNN classification is unspecified. The authors should clarify whether *k* is dataset-dependent (e.g., optimized per dataset) or method-dependent (e.g., fixed at *k*=5), as this impacts reproducibility and comparison fairness.

The paper’s strengths include a clear problem statement, a logical extension of WMD to supervised learning, and thorough empirical validation across multiple datasets. The proposed **w** component is particularly promising for handling lexical diversity. However, the current draft’s technical gaps—particularly the lack of intuition for Equation (5), unresolved parameter confusion, and insufficient justification for key claims—undermine the clarity of the contribution. Addressing these would significantly elevate the paper’s impact.

The authors’ responses to prior queries were thorough and helpful, resolving several initial concerns. For instance, they clarified the role of λ and the absence of the requested kNN baseline (which was deemed unnecessary post-revision). This responsiveness indicates a strong commitment to rigor. To maximize the paper’s value, the final version should integrate the following:  
1. A concise explanation of **w**’s role in Equation (5), contrasting it with **A**.  
2. Explicit definitions for all symbols (e.g., λ, *k*, "n"/"ne").  
3. A brief example illustrating how **w** handles syntactic variation (line 316).  
4. Correction of the table reference (line 257) to match Table 3’s content.  

Overall, this work presents a meaningful advancement in supervised text metric learning. With the proposed clarifications, it will make a valuable contribution to the field. I recommend acceptance pending minor revisions to address the points above. The method is sound, the experiments are comprehensive, and the code availability promise enhances reproducibility.

- Paper studies the compare-aggregate framework for sequence matching across QA and NLI, testing different comparison functions on four datasets and releasing code.
- Key finding: simple element-wise comparison functions (SUB and MULT) outperform more complex alternatives.
- Decision: Accept with minor revisions.
- Strength: Clear problem definition and motivation, highlighting gaps in prior work.
- Strength: Rigorous experimentation across diverse datasets (MovieQA, InsuranceQA, WikiQA, SNLI) demonstrating generalizability.
- Strength: Competitive or state-of-the-art results that challenge conventional wisdom about complex comparison functions.
- Strength: Ablation studies help explain component contributions.
- Strength: Code availability promotes reproducibility.
- Revision: Explain and motivate the preprocessing/gating layer more clearly, including impact on different word types (e.g., stopwords).
- Revision: Provide a deeper, intuitive analysis of why SUB and MULT work well and how dataset/task characteristics affect this.
- Revision: Report statistical significance tests for reported performance differences.
- Revision: Discuss hyperparameter sensitivity (embedding size, CNN filter sizes) and detail CNN aggregation configuration.
- Revision: Expand related work to include recent sequence-matching advances (e.g., transformer-based models).
- Questions for authors: clarify why an order-insensitive preprocessing layer performs well, share example gate values, compare computational cost of comparison functions, and report any statistical tests performed.

- Paper studies a compare-aggregate framework across four tasks (MovieQA, InsuranceQA, WikiQA, SNLI): attention align, compare with f, aggregate with CNN.
- Main contribution is a systematic evaluation of six comparison functions (NN, NTN, EUCCOS, SUB, MULT, SUBMULT+NN).
- Empirical finding: simple element-wise operations (SUB, MULT, SUBMULT+NN) are strong and often outperform more expressive neural comparators.
- Additional architecture: lightweight order-insensitive gating preprocessing, one-way attention, and CNN aggregation.
- Decision: Weak Accept.
- Key reason for acceptance: solid empirical study isolating comparison functions with consistent gains and competitive-to-SOTA results.
- Framing and motivation are appropriate — comparator-focused study fills a gap relative to existing pairwise models.
- Method is clear: gate-based preprocessing, one-way soft attention, token-level comparison, CNN aggregation; comparator choices span reasonable spectrum.
- Empirical support is convincing: across datasets SUBMULT+NN generally best; ablations show attention and preprocessing matter; code released.
- Practical implication: element-wise subtraction/multiplication are simple, efficient, and robust across tasks, useful for long sequences.
- Limitation: novelty is incremental; architecture is close to decomposable attention and related prior work.
- Fairness concerns: embeddings frozen (inconsistent with some baselines), comparator outputs have unequal dimensionality (EUCCOS 2-D vs others l-D), and no statistical significance tests.
- Context modeling concern: preprocessing is order-insensitive and attention is one-way; unclear if findings hold with richer contextual encoders or bidirectional/iterative attention.
- Experimental details under-specified: SNLI classification head, WikiQA negative sampling/filtering, dev/test splits, early stopping, dropout, etc.
- Suggestion: control capacity by projecting all token-level comparisons to a common dimension before CNN.
- Suggestion: include stronger baselines (bi-directional/decomposable attention, ESIM-like variants) to test comparator robustness with context.
- Suggestion: report significance/robustness checks (CIs, multiple runs, varying embedding fine-tuning, different l and CNN kernels).
- Suggestion: add error analysis (phenomena-specific on SNLI, length sensitivity on MovieQA) and runtime/memory comparisons.
- Suggestion: clarify implementation details (attention softmax axis, SNLI head, splits, tuned hyperparameters, early stopping).
- Questions for authors: did you project comparator outputs to equal dimensionality; try bi-directional/iterative attention; replace preprocessing with BiLSTM/GRU; fine-tune embeddings; follow standard WikiQA filtering; run statistical tests; report training/inference time and memory?

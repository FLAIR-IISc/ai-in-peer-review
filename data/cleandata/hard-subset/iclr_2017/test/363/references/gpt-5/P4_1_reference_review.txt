1. Summary of contributions
The paper investigates a general compare-aggregate architecture for matching text sequences and evaluates it across four distinct tasks/datasets: MovieQA (machine comprehension), InsuranceQA and WikiQA (answer selection/ranking), and SNLI (natural language inference). The model consists of: (i) a preprocessing layer with gating to modulate word importance, (ii) a one-directional attention mechanism aligning each token in one sequence to an attention-weighted summary of the other, (iii) a token-level comparison function, and (iv) a CNN aggregation module for final prediction. The central contribution is a systematic study of six comparison functions (NN, NTN, Euclidean+cosine, element-wise subtraction, element-wise multiplication, and a combined SUBMULT+NN). Empirically, the element-wise operations (SUB, MULT) and their combination perform best overall, often outperforming more flexible neural comparators and yielding state-of-the-art or competitive results on three QA datasets and competitive performance on SNLI. The paper includes ablations indicating the importance of the attention and preprocessing gates, and provides qualitative visualizations.

2. Decision
Accept. Key reasons: (1) Strong empirical results across multiple, diverse benchmarks demonstrating that a simple, general compare-aggregate architecture with principled comparison functions performs very well; (2) A clear, systematic evaluation of comparison functions that provides actionable guidance for the community (element-wise SUB/MULT are robust and effective).

3. Supporting arguments
- Clarity: The high-level model is clearly described, with explicit equations for preprocessing, attention, comparison, and aggregation. The motivation for focusing on comparison functions is well articulated and contextualized against recent literature (e.g., decomposable attention, match-LSTM, pairwise interaction models).
- Technical soundness: The components (gated preprocessing, attention, element-wise comparisons, CNN aggregation) are standard and correctly specified. The comparison functions are well-motivated: SUB/MULT preserve dimension-wise information related to Euclidean distance and cosine similarity without over-parameterization. The choice of one-directional attention is reasonable for the evaluated tasks.
- Empirical rigor: Results on four datasets provide breadth. The study compares six comparison functions and includes ablations (removing preprocessing/attention) with qualitative insights into when each module matters (e.g., attention helping more when sequence lengths differ). The model achieves SOTA or very competitive results on several datasets. Implementation details (optimizer, hidden size, embeddings) are provided; code is released.
- Novelty: The overall framework is close to prior “compare-aggregate” models (e.g., Parikh et al., Wang & Jiang, He & Lin). The primary novelty lies in the systematic analysis of the comparison function and demonstrating cross-task generality. While incremental, the findings are practical and non-obvious (simple element-wise operations beating more complex comparators and even distance/similarity scalars).
- Limitations/concerns: Some training details are missing or ambiguous (e.g., loss design for WikiQA with multiple correct answers; specifics of CNN aggregation). Baselines are taken from reported numbers rather than re-implemented under a common setting. No statistical significance tests are reported. The preprocessing for three tasks is order-insensitive, yet the model performs strongly; this is interesting but would benefit from deeper analysis and controlled comparisons (e.g., replacing CNN with simple pooling to isolate where order helps).

4. Additional feedback to improve the paper (helpful, not necessarily decision-critical)
- Clarify the training objective per dataset, especially for WikiQA where multiple correct answers exist. If using a softmax over candidates, explain how instances with multiple positives are handled (e.g., masking, multi-label loss, pairwise ranking loss).
- Specify attention normalization (over which dimension the softmax is applied) and any normalization of vectors prior to EUCCOS/SUB/MULT (e.g., L2 normalization).
- Provide more details on the aggregation CNN: number of filters, window sizes, pooling strategy. A small ablation comparing CNN aggregation to sum/max pooling or BiLSTM aggregation would help generalize the architectural takeaway beyond the comparison layer.
- Report hyperparameter selection protocol and whether all methods used the same dev sets. Include variance across runs and significance tests for key comparisons.
- For MovieQA, discuss whether direct Q–A matching (in addition to Q–P and A–P) helps, and whether two-directional attention changes results.
- Provide computational cost analysis (training/inference time, memory) and scaling behavior with sequence length, given the attention and CNN modules.
- Consider additional sanity checks: normalizing embeddings, the effect of fine-tuning embeddings, and regularization for NTN to ensure a fair comparison.
- Include brief error analyses per task to illustrate where the element-wise comparators succeed/fail, and whether errors correlate with lexical overlap versus paraphrastic phenomena.

Questions for the authors
- For WikiQA, what exact loss function was used during training given multiple correct answers per question? If using a softmax over K candidates, how are multiple positives handled?
- Along which axis is the softmax applied in Eq. (2) for G? Is the normalization per column (per token in A) so columns sum to 1?
- Are aj and hj normalized before applying EUCCOS, SUB, MULT? If not, did you try normalization and how did it affect performance?
- What are the CNN aggregation details (filter widths/counts, pooling)? How sensitive are results to these choices?
- Did you try bidirectional attention (both Q→A and A→Q), and if so, how did performance and computation compare?
- For MovieQA, did including a direct Q–A comparison path change results?
- How were hyperparameters tuned and were multiple random seeds evaluated? Any variance statistics?
- For NTN, what regularization and parameter initialization were used? Could its weaker performance stem from over-parameterization relative to data scale?
- How is class imbalance handled on SNLI (if at all)? Any constraints on maximum sequence length and truncation policy?
- Do the conclusions about SUB/MULT still hold if word embeddings are fine-tuned, or if contextual encoders (e.g., LSTM in preprocessing) are used across all tasks?



# Peer Review for ICLR 2017

## 1. Summary of the Paper's Claims and Contributions

The paper introduces the "neural statistician," a neural network that learns to compute summary statistics of datasets in an unsupervised manner. The authors propose extending variational autoencoders to model datasets rather than individual data points, using a hierarchical generative process where each dataset is assumed to be drawn from a common underlying generative process with context c. The neural statistician takes a dataset as input and outputs a vector of summary statistics (mean and variance of a Gaussian in latent space) that encapsulate a generative model for that dataset. The authors claim this approach enables efficient learning from new datasets for unsupervised and supervised tasks, demonstrating applications in clustering datasets, transferring generative models, selecting representative samples, and classifying unseen classes. The paper positions itself as an unsupervised, data-efficient, parameter-efficient method capable of few-shot learning.

## 2. Decision

Reject. The main reason is that the proposed method doesn't clearly demonstrate a significant advancement over existing approaches for the tasks it claims to solve, and the experimental evaluation lacks proper comparison to relevant baselines.

## 3. Supporting Arguments for the Decision

The paper's core idea—extending VAEs to model datasets rather than individual points—is conceptually straightforward but lacks a compelling justification for why this specific architecture is necessary or superior to simpler alternatives. The key limitation is that the paper doesn't properly compare against a natural baseline: training a VAE on each dataset individually and then using the latent representations for classification. For instance, in the OMNIGLOT few-shot classification experiments (Table 1), the authors compare against matching networks and other few-shot learning methods but don't include a baseline of using class-specific VAEs for classification, which would provide a much clearer benchmark for the "neural statistician" approach.

The claim that the method is "unsupervised" is misleading. While the statistic network is trained without explicit dataset labels, the method fundamentally requires datasets to be provided as input, which is not the same as traditional unsupervised learning. The paper doesn't adequately distinguish between unsupervised learning of statistics versus unsupervised learning of dataset similarity.

The paper's experimental evaluation is limited in scope and doesn't address a critical limitation mentioned in the conclusion: the "dataset hungry" nature of the approach. The authors don't provide experiments showing how performance scales with the number of datasets (e.g., 100 vs. 10,000 datasets), which is crucial for understanding the method's practical utility.

The few-shot classification results (Table 1) show the neural statistician performs worse than matching networks on 20-way tasks, but the explanation ("it can exaggerate differences between similar classes") lacks empirical support and is not sufficiently justified. The paper also doesn't provide evidence that the hierarchical structure they propose is necessary for the observed performance.

## 4. Additional Feedback for Improvement

- **Clarify the "unsupervised" claim**: The paper should explicitly define what "unsupervised" means in this context and distinguish it from traditional unsupervised learning. The method requires datasets as input, which is different from unsupervised learning where no structured data is provided.

- **Include proper baselines**: The most critical improvement would be to compare against the baseline of training a VAE on each dataset individually and using the resulting latent representations for classification. This would provide a clear benchmark for whether the hierarchical structure offers any advantage.

- **Address the dataset size limitation**: The paper mentions that the method is "dataset hungry" but doesn't provide experiments showing how performance depends on the number of datasets. Including such experiments would greatly strengthen the paper's claims.

- **Provide theoretical justification**: The paper would benefit from a theoretical analysis explaining why the hierarchical structure is necessary for the tasks they're considering, rather than just demonstrating empirical results.

- **Clarify the generative model transfer claim**: The paper claims the method can "transfer generative models to new datasets," but the experiments don't clearly demonstrate this transferability beyond simply generating samples from new datasets.

## 5. Questions for the Authors

1. How does the performance of the neural statistician scale with the number of datasets? What is the minimum number of datasets needed for the method to work effectively, and how does performance change as the number of datasets increases?

2. In the few-shot classification experiments, how does the neural statistician compare to a baseline of training a VAE on each class individually and using the class embeddings for classification? This comparison would be crucial for establishing the method's value.

3. The paper claims the method is "unsupervised," but it still requires datasets as input. How does this differ from simply training a VAE on each dataset individually (which would also be unsupervised)? What specific advantage does the hierarchical structure provide in this context?

4. In the YouTube Faces experiment, the authors use a Gaussian likelihood with shared variance across all datapoints. How does this affect the model's ability to generate diverse faces, and would using a per-datapoint variance improve the results?

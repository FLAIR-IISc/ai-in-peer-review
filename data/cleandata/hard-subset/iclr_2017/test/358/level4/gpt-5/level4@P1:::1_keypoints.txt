- Proposal: hierarchical generative model treating datasets as first-class objects with dataset-level context and example-level latents via a “double” variational formulation and a statistic network.  
- Decision: Accept.  
- Main strength: tackles the important, under-explored problem of dataset-level representations with a well-motivated hierarchical VAE extension.  
- Main strength: clean presentation of the double variational bound and statistic network, broadly applicable.  
- Empirical support: promising few-shot learning results suggesting the framework captures dataset-level structure and generalizes across datasets.  
- Modeling choice: statistic network enforces exchangeability via pooling, appropriate for unordered sets.  
- Primary concern: training the statistic network on minibatches may miss rare “existence”-type features and could bias the gradient relative to the full-dataset objective.  
- Clarification requested: does q(c|D) use all examples in a dataset or minibatches/subsets during training?  
- Clarification requested: if minibatches are used, is the resulting gradient estimator unbiased for the full-dataset objective, and under what assumptions?  
- Suggestion: provide an ablation on minibatch size per dataset and its effect on performance to quantify the trade-off between compute and expressivity.  
- Suggestion: mean pooling can be limiting for “any”-type features under minibatching; consider and evaluate alternatives (max pooling, learned attention).  
- Suggestion: discuss/experiment with pooling operators’ impact on performance, especially when rare features matter.  
- Practical suggestion: discuss applicability to hierarchical forecasting (e.g., electricity/sales) and required adaptations.  
- Question: how are within-dataset minibatches formed and how many examples per set are used when computing q(c|D)?  
- Question: can you provide theoretical or empirical analysis of any bias introduced by minibatch training of the statistic network?  
- Question: how would the statistic network learn presence/existence features if minibatches frequently exclude rare instances?  
- Question: can you compare training the statistic network on full sets versus minibatched subsets and report any performance gap?

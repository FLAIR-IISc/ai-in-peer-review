This paper proposes a hierarchical generative model where the lower level contains points within the dataset, and the higher-level model consists of unordered sets of datasets. The core idea is to use a "double" variance constraint, where a high-level latent variable describes the dataset and a low-level latent variable describes the individual examples.

Hierarchical modeling is an important and high-impact problem, and I feel it is under-researched in the deep learning literature.

Pros:
- Some shot-learning results look good, but I am not an expert in this field.
- The idea of using a "double" variance constraint in hierarchical generative models is well-presented and seems to be widely applicable.

Cons:
- When training a neural network, are minibatches (i.e., subsets of examples) used?
- If not, does using minibatches actually give you a reasonable approximation of the full gradient (if you used all the examples)? For example, what would happen if the neural network wanted to extract any example from the dataset that has a certain attribute and considers it as a feature? If your neural network is trained on minibatches, it will not be able to learn this feature, because some examples from the dataset will be missing in the given minibatch. Using minibatches to train the neural network (as opposed to using all the examples in the dataset) seems to limit the expressive power of the model.

This research paper proposes a hierarchical generative model where the lower level contains points within the dataset, and the higher level models are unordered sets of datasets. The core idea is to use a "double" variance bound where a higher-level latent variable describes the dataset and a lower-level latent variable describes the individual examples.

Hierarchical modeling is a significant and high-impact problem, and I feel it has been under-explored in the deep learning literature.

Pros:
- Some shot-learning results look good, but I am not an expert in this field.
- The idea of using a "double" variance bound in hierarchical generative models is well-presented and seems to be widely applied.

Cons:
- When training a statistical network, are minibatches (i.e., subsets of examples) used?
- If not, does using minibatches actually give you a fair estimator of the full gradient (if you had used all the examples)? For example, what would happen if the statistical network wanted to extract any example from the dataset that has a certain attribute and considers it as a feature? If your statistical network is trained on minibatches, it will not be able to learn this feature, because some examples from the dataset will be missing in a given minibatch. Using minibatches to train the statistical network (as opposed to using all examples in the dataset) seems to limit the expressive power of the model.

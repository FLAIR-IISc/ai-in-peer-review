The paper details a process for refining the controller RNN after training: “After the controller RNN is done training, we take the best RNN cell according to the lowest validation perplexity and then run a grid search over learning rate, weight initialization, dropout rates and decay epoch. The best cell found was then run with three different configurations and sizes to increase its capacity.” 
To facilitate replication, it would be valuable to include the specific hyperparameters used during this grid search, as well as the type of dropout employed (e.g., recurrent dropout, embedding dropout). Without this information, reproducing the results would necessitate extensive experimentation. Following the precedent set by work such as Zaremba et al. (2014), providing a baseline set of hyperparameters would significantly benefit future research in this area. 
This request for detailed hyperparameters extends to the other experiments presented, such as the character language model.
This paper addresses an important problem – the automation of neural architecture search. While the current computational cost is significant, this is a reasonable trade-off given the likely improvements in hardware and algorithms in the near future. 
The work demonstrates the technique’s effectiveness across both standard vision and text tasks, achieving gains on numerous benchmark datasets by exploring architectures beyond conventional RNNs and CNNs. While application to a wider range of datasets would be beneficial, the current results provide strong evidence that this approach is not only competitive with human-designed architectures, but may even outperform them. This suggests a promising path toward tailoring architectures to specific datasets without the need for extensive manual design.
Overall, this is a well-written paper on a compelling topic with robust results. I recommend acceptance.

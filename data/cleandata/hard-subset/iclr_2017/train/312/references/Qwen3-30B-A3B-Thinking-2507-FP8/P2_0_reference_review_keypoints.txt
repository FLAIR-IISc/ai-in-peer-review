- Proposes Neural Architecture Search (NAS) using a controller RNN that generates variable-length architecture descriptions and is trained with reinforcement learning.
- Main contributions: automated discovery of convolutional architectures (CIFAR-10) and recurrent cell architectures (Penn Treebank).
- CIFAR-10 result: 3.65% test error, claimed to outperform human-designed models and be 1.05x faster than DenseNet.
- Penn Treebank result: 62.4 perplexity, reported as 3.6 points better than previous state-of-the-art.
- Demonstrates flexibility, scalability, robustness, and transferability through multiple control experiments and transfer learning results.
- Positions work clearly relative to hyperparameter optimization, neuro-evolution, and program synthesis.
- Provides clear technical explanations of the controller RNN, RL training, skip connections, and recurrent cell extensions.
- Decision: Accept with minor revisions.
- Key reasons to accept: tackles an important problem with a novel, well-motivated approach and strong empirical results.
- Significance: could automate expert-intensive architecture design, democratizing access to high-performing models.
- Empirical evaluation is thorough: strong baselines, benchmarks (CIFAR-10, PTB), random-search comparison, and transfer experiments.
- Paper quality: well-written, well-structured, and described in sufficient detail for reproducibility.
- Suggestion 1: add a detailed computational cost analysis (GPU hours, comparison to human design).
- Suggestion 2: explain how the controller efficiently navigates an enormous search space (≈6×10^16 vs 12,800 evaluated architectures).
- Suggestion 3: include an explicit limitations discussion and scalability analysis (e.g., ImageNet experiments or subsets).
- Suggestion 4: compare/acknowledge more recent NAS developments (e.g., DARTS).
- Question 1: clarify which “other methods” cannot design models from scratch and why.
- Question 2: provide a fairer CIFAR-10 comparison controlling for DenseNet’s 1×1 convolution parameter reductions.
- Question 3: give details for the recurrent cell speed claim (inference time per token, FLOPs, etc.).
- Question 4: state the exact previous state-of-the-art perplexity value referenced in the 3.6 improvement claim.
- Question 5: justify the schedule of increasing child model depth every 1,600 samples and whether alternative schedules were tested.

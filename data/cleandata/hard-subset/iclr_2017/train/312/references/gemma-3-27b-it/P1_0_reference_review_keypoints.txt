- Introduces a gradient-based Neural Architecture Search (NAS) that uses an RNN controller and policy gradients to generate and optimize network architectures.
- Demonstrates strong empirical results on CIFAR-10 and Penn Treebank, achieving competitive or state-of-the-art performance.
- Decision: Accept with minor revisions — novel, well-executed, and reproducible with strong empirical validation.
- Novelty: Advances automated architecture discovery beyond hyperparameter tuning using reinforcement learning and RNN controllers.
- Strong empirical evidence that NAS can find architectures that outperform human-designed ones, notably on language modeling.
- Technical soundness: Clear policy gradient formulation, variance reduction (baseline), and handling of variable-length architectures.
- Comprehensive scope: Extends core NAS to skip connections, parallel training, and recurrent cell design with detailed controller generation.
- Computational cost concern: Request for clearer, concrete reporting of total compute (e.g., GPU hours) and comparison to other NAS/manual search costs.
- Search space limitations: Notes constrained layer types and asks for discussion on expanding search space and associated challenges.
- Ablation studies: Recommends more experiments (e.g., removing skip connections, varying controller architecture) to assess component importance.
- Code availability: Encourages prompt public release of code and trained models for reproducibility.
- Baseline clarity: Suggests a clearer definition of the random search baseline (sampling strategy, number of trials).
- Scalability question: Asks about limits on dataset size and model complexity and applicability to larger datasets like ImageNet.
- Generalization and stability: Requests evaluation of transferability to other tasks, sensitivity to random seeds, and reproducibility measures.
- Controller sensitivity and skip-connection behavior: Requests experiments on controller architecture sensitivity and intuition/analysis for why dense skip connections slightly hurt performance.

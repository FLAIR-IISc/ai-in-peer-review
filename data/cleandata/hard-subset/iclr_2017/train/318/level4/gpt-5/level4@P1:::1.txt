Summary of contributions
- The paper introduces a set of differentiable graph transformations that enable learning graph-to-graph mappings via gradient descent. This lets the model construct, modify, and reason over graphs as an internal state, rather than only consuming fixed graphs.
- Building on GGS-NN, the proposed Gated Graph Transformer Neural Network (GGT-NN) uses graphs as intermediate representations and can produce either textual or graph-structured outputs. This bridges graph-based reasoning and sequence prediction.
- The approach naturally applies to settings where states evolve as graphs. The cellular automaton example (Rule 30) illustrates this: the graph grows at each step (new nodes for new cells), with edges to neighbors and special nodes for 0/1 values. The model is trained to generate the sequence of graphs.
- On bAbI tasks, the model leverages graph-structured intermediate states and textual inputs to answer questions, and is reported to outperform prior work.

Decision: Accept (weak)
Key reasons:
- A clear, useful extension of GGS-NN that makes graph-structured intermediate states first-class, with differentiable graph construction and transformation.
- Strong empirical performance on bAbI tasks when using the proposed architecture.

Supporting arguments
- Problem addressed and motivation: Many tasks require tracking and updating relational structure over time. The paper tackles learning differentiable operators for graph construction and transformation, enabling graph-to-graph prediction and downstream text generation. This is well motivated and well positioned relative to GNN/GGS-NN work that assumes fixed graphs.
- Approach: The paper defines five differentiable graph transformations (node addition, node state update, edge update, propagation, and aggregation) and composes them into GGT-NN. The mapping from text to graph updates (including direct reference to node types) and from graph to output is coherent and enables end-to-end training.
- Empirical support: Results on the bAbI suite indicate that using a graph-structured intermediate state is effective; the reported model achieves state-of-the-art performance on these tasks. The cellular automaton experiment is a natural fit for graph-to-graph learning and demonstrates the intended use case of the proposed operators.
- Limitations: In the cellular automaton setting (Rule 30), which is an especially natural graph-to-graph problem, performance is far from solving the task. This highlights open questions about stability, training signals, or model capacity for more demanding graph dynamics.

Additional feedback to improve the paper (not all part of the accept decision)
- Clarify supervision and training signals. It would help to describe precisely what supervision is available in each experiment (e.g., whether full intermediate graphs are provided during training on bAbI, the role of direct references, and how node/edge losses are combined with answer losses).
- Ablations. Please include ablations for:
  - With vs. without direct reference on bAbI.
  - With vs. without intermediate propagation.
  - Contributions of each transformation (e.g., remove Tadd or TC) on the cellular automaton task.
- Error analysis. For bAbI tasks where the model underperforms (e.g., positional reasoning) and for Rule 30, provide qualitative failure cases and hypothesize whether errors stem from graph growth, edge typing, or propagation depth.
- Scalability. Discuss time/space complexity as graphs grow (e.g., O(|V|^2) edge updates), and whether sparse operations or pruning could mitigate this. Practical constraints will matter for larger real-world graphs.
- Hyperparameters and training stability. Include details on propagation steps, hidden sizes, learning rates, and regularization that were important for convergence, especially in the graph-growing setting.
- Comparisons. Since the core novelty is learning graph-to-graph transformations, consider adding baselines that operate on evolving graphs (e.g., fixed-topology GNN with externally provided graph updates, or rule-based graph updates) to isolate the benefits of learned graph construction.
- Generalization. For the rule discovery tasks, report generalization beyond training horizons and to larger graphs in more detail (e.g., accuracy as a function of steps and graph size), since this is a key selling point.

Questions for the authors
- Supervision:
  - For bAbI, do you provide the full ground-truth graph at each sentence during training? If so, how critical is this for reaching the reported performance?
  - How are the node strength and type distributions supervised in practice, and how sensitive is training to the ordering of added nodes?
- Direct reference:
  - How are node types mapped from words in cases of ambiguity or multi-word entities? How much does direct reference contribute relative to learned attention alone?
  - Can the model learn without direct reference for tasks that do not expose entity names reliably?
- Cellular automaton (Rule 30):
  - What do you believe limits performance here? Is it lack of depth in propagation, insufficient edge typing, or optimization issues as the graph grows?
  - Did you try curriculum learning (short to long horizons) or stronger regularization to stabilize graph growth?
- Propagation depth and capacity:
  - How many propagation steps are used per sentence and at query time? How does accuracy trade off with depth and hidden size?
- Efficiency:
  - What are the practical memory and runtime costs as the graph size increases in the automaton and bAbI tasks? Any evidence that sparse connectivity or neighborhood sampling would preserve performance?
- Reproducibility:
  - Are the code and exact graph schemas for bAbI available to facilitate comparison and follow-up work?

Overall, the paper presents a compelling idea—differentiable graph transformations for building and using intermediate graph states—and demonstrates its promise on bAbI. Strengthening the analysis of the graph-to-graph learning (especially Rule 30), clarifying supervision, and adding ablations would make the contribution more convincing.

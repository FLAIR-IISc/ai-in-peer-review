- Introduces a neural architecture that learns to construct and update a graph-structured internal state from text using differentiable graph transformations as an explicit memory queried to produce answers.  
- Extends GG-NN/GGS-NN by treating the graph as an evolving, differentiable representation manipulated during inference rather than a fixed input graph, driven by sentence encoders.  
- Claims to be the first implementation treating memory explicitly as a differentiable graph (vs. stack or flat memory array), demonstrated primarily on bAbI tasks.  
- The paradigm is original, thought-provoking, and technically sound, potentially more expressive than stack-based or standard memory networks.  
- Decision: Accept — novelty and potential impact of an explicit differentiable graph memory outweigh preliminary results and added complexity.  
- Paper improved substantially in clarity from the initial version and is intellectually stimulating despite higher complexity.  
- Method jointly learns sentence encodings and graph-transformation parameters, enabling building/modifying a relational state through a dialogue/story.  
- Empirically, performance on bAbI is broadly comparable to strong memory-network baselines, with no consistent accuracy gains across tasks yet.  
- The approach is significantly more complex conceptually and computationally than memory-network-style systems, raising efficiency vs. benefit questions.  
- Explicit graph memory is a compelling design that could ultimately be more powerful, and the revised presentation is technically careful.  
- Clarify exposition with an intuitive walkthrough and a figure mapping a short story to successive graph states before formalism.  
- Tighten novelty claims by clearly delineating differences from GGS-NNs (graph-as-input vs. evolving internal state) and DNC/Memory Networks (implicit vs. explicit structure).  
- Add ablations (e.g., remove direct reference, edge update, propagation) to identify which transformations drive gains.  
- Report computational cost (time/memory) relative to memory networks on the same hardware to quantify complexity trade-offs.  
- Include or discuss comparisons to traditional rule-induction baselines mentioned in the draft, or explain expected gaps if unavailable.  
- Visualize learned graphs on select bAbI tasks to illustrate reasoning paths and how reasoning unfolds.  
- Discuss generalization: whether graphs stabilize to task-meaningful structures and sensitivity to lexical variation.  
- Consolidate notation early, reduce symbol overload, and type/dimension matrices/vectors on first use; ensure comprehensive references for “first differentiable graph memory” claims.  
- Ask authors to precisely define the boundary of “differentiable memory as a graph” relative to GGS-NN and DNC/MemN2N, and cite prior work if any uses an evolving graph as memory.  
- Request asymptotic and empirical runtime/memory overhead vs. MemN2N/EntNet on representative bAbI tasks and insights on scalability bottlenecks.  
- Probe dependence on supervision: how performance changes with weaker/no graph supervision and what inductive biases help.  
- Ask for evidence that graph memory confers better systematic generalization (e.g., longer stories, compositional variations) beyond bAbI.  
- Investigate which bAbI tasks the approach underperforms rule-induction methods on and whether issues stem from search/credit assignment or representation mismatch.  
- Query how critical specific transformations are (edge toggle vs. continuous update, per-sentence propagation) and whether there are stability issues during training.  
- Request case studies where extracted graphs provide interpretability beyond memory-network attention maps.  
- Explore whether sparse connectivity or subgraph updates could make the approach practical on larger datasets and how that would integrate with the current formulation.

- Introduces Gated Graph Transformer Neural Network (GGT-NN), extending GGS-NNs to enable graph-structured intermediate representations that are constructed and modified from textual input.  
- Demonstrated on three domains: bAbI QA tasks, Rule 30 cellular automaton, and arbitrary 4-state Turing machines.  
- Achieves 95%+ accuracy on most bAbI tasks and particularly strong results on pathfinding (task 19), outperforming nearly all competing models.  
- Successfully learns underlying rules for the cellular automaton and Turing machine tasks without explicit rule specifications.  
- Recommendation: accept with minor revisions.  
- Contribution: meaningful extension allowing dynamic graph construction and transformation rather than only graph processing.  
- Technical approach: well-motivated, building on prior GNN, GGS-NN, and memory network work.  
- Experimental results: thorough and demonstrate versatility across different task types.  
- Strength: novel ability to perform graph transformations as part of the model.  
- Strength: strong empirical performance across multiple domains, especially where graph structure is advantageous.  
- Strength: clear demonstration of unsupervised rule learning capability.  
- Strength: thoughtful comparison with prior work, highlighting explicit graph structure vs implicit memory.  
- Weakness: explanation of the direct reference mechanism is unclear despite its apparent performance importance.  
- Weakness: discussion of computational complexity is brief; quadratic complexity is a noted limitation.  
- Weakness: limited comparative analysis of trade-offs versus existing memory network architectures.  
- Suggestion: add detailed explanation of direct reference mechanism implementation and rationale.  
- Suggestion: expand discussion of computational complexity and possible optimization strategies.  
- Suggestion: include a more comprehensive comparison of trade-offs between GGT-NN and other approaches.  
- Suggestion: clarify relationship between GGT-NN architecture and the graph sequence input extension in Appendix D.  
- Key question: how will the quadratic complexity be addressed for larger graphs?  
- Key question: can you provide more intuition for why direct reference improves performance?  
- Key question: how would the model scale to much larger, more complex graphs and what architectural changes would be needed?  
- Overall: strong, novel contribution that advances graph-based neural networks and provides a solid foundation for future work.



This paper introduces on-the-fly learning to represent dialogues as graphs serving as memory, first evaluated on the bAbI tasks. The graph structure is learned during inference, though long-term representation learning optimizes graph transformation parameters and sentence encodings. This represents the first implementation of a graph-based differentiable memory, though its complexity appears disproportionate to the marginal performance gains observed on bAbI tasks compared to established Memory Networks. While the approach remains preliminary, the graph-based memory representation offers greater expressive power than simpler structures like stacks.  

Clarity was initially a significant issue—the original draft was technically sound but read more like machine code than prose. The authors substantially revised the paper, resulting in a much more readable and coherent presentation. Despite the complexity, the work is technically accurate (within my understanding), conceptually novel, and thought-provoking. The core idea merits publication.  

However, the preliminary results do not yet demonstrate whether the complex graph-based memory provides superior learning or generalization capacity over alternatives. Performance on bAbI remains comparable to state-of-the-art Memory Networks but lags behind traditional rule-based methods (see [citation]). The authors should clarify whether the graph’s expressiveness translates to tangible benefits in more complex settings or if the added complexity is unjustified for current benchmarks. Future work should address this gap through rigorous comparisons on larger, more challenging datasets beyond bAbI.  

Overall, the paper presents a promising but unproven direction. The revised version significantly improves accessibility, and the underlying concept—using graphs as dynamic, learnable memory—deserves attention from the community. With further validation, this approach could inspire new architectures for structured memory in dialogue systems.



The paper proposes an extension to the Gated Graph Sequence Neural Network (GGNN) by enabling the model to learn complex, differentiable graph transformations as part of its internal representation. The core idea is to allow the model to dynamically build or modify a graph-structured state during computation, specifically for solving question-answering tasks (e.g., the bAbI dataset). The authors introduce five distinct differentiable graph transformations that are learned in a supervised manner, where the graph state is provided at each timestep during training. A specific instantiation of the model processes a sequence input, iteratively updates an internal graph state, and produces a final prediction; this approach achieves promising results on bAbI question-answering tasks.

The approach is conceptually interesting, as it successfully combines continuous learning (via gradient descent) with symbolic representation (through evolving graph structures). It represents a notable advancement over recent efforts to incorporate symbolic elements into differentiable models (e.g., Memory Networks, Neural Turing Machines), particularly because the model's state structure is not fixed but can adapt dynamically during computation. 

My primary concern relates to the training methodology: the model requires the graph state to be provided at every timestep during training. While feasible for specific tasks like bAbI (where the target graph transformations are known), this requirement is impractical for more complex problems where the desired graph evolution is unknown and must be learned from scratch. This limitation significantly restricts the model's broader applicability. 

Additionally, the paper's content and density suggest it would be more appropriate for a journal venue rather than a conference. The current presentation is overly dense and challenging to follow, which hinders clear communication of the technical contributions.

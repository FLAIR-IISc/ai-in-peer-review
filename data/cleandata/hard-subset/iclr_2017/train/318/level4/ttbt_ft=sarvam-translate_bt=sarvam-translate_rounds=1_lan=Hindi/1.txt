The main contribution of this paper appears to be the introduction of a set of differential graph transformations that allow you to learn graph-to-graph classification tasks using gradient descent. This naturally converts a cellular automaton represented as a sequence of graphs into a learning task. In that task, the graph of nodes grows with each iteration, with nodes pointing to neighbors and special nodes representing 0/1 values. The proposed architecture allows learning this sequence of graphs, although in practice, this task (rule 30) was far from being solved.

This idea is combined with the ideas of previous papers (GGS-NN) to allow the model to generate text output instead of graph output, and to use graphs as intermediate representations, which allows it to beat state-of-the-art technology on BaBi tasks.

- Proposes Gated Graph Transformer Neural Network (GGT-NN) that learns differentiable graph transformations to dynamically manipulate graph structures from sequential inputs.
- Extends GGS-NNs by using graphs as flexible internal representations rather than only as inputs.
- Introduces five differentiable transformations: node addition, state update, edge update, propagation, and aggregation.
- Demonstrates strong performance on rule-discovery tasks (Cellular Automaton, Turing machines) with good generalization to longer sequences.
- Achieves competitive or state-of-the-art results on bAbI QA tasks, outperforming prior methods on several tasks.
- Decision: Accept — judged a well-motivated, significant contribution with compelling empirical validation.
- Architecture is clearly described and builds logically on GRUs and prior GGS-NN work.
- Authors are transparent about using strong supervision, which affects comparisons to end-to-end models.
- Major limitation: heavy reliance on step-by-step, ground-truth graph supervision; sensitivity to weaker supervision is untested.
- Scalability concern: model complexity scales quadratically with node count; lacks empirical runtime/memory analysis.
- Evaluation gap: brief failure analysis on bAbI Task 17 — deeper investigation and visualization of incorrect graphs recommended.
- Direct reference mechanism relies on brittle string matching; suggest more robust alternatives (e.g., attention over embeddings).
- Question: How sensitive is performance to the design of hand-crafted graph schemas (e.g., linked-list for Task 3)?
- Question: For node addition, does the network learn a consistent canonical ordering or remain permutation-arbitrary in practice?
- Question: Request visualization/analysis of a learned Turing machine graph to confirm correct FSM encoding of transitions.

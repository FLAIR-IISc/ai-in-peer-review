- Proposes Gated Graph Transformer Neural Network (GGT-NN), extending GGS-NNs with differentiable graph transformations as internal representations.  
- Introduces five differentiable transformation types: node addition, node state update, edge update, propagation, and aggregation.  
- Trains in a supervised setting with graph states provided at each timestep; applied to QA (bAbI) with promising results.  
- Recommendation: Accept — novel, well-motivated extension placed well in the literature with a clear, concise architectural explanation.  
- Empirical evidence: strong results showing the model can learn complex graph transformations and achieve high accuracy.  
- Strength: maintains evolving, complex graph states while remaining fully differentiable and trainable by gradient descent.  
- Advantage: effectively mixes continuous and symbolic representations and is more general than fixed-shape memory models (e.g., Memory Networks, Neural Turing Machines).  
- Evaluation thoroughness: includes ablation study and comparisons with state-of-the-art.  
- Major concern: requires ground-truth graph states at each timestep, limiting applicability where such supervision is unavailable.  
- Readability concern: paper is dense and might be more suitable for a journal than a conference.  
- Suggestion: provide more implementation details, including architectures used for each transformation type.  
- Suggestion: discuss potential applications beyond question-answering (e.g., graph generation, graph-based reasoning).  
- Suggestion: add more visualizations of model behavior (graph visualizations, attention heatmaps).  
- Suggestion: address the supervision requirement for ground-truth graph states and discuss ways to relax it.  
- Questions for authors: provide details on architectures for each transformation and how they were chosen.  
- Questions for authors: how will you address the ground-truth graph-state requirement?  
- Questions for authors: have you considered applying the model to tasks beyond QA, such as graph generation or reasoning?  
- Questions for authors: can you provide more visualizations of model performance (graph visualizations, attention heatmaps)?



# Review of "Learning Graphical State Transitions"

## 1. Summary of Contributions

The paper introduces the Gated Graph Transformer Neural Network (GGT-NN), a novel architecture that represents dialog and world states as differentiable graphs, serving as a memory structure that can be constructed and modified during inference. This represents a significant conceptual advance over previous approaches like memory networks, as it explicitly models the internal state as a graph with labeled nodes and edges rather than a fixed-size list of vectors. The authors demonstrate the model's capabilities on bAbI tasks (achieving comparable results to state-of-the-art memory networks), as well as on rule discovery tasks like cellular automata and Turing machines. The key innovation is the introduction of differentiable graph transformations (node addition, state updates, edge modifications, etc.) that allow the network to build and modify its graphical internal state based on textual input. The work is technically sound, well-motivated, and represents an important step toward more flexible neural architectures that can handle structured data.

## 2. Decision

Accept

Key reasons: (1) The paper presents a novel approach to differentiable memory representation as a graph, which is conceptually significant and appears to be the first implementation of this idea. (2) Despite modest performance gains on bAbI tasks, the work is technically accurate, thought-provoking, and makes a constructive contribution to the field.

## 3. Supporting Arguments

The paper's primary contribution is the introduction of a differentiable graph structure as a memory mechanism that can be constructed during inference, rather than being fixed or represented as a simple list. This is a significant conceptual advance over previous memory networks, which store information in a fixed-size list of vectors that cannot be easily interpreted or visualized. The GGT-NN's explicit graph structure allows for potential downstream applications that require graph-structured input.

The technical execution is sound, with the authors providing detailed descriptions of the graph transformations and demonstrating the model's capabilities on multiple tasks. While the performance on bAbI tasks is comparable to top memory networks (with some tasks showing modest improvements), the work is still preliminary and doesn't yet demonstrate superior learning or generalization capacity. However, the paper's main contribution is conceptual rather than empirical, and it opens up new directions for research.

The authors acknowledge the model's complexity (substantially more complex than prior methods) and the quadratic computational cost of graph operations, while also suggesting promising directions for future work to address these limitations. The paper is well-placed in the literature, with a thorough comparison to related work that clearly articulates the advantages of their approach.

## 4. Additional Feedback

To improve the paper, I recommend:

1. Clarify the relationship between GGT-NN and prior work on graph neural networks. While the paper mentions Li et al. (2016), it would be helpful to more explicitly state how GGT-NN differs from GGS-NNs and other graph-based approaches.

2. Provide more detailed analysis of why the model achieves better performance on certain tasks (like pathfinding) but not others (like positional reasoning). The current paper notes that task 17 has more entities, but a deeper analysis would strengthen the argument for the graph-based approach.

3. Include a more thorough discussion of the computational complexity trade-offs. While the paper mentions quadratic scaling, it would be helpful to see empirical evidence of how the model's training time and memory requirements scale with graph size.

4. Consider adding a visualization of the learned graphs for some of the bAbI tasks to demonstrate how the model structures the knowledge internally.

5. The paper mentions that the model with direct reference performs better than without, but it's unclear whether this is a general phenomenon or specific to the bAbI tasks. A more systematic analysis of this aspect would be valuable.

## Questions for Authors

1. The paper states that the GGT-NN with direct reference outperforms the model without it on 19/20 tasks. Could you provide more insight into why direct reference is so beneficial for most tasks, and whether this benefit might diminish with larger datasets?

2. For the rule discovery tasks, the paper shows the model successfully learns Rule 30 and Turing machines. Could you elaborate on how the model generalizes to longer sequences (20-30 timesteps) compared to the training sequences (7-6 timesteps)? What specific challenges does this pose for the model?

3. The paper mentions that the GGT-NN's distributed nature makes it less time and space efficient than memory networks. Could you provide more concrete evidence of this trade-off through empirical measurements of training time and memory usage for comparable tasks?

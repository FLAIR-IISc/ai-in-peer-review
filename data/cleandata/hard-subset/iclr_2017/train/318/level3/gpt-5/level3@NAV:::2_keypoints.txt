- Proposes GGT-NN: learning differentiable, graph-structured memory built on-the-fly from text with node addition, state/edge updates, propagation, and aggregation.  
- Novelty: first concrete implementation of an explicit, learned graph memory (vs. vector memories), building on GGS-NNs.  
- Technical contribution: differentiable graph transformations applied during inference while learning transformation parameters and sentence encoders.  
- Demonstrates end-to-end use of graph memory on bAbI QA and on rule-discovery tasks (cellular automata, Turing machines).  
- Decision: Accept — novelty, significance, and technical soundness justify acceptance despite limited bAbI gains.  
- Empirical results: comparable to top memory networks on bAbI, strong on pathfinding and induction, but not overall superior and worse than rule-induction baselines.  
- Limitations: training requires strong graph supervision and a direct-reference mechanism; approach is more complex and computationally costly than memory networks.  
- Presentation: latest draft is substantially clearer; claims are appropriately scoped and results appear correct.  
- Ask to quantify computational and memory complexity vs. MemN2N/DNC and report wall-clock and peak memory.  
- Ask to clarify performance degradation without graph supervision or with partial supervision.  
- Request ablations: which transformations (Tadd/TC/Tprop) matter most and sensitivity to edge sparsity.  
- Suggest additional evaluations: tasks favoring rule-induction to probe generalization and robustness of direct-reference to noisy mentions/coreference.  
- Reproducibility request: release code and exact hyperparameters.

- Proposes Gated Graph Transformer Neural Networks (GGT-NNs): a differentiable, typed graph memory constructed from text that can add nodes/edges, propagate information, and produce answers.
- Core contribution: a toolkit of differentiable graph transformations (node addition, node-state updates, edge updates, propagation, aggregation) trained end-to-end.
- Model can construct and use symbolic-looking graph state for reasoning directly from unstructured input.
- Achieves near-SOTA accuracy on bAbI with strong supervision, especially on graph-aligned tasks like pathfinding.
- Demonstrates rule discovery and simulation capabilities: learns Rule 30 and 2-symbol 4-state Turing machines with some generalization to longer rollouts.
- Technically careful and novel in explicitly positioning a learned, typed graph as core memory.
- Decision: Weak accept.
- Key reasons for acceptance: conceptual novelty, clear technical contribution, and solid empirical validation across diverse domains.
- The approach is likely to stimulate further research into explicit structured memories despite modest bAbI gains over strong baselines.
- Tackles unified problem of building, transforming, and reading graph-structured state from text in a single differentiable model.
- Differentiates from Memory Networks/DNC by making structure explicit and interpretable rather than implicit via attention/pointers.
- Contrasts with prior GNNs that assume given graphs by learning and updating graphs from text.
- Results support claims: competitive bAbI performance, convincing rule-simulation generalization, and well-defined differentiable operators.
- Supervision strategies and set-loss for node orderings are appropriate and carefully explained.
- Caveat: bAbI improvements over strong baselines are modest and model complexity + supervision requirements are higher.
- Caveat: Computational and memory complexity scale quadratically with number of nodes; scalability to larger problems not demonstrated.
- Caveat: More analysis needed to distinguish learning compact rules vs. local pattern replication in rule-discovery tasks.
- Suggested clarity improvement: add a concise “design goals and trade-offs” section explaining when explicit graphs help and associated costs.
- Suggested experiments: fuller ablation table (transformations, direct reference, propagation, supervision), variance over seeds, hyperparameter sensitivity, and graph-size caps.
- Fairness suggestion: clarify supervision used across baselines and, if possible, compare MemNN/DNC variants with same supervision.
- Efficiency suggestion: report wall-clock, memory usage vs. nodes/edges, and try sparsification or neighborhood truncation strategies.
- Interpretability suggestion: visualize learned graphs over time and highlight differences from ground truth to illustrate failure modes.
- Generalization suggestion: extend rollouts for Rule 30/TMs, test unseen rules and longer tapes, and quantify degradation.
- Weaker-supervision suggestion: explore semi-supervised training, regularizers for sparsity/consistency, or reduced graph supervision.
- Key questions for authors: how performance degrades with reduced graph supervision or no direct reference; preliminary semi-supervised results?
- Key questions for authors: which transformations are most critical (e.g., remove Tadd/TC), and how many propagation steps are needed?
- Key questions for authors: typical nodes/edges per story and compute/memory footprints; any sparsity mechanisms tried?
- Key questions for authors: compare to algorithmic baselines (e.g., BFS on extracted graph) and behavior with larger bAbI training sets (10k)?
- Key questions for authors: diversity of Turing machine rules used, correlation of failures with transition structures, and evidence model learned abstract transitions vs. memorizing sequences?
- Key questions for authors: sensitivity of direct reference to entity surface forms and potential for learned entity linking to replace exact matches.

1. Summary of contributions
The paper proposes learning graphical state transitions that construct and manipulate a graph-structured, fully differentiable memory on-the-fly from text. Building on GGS-NNs, the GGT-NN introduces differentiable graph transformations (node addition, state update, edge update, propagation, aggregation) applied during inference, while learning the transformation parameters and sentence encoders over the long term. It demonstrates end-to-end use of graph memory for bAbI QA, and for rule-discovery tasks (cellular automata, Turing machines). To my knowledge, this is the first concrete implementation where the model’s internal memory is an explicit, learned graph rather than a set/vector memory, and it is technically careful and thought-provoking. The latest draft is substantially clearer than earlier versions.

2. Decision: Accept
Key reasons: (i) Novelty and significance of an explicit differentiable graph memory learned on-the-fly, which is well-motivated and well-situated relative to GNNs, memory networks, and DNC; (ii) Technical soundness and a compelling set of analyses despite preliminary performance parity on bAbI.

3. Supporting arguments
- Problem: How to endow neural models with an explicit, differentiable, graph-structured memory that can be built and transformed from text and then queried.
- Motivation and placement: The paper situates the work clearly against GG(GS)-NNs (fixed graphs) and neural memories (implicit vectors). Graph memory is plausibly more expressive than stacks/arrays and fits relational reasoning tasks (e.g., pathfinding).
- Evidence and rigor: On bAbI, accuracy is comparable to top memory networks, with notable strengths on pathfinding and induction, but not surpassing memory networks overall and worse than rule-induction baselines. The rule-discovery tasks show the machinery can learn and generalize basic algorithmic transformations. Training requires strong supervision of graphs and a “direct reference” mechanism, and the approach is substantially more complex and costly than memory networks. Claims are appropriately scoped; results are correct as far as I can assess.

4. Feedback and questions (to improve the paper; not decisive)
- Please quantify computational and memory complexity vs. MemN2N/DNC and report wall-clock and peak memory.
- Clarify the supervision gap: how does performance degrade without graph supervision or with partial supervision?
- Provide ablations: which transformations matter most (Tadd/TC/Tprop), and how sensitive are results to edge sparsity?
- Could you evaluate on tasks where rule-induction excels to probe generalization beyond bAbI?
- How robust is the direct-reference mechanism to noisy entity mentions and coreference?
- Please release code and exact hyperparameters to aid reproducibility.

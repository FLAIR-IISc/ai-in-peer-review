This paper proposes a novel approach to representing dialog state as a graph, enabling differentiable memory for question answering and rule discovery tasks. The Gated Graph Transformer Neural Network (GGT-NN) learns to construct and modify graphs based on textual input, demonstrating the potential to capture complex relationships between entities. The core contribution lies in implementing a differentiable memory represented as a graph, a significant step beyond prior work utilizing stacks or lists.

**Accept**

My recommendation is to accept this paper for publication, despite the current lack of substantial performance gains over existing methods. The novelty of the approach – representing memory as a differentiable graph – is compelling and potentially groundbreaking. While the empirical results are not yet state-of-the-art, the work lays a solid foundation for future research in this direction, and the technical soundness warrants publication.

**Supporting Arguments:**

The GGT-NN model represents a significant increase in complexity compared to established memory network architectures. This complexity is justified by the theoretical advantage of a graph-based memory representation, which is more expressive than simpler structures like stacks. The author acknowledges the current performance is comparable to strong memory networks but inferior to rule induction methods on bAbI tasks. However, the preliminary nature of the results is explicitly stated, and the successful application to rule discovery tasks (cellular automaton and Turing machines) hints at the potential of the approach for problems where relational reasoning is crucial. 

The paper demonstrates a clear understanding of the existing literature, positioning the GGT-NN within the context of graph neural networks, memory networks, and differentiable neural computers. The detailed description of the graph transformations and the implementation details (including the attention mechanism and GRU updates) indicate a thorough and technically accurate approach. The improvement in clarity from earlier drafts, as noted in the submission materials, is also commendable. The inclusion of detailed appendices explaining the graph format for different tasks further strengthens the paper.

The demonstration of learning complex update rules in the cellular automaton and Turing machine tasks is particularly noteworthy. This suggests the GGT-NN can not only store relational information but also *reason* about it, a capability not readily available in simpler memory models. While these tasks are somewhat artificial, they provide a crucial proof-of-concept for the model’s ability to learn and manipulate graph-structured representations.

**Additional Feedback:**

While the paper is generally well-written, some areas could benefit from further clarification. The interplay between the different graph transformations (node addition, state update, edge update, propagation, aggregation) could be explained more intuitively. A visual example tracing the flow of information through these transformations for a simple bAbI task would be highly beneficial.

The discussion of computational complexity is crucial, given the quadratic scaling with graph size. Exploring potential optimizations (e.g., sparse graph representations, attention mechanisms to limit propagation) would significantly strengthen the paper. A more detailed comparison of the GGT-NN’s computational cost with other memory models (Memory Networks, DNCs) would be valuable.

The observation that direct reference improves performance is important. Investigating *why* direct reference is so effective – does it help with initialization, gradient flow, or something else? – could lead to further insights.

**Questions for the Authors:**

1.  The paper mentions exploring different configurations of graph transformations. Could you provide a more detailed discussion of these configurations and the trade-offs between them? Specifically, under what conditions would you recommend disabling propagation or direct reference?
2.  The cellular automaton and Turing machine tasks demonstrate rule learning. Could you quantify the complexity of the rules learned? Is there a limit to the complexity of rules the GGT-NN can learn, and if so, what factors influence this limit?
3.  How sensitive is the model’s performance to the initial graph structure (e.g., the initial node types and edges)? Does pre-training the graph embeddings improve performance on downstream tasks?
4.  The extended model using the sequence of graphs showed slightly worse performance than the original. What steps can be taken to resolve this performance issue? Would techniques like attention over the sequence of graphs help?
5.  In the paper, it states that *“Setting both [cv,v′,y and rv,v′,y] to zero results in no change for that edge, and setting both to 1 toggles the edge state.”* Is the intent that only one of them is set to 1 at a time? If both are set to 1, does this cause any issues with the gradient flow?

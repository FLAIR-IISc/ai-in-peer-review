### Review of "LEARNING GRAPHICAL STATE TRANSITIONS"

**1. Summary of Contributions**

This paper introduces the Gated Graph Transformer Neural Network (GGT-NN), a novel architecture that extends Gated Graph Sequence Neural Networks (GGS-NNs) to learn and manipulate dynamic, graph-structured internal representations. The core contribution is a set of five differentiable graph transformations—node addition, node state update, edge update, propagation, and aggregation—that allow the model to learn complex state transitions entirely through gradient descent. The authors present a model instance that processes a sequence of textual inputs, iteratively constructing and modifying an internal graph that represents the evolving state of the world described in the text. This model is applied to the bAbI question-answering tasks, where it achieves strong performance, particularly on tasks that are naturally suited to a graph representation like pathfinding. The paper further demonstrates the model's capacity to learn complex, symbolic rules by successfully modeling a 1D cellular automaton and the behavior of arbitrary 4-state Turing machines, purely from sequential examples of their state transitions.

**2. Decision and Key Reasons**

**Decision:** Accept

**Key Reasons:**

1.  The primary reason for acceptance is the proposal of a novel, fully differentiable framework for learning complex graph transformations as an internal memory. This represents a significant conceptual advance over models that use fixed-topology graphs (like GGS-NNs) or unstructured, fixed-size memories (like NTMs and Memory Networks), enabling the model to explicitly and dynamically represent relationships between entities.

2.  The model demonstrates a powerful ability to mix symbolic structures (the graph topology) with continuous representations (node and edge states) and learn complex, algorithmic state updates from data, as convincingly shown by the Turing machine and cellular automaton experiments.

**3. Supporting Arguments**

The main strength of this work lies in its ambitious goal and successful implementation of a learnable, dynamic graph memory. While prior work like Memory Networks and DNCs maintain an internal state, that state is often implicit in a set of vectors. In contrast, the GGT-NN explicitly models the internal state as a graph with labeled nodes and edges. This is a more expressive and interpretable representation, and the experiments show its practical benefits. The outstanding performance on the bAbI pathfinding task (Task 19), where GGT-NN substantially outperforms the strongly-supervised MemNN baseline, is strong evidence that an explicit graph structure is highly advantageous for certain reasoning problems. The rule discovery experiments are equally compelling, showing that the proposed set of transformations is general enough to capture the complex logic of Turing machines and cellular automata, moving beyond simple fact storage towards learning algorithmic procedures.

However, the paper's primary contribution is tempered by a significant practical limitation: the training procedure relies on dense, ground-truth graph supervision at every timestep. For each sentence in a bAbI story, the model is provided with the correct graph it should have produced. While the authors are transparent about this requirement, it significantly limits the model's applicability to real-world tasks where such detailed supervision is unavailable. This makes the comparison to end-to-end models like MemN2N and DNC somewhat indirect, as the GGT-NN receives a much stronger learning signal. Despite this, the work stands as a valuable proof-of-concept for a new class of models and lays the groundwork for future research on reducing this supervision requirement.

**4. Additional Feedback and Suggestions for Improvement**

These points are intended to help strengthen a future version of the paper and are not central to the acceptance decision.

*   **Clarity and Density:** The paper is quite dense, introducing a large amount of new machinery (five distinct transformations, each with its own neural network architecture) in a limited space. While the appendices provide crucial details, moving a more concise mathematical formulation or a more detailed diagram of the transformations into the main body could improve readability. The current format may be better suited for a journal submission where space is less constrained.

*   **Analysis of Failure Cases:** The model fails on bAbI Task 17 (Positional Reasoning). The paper speculates this is due to the large number of entities and short stories. A deeper analysis here would be welcome. Is the model failing to create the correct nodes, connect them properly, or propagate information effectively? Visualizing the generated graphs on this task could provide valuable insights into the model's limitations.

*   **Computational Complexity:** The conclusion briefly mentions that the edge update and propagation steps scale quadratically with the number of nodes, which is a major scalability concern. It would be beneficial to include a brief discussion of the practical implications of this in the main experiments section (e.g., average graph sizes, training times) to give readers a better sense of the model's computational footprint.

*   **Ablation Studies:** The "direct reference" mechanism provides a significant performance boost. An ablation study that explores the contribution of each of the five transformation types would be highly informative. For instance, how crucial is the propagation step (T_prop) for non-pathfinding tasks? Understanding the role of each component would further solidify the paper's contributions.

**5. Questions for the Authors**

1.  The reliance on dense supervision is the main concern. Have you explored any preliminary ideas for reducing or removing this requirement? For example, could a curriculum or annealing strategy be used where graph supervision is faded out over time? Or could a reinforcement learning approach be used, where the model is only rewarded for the final correct answer, forcing it to learn a useful intermediate graph representation on its own?

2.  During training, the correct graph state is substituted in at each step. How does this affect the model at test time, when it must rely on its own, potentially flawed, generated graphs for subsequent steps? Does error accumulate rapidly? Have you investigated the model's robustness to its own mistakes?

3.  How sensitive is the model to the specific graph schema chosen for a task? For the bAbI tasks, the graph representation was derived directly from the simulator's internal "Knowledge" object. If a human were to design a slightly different but equally valid graph schema for the same task, do you have a sense for how robust the training process would be?

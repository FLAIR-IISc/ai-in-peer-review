This paper introduces the Gated Graph Transformer Neural Network (GGT-NN), a novel architecture that extends Gated Graph Sequence Neural Networks to learn and manipulate complex, graph-structured internal states. The core contribution is a set of five differentiable graph transformations (e.g., node addition, edge update, propagation) that allow the model to dynamically build and modify its internal graph based on sequential input, such as text. The authors demonstrate the model's effectiveness on the bAbI question-answering suite and other rule-discovery tasks, where it successfully learns to construct intermediate graph representations and use them for reasoning, achieving promising results.

I recommend **accepting** this paper. The primary reason is the novelty of the proposed framework, which maintains an evolving, explicit graph state while remaining fully differentiable and trainable via gradient descent. This provides a powerful and more general alternative to fixed-shape memory architectures like Memory Networks or NTMs, effectively bridging symbolic, relational structures with continuous representations.

The main strength of this work is its ability to learn sophisticated graph manipulations as part of an end-to-end model. The results on the bAbI tasks, particularly pathfinding, convincingly demonstrate that an explicit graphical representation is superior for problems with inherent relational structure. This work opens up interesting possibilities for tasks requiring complex, dynamic state tracking. However, the model's primary weakness is its reliance on strong supervision; training requires ground-truth graph states at every timestep. This significantly limits its applicability to real-world tasks where such detailed supervision is rarely available. The authors are transparent about this limitation, but it remains a major hurdle for broader adoption.

For future improvement, I suggest the following:
The paper is quite dense and packed with information, which can hinder readability for a conference format. It might be better structured as a journal paper, but for this submission, I would recommend streamlining the main text and moving more implementation details to the appendix to improve clarity and focus on the core concepts.

Finally, I would like to ask the authors:
1.  Have you explored methods to relax the strong supervision requirement? For instance, could a model be trained with a combination of fully supervised and weakly supervised (i.e., answer-only) examples?
2.  The edge update transformation appears to scale quadratically with the number of nodes. Could you comment on the computational complexity and the model's scalability to larger, more complex graph reasoning problems?

1. Introduces Gated Graph Transformer Neural Network (GGT-NN), extending GGS-NNs to learn and manipulate explicit, graph-structured internal states.  
2. Core contribution: five differentiable graph transformations (node addition, node state update, edge update, propagation, aggregation).  
3. Enables dynamic construction and modification of an internal graph from unstructured sequential input while remaining end-to-end differentiable.  
4. Demonstrates strong empirical results: near-perfect on 19/20 bAbI tasks and success on rule-discovery tasks (cellular automata, Turing machines).  
5. Decision: Recommend Accept (Borderline).  
6. Primary acceptance reason: novel, powerful framework for dynamic graph-structured reasoning that integrates symbolic structure with deep learning.  
7. Primary reservation: heavy reliance on strong, step-by-step graph supervision limits practical applicability and comparability to end-to-end models.  
8. Strength: modular, interpretable explicit graph state that's inspectable compared to latent memories in DNCs or Memory Networks.  
9. Strength: excellent performance on graph-suited tasks (e.g., pathfinding) and on learning deterministic state-transition rules.  
10. Weakness: model learns to map text to predefined graphs using graph-matching loss, reducing claims about autonomous reasoning and generalizability.  
11. Practical limitation: computational scaling is quadratic in number of nodes, hindering application to larger problems.  
12. Suggestion: improve clarity by moving implementation details to appendix and adding a detailed data-flow diagram for Algorithm 1.  
13. Suggestion: include ablation on supervision sparsity to quantify dependence on strong intermediate supervision.  
14. Suggestion: provide qualitative error analysis for failure on bAbI task 17 to diagnose graph construction errors.  
15. Question: how is the computational cost of the node addition loss (max permutation) handled in practice when multiple nodes are added?  
16. Question: how robust is the direct reference mechanism to language variations like pronouns or synonyms?  
17. Question: have authors considered alternative training schemes (pretraining with graph supervision, fine-tuning without it, or reinforcement learning) to reduce supervision needs?

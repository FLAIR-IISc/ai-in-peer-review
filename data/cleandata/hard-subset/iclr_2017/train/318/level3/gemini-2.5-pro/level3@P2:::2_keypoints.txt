- Introduces Gated Graph Transformer Neural Network (GGT-NN) that learns to build and modify an explicit graph-structured internal state from sequential unstructured input.
- Extends GGS-NNs by adding differentiable graph transformations for dynamic node/edge creation and modification, learned end-to-end.
- Demonstrates capability by solving 19/20 bAbI tasks and by discovering rules for a cellular automaton and a family of Turing machines.
- Core contribution: a general framework for learning to manipulate explicit, interpretable graphical representations within a neural network.
- Decision: Accept.
- Key reason 1: Novel and significant idea — first system (to reviewer’s knowledge) that learns and modifies graph structure on-the-fly as differentiable memory.
- Key reason 2: Framework is general — effective on QA and algorithmic rule-discovery tasks, showing ability to learn algorithmic state transformations.
- Well-motivated and well-situated in literature, building on GGS-NNs and relating to Memory Networks/DNCs.
- Empirical support is rigorous; bAbI performance competitive and especially strong on pathfinding and induction tasks where explicit graphs help.
- Rule-discovery results strongly suggest the model learns algorithmic graph manipulations rather than task-specific shortcuts; technical details are thorough in appendices.
- Concern: Model complexity is substantially higher than alternatives while bAbI performance gains are not consistently large; need clearer discussion of this trade-off and the value of interpretability.
- Limitation: Requires strong supervision with ground-truth graphs at every timestep, which hinders applicability to real-world tasks.
- Practical issue: Quadratic complexity in node count raises scalability concerns; paper should report practical limits (max graph size, training time).
- Clarity: Paper is much improved but would benefit from simplified notation and more illustrative diagrams given model complexity.
- Question 1: Request elaboration on performance vs. complexity and tasks where explicit graph structure is crucial over DNCs/Memory Networks.
- Question 2: Ask about robustness to noisy or incomplete graph supervision and how performance degrades with missing edges.
- Question 3: Wonders if the "direct reference" mechanism (linking text mentions to node types) could be replaced by a learned attention mechanism to avoid exact name matching.

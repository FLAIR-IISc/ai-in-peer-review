- Introduces Gated Graph Transformer Neural Network (GGT-NN) that learns and manipulates graph-structured internal states for sequential reasoning.  
- Proposes differentiable graph transformations: node addition, state update, edge update, propagation, and aggregation.  
- Builds graph-structured internal state incrementally from natural language input.  
- Demonstrates strong performance on bAbI tasks and impressive rule-discovery on cellular automata and Turing machines.  
- Uses graph-level supervision at each timestep and handles node order considerations.  
- Decision: Accept with Minor Revisions.  
- Novelty/significance: allows the graph structure itself to be learned and transformed, extending prior GNN-based sequential models.  
- Empirical strength: competitive with SOTA on bAbI and convincing capability to learn complex state-transition rules.  
- Methodology: clear description of architecture and transformations.  
- Related work: thorough and well-situated in GNN, sequence modeling, and memory networks literature.  
- Lack of ablation studies to isolate contributions of specific graph transformations and propagation steps.  
- No discussion of hyperparameter sensitivity (learning rates, hidden dimensions, regularization).  
- Computational cost concern: quadratic scaling with graph size; more details and mitigation strategies needed.  
- Request to clarify “direct reference” with concrete examples/attention-weight snippets.  
- Figures 1 and 2 are abstract and would benefit from concrete visual examples of graph transformations.  
- Appendix B is useful but should include pseudocode for implementation clarity.  
- Table formatting: Table 2 should more clearly highlight best results per task.  
- Code availability: ensure a public repository with clear replication instructions.  
- Question: practical scalability limits and techniques explored (graph partitioning, sparse connectivity).  
- Question: generalization beyond bAbI—any experiments on other datasets/tasks?  
- Question: sensitivity to initial graph representation choices (node/edge types, initial connections).  
- Question: comparison and trade-offs versus symbolic approaches for implementing Turing machines.

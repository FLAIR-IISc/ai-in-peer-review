- Introduces Gated Graph Transformer Neural Network (GGT-NN) enabling dynamic, learnable graph-structured intermediate representations and outputs.  
- Core contribution: differentiable graph transformations (node addition, state/edge update, propagation, aggregation) that can be composed and learned end-to-end.  
- Demonstrates effectiveness on bAbI tasks, a 1D cellular automaton, and 4-state Turing machines, showing versatile state-transition learning.  
- Decision: Accept with Minor Revisions — novel and promising approach with compelling experiments but needs clearer presentation and comparisons.  
- Novelty: Advances beyond fixed-graph architectures by learning to build and modify internal graph representations.  
- Experimental validation: Strong empirical results across diverse tasks and insightful ablations (e.g., direct reference).  
- Theoretical foundation: Transformations are well-defined and differentiable, supporting backpropagation.  
- Motivation: Clear case made for graph structures to represent state transitions and limitations of traditional methods.  
- Related work: Needs broader discussion, including recurrent state-space models, Kalman filters, and methods that discover graph structure.  
- Implementation details: Lacks specifics on network architectures and hyperparameters; code is provided but inline explanations would aid reproducibility.  
- Scalability: Authors acknowledge limits; paper should discuss optimizations (sparse representations, efficient propagation) and compare computational cost to alternatives like LSTMs.  
- Visualization: Suggests adding visualizations of learned graphs for cellular automaton and Turing machine tasks for interpretability.  
- Loss function: Node adjustment loss (Lnode) needs clearer explanation, especially permutation max implementation and ordering effects.  
- Question — transformation choices: Why this specific set of graph transformations and were alternatives considered?  
- Question — direct reference: How sensitive is performance to accuracy/noise in the direct reference mechanism?  
- Question — scalability trade-offs: What are the main bottlenecks and have techniques like graph sampling or ANN been tried?  
- Question — generalization: Any anticipated challenges applying GGT-NN to domains like robotics or game playing?  
- Question — hyperparameter sensitivity: How robust are results to choices like learning rate, propagation steps, and hidden state dimensionality?

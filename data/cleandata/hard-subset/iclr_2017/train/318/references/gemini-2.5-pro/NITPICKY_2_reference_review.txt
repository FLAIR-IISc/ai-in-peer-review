**Summary**

The paper introduces the Gated Graph Transformer Neural Network (GGT-NN), a novel architecture designed to learn and manipulate graph-structured state representations from sequential data, such as natural language. The core of the model is a set of five differentiable graph transformations: node addition, node state update, edge update, propagation, and aggregation. These primitives can be composed to construct a network that incrementally builds a graph from input sentences and then uses this graph to answer queries or predict future states. The authors evaluate the GGT-NN on the bAbI question-answering dataset, where it shows strong performance, and on two rule-discovery tasks (a 1D cellular automaton and Turing machine simulation), demonstrating its ability to learn complex, deterministic state transitions. The primary contribution is a framework that maintains an explicit and interpretable graphical internal state, which can be dynamically modified in a differentiable manner.

**Decision: Reject**

While the conceptual framework of differentiable graph transformations is interesting and potentially impactful, the paper in its current form is substantially below the standard for publication at ICLR. The decision for rejection is based on two key reasons:
1.  The submission is critically incomplete and riddled with severe formatting and presentation errors, making a proper scientific evaluation impossible.
2.  The experimental success of the model relies on a level of supervision and task-specific engineering that is not transparently presented and arguably undermines the paper's central claims.

**Supporting Arguments for Decision**

**1. Severe Presentation and Formatting Issues:**
The manuscript is not in a reviewable state.
*   **Missing Figures and Tables:** All figures (1-7) and tables (1-4) are referenced in the text but are not included. These are essential for understanding the model's architecture (Figs. 1, 2, 4), interpreting the results (Tables 1-4), and visualizing the learned representations (Figs. 3, 5, 6, 7). Without this information, the empirical claims made in Section 5 are completely unverifiable.
*   **Corrupted References:** The bibliography is unusable. It appears to be a malformed export from a reference manager, with raw BibTeX-like macros (`? \Q2016\E`) and repeated, garbled entries. This demonstrates a profound lack of care in the preparation of the manuscript.
*   **Technical Typos and Inconsistencies:** There are numerous errors in the technical presentation. For instance, the GRU equations in Section 2.1 are missing the element-wise multiplication operator, a crucial detail. Algorithm 1 contains line numbering errors. The mathematical notation is occasionally non-standard (e.g., `>` for matrix transpose) and insufficiently explained. The paper also lacks an abstract, which is a standard requirement.

**2. Methodological Weaknesses and Lack of Transparency:**
The paper's claims of learning to build graphical states from "unstructured input" are not fully supported by the methodology.
*   **Heavy Reliance on Engineered Supervision:** The model's success hinges on strong supervision, where it is provided with the ground-truth graph at every single timestep. As detailed in Appendix C, the generation of these supervision graphs requires significant, task-specific manual engineering. For example, the authors state they had to add new linked-list structures to represent history for bAbI task 3 and disambiguate relation types by hand based on context. This level of intervention suggests the model is not discovering structure from raw text but rather learning to map text to a pre-designed, highly engineered graphical schema.
*   **Teacher Forcing:** The model uses teacher forcing not just for the output but for the intermediate graph structure itself ("the correct values... are substituted into the graph for further processing"). This means the model never has to learn to recover from its own structural prediction errors, raising serious doubts about its robustness and ability to function in a truly end-to-end setting. The paper admits that end-to-end training was unsuccessful, but the extent of this crutch is a major limitation of the current work.
*   **Unaddressed Complexity:** The node addition loss (`Lnode`) involves a maximization over all permutations (`max π`) of the new nodes. The authors do not discuss the computational complexity of this step, which is naively factorial. It is unclear if an approximation is used or if the number of new nodes is always trivially small. This lack of detail on a core component of the loss function is a notable omission.

**Additional Feedback for Improvement**

*   **Clarity and Style:** The use of the first-person singular ("I") is unconventional. Adopting the plural ("we") would improve the paper's formal tone. The model's name, "Gated Graph Transformer NN," is likely to cause confusion with the now-ubiquitous Transformer architecture (Vaswani et al., 2017). A more descriptive name, such as "Gated Graph Transformation NN," would be clearer.
*   **Future Experiments:** To strengthen the paper's claims, the authors should investigate reducing the level of supervision. Experiments showing how performance degrades as graph supervision is removed (e.g., provided only every N steps, or only at the end) would provide a much clearer picture of what the model is actually learning.
*   **Complexity Analysis:** A discussion of the model's quadratic complexity is present, but it would be beneficial to expand on this with concrete timings and analysis of potential optimizations (e.g., sparse edge matrices or localized updates) that could make the approach more scalable.

**Questions for the Authors**
1.  How was the `max π` operation in the node loss function implemented in practice? What was the typical number of new nodes added at each step in the bAbI experiments, and what was the computational cost of this loss component?
2.  Can you provide a more systematic account of the manual engineering required to create the ground-truth graphs for all 20 bAbI tasks? Understanding the full extent of this pre-processing is critical to evaluating the model's level of autonomy.
3.  Could you quantify the performance of the model when trained with only final answer supervision, even if it is poor? This would establish a crucial baseline and highlight the specific contribution of the intermediate graph supervision.

### Review of "LEARNING GRAPHICAL STATE TRANSITIONS"

#### 1. Summary of Contributions

This paper introduces the Gated Graph Transformer Neural Network (GGT-NN), a novel architecture designed to learn and manipulate dynamic, graph-structured internal states. The core contribution is a framework of "differentiable graph transformations" that allow a model to incrementally build and modify a graph based on sequential input, such as natural language. These transformations include operations for adding nodes, updating node states, modifying edges, propagating information between nodes, and aggregating a graph-level representation.

The authors propose a "soft" graph representation where nodes have existence strengths and edges are represented by a connectivity tensor with continuous values, making the entire structure amenable to gradient-based optimization. The GGT-NN model is demonstrated on two distinct sets of tasks: 1) the bAbI question-answering dataset, where it learns to parse stories into an internal knowledge graph and answer queries, and 2) rule-discovery tasks, where it learns the transition rules for a 1D cellular automaton and arbitrary 4-state Turing machines. The paper claims that by making the internal state explicitly graph-structured and manipulable, the GGT-NN can outperform or be competitive with state-of-the-art memory-augmented networks on tasks that have a natural graphical representation, such as pathfinding.

#### 2. Decision and Key Reasons

**Decision:** Accept

**Key Reasons:**

1.  **Significant Conceptual Novelty:** The paper presents a compelling and well-motivated extension of Graph Neural Networks from operating on static graphs to learning to dynamically construct and transform them. The proposed set of differentiable graph transformations is an elegant and general framework that addresses a fundamental limitation of prior GNN models. This shift from processing a fixed graph to learning the graph-building process itself is a significant contribution to the field.
2.  **Strong Empirical Validation:** The claims are well-supported by a thoughtful set of experiments. The success on bAbI tasks, particularly the high performance on inherently graphical problems like pathfinding (Task 19) and induction (Task 16), provides strong evidence for the benefits of an explicit graphical state. Furthermore, the experiments on cellular automata and Turing machines serve as a powerful proof of concept, demonstrating that the model can learn complex, generalizable algorithmic rules for state transitions, which is the central premise of the work.

#### 3. Supporting Arguments

The primary strength of this work lies in its novel formulation. While previous models like GG-NNs and GCNs operate on pre-defined graph structures, and memory-augmented networks like DNCs and MemN2N have dynamic internal states, the GGT-NN bridges a critical gap. Its internal state is not just a collection of memory vectors but an explicit, interpretable graph with nodes and typed edges. This allows for distributed computation across the graph (as shown by the propagation step) and provides a structured representation that is potentially more aligned with certain reasoning tasks. The modularity of the proposed transformations (add, update, propagate) is a key feature, allowing the architecture to be flexibly configured for different problems.

The experimental design is strong. Using bAbI showcases the model's ability to ground natural language into a structured knowledge base. The comparison against a wide range of SOTA models is thorough. The authors are transparent about the fact that their model, like the original MemNN, uses strong supervision, which is a crucial detail for fair comparison. The rule-discovery tasks are particularly compelling. By removing the ambiguity of natural language, these experiments isolate the model's ability to learn the core graph transformation logic. The model's success in generalizing to longer simulation sequences on these tasks indicates that it is not merely memorizing trajectories but learning the underlying update rules.

However, the paper's primary limitation is its reliance on strong, step-by-step graph supervision during training. For each sentence in a bAbI story, the model is provided with the complete ground-truth graph it is expected to produce. This is a very powerful supervisory signal that is unavailable in most real-world scenarios. The authors rightly acknowledge this, but it significantly constrains the immediate applicability of the proposed method. The learning problem is framed as "learn to produce this sequence of graphs," rather than "discover a useful graph structure to solve the task." While this is a reasonable first step for such a complex model, it is a major caveat.

#### 4. Additional Feedback for Improvement

*   **Scalability and Complexity:** The edge update transformation (`TC`) and the propagation step (`Tprop`) appear to scale quadratically with the number of nodes, as they consider all pairs of nodes. This is a significant bottleneck that limits the model to small graphs. While the conclusion mentions this as future work, a brief discussion in the main paper about potential mitigations (e.g., using sparse attention mechanisms, locality-sensitive updates) would strengthen the paper.
*   **The "Transformer" Naming:** The name "Gated Graph Transformer Neural Network" could be confusing, especially in light of the now-dominant attention-based Transformer architecture (Vaswani et al., 2017). The model "transforms" graphs, but it does not use the self-attention mechanism that defines the Transformer. This is a minor point but could be worth clarifying.
*   **Discussion of Appendix D:** The model variant in Appendix D, which processes a sequence of graphs to answer a query, is a very interesting idea for handling temporal tasks without complicating the graph structure itself. This feels like more than an appendix-level contribution and could be worth summarizing in the main body, as it presents an alternative solution to the temporal reasoning challenges in tasks like bAbI #3.
*   **Ablation Studies:** The GGT-NN is a complex system with many moving parts (five transformation types, direct reference, intermediate propagation). An ablation study, even on a single bAbI task, would provide valuable insight into the relative importance of each component. For instance, how crucial is the explicit edge update (`TC`) compared to just relying on node state updates and propagation?

#### 5. Questions for the Authors

1.  **Sensitivity to Supervision:** How critical is the dense, step-by-step graph supervision? Have you experimented with sparser supervision schemes, such as providing the ground-truth graph only after every few sentences, or only the final graph before the query? Understanding how performance degrades as supervision is removed would be crucial for assessing the model's potential for wider application.
2.  **Qualitative Error Analysis:** For the tasks where the model struggled, like bAbI Task 17 (Positional Reasoning), can you provide a qualitative analysis of the generated graphs? Does the model fail to create the correct nodes, or does it struggle with inferring the correct relational edges? Visualizing an incorrect graph could provide significant insight into the model's failure modes.
3.  **Interpretability of Learned Rules:** In the Turing machine experiments, have you attempted to analyze the learned weights of the propagation and update functions? Is it possible to interpret the learned rules and see if they correspond in some way to the actual symbolic rules of the given Turing machine? This could be a powerful demonstration of the model's interpretability.

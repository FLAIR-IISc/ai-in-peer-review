### **Review of "Learning Graphical State Transitions"**

#### **1. Paper Summary**

This paper introduces the Gated Graph Transformer Neural Network (GGT-NN), a novel architecture designed to build, maintain, and reason over a dynamically changing graphical internal state. The core contribution is a set of differentiable graph transformations that allow a neural network to manipulate a graph structure in response to sequential input, such as natural language. These transformations include adding nodes, updating node states, modifying edges, propagating information between nodes, and aggregating a graph-level representation.

The authors demonstrate the GGT-NN's capabilities on two distinct classes of problems. First, on the bAbI question-answering dataset, the model learns to parse stories, construct an internal knowledge graph representing the state of the world, and answer questions based on this graph. Second, on two rule-discovery tasks—simulating a 1D cellular automaton and arbitrary 4-state Turing machines—the model demonstrates its ability to learn complex, generalizable state transition rules purely from graphical examples. The paper positions this work as an extension of Gated Graph Neural Networks (GG-NNs) to dynamic graphs and as an alternative to memory-augmented networks (like DNCs or Memory Networks) that provides a more explicit, structured, and interpretable internal state.

#### **2. Decision and Key Reasons**

**Decision:** Accept

**Key Reasons:**

1.  **Novel and Significant Contribution:** The paper presents a well-defined and powerful framework for learning dynamic graph manipulations. The concept of "differentiable graph transformations" is a novel and significant contribution that elegantly extends the capabilities of graph neural networks from static processing to dynamic construction and reasoning. This opens a promising research direction for models that need to maintain an explicit, structured world model.
2.  **Strong Empirical Validation:** The claims are supported by extensive experiments on a diverse set of challenging tasks. The model achieves state-of-the-art or competitive performance on the bAbI tasks, notably excelling in tasks with a natural graph structure like pathfinding (Task 19). Furthermore, the success on the cellular automaton and Turing machine tasks provides compelling evidence that the framework can learn complex, distributed algorithms, which is a strong testament to its expressive power.

#### **3. Supporting Arguments for Decision**

**Strengths:**

*   **Explicit and Interpretable State:** A key advantage of the GGT-NN over related architectures like DNCs or Memory Networks is its explicit graphical state. While memory networks have an unstructured memory and DNCs have implicit temporal/content-based links, the GGT-NN maintains a graph with typed nodes and edges. This structure is not only powerful for relational reasoning but is also highly interpretable and can be visualized, as demonstrated in Figure 3. This allows for direct inspection of what the model has learned.
*   **Compositionality and Modularity:** The framework is built on a set of modular transformations (add, update, propagate, etc.). As the authors note, these components can be reconfigured to tackle different problem settings, making the architecture flexible. This design is principled and well-conceived.
*   **Demonstrated Algorithmic Learning:** The experiments on cellular automata and Turing machines are particularly impressive. These tasks require the model to learn precise, local rules that result in complex global behavior. The model's ability to learn these rules and generalize to longer simulation times indicates that the propagation and update mechanisms are effective at capturing algorithmic state transitions.
*   **Honest Evaluation and Comparison:** The authors are transparent about the model's reliance on strong supervision for the bAbI tasks. They correctly position their results alongside other strongly supervised methods (MemNN) and distinguish them from end-to-end models. This transparency is commendable and allows for a fair assessment of the results.

**Weaknesses and Counterarguments:**

*   **Reliance on Strong Supervision:** The most significant limitation is the need for full, step-by-step graphical supervision during training. For each sentence in a bAbI story, the model is provided with the ground-truth graph it is supposed to produce. This requirement is a major obstacle for applying the model to real-world problems where such detailed supervision is unavailable. While the authors acknowledge this, the paper would be stronger if it explored the effects of sparser supervision. However, as a proof of concept for a new modeling paradigm, this is an acceptable simplification. The framework itself is the main contribution, and future work can address the supervision challenge.
*   **Scalability:** The edge update transformation (`TC`) considers all pairs of nodes, resulting in a time and space complexity of O(|V|^2) per step, where |V| is the number of nodes. This quadratic scaling limits the model to relatively small graphs. The authors acknowledge this downside, and while it is a practical limitation, it does not invalidate the core contribution for problems within its feasible scale.

#### **4. Additional Feedback and Suggestions for Improvement**

*   **Clarity on the term "Transformer":** The name "Gated Graph Transformer Neural Network" could be misleading given the now-ubiquitous "Transformer" architecture (Vaswani et al., 2017), which is based on self-attention. This paper's use of "transform" refers to the graph transformations, not the specific attention-based mechanism. A brief note in the introduction or related work could clarify this to prevent confusion for modern readers.
*   **Deeper Analysis of Failure Case:** The model fails to achieve high accuracy on bAbI Task 17 (Positional Reasoning). The authors suggest this is due to the large number of entities and short stories. A more detailed error analysis would be beneficial. Is the model failing to create the correct nodes, establish the correct edges, or is the final reasoning step (propagation/aggregation) insufficient? This could reveal more about the model's limitations.
*   **Exploring Weaker Supervision:** The future work section rightly points to reducing the supervision level. Even a small preliminary experiment showing how performance degrades as graph supervision is removed (e.g., only providing the final graph, or supervising only 10% of steps) would significantly strengthen the paper by outlining the boundaries of the current approach.

#### **5. Questions for the Authors**

1.  The reliance on full, step-by-step graphical supervision is a key point of discussion. Have you performed any experiments to test the model's robustness to sparser supervision signals? For example, how does the model perform if trained end-to-end with only the final question-answer pair as supervision, possibly with regularization to encourage graph-like structures?
2.  Could you elaborate on the practical scalability limits of the model? What was the maximum number of nodes and edges in your experiments, and what were the corresponding training times? Do you see a path toward mitigating the O(|V|^2) complexity, perhaps through sparse updates or an attention mechanism that selects which node pairs to update?
3.  For the pathfinding task (bAbI Task 19), how many propagation steps (`T_prop`) were required for the model to find the solution? Does this number correlate with the length of the shortest path in the ground-truth graph? This could provide insight into whether the model is learning a distributed bellman-ford or breadth-first search-like algorithm.

### **ICLR Review**

**Paper: LEARNING GRAPHICAL STATE TRANSITIONS**

**1. Summary of Contributions**

This paper introduces the Gated Graph Transformer Neural Network (GGT-NN), a novel architecture designed to process sequential, unstructured input (such as natural language), dynamically construct and modify an explicit graph-structured internal state, and then use this state for reasoning and question answering. The core contribution is a set of differentiable graph transformations: node addition, node state update (including a "direct reference" mechanism), edge update, propagation, and aggregation. These transformations serve as modular building blocks that allow the model to learn how to manipulate its internal graph representation in response to input.

The authors demonstrate the GGT-NN's capabilities on two distinct sets of tasks. First, on the 20 bAbI question-answering tasks, the model achieves state-of-the-art or competitive performance, excelling on tasks that are naturally suited to a graphical representation, such as pathfinding and induction. Second, on rule-discovery tasks (simulating a 1D cellular automaton and arbitrary Turing machines), the model shows it can learn complex, deterministic update rules purely from sequential graph states, and generalize these rules to longer sequences. The paper positions the GGT-NN as a bridge between unstructured data processing and explicit, interpretable, structured reasoning, distinguishing it from GNNs that operate on fixed graphs and from memory-augmented networks (like DNCs or MemNets) that use implicit, unstructured memory stores.

**2. Decision and Key Reasons**

**Decision:** Accept

**Key Reasons:**
1.  **High Novelty and Significance:** The paper proposes a novel and powerful framework for learning to manipulate explicit graph structures. The concept of a composable, differentiable set of graph transformations is a significant contribution that opens a new direction for reasoning models. It elegantly combines the strengths of sequence models for input processing with the explicit relational reasoning capabilities of graph networks, while crucially allowing the graph structure itself to be dynamic.
2.  **Strong Empirical Support for Claims:** The model's effectiveness is well-supported by strong results on the bAbI benchmark, particularly its superior performance on graph-centric tasks (e.g., Task 19, pathfinding). Furthermore, the rule-learning experiments convincingly demonstrate that the proposed transformations are expressive enough to capture complex algorithmic processes, which validates the fundamental design of the architecture.

However, the primary reason this is not a strong accept is the model's heavy reliance on full, step-by-step graph supervision during training. While the authors are transparent about this limitation, it significantly lowers the bar for the learning problem and raises questions about the model's applicability to tasks where such detailed supervision is unavailable.

**3. Supporting Arguments for Decision**

The main strength of this work lies in its conceptual advance. Traditional GNNs are powerful feature extractors for data that is *already* structured as a graph. Memory networks and DNCs can reason over sequences but store information in a relatively unstructured memory bank, making explicit relational reasoning difficult to interpret and enforce. The GGT-NN carves out a compelling middle ground: it learns to *create the structure* from unstructured data. The modular transformations (T_add, T_C, T_prop, etc.) are a well-designed, intuitive toolkit for this purpose. The results on bAbI Task 19 (Pathfinding) and Task 16 (Induction) are particularly telling; the model outperforms many strong baselines, which strongly suggests that an explicit graphical representation provides a powerful inductive bias for these types of relational reasoning.

The experiments on cellular automata and Turing machines are also a crucial piece of evidence. They demonstrate that the GGT-NN is not just a pattern-matcher for specific NLP tasks but a more general framework capable of learning algorithmic state transitions. The ability to learn these rules and generalize to longer time horizons shows that the propagation and update mechanisms are capturing the underlying dynamics of the systems, which is a very strong result.

The main weakness, which must be acknowledged, is the supervision requirement. At each step of processing the bAbI stories, the model is given the ground-truth graph it is supposed to have constructed. The loss is computed on the model's ability to produce this intermediate graph. This means the model is not discovering the relational structure from the end-task signal alone; rather, it is learning a complex, supervised mapping from text to pre-defined graph structures. This makes the comparison with end-to-end trained models like DNCs and EntNets somewhat difficult, though the authors are fair in highlighting the comparison to the similarly-supervised MemNN. While this limitation is significant, the novelty of the architecture and the demonstration of its potential are strong enough to warrant acceptance. The paper rightfully frames a reduction in supervision as a key direction for future work.

**4. Additional Feedback and Questions for the Authors**

The following points are intended to help improve the paper and are not critical to my decision.

*   **Complexity Analysis:** The paper correctly notes that the model's complexity scales quadratically with the number of nodes due to the all-pairs edge update transformation. It would be very helpful to include statistics on the typical/maximum graph sizes (number of nodes and edges) encountered in the bAbI and rule-discovery experiments. This would provide context for the practical scalability of the current approach.
*   **Analysis of "Direct Reference":** The direct reference mechanism appears crucial for good performance, as shown in Tables 1 and 2. The description in Section 4 is a bit brief. It seems to function as a simple but effective form of attention, routing information from specific words to nodes of the corresponding type. Could you elaborate on why this is so much more effective than allowing the model to learn this routing implicitly through the general node state update (Th)? Does the general update struggle to associate words like "Mary" with the "Mary" node type without this explicit pointer?
*   **Failure Case Analysis:** The model fails to achieve high accuracy on bAbI Task 17 (Positional Reasoning). The paper hypothesizes this is due to the large entity space and short stories. This is plausible, but a more detailed analysis would be welcome. For example, by inspecting the generated graphs, can you identify if the model fails to create the correct nodes, connect them with the right edges, or fails during the final reasoning/propagation steps?
*   **Node Addition Stability:** The node addition transformation (T_add) uses a recurrent function that learns to set a node's strength `s_v` to 0 to indicate that no more nodes should be added. How stable is this mechanism? Does it sometimes fail by adding too many or too few nodes, and how sensitive is it to initialization or other hyperparameters?

**Clarifying Questions:**

1.  Could you elaborate on the decision to replace the learned propagation matrices from GG-NNs with learned functions (`f_fwd`, `f_bwd`) in the propagation step (T_prop)? What advantage did this provide in your experiments?
2.  Regarding the supervision, have you experimented with curriculum learning or annealing strategies where the graph supervision is gradually faded out, forcing the model to rely more on the final answer's loss signal? This seems like a promising first step towards reducing the supervision requirement.

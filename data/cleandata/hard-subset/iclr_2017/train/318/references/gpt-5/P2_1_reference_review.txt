1) Summary of contributions
- Problem: The paper proposes learning to build and update an explicit, differentiable graph-structured state while processing text, and then using that state for reasoning and generation. This goes beyond GG-NN/GGS-NN by allowing the graph’s structure (nodes, edges, strengths) to be modified over time, not just propagate over a fixed graph.
- Method: The Gated Graph Transformer Neural Network (GGT-NN) defines a set of differentiable graph transformations: node addition, node state update (including a “direct reference” variant), edge update, propagation, and aggregation. Nodes/edges carry continuous “existence”/strengths in [0,1]. The model processes sentences with a GRU, applies graph transformations per sentence, and answers queries via further propagation/aggregation. Training uses strong supervision with losses on edges and newly added nodes (with permutation-invariant assignment following Vinyals et al., 2016).
- Empirical results: On bAbI, with full access to the generator’s knowledge object (converted to graphs), GGT-NN attains high accuracy on most tasks; direct reference and intermediate propagation improve several tasks. On synthetic rule-learning tasks, the model learns 1D CA (Rule 30) and 2-symbol 4-state Turing machines from graph sequences and generalizes to longer rollouts.

Overall, this is an ambitious and interesting attempt to unify text processing with explicit, learned graph construction and transformation, with a modular, well-described set of differentiable operators.

2) Decision: Reject
Key reasons:
- Heavy reliance on strong supervision and engineered signals (full access to ground-truth graphs; direct-reference based on string matches) undermines the central claim of learning to construct useful graph states from unstructured input.
- Novelty over prior graph neural networks and differentiable memory models appears incremental; the empirical evaluation lacks rigorous ablations and fair comparisons to strong end-to-end baselines without privileged supervision.

3) Supporting arguments
- Validity and rigor of claims:
  - The primary claim is that GGT-NNs can learn to incrementally build and use graph-structured internal states from text. However, the bAbI experiments require graph-level supervision at each step (including gold edges and node labels) and “direct reference” built from exact string matches to node types. This supervision largely defines the target graph and shortcuts the hard part of learning to ground text into entities/relations. The model without direct reference struggles on many tasks, reinforcing that the approach depends on these privileged signals.
  - Comparisons: The paper compares to models trained with weak supervision (e.g., MemN2N, DNC) and notes MemNN also uses strong supervision (supporting facts). But the GGT-NN uses even stronger supervision (full graphs); thus cross-model comparisons are not apples-to-apples. It is unclear how GGT-NN performs with only answer supervision (end-to-end), which is the more realistic QA setting.
  - Synthetic tasks (CA, Turing machines) are useful to probe rule learning. The results suggest the operators can encode local transition rules and generalize depth. However, experimental rigor is limited: no reporting of variance, error modes, or sensitivity to hyperparameters; details about model capacity, propagation steps, and complexity are sparse in the main text.
- Novelty and positioning:
  - The core idea—differentiable, dynamic graph construction with message passing—is a natural extension of GG-NN/GGS-NN. The specific design choices (continuous node/edge strengths; node-addition RNN; edge set/reset networks; “direct reference” typed updates) are sensible but feel like engineering variants rather than a fundamentally new principle. The related work discussion is broad but does not engage with contemporaneous works on learning dynamic graphs or structured state machines at scale; citations are inconsistently formatted and several key recent graph reasoning works are missing.
- Clarity and completeness:
  - The paper is generally clear about the transformations and provides pseudocode and appendix details. However, essential training details are missing from the main paper (hyperparameters, propagation steps, size of hidden states, edge types per task, enforcement of simplex constraints on node types). Tables referenced are not shown here; reproducibility depends on code, but it is not evident that all settings are documented.
- Practicality and scaling:
  - The authors acknowledge O(|V|^2) time/space behavior. There is no empirical scaling analysis (runtime/memory vs. graph size) or experiments on larger, real-world datasets with noisy entity mentions and relations. The approach’s reliance on direct reference and exact node types raises concerns about robustness to natural language variation.

4) Additional feedback (to help improve the work)
- Demonstrate end-to-end learning without graph-level supervision on bAbI (1k and 10k) using only answer supervision; report the performance gap and ablate the contribution of each transformation (node add, edge set/reset, propagation depth, direct reference).
- Replace string-matching “direct reference” with learned mention grounding (e.g., attention over tokens) and show performance and robustness to paraphrases and multiword entities.
- Provide fair comparisons where all models have similar supervision levels; if strong supervision is essential, include alternative baselines that also receive the same graph targets (e.g., GGS-NN over gold graphs; seq2seq graph parser).
- Include qualitative analyses: visualizations of learned graphs over time; failure cases; learned edge confidences; whether the model creates spurious nodes/edges.
- Report training details: architecture sizes, number of propagation steps per phase, optimizer, learning rates, regularization, early stopping, and compute resources. Add variance over multiple seeds.
- Address scalability: sparse adjacency, local neighborhoods, or sampling strategies to move beyond quadratic costs.
- Clarify how constraints are enforced (e.g., node type distributions summing to 1; [0,1] bounds on strengths) and whether projection/penalties are used.

Questions for the authors
- How does GGT-NN perform on bAbI with only answer supervision (no graph supervision, no direct-reference strings)? Can you report results and learning curves?
- How is the simplex constraint on node types and [0,1] constraints on node/edge strengths enforced in training? Sigmoid/softmax layers or explicit projection?
- What are the exact hyperparameters (hidden sizes, number of propagation steps, number of nodes added per step, edge MLP sizes)? How sensitive are results to these?
- How many edge types are used per task, and how were they designed (especially for bAbI where relation names were split into multiple edge types)?
- For the Turing machine experiment: how is generalization measured for 20/30-step rollouts? Are the rule graphs fixed and only states evolve, or does the model also have to maintain rule edges? How does accuracy degrade as rollout length increases?
- Can the method handle ambiguity/noise in mentions (coreference, paraphrases) without direct reference? Any preliminary experiments?
- What is the runtime and memory footprint vs. graph size, and how large a graph can you handle in practice?

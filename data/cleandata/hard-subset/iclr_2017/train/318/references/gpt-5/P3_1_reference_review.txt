1) Summary and contributions
The paper proposes Gated Graph Transformer Neural Networks (GGT-NNs), an extension of GG-NNs/GGS-NNs that maintains a fully differentiable, editable graph state with soft node types, node existence strengths, and soft, typed edges. It defines a modular set of differentiable transformations: node addition (Tadd), node state update with optional direct reference (Th, Th,direct), edge update (TC), message passing/propagation (Tprop), and graph-level aggregation (Trepr). A particular configuration is used to parse sequences of sentences into a graph and answer queries. Training can be strongly supervised by providing ground-truth intermediate graphs; a permutation-invariant node-addition loss is described, and edges are trained with a Bernoulli cross-entropy. Experiments cover bAbI QA (with graph supervision) and two “rule discovery” tasks (Rule 30 cellular automaton and 2-symbol 4-state Turing machines), showing strong accuracy on 19/20 bAbI tasks with direct reference and promising generalization to longer trajectories on the synthetic dynamics tasks. The paper argues for advantages over memory-based architectures due to explicit, interpretable graph state and distributed computation over nodes.

2) Decision: Reject
Key reasons:
- The central empirical results rely on extensive strong supervision (ground-truth graphs and oracle entity “direct reference”) and heavily engineered graph schemas, limiting the significance of the claims about learning to construct/manipulate graphs from unstructured text.
- The novelty relative to existing graph neural network families (GG-NNs/GGS-NNs, message passing neural nets, dynamic graph models) is incremental and insufficiently disentangled experimentally; important methodological details (e.g., the permutation matching for node addition) are underspecified.

3) Supporting arguments
- Motivation and placement. The goal—learning state transitions over explicit graphs built from text—is compelling. However, similar capabilities have been explored in: GG-NN/GGS-NN (Li et al., 2016) for sequential outputs; message passing architectures (Gilmer et al., 2017), dynamic graph learning (edge inference; e.g., NRI/Kipf et al., 2018), and the Graph Networks framework (Battaglia et al., 2018) that factorizes node/edge updates and supports dynamic edges. The incremental elements here are (i) soft node existence/type distributions, (ii) a modular set of transformations including node addition and edge update, and (iii) the training recipe with strong supervision. The paper does not convincingly differentiate GGT-NN’s capabilities from these predecessors in settings without strong supervision.
- Strong supervision and engineered schemas. For bAbI, the model is trained to match the generator’s hidden “knowledge” object converted into a bespoke graph schema with many typed edges, and “direct reference” is provided via lexical matching from entity mentions to node types. The main positive results (19/20 tasks above 95%) depend on both supervision and direct reference. Without direct reference, 9/20 tasks exceed 95%, and the model needs more data. This makes the contribution less about learning to parse and structure text, and more about consuming a near-oracle graph supervision signal. The comparison to end-to-end models (MemN2N, DNC, etc.) is therefore not on equal footing; the paper notes this but still draws conclusions about representational superiority from those numbers.
- Missing or underspecified details. The node-addition loss uses a max over permutations to match an unordered set of newly added nodes to ground-truth nodes. How is this implemented computationally? A naive max over permutations is infeasible; a linear assignment/Hungarian relaxation or bipartite matching is standard, but this is not described. Moreover, after computing the loss, the method “substitutes” ground-truth nodes/edges into the state during training (teacher forcing), which can induce exposure bias; there is no analysis of mismatch at test time when the model’s own predictions feed subsequent steps.
- Ablations and analysis. Beyond the direct-reference on/off toggle and intermediate propagation for selected tasks, there is no ablation to quantify the contributions of Tadd vs TC vs Tprop, the impact of soft node strengths, or supervision choices (e.g., training only with answer supervision). There is also no comparison to text-to-graph baselines (e.g., neural semantic parsers) or to dynamic edge inference baselines for the synthetic tasks. The extension to sequence-of-graphs (Appendix D) shows slightly worse results and is not deeply analyzed.
- Scalability and practicality. Time/space complexity is quadratic in the number of nodes and multiplicative in the number of edge types, and each edge type has its own parameters (f_fwd/f_bwd), which can explode in realistic schemas. The paper acknowledges scalability issues but does not provide strategies (e.g., sparse ops, neighbor sampling, type sharing, learned low-rank parameterization) or empirical scaling curves. The bAbI and synthetic tasks are very small.
- Empirical scope. The rule discovery tasks are nice demonstrations that the model can learn local update rules and sometimes generalize to longer horizons. However, they encode the transition table as explicit rule nodes/edges and supervise the next-step graphs, which makes them closer to differentiable emulation of given rules than learning latent structure from observations. No baselines or error analyses are provided (e.g., where/why generalization fails).

4) Feedback to improve the paper
- Clarify the permutation-invariant matching for node addition. If you use Hungarian/bipartite matching, specify the cost and differentiability. If you approximate the max, describe the algorithm and its complexity.
- Report training/inference complexity and memory with respect to |V| and |E|, number of edge types, and provide wall-clock times. Explore sparse message passing or subgraph sampling.
- Provide ablations isolating: edge update vs propagation; with/without node strengths; per-type vs shared f_fwd/f_bwd; replacing ground-truth substitution with scheduled sampling; removing graph supervision (answer-only training). Quantify performance drops.
- Fair baselines for bAbI under similar supervision (e.g., MemNN with knowledge-graph supervision, or GGS-NN on the same graph inputs), and end-to-end baselines with an entity linker rather than oracle direct reference. Consider modern text-to-graph parsers on AMR, OpenIE, or knowledge base population to demonstrate broader applicability.
- Reduce engineered edge-type explosion by parameter sharing or compositional relation embeddings; compare to relation-embedding approaches.
- Include qualitative analyses: visualize learned graphs, attention/propagation paths used to answer queries, and failure cases.
- Discuss robustness: performance under noisy entity mentions or missing supervision; impact of mismatched node ordering at train/test.

5) Questions for the authors
- How exactly do you compute the max over permutations in Lnode? Is it reduced to a linear assignment problem? Is the procedure differentiable or used only to select targets?
- During training you “substitute” correct nodes/edges into the graph after computing the loss. At test time these are not available. Did you observe exposure bias? Did you try scheduled sampling or partial substitution?
- What are the parameter counts and per-step runtime/memory as functions of node count and edge types? How large a graph can you handle in practice?
- Are f_fwd/f_bwd shared across edge types, or distinct per type? If distinct, how do you prevent overfitting and parameter blowup when Y is large?
- Can the model be trained with only answer supervision on bAbI (no graph supervision)? If so, what accuracies do you obtain, with and without direct reference?
- How sensitive are results to errors in direct reference (e.g., ambiguous names, coreference)? Can you replace direct reference with a learned pointer/entity linker?
- In the Turing machine task, do you ever withhold the explicit rule nodes/edges and ask the model to infer them from trajectories? If not, could you comment on feasibility?
- Could you compare to a GGNN/GGSNN variant that receives the ground-truth graph as input (no construction) to isolate the benefit of learning state transitions vs using provided structure?
- What is the effect of intermediate propagation on tasks beyond 3 and 5? Any guidance on when to enable it?

Overall, the paper presents an appealing modular view of graph-state construction and manipulation, but the heavy reliance on strong supervision, limited experimental scope, and missing details reduce its impact in its current form. I encourage the authors to strengthen the empirical and methodological clarity as above.



# Review of "Learning Graphical State Transitions"

## 1. Summary of Contributions
The paper introduces the Gated Graph Transformer Neural Network (GGT-NN), which extends previous graph neural network approaches by allowing graph-structured intermediate representations and graph-structured outputs. The key innovation is a set of differentiable graph transformations (node addition, node state update, edge update, propagation, and aggregation) that enable the model to modify graph structures during processing. The authors demonstrate the model's effectiveness on bAbI tasks (reaching 95%+ accuracy on most tasks), cellular automata, and Turing machines, arguing that the graphical representation provides advantages for certain structured reasoning tasks.

## 2. Decision
Reject. The paper contains numerous technical inconsistencies and methodological ambiguities that undermine confidence in the proposed approach.

## 3. Supporting Arguments

The paper suffers from multiple technical inconsistencies that make it difficult to assess the validity of the proposed method:

1. **Inconsistent mathematical formulations**: The paper uses inconsistent spacing around mathematical operators (e.g., "h̃(t) = φ ( Wx + U ( r(t) h(t−1) ) + b )") and inconsistent notation (e.g., "GG-NN" vs. "GGNN"). In the node addition transformation (B.1), the paper states that "node hidden states hv are initialized to zero" but then uses "hi" in equation (4), which is confusingly defined as "recurrently computing sv and xv for each new node." This suggests a circular dependency that isn't properly explained.

2. **Confusion between belief and probability**: The paper defines node strength "sv ∈ [0, 1]" as "the level of belief that node v should exist," but then uses it in a loss function as if it were a probability (Lnode = −s*π(v) ln(sv) + (1− s*π(v)) ln(1− sv)). This conflates belief (which may not be probabilistic) with probability, creating a conceptual inconsistency.

3. **Ambiguous graph construction**: For the bAbI tasks, the paper claims to use "graphical representations" but provides insufficient detail about how these graphs are constructed from the textual input. The paper states "the knowledge object provides names for each node type, and direct reference was performed based on these names," but doesn't explain how these node types were derived or how the graph structure was determined for training.

4. **Insufficient experimental details**: The paper claims "the GGT-NN model was trained with full graph information" but fails to specify how this graph information was generated or what the graph representations actually look like for the bAbI tasks. For the rule discovery tasks, the paper states "no additional information was provided as textual input," but doesn't explain how the initial graph structure was generated for training.

5. **Inconsistent edge update formulation**: In the edge update transformation (B.3), the paper describes "cv,v′,y gives the level of belief in [0, 1] that an edge from v to v′ of type y should be created if it does not exist," but the equation "C′v,v′ = (1− Cv,v′) cv,v′ + Cv,v′ (1− rv,v′)" suggests cv,v′ is the probability of creating an edge if it doesn't exist and rv,v′ is the probability of removing an edge if it exists. The description doesn't match the equation.

These inconsistencies are not merely superficial; they undermine the core technical contribution of the paper. The paper's claim that the model can learn to manipulate graph structures is questionable when the graph structure itself isn't properly defined or explained.

## 4. Additional Feedback

- **Clarify the graph construction process**: Provide a detailed description of how the graph representations are generated from the raw input for each task. Include examples of the graph structure for specific bAbI examples.

- **Fix mathematical notation consistency**: Standardize spacing around mathematical operators, use consistent notation (e.g., always use "GG-NN" or "GGNN"), and ensure all equations match their descriptions.

- **Explain the loss functions more carefully**: For the node adjustment loss, clarify why the model can treat "strength" as a probability when it was defined as a "level of belief."

- **Provide more experimental details**: Include exact accuracy values for all tasks, not just a subset. Clarify the validation procedure and hyperparameter tuning process.

- **Address the graph initialization issue**: Explain how the graph structure is initialized for the rule discovery tasks (cellular automata and Turing machines) without using the "full graph information" that was used for the bAbI tasks.

## 5. Clarification Questions

1. In the node addition transformation (B.1), you state that "node hidden states hv are initialized to zero" but then use "hi" in equation (4). Could you clarify how the recurrence works when the hidden state is being initialized to zero?

2. For the rule discovery tasks, how is the initial graph structure generated for training? The paper states "no additional information was provided as textual input," but the model still needs some initial graph structure to learn from.

3. In the edge update transformation (B.3), the description of cv,v′,y doesn't match the equation. Could you clarify what cv,v′,y and rv,v′,y actually represent in the equation?

4. For the bAbI tasks, could you provide an example of how the graph representation is constructed from a specific input sentence? The paper mentions "direct reference" but doesn't show how the graph structure reflects this.

5. In the node adjustment loss (4.1), you state "only the edges and the new nodes are replaced by the supervision." What happens to the hidden states of existing nodes during training? The paper mentions they are "propagated without adjustment," but this seems inconsistent with the backpropagation process.

The paper has interesting ideas, but the technical inconsistencies and methodological ambiguities are too significant to recommend acceptance. Addressing these issues would substantially strengthen the paper.

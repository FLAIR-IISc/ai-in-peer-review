- Introduces Gated Graph Transformer Neural Network (GGT-NN) that enables differentiable graph structure transformations (node add/update, edge update, propagation, aggregation) to build internal graph states from unstructured input.  
- Shows strong empirical performance: 95%+ on 19/20 bAbI tasks and large improvement on pathfinding (Task 19), plus generalization to cellular automata (Rule 30) and Turing machine simulation.  
- Main contribution: dynamic, learnable graph-structured internal states extending GNNs beyond fixed-input graphs.  
- Recommendation: Accept — significant, well-motivated contribution with thorough evaluation.  
- Empirical evaluation is comprehensive and demonstrates clear advantages of graphical representations for relational/spatial tasks.  
- Comparison to related work (memory networks, DNCs, etc.) is fair and highlights the explicit graph-structured internal state as a differentiator.  
- Technical formulations and implementation details are sound, including supervised training with graph-level supervision for nodes and edges.  
- Paper acknowledges quadratic time/space complexity and suggests future directions but lacks detailed scaling analysis.  
- Suggestion: Clarify differences between GGT-NN and GGS-NN (e.g., add a comparative figure).  
- Suggestion: Provide more thorough analysis of computational complexity and practical scalability to larger graphs (e.g., 100+ nodes).  
- Suggestion: Include ablation studies to quantify the contribution of each of the five graph transformations.  
- Suggestion: Improve presentation of results/tables to more clearly highlight which tasks benefit most from graph structure.  
- Question: Why does the direct reference mechanism yield only minimal improvement on Task 17 (Positional Reasoning) despite complex entity representation?  
- Question: Request quantitative metrics showing how model performance changes with increasing input size (e.g., more timesteps for cellular automaton).  
- Question: Have techniques been explored to mitigate quadratic complexity (sparse graphs, sampling, approximations) to enable larger-graph applicability?  
- Question: Request detailed explanation of how bAbI knowledge objects were mapped to graph format, especially for history-dependent tasks like Task 3.



# Peer Review: "LEARNING GRAPHICAL STATE TRANSITIONS"

## 1. Summary of Claims and Contributions

This paper introduces the Gated Graph Transformer Neural Network (GGT-NN), a novel architecture that extends previous graph neural network approaches (GG-NN and GGS-NN) by enabling the learning of graphical state transitions. The key contribution is a differentiable framework for graph transformations that allows the model to build and modify graph structures from unstructured input (e.g., text), rather than requiring fixed graph structures as input. The paper formalizes five graph transformations (node addition, node state update, edge update, propagation, and aggregation), which can be composed to create flexible architectures. The authors demonstrate the model's effectiveness on bAbI tasks (reaching 95%+ accuracy on most tasks), cellular automaton rule discovery, and Turing machine simulation, showing that graphical representations are particularly effective for tasks like pathfinding. The paper makes a compelling case that explicit graph representations offer advantages over implicit memory structures for certain structured reasoning tasks.

## 2. Decision

**Reject**. While the paper presents an interesting extension to graph neural networks with a well-motivated problem statement, the technical contribution lacks sufficient novelty, the experimental validation is incomplete and potentially misleading, and the paper fails to adequately address significant limitations of the approach.

## 3. Supporting Arguments

**Novelty and Motivation**: The paper's core contribution—differentiable graph transformations—is not sufficiently novel. The authors build on existing work (GG-NN, GGS-NN), but the differentiable graph definition they propose (with node strengths and fractional edge weights) is a straightforward extension of prior work. The paper does not convincingly demonstrate how this differs from or improves upon existing approaches to differentiable graph manipulation (e.g., the work on differentiable graph operations by Xu et al. or the GNNs with learnable edge types in the literature). The motivation for "learning graphical state transitions" is valid, but the paper doesn't clearly distinguish its approach from existing methods that handle dynamic graph structures.

**Experimental Validation**: The experimental section has significant issues. The bAbI task results are misleading because the GGT-NN model was trained with full graph-level supervision (including the correct graph at each timestep), while the comparison models (MemNN, MemN2N, etc.) were trained end-to-end without this additional supervision. This creates an unfair comparison—effectively, the GGT-NN is trained with a much stronger signal. The paper acknowledges this but fails to demonstrate that the model can achieve similar performance with less supervision. The ablation study on "direct reference" is superficial and doesn't quantify how much of the performance gain comes from this mechanism versus the graph representation itself.

**Computational Limitations**: The paper acknowledges the quadratic complexity of the model (Section 6), which is a severe limitation for practical applications. However, it provides no empirical evidence of this limitation's impact on performance with larger graphs, nor does it propose any strategies to mitigate it (e.g., sparsity, sampling). The paper claims the model "can be used as-is for tasks consisting of a sequence of input sentences and graphs," but the computational constraints would make this impractical for most real-world applications.

**Rule Discovery Tasks**: The cellular automaton and Turing machine experiments are interesting but lack sufficient detail. The paper doesn't explain how the graphical representation was designed for these tasks, making it impossible to determine whether the model is learning the underlying rules or simply memorizing the graph transformations. The paper also doesn't provide error analysis or compare to baselines that might use similar graphical representations.

## 4. Additional Feedback for Improvement

**1. Clarify the novelty**: The paper should more clearly distinguish its contribution from existing work on differentiable graph operations. A thorough comparison with recent methods (e.g., GraphSAGE, GAT, or differentiable graph operations in the literature) is essential to establish the novelty.

**2. Address the supervision issue**: The paper must provide experiments demonstrating that the model can achieve comparable performance with less supervision. A fair comparison would train the GGT-NN without the full graph-level supervision, or include a study on how much supervision is needed for good performance.

**3. Quantify the "direct reference" mechanism**: The paper claims this is crucial for performance, but lacks a proper ablation study. It should measure the performance drop when this mechanism is removed to determine its actual contribution.

**4. Discuss computational limitations**: The paper should include experiments showing how performance and training time scale with graph size. It should also propose or discuss potential solutions to the quadratic complexity problem (e.g., sparse connections, sampling).

**5. Provide more detail on the graphical representations**: For the rule discovery tasks, the paper should explain how the graphical representation was designed for each task and why it's appropriate. A visualization of the graph structure during training would help clarify what the model is learning.

**6. Compare to more recent methods**: The paper should compare against more recent graph neural network approaches (e.g., GraphSAGE, GAT, or GNNs with learnable edge types) to demonstrate the relative advantages of the proposed framework.

**Questions for Authors**:
- How does the GGT-NN's performance compare to a baseline that uses the same graphical representation but without the differentiable graph transformations (e.g., using a fixed graph structure with GNNs)?
- What specific ablation experiments would you run to demonstrate that the model is learning the underlying rules of the cellular automaton and Turing machine, rather than just memorizing the graph transformations?
- Could you provide evidence that the model can learn to build graphs without the full graph-level supervision that was used in the bAbI experiments?

The paper presents an interesting direction for graph-based neural networks, but the current version lacks sufficient novelty, experimental rigor, and acknowledgment of limitations to warrant acceptance at ICLR. With the suggested improvements, particularly addressing the supervision issue and computational limitations, the paper could make a stronger contribution to the field.

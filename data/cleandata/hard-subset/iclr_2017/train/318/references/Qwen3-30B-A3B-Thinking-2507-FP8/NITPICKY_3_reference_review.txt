

# Review of "LEARNING GRAPHICAL STATE TRANSITIONS"

## 1. Summary of Contribution

The paper proposes the Gated Graph Transformer Neural Network (GGT-NN), which extends previous graph neural network approaches by introducing differentiable graph transformations that allow for graph-structured intermediate representations and outputs. The authors claim that GGT-NN can process unstructured input (like text) to construct a graph-based internal state, which can then be used to answer questions or model complex systems like cellular automata and Turing machines. The paper demonstrates the approach on the bAbI tasks and two rule discovery tasks, showing competitive performance compared to existing methods.

## 2. Decision

Reject. The paper contains fundamental technical errors in the background section that undermine the credibility of the entire approach, and there are multiple inconsistencies in notation, definitions, and experimental descriptions that make it difficult to assess the validity of the claims.

## 3. Supporting Arguments

The most critical issue is a fundamental technical error in the background section (Section 2.1). The GRU equations are presented incorrectly:

"h̃(t) = φ ( Wx + U ( r(t) h(t−1) ) + b )"

This equation is missing the input vector x(t), which is essential for the GRU operation. The correct equation should be:

"h̃(t) = φ ( Wx x(t) + U ( r(t) ⊙ h(t−1) ) + b )"

This is a severe error in a background section that describes a fundamental building block of the proposed model. The authors have misstated the basic operation that their model relies on, which raises serious doubts about the technical soundness of the paper.

Additionally, the paper contains multiple instances of inconsistent notation and unclear definitions:

- In Section 3, the paper states "Each node also has a strength sv ∈ [0, 1]" but later refers to "s(v)" in the loss function, creating confusion about the notation.
- The paper states "direct reference enabled" in Algorithm 1 without defining what this means or how it's implemented, creating ambiguity in the algorithm's operation.
- The paper claims to be trained with "1,000 training examples" for bAbI tasks, but this is ambiguous - does this mean 1,000 examples per task or 1,000 examples total? The bAbI dataset has 1,000 examples per task, so this phrasing is misleading.
- In Section 4.1, the paper states "only the edges and the new nodes are replaced by the supervision. The hidden states of all existing nodes are propagated without adjustment." This is contradictory because if hidden states are "propagated," they should be updated, not left unchanged.

The paper also has formatting inconsistencies throughout, including missing parentheses in pseudocode (Algorithm 1) and inconsistent use of "GGT-NN" vs. "Gated Graph Transformer Neural Network" in the text.

## 4. Additional Feedback

To improve the paper, the authors should:

1. **Correct the fundamental GRU error in Section 2.1** and ensure all mathematical formulations are technically precise. This is non-negotiable for a technical paper.

2. **Clarify all notation** consistently throughout the paper. Define all variables before first use and maintain consistent notation. For example, use either "sv" or "s(v)" consistently for node strength.

3. **Revise ambiguous claims** about training data. Be precise about the number of training examples per task (e.g., "1,000 examples per task" rather than "1,000 training examples").

4. **Fix the algorithm pseudocode** by adding missing parentheses and clarifying the meaning of "direct reference enabled."

5. **Provide more detail on how the model learns update rules** for the cellular automaton and Turing machine tasks. The paper states the model "learns complex update rules" but doesn't explain the learning process or how the graph transformations enable this learning.

6. **Address the contradiction** in Section 4.1 regarding "propagation without adjustment" of hidden states. If hidden states are propagated, they should be updated according to the propagation equations.

7. **Proofread the paper** carefully for minor grammatical errors and formatting inconsistencies, which are particularly noticeable in technical papers.

## Questions for Authors

1. Can you clarify the exact meaning of "direct reference enabled" in Algorithm 1 and how it's implemented in the model?

2. How exactly does the model learn the update rules for the cellular automaton and Turing machine tasks? The paper states the model "learns complex update rules" but doesn't explain the learning process in sufficient detail.

3. In Section 4.1, you state "the hidden states of all existing nodes are propagated without adjustment." This contradicts the propagation operation described in Section 2.2 and Appendix B.4. Can you clarify this apparent contradiction?

4. For the bAbI tasks, was the model trained with 1,000 examples per task (as is standard) or 1,000 examples total across all tasks? This needs to be made clear.

The paper presents an interesting idea, but the fundamental technical error in the background section and the numerous inconsistencies in notation, definitions, and experimental descriptions make it impossible to assess the validity of the claims. I recommend rejection due to these substantial technical flaws.

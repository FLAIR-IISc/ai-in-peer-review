

# Review of "LEARNING GRAPHICAL STATE TRANSITIONS"

## 1. Summary of Contributions

This paper introduces the Gated Graph Transformer Neural Network (GGT-NN), which extends Gated Graph Neural Networks (GG-NNs) by enabling the construction and modification of graph structures based on unstructured input. The key contribution is a set of differentiable graph transformations (node addition, node state update, edge update, propagation, and aggregation) that allow the network to build and manipulate graph-structured internal states. The authors demonstrate the model's effectiveness on three distinct tasks: the bAbI question-answering tasks (where the model achieves 95% accuracy on 19/20 tasks with direct reference), a 1D cellular automaton (Rule 30), and arbitrary 4-state Turing machines. The paper positions GGT-NNs as addressing a significant gap in existing approaches by allowing graph structures to be learned from unstructured input rather than requiring fixed graph structures.

## 2. Decision

Reject. While the paper presents a novel architecture with promising empirical results, it lacks sufficient theoretical analysis and fails to adequately address the computational complexity of the approach, which is a significant limitation for practical applications.

## 3. Supporting Arguments

The paper's main strengths lie in its clear motivation and strong empirical results. The GGT-NN model effectively demonstrates the ability to learn graph-structured representations from textual input and solve tasks where graph representations are naturally suited (like pathfinding in bAbI task 19). The comparison with existing methods on the bAbI tasks is well-executed, showing that the GGT-NN with direct reference outperforms several baselines, particularly on tasks where graphical representations provide advantages.

However, the paper has significant weaknesses that prevent acceptance. First, it lacks any theoretical analysis of the model's properties. The authors introduce several graph transformations but provide no analysis of their representational capacity, convergence guarantees, or theoretical bounds on the model's capabilities. This is particularly concerning for a conference like ICLR, where theoretical foundations are highly valued.

Second, the paper fails to quantify the computational complexity of the GGT-NN architecture. The authors mention in the conclusion that "the time and space required to train the model increase very quickly as the complexity of the task increases," but provide no empirical evidence of this scaling behavior. Without this analysis, it's impossible to assess the practical limitations of the approach, especially for larger graph structures.

Third, some aspects of the method are not clearly explained. For instance, the node addition process and the handling of unordered sets of nodes could be clarified with more detailed examples or pseudocode.

## 4. Additional Feedback for Improvement

To strengthen the paper for future submission, the authors should:

1. Provide a thorough theoretical analysis of the GGT-NN architecture, including representational capacity and convergence properties of the graph transformations.

2. Quantify the computational complexity of the model by providing runtime and memory measurements as a function of graph size and number of transformations. This would help readers understand the practical limitations of the approach.

3. Clarify the node addition process, particularly how the network determines when to add new nodes and how it handles the arbitrary ordering of nodes (which is currently described as "the network arbitrarily chooses an ordering").

4. Expand the comparison with related work to include more recent graph neural network approaches (e.g., GraphSAGE, Graph Attention Networks) to better position the contribution.

5. Consider providing ablation studies to isolate the contribution of each graph transformation to the overall performance.

## 5. Questions for Authors

1. Can you provide a more detailed analysis of the computational complexity of the GGT-NN model, particularly how time and space requirements scale with graph size (number of nodes and edges) and the number of transformations applied?

2. How does the model handle graphs with a variable number of nodes during inference, and what are the limitations of this approach for tasks requiring large-scale graph structures?

3. Could you elaborate on the specific mechanisms that allow the GGT-NN to learn complex update rules (like Rule 30) without explicit rule specification, and how this compares to other approaches for learning rule-based systems?

4. The paper mentions "strong supervision" with full graph information as a requirement. What specific experiments could be conducted to reduce this supervision requirement (e.g., using semi-supervised learning or self-supervision), and how might this affect performance?

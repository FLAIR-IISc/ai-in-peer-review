- Introduces Gated Graph Transformer Neural Network (GGT-NN) for differentiable graph construction and modification.
- Redefines graphs with continuous node "strengths" and edge "beliefs" to enable differentiability.
- Proposes five differentiable graph transformations: node addition, node state update, edge update, propagation, and aggregation.
- Presents a framework combining these transformations to build/manipulate graphs from unstructured input (e.g., text).
- Demonstrates GGT-NN on bAbI QA tasks, Rule 30 cellular automata, and Turing machines.
- Recommends acceptance with minor revisions due to strong contribution and evaluation.
- Clearly motivates the problem: existing GNNs’ fixed structures vs. need for learned graph construction, with bAbI as a compelling example.
- Technical innovation: continuous strengths and differentiable transformations are substantial and described with reproducible detail.
- Empirical evaluation is comprehensive: high bAbI accuracy (95%+), large improvement on pathfinding (task 19), and successful rule discovery.
- Ablation study (with/without direct reference) offers useful insight into model behavior.
- Thorough comparison to related work (GNNs, GGS-NNs, Memory Networks, DNCs), clearly distinguishing GGT-NN’s contributions.
- Needs clearer explanation of how node addition and other transformations are made differentiable, especially stopping criteria.
- Should include comparisons with weaker/partial graph supervision to address fairness vs. end-to-end baselines.
- Should discuss scalability and complexity beyond noting quadratic cost, including potential optimizations or approximations.
- Improve readability: simplify dense algorithm descriptions and add intuitive explanations or diagrams.
- Asks how the model handles differentiable node addition and learning when to stop adding nodes.
- Requests discussion of related works in differentiable programming/neural program synthesis and how GGT-NN differs.
- Seeks explanation for why direct reference improves performance—attention vs. deeper graph-construction reasons.
- Asks for specific scaling optimizations (e.g., whether sparse representations suffice or more advanced methods are needed).

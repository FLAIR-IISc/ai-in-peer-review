

# Review of "LEARNING GRAPHICAL STATE TRANSITIONS"

## 1. Summary of Contribution

This paper introduces the Gated Graph Transformer Neural Network (GGT-NN), which extends prior graph neural network approaches by enabling neural networks to construct and modify graph-structured internal states from unstructured input (such as natural language). The authors propose a set of differentiable graph transformations (node addition, node state update, edge update, propagation, and aggregation) that allow the network to incrementally build and modify graphs. The GGT-NN model is demonstrated on three types of tasks: bAbI question-answering tasks (where the model constructs a graph representing the world state from natural language input), 1D cellular automaton learning (Rule 30), and simulating 4-state Turing machines. The paper shows that the GGT-NN model achieves high accuracy on these tasks, particularly excelling at pathfinding (bAbI task 19) and tasks requiring relational reasoning (bAbI task 16), which benefit from the graphical representation.

## 2. Decision

Accept. The paper presents a technically sound extension to graph neural networks with clear novelty in enabling graph construction and modification from unstructured input, and provides strong empirical evidence supporting the claims.

## 3. Supporting Arguments

**Technical Soundness**: The paper provides clear mathematical formulations for the differentiable graph transformations and their implementation details. The model architecture is well-motivated as an extension of prior work (GG-NNs and GGS-NNs), with the key innovation being the ability to construct graphs rather than process fixed graphs. The supervision method for learning graph structures (using the Vinyals et al. approach for unordered sets) is appropriate and well-explained. The implementation details in Appendix B are thorough, allowing for reproducibility.

**Novelty**: The paper clearly differentiates itself from prior work. Unlike GNNs, GGS-NNs, or Memory Networks, the GGT-NN model can construct and modify graph structures from unstructured input rather than processing fixed graphs. The paper's comparison with related work (Section 6) effectively positions this contribution within the literature. The ability to use graph-structured internal states for tasks that benefit from graphical representations (like pathfinding) is a significant contribution.

**Empirical Rigor**: The experimental evaluation is comprehensive. The bAbI task results (Table 2) show the GGT-NN model achieves 95%+ accuracy on 19/20 tasks (with direct reference), outperforming a simple LSTM baseline and showing competitive results against state-of-the-art approaches. The model's success on the pathfinding task (bAbI task 19) is particularly compelling, as it demonstrates the model's ability to solve a task naturally suited to graphical representation. The cellular automaton and Turing machine tasks (Table 3) further validate the model's ability to learn complex graph transformation rules.

## 4. Additional Feedback

- The paper would benefit from a more detailed discussion of the computational complexity tradeoffs. While the quadratic scaling is acknowledged, a quantitative analysis of how this affects training time and memory requirements across different task sizes would strengthen the practical implications.

- The paper mentions that the GGT-NN model with direct reference outperforms the model without it, but the specific mechanisms by which direct reference improves performance could be more clearly explained. A case study showing how direct references are used in specific examples would help readers understand this component.

- The paper could include a qualitative analysis of the constructed graphs, showing examples of how the graph structure evolves during the processing of input sentences, to better illustrate the model's internal state manipulation.

## 5. Questions for Authors

1. In the bAbI experiments, the model with direct reference achieves significantly better results than the model without it (Table 2). Could you provide more insight into how the direct reference mechanism specifically improves performance, perhaps by showing examples of sentences where direct reference makes a critical difference?

2. The paper notes that the GGT-NN model's quadratic complexity limits its applicability to larger graphs. Could you provide more details about the maximum graph size your model was able to handle in your experiments, and how this compares to the graph sizes used in the bAbI tasks?

3. For the Turing machine simulation task, the paper states that the model was trained on 20,000 examples. Given the complexity of the task, how many unique Turing machine rules were included in the training data, and how did the model generalize to rules not seen during training?

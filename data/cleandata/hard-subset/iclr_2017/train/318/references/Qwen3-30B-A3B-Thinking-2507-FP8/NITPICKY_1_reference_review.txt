

# Review of "Learning Graphical State Transitions"

## 1. Summary of Contributions
The paper introduces the Gated Graph Transformer Neural Network (GGT-NN), which extends Graph Neural Networks by allowing graph-structured intermediate representations and graph-structured outputs. The authors propose a set of differentiable graph transformations (node addition, node state update, edge update, propagation, and aggregation) that can be combined to process graph structures in complex ways. The paper demonstrates the model on the bAbI tasks, a cellular automaton task, and Turing machine simulation, claiming that the GGT-NN can learn to construct meaningful graphical internal states from natural language input.

## 2. Decision
Reject. The paper contains significant technical inconsistencies and lacks sufficient experimental rigor to support its claims.

## 3. Supporting Arguments

### Critical Technical Inconsistencies
1. **Graph Definition Contradictions**: The paper defines "each node has an annotation xv ∈ RN" and "∑N j=1 xv,j = 1" (Section 3), but later uses N as an index in "xv,j" while N is also used as the dimensionality of the annotation vector. This is a fundamental notational confusion that undermines the technical foundation of the paper.

2. **Node Strength Contradiction**: The paper states "sv = 1 means the node exists, and sv = 0 indicates that the node should not exist" (Section 3), yet later in Section B.1, it claims "the recurrent function fadd can learn to output sv = 0 for some nodes to create fewer nodes." This is logically inconsistent—how can a node with sv=0 exist in the graph if it's supposed to not exist?

3. **Edge Loss Function**: The edge loss function is defined as "Ledge = −C* · ln(C)− (1− C*) · ln(1− C)" (Section 4.1). The use of the dot product (·) is incorrect here—it should be element-wise operations. This is a critical technical error that would prevent proper implementation.

### Experimental Rigor Issues
1. **Incomplete Task 17 Analysis**: The paper states "the only task that the GGT-NN was unable to solve with 95% accuracy was task 17" but fails to provide the actual accuracy numbers for this task. Without these numbers, it's impossible to assess whether the failure is significant or merely a minor performance gap.

2. **Insufficient Comparison**: The paper claims "the GGT-NN model was trained using only 1,000 training examples" but doesn't provide a fair comparison with other models trained on the same number of examples. The comparison with other models is made without controlling for training data size, making the claims about efficiency and performance questionable.

3. **Lack of Statistical Significance**: The paper claims "the GGT-NN model with direct reference reaches 95% accuracy on 19/20 of the bAbI tasks" but provides no statistical significance tests to confirm these results are meaningful.

### Presentation and Clarity Issues
1. **Inconsistent Notation**: The paper uses "h̃(t)" in Section 2.1 but fails to properly format the tilde in some equations, creating confusion about what the notation represents.

2. **Undefined Symbols**: The paper introduces "Γ" as the set of graphs in Section 3 without defining it previously. The symbol is used without explanation, making it difficult for readers to follow.

3. **Ambiguous References**: The paper frequently uses "it" without clear antecedents, making it difficult to follow the logic of the proposed methods.

## 4. Additional Feedback

### Key Questions for Authors
1. Can you clarify the relationship between N (the dimension of the node annotation vector) and the index j used in "∑N j=1 xv,j = 1"? Is N the number of node types or the dimensionality of the annotation vector?

2. How does the model handle node addition when sv = 0 for a node? If the node doesn't exist, how can it be part of the graph structure?

3. Can you provide the exact accuracy numbers for task 17 (Positional Reasoning) to help assess the significance of the failure?

4. What ablation studies have you performed to determine which graph transformation components are most important for the model's performance?

### Suggested Improvements
1. **Fix Notational Inconsistencies**: Clearly define all symbols before their first use and maintain consistent notation throughout the paper.

2. **Provide Complete Experimental Details**: Include exact accuracy numbers for all tasks, especially task 17, and provide statistical significance tests for all claims.

3. **Clarify Graph Transformation Mechanics**: Provide more detailed explanation of how node strength affects the graph structure, particularly for nodes with sv = 0.

4. **Improve Comparison with Baselines**: Ensure fair comparison by training all models on the same number of examples and reporting results consistently.

5. **Add Qualitative Analysis**: Include examples of the graphs constructed by the model to demonstrate that the graphical representations are meaningful and useful for the tasks.

The paper presents an interesting idea for extending graph neural networks, but the technical inconsistencies and lack of rigorous evaluation prevent me from recommending acceptance. The authors should address these fundamental issues before resubmitting.

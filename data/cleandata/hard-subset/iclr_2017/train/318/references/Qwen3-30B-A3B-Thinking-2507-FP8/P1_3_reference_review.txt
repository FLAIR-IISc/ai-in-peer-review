

# Peer Review: "LEARNING GRAPHICAL STATE TRANSITIONS"

## 1. Summary of Contributions

This paper introduces the Gated Graph Transformer Neural Network (GGT-NN), a novel neural network architecture that extends Graph Neural Networks (GNNs) and Gated Graph Sequence Neural Networks (GGS-NNs) by allowing the model to learn and manipulate graph-structured internal states. The key contribution is a set of differentiable graph transformations that enable the model to dynamically build and modify graph structures during processing, rather than being limited to fixed graph inputs. The authors demonstrate the model's capabilities on three main tasks: the bAbI question-answering tasks (where the model constructs a graph from natural language input), a 1D cellular automaton (Rule 30), and arbitrary 4-state Turing machines. The model achieves strong results, particularly on tasks where graph representation is naturally suited (like pathfinding), and outperforms baselines on several bAbI tasks. The paper makes a compelling argument for the value of explicit graph-structured internal representations compared to more implicit representations used in other architectures like Memory Networks.

## 2. Decision

**Accept** with minor revisions.

The paper makes a significant contribution by extending the scope of graph neural networks to allow for dynamic graph structure learning. The work is well-motivated, technically sound, and demonstrates clear empirical advantages over previous approaches on relevant tasks.

## 3. Supporting Arguments for Decision

The paper addresses a meaningful limitation of current graph neural network approaches: the inability to learn graph structures from unstructured input. The authors clearly position their work within the literature, showing how it extends previous GNN and GGS-NN work (Li et al., 2016) while addressing their limitations. The theoretical foundation for the differentiable graph transformations is sound, and the paper provides sufficient detail on the implementation of the five transformation types (node addition, node state update, edge update, propagation, and aggregation).

The empirical results are scientifically rigorous and well-structured. The authors provide:
- Strong baselines on the bAbI tasks (including comparisons to 10+ state-of-the-art approaches)
- Ablation studies showing the importance of direct reference (improving accuracy from 9/20 tasks to 19/20 at 95% accuracy)
- Two additional tasks (cellular automaton and Turing machines) demonstrating the model's ability to learn complex graph-based rules
- Analysis of generalization to longer sequences in the cellular automaton task

The most compelling result is the performance on task 19 (pathfinding), where the GGT-NN model with direct reference outperforms all other models except one (DMN+), showing a 64% improvement over the MemNN baseline. This is particularly significant because it demonstrates the model's ability to solve a task naturally suited to graph representation using textual input without requiring pre-processed graphical data.

The paper also makes a strong argument for the explicit graph representation by showing that the GGT-NN model's internal state can be extracted and visualized, unlike the implicit representations in Memory Networks or DNCs.

## 4. Additional Feedback for Improvement

While the paper is strong, I recommend the following improvements to enhance clarity and impact:

1. **Clarity on Differentiability**: The paper states the graph transformations are differentiable but doesn't provide sufficient detail on how the loss functions are backpropagated through the graph structure modifications (especially for node ordering in the node loss function). A brief explanation of the computational graph for the node loss would strengthen the technical contribution.

2. **Computational Complexity Analysis**: The paper mentions the quadratic complexity with respect to the number of nodes but doesn't provide any analysis of how this impacts practical applications. A discussion of scalability (e.g., how the model performs with graphs containing 100+ nodes) would be valuable.

3. **Hyperparameter Sensitivity**: The paper uses 1,000 training examples for bAbI tasks, but doesn't discuss how the model's performance changes with different amounts of data. A small ablation study showing how performance scales with training data size would strengthen the results.

4. **Conciseness**: Some sections (like the background on GRUs and GG-NNs) could be shortened to focus more on the novel contributions. The paper would benefit from tighter editing to make the key contributions more prominent.

5. **Comparison to Graph-Transformer Architectures**: The paper doesn't discuss how the GGT-NN relates to recent graph transformer architectures (like those in the "Graph Transformer" family). A brief comparison would help situate the work within the current landscape of graph neural network research.

## 5. Questions for the Authors

1. The paper mentions that the model can learn arbitrary graph-based rules (as demonstrated in the cellular automaton and Turing machine tasks). Could the authors elaborate on the theoretical limits of the model's ability to learn different types of graph transformations, particularly for more complex rule sets?

2. In the bAbI tasks, the authors note that task 17 (Positional Reasoning) was particularly challenging due to the large number of possible entities. How might the model be extended to handle even larger numbers of entities without performance degradation?

3. The paper states that the graph transformations are differentiable, but doesn't discuss potential numerical stability issues when training with the edge loss function (Ledge = -C*·ln(C) - (1-C*)·ln(1-C)). Have the authors encountered any stability problems during training, and if so, how did they address them?

4. The paper mentions that the GGT-NN model with direct reference outperforms the model without it on 19/20 tasks. What specific aspects of the direct reference mechanism are most responsible for this improvement, and how might this be generalized to other tasks?

Overall, this is a strong contribution that extends the capabilities of graph neural networks in a meaningful way. The paper makes a compelling case for the value of explicit graph-structured internal representations and demonstrates the model's effectiveness on multiple challenging tasks. With the suggested revisions, this paper would make a valuable contribution to ICLR.

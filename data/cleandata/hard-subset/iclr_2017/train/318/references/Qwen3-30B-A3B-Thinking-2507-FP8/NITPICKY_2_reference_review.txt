

# Review of "Learning Graphical State Transitions"

## Summary of Contributions
The paper introduces a novel Gated Graph Transformer Neural Network (GGT-NN) model that extends Graph Neural Networks by enabling graph-structured intermediate representations and graph-structured outputs. The key contribution is a set of differentiable graph transformations (node addition, node state update, edge update, propagation, and aggregation) that can be composed to process graphs in complex ways. The authors demonstrate the model on bAbI tasks, cellular automata, and Turing machines, claiming significant improvements over baseline models on certain tasks, particularly those involving graph-structured reasoning.

## Decision
Reject. The paper contains multiple technical inconsistencies and formatting errors that undermine confidence in the methodology and results.

## Supporting Arguments

The paper's technical presentation contains numerous critical inconsistencies that would make replication difficult and cast doubt on the validity of the results:

1. **Notational Inconsistency**: The paper defines a graph as G = (V, C) in Section 3 but then uses G = (V, E) in Section 2.2, creating confusion about the graph representation. The notation for edge types also shifts between "ye" (Section 2.2) and "Cv,v',y" (Section 3) without clear explanation.

2. **Technical Errors in Formulas**: In Section 2.1, the GRU equations contain a clear notational error: "h̃(t) = φ ( Wx + U ( r(t) h(t−1) ) + b )" should be "h̃(t) = φ ( Wx(t) + U ( r(t) ⊙ h(t−1) ) + b )" where ⊙ denotes element-wise multiplication (as noted in the text but missing in the equation).

3. **Confusing Loss Function**: Section 4.1 presents a node adjustment loss function with inconsistent indexing: "Lnode = −max π |Vnew|∑ v=|Vold|+1 s*π(v) ln(sv) + (1− s*π(v)) ln(1− sv) + x*π(v) · ln(xv)". The notation "s*π(v)" is undefined (π appears to be a permutation, but this is never explained), and the summation index is unclear.

4. **Misrepresentation of Rule 30**: In Section 5.2.1, the paper describes Rule 30 as "the binary cellular automaton known as Rule 30" but Rule 30 is not binary - it's a cellular automaton with two states (0 and 1) but the rule applies to 8 possible neighborhoods, not binary inputs.

5. **Inconsistent Training Details**: The paper states "the model was trained using only 1,000 training examples" (Section 5.1) but then says "results using 10,000 examples have also been reproduced here for comparison" (Section 5.1.1), without clarifying which results correspond to which training set size.

6. **Unjustified Claims**: The paper claims "the model with direct reference reaches 95% accuracy on 19/20 of the bAbI tasks" without providing statistical significance testing for these improvements over the non-direct-reference model.

7. **Formatting Inconsistencies**: References contain inconsistent formatting (e.g., "Vinyals et al.,? \Q2016\E, Vinyals et al\." vs. "Vinyals et al. (2016)"), and the paper uses both "GGS-NN" and "GGS-NNs" without clear distinction.

These inconsistencies are not merely cosmetic; they represent fundamental issues with the paper's technical presentation that would make replication and verification difficult. As a reviewer, I am required to be meticulous about these details, and the accumulation of these minor but significant flaws leads me to conclude that the paper does not meet ICLR's standards for technical rigor.

## Additional Feedback

1. The paper would benefit from a more precise mathematical formalization of the graph transformations, including clear definitions of all variables and consistent notation throughout.

2. The claims about the superiority of the GGT-NN model should be supported by statistical significance tests for the reported performance differences.

3. The relationship between the proposed model and existing graph neural network approaches needs to be more clearly articulated. The comparison with related work (Section 6) is descriptive rather than analytical.

4. The paper should clarify the exact training details, including how many examples were used for the main results versus the 10,000 examples mentioned for comparison.

5. The claim that "any ordering should be sufficient" for the node ordering (Section 4.1) contradicts the earlier statement that "the network arbitrarily chooses an ordering which may not be the optimal ordering for the task."

These issues, while seemingly minor, collectively undermine the paper's technical credibility. A paper of this nature requires precise mathematical formulation and consistent presentation to be considered for publication at ICLR. The current version would benefit from significant revision to address these concerns before reconsideration.

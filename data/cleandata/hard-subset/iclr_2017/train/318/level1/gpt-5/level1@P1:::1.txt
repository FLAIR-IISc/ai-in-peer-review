Summary of contributions
The paper proposes the Gated Graph Transformer Neural Network (GGT-NN), a neural architecture that: (i) maintains an explicit, differentiable graph as an internal state; (ii) incrementally constructs and updates this graph from unstructured (text) input; and (iii) uses it to generate answers or graph outputs. Key technical contributions include: a soft graph representation with node-type distributions and node/edge strengths; a suite of differentiable graph transformations (node addition, node-state update with an optional “direct reference” mechanism, edge update, propagation, and aggregation); and a training scheme with strong supervision that injects ground-truth graph structure at intermediate steps, including a permutation-invariant node addition loss. Empirically, the model achieves strong results on many bAbI tasks (95%+ on 19/20 with direct reference), and learns transition rules for synthetic graphical systems (Rule 30 cellular automaton and 2-symbol, 4-state Turing machines), with some generalization to longer rollouts. The paper articulates how GGT-NN relates to GNN/GGS-NN, memory networks, and DNCs, highlighting interpretability and distributed computation over an explicit graph.

Decision: Reject
Key reasons: (1) The main empirical results rely on privileged, task-specific strong supervision (ground-truth graphs at every step and engineered graph schemas), making it unclear whether the approach works end-to-end from answers only. (2) The paper lacks crucial experimental rigor (ablations, hyperparameters, training details, compute/time/memory complexity, variance across seeds) and fair comparisons; consequently, the claims about generality and superiority are insufficiently supported.

Supporting arguments
- Problem and motivation: Using graphs as an explicit, learned latent state for text-based reasoning is timely and well motivated. Extending GGS-NN beyond fixed input graphs to graph construction is a meaningful step, and the modular transformation view is clear and potentially useful.

- Positioning with literature: The paper situates itself relative to GNN/GGS-NN, MemNN/MemN2N, DNC, and HAM, and argues for interpretability and distributed computation on graphs. However, the comparison to prior works that also perform structured memory manipulation is mostly qualitative; important baselines (e.g., GGS-NN with text-to-graph parser at test time; end-to-end baselines with entity linking) are not provided.

- Empirical support of claims:
  • bAbI: Results are reported with strong supervision using the bAbI “Knowledge” object converted to a graph, plus task-specific graph engineering (e.g., added history linked lists for temporal tasks). This deviates from the standard end-to-end setting and confounds the claimed advantage over baselines trained without supervision. The direct reference mechanism further injects near-oracle entity linking. Without direct reference, performance drops substantially (95% on 9/20 tasks), and no end-to-end training (answers only) results are shown.
  • Rule discovery tasks: The CA and TM experiments are interesting but synthetic, with constrained schemas and small horizons. There are no baselines (e.g., DNC, MemN2N, or GGS-NN with fixed graphs) to calibrate difficulty, and no analysis of failure modes or sensitivity to longer rollouts beyond brief accuracy numbers.
  • Rigor: Hyperparameters, hidden sizes, number of nodes added per sentence, maximum graph size, propagation steps, optimization details, training time, and memory footprint are largely missing from the main text; reproducibility, while code is referenced, is not supported by sufficient detail. No ablations quantify the contribution of each transformation or the supervision components (e.g., node vs edge losses, permutation-invariant matching, teacher-forced graph injection).

- Claims vs evidence: The core claim—that learning to construct and reason with internal graphs from text is effective—is only supported under privileged supervision, with task-specific schemas and direct references. It remains unproven under the commonly accepted end-to-end QA setup, limiting generality and impact.

Additional feedback (to improve the paper; not all are decision-critical)
- Provide end-to-end results: Train without graph-level supervision (answers only), with and without direct reference/entity linking, and report accuracy gaps. Semi-supervised variants (as suggested) would strengthen the case.
- Ablations: Quantify the effect of each transformation (Tadd, TC, Th vs Th,direct, Tprop) and supervision components. Show performance when removing the task-specific history graph augmentation and using the sequence-of-graphs variant (Appendix D) as the default.
- Fair comparisons: For bAbI, compare to baselines with similar degrees of supervision (e.g., supervised MemNN) and to strong end-to-end baselines under the same data regime (1k examples). Include GGS-NN with graph inputs constructed by an external parser at test time, if feasible.
- Complexity and scalability: Report time and space as functions of |V| and |E|, actual graph sizes per task, training/inference throughput, and memory usage. Explore sparse connectivity/pruning to mitigate O(|V|^2).
- Generalization and robustness: Provide accuracy vs sequence length/graph size curves, multiple random seeds, and error analyses (e.g., failure patterns on task 17).
- Clarity and reproducibility: Consolidate key implementation details (hidden dimensions, propagation steps, optimizer, learning rate, number of nodes added per step, stopping criteria for Tadd, regularization). Clarify how node and edge type vocabularies are defined per task.
- Visualization: Show qualitative examples of learned graphs over time for bAbI, not just synthetic tasks, to support interpretability claims.

Questions for the authors
1) How many nodes can Tadd create per sentence, and how is this maximum chosen? Is there any penalty encouraging parsimony in node additions?
2) How are node types and edge types defined for each dataset? Are they fixed a priori from the task generator, or learned/induced?
3) At test time on bAbI, do you still rely on exact string matching for direct reference? How does the model handle OOV entities or paraphrases?
4) Can you report results trained end-to-end without graph-level supervision (answers only), with/without direct reference? If not competitive, what fails?
5) What are the main hyperparameters (hidden sizes, Tprop steps, learning rates), training time, and memory usage for representative tasks (e.g., task 19)?
6) How sensitive is performance to the history-linked-list engineering for temporal tasks? With the Appendix D sequence-of-graphs variant, can you reach parity on tasks 3/5?
7) Do you observe instability or label switching due to the permutation-invariant node matching across steps? How do you ensure consistent node identities over time?
8) For Rule 30 and Turing machines, how large are the tapes/sequences at train vs test, and how does accuracy degrade with length? Any baseline comparisons (e.g., DNC)?
9) Are results averaged over multiple runs with different seeds? Please report means and standard deviations.
10) Could the edge update TC be made sparse (top-k) without hurting accuracy? Any preliminary evidence?



# Review

This paper introduces the Gated Graph Transformer Neural Network (GGT-NN), a novel extension of Gated Graph Sequence Neural Networks (GGS-NNs) that enables learning to construct and modify graph-structured internal representations from textual input. The key contribution is a set of differentiable graph transformations (node addition, state update, edge update, propagation, and aggregation) that allow the model to incrementally build graphs and use them for various tasks, rather than just processing existing graphs.

The paper makes a clear contribution to the field by addressing a significant gap in graph neural network research: the ability to construct graphs from unstructured input rather than requiring graph inputs. The authors demonstrate the model's effectiveness on two main task categories: the bAbI tasks (where the GGT-NN achieves 95% accuracy on 19/20 tasks) and rule discovery tasks (cellular automaton and Turing machines), where the model successfully learns the underlying update rules without explicit supervision.

I recommend acceptance of this paper. The key reasons are: (1) the paper clearly identifies and addresses a meaningful gap in graph neural network research, and (2) the empirical results robustly support the claims, particularly the model's ability to learn graph-structured representations for tasks where such representations are naturally suited (e.g., pathfinding).

The paper is well-motivated and places the work well within the literature, clearly distinguishing GGT-NNs from related approaches like GNNs, GGS-NNs, memory networks, and DNCs. The differentiable graph transformations are well-defined and the model architecture is clearly described. The evaluation on bAbI tasks is particularly strong, with the model outperforming strong baselines on several tasks (especially task 19, pathfinding), which is reasonable given the natural fit of the graphical representation to the task. The rule discovery tasks further validate the model's ability to learn complex graph-based update rules.

For additional improvements, I suggest:
- A more detailed discussion of the model's scalability limitations. While the paper mentions the quadratic complexity of the propagation step, it would be helpful to provide concrete examples of how this affects performance on larger graphs (e.g., how the model's performance degrades as the graph size increases).
- Clarification on the implementation of "direct reference" updates. The paper mentions this as a key component that improves performance, but doesn't fully explain how it's implemented or why it's necessary. A more detailed description of the mechanism would help readers understand this important aspect of the model.
- A more direct comparison with GGS-NNs in terms of computational requirements. The paper notes that GGS-NNs require preprocessing into graphical form, but doesn't provide a clear comparison of training time or memory requirements between the two approaches.

I have two specific questions for the authors:
1. The paper mentions that the model requires "strong supervision" for the bAbI tasks (using the correct graph at each timestep). How much of this supervision is actually necessary for the model to learn meaningful graph representations? Could the model learn to construct the graph with less supervision (e.g., only at the final timestep)?
2. For the rule discovery tasks (cellular automaton and Turing machines), how does the model handle cases where the graph structure changes significantly between timesteps (e.g., new nodes being added)? Is there a specific mechanism to handle these structural changes, or does the model rely on the graph transformations to handle these changes implicitly?

Overall, this is a strong contribution that advances the field of graph neural networks by enabling the construction of graph-structured internal states from textual input. The model is well-defined, the experiments are thorough, and the results are convincing. The paper provides a valuable framework for future work on graph-structured representations.

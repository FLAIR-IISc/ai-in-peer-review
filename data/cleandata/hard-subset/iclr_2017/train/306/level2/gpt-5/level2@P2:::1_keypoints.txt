- Tackles few-shot N-way classification by meta-learning a coordinate-wise LSTM optimizer and an initialization for rapid adaptation with few gradient steps.  
- Key idea: use an LSTM whose cell state holds learner parameters and whose gates are conditioned on loss, gradient, parameters, and previous gates to produce learned per-parameter updates.  
- Meta-objective: minimize held-out query loss after T support-set updates, aligning training with few-shot evaluation.  
- Meta-learns both the optimizer and the learner initialization, enabling fast task-specific adaptation.  
- Uses parameter sharing across coordinates, gradient/loss preprocessing, and careful per-episode batch-norm handling.  
- Achieves competitive performance with Matching Networks on Mini-ImageNet and outperforms baselines, especially in 5-shot settings.  
- Provides qualitative visualizations of learned gates and releases code for reproducibility.  
- Decision: Accept.  
- Reason: Clear, well-motivated formulation that jointly learns optimizer and initialization with competitive empirical results and careful design matching train/test conditions.  
- Problem/motivation: Addresses limitations of standard gradient-based training in few-shot regimes by optimizing post-adaptation test loss directly.  
- Positioning/novelty: Distinct from prior LSTM-optimizer work by representing parameters in LSTM cell state and conditioning gates on loss/gradient/history for fixed-step rapid generalization.  
- Technical soundness: Training via unrolled updates and meta-loss on query set is coherent; gradient-independence approximation is standard and empirically justified.  
- Empirical support: Appropriate protocol (class splits, many episodes, CIs) and ablations showing benefit of training to the test-step count; BN handling avoids cross-episode leakage.  
- Suggestion — clarify update sign: Ambiguity whether c̃t = −∇θLt or ∇θLt; provide final implemented update equation.  
- Suggestion — fairness/capacity: Learner uses 32 filters vs 64 in baselines; align architectures, report parameter counts, or control for capacity.  
- Suggestion — baseline strength: Improve fine-tuning baseline with standard few-shot regularization (weight decay, early stopping, normalization strategies).  
- Suggestion — ablations: Isolate learned init vs learned optimizer, test gate input importance, and robustness to varying T (e.g., train with T distribution).  
- Suggestion — generality: Add results on other datasets (Omniglot, tiered-ImageNet) or discuss expected transferability.  
- Suggestion — complexity/stability: Report memory/compute overhead of meta-training and any stabilization techniques beyond gate bias init and gradient clipping.  
- Suggestion — clarify architecture: Explain roles of the two LSTM layers, batching per-coordinate, and provide a schematic or explicit equations.  
- Suggestion — batch-norm details: Specify per-episode BN decay/momentum and quantify impact via ablation.  
- Question — update sign/gating: Is an explicit negative sign applied to gradients, or can the model produce negative effective learning rates? Provide the exact update equation.  
- Question — final classification layer: How are variable N final-layer parameters handled across episodes; are they LSTM-updated and re-initialized per-episode from learned c0?  
- Question — coordinate-wise sharing: Are gates/hiddens independent per-parameter or coupled (e.g., via first-layer LSTM)?  
- Question — sensitivity to T: How does performance change when test-time updates differ from meta-training T?  
- Question — generalization across tasks: Does a model trained on 5-way transfer to 10-way tasks without re-meta-training?  
- Question — baseline tuning: What regularization or optimizers were used for the fine-tuning baseline (e.g., Adam, per-parameter rates)?  
- Question — failure modes: Any episodes show instability/divergence, and do gate visualizations correlate with success/failure?

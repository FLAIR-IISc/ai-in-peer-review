Summary
- The paper tackles few-shot classification via meta-learning, proposing an LSTM-based meta-learner that learns both an initial parameter vector for a CNN “learner” and an update rule that maps per-step gradients and losses to parameter updates. The update is cast to resemble an LSTM cell update with input/forget gates. The meta-learner is trained end-to-end to minimize the learner’s test loss after a fixed number of within-episode updates. The approach includes per-parameter coordinate-wise parameter sharing, gradient/loss preprocessing, a “gradient independence” approximation to avoid second derivatives, and careful handling of batch normalization statistics to avoid leakage across episodes. Experiments on a Mini-ImageNet variant claim competitive or better results than strong baselines (nearest neighbor, fine-tuning) and Matching Networks, particularly on 5-shot.

Decision: Reject
Key reasons:
1) Methodological inconsistencies and likely technical errors in the core update rule, together with numerous presentation/formatting issues, undermine confidence in correctness and reproducibility.
2) Insufficient empirical evidence and missing details: core results are not reported numerically; important ablations and fair baselines are missing; several experimental statements contradict each other.

Supporting arguments
- Core update/sign inconsistency: Section 3.1 maps gradient descent θt = θt−1 − αt∇Lt to an LSTM with ct = ft ct−1 + it c̃t requiring c̃t = −∇Lt. However, the paper subsequently sets c̃t = ∇Lt (without the negative). With it = σ(·) ∈ (0,1), the update would add a positive multiple of the gradient, not subtract it, unless the sign is handled elsewhere (not explained). This is a critical ambiguity/error.
- Gate parameterization: Using σ for the input gate bounds the effective learning rate in (0,1) and enforces non-negativity. Without an explicit mechanism to flip the sign (especially given c̃t = ∇Lt), the optimizer may not implement descent. This design choice needs justification or correction.
- Inconsistent experimental details:
  - Section 5 first defines the learner CNN to have 32 filters per layer, but the baselines (and Matching Networks) are stated to have 64 filters. This creates capacity mismatch and an unclear comparison. Later text says the learner uses the entire Dtrain per update, while Algorithm 1 samples random batches Xt,Yt at each step.
  - BN protocol differs between meta-train and meta-test in ways that are not fully aligned (meta-train test-time uses batch stats; meta-test uses running averages computed on train). This mismatch may confound evaluation. No sensitivity analysis is provided.
- Missing quantitative results: Table 1 is referenced but not included; no mean accuracies, standard deviations, or confidence intervals are provided. Claims of outperforming Matching Networks on 5-shot and being competitive on 1-shot cannot be assessed.
- Limited baselines and ablations:
  - No comparison to a meta-learned initialization with simple SGD updates (e.g., a baseline that learns c0 but uses fixed update rules), which would isolate the benefit of learning the optimizer from learning the initialization.
  - No comparison to memory-augmented approaches (Santoro et al.), nor to later, simple metric-learning baselines like Prototypical Networks (if contemporaneous versions are allowable, at least a discussion).
  - No ablations of ft = 1 vs learned forget gate, effect of gradient preprocessing, effect of the “gradient independence” approximation, or number of unrolled steps T.
- Clarity and reproducibility:
  - Crucial meta-learner architectural details are missing: hidden size per-parameter LSTM, dimensionality of inputs after preprocessing, exact parameter sharing pattern, and how a 2-layer LSTM is applied per-coordinate at ImageNet-scale CNNs without prohibitive memory.
  - Data processing details (image resizing/cropping, augmentation), optimizer hyperparameters for baselines, BN momentum/epsilon, and exact meta-optimization schedule are not fully specified.
- Claims and writing:
  - Overclaim: early text suggests a “guarantee that the initialization is an optimal starting point.” The method optimizes a surrogate objective; no guarantee is provided.
  - Multiple typos and formatting issues (“neutral networks,” inconsistent capitalization of ImageNet, incomplete references, malformed citations). References include incomplete entries (“Bosc?”), missing venues/years standardization, and broken LaTeX artifacts. This level of polish falls below ICLR standards.

Additional feedback to improve the paper
- Fix the update rule: clarify the sign of c̃t and the range/sign of it. If c̃t = ∇Lt is desired, provide a learnable transformation that permits negative effective learning rates or explicitly set c̃t = −∇Lt as per the mapping. Provide intuition/ablation on gate nonlinearities (e.g., tanh vs sigmoid, or unconstrained α via exp).
- Report complete quantitative results with confidence intervals on standard Mini-ImageNet splits. If using custom splits, release them and report both. Include number of episodes used for evaluation.
- Ensure fair model capacity across methods; use identical backbones (filters, normalization) for all baselines and your method, or justify differences.
- Add ablations: learned init vs learned optimizer; ft fixed vs learned; preprocessing on/off; effect of ignoring second derivatives; sensitivity to T.
- Expand baselines: at minimum, a meta-learned initialization + SGD (MAML-like without second derivatives if contemporaneous), and prototypical networks or a strong metric-learning baseline using the same backbone.
- Clarify BN handling and provide sensitivity when batch sizes are tiny; consider alternatives (GroupNorm/LayerNorm) and report results.
- Provide full implementation details: LSTM hidden sizes, parameter counts, memory/time footprint, meta-optimization schedule, early stopping, gradient clipping rationale.
- Clean up the writing, notation, and references; include all figures/tables.

Questions for the authors
1) Please resolve the sign inconsistency: is c̃t = ∇Lt or −∇Lt? If the former, how do you implement descent with it = σ(·) ≥ 0?
2) What are the exact numeric results (mean ± 95% CI) for 1-shot and 5-shot 5-way on your Mini-ImageNet split for all methods? How many test episodes?
3) Are the learner and baselines using the same CNN backbone (filters, BN, dropout)? If not, why, and what happens when they are matched?
4) In Algorithm 1 you sample batches from Dtrain, but elsewhere you say the full Dtrain is used at each step. Which is correct, and does it affect results?
5) Provide the meta-learner architecture details: per-parameter hidden state size, input features per coordinate, and how you scale to tens of thousands of parameters without prohibitive memory.
6) How sensitive are results to the BN protocol and to the number of unrolled steps T during meta-train vs meta-test?
7) Did you evaluate a baseline that only meta-learns the initialization with fixed SGD updates? If so, how does it compare?
8) Is the GitHub repository and the exact class splits for Mini-ImageNet publicly available and sufficient to reproduce your numbers?

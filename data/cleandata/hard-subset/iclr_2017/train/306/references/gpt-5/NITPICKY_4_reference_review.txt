Summary
- The paper tackles few-shot classification by framing it as meta-learning: an LSTM meta-learner learns to update a base learner’s parameters quickly from few examples. The key idea is to interpret the learner parameters as the LSTM cell state and to learn input/forget gates that implement an update rule akin to gradient descent, with shared per-parameter optimizer weights across coordinates and gradient/loss preprocessing (following Andrychowicz et al., 2016). The meta-learner is trained end-to-end on episodic tasks to minimize the test loss after a fixed number of inner updates, and it also learns a good task-common initialization c0 for the learner. The paper proposes an episodic batch-normalization protocol to avoid leakage across tasks. On 5-way Mini-ImageNet 1-shot and 5-shot, the method is reported as competitive with or better than Matching Networks and baselines, with code promised.

Decision: Reject
Key reasons:
1) The experimental evidence is insufficient and not convincingly presented: critical quantitative results are missing (Table 1 not present; no exact numbers, variance, seeds, or episode counts), and comparison fairness is questionable (backbone mismatch, handicapped baselines).
2) Multiple technical and presentation inconsistencies/omissions (update rule sign mismatch, algorithmic inputs vs described gates, BN protocol mismatch, incomplete training details, malformed references), which undermine confidence in correctness and reproducibility.

Supporting arguments
- Missing or incomplete results:
  - Table 1 is referenced but not included; no numeric accuracies or confidence intervals are provided, making it impossible to verify claims.
  - No details on number of evaluation episodes, random seeds, or the method for computing confidence intervals.
  - Only Mini-ImageNet is considered; no results on Omniglot or other standard few-shot benchmarks.

- Fairness and comparability issues:
  - The learner architecture for the meta-learner is described as a 4-layer CNN with 32 filters, while baselines and Matching Networks use 64 filters to “remain consistent with Vinyals et al. (2016).” This confounds capacity and optimization effects and is not an apples-to-apples comparison. Either all methods should share the same backbone or provide matched-capacity variants.
  - Matching Networks are evaluated without the optional fine-tuning the original paper permits, potentially handicapping that baseline. Conversely, regularization (e.g., dropout) is added only to Matching Networks, not to the learner, further complicating fairness.
  - The nearest-neighbor and fine-tuning baselines are relatively weak; stronger recent meta-learners (e.g., MAML/Reptile-family) are not compared.

- Methodological inconsistencies and underspecification:
  - Update rule/sign: In Sec. 3.1 the mapping from SGD to LSTM (Eq. 2) requires c̃t = −∇θt−1Lt, but the paper later sets c̃t = ∇θt−1Lt. This sign inconsistency should be resolved; it is central to the update.
  - Inputs to gates vs algorithm: Gates it and ft are defined as functions of [∇Lt, Lt, θt−1, previous gate], but Algorithm 1 shows the meta-learner receiving only (∇Lt, Lt). Clarity is needed on what the LSTM actually consumes and how θt−1 enters.
  - Two-layer LSTM design is unclear: the role of the first “normal” LSTM vs the second “modified” LSTM is not precisely specified (hidden sizes, interfaces, which features feed which layer).
  - Gradient independence assumption: The paper follows Andrychowicz et al. (2016) in ignoring certain higher-order terms, but provides no ablation to justify that this approximation doesn’t harm performance in the few-shot setting.
  - BN protocol: The text claims to match train/test conditions but describes: meta-training uses batch statistics for both train and test; meta-testing uses batch statistics on the train set and running averages for test. That is not matched. A precise, leakage-safe, matched protocol (ideally using episode-wise statistics) should be stated and ablated.
  - Scalability: Per-parameter optimizer state (hidden and cell states for each coordinate) scales linearly with the number of learner parameters; no memory/runtime measurements, meta-batch sizes, number of steps T, or wall-clock training cost are reported.

- Positioning and novelty:
  - The core idea—learning an optimizer via an LSTM with coordinate sharing and input preprocessing—builds heavily on Andrychowicz et al. (2016). The paper’s main adaptation is the episodic meta-objective for few-shot generalization and learning an initialization c0. This is close to contemporaneous few-shot meta-optimization lines (e.g., MAML and related), which are not discussed. The related work section should more carefully contrast contributions and limitations.

- Clarity and presentation quality:
  - Numerous typos and inconsistencies (e.g., “neutral networks,” “Imagenet,” “metaset”), equation cross-references, and missing minus signs.
  - References are poorly formatted and in places malformed (LaTeX escapes, missing venues/years).
  - Figures and tables referred to are absent in the submission provided; algorithm pseudocode is minimal and ambiguous.

Additional feedback (to help improve the paper)
- Provide full quantitative results with means, 95% CIs, number of evaluation episodes, and results across at least three random seeds. Include ablations on:
  - learned initialization vs learned optimizer (freeze c0 vs gates),
  - number of inner steps T and matching train/test T,
  - effect of the gradient independence assumption (with/without higher-order terms for smaller models),
  - BN handling variants,
  - gradient/loss preprocessing parameter p.
- Standardize backbones across all methods. Report results for shared-capacity backbones, and optionally for multiple capacities to study scaling.
- Compare to stronger meta-learning baselines (e.g., gradient-based meta-learners). Include Omniglot and, if possible, tieredImageNet for breadth.
- Precisely specify the two-layer LSTM architecture, hidden sizes, input features to each gate, and how θ is injected. Fix the sign in the update and ensure Algorithm 1 matches the text.
- Clearly describe the episodic BN protocol to avoid leakage and to match train/test conditions, with exact formulas and momentum settings.
- Report memory and compute overhead, meta-batch sizes, number of meta-iterations, and wall-clock training time.
- Release the exact Mini-ImageNet splits used and ensure the code link is functional and sufficient for reproduction.

Questions for the authors
1) Please provide the exact numerical results (means, 95% CIs), number of evaluation episodes, and number of random seeds for all methods in Table 1.
2) Are all methods evaluated with the same CNN backbone? If not, can you report results with matched backbones (same number of filters, params, regularization)?
3) Clarify the parameter update: should c̃t = −∇θt−1Lt? Please give the final equation used in code and how the two-layer LSTM interfaces implement it.
4) What inputs are actually fed to the gates at each step? Does the meta-learner receive θt−1 explicitly, and how are previous gate values provided?
5) Detail the episodic BN protocol precisely. Are running averages computed only on Dtrain within an episode? What momentum/weighting is used and why?
6) How many meta-training episodes, meta-batch size, training epochs, and what is the total training time and memory footprint?
7) How sensitive is performance to the gradient independence assumption? Can you provide a small-scale experiment including second-order terms?
8) How is the final linear layer initialized given label permutations across episodes? Do you learn c0 for that layer as well, and how does it generalize across tasks?

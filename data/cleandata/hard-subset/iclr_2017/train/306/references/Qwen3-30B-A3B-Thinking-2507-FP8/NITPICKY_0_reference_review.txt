

# Review for ICLR

## 1. Summary of Contributions

This paper proposes an LSTM-based meta-learner optimizer for few-shot learning, framing the problem within a meta-learning setting. The authors aim to address the limitations of gradient-based optimization in the few-shot regime by learning an effective initialization and optimization strategy. They draw an interesting analogy between LSTM cell state updates and gradient descent updates, proposing to use an LSTM to learn a task-specific optimization algorithm. The paper presents empirical results on Mini-ImageNet showing competitive performance against baselines and Matching Networks, particularly for 5-shot learning. The work is well-positioned within the meta-learning literature and makes a novel connection between LSTM architectures and optimization algorithms.

## 2. Decision

Reject. The paper contains multiple methodological inconsistencies that undermine the technical validity of the proposed approach, and the experimental results lack sufficient detail to verify the claims.

## 3. Supporting Arguments

The primary issue is a fundamental inconsistency in the model formulation that affects the entire approach. In Section 3.1, the authors claim: "ct = ft ct−1 + it c̃t, (2) if ft = 1, ct−1 = θt−1, it = αt, and c̃t = −∇θt−1Lt." However, in the actual parameterization of the gates (Equations 3 and 4), they define "c̃t = ∇θt−1Lt" without the negative sign. This creates a critical contradiction: the model as described would be learning to update parameters using "c̃t = ∇θt−1Lt" but the analogy claims "c̃t = −∇θt−1Lt." This inconsistency fundamentally undermines the proposed model's theoretical foundation and could significantly impact the optimization behavior.

Additionally, the paper makes an unsupported simplifying assumption (Section 3.3.1) that the gradients of the meta-learner are independent of the learner's loss and gradients, which is a significant technical flaw. The authors acknowledge that this would normally require second derivatives but claim they can ignore this dependency without justification. Without a thorough analysis of how this assumption affects the model's performance or convergence, the experimental results cannot be trusted.

The paper also contains several other methodological inconsistencies:
- The training algorithm (Algorithm 1) contains a typo: "Θd ← random initialization" followed by "Θd−1" in the next line.
- The paper claims to use "a fixed number of updates for fine tuning" in the baseline but never specifies what this number is.
- The Mini-ImageNet dataset creation process is not sufficiently described (100 classes selected from ImageNet, but no details on selection methodology).
- The claim "We consider 1-shot and 5-shot classification for 5 classes" is confusing, as 5-shot classification for 5 classes would mean 5 examples per class for 5 classes, which is a very specific setting not well-explained in the context of standard few-shot learning benchmarks.

## 4. Additional Feedback for Improvement

1. **Correct the fundamental model inconsistency**: The authors must resolve the sign error in the gradient term (c̃t = ±∇θt−1Lt) and provide a clear theoretical justification for the correct sign.

2. **Clarify the gradient independence assumption**: Provide a detailed analysis of how the gradient independence assumption affects the model's performance, and consider experiments to quantify the impact of this assumption.

3. **Improve methodological details**: 
   - Specify the dimensions of the LSTM cells used in the meta-learner.
   - Clearly define the "fixed number of updates" for the baseline fine-tuning.
   - Provide complete details about Mini-ImageNet dataset creation (how classes were selected, handling of class imbalance).
   - Clarify the meaning of "5-shot classification for 5 classes" in the context of standard few-shot learning benchmarks.

4. **Strengthen the experimental validation**:
   - Add ablation studies to demonstrate the contribution of different components of the meta-learner.
   - Provide more detailed comparison with recent state-of-the-art methods beyond Matching Networks.
   - Include convergence analysis showing how quickly the meta-learner converges compared to standard optimization methods.

5. **Fix formatting issues**:
   - Correct inconsistent capitalization of "meta-learning" throughout the paper.
   - Fix the reference formatting (e.g., "Andrychowicz et al\.,? \Q2016\E" is not standard).
   - Correct the misattribution of "Hochreiter et al. (2001)" in the related work.

## Questions for Authors

1. In Equation 2, the paper states "c̃t = −∇θt−1Lt" but in Equations 3 and 4, the meta-learner uses "c̃t = ∇θt−1Lt" without the negative sign. Which is correct, and what is the theoretical justification for the sign?

2. The paper claims to use batch normalization in the learner network but doesn't explain how batch statistics are handled during meta-testing to prevent information leakage between datasets. How exactly did you implement this, and what evidence do you have that this implementation doesn't leak information between datasets?

3. In Algorithm 1, the meta-learner parameters are initialized as "Θ0 ← random initialization" but then the algorithm uses "Θd−1" for the meta-learner parameters. This appears to be a typo. Please clarify the correct initialization procedure.

4. The paper states "We consider 1-shot and 5-shot classification for 5 classes" but later mentions "For 5-shot, we are able to do much better than Matching Networks." How does the "5 classes" specification relate to the standard few-shot learning setting (where N is the number of classes)? Please clarify this setting.



In light of the authors' responsiveness and manuscript revisions—particularly the clarification of the meta-learning task—I am updating my score to 8.  

This paper addresses few-shot learning via meta-learning, a concept experiencing renewed interest over the past year. The authors frame few-shot learning as a sequential meta-learning problem: each "example" consists of a sequence of training batches followed by a test batch. At each step, inputs include the base learner’s outputs (e.g., training loss, gradients) and its current state (parameters). The authors employ an LSTM to model this meta-learning process, using the second-layer memory cells to directly represent the base learner’s parameter updates. They highlight parallels between LSTM memory cell updates and gradient descent. Meta-learner updates are computed using the base learner’s prediction loss on the test batch. Key simplifications include weight sharing across second-layer cells (analogous to a uniform learning rate) and the recreation of the Mini-ImageNet dataset (Vinyals et al., 2016). Results show the LSTM meta-learner is competitive with the state-of-the-art (Matching Networks, Vinyals et al., 2016) on 1- and 5-shot Mini-ImageNet tasks.  

**Strengths:**  
- The sequential meta-learning formulation is intuitively natural and advances the field. While not the first to propose persistent learning across tasks, the authors’ implementation represents a meaningful contribution.  
- The approach achieves competitive results, outperforming Vinyals et al. (2016) in 1- and 5-shot settings. Crucially, the base learner (a simple ConvNet) differs significantly from the nearest-neighbor embedding approach in Vinyals et al., offering a fresh path to state-of-the-art results.  
- The insight linking LSTM memory cells to gradient descent updates is novel and insightful.  
- The paper provides valuable practical guidance for replicating the method, including proper initialization, weight sharing, loss/gradient normalization, and simulating test conditions during meta-training. While some points (e.g., meta-learner/base-learner parameter independence) are known, the discussion here is exceptionally useful for practitioners.  

**Weaknesses:**  
- The writing is occasionally opaque. The core problem description (Section 2, p. 2) relies on highly abstract notation unfamiliar to broader audiences. A concrete example (e.g., using MNIST) paired with a diagram illustrating the sequential few-shot task would greatly improve accessibility.  
- The Mini-ImageNet setup requires clarification. The dataset has 100 classes (64 for meta-training, 16 for validation, 20 for testing). Does meta-training only use 64 classes (with the base learner’s softmax outputting 64 units), or are all 100 classes observed across meta-training batches (with 36 never seen)? This ambiguity affects the interpretation of the base learner’s architecture.  
- Figure 2’s plots lack explanatory power, and the discussion offers minimal insight into their significance.  

This is a strong paper with compelling results. While the technical contributions are sound and the approach is well-motivated, the presentation could be significantly improved. I will maintain my updated score of 8 if the writing is refined to clarify the task formulation and dataset setup.

- Novel formulation: few-shot learning framed as sequential meta-learning using an LSTM-based meta-learner.  
- Core contribution: LSTM meta-learner learns both a good initialization and a dynamic update rule for rapid convergence with few updates.  
- Practical focus: targets scenarios with a limited number of parameter updates (e.g., 12 updates for 1-shot, 5 for 5-shot).  
- Empirical result: competitive performance on Mini-ImageNet, sometimes outperforming Matching Networks.  
- Acceptance: reviewer recommends accept.  
- Strong motivation: gradient-based optimization struggles with limited data; learned optimization is a plausible solution.  
- Theoretical insight: analogy between LSTM cell updates and gradient descent is compelling.  
- Experimental setup: uses Mini-ImageNet benchmark and reports improvements over baselines.  
- Analysis: ablation/visualization of meta-learner gate values provides insight into learned behavior.  
- Practical contributions: useful implementation/training tips (initialization, weight sharing, normalization) and code release for reproducibility.  
- Positioning: paper situates itself well within meta-learning and few-shot literature with relevant citations.  
- Presentation issues: initial task description is abstract; needs a concrete step-by-step episode example and diagram.  
- Ambiguity in experimental details: unclear N-class few-shot setup, class split usage, and softmax output size per episode.  
- Figure 2 critique: computational graph figure is uninformative and needs more detail and annotation.  
- Writing style: phrasing is dense and opaque; sections should be rewritten for clarity and conciseness.  
- Questions for authors: rationale for chosen update counts (12 and 5), class split/disjointness across meta-train/val/test, sensitivity to gradient/loss preprocessing, impact of assumed gradient independence, and whether regularization could reduce fine-tuning baseline overfitting.

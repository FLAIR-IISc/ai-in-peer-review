- Proposes an LSTM-based meta-learner that interprets gradient descent as gated recurrent cell updates for few-shot adaptation.  
- Decision: Accept — compelling, original approach with sound methodology and competitive results.  
- Clear motivation: addresses limitations of standard gradient-based optimization in low-data regimes.  
- Insightful analogy between gradient descent updates and recurrent cell updates.  
- Experiments on Mini-ImageNet show competitive performance versus state-of-the-art (e.g., Matching Networks).  
- Practical recommendation: explicitly train the meta-learner for a specific number of updates; code release would improve reproducibility.  
- Suggest clarifying LSTM vs GRU resemblance or re-framing/experimenting with GRU variants.  
- Recommend adding a visual illustration of meta-train / meta-val / meta-test class/data separation.  
- Request deeper parameter sensitivity analysis, especially learned input (it) and forget (ft) gate values across tasks.  
- Revise Figure 2 to better illustrate gradient and parameter flow between meta-learner and learner.  
- Minor typo: “it” should be “its” in Section 3.2.  
- Question: justify LSTM architecture choices (layers, hidden units) and sensitivity to these choices.  
- Question: provide examples/visualizations of the learned task-common initialization vs random initialization.  
- Question: did you try including second-order gradient terms, and what were the results and computational costs?  
- Question: is there an optimal number of meta-training updates, and how does performance change if meta-test updates differ?  
- Question: clarify current limitations relative to Matching Networks and outline directions to improve performance.

- Paper formulates few-shot learning as meta-learning where an LSTM "meta-learner" learns an optimization algorithm for a classifier ("learner"), exploiting similarity between LSTM updates and gradient descent.
- Meta-learner learns both the learner's initial weights (c0) and a task-specific update rule, trained end-to-end to minimize test loss after few updates.
- Method achieves competitive 1-shot and superior 5-shot performance versus strong baselines (e.g., Matching Networks) on Mini-ImageNet.
- Paper offers practical training insights (parameter sharing, input preprocessing, LSTM initialization).
- Recommendation: Accept.
- Main reasons for acceptance: (1) an elegant, principled conceptual contribution (LSTM-gradient descent analogy); (2) convincing empirical results validating the approach.
- Key supporting argument: the approach learns an optimization process rather than an embedding, with LSTM gates acting like dynamic learning rates/momentum and c0 providing a better starting point than standard pretraining.
- Experiments show standard fine-tuning from pretraining overfits in few-shot settings, motivating meta-learned optimization; results position the method as a state-of-the-art contender.
- Improvement suggestion: clarify the meta-learning task with a concrete step-by-step episode example and an illustrative diagram.
- Improvement suggestion: simplify dense prose and more clearly define model components (e.g., roles of each LSTM layer).
- Improvement suggestion: replace or augment Figure 3 with more informative visualizations or analyses linking gate values to input/gradient properties.
- Question for authors: clarify N-way setup — is the final softmax sized N and how are episode classes mapped to output units across updates?
- Question for authors: why recompute BatchNorm statistics per episode at meta-test rather than use running averages from meta-training?
- Question for authors: how sensitive is performance to the fixed training horizon T, and does a model trained for T generalize if run for different numbers of steps at test time?

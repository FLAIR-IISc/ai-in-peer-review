Summary
The paper proposes “Direct Future Prediction” (DFP): recasting sensorimotor control as supervised prediction of future measurement vectors (e.g., health, ammo, frags) at multiple horizons, conditioned on current observation and a goal vector. Actions are chosen by maximizing the inner product between the goal and the predicted future measurements. The model uses a CNN perception module and a dueling-style prediction module split into expectation and action streams. On several ViZDoom scenarios, DFP reportedly outperforms DQN, A3C, and DSR, shows generalization across textures/environments, and can operate with goal-agnostic training to support novel goals at test time. An agent using this approach won the Visual Doom AI Competition (Full Deathmatch).

Decision: Reject
Key reasons:
1) Substantial methodological and presentation inconsistencies (goal-conditioning and supervision mismatch; action-conditioned vs all-action outputs; missing action count; unclear loss/weighting), undermining clarity and reproducibility.
2) Experimental evaluation and baseline comparisons lack crucial details and controls (fairness, ablations, error bars, hyperparameters), making the reported advantages difficult to verify.
3) Numerous formatting and reference issues throughout (broken citations, stray text) fall below ICLR standards.

Supporting arguments
- Goal-conditioning inconsistency: Eq. (2) defines F(o, a, g) with target f (measurement deltas). Since f does not depend on g, a pure L2 loss (Eq. 4) provides no learning signal to use g; the optimal predictor can ignore g. Yet the paper claims benefit from randomized goals and goal-conditioned prediction. If the network truly predicts f, please clarify how g influences training. Alternatively, if the network predicts goal-weighted outcomes u = g^T f, the equations and loss are inconsistent with the text.
- Action supervision vs all-action outputs: Sec. 3.2 says the network “emits predictions of future measurements for all actions at once,” but Eq. (2)/(4) imply conditioning on a single ai and supervising only that branch. How is the loss computed: only for the executed action’s head, or for all actions? If only for ai, what regularization/cross-action coupling (beyond Eq. 6 normalization) prevents degenerate behavior for unobserved actions?
- Normalization layer ambiguity: Eq. (6) reuses Ai both pre- and post-normalization, leading to notational confusion. Also using A both as the action set and the action stream name is confusing.
- Temporal targets and loss weighting: You predict at τ = {1,2,4,8,16,32} but “only the latest three time steps contribute … with coefficients (0.5, 0.5, 1).” “Latest” is ambiguous (largest τ?). The precise composition of f and per-horizon weighting in Eq. (4) needs to be spelled out.
- Action space ambiguity: In D3/D4, the paper states there are eight binary sub-actions and “any combination … resulting in possible actions,” but the number is missing. If all 8 binary sub-actions are combinable, |A| = 256; please confirm. This matters for architecture size and difficulty.
- Measurement scaling/normalization: D3/D4 use three “normalized” measurements; the normalization scheme (ranges, per-episode normalization, clipping) is not described. This is crucial for balanced training and fair comparisons against baselines (which also use linearized rewards).
- Baseline fairness and reporting: A3C implementation is third-party; search spaces differ across tasks; details such as frame stacking, action repeat, input resolution, reward clipping, entropy coefficients, optimizer configuration, and number of actors are not clearly matched to your method. Training speed “million steps/day” is implementation/hardware dependent and not an apples-to-apples metric without controlling for environment step cost and compute. No confidence intervals/error bars are shown, and only three seeds are reported.
- Metric choice: In the main comparisons for D3/D4, only “frags at end of episode” is reported, while the training objective includes ammo/health. This can hide trade-offs. Although the goal-agnostic section reports vectors, the main claims should include all three or a principled scalarization.
- Off-policy concerns: Experience memory is “small,” details missing (size M, update period k, batch size N). Given off-policy data and changing policies/goals, what biases are introduced? Any target network or stabilization techniques used? None are described.
- Reproducibility/appendices: The text relies on Appendix A for key specifics (architecture, hyperparameters), but Appendix A is not included here; Appendix B is partial. Code availability is not mentioned.
- References/formatting: The bibliography contains numerous malformed entries (placeholders “? \Q2016\E,” incomplete venues, stray text “search. Nature,” duplicated and truncated author lists, inconsistent punctuation), and several typographical/formatting errors in the main text (e.g., “threedimensional,” missing action count, spacing in URLs). This is substantially below ICLR standards.

Additional feedback (to help improve the paper)
- Clarify the learning target: either predict f (goal-independent) and drop g from F, or predict goal-conditioned outcomes u and adjust equations/loss accordingly. If g remains as input for multi-task feature selection, justify and demonstrate that it is used (e.g., with ablations removing g).
- Precisely specify the loss over multi-horizon predictions, per-dimension weights, and the supervision applied to action heads. Provide an ablation of single-action-head vs all-action-head outputs.
- Report all three measurements for D3/D4 in main results, and include error bars over ≥5 seeds. Provide statistical tests or CIs.
- Ensure fair baselines: match input pre-processing (frame stacking/gray-scale/resolution), action repeat, exploration, optimizer, network capacity, and compute budget. Consider A3C-LSTM and dueling DQN baselines given partial observability and your own use of dueling-like structure.
- Provide full details of the action space cardinality in each scenario, measurement normalization, replay memory M, update schedule k, batch size N, optimizer and LR schedule, image resolution, frame skip, and reward shaping used for baselines.
- Discuss limitations: dependence on having meaningful dense measurements; handling of counterfactuals for unexecuted actions; potential extrapolation errors.
- Clean up all references and formatting to meet conference standards.

Questions for the authors
1) What exact quantity does F predict: the full measurement vector f or a goal-weighted scalar? If f, why include g in F’s inputs, and how do you ensure the network uses it?
2) Do you supervise only the executed action’s output head or all action heads? If only one, how do you prevent garbage predictions for never/rarely taken actions?
3) What is the exact number of actions in D3/D4? Are all 8 sub-actions truly combinable at each step (|A|=256)?
4) How are measurements normalized in D3/D4? Are normalization params fixed across train/test environments?
5) What are M, N, k, optimizer, LR schedule, image resolution, frame skip/repeat, and stochasticity settings?
6) For A3C/DQN baselines, did you use frame stacking or recurrence? If not, why; if yes, what stack length? Did you match network capacity to DFP?
7) Can you provide error bars across more seeds and full measurement vectors for the main comparisons?
8) How sensitive is performance to horizon set {τ} and the weighting of horizons in the loss?
9) Did you try removing g from F or training with/without goal randomization to quantify its effect?

- Main claim: Incorporating recursion into neural programming (NPI) enables provable generalization for discrete tasks with finite base cases.  
- Verification method: Proves generalization by testing only a finite set of base cases and reduction rules.  
- Empirical result: Recursive NPI achieves perfect generalization on four discrete tasks (addition, bubble sort, topological sort, quicksort).  
- Data efficiency: Models generalize from very small training sets (e.g., 200 traces for addition, 6 traces for topological sort).  
- Recursive vs non-recursive: Non-recursive models fail to generalize beyond modest input sizes, while recursive models generalize to arbitrarily large inputs.  
- Example highlight: Recursive topological sort trained on 6 traces generalizes to graphs up to 70 vertices; non-recursive fails past size 8.  
- Verification confirms semantics: The verification step shows models learned correct program semantics rather than spurious patterns.  
- Theoretical significance: Provides a novel framework moving neural programming toward formal, provable guarantees.  
- Practical contribution: Recursion can be implemented in NPI by modifying training traces, requiring minimal architectural changes.  
- Limitation: Verification is infeasible for continuous input spaces (e.g., MNIST, speech), limiting applicability.  
- Requested clarity: Paper should more explicitly state the problem class and precise scope/limitations of the verification procedure.  
- Distinction needed: Paper should better separate the theoretical result (recursion → provable generalization) from the practical implementation in NPI.  
- Question 1: How could the verification framework be extended to continuous input spaces and what challenges would arise?  
- Question 2: Why is the LSTM hidden-state reset at subprogram calls critical for recursion, and is this property generalizable to other architectures?  
- Question 3: How to adapt verification for tasks with extremely large or infinite base cases, and what are the practical limits of the current approach?

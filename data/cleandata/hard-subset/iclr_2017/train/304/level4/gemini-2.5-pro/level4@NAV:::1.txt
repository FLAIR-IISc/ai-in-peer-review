This work posits that the capacity to handle recursion is a critical component for neural programming architectures, asserting that this capability facilitates robust generalization to out-of-distribution test cases and enables learning from smaller training datasets. The methodology is largely an extension of the Neural Programmer-Interpreter (NPI) framework by Reed & de Freitas (ICLR 2016), which learns from program execution traces. The central contribution here is the training of NPI models on traces that explicitly feature recursive calls. The authors demonstrate that the correctness of the learned programs can be verified by evaluating them against a minimal set of base cases and reduction rules. Impressively, this approach allows the NPI architecture to perfectly infer algorithms for problems such as Bubblesort and the Tower of Hanoi.

A significant strength of this work is its simplicity; as the authors note, the sole modification is to the execution traces provided to the training pipeline. This, however, raises a fundamental question about the main takeaway: does access to execution traces mean the neural programming problem is effectively solved, or was the problem formulation perhaps too constrained from the outset? For instance, consider a larger input domain like MNIST, with the task of inferring a program to sort digits from highest to lowest value. In this setting, providing execution traces would essentially decouple the perceptual problem of digit recognition from the algorithmic problem of inferring the program logic. The task would then be no more difficult than learning an MNIST classifier and learning to bubblesort symbols independently. This leads to a critical question: what is a problem for which execution traces are available, but the proposed method would still be unable to infer the correct program?

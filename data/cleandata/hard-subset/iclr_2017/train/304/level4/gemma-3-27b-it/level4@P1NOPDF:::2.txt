This is a very interesting and readily accessible paper. 
The authors present a small, yet effective, approach to significantly enhance the power of Neural Programming Interpreters. By incorporating recursion, NPI generalizes better from a limited number of execution traces.
This work is a compelling demonstration of how a modest, but non-trivial, extension can substantially improve the practical applicability of a machine learning method.
I particularly appreciate the consistent notation used throughout this paper and the original DeepMind paper. As someone not deeply familiar with this field, it was helpful to be able to consult the original work alongside this one. 
My primary concern lies with the generality of the provided proofs. While the current proofs hold for the numerical examples presented—where one can iterate over all possible execution paths until the next recursive call—it’s unclear how this approach would generalize to continuous input spaces (e.g., the 3D car example from the original paper). It seems a proof of generalization would remain intractable in such continuous cases. 
Would the authors consider releasing the source code for this work?

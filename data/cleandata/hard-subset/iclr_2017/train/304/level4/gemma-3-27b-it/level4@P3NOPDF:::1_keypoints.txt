- Paper shows recursion in training traces improves neural program learning, yielding stronger generalization and less data needed.
- Builds on the Neural Programmer-Interpreter (NPI) framework, training NPIs on execution traces that include recursive calls.
- Empirically demonstrates perfect inference for Bubblesort and Tower of Hanoi using recursive traces.
- Main contribution: a verification method that proves learned program correctness by testing a limited set of base cases and reduction rules.
- Strength: conceptual simplicity—only adding recursion to traces produces large gains.
- Raises the core question whether program learning is effectively “solved” when full execution traces are available.
- Concern that trace supervision can decouple tasks (e.g., MNIST sorting separates perception from algorithm), reducing problem difficulty artificially.
- Lack of analysis of problem classes where trace-based recursion fails—need to identify tasks that expose limitations.
- Need to evaluate scaling to larger input spaces, more complex logic, richer data structures, and nuanced conditionals.
- Verification scalability is unclear—how required base cases and reductions grow with program complexity needs discussion.
- Overall positive demonstration but requires further work on generality, challenging benchmarks, and verification scalability.

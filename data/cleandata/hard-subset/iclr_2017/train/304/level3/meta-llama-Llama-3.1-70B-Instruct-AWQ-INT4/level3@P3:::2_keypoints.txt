- Proposes incorporating recursion into Neural Programming Interpreters (NPI) to improve generalization and interpretability with limited training data.
- Paper is well-written, accessible, and reuses DeepMind notation to aid readability.
- Introduction clearly frames the generalization problem in neural programming architectures and current limitations.
- Key contribution: modify training to include recursion so NPI can learn recursive programs that generalize perfectly to unseen inputs.
- Provides analysis that recursion reduces problem complexity and makes reasoning more tractable.
- Experimental validation on four tasks (grade-school addition, bubble sort, topological sort, quicksort) shows recursive NPI outperforms non-recursive versions, especially on longer inputs.
- Strength: proposes a verification procedure offering provable guarantees about learned program behavior on given input sets.
- Limitation: generalization proofs are vague and need greater rigor.
- Limitation: unclear how the approach handles continuous input spaces (e.g., 3D car models).
- Limitation: lacks information on computational cost and scalability to larger problems.
- Recommendation: accept the paper.
- Requested additions/feedback: report computational cost/scalability, strengthen proofs, address continuous inputs, and clarify code release plans.
- Questions for authors: details on computational cost/scalability, handling continuous inputs, specifics of the verification procedure, and whether source code will be released.

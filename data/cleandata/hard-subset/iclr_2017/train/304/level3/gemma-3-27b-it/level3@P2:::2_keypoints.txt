- Decision: Accept — novel, compelling approach showing recursion improves neural program synthesis generalization and interpretability with provable guarantees.
- Proposal: Augment Neural Programmer-Interpreter (NPI) architectures with recursion.
- Tasks/evaluation: Tested on grade-school addition, bubble sort, topological sort, and quicksort.
- Empirical result: Models with recursion achieve superior performance and 100% accuracy on valid inputs after limited training.
- Data efficiency: Strong generalization from small amounts of training data.
- Interpretability: Recursion leads to more interpretable learned programs.
- Provable generalization: Framework provided to guarantee correctness via verification of base cases and reduction rules.
- Verification: Authors detail a verification procedure and construction of a verification set to support provable claims.
- Strengths: Clear motivation, well-placed literature context, and impactful contribution to trustworthy neural programs.
- Limitations — scope: Tasks are relatively simple; experiments should be expanded to more complex or realistic inputs and other architectures beyond NPI.
- Limitations — verification: Construction of the verification set needs more concrete explanation and has limitations for large/continuous input spaces.
- Claim caution: The statement claiming to be the "first" on provable generalization should be scrutinized against related work (e.g., formal verification of neural networks).
- Reproducibility: Releasing source code is requested to facilitate reproducibility and further research.
- Questions for authors: (1) How is the verification set constructed for complex tasks (topological sort, quicksort) and how is coverage ensured?
- Questions for authors: (2) Any strategies to handle verification for large/infinite base cases (e.g., formal verification/model checking)?
- Questions for authors: (3) How sensitive is performance to hyperparameters (learning rate, LSTM size)?
- Questions for authors: (4) Do benefits of recursion extend to other neural program synthesis architectures; which architectures might benefit most?
- Questions for authors: (5) Will you release the implementation source code?

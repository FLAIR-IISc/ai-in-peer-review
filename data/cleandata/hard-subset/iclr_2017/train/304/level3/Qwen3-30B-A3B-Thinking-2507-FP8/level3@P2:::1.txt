

# Review for ICLR 2017

## 1. Summary

This paper makes a significant contribution to neural program synthesis by demonstrating that incorporating recursion as a fundamental abstraction enables provable perfect generalization for neural programs. The authors extend the Neural Programmer-Interpreter (NPI) framework by modifying training traces to include recursive elements, allowing the model to learn recursive programs. They demonstrate that for four tasks (grade-school addition, bubble sort, topological sort, and quicksort), the recursive neural programs achieve perfect generalization from very small training sets (as few as 4-200 examples), and crucially, they provide a verification procedure that proves the generalization behavior by testing only a finite set of base cases and reduction rules. The paper is well-structured, with clear theoretical motivation and convincing empirical results for the specific tasks considered.

## 2. Decision

I recommend acceptance with minor revisions. The paper makes a novel and important contribution to the field by introducing recursion as a key abstraction for neural programs and demonstrating provable generalization for the first time. The theoretical insight is significant, and the empirical results are compelling for the specific tasks they consider.

## 3. Supporting Arguments

The paper successfully addresses a critical limitation in neural program synthesis: the poor generalization of existing models beyond certain input complexities. The key insight that recursion enables provable generalization is well-motivated by the literature and correctly identifies the problem with current approaches (which learn spurious dependencies on training set characteristics like input length).

The theoretical contribution is particularly strong: the verification procedure they propose dramatically reduces the verification burden by leveraging the recursive structure. For recursive addition, they demonstrate that proving correctness for a small set of base cases (20,181 problems) guarantees correctness for all possible input lengths. This is a significant methodological advance.

The empirical results are convincing for the specific tasks they consider. For example, the recursive NPI model for bubble sort achieves 100% accuracy on arrays of all lengths tested (up to 5000 elements) from training on only 100 examples, while the non-recursive version fails beyond length 8. The verification procedure successfully proves perfect generalization for the recursive models, which is a first in the field.

However, I have some concerns about the broader applicability of the approach. The paper's success relies on having access to execution traces that explicitly contain recursive structure, which may not be available in more realistic settings where the input is raw data (like images or speech) rather than structured program execution traces. The reviewer's concern about larger input domains (e.g., sorting MNIST digits) where execution traces decouple perception from program logic is valid - in such cases, the problem might be trivialized into separate perception and program logic components.

## 4. Additional Feedback

To strengthen the paper, I recommend the following improvements:

1. More clearly articulate the limitations of the approach, particularly regarding the requirement for execution traces containing recursive structure. Discuss how this might limit applicability to real-world problems where such traces are not available.

2. Include a more detailed analysis of the trade-offs between the simplicity of the approach (just changing training traces) and the constraints it imposes on the problem setup.

3. Consider discussing how the approach might be extended to domains where execution traces are not directly available, such as by learning to generate these traces from raw inputs.

4. Add a brief comparison to other methods for handling program generalization (e.g., curriculum learning, different architectures) to better position their contribution.

## 5. Clarification Questions

To better assess the paper's contribution and limitations, I would like the authors to address the following questions:

1. How would the approach handle a problem like sorting MNIST digits where the input is raw images and the execution trace would need to be generated from the image data? Would the problem become trivialized into separate perception and program logic components?

2. Are there any tasks where execution traces are available but the recursive approach still fails to learn the correct program? If so, what are the characteristics of such tasks?

3. How would the verification procedure scale to more complex domains where the base cases and reduction rules might be very numerous or infinite? What would be the practical limitations of the verification approach?

The paper makes an important theoretical contribution to neural program synthesis, and I believe it will be valuable to the community. With the minor revisions suggested, it will be a strong contribution to ICLR 2017.

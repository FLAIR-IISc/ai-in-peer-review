- Problem: Neural program learners fail to generalize from small training inputs to more complex inputs.  
- Proposal: Incorporate recursion into neural architectures via an extension of the Neural Programmer-Interpreter (NPI) to improve generalization.  
- Key contribution: Recursive neural programs can achieve provably perfect generalization for multiple tasks (addition, bubble sort, topological sort, quicksort).  
- Verification: Generalization can be verified by checking a finite set of base cases and reduction rules.  
- Motivation: Existing methods learn spurious dependencies (e.g., input length); recursion decomposes problems into manageable subproblems.  
- Recommendation: Strong accept—paper makes a significant first demonstration of provably perfect generalization in neural program induction.  
- Empirical strength: Recursive models achieve perfect generalization with minimal training data (e.g., 200 examples for addition, 4 for quicksort); non-recursive models fail on larger inputs.  
- Verification procedure: Provides a theoretically sound method to guarantee generalization without exhaustive testing; applied consistently across tasks with matching results.  
- Formal verification framework: Paper’s strongest contribution—reduces verification to a finite set of base cases and reductions, enabling proofs of perfect generalization.  
- Verification set details: Addition verification set (20,181 inputs) was analytically constructed; other tasks used environment-observation-derived sets.  
- Task-specific findings: Bubble sort needs full recursion for perfect generalization; partial recursion is insufficient.  
- Generality: Topological sort generalizes from a single 5-node trace to 70-node graphs; quicksort results show approach handles non-tail recursion.  
- Revisions: Authors added topological sort and quicksort and provided thorough implementation details and trace-generation functions in the appendix.  
- Suggested improvements: Provide deeper formal analysis of why recursion reduces complexity and discuss verification intractability for very large/infinite input domains (e.g., MNIST).  
- Questions for authors: Clarify whether verification can be (partially) automated, how verification sets are constructed for infinite domains, computational cost for large problems, and whether probabilistic guarantees can be provided when perfect generalization is impossible.



This paper makes a significant contribution by demonstrating how incorporating recursion into neural programming architectures can dramatically improve generalization. The authors introduce a novel approach to enable recursive neural programs within the Neural Programmer-Interpreter (NPI) framework by modifying training traces to explicitly include recursive elements. Their key insight is that recursion reduces problem complexity by decomposing tasks into smaller instances with finite base cases and reduction rules, enabling provable guarantees about program behavior without exhaustive input testing.

The paper successfully demonstrates this approach across four diverse tasks: grade-school addition, bubble sort, topological sort, and quicksort. The recursive NPI implementations achieve perfect generalization from minimal training data (e.g., 200 traces for addition, 6 traces for topological sort), outperforming non-recursive versions that fail on inputs beyond a certain complexity threshold. Crucially, the authors provide a verification procedure that allows them to prove perfect generalization by testing only a finite set of base cases and reduction rules, which is a novel contribution to neural programming.

The paper is exceptionally well-written and clearly explains the concepts. The authors effectively reuse DeepMind's NPI notation, making the paper accessible to non-experts while maintaining technical precision. The experiments are well-designed and convincingly demonstrate the benefits of recursion, with the recursive versions consistently achieving 100% accuracy across all tested input complexities, while non-recursive versions fail for longer inputs.

I recommend acceptance with confidence. The paper makes a clear and significant contribution to the field of neural programming, addressing a critical limitation (poor generalization) with an elegant solution (recursion). The verification procedure for proving perfect generalization is particularly noteworthy as it represents a first in neural programming research.

The main limitation to note is that the verification approach appears intractable for continuous input spaces (e.g., 3D object manipulation), which would be valuable to address in future work. The authors should clarify these limitations more explicitly in the discussion.

To improve the paper, I suggest:
1) Providing more detailed examples of the verification set construction process for the quicksort task, which involves non-tail recursion.
2) Expanding the discussion on how recursion improves interpretability beyond what's currently stated.
3) Including a brief comparison with recent work on neural program synthesis (e.g., from the 2016 NeurIPS papers).

I have two specific questions to clarify:
1) The paper claims recursive programs achieve perfect generalization, but it's unclear how the verification procedure handles cases where reduction rules might depend on specific input values (e.g., for quicksort, how are pivot selection rules verified?).
2) Will the authors release the source code for their implementation, including the verification procedure, to enable reproducibility?

The paper's contribution is both theoretically and practically significant, demonstrating how a small but crucial modification (incorporating recursion) can make neural programming approaches more robust and reliable. This work represents an important step toward building neural architectures capable of learning program semantics that generalize perfectly to unseen inputs.

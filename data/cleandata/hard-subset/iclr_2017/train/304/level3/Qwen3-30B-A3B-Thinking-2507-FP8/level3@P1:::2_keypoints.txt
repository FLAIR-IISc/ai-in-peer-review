1) Main contribution: Incorporating recursion into neural programming (NPI) dramatically improves generalization.  
2) Method: Enable recursive neural programs by modifying training traces to explicitly include recursion.  
3) Key insight: Recursion reduces problem complexity via finite base cases and reduction rules, allowing provable behavior guarantees.  
4) Verification: Introduces a procedure to prove perfect generalization by testing a finite set of base cases and reductions—a novel contribution.  
5) Tasks evaluated: Demonstrated on addition, bubble sort, topological sort, and quicksort.  
6) Empirical results: Recursive NPIs achieve perfect generalization from minimal data (e.g., 200 traces for addition, 6 for topological sort).  
7) Baseline comparison: Non-recursive versions fail on longer/more complex inputs while recursive versions reach 100% accuracy.  
8) Presentation: Paper is exceptionally well-written, reuses NPI notation effectively, and is accessible yet precise.  
9) Recommendation: Accept with confidence; work is a clear, significant contribution to neural programming.  
10) Theoretical/practical significance: Small modification (recursion) yields robust, provable program semantics that generalize.  
11) Limitation: Verification approach appears intractable for continuous input spaces (e.g., 3D object manipulation) and needs clearer discussion.  
12) Suggested improvements: (a) Show detailed verification-set construction for quicksort (non-tail recursion), (b) expand discussion on recursion and interpretability, (c) compare briefly with recent neural program synthesis work (2016 NeurIPS).  
13) Open questions: (a) How does verification handle reduction rules dependent on specific input values (e.g., pivot selection in quicksort)? (b) Will the authors release source code and verification tooling for reproducibility?

- Proposes adding recursion to neural programming architectures to fix poor generalization in learned algorithms.
- Recursion forces learning of base cases and reduction rules, decoupling logic from input size and simplifying the learning task.
- Implements recursion by using recursive training traces and resetting the NPI controller state on subprogram calls.
- Demonstrates near-perfect generalization from few examples on addition, bubble sort, topological sort, and quicksort, outperforming non-recursive models.
- Provides a methodology to construct a finite verification set via recursion, enabling provably perfect generalization.
- Decision: Accept.
- Key reason: Simple, well-motivated idea that effectively addresses a core limitation in neural program synthesis.
- Key reason: Introduces a concrete framework bridging learning and formal verification for neural programs.
- Empirical support: Strong, qualitative improvements and ablation studies (notably on bubble sort) show recursion’s impact.
- Limitation: Requires fully specified recursive execution traces (strong supervision).
- Limitation: Manual construction of the verification set V may not scale and needs discussion of automation.
- Limitation: Approach depends on NPI’s reset-on-call behavior; applicability to architectures with persistent controller state is unclear.
- Question: How practical/difficult is ensuring the verification set truly covers all base cases and reductions?
- Question: Can the model infer recursion from non-recursive traces or input-output pairs, reducing supervision?
- Question: Does the need to reset controller state imply architectures should separate short-term stateless controllers from long-term structured memory?

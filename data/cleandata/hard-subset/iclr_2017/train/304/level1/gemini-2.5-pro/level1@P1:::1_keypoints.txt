- Proposes explicitly incorporating recursion into learned program execution to improve generalization.  
- Adapts the Neural Programmer-Interpreter (NPI) to train on recursive execution traces.  
- Recursive NPI models generalize dramatically better than iterative-trained models on algorithmic tasks.  
- Empirical validation on addition, bubble sort, topological sort, and quicksort shows near-perfect accuracy from few simple examples.  
- Key claim: recursion makes formal verification tractable by reducing infinite input domains to a finite verification set of base cases and reduction rules.  
- Presents a novel verification methodology for neural program models and demonstrates it as a proof-of-concept.  
- Strong experimental evidence: non-recursive models’ performance collapses with input length while recursive models maintain 100% accuracy in some tasks.  
- Paper is well-written, builds on NPI, and introduces a transformative idea with significant impact.  
- Limitation: relies on full, correct recursive execution traces (strong supervision); needs discussion on sensitivity and learning from partial/noisy traces.  
- Request to clarify and exemplify the verification-set construction (e.g., one base case and one reduction rule for bubble sort).  
- Appendix detail: bubble sort verification used a simplified encoder observing comparisons directly; this abstraction should be noted in main text.  
- Ask authors to state concrete sizes of training vs. verification sets for tasks (e.g., topological sort: 6 training traces vs. 73 verification graphs).  
- Ask whether recursive behavior can be induced with less supervision or learned from iterative traces via architectural bias.  
- Ask how authors ensured empirically generated verification sets truly cover all base cases and reduction rules (completeness of V).

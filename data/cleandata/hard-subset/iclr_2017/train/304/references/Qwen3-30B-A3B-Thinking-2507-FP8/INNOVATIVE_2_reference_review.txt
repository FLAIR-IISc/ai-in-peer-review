

# Review of "MAKING NEURAL PROGRAMMING ARCHITECTURES GENERALIZE VIA RECURSION"

## Summary of Contribution

This paper presents a bold and novel approach to neural programming by explicitly incorporating recursion into neural architectures to achieve provably perfect generalization. The authors correctly identify a fundamental limitation of current neural programming models: they fail to generalize beyond a certain input complexity due to learning spurious dependencies on training examples rather than the true program semantics. The paper's key insight is that recursion, a core concept in programming languages, is essential for neural programs to generalize and that it enables provable guarantees about their behavior. The authors demonstrate that by modifying training traces to include recursive elements, the Neural Programmer-Interpreter (NPI) architecture can learn recursive programs that achieve 100% accuracy on all tested inputs (including inputs of arbitrary length) for tasks like grade-school addition, bubble sort, topological sort, and quicksort. Crucially, the paper introduces a verification procedure to prove that these recursive neural programs generalize perfectly, which the authors claim is the first time such provable guarantees have been provided for neural programs.

## Decision

Accept

## Supporting Arguments

This paper should be accepted for ICLR because it introduces a significant new concept that fundamentally changes how we think about neural programming architectures. The core contribution—explicitly incorporating recursion as a key abstraction to achieve provable generalization—is novel, bold, and potentially transformative for the field. While the verification procedure currently applies only to tasks with finite base cases (as acknowledged by the authors), the paper's core insight about recursion as a necessary component for neural program generalization is highly valuable.

The paper's novelty is well-justified by the literature review, which correctly identifies that previous work on neural programming (including the NPI paper) did not explicitly leverage recursion as a key abstraction for generalization. The empirical results are strong for the tasks tested, showing that the recursive versions achieve perfect generalization where non-recursive versions fail, and the verification procedure provides a concrete method to validate the generalization claim. This combination of novel idea, strong empirical results, and the claim of provable guarantees makes the paper stand out as a significant contribution to the field.

The paper's approach aligns with ICLR's emphasis on novel ideas over methodological perfection. While the verification procedure might not generalize to all possible tasks (as the authors acknowledge), the core insight about recursion as a key abstraction for generalization is sufficiently compelling to warrant acceptance.

## Additional Feedback

1. The paper would benefit from a more detailed discussion of the limitations of the verification procedure, particularly regarding how it might scale to more complex tasks. While the authors acknowledge this limitation, a more thorough analysis would strengthen the paper.

2. It would be helpful to see a clearer comparison between the verification set size and the training set size for each task. The paper mentions that the training sets are very small (e.g., 200 traces for addition), while the verification sets are larger but still finite. A table showing the size of training sets, verification sets, and the number of inputs tested would help readers understand the generalization capability.

3. The paper mentions that they "could provide proofs that these learned programs generalize perfectly" for the tasks tested. For readers to fully appreciate this claim, it would be valuable to include a small example of the verification process for one of the tasks (e.g., for addition), showing how the base cases and reduction rules were identified and verified.

4. The paper claims this is the first time provable guarantees of perfect generalization have been provided for neural programs. To strengthen this claim, a more detailed comparison with other approaches that might claim similar guarantees (even if not fully realized) would be helpful.

## Questions for Authors

1. How do you envision extending the verification procedure to handle tasks with potentially infinite state spaces (e.g., natural language processing or image recognition tasks), where the base cases and reduction rules might be unbounded?

2. The paper states that the NPI architecture naturally supports recursion, but previous work did not consider it explicitly. Could you elaborate on why previous researchers might have overlooked recursion as a key abstraction, and how your work changes the design philosophy of neural programming architectures?

3. The paper notes that "in this paper, we provide as an example one way to train a recursive neural program, by providing a certain training execution trace." What are your specific plans for reducing supervision in future work, and how do you anticipate this might impact the model's ability to discover recursive structures on its own?

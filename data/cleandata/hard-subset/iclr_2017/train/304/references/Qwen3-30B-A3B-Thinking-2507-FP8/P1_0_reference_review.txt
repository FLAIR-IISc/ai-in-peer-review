

# Peer Review for ICLR

## 1. Summary of the Paper's Claims and Contributions

The paper "Making Neural Programming Architectures Generalize via Recursion" tackles a fundamental limitation of current neural programming approaches: their poor generalization beyond a certain complexity threshold. The authors argue that existing neural programs often learn spurious dependencies on specific characteristics of training examples (such as input length) rather than the true program semantics. To address this, they propose explicitly incorporating recursion into neural programming architectures, which they claim enables provable perfect generalization.

The key contributions are:
1. A novel approach to neural programming that incorporates recursion, allowing neural programs to learn true program semantics rather than spurious dependencies
2. A verification procedure that enables proving perfect generalization for neural programs by testing only a finite set of base cases and reduction rules
3. Empirical demonstrations that recursive neural programs achieve perfect generalization on four tasks (addition, bubble sort, topological sort, and quicksort) after training on very small datasets
4. The first paper to provide provable guarantees of perfect generalization for neural programs

The authors implement their approach in the Neural Programmer-Interpreter (NPI) architecture by modifying training traces to include recursive elements. They demonstrate that recursive versions of the NPI model achieve perfect generalization on inputs of arbitrary complexity, while non-recursive versions fail for longer inputs. Crucially, they show that with their verification procedure, they can formally prove the recursive neural programs' perfect generalization.

## 2. Decision

I recommend **acceptance** of this paper with minor revisions.

Key reasons:
1. The paper addresses an important and well-motivated problem in neural programming (generalization limitations)
2. The approach of using recursion to achieve provable generalization is novel and well-placed within the literature
3. The empirical results are strong and clearly demonstrate the advantages of the recursive approach

## 3. Supporting Arguments

The paper makes a compelling case for the importance of recursion in neural programming. The authors correctly identify a critical limitation of current approaches: neural programs often fail to generalize beyond a certain complexity threshold because they learn spurious dependencies on specific training characteristics rather than the true program semantics. This is well-supported by the literature they cite (Vinyals et al., Kaiser & Sutskever, Reed & de Freitas, Graves et al., Zaremba et al.).

The proposed solution is well-motivated. Recursion is a fundamental programming concept that decomposes problems into simpler instances, which aligns with the authors' hypothesis that it would help neural programs learn the true program semantics. The paper provides a clear explanation of how recursion can be implemented within the NPI architecture, and how it naturally supports recursive program learning.

The empirical results are robust and convincing. For all four tasks, the recursive versions achieve perfect generalization on inputs of arbitrary complexity (up to the limits of the experiments), while non-recursive versions fail for longer inputs. For example, in the addition task, the recursive version can generalize from 1-digit examples to 5000-digit examples, while the non-recursive version fails on longer inputs. The paper also provides a clear verification procedure that allows them to prove perfect generalization by testing only a finite set of base cases and reduction rules.

The verification procedure is particularly significant. It's the first time one can provide provable guarantees of perfect generalization for neural programs, which is a major contribution to the field. The paper clearly explains how the recursive structure makes verification tractable by reducing the problem to checking a finite number of cases rather than an infinite number of inputs.

The paper is well-written and clearly organized. The authors effectively connect their theoretical claims with empirical results, and the verification procedure is well-explained and justified.

## 4. Additional Feedback for Improvement

While the paper is strong overall, I have a few suggestions to improve it further:

1. **Clarify the scope of "provably perfect generalization"**: The paper acknowledges that the verification procedure may not work for tasks with very large input domains (like MNIST digits), but this limitation should be more prominently discussed. It would be helpful to provide a more precise characterization of which types of tasks are amenable to this verification approach and which are not.

2. **Provide more details about the verification set construction**: For tasks like topological sort and quicksort, where the verification set was constructed through random generation rather than analytical determination, the paper could provide more details about how they ensured the verification set covered all necessary base cases and reduction rules. A table comparing the size of the training set, verification set, and the total number of possible inputs would be valuable.

3. **Discuss the relationship to existing recursive neural networks**: While the paper mentions recursive neural networks in the context of parsing, it doesn't clearly distinguish between their approach and existing recursive neural network architectures. Clarifying how their approach differs from and complements existing work in this area would strengthen the paper's positioning.

4. **Include a more detailed comparison with related work**: The paper could benefit from a more thorough comparison with other approaches to improving generalization in neural programming, such as curriculum learning or architectural modifications.

5. **Clarify the "training set size" claim**: The paper states that the training set can be "considerably smaller than the verification set," but it would be helpful to provide specific numbers for the training set size versus verification set size for each task to make this claim more concrete.

## 5. Questions for the Authors

1. The paper mentions that for tasks with very large input domains (like MNIST), the verification procedure may be infeasible. How do you think the field might address this limitation? Are there alternative verification approaches that could be developed for such tasks?

2. For the bubble sort task, you note that partial recursion (only making one recursive call) is insufficient for perfect generalization, while full recursion (making three recursive calls) is sufficient. Could you elaborate on why the specific number of recursive calls matters, and whether this suggests a general principle about the number of recursive calls needed for perfect generalization?

3. The paper states that "the training set can often be considerably smaller than the verification set." Could you provide specific numbers for the size of the training set versus the verification set for each of the four tasks? This would help clarify the practical implications of your approach.

4. How do you think your approach might scale to more complex tasks that require multiple levels of recursion or more intricate recursive structures?

5. The paper mentions that the NPI model naturally supports recursion, but the authors of NPI didn't explicitly consider this aspect. How do you think the NPI architecture might be modified to more directly incorporate recursion from the start, rather than relying on modified training traces?



# Review of "MAKING NEURAL PROGRAMMING ARCHITECTURES GENERALIZE VIA RECURSION"

## 1. Summary of Contributions
The paper proposes incorporating recursion into neural programming architectures to achieve provable generalization. The authors demonstrate that by modifying training traces to include recursive elements, the Neural Programmer-Interpreter (NPI) architecture can learn recursive programs that generalize perfectly to inputs of arbitrary complexity for specific tasks (grade-school addition, bubble sort, topological sort, and quicksort). They claim that their approach enables "provably perfect generalization" by verifying that the neural program handles all base cases and reduction rules, which they argue is the first time such provable guarantees have been provided for neural programs.

## 2. Decision
Reject.

Key reasons: The claims about "provably perfect generalization" are misleading and not supported by the evidence presented, as the verification procedure is only feasible for specific tasks with discrete, small input spaces and cannot be generalized to most practical programming problems.

## 3. Supporting Arguments

The paper makes a valuable empirical observation: recursion helps neural programs generalize better on specific tasks. The results on addition, bubble sort, topological sort, and quicksort demonstrate that recursive NPI versions outperform non-recursive versions on generalization, and they can verify perfect generalization for these specific tasks.

However, the paper's core claim—that recursion enables "provably perfect generalization" for neural programs—overstates the contribution. The verification procedure they describe is only applicable to tasks with finite, small verification sets (e.g., addition with 20,181 inputs, bubble sort with a single array of size 10). For most practical programming tasks (like sorting arbitrary strings, image processing, or natural language tasks), the verification set would be prohibitively large or infinite, making their approach infeasible.

The paper states: "this is the first time one can provide provable guarantees of perfect generalization for neural programs." This claim is misleading because the verification is limited to specific tasks with small input spaces where the verification set is manageable. It does not represent a general method for verifying neural programs. The authors should clarify that their "provability" is limited to tasks where the verification set can be constructed and tested, rather than implying a general theoretical framework.

The paper also fails to demonstrate how their approach would scale to more complex programming tasks. For instance, the verification set for addition requires 20,181 input problems, which is already non-trivial. For a task like sorting strings of arbitrary length with arbitrary characters, the verification set would be infinite, making their approach inapplicable.

## 4. Additional Feedback

To improve the paper, the authors should:
- Clarify the scope of their "provably perfect generalization" claim. Instead of presenting it as a general solution, they should explicitly state that it applies only to tasks with discrete, finite input spaces where the verification set can be constructed.
- Discuss the limitations of their verification procedure more thoroughly, especially for tasks with continuous inputs or infinite domains.
- Provide a more formal statement of the conditions under which their verification approach would work, and how to determine if a programming task falls within those conditions.
- Compare their verification procedure more explicitly with existing work on neural network verification (e.g., formal verification of neural networks for safety properties) to clarify what is novel about their approach.

## 5. Questions for Authors

1. How would your verification procedure scale to a task like sorting strings or images, where the input space is continuous and potentially infinite? What would be the computational complexity of verifying such a program?

2. The paper claims that "this is the first time one can provide provable guarantees of perfect generalization for neural programs." How does this differ from previous work on verifying neural networks (e.g., for safety properties), which also provides formal guarantees under specific conditions?

3. For the bubble sort task, the paper states that "the non-recursive program's correctness degrades for length 11 and beyond." How does this compare to the non-recursive NPI results from Reed & de Freitas (2016), which reported generalization up to 3000 digits for addition but didn't test beyond length 8 for bubble sort? Why does the non-recursive NPI fail at length 11 for bubble sort but not for addition?

4. The paper states that "the verification procedure we have described is inadequate" for tasks with "very large input domains." What specific changes would be needed to make the verification procedure applicable to such tasks, and would those changes fundamentally alter the approach?



# Peer Review: MAKING NEURAL PROGRAMMING ARCHITECTURES GENERALIZE VIA RECURSION

## 1. Summary of Claims and Contribution

This paper addresses a fundamental challenge in neural programming: the inability of current models to generalize beyond the complexity seen in training data. The authors propose that incorporating recursion into neural programming architectures enables provable generalization guarantees. They demonstrate this by modifying the Neural Programmer-Interpreter (NPI) training procedure to learn recursive programs for tasks including grade-school addition, bubble sort, topological sort, and quicksort. The key contribution is showing that with appropriate training traces containing recursive elements, NPI can learn programs that achieve perfect generalization on all valid inputs (verified via a procedure that checks base cases and reduction rules), whereas non-recursive versions fail on inputs beyond certain complexity thresholds. This is presented as the first work to provide provable guarantees for neural programs' generalization.

## 2. Decision

**Reject**. While the paper presents a theoretically interesting approach with promising empirical results on specific tasks, the work lacks sufficient novelty and generalizability to warrant acceptance. The core insight (using recursion for better generalization) is not novel in computer science, and the paper's main contribution is applying this concept to an existing architecture (NPI) with a specific training modification. The verification procedure is limited to very small input spaces and doesn't scale to more complex programming tasks.

## 3. Supporting Arguments

**Lack of Novelty in Core Concept**: Recursion is a fundamental concept in computer science that has been used for decades to enable generalization in programming. The paper's key contribution is applying this concept to neural programs, but the authors acknowledge that NPI already inherently supports recursion (Section 2.2). The main innovation appears to be modifying training traces to encourage recursive learning, not introducing a new neural architecture or fundamentally new concept.

**Limited Scope and Scalability**: The verification procedure only works for tasks with small, analytically tractable verification sets (e.g., addition with 20,181 input problems, bubble sort with a single array of size 10). The authors themselves acknowledge this limitation: "For tasks with very large input domains... the state space of base cases and reduction rules could be prohibitively large." The paper's "provably perfect generalization" claim is thus confined to very specific, simple tasks, not the broader neural programming domain.

**Insufficient Comparison to Baselines**: The paper compares recursive NPI to non-recursive NPI but doesn't compare to other generalization techniques like curriculum learning, which has been shown to improve generalization in previous work (e.g., Zaremba et al., 2016). Without showing that recursion provides a clear advantage over existing methods, the contribution appears incremental.

**Supervision Requirement**: The approach requires explicitly providing recursive training traces, which is a significant limitation. The authors acknowledge this as future work ("decrease supervision"), but the current method relies on a strong form of supervision that may not be practical for complex tasks.

## 4. Additional Feedback for Improvement

The paper would benefit from:

1. **Clearer Distinction of Contribution**: Explicitly state what is novel (the specific implementation of recursion in NPI via trace modification) and what is not (the concept of recursion itself).

2. **Broader Task Evaluation**: Demonstrate results on more complex programming tasks beyond the four simple examples provided. Show how the verification procedure scales (or fails to scale) with task complexity.

3. **Comparison to Alternative Generalization Techniques**: Compare to other approaches for improving generalization in neural programming (e.g., curriculum learning, architectural modifications beyond recursion).

4. **Discussion of Limitations**: More thoroughly discuss the limitations of the verification procedure for real-world programming tasks with large input spaces.

5. **Clarification of "Provable Generalization"**: Explicitly state that the proof only applies to the specific tasks with small verification sets, not to neural programs in general.

## 5. Questions for Authors

1. The paper claims recursion enables "provably perfect generalization" but acknowledges the verification procedure becomes infeasible for large input spaces. How do you propose to handle tasks with large input domains (e.g., sorting arrays of length 1000+ or processing graphs with 100+ vertices) where the verification set would be prohibitively large?

2. The paper states that NPI "naturally supports recursion" but the authors of NPI did not consider it. What specific architectural or training modifications were necessary to make NPI learn recursive programs? How different is this from simply providing recursive training traces?

3. For the bubble sort task, the paper shows that full recursion (with three recursive calls) is needed for perfect generalization, while partial recursion (with only one recursive call) is insufficient. How can one automatically determine the correct level of recursion required for a given programming task?

4. The paper claims to achieve "perfect generalization from a very small number of training examples" (e.g., 200 traces for addition). However, the verification set for addition contains 20,181 input problems. How does the training set size compare to the verification set size, and why is the model able to generalize from such a small training set to a much larger verification set?

These questions are crucial for understanding the true scope and limitations of the proposed approach. The paper's claims about "provably perfect generalization" require careful qualification given the limitations of the verification procedure.

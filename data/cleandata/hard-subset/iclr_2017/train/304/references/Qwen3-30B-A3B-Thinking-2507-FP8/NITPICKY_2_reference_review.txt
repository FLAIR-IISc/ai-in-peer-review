

# Review of "MAKING NEURAL PROGRAMMING ARCHITECTURES GENERALIZE VIA RECURSION"

## 1. Summary of Contributions
The paper proposes incorporating recursion into neural programming architectures to achieve provably perfect generalization. The authors argue that current neural programming approaches fail to generalize beyond a certain complexity threshold due to spurious dependencies on input characteristics like length. They demonstrate that by modifying training traces to include recursive elements, neural programs can achieve perfect generalization on four tasks (grade-school addition, bubble sort, topological sort, and quicksort) with small training sets. They further propose a verification procedure to prove generalization by testing only a finite set of base cases and reduction rules, which they claim is the first such provable guarantee for neural programs.

## 2. Decision
Reject. The paper makes an overreaching claim of "provably perfect generalization" that is not fully supported by the evidence, and contains multiple methodological inconsistencies that undermine the core contribution.

## 3. Supporting Arguments

### Fundamental Methodological Inconsistency
The paper claims the non-recursive programs "generalize on all input lengths we tried, up to 5000 digits" (Section 4.1), yet simultaneously claims "there is no provable guarantee that the non-recursive learned program will generalize to all inputs" (Section 4.1). This is a direct contradiction. If the non-recursive program generalizes well on all tested inputs (up to 5000 digits), then the paper's own empirical evidence contradicts its central claim about the necessity of recursion for generalization. This inconsistency severely undermines the paper's core contribution.

### Verification Procedure Limitations
The paper claims "this is the first time one can provide provable guarantees of perfect generalization for neural programs," but fails to acknowledge that their verification procedure is only applicable to specific, simple tasks where base cases and reduction rules can be enumerated (Section 3.3). They explicitly state that for "tasks with very large input domains, such as ones involving MNIST digits or speech samples, the state space of base cases and reduction rules could be prohibitively large, possibly infinite" (Section 4.2). This limitation is not adequately addressed, making the claim of "provably perfect generalization" misleading and limited to very narrow domains.

### Implementation and Presentation Flaws
Multiple formatting inconsistencies and technical inaccuracies weaken the paper's credibility:
- Inconsistent citation formats (e.g., "Vinyals et al. (2015)" vs. "Vinyals et al\.,? \Q2015\E" in references)
- Inconsistent use of K=10 vs. K=11 for the grade-school addition domain-specific encoder
- Unclear description of recursive implementation (e.g., "adding a tail recursive call into the trace for the ADD program" without explaining how this differs from the non-recursive trace)
- The paper states "the NPI model naturally supports recursion" but provides no clear evidence that the NPI architecture inherently supports recursion without modification, contradicting their own claim that "the authors of NPI did not consider this aspect explicitly"

### Empirical Evidence Gaps
The paper lacks proper statistical validation of results. For example, for topological sort, they state "the recursive version solves all graph instances we tried, from graphs of size 5 through 70," but provide no statistical significance testing to support this claim. Similarly, for bubble sort, they claim "the fully recursive version is able to sort every array given to it" but only test up to arrays of length 10 (Table 1), which is insufficient to claim perfect generalization.

## 4. Additional Feedback

### Key Clarifications Needed
1. Please clarify the contradiction between the empirical generalization of non-recursive programs on the test sets and the claim that they lack provable guarantees. How can they generalize well on all tested inputs but lack a provable guarantee?

2. Provide a clear mathematical definition of "provably perfect generalization" that encompasses both the verification procedure and its limitations. What is the formal guarantee provided by the verification procedure?

3. Explain exactly how the NPI architecture "naturally supports recursion" without modification, given that the authors had to change the training traces to enable recursion. The paper states "the NPI model naturally supports recursion" but then describes modifying the training traces, which is a significant change.

4. For the verification set construction, provide more details about how the verification set for each task was constructed and why it's sufficient. For example, for addition, you state "the verification set contains 20,181 input problems" - please explain how this number was determined to be sufficient.

### Minor Corrections Needed
- Fix inconsistent citation formats throughout the paper
- Clarify the K=10 vs. K=11 discrepancy in the grade-school addition description
- Fix grammatical errors (e.g., "the only metric that has been used is generalization of expected behavior to inputs of greater complexity" should be "the only metric that has been used is the generalization of expected behavior to inputs of greater complexity")
- In Figure 1, clarify the meaning of "ADD" on line 13 of the recursive trace

The paper presents an interesting idea with potential value for neural programming, but the current version contains substantial methodological inconsistencies and overclaims that prevent it from being accepted for publication at ICLR. The authors should address these issues before resubmission.

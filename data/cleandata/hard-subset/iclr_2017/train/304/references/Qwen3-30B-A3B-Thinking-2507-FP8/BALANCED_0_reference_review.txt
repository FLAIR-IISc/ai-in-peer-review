

# Review of "MAKING NEURAL PROGRAMMING ARCHITECTURES GENERALIZE VIA RECURSION"

## 1. Summary of Contributions

This paper addresses the critical challenge of generalization in neural programming architectures. The authors propose incorporating recursion into neural programming models as a means to achieve provably perfect generalization. They demonstrate that by modifying training traces to include recursive elements, the Neural Programmer-Interpreter (NPI) architecture can learn recursive neural programs that outperform non-recursive versions on tasks including grade-school addition, bubble sort, topological sort, and quicksort. The key contribution is introducing a verification procedure that, given a finite set of base cases and reduction rules, can prove perfect generalization for neural programs—a novel concept in neural programming research.

## 2. Decision

**Reject**. While the paper identifies an important problem and presents a promising direction, the claims of "provably perfect generalization" are not sufficiently supported by the evidence provided. The verification procedure described does not constitute a mathematical proof of generalization for all possible inputs, and the paper overstates its contribution in this regard.

## 3. Supporting Arguments

The paper makes a strong case for recursion as a solution to the generalization problem in neural programming. The motivation is well-placed in the literature, and the empirical results demonstrate that recursive neural programs do generalize better than non-recursive versions on the tasks tested. However, the paper fails to properly distinguish between verified generalization (on a finite set of inputs) and provable generalization (for all possible inputs), which is the core claim of the paper.

The main issue lies in Section 3.3, where the authors claim to "prove" perfect generalization by verifying the neural program on a verification set. For addition, they analytically construct a verification set of 20,181 inputs, which is impressive but still finite. For other tasks, they rely on "randomly generating input candidates until they completely cover the base cases and reduction rules," which is not a rigorous verification method. The paper acknowledges this limitation in Section 5 ("for tasks with very large input domains... the verification procedure we have described is inadequate"), yet continues to claim "provably perfect generalization" as if the verification procedure provides a mathematical proof.

The paper also lacks sufficient comparison to other neural programming architectures beyond the NPI. While it's true that NPI naturally supports recursion, the paper doesn't address whether other architectures could similarly benefit from recursive training or if this approach is uniquely applicable to NPI.

## 4. Additional Feedback for Improvement

The paper has significant potential but needs to address the following issues:

1. **Clarify the meaning of "provably perfect generalization"**: The paper should distinguish between mathematical proof of generalization (which would require showing correctness for all inputs) and verification on a finite set of inputs. The current usage conflates these concepts, leading to overstated claims.

2. **Strengthen the verification procedure**: For tasks other than addition, the paper relies on heuristic methods to construct verification sets. The authors should provide a more rigorous method for constructing these sets, potentially using formal methods or symbolic verification techniques to ensure completeness.

3. **Expand comparisons**: The paper should compare with other neural programming architectures (e.g., Neural Turing Machines, Differentiable Neural Computers) to demonstrate whether the recursive approach is uniquely beneficial to NPI or more broadly applicable.

4. **Address scalability**: The paper acknowledges that the verification procedure becomes infeasible for large input domains, but doesn't provide any solution or alternative approach for such cases. This is a critical limitation that needs addressing.

5. **Provide theoretical analysis**: While the paper makes an informal argument that recursion helps by decomposing problems into base cases and reduction rules, it lacks a theoretical analysis of why this decomposition improves generalization. A more formal treatment would strengthen the contribution.

## Questions for Authors

1. How do you respond to the criticism that the verification procedure described does not constitute a mathematical proof of generalization for all possible inputs, but only for a specific finite set of inputs? How would you revise the paper to more accurately reflect the nature of the verification?

2. For the bubble sort task, you modified the domain-specific encoder to directly expose comparisons between elements. How does this affect the generality of your approach, and what would happen if you used the original encoder as described in Reed & de Freitas (2016)?

3. For the topological sort task, you claim to have constructed a verification set of size 73 that covers all necessary cases. What specific theoretical guarantees do you have that this set truly covers all base cases and reduction rules, rather than just being sufficient for the specific graphs you tested?

4. How would you adapt your verification procedure to handle tasks with continuous input spaces (e.g., sorting images or processing speech), where the input domain is unbounded and the number of possible base cases is infinite?

This paper addresses an important problem in neural programming with a potentially valuable approach, but the claims of "provably perfect generalization" need to be tempered with a more accurate description of what the verification procedure actually accomplishes. With these clarifications and extensions, the paper could make a strong contribution to the field.

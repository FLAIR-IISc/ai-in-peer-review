- Paper goal: address poor generalization of neural programs by explicitly incorporating recursion into neural programming architectures.  
- Main claim: recursion enables neural programs to learn true program semantics instead of spurious dependencies, yielding provable perfect generalization.  
- Contribution 1: a novel recursive neural programming approach that aims to capture true program semantics.  
- Contribution 2: a verification procedure that proves perfect generalization by testing a finite set of base cases and reduction rules.  
- Contribution 3: empirical results showing recursive models achieve perfect generalization on addition, bubble sort, topological sort, and quicksort after training on small datasets.  
- Contribution 4: claims to be the first paper to provide provable guarantees of perfect generalization for neural programs.  
- Implementation: recursion implemented in the Neural Programmer-Interpreter (NPI) via modified training traces.  
- Empirical highlight: recursive NPI generalizes to arbitrary input complexity (e.g., addition generalizes from 1-digit to 5000-digit), while non-recursive NPI fails on longer inputs.  
- Decision: recommend acceptance with minor revisions.  
- Reason for decision 1: addresses an important, well-motivated problem in neural programming generalization.  
- Reason for decision 2: using recursion for provable generalization is novel and well-situated in prior work.  
- Reason for decision 3: empirical results convincingly demonstrate the recursive approach’s advantages.  
- Supporting argument: recursion naturally decomposes problems and helps learn true semantics, aligning with cited literature on generalization failures.  
- Supporting argument: verification procedure is significant because it reduces infinite input verification to a finite check via recursion.  
- Writing: paper is well-written, clearly organized, and connects theory to experiments.  
- Suggestion 1: clarify scope and limitations of “provably perfect generalization,” especially for large input domains like MNIST.  
- Suggestion 2: provide more detail on construction of verification sets (especially for topological sort and quicksort) and compare sizes of training, verification, and possible input sets.  
- Suggestion 3: better distinguish this approach from existing recursive neural network architectures and clarify complementarities/differences.  
- Suggestion 4: include a more detailed comparison with other generalization-improving methods (e.g., curriculum learning, architectural changes).  
- Suggestion 5: clarify the claim that training sets can be considerably smaller than verification sets with concrete per-task numbers.  
- Question 1: how might verification scale or be adapted for tasks with very large input domains, and are alternative verification approaches possible?  
- Question 2: why does the number of recursive calls (e.g., partial vs full recursion in bubble sort) affect perfect generalization, and is there a general principle?  
- Question 3: please provide exact training set vs verification set sizes for each of the four tasks.  
- Question 4: how would the approach scale to tasks needing multiple recursion levels or more complex recursive structures?  
- Question 5: could the NPI architecture be modified to natively support recursion rather than relying on modified training traces, and how?

### Review

**1. Summary of Contributions**

This paper addresses the critical problem of generalization in neural program synthesis. The authors posit that existing neural programming architectures fail to generalize to more complex inputs because they learn spurious correlations from the training data instead of the true, underlying algorithmic semantics. The core contribution is the proposal to explicitly incorporate recursion into neural programming architectures to resolve this issue. The authors argue that recursion not only improves empirical generalization but also makes it possible to formally verify the learned programs, leading to provable guarantees of perfect generalization. As a concrete demonstration, they adapt the Neural Programmer-Interpreter (NPI) architecture to learn recursive programs for four tasks: grade-school addition, bubble sort, topological sort, and quicksort. This is achieved by supervising the NPI model with execution traces that are themselves recursive. The authors claim that their recursive models achieve 100% accuracy on complex inputs after training on a very small number of simple examples. Furthermore, they outline and claim to have executed a verification procedure that, by checking a finite set of base cases and reduction rules, proves that the learned recursive programs generalize perfectly to all valid inputs of arbitrary complexity.

**2. Decision and Key Reasons**

**Decision:** Reject

**Key Reasons:**

1.  **Critical Methodological Flaws:** The design of the domain-specific encoders for several tasks appears to trivialize the learning problem by feeding pre-computed logical conditions to the network, rather than raw environmental state. Furthermore, the verification procedure for at least one task is performed on a different model architecture than the one used for the main generalization experiments, undermining the central claim of the paper.
2.  **Incomplete and Poorly Presented Manuscript:** The paper is unpublishable in its current form due to missing figures (1, 2) and tables (1, 2, 3, 4), all of which are referenced in the text and are essential for evaluating the empirical claims. Additionally, the reference section contains severe formatting errors, indicating a lack of careful preparation.

**3. Supporting Arguments**

While the core idea of using recursion for generalization and verification is highly compelling, the execution and presentation of this work are marred by significant issues.

*   **Trivialized Learning through Encoder Design:** The domain-specific encoders for the more complex tasks seem to perform the majority of the algorithmic logic externally. For quicksort, the encoder's input is `MLP ([Qarray(pj) ≤ Qarray(phi), pj == phi, ...])`. The network is not learning to compare values; it is explicitly being told the result of the comparison as a boolean input. Its task is reduced to mapping these pre-computed flags to the next program in the sequence. This is less about learning a sorting algorithm and more about learning a simple controller for an externally defined procedure. The same issue exists for topological sort. This design choice significantly weakens the claim that the model is "learning a program" in any meaningful sense and likely explains why it can be trained on a minuscule number of examples.

*   **Inconsistent Model for Verification:** In Appendix A.6, the authors state that for verifying the bubble sort program, they replaced the original domain-specific encoder with a new one that provides the boolean `Q(1, i1) ≤ Q(1, i2)` as input. This is a classic "bait and switch." The paper claims to provide provable guarantees for the models that show impressive generalization in the main experiments, but the verification is performed on a different, likely easier-to-verify model. This invalidates the link between the empirical results and the theoretical claims for this task. All verification must be performed on the exact same model that was trained and tested.

*   **Questionable Verification Sets:** The verification set for quicksort is claimed to be a single 10-element array. It is highly improbable that a single, fixed array of this size can cover all base cases and reduction rules of the Lomuto partition scheme for all possible inputs. For instance, this single array cannot possibly explore all permutations of value orderings that affect the pivot choices and recursive calls. This claim requires substantial theoretical justification which is currently absent, casting doubt on the rigor of the entire verification procedure.

*   **Missing Empirical Evidence:** The paper's core empirical claims are unsubstantiated due to the absence of Tables 1, 2, 3, and 4, and Figures 1 and 2. Without these, it is impossible to assess the performance of the non-recursive, partially recursive, and fully recursive models. For instance, the text claims Table 1 shows non-recursive bubble sort failing beyond length 8, but this table is not provided. Reviewing a paper with such critical components missing is infeasible.

*   **Formatting and Professionalism:** The manuscript is not ready for review. The title is in all caps. The citation style in the reference list is broken (e.g., `Andrychowicz and Kurach.,? \Q2016\E, Andrychowicz and Kurach.,`), suggesting a problem with the bibliography generation process that was not caught by the authors. Such errors detract from the credibility of the work and indicate a rushed submission.

**4. Additional Feedback for Improvement**

The central premise of this paper is excellent and addresses a fundamental challenge in the field. I encourage the authors to revise their work substantially, as the direction is promising.

*   **Revise Encoders:** The encoders must provide the model with a more direct view of the environment state (e.g., the actual numerical values at pointer locations), forcing the network to learn the necessary logical operations (like comparison) itself. Success under these more realistic conditions would make the results far more compelling.
*   **Ensure Methodological Consistency:** The exact same model architecture and configuration must be used for training, generalization testing, and formal verification. Any deviation must be rigorously justified.
*   **Strengthen Verification Claims:** The construction of the verification set `V` needs to be more transparent and justified. For tasks like quicksort, provide a formal argument for why the chosen set is sufficient to cover all relevant execution paths.
*   **Complete the Manuscript:** Before resubmission, the paper must be thoroughly proofread. All figures, tables, and references must be correctly included and formatted. The pseudocode in Algorithm 1 should be clarified (e.g., `function RUN` call syntax).

**5. Questions for the Authors**

1.  Could you justify the decision to provide pre-computed boolean conditions to the encoders for quicksort and topological sort? Did you experiment with encoders that provide raw values (e.g., `Qarray(pj)` and `Qarray(phi)`), and if so, how did the models perform?
2.  What was the rationale for using a different encoder for the verification of the bubble sort model than the one used for the main generalization experiments?
3.  Could you provide a more detailed argument for why a single 10-element array constitutes a complete verification set for quicksort, covering all necessary base cases and reduction rules? Which specific cases does this array cover?
4.  The current submission is missing Figures 1 and 2, and Tables 1, 2, 3, and 4. Are the textual descriptions of their contents fully accurate?

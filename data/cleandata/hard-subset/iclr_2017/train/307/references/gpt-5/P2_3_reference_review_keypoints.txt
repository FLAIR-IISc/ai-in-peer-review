- Addresses evaluation of end-to-end goal-oriented dialogue systems, focusing on state tracking, KB interaction, and task completion rather than chit-chat.
- Proposes five synthetic controlled tasks (slot elicitation/updating, KB querying, sorting KB returns, providing attributes, full-dialog) in a restaurant domain.
- Introduces an OOV test split to assess generalization to unseen entities.
- Reformats DSTC2 into the same task framework (Task 6) for comparative evaluation.
- Evaluates on a proprietary Concierge dataset to test ecological validity.
- Provides baselines: rule-based, IR, supervised embeddings, MemN2N, and a “match type” feature for entity handling.
- Finds models can learn API call issuance/updates and handle OOVs with type features but struggle with interpreting KB returns and multi-step reasoning.
- Reports low per-dialog success on the harder tasks despite reasonable per-response accuracy.
- Decision: Accept.
- Reason: The benchmark fills a reproducible, diagnostic evaluation gap and should have strong community impact.
- Empirical study is rigorous: diverse baselines, OOV testing, synthetic and real data, and careful hyperparameter tuning.
- Uses complementary metrics (per-response and per-dialog), highlighting a gap important for goal completion.
- OOV evaluation uses principled disjoint KB splits to probe generalization.
- Ablations (hops, match-type features) substantiate claims about reasoning limits and entity handling benefits.
- Limitation: Synthetic tasks use fixed templates and deterministic orders, possibly under-representing real dialogue variability.
- Limitation: Ranking setup simplifies generation and may not reflect fully generative system performance.
- Limitation: Match-type features inject domain knowledge and assume KB access, reducing purity of end-to-end approach.
- Limitation: DSTC2 conversion and split changes prevent direct leaderboard comparison.
- Suggested improvements: increase simulator variability, add generative baselines with copy mechanisms, and provide finer-grained error analyses for hard tasks.
- Suggested improvements: report per-dialog accuracy for DSTC2/Concierge and clarify candidate set size effects on ranking difficulty.
- Suggested analysis: relate match-type features to copy/pointer networks and assess model-agnostic alternatives.
- Questions: sensitivity to removing simulator determinism and whether MemN2N gains persist.
- Questions: practical entity identification method for type features and its error rate on real data.
- Questions: whether training access to both KBs implies type lexicon exposure to OOV entities and how that affects performance.
- Questions: request per-dialog accuracy for DSTC2/Concierge, hop attention analysis for Task 3, and code/data release for reproducibility.

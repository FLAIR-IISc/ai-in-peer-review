- Proposes an open, reproducible benchmark for end-to-end goal-oriented dialog focused on restaurant booking, decomposed into five synthetic subtasks plus two real-data evaluations.
- Introduces an OOV split forcing generalization to unseen entities (locations, cuisines, restaurants).
- Evaluates diverse baselines (rule-based, IR, supervised embeddings) and end-to-end Memory Networks (MemN2N) with a simple match-type feature for entities.
- Aims to enable consistent comparisons, diagnose missing capabilities, and test whether synthetic tasks correlate with real-dialog performance.
- Decision: Accept — benchmark is well-motivated and fills a clear gap for KB-grounded, OOV-capable goal-oriented dialog evaluation.
- Experimental setup is careful with diverse baselines and metrics, yielding insightful findings about model capabilities and failures.
- Evidence shows performance patterns on synthetic tasks transfer to DSTC2 and Concierge, supporting external validity.
- Positions itself well relative to prior work (POMDP/slot-filling, chit-chat, dialog QA, DSTC) and adapts the bAbI philosophy to dialog+KB operations.
- Evaluation uses per-response and per-dialog accuracy, a clear OOV split, and a transparent, reproducible methodology.
- Key empirical findings: IR is weak, supervised embeddings struggle on KB reasoning, MemN2N can issue/modify API calls but struggles with KB interpretation/sorting, and match-type features are crucial for OOV.
- Match-type features act as a simple but effective mechanism to handle entities/OOVs.
- IR methods fail due to rapid goal-directed dialog moves and low lexical overlap.
- Supervised embeddings do moderately well but fail at KB reasoning and sorting tasks.
- MemN2N advances substantially on API issuance/updating but has limits on interpreting and sorting KB results.
- Suggest clarifying candidate ranking vs generation choice and reporting auxiliary metrics like Recall@k or small candidate-pool studies.
- Candidate set construction (including test candidates) needs justification and sensitivity analysis to rule out leakage.
- Request more granular error analysis (per-turn confusion for T3/T5 and typical failure modes in sorting and option acceptance/termination).
- Dataset-design critique: deterministic slot prompt order in T1 risks overfitting; recommend paraphrastic variation and policy randomness.
- For DSTC2, quantify artifacts introduced by adding ratings and the fraction of non-deterministic bot behavior.
- For Concierge, removing named entities via NER may obscure entity handling; suggest releasing de-identified but typed entities.
- Request learning curves (1k–10k+) to show data efficiency and whether scaling reduces T3/T5 gaps.
- Modeling suggestion: compare to pointer/copy networks or key–value memory networks and ablate against genuine copy mechanisms.
- For sorting (T3), analyze whether failures stem from inability to learn ordering vs long-context reasoning limits; try ranking supervision or differentiable sort modules.
- Report sensitivity to number of hops, memory size, and stability across random seeds.
- Reproducibility demands: release code, data generation scripts, KBs (both splits), DSTC2 conversion, and document licenses and lexicons.
- Clarify whether training access to both KBs artificially eases OOV handling and whether that setting reflects realistic deployment.
- Ask authors to justify including test candidates in candidate sets and report results when candidates are restricted to training/templates.
- Ask whether exposing both KBs during training eases OOV and how performance changes if the second KB is hidden until test time.
- Ask whether MemN2N attention indicates any learned ordering for T3 and whether auxiliary losses or architecture variants were tried.
- Ask DSTC2 conversion authors to quantify how often original dialogs disagreed with "pick highest rating" and how such cases were handled.
- Ask about robustness to paraphrase diversity, ASR-like noise, and longer dialogs.
- Ask whether models trained on T1–T4 transfer zero-shot to other domains (e.g., hotels) given only a new KB schema and type lexicon.
- Overall assessment: a timely, well-executed benchmark with solid baselines and useful insights likely to catalyze progress on goal-oriented dialog.

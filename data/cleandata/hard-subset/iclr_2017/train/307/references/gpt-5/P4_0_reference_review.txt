1. Summary of claimed contributions
The paper introduces an open, lightweight, and reproducible benchmark suite for evaluating end-to-end goal-oriented dialog systems in a restaurant reservation domain. It decomposes the overall objective into five synthetic tasks (issuing/updating API calls, displaying sorted options from a KB, providing requested KB attributes, and full dialogs), plus two evaluations on real dialogs: a reformulated version of DSTC2 and a proprietary concierge dataset. The benchmark isolates core capabilities such as dialog management without explicit state, KB querying and interpretation, handling OOV entities, and multi-turn coordination. The paper also reports extensive baselines (rule-based, IR, supervised embeddings) and evaluates End-to-End Memory Networks (MemN2N), proposing a simple but effective “match type” feature to better handle entities and OOVs. Key findings are that MemN2N performs best among learned methods, can learn to issue/update API calls and handle some OOVs, but still struggles with ranking/sorting KB outputs and achieving end-to-end per-dialog success. The authors argue the suite enables focused error analysis and serves as a proxy for performance on real data.

2. Decision
Accept. Primary reasons: (i) a clearly described, well-motivated, and open benchmark that targets core capabilities of goal-oriented dialog with controlled, analyzable tasks, and (ii) thorough experimental evaluation that supports the claims and yields actionable insights on model strengths/limitations, including OOV handling.

3. Supporting arguments
- Problem and motivation: The paper tackles a central question—can end-to-end models replace traditional slot-filling methods in narrow goal-oriented domains, and where do they fall short? The decomposition into subtasks is well grounded and aligned with the literature on dialog management vs. KB reasoning (POMDPs, DSTC). The work addresses a gap: lack of standardized, reproducible resources for training/testing end-to-end models in goal-oriented settings.
- Placement in literature: The benchmark is positioned relative to chit-chat datasets (Twitter, Ubuntu), DSTC resources, user simulators, and the bAbI QA tasks. The choice to assess MemN2N builds on prior successes in QA and non-goal dialog, and the match-type augmentation is motivated by known issues with entities in embeddings.
- Technical soundness: The task generator, KB schema, and OOV protocol (disjoint cuisines/locations across KBs) are clearly specified. The training/evaluation setup (ranking among a global candidate set, per-response/per-dialog metrics) is well defined. Baselines span rule-based, IR, and learned embedding models. Hyperparameters are cross-validated and architectural details are provided.
- Claims vs. evidence: Results consistently show MemN2N outperforming IR and supervised embeddings on synthetic and real tasks; rule-based systems solve synthetic tasks but perform worse than MemN2N on DSTC2. The match-type feature improves OOV handling and Task 4 (entity attribute retrieval) as claimed. The persistent per-dialog failures on T3/T5 substantiate the claim that current end-to-end models still struggle with KB-result interpretation/sorting and reliable full-dialog completion. The pattern of relative performance carries over to DSTC2 and Concierge, reinforcing the proxy claim.

4. Additional feedback (to help improve the paper; not all points affect the decision)
- Scope and realism: The synthetic tasks are necessarily templated and partly deterministic (e.g., fixed slot-question order in T1, sorted listing in T3). Consider adding variants with randomized prompt orders, linguistic paraphrasing, and noise to stress robustness and reduce overfitting to templates.
- Evaluation modality: The ranking setup simplifies output control and avoids NLG complexities. Reporting complementary seq2seq/transformer generation baselines (with copy/pointer) would strengthen conclusions about generalizability beyond ranking.
- Stronger entity baselines: Given the centrality of entities, compare against pointer/copy models and key-value memory networks to contextualize the match-type feature.
- Candidate set composition: Using a global candidate set including test utterances is conventional in retrieval evaluation but worth explicitly discussing in terms of potential leakage and comparability to real deployment scenarios.
- DSTC2 conversion: The addition of ratings to enable deterministic option ranking is pragmatic; reporting an ablation on this design choice (or a non-deterministic variant) would be informative.
- Error analysis: Include qualitative breakdowns for T3/T5 errors (e.g., failures in sorting vs. failure to attend to relevant KB facts vs. dialog state tracking) and per-turn confusion categories.
- Reproducibility and access: The synthetic tasks and converted DSTC2 are open; clarify the availability or a surrogate for the Concierge dataset to allow community benchmarking on real data.

Questions for the authors
- Candidate set: Does the candidate pool include all test responses verbatim? If so, can you quantify how candidate set size/composition impacts performance, and do you provide a fixed candidate set for fair comparison?
- OOV protocol: Models have access to both KBs during training for typing. Could you report performance when KB2 (OOV entities) is withheld during training to isolate generalization from type-only signals?
- Sorting behavior in T3: Did you analyze whether MemN2N failures are due to inability to learn sorting, attention misalignment, or insufficient memory capacity? Have you tried architectural biases for sorting/ranking (e.g., pointer networks)?
- Real-data robustness: On DSTC2/Concierge, how sensitive are results to ASR errors, typos, and paraphrases? Any experiments with adversarial paraphrasing or noise augmentation?
- Ablations: Can you provide ablations for time features, number of hops, and the match-type feature per entity type? Which types contribute most to OOV gains?
- Generalization: How does performance scale with training set size and dialog length? Any evidence of compositional generalization to unseen combinations of slots beyond those sampled?
- Release: Will code for data generation, task conversion, and model training be released alongside the datasets to ensure full reproducibility?

Overall, the paper provides a useful benchmark and careful empirical study that are likely to catalyze progress on end-to-end goal-oriented dialog. The methodological novelty is limited, but the resource and analysis are strong and well supported.

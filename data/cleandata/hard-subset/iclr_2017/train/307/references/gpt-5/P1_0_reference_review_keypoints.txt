- Introduces a lightweight, reproducible testbed for end-to-end goal-oriented dialogue focused on restaurant reservation.
- Proposes five synthetic tasks (T1–T5) targeting slot elicitation, updating, KB querying, option ranking, and using KB facts with in-vocab and OOV splits.
- Converts DSTC2 into the same format (T6) and evaluates on an anonymized concierge dataset to test realism and robustness.
- Standardizes evaluation via per-response and per-dialog accuracy in a next-utterance ranking setup.
- Provides extensive baselines: rule-based, TF-IDF, nearest neighbor, supervised embeddings, and MemN2N with/without match type features.
- Finds MemN2N outperforms IR and supervised embeddings, can learn API call behaviors and handle OOVs with match types.
- Identifies persistent failures: option display/sorting and end-to-end dialog performance; per-dialog accuracy remains low.
- Observes that performance trends on synthetic tasks transfer to DSTC2 and Concierge, making the benchmark a useful proxy.
- Decision: Accept.
- Key reason 1: Provides a well-motivated, reproducible benchmark enabling fine-grained error analysis and OOV tests.
- Key reason 2: Thorough empirical study that yields actionable insights about end-to-end models’ strengths and limitations.
- Problem/motivation: Addresses gap between chit-chat benchmarks and goal-oriented dialogue needs (slot tracking, KB access, entity handling).
- Placement in literature: Well positioned relative to bAbI, chit-chat corpora, DSTC, and user simulation work; argues prior resources are insufficient.
- Scientific rigor: Comprehensive experiments, OOV splits, ablations, multiple datasets, and detailed metrics and error analyses.
- Novelty/insight: Match type features are simple but effective; main contribution is empirical insight rather than model novelty.
- Reproducibility/usefulness: Releases synthetic tasks and DSTC2 conversion, decomposing dialog competencies for targeted progress.
- Suggestion: Increase linguistic variability, non-deterministic slot order, multiple valid next actions, and ASR-like noise to improve realism.
- Suggestion: Clarify implications of the ranking setup and consider train-only or scalable retrieval candidate sets to avoid bias.
- Suggestion: Include stronger neural baselines (pointer/copy, key–value memories, transformers, pointer networks) for copying and sorting tasks.
- Suggestion: Discuss potential biases from DSTC2 rating tie-breakers and test alternative deterministic or non-deterministic evaluations.
- Suggestion: Report learning curves and robustness to training size, candidate set size, hops, memory size, and use of history vs last utterance.
- Suggestion: Release DSTC2 conversion scripts and rule/patterns for rule-based baselines for transparency.
- Question: Does including test utterances in the candidate pool risk leakage or bias; what changes with train-only or large distractor pools?
- Question: Can you quantify stricter OOV where OOV KB entities are absent at train time and when KB schema changes?
- Question: Provide diagnostics on MemN2N failures in sorting vs tracking proposed options vs conditioning on accept/reject.
- Question: How sensitive are DSTC2 results to rating tie-breakers and original system non-determinism; any sanity checks on converted API calls?
- Question: For Concierge, share intent distribution, proportion of KB-required turns, operator style consistency, and impact on performance gap.
- Question: Do match type features act as exact-match copy signals; have you tried combining with character/subword embeddings for OOVs?
- Question: Have you considered end-to-end interaction with a user simulator to measure task success and dialog length beyond ranking metrics?

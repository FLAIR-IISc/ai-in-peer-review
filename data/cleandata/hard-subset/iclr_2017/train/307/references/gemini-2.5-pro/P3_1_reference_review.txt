### Summary of Contributions

This paper introduces a valuable new resource for the dialogue systems community: a set of goal-oriented dialogue tasks designed to benchmark and analyze end-to-end models. Drawing inspiration from the bAbI tasks for question answering, the authors propose a suite of five synthetic tasks in the restaurant reservation domain, which systematically test different capabilities such as dialogue management, issuing API calls, and interpreting knowledge base (KB) results. A key strength of the contribution is the careful design, which includes a standard test set and an Out-Of-Vocabulary (OOV) test set to specifically probe model generalization to unseen entities. Crucially, the authors validate their synthetic benchmark by showing that the performance patterns of various models on these tasks correlate well with their performance on two real-world datasets: a converted version of DSTC2 and a new dataset from a human-operated concierge service. The paper provides strong baselines, including IR methods, supervised embeddings, and Memory Networks, demonstrating that while current end-to-end models are promising—outperforming a rule-based system on real data—they still fall short of solving complex, multi-turn goal-oriented dialogues, as highlighted by low per-dialogue accuracy scores.

### Decision and Key Reasons

**Decision:** Accept

1.  **Addresses a Critical Need:** The paper tackles a significant and well-recognized problem in the field: the lack of standardized, reproducible, and analyzable benchmarks for goal-oriented dialogue. By providing an open-source, lightweight testbed, it facilitates rigorous comparison and focused research, moving beyond less-reproducible human evaluations or datasets not suited for end-to-end training.

2.  **Excellent Methodological Rigor:** The design of the task suite is thoughtful and methodologically sound. The decomposition of a full dialogue into subtasks (T1-T5) enables targeted error analysis. Furthermore, the inclusion of the OOV split and the validation against two real-world datasets (DSTC2 and Concierge) provide strong evidence that the proposed synthetic benchmark is a meaningful proxy for real-world challenges, which is a major strength of this work.

### Supporting Arguments

The primary contribution of this paper is the dataset and evaluation framework, and it excels in this regard. The motivation is compelling: while end-to-end models have shown success in chit-chat, their ability to handle the structured reasoning required for goal-oriented tasks is an open and critical question. This paper provides the tools to start answering it systematically.

The task design is a clear strength.
*   **Decomposition:** Breaking down the problem into issuing API calls (T1), updating them (T2), displaying options (T3), and providing details (T4) before combining them (T5) is highly effective. It allows researchers to pinpoint model weaknesses. The experimental results clearly show this, with models succeeding on T1/T2 but failing on T3, indicating that interpreting and acting upon KB results is a key bottleneck.
*   **OOV Generalization:** The creation of disjoint KBs to test OOV generalization is a crucial feature. It directly addresses a known weakness of embedding-based models and provides a concrete way to measure progress on this front. The success of the "match type features" demonstrates the utility of this test setup.
*   **Validation:** The effort to show that performance trends on the synthetic tasks mirror those on DSTC2 and the Concierge dataset is commendable and significantly elevates the paper's impact. It provides confidence that improvements on these tasks are likely to translate to real-world performance gains.

The experimental evaluation is thorough. The choice of baselines covers a reasonable spectrum from classic IR to more recent neural architectures like Memory Networks. The results are insightful:
*   The superiority of MemNNs over supervised embeddings and IR methods confirms the value of more structured architectures with explicit memory for this type of task.
*   The fact that the hand-crafted rule-based system fails to outperform the MemNN on the noisy, real-world DSTC2 data is a powerful argument for the potential of end-to-end learning approaches.
*   The stark contrast between per-response and per-dialogue accuracy is an important finding for the community. It highlights that achieving local correctness is insufficient and that maintaining coherent state and strategy over multiple turns is the real challenge, which this benchmark effectively measures.

### Additional Feedback for Improvement

While this is a strong paper, the following points could be addressed to further improve its clarity and impact:

1.  **Ranking vs. Generation:** The evaluation is framed as a ranking task over a candidate set. This is a practical and standard evaluation choice, but it is a simplification of the full dialogue problem, which often requires generation. It would be beneficial for the authors to explicitly discuss this limitation and speculate on how the challenges might differ in a generative setting. For instance, would models be more prone to hallucinating entities or failing to adhere to the KB?

2.  **Analysis of Task 3 Failure:** The paper notes that models struggle with Task 3 (Displaying options), which involves sorting restaurants by rating. The analysis could be deepened here. Is the failure due to the symbolic reasoning required for sorting, the need to iterate through a list of options, or something else? A more detailed error analysis on this specific task would be very insightful. For example, does the model propose unsorted options, or does it fail to propose any valid options at all?

3.  **Nature of the Concierge Data:** The processing of the Concierge dataset involves removing NER entities. While this is necessary for anonymization, it potentially removes much of the semantic grounding (e.g., specific locations or restaurant names) that the models are supposed to learn to handle. A brief discussion on how this might affect the results and the comparison to the KB-grounded tasks would be helpful.

### Questions for the Authors

1.  The "match type features" are a simple yet effective mechanism. Could you elaborate on why this feature provides such a significant boost for the Memory Network but has a lesser impact on the other models? Does it suggest that the MemNN architecture is better poised to integrate such discrete, symbolic features alongside learned distributed representations?

2.  Regarding the failure on Task 3, did you investigate whether the sorting requirement was the primary cause of errors? Have you considered a variant of Task 3 without the sorting constraint to isolate this difficulty?

3.  The user simulator is based on a limited set of templates. How do you think the performance of the tested models would change if the linguistic variety of the user utterances was significantly increased, even while keeping the underlying semantics the same? Does the current setup risk rewarding models that learn surface-level template matching?

### Summary of Contributions

This paper introduces a new benchmark suite, in the spirit of the bAbI tasks, for evaluating end-to-end goal-oriented dialog systems. The authors argue that while end-to-end models have shown promise in chit-chat settings, their capabilities in goal-oriented scenarios are poorly understood due to a lack of suitable, reproducible evaluation frameworks. To address this, they propose a set of six tasks centered around restaurant reservations. The first five tasks are generated via a simulator, allowing for a controlled environment to test specific sub-skills: issuing API calls (Task 1), updating them (Task 2), displaying options from a Knowledge Base (KB) (Task 3), providing extra information (Task 4), and conducting a full dialog (Task 5). Crucially, the benchmark includes an out-of-vocabulary (OOV) test set to evaluate generalization to unseen entities. The final task (Task 6) is derived from the real-world DSTC2 dataset, and the authors also provide results on a proprietary Concierge dataset to validate that the patterns observed on the synthetic tasks are a reasonable proxy for real-world performance. The paper provides comprehensive baseline results using IR methods, supervised embeddings, and Memory Networks, demonstrating that even strong end-to-end models struggle with these tasks, thereby highlighting key areas for future research.

### Decision and Justification

**Decision: Accept**

This paper represents exactly the kind of foundational contribution that pushes a field in a new and productive direction. My decision is based on two key reasons:

1.  **A Bold and Necessary Conceptual Shift:** The paper's primary contribution is not an incremental improvement to an existing model, but a bold proposal for a new way to measure progress in goal-oriented dialog. By creating a structured, analyzable, and reproducible testbed, it moves the community away from monolithic evaluations on noisy data towards a more scientific approach where model capabilities can be isolated and rigorously tested. This is a significant service to the field that has the potential to accelerate meaningful progress.
2.  **Insightful Task Design and Analysis:** The decomposition of the dialog problem into distinct subtasks is highly insightful. This design provides a built-in mechanism for error analysis, allowing researchers to understand *why* a model fails, rather than just observing *that* it fails. The results, showing that models can handle state tracking for API calls but fail at reasoning over KB results, offer a crucial diagnostic. The explicit inclusion of an OOV setting to test generalization is another forward-thinking and critical feature that addresses a major real-world challenge.

### Supporting Arguments

The main strength of this work lies in its vision. For too long, progress in end-to-end dialog has been hampered by a lack of standardized benchmarks that are both challenging and interpretable. This work takes a direct and effective shot at solving this problem. The analogy to the bAbI tasks is apt; just as bAbI provided a clear set of "prerequisite" skills for question answering, these tasks provide a clear set of skills for goal-oriented conversation. This provides a clear roadmap for researchers: build models that can solve these tasks sequentially and in combination.

The experimental results powerfully demonstrate the utility of the benchmark. The authors show that simple IR baselines are surprisingly competitive, while standard supervised embedding models fail on almost all tasks in terms of per-dialog accuracy. The Memory Network results are particularly illuminating. They succeed at the simpler state-tracking tasks but fail at tasks requiring reasoning over retrieved KB information (Task 3) or combining all skills (Task 5), with per-dialog accuracies remaining near zero. This is not a failure of the paper, but a success of the benchmark in exposing the brittleness of current models. The performance jump seen with the simple but effective "match type features," especially on OOV entities, further highlights how current neural architectures struggle with symbolic reasoning and generalization, providing a clear target for future architectural innovations.

Finally, the inclusion of experiments on DSTC2 and the Concierge dataset provides an important sanity check. The fact that the relative performance of the different models remains consistent across the synthetic and real datasets lends significant credibility to the benchmark as a valid proxy for real-world challenges, justifying its use as a primary tool for development and analysis.

### Additional Feedback for Improvement

While I am very enthusiastic about this paper, the following points could further strengthen it:

*   **Generation vs. Ranking:** The current evaluation is framed as a ranking task over a large set of candidates. This is a practical choice but sidesteps the harder problem of generation. It would be beneficial for the authors to explicitly discuss this limitation and perhaps outline how the benchmark could be adapted for generative evaluation in the future.
*   **Linguistic Diversity:** The simulator uses a limited set of templates to generate utterances. This controlled setup is a strength for isolating reasoning capabilities, but it is also a significant simplification. The authors should be more upfront about this limitation in the main text and could suggest that future work might involve incorporating paraphrasing models or more complex user simulators to increase linguistic variety.
*   **Analysis of "Match Type Features":** The hybrid nature of adding symbolic "match type features" to the Memory Network is fascinating and effective. A deeper analysis of this mechanism would be welcome. Is the model simply learning to copy entities of a certain type, or is it learning a more abstract rule about how to handle, for instance, a "phone number" entity? A small ablation study or visualization could provide valuable insights.

### Questions for the Authors

*   The user simulator has deterministic elements (e.g., the order of questions in Task 1). To what extent do you believe models might be exploiting these regularities, and have you considered introducing more stochasticity into the user and bot simulators to test for more robust strategies?
*   The per-dialog accuracy is a very strict, all-or-nothing metric. While it clearly shows the task is unsolved, have you considered a more relaxed "task success" metric? For example, a dialog could be considered successful if the correct restaurant is eventually identified, even if there are minor errors in intermediate turns. This might provide a more granular measure of progress.
*   The candidate set for ranking is composed of all possible responses from all tasks combined. Could this large and often out-of-context set of distractors make the task artificially difficult or easy in certain ways? How might the results change if a more contextually-aware method for sampling negative candidates was used?

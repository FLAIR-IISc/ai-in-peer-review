### 1. Summary of Contributions

This paper introduces a valuable open-source resource for testing and analyzing end-to-end goal-oriented dialog systems. The core contribution is a suite of six tasks designed in the context of restaurant reservations. The first five tasks are synthetically generated using a simulator and a knowledge base (KB), systematically breaking down the dialog process into distinct sub-skills: issuing API calls (Task 1), updating them (Task 2), presenting options from KB results (Task 3), providing extra information (Task 4), and conducting a full dialog (Task 5). This bAbI-style design facilitates fine-grained error analysis. The framework also includes an Out-Of-Vocabulary (OOV) test set to evaluate model generalization to unseen entities.

To validate the relevance of the synthetic tasks, the authors supplement them with two datasets based on real interactions: a conversion of the DSTC2 dataset (Task 6) and a new dataset from a human-operated online concierge service. The paper provides a thorough empirical evaluation on this new benchmark, comparing classical IR baselines, supervised embedding models, and Memory Networks. It also proposes a "match type" feature to improve the handling of KB entities, particularly in OOV scenarios. The results demonstrate that while modern end-to-end models like Memory Networks outperform other methods, they still struggle with key aspects of goal-oriented dialog, such as interpreting and utilizing KB query results, as highlighted by the low per-dialog completion rates.

### 2. Decision and Key Reasons

**Decision:** Accept

1.  **Well-Motivated and High-Quality Benchmark:** The paper addresses a clear and significant gap in the literature: the lack of a standardized, reproducible, and analyzable benchmark for end-to-end goal-oriented dialog systems. The proposed task suite is well-designed, allowing for controlled experiments and targeted analysis of model capabilities, which is a substantial improvement over evaluating on noisy, non-reproducible, or chit-chat-focused datasets.
2.  **Thorough and Insightful Empirical Analysis:** The paper provides a comprehensive set of baselines and a rigorous evaluation that yields valuable insights. The comparison across synthetic and real-world datasets validates the utility of the proposed tasks as a proxy for real-world performance. The analysis clearly highlights the specific failure modes of current end-to-end models, providing a clear path forward for future research.

### 3. Supporting Arguments for the Decision

The paper's strength lies in its meticulous construction of a new resource and the depth of the accompanying analysis.

*   **Clarity and Soundness of Task Design:** The motivation for the benchmark is exceptionally clear. The paper effectively argues why existing resources are insufficient for systematically training and evaluating end-to-end goal-oriented systems. The decomposition of a complex dialog into five distinct, progressively harder tasks is logically sound and serves its intended purpose of enabling targeted error analysis. For example, the contrast in model performance between Task 2 (updating API calls) and Task 3 (displaying options) clearly pinpoints that reasoning over KB results is a major bottleneck, a conclusion that would be difficult to draw from an undecomposed, monolithic task. The inclusion of the OOV test set is crucial for assessing a model's ability to generalize beyond its training vocabulary, a critical requirement for real-world deployment.

*   **Technical Soundness and Rigor of Evaluation:** The experimental setup is scientifically rigorous. The paper evaluates a sensible range of models, from simple IR baselines (TF-IDF, Nearest Neighbor) to strong supervised learning models (Supervised Embeddings, Memory Networks). The inclusion of a rule-based system provides an important point of comparison, demonstrating the performance ceiling on the synthetic tasks and offering a realistic baseline for the noisier DSTC2 task. The choice of evaluation metrics—per-response and per-dialog accuracy—is appropriate and effectively illustrates the core challenge: stringing together individually correct responses to achieve a final goal is non-trivial. The paper's claim that performance patterns on the synthetic tasks are a good proxy for real data is strongly supported by the consistent relative rankings of the models across tasks T1-T5, T6 (DSTC2), and the Concierge dataset.

*   **Novelty and Impact:** The primary contribution is a novel dataset and evaluation framework. While inspired by the bAbI tasks for question answering, its application to and design for goal-oriented dialog is a novel and impactful contribution. It provides the community with a much-needed tool to move beyond chit-chat evaluation and make concrete, measurable progress on goal-oriented systems. The proposed "match type" feature, while a minor technical contribution, is a simple yet effective technique that directly addresses a known weakness of embedding-based models in handling entities, further strengthening the paper's contribution.

### 4. Additional Feedback for Improvement

These points are intended to be constructive and are not critical to the decision.

*   The description of the candidate set is that it contains all bot utterances and API calls from all train, validation, and test sets combined. It would be helpful to report the size of this candidate set for each task, as this is a key factor in the task's difficulty.
*   In Task 1, the bot is said to ask for information in a "deterministic order". This might make the task easier than a real-world scenario. A future version of the dataset could include a variant with a randomized order of questions to test more robust dialog management.
*   The paper's focus on a ranking-based evaluation ("Next-Utterance-Classification") is a valid and common simplification. However, a brief discussion on the challenges of moving to a generative setting for these tasks could add valuable context for future work.

### 5. Questions for the Authors

*   The per-dialog accuracy for the Concierge dataset is not reported in Table 2. Is this because the notion of a single, clear "dialog goal" is ill-defined in this more complex human-human dataset, or was this metric simply omitted?
*   In the Memory Network implementation (Appendix A), KB facts resulting from an API call are stored in memory as "user utterances". This seems like a potentially lossy or unnatural representation. Did you experiment with alternative representations, such as storing KB facts in a separate, more structured memory, and if so, how did that affect performance?
*   Could you provide more detail on how the "match type features" were integrated into the non-MemNN baselines like TF-IDF Match and Supervised Embeddings? The description in Section 4.5 and Appendix D is brief, and a clearer explanation of the feature engineering for these models would be beneficial.

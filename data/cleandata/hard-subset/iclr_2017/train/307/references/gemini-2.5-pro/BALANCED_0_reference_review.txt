### **Summary**
This paper introduces a valuable new resource for the research community: a set of diagnostic tasks for evaluating end-to-end, goal-oriented dialog systems. The authors rightly point out that while end-to-end models have shown promise in chit-chat settings, their capabilities in practical, goal-oriented scenarios are less understood, partly due to a lack of suitable benchmarks. Drawing inspiration from the bAbI tasks for question answering, the paper proposes a suite of five synthetic tasks in the restaurant reservation domain. These tasks are designed to test specific capabilities in a controlled manner, such as issuing API calls, updating requests, interpreting KB query results, and providing information. Crucially, the benchmark also includes an out-of-vocabulary (OOV) test set to assess model generalization to unseen entities. The paper provides a thorough empirical evaluation of several baselines, from classic IR methods to Memory Networks, offering a clear analysis of their relative strengths and weaknesses. The authors further validate the utility of their synthetic benchmark by demonstrating that the performance patterns of these models carry over to two datasets based on real human interactions: a converted version of the DSTC2 dataset and a novel dataset from an online concierge service. The key finding is that while current end-to-end models can learn some aspects of goal-oriented dialog, they still fall short on crucial skills like reasoning over knowledge base results, leading to low overall task completion rates.

### **Decision and Key Reasons**
**Decision:** Accept

1.  **A Well-Designed and Much-Needed Benchmark:** The primary contribution is a public, diagnostic benchmark for goal-oriented dialog, a critical research area where standardized evaluation has been a significant challenge. The bAbI-style breakdown of a complex dialog into distinct sub-skills is an excellent design choice that facilitates targeted model development and rigorous error analysis.
2.  **Insightful and Comprehensive Empirical Analysis:** The paper accompanies the dataset with a strong set of experiments that establish competitive baselines and, more importantly, leverages the task design to provide clear insights into the specific failure modes of current end-to-end models. The validation of these findings on two real-world datasets substantially strengthens the claim that the proposed synthetic tasks are a useful proxy for real-world challenges.

### **Supporting Arguments for the Decision**
The paper successfully identifies and addresses a clear gap in the existing resources for dialog system research. Traditional goal-oriented datasets (like DSTC) were not designed for end-to-end learning, while large-scale chit-chat corpora lack the goal-oriented structure and grounding in a knowledge base needed for many practical applications. This paper’s contribution provides a middle ground that is both structured and suitable for modern neural models.

The design of the tasks is a major strength. By isolating skills like API call generation (Task 1), state updates (Task 2), and KB result presentation (Task 3), the benchmark allows researchers to pinpoint exactly where their models are failing. For instance, the results clearly show that Memory Networks are relatively successful at tasks requiring implicit state tracking (T1, T2) but struggle significantly with tasks that require interpreting and acting upon the structured information returned from a KB (T3, T5), as evidenced by the sharp drop in per-dialog accuracy. This level of detailed analysis is invaluable for guiding future research.

The inclusion of an OOV test set is another significant strength. It forces models to move beyond memorization and addresses the critical challenge of handling new entities, which is fundamental for any real-world deployment. The paper's solution, "match type features," provides a strong baseline for this problem while also highlighting the limitations of standard embedding-based approaches.

Finally, the experimental comparison to real-world data is crucial. A common criticism of synthetic datasets is their potential lack of relevance. By showing that the relative performance of different models (MemNN > Supervised Embeddings > IR) holds across the synthetic tasks, the noisier DSTC2 data, and the human-human Concierge data, the authors make a compelling case for their benchmark’s utility as a research tool. The finding that a well-tuned MemNN outperforms a hand-crafted rule-based system on the DSTC2 task (T6) is a powerful demonstration of the potential of end-to-end approaches, even as the overall results highlight their current limitations.

### **Additional Feedback for Improvement**
These points are intended as constructive suggestions for future work and do not detract from the quality of the current submission.

*   **Linguistic Diversity:** The synthetic tasks are generated from a limited set of templates. While this is a necessary simplification for creating a controlled environment, future versions of the benchmark could be enhanced by incorporating paraphrasing, linguistic noise, or more complex sentence structures to better approximate the richness of human language.
*   **Generative Evaluation:** The current evaluation is based on ranking a predefined set of candidate responses. This is a practical and widely used methodology, but it sidesteps the challenges of natural language generation. Extending the benchmark to include a generative evaluation component, perhaps using metrics like BLEU/ROUGE alongside human evaluation, would be a valuable future direction.
*   **Model Architectures:** While Memory Networks were a strong choice at the time of this research, the field has evolved rapidly. It would be highly beneficial for the community if future work could establish baselines on this dataset using more recent architectures, such as Transformer-based models (e.g., BERT, GPT), to see how they handle these structured reasoning tasks.
*   **Handling OOV Entities:** The "match type features" are an effective, explicit mechanism for handling OOV entities by leveraging exact matches against the KB. An interesting research direction would be to explore methods that can learn this matching and generalization capability more implicitly, without requiring this feature engineering step.

### **Questions for the Authors**
I would appreciate clarification on the following points to ensure a complete understanding of the work.

1.  **Rule-Based System Effort:** The paper mentions that the rule-based system for Task 6 consists of 28 rules and 21 patterns. Could you provide a rough estimate of the human effort (e.g., person-hours) required to develop and tune this system? This would provide valuable context for comparing its performance against the automatically learned models.
2.  **Concierge API Call Evaluation:** For the Concierge dataset, the paper states that API calls are predicted "without any argument" (e.g., just "Yelp"). How is the correctness of such a prediction evaluated at test time if the ground truth is simply a label indicating that an external service was used, without specifics?
3.  **Candidate Set Construction:** The candidate set for ranking is created from all bot utterances and API calls across the combined training, validation, and test sets (including OOV). Does this not risk a subtle form of information leakage, where a model might learn to rank a response containing an OOV test entity highly simply because that complete response string is present in the global candidate set? Have you considered an alternative where the candidate set for the OOV test is restricted to avoid exposing OOV entities that are not present in the dialog history or KB?

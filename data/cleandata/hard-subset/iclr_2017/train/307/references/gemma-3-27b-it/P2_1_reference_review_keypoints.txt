- Introduces a novel benchmark of five synthetic goal-oriented dialog tasks (restaurant reservation) plus evaluations on DSTC2 and an internal concierge dataset.
- Motivates benchmark by pointing out limitations of existing datasets: poor reproducibility, hard error analysis, and narrow domains.
- Tasks are designed to isolate core capabilities (API calling, KB querying, entity handling) and include an OOV split to force generalization.
- Evaluates multiple baselines (rule-based, IR, supervised embeddings, Memory Networks) to highlight current end-to-end model challenges.
- Decision: Accept with Minor Revisions.
- Strength: Clear problem statement and well-argued motivation for a controlled, interpretable benchmark.
- Strength: Well-designed decomposition into subtasks enables targeted error analysis and realism via real-world datasets.
- Strength: Thorough experiments with insightful analysis; match-type feature ablation is particularly valuable.
- Strength: Commits to reproducibility by providing data and resources.
- Key finding: End-to-end models can handle some aspects (e.g., API calling) but struggle with KB reasoning and completing full dialogs.
- Suggestion: Provide a more detailed discussion of task complexity (e.g., search space size, number of possible API calls).
- Suggestion: Move more Memory Network architecture and hyperparameter details from the appendix into the main paper.
- Suggestion: Add qualitative error analysis with concrete test-set examples to show where models fail.
- Suggestion: Explicitly discuss limitations of synthetic data (simplified language) and effects on generalizability.
- Question: Why were Memory Networks chosen as the primary end-to-end model and were other architectures (e.g., Transformers) considered?
- Question: How sensitive are results to the specific language patterns used to generate synthetic utterances?
- Question: Were attempts made to train models to generalize to new entities (e.g., meta-learning or transfer learning), and what were the results?
- Question: Please detail the concierge dataset anonymization process and any steps taken to mitigate anonymization-induced biases.

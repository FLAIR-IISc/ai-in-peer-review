- Introduces a synthetic, KB-grounded benchmark of five progressive goal-oriented dialog tasks for restaurant reservation to evaluate end-to-end systems.
- Validates synthetic tasks against DSTC2 and a proprietary concierge dataset to show correlation with real-world performance.
- Benchmarks Memory Networks on the tasks, demonstrating promising but incomplete performance.
- Decision: Accept with minor revisions; fills a significant evaluation gap and offers a valuable, reproducible resource.
- Clear problem definition and motivation for a controlled environment to isolate dialog capabilities.
- Strong technical contribution: well-defined, progressively complex tasks including OOV splits for realism.
- Rigorous evaluation using per-response and per-dialog accuracy and validation on real datasets.
- Reproducibility: data and code publicly released (http://fb.ai/babi).
- Well-written and organized with clear task and experimental descriptions.
- Discuss limitations more deeply, e.g., deterministic question order may not reflect real user behavior.
- Include a more comprehensive ablation study on Memory Network components (hops, embedding size, etc.).
- Provide more detailed error analysis categorizing KB-query, interpretation, and generation errors.
- Report hyperparameter search details (search space, optimization method) to improve reproducibility.
- Expand discussion linking the tasks to challenges in POMDP-based dialog systems.
- How will deterministic task elements (Task 1) affect generalizability and will variability be added?
- How sensitive are results to KB size and complexity (more restaurants, locations, cuisines)?
- Provide details on converting concierge logs to task format and how API calls were inferred.
- Specify the negative sampling strategy used for Supervised Embedding models.
- What future extensions (complex reasoning, long-term planning) are planned for the benchmark?

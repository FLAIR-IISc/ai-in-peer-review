- Proposes a novel, KB-grounded benchmark of five tasks for evaluating goal-oriented end-to-end dialog systems (restaurant reservation).  
- Targets core dialog capabilities: API querying, knowledge interpretation, and dialog management, with synthetic and real-world datasets.  
- Evaluates Memory Networks (MemNNs) against rule-based, IR, and supervised embeddings baselines.  
- Key finding: end-to-end models show promise but fall short of traditional methods on reliable goal completion (per-dialog accuracy).  
- Decision: Accept with minor revisions — benchmark is valuable, experiments thorough, and results informative.  
- Strength: Clear problem definition and strong motivation for a goal-oriented benchmark beyond chit-chat metrics.  
- Strength: Well-designed tasks that enable targeted error analysis and include an OOV test set to probe generalization.  
- Strength: Rigorous experiments with multiple baselines and ablation studies (e.g., match-type features).  
- Strength: Compelling, consistent results across synthetic and real-world datasets.  
- Strength: Reproducibility ensured by public release of the dataset (http://fb.ai/babi).  
- Revision: Provide more detailed hyperparameter tuning procedures and sensitivity analysis.  
- Revision: Include deeper qualitative error analysis with specific failure examples and root causes.  
- Revision: Expand discussion of limitations, including consideration of other architectures (e.g., Transformers).  
- Revision: Give more details about the concierge dataset (service context, user/operator characteristics, anonymization).  
- Questions for authors: Assess task difficulty differences, design and scope of match-type features, scalability to larger KBs, and real-world deployment challenges beyond improving per-dialog accuracy.

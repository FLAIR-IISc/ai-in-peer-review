## Review of "Goal-Oriented Dialog Tasks"

**1. Summary:**

This paper presents a novel benchmark for evaluating end-to-end dialog systems in a goal-oriented setting, specifically restaurant reservation. Recognizing the limitations of existing datasets – lack of reproducibility, difficulty in error analysis, and domain specificity – the authors introduce a set of five synthetic tasks designed to isolate key capabilities required for successful goal-oriented dialog, such as API calling, knowledge base querying, and entity handling. They supplement these tasks with evaluations on two real-world datasets (DSTC2 and an internal concierge service) to demonstrate the generalizability of their findings.  The authors experiment with several baselines, including rule-based systems, information retrieval methods, supervised embeddings, and Memory Networks, highlighting the challenges remaining for end-to-end models.

**2. Decision:**

**Accept with Minor Revisions.** The paper addresses a crucial gap in the evaluation of end-to-end dialog systems. The proposed benchmark is well-motivated, thoughtfully designed, and provides a valuable resource for the community.  The experimental results are insightful, clearly demonstrating the strengths and weaknesses of current methods in a goal-oriented context.

**3. Supporting Arguments:**

* **Clear Problem Statement & Motivation:** The paper explicitly identifies the need for a more controlled and interpretable benchmark for evaluating goal-oriented dialog systems, moving beyond the limitations of existing datasets. The motivation is well-articulated, referencing the difficulties of scaling slot-filling approaches and the challenges faced by end-to-end models.
* **Well-Designed Benchmark:** The decomposition of the goal-oriented task into smaller, isolated subtasks (Tasks 1-5) is a strength. This allows for targeted error analysis and facilitates understanding which specific capabilities models struggle with.  The inclusion of both synthetic and real-world datasets strengthens the validity of the findings. The OOV split is particularly clever, forcing models to generalize beyond memorization.
* **Thorough Experiments:** The authors evaluate a reasonable range of baseline models, demonstrating a clear understanding of existing approaches. The comparisons are well-presented and the analysis of results is insightful. The ablation study with match-type features is particularly valuable.
* **Reproducibility:**  The authors explicitly state the intent to provide an open resource, and provide a link to the data. This commitment to reproducibility is a significant positive.
* **Strong Results & Analysis:** The findings consistently show that while end-to-end models can handle some aspects of goal-oriented dialog (e.g., API calling), they still struggle with tasks requiring reasoning about knowledge base results and completing full dialogs. This analysis is valuable for guiding future research.

**4. Additional Feedback (for improvement):**

* **Task Complexity:** While the decomposition into subtasks is valuable, a more detailed discussion of the inherent complexity of each task (e.g., search space size, number of potential API calls) would be helpful. This could provide a more nuanced understanding of the results.
* **Memory Network Details:** While the appendix provides some implementation details, a slightly more in-depth description of the Memory Network architecture and hyperparameters (e.g., embedding dimensions, hop number) within the main body would improve clarity.
* **Error Analysis:** While the paper identifies areas where models struggle, a more detailed qualitative error analysis with specific examples from the test set would be beneficial. Showing *where* the models fail would be incredibly insightful.
* **Discussion of Limitations:**  A more explicit discussion of the limitations of the synthetic data (e.g., simplified language patterns) and how these might affect the generalizability of the findings would be valuable.



**Questions for the Authors:**

1.  **Regarding the choice of Memory Networks:** Why were Memory Networks specifically selected as the primary end-to-end model?  Were other architectures (e.g., Transformers) considered and, if so, why were they not included in the experiments?
2.  **Regarding the synthetic data generation:** How sensitive are the results to the specific language patterns used to generate user and bot utterances?  Would using more diverse or realistic language patterns significantly alter the performance of the models?
3. **Regarding the OOV split:** Were any attempts made to explicitly train the models to generalize to new entities (e.g., using meta-learning or transfer learning techniques)? If so, what were the results?
4.  **Regarding the Concierge Dataset:** Could you provide a little more detail on the nature of the anonymization process for the concierge data? Were there any steps taken to mitigate potential biases introduced during anonymization?

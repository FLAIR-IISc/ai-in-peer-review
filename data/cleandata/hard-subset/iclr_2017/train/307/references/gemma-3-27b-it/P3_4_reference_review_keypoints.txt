- Introduces a diagnostic dataset of five synthetic restaurant-reservation tasks plus evaluations on DSTC2 and a concierge dataset to analyze goal-oriented dialogue systems.
- Decomposes goal-directed dialogues into smaller, interpretable subtasks to better identify model strengths and weaknesses.
- Explicitly aims to provide an evaluation framework rather than push state-of-the-art performance.
- Decision: Accept with minor revisions.
- Clear motivation: argues convincingly that end-to-end dialogue evaluation needs a controlled, interpretable testbed beyond chit-chat metrics.
- Well-designed testbed: five tasks target API calling, updating calls, displaying options, providing info, and full dialog; includes OOV entities and real-data conversion.
- Rigorous evaluation: uses per-response and per-dialog accuracy and compares rule-based, IR, supervised embedding, and Memory Network baselines, plus ablation studies.
- Significant findings: end-to-end models handle basic tasks but struggle with KB interaction, dialogue coherence, and generalizing to unseen entities.
- Reproducibility: dataset publicly released (http://fb.ai/babi).
- Limitation: paper should discuss how representative and potentially biased the simulated user behavior is.
- Error analysis: recommend expanding beyond aggregate metrics to categorized failure examples (e.g., wrong KB query, state-tracking errors).
- Hyperparameter sensitivity: suggest reporting Memory Network sensitivity to hops, embedding size, etc.
- POMDPs: recommend clearer connections between tasks and POMDP components (state, action, reward).
- Question—User simulation: what steps ensured realistic, unbiased simulated user behavior?
- Question—Scalability: how easily can the testbed be extended to domains beyond restaurants and what changes are needed?
- Question—Future work: plans to add complex phenomena like clarifications or error recovery?
- Question—Match-type feature impact: why did match-type features hurt Task 2 performance; is there a trade-off between handling OOV entities and maintaining dialogue coherence?
- Conclusion: valuable, well-executed diagnostic tool that would be strengthened by addressing the suggested analyses and limitations.

- Introduces a benchmark for goal-oriented dialog (restaurant reservation) with five synthetic tasks testing dialog management, KB querying, and entity handling.
- Core contribution: a controlled, reproducible environment to assess end-to-end models in goal-directed dialogs.
- Experiments use Memory Networks and compare against rule-based and IR baselines.
- Validates synthetic tasks against real-world datasets (DSTC2 and a proprietary concierge service).
- Key finding: models achieve strong per-response accuracy but weak per-dialog accuracy, indicating failure to complete tasks end-to-end.
- OOV generalization tests are included and highlight limitations of current models.
- Decision: Accept with minor revisions.
- Strength: Clear problem definition and motivation, analogous to bAbI tasks for decomposition of goals.
- Strength: Strong empirical validation showing consistent result patterns across synthetic and real data.
- Strength: Open-source dataset and task definitions, improving reproducibility.
- Insight: Analysis points to shortcomings in KB reasoning and maintaining dialog coherence.
- Suggestion: Provide more details on hyperparameter search (search space, algorithm) and integrate key settings into main text.
- Suggestion: Expand systematic error analysis to identify common failure modes (API call errors, entity handling).
- Suggestion: Explicitly discuss limitations of the synthetic benchmark and how it may differ from real user complexity.
- Suggestion: Briefly summarize Memory Network architectural choices in the main text for readability.
- Questions: Impact of increasing dataset scale and whether performance plateaus.
- Questions: How model performance would change with more complex KBs and whether OOV performance would worsen.
- Questions: Are tasks independent or interdependent, and how might that affect evaluation?
- Questions: What specific insights or discrepancies emerged from DSTC2 and Concierge analyses?
- Questions: Have other end-to-end architectures been tried and how do they compare?

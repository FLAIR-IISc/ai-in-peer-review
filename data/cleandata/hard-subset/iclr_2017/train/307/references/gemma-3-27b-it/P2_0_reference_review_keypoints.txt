- Introduces a new benchmark of five tasks and two datasets to evaluate goal-oriented end-to-end dialog systems in the restaurant reservation domain.
- Argues current evaluation methods are insufficient and that breaking goal-oriented dialog into structured subtasks is necessary.
- Demonstrates benchmark usability by applying Memory Networks and comparing to rule-based and IR baselines.
- Shows end-to-end models are promising but still fail to achieve reliable goal completion (low per-dialog accuracy).
- Validates benchmark correlation with real-world data.
- Decision: Accept with minor revisions.
- Clear problem definition: highlights limitations of existing evaluations in isolating skills and reproducibility.
- Well-motivated task design that mirrors dialog sub-components (API calling, KB interpretation), analogous to bAbI tasks.
- Strong empirical evaluation across benchmark and real-world datasets supporting the claims.
- Open-source release promotes reproducibility and further research.
- Corroboration using DSTC2 and a proprietary concierge dataset strengthens relevance.
- Limitation: restaurant domain simplicity—needs discussion on generalization to more complex domains.
- Suggestion: briefly summarize key hyperparameter choices and their impact in the main paper.
- Suggestion: deeper error analysis (e.g., attention visualizations, specific error types) would provide actionable insights.
- Observation: the impact of match type features (handling OOV entities) is large and underemphasized.
- Question: How sensitive are results to training data size—would a learning curve analysis help?
- Question: Have other OOV entity handling techniques been explored beyond match type features?
- Question: Why choose Memory Networks as the primary model; were other architectures considered?
- Question: Are there plans to release the concierge dataset or provide a more detailed description of it?

- Introduces a diagnostic dataset of five tasks for systematically evaluating end-to-end goal-oriented dialogue systems, validated on simulated and real-world data.
- Uses Memory Networks as a baseline to demonstrate strengths and highlight shortcomings, especially low per-dialog success despite decent per-response accuracy.
- Decision: Accept with minor revisions.
- Clear motivation: addresses evaluation challenges and argues for decomposing goal-oriented dialogue into diagnostic sub-tasks.
- Well-designed tasks: progressively harder, include OOV splits, and combine simulated plus real datasets for validity.
- Strong baseline & analysis: Memory Networks with ablation (match-type features) give useful insights into model capabilities and limitations.
- Reproducibility: authors commit to releasing dataset and code, enabling replication and extension.
- High community relevance: fills a need for standardized, controllable evaluation benchmarks in dialogue research.
- Suggest increasing task granularity (e.g., split Task 3 into sorting, filtering, presenting) for finer diagnostics.
- Recommend moving key Memory Network hyperparameter and architecture details into the main text and discussing effects of memory size and hops.
- Ask for expanded qualitative error analysis with concrete examples of API-querying, KB reasoning, and multi-turn failures.
- Recommend adding stronger baselines (e.g., BERT-based slot-filling) for more robust comparisons.
- Questions for authors: scalability of the simulator and ease of scaling to other domains.
- Questions for authors: sensitivity of results to KB structure and experiments with different KB complexities/relations.
- Questions for authors: impact of increasing natural language variation on model performance and the controllability–realism trade-off.
- Questions for authors: request granular statistics to pinpoint whether per-dialog failures stem from API querying, KB reasoning, or dialogue management.

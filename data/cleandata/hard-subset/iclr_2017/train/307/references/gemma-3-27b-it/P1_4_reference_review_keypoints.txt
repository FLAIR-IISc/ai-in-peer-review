- Introduces a new synthetic benchmark of five goal-oriented dialog tasks (restaurant reservation) plus evaluations on DSTC2 and a concierge dataset to test API querying, state tracking, and entity handling.  
- Decision: Reject—encouraged to revise and resubmit because the benchmark is valuable but empirical results and analysis are insufficient.  
- Limited novelty: task combination is useful but individual components (synthetic dialogs, KB interaction) are not novel; main contribution is the task design.  
- Disappointing empirical performance: end-to-end models show low per-dialog accuracy and the paper lacks analysis explaining why.  
- Match-type feature helps but is an incremental improvement.  
- Insufficient baselines: comparisons are incomplete (rule-based only on DSTC2, simplistic IR baselines) and Memory Network experiments are limited.  
- Overly optimistic claim: correlation between synthetic and real-world performance is observed but not convincingly strong enough to replace real-world evaluation.  
- Need deeper error analysis: identify whether failures stem from KB querying, entity disambiguation, dialog state tracking, etc., with concrete failure examples.  
- Include more sophisticated baselines: test modern end-to-end architectures (e.g., Transformers, advanced memory networks) and varied hyperparameters.  
- Perform ablation studies on match-type features to determine which entity types benefit and any negative effects.  
- Expand real-world validation across more diverse datasets to strengthen conclusions.  
- Clarify KB interaction: detail API call representations to the model and limitations of the KB.  
- Discuss generalization and OOV sets: explain construction of OOV tests and strategies for handling unseen entities.  
- Questions for authors: provide example Memory Network errors (esp. tasks 3 & 5), sensitivity to training set size, whether KB limitations affect accuracy, any reinforcement learning attempts, and computational costs of training/evaluation.

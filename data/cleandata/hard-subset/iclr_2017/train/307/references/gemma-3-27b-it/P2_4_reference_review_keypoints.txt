- Proposes a new open-source dataset and five tasks in a restaurant reservation domain to evaluate end-to-end goal-oriented dialog systems.
- Tasks are designed to test KB querying, dialog management, handling OOV entities, and other core dialog capabilities; includes an OOV test set.
- Evaluates on DSTC2 and a proprietary concierge dataset, showing synthetic-task performance correlates with real-world performance.
- Establishes baselines and evaluates rule-based, IR, supervised embeddings, and Memory Networks.
- Recommendation: Accept with minor revisions.
- Strength: Clear problem definition and motivation for a new benchmark.
- Strength: Well-designed tasks that facilitate error analysis and targeted improvements.
- Strength: Strong empirical evaluation across multiple methods.
- Strength: Demonstrated correlation with real data increases benchmark validity.
- Strength: Dataset and code are publicly released, promoting reproducibility.
- Improvement: Provide a more thorough discussion of limitations of the synthetic environment and simplified KB.
- Improvement: Include detailed ablation studies for Memory Network hyperparameters (e.g., hops, embedding size).
- Improvement: Conduct a deeper error analysis with concrete examples of consistent model failures.
- Improvement/Future work: Expand discussion on extending KB complexity and exploring reinforcement learning approaches.
- Question: Why was the restaurant reservation domain chosen and were other domains considered?
- Question: How is reproducibility ensured given stochastic dialog generation?
- Question: Are there plans to expand the KB to include more complex relationships or entities?
- Question: How was DSTC2 converted into the task format, specifically how were API calls and outputs determined from dialog states?

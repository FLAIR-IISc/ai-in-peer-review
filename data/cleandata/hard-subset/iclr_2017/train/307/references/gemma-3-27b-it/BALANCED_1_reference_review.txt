This paper introduces a set of tasks and a corresponding dataset for evaluating goal-oriented dialog systems, aiming to bridge the gap between chit-chat performance and real-world applicability. The core idea is to create a simplified restaurant reservation scenario with well-defined subtasks (API calls, knowledge base querying, option presentation) that isolate key capabilities. The authors also provide baseline results using Memory Networks and compare their performance against rule-based and information retrieval methods, demonstrating the dataset’s utility. Overall, this is a well-motivated and valuable contribution to the field.

**Reject** with a recommendation for revision. While the paper presents a solid idea and a useful resource, the empirical results, while illustrative, are not compelling enough to demonstrate a significant advance or new understanding. The performance gap between the proposed methods and a simple rule-based system on the synthetic tasks is concerning, and the analysis of *why* the Memory Networks struggle is relatively limited. The paper feels more like a dataset release with preliminary experiments than a full research contribution.

The primary strength of this work lies in its attempt to address the lack of standardized evaluation for goal-oriented dialog systems. The bAbI-inspired task decomposition is clever, allowing for targeted analysis of system capabilities. The creation of the dataset, including both synthetic and real-world data (DSTC2 and the concierge dataset), is a valuable contribution to the community. The inclusion of OOV testing is a thoughtful addition, highlighting the challenge of generalization.

However, several weaknesses detract from the overall impact. First, the Memory Network baseline, while reasonable, doesn't achieve state-of-the-art performance on these tasks, and its limitations aren't fully explored.  The authors acknowledge low per-dialog accuracy, but don't deeply investigate *why* the models fail to complete full conversations. The addition of match-type features provides some improvement, but the underlying issues seem deeper. Second, the comparison to a simple rule-based system on the synthetic tasks reveals that the neural networks don't outperform it. This raises questions about the utility of these tasks for *discriminating* between different neural architectures. If a rule-based system can achieve perfect performance, the task may be too easy or not sufficiently representative of the complexities of real-world dialog.  Third, the analysis of the real-world datasets (DSTC2 and the concierge dataset) is brief, and it’s unclear how representative these datasets are of broader goal-oriented dialog scenarios.  The Concierge dataset, in particular, sounds very challenging due to its noise and variability, but the paper doesn't fully acknowledge or address these challenges.

Here are some suggestions for improving the paper:

*   **Deeper Error Analysis:**  Conduct a more detailed error analysis of the Memory Network on each task. What specific types of errors does it make?  Are there patterns in the failures?  Provide examples of incorrect predictions and explain why the model likely made those errors.
*   **Stronger Baselines:**  Consider including more sophisticated baseline models. For example, exploring more recent transformer-based architectures (e.g., BART, T5) fine-tuned for dialog generation could provide a more meaningful comparison.  Investigate if augmenting the Memory Networks with attention mechanisms over the KB directly could improve performance.
*   **Task Complexity & Difficulty:** Re-evaluate the task design. Are the tasks truly challenging enough to differentiate between different learning approaches? Consider increasing the complexity of the tasks, perhaps by introducing more ambiguity, requiring more reasoning steps, or including more diverse user requests.
*   **Dataset Analysis:** Provide a more thorough analysis of the DSTC2 and Concierge datasets.  What are the key characteristics of these datasets? What are their limitations? How do they compare to other available datasets for goal-oriented dialog?
*   **Ablation Studies:** Conduct ablation studies to better understand the contribution of different components of the Memory Network architecture (e.g., the number of hops, the embedding dimension, the use of match-type features).
* **Clarify Evaluation Metric Rationale:** While per-response and per-dialog accuracy are reasonable metrics, discuss their limitations. A low per-dialog accuracy doesn't necessarily mean the model is *completely* failing; it could be making minor errors that derail the conversation.

I have several questions for the authors:

*   Given that a rule-based system achieves perfect performance on the synthetic tasks, what is the goal of training neural networks on these tasks? Is it to achieve comparable performance, or to learn representations that generalize better to unseen scenarios?
*   Could you provide more examples of the types of errors made by the Memory Network on tasks 3 and 5? What specific aspects of these tasks are proving most difficult for the model?
*   What were the primary challenges encountered when converting the DSTC2 data into the proposed task format? Were any simplifying assumptions made that might have affected the results?
*   The paper mentions that the concierge data is noisy. How was this noise addressed during the data processing and evaluation stages?
*   How sensitive are the results to the size of the training set? Would increasing the size of the training set significantly improve the performance of the Memory Network?

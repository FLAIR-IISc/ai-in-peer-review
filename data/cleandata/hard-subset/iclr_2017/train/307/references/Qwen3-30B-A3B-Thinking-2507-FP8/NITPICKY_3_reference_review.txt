

# Review

## Summary
This paper proposes a novel benchmark dataset and set of tasks for evaluating end-to-end goal-oriented dialog systems. The authors introduce five synthetic tasks within a restaurant reservation domain, designed to test specific capabilities like dialog management, KB querying, and entity handling. They also supplement this with two real-world datasets (DSTC2 and a concierge service dataset) to validate the synthetic tasks as a proxy for real-world performance. The authors evaluate several baseline models (rule-based, IR, supervised embeddings, and Memory Networks) on these tasks, demonstrating that while Memory Networks show promise, they still struggle with key aspects of goal-oriented dialog.

## Decision
Reject

## Supporting Arguments
While the paper addresses an important gap in the field by providing a systematic testbed for goal-oriented dialog systems, several significant methodological and presentation flaws undermine the contribution. As a perfectionist reviewer, I must note that the paper fails to meet ICLR's standards for rigorous presentation and reproducibility.

First, there are numerous formatting inconsistencies that suggest poor attention to detail. "Out-Of-Vocabulary" should be "Out-of-Vocabulary" (hyphenation inconsistency), "end-to-end" is inconsistently hyphenated, and "MemNNs" is used without definition. The reference list contains formatting errors like "Bai et al\.,? 2009" instead of "Bai et al., 2009," which is unprofessional for a top-tier venue.

Second, there are critical methodological inconsistencies. The paper states that "party size: six" in Figure 1, but the KB description specifies "party size: 2, 4, 6 or 8 people" (with numerical values). This inconsistency between textual description and example is confusing and undermines reproducibility. Similarly, the paper claims that Memory Networks "can be trained to perform non-trivial operations such as issuing API calls to KBs and manipulating entities unseen in training" but Table 2 shows per-dialog accuracy remains near zero for most tasks, contradicting this claim.

Third, the paper makes significant claims without supporting evidence. For instance, "Memory Networks with match type features give two performance gains..." is stated without specific numerical evidence. The claim that "On the OOV tasks again performance is improved, but this is all due to better performance on known words" lacks supporting data. The paper also states that the synthetic tasks are "a good proxy for what would be observed on real data" but doesn't provide sufficient statistical evidence to support this claim.

Fourth, the paper's experimental design has issues. The authors claim they "use a slightly different training/validation split" for DSTC2 but don't explain how it differs, making it impossible to replicate their experiments. The description of how user and bot utterances are generated from patterns (43 user patterns, 20 bot patterns) lacks detail, making it difficult to assess the quality of their synthetic data.

## Additional Feedback
1. Please provide consistent hyphenation throughout the paper (e.g., "Out-of-Vocabulary" instead of "Out-Of-Vocabulary").
2. Define all abbreviations on first use (e.g., "MemNNs" should be defined as "Memory Networks" with the abbreviation in parentheses).
3. Provide specific numerical evidence to support all claims about performance improvements (e.g., "Memory Networks with match type features give two performance gains..." should include specific accuracy numbers).
4. Clarify the inconsistency between "party size: six" and the numerical representation of party size (2, 4, 6, or 8).
5. Explain exactly how the training/validation split for DSTC2 differs from the original split.
6. Provide details about the 43 user patterns and 20 bot patterns to demonstrate the quality and diversity of the synthetic data.

## Questions for Authors
1. How exactly did you generate the 43 user patterns and 20 bot patterns, and what criteria did you use to ensure they cover the necessary linguistic variations?
2. Can you provide the exact numerical values for the two performance gains mentioned in Section 5 regarding Memory Networks with match type features?
3. Why does the paper claim that Memory Networks can "manipulate entities unseen in training" when the per-dialog accuracy for OOV tasks remains low (as shown in Table 2)?
4. How did you determine that the synthetic tasks are a good proxy for real-world performance, and what statistical tests did you perform to validate this claim?

The paper has a valuable contribution in proposing a benchmark for goal-oriented dialog systems, but the numerous methodological inconsistencies, lack of specific evidence for key claims, and formatting errors significantly undermine its credibility and reproducibility. For ICLR, which prioritizes technical rigor and presentation quality, these issues are substantial enough to warrant rejection.

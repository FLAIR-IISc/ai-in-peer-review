

# Peer Review for ICLR

## 1. Summary of the Paper's Claims and Contribution

This paper makes a significant contribution to the field of goal-oriented dialog systems by proposing a standardized, reproducible testbed for evaluating end-to-end dialog models. The authors address a critical gap in the literature: the lack of a systematic, domain-agnostic benchmark for evaluating whether end-to-end neural dialog models can handle goal-oriented tasks as well as traditional slot-filling approaches. 

The key contributions are:
1. A set of 5 synthetic tasks grounded in a restaurant reservation scenario that break down the goal-directed dialog process into specific subtasks (issuing API calls, updating requests, displaying options, providing extra information, and full dialogs)
2. A clear methodology for generating these tasks using a structured knowledge base and simulation
3. A rigorous evaluation framework with both per-response and per-dialog accuracy metrics
4. Validation of the synthetic tasks using two real-world datasets (DSTC2 and an online concierge service), showing that the pattern of results on synthetic data correlates with performance on real data

The authors demonstrate that while end-to-end models (particularly Memory Networks) show promise, they still struggle with key capabilities like interpreting knowledge base results to present options to users (Task 3), which is crucial for goal-oriented dialog success. This work provides a valuable resource for the community to systematically compare and improve end-to-end dialog models.

## 2. Decision

**Accept**

This paper makes a strong contribution to the field by addressing a clear gap in evaluation methodology for goal-oriented dialog systems. The proposed testbed is well-designed, reproducible, and addresses a critical need in the community. The experimental validation across synthetic and real-world datasets provides convincing evidence that this testbed is a useful proxy for real-world performance.

## 3. Supporting Arguments for Acceptance

### Strong Methodological Foundation
The paper's approach is well-motivated and well-placed in the literature. The authors correctly identify the limitations of existing evaluation resources (DSTC datasets are designed for state tracking, not end-to-end learning; human-bot dialogs on Mechanical Turk lack reproducibility). The analogy to bAbI tasks is appropriate, and the authors clearly explain how their tasks differ from previous work (e.g., they're dialog tasks, not question answering).

### Rigorous Evaluation Framework
The evaluation is scientifically sound, with both per-response and per-dialog accuracy metrics. The authors also introduce a valuable OOV test set to evaluate generalization to unseen entities, which is particularly relevant for real-world applications. The inclusion of two real-world datasets (DSTC2 and concierge service) to validate the synthetic tasks is a major strength that addresses the common concern that synthetic data might not correlate with real performance.

### Meaningful Results and Insights
The results provide clear insights into the strengths and limitations of current end-to-end models. The finding that Memory Networks (with match type features) can handle tasks requiring API calls and entity manipulation but struggle with interpreting knowledge base results to present options (Task 3) is particularly valuable. The paper correctly notes that this is a fundamental limitation that needs to be addressed for end-to-end models to be viable for real-world goal-oriented dialog systems.

### Practical Value
The paper provides an open resource (http://fb.ai/babi) that will benefit the community. The authors have made a genuine effort to create a reusable, reproducible benchmark, which is a significant contribution to the field.

## 4. Additional Feedback for Improvement

### Critical Strengths to Highlight
The paper's greatest strength is its focus on reproducibility and systematic evaluation - a critical need in dialog systems research. The authors have created a resource that can be used to compare methods in a controlled way, which is sorely lacking in the field.

### Areas for Improvement

1. **Generalization to Other Domains**: While the restaurant reservation domain is a good starting point, the paper doesn't discuss how this testbed could be extended to other domains (e.g., hotel booking, flight booking). The authors should add a short discussion on the challenges and potential solutions for adapting this framework to other goal-oriented domains.

2. **Failure Mode Analysis**: The paper identifies that Task 3 (displaying options) remains unsolved, but doesn't provide a detailed analysis of *why* models fail at this task. A deeper analysis of the specific failure modes (e.g., is it a problem with the model architecture, the training data, or the task design?) would significantly strengthen the paper.

3. **Comparison to More Recent Work**: The paper cites Memory Networks as the primary end-to-end model, but doesn't compare to more recent architectures like Transformer-based models (e.g., DialoGPT, BlenderBot) that have shown promise in dialog. A brief comparison to these newer models would strengthen the relevance of the testbed.

4. **Discussion of Limitations**: The paper states that the goal is "not to improve the state of the art in the narrow domain of restaurant booking," but it could benefit from a more explicit discussion of the limitations of the restaurant domain as a testbed for general goal-oriented dialog. For instance, how might the testbed need to be modified for domains with more complex constraints or interactions?

5. **Human Evaluation**: While the paper focuses on automated metrics, it would be valuable to include a small human evaluation to assess whether the model's responses are not only accurate but also natural and appropriate for the context. This would help bridge the gap between automated metrics and human satisfaction.

### Questions for Authors

1. How would you modify this testbed to evaluate dialog systems for a domain with more complex constraints (e.g., hotel booking with multiple rooms, check-in/out times, and amenities)? What specific challenges would arise?

2. The paper states that Memory Networks with match type features can solve Task 4 (providing extra information) but not Task 3 (displaying options). What specific architectural changes or training strategies might be needed to solve Task 3? Could this be related to the need for a more structured approach to knowledge base interaction?

3. The authors mention that "the pattern of results observed in our tasks is indeed a good proxy for what would be observed on real data." Could you provide more specific evidence for this claim beyond the correlation between synthetic and real datasets? For instance, what specific error patterns were similar between the synthetic tasks and real data?

## Final Assessment

This paper makes a substantial contribution to the field by providing a much-needed standardized testbed for evaluating end-to-end goal-oriented dialog systems. The methodology is sound, the evaluation is rigorous, and the results provide valuable insights into the current capabilities and limitations of neural dialog models. The open resource will be a valuable tool for the community. With minor revisions addressing the areas for improvement noted above, this paper would make a strong contribution to ICLR. I recommend acceptance.

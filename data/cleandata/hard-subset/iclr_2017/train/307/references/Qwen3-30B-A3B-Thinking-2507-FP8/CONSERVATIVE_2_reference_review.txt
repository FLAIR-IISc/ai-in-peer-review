

# Review

## Summary
The paper proposes a new dataset and task set for evaluating end-to-end goal-oriented dialog systems, designed to break down the dialog process into specific subtasks within a restaurant reservation domain. The authors introduce five synthetic tasks (issuing API calls, updating API calls, displaying options, providing extra information, and conducting full dialogs) and validate them on two real-world datasets (DSTC2 and a concierge service). They evaluate multiple baseline models (rule-based, IR methods, supervised embeddings, Memory Networks) on these tasks, with the goal of creating a reproducible, lightweight evaluation framework.

## Decision
Reject. The paper does not provide sufficient evidence that its proposed task set represents a meaningful advance over existing evaluation methods for goal-oriented dialog systems.

## Supporting Arguments
The primary issue with this paper is that it fails to establish why its specific task set is superior to existing evaluation approaches. While the authors acknowledge they are not proposing a new state-of-the-art method, their contribution as a dataset alone is insufficient for ICLR, which typically prioritizes methodological advances. The paper makes several claims that are not adequately supported by evidence:

First, the paper claims their task set is a "good proxy" for real-world performance, but the evidence provided is weak. They show that the relative performance of models is similar across their synthetic tasks and real-world datasets (DSTC2 and Concierge), but this is a much weaker claim than the paper implies. The authors don't demonstrate that their synthetic tasks correlate with actual human satisfaction or task completion rates in real-world settings, which would be necessary to establish their value as an evaluation framework.

Second, the synthetic tasks are highly constrained and may not reflect the complexity of real-world dialog. The paper's evaluation shows that even the best models (Memory Networks) have very low per-dialog accuracy (close to 0%), which suggests the problem remains fundamentally challenging. This isn't new information, and the authors don't explain why their task set would help solve this fundamental problem rather than just providing another way to measure it.

Third, the paper doesn't sufficiently justify why their specific set of five tasks is better than other possible task sets. The task definitions seem somewhat arbitrary, and the paper doesn't provide evidence that these particular subtasks capture the most important aspects of goal-oriented dialog. The motivation for the specific task design is not well-argued.

Finally, the paper's evaluation methodology is limited in its scope. The training sets are very small (1,000 examples per task), which creates unrealistic learning conditions. The paper doesn't discuss how these results would scale to larger, more realistic datasets or how the task set would perform with more complex domains.

## Additional Feedback
1. The paper would benefit from a more rigorous comparison with existing evaluation methods for goal-oriented dialog systems, not just a comparison of model performance. The authors should clearly articulate what specific limitations of current evaluation methods their task set addresses.

2. The authors should provide more evidence about how their task set would help advance the field beyond just providing another evaluation benchmark. What specific insights about model behavior would this task set provide that would not be available from existing evaluation methods?

3. The paper should thoroughly discuss the limitations of their synthetic tasks in capturing real-world dialog complexity, rather than presenting them as a perfect proxy.

4. The authors should consider how their task set would scale to more complex domains beyond restaurant reservation, and whether the same five tasks would be sufficient or if additional tasks would be needed.

## Questions for Authors
1. How does your task set specifically address the limitations of existing evaluation methods for goal-oriented dialog, beyond just providing another dataset? What evidence do you have that your task set leads to more reliable or informative evaluations than current methods?

2. The paper claims that Memory Networks can "learn to manipulate entities unseen in training" with the match type features. However, the results show that even with these features, the models still fail on key tasks like displaying options (Task 3) and full dialogs (Task 5). How does this align with the claim that the task set is a good proxy for real-world performance?

3. The paper states that "the pattern of results observed in our tasks is indeed a good proxy for what would be observed on real data." What specific metrics or evidence do you have that this pattern would hold for more complex domains or more challenging real-world scenarios beyond DSTC2 and the concierge dataset?

4. Given that the paper's main contribution is a dataset, how would you argue that this dataset represents a significant advance over existing datasets like DSTC2, which was designed for dialog state tracking rather than end-to-end learning?

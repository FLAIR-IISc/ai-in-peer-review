- Introduces a reproducible benchmark for end-to-end goal-oriented dialog systems in a restaurant reservation domain.  
- Proposes five tasks testing dialog management, KB querying, interpreting results, and handling new entities.  
- Provides synthetic data with controlled properties (including OOV testing) and validation on two real-world datasets (DSTC2, concierge).  
- Demonstrates that synthetic tasks correlate well with real-world performance patterns.  
- Finds end-to-end models can learn to issue API calls but struggle to interpret KB results and present options (notably Task 3).  
- Memory Networks outperform other baselines but still have significant limitations on certain tasks.  
- Benchmark design is scientifically rigorous: distinct tasks, OOV testing, multiple evaluation metrics, real-data validation, baseline comparisons, and ablations.  
- Decision: Accept — paper fills a clear gap with a well-motivated, reproducible, lightweight benchmark.  
- Suggests discussing limitations of synthetic data, especially fixed patterns versus natural language variation.  
- Recommends comparative analysis of divergences and specific failure modes between synthetic and real data.  
- Suggests considering benchmark extensions to other domains with richer KBs and complex flows.  
- Notes need to elaborate how shorter dialogs in the concierge dataset affect model performance.  
- Asks authors about scalability to more complex domains (e.g., multi-constraint hotel booking).  
- Questions how models handled concierge cases without API calls and what this reveals about benchmark limits.  
- Asks how the deterministic question order in Task 1 affects generalization and how to address non-deterministic user behavior.  
- Requests clarification on how much OOV improvements come from match type features versus other model aspects.

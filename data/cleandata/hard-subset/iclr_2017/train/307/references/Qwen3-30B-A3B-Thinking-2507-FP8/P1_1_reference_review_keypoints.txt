- Introduces an open-source, reproducible benchmark for end-to-end goal-oriented dialog systems, focused on restaurant reservation scenarios.  
- Identifies a gap: lack of controlled benchmarks for evaluating goal-oriented end-to-end dialog models.  
- Proposes five synthetic tasks targeting dialog management, KB querying, result interpretation, information retrieval, and combined capabilities.  
- Includes OOV test sets to force handling of new entities, addressing real-world generalization.  
- Supplements synthetic tasks with two real-world datasets (DSTC2 and Concierge) to validate proxy usefulness.  
- Evaluates multiple model families (rule-based, IR, supervised embeddings, Memory Networks) across tasks.  
- Key finding: end-to-end models can learn API-call behavior but struggle to interpret and present KB results (e.g., sorting/displaying options).  
- Benchmark is lightweight, reproducible, and designed for targeted error analysis.  
- Decision: Accept.  
- Primary acceptance reasons: addresses an important gap and provides a well-designed benchmark that correlates with real-world performance.  
- Supporting argument: clear articulation of limitations of prior work and why existing datasets are unsuitable for end-to-end evaluation.  
- Supporting argument: task design systematically isolates capabilities (Tasks 1–5) and uses OOV to test entity handling.  
- Supporting argument: thorough experimental setup with proper splits, hyperparameter tuning, and ablations (e.g., match type features).  
- Validation: relative model performance on synthetic tasks matches performance on real datasets, suggesting good proxy behavior.  
- Suggestion: clarify practical deployment implications of low per-dialog performance (e.g., failure rates).  
- Suggestion: expand discussion on why Task 3 (displaying options) is especially challenging for models.  
- Suggestion: provide a more detailed breakdown of model failures in result tables to show specific error modes.  
- Suggestion: better position and compare dataset to closest prior work (Dodge et al., 2016).  
- Question: clarify which dialog aspects cause near-zero per-dialog accuracy on T3/T4 (sorting, presentation, or user acceptance?).  
- Question: specify which metrics/aspects show consistent relative performance between synthetic and real datasets.  
- Question: give examples of Concierge user requests that fall outside the synthetic task scope.  
- Question: clarify practicality of match type features for real-world deployment (e.g., KB access at inference).

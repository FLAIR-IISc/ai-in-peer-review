- Introduces a reproducible benchmark of five synthetic goal-oriented dialog tasks (plus a sixth real-data task) for restaurant reservation to test dialog management, KB querying, result interpretation, and OOV generalization.  
- Benchmark is inspired by bAbI tasks, carefully structured to enable fine-grained error analysis and end-to-end evaluation.  
- Fills a gap: existing datasets are noisy, domain-specific, or component-focused rather than supporting end-to-end learning.  
- Experimental validation is robust: evaluations on synthetic tasks and two real-world datasets (DSTC2 and a concierge dataset) show consistent performance patterns.  
- Memory Networks with match type features perform best overall but still struggle with interpreting KB results (Task 3).  
- Ablation study on match type features provides useful insights into entity handling for end-to-end models.  
- Evaluation uses appropriate metrics (per-response and per-dialog accuracy) and strong OOV test sets to assess generalization.  
- Decision: Accept — paper makes a strong, well-motivated contribution and offers a useful benchmark for the community.  
- Recommend clarifying how the benchmark could be extended to other domains (e.g., hotels, travel) to show broader impact.  
- Suggest deeper analysis of why match type features slightly worsen performance on Task 2 (updating API calls).  
- Advise quantifying the claimed greater variability of the Concierge dataset (e.g., statistics on request diversity).  
- Recommend discussing limitations of synthetic language patterns in capturing natural language variability.  
- Ask authors to provide examples showing how wider Concierge user requests affect model performance and which request aspects are most challenging.  
- Request explanation for why match type features degrade Task 2 performance and whether this indicates limits of that approach.  
- Ask how the benchmark would transfer to other domains and what adjustments would be needed for different entity types/dialog structures.  
- Request more evidence comparing error patterns between synthetic and real datasets to support the claim that synthetic tasks are a good proxy.  
- Concludes that the benchmark is a valuable, standardized, reproducible resource likely to become a standard in the field (http://fb.ai/babi).

- Introduces a reproducible, lightweight benchmark for end-to-end goal-oriented dialog systems addressing a gap in evaluation resources.
- Proposes five synthetic restaurant-reservation tasks testing specific capabilities: API issuing, API updating, displaying options, providing extra info, and full dialogs.
- Includes an Out-of-Vocabulary (OOV) test set to evaluate generalization to unseen entities.
- Evaluates multiple baselines (rule-based, IR, supervised embeddings, Memory Networks) and validates findings on DSTC2 and a concierge dataset.
- Finds Memory Networks can handle some tasks (e.g., issuing API calls) but struggle with interpreting KB results to display options (Task 3).
- Benchmark is designed to enable clear error analysis by breaking goal-oriented dialog into subtasks.
- Paper is well-motivated and positioned as a "bAbI-like" testbed for goal-oriented dialog with thorough related work.
- Experimental design is scientifically rigorous, including ablation studies and OOV evaluations.
- Validation on real-world datasets suggests the synthetic tasks are a reasonable proxy for real performance.
- Contribution is practical and foundational, enabling focused research on specific dialog capabilities rather than just overall accuracy.
- Recommends acceptance with minor revisions.
- Suggests adding a discussion of benchmark limitations and generalization to domains beyond restaurants.
- Recommends clarifying why the restaurant domain was chosen and its suitability.
- Asks for guidance on how researchers should build on the benchmark and next steps after evaluation.
- Requests more detailed error analysis explaining why Memory Networks with match type features fail on Task 3.
- Questions why the authors focused on evaluation rather than improving state-of-the-art in restaurant booking.
- Asks how the benchmark could be extended to test cross-domain generalization (e.g., to hotel booking).
- Requests statistical significance details to support the claim that synthetic tasks proxy real data.
- Queries why 3–4 hops in Memory Networks work best and whether more hops would help or are theoretically limited.

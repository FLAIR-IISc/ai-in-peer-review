

# Review for ICLR

## 1. Summary of Contribution
This paper makes a valuable contribution by introducing a well-designed, reproducible benchmark for evaluating end-to-end goal-oriented dialog systems. The authors address a critical gap in the literature: existing datasets for goal-oriented dialog are either noisy (designed for state tracking), domain-specific, or not suitable for end-to-end learning. They propose a set of five tasks within a restaurant reservation domain that test crucial capabilities like dialog management, KB querying, interpreting results, and handling new entities. The benchmark includes both synthetic data with controlled properties (including OOV testing) and validation on two real-world datasets (DSTC2 and an online concierge service). The results demonstrate that while end-to-end models can handle some aspects (like issuing API calls), they struggle with interpreting KB results to present options, and the synthetic data correlates well with real-world performance patterns.

## 2. Decision
**Accept**. This paper makes a significant contribution to the field by providing a reproducible benchmark that addresses a clear gap in the literature. The work is well-motivated, the benchmark is carefully designed, and the results provide valuable insights into the current limitations of end-to-end dialog systems.

## 3. Supporting Arguments
**Problem and Motivation (Question 1 & 2):** The paper clearly articulates the problem: end-to-end dialog systems have shown promise in chit-chat settings but lack proper evaluation in goal-oriented contexts. The authors effectively position their work against prior efforts (bAbI tasks, DSTC, end-to-end dialog systems) and demonstrate how their benchmark differs by being designed for end-to-end learning, reproducible, and lightweight. The motivation is strong and well-supported by the literature review.

**Scientific Rigor (Question 3):** The benchmark is scientifically sound. The authors:
- Carefully designed five distinct tasks covering different dialog stages
- Included OOV testing to evaluate generalization to unseen entities
- Used multiple evaluation metrics (per-response and per-dialog accuracy)
- Validated the synthetic data against two real-world datasets
- Compared multiple baselines (rule-based, IR, supervised embeddings, Memory Networks)
- Provided clear ablation studies (e.g., the impact of match type features)

The correlation between synthetic and real-world data (DSTC2 and concierge) is particularly compelling evidence that their benchmark is useful. The paper demonstrates that Memory Networks outperform other baselines, but still struggle with interpreting KB results to present options (Task 3), which aligns with the authors' initial hypothesis about the limitations of end-to-end systems.

## 4. Additional Feedback
To strengthen the paper further:
- Include a more detailed discussion of the limitations of the synthetic data (e.g., how fixed patterns might not capture natural language variation).
- Provide a comparative analysis of where the synthetic and real data diverge (e.g., specific failure modes that appear only in real data).
- Consider discussing how this benchmark could be extended to other domains beyond restaurant reservation.
- The paper mentions that the concierge dataset has "shorter dialogs" but doesn't elaborate on how this affects model performance.

## 5. Questions for Authors
1. How would you expect this benchmark to scale to more complex domains with richer KBs and more intricate dialog flows (e.g., hotel booking with multiple constraints)?
2. In the concierge dataset, there are cases where operators don't use API calls. How did the models handle these cases, and what does this reveal about the limitations of the benchmark?
3. The paper mentions a "deterministic order" of questions in Task 1. How does this affect the model's ability to generalize to non-deterministic user behavior, and how might the benchmark be modified to address this?
4. The authors note that Memory Networks with match type features perform better on OOV, but how much of this improvement is due to the match type features versus other aspects of the model?

This paper makes a valuable contribution to the field by providing a reproducible benchmark that will help advance research in goal-oriented dialog systems. The work is well-executed, well-motivated, and addresses a clear gap in the literature. I recommend acceptance.

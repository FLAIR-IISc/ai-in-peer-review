

# Review for ICLR

## Summary of Contribution
This paper makes a significant contribution by introducing a novel, open-source testbed for evaluating end-to-end goal-oriented dialog systems. The authors recognize a critical gap in the field: the lack of reproducible, lightweight resources to evaluate how well end-to-end models handle goal-directed conversations. They propose a set of five carefully designed tasks within the restaurant reservation domain that systematically break down the goal-oriented conversation into crucial subtasks (dialog management, KB querying, interpreting results, handling new entities, etc.). The authors demonstrate that their synthetic tasks correlate well with real-world performance on DSTC2 and a concierge dataset, making this a valuable resource for the community. The paper's focus on providing a reproducible benchmark rather than improving state-of-the-art performance is particularly refreshing and aligns with the need for better evaluation frameworks in dialog research.

## Decision
Accept. The paper introduces a novel, impactful contribution to the field of dialog systems by providing a reproducible benchmark for evaluating end-to-end goal-oriented dialog systems. The core idea of creating a systematic, task-based testbed inspired by bAbI tasks is bold and addresses a critical gap in the literature.

## Supporting Arguments
The paper excels in novelty and conceptual contribution. While previous work has focused on either traditional slot-filling approaches or end-to-end models for chit-chat, this paper identifies and fills a clear gap: a standardized, reproducible evaluation framework for goal-oriented end-to-end dialog systems. The authors successfully motivate why existing datasets are inadequate for this purpose, citing issues with reproducibility, noise, and lack of structure.

The task design is thoughtful and well-executed. By breaking down the restaurant reservation task into five specific subtasks (issuing API calls, updating API calls, displaying options, providing extra information, and conducting full dialogs), the authors enable fine-grained error analysis that is impossible with existing datasets. The inclusion of Out-of-Vocabulary (OOV) testing is particularly insightful, as it directly addresses a key challenge for end-to-end models.

The validation across synthetic tasks and real datasets (DSTC2 and concierge service) is strong evidence that their framework provides a meaningful proxy for real-world performance. The consistent pattern of relative performance across synthetic and real data is compelling evidence for the framework's utility.

While the paper acknowledges that it's not improving state-of-the-art for restaurant booking, this is precisely the point—it's providing the infrastructure to better evaluate and improve such systems. The paper's contribution is exactly the kind of foundational work that enables future progress, which is what ICLR values highly.

## Additional Feedback
1. The paper would benefit from a more detailed discussion of how the proposed tasks might generalize beyond restaurant reservation. While the authors acknowledge the narrow domain, a brief exploration of potential extensions to other domains would strengthen the argument for broader applicability.

2. The paper could provide more guidance on how researchers might use this testbed for specific research questions. For example, what types of model architectures or training techniques might be most suited to the different tasks?

3. The paper mentions that Memory Networks with match type features improve performance on OOV tasks, but it would be helpful to see a more detailed analysis of what specific errors the model still makes and why. Understanding the precise limitations would help guide future research.

4. While the paper acknowledges the synthetic nature of the data, it might be valuable to include a brief analysis of how the synthetic data differs from human conversations beyond the vocabulary (e.g., in terms of dialogue structure, common errors, etc.).

## Questions for Authors
1. The paper mentions that the synthetic data has a different structure than human conversations (e.g., deterministic bot behavior). How might you modify the synthetic data generation to better capture the variability of human conversations while maintaining reproducibility?

2. The authors note that Memory Networks with match type features improve performance on OOV tasks. Could you elaborate on how the match type features interact with the Memory Network's attention mechanism to enable this improvement?

3. Given that the task design is deterministic (unlike real conversations), how would you expect the results on this testbed to correlate with human evaluation metrics (e.g., task completion rate, user satisfaction) in a real-world deployment?

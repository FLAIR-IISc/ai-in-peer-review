- Paper proposes a standardized, reproducible benchmark of five synthetic tasks in a restaurant reservation domain to evaluate end-to-end goal-oriented dialog systems.  
- The benchmark targets specific capabilities: dialog management, KB querying, interpreting results, and handling new entities (OOV).  
- Authors validate the synthetic tasks against two real-world datasets (DSTC2 and Concierge), showing similar performance patterns.  
- Contribution is a testbed/resource (publicly released at http://fb.ai/babi) to enable rigorous comparisons, not a state-of-the-art model.  
- Recommendation: Accept — the paper fills a clear gap and is well-motivated and rigorous.  
- Strength: Clearly identifies limitations of existing datasets and motivates end-to-end evaluation for goal-oriented dialog.  
- Strength: Thoughtful task design breaking down goal-oriented dialog into specific, testable components, with valuable OOV testing.  
- Strength: Rigorous experimental setup with multiple baselines (rule-based, IR, supervised embeddings, Memory Networks).  
- Strength: Real-world validation demonstrates synthetic tasks are a reasonable proxy for practical performance.  
- Finding: Current end-to-end models, especially Memory Networks, struggle with interpreting KB results and producing correct per-dialog outputs.  
- Observation: Memory Networks achieve low per-dialog accuracy (close to 0) on tasks T3 and T4.  
- Request: Provide details on the rule-based system used for T6 (rule construction, count, prioritization) to clarify its limitations.  
- Request: Include concrete failure examples where models fail on T3 and T4 to illustrate specific weaknesses.  
- Request: Explain match type features in more detail and how they are integrated into models to enable improvements.  
- Request: Discuss limitations of the synthetic tasks and explicitly state which real-world dialog aspects are not captured.  
- Request: Quantify the similarity between synthetic and real-world performance patterns (e.g., correlation coefficients).  
- Question: How exactly do match type features enable Memory Networks to solve T4 and improve OOV handling?  
- Question: Can you provide specific failure cases for Memory Networks on T3 and T4 and explain the failing aspects?  
- Question: Please provide a quantitative measure of similarity between Concierge and synthetic task performance.  
- Question: How does the wider set of user requests in Concierge affect model performance, and which request types are most challenging?

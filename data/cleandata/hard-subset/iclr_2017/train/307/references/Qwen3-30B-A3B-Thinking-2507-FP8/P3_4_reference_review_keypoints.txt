- Introduces a standardized, reproducible benchmark of five synthetic restaurant reservation tasks to evaluate end-to-end goal-oriented dialog systems.  
- Provides two real-world datasets (DSTC2 and Concierge) to validate that synthetic tasks proxy real-world performance.  
- Focuses on OOV generalization and task decomposition to enable targeted error analysis.  
- Decision: Accept with minor revisions.  
- Addresses a gap in existing datasets by offering a controlled, domain-focused benchmark versus ad-hoc or component-specific data.  
- Task decomposition (T1–T5) effectively isolates key capabilities like dialog management, KB querying, and entity handling.  
- Methodology is rigorous: transparent synthetic data generation, well-designed OOV test sets, and inclusion of real data for validation.  
- Results show end-to-end models (Memory Networks with match-type features) can issue API calls but struggle with interpreting KB results and displaying options (T3).  
- The claim that synthetic-to-real transferability holds is plausible but needs more quantitative analysis linking specific synthetic failures to real-world errors.  
- Clarify whether using ranking over candidate responses constitutes true "end-to-end" evaluation or a practical deployment choice.  
- Expand error analysis for T3 to determine if difficulty arises from sorting, KB interpretation, or other factors.  
- Discuss how easily the benchmark and task design generalize to other domains beyond restaurant reservations.  
- Compare baseline results to more recent architectures (e.g., Transformer-based models) to situate the work in current SOTA.  
- Question: Which specific failure modes in synthetic tasks (notably T3) map to errors in real datasets (T6/Concierge)?  
- Question: How would match-type features scale to domains with unknown or more complex KB structures?  
- Question: Why do match-type features help T4 (providing information) but not T3 (displaying options)?  
- Question: Provide context for "per-dialog accuracy remains low" by comparing to traditional slot-filling system performance on the same tasks.

- Introduces a systematic, reproducible testbed for end-to-end goal-oriented dialog systems.  
- Proposes five synthetic restaurant-reservation tasks targeting specific dialog capabilities (API calls, updating requests, displaying options, providing info, full dialog).  
- Designs tasks to isolate capabilities for error analysis and includes OOV testing for generalization assessment.  
- Validates synthetic tasks with two real datasets (DSTC2 and an online Concierge) to show proxy alignment with real-world performance.  
- Evaluates multiple model classes (rule-based, IR, supervised embeddings, Memory Networks) comprehensively.  
- Releases an open dataset and code repository to facilitate reproducibility and future research.  
- Recommendation: Accept.  
- Paper addresses a significant gap in reproducible evaluation for goal-oriented dialog and is well-motivated.  
- Methodology is rigorous with careful task design, appropriate baselines, and proper data splits and metrics.  
- Results show end-to-end models can learn API-calling but struggle with interpreting/presenting knowledge-base information.  
- Synthetic task performance patterns align with real dataset results, supporting the testbed’s proxy validity.  
- Contribution is significant because it provides a standardized benchmark to focus research on weak dialog capabilities.  
- Suggest discussing limitations of synthetic tasks relative to real-world dialog complexity.  
- Recommend expanding discussion and examples of specific model error types beyond Appendix B.  
- Ask authors to address scalability of the testbed to more complex domains beyond restaurant booking.  
- Request more descriptive examples and analysis of the Concierge dataset’s diversity and challenges.  
- Suggest providing concrete directions for future work and architectural changes to address identified limitations.  
- Question: Under what conditions or domains might the synthetic tasks fail to be good proxies for real-world performance?  
- Question: Why was the 2015 MemN2N Memory Network chosen over more modern architectures, and would newer models change results?  
- Question: How does the wider set of user requests in Concierge affect evaluation and which request types are especially challenging?  
- Question: Why do match-type features slightly hurt performance on T2 (updating API calls) but help on other tasks?  
- Question: What specific capabilities must improve for models to achieve high per-dialog accuracy on T3 and T4?



**1. Summary of Contributions**

This paper addresses a critical gap in dialog system research by proposing a standardized, reproducible testbed for evaluating end-to-end goal-oriented dialog systems. The authors recognize that while end-to-end models have shown promise in chit-chat settings, their performance in goal-oriented contexts—where traditional slot-filling systems excel—remains unclear. They design a set of five synthetic tasks within a restaurant reservation domain, grounded in a knowledge base, to systematically evaluate key capabilities like dialog management, KB querying, and result interpretation. Crucially, they validate that these tasks correlate with performance on real-world datasets (DSTC2 and a concierge service), demonstrating their utility as an evaluation proxy. The paper provides a valuable resource for the community, with datasets, clear task definitions, and thorough baselines.

**2. Decision**

I recommend **Acceptance**. The paper addresses a significant gap in the literature with a well-motivated, rigorously designed testbed that has clear utility for the community. The validation across multiple datasets provides strong evidence that the proposed tasks effectively proxy real-world performance.

**3. Supporting Arguments**

The paper's strengths justify acceptance. First, the motivation is excellent: it clearly articulates the disconnect between chit-chat and goal-oriented dialog evaluation, and justifies the need for a standardized testbed. The authors properly contextualize their work within the literature, citing relevant prior work on slot-filling systems, end-to-end dialog models, and evaluation challenges.

Second, the testbed design is thoughtful and comprehensive. The five tasks systematically break down the goal-oriented dialog process, allowing for granular error analysis. The inclusion of OOV test sets is particularly valuable for evaluating generalization capabilities. The validation on real-world datasets (DSTC2 and concierge) is a strong point, showing that the synthetic tasks correlate with real performance—this is a significant contribution.

Third, the experimental setup is rigorous. The authors compare multiple baselines (rule-based, IR methods, supervised embeddings, Memory Networks) using both per-response and per-dialog accuracy metrics. Their results clearly show that end-to-end models can handle some aspects of goal-oriented dialog (e.g., issuing API calls) but struggle with others (e.g., interpreting API results to present options), aligning with their stated goal of identifying where end-to-end systems fall short.

**4. Additional Feedback**

To further strengthen the paper:

- A deeper analysis of *why* end-to-end models struggle with certain tasks (e.g., Task 3: Displaying options) would add significant value. A qualitative error analysis would provide more insight into the limitations of current approaches beyond just reporting accuracy numbers.

- While the restaurant domain is well-chosen for its simplicity, the paper could discuss challenges in extending this framework to more complex domains (e.g., travel booking with multi-step constraints). This would help readers understand the testbed's scalability.

- The comparison should include more recent end-to-end dialog models (e.g., transformer-based architectures) beyond Memory Networks. This would strengthen the claim that the testbed is useful for evaluating state-of-the-art approaches.

**5. Questions for Authors**

- Could you elaborate on the specific challenges that make Task 3 (Displaying options) particularly difficult for end-to-end models? What aspects of the task design (e.g., the need to sort by rating, handle partial user acceptance) contribute most to the performance gap?

- How would you expect the proposed testbed to scale to domains with more complex knowledge bases (e.g., travel booking with flight, hotel, and car rental constraints)? What modifications would be necessary?

- The paper shows that match type features improve OOV performance. Could you clarify how these features interact with the Memory Network's attention mechanism—specifically, how they influence the memory reads during the "hops" process?

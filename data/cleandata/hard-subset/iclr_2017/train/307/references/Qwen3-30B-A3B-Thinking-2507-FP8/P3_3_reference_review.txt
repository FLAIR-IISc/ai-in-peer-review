

# Peer Review: A Set of Goal-Oriented Dialog Tasks

## 1. Summary of Contributions

This paper makes a significant contribution to the field of dialog systems by introducing a well-designed, reproducible benchmark for evaluating end-to-end goal-oriented dialog models. The authors propose a set of five synthetic tasks grounded in restaurant reservation, designed to test specific capabilities critical for goal-directed conversations (dialog management, KB querying, result interpretation, and OOV generalization). The benchmark is carefully structured to enable error analysis by breaking down the goal-oriented dialog into specific subtasks, with an additional sixth task using real-world data (DSTC2 and a concierge dataset) to validate the synthetic tasks as a meaningful proxy for real performance.

The paper addresses a critical gap in the literature: the lack of standardized, reproducible evaluation frameworks for end-to-end goal-oriented dialog systems. The authors convincingly argue that existing datasets are either too noisy, domain-specific, or designed for component-level evaluation rather than end-to-end learning. Their benchmark, inspired by the bAbI tasks, provides a controlled environment for systematic evaluation while maintaining relevance to real-world applications.

## 2. Decision

**Accept**. The paper makes a strong contribution to the field by providing a well-motivated, rigorously designed benchmark that addresses a clear and important gap in dialog system evaluation. The experimental validation across synthetic and real-world datasets provides compelling evidence of the benchmark's utility.

## 3. Supporting Arguments

The paper's contribution is exceptionally well-motivated and positioned within the literature. The related work section thoroughly analyzes existing datasets and their limitations, making a compelling case for the need for a new benchmark. The authors correctly identify that while end-to-end models have shown promise in chit-chat settings, their performance in goal-oriented dialog remains unclear and poorly evaluated.

The experimental design is robust and scientifically rigorous. The authors not only evaluate on their synthetic tasks but also validate findings on two real-world datasets (DSTC2 and a concierge service dataset), demonstrating consistent performance patterns across synthetic and real data. This validation is crucial for establishing the benchmark's relevance to practical applications.

The ablation study on match type features provides valuable insights into handling entities in dialog systems—a critical challenge for end-to-end approaches. The finding that Memory Networks with match type features outperform other methods but still struggle with interpreting KB results (Task 3) is particularly insightful and points to a key direction for future work.

The evaluation metrics (per-response and per-dialog accuracy) are appropriate for goal-oriented dialog, with the latter tracking completion of the actual goal. The careful design of OOV test sets to evaluate generalization to unseen entities is particularly strong.

## 4. Additional Feedback

To further strengthen the paper, I recommend:

1. **Clarify the domain extension**: While the paper correctly states that the goal is not to improve restaurant booking, briefly discussing how this benchmark could be extended to other domains (e.g., hotel booking, travel planning) would strengthen the paper's broader impact.

2. **Deepen the analysis of T2 performance drop**: The paper mentions that Memory Networks with match type features perform slightly worse on Task 2 (updating API calls). A more detailed explanation of why this occurs would be valuable, as it challenges the general assumption that match type features always improve performance.

3. **Quantify real-world complexity**: The paper states the Concierge dataset is "more variable" but doesn't quantify this. Providing statistics on the diversity of user requests or dialog structures would better justify the benchmark's real-world relevance.

4. **Discuss synthetic language limitations**: While the synthetic language patterns (43 user, 20 bot) create thousands of utterances, it would be helpful to discuss potential limitations in capturing natural language variability compared to real user data.

## 5. Questions for Authors

1. The paper mentions that "the set of user requests is much wider" in the Concierge dataset. Could you provide specific examples of how this wider range affects model performance, and which aspects of the requests are most challenging for end-to-end models?

2. You observe that match type features slightly degrade performance on Task 2 (updating API calls). Could you elaborate on why this might occur, and whether this suggests a limitation of the match type feature approach in certain dialog contexts?

3. How would you expect the benchmark to perform when applied to a different domain (e.g., hotel booking) with different entity types and dialog structures? What adjustments might be needed to maintain the benchmark's effectiveness?

4. The paper states that "the pattern of results observed in our tasks is indeed a good proxy for what would be observed on real data." Could you provide more evidence for this claim, perhaps by comparing the error patterns between synthetic and real datasets?

This paper makes a valuable contribution to the field by providing a standardized, reproducible benchmark for evaluating end-to-end goal-oriented dialog systems. The careful design of the tasks, thorough experimental validation, and clear insights into current model limitations make this work highly relevant and useful for future research in dialog systems. The open resource (http://fb.ai/babi) will likely become a standard benchmark in the field.

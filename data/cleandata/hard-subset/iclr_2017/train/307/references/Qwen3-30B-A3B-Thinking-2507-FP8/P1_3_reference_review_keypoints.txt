- Proposes a systematic, reproducible benchmark for end-to-end goal-oriented dialog with five synthetic restaurant-reservation tasks targeting specific capabilities (T1–T5) and two real-world datasets (DSTC2, Concierge).  
- Tasks are grounded in a structured knowledge base and designed to be broken into testable subtasks (API calls, updates, displaying options, extra info, full dialogs).  
- Main contribution is a standardized evaluation framework, not state-of-the-art performance on restaurant booking.  
- Evaluates multiple baselines (rule-based, IR, supervised embeddings, Memory Networks) to enable rigorous comparisons.  
- Finds end-to-end models can handle issuing API calls but struggle with interpreting KB results to display options (T3) and updating information (T2).  
- Memory Networks with match type features improve on T1–T2 but still fail on T3–T4, indicating substantial room for improvement.  
- Synthetic-task results generalize to real-world datasets (DSTC2, Concierge), supporting the benchmark’s validity.  
- Provides an open resource (http://fb.ai/babi) for reproducible future research.  
- Decision: Accept with minor revisions.  
- Strength: Strong motivation—fills a gap in evaluating end-to-end models in goal-oriented settings, analogous to bAbI for QA.  
- Strength: Rigorous experimental design with per-response and per-dialog accuracy and OOV test sets for generalization.  
- Strength: Results give actionable insights into limitations of current end-to-end approaches.  
- Suggestion: Add a more detailed discussion of the synthetic dataset’s limitations and generalizability to other domains.  
- Suggestion: Report computational complexity, inference time, and memory requirements, especially for multi-hop Memory Networks.  
- Suggestion: Strengthen the claim that synthetic-task patterns predict real-data behavior by analyzing which synthetic features best predict real-world performance.  
- Suggestion: Discuss extensions to handle more complex dialog structures (multi-domain, multiple API calls).  
- Question: Explain how match type features work and whether they generalize to other entity types (people, organizations, domain-specific entities).  
- Question: How would the benchmark adapt to ambiguous or open-ended user requests that don’t fit the predefined structure?  
- Question: How to modify the benchmark to better capture the diversity and challenges of the Concierge dataset while preserving reproducibility?

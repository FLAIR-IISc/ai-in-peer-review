- Introduces a new open resource (bAbI dialogs) for systematically evaluating end-to-end goal-oriented dialog systems in the restaurant domain.  
- Proposes five tasks testing dialog management, KB querying/interpretation, and full dialog completion, with OOV tests for generalization.  
- Validates synthetic tasks against real-world datasets (DSTC2, Concierge), showing similar relative performance patterns.  
- Evaluates multiple baselines (rule-based, IR, supervised embeddings, Memory Networks) using per-response and per-dialog accuracy.  
- Shows end-to-end models can track implicit dialog state and handle OOV entities (Tasks 1–2).  
- Finds models struggle with interpreting KB results and presenting options to users (Task 3), hindering goal completion.  
- Memory Networks perform well on some tasks but have notable limitations; match-type features yield gains on Tasks 3 and 4.  
- Resource is publicly released (http://fb.ai/babi) to enable reproducible comparisons.  
- Decision: Accept with minor revisions; contribution is timely and valuable.  
- Recommend more detail on how synthetic natural language patterns were created and validated against real language.  
- Request clarification on Task 3: how the bot obtains restaurant ratings before/after KB queries.  
- Suggest deeper discussion of limitations and concrete model/techniques to address KB interpretation failures.  
- Ask for statistical significance testing to support claims about consistent performance patterns across datasets.  
- Advise comparison to more recent models (e.g., Transformer-based) to better position the work.  
- Specific author questions: clarify Task 3 rating handling, provide examples of Concierge out-of-domain requests, describe utterance pattern selection, supply example dialog failures (esp. Task 3), and explain how match-type features improve Tasks 3–4 and generalize.

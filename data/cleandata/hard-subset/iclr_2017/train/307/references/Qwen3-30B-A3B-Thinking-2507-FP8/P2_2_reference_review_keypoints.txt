- Contribution: Introduces a well-designed, open-source testbed for evaluating end-to-end goal-oriented dialog systems.  
- Gap addressed: Fills the lack of standardized, reproducible evaluation methods for end-to-end goal-oriented dialog models.  
- Task design: Proposes five structured restaurant-reservation tasks testing API calls, updates, option display, extra info, and full dialogs.  
- OOV evaluation: Includes an Out-of-Vocabulary (OOV) test set to assess generalization to unseen entities.  
- Real-data validation: Demonstrates that synthetic-task performance patterns correlate with results on DSTC2 and a concierge dataset.  
- Baselines and findings: Provides thorough baselines (rule-based, IR, embeddings, Memory Networks); Memory Networks perform best.  
- Key limitation found: End-to-end models struggle with interpreting KB results to present options to users (Task 3).  
- Decision: Recommend acceptance based on importance, rigor, and validation.  
- Scientific rigor: Tasks are carefully designed, validated, and shown to be a useful proxy for real-world performance.  
- Reproducibility: Dataset, task definitions, and implementation details are provided and publicly released.  
- Evaluation metrics: Uses appropriate per-response and per-dialog accuracy metrics.  
- Presentation: Paper is well-written and well-organized.  
- Suggested improvements: Add a detailed discussion of testbed limitations and domain aspects not captured by synthetic tasks.  
- Suggested improvements: Discuss how to transfer or extend the testbed to other domains (e.g., hotels, travel).  
- Suggested improvements: Provide a brief implementation/example of using the testbed with a simple ML framework.  
- Suggested improvements: Add a short broader-impact discussion on how the testbed could accelerate research.  
- Questions for authors: Clarify what makes the concierge dataset more challenging and how to incorporate those aspects into the testbed.  
- Questions for authors: Explain how match-type features in Memory Networks handle OOV entities and why they help.  
- Questions for authors: Provide a short example of integrating the testbed into an existing evaluation pipeline.  
- Questions for authors: Describe how researchers developing new end-to-end models should use the testbed and what unique insights it provides.

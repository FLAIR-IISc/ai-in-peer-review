- Introduces a novel, reproducible benchmark and five tasks for evaluating end-to-end goal-oriented dialog systems in a restaurant-reservation domain.  
- Addresses a gap: lack of standardized evaluation for goal-oriented end-to-end models and converts DSTC2 and concierge data to validate the benchmark.  
- Decision: Accept with minor revisions.  
- Strong motivation: clearly contrasts slot-filling limits and unclear end-to-end performance, situating work as a dialog-domain extension of bAbI tasks.  
- Methodology is rigorous: five targeted tasks, controlled synthetic data (including OOV testing), real-data validation, clear metrics (per-response and per-dialog), and multiple baselines.  
- Results support claims: Memory Networks handle API calls and request updates but fail at interpreting API results to display options; match-type features help OOV performance.  
- Benchmark improves reproducibility and offers a standardized framework for comparing future dialog systems.  
- Suggest clarifying practical utility and adoption pathways (e.g., integration into competitions or community use).  
- Recommend expanding discussion of benchmark limitations and generalization beyond restaurant domain.  
- Ask authors to address scalability: how easily the framework extends to other domains.  
- Question 1: Why is Task 3 (displaying options) especially challenging for end-to-end models and what architectural changes might help?  
- Question 2: Provide more details on validating the synthetic-to-real correlation (e.g., statistical tests) and results on datasets beyond DSTC2 and concierge.  
- Question 3: How were wider user requests in the concierge data handled, and are there plans to include broader request types in future benchmarks?

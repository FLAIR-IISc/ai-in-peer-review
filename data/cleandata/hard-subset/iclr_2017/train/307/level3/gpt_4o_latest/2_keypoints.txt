- Decision: Paper accepted for its significant contributions to Dialog/NLP and ML.  
- Main contribution: Critical examination of end-to-end ML for goal-oriented dialog with a comprehensive testbed based on a restaurant reservation scenario.  
- Conjectures: (1) End-to-end ML methods alone are insufficient for robust goal-oriented dialog; (2) performance on synthetic data can predict performance on natural data.  
- Evaluation critique: Existing dialog evaluation metrics are inadequate and the paper proposes more meaningful alternatives.  
- Algorithmic addition: Introduces "match type" to improve Memory Networks, addressing OOV entities and brittle entity matching.  
- Experimental evidence: Experiments show the effectiveness of "match type," particularly on OOV tasks.  
- Baselines and rigor: Uses multiple baselines (rule-based, IR, supervised embeddings, Memory Networks) and tests on both synthetic and real datasets, strengthening claims.  
- Clarity: Well-written, accessible paper with a helpful appendix summarizing Memory Networks.  
- Reproducibility/value: Testbed provides a realistic, reproducible framework for evaluation and comparison.  
- Suggestion — vocabulary mismatch: Identifies mismatch between user vocabulary and KB but does not solve it; recommends exploring representation learning.  
- Suggestion — metrics: Could propose concrete, revenue- or outcome-oriented metrics (e.g., minimize user rejections or dialog length).  
- Suggestion — generalization: Experiments limited to restaurant domain; should be tested on broader goal-oriented domains.  
- Suggestion — protocol sensitivity: Per-dialog accuracy is sensitive to protocols; recommend discussing standardization across datasets for reproducibility.  
- Question — KB complexity: How does "match type" generalize to complex or hierarchical knowledge bases?  
- Question — Task 2 drop: Request more insight into the observed performance drop on Task 2 with "match type."  
- Question — reinforcement learning: Have authors considered RL for dialog management tasks like displaying options (Task 3)?  
- Question — real-world alignment: How do proposed metrics correlate with user satisfaction in real deployments?

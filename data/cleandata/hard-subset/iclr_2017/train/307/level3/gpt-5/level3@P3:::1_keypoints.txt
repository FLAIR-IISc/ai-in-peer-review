- Introduces a public, rule-based synthetic dataset and task suite for end-to-end restaurant-booking dialogues.
- Tasks decompose capabilities: issuing/updating API calls, sorting/displaying options, retrieving entity attributes, plus a full-dialog task.
- Provides complementary, more realistic evaluations by converting DSTC2 and including a Concierge dataset.
- Supplies extensive baselines: TF–IDF, nearest neighbor, supervised embeddings, Memory Networks (with/without match-type features).
- Reports both per-response and per-dialog accuracies.
- Key finding: Memory Networks learn non-trivial behaviors (API calls, some KB reasoning) but struggle on end-to-end full-dialog success.
- Key finding: Match-type features significantly help with OOV entities and attribute retrieval.
- Paper is well written; dataset likely to catalyze progress; cross-dataset evaluation is informative.
- Decision: Accept.
- Primary reason 1: The dataset and task suite are timely, useful, and well-engineered for reproducible, fine-grained evaluation.
- Primary reason 2: Broad, careful empirical study and triangulation across synthetic, DSTC2, and Concierge data validate the testbed’s relevance.
- Contribution: Fills a gap where existing corpora lack end-to-end evaluation or fine-grained task decomposition; ties to bAbI-style tasks appropriate.
- Evidence: Experiments show MemNNs outperform simple IR and embedding baselines; per-dialog metrics reveal remaining gaps (notably Tasks 3 and 5).
- Caveat: Tasks 1–5 are fully deterministic and do not test robustness to ASR noise, ambiguity, user goal uncertainty, or repair.
- Caveat: Neural model superiority on small datasets may reflect inductive biases/regularization rather than inherent suitability.
- Caveat: Missing strong non-neural baselines that use word order/compositional features (e.g., logistic/SVM with uni/bi-grams and match-types).
- Caveat: Claim that prior work lacks well-defined evaluation measures is overstated; relevant prior evaluation exists.
- Suggestion: Add a non-neural benchmark (logistic regression with pre-trained embeddings, bi-grams, match-types; optional CRF/SVM slot tagger pipeline).
- Suggestion: Report learning curves to show sample complexity and data needs.
- Suggestion: Add noisy/ambiguous task variants (ASR errors, paraphrase, conflicting slots, user corrections) to test robustness and repair.
- Suggestion: Expand evaluation to include generation settings and analyze calibration between per-response and per-dialog metrics.
- Suggestion: Clarify candidate set pooling to discuss potential template-knowledge leakage and its impact on difficulty.
- Suggestion: Soften the claim about absence of metrics and cite prior evaluation efforts (e.g., Wen et al., Liu et al., Yu et al.).
- Suggestion: Release simulator, language templates, DSTC2 conversion scripts, document user/bot patterns, and clarify Concierge licensing.
- Questions for authors: How do results change with a strong linear baseline including bi-grams and match-type features?
- Questions for authors: Can you provide learning curves to compare sample efficiency across models?
- Questions for authors: How sensitive are MemNN results to candidate set size and cross-task pooling?
- Questions for authors: Do match-type features leak KB structure, making OOV generalization easier than in real deployments?
- Questions for authors: Can you add noisy/ambiguous variants to assess robustness and dialogue repair?
- Questions for authors: For Task 3, have you tried pointer/copy or sorting-aware architectures to improve performance?
- Questions for authors: In DSTC2 conversion, how often do non-deterministic bot behaviors violate assumptions, and how does that affect errors?
- Questions for authors: Will you release Concierge data or a comparable anonymized subset with clear licensing?
- Reviewer update: Score increased after rebuttal and added analyses; final recommendation remains accept based on dataset contribution and multi-dataset evaluation.

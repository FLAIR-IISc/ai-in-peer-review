- Introduces a public, controlled end-to-end goal-oriented dialogue testbed focused on restaurant booking, decomposed into five structured tasks plus DSTC2 conversion and a real concierge evaluation.
- Evaluates rule-based, IR (TF-IDF, nearest neighbor), supervised embeddings, and end-to-end Memory Networks, and proposes match-type features for entity/OOV handling.
- Empirical results show MemNNs learn non-trivial behaviors (API calls, OOV handling with type features) but struggle on harder skills like sorting and KB-result interpretation.
- The resource is well-motivated, reproducible, and provides built-in subtasks for error analysis, making it a valuable community benchmark.
- Decision: Accept.
- Key reason 1: The dataset and task suite fill a gap in reproducible end-to-end goal-oriented dialogue benchmarking with clear evaluation protocols.
- Key reason 2: The empirical analysis is careful and honest, with OOV and real-data evaluations that substantively support the paper’s claims.
- Paper question: Tests whether end-to-end neural models can replace slot-filling systems in narrow goal-oriented tasks and identifies specific failure modes.
- Placement in literature: Well-motivated and connected to prior slot-filling, POMDP, and end-to-end dialogue work, though the claim about prior evaluation gaps is slightly overstated.
- Results support claims: Controlled tasks and OOV splits allow precise measurement; MemNNs improve with match-type features but fail on T3/T5, with similar patterns on DSTC2/concierge.
- Baselines/fairness concern: Inclusion of diverse baselines is good, but the absence of a strong non-neural linear/log-linear baseline leaves some questions about modeling vs. regularization.
- Baseline suggestion: Add a log-linear model with uni/bi-grams, context–candidate interactions, entity overlap/type, turn index, and simple KB features.
- Retrieval/modeling suggestion: Consider stronger retrieval (BM25+RM3) and simple CNN/RNN encoders to assess the role of sequence modeling.
- Determinism concern: Tasks are deterministic; adding stochastic variants (random prompts, paraphrases, ASR noise, ambiguous goals) would improve ecological validity.
- Evaluation protocol suggestions: Ablate candidate set sensitivity, include generation-based/human-in-the-loop evaluations, and report robustness to candidate pool changes.
- Claim scope: Soften statements that prior work lacks well-defined evaluation and acknowledge complementary efforts.
- Analysis needs: Deeper error analysis for T3/T5 (sorting vs. entity grounding), per-skill breakdowns, and sample-efficiency plots.
- Questions for authors: How sensitive are results to candidate set size/composition and in-domain vs. union candidates?
- Question: Can you report explicit sorting accuracy for T3 and whether models learn true ordering vs. spurious cues?
- Question: How robust are match-type features with larger/noisier or partially unavailable KBs?
- Question: Do relative gains persist under randomized question order and paraphrased prompts?
- Question: Will you add a log-linear non-neural baseline to better calibrate the gap to MemNNs?
- Question: Will you release DSTC2 conversion scripts/splits and comment on non-deterministic system behavior affecting ranking?
- Question: Can you share an anonymized or synthetic proxy of the concierge dataset for independent validation?
- Note: Added bigram and match-type variants for supervised embeddings addressed some baseline concerns and increased confidence, prompting acceptance.

- Introduces an open-source restaurant reservation testbed broken into five subtasks with an OOV split.  
- Testbed isolates key capabilities: issuing API calls, handling updates, and using KB results.  
- Provides an out-of-vocabulary (OOV) test set to evaluate generalization to unseen entities.  
- Thorough empirical evaluation across baselines: TF-IDF, Nearest Neighbor, supervised embeddings, and Memory Networks.  
- Proposes a simple "match type" feature for MemNNs to leverage KB information and better handle OOV entities.  
- Shows current end-to-end models are not yet robust enough for reliable goal-oriented dialog.  
- Finds that synthetic task performance predicts real-world performance (DSTC2 and an online concierge), validating the testbed.  
- Experimental protocol is strong and learning-based methods outperform IR baselines, with MemNNs typically best.  
- Match type feature is effective particularly in OOV settings and tasks requiring grounding in KB facts (e.g., Task 4).  
- Per-response accuracies can be encouraging, but per-dialog accuracies are near-zero on complex tasks, highlighting major challenges.  
- Paper is well-written and includes a helpful appendix with implementation details and examples.  
- Evaluation metrics are brittle: per-response can over-penalize fillers and per-dialog is overly harsh; more nuanced, goal-oriented metrics are needed.  
- The match type feature depends on exact string matches and fails to address semantic/synonym mismatches (vocabulary mismatch).  
- Recommends exploring loss functions tied to business goals (e.g., weighting API-call errors higher than conversational filler).  
- Requests clarification on per-dialog accuracy sensitivity to task definitions and alternative goal-completion metrics.  
- Asks how match types could be extended to handle semantic/paraphrase matches (synonyms, character-level models, embeddings).  
- Asks why ranking-based evaluation was used for the Concierge dataset and how results might change under a strict top-1 metric.

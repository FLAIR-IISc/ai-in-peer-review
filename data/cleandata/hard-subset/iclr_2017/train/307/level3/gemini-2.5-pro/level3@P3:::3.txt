### **Review**

**1. Summary of the paper's claims and contributions**

This paper introduces a valuable new resource for the community: a set of tasks and datasets designed to benchmark end-to-end, goal-oriented dialog systems. The authors create a simulated environment in the restaurant reservation domain, complete with an underlying knowledge base (KB), from which they generate a series of five diagnostic tasks. These tasks are designed to test specific capabilities, such as issuing API calls, handling KB query results, and conducting a full conversation. The dialogs are generated using a rule-based system for both the user and the bot, ensuring a controlled and reproducible testbed. The core task is framed as ranking a candidate response at each turn, with performance measured by both per-response and per-dialog accuracy. The authors provide strong baselines, including classic IR methods and supervised embeddings, and demonstrate that Memory Networks outperform these simpler models. The paper also validates its findings by adapting the real-world DSTC2 dataset to its format and testing on data from a commercial concierge service, showing that the performance patterns observed on the synthetic data carry over to more realistic settings.

**2. Decision and key reasons**

**Decision:** Accept

**Key Reasons:**
1.  The paper addresses a significant gap in the field by providing a publicly available, well-structured, and reproducible benchmark for evaluating end-to-end goal-oriented dialog systems. The bAbI-style breakdown of tasks is particularly useful for diagnostic analysis.
2.  The experimental results and analysis provide valuable insights into the current capabilities and limitations of end-to-end models like Memory Networks in a transactional setting, showing that while they are promising, they still fall short of the reliability needed for deployment.

**3. Supporting arguments for the decision**

The main contribution of this work is the creation and release of a much-needed testbed. Progress in end-to-end dialog has been hampered by a lack of standardized goal-oriented datasets that are large enough for neural models but controlled enough for clear analysis. This work provides an excellent solution. The decomposition into five subtasks is a major strength, as it allows researchers to isolate specific model failings, such as the difficulty in reasoning over KB results (Task 3) versus simply managing dialog state (Tasks 1 and 2). The inclusion of an Out-Of-Vocabulary (OOV) test set is another strong feature that directly tests a model's ability to generalize to new entities, a crucial requirement for real-world scalability.

The empirical evaluation is thorough and provides a clear picture of where current models stand. The authors show that Memory Networks can learn non-trivial behaviors like issuing API calls and tracking state, significantly outperforming bag-of-words baselines. However, the reported low per-dialog accuracies are just as illuminating, serving as a sober reminder of the challenges that remain. This work rightly highlights that a single incorrect turn can derail an entire goal-oriented conversation, a critical insight that per-response metrics can obscure.

Despite these strengths, I have two main reservations. First, I am skeptical about the near-term value of purely end-to-end training for narrow-domain, high-stakes applications like booking. The paper's own results show that while end-to-end models are improving, they are far from perfect, and a carefully hand-engineered system would likely yield much higher user satisfaction and task success rates today. The paper's goal is to gauge the weaknesses of end-to-end systems, which it does effectively, but it also reinforces the view that modular, interpretable systems remain superior for such critical tasks.

Second, the choice to frame the problem as response ranking rather than generation is a significant limitation. While ranking simplifies evaluation, it sidesteps the harder and more realistic challenge of generation. This formulation is critically dependent on a pre-collected set of candidate responses. This dependency limits the model's ability to produce novel utterances and undermines the core promise of end-to-end systems to generalize and scale easily to new domains without extensive manual collection of potential responses.

**4. Additional feedback for improvement**

*   The user simulator's reliance on a fixed set of rule-based NL generation patterns (43 for the user, 20 for the bot) significantly restricts the linguistic diversity of the training data. This may give an overly optimistic view of model performance on language that deviates from these patterns. While this is a reasonable trade-off for reproducibility, publishing the list of generation rules would be highly beneficial for transparency and would allow other researchers to better understand the dataset's constraints.
*   The "match type features" are an effective addition but feel like a form of feature engineering that moves away from a purely end-to-end paradigm. A brief discussion on this tension and the trade-offs involved would strengthen the paper.

**5. Questions for the authors**

*   The response ranking framework requires a comprehensive list of candidate responses. In a real-world scenario where one is building a system for a new domain, where would this list of candidates come from if not from an existing system or extensive manual curation? Doesn't this dependency on a pre-existing corpus limit the scalability advantage that motivates end-to-end systems in the first place?
*   Could you elaborate on the decision to use a ranking paradigm over a generation one? Was this choice motivated primarily by the difficulty of evaluating generated responses, or were there other factors?
*   Would you consider releasing the NL-generation rules used by the simulator? This would greatly help the community understand the linguistic scope of the dataset.

### 1. Summary of the paper's claims/contributions

This paper presents a valuable and timely contribution to the dialog systems community by providing a rigorous testbed for evaluating end-to-end models in goal-oriented settings. The authors correctly identify that while end-to-end systems have shown promise in chit-chat scenarios, their applicability to structured, transactional tasks like restaurant booking remains an open question. The core contribution is a set of five synthetic, bAbI-style tasks that systematically break down the problem into key capabilities: issuing API calls, updating them, displaying options, providing information, and conducting a full dialog.

Crucially, the authors validate their synthetic benchmark by showing that the relative performance of different models (IR, supervised embeddings, Memory Networks) on these tasks is a strong predictor of their performance on two real-world datasets (DSTC2 and a proprietary concierge service). This provides strong evidence that the proposed testbed is a useful proxy for real-world challenges. The paper provides a sobering and necessary reality check, demonstrating that current end-to-end methods, while promising, are not yet robust enough to reliably handle goal-oriented dialogs. The proposed "match type" feature for Memory Networks is a simple but effective technique for handling OOV entities, and its evaluation is thorough. The clarity of the experimental setup and the public release of the data are commendable and will undoubtedly foster reproducible research in this critical area.

### 2. Decision and key reasons

**Decision:** Accept

**Key Reasons:**
1.  The paper introduces a much-needed, well-designed benchmark for a crucial and challenging area—goal-oriented dialog. It provides a clear methodology for analyzing the strengths and weaknesses of end-to-end models, a significant contribution to both the ML and NLP/Dialog communities.
2.  The work rigorously supports its central conjectures: that current end-to-end models are not yet sufficient for reliable goal-oriented tasks, and that performance on well-designed synthetic data can effectively predict relative performance on real-world data. This validation makes the proposed testbed a credible tool for future research.

### 3. Supporting arguments for the decision

The main value of this paper is not a novel algorithm but its methodological rigor and the important questions it raises. It systematically investigates whether the hype around end-to-end dialog models translates to practical, goal-oriented applications. The answer, backed by strong empirical evidence, is "not yet," and the paper provides the tools to measure progress towards that goal.

The breakdown of the full dialog problem into five distinct subtasks is an excellent design choice. It enables fine-grained error analysis, as demonstrated by the varying performance of the Memory Network across Tasks 1-5. For example, the model succeeds at issuing and updating API calls (T1, T2) but struggles with interpreting KB results to display options (T3), pinpointing a specific area for future improvement.

Furthermore, the paper's comparison against multiple strong baselines, including classic IR methods, supervised embeddings, and a rule-based system, is thorough. The finding that Memory Networks outperform the rule-based baseline on the noisy, real-world DSTC2 data (Task 6) is a compelling argument for the potential of learning-based approaches. The consistency of relative model performance across the synthetic tasks, DSTC2, and the Concierge dataset is a powerful result, giving researchers confidence that improvements on the bAbI-style tasks will likely translate to real-world gains. The paper is exceptionally well-written, with clear explanations, informative tables, and a helpful appendix that enhances reproducibility.

### 4. Additional feedback for improvement

These points are intended to help strengthen the paper and are not central to my decision to accept.

**On Evaluation Metrics:** The paper relies on per-response and per-dialog accuracy, which are standard but can be brittle.
*   **Per-response accuracy** can be misleading. It equally penalizes an incorrect "okay, one moment" and a completely wrong API call, even though the latter is a critical failure and the former is benign.
*   **Per-dialog accuracy** is extremely harsh and highly sensitive to the task definition. For instance, the result that the full dialog task (T5) has a higher success rate than the subtask of displaying options (T3) is counterintuitive. As I understand it, this is an artifact of T3 requiring API calls to return at least 3 options, making it statistically harder to complete every turn perfectly compared to T5, which only requires 1 option. This highlights how the metric can be swayed by the experimental protocol rather than just model capability.

I would encourage the authors to discuss these limitations and suggest that future work could explore more nuanced, application-driven metrics. For instance, a weighted loss function that heavily penalizes API call errors, rewards task completion in fewer turns, and lightly penalizes conversational miscues could better reflect real-world utility.

**On Vocabulary Mismatch:** The proposed "match type" feature is a clever way to handle OOV entities but relies on exact string matches between the user utterance and the KB. This doesn't address the common problem of vocabulary mismatch (e.g., user says "cheap," but KB contains "moderate"; user says "near downtown," but KB has specific neighborhood names). This is a key challenge in real-world systems, and it would strengthen the paper to acknowledge this as a limitation of the current approach and a direction for future research.

### 5. Questions for the authors

*   The "match type" feature is effective for OOV entities present in the KB. How do you envision this approach extending to handle synonyms or paraphrases that are not exact KB string matches (e.g., a user asking for an "inexpensive" restaurant when the KB only contains the price-range terms "cheap", "moderate", "expensive")? Have you considered methods to learn these semantic mappings?
*   For the Concierge dataset, evaluation is based on selecting the correct human operator response from a large candidate set. Given the high variability of human language, how appropriate is this exact-match metric? Would it be possible to devise an evaluation that instead measures whether the model's chosen response is semantically correct or moves the dialog towards the goal, even if it's not a verbatim match?

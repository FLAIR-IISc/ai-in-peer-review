- Introduces a benchmark dataset for end-to-end goal-oriented dialog in a simulated restaurant reservation domain with an underlying KB and five diagnostic tasks.  
- Dialogs are generated by rule-based user and bot agents, providing a controlled, reproducible testbed.  
- Core task: rank candidate responses at each turn; evaluation uses per-response and per-dialog accuracy.  
- Provides baselines (IR, supervised embeddings) and shows Memory Networks outperform simpler models.  
- Validates findings by adapting DSTC2 and testing on commercial concierge data, showing patterns generalize to realistic settings.  
- Decision: Accept.  
- Key reason: fills a gap with a public, well-structured, reproducible benchmark; bAbI-style task breakdown aids diagnostic analysis.  
- Key reason: experimental results give useful insights into strengths and weaknesses of end-to-end models like Memory Networks.  
- Strength: decomposition into five subtasks isolates specific model failings (e.g., reasoning over KB results vs. dialog state tracking).  
- Strength: inclusion of an OOV test set tests generalization to new entities, important for real-world scalability.  
- Empirical finding: Memory Networks can learn API calls and state tracking but per-dialog accuracies remain low, revealing fragility.  
- Reservation: skeptical of near-term value of purely end-to-end training for narrow, high-stakes tasks where engineered modular systems outperform.  
- Reservation: framing as response ranking limits realism, depends on a pre-collected candidate set, and sidesteps the generation challenge.  
- Feedback: simulator’s limited NL generation patterns (43 user, 20 bot) restrict linguistic diversity; publishing rules would improve transparency.  
- Feedback: "match type features" act like feature engineering and conflict with a purely end-to-end claim; discuss this trade-off.  
- Question: In new domains, where would a comprehensive candidate response list come from without an existing system or heavy manual curation?  
- Question: Why choose ranking over generation—was this primarily due to evaluation difficulties or other factors?  
- Question: Will the authors release the NL-generation rules used by the simulator?

- Introduces a well-designed testbed of five restaurant-reservation tasks to evaluate end-to-end goal-oriented dialog systems.  
- Provides synthetic data, OOV test sets, and validation on DSTC2 and an online concierge dataset.  
- Shows Memory Networks with "match type" features are promising but still underperform rule-based systems in this domain.  
- Recommendation: accept — the work is a valuable, comprehensive testbed for goal-oriented dialog research.  
- Fills a critical gap by providing a standardized framework for evaluating end-to-end dialog in goal-oriented settings.  
- Empirically supports two key claims: end-to-end ML is currently inadequate for goal-oriented dialogs, and synthetic performance predicts natural-data performance.  
- Experimental protocol is strong: reproducible, includes OOV sets, and thorough ablation studies (e.g., Memory Network hops).  
- Primary contribution is the systematic evaluation framework and insights, with "match type" as the main algorithmic tweak.  
- Suggest expanding evaluation metrics beyond per-response accuracy (e.g., booking success rate or API-call correctness).  
- Request deeper analysis of why Task 3 (displaying options) is particularly challenging compared to other tasks.  
- Note limitation of "match type" relying on exact KB occurrences and recommend addressing vocabulary/semantic mismatches.  
- Propose exploring application-driven loss functions tied to real-world metrics and considering human users or realistic simulators for evaluation.  
- Questions for authors: how to modify metrics for real-world success, why Task 3 is hard, and how to extend the testbed to more complex dialog domains.

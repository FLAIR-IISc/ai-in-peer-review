

# Review for ICLR 2017 Paper: "A Testbed for Goal-Oriented Dialog Systems"

## 1. Summary of Claims and Contributions

This paper makes a vital contribution by providing a systematic testbed to evaluate end-to-end learning approaches for goal-oriented dialog systems. The authors address a critical gap in the literature: while end-to-end neural models have shown promise in chit-chat dialog, their applicability to goal-oriented tasks remains unclear. The paper introduces five well-defined synthetic tasks within a restaurant reservation domain that break down the capabilities required for successful goal-oriented dialog (issuing API calls, updating requests, displaying options, providing information, and conducting full dialogs). Crucially, the authors demonstrate that end-to-end models, specifically Memory Networks with a novel "match type" feature, can learn to perform non-trivial operations in these tasks, though they fall short of traditional rule-based systems in achieving the ultimate goal (per-dialog accuracy). The paper also establishes a strong correlation between performance on the synthetic tasks and real-world datasets (DSTC2 and concierge data), making the testbed a valuable resource for the community. The experimental protocol is exceptionally clear, with thorough ablation studies and insightful error analysis.

## 2. Decision

Accept. The paper makes a significant contribution to the field by providing a valuable reality check on the current state of end-to-end learning for goal-oriented dialog, and its testbed provides a much-needed reproducible benchmark for future work.

## 3. Supporting Arguments

The paper's key contribution is its thorough evaluation of end-to-end approaches in goal-oriented dialog, which directly addresses a critical question in the field: "Can end-to-end models be competitive with traditional methods in well-defined narrow-domain tasks where they excel?" The authors provide compelling evidence that current end-to-end approaches are still insufficient for goal-oriented dialog (conjecture 1), and that performance on well-matched synthetic data predicts performance on natural data (conjecture 2). This is a significant contribution for both the Dialog/NLP and ML communities.

The experimental design is exemplary. The synthetic testbed allows for controlled evaluation of specific capabilities (e.g., handling OOV entities, interpreting API results), and the authors carefully evaluate multiple baseline approaches (rule-based, IR, supervised embeddings, Memory Networks). The paper's analysis of metrics is particularly valuable - it correctly identifies that per-response accuracy is misleading for goal-oriented tasks, as it over-penalizes intermediate responses when only API call correctness matters for the overall goal. The paper's demonstration of how per-dialog accuracy is sensitive to experimental protocol (e.g., the counterintuitive T3 vs T5 results) is a crucial methodological contribution.

The paper's algorithmic contribution (the "match type" feature for Memory Networks) is simple yet effective, and it's the first thorough deployment and test of Memory Networks on goal-oriented dialog tasks. The clear presentation of the model architecture, with helpful appendix and informative tables (like the impact of number of hops), makes the paper highly accessible.

## 4. Additional Feedback for Improvement

While the paper is strong overall, there are a few areas where it could be improved:

1. The paper raises important questions about evaluation metrics but doesn't fully address the implications. It would be valuable to explicitly discuss how the authors' findings might influence future evaluation frameworks in the field.

2. The paper suggests more meaningful application-driven loss functions, but doesn't provide a clear path forward. It would strengthen the paper to sketch a preliminary approach for incorporating such loss functions into the testbed.

3. The vocabulary mismatch issue between user language and KB terms is well-identified but not fully explored. The authors could consider discussing potential extensions to the match type feature to handle semantic associations (e.g., by incorporating word embeddings for KB terms).

4. The paper could provide more guidance on how to adapt the testbed to other domains beyond restaurant reservation, including potential challenges in doing so.

## 5. Questions for Authors

1. Regarding the concierge dataset evaluation: you note that "correct" should mean the single best response rather than being among the 5-best. How would you modify the evaluation protocol to implement this, and how might it change the relative performance of the models?

2. You identify that per-dialog accuracy is highly sensitive to experimental protocol, as illustrated by the T3 vs T5 results. Could you elaborate on how you would design a more robust evaluation protocol that minimizes such protocol-dependent artifacts?

3. The paper suggests that more meaningful loss functions tied to real-world revenue would be valuable. Could you provide a specific example of how such a loss function might be constructed for the restaurant reservation domain, and what additional data would be required to implement it?

4. Regarding the vocabulary mismatch issue: you note that match type features don't learn semantic associations automatically. How might you extend the match type feature to incorporate semantic similarity between entities (e.g., recognizing that "Italian" and "Italian cuisine" are related), and what would be the impact on OOV performance?

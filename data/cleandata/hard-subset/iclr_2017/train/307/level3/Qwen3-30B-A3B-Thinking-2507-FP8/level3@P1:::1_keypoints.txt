- Introduces a new reproducible testbed for evaluating end-to-end goal-oriented dialogue systems.  
- Proposes five synthetic tasks in a restaurant reservation domain that decompose required dialog capabilities.  
- Tasks isolate capabilities (issuing API calls, updating requests, displaying options, providing information, full dialogs) to enable precise error analysis.  
- Provides two real-world datasets (DSTC2 and concierge) to validate correlation between synthetic and real performance.  
- Experiments show Memory Networks can learn API calls and entity manipulation but struggle to interpret KB results for displaying options or providing information.  
- Decision: Accept (Weak Accept).  
- Key reason: Significant, well-structured, reproducible contribution filling a clear gap in the literature.  
- Supporting argument: Synthetic tasks allow precise error analysis that real-world datasets make difficult.  
- Supporting argument: Testbed correlates well with real-world data, validating its usefulness as an evaluation proxy.  
- Supporting argument: Findings offer clear guidance on strengths and limitations of current end-to-end models; data and code are publicly released.  
- Feedback: Claim that “existing work has no well defined measures of performance” is inaccurate—prior work (e.g., Wen et al., Liu et al., Yu et al.) provides evaluation protocols and should be cited/qualified.  
- Feedback: Missing comparison to stronger non-neural baselines that capture word order (e.g., logistic regression with bi-grams and match-type features); TF-IDF baseline is too weak.  
- Feedback: Tasks are fully deterministic, limiting realism; the paper should discuss how this affects generalizability to noisy, ambiguous real-world interactions.  
- Feedback: Clarify conflicting statements about end-to-end models not being competitive versus Memory Networks outperforming rule-based baselines on DSTC2.  
- Question: How do authors justify the claim that “existing work has no well defined measures of performance” given extensive prior evaluation work?  
- Question: What is the expected impact on results if a non-neural baseline with word-order features (e.g., logistic regression with bi-grams) is included?  
- Question: How should researchers adapt this lightweight synthetic testbed to other domains beyond restaurant reservations?  
- Question: Are there limitations to match-type features for OOV/entity handling when scaling to domains with more complex entity types?

- Introduces a valuable synthetic testbed for goal-oriented dialog (restaurant reservation) with five well-defined subtasks.  
- Creates a controlled environment that systematically breaks down complex goal-oriented dialog into manageable tasks.  
- Shows current end-to-end models are promising but fail at interpreting KB query results to present options to users.  
- Demonstrates strong correlation between model performance on the synthetic testbed and real-world data (DSTC and an online concierge), validating the proxy.  
- Provides a significant, standardized evaluation framework beneficial to dialog systems and ML communities.  
- Recommends acceptance due to addressing a major evaluation gap for end-to-end dialog systems.  
- Finds end-to-end approaches can learn implicit dialog states and handle OOV entities but still struggle with critical goal-oriented behaviors.  
- Experimental protocol is excellent: careful task design, clear metrics, meaningful model comparisons, and built-in error analysis.  
- Thorough ablation studies (e.g., match type features, Memory Network hops) yield useful insights into model behavior.  
- Paper is exceptionally clear, with precise task definitions grounded in a realistic KB and helpful appendix.  
- Suggests clarifying evaluation metrics: per-response accuracy can mislead and per-dialog accuracy is sensitive to task definitions; relate metrics to real-world success like booking completion.  
- Notes "match type" features help OOV handling but do not solve vocabulary/semantic mismatch or scalability to more complex domains.  
- Asks two clarifying questions: why match type decreases Task 2 performance, and why not use top-k (e.g., top-5) metrics for concierge evaluation instead of exact match.

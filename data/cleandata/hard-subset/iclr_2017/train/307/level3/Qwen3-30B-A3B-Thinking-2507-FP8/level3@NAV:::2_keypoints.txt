- Presents a compelling testbed for evaluating end-to-end goal-oriented dialog systems focused on restaurant reservation tasks.  
- Introduces five synthetic tasks that decompose required capabilities (API call generation, KB querying, response generation).  
- Demonstrates end-to-end models (especially Memory Networks) show promise but fall far short of rule-based systems at completing full dialog goals.  
- Provides an important reality check on current ML limitations for goal-oriented dialogs relevant to Dialog/NLP and ML communities.  
- Recommends acceptance.  
- Key contribution 1: first thorough deployment and testing of end-to-end models on goal-oriented dialog tasks.  
- Key contribution 2: establishes a valuable, reproducible testbed and experimental protocol.  
- Shows that relative performance on synthetic data predicts performance on natural data, supporting the paper’s main conjecture.  
- Experimental design is exemplary with clear task definitions, a helpful appendix, and informative tables.  
- Highlights that per-response accuracy is misleading because it over-penalizes intermediate responses when API call correctness determines goal completion.  
- Critiques the handling of vocabulary mismatch between user language and KB terms; the “match type” feature needs exact KB occurrences rather than learned semantic associations.  
- Recommendations for improvement: explicitly define concierge evaluation criterion (single best vs. top-5), explore application-driven loss functions tied to outcomes, and discuss approaches to handle vocabulary mismatches beyond “match type.”

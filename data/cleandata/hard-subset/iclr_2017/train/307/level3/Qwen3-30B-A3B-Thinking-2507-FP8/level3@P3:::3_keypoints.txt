- Introduces a new simulated restaurant-domain dataset/testbed with five tasks to evaluate end-to-end goal-oriented dialog systems.  
- Dataset grounded in a KB of restaurant properties and generated using rule-based NLG for controlled evaluation.  
- Tasks isolate key dialog capabilities: issuing/updating API calls, showing options, providing extra info, and full dialogs.  
- Evaluates multiple models (rule-based, IR, supervised embeddings, Memory Networks) using per-response and per-dialog accuracy.  
- Main finding: Memory Networks outperform bag-of-words baselines and can learn non-trivial KB manipulations and API call behavior.  
- Provides a reproducible testbed intended to fill a gap in systematic evaluation of end-to-end goal-oriented dialog systems.  
- Decision: Accept — testbed is valuable and experiments give meaningful insights into current approaches’ capabilities and limits.  
- Strong motivation: identifies limitations of existing datasets and the need for systematic evaluation beyond slot-filling or chit-chat work.  
- Methodology is rigorous: five tasks enable detailed error analysis and controlled experiments (including OOV settings).  
- Use of both synthetic and real-world data (DSTC2, concierge) strengthens validity; similar performance trends observed across datasets.  
- OOV experiments highlight current models’ limitations in handling new entities, important for real-world deployment.  
- Concern: Skepticism about the practical value of end-to-end training in narrow domains; authors should discuss maintenance/scalability benefits vs hand-crafted systems.  
- Limitation: Ranking-based evaluation (selecting from predefined candidates) may not reflect generation-based settings; justification and alternatives needed.  
- Request to publish rule-based NL-generation rules as supplementary material to ensure full reproducibility.  
- Practical scalability question: clarify how candidate response lists would be obtained in new domains and whether the testbed can adapt to generation-based evaluation.

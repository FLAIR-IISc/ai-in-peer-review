- Introduces a novel testbed for end-to-end goal-oriented dialog in a restaurant reservation domain with a structured KB.  
- Defines five tasks that decompose dialog into subtasks requiring manipulation of natural language and KB symbols (API calls, request updates, showing options, providing info, full dialogs).  
- Main contribution is a synthetic dataset enabling reproducible evaluation with per-response and per-dialog accuracy metrics.  
- Experiments show Memory Networks outperform classical IR and supervised embedding baselines, especially when augmented with match type features for entities.  
- Recommendation: accept — addresses a gap where prior work focused on chit-chat rather than goal-oriented dialogs.  
- Clearly positions the work relative to existing datasets, arguing theirs is less noisy, more suited for end-to-end learning, and systematically designed.  
- Rigorous experiments across simulated tasks, DSTC2, and a real concierge dataset show consistent performance patterns and some real-world generalization.  
- Ablation studies (match type features, OOV performance) give useful insights into model capabilities and limitations.  
- Methodology is scientifically rigorous with careful data splits, evaluation metrics, and baselines.  
- Primary concern: ranking-based evaluation relies on pre-collected candidate responses, limiting practicality and undermining claim of avoiding domain-specific handcrafting.  
- Suggestion: paper would be stronger if it demonstrated generation capabilities or a clear path to obtain candidate responses in new domains.  
- Concern: rule-based NL generation for synthetic data limits linguistic variability versus human dialogues; request full list of NL generation patterns for reproducibility and bias assessment.  
- Skepticism remains about the practical value of end-to-end approaches in narrow domains where hand-crafted systems perform well; authors should discuss when end-to-end is preferable.  
- Recommended improvements: clarify how to obtain candidate responses in new domains, publish NL generation rules, and add a nuanced trade-off discussion between end-to-end and hand-crafted systems.  
- Two questions for authors: how to obtain candidate response lists without handcrafting, and how models trained on rule-based NL will generalize to diverse human language?



**Review**

This paper presents a compelling testbed for evaluating end-to-end dialog systems in goal-oriented applications, specifically for restaurant reservation tasks. The authors introduce five well-designed synthetic tasks that systematically break down the capabilities required for effective goal-directed dialog, including API call generation, knowledge base querying, and response generation. They demonstrate that while end-to-end models (particularly Memory Networks) can achieve promising performance on these tasks, they remain significantly inferior to rule-based systems in completing full dialog goals. The paper makes a valuable reality check about current end-to-end ML limitations for goal-oriented dialogs, which is highly significant for both Dialog/NLP and ML communities.

I recommend acceptance. The paper makes two key contributions: (1) it provides the first thorough deployment and testing of end-to-end models on goal-oriented dialog tasks, and (2) it establishes a valuable testbed with excellent experimental protocol that enables reproducible evaluation. The authors convincingly demonstrate that relative performance on synthetic data predicts performance on natural data, supporting their key conjecture about end-to-end ML's current inadequacy for goal-oriented dialogs.

The paper's experimental design is exemplary, with clear task definitions, helpful appendix, and informative tables. The authors effectively show that standard metrics like per-response accuracy are misleading, as they over-penalize intermediate responses when only API call correctness matters for goal completion. However, the paper could better address the vocabulary mismatch issue between user language and KB terms, as the "match type" feature requires exact KB occurrences rather than learning semantic associations.

To improve the paper, I recommend: (1) explicitly defining the evaluation criterion for concierge data (single best response vs. among 5-best), (2) exploring more application-driven loss functions tied to real-world outcomes, and (3) discussing how to handle vocabulary mismatches beyond the current "match type" approach. These refinements would strengthen the paper's contribution to the field.

- Proposes a testbed of five restaurant-reservation tasks to evaluate end-to-end goal-oriented dialog systems.  
- Argues current chatbots and end-to-end ML models are insufficient for goal-oriented dialogs and need thorough evaluation.  
- Introduces a simple "match type" addition to memory networks and provides the first thorough deployment on goal-oriented tasks.  
- Shows the model can learn non-trivial operations like issuing API calls and handling entities unseen in training.  
- Experimental protocol and presentation are excellent, with a helpful appendix and informative tables.  
- Decision: accept—paper gives a valuable reality check and a well-motivated, well-designed testbed for future research.  
- Limitation: the "match type" requires exact KB occurrences and does not address vocabulary mismatch between user language and KB terms.  
- Suggests current metrics (per-response and per-dialog accuracy) may be insufficient; recommends application-driven loss functions (e.g., weighted API errors, dialog turns, user rejections).  
- Requests from authors: plans for vocabulary-mismatch handling, implementation details of "match type," extensibility to other goal-oriented domains, and deeper analysis of OOV test-set results.

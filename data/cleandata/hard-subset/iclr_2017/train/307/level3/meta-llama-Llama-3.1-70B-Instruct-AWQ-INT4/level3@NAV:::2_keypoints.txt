- Proposes a testbed for evaluating end-to-end dialog systems in goal-oriented applications.
- Introduces five restaurant-reservation tasks requiring sentence/symbol manipulation, API calls, and use of call outputs.
- Compares rule-based systems, classical IR models, supervised embeddings, and end-to-end Memory Networks.
- Finds Memory Networks outperform other baselines but still struggle with goal-oriented dialog.
- Decision: Accept.
- Reason 1: Provides a valuable reality check on limitations and challenges of end-to-end dialog systems.
- Reason 2: Offers a thorough, well-designed testbed likely to facilitate future research.
- Contributions are significant to Dialog/NLP and ML communities and supported by experiments.
- Experimental protocol and clarity are excellent and easy to reproduce.
- Suggestion: Address vocabulary mismatch between user language and KB terms.
- Suggestion: Explore application-driven loss functions tied to real-world metrics (e.g., revenue), possibly requiring human users or realistic simulators.
- Suggestion: Provide more analysis, e.g., impact of number of hops on Memory Network performance.
- Question: How will you address vocabulary mismatch in future work?
- Question: Please provide implementation details for match type features and their use in Memory Networks.
- Question: How would results change with a different evaluation metric (weighted API errors, dialog turns, user rejections)?

- Introduces a new restaurant-reservation testbed for evaluating end-to-end goal-oriented dialog systems.  
- Motivates the work by questioning whether chit-chat successes transfer to complex goal-driven interactions and highlighting scalability limits of hand-crafted systems.  
- Defines five KB-grounded tasks to assess capabilities like API querying and entity manipulation.  
- Shows a Memory Network can achieve promising but imperfect performance on these tasks.  
- Memory Network outperforms a rule-based slot-filling baseline on related datasets.  
- Decision: Accept.  
- Key contribution: a well-defined, reproducible evaluation environment enabling granular analysis of goal-oriented dialog performance.  
- Task breakdown (API calls, updates, displaying options, providing info) isolates core dialog components for targeted evaluation.  
- Comparison to slot-filling highlights scalability issues of hand-crafted systems and suggests end-to-end potential.  
- Validation on DSTC2 and a proprietary concierge dataset supports generalizability.  
- OOV experiments reveal challenges with unseen entities; Memory Networks show some promise but need improvement.  
- Concern: ranking-based evaluation limits generalization; generation-based evaluation should be explored.  
- Concern: simulator uses fixed NLG patterns, which may bias results; publish patterns or examples.  
- Request more detail on hyperparameter tuning and sensitivity for reproducibility.  
- Request deeper error analysis with failed-dialog examples and error categorization.  
- Questions for authors: why choose Memory Networks over other architectures, share simulator patterns, methods for handling dynamic/OOV entities, confidence that ranking reflects response quality, and computational cost of training/evaluation.

- Investigates end-to-end dialog systems for goal-oriented tasks (restaurant reservation), noting chit-chat success doesn't imply task success.
- Introduces a five-task testbed isolating API calling, KB interaction, and dialog management, with both in-vocabulary and out-of-vocabulary sets.
- Memory Networks augmented with “match type” features achieve encouraging but imperfect results, showing promise yet significant remaining challenges.
- Validates synthetic findings on DSTC2 and a proprietary concierge dataset, demonstrating consistent performance patterns.
- Paper's aim is diagnostic analysis of strengths/weaknesses rather than achieving state-of-the-art booking performance.
- Recommendation: Accept with minor revisions.
- Paper is well-motivated, clearly articulating the gap between slot-filling systems and end-to-end learning for goal tasks.
- Testbed is a strong contribution enabling more granular and controlled evaluation of dialog capabilities.
- Experiments are thorough, comparing against strong baselines (rule-based, IR, supervised embeddings) and including ablation studies.
- Ablation shows match-type features effectively incorporate KB signals and help handle OOV entities.
- Detailed hyperparameter reporting and appendix material improve reproducibility.
- Reporting both per-response and per-dialog accuracies gives a more nuanced performance picture than a single metric.
- Suggest expanding discussion on why per-dialog accuracy is highly sensitive to task definitions and metric strictness.
- Recommend evaluating the concierge dataset using top-5 responses (or similar relaxed metrics) to capture practical correctness.
- KB vocabulary mismatch deserves more exploration; propose investigating semantic association methods beyond match-type features.
- Encourage discussion of application-driven loss functions (e.g., weighted API errors, dialog turns, rejections) and their trade-offs.
- Request more concrete error analysis with example failure modes (misinterpreted intent, incorrect API calls).
- Questions for authors: did you evaluate concierge results using top-5 responses, and if so, how did outcomes change?
- Questions for authors: how do match-type features handle synonyms or related KB terms in practice?
- Questions for authors: what are the computational costs and any observed bottlenecks from adding match-type features?
- Questions for authors: any preliminary ideas on designing application-driven loss functions aligned with real-world goals?

- Introduces a new dataset and tasks for end-to-end goal-oriented dialogue in restaurant reservation settings.  
- Motivates work by noting limitations of hand-crafted systems and uncertain performance of end-to-end models in goal-directed tasks.  
- Tasks evaluate API calling, knowledge base querying, and dialogue state management.  
- Provides both in-vocabulary and out-of-vocabulary test sets to assess generalization.  
- Presents experiments using Memory Networks and compares them to a rule-based baseline and simple baselines.  
- Includes results on existing datasets (DSTC2 and a concierge dataset) to validate findings.  
- Finds end-to-end systems struggle to achieve full dialogue success (low per-dialog accuracy).  
- Contribution is valuable for facilitating research in goal-oriented dialogue despite experimental weaknesses.  
- Recommendation: Reject with strong encouragement to resubmit after addressing key concerns.  
- Primary weakness: lack of sufficiently competitive, non-neural baselines for core synthetic tasks (T1–T5).  
- TF-IDF and supervised embeddings comparisons are underwhelming and likely miss word-order/ngram information.  
- Without stronger baselines, Memory Network performance conclusions are inconclusive.  
- Claim that existing work lacks performance measures is inaccurate and overlooks relevant prior work.  
- Deterministic nature of tasks 1–5 is a limitation and does not capture real-world ambiguity, noise, or dialogue repair.  
- Lack of evaluation under noisy or ambiguous interactions limits assessment of real-world generalization.  
- Suggested improvements: add stronger baselines incorporating word order (e.g., logistic regression with bigrams and match-type features).  
- Suggested improvements: introduce task variations with ambiguity and correction scenarios.  
- Suggested improvements: provide detailed error analysis of Memory Network failures and discuss hyperparameter sensitivity.  
- Suggested improvements: clarify the impact of match-type features on OOV performance and remaining limitations.  
- Questions for authors: why not include n-gram/non-neural baselines; plans to add realistic/noisy interactions; specifics of Memory Network errors; hyperparameter sensitivity; detailed effect of match-type features on OOV improvements.

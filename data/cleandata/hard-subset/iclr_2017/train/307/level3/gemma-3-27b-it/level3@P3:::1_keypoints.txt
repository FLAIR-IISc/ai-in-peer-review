- Introduces a new public, rule-based synthetic dataset and tasks for goal-oriented dialogues in a restaurant reservation scenario.  
- Focuses evaluation on API calls, option display, and full dialogue management.  
- Motivates work by scalability limits of handcrafted systems and uncertainty about neural end-to-end models transferring to goal-oriented tasks.  
- Provides a valuable benchmark and comparative analysis using Memory Networks and several baselines.  
- Recommendation: Accept with reservations.  
- Strength: Controlled, reproducible testbed where synthetic data allows precise complexity control and isolation of specific dialogue skills.  
- Strength: Breaking the task into Tasks 1–5 aids error analysis and pinpointing model weaknesses.  
- Strength: Inclusion of real-world DSTC2 and concierge data supports generalizability.  
- Strength: Uses per-response and per-dialog accuracy, appropriate evaluation metrics.  
- Strength: Memory Network comparison to TF-IDF and supervised embeddings is a reasonable starting point; performance patterns are consistent across synthetic and real data.  
- Major weakness: Lack of a strong non-neural baseline (e.g., logistic regression with word-order features) undermines interpretation of neural model gains.  
- Weakness: Tasks 1–5 are deterministic and do not assess handling of noise, ambiguity, user goal inference, or dialogue repair.  
- Weakness: Claim that prior work has no well-defined performance measures is inaccurate.  
- Suggestion: Add a logistic regression baseline with bigrams/trigrams and position features to provide a fairer comparison.  
- Suggestion: Introduce task variations that add uncertainty or underspecified requests to better simulate real-world dialogue.  
- Suggestion: Provide a more detailed error analysis of Memory Network failure modes.  
- Suggestion: Evaluate sensitivity to knowledge-base size and complexity.  
- Suggestion: Report ablation results for the seven match-type features.

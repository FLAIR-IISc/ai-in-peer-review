- Introduces a new dataset and tasks for end-to-end goal-oriented dialog in the restaurant reservation domain.  
- Motivates the work by highlighting limits of hand-crafted slot-filling systems and need to evaluate end-to-end approaches.  
- Presents a simulated environment with an underlying knowledge base and uses per-response and per-dialog accuracy metrics for evaluation.  
- Core contribution is a testbed to break down strengths and weaknesses of end-to-end dialog systems, demonstrated with Memory Networks.  
- Compares Memory Networks to a rule-based baseline and simpler methods to provide meaningful benchmarks.  
- Decision: Accept — the paper makes a valuable contribution to dialog system evaluation methodology.  
- The dataset and tasks are well-defined and reproducible, helping address data scarcity for dialog research.  
- Experimental results are not state-of-the-art but offer insightful comparisons; Memory Networks can manipulate symbolic info but struggle to complete full dialogs.  
- Extensions to DSTC2 and an online concierge dataset strengthen claims that synthetic data can be a useful proxy for real-world scenarios.  
- Out-of-vocabulary (OOV) tests reveal current approaches’ limitations in generalizing to unseen entities.  
- Ranking-task formulation is a limitation for real-world deployment; adapting the framework to generation warrants exploration.  
- The simulator’s rule-based natural language generator may introduce bias; a more stochastic NLG and publishing the NLG rules would increase transparency.  
- Paper could benefit from more detailed error analysis (e.g., incorrect API calls, KB misinterpretation) to guide future model improvements.  
- Questions posed to authors: strategies for adapting ranking to generation; provide full NLG rules; most frequent Memory Network errors in tasks 3 and 5; alternatives to match-type features for OOV (e.g., subword/char models); insights on transferability between simulated and real datasets.

1. Summary and claimed contributions
The paper introduces an open, reproducible testbed for evaluating end-to-end goal-oriented dialog systems, centered on restaurant reservation. The authors decompose the problem into five synthetic tasks (API issuance, updates, option display with sorting, KB-grounded Q/A, and full dialogs), plus a DSTC2-based conversion (Task 6) and a real concierge dataset. They claim: (a) the testbed enables fine-grained diagnosis of capabilities (state tracking, KB usage, OOV handling); (b) Memory Networks with match-type features achieve promising but imperfect performance, particularly on API-related subtasks and OOV entities; (c) performance patterns on synthetic tasks translate to real data, suggesting the testbed is a useful proxy.

2. Decision: Accept
Reason: The paper provides a well-motivated, much-needed, and carefully designed benchmark with clear task decomposition and comprehensive baselines. The empirical analysis is informative, reproducible, and highlights concrete gaps in current end-to-end methods.

3. Supporting arguments
- Problem and motivation: Clearly articulated gap—chit-chat success does not transfer to goal-directed dialog; existing datasets are unsuitable for end-to-end evaluation. The breakdown into capabilities is thoughtful and actionable.
- Support for claims: Extensive comparisons (IR, supervised embeddings, MemN2N±type features) across synthetic, DSTC2-derived, and concierge data. Results consistently show MemN2N advantages, the necessity of type features for entities/OOVs, and difficulty with option ranking and full dialogs—aligning with claims.
- Usefulness and novelty: The dataset design (ranking setup, OOV split, API grounding, sorting) and match-type features for entity handling are practical and impactful. Public release supports community progress and fair comparison.
- Completeness and rigor: Hyperparameters and appendices aid reproducibility. While statistical significance tests are not reported, cross-dataset consistency lends credibility. Limitations are acknowledged.

4. Additional feedback (to help improve; not all are decision-critical)
- Consider stronger baselines with copy/pointer mechanisms, span selection, and constrained decoding; compare to seq2seq/transformers with copy and KB grounding.
- Report statistical significance and confidence intervals; add ablations on hops, memory size, candidate set size, and training set size to quantify sample efficiency.
- Reduce determinism in synthetic tasks (e.g., randomize question order, linguistic variability) to better reflect real-world variance.
- Complement ranking with generation-based evaluation and human judgments for Concierge; report goal success vs. turns/trade-offs.
- Clarify DSTC2 conversion (e.g., added ratings) and its potential biases.

Questions for authors
- What is the average candidate set size at test time across tasks, and how does performance scale with it?
- In the OOV setting, are all OOV surface forms guaranteed to appear in the KB at inference? How would the approach fare if some entities are unseen in both dialogs and KB?
- How sensitive are results to KB size/option cardinality and to training set size (learning curves)?
- On Task 3, did you try explicit supervision for ranking (e.g., pairwise/listwise losses) within MemN2N?
- For DSTC2, how often does the original bot behavior deviate from the deterministic policy, and does that drive most errors?

Summary and contributions
The paper introduces an open, reproducible testbed for evaluating end-to-end goal-oriented dialog systems in a controlled restaurant-reservation domain. The testbed decomposes the problem into five synthetic tasks that cover key sub-skills (dialog management, issuing and updating API calls, sorting/displaying options based on KB facts, and retrieving specific attributes), plus a sixth task derived from real DSTC2 dialogs, and an additional evaluation on a proprietary concierge dataset. The work provides a careful comparison of several baselines—TF-IDF, nearest neighbor retrieval, supervised embeddings, rule-based, and end-to-end Memory Networks (MemN2N)—and augments MemN2N with simple but effective match-type features to improve handling of entities and OOV terms. Results show that MemN2N achieves promising per-response accuracy, learns nontrivial behaviors such as API call construction and KB result usage, and outperforms a hand-crafted slot-filling system on DSTC2-based data. The paper emphasizes per-dialog accuracy to reflect task completion and highlights remaining gaps on tasks requiring multi-step reasoning over KB facts (e.g., sorted option presentation).

Decision: Accept
Key reasons: (1) The testbed fills a clear gap for reproducible, fine-grained evaluation of end-to-end methods in goal-oriented settings and is well-motivated and positioned in the literature. (2) Empirical results are thorough, carefully analyzed, and support the claims; the pattern of findings transfers from synthetic tasks to real dialog data (DSTC2 and concierge), making the resource practically useful.

Supporting arguments
- Problem significance: Scaling goal-oriented dialog beyond handcrafted slot-filling is an important community goal. Existing datasets either target state tracking modules, rely on noisy interactions with legacy systems, or lack reproducibility. A controlled, modular benchmark that stresses entity manipulation, API use, sorting, and OOV handling addresses an evident need.
- Placement and motivation: The work is well-situated relative to POMDP/slot-filling systems (Young et al., 2013) and to end-to-end chit-chat datasets (e.g., Twitter/Reddit, Ubuntu). The distinction between chit-chat and goal-oriented behaviors is explicit, and the test suite is designed to probe goal-driven capabilities rather than language modeling alone. The relation to bAbI QA tasks is appropriate, highlighting why dialog-specific behaviors (asking questions, API calls) are needed.
- Methodological soundness: The tasks are clearly specified with controllable difficulty, and the OOV split is carefully designed to probe generalization to unseen entities using KB grounding. The ranking setup, per-response and per-dialog metrics, and model/hyperparameter reporting in the appendix support reproducibility. The MemN2N variant and match-type features are appropriate baselines that target known weaknesses of embedding models with entities.
- Empirical support: Results consistently show: TF-IDF < nearest neighbor < supervised embeddings < MemN2N; match-type features yield substantial OOV gains and unlock Task 4. The failure modes on Tasks 3 and 5 (sorting and multi-step KB interpretation in end-to-end settings) are convincingly demonstrated. Importantly, the relative ordering of methods persists on DSTC2 and the concierge dataset, strengthening external validity. A rule-based system solves synthetic tasks but underperforms MemN2N on DSTC2, underscoring the utility of learning-based approaches in realistic conditions.
- Impact: By releasing an open resource with built-in error analysis via subtasks, the work can catalyze research on entity handling, KB integration, and robust dialog management.

Concerns and limitations (non-blocking)
- The simulator includes deterministic behaviors (e.g., slot query order), which may simplify dialog management. However, Tasks 3–5 and DSTC2 introduce non-determinism and noise, and the paper is explicit that the goal is controlled analysis rather than maximal realism.
- The evaluation is framed as next-utterance ranking with a candidate pool that includes responses from train/validation/test/OOV across tasks. While common in NUC setups, this choice should be discussed as it differs from generative or interactive evaluations and may favor retrieval-style methods.
- The concierge dataset is proprietary; release constraints limit broad benchmarking. The open synthetic/DSTC2 resources partially offset this.

Suggestions to improve the paper
Essential (feasible within rebuttal/camera-ready):
- Clarify candidate set construction and justify inclusion of test utterances in the candidate pool. Report sensitivity to candidate set size (e.g., random augmentation) and, if possible, an alternative evaluation where the candidate set is limited to training utterances to gauge robustness.
- Provide a brief ablation on MemN2N showing the effect of hops, time/speaker features, and match-type features per task, to better link architectural choices to observed gains.
- Expand the discussion on limitations of deterministic simulation choices and how Task 6 and concierge results mitigate these concerns.

Nice-to-have (optional):
- Include a pointer/copy baseline (or discuss expected performance) given its relevance to entity/OOV handling in goal-oriented dialog.
- Add variants with injected noise (paraphrases/ASR-like errors) to bridge synthetic and real conditions.
- Provide a short error taxonomy for Tasks 3 and 5 (e.g., sorting mistakes vs. option-tracking vs. stopping criterion).

Questions for clarification
- How large is the candidate set per task at test time, and how does per-response performance change when varying it? Is there leakage risk from including test-only candidates that share templates with training?
- In DSTC2 conversion, ratings were added to enforce deterministic option ordering. Could non-deterministic or multi-criterion ranking be supported to better reflect real recommendation behavior?
- For the OOV split, models have access to both KBs during training. How sensitive are results to withholding the OOV KB during training (i.e., testing pure generalization via type features without entity exposure)?
- Are there failure analyses on Task 3 that isolate sorting vs. dialog-control errors (e.g., wrong option order vs. offering too few/many options)?
- How stable are conclusions across different random seeds for simulator generation and model training?

Overall, this paper presents a timely, useful benchmark with clear insights and solid empirical execution. The resource is likely to become a reference point for goal-oriented dialog research.

- Introduces an open, reproducible restaurant-reservation testbed decomposed into five synthetic tasks plus a DSTC2 conversion and a real concierge dataset.
- Claims the testbed enables fine-grained diagnosis of state tracking, KB usage, and OOV handling.
- Finds Memory Networks with match-type features perform well but imperfectly, especially on API-related subtasks and OOV entities.
- Observes that performance patterns on synthetic tasks translate to real data, suggesting the testbed is a useful proxy.
- Decision: Accept—well-motivated benchmark with clear task decomposition and comprehensive, reproducible baselines.
- Clearly articulates the gap between chit-chat and goal-directed dialog and provides a thoughtful capability breakdown.
- Provides extensive comparisons (IR, supervised embeddings, MemN2N±type) across datasets, showing MemN2N advantages and the necessity of type features.
- Demonstrates difficulty for current models on option ranking and full dialogs, aligning with claimed limitations.
- Dataset design (ranking setup, OOV split, API grounding, sorting) and match-type features are practical and impactful; public release supports community use.
- Reproducibility aided by hyperparameters and appendices, though statistical significance tests are not reported.
- Suggests adding stronger baselines: copy/pointer mechanisms, span selection, constrained decoding, and seq2seq/transformers with KB grounding.
- Recommends reporting statistical significance/confidence intervals and ablation studies (hops, memory size, candidate set size, training size).
- Advises reducing determinism in synthetic tasks (randomize question order, add linguistic variability) to better reflect real-world variance.
- Suggests complementing ranking with generation-based evaluation and human judgments for Concierge; report goal success vs. turns and trade-offs.
- Asks for clarification on the DSTC2 conversion details (e.g., added ratings) and potential biases introduced.
- Requests info on average candidate set size at test time and how performance scales with it.
- Asks whether OOV surface forms always appear in the KB at inference and how the approach fares if entities are unseen in both dialogs and KB.
- Queries sensitivity of results to KB size/option cardinality and to training set size (learning curves).
- Asks if explicit ranking supervision (pairwise/listwise losses) was tried for Task 3 within MemN2N.
- Asks how often DSTC2 original bot behavior deviates from a deterministic policy and whether that drives most errors.

- Introduces a new testbed for evaluating end-to-end goal-oriented dialog systems in a restaurant reservation domain.  
- Proposes five diagnostic tasks generated by a KB-grounded simulator to isolate abilities like API calls, updates, and using KB info.  
- Provides baselines (IR, supervised embeddings, Memory Networks) and finds MemNNs have good per-response accuracy but poor per-dialog completion.  
- Validates the synthetic benchmark by showing model performance patterns match DSTC2 and a proprietary Concierge dataset.  
- Decision: Accept.  
- Addresses the need for standardized, reproducible, and analyzable benchmarks for goal-oriented dialog.  
- Strong methodology: diagnostic subtasks inspired by bAbI and comprehensive experimental evaluation.  
- Inclusion of an OOV test set to evaluate generalization to new entities.  
- Experiments reveal a large gap between per-response and per-dialog accuracy, showing single errors can derail conversations.  
- Introduces "match type features" to inject KB knowledge and help generalize to OOV entities.  
- Grounding the synthetic tasks in real datasets increases the benchmark’s credibility and practical relevance.  
- Limitation: evaluation framed as ranking from a fixed candidate set rather than full generative modeling—should be acknowledged.  
- Limitation: simulated language is template-based and lacks real-world linguistic diversity; suggest augmenting with paraphrasing.  
- Suggestion: add a brief contrast between Memory Networks and other architectures (e.g., seq2seq) to strengthen model choice justification.  
- Question: clarify the size of the candidate set and overlap between test correct responses and training candidates.  
- Question: consider non-KB approaches for handling OOV entities, such as character- or subword-level models (e.g., BPE).

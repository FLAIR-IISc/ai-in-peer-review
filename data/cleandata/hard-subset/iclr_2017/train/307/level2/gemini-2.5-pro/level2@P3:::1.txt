### Summary

This paper introduces a valuable new testbed for evaluating end-to-end goal-oriented dialog systems. Recognizing the gap between the success of such systems in chit-chat settings and the complex requirements of task-oriented conversations, the work proposes a series of diagnostic tasks in the restaurant reservation domain. The contribution is twofold: first, a set of five synthetically generated tasks that systematically break down the dialog process into sub-problems like issuing API calls, handling updates, and using knowledge base (KB) results. These tasks include an important out-of-vocabulary (OOV) condition to test generalization to unseen entities. Second, the paper adapts real-world data from the Dialog State Tracking Challenge 2 (DSTC2) and a proprietary concierge service into the same framework, allowing for a comparison between simulated and real-world performance. The paper provides a thorough empirical evaluation of several baselines, including classical IR methods, supervised embeddings, and Memory Networks. The results demonstrate that while Memory Networks are a promising architecture, significant challenges remain, as highlighted by the large gap between per-response and per-dialog accuracy. A key finding is that the performance patterns observed on the synthetic tasks are a good proxy for those on real data, validating the testbed's utility for future research.

### Decision: Accept

This is a strong paper that makes a clear and useful contribution to the field. The key reasons for this decision are:

1.  **A Well-Designed and Necessary Resource:** The work addresses a critical need for a standardized, reproducible, and diagnostic evaluation framework for goal-oriented dialog systems. The bAbI-style task decomposition is a principled approach to pinpointing model weaknesses, and the inclusion of both synthetic and real-world data, along with a dedicated OOV test set, makes the resource robust and highly valuable for the community.
2.  **Thorough and Insightful Experiments:** The empirical evaluation is comprehensive, comparing a representative end-to-end model against several strong baselines. The analysis provides important insights, such as the major challenge of achieving per-dialog correctness even with high per-response accuracy, and the effectiveness of a simple "match type feature" for handling OOV entities. These findings serve as an excellent benchmark for future work.

### Supporting Arguments for Decision

The paper is well-motivated, well-executed, and the contributions are significant. The lack of suitable benchmarks has been a hindrance to progress in end-to-end goal-oriented dialog, and this work provides a clear path forward. The design of the tasks is thoughtful, isolating specific capabilities that are crucial for successful task completion. For example, separating API call generation (Task 1) from using the results of an API call (Task 3) allows for a fine-grained analysis of model capabilities.

The experimental results strongly support the paper's claims. The consistent trend of Memory Networks outperforming supervised embeddings, which in turn outperform classical IR methods, is shown across both synthetic and real datasets. This validates the central premise that the synthetic testbed can act as an effective proxy for development, allowing for faster and more controlled experimentation. The reported performance numbers, particularly the near-zero per-dialog accuracy on more complex tasks like T3 and T5, clearly illustrate the difficulty of the problem and the limitations of current models, grounding the discussion in concrete evidence. The introduction and successful application of match type features to improve OOV performance is a nice, practical contribution that directly addresses a known weakness of embedding-based models.

### Additional Feedback for Improvement

The following points are suggestions aimed at further strengthening the paper and are not critical for the acceptance decision.

*   **Clarification on the Candidate Set:** The paper states that at each turn, candidates are ranked from a set of all bot utterances and API calls appearing in the combined training, validation, and test sets. This methodology could be clarified further. While this setup is reasonable for a next-utterance classification task, it may allow the model to select a correct response from the test set that contains entities or phrasing it has never been trained to associate with a given context. It would be helpful to briefly discuss the rationale for this choice over a more constrained candidate set (e.g., only those seen in training) and its potential implications for evaluating generalization.
*   **Qualitative Error Analysis:** The paper provides a strong quantitative analysis. This could be complemented by a brief qualitative error analysis in the main text. For instance, on Task 3 ("Displaying options"), where per-dialog accuracy is consistently 0%, what are the specific failure modes? Does the model fail to sort the options correctly, suggest a non-existent restaurant, or fail to recognize when the user accepts an option? A summary of common errors for the most difficult tasks would provide deeper insight into the models' shortcomings.
*   **Discussion on Generative Models:** The evaluation is framed as a ranking problem. This is a practical and widely used approach. However, a brief discussion on how this testbed could be adapted or used to evaluate generative dialog models would be a valuable addition. This would broaden the paper's applicability and connect it to another major thread of dialog research.

### Questions for the Authors

To help clarify some aspects of the work, it would be beneficial if the authors could address the following:

*   In the OOV experiments, the model has access to the full KB, including entities not seen in the training dialogs. Could the paper elaborate on precisely how the Memory Network is expected to leverage this KB information beyond the "match type features"? For instance, are there mechanisms for incorporating the KB facts directly into the memory representation?
*   The rule-based system for DSTC2 (Task 6) is a strong baseline. Could the paper briefly characterize the types of errors it makes? Understanding where a carefully crafted rule-based system fails could provide further motivation for the machine learning approaches.

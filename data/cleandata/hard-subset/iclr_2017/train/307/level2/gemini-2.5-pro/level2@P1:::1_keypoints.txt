- Introduces an open-source, reproducible testbed for evaluating end-to-end, goal-oriented dialog systems.  
- Proposes five synthetic bAbI-style restaurant reservation tasks targeting specific skills (API calls, query updates, KB result presentation, entity details, full dialog).  
- Includes an OOV setting to test generalization to unseen entities.  
- Provides evaluations on two real-world datasets (DSTC2 converted and a Concierge dataset) to validate the synthetic tasks.  
- Empirically compares IR baselines, supervised embeddings, and Memory Networks.  
- Finds Memory Networks with match type features are promising but still fail on complex tasks.  
- Highlights a notable gap between per-response and per-dialog accuracy, indicating difficulty maintaining multi-turn coherence.  
- Shows consistent relative model performance across synthetic and real datasets, supporting the benchmark’s validity.  
- Decision: Accept.  
- Key reason: Significant community contribution by filling a benchmark gap for goal-oriented dialog research.  
- Key reason: Effective task decomposition enables fine-grained error analysis and reveals specific failure modes (e.g., using KB results).  
- Strength: Thorough experimental evaluation providing strong baselines and a simple, effective match-type feature for handling entities/OOVs.  
- Suggestion: Discuss how to adapt and evaluate the testbed for generative models (e.g., seq2seq).  
- Suggestion: Acknowledge simulator language simplicity and discuss mitigation via real-dataset consistency.  
- Suggestion: Provide deeper error analysis for Task 3 (sorting/displaying options) to pinpoint failure causes.  
- Suggestion: Give more details on the rule-based system optimization (effort and tuning) used for Task 6.  
- Question: Why did match type features reduce Task 2 per-response accuracy (99.6% → 96.1%)?  
- Question: Have you tried randomizing the bot’s order for requesting missing info to increase task difficulty?  
- Question: Consider evaluating other end-to-end architectures (e.g., seq2seq with attention) for comparison.

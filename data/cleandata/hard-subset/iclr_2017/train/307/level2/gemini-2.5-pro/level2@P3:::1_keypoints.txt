- Introduces a new diagnostic testbed for end-to-end goal-oriented dialog in the restaurant domain.  
- Proposes five synthetic tasks that decompose dialog skills (API calls, updates, KB use) and include an OOV condition.  
- Adapts real-world datasets (DSTC2 and a concierge service) into the same framework for comparison.  
- Provides a comprehensive empirical evaluation against baselines: IR, supervised embeddings, and Memory Networks.  
- Finds Memory Networks are promising but large gaps remain, especially between per-response and per-dialog accuracy.  
- Shows that performance trends on synthetic tasks are a good proxy for real data, validating the testbed.  
- Decision: Accept — the paper makes a clear, useful contribution to the field.  
- Strength: Well-designed, reproducible resource with principled bAbI-style task decomposition and OOV testing.  
- Strength: Thorough experiments that reveal key limitations (e.g., near-zero per-dialog on complex tasks) and validate model rankings.  
- Practical contribution: Match type features effectively improve OOV handling for embedding-based models.  
- Supporting evidence: Memory Networks consistently outperform supervised embeddings, which outperform classical IR, across datasets.  
- Suggestion: Clarify candidate set construction and discuss implications for generalization vs. a more constrained candidate set.  
- Suggestion: Add a brief qualitative error analysis for the hardest tasks (e.g., Task 3) to illustrate common failure modes.  
- Suggestion: Discuss how the testbed could be adapted to evaluate generative dialog models.  
- Question: Clarify how Memory Networks leverage the full KB beyond match type features and how KB facts are represented in memory.  
- Question: Characterize the error types made by the rule-based DSTC2 baseline to motivate ML approaches.

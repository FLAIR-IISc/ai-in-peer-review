- Paper proposes a five-task testbed for evaluating end-to-end goal-oriented dialog systems in a restaurant-reservation context.
- Main contribution: a systematic, controlled evaluation framework for goal-oriented end-to-end dialog systems.
- Decision: accept.
- Reason 1: Addresses the important gap of lacking systematic evaluation for goal-oriented dialog systems.
- Reason 2: Testbed is well-motivated and well-designed, covering dialog management, knowledge-base querying, and interpreting API outputs.
- Support: Clear introduction that highlights limitations of traditional evaluations (poor reproducibility, hard-to-interpret errors).
- Support: Five tasks are designed to address these limitations with clear task descriptions and requirements.
- Support: Evaluation on two additional real-user datasets demonstrates the testbed’s real-world applicability.
- Feedback: Provide a more detailed explanation of the Memory Network architecture and its implementation/hyperparameters.
- Feedback: Include deeper result analysis (error analysis) and comparisons with other state-of-the-art models.
- Feedback: Give more information about resources required to run the testbed and expected computational cost.
- Question: Can the authors provide Memory Network implementation details and hyperparameter choices?
- Question: How will the testbed be extended to cover more goal-oriented dialog scenarios?
- Question: What are the resource and computational costs to run the testbed?

- Paper studies end-to-end dialog systems for goal-oriented tasks (restaurant reservations) and introduces a testbed with five synthetic tasks plus two real datasets.
- Authors evaluate Memory Networks against traditional slot-filling baselines and other machine learning methods.
- Memory Networks perform well on issuing/updating API calls but struggle with interpreting API results and conducting full dialogs.
- Conclusion: end-to-end systems need further advances to match or surpass traditional methods in goal-oriented settings.
- Decision: Accept.
- Novel contribution: provides a systematic, reproducible testbed valuable to the community.
- Rigorous evaluation: compares rule-based, classical IR, supervised embeddings, and Memory Networks to reveal strengths and weaknesses.
- Testbed design: breaks down dialog tasks into subtasks for targeted evaluation and error analysis.
- Real-world grounding: comparison with DSTC2 and Concierge ensures practical relevance beyond synthetic tasks.
- Technical insight: using match type features in Memory Networks helps handle out-of-vocabulary entities.
- Improvement requested: more detailed analysis of why Memory Networks fail on displaying options and full dialogs.
- Improvement requested: include comparisons with other state-of-the-art architectures (e.g., transformers).
- Improvement requested: expand dataset diversity to better capture noisy, variable real-world conversations.
- Question for authors: plans to use/extend the testbed to other domains beyond restaurant reservations?
- Question for authors: were techniques tried to improve Memory Networks on Task 3 (displaying options) or Task 5 (full dialogs), and what were results?
- Question for authors: could reinforcement learning or user simulators improve end-to-end system performance?

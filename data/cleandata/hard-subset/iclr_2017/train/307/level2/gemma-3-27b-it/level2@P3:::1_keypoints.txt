- Introduces a reproducible, lightweight testbed for evaluating end-to-end goal-oriented dialog (restaurant reservation).
- Proposes five synthetic tasks designed to isolate key dialog capabilities and evaluates on DSTC2 and a concierge dataset.
- Demonstrates Memory Networks’ performance, comparing to a rule-based baseline and other methods.
- Decision: accept with minor revisions.
- Strength: clear motivation and careful experimental design that isolates model behavior.
- Strength: inclusion of multiple datasets adds robustness to findings.
- Strength: match type features for handling OOV entities are a useful and insightful addition.
- Weakness: hyperparameter tuning process is insufficiently detailed—unclear how optimal settings were chosen.
- Weakness: task descriptions need more illustrative dialog examples to clarify differences between tasks.
- Weakness: impact of match type features lacks thorough ablation analysis per individual task.
- Weakness: conclusion could better expand on future work beyond merely improving models (e.g., alternative formulations/architectures).
- Question: please detail the hyperparameter tuning method (grid search, random search, etc.).
- Question: clarify rationale for choosing 3–4 hops in Memory Networks and trade-offs with performance.
- Question: how do match type features affect in-vocabulary entity performance—any degradation for known entities?
- Question: what are the specific error modes of Memory Networks on DSTC2, and how can they guide future development?
- Question: why was the rule-based system implemented only for DSTC2 and not for other datasets or synthetic tasks; would a rule-based baseline strengthen comparisons?

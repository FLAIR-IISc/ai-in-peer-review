- Introduces a well-designed testbed to evaluate end-to-end goal-oriented dialog systems, addressing a gap in suitable evaluation frameworks.  
- Proposes five synthetic restaurant-reservation tasks that isolate key capabilities (API calls, updating requests, displaying options, providing info, full dialogs) plus two real-world datasets for validation.  
- Demonstrates Memory Networks can perform well, especially when augmented with match type features for entity handling.  
- Highlights a persistent challenge: models struggle to interpret API results to display options (Task 3).  
- Recommends acceptance because the paper addresses a significant gap and provides a rigorous, useful testbed for the community.  
- Strength: thoughtful task design that systematically isolates required dialog capabilities and emphasizes reproducibility and interpretability.  
- Strength: thorough experimental setup including ablation studies, OOV analysis, and validation on DSTC2 and Concierge datasets.  
- Results: Memory Networks can learn implicit dialog state and issue API calls but fail at interpreting API results for option display.  
- Suggestion: include a more detailed analysis of why Task 3 is difficult, with specific failure cases (model vs. API-result interpretation).  
- Suggestion: report precise quantitative gains from match type features (e.g., exact accuracy improvements per task).  
- Suggestion: expand discussion of the rule-based DSTC2 baseline (40.7% per-response accuracy) and why it’s a reasonable real-world baseline.  
- Suggestion: clarify how the proposed testbed complements existing dialog state tracking benchmarks rather than replacing them.  
- Question: provide one or two concrete examples where the model generated correct API calls but misinterpreted the results.  
- Question: explain why OOV improvements with match type features are attributed to better performance on known words and prospects for handling truly novel entities.  
- Question: clarify why consistent relative performance across synthetic tasks and real Concierge data is significant beyond validating the synthetic proxy.

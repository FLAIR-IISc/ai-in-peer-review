This paper introduces a new dataset designed for evaluating end-to-end goal-oriented dialog systems. The dataset focuses on the restaurant reservation domain, where the objective is to determine availability and book a table based on user-provided preferences expressed throughout a dialog. Data generation employs a simulation driven by an underlying knowledge base, sampling parameter variations (cuisine, price range, etc.) and applying rule-based transformations to produce natural language utterances. The task is framed as ranking candidate responses for each dialog turn, with performance evaluated using per-response and per-dialog accuracy. The authors demonstrate that Memory Networks outperform basic bag-of-words baselines.

I thank the authors for this interesting contribution. However, I question the value of end-to-end trained systems within narrow domains. While end-to-end learning is compelling for open-domain scenarios where manual state and response coding is impractical, narrow domains are typically well-understood, and the primary goal is maximizing user satisfaction. In such cases, wouldn't leveraging domain knowledge to engineer an optimal system be more effective?

Furthermore, given the restricted domain, I find the choice to RANK responses rather than GENERATE them somewhat limiting, although I recognize the simplification this offers for evaluation. The practical sourcing of these candidate responses is also unclear. The models appear to rank all responses observed during training, validation, and testing (as noted in the last sentence before Section 3.2). Since a key benefit of end-to-end training is scalability to new domains without manual re-engineering, it’s unclear how this response set would be obtained in a novel domain. Generating responses would facilitate better generalization compared to ranking a pre-defined list of generic responses, and this aspect represents the primary weakness of the work.

Finally, the simulation-based data generation process, relying on expanding tuples like (cuisine, price) using natural language generation rules, inherently limits the variability of training responses. While this is a trade-off for the ability to generate an unlimited dataset, the specific rules used were not presented. Publishing these rules would be beneficial. 

Overall, despite my reservations, I believe this is a worthwhile contribution suitable for publication at the conference.

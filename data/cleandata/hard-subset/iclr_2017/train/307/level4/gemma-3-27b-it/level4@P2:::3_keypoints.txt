- Introduces a new dataset and evaluation framework for goal-oriented end-to-end dialog systems in the restaurant reservation domain.
- Dataset is generated via a knowledge-base-driven simulation, enabling controlled, large-scale data creation.
- Frames the task as ranking candidate responses rather than generation.
- Demonstrates Memory Networks outperform simpler baselines (IR and supervised embeddings).
- Recommendation: accept.
- Main strength: systematic, controlled evaluation approach and a valuable testbed for focused analysis.
- Effectively motivates the need for a dedicated benchmark for narrow, goal-oriented domains versus open-domain systems.
- Experimental setup includes both synthetic and real-world datasets (DSTC2, online concierge), supporting proxy validity of synthetic data.
- Ablation studies (e.g., match-type features) offer insights into model behavior.
- Analysis contrasts per-response vs per-dialog accuracy, highlighting dialog coherence as a remaining challenge.
- Critique: ranking-based evaluation is restrictive and may limit generalization to novel domains.
- Suggestion: publish the natural language generation rules from the simulation to improve reproducibility and dataset analysis.
- Suggestion: include detailed error analysis to identify whether failures are due to API interpretation, dialog state management, or unseen entities.
- Questions for authors: challenges and modifications needed to extend evaluation to generation tasks.
- Questions for authors: how to incorporate more realistic, stochastic user behavior versus the deterministic simulation.
- Questions for authors: request concrete examples of Memory Network errors, especially on harder tasks like displaying options.
- Questions for authors: what is the computational cost of training Memory Networks relative to simpler baselines?
- Questions for authors: how was anonymization performed for the concierge dataset and what biases might that introduce?

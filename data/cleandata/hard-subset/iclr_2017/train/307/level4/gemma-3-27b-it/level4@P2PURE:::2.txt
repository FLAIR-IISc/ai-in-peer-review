The proliferation of chatbots as a universal solution for human-computer interaction was a prominent trend in 2016, accompanied by claims of their ability to handle dialogs far beyond simple conversation. This paper provides a valuable and sobering assessment of the current state of affairs. While its primary relevance lies within the Dialog/Natural Language Processing communities – offering important insights for software engineers regarding the limitations of existing chatbots – it is also highly pertinent to the Machine Learning community, highlighting the necessity for more rigorous validation when applying ML techniques to dialog systems. Consequently, I assess this work as being of high significance.

The paper is underpinned by two key conjectures, articulated by Antoine Bordes during a NIPS workshop presentation on this work, though not explicitly stated within the paper itself. These conjectures are likely to stimulate further research, particularly given the chosen metrics:
1) The performance of end-to-end machine learning approaches remains inadequate for goal-oriented dialogs.
2) Relative performance on synthetic data accurately predicts performance on natural data – a potentially significant departure from prior observations, though the authors have made considerable effort to align the synthetic and natural conditions.

While the algorithmic contribution is relatively modest, consisting of a single, straightforward addition to memory networks (the match type), this is the first instance of these networks being deployed and evaluated on a goal-oriented dialog task. The experimental protocol is exceptionally well-designed. The paper is also remarkably clear and accessible to readers with backgrounds beyond machine learning and dialog research. I was particularly impressed by the concise and informative appendix summarizing memory networks, followed by the tables clearly illustrating the impact of the number of hops.

This paper represents the state-of-the-art in exploring more robust metrics for dialog modeling, while simultaneously demonstrating the inherent fragility and arbitrariness of existing metrics. This is more a suggestion for future research directions than a request for revision.

The use of per-response accuracy (essentially next utterance classification from a fixed set of responses) is problematic. Table 3 vividly demonstrates its limitations: success hinges on a correct API call and a concise dialog, yielding only around 1/7 accuracy given the need for exact matches across all six bot responses preceding the API call. 

Would per-dialog accuracy, requiring correctness across *all* responses, be preferable? Table 2 demonstrates its sensitivity to the experimental setup. The initially puzzling disparity between accuracy for subtask T3 (0.0) and the full dialog T5 (19.7) became clear upon reviewing the task definitions (3.1.1), which revealed that T3 requires displaying three options, whereas T5 requires only one.

For the concierge data, the metric’s reliance on being within the 5-best responses raises the question of what would happen if 'correct' instead meant being the *best* response.

While I do not fault the authors for employing standard dialog metrics, nor for acknowledging the potential for overly pessimistic new metrics, I suggest exploring alternative representations of dialogs that might yield more meaningful evaluations for goal-oriented systems. For example, if one were to commercialize Virtual Assistants based on successful dialog completion, the relevant metric would be one that maximizes revenue – perhaps a weighted sum of errors in the API call, the number of turns required, and the number of rejected user options. However, such a loss function is difficult to assess on pre-defined dialogs and would necessitate either real human users or a realistic dialog simulator.

Finally, a closely related issue regarding representation learning that the paper doesn’t fully address is the challenge of vocabulary mismatches between user input and the knowledge base. Specifically, for the match type algorithm to identify 'Indian' as a 'type of cuisine,' that word must appear verbatim in the knowledge base. Situations arise where the knowledge base employs obfuscated terminology, and we would prefer machine learning to infer the associations rather than relying on manual specification.

This paper introduces a new dataset and benchmark for evaluating end-to-end goal-oriented dialog systems, specifically within the restaurant reservation domain. The work focuses on systematically assessing the capabilities of these systems through a series of tasks designed to isolate key challenges like API calling, knowledge base querying, and dialog state management. The authors demonstrate the performance of Memory Networks on these tasks, comparing them to rule-based and information retrieval baselines, and show improvements over these baselines.

Reject. While the paper presents a valuable resource and analysis, the chosen evaluation paradigm—ranking from a predefined candidate set—limits the practical relevance and generalizability of the findings for truly end-to-end dialog systems. The focus on a ranking task, rather than generation, weakens the argument for the scalability benefits typically associated with end-to-end approaches.

The paper’s primary contribution lies in the creation of a structured testbed for goal-oriented dialog, which is a positive step towards more rigorous evaluation in this area. The tasks are well-defined and the synthetic data generation process allows for controlled experimentation and reproducibility. The comparison with a rule-based baseline and several information retrieval methods provides a useful point of reference. The observation that Memory Networks show improvements over these baselines is encouraging. Further, the validation of the synthetic data using real-world datasets from DSTC2 and an online concierge service strengthens the findings. 

However, the core methodology of ranking from a fixed candidate set raises several concerns. In a real-world deployment, a dialog system must *generate* responses, not select from a predetermined list. The method for constructing this candidate set—drawing from all utterances in the training, validation, and test sets—is not scalable to new domains, undermining the purported advantage of end-to-end learning. The authors acknowledge this limitation, but it remains a significant weakness. To strengthen the work, it would be beneficial to explore generative models or consider methods for dynamically generating candidate responses. Further, the specific natural language generation rules used to create the training data should be published to allow for reproducibility and analysis of their potential impact on model performance. Additionally, the effect of the match type features on the OOV performance could be further investigated. Is the improvement merely due to the fact that the KB entities can be recognised, or does it lead to better reasoning across the different tasks? Finally, more detailed error analysis of the Memory Network predictions would be helpful in pinpointing specific areas where the model struggles.

I have several clarifying questions for the authors: 1) How computationally expensive is generating the candidate set in practice, and how does this scale with the size of the training data? 2) Could the authors comment on the diversity of the candidate set and the potential for models to exploit biases within it? 3) What are the specific natural language generation rules used to create the training data, and how might these rules affect the generalizability of the results? 4) Is there a reason why the model performance dropped when matching type features were added in T2? 5) Did the authors evaluate any other generative models alongside the Memory Networks?

The recent surge in applying chatbots to a wide range of human-computer interaction scenarios, with claims of solving complex dialog tasks beyond simple conversation, necessitates a realistic evaluation of their capabilities. This paper provides a valuable and sobering assessment. Its relevance extends to both dialog/natural language processing communities – to inform software engineers about current chatbot limitations – and machine learning venues – to highlight the need for more rigorous validation of ML techniques applied to dialog systems. Consequently, I assess this work as highly significant.

The paper implicitly builds upon two key conjectures, articulated by Antoine Bordes during a NIPS workshop presentation: 
1) Current end-to-end machine learning approaches remain inadequate for goal-oriented dialogs.
2) Relative algorithmic performance on synthetic data can reliably predict performance on real-world data, a finding that, if substantiated, would contrast with previous observations. The authors have made a substantial effort to align synthetic and natural conditions to investigate this claim.

While the algorithmic contribution – a simple addition of a match type to memory networks – is incremental, this represents the first deployment and evaluation of this approach in a goal-oriented dialog setting. The experimental protocol is exceptionally well-designed. The paper is clearly written and accessible to a broad audience, including researchers beyond machine learning and dialog. The concise appendix summarizing memory networks and the tables illustrating the impact of the number of hops are particularly commendable. 

This work establishes a state-of-the-art exploration of rigorous metrics for dialog modeling while simultaneously demonstrating the inherent fragility and arbitrariness of such metrics. This observation is more a direction for future research than a request for revision.

The use of per-response accuracy (next utterance classification) is problematic, as Table 3 illustrates. Success is largely defined by a correct API call and a short dialog length, yielding a potentially misleading 1/7 accuracy given the need for exact matches across multiple bot responses. 

Would a stricter per-dialog accuracy, requiring correctness across all responses, be preferable? Table 2 reveals its sensitivity to the experimental setup. The discrepancy between the lower accuracy for subtask T3 (0.0) and the higher accuracy for the full dialog T5 (19.7) initially raised questions, but the task definitions (3.1.1) clarified that T3 requires displaying three options while T5 requires only one.

For the concierge data, considering ‘correct’ as achieving the *best* outcome, rather than simply being among the top five, warrants investigation. 

While the authors’ use of standard dialog metrics is justified, and the potential for overly pessimistic alternatives is acknowledged, I propose considering a revenue-maximization approach for representing goal-oriented dialogs. If offering Virtual Assistants as a service, paid upon successful completion, the metric should reflect this. In the restaurant scenario, a loss function encompassing the number of errors in the API call, the number of turns, and the number of rejected options by the user could be appropriate. However, such a loss function requires either real user interactions or a realistic simulator, precluding its use with static datasets.

Finally, the paper does not adequately address the challenge of vocabulary mismatch between user input and the knowledge base, a crucial aspect of representation learning. Specifically, the ‘match type’ algorithm requires exact lexical overlap – for example, the presence of ‘Indian’ in the knowledge base to associate it with ‘type of cuisine.’ In practical scenarios, knowledge bases may employ obfuscated terminology, necessitating machine learning models to infer associations rather than relying on explicit human-defined mappings.

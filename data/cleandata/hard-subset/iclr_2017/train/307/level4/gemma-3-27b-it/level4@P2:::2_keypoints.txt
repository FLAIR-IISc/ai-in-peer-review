- Valuable, timely reality check on deploying chatbots for goal-oriented dialogs, relevant beyond Dialog/NLP to ML community.  
- High significance rests on two conjectures: end-to-end ML is insufficient for goal-oriented dialog; relative performance on synthetic data predicts real-data performance.  
- Primary contribution: systematic evaluation of end-to-end dialog systems (Memory Networks with match-type features) in a restaurant reservation setting.  
- Experimental protocol is exceptionally well designed and the paper is clearly written and accessible.  
- Concise appendix and detailed tables (e.g., influence of number of hops) are particularly helpful.  
- Recommendation: accept — strong evaluation framework and analysis outweigh limited algorithmic novelty.  
- Demonstrates challenges of end-to-end learning in goal-oriented dialog, exposing weaknesses versus handcrafted systems.  
- Comprehensive comparison across methods: rule-based, IR, supervised embeddings, and Memory Networks.  
- Results validated on both synthetic and real-world datasets (DSTC2 and concierge service); relative performance trends are consistent.  
- Limitation: reliance on per-response accuracy metric, which can be misleading compared to per-dialog success.  
- Per-dialog accuracy is more informative but sensitive to task design (e.g., mismatch between subtasks and full dialog).  
- Suggestion: explore alternative metrics reflecting real-world goals (weighted API errors, dialog turns, user rejections), though hard to apply to canned dialogs.  
- Vocabulary mismatch issue: match-type features require exact lexical matches to KB, limiting generalization.  
- Recommendation to investigate learning associations between user vocabulary and KB concepts rather than hand-coded matches.  
- Questions posed: (1) effect of evaluating concierge results using top-5 suggestions vs top-1; (2) potential benefits/challenges of adding reinforcement learning and reward design; (3) what Memory Network modifications could improve complex reasoning/KB interaction (e.g., Task 3); (4) plans to learn mappings from user utterances to KB entities without exact lexical matches.

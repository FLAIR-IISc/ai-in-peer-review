- Presents a new publicly available dataset and tasks for evaluating goal-oriented dialogue systems in a controlled, rule-based environment.  
- Aims to provide reproducible, interpretable evaluation of capabilities like API calling, information presentation, and dialogue management.  
- Demonstrates the testbed by applying a Memory Network and comparing it to several baselines.  
- Recommendation: Reject — results do not show a clear advantage for end-to-end neural models over simpler methods.  
- Memory Network shows some improvements but is not consistently or significantly superior given its complexity and data needs.  
- Per-dialog accuracy remains low, indicating limited success on complete dialogues.  
- Tasks as currently defined may not be sufficiently challenging to highlight deep learning strengths.  
- Strength: well-motivated dataset breakdown into testable sub-components and comparison to rule-based and IR baselines.  
- Strength: inclusion of Dialog State Tracking Challenge and online concierge data supports generalizability.  
- Inclusion of match-type features improves performance but does not change the core conclusion.  
- Initial lack of a strong baseline capturing word order weakened the original assessment (later addressed).  
- Suggestion: increase task complexity (ambiguity, implicit confirmations, multi-turn reasoning).  
- Suggestion: explore more robust non-neural baselines that capture word order (n-grams, syntactic features, logistic regression with combined features).  
- Suggestion: perform detailed error analysis to identify model weaknesses and guide improvements.  
- Suggestion: run ablation studies to measure contributions of components (match-type features, number of hops).  
- Suggestion: clarify evaluation metrics and consider additional metrics like task completion rate or simulated user satisfaction.  
- Questions for authors: are current tasks sufficiently complex for end-to-end systems, and what modifications would increase difficulty?  
- Questions for authors: how will the testbed evaluate techniques for OOV handling (subword modeling, entity linking)?  
- Questions for authors: can you detail recurring error patterns made by the Memory Network?  
- Questions for authors: how sensitive are results to Memory Network hyperparameters and was a thorough hyperparameter search performed?

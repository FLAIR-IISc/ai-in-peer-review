1. Provides a reality check showing significant limitations of current end-to-end ML approaches for complex, goal-oriented dialog and calls for more realistic validation methods.  
2. Puts forward two stimulating conjectures: (a) end-to-end ML performance is insufficient for true goal-oriented dialog, and (b) relative algorithm performance on synthetic data predicts performance on natural data.  
3. Algorithmic contribution is modest (a “match type” addition to memory networks) but the work is the first rigorous deployment/testing of this architecture on goal-oriented dialog with a strong experimental protocol.  
4. Paper is exceptionally clear and accessible, with a helpful appendix and illustrative tables (e.g., effect of number of memory hops).  
5. Establishes a state-of-the-art benchmark while highlighting the fragility and arbitrariness of current dialog evaluation metrics.  
6. Standard metrics (e.g., per-response accuracy) can be misleadingly high because correct API calls can inflate scores despite irrelevant preceding responses.  
7. Accuracy is highly sensitive to experimental protocol and task definitions (example: subtask T3 vs T5); evaluation choices (e.g., top-1 vs top-5) matter for datasets like concierge.  
8. Recommends exploring alternative, application-driven metrics (e.g., revenue-weighted loss combining API errors, dialog length, and user rejections), which would require real users or realistic simulators.  
9. Critiques reliance on exact lexical matches: the match-type mechanism doesn’t solve vocabulary/KB mismatch and hand-defined mappings are problematic—models should learn such associations.

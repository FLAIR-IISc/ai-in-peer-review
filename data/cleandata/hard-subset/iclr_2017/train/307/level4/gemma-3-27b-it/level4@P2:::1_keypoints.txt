- Contribution: Introduces a new public dataset and task suite for goal-oriented restaurant-dialogue evaluation.  
- Decision: Reject — experimental results do not convincingly show end-to-end neural models outperform simpler approaches.  
- Main concern: Memory Networks show low per-dialog accuracy, indicating difficulty maintaining coherent goal-oriented conversations.  
- Baseline issue: Initial lack of word-order baseline was fixed; logistic regression with bi-grams and match-type features is competitive.  
- Implication: Competitive performance of simple models questions the necessity of the proposed complex neural architecture.  
- Limitation: Tasks 1–5 are deterministic/synthetic, limiting generalizability to noisy, ambiguous real-world dialogues.  
- Error analysis needed: More detailed analysis of Memory Network failures, especially interpreting API call results (Tasks 3 & 4).  
- KB complexity: Current knowledge base is simple; richer, larger KBs would provide a more realistic challenge.  
- Dataset diversity: Should be expanded beyond DSTC2 and Concierge to cover more domains and variability.  
- Evaluation metrics: Recommend adding task-completion rate and dialogue efficiency (e.g., number of turns) alongside per-response and per-dialog accuracy.  
- Clarification needed: Explain how “match type” features are integrated into the model and why they improve performance.  
- Questions for authors: What architectural/training changes could improve Memory Networks?  
- Questions for authors: Provide detailed error breakdowns per task to guide future model design.  
- Questions for authors: How robust are results to training data size/distribution and would larger/diverse datasets help?  
- Questions for authors: Has the dataset been released and how can others access it?

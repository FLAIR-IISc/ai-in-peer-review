SYNOPSIS:
This paper introduces a new dataset for evaluating end-to-end goal-oriented dialog systems. The data focuses on the restaurant domain, where the goal is to find availability and book a table based on user-provided parameters within a dialog. The dataset is generated through simulation, leveraging an underlying knowledge base to produce samples for various parameters (cuisine, price range, etc.), followed by rule-based transformations to render natural language descriptions. The task is to rank a set of candidate responses for each dialog turn, and evaluation is reported using per-response and per-dialog accuracy. The authors demonstrate that Memory Networks outperform basic bag-of-words baselines.

THOUGHTS:
I would like to thank the authors for this interesting contribution. However, I remain somewhat skeptical about the benefits of end-to-end trained systems in narrow-domain settings. In open-domain scenarios, there’s a strong case to be made that hand-coding states and responses wouldn’t scale effectively, making end-to-end learning a sensible approach. But in narrow domains, we typically possess a strong understanding of the subject matter, and the primary goal is maximizing user satisfaction. Wouldn’t it then be more effective to leverage this domain knowledge to engineer the optimal system possible?

Given the already restricted domain, I was also a little disappointed to see the task framed as ranking responses rather than generating them, although I recognize that ranking simplifies evaluation considerably. I’m also unclear how candidate responses would be obtained in a practical deployment. The models appear to rank the complete set of responses observed during training, validation, and testing (as noted in the last sentence before Section 3.2). Since a key argument for end-to-end training is its ability to scale to new domains without extensive manual re-engineering, how would this set of candidates be acquired for a novel domain in practice? Generating responses would facilitate much better generalization to new domains compared to simply ranking a predetermined list of hand-collected responses, and this is, in my opinion, the weakest aspect of this work.

Finally, because the data is generated using a simulation that expands tuples (e.g., cuisine, price) with natural language generation rules, the variability in training responses is inherently constrained. While this is a trade-off for the ability to generate an unlimited amount of data, I could not locate the list of rules used in the paper. Publishing these rules would be beneficial.

Overall, despite my reservations, I believe this is an interesting contribution that warrants publication at the conference.

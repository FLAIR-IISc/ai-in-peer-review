- Paper provides a timely reality check on end-to-end dialog systems by tackling goal-oriented conversations beyond chit-chat.  
- Algorithmic contribution is incremental (match-type features added to Memory Networks).  
- Main strength is a carefully designed testbed, thorough experiments, and insightful analysis.  
- Work is significant for NLP and ML communities, stressing need for realistic validation of dialog models.  
- Decision: Accept; paper is well-motivated, convincingly supported, and clearly written.  
- Demonstrates current end-to-end models struggle with goal-oriented dialog even in narrow domains (restaurant reservation).  
- Frames two key conjectures: current performance is insufficient, and synthetic data predicts real-world performance.  
- Introduces a five-task testbed and evaluates on both synthetic and real datasets (DSTC2 and concierge data).  
- Consistent results across datasets suggest synthetic-task performance is a reliable proxy for real-world performance.  
- Ablation studies, especially match-type features, illuminate Memory Network strengths and weaknesses.  
- Analysis contrasting per-response vs per-dialog accuracy highlights limitations of standard evaluation metrics.  
- Suggests exploring alternative, value-oriented metrics (e.g., cost- or revenue-based) to better reflect real-world utility.  
- Notes vocabulary mismatch between user input and KB as a limitation and recommends exploring entity mapping or synonym handling.  
- Calls for more detailed error analysis to identify scenarios where the model fails (complex reasoning, ambiguous requests).  
- Requests clarification on how words are designated as 'typed' for match-type features beyond appearing as KB entities.  
- Questions for authors: explain sensitivity of per-dialog accuracy to protocol and how to design less-sensitive tasks.  
- Questions for authors: how results change if correctness is defined as top-N (e.g., top-5) rather than exact match for concierge data.  
- Questions for authors: provide a concrete example of a cost-based metric for restaurant reservations with specific weighting factors.  
- Questions for authors: propose strategies to address vocabulary mismatch so models associate user terms with KB entities without exact matches.

- Paper provides a reality check on using chatbots for goal-oriented dialogs and is highly significant for dialog and ML communities.  
- Two key conjectures: end-to-end ML is insufficient for goal-oriented dialogs, and synthetic data performance predicts natural data performance.  
- Strong effort to align synthetic and natural conditions to test those conjectures.  
- Main algorithmic contribution: introducing match type features to Memory Networks for goal-oriented dialog tasks.  
- Experimental protocol is excellent, with clear, accessible presentation and a concise appendix explaining Memory Networks and hop effects.  
- Paper highlights that dialog evaluation metrics remain brittle and somewhat arbitrary.  
- Per-response accuracy (next-utterance classification) is limited and can be misleading in practice.  
- Per-dialog accuracy might be a better metric but is highly sensitive to task definitions and protocol.  
- Question raised: would allowing “correct” to be top-5 responses change results on the concierge dataset?  
- Proposes a more realistic loss function for deployed assistants (weighted errors on API calls, turn count, rejected options) requiring humans or realistic simulators.  
- Notes vocabulary mismatch issue: match type features need exact KB wording and ML should learn associations instead of relying on hand-defined mappings.  
- Decision: Accept — paper tackles an important problem and provides a valuable, reproducible testbed.  
- Supporting strengths: addresses a key problem, well-motivated testbed focused on reproducibility, and rigorous experiments including rule-based baselines and multiple datasets.  
- Additional suggestions: consider reinforcement learning with task-level rewards, explore methods to bridge user–KB vocabulary gaps, and further discuss challenges in creating meaningful goal-oriented dialog metrics.

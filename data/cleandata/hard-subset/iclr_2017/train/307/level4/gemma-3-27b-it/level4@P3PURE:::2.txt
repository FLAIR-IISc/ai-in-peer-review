Attempts to apply chatbots to all forms of human-computer interaction were a prominent trend in 2016, with claims that they could handle dialogs far beyond simple chit-chat. This paper provides a crucial reality check. While most relevant for dialog and natural language processing venues—to educate software engineers about the limitations of current chatbots—it is also highly significant for machine learning venues, as it highlights the need for more realistic validation of machine learning approaches applied to dialog systems. Therefore, I would consider this work to be of high significance.

Two important conjectures underpin this paper and are likely to stimulate further research. Though not explicitly stated, Antoine Bordes clearly articulated them during a NIPS workshop presentation on this work: (1) the performance of end-to-end machine learning approaches remains insufficient for goal-oriented dialogs, and (2) relative performance on synthetic data is a good predictor of performance on natural data. This latter point would represent a departure from previous observations, but the authors have made a strong effort to align the synthetic and natural conditions. 

While the original algorithmic contribution—a relatively simple addition to memory networks (the match type)—is modest, this paper represents the first deployment and thorough testing of this architecture on a goal-oriented dialog task. The experimental protocol is excellent. The paper is exceptionally clear and accessible to a broad readership, including researchers beyond machine learning and dialog systems. I was particularly impressed by the concise and informative appendix on memory networks, which effectively summarized the architecture, followed by tables illustrating the influence of the number of hops.

This paper not only establishes the state-of-the-art in rigorously evaluating dialog modeling but also reminds us how brittle and, to some extent, arbitrary current metrics remain. This is more a suggestion for future research than a request for revision. 

The authors employ per-response accuracy (essentially next utterance classification from a fixed list of responses). Table 3 clearly demonstrates how inadequate this metric can be in practice: success hinges on a correct API call and a reasonably short dialog, yet this yields only a 1/7 accuracy because the preceding six bot responses must also be exact. 

Would per-dialog accuracy—requiring all responses to be correct—be more informative? Table 2 reveals its sensitivity to the experimental protocol. I was initially puzzled by the significantly lower accuracy for subtask T3 (0.0) compared to the full dialog task T5 (19.7), until the authors clarified the task definitions (3.1.1); T3 requires displaying three options, while T5 requires only one.

For the concierge dataset, what would happen if ‘correct’ meant identifying the *best* response, rather than one within the top five? 

While I cannot fault the authors for employing standard dialog metrics, or for recognizing the potential for overly pessimistic new metrics, I believe there is a way to represent dialogs that could lead to more meaningful evaluation in goal-oriented scenarios. For example, consider selling Virtual Assistants as a service, with payment contingent upon successful dialog completion. The optimal metric would then be one that maximizes revenue, potentially a weighted sum of errors in the API call, the number of turns required to reach the call, and the number of rejected options by the user. However, such a loss function cannot be directly measured on canned dialogs and would necessitate either a real human user or a realistic simulator.

Another issue, closely related to representation learning, that this paper does not fully address is the handling of vocabulary mismatches between user input and the knowledge base. Specifically, for the match type algorithm to identify ‘Indian’ as a ‘type of cuisine’, that word must appear exactly within the knowledge base. It is conceivable that the knowledge base might employ obfuscated terminology, and it would be desirable for machine learning to *learn* these associations rather than relying on explicit human annotation.

- Provides a necessary reality check on overclaims about chatbots solving complex goal-oriented dialogs.
- Highly significant and relevant to both Dialog/NLP and ML communities; educates engineers about current chatbot limitations.
- Two important conjectures: end-to-end ML is still insufficient for goal-oriented dialogs; relative performance on synthetic data can predict natural-data performance if conditions are well-matched.
- Authors carefully match synthetic and natural conditions, a departure from prior work.
- Algorithmic contribution (match type) is simple but notable as the first deployment of memory networks on goal-oriented dialog.
- Experimental protocol and paper clarity are excellent, with a helpful appendix and informative tables (e.g., effect of number of hops).
- Advances state-of-the-art in rigorous dialog metrics while exposing their brittleness and arbitrariness.
- Per-response accuracy is misleading in practice because many dialog successes depend on a single final API call despite multiple intermediate responses.
- Per-dialog accuracy may be more meaningful but is highly sensitive to experimental protocol and task definitions.
- Task definitions explain discrepancies (e.g., T3 requiring three options vs T5 requiring one), affecting reported accuracies.
- For concierge data, using "best" instead of "5-best" correctness could change results and merits exploration.
- More realistic metrics (e.g., revenue-weighted loss combining API errors, dialog length, and rejected options) would be preferable but require real users or realistic simulators.
- Representation-learning limitation: match type relies on exact vocabulary overlap between user utterances and KB, failing when KB uses different/obfuscated terms.
- Desirable for ML to learn associations between user language and KB entries instead of relying on manual KB curation.

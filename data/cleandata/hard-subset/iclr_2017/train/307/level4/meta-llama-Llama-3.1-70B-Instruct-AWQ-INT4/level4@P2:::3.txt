This paper proposes a testbed for evaluating end-to-end goal-oriented dialog systems in a restaurant reservation setting. The authors generate data using a simulation based on an underlying knowledge base, which allows for the creation of various tasks that test different aspects of dialog systems. They evaluate several models, including rule-based systems, classical information retrieval methods, supervised embeddings, and end-to-end Memory Networks, on these tasks. The results show that Memory Networks outperform other models, but still struggle with tasks that require interpreting knowledge about entities and presenting results to the user.

The paper makes an interesting contribution to the field of dialog systems, but I have some reservations about the utility of end-to-end trained systems in narrow-domain settings. In such settings, it is often possible to engineer high-quality systems using domain knowledge, which may be more effective than relying on end-to-end trained methods. Additionally, the paper's focus on ranking candidate responses rather than generating them may limit the generalizability of the approach to new domains.

I also have some concerns about the simulation-based approach used to generate data. While it allows for the creation of unlimited data, it may constrain the variability in the training responses. It would be helpful to see the list of rules used to generate the data, as well as more information about how the candidate responses were obtained.

Overall, I think the paper is worthy of publication, but I would like to see some revisions to address these concerns. Specifically, I would like to see more discussion of the trade-offs between end-to-end trained systems and engineered systems in narrow-domain settings, as well as more information about the data generation process and the candidate responses.

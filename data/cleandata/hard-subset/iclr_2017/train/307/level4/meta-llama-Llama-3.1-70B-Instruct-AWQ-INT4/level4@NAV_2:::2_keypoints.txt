- Paper provides a significant reality check on current chatbots’ ability to handle goal-oriented dialogs.
- Proposes a restaurant-reservation testbed for evaluating end-to-end dialog systems.
- Testbed contains five tasks requiring sentence/symbol manipulation, issuing API calls, and using API outputs.
- Evaluates rule-based systems, classical IR, supervised embeddings, and end-to-end Memory Networks.
- Experimental protocol is excellent and the paper is clear and accessible beyond ML/dialog experts.
- Small algorithmic contribution: a simple addition to Memory Networks, first tested on goal-oriented dialog.
- Uses per-response and per-dialog accuracy metrics and highlights their limitations (per-response can be misleading).
- End-to-end Memory Networks outperform IR and supervised embeddings but still struggle with interpreting KB entity info from API returns.
- Results are validated on a real-world concierge dataset, showing similar relative performance patterns.
- Critique: reliance on standard dialog metrics may be inadequate; suggests alternative weighted error metrics (errors in API call, turns to API call, rejected options) which require humans or realistic simulators.
- Critique: match-type algorithm requires exact word matches in the KB and fails when user vocabulary doesn’t match KB vocabulary.
- Suggestion: explore approaches that learn associations between user words and KB entities instead of exact matching.
- Request for more details on the match-type algorithm and handling of out-of-vocabulary words.
- Open questions: how authors will address metric limitations, extend to multi-API/more complex dialogs, and evaluate with real human users or realistic simulators.
- Recommendation: Accept — due to the paper’s reality-check significance, valuable testbed, and strong experimental protocol.

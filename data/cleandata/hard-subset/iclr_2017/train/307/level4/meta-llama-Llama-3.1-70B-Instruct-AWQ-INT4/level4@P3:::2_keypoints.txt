- Presents a testbed for evaluating end-to-end goal-oriented dialog systems, focused on restaurant reservation.
- Testbed consists of five tasks requiring sentence/symbol manipulation, API calls, and use of API outputs.
- Evaluates a Memory Network model (with added match type features) against a hand-crafted slot-filling baseline and other baselines on DSTC2 data.
- Main algorithmic contribution: match type features for Memory Networks, which improve performance.
- Memory Networks with match type features outperform classical IR methods and supervised embedding models.
- Demonstrates that end-to-end machine learning approaches remain insufficient for robust goal-oriented dialogs.
- Finds that relative performance on synthetic data can predict performance on natural data.
- Experimental protocol is excellent and the paper is very clear and accessible beyond ML/dialog researchers.
- Highlights brittleness and arbitrariness of common evaluation metrics for dialog modeling.
- Criticizes per-response accuracy as unrealistic because it requires exact matches for non-critical responses.
- Notes per-dialog accuracy is sensitive to experimental protocol and may not reflect true system performance.
- Suggests more meaningful, user-centric metrics (e.g., weighted sum of API errors, turns to API call, rejected options).
- Raises an unaddressed issue: handling vocabulary mismatches between user utterances and the knowledge base.
- Requests more details on Memory Network implementation, match type feature implementation, and hyperparameters.
- Overall recommendation: accept — significant contribution, strong protocol, clear writing, and useful insights into chatbot limitations.

This paper serves as a reality check on the recent trend of using chatbots for various forms of human-computer interaction, which has been gaining momentum since 2016. The authors' work is highly significant, as it highlights the limitations of current chatbots in handling goal-oriented dialogs. While the paper is primarily relevant to the Dialog/Natural Language community, it also holds value for the Machine Learning community, as it emphasizes the need for more realistic validation of machine learning models applied to dialogs.

Two underlying conjectures in this paper, as stated by Antoine Bordes during a NIPS workshop presentation, are likely to inspire further research. These conjectures are: (1) the performance of end-to-end machine learning approaches is still insufficient for goal-oriented dialogs, and (2) relative performance on synthetic data can be a good predictor of performance on natural data, provided that the synthetic and natural conditions are carefully matched. The authors have made a concerted effort to align these conditions, making their findings noteworthy.

Although the paper's original algorithmic contribution is relatively simple, consisting of a modification to memory networks (match type), it is the first time these networks have been deployed and tested on a goal-oriented dialog. The experimental protocol is excellent, and the paper's clarity is outstanding, making it accessible to a broad readership beyond machine learning and dialog researchers. The concise appendix on memory networks and the accompanying tables effectively explain the influence of the number of hops.

This paper represents the state-of-the-art in exploring more rigorous metrics for dialog modeling. However, it also underscores the brittleness and arbitrariness of these metrics. This is more a recommendation for future research than a suggestion for revision.

The authors employ per-response accuracy, which can be misleading in practice, as demonstrated in Table 3. A correct API call and a reasonably short dialog are often the primary objectives, but this would only yield a 1/7 accuracy, as the six bot responses leading to the API call must also be exact. Per-dialog accuracy, where all responses must be correct, may be a better metric, but Table 2 shows its sensitivity to the experimental protocol.

The paper raises questions about the concierge data, such as what would happen if 'correct' meant being the best, rather than among the 5-best options. While the authors cannot be faulted for using standard dialog metrics, it is possible to envision alternative representations that could lead to more meaningful metrics in goal-oriented dialogs. For instance, if one were to sell Virtual Assistants as a service, with payment upon successful completion of a dialog, the metric that would maximize revenue would likely be a weighted sum of the number of errors in the API call, the number of turns to reach that API call, and the number of rejected options by the user. However, such a loss function cannot be measured on canned dialogs and would require either real human users or a realistic simulator.

The paper also overlooks an issue closely related to representation learning: what happens when the vocabulary used by the user does not match exactly the vocabulary in the knowledge base. For example, the match type algorithm requires the word 'Indian' to occur exactly in the knowledge base to code it as 'type of cuisine.' This limitation may lead to issues when the knowledge base uses obfuscated terminology, and it would be desirable for machine learning models to learn the associations rather than relying on manual descriptions.

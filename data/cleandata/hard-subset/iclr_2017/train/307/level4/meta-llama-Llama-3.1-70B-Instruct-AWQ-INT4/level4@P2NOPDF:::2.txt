This paper provides a much-needed reality check on the use of chatbots for human-computer interaction, a trend that gained significant attention in 2016. While its primary relevance lies in Dialog/Natural Language venues, where it educates software engineers about the limitations of current chatbots, it also holds value for Machine Learning venues, as it highlights the need for more realistic validation of ML applied to dialogs. Consequently, I consider this work to be of high significance.

Two crucial conjectures underlie this paper, which were explicitly stated by Antoine Bordes during a NIPS workshop presentation. These conjectures are likely to inspire further research. The first conjecture suggests that the performance of end-to-end ML approaches remains insufficient for goal-oriented dialogs. The second conjecture proposes that relative performance on synthetic data can be a reliable predictor of performance on natural data, which would be a departure from previous observations. The authors made a concerted effort to match synthetic and natural conditions, lending credence to this claim.

Although the paper's original algorithmic contribution is relatively simple, consisting of a modification to memory networks (match type), it is the first time these networks have been deployed and tested on a goal-oriented dialog. The experimental protocol is excellent, and the overall paper clarity is outstanding, making it accessible to a broad readership beyond ML and dialog researchers. The concise appendix on memory networks is particularly noteworthy, as it provides a clear summary of the concept. The tables explaining the influence of the number of hops are also well-presented.

This paper represents the state-of-the-art in exploring more rigorous metrics for dialog modeling. However, it also highlights the brittleness and arbitrariness of these metrics, which is more a recommendation for future research than a criticism of the paper.

The authors employ per-response accuracy as a metric, which can be misleading in practice. As shown in Table 3, this metric prioritizes correct API calls and reasonably short dialogs, but it would only yield a 1/7 accuracy, as the six bot responses leading to the API call must also be exact.

Per-dialog accuracy, which requires all responses to be correct, may be a more suitable metric. However, as demonstrated in Table 2, this metric is highly sensitive to the experimental protocol. The authors' task definitions (Section 3.1.1) clarify the discrepancy between the accuracy for subtask T3 (0.0) and the full dialog T5 (19.7), where T3 requires displaying three options, while T5 only requires displaying one.

For the concierge data, it would be interesting to explore the implications of defining 'correct' as the best response, rather than one of the five best.

While the authors' use of standard dialog metrics is not flawed, and their introduction of new, more pessimistic metrics is a valuable contribution, I propose an alternative approach to representing dialogs. If I were to sell Virtual Assistants as a service, with payment contingent upon successful dialog completion, the metric that would maximize revenue would likely be a weighted sum of the number of errors in the API call, the number of turns to reach the API call, and the number of rejected options by the user. However, such a loss function cannot be measured on canned dialogs and would require either real human users or a realistic simulator.

Another issue closely related to representation learning that this paper does not adequately address is the scenario where the user's vocabulary does not exactly match the vocabulary in the knowledge base. Specifically, the match type algorithm requires the word 'Indian' to be coded as 'type of cuisine' if it appears exactly in the KB. In situations where the KB uses obfuscated terminology, it would be desirable for ML to learn the associations rather than relying on human-defined descriptions.

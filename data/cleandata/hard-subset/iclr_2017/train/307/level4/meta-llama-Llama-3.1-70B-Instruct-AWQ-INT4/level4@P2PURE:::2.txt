The recent trend of utilizing chatbots for various forms of human-computer interaction has been a prominent phenomenon in 2016, with proponents claiming their ability to tackle complex dialogs beyond mere chit-chat. This paper provides a much-needed reality check, offering valuable insights that can benefit both dialog/natural language processing and machine learning communities. Its significance lies in its ability to educate software engineers about the limitations of current chatbots and to inform researchers about the necessity of more realistic validation for machine learning applications in dialogs.

Two crucial conjectures underlie this paper, which were explicitly stated by Antoine Bordes during a NIPS workshop presentation covering this work. These conjectures are: (1) the performance of end-to-end machine learning approaches is still insufficient for goal-oriented dialogs, and (2) relative performance on synthetic data can be a good predictor of performance on natural data, provided that the conditions are carefully matched. The authors have made a concerted effort to align the synthetic and natural conditions, making this finding noteworthy.

Although the algorithmic contribution of this paper is relatively simple, consisting of a single addition to memory networks (match type), it is the first time these networks have been deployed and tested on a goal-oriented dialog. The experimental protocol is excellent, and the paper's clarity is outstanding, making it accessible to a broad readership beyond machine learning and dialog researchers. The concise appendix on memory networks and the accompanying tables effectively explaining the influence of the number of hops are particularly noteworthy.

This paper represents the state-of-the-art in exploring more rigorous metrics for dialog modeling. However, it also highlights the brittleness and arbitrariness of these metrics, which is more a recommendation for future research than a revision suggestion. The authors employ per-response accuracy, which can be misleading in practice, as demonstrated in Table 3. A correct API call and a reasonably short dialog may only yield a 1/7 accuracy, despite the six bot responses required to reach the API call needing to be exact.

The alternative metric of per-dialog accuracy, where all responses must be correct, is also sensitive to the experimental protocol, as shown in Table 2. The significant disparity in accuracy between subtask T3 (0.0) and the full dialog T5 (19.7) can be attributed to the task definitions, where T3 requires displaying three options, whereas T5 only requires displaying one.

For the concierge data, it would be interesting to explore the implications of defining 'correct' as being the best option, rather than among the top 5. While the authors' use of standard dialog metrics is understandable, and their proposed new metrics may be overly pessimistic, an alternative approach could be to represent dialogs in a way that yields more meaningful metrics for goal-oriented dialogs. For instance, if a virtual assistant service is paid upon successful completion of a dialog, the optimal metric would likely be a weighted sum of the number of errors in the API call, the number of turns to reach the API call, and the number of rejected options by the user. However, such a metric cannot be measured on canned dialogs and would require either real human users or a realistic simulator.

Another issue closely related to representation learning that this paper fails to address is the scenario where the user's vocabulary does not exactly match the vocabulary in the knowledge base. Specifically, for the match type algorithm to effectively code 'Indian' as 'type of cuisine,' the word 'Indian' would need to occur exactly in the knowledge base. This limitation may lead to issues when the knowledge base employs obfuscated terminology, and it would be desirable for machine learning to learn the associations rather than relying on manual descriptions.

This paper provides a much-needed reality check on the recent trend of using chatbots for various forms of human-computer interaction, which has been claimed to solve many forms of dialogs beyond simple chit-chat. The work is highly significant and relevant to both Dialog/Natural Language and Machine Learning venues, as it educates software engineers about the limitations of current chatbots and highlights the need for more realistic validation of machine learning applied to dialogs.

Two important conjectures underlying this paper, as stated by Antoine Bordes during a NIPS workshop presentation, are likely to open up new avenues for research. These conjectures are: (1) the performance of end-to-end machine learning approaches is still insufficient for goal-oriented dialogs, and (2) relative performance on synthetic data can be a good predictor of performance on natural data, provided that the synthetic and natural conditions are carefully matched. The authors have made a strong effort to match these conditions, which is a departure from previous observations.

Although the original algorithmic contribution of the paper is a relatively simple addition to memory networks (match type), it is notable that this is the first time these networks have been deployed and tested on a goal-oriented dialog. The experimental protocol is excellent, and the paper's clarity is excellent, making it accessible to a readership beyond machine learning and dialog researchers. The appendix on memory networks provides a clear summary, and the tables effectively explain the influence of the number of hops.

This paper represents the state-of-the-art in exploring more rigorous metrics for dialog modeling. However, it also highlights the brittleness and arbitrariness of these metrics, which is more a recommendation for future research than a criticism of the paper. The authors use per-response accuracy, which can be misleading in practice, as shown in Table 3. A correct API call and a reasonably short dialog may only yield a 1/7 accuracy, as the six bot responses needed to reach the API call must also be exact.

Per-dialog accuracy, where all responses must be correct, may be a better metric, but Table 2 shows its sensitivity to the experimental protocol. The authors' task definitions (Section 3.1.1) clarify the discrepancy between the accuracy for subtask T3 (0.0) and the full dialog T5 (19.7), as T3 requires displaying three options while T5 only requires displaying one.

For the concierge data, it would be interesting to explore what would happen if 'correct' meant being the best, rather than among the 5-best. While the authors cannot be faulted for using standard dialog metrics, it is possible to conceive of alternative metrics that could be more meaningful for goal-oriented dialogs. For instance, if one were to sell Virtual Assistants as a service, being paid upon successful completion of a dialog, the metric that would maximize revenue would likely be a weighted sum of the number of errors in the API call, the number of turns to reach that API call, and the number of rejected options by the user. However, such a loss function cannot be measured on canned dialogs and would require either a real human user or a realistic simulator.

Another issue closely related to representation learning that this paper fails to address is what happens when the vocabulary used by the user does not match exactly the vocabulary in the knowledge base. In particular, for the match type algorithm to code 'Indian' as 'type of cuisine', this word would have to occur exactly in the KB. This limitation could be problematic in situations where the KB uses obfuscated terminology, and it would be desirable for machine learning to learn the associations rather than relying on human hand-curation.

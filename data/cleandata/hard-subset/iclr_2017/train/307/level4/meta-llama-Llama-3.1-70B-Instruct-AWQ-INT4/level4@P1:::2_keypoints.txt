Paper provides a reality check on end-to-end ML for goal-oriented dialogs, showing limits beyond chit‑chat.  
Highly relevant to Dialog/NLP and ML communities; educates engineers and researchers about the need for more realistic validation.  
Conjecture 1: end-to-end ML performance is still insufficient for complex goal-oriented dialogs; models struggle on tasks requiring reasoning/decision-making.  
Conjecture 2: relative performance on well-matched synthetic data can predict performance on natural data.  
Original contribution: match type features added to memory networks — simple, effective, first deployment on goal-oriented dialog.  
Experimental protocol is systematic, controlled, and enables fair model comparisons.  
Paper clarity is excellent and accessible to non-specialists.  
Major concern: chosen evaluation metrics (per-response, per-dialog, concierge top-5) can be misleading or insufficient for goal-oriented tasks.  
Per-dialog accuracy is sensitive to protocol (e.g., subtask T3 much lower than full dialog T5), indicating metric/task-definition issues.  
Recommendation: adopt more informative, task-aware metrics (e.g., weighted objective reflecting API errors, dialog length, user rejections) which require realistic simulators or human data.  
Limitation: representation learning is under-addressed—match type requires exact KB vocabulary (e.g., 'Indian') and fails with obfuscated or mismatched terms.  
Desirable: methods that learn KB–user vocabulary associations instead of relying on hand-annotation.  
Overall judgment: valuable contribution with no major flaws; reviewer recommends Accept.  
Questions for authors: detail match type implementation and performance impact; plans for handling vocabulary mismatch; discuss applications/implications for virtual assistants.

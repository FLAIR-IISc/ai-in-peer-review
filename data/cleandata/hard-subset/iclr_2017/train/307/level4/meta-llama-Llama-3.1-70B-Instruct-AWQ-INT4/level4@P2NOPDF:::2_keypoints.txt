- Paper provides a needed reality check on chatbot use in HCI, significant for Dialog/NLP and ML communities.  
- Two key conjectures: end-to-end ML is inadequate for goal-oriented dialogs; relative performance on synthetic data may predict natural data performance.  
- Authors largely matched synthetic and natural conditions, supporting the second conjecture.  
- Algorithmic contribution is simple (match-type modification to memory networks) but novel in applying memory networks to goal-oriented dialogs.  
- Experimental protocol is excellent and paper clarity is outstanding, with a helpful appendix summarizing memory networks.  
- Tables on the effect of the number of hops are well-presented and informative.  
- Work advances dialog evaluation metrics but highlights their brittleness and arbitrariness, suggesting future research directions.  
- Per-response accuracy is misleading in practice because it can overvalue getting the API call right while ignoring intermediate response errors.  
- Per-dialog accuracy is more appropriate but highly sensitive to task definitions and experimental protocol.  
- Task definition sensitivity exemplified by the T3 (0.0) vs T5 (19.7) discrepancy due to differing requirements for displayed options.  
- For concierge data, consider defining correctness as the single best response rather than any of the top five.  
- A more realistic loss for deployed assistants would weight API errors, turns to completion, and rejected options, but requires real users or realistic simulators.  
- The paper does not fully address vocabulary mismatch between user language and KB terms; ML should learn associations rather than rely on human-defined type mappings.

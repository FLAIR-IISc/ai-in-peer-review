1. Paper provides a necessary reality check on chatbot capabilities and is highly significant for Dialog/NLP and ML venues.  
2. Conjecture 1: End-to-end ML performance remains insufficient for goal-oriented dialog under the used metrics.  
3. Conjecture 2: Relative algorithm performance on well-aligned synthetic data can predict performance on natural data.  
4. Main algorithmic contribution: a simple match-type addition to memory networks, first tested on goal-oriented dialog.  
5. Experimental protocol and exposition are excellent and accessible; concise appendix and tables (e.g., hops impact) are particularly strong.  
6. Current dialog metrics are brittle and sometimes arbitrary, limiting meaningful evaluation.  
7. Per-response accuracy is problematic—doesn't capture importance of correct API call or dialog brevity (Table 3 example).  
8. Per-dialog accuracy is highly sensitive to protocol/task definitions, explaining counterintuitive results (e.g., T3 vs T5).  
9. Evaluation ambiguity: concierge data scoring (top-5 vs single-best correct) could substantially change conclusions.  
10. Proposes a more meaningful loss for goal-oriented dialog tied to revenue: weighted API errors, dialog length, and user rejections—requiring real users or realistic simulators.  
11. Vocabulary mismatch between user language and KB is under-addressed; match-type relies on exact KB terms instead of learned associations.

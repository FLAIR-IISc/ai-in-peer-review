Summary
The paper introduces an open, reproducible testbed for evaluating end-to-end, goal-oriented dialog systems. Within a restaurant reservation scenario, it defines five synthetic tasks that isolate key capabilities (issuing and updating API calls, reading and using KB outputs, sorting options, handling OOV entities), plus two evaluations on real data (DSTC2 converted into the task format, and a proprietary Concierge dataset). The authors compare several baselines—TF–IDF, nearest-neighbor IR, supervised embeddings, and Memory Networks (with and without match-type features). They report that Memory Networks achieve promising per-response accuracy, learn non-trivial operations (e.g., API calls, some OOV handling with match types), outperform a hand-crafted rule-based baseline on DSTC2, but still exhibit low per-dialog accuracy, indicating remaining gaps for reliable goal completion.

Decision: Reject (as-is)
Key reasons:
1) Missing strong non-neural baseline with word-order information in the main evaluation, which undermines the fairness of the comparison and may overstate the advantage of neural models.
2) The synthetic tasks (1–5) are fully deterministic and do not probe crucial aspects of real dialog such as uncertainty, ambiguity, and repair strategies.

Supporting arguments
- Baselines and fairness of evaluation: The current non-neural baselines are bag-of-words TF–IDF and nearest neighbor. Without a competitive word-order-aware benchmark, the evaluation likely overestimates the gains from representation learning in the proposed neural models. A simple, well-regularized non-neural classifier that incorporates word order (e.g., bigrams) and entity typing could plausibly be competitive with, or even surpass, some of the neural systems on these deterministic tasks. This is especially pertinent given the relatively small training sets; with limited data, performance gaps between neural architectures can reflect regularization choices more than modeling advances.
- Determinism and task scope: Tasks 1–5 are designed to be completely predictable (and solvable at 100% with rules), which is useful for controlled analysis but does not evaluate a system’s ability to handle noise, ambiguity, or user behavior variation. Important dialog competencies—inferring distributions over user goals, managing ASR/SLU errors, and performing repair—are not measured here. As a result, it is unclear how performance on these tasks translates to more realistic, uncertain scenarios.
- Claims and positioning: The paper argues that existing work lacks well-defined performance measures. This is not entirely accurate. End-to-end task-oriented systems do have clear metrics (e.g., Wen et al., “A Network-based End-to-End Trainable Task-oriented Dialogue System”), and non-task-oriented settings can be evaluated with human studies (e.g., Liu et al., 2016, for Twitter; see also Yu et al., “Strategy and Policy Learning for Non-Task-Oriented Conversational Systems”). The testbed is valuable, but the framing should be more precise.

Additional feedback to improve the paper (not primary to the decision)
- Add a word-order-aware non-neural baseline: To fairly assess the value of representation learning, include a strong, simple benchmark such as logistic regression or linear SVM over:
  1) averaged pre-trained or supervised word embeddings (as in the Supervised Embeddings model),
  2) bigram features (to capture basic word order),
  3) match-type features (to handle entities and OOVs).
  Reporting this baseline would materially strengthen the empirical claims. If included, I would raise my rating (as noted below, I have updated my score based on new results).
- Clarify generalization beyond determinism: While the DSTC2 and Concierge results help, please discuss how performance on the synthetic tasks predicts robustness to noisy inputs and non-deterministic user behavior. For example, can you simulate noise (paraphrases, typos, slot confusions) to quantify brittleness?
- Error analysis and per-dialog failures: Since per-dialog accuracy remains low, a deeper qualitative error analysis would be helpful. Where do models fail most often (e.g., ranking options after KB calls, state updates after corrections, handling multi-constraint changes)? Which errors directly cause goal failure?
- Data scale and sample efficiency: Given the small training sets, discuss the sensitivity of each model (especially neural) to data size and regularization. Are improvements over IR baselines robust as training size varies?
- Evaluation claim: Reword the conclusion regarding “no well-defined measures of performance” to acknowledge prior work in task-oriented evaluation and human evaluation for chit-chat, and to clarify how your testbed complements these efforts.

Questions for the authors
- Baselines:
  • Did you evaluate a simple linear model with bigrams and match-type features? If not, can you add it? How close does it get to Memory Networks on deterministic tasks and OOV settings?
  • For supervised embeddings, did incorporating bigrams or constrained n-grams materially change results across tasks (including DSTC2/Concierge)?
- OOV handling:
  • How sensitive are results to the specific OOV split (cuisine/location halves)? Did you try alternative partitions or larger KBs?
  • Can match-type features be extended to capture entity spans and canonicalization errors (e.g., minor misspellings)?
- Robustness:
  • How do models perform with injected noise (paraphrases, slot confusions, ASR-like errors)? Are there preliminary results on noisy synthetic variants?
- Reporting:
  • For per-dialog failures, what proportion stems from a single critical mistake vs. cumulative minor errors?
  • How does candidate set size influence ranking performance and error modes?

Post-rebuttal note: I have updated my score in light of the new results added in the paper. Including a strong non-neural, word-order-aware baseline materially strengthens the empirical case.

- Introduces an open, reproducible testbed for end-to-end goal-oriented dialog focused on restaurant reservation, with five synthetic subtasks plus a full-dialog task and evaluations on DSTC2 and a concierge dataset.
- Benchmark targets state tracking without explicit annotations, KB interaction, handling unseen entities (OOV), and dialog completion.
- Compares baselines (IR, supervised embeddings, rule-based) and end-to-end Memory Networks, and introduces match-type features for better entity handling.
- Data, tasks, and methodology are clearly described and valuable to the community.
- Decision: Accept.
- Key reason: provides a well-motivated public benchmark with clear task decomposition and OOV protocols, filling a reproducibility gap for goal-oriented dialog.
- Key reason: thorough empirical study that illuminates strengths and limitations of end-to-end models, with results carrying over to real datasets.
- Task decomposition enables targeted error analysis (dialog management vs. KB interaction vs. OOV handling).
- Synthetic deterministic tasks are intentionally solvable by rules to diagnose ML methods; inclusion of DSTC2 and concierge datasets adds realism.
- Match-type features are a sensible, effective design to mitigate entity/OOV issues in embedding-based models.
- Results show MemNNs learn API issuance/updates and benefit from multi-hop attention; match-type features improve OOV and KB-answering (Task 4).
- MemNNs and other methods struggle on option ranking (Task 3) and full-dialog success (Task 5), highlighting KB interpretation and end-to-end completion challenges.
- The same ranking of methods holds across synthetic and real datasets, supporting the synthetic tasks as useful proxies.
- Suggestion: strengthen non-neural baselines with word-order features (e.g., averaged embeddings + bigrams + match-type) and present those results prominently.
- Suggestion: add noisy variants (paraphrases, slot retractions, ASR-like perturbations, off-topic turns) to probe robustness and dialog repair.
- Suggestion: complement next-utterance ranking with generation-based assessment to test generalization beyond fixed candidate pools.
- Suggestion: provide finer-grained error breakdowns for Tasks 3 and 5, ablate match-type features by entity type, and report sensitivity to hops and candidate set size.
- Clarification: soften the claim about “no well-defined measures” and acknowledge existing task-oriented metrics and evaluation protocols.
- Reproducibility request: confirm data/code license, provide exact candidate set sizes per task, and ensure no leakage across tasks/splits.
- Question: what are exact candidate set sizes per task (including OOV) and chance performance; how sensitive are results to enlarging candidate pools with paraphrases?
- Question: how often are OOV tokens matched via type features versus memorized embeddings?
- Question: do match-type features rely on exact string matching only, and how would they handle typos, morphology, ASR errors, or multi-token entities?
- Question: in Task 3, where do MemNNs fail most—rating reading, sorting stability, or decision to stop offering options—and were differentiable sorting or pointer mechanisms tried?
- Question: how do results scale with larger KBs, longer dialogs, more slot types, or different bot policies (e.g., party-size distributions)?
- Question: please report variability across runs and statistical significance for key comparisons (e.g., MemNN with/without match-type).
- Question: were sequence models (GRU/LSTM with attention) or key–value memory variants for KB interactions tried, and how do they compare?

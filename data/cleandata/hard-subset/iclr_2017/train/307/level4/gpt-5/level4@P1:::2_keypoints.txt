- Introduces a reproducible restaurant-reservation testbed decomposed into five subtasks and validated on DSTC2 and Concierge real datasets.  
- Evaluates Memory Networks (with/without a simple match-type feature) against strong baselines (rule-based, IR, supervised embeddings) under IV and OOV conditions.  
- Shows end-to-end models can learn nontrivial behaviors (issuing/updating API calls, some OOV handling) but struggle with interpreting API results and achieving robust per-dialog success.  
- Paper is clearly written with an excellent experimental protocol and a helpful Memory Network appendix explaining multi-hop attention.  
- Decision: Accept.  
- Significance: Provides a timely, rigorous, reproducible yardstick for goal-oriented dialog research.  
- Careful evaluation: Synthetic tasks mirror essential capabilities and relative performance generalizes to real data when conditions match.  
- Main contribution is experimental framework and comparative evaluation rather than algorithmic novelty (match-type is a small addition).  
- Empirical conjecture 1: End-to-end ML approaches remain insufficient for reliable goal-oriented dialog, especially per-dialog success.  
- Empirical conjecture 2: Relative performance on well-matched synthetic tasks predicts relative performance on natural data.  
- Error-analysis-by-design: The five subtasks isolate core competencies; MemNNs excel at API issuing/updating (T1–T2) and improve with match types for KB use/OOV (T4) but fail at sorting/displaying options and full-dialog success (T3, T5).  
- Real-data validation: Relative performance patterns on synthetic tasks carry over to DSTC2 and Concierge, supporting the testbed's realism.  
- Evaluation feedback: Per-response accuracy can be misleading; per-dialog accuracy is sensitive to task definitions; recommend reporting top-1/top-k/MRR and proposing goal-focused utility metrics combining slot errors, turns, and rejected options.  
- Entity handling feedback: Match-type relies on exact KB matches; suggest discussing/experimenting with learned mappings, NER/EL, synonym expansion, or fuzzy matching and quantifying degradation with KB surface-form mismatches.  
- Questions for authors: Provide quantitative synthetic-to-natural correlation (e.g., rank correlation) across methods.  
- Questions for authors: Ablate sensitivity of T3/T5 to number of required options, acceptance probabilities, and near-ties.  
- Questions for authors: Specify which ranking metrics were used for Concierge (top-1 vs top-k, MRR) and report stricter evaluations.  
- Questions for authors: Analyze sensitivity to candidate set size/composition and negative sampling, and effects of dialog-state constrained candidates.  
- Questions for authors: Report experiments (or plans) with alias tables/learned entity-linking and integration of match-type features.  
- Questions for authors: Did you try pointer/copy mechanisms or key-value memory variants to better ground KB facts for T3/T4?

SYNOPSIS:
This paper presents a new dataset for evaluating end-to-end goal-oriented dialog systems. All dialogs are set in the restaurant domain, where the objective is to check availability and ultimately book a table based on user-specified parameters (e.g., cuisine, price range). The data are generated by simulating interactions using an underlying knowledge base to sample parameter tuples, followed by rule-based transformations to render natural-language utterances. The task is to rank a set of candidate responses at each dialog turn, with evaluation reported via per-response accuracy and per-dialog accuracy. The authors show that Memory Networks outperform simple bag-of-words baselines.

THOUGHTS:
Thank you for an interesting contribution. That said, I remain skeptical about the utility of end-to-end training in narrow domains. In open domains, hand-coding all states and responses is infeasible, so end-to-end methods are compelling. But in narrow domains, we typically have deep domain knowledge and aim for high user satisfaction. In such cases, doesn’t it make more sense to leverage domain expertise to engineer the best system rather than rely solely on end-to-end training?

Given the restricted domain, I am also disappointed that the task is to rank rather than generate responses, though I understand this simplifies evaluation. I am uncertain how the candidate responses are obtained in practice. As stated (see the last sentence before Sec. 3.2), the models appear to rank the set of all responses from train/validation/test. If a central argument for end-to-end methods is easy portability to new domains without manual re-engineering, where would this candidate set come from for a new domain? Generative models would plausibly generalize better to unseen domains than systems that rank a hand-collected list of generic responses. In my view, this reliance on ranking is the weakest aspect of the work.

Finally, because the data are produced by expanding (cuisine, price, etc.) tuples via rule-based natural-language generation, the variability of training utterances is inherently constrained, even if the simulator allows unlimited data. I could not find the list of rules used; publishing them would improve transparency and reproducibility.

Overall, despite these reservations, this is an interesting and useful contribution that merits acceptance to the conference.

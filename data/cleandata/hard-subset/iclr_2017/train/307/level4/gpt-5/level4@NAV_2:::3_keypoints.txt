- New evaluation resource for end-to-end, goal-oriented dialog in a restaurant-booking domain using simulated users, a KB, and rule-based NL templates.
- Task formulation: next-response selection where models rank a fixed set of candidate system utterances (including API calls) and are evaluated with per-response and per-dialog accuracy.
- Baselines: Memory Networks outperform bag-of-words and classic IR baselines on this setup.
- Decision: Accept.
- Key reasons for acceptance: provides a reusable, controlled testbed useful for comparability and diagnostics.
- Key reasons for acceptance: empirical study is careful and shows clear gains of an end-to-end model over strong, simple baselines under a well-defined protocol.
- Problem targeted: whether end-to-end models can perform transactional dialog capabilities (constraint tracking, issuing/updating API calls, using KB facts).
- Motivation/placement: examines transfer of end-to-end benefits from open-domain to narrow, slot-filling domains.
- Evidence: ranking setup allows unambiguous automatic evaluation; MemNets show consistent per-response and some per-dialog improvements.
- Suggestion: add generation-based experiments to test scalability and domain transfer beyond fixed candidate pools.
- Suggestion: clarify how candidate response sets would be obtained in practical deployments and their alignment with “minimal manual effort.”
- Suggestion: publish the full set of NL templates/rules or provide statistics on template coverage/variability to support reproducibility and robustness checks.
- Suggestion: discuss scenarios where hand-engineered, domain-specific systems may still be preferable (e.g., extreme reliability requirements).
- Questions for authors: how is the candidate response set constructed for new domains and real deployments?
- Questions for authors: any plans or preliminary results for generation (seq2seq or retrieval-augmented) to test generalization beyond fixed pools?
- Questions for authors: can you release templates or at least provide template coverage/variability statistics?
- Questions for authors: how do you see end-to-end models complementing or replacing hand-engineered systems in narrow domains with strict user-satisfaction targets?

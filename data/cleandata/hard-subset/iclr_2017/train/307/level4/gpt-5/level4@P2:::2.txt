Summary of contributions
- The paper introduces a clean, reproducible testbed for evaluating end-to-end dialog systems in goal-oriented settings, centered on restaurant reservation. The tasks decompose the problem into interpretable subtasks (dialog management, API calls, ranking options, using KB outputs, and handling OOV entities), plus two evaluations on realistic data (DSTC2 and a real concierge service).
- It provides a careful empirical study across rule-based systems, IR baselines, supervised embeddings, and Memory Networks, including a simple but effective “match type” extension for entity handling.
- The results show that end-to-end neural models can learn nontrivial operations (issuing API calls, updating them, and using KB outputs), but still struggle to achieve high per-dialog success in more complex tasks. The performance pattern on synthetic tasks mirrors that on DSTC2 and the concierge data, supporting the premise that these synthetic tasks are a useful proxy.
- The paper is clearly written, well-situated in the literature, and offers an exemplary experimental protocol. The appendix gives a concise, accessible overview of Memory Networks and ablations on number of hops.

Decision: Accept
Primary reasons:
1) The testbed and analysis fill a real gap: a reproducible, interpretable framework for goal-oriented dialog that bridges synthetic and real data. This is valuable to both ML and dialog communities.
2) The study offers a sober, evidence-based reality check on end-to-end methods in goal-directed settings and advances methodology (match type features) with clear takeaways.

Supporting arguments
- Problem and motivation: The paper tackles whether end-to-end systems trained on dialogs can replace traditional slot-filling in narrow domains, and which capabilities remain lacking. This is timely and well-motivated given the surge of interest in chatbots.
- Positioning: It complements prior “chit-chat” evaluations and bAbI-style QA, addressing dialog-management, KB interaction, and OOV entities—precisely where end-to-end models must excel for practical deployment.
- Methodology: The design of five controlled tasks plus two realistic datasets enables rigorous, comparable evaluation and built-in error analysis. The OOV split is particularly insightful.
- Evidence: Across tasks, Memory Networks outperform IR and supervised embeddings; match type features substantially improve OOV handling and KB lookup responses (e.g., T4). Yet per-dialog accuracy remains modest on harder tasks (especially T3/T5), highlighting current limitations. The relative performance ordering carries from synthetic tasks to DSTC2 and concierge data, a notable and useful finding.

Feedback to improve the paper (not required for acceptance)
- Metrics and success criteria:
  - Per-response accuracy (next-utterance classification) can be misleading for goal-oriented dialogs: correctness often hinges on producing the right API call with minimal detours, not verbatim matching of intermediary prompts. Per-dialog accuracy is informative but highly sensitive to task design (e.g., T3 requires 3 options; T5 requires only one), which complicates cross-task comparison.
  - Consider task-success metrics more aligned with application value, e.g., weighted penalties for API-call errors, turns to first correct API call, and number of rejected options before acceptance. Such a metric would better reflect real deployment objectives, even if it requires a simulator or human evaluation rather than canned transcripts.
  - For the concierge dataset, report top-k next-utterance classification (e.g., top-1 vs. top-5 vs. MRR) to quantify whether models place the correct response among a small shortlist, which is actionable for retrieval-based agents.
- Entity handling and vocabulary mismatch:
  - Match type features require exact string/type matches in the KB. In realistic settings, user vocabulary may not align with KB terminology (synonyms, abbreviations, obfuscations). Consider evaluating or discussing learned entity linking/canonicalization (e.g., synonym mapping, ontology alignment, or neural entity linking) to bridge KB–user vocabulary gaps without manual curation.
- Analysis depth:
  - Add finer-grained error analyses by capability (API call formation, result interpretation, sorting, stopping criteria) and by entity type. This would clarify where MemNNs fail in T3/T5.
  - Provide ablations for match type components (exact match only vs. types only vs. both) and interactions with number of hops, to isolate what drives gains.
  - Report sensitivity to candidate set size and retrieval bias in the ranking setup, especially for the concierge data.

Questions for the authors
1) Generalization and proxy validity: Beyond restaurants, have you piloted analogous tasks in other domains (e.g., travel, movie booking)? Do relative method rankings persist?
2) Ranking vs. generation: How sensitive are results to the size/content of the candidate set? Would models trained for generation (with constrained decoding) show similar per-dialog patterns?
3) T3/T5 difficulty: In T3, failures stem from reading and sorting KB outputs and deciding when to stop. Which specific operations cause the most errors? Could adding explicit supervision for sorting or pointer mechanisms help?
4) OOV handling: How are type features constructed for OOV entities at test time—do you assume access to the full KB string list? How would performance change if KB strings are noisy or partially missing?
5) DSTC2 noise: Which error sources (ASR errors vs. bot nondeterminism) most degrade performance in T6? Any evidence that increasing hops helps more under noise?
6) Concierge evaluation: Could you report top-k metrics and qualitative analyses of near-miss candidates to assess practical utility in retrieval-based assistants?
7) Mapping user terms to KB: Have you tried weakly supervised or embedding-based alignment to learn mappings between user surface forms and KB schema values (e.g., “NoMad” → “Flatiron,” “Ind.” → “Indian”)? If not, what obstacles did you foresee?

Overall, this is a valuable, well-executed study that should catalyze more rigorous, application-oriented evaluation for end-to-end goal-oriented dialog.

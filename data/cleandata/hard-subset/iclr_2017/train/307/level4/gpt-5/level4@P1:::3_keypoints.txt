- Introduces an open, reproducible testbed for end-to-end goal-oriented dialog with tasks in restaurant reservation covering dialog management, API calls, KB interpretation, and OOV handling.  
- Evaluation uses per-response and per-dialog accuracy in a ranking setup to enable clear comparisons and error analysis.  
- Benchmarks include rule-based, IR, supervised embeddings, and Memory Networks on synthetic tasks, DSTC2, and real concierge logs.  
- Memory Networks outperform IR and bag-of-words baselines and learn non-trivial ops (API calls, OOV handling with match-type features) but per-dialog success is limited on harder tasks.  
- Decision: Accept — testbed is useful, well-structured, openly available, and facilitates reproducibility and ablation studies.  
- Experiments are thorough across synthetic, semi-real, and real data, revealing strengths and limitations of current end-to-end approaches.  
- Problem framing is clear: tests whether end-to-end models for chit-chat can meet goal-directed dialog demands via a task decomposition (T1–T5) and OOV splits.  
- Work is well positioned in dialog literature, contrasting slot-filling/POMDP systems and adapting bAbI-like tasks to dialog.  
- Empirical results support claims: Memory Networks excel on API/OOV tasks but struggle on KB interpretation/sorting, aligning with conclusions about current limits.  
- The synthetic tasks’ performance patterns align with DSTC2 and concierge results, supporting external validity.  
- Testbed utility: modular tasks, OOV split, and ranking evaluation should catalyze progress through consistent comparisons.  
- Concern: evaluating only response ranking (not generation) limits applicability; authors should justify and discuss extensions to generation or hybrid approaches.  
- Concern: candidate set construction (union across splits) may not scale or generalize; request clarity on construction, size statistics, and practical maintenance.  
- Concern: simulator’s deterministic templates and policies limit linguistic variability and may bias models; request publication of NL templates/policies and experiments with increased variability.  
- Suggestion: add metrics like goal success rate, turns-to-success, and error-type analysis; recommend human evaluation on DSTC2/concierge subsets.  
- Positive: match-type features effectively improve OOV performance; ask about sensitivity to KB coverage, entity-linking errors, and combination with subword models.  
- Suggest adding traditional slot-filling or neural belief-tracking baselines for better contextualization in narrow domains.  
- Suggest discussing how to adapt the task suite and OOV/KP grounding to other domains (travel, retail) and required resources.  
- Clarification requests: exact candidate set definition per task/dataset and leakage prevention across splits.  
- Clarification requests: typical candidate set size at inference and performance dependence on pool size.  
- Clarification requests: release of full user/bot templates and dialog generation code.  
- Clarification requests: whether a generation track is planned and practical strategies for candidate construction in new domains.  
- Clarification requests: whether explicit reasoning modules (differentiable sorting, pointer mechanisms) were tried for T3/T5.  
- Clarification requests: analysis on DSTC2 separating ASR artifact errors from modeling limitations.  
- Clarification requests: model behavior under randomized bot question order or increased paraphrase variability.

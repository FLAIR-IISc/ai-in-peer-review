- Introduces a reproducible testbed for goal-oriented dialog (restaurant domain), decomposing goals into focused tasks and using synthetic, DSTC2, and Concierge datasets.  
- Evaluates baselines and an end-to-end Memory Network; proposes match type features to handle entities and OOV words.  
- Experimental protocol is careful: uses per-response and per-dialog accuracy, OOV stress-testing, synthetic vs. real comparisons, and hop-count analysis.  
- Writing is clear and accessible, with a concise, effective summary of Memory Networks and hop effects.  
- Decision: Accept — the paper delivers a high-quality, broadly useful testbed and a careful empirical study.  
- Significance: Provides a rigorous reality check for goal-directed dialog amid broad chatbot claims.  
- Central takeaways: (1) end-to-end methods are not yet sufficient for reliable goal-oriented dialog; (2) method rankings on well-crafted synthetic tasks can predict rankings on natural dialogs.  
- Methodological contribution: Match type features are a simple but impactful addition for entity handling and OOV generalization in MemNNs.  
- Experimental rigor: Task design enables granular error analysis; results are consistent across synthetic, DSTC2, and Concierge; MemNN outperforms an engineered rule-based system on DSTC2.  
- Metric insight: Standard metrics are brittle—per-response classification can hide task success, while per-dialog accuracy is sensitive to task design choices.  
- Suggestion: Use success-oriented metrics aligned with task utility (e.g., cost-weighted API field errors, turns to correct API call, rejected options).  
- Suggestion: Report top-k evaluation (MRR/Recall@k) on Concierge to better capture near-misses given varied operator responses.  
- Suggestion: Address vocabulary–KB mismatch by discussing or experimenting with learned mappings, synonymy, normalization, or ontology linking.  
- Suggestion: Make the two guiding conjectures explicit to strengthen the narrative and aid future comparisons.  
- Suggestion: Provide an error analysis showing where full-dialog failures accumulate (API fields vs option ordering vs attribute retrieval).  
- Question: Please report top-k measures (Recall@1/5/10, MRR) and how often the gold response appears just outside top-5.  
- Question: How sensitive is per-dialog accuracy to small relaxations (template variations counted correct or correct API call within bounded turns)?  
- Question: In T3/T5, how much failure is due to option sorting vs tracking proposed options vs stopping criteria; any isolating ablations?  
- Question: On OOV generalization, how much gains come from type features alone versus learned generalization; have you tried entity normalizers or synonym dictionaries?  
- Question: Since models access KBs at training time, clarify risk of implicit leakage (memorizing KB strings) and how fairness across methods was ensured.  
- Question: How robust are MemNN results to increased candidate set size and more diverse, non-deterministic bot language; do additional hops consistently help?  
- Question: Would a simulator that accepts multiple semantically appropriate responses (rather than a single canonical utterance) change per-dialog success rates?

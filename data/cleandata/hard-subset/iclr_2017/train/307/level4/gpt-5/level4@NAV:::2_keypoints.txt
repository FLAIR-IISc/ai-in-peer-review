- Paper offers a much-needed reality check on chatbot capabilities, relevant and significant for both Dialog/NLP and ML communities.  
- Two core conjectures: (1) end-to-end ML is still inadequate for goal-oriented dialogues; (2) relative performance on synthetic data can predict performance on natural data when conditions are aligned.  
- Algorithmic novelty is small (match type added to memory networks), but this is the first deployment/evaluation on goal-oriented dialogue with an excellent experimental protocol.  
- Paper is clear and accessible, with a useful appendix summarizing memory networks and effects of number of hops.  
- Current dialogue metrics are brittle and somewhat arbitrary, highlighting the need for better evaluation methods.  
- Per-response accuracy (next-utterance classification) can be misleadingly pessimistic because a correct API call can still yield low score if intermediate responses aren’t exact.  
- Per-dialog accuracy is highly sensitive to task definitions and experimental protocol (e.g., the T3 vs T5 discrepancy).  
- For concierge data, evaluation choice matters: treating “correct” as single-best vs top-5 could change results.  
- Proposed alternative metric: a revenue-style loss combining API-call errors, dialogue length, and user-rejected options — but this needs real users or realistic simulators.  
- Paper does not sufficiently handle vocabulary mismatch between user language and KB entries; match-type requires verbatim KB terms instead of learned associations.

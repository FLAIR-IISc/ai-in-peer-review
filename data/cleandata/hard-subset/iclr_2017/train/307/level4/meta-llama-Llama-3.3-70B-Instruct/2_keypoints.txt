- Timely reality check on chatbot claims, highlighting limitations of current systems for complex dialogs.  
- Highly significant for Dialog/NLP and ML communities, educating engineers and pushing for realistic validation.  
- Authors argue end-to-end ML remains insufficient for goal-oriented dialogs.  
- Relative algorithm performance on synthetic data can predict performance on natural data, contrary to past findings.  
- Careful effort to closely match synthetic and natural experimental conditions.  
- Algorithmic novelty is small (match type added to memory networks) but its application to goal-oriented dialog is novel.  
- Experimental protocol is rigorous and the paper is exceptionally clear and accessible.  
- Appendix and analysis (e.g., effect of number of hops) are valuable contributions.  
- Current metrics are brittle/arbitrary: per-response accuracy can be misleading (Table 3 example).  
- Per-dialog accuracy is sensitive to task definitions and protocol choices (T3 vs T5 discrepancy in Table 2).  
- For concierge data, evaluating “best” rather than fixed top-k responses could give more nuanced results.  
- Proposes developing more meaningful metrics (e.g., weighted loss combining API correctness, turns, rejected options), but these require real users or realistic simulators.  
- Paper overlooks representation-learning issues: vocabulary mismatches between users and KBs (e.g., coding “Indian” as cuisine) need learned associations, not exact string matches.

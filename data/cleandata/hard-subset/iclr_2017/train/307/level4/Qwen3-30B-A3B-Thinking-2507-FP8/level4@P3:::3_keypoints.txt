- Introduces a reproducible testbed of five synthetic tasks for end-to-end goal-oriented dialog in the restaurant domain.  
- Tasks target key dialog capabilities: issuing API calls, updating requests, displaying options, providing extra info, and full dialogs.  
- Testbed is grounded in a restaurant knowledge base and designed for easy reproducibility.  
- Memory Networks achieve strong per-response accuracy but fall short on per-dialog performance.  
- Results on synthetic tasks generalize to real-world datasets (DSTC2 and a concierge service).  
- Paper fills a gap by providing a standardized benchmark for end-to-end goal-oriented dialog evaluation.  
- Careful task decomposition enables targeted error analysis not possible with existing datasets.  
- Experimental setup is rigorous, using both per-response and per-dialog metrics to assess component and end-to-end performance.  
- Comparison across baselines (rule-based, IR, supervised embeddings, Memory Networks) is comprehensive and well contextualized.  
- Memory Networks benefit significantly from match type features, improving performance on several tasks.  
- Even best models struggle with interpreting API call results (notably Tasks 3 and 5), revealing current limitations.  
- Suggests discussing limitations of ranking vs. generation approaches and candidate set construction for open domains.  
- Requests clarification on how match type features interact with Memory Networks and details of NLG rules for synthetic data.  
- Asks authors how candidate responses would be obtained for new domains, why the rule-based DSTC2 baseline underperforms, and how to handle truly OOV entities without a predefined KB.



This paper introduces a novel testbed for systematically evaluating end-to-end goal-oriented dialog systems in a controlled, reproducible manner. The authors propose a set of five tasks grounded in restaurant reservation, where the goal is to manipulate natural language and symbolic knowledge base (KB) elements to successfully complete dialogues. The testbed enables error analysis by breaking down complex dialog capabilities into specific subtasks, including issuing API calls, updating requests, displaying options, providing extra information, and conducting full dialogs. The authors demonstrate that Memory Networks can achieve promising performance on these tasks, outperforming rule-based baselines on real-world datasets while revealing key limitations in handling dialog state tracking and knowledge interpretation.

I recommend acceptance with minor revisions. The paper makes a valuable contribution to the field by providing a reproducible framework for evaluating end-to-end dialog systems in goal-oriented settings, and the experimental results provide meaningful insights into the strengths and weaknesses of current approaches.

The paper addresses a genuine gap in the literature regarding standardized evaluation of end-to-end dialog systems in goal-oriented contexts. The authors successfully demonstrate that their testbed can be used to systematically analyze model capabilities, with results showing consistent patterns across synthetic tasks and real-world datasets from DSTC2 and an online concierge service. The comparative results with Memory Networks, supervised embeddings, and classical IR methods provide useful benchmarks for future work, and the OOV evaluation effectively highlights generalization challenges.

To improve the paper, please explicitly discuss the practical implications of the ranking-based evaluation approach. The paper should clarify how the candidate response set is constructed for new domains, addressing the concern that this method may not scale well to truly novel domains where candidate responses would need to be generated rather than ranked. Additionally, provide a more detailed description of the NL-generation rules used to create natural language utterances from the KB parameters, as this is critical for reproducibility and understanding the dataset's limitations.

1. How exactly would the candidate response set be constructed for a new domain not covered by the current testbed? Would the authors expect domain experts to manually curate candidate responses for every new domain, or is there a way to generate these candidates automatically?
2. Could you provide the complete list of NL-generation rules used to create natural language utterances from the KB parameters, as this would greatly enhance reproducibility and help readers understand the dataset's linguistic diversity?
3. The paper mentions that Memory Networks with match type features improve performance on OOV tasks, but the improvement is still limited. Could the authors elaborate on why this limitation persists and what specific aspects of the model would need to change to overcome it?

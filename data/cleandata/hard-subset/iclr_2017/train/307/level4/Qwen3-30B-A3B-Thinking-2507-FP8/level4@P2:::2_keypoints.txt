- Proposes a systematic, reproducible testbed of five KB-grounded restaurant reservation tasks to evaluate end-to-end goal-oriented dialog systems.  
- Targets capabilities like dialog management, API querying, and entity manipulation beyond chit-chat and slot-filling.  
- Shows Memory Networks achieve promising per-response accuracy but suffer low per-dialog accuracy, revealing significant weaknesses in dialog-level behavior.  
- Validates synthetic findings on two real-world datasets (DSTC2, concierge service), showing consistent performance patterns.  
- Decision: Accept — paper makes a significant contribution by providing a well-designed evaluation framework.  
- Addresses a critical literature gap: lack of standardized evaluation for end-to-end goal-oriented dialogs.  
- Well-motivated positioning that contrasts limitations of traditional slot-filling and prior end-to-end methods.  
- Rigorous experiments comparing rule-based, IR, supervised embeddings, and Memory Networks with appropriate metrics.  
- Demonstrates specific failure modes: models can issue API calls but struggle to interpret API results and manage multi-step dialog outcomes.  
- Suggests synthetic testbed is a good proxy for real-world performance, strengthening external validity.  
- Recommendation to clarify relationship between per-response and per-dialog accuracy, since metric sensitivity affects interpretation.  
- Suggests more concrete examples of how the "match type" feature helps/hurts specific tasks and discussion on extending the testbed to other domains.  
- Asks authors to propose a more consistent, real-world-reflective success metric beyond requiring every response to be perfect.  
- Requests guidance on modifications needed to adapt the testbed to diverse real-world domains (e.g., concierge language variability).  
- Encourages exploration of integrating match-type features with advanced techniques (entity linking, knowledge-graph embeddings) for OOV handling.

- Introduces a systematic testbed for evaluating end-to-end goal-oriented dialog systems, focused on restaurant reservation.
- Proposes five well-defined tasks grounded in a restaurant knowledge base to test API call handling, dialog management, KB interaction, and response generation.
- Demonstrates Memory Networks achieve promising but imperfect performance on these tasks.
- Validates synthetic testbed findings on real-world datasets (DSTC2 and an online concierge), showing the testbed is a good proxy for real performance.
- Recommends acceptance, citing the testbed's suitability for systematic evaluation of goal-oriented dialog systems.
- Recommends acceptance, citing rigorous experimental protocol and cross-dataset validation supporting the claims.
- Provides a reproducible framework enabling controlled evaluation of end-to-end dialog systems in goal-oriented settings.
- Shows end-to-end models can learn implicit dialog state tracking and basic API call operations but struggle to interpret KB results and present user options.
- Paper is clear and accessible, valuable to dialog and ML researchers interested in rigorous evaluation methods.
- Suggests clarifying the relationship between per-response and per-dialog accuracy metrics in the main text (Table 2 footnote insufficient).
- Suggests adding a brief discussion of limitations in generalizing the testbed to other goal-oriented domains.
- Requests clarification on how match type features help handle OOV entities and whether this extends to other entity types.
- Asks how the testbed would need modification to evaluate other domains (e.g., travel booking, customer support) and what generalization challenges might arise.

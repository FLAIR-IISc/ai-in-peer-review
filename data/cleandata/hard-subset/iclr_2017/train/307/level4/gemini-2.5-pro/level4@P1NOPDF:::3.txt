**SYNOPSIS**

This paper introduces a new dataset for evaluating end-to-end, goal-oriented dialogue systems in the restaurant domain. The task involves a user interacting with a bot to find and book a table based on specified constraints (e.g., cuisine, price range). The dataset is created via simulation, where a knowledge base is used to generate semantic tuples that are then converted into natural language utterances using a set of rule-based transformations. The proposed task is to rank a list of candidate responses at each turn of the dialogue. The authors evaluate performance using per-response and per-dialogue accuracy, demonstrating that Memory Networks outperform bag-of-words baselines.

**THOUGHTS**

This is an interesting and timely contribution, and I thank the authors for their work. However, I have some reservations regarding the overall approach and its practical utility, which I hope the authors can address.

My primary concern relates to the applicability of end-to-end training for narrow-domain dialogue. While end-to-end methods are well-justified in open-domain settings where manually engineering all states and responses is intractable, their advantage is less clear in narrow domains. In settings like restaurant booking, the domain is well-understood, and the primary objective is high user satisfaction. This raises the question of whether it would be more practical to leverage this explicit domain knowledge to engineer a more robust and reliable system.

A significant limitation of the current work is the choice to frame the task as ranking candidate responses rather than generating them. While this simplifies evaluation, it introduces practical challenges. The paper states that models rank the set of all responses found in the training, validation, and test sets. A crucial question is how this candidate set would be sourced in a real-world application, especially when scaling to a new domain. One of the main arguments for end-to-end systems is their scalability and reduced engineering effort. However, if a comprehensive list of candidate responses must be manually created for each new domain, this benefit is diminished. A generative approach would offer far better generalization and be more aligned with the goal of building easily adaptable dialogue systems. This reliance on a pre-existing response set is arguably the most significant weakness of the proposed framework.

Finally, regarding the data generation process, the use of a simulator to expand semantic tuples via natural language generation rules inevitably constrains the linguistic diversity of the dialogue. While this is a reasonable trade-off for the ability to generate a large-scale dataset, the specifics of the generation process are unclear. The paper does not appear to include the set of rules used for generation. Publishing these rules, perhaps in an appendix, would be valuable for reproducibility and for understanding the inherent biases and limitations of the dataset.

Overall, despite these reservations, the paper introduces a valuable resource and benchmark for the community. I believe it is an interesting contribution worthy of publication at the conference.

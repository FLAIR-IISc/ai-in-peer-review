- Introduces a testbed for restaurant reservation dialogs, decomposing the task into five sub-problems to probe API call generation, state updates, and KB integration.
- Evaluates a Memory Network model against baselines on both simulated and real-world datasets (DSTC2, Concierge), revealing current limitations of end-to-end approaches.
- Primary contribution is an open, reproducible resource and analysis framework rather than a novel algorithm.
- Recommendation: accept.
- Strength — Significance and timeliness: shifts focus from open-domain chit-chat to practical goal-oriented dialog challenges and provides a reality check on model capabilities.
- Strength — High-quality experimental framework: public testbed and task decomposition enable detailed error analysis and guide future research (e.g., KB result interpretation/presentation).
- Rigorous experimental protocol with appropriate baselines (IR to rule-based) and evidence that synthetic task performance predicts real-world performance.
- Paper is exceptionally well-written; appendix succinctly summarizes Memory Networks and hop analysis illuminates model reasoning.
- Critique — Evaluation metrics: per-response and per-dialog accuracy have significant limitations and can misrepresent task success; calls for more nuanced, task-oriented metrics (e.g., weighted loss reflecting API correctness, turns, rejections).
- Empirical observation: inconsistent scores across tasks (e.g., 0% on Task 3 vs. 19.7% on Task 5) illustrate metric sensitivity to task design.
- Critique — Robustness to vocabulary mismatch: the match-type feature relies on exact string matches, raising concerns about handling linguistic variation between user utterances and KB terminology without hand-crafted mappings.

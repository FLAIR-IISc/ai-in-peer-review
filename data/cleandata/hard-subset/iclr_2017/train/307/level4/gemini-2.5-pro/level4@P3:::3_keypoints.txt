- Introduces a new public dataset and task suite for evaluating end-to-end goal-oriented dialog systems in the restaurant reservation domain.  
- Main contributions: (1) five simulated tasks breaking down dialog sub-problems, (2) an OOV test set for unseen-entity generalization, (3) evaluation of multiple baselines.  
- The five tasks isolate specific capabilities (e.g., issuing API calls, handling updates, using KB results) for fine-grained analysis.  
- Inclusion of an OOV split explicitly tests model generalization to unseen entities.  
- Baselines evaluated include IR methods, supervised embeddings, and Memory Networks, plus experiments on DSTC2 and a real-world concierge dataset.  
- Memory Networks outperform simpler baselines but still struggle with complex reasoning, highlighting research gaps.  
- Decision: Accept — the benchmark is well-designed and fills a critical need.  
- Reason 1 for accept: Thoughtful benchmark design and OOV split enable detailed model analysis lacking in the field.  
- Reason 2 for accept: Solid experimental baselines provide clear insights into strengths and weaknesses of end-to-end models.  
- Strength: Public release addresses the lack of rigorous evaluation resources for end-to-end goal-oriented dialog.  
- Strength: Task decomposition and real-data grounding allow precise error diagnosis and validation of synthetic findings.  
- Weakness: Framing the task as ranking from a predefined candidate set rather than generation is a major concern.  
- Practicality concern: A complete candidate list is unrealistic in new domains and would require manual curation, reintroducing handcrafting.  
- Generalization concern: Ranking limits assessment of a model’s true generative and generalization abilities.  
- Tension noted: Rule-based system achieves perfect accuracy on simulated tasks, raising questions about the practical need for end-to-end approaches in narrow, well-understood domains.  
- Reproducibility suggestion: Release the user simulator code or detailed NLG patterns to allow others to reproduce and extend the data/simulation.  
- Questions for authors requested: justify ranking vs generation and explain how candidate sets would be built in new domains; clarify the practical niche for end-to-end models given handcrafted systems’ performance.

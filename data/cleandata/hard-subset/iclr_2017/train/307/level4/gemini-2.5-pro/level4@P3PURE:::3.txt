**Summary**

This paper introduces a new dataset for evaluating end-to-end goal-oriented dialog systems. The dataset is synthetically generated for the restaurant booking domain, where the system's goal is to find availability and book a table based on user-specified constraints (e.g., cuisine, price range). The data generation process involves using a simulator and an underlying knowledge base to create structured dialog states, which are then rendered into natural language using rule-based transformations. The proposed task is to rank a set of candidate responses at each turn of the dialog, with evaluation performed using per-response and per-dialog accuracy. Experiments demonstrate that Memory Networks can outperform simpler bag-of-words baselines on this task.

**Critique**

The authors are commended for this interesting contribution and for releasing a new resource to the community. However, this work raises some important questions about the utility of end-to-end trained systems in narrow-domain settings. For open-domain dialog, there is a compelling argument that hand-coding all possible states and responses is intractable, making end-to-end learning a necessity. In contrast, for narrow domains like restaurant booking, the domain is well-understood. In such cases, it is often more practical to leverage this domain knowledge to engineer a system optimized for high user satisfaction.

A significant limitation of the proposed task is its formulation as a ranking problem rather than a generation problem, a choice presumably made to simplify evaluation. This leads to a critical question regarding practical application: how is the set of candidate responses obtained? The paper indicates that the models rank the set of all responses present in the training, validation, and test sets. This approach is not scalable and undermines a key motivation for end-to-end systems: the ability to easily adapt to new domains without extensive manual engineering. If a new domain requires a manually curated list of all possible responses, the benefit of the end-to-end approach is greatly diminished. A generative model would offer far better generalization to new domains than one that simply ranks a pre-existing, domain-specific list of responses. This reliance on a pre-defined candidate set is arguably the most significant weakness of the current work.

Finally, the data generation process, while enabling the creation of a large-scale dataset, introduces certain limitations. Since the data is generated by expanding structured tuples (e.g., cuisine, price) using a fixed set of natural language generation rules, the linguistic diversity of the resulting dialogs is inherently constrained. While this is a reasonable trade-off for the ability to generate unlimited data, the specifics of this process are not fully transparent. For reproducibility and to allow the community to better understand this trade-off, it would be beneficial for the authors to publish the set of rules used in the simulation.

Despite these concerns, this paper presents a valuable contribution. The introduction of a new, large-scale dataset and the baseline results are of interest to the community and merit publication.

This paper serves as a serious reality check on the 2016 trend of applying chatbots to all forms of human-computer interaction, which has been accompanied by claims that they can solve many types of dialogue beyond simple chit-chat. While highly relevant for Dialog and Natural Language Processing venues—to educate software engineers on the limitations of current chatbots—it would also be a valuable contribution to Machine Learning venues, as it highlights the need for more realistic validation of ML applied to dialogue systems. For these reasons, I consider this work to be highly significant.

Two important conjectures, likely to inspire further research, underlie this paper. While not stated explicitly in the text, they were articulated by Antoine Bordes during a NIPS workshop presentation covering this work. Based on the metrics and results presented, these conjectures are:
1) The performance of end-to-end ML approaches is still insufficient for goal-oriented dialogue.
2) When comparing algorithms, relative performance on synthetic data is a good predictor of performance on natural data. This latter point marks a departure from previous observations, but the authors make a commendable effort to align the synthetic and natural data conditions.

While the paper's original algorithmic contribution consists of a relatively simple addition to memory networks (the "match type" feature), this work represents the first time these models have been deployed and tested on a goal-oriented dialogue task. The experimental protocol is excellent. Furthermore, the paper's clarity is outstanding, making it accessible to a readership beyond ML and dialogue researchers. I was particularly impressed by the concise and effective summary of memory networks in the appendix, as well as the tables clarifying the influence of the number of hops.

Although this paper represents the state-of-the-art in the exploration of more rigorous metrics for dialogue modeling, it also highlights how brittle and somewhat arbitrary these metrics can be. (Note: This critique is intended as a suggestion for future research rather than a requirement for revision.)

First, the authors use per-response accuracy, which measures the classification of the next utterance from a fixed list of responses. Table 3 illustrates the practical limitations of this metric. In a successful dialogue, the critical outcome is a correct API call within a reasonable number of turns. However, under this metric, such a dialogue might achieve a score as low as 1/7 simply because the six preceding bot responses did not exactly match the ground truth, even if they successfully led to the correct API call.

The per-dialogue accuracy, which requires all responses to be correct, is also shown to be problematic. As Table 2 demonstrates, this metric is highly sensitive to the experimental protocol. For example, it is initially puzzling that the accuracy for subtask T3 is 0.0%, while the accuracy for the full dialogue (T5) is 19.7%. This counterintuitive result is explained by the task definitions (Section 3.1.1): T3 requires displaying three options, making it a more difficult task than T5, which only requires one. A similar question arises from the concierge data: how would performance change if 'correct' was defined as the single best response, rather than being among the top five?

While the authors cannot be faulted for using standard dialogue metrics—and indeed for proposing new ones, even if they appear overly pessimistic—it is worth considering alternatives that could yield more meaningful metrics for goal-oriented dialogue. Consider a business-oriented metric: if a virtual assistant service is paid upon successful task completion, what metric would maximize revenue? For the restaurant-booking task, a suitable loss function might be a weighted sum of errors in the final API call, the number of turns required to reach it, and the number of options rejected by the user. However, such a loss function cannot be measured on static, pre-recorded dialogues and would require interaction with either a real human user or a realistic user simulator.

Finally, another issue related to representation learning that the paper does not sufficiently address is the challenge of vocabulary mismatch between the user's utterance and the knowledge base (KB). Specifically, the "match type" feature appears to rely on a word like 'Indian' occurring verbatim in the KB to be coded as a 'type of cuisine'. In real-world scenarios, a KB might use specialized or non-intuitive terminology. An ideal system should leverage machine learning to learn the association between user vocabulary and KB terms, rather than relying on exact matches or manual annotations.

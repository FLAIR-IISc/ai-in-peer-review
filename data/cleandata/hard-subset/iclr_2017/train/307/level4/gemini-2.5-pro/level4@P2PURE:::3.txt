**SYNOPSIS:**

This paper introduces a new dataset for evaluating end-to-end, goal-oriented dialog systems in the restaurant booking domain. The task involves finding and booking a table based on user-specified constraints. The dataset is constructed via simulation, where a knowledge base is used to sample parameters (e.g., cuisine, price range) which are then rendered into natural language utterances using rule-based transformations. The proposed task is to rank a set of candidate responses for each turn in a dialog. The authors report per-response and per-dialog accuracy, demonstrating that Memory Networks outperform bag-of-words baselines on this task.

**THOUGHTS:**

I thank the authors for their interesting contribution. However, I am skeptical about the utility of end-to-end trained systems for narrow-domain settings. While end-to-end methods are well-suited for open-domain dialog where hand-crafting states and responses is intractable, narrow domains are typically well-understood. In such cases, where the primary goal is high user satisfaction, it seems more practical to leverage this domain knowledge to engineer an optimal system.

A significant limitation of this work is the choice to frame the task as ranking responses rather than generating them, given the restricted domain. While this approach simplifies evaluation, it raises questions about practical application. Specifically, it is unclear how the set of candidate responses would be sourced. The paper suggests that the models rank the set of all responses present in the training, validation, and test sets. This reliance on a pre-defined candidate set undermines a key motivation for end-to-end systems: the ease of scaling to new domains without extensive manual re-engineering. It is not clear how this candidate list would be obtained for a new domain. A generative approach would offer far better generalization, and the decision to focus on ranking is, in my view, the weakest aspect of this work.

Finally, the data generation process, which expands attribute tuples using natural language generation rules, necessarily limits the linguistic variability of the training data. This is an understandable trade-off for the ability to generate a large volume of data via simulation. However, the paper does not include the list of rules used for this generation. Publishing these rules would significantly improve the reproducibility of the work.

Overall, despite these concerns, I believe this is an interesting contribution worthy of publication at the conference.

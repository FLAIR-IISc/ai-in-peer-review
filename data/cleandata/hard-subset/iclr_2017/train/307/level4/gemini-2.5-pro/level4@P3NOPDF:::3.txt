### Review

**Summary**

This paper introduces a new dataset and benchmark task for evaluating end-to-end goal-oriented dialog systems. The task is set in the restaurant booking domain, where an agent must assist a user in finding and booking a table based on constraints like cuisine, price range, and location. The dataset is synthesized by simulating dialogs against an underlying knowledge base and then rendering system and user turns into natural language using a set of rule-based transformations. The proposed task is to rank a set of candidate responses for each turn in a dialog, with evaluation performed using per-response and per-dialog accuracy. The authors establish several baselines, including a bag-of-words model, and demonstrate that Memory Networks can achieve superior performance on this task.

**Strengths**

1.  **New Public Dataset and Benchmark:** The primary contribution of this work is the creation and release of a new, well-defined task and dataset for goal-oriented dialog. This provides a valuable benchmark for the community, allowing for standardized evaluation and comparison of end-to-end models in this space.
2.  **Solid Experimental Setup:** The paper provides a clear experimental methodology, including reasonable baselines and the application of a more sophisticated model (Memory Networks). This offers a solid starting point for future research to build upon.

**Weaknesses and Major Concerns**

1.  **Motivation for End-to-End Systems in Narrow Domains:** My main reservation concerns the fundamental motivation for using a purely end-to-end trained system in such a narrow and well-defined domain. In open-domain dialog, the argument for end-to-end learning is compelling, as hand-crafting the vast number of possible states and responses is intractable. However, for a constrained domain like restaurant booking, the state space is well-understood and manageable. In such settings, where high user satisfaction and task success are paramount, it is not clear that an end-to-end approach is superior to a more traditional, engineered system that can leverage domain knowledge for greater control, reliability, and debuggability. The paper could be strengthened by a more thorough discussion of the trade-offs and a clearer justification for why an end-to-end approach is preferable in this context.

2.  **Ranking vs. Generation:** A significant weakness of the proposed task is the choice to frame it as ranking candidate responses rather than generating them. While this choice greatly simplifies evaluation, it raises serious questions about the practical application and scalability of the developed models. A key argument for end-to-end systems is their potential for easy adaptation to new domains. However, the paper states that the candidate set is composed of all unique responses present in the training, validation, and test sets. This raises a critical question: how would this candidate set be sourced for a new domain? If it requires manually collecting or generating a comprehensive list of all possible responses, it significantly undermines the "low-engineering" scalability argument that motivates end-to-end approaches. A generation-based approach would offer more plausible generalization to new entities and domains, making it a more compelling demonstration of the power of these systems. This framing as a ranking task is, in my view, the weakest aspect of the work.

3.  **Limited Diversity of Synthetic Data:** The dataset is generated synthetically by expanding structured tuples (e.g., cuisine, price) into natural language via a simulator with fixed rules. While this method allows for the creation of a large corpus, it also inherently constrains the linguistic diversity and variability of the dialogs. This may not reflect the full range of expressions found in real user interactions. To better allow the community to understand the properties and potential biases of the dataset, it would be beneficial for the authors to publish the set of natural language generation rules used in the simulation.

**Recommendation**

Despite the significant conceptual concerns raised above, particularly regarding the ranking-based task formulation and its implications for scalability, I believe this paper is a valuable contribution to the community. The release of a new public dataset and a clear evaluation benchmark for goal-oriented dialog is a positive and needed step for the field. The work provides a useful testbed for future research, even if the specific framing of the problem has notable limitations.

I thank the authors for their detailed response, which has clarified some of my questions. Following these clarifications and the inclusion of new results in the revision, I have updated my score and recommend acceptance.

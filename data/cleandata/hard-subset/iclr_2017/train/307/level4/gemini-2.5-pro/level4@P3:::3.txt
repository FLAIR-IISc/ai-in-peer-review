### Summary of Contributions

This paper introduces a valuable new public dataset and a suite of tasks for evaluating end-to-end goal-oriented dialog systems. The authors correctly identify that while end-to-end models have shown promise in open-domain chit-chat, their capabilities in structured, goal-oriented settings are less understood, and existing datasets are often unsuitable for end-to-end training. To address this, the paper presents a testbed in the restaurant reservation domain. The contribution is threefold: 1) A set of five simulated tasks that systematically break down the dialog process into sub-problems like issuing API calls, handling updates, and presenting options from a knowledge base (KB). 2) The inclusion of an out-of-vocabulary (OOV) test set to specifically measure generalization to unseen entities. 3) An evaluation of several baselines, including classical IR methods, supervised embeddings, and Memory Networks, on these tasks as well as on converted data from the DSTC2 challenge and a real-world concierge service. The results demonstrate that Memory Networks outperform simpler baselines but still struggle with complex reasoning, highlighting key areas for future research.

### Decision and Key Reasons

**Decision: Accept**

This paper provides a well-designed and much-needed resource for the community. My decision is based on two key reasons:

1.  The work introduces a valuable and thoughtfully constructed benchmark for a critical research area. The breakdown into distinct sub-tasks and the inclusion of an OOV split provide a framework for detailed model analysis that is currently lacking in the field of goal-oriented dialog.
2.  The experimental evaluation provides a solid set of baselines and offers clear insights into the current strengths and weaknesses of end-to-end models, successfully demonstrating the utility of the proposed testbed.

### Supporting Arguments

**Strengths:**
The primary strength of this paper is the public release of a high-quality dataset and testbed. The motivation is clear: traditional slot-filling systems are brittle and do not scale easily, while end-to-end systems, despite their potential, lack rigorous evaluation benchmarks in the goal-oriented setting. This work directly fills that gap.

The design of the five simulated tasks is particularly insightful. By isolating specific capabilities (e.g., Task 1: issuing API calls; Task 4: using information from KB results), the authors enable researchers to perform fine-grained error analysis and pinpoint exactly where a given model fails. This is a significant improvement over evaluating on a single, monolithic task where the causes of failure can be difficult to disentangle. Furthermore, grounding the experiments with results on real data from DSTC2 and the Concierge service helps validate that the patterns observed on the synthetic data are relevant to real-world challenges.

**Weaknesses / Concerns:**
My main reservation concerns the choice to frame the core task as ranking from a pre-defined set of candidates, rather than response generation. The paper states that the candidate set is composed of all bot utterances from the combined training, validation, and test sets. This setup has two significant drawbacks:

1.  **Practicality:** In a real-world deployment for a new domain, this exhaustive list of possible responses would not be available beforehand. It would need to be manually curated, which re-introduces the kind of domain-specific handcrafting that end-to-end systems are meant to avoid.
2.  **Generalization:** This approach limits the assessment of a model's true generative and generalization capabilities. A key promise of end-to-end dialog is the ability to formulate novel responses. By constraining the model to a fixed list, the benchmark cannot measure this crucial aspect.

Additionally, while I understand the paper's goal is to analyze end-to-end systems, there is a tension in the motivation for applying them to narrow-domain tasks. As the authors’ rule-based system achieves perfect accuracy on the simulated tasks, it raises the question of whether the complexity of end-to-end learning is justified in settings where domain knowledge is readily available and high reliability is paramount. The paper would be strengthened by a more direct discussion of this trade-off.

Despite these concerns, the value of the dataset as a tool for analysis and comparison outweighs these limitations. The work provides a solid foundation that the community can build upon.

### Additional Feedback and Questions

The following points are intended to help improve the paper and are not central to my decision.

*   **Reproducibility of Simulation:** The data is generated via a simulation based on patterns and a knowledge base. To maximize the utility and reproducibility of this work, it would be highly beneficial to release the code for the user simulator, or at least provide a detailed description of the natural language generation patterns used. This would allow other researchers to generate more data or adapt the simulation to new domains.
*   **Questions for the Authors:**
    1.  Could you elaborate on the decision to formulate the task as ranking rather than generation? How do you envision the candidate set being constructed in a practical deployment for a completely new domain, where a pre-existing corpus of all possible system responses is unavailable?
    2.  Given that a hand-crafted system can achieve 100% accuracy on your simulated tasks, what is your perspective on the practical niche for end-to-end models in such well-defined, narrow-domain applications where user satisfaction is critical and the domain rules are well understood? Are they primarily a tool for analysis, or do you see a path for them to outperform engineered systems in reliability?

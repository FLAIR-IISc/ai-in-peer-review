This paper presents a timely and crucial contribution by systematically evaluating the capabilities of end-to-end dialog systems in goal-oriented settings. The authors introduce a well-designed testbed for restaurant reservations, breaking down the complex task into five distinct sub-problems that probe specific system abilities like API call generation, state updates, and knowledge base integration. By testing a Memory Network-based model against several baselines on both simulated and real-world datasets, the paper provides a sobering but valuable analysis of the current state of the art. The primary contribution is not a novel algorithm, but rather the creation of an open resource that facilitates reproducible research and provides a clear framework for analyzing the strengths and weaknesses of goal-oriented dialog models.

I recommend accepting this paper for publication. My decision is based on two key strengths:
1.  **Significance and Timeliness:** The work serves as an essential reality check on the performance of current end-to-end models, moving the focus from open-domain chit-chat to the more structured, practical challenges of goal-oriented dialog.
2.  **High-Quality Experimental Framework:** The paper introduces an excellent and much-needed public testbed. The decomposition into sub-tasks is insightful, enabling detailed error analysis and guiding future research toward specific weaknesses, such as interpreting and presenting KB query results.

The paper’s core arguments are well-supported by a rigorous experimental protocol. The choice of baselines, from simple IR methods to a rule-based system, provides necessary context for evaluating the performance of the Memory Network model. The inclusion of both synthetic and real-world data (DSTC2, Concierge) strengthens the findings, and the authors make a compelling case that relative performance on their synthetic tasks is a good predictor of performance on real data. This suggests the testbed is a valuable proxy for more complex, noisy environments. Furthermore, the paper is exceptionally well-written and clear. The appendix provides a concise and effective summary of Memory Networks, and the analysis of how performance varies with the number of "hops" offers valuable insight into the model's reasoning process.

The following points are intended as constructive feedback to help improve the paper and stimulate future research; they are not central to my decision to accept.

*   **Critique of Evaluation Metrics:** The paper relies on standard metrics like per-response and per-dialog accuracy, but also highlights their limitations. Per-response accuracy can be misleading, as it weights all turns equally; a correct greeting is counted the same as a correct, task-critical API call. A successful dialog with seven turns might achieve only 14% accuracy if only the API call is considered truly essential. Conversely, per-dialog accuracy seems overly harsh and sensitive to task design. For instance, the model scores 0% on Task 3 but 19.7% on the more complex Task 5, a discrepancy the authors explain is due to a difference in the number of required options to be presented. It would be beneficial for future work to explore more nuanced, task-oriented metrics. One could envision a loss function that more directly reflects task success, perhaps a weighted sum of errors in the final API call parameters, the number of turns taken, and the number of user rejections.
*   **Robustness to Vocabulary Mismatch:** The proposed "match type" feature is a simple and effective way to handle out-of-vocabulary entities. However, it appears to rely on exact string matching between the user's utterance and entities in the knowledge base. A question for the authors would be: how would the system perform if the KB vocabulary did not perfectly align with the user's vocabulary (e.g., the user requests a "cheap" restaurant, but the KB only contains the term "moderate")? This points to a broader research challenge in learning to associate linguistic variations with canonical KB entities without explicit hand-coding.

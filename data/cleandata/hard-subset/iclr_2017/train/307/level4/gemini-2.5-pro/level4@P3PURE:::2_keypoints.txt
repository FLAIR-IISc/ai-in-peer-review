- Paper is an important reality check on applying chatbots to diverse HCI tasks, relevant and significant for Dialog/NLP and ML communities.  
- Two key conjectures: (1) end-to-end ML is still insufficient for goal-oriented dialogue; (2) relative performance on synthetic data predicts performance on natural data.  
- The main algorithmic change is a simple "match type" addition to memory networks; this is the first deployment/testing of these models on goal-oriented dialogue.  
- Experimental protocol and paper clarity are excellent, with a useful memory network summary and clear analysis of hop effects.  
- Current dialogue metrics are brittle and can be arbitrary, limiting meaningful evaluation.  
- Per-response accuracy is misleading because it penalizes dialogues that reach correct API outcomes despite differing intermediate responses.  
- Per-dialogue accuracy is highly sensitive to task definitions and protocol (illustrated by T3 vs T5 and concierge top-1 vs top-5 issues).  
- More meaningful, application-driven metrics (e.g., weighted loss on final API correctness, turns taken, options rejected) are needed but require interactive evaluation or realistic simulators.  
- Representation learning shortcomings: "match type" relies on exact KB term matches, ignoring vocabulary mismatch; models should learn associations between user language and KB terminology rather than use exact matches or manual annotations.

**Summary**

This paper presents a new dataset for evaluating end-to-end, goal-oriented dialog systems. The dataset is situated in the restaurant booking domain, where an agent must assist a user in finding and booking a table based on constraints like cuisine and price range. The data is synthetically generated through a simulation that uses an underlying knowledge base to create task-specific parameter tuples (e.g., cuisine, location, party size). These tuples are then converted into natural language utterances using a set of rule-based transformations.

The primary task is formulated as ranking a set of candidate responses for each turn in the dialog. Evaluation is performed using per-response and per-dialog accuracy metrics. The authors demonstrate that Memory Networks outperform simpler bag-of-words baselines on this task.

**Review and Critique**

While this is an interesting and valuable contribution, the paper raises some fundamental questions about the utility of end-to-end trained systems in narrow-domain settings. The motivation for end-to-end systems is particularly strong in open-domain dialog, where manually engineering states and responses is intractable. However, in a narrow, well-defined domain like restaurant booking, the argument is less clear. For such tasks, where the primary goal is maximizing task success and user satisfaction, leveraging domain knowledge to engineer a robust, high-performance system is often a more practical approach. It would be beneficial for the authors to further articulate the advantages of an end-to-end approach in this specific context over a more traditional, modular system.

A significant concern is the choice to frame the task as ranking rather than generation. While this simplifies evaluation, it raises practical questions about the system's real-world applicability and scalability. The paper states that the models rank the set of all responses present in the dataset. This leads to a critical question: how would this set of candidate responses be sourced for a new domain? A core promise of end-to-end training is the ease of porting a system to new domains with minimal re-engineering. However, if each new domain requires a manually curated list of all possible system responses, this benefit is significantly diminished. A generative model, while harder to evaluate, would offer a more scalable and generalizable solution, better aligning with the stated goals of end-to-end dialog systems. This reliance on a pre-existing candidate set appears to be a major limitation of the current framework.

Finally, the reliance on a simulator with rule-based natural language generation warrants discussion. While this approach allows for the creation of a large-scale dataset, it inherently limits the linguistic diversity of the dialogs. The variability is constrained by the predefined generation rules. To better assess the complexity and diversity of the dataset, and to facilitate reproducibility, it would be highly beneficial for the authors to include or publish the set of rules used by the simulator.

In summary, despite the aforementioned concerns regarding the problem formulation and the applicability of end-to-end methods in this context, the paper introduces a useful new dataset and provides valuable baseline results. It is an interesting contribution that stimulates important discussion in the field and is worthy of publication.

- Introduces an open-source testbed of five synthetic restaurant-reservation tasks to evaluate end-to-end, goal-oriented dialog systems.  
- Validates the testbed on two real-world datasets (DSTC2 and a proprietary concierge dataset).  
- Benchmarks several models (IR baselines, supervised embeddings, Memory Networks) and analyzes their relative strengths and weaknesses.  
- Decision: Accept.  
- Addresses a critical need for standardized, lightweight, reproducible testbeds for goal-oriented dialogue benchmarking.  
- Enables fine-grained analysis by decomposing dialog into distinct sub-tasks, allowing diagnosis of specific model failures.  
- Task design probes abilities like issuing/updating API calls and integrating KB outputs (Tasks 1–5).  
- Experiments show Memory Networks are promising but struggle with reasoning over KB query results, especially on Tasks 3 and 5.  
- Inclusion of OOV test sets and match-type features helps evaluate generalization to unseen entities.  
- Memory Network outperforms rule-based systems on noisy DSTC2, highlighting end-to-end approaches’ potential on realistic data.  
- Suggests adding stronger non-neural baselines (e.g., logistic regression with n-gram and match-type features) to better contextualize neural gains.  
- Notes limitation: deterministic simulations lack noise, ambiguity, and dialogue-repair dynamics of real interactions; DSTC2/Concierge only partially mitigate this.  
- Recommends clarifying the conclusion about existing performance measures, distinguishing chit-chat metrics from task-oriented objective metrics.  
- Asks authors to comment on expected performance of strong feature-engineered baselines and on extending the testbed to model user ambiguity, noise, and dialogue repair.

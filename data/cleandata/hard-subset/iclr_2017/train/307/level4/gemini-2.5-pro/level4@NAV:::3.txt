SYNOPSIS:
This work presents a novel dataset designed for the evaluation of end-to-end, goal-oriented dialogue systems. The dataset is situated in the restaurant domain, with the objective of finding and booking a table according to user-specified constraints (e.g., cuisine, price range). The data is synthetically generated through a simulation that leverages an underlying knowledge base to create parameter samples, which are then converted into natural language utterances via rule-based transformations. The core task is defined as ranking a list of candidate responses for each subsequent turn in the dialogue, and performance is measured using per-response and per-dialogue accuracy. The authors demonstrate that Memory Networks outperform basic bag-of-words baselines on this task.

THOUGHTS:
The authors are commended for this interesting contribution. Nevertheless, I have reservations regarding the applicability of purely end-to-end trained systems in such a constrained domain. While the argument for end-to-end methods is strong in open-domain settings where manual engineering is intractable, their advantage is less clear in narrow domains. In settings like this, where domain knowledge is well-understood, it seems more practical to leverage this knowledge directly to engineer an optimal system aimed at maximizing user satisfaction.

Given the restricted domain, it is also somewhat disappointing that the task is framed as response ranking instead of generation, though I acknowledge this choice simplifies the evaluation process. A key question is how the candidate response set would be procured in a real-world application. The paper suggests that models rank from a global pool of all responses present in the training, validation, and test sets. This is particularly problematic given that a primary motivation for end-to-end systems is their scalability to new domains without extensive re-engineering. It is unclear how this candidate list would be sourced for a new domain. A generative approach would offer superior generalization capabilities and, in my view, its absence represents the most significant weakness of this work.

Finally, the reliance on a simulation that expands tuples (e.g., cuisine, price) using natural language generation rules inevitably limits the linguistic diversity of the resulting dataset. This is an understandable trade-off for the benefit of generating a large-scale corpus. However, the specific rules used for generation were not provided. For transparency and reproducibility, it would be beneficial to publish this information.

In summary, while I maintain some reservations about the overall approach, the paper is a valuable contribution to the field and is suitable for publication at this conference.

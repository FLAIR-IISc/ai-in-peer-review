- Paper is a timely, high-significance reality check on overextending chatbots for all HCI, relevant to Dialogue/NLP and ML venues.  
- Two implicit but important conjectures: (1) end-to-end ML methods are still inadequate for goal-oriented dialogues; (2) relative algorithm performance on well-aligned synthetic data can predict performance on natural data.  
- Introduces a simple yet novel match-type mechanism added to memory networks and is the first rigorous deployment/testing of such networks in goal-oriented dialogue.  
- Experimental protocol is exemplary and the paper is exceptionally clear and accessible (notably the concise appendix and hop-impact tables).  
- Advances more rigorous dialogue metrics but also demonstrates those metrics are brittle and somewhat arbitrary, pointing to a research need rather than a fatal flaw.  
- Per-response accuracy is impractical for goal-oriented tasks (e.g., a correct API call can be masked by many required exact intermediate responses).  
- Per-dialog accuracy might be better but is highly sensitive to task definitions and experimental protocol (explains T3 vs T5 discrepancy).  
- Requests clarification/evaluation: for the concierge dataset, how do results change if "correct" means selecting the single best response instead of being among top five?  
- Proposes alternative, utility-driven metrics (e.g., revenue-maximizing loss combining API correctness, turns to completion, user rejections) which require real users or realistic simulators.  
- Notes an unaddressed representation-learning issue: vocabulary mismatch between user language and KB (match-type relies on exact token matches).  
- Recommends ML systems should learn KB-to-user-word associations automatically rather than depending on manual annotations or exact string matches.

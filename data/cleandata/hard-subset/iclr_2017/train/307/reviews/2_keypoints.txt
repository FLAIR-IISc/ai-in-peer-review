- Paper provides a valuable reality check: current chatbots and end-to-end ML are insufficient for goal-oriented dialogs and the work is significant for both Dialog/NLP and ML communities.
- Two key conjectures: (1) end-to-end ML performance is still inadequate for goal-oriented dialogs; (2) relative performance on well-matched synthetic data predicts performance on natural data.
- Main algorithmic contribution is a simple "match type" addition to memory networks, but it's the first thorough deployment and test on goal-oriented dialog tasks.
- Experimental protocol and paper clarity are excellent, with a helpful appendix and informative tables (e.g., impact of number of hops).
- Common dialog metrics are brittle and sometimes arbitrary; per-response accuracy can be misleading because it over-penalizes intermediate responses despite only API call correctness mattering.
- Per-dialog accuracy is highly sensitive to experimental protocol, illustrated by the counterintuitive T3 vs T5 results driven by task definitions.
- Question raised about evaluation on concierge data: should "correct" mean the single best response rather than being among the 5-best?
- Suggests more meaningful, application-driven loss functions (e.g., weighted sum of API errors, dialog turns, user rejections) tied to real-world revenue, which would require human users or realistic simulators.
- Paper does not adequately address vocabulary mismatch between user language and KB terms; match type needs exact KB occurrences and doesn't learn semantic associations automatically.

- Introduces a benchmark for goal-oriented dialog (restaurant reservation) with five KB-simulated tasks testing API calls, info retrieval, and dialog management.  
- Evaluates on DSTC2 and a proprietary concierge dataset and compares a Memory Network to a hand-crafted slot-filling baseline and other methods.  
- Finds end-to-end models promising but substantially behind traditional methods in achieving full dialog success; Memory Network shows low per-dialog accuracy.  
- Recommendation: Reject — testbed is useful but results are not substantial enough for acceptance at a top conference.  
- Principal reason: limited novelty and impact — paper mainly demonstrates a testbed rather than advancing state-of-the-art; improvements are incremental.  
- Approach seen as relatively simple: uses established Memory Networks and match-type features without exploring more advanced architectures.  
- Requests more detailed ablation studies to determine which aspects of match-type features drive gains (entity type matching vs. OOV handling).  
- Requests more systematic error analysis with concrete examples, especially failures in interpreting KB results.  
- Recommends comparison to stronger baselines (e.g., transformer-based models) for a more comprehensive evaluation.  
- Suggests discussing scalability and strategies for extending the approach to more complex domains.  
- Urges releasing code and datasets to improve reproducibility and impact.  
- Asks specific author questions on Memory Network limitations and alternatives; why match-type features help; noise handling in concierge dataset; hyperparameter sensitivity; and adding realistic simulation complexities.

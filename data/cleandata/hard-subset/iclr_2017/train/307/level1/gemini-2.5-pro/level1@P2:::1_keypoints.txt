- Introduces a public dataset of five synthetic restaurant reservation tasks to test specific dialog capabilities (API calls, user updates, KB interpretation, full conversation).  
- Proposes an OOV setting to evaluate model generalization to unseen entities crucial for real-world scalability.  
- Provides thorough empirical evaluation comparing IR baselines, supervised embeddings, and end-to-end Memory Networks (MemNN).  
- Validates the synthetic tasks by showing preserved relative model rankings on DSTC2 and a proprietary Concierge dataset.  
- Finds MemNNs, especially with match type features for entities, are promising but still fail to achieve perfect dialog completion.  
- Decision: Accept, motivated by the benchmark’s community value and rigorous experiments.  
- The bAbI-style task decomposition is a strong methodological choice that helps pinpoint specific model failures.  
- The OOV test is a well-executed feature addressing practical scaling challenges.  
- Using both per-response and per-dialog accuracy reveals system brittleness where one error can break an entire dialog.  
- Match type features are a simple, effective engineering fix that improves entity handling in embedding models.  
- Suggest discussing how the benchmark could be adapted for generative models and how to evaluate them (e.g., BLEU or semantic checks).  
- Recommend comparing MemNNs to a well-tuned sequence-to-sequence with attention baseline for broader context.  
- Ask for an estimate of human effort required to build the DSTC2 rule-based baseline to contextualize trade-offs.  
- Propose testing a variant with randomized questioning order to assess robustness to dialog path variability.  
- Question whether models had access to the OOV KB entity list during training and how that affects interpretation of OOV results.  
- Ask what architectural changes (external module, attention variant) might help end-to-end models perform structured operations like sorting KB results (Task 3).  
- Request a hypothesis for the slight performance drop on Task 2 when using match type features (98.6% vs. 99.4%).

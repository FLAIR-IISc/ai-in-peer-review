- Introduces a valuable, open-source testbed for evaluating end-to-end goal-oriented dialog systems.  
- Correctly notes that chit-chat success doesn't guarantee performance in transactional dialogs requiring KB queries and dialog state management.  
- Proposes five synthetic restaurant-reservation tasks (T1–T5) to systematically diagnose capabilities like API calls, OOV handling, and using KB results.  
- Provides a thorough empirical analysis benchmarking a Memory Network against several baselines.  
- Validates findings on two real-world datasets (DSTC2 and a concierge service), showing synthetic tasks track real-world patterns.  
- Results show modern end-to-end models are promising but still fail to reliably complete full dialogs.  
- Strong recommendation to accept based on providing a well-designed diagnostic tool that fills an evaluation gap.  
- Experimental analysis is rigorous and insightful, highlighting specific strengths and critical weaknesses of current approaches.  
- Main strength: structured, fine-grained evaluation by breaking goal-oriented dialog into subtasks for detailed error analysis.  
- Inclusion of both per-response and per-dialog accuracy metrics reveals that single-turn success does not ensure dialog-level completion.  
- OOV experiments and the introduction of "match type features" address a practical deployment challenge and strengthen the analysis.  
- Validation on DSTC2 and Concierge datasets supports that synthetic tasks are a good proxy for real-world performance.  
- Suggested improvement: clarify the size of the candidate set used for ranking bot utterances and API calls.  
- Suggested improvement: discuss generalizability of match type features to domains beyond restaurant reservation.  
- Questions for authors: how sensitive are models (particularly MemNN) to increased linguistic variation in the user simulator?  
- Questions for authors: what types of errors does the rule-based system make on noisy DSTC2 that the learned MemNN overcomes?

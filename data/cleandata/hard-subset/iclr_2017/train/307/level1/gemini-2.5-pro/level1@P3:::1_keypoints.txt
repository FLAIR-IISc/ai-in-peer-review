- Paper tackles rigorous evaluation of end-to-end goal-oriented dialog, addressing the lack of reproducible, diagnostic benchmarks.  
- Main contribution: open-source restaurant-reservation testbed with five bAbI-style simulated tasks probing specific dialog capabilities.  
- Testbed includes an Out-Of-Vocabulary (OOV) setting to evaluate generalization to unseen entities.  
- Real-world grounding: converted DSTC2 and an online Concierge dataset into the framework.  
- Empirical evaluation spans IR baselines, supervised embeddings, and end-to-end Memory Networks (MemNN).  
- Introduces a simple "match type feature" to help embedding models handle OOV entities.  
- Results show MemNNs are promising and can learn implicit state-tracking but still fail to solve the full task.  
- Key finding: high per-response accuracy contrasts with very low per-dialog accuracy, revealing model brittleness.  
- Synthetic tasks are diagnostic: Task 1 tests API calls/state tracking, Task 2 tests updates, Tasks 3–4 test KB integration and result reasoning.  
- OOV test sets force models to generalize rather than memorize entities.  
- Match type features are practical and effective, demonstrating KB-assisted handling of unseen entities.  
- Performance trends are consistent across synthetic and real datasets, validating the synthetic testbed as a proxy for real-world performance.  
- Paper is well-written and likely to become a standard benchmark resource for the field.  
- Feedback: user simulator’s fixed NL patterns limit linguistic variability; discuss this limitation and mitigation via DSTC2/Concierge.  
- Feedback: ranking-over-candidates evaluation simplifies the task and masks generation challenges; acknowledge limitations of a generative setting.  
- Feedback: consider deeper integration of match type features (e.g., separate entity embeddings or attention modulation) with MemNN reasoning.  
- Question: request qualitative error analysis comparing MemNN vs rule-based system on DSTC2—where each excels or fails.  
- Question: explain why MemNN performance on Task 2 drops when adding match type features.  
- Question: clarify Concierge evaluation—how API calls are judged correct when predicted “without any argument.”  
- Question: comment on sensitivity to using up to 1000 time features and the importance of fine-grained temporal encoding.

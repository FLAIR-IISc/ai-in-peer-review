- Introduces a new public, reproducible dataset and diagnostic tasks for evaluating end-to-end goal-oriented dialog systems.  
- Proposes five synthetic restaurant-reservation tasks of increasing complexity testing API calls, user updates, API result interpretation, and KB information retrieval.  
- Validates synthetic tasks against two real-world datasets (DSTC2 conversion and a Concierge dataset) to demonstrate practical relevance.  
- Evaluates several baselines (IR, supervised embeddings, Memory Networks) and reports comprehensive per-response and per-dialog metrics.  
- Memory Networks with a proposed "match type feature" perform best among learning models but still fail on more complex tasks.  
- Highlights a key weakness of current end-to-end models: reasoning over knowledge returned from API calls.  
- Decision: Accept.  
- Major reason: significant resource contribution filling a gap in standardized benchmarks for goal-oriented dialog.  
- Major reason: thorough, rigorous experimental evaluation with strong baselines and validation on real data.  
- Task decomposition (bAbI-style) is effective for diagnostic error analysis and isolating model deficiencies.  
- OOV test set and "match type feature" address generalization to unseen entities and show clear performance gains.  
- Consistent performance patterns between synthetic and real datasets support the testbed’s validity as a proxy for real challenges.  
- Suggestion: provide a more detailed error analysis for full-dialog Task 5 to identify which sub-task causes failures.  
- Suggestion: include an ablation study on match type features (matching input vs. memory vs. both) to clarify its effect.  
- Suggestion: discuss implications of using all bot utterances as the candidate set versus dynamic or larger real-world candidate generation.  
- Question: how are match type features applied to multi-word entities—by exact full-string matching or otherwise?  
- Question: does Memory Network attention indicate any ability to learn sorting/comparison needed for Task 3, or does it fail fundamentally?  
- Question: what were the main challenges in converting noisy DSTC2 and Concierge interactions into structured API calls and KB results?

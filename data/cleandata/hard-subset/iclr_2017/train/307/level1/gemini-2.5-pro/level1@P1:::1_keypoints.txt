- Introduces a new public testbed for goal-oriented dialog in the restaurant domain with five synthetic diagnostic tasks.  
- Tasks isolate issuing API calls, updating requests, presenting KB options, providing extra info, and full-dialog behavior.  
- Dataset includes an OOV test set to measure generalization to unseen entities.  
- Thorough experimental evaluation benchmarking IR, supervised embeddings, and Memory Networks (MemNNs).  
- MemNNs outperform other learning-based baselines but still struggle on complex tasks, with low per-dialog accuracy.  
- Introduces "match type features" that substantially improve MemNN handling of OOV entities.  
- Validates synthetic testbed by showing consistent model performance patterns on converted DSTC2 and a proprietary Concierge dataset.  
- Primary contribution is a well-designed, reproducible dataset and evaluation framework that fills a community need.  
- Empirical analysis diagnoses specific MemNN weaknesses (e.g., interpreting and presenting KB results) and strengths (e.g., state tracking for API calls).  
- OOV split is a crucial element pushing evaluation beyond pattern matching toward real generalization.  
- Match type features are a simple, practical contribution for handling entity generalization.  
- Recommendation: Accept.  
- Suggest discussing limitations of ranking-based evaluation and how the framework might be extended to generative models.  
- Suggest addressing the template-based user simulator limitation and exploring more diverse, learned, or adversarial simulators.  
- Suggest adding a brief qualitative analysis of multi-hop MemNN attention in the main text to illustrate how hops contribute to decisions.  
- Question: Why does Nearest Neighbor IR outperform TF-IDF here—was turn-by-turn vocabulary overlap measured versus chit-chat datasets?  
- Question: Were alternative entity-handling methods tried (e.g., special entity markers or separate entity memory) to clarify whether benefits come from type identification or exact string matching?

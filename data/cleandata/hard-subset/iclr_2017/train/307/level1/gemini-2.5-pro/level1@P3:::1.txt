### 1. Summary of Contributions

This paper addresses a crucial and timely problem: how to rigorously evaluate and analyze end-to-end models for goal-oriented dialog. While recent neural network models have shown promise in open-domain "chit-chat," their applicability to structured, task-driven conversations remains an open question. The authors argue that a primary obstacle to progress is the lack of suitable public benchmarks that are reproducible, diagnostic, and representative of the challenges in this domain.

The main contribution is a new, open-source testbed for goal-oriented dialog, framed in the context of restaurant reservations. Inspired by the bAbI tasks for question answering, the authors propose a suite of five simulated tasks designed to probe specific capabilities of a dialog agent, such as issuing API calls, handling user-driven updates, and utilizing knowledge base (KB) query results. Crucially, the testbed includes an Out-Of-Vocabulary (OOV) setting to test generalization to unseen entities, a common real-world challenge. The authors supplement these simulated tasks by converting data from the Dialog State Tracking Challenge 2 (DSTC2) and a real-world online concierge service into their framework, grounding their work in realistic conversational data.

To demonstrate the utility of their testbed, the paper provides a thorough empirical evaluation of several models, ranging from classical IR baselines to supervised embedding models and, most notably, an end-to-end Memory Network (MemNN). They also introduce a simple yet effective "match type feature" mechanism to help embedding-based models handle OOV entities. The results provide a clear and insightful snapshot of the current state of the art: while MemNNs are promising and can learn non-trivial behaviors like implicit state tracking, they still fall short of solving the full task, with per-dialog accuracy remaining a significant challenge. This work provides the community with both a valuable resource and a clear set of challenges for future research.

### 2. Decision and Key Reasons

**Decision:** Accept

**Key Reasons:**

1.  **Significant Community Contribution:** The paper introduces a well-designed, publicly available dataset and benchmark that fills a critical void in goal-oriented dialog research. The bAbI-style breakdown of tasks provides a much-needed framework for diagnostic evaluation and error analysis, enabling more focused and reproducible research on the limitations of current end-to-end models.
2.  **Insightful Experimental Analysis:** The paper presents a strong set of experiments that clearly highlight the strengths and weaknesses of current end-to-end approaches. The distinction between high per-response accuracy and very low per-dialog accuracy is a powerful illustration of the brittleness of these models. The findings—that models can learn to manage dialog state for queries but fail at reasoning over the returned results—provide a clear direction for future work.

### 3. Supporting Arguments for Decision

The primary strength of this paper is its contribution of a new resource that directly enables progress on an important problem. The motivation is compelling: the field needs to move beyond chit-chat and tackle goal-oriented dialog, but lacks the tools to do so systematically. The proposed dataset is thoughtfully constructed. The five sub-tasks are well-chosen to isolate key dialog capabilities: Task 1 (API calls) tests state tracking, Task 2 (updates) tests robustness to conversational turns, and Tasks 3 and 4 (displaying options, providing info) test the crucial ability to integrate and use external knowledge from a KB. This diagnostic design is a major step up from evaluating on monolithic, noisy dialog corpora.

The inclusion of OOV test sets is a particularly strong feature. It forces models to move beyond memorizing training set entities and learn more generalizable representations, a prerequisite for any real-world deployment. The proposed "match type features" are a simple, practical, and effective technique for addressing this challenge, demonstrating how models can leverage a KB to handle unseen entities.

The experimental evaluation is comprehensive and provides valuable baselines for future work. The comparison spans from non-learning IR methods to a state-of-the-art neural architecture (MemNN). The results convincingly support the paper's central claims. The superiority of MemNNs over simpler baselines demonstrates that sophisticated architectures are indeed necessary. However, the low per-dialog accuracy, especially on tasks requiring use of KB results (T3, T5), starkly illustrates the remaining challenges. The fact that the relative performance of the models is consistent across the synthetic tasks and the real-world DSTC2 and Concierge datasets validates the use of the synthetic testbed as a reliable proxy for real-world performance, strengthening its value as a research tool.

Finally, the paper is well-written and clear. The problem, proposed solution, and experimental findings are easy to follow. This work has the potential to become a standard benchmark in the field, much like the bAbI tasks did for QA.

### 4. Additional Feedback for Improvement

These points are intended as constructive suggestions and do not detract from my positive assessment of the paper.

*   **User Simulation Complexity:** The user simulator relies on a fixed set of natural language patterns. While this provides excellent experimental control, it may not fully capture the linguistic variability of real users. It would be beneficial to include a brief discussion on this limitation and how the inclusion of the DSTC2 and Concierge datasets helps to mitigate it.
*   **Ranking vs. Generation:** The evaluation is framed as a ranking task over a pre-defined set of candidate responses. This is a practical simplification but masks the significant challenge of natural language generation. A short paragraph in the conclusion acknowledging this and discussing the additional difficulties of a fully generative setting would provide a more complete picture of the path towards truly end-to-end systems.
*   **Integration of Match Type Features:** The match type features are appended to the bag-of-words representation. This is effective, but it feels somewhat disconnected from the core reasoning mechanism of the MemNN. Have the authors considered more deeply integrating this entity type information, for example, by creating separate entity embeddings or using the types to modulate the attention mechanism? This could be an interesting direction for future work.

### 5. Questions for the Authors

*   The MemNN outperforms the rule-based system on DSTC2 (Task 6), which is a significant result. Could you provide a brief qualitative analysis of the types of errors each system makes? For instance, where does the MemNN's ability to generalize from data surpass the hand-crafted rules, and in what situations do the hard-coded rules prove more robust?
*   In Table 2, the performance of the MemNN on Task 2 (Updating API calls) drops slightly when match type features are added (from 96.1% to 94.2%). This is counter-intuitive, as the features help on almost all other tasks. Do you have a hypothesis for this performance degradation?
*   Could you provide more detail on the evaluation for the Concierge dataset? The paper mentions that API calls must be predicted but "without any argument." How is a correct prediction defined in this case? Is the model simply predicting a special token like `api_call_opentable` from the candidate set?
*   The MemNN implementation uses up to 1000 "time features" to encode the position of an utterance in the dialog history. Could you comment on the sensitivity of the model to this hyperparameter? How important is this fine-grained temporal information for solving these tasks?

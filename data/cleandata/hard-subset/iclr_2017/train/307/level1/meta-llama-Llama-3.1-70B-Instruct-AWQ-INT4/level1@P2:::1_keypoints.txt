- Proposes a testbed to evaluate end-to-end goal-oriented dialog systems in restaurant reservation scenarios.  
- Designs five tasks requiring sentence/symbol manipulation, API calls, and use of API outputs.  
- Evaluates models on DSTC2 data and an online concierge service.  
- Aims to dissect strengths and shortcomings of end-to-end dialog systems in goal-oriented settings.  
- Reviewer accepts the paper for two reasons: a well-motivated, well-designed testbed and thorough evaluation against multiple models and datasets.  
- Thorough evaluation includes Memory Networks, IR models, supervised embeddings, and a hand-crafted slot-filling baseline.  
- Results: Memory Networks outperform other models on most tasks but still struggle with interpreting entity knowledge from API call returns.  
- Main weakness: lack of detailed error analysis and breakdown of error types.  
- Suggested improvements: more detail on Memory Network architecture and its adaptation to dialog tasks.  
- Suggested improvements: provide insights into why Memory Networks outperform others.  
- Suggested improvements: include a detailed error breakdown and discuss testbed limitations and possible extensions.  
- Questions for authors: clarify Memory Network adaptations, plans to fix entity interpretation issues from API calls, reasons for Memory Network superiority, and plans to extend the testbed.

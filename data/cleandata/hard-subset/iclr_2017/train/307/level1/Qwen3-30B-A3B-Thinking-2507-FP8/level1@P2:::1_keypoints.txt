- Introduces a systematic testbed for evaluating end-to-end goal-oriented dialog systems, centered on restaurant reservations.
- Proposes five tasks that require non-trivial operations: issuing API calls, updating API calls, displaying options, providing extra information, and conducting full dialogs.
- Provides synthetic datasets (with in-vocab and out-of-vocab splits) and real-world data from DSTC2 and an online concierge service.
- Demonstrates Memory Networks can learn to manipulate symbols and perform basic dialog operations but struggle on more complex tasks like interpreting API results to display options.
- Decision: Accept — the testbed addresses a significant gap with a well-motivated, rigorous, and reproducible evaluation framework.
- Strong experimental design: careful synthetic data construction, IV/OOV evaluation, multiple baselines (rule-based, IR, embeddings, Memory Networks), and clear metrics (per-response and per-dialog accuracy).
- Positions work well in literature by extending bAbI-style evaluation to goal-oriented dialog and addressing limitations of existing noisy or non-goal datasets.
- Real-world evaluations corroborate synthetic-task findings, indicating the testbed is a reasonable proxy for real performance.
- Recommendation: include a more detailed discussion of the testbed’s limitations and how well it generalizes beyond the restaurant domain.
- Recommendation: provide deeper error analysis of model failures, especially on Task 3 (displaying options).
- Recommendation: perform an ablation study to quantify how much improvement comes from match type features versus the Memory Network architecture.
- Questions for authors: clarify the specific challenges of Task 3 and possible remedies; justify generalizability of observed method rankings to other domains; explain whether Task 3’s design (sorting/presenting restaurants) fundamentally limits end-to-end models or can be overcome.

- Proposes a reproducible, lightweight synthetic testbed of five structured restaurant reservation tasks to evaluate end-to-end goal-oriented dialog systems.  
- Identifies a gap: end-to-end neural models work for chit-chat but are under-evaluated for goal-oriented tasks requiring API/KB interactions.  
- Designs tasks to break goal-oriented conversation into component subtasks (API call, API update, display options, provide info, full dialog) for error analysis.  
- Includes OOV scenarios to test generalization and uses per-response and per-dialog accuracy as evaluation metrics.  
- Validates the synthetic testbed by showing results correlate with real-world datasets (DSTC2 and an online concierge service).  
- Clarifies contribution: not a state-of-the-art restaurant system but a standardized benchmark for evaluating goal-oriented dialog models.  
- Decision: Accept — testbed meaningfully advances systematic evaluation and error analysis for conversational agents.  
- Methodology is rigorous with clear task rationale, reproducibility focus, and appropriate evaluation metrics.  
- Empirical comparison across rule-based, IR, supervised embeddings, and Memory Networks with ablations for match-type features.  
- Key empirical finding: Memory Networks can learn dialog state tracking and API-like symbol manipulation but struggle with interpreting KB results and displaying options.  
- Demonstrates limited OOV improvement from match-type features, highlighting remaining generalization challenges.  
- Suggests deeper error analysis to understand Memory Networks' failures on tasks like displaying options.  
- Notes limitation: the restaurant domain is constrained, so cross-domain generalization (e.g., travel, customer service) is unclear.  
- Recommends discussing integration with reinforcement learning frameworks for future work.  
- Asks authors to (1) compare specific error patterns between synthetic and real data, (2) explain fundamental limitations of current architectures and propose fixes, and (3) detail how error profiles differ between rule-based and neural models.

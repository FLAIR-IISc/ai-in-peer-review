- Introduces a valuable, controlled testbed for systematic evaluation of end-to-end goal-oriented dialog systems.  
- Proposes five well-designed subtask-oriented challenges in the restaurant reservation domain to decompose complex dialog capabilities.  
- Enables reproducible comparisons and granular error analysis through task decomposition.  
- Evaluates multiple model types (rule-based, IR, supervised embeddings, Memory Networks) across synthetic and real-world (DSTC2, concierge) datasets.  
- Reports both per-response and per-dialog accuracy metrics for comprehensive evaluation.  
- Recommends acceptance: fills a significant gap by offering a reproducible framework missing in end-to-end dialog research.  
- Provides evidence that end-to-end models can learn non-trivial operations (API calls, entity manipulation) but still fail at full dialog success, especially in interpreting KB results and presenting options.  
- Strengths: clear task definitions, thoughtful dataset construction with OOV challenges, and thorough multi-architecture evaluation.  
- Shows synthetic testbed is a valid proxy for real-world performance with consistent relative model rankings across datasets.  
- Analysis of model successes and failures yields important insights for the field.  
- Suggested improvement: provide more detail on how match type features enable OOV handling beyond asserting their importance.  
- Suggested improvement: strengthen the connection between synthetic tasks and specific real-world dialog challenges.  
- Questions for clarification: how to quantify difficulty differences between synthetic tasks and concierge data, especially variability in user requests?  
- Question: why do Tasks 3 and 5 remain hard even with match type features, and which task aspects are most problematic for current models?  
- Question: how should the community build on this evaluation-focused testbed to drive future progress in end-to-end goal-oriented dialog systems?

- Proposes a systematic testbed of five tasks for evaluating end-to-end goal-oriented dialog systems in a restaurant reservation domain.  
- Testbed uses a simulation to generate synthetic data and is validated on DSTC2 and an online concierge dataset.  
- Compares multiple approaches: rule-based systems, IR, supervised embeddings, and Memory Networks.  
- Main contribution: a reproducible evaluation framework addressing gaps in assessing goal-directed end-to-end dialog.  
- Decision: Accept — significant contribution with rigorous empirical validation.  
- Tasks are designed to isolate capabilities: issuing API calls, tracking state, handling OOV entities, updating requests, and presenting options.  
- Methodology is rigorous: defined KB, controlled synthetic generation (including OOV), appropriate metrics (per-response and per-dialog), and ablation studies.  
- Memory Networks can learn API calls and handle OOV entities but struggle to interpret KB results and present options (Task 3).  
- Results are consistent across synthetic and real-world datasets, supporting the testbed as a proxy for real performance.  
- Suggested improvements: detail simulation limitations, summarize Memory Network choices in main text, discuss extensions to other domains, and propose model-improvement directions.  
- Questions for authors: how to extend to complex/multi-domain scenarios; clarify Memory Network failure modes on Task 3 (architecture vs data); recommendations to improve low per-dialog accuracy.

- Proposes a well-designed synthetic testbed for evaluating end-to-end goal-oriented dialog systems.  
- Identifies a gap: end-to-end models are evaluated mainly on chit-chat, not goal-oriented tasks.  
- Defines five restaurant-domain tasks targeting API calls, request updates, option display, extra info, and full dialogs.  
- Tasks require handling natural language and symbolic KB manipulation.  
- Shows Memory Networks can learn non-trivial dialog operations but have notable shortcomings.  
- Validates synthetic findings on real datasets (DSTC2 and concierge) to confirm proxy relevance.  
- Releases dataset, code, and task definitions for reproducibility.  
- Reviewer recommends acceptance.  
- Paper fills a genuine need with a well-motivated, reproducible evaluation framework.  
- Experimental design is rigorous: controlled synthetic data, OOV test set, multiple baselines, and error analysis.  
- Key finding: Memory Networks track implicit dialog state and symbols but struggle to interpret API results for displaying options.  
- Observed patterns on synthetic data correlate with real-world performance, supporting benchmark validity.  
- Suggest adding discussion on adapting the testbed to other domains (e.g., travel, customer service).  
- Suggest elaborating how synthetic data may differ from real conversations and impacts on performance.  
- Recommend comparing to more recent end-to-end models (e.g., transformer-based architectures).  
- Question: Why do match type features improve apparent OOV performance by aiding known words but not actual OOV words?  
- Question: How were cases without API usage in the concierge dataset handled, and how does that variability affect learning?  
- Question: Why do match type features worsen performance on Task 2 (updating API calls), and is this specific to that task?

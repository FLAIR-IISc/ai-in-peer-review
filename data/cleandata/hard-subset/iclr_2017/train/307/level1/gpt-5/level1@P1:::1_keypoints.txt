- Introduces a lightweight, reproducible testbed for end-to-end goal-oriented restaurant dialog decomposed into five subtasks, plus DSTC2 conversion (Task 6) and a real concierge dataset.
- Designed to probe beyond chit-chat: implicit dialog state tracking, KB grounding, symbolic manipulation (sorting), and handling OOV entities via an OOV split.
- Provides extensive baselines: rule-based, TF-IDF, nearest-neighbor, supervised embeddings, and MemN2N with/without KB-aware “match type” features.
- Evaluation uses a ranking setup with per-response and per-dialog accuracy, the latter measuring goal completion.
- Results: MemN2N outperforms IR and supervised embeddings and learns non-trivial behaviors (e.g., issuing/updating API calls).
- Match-type features significantly improve OOV handling and Task 4 (KB lookup answers).
- Per-dialog accuracy remains low on harder tasks (e.g., sorted option display), showing current model limitations.
- Similar relative performance patterns on DSTC2 and concierge data support external validity.
- Decision: Accept.
- Key reasons for accept: a needed, carefully designed, reproducible benchmark isolating key competencies with meaningful OOV and KB challenges.
- Key reasons for accept: solid, transparent empirical study with clear metrics and validation on real data likely to catalyze progress.
- Well-motivated problem targeting the gap between chit-chat end-to-end systems and goal-oriented dialog requiring state tracking and KB use.
- Proper placement in literature connecting to bAbI, DSTC, and data-driven dialog; fair comparative evaluation of models.
- Empirical support strengthened by hyperparameter/ablation studies (hops, match types) and reported metrics.
- Rigor and transparency: explicit candidate ranking setup, OOV protocol, released data for Tasks 1–6, and a rule-based upper bound.
- Caveat: Simulator uses deterministic, template-based language that may not capture real linguistic variability.
- Caveat: Ranking-based evaluation sidesteps generation/realization and may overstate capabilities.
- Caveat: Match-type mechanism assumes typed KB and hand-enumerated types; scalability to new slots/types is unclear.
- Caveat: Limited analysis of statistical significance, detailed error breakdowns, and sensitivity to training size/paraphrase diversity.
- Suggestion: Expand error analysis to quantify which dialog steps (sorting, grounding, state tracking) cause failures and include per-turn breakdowns.
- Suggestion: Run robustness studies varying paraphrase diversity, bot stochasticity, noise (ASR-like), and training data size.
- Suggestion: Add competitive generative baselines with copy/pointer mechanisms and entity-aware seq2seq models.
- Suggestion: Complement ranking evaluation with generative evaluation, constrained decoding success rates, and human evaluation on a subset.
- Suggestion: Provide DSTC2 conversion details and release conversion scripts; report candidate set sensitivity (size/composition) and accuracy scaling.

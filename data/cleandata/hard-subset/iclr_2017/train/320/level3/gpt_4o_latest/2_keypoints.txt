- Paper tackles visual servoing using learned visual representations and predictive dynamics models.
- Introduces an action-conditional bilinear model built on pre-trained VGG features.
- Proposes a multi-scale, multi-channel, locally-connected variant to improve dynamics prediction.
- Introduces a weighted objective incorporating long-term policy values via a fitted-value approach.
- Presents a novel trust-region fitted-Q iteration algorithm for sample-efficient RL.
- Empirical results claim improved generalization, robustness to visual variations/occlusions, and sample efficiency.
- Decision: Reject due to misalignment with ICLR’s emphasis on representation learning.
- Key strength: Well-motivated use of pre-trained VGG features showing strong empirical robustness.
- Key strength: Trust-region fitted-Q iteration is a novel, valuable contribution for robotics RL.
- Key strength: Thorough evaluation including comparisons to classical and end-to-end RL baselines.
- Key weakness: Focuses on control and predictive dynamics rather than advancing representation learning.
- Key weakness: Reliance on fixed pre-trained features and linear value approximation limits end-to-end novelty.
- Key weakness: Dependence on pre-trained features raises questions about applicability where such features aren’t available.
- Suggested improvement: Explore end-to-end learning of visual representations tailored to servoing.
- Suggested improvement: Jointly optimize feature representation and control policy using unsupervised/self-supervised methods.
- Suggested improvement: Add experiments showing applicability when pre-trained features are unavailable or less effective.
- Suggested improvement: Include detailed ablation studies isolating the multi-scale model, weighted objective, and trust-region algorithm.
- Questions for authors: How does performance compare using end-to-end learned features vs. pre-trained VGG?
- Questions for authors: Can the trust-region fitted-Q iteration be adapted when pre-trained features aren’t available?
- Questions for authors: How does the method perform in highly dynamic or cluttered environments where pre-trained features may fail?

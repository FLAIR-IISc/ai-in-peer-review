### Summary of Contributions

This paper presents a novel and highly sample-efficient method for learning visual servoing controllers. The core problem is to enable a robot to follow a target using visual input, while being robust to appearance changes and distractors, and requiring minimal task-specific training data. The authors propose a three-part solution: (1) leveraging pre-trained deep convolutional features (from VGG) to provide a rich and robust visual representation without task-specific feature engineering; (2) learning a simple, multi-scale bilinear model to predict the short-term dynamics of these features in response to robot actions; and (3) using a specialized fitted Q-iteration (FQI) algorithm to learn a set of weights over the feature channels and scales. These weights allow the controller to focus on task-relevant visual information. The key insight is that the servoing policy, which minimizes a weighted one-step prediction error, has a structure that permits an efficient, linear Q-function approximation. This leads to a stable and extremely data-efficient RL procedure. The method is validated on a challenging simulated car-following task, where it demonstrates substantial improvements in performance and a more than two-orders-of-magnitude improvement in sample efficiency compared to standard model-free deep RL algorithms like TRPO, and also outperforms classical visual servoing baselines.

### Decision and Key Reasons

**Decision:** Accept

1.  **Exceptional Sample Efficiency:** The paper compellingly demonstrates a method that learns a competent visual servoing policy from only 20 trajectories. This represents a massive improvement (over 100x) compared to standard deep RL approaches and is a significant step towards making learned robotic controllers practical for real-world applications where data is expensive to collect.
2.  **Elegant and Effective Formulation:** The combination of pre-trained features, a simple predictive model, and a carefully structured Q-function is highly effective. The design of the Q-function approximator to be linear in the feature-error terms is a particularly clever insight that makes the FQI approach computationally efficient and stable, directly addressing a common pain point in value-function-based RL.

### Supporting Arguments for Decision

The main strength of this paper lies in its well-motivated and pragmatic approach to a difficult problem. Instead of attempting to solve the visual servoing problem with a monolithic, end-to-end deep RL model that is notoriously data-hungry, the authors decompose the problem into more manageable parts, leveraging the strengths of different techniques.

Using pre-trained features from VGG is a well-established practice for transfer learning, but its application here, combined with a learned dynamics model in that feature space, is powerful. The choice of a bilinear model for the dynamics is justified by its simplicity and linearity in the control inputs, which makes the resulting policy optimization (Equation 3) a simple quadratic program that can be solved efficiently. This is a critical design choice that enables the rest of the framework.

The most novel contribution is the FQI algorithm for learning the feature weights. The authors correctly identify that simply minimizing prediction error across all feature channels is suboptimal, as distractors would be weighted equally with the target. By framing the problem of learning these weights as a Q-learning problem, they directly optimize for long-term task performance. The specific Q-function parametrization (Section 5.2) is the key technical insight. It aligns the structure of the value function with the structure of the policy, allowing the greedy action selection `argmin(Q)` to be exactly the desired servoing policy. This avoids the need for a separate, complex policy network and leads to a simple least-squares problem for the FQI updates (Equations 5 and 6), which is much more stable and sample-efficient than gradient-based policy optimization methods.

The experimental evaluation is thorough and convincing. The authors compare their method against a strong set of baselines, including:
*   Ablations using different features (pixels, various VGG layers).
*   Alternative learning algorithms for the weights (CEM, TRPO).
*   Model-free end-to-end RL (TRPO on a CNN).
*   Classical robotics methods (IBVS with ORB features and with a state-of-the-art tracker).

The results clearly show the superiority of the proposed approach. The learning curve comparison in Figure 7, showing the proposed method achieving strong performance in just 2 sampling iterations while TRPO barely improves, is particularly impactful and strongly supports the central claim of sample efficiency. The release of the code and environment is also a commendable contribution to the community.

### Additional Feedback for Improvement

The following points are intended to help improve the paper and are not critical for the acceptance decision.

1.  **Clarity on Dynamics Model Generality:** The paper states that a single dynamics model was learned for all training cars. It would be helpful to clarify the intended scope of this model. Is the model specific to the "car-following-in-this-city" domain, or is the claim that it represents a more general model of visual dynamics under ego-motion? Clarifying this would help set reader expectations about when the dynamics model would need to be re-trained (e.g., for a different robot, a flying task, or following a different object category).

2.  **Discussion of Limitations:** The paper could benefit from a more explicit discussion of its limitations. For instance, the RL training relies on a cost function derived from ground-truth 3D positions of the car (Appendix B). This is a standard and acceptable practice for simulated experiments, but it's an important detail that has implications for real-world deployment, which would likely require an alternative reward source (e.g., from human labels or a learned reward model). Similarly, the bilinear dynamics model may not be sufficient for environments with more complex, non-linear dynamics. Acknowledging these points would further strengthen the paper.

3.  **Real-World Transfer:** While out of scope for this work, a brief discussion on the potential challenges and strategies for transferring this system to a real robot would be valuable. For example, would the pre-trained VGG features and bilinear dynamics assumption hold up under real-world camera noise and lighting conditions?

### Questions for the Authors

To further solidify my understanding, I would appreciate answers to the following questions:

1.  The data for training the dynamics model was collected using a "hand-coded policy." How sensitive is the quality of the final servoing policy to the data distribution used for learning the dynamics? For instance, if the dynamics were trained on data from a purely random exploration policy, would you expect a significant degradation in performance?
2.  The cost function in Appendix B penalizes the system for getting too close to the car (`||pt+1||2 ≥ τ`). Does this create an incentive for the policy to maintain a specific distance, or is the primary driver of behavior the minimization of the visual feature error relative to the goal image `x*` taken at the start?
3.  The locally connected bilinear model (Equation 2) seems like a good compromise between expressivity and parameter count. The experiments, however, use fully connected dynamics for the VGG features due to memory constraints. Could you comment on whether you believe a locally connected version for VGG features (if memory permitted) would further improve performance or if the global receptive field of deeper features makes this distinction less important?

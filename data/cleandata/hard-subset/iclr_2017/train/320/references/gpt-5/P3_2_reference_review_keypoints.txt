- Paper combines pretrained CNN features, learned bilinear multi-scale feature dynamics, and a sample-efficient RL (FQI) to learn a weighted one-step servoing objective for visual servoing without hand-engineered features.
- Key ideas: use semantic feature maps as observations, learn per-channel/scale bilinear dynamics, servo by minimizing predicted next-step feature error to a target image, learn feature weights and action regularization via FQI enabling closed-form argmin.
- Empirical claim: strong performance on a synthetic quadrotor-car-following benchmark with high sample efficiency (RL uses 20 trajectories) and outperforms pixel-based servoing, classical IBVS, and model-free deep RL baselines; code and videos released.
- Decision: Weak accept — coherent, well-motivated integration with convincing synthetic-domain results, but some experimental and methodological caveats.
- Strength: Good placement at intersection of visual servoing, predictive modeling, and RL, addressing IBVS and model-free RL limitations.
- Strength: Bilinear dynamics and locally-connected formulation are appropriate for short-horizon camera motion predictions and yield a convex quadratic in actions with closed-form argmin.
- Strength: Q-function parameterization (nonnegative per-channel/scale weights + quadratic action penalty) aligns greedy action selection with the servo objective, and the FQI update (rescale + bias) is reasonable.
- Strength: Using target images formalizes instance-following without explicit labels, and RL stage learns which feature channels matter for long-horizon objectives.
- Empirics: Ablations over VGG layers show deeper features up to conv4_3 help while conv5_3 degrades; comparisons include multiple baselines (CEM, TRPO, IBVS with ORB/C-COT, PBVS upper bound).
- Empirics: Generalization to novel cars indicates semantic features improve robustness over pixels.
- Reproducibility: Paper is mostly clear, includes algorithmic details, and releases code and simulator.
- Concern: Sample-efficiency claim omits cost of dynamics pretraining (~100 trajectories / 10k triplets); total data footprint and wall-clock should be reported and compared to baselines.
- Concern: Baseline fairness — IBVS/C-COT given GT depth and box while the method uses pre-trained dynamics and features; clarify priors and consider variants for fair comparison.
- Concern: Action constraints — argmin appears unconstrained while simulator clamps velocities; clarify whether box constraints are enforced in optimization and effect on stability.
- Concern: Dynamics model variants — inconsistent description of locally-connected vs fully-connected bilinear dynamics; specify which was used per VGG layer and provide parameter counts/training times.
- Concern: Reward uses GT car pose (sim-only), limiting transferability; discuss image-only proxy costs for real-world deployment.
- Suggested ablations: remove learned dynamics (finite-difference Jacobians), vary dynamics pretraining data size, compare per-scale vs per-channel weights, test spatial masks/attention, try alternative backbones/self-supervised features.
- Suggested measurements: report parameter counts, runtime (GPU/CPU), inference-time breakdown (feature extraction vs control solve), and variance across seeds.
- Suggested robustness tests: occlusions, distractors, lighting/viewpoint changes, camera roll/pitch, and analysis of failure cases.
- Suggestion on theory: discuss convergence properties of the FQI variant with nonnegativity constraints and target rescaling.
- Questions for authors:
  1) What action constraints are enforced at execution and are they included in the argmin optimization?
  2) For each VGG layer, which bilinear dynamics variant was used (fully vs locally connected), and what were parameter counts/training times?
  3) How many parameters are in θ (channels × scales + action dims) and what are typical values/sparsity patterns?
  4) How sensitive is performance to regularization ν and the nonnegativity constraint on θ; any instability without them?
  5) Can you report combined sample usage (dynamics pretraining + RL) vs TRPO baselines, and give baselines access to pretraining to isolate FQI effects?
  6) Have you tried spatial masks/attention to avoid penalizing distractors and would spatially varying weights help generalization?
  7) How would you define/learn the RL cost in real-world settings without GT pose, and do you expect transfer from CitySim3D?

This is an excellent paper demonstrating an end-to-end trained image compression and decompression system. It achieves better bit-rates and quality than established image compression algorithms (such as JPEG-2000). In addition to showcasing the effectiveness of 'deep learning' for a new application, a significant contribution of this paper is the introduction of a differentiable version of the "rate" function. The authors show that this can be used for efficient training with various rate-distortion tradeoffs. I hope it will not only be limited to compression applications but will also have implications in other fields where differentiable approximations of similar functions can be beneficial.

The authors have provided a thoughtful response to my pre-review questions. I would still say that to reduce distortion under a specific range and quantization, a sufficiently complex network will automatically learn to generate code within a specific range with the highest possible entropy (i.e., it will meet the upper limit). However, the second argument is plausible—doing so imposes a certain "form" on how the compressor output is used, which would require a more complex network to match the effective compression of the current system. This network would be able to perform the calculations done by a separate variable rate encoder currently used to store q.

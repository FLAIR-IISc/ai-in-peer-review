

# Review of "END-TO-END OPTIMIZED IMAGE COMPRESSION"

## 1. Summary of Contributions

The paper proposes an end-to-end optimized image compression framework using nonlinear transforms with a continuous relaxation of quantization to enable gradient-based optimization. The authors claim their method outperforms JPEG and JPEG 2000 in rate-distortion performance and visual quality, even when optimized for MSE. The approach uses cascaded linear convolutions with generalized divisive normalization (GDN) as the nonlinear transform, followed by uniform scalar quantization. The key innovation is the use of a continuous relaxation with additive uniform noise to approximate the quantization step for training purposes, while still evaluating using actual bit rates.

## 2. Decision

Reject. The paper contains fundamental mathematical errors that undermine its technical credibility, particularly the incorrect dimensionality reduction claim, and lacks sufficient justification for key design choices.

## 3. Supporting Arguments

The most critical flaw is the claim that "the net output thus has half the dimensionality of the input" (section 2). With the described architecture (first stage downsampling by 4, then by 2, then by 2), the output dimensionality would be 1/16 of the input spatial dimension, not 1/2. For a 256×256 input image, this would result in a 16×16 output (256 pixels), not 128×128 (16,384 pixels) as implied by "half the dimensionality." This is a significant mathematical error that casts doubt on the authors' understanding of their own architecture.

Additionally, the paper mischaracterizes the LISTA algorithm in the appendix (6.1), claiming the synthesis transform is constructed "by applying a principle known from the LISTA algorithm" to invert the GDN transform. LISTA (Gregor and LeCun, 2010) is designed for solving sparse coding problems, not for inverting normalization transforms. This mischaracterization undermines the technical justification for the synthesis transform architecture.

The paper also contains a fundamental error in section 3.1, where it claims "the first term in the Kullback-Leibler divergence is constant." This is incorrect; the first term (E_q[log q(y|x)]) is the entropy of the approximate posterior, which varies with the parameters being optimized.

Furthermore, the paper lacks justification for the choice of "one iteration" in the inverse transform (IGDN), which is a critical design decision for reconstruction quality. The authors provide no evidence or explanation for why this single iteration is sufficient.

The paper also contains numerous minor but significant inconsistencies: inconsistent parameter notation (using both φ and θ for different parameter sets without clear justification), unclear architectural details (downsampling factors not specified in main text), and a misrepresentation of CABAC implementation (section 6.2) that does not match standard CABAC.

## 4. Additional Feedback

To improve the paper, the authors should:

1. Correct the dimensionality reduction error and provide accurate architectural specifications for the transforms. The current description would lead to an output dimensionality of 1/256 of the input, not 1/2.

2. Clarify the relationship between the synthesis transform and the LISTA algorithm, or remove the incorrect reference to LISTA.

3. Correct the explanation of the KL divergence terms in section 3.1.

4. Justify the choice of one iteration for the inverse transform with empirical evidence or theoretical analysis.

5. Standardize the parameter notation throughout the paper (either use φ for all parameters or clearly distinguish between analysis and synthesis parameters).

6. Provide a more detailed description of the entropy code implementation to clarify how actual bit rates are calculated.

7. Verify all claims about the dimensionality and computational complexity of the transforms.

8. Remove the incorrect reference to the LISTA algorithm and replace it with an accurate technical justification for the synthesis transform design.

## Questions for Authors

1. The paper claims "the net output thus has half the dimensionality of the input" (section 2). For a 256×256 input image, what is the exact spatial dimensionality of the output? Please provide the calculation.

2. What is the theoretical or empirical justification for using only a single iteration in the inverse transform (IGDN)? How does this compare to using multiple iterations?

3. The paper states that "the continuous relaxation using additive noise was used only for training purposes." However, the loss function (11) still includes the additive noise during training. Please clarify this point.

4. The paper claims to implement "a simple entropy code based on the context-based adaptive binary arithmetic coding framework (CABAC)." However, the description in section 6.2 does not match standard CABAC implementation. Please clarify how the entropy code differs from standard CABAC and provide evidence that the actual bit rates calculated are comparable to those of standard CABAC implementations.

5. The paper states "the set of ĥ, ĉ, β̂, and γ̂ make up the parameter vector θ" (section 2), but earlier it states "the full set of h, c, β, and γ parameters (across all three stages) constitute the parameter vector φ to be optimized" (section 2). Please clarify the distinction between these parameter sets and why the synthesis transform parameters use different notation.

1) Summary of claims and contributions
The paper proposes an end-to-end optimized image compression framework based on nonlinear analysis/synthesis transforms with generalized divisive normalization (GDN/IGDN), uniform scalar quantization, and an entropy coder. To make the non-differentiable quantization amenable to SGD, the authors replace rounding during training with additive uniform noise and optimize a relaxed rate–distortion objective. They connect the relaxation to VAEs, argue the method approximates the discrete problem across the RD curve, and report actual bitrates using a CABAC-style entropy coder. Experiments on Kodak (and some standard images) compare against JPEG and JPEG 2000, showing improved PSNR at many rates and substantially better MS-SSIM and perceived quality, especially at low bitrates. The method is extended to color. The paper emphasizes the qualitative advantages (reduced blocking/ringing) and posits that GDN captures image statistics efficiently.

2) Decision: Reject
Key reasons:
- Technical correctness and presentation issues (equation/sign errors, incomplete operators, misuse of density/PMF equality) coupled with numerous formatting/reference problems undermine confidence in the rigor and clarity of the work.
- Empirical evidence is not summarized rigorously: no standard aggregate metrics (e.g., BD-rate on Kodak), limited baselines, per-λ architecture changes confound fairness, and key practical aspects (runtime, model size, reproducibility) are missing.

3) Supporting arguments
Technical/presentation issues:
- Eq. (3) (GDN) appears incorrect: divisive normalization should be u^{(k+1)}_i = w^{(k)}_i / (β + Σ_j γ w_j^2)^{1/2} (i.e., exponent −1/2). As written, it multiplies by (·)^{+1/2}, which matches IGDN, contradicting the text (“divisive normalization”). Eq. (4) (IGDN) uses the multiplicative +1/2 form, so the mismatch in Eq. (3) is likely a sign error but is non-trivial.
- Eq. (5) (upsampling) is incomplete: v̂ is defined for integer coordinates only; the “otherwise” branch is blank. If zeros are inserted, say so; if learned transposed convolutions are used, specify. As written, the operator is undefined.
- Eq. (10) states p_{ỹ}(n) = P_q(n) for n ∈ Z^M. A differential density evaluated at integers is not equal to a PMF mass; the correct statement is that ∫_{n−1/2}^{n+1/2} p_{y}(t) dt = P_q(n) and p_{ỹ} = p_y ∗ U(0,1). The text later acknowledges bias but the equality as written is mathematically incorrect and misleading.
- Several notational inconsistencies and typos (e.g., “continous,” “representing values,” inconsistent vector/scalar notation, missing parentheses) impede readability.
- References are in poor condition: many entries have missing years, malformed author lists, placeholders (e.g., “? \Q2016\E”), missing venues/DOIs, and duplicated/mis-attributed citations. This is far below ICLR standards.

Methodology/evaluation concerns:
- The entropy model assumes fully factorized marginals across space and channels; while the transform aims to Gaussianize/independize, there is no quantitative analysis of residual dependencies or ablations showing impact of better context models. The CABAC scheme uses a simple raster-scan with weak context; JPEG 2000 uses sophisticated context models. Yet the paper claims superiority without carefully controlling for entropy coding strengths.
- Per-λ architecture changes (increasing channels for large λ and RGB) confound fairness across the RD curve and vs baselines. A fixed-capacity model per dataset is preferable; otherwise capacity becomes a hidden knob affecting the RD curve.
- The experimental presentation is qualitative-heavy; key quantitative summaries are missing. There is no aggregated RD analysis (e.g., BD-rate over Kodak), no tables reporting average PSNR/MS-SSIM vs bpp, and no standard error bars. The paper defers to a website for numerous results, which is problematic for archival rigor and reproducibility.
- Baselines are limited: JPEG and JPEG 2000 only. By 2016/2017, HEVC intra/BPG and WebP were known relevant intra-image baselines and often stronger than JPEG 2000; their omission weakens the empirical case.
- Practical aspects are not reported: inference time, encoder/decoder complexity, model size, memory footprint, training cost, and robustness across seeds. For a compression method, these are important.
- Variable-rate practicality: the approach requires a separate model per λ; while noted, there is no demonstration of a single model operating across rates or of a principled rate control mechanism.

4) Additional feedback (to help improve the paper; not all are decision-critical)
- Fix Eq. (3), fully specify Eq. (5), and correct Eq. (10)’s statement. Clarify all operators and boundary conditions; include exact implementation details (e.g., whether upsampling is zero-insertion plus convolution, strided/deconvolution, or resize-convolution).
- Clean up all references to ICLR standard. Ensure all citations are complete and consistent.
- Provide a rigorous quantitative summary: report BD-rate vs JPEG/JPEG 2000 (and ideally BPG/HEVC-intra and WebP) on Kodak, for PSNR and MS-SSIM; include per-image plots in appendix but keep main-text tables/figures self-contained.
- Control model capacity across λ to avoid confounds; alternatively, explicitly analyze capacity’s effect on the RD frontier.
- Include ablations: GDN vs ReLU/sigmoid, number of stages, piecewise-linear marginal resolution, context model strength, continuous relaxation vs straight-through estimator, and uniform-noise width sensitivity.
- Analyze the approximation gap: quantify bias/variance of h(ỹ) vs H(q) across rates; provide conditions under which the relaxation is accurate and where it fails.
- Report runtime, memory, and model size, and discuss deployment implications. Release code and trained models.
- Evaluate residual dependencies in y (e.g., mutual information across channels/space) to justify the factorized model or motivate stronger entropy models.

Questions for the authors
- Please confirm the correct GDN form in Eq. (3) (should it be exponent −1/2?). If so, how did this affect training/implementation?
- Precisely define the upsampling in Eq. (5). If zeros are inserted, what anti-aliasing/filtering is used?
- How large is the gap between differential and discrete entropy in practice? Can you report average h(ỹ) − H(q) across Kodak per λ?
- Did you evaluate stronger entropy models (e.g., autoregressive context across space/channels) and, if so, how much bitrate gain did they yield?
- How sensitive are results to the independent-marginal assumption? Any evidence of residual correlations in y?
- Can you provide aggregated RD numbers (e.g., BD-rate) on Kodak and include BPG/HEVC-intra as a baseline?
- What are encoding/decoding times and model sizes for grayscale and RGB? How do they scale with image size?
- Can a single model support multiple rates (e.g., via learned scale parameters or latent dropping), and how does its RD performance compare to per-λ models?

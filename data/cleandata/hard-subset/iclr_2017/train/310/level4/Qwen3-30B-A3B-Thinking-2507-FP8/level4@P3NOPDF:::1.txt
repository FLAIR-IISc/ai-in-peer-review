

This paper presents a timely investigation into multi-agent communication dynamics, addressing a foundational question in reinforcement learning: whether machine problem-solving strategies resemble human cognitive processes. The work's core contribution lies in its focus on communication efficiency between agents, a critical yet underexplored aspect as RL applications proliferate.  

**Strengths**  
The paper excels in its conceptual clarity and empirical design. The task—requiring one agent to convey information about an image to a second agent—strikes an optimal balance: simple enough to enable intuitive post-learning analysis yet complex enough to reveal meaningful communication patterns. A particularly compelling finding is the differential symbol usage between informed and agnostic agents. Informed agents leverage multiple symbols to encode nuanced distinctions, while agnostic agents converge on a binary symbol system. This contrast provides concrete evidence of how prior knowledge shapes communication protocols, offering valuable insights into the relationship between agent capabilities and emergent language. The task design effectively isolates communication as the primary variable, enabling clear causal interpretation of the results.  

**Key Concerns**  
The primary limitation concerns the task's alignment with the paper's stated goal. The core problem reduces to a standard image classification challenge: agent A must map an input image to a symbol, and agent B must map that symbol back to the correct image class. Crucially, the symbols themselves function as proxies for image categories (e.g., "symbol 1" = "cat," "symbol 2" = "dog"), rendering the communication channel redundant. This reduction undermines the paper's contribution to *communication research*, as the solution hinges entirely on classifying images via an intermediate symbol layer—effectively making it a transfer learning problem. The authors acknowledge this by noting the task's simplicity, but they do not adequately address how this differs from conventional image classification pipelines.  

More critically, the reinforcement learning (RL) framework appears unnecessarily complex for this task. If the symbols directly encode class labels, a supervised classifier (e.g., a simple CNN) could achieve identical performance with orders of magnitude less training data and computational cost. The paper’s RL approach does not demonstrate any advantage over direct classification; instead, it adds layers of complexity without evidence of superior communication dynamics. This misalignment is particularly problematic because the paper positions itself as advancing *communication* understanding, not classification. The authors should either: (1) redesign the task to avoid this reduction (e.g., by introducing ambiguity where symbols *cannot* directly map to classes), or (2) explicitly justify why RL is necessary for a problem that reduces to classification.  

**Recommendation**  
This paper identifies an important research direction but falls short in execution. The communication insights are compelling *only if* the task genuinely requires emergent symbol use beyond simple classification. As currently framed, the work risks conflating communication with classification, weakening its contribution to the field. I recommend *rejection* unless the authors provide a thorough revision addressing the task’s fundamental reduction to image classification. Specifically, they must either:  
- Demonstrate a task variant where symbols cannot be mapped to class labels (e.g., by introducing partial observability or ambiguous inputs),  
- Or explicitly compare their RL approach against supervised baselines to prove its necessity for communication learning.  

Without this, the paper’s core argument—that RL reveals meaningful communication strategies—lacks foundation. The current results merely show RL can replicate classification via symbols, not that it uncovers new communication principles. While the problem space is valuable, the paper’s current implementation does not advance it. A revised version with a non-reducible task would be a strong contribution to ICLR.

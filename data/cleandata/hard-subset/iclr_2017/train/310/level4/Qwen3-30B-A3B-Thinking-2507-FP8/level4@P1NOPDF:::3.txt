

This paper introduces a referential communication game between two agents operating on visual inputs. The sender observes two images and a binary target variable (t), then transmits a symbolic message to the receiver. The receiver uses this message to predict t, with both agents rewarded when the prediction is correct. The authors parameterize the agents as neural networks using pretrained image features and train them via REINFORCE. The results demonstrate that the agents learn optimal communication policies, with the transmitted symbols encoding meaningful concepts. The framework is extended to a variant grounded on distinct image classes, where the agents learn even more interpretable symbolic representations. Furthermore, in a multi-game setup where the sender alternates between communication and a supervised image classification task, the symbolic messages become increasingly meaningful—consistent with the grounding provided by the classification objective.

This work addresses a compelling challenge: learning shared communication protocols in multi-agent systems, which is significantly more complex than standard supervised or single-agent reinforcement learning. The authors’ choice to begin with a simplified referential game is well-justified. Crucially, their approach—first learning agent-to-agent communication and later grounding it in human language—represents a novel paradigm distinct from conventional sequence-to-sequence models, which often prioritize statistical patterns over functional semantics. The paper’s contributions are substantial: it establishes a clear experimental framework for studying grounded communication, provides rigorous analysis of learned symbols, and offers insightful visualizations of their semantic content. This constitutes a valuable foundational step for future research on interpretable agent communication. I recommend acceptance.

**Specific Comments:**
- **Target integration:** Clarify how the binary target t is incorporated into the sender network architecture (e.g., as an input feature, a conditional parameter, or through a separate embedding).
- **Table 1 vs. Table 2:** The percentage formatting is inconsistent. Table 1 reports values in [0, 100] (e.g., 85%), while Table 2 uses [0, 1] (e.g., 0.85). Standardize to one format (e.g., [0, 1] for consistency with Table 2) or explicitly state the range for each table. Additionally, the "obs-chance purity" column in Table 1 contains values near zero (e.g., 0.0001), which appears anomalous—verify if this reflects a data error or is intentional.

### Review

#### Summary of Contributions

This paper introduces a compelling framework for studying the emergence of language through multi-agent communication. The authors propose a cooperative, two-player referential game where a "sender" agent must generate a symbolic message to help a "receiver" agent identify a target image from a pair. The agents, implemented as simple neural networks, are trained from scratch using reinforcement learning, with task success as the only reward signal.

The paper makes several key contributions. First, it demonstrates that agents can successfully develop a discrete communication protocol to solve the referential task, converging to near-optimal policies. Second, through a series of well-designed analyses (including purity metrics, SVD, and t-SNE visualizations), the paper provides evidence that the emergent language captures meaningful semantic properties of the images, moving beyond low-level visual features. Third, the authors show that modifying the game's environment—by forcing agents to coordinate on object-level properties rather than specific image instances—can encourage the development of more abstract and generalizable symbol meanings. Finally, and perhaps most significantly, the paper presents a novel and promising strategy for grounding the agents' emergent language in human language by interleaving the referential game with a supervised image labeling task. This hybrid approach is shown to not only make the agents' code more interpretable but also to enable a degree of successful communication with human participants on novel tasks.

#### Decision and Key Reasons

**Decision: Accept**

This paper should be accepted for two primary reasons:

1.  **Novel and Important Problem Formulation:** The work tackles the fundamental problem of how communication can emerge and be grounded. The proposed multi-agent, game-based framework is a well-motivated and refreshing alternative to purely supervised approaches to language, focusing on the functional aspects of communication rather than just statistical pattern matching. The strategy of first allowing a language to emerge and then grounding it is a novel contribution with significant potential.
2.  **Strong and insightful Empirical Results:** The experiments are thoughtfully designed and progressively build a strong case for the paper's claims. The analysis goes beyond simply reporting task performance, offering valuable insights into the semantic structure of the learned communication protocol. The final experiment, demonstrating that grounding on one dataset can generalize to enable human-agent communication on another, is particularly compelling.

#### Supporting Arguments for the Decision

The paper is well-situated in the literature, drawing connections to multi-agent RL, game theory, and cognitive science. The motivation to move beyond "canned" conversational data towards interactive learning is strong and timely. The proposed referential game is simple enough to be tractable and analyzable, yet complex enough to allow for the emergence of non-trivial communication.

The empirical validation is thorough. The authors first establish a baseline result: tabula rasa agents can learn to communicate effectively (Section 4). They then dissect *what* is being learned. The use of purity scores to show that the learned symbols align with human-defined semantic categories is a clever method of evaluation. The SVD analysis provides further evidence that the communication system is not trivial, possessing a multi-dimensional structure.

The experiment in Section 4.1, which modifies the "common knowledge" between agents to encourage coordination on class-level properties, is a creative and insightful demonstration of how the environment shapes the emergent language. This is a crucial step, showing that the properties of the learned language are not arbitrary but can be deliberately guided by the task design.

The final set of experiments on grounding the communication (Section 5) is the paper's most significant contribution. The idea of combining interactive self-play with supervised learning, inspired by successes like AlphaGo, is applied effectively here. The results are striking: purity dramatically increases, and more importantly, the agents' code becomes interpretable to humans. The crowd-sourced experiment provides strong quantitative evidence that the grounding generalizes, allowing humans to correctly interpret the agent's messages even when used in novel, "metonymic" ways (e.g., using "dolphin" to refer to the sea). This finding is an encouraging step towards building machines that can flexibly communicate with people.

#### Additional Feedback for Improvement

The following points are intended as constructive feedback to further strengthen the paper. They are not critical for the acceptance decision.

*   **Clarity on Table Formatting:** There appears to be an inconsistency in how percentages are reported. In Table 1, values seem to be on a [0, 100] scale, while in Table 2, they seem to be on a [0, 1] scale. Standardizing this would improve clarity.
*   **Minor Typos:** I noticed a few minor typos, such as "assest" which should be "assess", and "usufal" which should be "usual". A careful proofread would be beneficial.
*   **Future Directions:** The agent architectures are intentionally simple. It would be interesting if the authors could briefly speculate in the discussion about how more complex architectures (e.g., recurrent networks) might enable more sophisticated communication protocols, such as multi-turn dialogue or compositional language, within this same framework.

#### Questions for the Authors

Your responses to these questions would help clarify my understanding of the work.

1.  Could you please clarify how the target information `t` is provided as input to the sender network? The text mentions the input is `θS(iL, iR, t)`, and the caption for Figure 1 states that the target and distractor are always presented in a fixed order, but a more explicit description of the input representation would be helpful for reproducibility. For instance, is `t` concatenated as a binary flag, or is it only implicitly provided by the input order?
2.  In Table 1, the values in the "obs-chance purity" column are extremely small (e.g., 0.0003). Is this a typo? If not, could you please briefly explain the calculation that leads to such small difference values, especially when the observed purity is significantly above chance? It seems surprising given that the observed purity is sometimes more than double the chance level.

- Task setup: Introduces a referential game where a sender sees the target image and emits a discrete symbol; receiver sees both images and the symbol and must pick the target.  
- Agents and training: Both sender and receiver are neural nets trained end-to-end with REINFORCE using pretrained CNN (VGG) features and a discrete communication bottleneck.  
- Main result — coordination: Tabula-rasa agents reliably learn to coordinate and reach near-perfect accuracy.  
- Main result — semantics: Emergent symbols show semantic regularities correlated with object categories and visual similarity (supported by purity analyses, SVD, t-SNE).  
- Main result — abstraction: Showing different instances of the same ImageNet class to sender and receiver encourages more abstract, class-level symbol meanings.  
- Main result — grounding: Combining self-play with supervised object naming grounds symbols in natural language, improving purity, interpretability, and enabling partial human–agent communication.  
- Human evaluation: A crowdsourced study shows partial human understanding of symbols (68% success), supporting interpretability claims.  
- Motivation and positioning: Addresses limitations of passive supervised models and situates work within signaling games, multi-agent RL, and linguistic theory.  
- Architecture choices: Uses simple sender variants (agnostic vs informed) and a discrete symbol bottleneck to isolate inductive biases relevant to language.  
- Use of pretrained features: Freezing VGG features focuses the study on communication rather than perception.  
- Evidence against trivial solutions: Analyses (informed sender uses more symbols, SVD shows multi-dimensionality, category purity above chance) argue against degenerate two-symbol solutions.  
- Contribution claim: Proposes a simple, scalable route to grounding emergent protocols without requiring humans during training.  
- Decision: Accept — novel, well-motivated framework with convincing empirical evidence and promising human evaluation.  
- Clarification requested — sender input: Specify exactly how target t is provided to the sender (ordering vs explicit flag) and image ordering for receiver.  
- Reporting issues: Standardize percent/scaling conventions across tables and verify “obs–chance purity” values and chance baseline computation.  
- Typos: Correct minor typos (e.g., “assest” → “assess”, “usufal” → “usual”).  
- Training details requested: State whether a variance-reduction baseline was used for REINFORCE, number of random seeds, variance across runs, and whether VGG features were frozen.  
- Hyperparameter sensitivity: Report sensitivity to temperature τ, vocabulary size, and embedding dimensionality.  
- Symbol usage metrics: Quantify effective vocabulary size (entropy, type–token ratio), distinguish polysemy vs redundancy, and consider mutual information measures or controlled synonymy experiments.  
- Statistical testing: Include confidence intervals and significance tests for purity and human-study results and detail permutation procedures for chance purity.  
- Human study details: Provide task instructions, participant demographics, quality control, agreement statistics, and examples of successes/failures; clarify symbol-to-word mapping.  
- Scope and limitations: Discuss constraints (single-symbol messages, two-image choice) and extensions to sequences, multi-turn dialog, and larger candidate sets.  
- Questions — sender input: Is the target always presented first and is there an explicit target flag to the sender?  
- Questions — REINFORCE stability: Did you use baseline/variance reduction and how stable are results across random seeds?  
- Questions — hyperparameter effects: How sensitive are convergence, purity, and interpretability to τ, vocabulary size (e.g., 10 vs 100), and embedding dimension?  
- Questions — symbol semantics: Can you quantify effective symbol count and amount of synonymy (e.g., via clustering of symbol–context distributions)?  
- Questions — grounding generalization: When supervised labels don’t overlap with game categories, does interpretability or accuracy degrade or produce negative transfer?  
- Questions — chance baseline: Precisely how is chance purity computed (permutation details) and what scaling is used for “obs–chance”?  
- Questions — human study sampling and prompts: How were image pairs chosen per symbol and what exact prompt did participants see; was per-word performance analyzed?  
- Questions — feature learning: Were CNN features frozen; have you tried joint visual-feature learning and its effect on semantic alignment?  
- Questions — generalization across distributions: Do agents trained on one image distribution transfer to another, especially under object-level reference?

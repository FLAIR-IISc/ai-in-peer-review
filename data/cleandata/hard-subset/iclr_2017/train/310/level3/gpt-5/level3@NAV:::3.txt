1. Summary
The paper studies emergent communication via a minimal referential game: a sender sees two images and a binary target t, sends one discrete symbol; a receiver must select the target. Both agents are neural networks operating on pretrained image features and trained end-to-end with REINFORCE. The authors show rapid convergence to near-optimal policies across architectures and vocabularies. Analyses (purity with respect to human categories, SVD of symbol–pair usage, t-SNE visualizations) suggest learned tokens capture meaningful, high-level concepts. A modified “object-level” game that hides image-specific details yields even more semantically coherent symbols. Finally, a multi-game setup that alternates the sender between the referential game and supervised ImageNet classification grounds symbols in human labels, markedly improving interpretability and enabling above-chance human–agent communication on a distinct dataset. The work argues this simple testbed is a justified starting point for shared communication in multi-agent settings and claims novelty in first learning agent-to-agent communication and then grounding it in human language as an alternative to seq2seq.

2. Decision: Accept
- Clear, well-motivated contribution with convincing empirical evidence that simple agents can develop meaningful, partially human-interpretable codes; grounding via multi-tasking is elegant and impactful.

3. Supporting arguments
- Positioning vs. prior work (e.g., differentiable comms, cheap-talk games) is appropriate; the focus on semantic interpretability and human evaluation strengthens significance.
- Results are consistent across conditions: convergence, increased purity with object-level setup, and strong improvements with supervised grounding (e.g., 47% exact label use when applicable; 68% human success on ReferIt-derived pairs).
- Methodological choices (discrete bottleneck, pretrained visual features) are reasonable and isolate the communication question.

4. Additional feedback (to help improve the paper)
- Clarify precisely how t is incorporated in the sender: is it only via ordering (target first), or also via an explicit binary input? How robust is performance if image order is randomized?
- Report variability across random seeds, confidence intervals for purity and human studies, and training stability (variance of REINFORCE, any baselines/entropy terms).
- Resolve reporting inconsistencies: Table 1 vs. Table 2 percentages ([0,100] vs. [0,1]); “obs-chance purity” in Table 1 appears numerically off.
- Include ablations: vocabulary size effects, fc vs. softmax features, informed vs. agnostic sender under identical capacities.
- Evaluate generalization: partner-swap (new receiver), unseen categories, and symbol stability across runs.
- Minor typos: “assest” → “assess”, “usufal” → “usual”. Please proofread references for formatting glitches.

Questions for authors
- How exactly is t encoded and fed to the sender? Any experiments with symmetric inputs or target indicator embeddings?
- How many runs per condition; what are mean±sd for coordination and purity?
- What temperature τ sensitivity did you observe? Any annealing or Gumbel-Softmax tried?
- Does supervised grounding hurt/help zero-shot reference to unseen classes?
- Can the emergent code support multi-symbol messages or multi-turn protocols in your framework?

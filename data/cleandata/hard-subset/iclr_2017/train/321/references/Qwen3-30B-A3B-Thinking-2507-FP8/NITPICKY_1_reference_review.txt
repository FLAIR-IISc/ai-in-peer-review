

# Review of "STOCHASTIC NEURAL NETWORKS FOR HIERARCHICAL REINFORCEMENT LEARNING"

## 1. Summary of Contributions

The paper proposes a novel framework for hierarchical reinforcement learning that learns a span of skills in a pre-training environment using Stochastic Neural Networks (SNNs) with an information-theoretic regularizer. The authors claim their approach can learn diverse, interpretable skills that can be reused across downstream tasks with sparse rewards, reducing sample complexity compared to training from scratch. The key technical contributions include using bilinear integration for latent variables, an MI-based regularizer to encourage skill diversity, and a hierarchical architecture for reusing learned skills. The paper demonstrates promising results on maze navigation and object gathering tasks, showing improved exploration and faster learning compared to baselines.

## 2. Decision

Reject. The paper contains fundamental technical errors in the proposed method (specifically in the MI regularizer implementation) and significant methodological inconsistencies that undermine the core claims of the work. The errors are not merely minor formatting issues but directly impact the validity of the proposed approach.

## 3. Supporting Arguments

The most critical issue is a fundamental technical error in the MI regularizer implementation. In Section 5.3, the authors propose modifying the reward as:

Rnt ← Rnt + αH log p̂(Z = zn|cnt)

However, since p̂(Z = zn|cnt) is a probability (between 0 and 1), log p̂(Z = zn|cnt) is negative. This means the proposed "reward bonus" would actually *decrease* the reward, which directly contradicts the paper's claim that it's encouraging the agent to learn diverse skills. The authors should have written αH * (-log p̂(Z = zn|cnt)) or αH * log(1/p̂(Z = zn|cnt)) to create a positive bonus. This is a fundamental error that would prevent the method from working as described, yet the paper fails to acknowledge this critical mistake.

A second major issue is the inconsistent and technically incorrect description of the posterior estimation. The paper states "mc(z) is an estimate of how many times each cell c is visited when latent code z is sampled." This is logically inconsistent - mc(z) should be a function of both the cell c and the latent code z, but the notation suggests it's only a function of z. The correct notation should be m_{c,z} (number of times cell c is visited with latent code z), but the paper uses inconsistent notation throughout, which would make the method impossible to implement correctly.

Additionally, the paper's claim that "the marginal distribution of π(a|s) is now a mixture of Gaussians instead of a simple Gaussian, and it may even become intractable to compute" is misleading. While the marginal is a mixture of Gaussians, it's not intractable to compute - the paper's solution of treating the latent code as part of the observation is a valid workaround, but the claim of intractability is exaggerated and could mislead readers.

## 4. Additional Feedback

1. **Critical Technical Correction**: The MI regularizer implementation must be fixed to use a positive bonus. The authors should correct the reward modification to Rnt ← Rnt - αH * log p̂(Z = zn|cnt), which would create a positive reward bonus. Without this correction, the method as described would not work.

2. **Clarify Notation**: The paper should consistently use proper notation for the posterior estimation (m_{c,z} instead of mc(z)) and clearly explain the relationship between the cell index, latent code, and visitation counts.

3. **Sample Complexity Claim**: The paper claims "the number of samples used to train a SNN is the same as a single feed-forward policy from Fig. 4(a)," but does not provide the actual sample count. This is critical for the sample efficiency claim. The authors should explicitly state the number of samples used for both the SNN and the Multi-policy baseline.

4. **Heess et al. (2016) Mischaracterization**: The paper incorrectly characterizes Heess et al. (2016) as having "independently proposed to learn a range of skills in a pre-training environment." The cited paper "Learning and transfer of modulated locomotor controllers" is about learning locomotor controllers for a single task, not about learning a span of skills for transfer to different tasks. This mischaracterization of related work undermines the paper's positioning in the literature.

5. **Clarity on Skill Adaptation**: The paper states "The weights of the low level and high level neural networks could also be jointly optimized" but then claims "frozen low-level policies are already sufficient." This creates confusion about how skills adapt to new tasks. The authors should clarify whether skills are adapted or not, and if not, how they handle task-specific skill requirements.

6. **Consistency in Terminology**: The paper uses "SNN" for both the architecture and the method, but doesn't consistently define the acronym. The authors should define "SNN" as "Stochastic Neural Network" at first use and maintain consistent usage.

These issues are not merely minor formatting inconsistencies but represent fundamental flaws that undermine the technical validity of the proposed approach. The paper's core contribution cannot be validated as described due to these errors, and the authors would need to substantially revise the methodological description and experimental validation to address these issues before the work could be considered for publication.

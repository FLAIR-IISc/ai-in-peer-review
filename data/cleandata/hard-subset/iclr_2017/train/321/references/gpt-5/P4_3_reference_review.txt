1. Summary of contributions
The paper addresses exploration and sample-efficiency in sparse-reward reinforcement learning by learning a reusable set of temporally extended skills in a pre-training phase and then composing them with a high-level policy on downstream tasks. The key ideas are:
- Model the low-level policy as a stochastic neural network (SNN) with a categorical latent code z that induces multimodal action distributions; integrate z with observations via bilinear interactions to improve expressivity.
- Encourage diversity and interpretability of skills by adding a mutual-information motivated reward bonus that makes z predictable from a small, task-relevant subset of the state (e.g., agent CoM position), estimated via discretized visitation counts.
- Freeze the learned SNN skills and train a high-level “Manager” that selects a skill every T steps, yielding temporally coherent exploration on sparse-reward tasks.
- Use TRPO for both phases, treating z as part of the observation to avoid intractable marginals.
Empirically, the paper shows: (i) SNNs with bilinear integration and MI bonus learn a broad, interpretable set of locomotion skills with the same sample budget as training a single Gaussian policy; (ii) the hierarchical manager over these skills improves exploration substantially; and (iii) on MuJoCo-based sparse tasks (mazes and gather) with Swimmer and Snake, the method learns faster and more reliably than strong baselines, including a proxy reward baseline and a multi-policy hierarchy that trains separate skills. The paper also analyzes sensitivity to the switch period T, reports variance across pre-training seeds, and discusses failure modes on unstable agents (Ant). Code and videos are provided.

2. Decision: Accept
Key reasons:
- Well-motivated, technically sound framework that leverages SNN multimodality and an MI-inspired regularizer to learn diverse skills with minimal supervision, and demonstrates clear empirical benefits for sparse-reward exploration.
- Clear ablations (concatenation vs bilinear integration, MI coefficient) and comparisons to reasonable baselines support the main claims; the approach is simple to implement and broadly applicable under a common agent-space assumption.

3. Supporting arguments
- Problem tackled: improve exploration and reduce sample complexity across multiple sparse-reward tasks by pretraining reusable skills using minimal domain knowledge, then composing them hierarchically. This is a relevant and important problem in RL.
- Motivation and placement: The paper situates itself well relative to options/HRL, skill discovery (HiREPS), intrinsic motivation (VIME), and pretraining approaches (e.g., goal-based). The distinguishing aspects are (i) multimodal policies via SNNs with shared parameters, (ii) MI-style regularization without requiring goals, and (iii) clean two-stage training with a frozen low-level for reuse.
- Technical soundness:
  - Using SNNs with categorical z and bilinear integration is a sensible way to realize multimodality with shared parameters; sampling z per episode induces temporally consistent skills.
  - The MI bonus is implemented as a log p̂(z|c) reward, where p̂ is estimated from batch visitation counts on a discretized agent-centric coordinate c. This is a pragmatic, low-variance estimator that aligns with maximizing I(Z; C) up to constants. While heuristic (non-stationary shaping, discretization choices), it is coherent and works empirically.
  - Treating z as part of the observation preserves the tractability of TRPO updates.
  - The hierarchical manager is standard, and the choice to freeze the low-level keeps the study focused on reuse and exploration.
- Empirical support:
  - Visitation plots convincingly show that SNNs with bilinear integration and MI produce diverse, interpretable skills, often broader than multiple independently trained unimodal policies with the same sample budget.
  - Random exploration coverage with a random manager over SNN skills is vastly larger than Gaussian action noise.
  - On sparse tasks (four mazes and gather) the hierarchical approach outperforms a strong proxy-reward baseline and a multi-policy hierarchy that used more pretrain samples. The gather task particularly benefits from MI-regularized skills (higher mean, lower variance).
  - Analyses on Snake show scalability; sensitivity to T is mild; limitations on Ant are candidly discussed.
- Novelty: Combining SNN multimodality, bilinear integration of a categorical latent, and an MI-style reward defined on an agent-centric state subset for unsupervised skill acquisition, followed by hierarchical reuse, is a meaningful and practical contribution beyond prior HRL/skill discovery methods.

4. Additional feedback (to help improve the paper; not all points affect the accept decision)
- Baselines and comparisons:
  - A more direct comparison to option-critic or other contemporary hierarchical methods on the same sparse tasks would strengthen the positioning.
  - Consider comparing to goal-conditioned policies learned in pretraining (e.g., via relabeling) to contextualize the advantage of the MI approach without explicit goals.
- MI bonus details:
  - Provide sensitivity analyses for grid resolution, batch size used for p̂(z|c), and αH. Discuss stability implications of non-stationary intrinsic rewards on TRPO.
  - Consider a learned discriminator/classifier to estimate p(z|c) in higher-dimensional c, with a simple cross-validation to select capacity.
  - Report a quantitative diversity metric (e.g., I(Z; C) estimate, trajectory dispersion) in addition to visitation plots.
- Generality and assumptions:
  - The proxy reward and c were tailored to locomotion; clarify how to choose c and the proxy reward in other domains (manipulation, vision-based tasks) and any instrumentation needed.
  - Discuss choosing K and the effect of over/under-specifying the number of skills; note whether continuous z helps.
- Hierarchy training:
  - Include limited results on end-to-end fine-tuning (even short runs) to quantify potential gains or stability issues.
  - Report sensitivity to the switch period T on more tasks and consider learning termination policies.
  - Analyze the manager’s observation space and whether excluding some agent states (to enforce abstraction) changes performance.

Questions for the authors
- How sensitive are performance and skill diversity to the discretization granularity for c and to the batch size used to estimate p̂(z|c)? Does smoothing (e.g., Laplace or kernel smoothing) help?
- Can you quantify the contribution of the MI bonus to downstream returns via an ablation in which the SNN skills are matched in pretrain return but differ only in MI?
- How do you select K in practice? Does increasing K beyond the number of distinct behaviors harm manager learning due to redundancy?
- Have you tried a learned classifier for p(z|c) rather than counts? Does it improve stability or diversity for higher-dimensional c?
- What is the effect of allowing the manager to condition on a running memory (RNN) versus feed-forward? Does it reduce dependence on T?
- For Ant, did you try pretraining with stochastic switching between skills to increase robustness to mid-trajectory switching? Would that mitigate falls?

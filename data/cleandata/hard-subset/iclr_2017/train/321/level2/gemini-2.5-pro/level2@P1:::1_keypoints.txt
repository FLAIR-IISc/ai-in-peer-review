- Presents a two-stage hierarchical RL framework: unsupervised pre-training of diverse low-level skills via a Stochastic Neural Network (SNN) with a proxy reward, then training a high-level manager to sequence frozen skills for downstream sparse-reward tasks.
- Decision: Accept.
- Key reason 1: Elegant, practical combination of skill discovery, information-theoretic regularization (mutual information bonus), and hierarchical control using a single SNN conditioned on a categorical latent.
- Key reason 2: Thorough and convincing experiments showing interpretable skills, improved exploration, and strong performance on challenging continuous-control benchmarks.
- Technical contribution: Using one SNN to represent multi-modal skills is more sample-efficient than training multiple independent policies.
- Technical contribution: Introducing an MI bonus between latent skill and agent state effectively encourages diverse, useful skill discovery.
- Architecture detail: Bilinear integration in the SNN and the MI bonus both positively affect learned skills, as shown by ablations.
- Experimental evidence: Visualizations and learning curves demonstrate dramatically improved exploration and faster solving of sparse-reward tasks versus a strong baseline.
- Baseline comparison: SNN matches or outperforms a multi-policy baseline while requiring fewer pre-training samples.
- Limitation 1 (feedback): Manager commits to a fixed skill duration T; appendix analyzes sensitivity but main text should summarize and note potential need for learned termination in dynamic tasks.
- Limitation 2 (feedback): MI bonus computed via 2D discretization may not scale to high-dimensional states; suggest discussing MLP-based posterior estimation as an extension.
- Limitation 3 (feedback): Stability issues for complex agents (e.g., Ant) when switching skills can cause falls; this should be acknowledged and noted as an open challenge.
- Question 1: How were skills for the "Multi-policy" baseline chosen—most diverse from many or simply different random seeds?
- Question 2: Have you considered mechanisms (e.g., novelty visitation rewards) to more explicitly encourage comprehensive coverage of behaviors beyond the MI bonus?
- Question 3: How does the manager use task-specific state info (S_rest) to select appropriate skills given low-level skills were trained without that input—does it learn intuitive mappings like wall detection → turn skill?

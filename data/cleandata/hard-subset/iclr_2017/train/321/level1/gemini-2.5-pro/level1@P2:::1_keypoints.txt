- Proposes a hierarchical RL framework that pre-trains diverse skills with a Stochastic Neural Network (SNN) conditioned on a discrete latent and then uses those frozen skills as temporally-extended actions for a high-level manager.  
- Core technical contributions: bilinear integration of latent code with state and an MI-based regularizer to encourage distinct, controllable skills.  
- Pre-training uses a simple proxy reward (e.g., moving quickly) to learn many reusable skills with minimal domain knowledge.  
- Decision: Accept.  
- Primary reason 1: Novel and effective unsupervised skill discovery via SNN + mutual information objective that reduces manual engineering.  
- Primary reason 2: Strong empirical validation showing improved exploration and faster learning on sparse-reward tasks.  
- Addresses a critical bottleneck in sparse-reward exploration by combining HRL, intrinsic motivation, and information-theoretic objectives.  
- Unsupervised pre-training avoids manual sub-goal specification and scales better than goal-conditioned baselines.  
- Ablations show bilinear integration and the MI bonus are crucial for learning diverse, separated skills.  
- Visualization evidence (visitation plots) demonstrates orders-of-magnitude improvement in exploration coverage versus naive strategies.  
- Downstream task performance: improved sample efficiency and success on tasks standard RL fails to solve; competitive with multi-policy baselines.  
- Experimental rigor: multiple seeds, several robot types, and honest reporting of failure modes (e.g., Ant instability).  
- Concern: MI bonus computed via discretized (x,y) visitation may not scale to higher-dimensional or less obvious distinguishing state spaces.  
- Concern: Fixed skill commitment time T could limit adaptability in dynamic tasks; learning termination could help.  
- Concern: Selecting which state variables to use for MI (C) may require domain insight; impact of noisy/less informative choices is unclear.  
- Question: Does estimating p(Z|c) from single-batch counts cause training instability, and did you try more stable estimators (e.g., running averages)?  
- Question: Does the mismatch between low-level skills trained on proprioception and a manager using full task sensors cause unpredictable behavior near obstacles?  
- Question: How to design a good proxy reward in non-locomotion domains (e.g., robotic manipulation) where movement alone is not the objective?

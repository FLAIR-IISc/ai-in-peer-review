- Introduces "nonparametric neural networks" that grow/prune units during a single training run using unit addition plus a group-l2 (fan-in) regularizer.  
- Theorem 1: proves a global minimum exists at finite network size with fan-in/fan-out regularization.  
- Proposes AdaRad optimizer that decouples radial (magnitude) and angular (direction) updates so new units can learn before being pruned.  
- Introduces CapNorm as a normalization detail that aids the method.  
- Empirical results: on benchmarks the method sometimes finds compact, high-performing architectures competitive with random search and better than same-size parametric nets.  
- Includes detailed training-dynamics analysis and a successful large-scale scalability test.  
- Decision: Accept.  
- Main justification: strong combination of rigorous theory (Theorem 1) and a practical, novel optimizer (AdaRad).  
- Main justification: thorough, transparent empirical evaluation including candid discussion of limitations.  
- Strength: addresses an important practical problem—automating layer width selection in one run to save compute.  
- Strength: novelty in unifying growth and pruning in a theoretically grounded framework; AdaRad is a genuine contribution.  
- Strength: technical quality—motivations clear, proof appears rigorous, experiments well-designed.  
- Weakness: inconsistent performance—notably fails on the 'convex' dataset with limited analysis of why.  
- Weakness: introduces new hyperparameters (unit addition rate ν, regularization λ, AdaRad step sizes αr and αφ) and relies on heuristics (αr = 150λ).  
- Suggestion: add analysis or hypotheses explaining the 'convex' failure mode (loss landscape, feature needs, or dynamics).  
- Suggestion: provide practical wall-clock time comparisons (nonparametric run vs full random-search cost).  
- Suggestion: experiment with smarter unit initialization strategies (e.g., error-driven initializations akin to boosting).  
- Suggestion: clarify AdaRad scaling claims—distinguish memory overhead (per-unit) from operations that depend on fan-in.  
- Questions for authors: how sensitive are results to the unit addition schedule ν and have adaptive schedules been tried?  
- Questions: justify and evaluate sensitivity of the heuristic αr = 150λ.  
- Questions: does the 'convex' failure stem from optimization (bad local minima/small network) or from model bias introduced by group-l2 regularization?

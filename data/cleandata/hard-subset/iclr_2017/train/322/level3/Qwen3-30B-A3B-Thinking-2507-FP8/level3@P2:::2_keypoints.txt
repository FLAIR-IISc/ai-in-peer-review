- Introduces a framework to automatically determine neural network size during a single training run by treating size as a continuous parameter that can grow/shrink.  
- Proves Theorem 1: with fan-in or fan-out L2 regularization the network converges to a finite size that minimizes training error.  
- Proposes a new training algorithm (AdaRad) and a modified normalization layer (CapNorm).  
- Experiments (MNIST, rectangles, convex datasets) show competitive performance versus exhaustive hyperparameter search and sometimes outperform fixed-size networks of the same final size.  

- Recommendation: accept with minor revisions.  

- Theorem 1 is a strong, well-motivated theoretical contribution addressing unbounded growth concerns.  
- Empirical results are promising, achieving similar performance with fewer parameters on some tasks.  
- Training dynamics analysis (Figure 3) supports the theoretical claims about network size evolution.  
- Paper is well-positioned relative to pruning, model selection, and regularization literature.  

- Add an intuitive, plain-English explanation and simple example for why L2 fan-in/fan-out prevents unbounded growth.  
- Clarify Figure 2 comparisons and provide a table with exact network sizes, validation, and test errors for each approach.  
- Add an experiment comparing nonparametric models to fixed-size networks with the same final size to isolate benefits of adaptive training.  
- Define “zero units” precisely and explain why both fan-in and fan-out must be zero to maintain network output.  

- Clarify how adding/removing zero units changes the L2 regularizer value (L2 of a zero vector is zero, so why does regularizer change?).  
- Does Theorem 1 extend to L1 regularization or only to L2, and why?  
- How sensitive is the final network structure to weight initialization and random seeds?  
- Discuss pros/cons of using fan-in vs. fan-out vs. both together, and whether the theoretical result still holds for combinations.  
- Is the regularization parameter λ predictable from data characteristics or must it be tuned, and how does it relate to final network size?

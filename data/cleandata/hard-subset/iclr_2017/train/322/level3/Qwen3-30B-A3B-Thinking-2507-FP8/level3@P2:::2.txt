

# Review for "NONPARAMETRIC NEURAL NETWORKS"

## 1. Summary of Contributions

This paper introduces a novel framework for automatically determining optimal neural network size during a single training run, eliminating the need for expensive global search over network architectures. The authors propose "nonparametric neural networks," where network size is treated as a continuous parameter that can grow and shrink during training. They prove a key theoretical result (Theorem 1) that with fan-in or fan-out L2 regularization, the network converges to a finite size that minimizes training error. The authors develop a novel training algorithm (AdaRad) and a modified normalization layer (CapNorm) to implement this framework. Experimental results on MNIST, rectangles images, and convex datasets demonstrate that nonparametric networks can achieve performance competitive with exhaustive hyperparameter search, and substantially outperform fixed-size networks of the same final size on some benchmarks.

## 2. Decision

I recommend acceptance with minor revisions. The paper presents a novel and theoretically grounded approach to network architecture selection that addresses a significant pain point in deep learning practice. The key theoretical contribution (Theorem 1) is well-motivated and provides a strong foundation for the proposed framework.

## 3. Supporting Arguments

The paper makes a meaningful contribution to the problem of automatic network architecture selection. The theoretical result (Theorem 1) is particularly compelling, as it rigorously establishes that the nonparametric approach will converge to a finite network size rather than growing indefinitely. This addresses a fundamental concern with approaches that dynamically grow networks.

The experimental results are promising, particularly on MNIST and rectangles images datasets, where the nonparametric approach achieves performance competitive with exhaustive hyperparameter search while using substantially fewer parameters. The analysis of the training process (Figure 3) provides valuable insight into how the network size evolves during training, which helps validate the approach's theoretical foundation.

The paper is well-motivated, with clear connections to related work on network pruning, model selection, and regularization. The authors effectively position their work as a solution to the two main drawbacks of black-box model selection: computational expense and inability to improve model performance during training.

## 4. Additional Feedback for Improvement

While the paper is strong, I have a few suggestions to improve clarity and impact:

- Theorem 1 is central to the paper, but its key insight is somewhat buried in the technical proof. I recommend adding a more intuitive, plain-English explanation of why the L2 regularizer with fan-in/fan-out constraints prevents unbounded growth, perhaps with a simple illustrative example.

- Figure 2 is confusing as presented. The comparison between nonparametric, parametric (same size), and parametric (best size) networks should be clearer. I recommend adding a separate table showing the exact network sizes, validation errors, and test errors for each approach.

- To better demonstrate the efficiency of the approach, please include an experiment comparing the nonparametric model to a fixed-size network with the same final size (not just the same initial size). This would more directly show whether the adaptive process provides performance benefits beyond just finding a good final size.

- The paper mentions "zero units" multiple times but doesn't clearly explain why both fan-in and fan-out must be zero to maintain network output. A more precise definition of "zero units" would strengthen the theoretical foundation.

## 5. Questions for Authors

To help me better understand and evaluate your work, I'd appreciate clarification on the following points:

1. The paper claims that adding zero units changes the value of the regularizer. However, with an L2 regularizer, adding a zero unit should not change the regularization term (since L2 norm of zero vector is zero). Could you clarify how the regularizer value changes when adding/removing zero units?

2. Theorem 1 is stated for L2 regularization, but the paper also mentions L1 as a potential alternative. Does the theoretical result hold for L1 regularization as well, or only for L2? If only for L2, why is this the case?

3. How sensitive is the final network structure to the initialization of weights and the random seed? Do different random starts yield substantially different network architectures and performance?

4. The paper suggests using only fan-in regularization, but the theoretical result applies equally to fan-out. Could you discuss the relative merits of using both fan-in and fan-out regularization together, and whether the theoretical result still holds?

5. The regularization parameter λ appears to be the primary control for final network size. Could you discuss whether λ can be predicted from data characteristics or if it requires tuning, and how it relates to the final network structure?

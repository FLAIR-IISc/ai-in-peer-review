- Decision: Accept — strong contribution with clear improvement over existing methods.  
- Proposal: UNREAL augments A3C with auxiliary control and reward-prediction tasks to learn richer representations.  
- Key idea: Simultaneously maximize multiple pseudo-reward functions to accelerate learning and improve performance.  
- Novelty: Combines diverse auxiliary tasks (control + reward prediction) learned simultaneously without using their policies.  
- Labyrinth results: 87% human-normalized performance and ~10x learning speedup.  
- Atari results: Average 880% of expert human performance, showing strong empirical gains.  
- Ablations: Studies show each auxiliary task contributes and combined effect is beneficial.  
- Robustness: UNREAL is less sensitive to hyperparameters than A3C.  
- Presentation: Well-written, clear technical exposition with useful appendix for implementation details.  
- Complexity: Architecture has many components and hyperparameters, making interactions and tuning challenging.  
- Practicality: Missing details on computational resources (CPUs/GPUs, training time) needed for replication.  
- Reproducibility: Releasing source code would greatly aid community adoption and follow-up work.  
- Question: Why does input reconstruction hurt performance — does it cause focus on irrelevant features or other issues?  
- Question: Did you try an LSTM-based reward-prediction network instead of a feedforward one, and what was the impact?

1. Summary of contributions
The paper proposes UNREAL, an A3C-based agent augmented with domain-independent auxiliary tasks that share the CNN+LSTM representation to improve feature learning under sparse extrinsic rewards. The main ideas are:
- Auxiliary control tasks: off-policy n-step Q-learning of separate policies for “pixel control” (maximizing local pixel-change in an n×n grid over the input) and “feature control” (maximizing activations of hidden units), trained from a shared replay buffer using deconvolutional heads.
- Reward-focused auxiliary tasks: a skewed-replay reward prediction head that predicts immediate reward (three-way classification), and value-function replay to accelerate value learning off-policy.
- A unified loss that combines on-policy A3C updates with off-policy auxiliary updates to train a shared representation, continuing to learn even when extrinsic rewards are absent.
Empirically, UNREAL substantially improves data efficiency, final performance, and hyperparameter robustness: 10× speedup and 87% human-normalized on Labyrinth, and new state-of-the-art on 57 Atari games (880% mean, 250% median expert human performance). The paper is clearly written, well positioned relative to Horde/UVFAs/successor representations/model-based auxiliary tasks, and includes ablations that illuminate the contributions of the proposed components.

2. Decision: Accept
Key reasons: (i) Strong, consistent empirical gains in both 3D Labyrinth and Atari with clear ablations; (ii) A simple, well-motivated and broadly applicable method that enhances representation learning in RL via auxiliary control and reward-focused prediction.

3. Supporting arguments
- Problem and motivation: The paper tackles representation learning in RL when extrinsic rewards are sparse and learning signals abundant in the sensorimotor stream. Rather than reconstructing observations, the agent learns to predict and control aspects of the stream via pseudo-reward functions that are better aligned with control and long-term goals. This is well motivated and distinct from prior unsupervised reconstruction objectives.
- Placement in the literature: The work is well connected to Horde (many value functions), UVFAs (factored value functions), successor representations (factored values under fixed policies), model prediction tasks, and recent 3D auxiliary-prediction work. The novelty lies in using off-policy auxiliary control tasks purely to shape a shared representation for the base policy/value, combined with reward prediction and value replay.
- Evidence and rigor: The empirical results are convincing: large improvements across 13 Labyrinth levels and 57 Atari games, with ablations showing contributions of pixel control, reward prediction, and value replay. The method improves both speed and robustness under substantial hyperparameter sweeps. Implementation details are adequate, and the evaluation protocol is clear. A minor weakness is that “feature control” has limited empirical validation, and the paper would benefit from more Atari ablations per component.

4. Additional feedback (to improve the paper; not all points affect the decision)
Clarity and correctness
- Clarify early (Abstract/Intro) that the agent learns separate policies for each auxiliary control task and that these are not used at test time, but solely to shape the shared representation.
- Off-policy updates: explicitly state that “Q-learning” uses n-step targets from replay (i.e., not strictly 1-step Q-learning), and explain why this choice is made.
- Loss definitions and notation:
  - Policy loss sign: Lπ is introduced as a term to be minimized; please write Lπ = −E[log π(a|s) Â] to avoid confusion.
  - Eq. 1: λC should be inside the summation (or made task-specific as λc) to reflect per-task weighting. Also, use r(t+k)^(c) (or r_{t+k}^{(c)}) rather than r_t^(c) immediately below Eq. 1.
  - Eq. 2: ensure LPC is explicitly defined in the equation and aligned with notation introduced earlier (sum over c of LQ^(c)).
  - Grammar: “the features discovered in this manner is shared” → “are shared.”
- Figure references and captions:
  - Reference Figure 2 in the main text; also reference Figure 1(d) in Section 3.3 when introducing replay/value replay.
  - Correct mismatches in Section 4.1.2 where “Figure 3 (right)” appears to refer to the top-left plot; unify left/right/top annotations consistently.
  - Figure 3: clarify what “Clip” denotes (e.g., reward clipping in Dueling DQN). Also clarify that percentages reported in the legend/panels refer to Labyrinth human-normalized performance.
  - Figure 5: the caption/text says “top 5 on three Labyrinth levels,” but the figure appears to show fewer; please fix the mismatch.
  - Section 4.2: add a comma after “methods.”
  - Appendix: specify where supplementary materials can be found.

Experimental scope and analysis
- Computational cost: quantify the wall-clock overhead and training speed impact of each auxiliary component (pixel control deconv head, reward prediction, value replay), and report GPU/CPU usage. Which auxiliary tasks dominate compute?
- Auxiliary weights: λRP and λVR appear fixed at 1; please justify this choice or provide sensitivity analyses. For λPC, report the selected values per domain (Atari/Labyrinth) and discuss how its scaling was chosen relative to pixel-change magnitude.
- Training horizon: several curves suggest performance was still improving. Please report longer runs and asymptotic performance for best configurations.
- Atari ablations: include per-component Atari ablations (pixel control only, reward prediction only, value replay only, and combinations) to assess whether contributions mirror Labyrinth.
- Feature-control stability: provide more empirical evidence for the “feature control” task (e.g., its ablation on more Labyrinth levels and on Atari), and discuss stability/convergence when the features and control targets co-evolve.

5. Questions for the authors
- Compute/efficiency: What is the wall-clock slowdown when adding pixel control, reward prediction, and value replay, respectively? How many extra updates per environment step does each add, and what is the total FLOPs increase?
- Sensitivity: How sensitive is performance to λRP, λVR, λPC, replay buffer size (2k), and the reward/non-reward skew for reward prediction? Did you try annealing λPC or disabling auxiliary control late in training?
- Feature control: Did you observe instability due to changing hidden activations as pseudo-rewards? Any evidence that feature control offers complementary benefits to pixel control on Atari?
- Generality: On games with low pixel dynamics (e.g., static screens), is pixel control still useful? How do you normalize pixel-change rewards across games to avoid domain-dependent scaling?
- Off-policy details: With n-step targets in the auxiliary Q-learning, did you observe any off-policy bias or divergence issues? Did importance sampling or target networks beyond θ− help?
- Asymptotic behavior: If trained longer, do auxiliary tasks remain beneficial, or do they mainly accelerate early learning?
- Options/utilization: Although auxiliary policies are not used at test time, did you explore using them as options or for exploration? Any preliminary results?

Overall, this work is a significant and practically valuable step toward leveraging auxiliary control and reward-focused prediction to improve deep RL data efficiency and robustness.

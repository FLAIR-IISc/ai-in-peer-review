- Introduces UNREAL: augmenting deep RL with auxiliary tasks to learn better feature representations.  
- Proposes pixel control (maximize changes in image regions) and feature control (maximize hidden-unit activations) as auxiliary control tasks.  
- Adds a supervised reward-prediction task learned from skewed replay to balance reward examples.  
- Empirical claim: UNREAL achieves 880% human-level on Atari (57 games) and 87% on Labyrinth, reaching baseline in ~10x fewer iterations.  
- Core insight: learning to control features via RL yields more useful representations than passive reconstruction, especially under partial observability.  
- Decision: Accept — significant contribution with strong empirical evidence across domains.  
- Addresses a meaningful limitation: RL typically learns only from extrinsic rewards while environments provide richer signals.  
- Ablation studies show pixel control outperforms pixel prediction/reconstruction, and feature control adds benefits.  
- Results are robust with appropriate baselines, hyperparameter analysis, and top-3 hyperparameter reporting.  
- UNREAL is a principled, natural extension of A3C that leverages auxiliary tasks to improve data efficiency and performance.  
- Clarify earlier definition of key losses (e.g., L_PC) rather than relegating them to the appendix.  
- Need more isolated analysis of each auxiliary task: why pixel control outperforms feature control and what specific features are learned.  
- Request deeper investigation of learned representations (metrics beyond RL performance, correlation with reward).  
- Clarify how the reward-prediction task differs from and complements the auxiliary control tasks.  
- Questions for authors: detail how pixel control differs from pixel-change prediction and why control helps; specify which hidden layer was used for feature control and why it underperforms; provide additional representation-quality metrics.

- Summary: Proposes UNREAL, an agent that learns auxiliary control (pixel and feature control) and reward-prediction tasks with skewed replay, yielding large gains (Atari 880% vs A3C 250%, Labyrinth 87% human) and ~10× faster learning.
- Decision: Recommend acceptance — significant RL contribution with strong, robust empirical results.
- Motivation/theory: Clearly distinguishes from prior work and argues convincingly that learning to control features (not just reconstruct) enables more flexible behavior.
- Empirical rigor: Strong experiments and ablations show each UNREAL component contributes, with robustness analyses and top-3 hyperparameter reporting addressing fairness concerns.
- Practical impact: Large speedup in data efficiency and compelling results on complex 3D Labyrinth tasks.
- Exposition critique: Key auxiliary-task details (L_PC and implementation differences between pixel vs feature control) should be moved into the main text for clarity.
- Analysis critique: Needs deeper, quantitative analysis of the auxiliary tasks themselves (which features are learned, metrics linking auxiliary-task performance to RL performance) and why pixel control outperforms feature control.
- Question 1: Provide specific metrics quantifying why controlling pixel changes is better than predicting immediate pixel changes (e.g., control-task accuracy, correlation with main-task performance).
- Question 2: Clarify how the "input change prediction" baseline relates to the reward-prediction task — are they predicting the same immediate reward or something different?
- Question 3: Explain how auxiliary tasks specifically drive exploration in Labyrinth (e.g., promoting maze layout discovery) versus merely improving representations.
- Question 4: Explain why reconstruction sped initial learning but worsened final scores here, despite prior work showing reconstruction can help in 3D environments.

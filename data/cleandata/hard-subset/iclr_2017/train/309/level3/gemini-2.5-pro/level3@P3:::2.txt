### **Review Summary**

This paper introduces UNREAL (Unsupervised Reinforcement and Auxiliary Learning), an agent that significantly enhances deep reinforcement learning by incorporating several domain-independent auxiliary tasks. The agent's core policy is trained with A3C, while its shared feature representation (a CNN followed by an LSTM) is simultaneously shaped by off-policy auxiliary tasks: learning to control pixel changes and network features, predicting immediate rewards, and replaying value function updates. This approach is shown to dramatically improve both data efficiency and final performance over a strong A3C baseline on the challenging Atari suite and a new, visually rich 3D environment, Labyrinth. The paper is well-written, the method is well-motivated, and the empirical results are impressive and convincing.

### **Decision**

**Clear Accept.**

The key reasons for this decision are:
1.  The paper presents a novel and effective architecture that successfully combines on-policy learning for the main objective with off-policy learning for several auxiliary tasks, leading to substantially improved representation learning.
2.  The claims are supported by extensive and rigorous experiments on two complex domains, demonstrating state-of-the-art performance, a remarkable ~10x speedup in learning, and improved robustness to hyperparameters.

### **Supporting Arguments**

The UNREAL agent's design is a powerful synthesis of recent advances in deep RL. By using A3C for the primary policy, it retains the benefits of stable, on-policy, recurrent learning. Simultaneously, it leverages the data efficiency of off-policy learning from a replay buffer for its auxiliary tasks. This hybrid on-policy/off-policy approach is a key strength and allows the agent to learn rich, general-purpose features from its stream of experience, even when extrinsic rewards are sparse.

The auxiliary tasks themselves are intuitive and general. The control tasks encourage the agent to learn about the causal structure of the environment, while reward prediction and value replay focus the representation on aspects relevant to the main objective. The empirical validation is thorough and compelling. The introduction of the Labyrinth environment is a valuable contribution, providing a challenging new benchmark for the community. The ablation studies on Labyrinth clearly demonstrate the additive benefits of each component. The generalization of these benefits to the 57-game Atari suite, where UNREAL sets a new state-of-the-art, strongly supports the paper's central claims.

### **Additional Feedback and Questions for the Authors**

The following points are intended to help improve the paper and are not critical to my decision.

**Major Points:**
*   **Computational Cost:** The addition of multiple auxiliary tasks presumably increases computational cost. Could you quantify this overhead? For instance, what is the wall-clock training time of UNREAL compared to the A3C baseline? Are certain auxiliary components (e.g., the deconvolutional network for pixel control) significantly more expensive than others?
*   **Feature Control Validation:** The empirical support for the feature control task feels preliminary. Figure 5 shows results on only one level, and the learning curve appears less stable than for pixel control. This raises a potential concern about convergence, as the control targets (the network activations) are non-stationary. Further analysis or experiments on this task would strengthen the paper.
*   **Hyperparameter Choices (λ weights):** The weights for value replay (λ_VR) and reward prediction (λ_RP) are fixed at 1.0. Could you comment on this choice? Were other values tested? It seems these could also be treated as tunable hyperparameters, similar to λ_PC.
*   **Atari Ablation Study:** The ablation studies on Labyrinth are excellent. To further solidify the claims, it would be highly valuable to see a similar ablation study on a subset of Atari games to confirm that the contributions of each auxiliary task generalize across domains.
*   **Training Duration:** The learning curves in Figure 3 suggest that agent performance was still climbing at the end of training. It would be interesting to know if performance saturates with longer training and to report the peak performance achieved by the best agents.

**Minor Points (Clarity and Corrections):**
*   **Clarity on Auxiliary Policies:** In the abstract and introduction, it would be helpful to explicitly state that the auxiliary control tasks learn separate policies (or, more accurately, value functions), rather than having a single agent policy that optimizes a weighted sum of extrinsic and pseudo-rewards. Equation 1 could be misinterpreted in this way.
*   **Off-Policy Method:** Please clarify that the auxiliary tasks are trained using n-step returns from off-policy data, which is an efficient approximation but differs from "true" 1-step Q-learning.
*   **Figure/Text Corrections:**
    *   Figure 2 is not referenced in the text. Figure 1(d) should be referenced in Section 3.3.
    *   The figure references in Section 4.1.1 are confusing (e.g., "Figure 3 (right)" appears to refer to the top-left plot). Please use explicit labels like "top-left", "top-right", etc.
    *   The legend of Figure 3 (top left) should clarify that the percentages (e.g., "UNREAL 87%") refer to mean scores on Labyrinth.
    *   The legend of Figure 3 (bottom right) mentions "Clip" for the Dueling DQN baseline. Please specify what this refers to (e.g., reward clipping).
    *   The text for Figure 5 claims results are shown for "three Labyrinth navigation levels," but the plot appears to show only two. Please correct this inconsistency.
*   **Equation/Formula Corrections:**
    *   Equation 1: The weight λ_C should be inside the summation over c ∈ C.
    *   Below Equation 1: The auxiliary return should likely use r_{t+k}^{(c)}, not r_t^{(c)}.
    *   Equation 2: L_PC is used but not explicitly defined in terms of the individual L_Q^(c) losses.
    *   The policy loss L_pi, derived from A3C, is presented as a minimization, but the underlying objective is a maximization. Using a negative sign (e.g., L_pi = -E[...]) would make this clearer.
*   **Typos and Grammar:**
    *   Section 3.2: "...the features discovered in this manner is shared..." should be "...are shared."
    *   Section 4.2: Missing comma after "...for all three methods".
    *   Appendix: The note "Further details are included in the supplementary materials" is vague. Please specify where these materials can be found.

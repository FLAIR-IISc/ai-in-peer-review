This paper introduces UNREAL, a deep reinforcement learning agent that enhances representation learning by incorporating several auxiliary tasks alongside the main policy objective. The core idea is that by training an agent to control and predict features of its own sensorimotor stream, it can develop a richer and more versatile representation that accelerates learning on the primary reward-maximization task. The proposed auxiliary tasks are domain-independent and include: 1) auxiliary control tasks (pixel control and feature control), where separate policies are learned to maximize changes in pixel values or hidden unit activations; 2) an auxiliary reward prediction task, which predicts the onset of immediate rewards from recent history; and 3) value function replay, which provides additional value-learning updates from a replay buffer. These tasks share the convolutional and recurrent layers of the main A3C-based agent, allowing the benefits of these unsupervised and self-supervised objectives to transfer to the main policy. The contributions are evaluated on the challenging 3D Labyrinth environment and 57 Atari games, showing substantial improvements over state-of-the-art baselines.

**Decision: Accept**

The primary reasons for this recommendation are the compelling empirical results demonstrating significant improvements in both data efficiency and final performance over a strong baseline (A3C) on two challenging domains, and the novel, domain-agnostic approach of using auxiliary control tasks for representation learning. The paper is well-written, the method is clearly explained, and the results are convincing.

**Supporting Arguments**

The paper's claims are well-supported by extensive experiments. The UNREAL agent achieves an average 10x learning speedup on Labyrinth tasks while increasing the mean human-normalized score from 54% (A3C) to 87%. On Atari, the agent achieves a mean of 880% expert human performance, substantially outperforming previous state-of-the-art methods. Beyond raw performance, the paper convincingly demonstrates that adding auxiliary tasks improves the agent's robustness to hyperparameter settings, a significant practical contribution. The ablation studies on Labyrinth clearly show that each auxiliary component contributes positively to the final performance, justifying their inclusion. The comparison against more traditional unsupervised tasks like input reconstruction in Figure 5 further strengthens the claim that learning to *control* aspects of the environment is a more effective auxiliary objective for RL than simply learning to *predict* or *reconstruct* them.

While the overall results are strong, the empirical validation for the "feature control" auxiliary task is less convincing than for the other components. The results in Figure 5 are preliminary and show only a slight improvement. This raises a potential concern about the stability of this approach: the agent is learning a policy to control hidden unit activations that are themselves changing as the network trains. It is not clear if or how this process converges, and the provided evidence is not sufficient to fully validate this specific component of the work. However, this is a minor point, as the pixel control task is well-validated and the overall UNREAL agent's performance is excellent even without feature control.

**Additional Feedback and Questions for the Authors**

The following points are intended to help improve the paper and are not central to my decision. I would appreciate clarification on these points in the author response.

*   **Computational Cost:** The auxiliary tasks introduce additional computational overhead. Could you quantify this cost? For example, how much slower is training a UNREAL agent compared to a baseline A3C agent in terms of wall-clock time or updates per second? Which auxiliary components are the most computationally expensive?
*   **Hyperparameter Choices:** The weights for the auxiliary losses (λVR, λRP) appear to be fixed at 1, while λPC is tuned. Was there any experimentation to validate the choice of λVR = 1 and λRP = 1, or could these also be treated as tunable hyperparameters?
*   **Training Duration:** In many of the learning curves (e.g., Figure 3), the UNREAL agent's performance is still clearly trending upwards at the end of training. This suggests the reported scores may not represent its final asymptotic performance. I would suggest either training for longer or at least commenting on this and reporting the performance of the best agents after more training steps.
*   **Atari Ablation Study:** The ablation study in Labyrinth is very insightful. A similar study on Atari would be valuable to understand if the contributions of each auxiliary component are consistent across different domains.
*   **Clarity on Policies:** The abstract states the agent "learns separate policies for maximising many other pseudo-reward functions". This could be misinterpreted as a single policy being learned to optimize a weighted sum of rewards. It would be clearer to state explicitly in the abstract or introduction that separate value functions/policy heads are learned for each auxiliary task, all sharing a common representation.

**Minor Corrections and Clarifications:**

*   **Off-Policy Method:** The auxiliary control tasks are trained using off-policy Q-learning updates from a replay buffer. However, since these updates use n-step returns computed from the buffer, it's not "true" Q-learning but rather an off-policy n-step value learning method. It would be more precise to mention this.
*   **Equation 1:** The λC term should be inside the summation. Also, in the definition of R(c), the reward term should be `r(c)t+k`.
*   **Equation 2:** The loss function references L_PC, but this term is not explicitly included in the equation itself, which instead shows the summation over individual L(c)Q terms. This could be clarified. The value of λPC used should also be explicitly stated.
*   **A3C Loss:** The policy loss `Lπ` in the description of LA3C is typically a maximization objective (i.e., maximizing log π * Advantage), but here it is presented as a component of a loss function to be minimized, which would imply `Lπ = - E[...]`. This should be written consistently.
*   **Figure References:** Figure 2 is not referenced in the main body text. Figure 1(d) should probably be referenced in Section 3.3 when discussing value function replay. In Section 4.1.1, the reference to "Figure 3 Top Right" seems to point to the top-left plot. Please ensure all figure references are correct.
*   **Figure 3 Legend:** In the bottom-right plot, what does "Clip" refer to for the Dueling DQN agent? The percentages (e.g., "54% (A3C)") in the top-left legend seem to refer to the final mean performance on Labyrinth; this should be made explicit.
*   **Figure 5:** The text says the figure shows results on "three Labyrinth navigation levels", but the plot seems to only show two (nav_maze_static_01 and nav_maze_random_goal_02). Please clarify.
*   **Grammar/Typos:** In Section 3.2: "the features discovered in this manner is shared" should be "are shared". In Section 4.2: "...for all three methods the right half..." should have a comma after "methods".
*   **Appendix:** A note in the appendix mentions "Further details are included in the supplementary materials." Please specify where these materials can be found.

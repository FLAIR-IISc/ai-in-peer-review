- Introduces UNREAL: an agent using self-supervised auxiliary tasks to improve sample efficiency, final performance, and robustness in deep RL.  
- Proposes auxiliary control tasks (pixel control, feature control) that learn off-policy value functions to maximize internal pseudo-rewards.  
- Introduces auxiliary prediction tasks (reward prediction, value replay) with prioritized replay to focus representation on task-relevant features.  
- Combines on-policy A3C for the main policy with off-policy learning for auxiliary tasks via a shared replay buffer.  
- Demonstrates strong empirical results on 57 Atari games and the Labyrinth 3D environment.  
- Achieves state-of-the-art Atari performance (avg 880% human) and Labyrinth (87% human).  
- Reports a ~10x average learning speedup on Labyrinth versus a strong A3C baseline.  
- Decision: Accept.  
- Key reason: Large performance and sample-efficiency gains addressing a core RL challenge.  
- Key reason: Novel, well-motivated architecture emphasizing control-based auxiliary objectives over passive prediction.  
- Well-situated in prior work, distinguishing itself by using pseudo-reward policies for shared representation learning rather than separate value functions.  
- Empirical rigor: strong benchmarks, comprehensive ablations showing each component helps and the full model is best.  
- Direct hypothesis tests show pixel control outperforms pixel prediction/reconstruction for representation learning.  
- Robustness analysis shows UNREAL is more stable across hyperparameter settings.  
- Feedback: System complexity with multiple loss weights—need discussion or guidance on setting λ hyperparameters.  
- Feedback: Clarify computational/wall-clock cost trade-offs of auxiliary tasks versus environment-step efficiency.  
- Feedback: Feature control results are preliminary; expanding them would strengthen claims about generality.  
- Question: Is pixel-control pseudo-reward robust to dynamic textures/noise, or would CNN-feature differences be better?  
- Question: Were prioritized replay schemes applied to other off-policy tasks (e.g., high TD-error prioritization for value replay/pixel control)?  
- Question: Could learned auxiliary control policies be leveraged for exploration or temporal abstractions/options rather than only for representation learning?

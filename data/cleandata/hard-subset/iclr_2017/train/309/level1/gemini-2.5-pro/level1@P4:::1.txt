### 1. Summary

This paper introduces UNREAL (Unsupervised Reinforcement and Auxiliary Learning), a novel deep reinforcement learning agent designed to improve learning speed, final performance, and robustness. The core contribution is the augmentation of a strong baseline agent (A3C) with a set of auxiliary tasks that are learned simultaneously with the main policy. These tasks are designed to build a richer and more versatile representation by leveraging the full sensorimotor stream, rather than focusing solely on the extrinsic reward signal.

The proposed auxiliary tasks are of two main types. First, **auxiliary control tasks** frame representation learning as a set of control problems, where the agent learns policies to maximize pseudo-rewards. Two specific instances are explored: "pixel control," which involves maximizing pixel changes in different regions of the input image, and "feature control," which involves maximizing the activations of hidden units in the agent's own network. These tasks are trained off-policy using Q-learning. Second, **auxiliary reward tasks** aim to focus the representation on important events. The paper introduces a "reward prediction" task, where the agent predicts the next immediate reward from a short sequence of observations. This is trained as a supervised classification problem on data rebalanced via experience replay to oversample rewarding events.

The UNREAL agent architecture integrates these components by combining the on-policy A3C algorithm for the main task with off-policy updates from an experience replay buffer for the auxiliary tasks and an additional value function regression loss. The paper provides extensive empirical validation on the 57-game Atari benchmark and a new, challenging 3D first-person environment called Labyrinth. The results show that the UNREAL agent achieves state-of-the-art performance, averaging 880% of expert human scores on Atari and 87% on Labyrinth, while demonstrating a mean learning speedup of 10x over the A3C baseline.

### 2. Decision and Key Reasons

**Decision:** Accept

1.  **Significant Conceptual Contribution:** The paper presents a novel and well-motivated framework for integrating self-supervised auxiliary objectives into a deep RL agent. The introduction of *control-based* auxiliary tasks (pixel/feature control) is a notable departure from more common predictive or reconstructive auxiliary tasks and proves to be highly effective. This reframing of representation learning as a collection of control problems is a valuable contribution to the field.
2.  **Strong and Rigorous Empirical Support:** The claims are substantiated by extensive experiments across two complex domains. The performance improvements over a state-of-the-art baseline (A3C) are substantial, not incremental. The ablation studies are thorough, clearly isolating the benefits of each auxiliary component and comparing the proposed control tasks against more traditional unsupervised methods like input reconstruction.

### 3. Supporting Arguments

The paper's approach is well-motivated by the problem of data inefficiency and sparse rewards in RL. The use of auxiliary tasks to shape a shared representation is a logical solution. The technical implementation is sound, effectively combining an on-policy policy-gradient method (A3C) for the main objective with more data-efficient off-policy value-based methods for the auxiliary tasks, all learned from a single stream of experience. This hybrid on-policy/off-policy design is a strength of the proposed architecture.

The empirical results are compelling. Achieving an average score of 880% on Atari establishes a new state of the art. The introduction of the visually rich and physically complex Labyrinth environment is itself a contribution, and the agent's high performance (87% human score) demonstrates the generality of the approach beyond 2D arcade games. The reported 10x speedup in learning on Labyrinth is a key practical result, directly addressing the data-efficiency problem. The ablation experiments in Section 4.1.1 provide clear evidence that learning to control pixel changes is more beneficial than merely predicting them or reconstructing the input, supporting the central hypothesis of the paper. Furthermore, the analysis of hyperparameter robustness (Figure 3) shows that the UNREAL agent is not only higher-performing but also more stable to train than the A3C baseline, which is a significant practical advantage.

### 4. Additional Feedback

These points are intended to help improve the paper and are not critical to the decision.

*   **Terminology:** The subtitle "Unsupervised Reinforcement Learning" in Section 4.1.1 could be clarified. While the pseudo-rewards for pixel control are generated without human labels (and are thus "unsupervised" in that sense), the agent still solves a reinforcement learning problem. This is distinct from more traditional unsupervised learning methods like autoencoders. A more precise phrasing might avoid potential confusion.
*   **Generality of Hyperparameters:** The weight for the pixel control loss, `lambda_PC`, was set differently for Atari and Labyrinth to account for differences in pixel dynamics. It would strengthen the method's generality if a normalization scheme for the pixel change pseudo-reward could be developed, potentially removing the need for this domain-specific hyperparameter tuning.
*   **Feature Control:** The results for feature control are presented as "preliminary." While the initial results are promising, a more complete analysis integrating it with the other auxiliary tasks into the full UNREAL agent would provide a fuller picture of its potential.

### 5. Questions for Authors

*   For the pixel control auxiliary task, a deconvolutional network head is used. Was this architectural choice critical? How does this compare to simpler alternatives, for instance, a shared MLP followed by independent linear heads for each grid cell's Q-values?
*   The reward prediction task relies on rebalancing the experience replay buffer to handle sparse rewards. How would this component adapt to environments with dense rewards where non-zero rewards are common? Would the rebalancing strategy still be effective or necessary?
*   The paper notes that the auxiliary control policies are used only to shape the representation and are not executed. Have you considered using these policies more directly, for example, as a source of intrinsic motivation or as options for hierarchical exploration?
*   The paper demonstrates robustness to learning rate and entropy cost. Could you comment on the sensitivity of the overall performance to the relative weights of the auxiliary losses (`lambda_VR`, `lambda_PC`, `lambda_RP`)? How critical is the tuning of these new hyperparameters?

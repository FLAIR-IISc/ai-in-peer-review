Summary and contributions
The paper proposes UNREAL, a deep RL agent that augments a standard A3C policy/value learner with auxiliary tasks to accelerate and stabilize representation learning. The auxiliary tasks are:
- Control tasks: off-policy n-step Q-learning on pseudo-rewards, notably “pixel control” (maximize local pixel changes in an n×n grid) and “feature control” (maximize activation of internal units).
- Reward-focused tasks: a supervised reward prediction head that predicts immediate reward from recent frames, with replay sampling biased to oversample rare rewards.
- Value replay: additional value function regression from replayed sequences.

All auxiliary tasks share the CNN (and LSTM for control), shaping the representation even when extrinsic rewards are sparse. Empirically, UNREAL delivers large gains in both final performance and data efficiency on 13 3D Labyrinth tasks (mean 10× speedup to A3C’s best) and across 57 Atari games (880% mean human-normalized performance), with improved robustness to hyperparameters.

Decision: Accept
Primary reasons:
- Strong, consistent empirical improvements in challenging visual-control domains, with careful ablations on Labyrinth and meaningful Atari comparisons.
- A well-motivated, literature-grounded approach that unifies representation shaping via auxiliary control and reward prediction with on-policy A3C, yielding clear data-efficiency benefits.

Supporting arguments
1) What problem is tackled? The paper addresses how to learn useful representations for RL when extrinsic rewards are sparse, by leveraging diverse training signals that align with control and reward prediction rather than pure reconstruction.

2) Is the approach well motivated and positioned? Yes. The method connects to options/pseudo-rewards, Horde, UVFA, successor representations, and model-based auxiliary prediction. The novelty lies in using many off-policy control tasks and biased reward prediction to shape the shared representation, while keeping the main policy on-policy and recurrent.

3) Do results support the claims? The Labyrinth experiments convincingly show large gains in speed and final scores, with ablations indicating contributions from pixel control, reward prediction, and value replay. Atari results show strong average improvements and robustness. The methodology (parallel A3C; replay for auxiliary tasks; explicit loss composition) is sound. One caveat: the auxiliary Q-learning is off-policy with n-step returns, which deviates from “true” 1-step Q-learning and may introduce bias; this should be acknowledged clearly (the authors note this elsewhere).

Additional feedback to improve the paper (helpful suggestions; not all affect my decision)
- Computational cost and wall-clock: Please quantify the extra compute and memory due to auxiliary heads and replay. How much slower per environment step is UNREAL versus A3C? Which auxiliary components dominate the cost?
- Clarify multi-policy learning: The abstract/intro could more clearly state that the agent learns separate policies/values for each auxiliary control task (rather than a single policy for all pseudo-rewards). Equation (1) helps but an earlier, explicit statement would avoid confusion.
- Feature control stability: The target features evolve as the network trains, which could create moving-target instability. Please discuss any stabilizing tricks (e.g., target networks, slower updates, normalization) and expand empirical validation beyond the preliminary result in Figure 5.
- Continue training to asymptote: Since the curves are still rising, consider extending training for best agents to show asymptotic performance and verify that the relative gains persist.
- Hyperparameters for auxiliary losses: Explain why λVR, λPC, λRP were fixed rather than tuned, and provide sensitivity analysis. If tuned in a range (as for λPC), report chosen ranges and default values per domain and how they affect results.
- Off-policy n-step Q-learning: Explicitly note that the auxiliary control tasks use off-policy, multi-step bootstrapping without importance sampling and discuss the expected bias/variance trade-offs and why this works well in practice.
- Atari ablations: Add more ablations on Atari (e.g., UNREAL minus pixel control; minus reward prediction; minus value replay) to assess whether contributions mirror Labyrinth.
- Replay details: Report replay buffer sizes per domain, sampling ratios for each auxiliary head, and any prioritization scheme beyond reward-balancing for prediction.

Editorial/clarity suggestions
- Policy gradient loss: “Policy gradient algorithms adjust the policy to maximise the expected reward” — Lπ is written as a loss to be minimized; clarify sign conventions.
- Equation (1): λC should either be inside the sum as λc (task-specific) or clearly defined as a scalar multiplying the sum. Also, use r(c)t+k (not r(c)t ) in R(c)t:t+n.
- Below Eq. (1): fix r(c)t+k superscripts/subscripts as noted.
- Define LPC: The text around Eq. (2) references LPC; make sure it is defined explicitly and appears in Eq. (2).
- Clarify “Clip” in Figure 3 legend for the dueling architecture.
- Figure references: Reference Figure 2 in the text; reference Figure 1(d) in Section 3.3; in Section 4.1.2 “Figure 3 (right)” should be “Figure 3 (top left)” and “Figure 3 Top” should be “Figure 3 Top Right.”
- Figure 3 legend: Make explicit that percentages reported there refer to Labyrinth results.
- Figure 5: The text says “three Labyrinth navigation levels,” but the plotted panels appear to address two (left and middle). Please reconcile.
- Section 4.2 sentence: “The left side shows the average performance curves of the top 5 agents for all three methods the right half shows ...” — add punctuation after “methods.”
- Appendix: If you mention “Further details are included in the supplementary materials,” provide a pointer or remove the claim.
- Notation/grammar: “the features discovered in this manner is shared” → “are shared.”
- Report λPC default/value(s) used per domain.

Questions for the authors
- What is the wall-clock training time and throughput relative to A3C for Labyrinth and Atari? How do compute costs scale with the number of auxiliary tasks?
- How sensitive are results to λVR, λPC, and λRP? Did you try learning per-task weights (e.g., uncertainty-based weighting) or normalizing task losses?
- For feature control, did you try slowing target feature updates (e.g., target network) or using layer normalization to mitigate drift? Any evidence of instability?
- How large is the replay buffer, how frequently are each auxiliary head’s updates applied per env step, and what are the sampling ratios? Did you test prioritized replay beyond reward balancing?
- How does performance vary with the pixel-control grid resolution and the normalization of pixel-change magnitudes across Atari games?
- Did you explore importance sampling or Retrace-style corrections for the off-policy n-step Q-learning on auxiliary tasks, and if so, with what effect?
- If training is extended further, do UNREAL’s gains persist, and what is the asymptotic gap versus A3C?

Overall, this is a strong, well-executed paper with clear contributions and substantial empirical impact. I recommend acceptance.

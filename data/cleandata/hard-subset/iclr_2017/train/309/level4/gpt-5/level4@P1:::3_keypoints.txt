- Proposal: augment RL agents with auxiliary tasks (pixel control, feature control, and reward prediction) learned from a shared representation to improve features for the main policy/value learning.  
- Auxiliary control tasks: control pixel-intensity changes on an n×n grid and control internal network activations, optimized off-policy with n-step Q-learning.  
- Auxiliary supervised task: predict immediate reward from short observation sequences using replay with skewed sampling to balance reward classes without biasing main updates.  
- Empirical results: large gains on visually rich discrete-action domains (Labyrinth and Atari), achieving comparable or better performance with roughly 10× fewer iterations.  
- Conceptual claim: control-based auxiliary objectives align better with downstream RL goals than passive reconstruction or model-based objectives under partial observability.  
- Decision: Accept.  
- Main reasons for accept: strong empirical improvements in data efficiency and robustness; novel, well-motivated use of control-based auxiliary objectives.  
- Novelty/motivation support: using pseudo-reward control problems to shape representations addresses reconstruction’s tendency to emphasize irrelevant visual details.  
- Methodological soundness: combining off-policy n-step Q-learning for auxiliaries with on-policy A3C and shared CNN/LSTM features is coherent; top-k hyperparameter evaluation helps fairness.  
- Empirical strength/supporting ablations: reported 10× speedup and ablations indicate pixel/feature control drives much of the benefit and UNREAL’s combination performs best.  
- Request for more implementation detail: include auxiliary control loss definitions (e.g., LPC), pseudo-reward formulas, n-step targets, and any target-network specifics in the main text.  
- Deeper experimental analysis recommended: analyze representational changes, linear separability/value prediction accuracy, and more granular per-task ablations (pixel vs feature vs both).  
- Compare objectives: provide broader analysis comparing control vs prediction vs reconstruction to strengthen claims that control objectives are better aligned with RL.  
- Clarify skewed sampling: explain how reward-prediction skewing affects learning stability and whether similar skewing was tried for other tasks.  
- Move practical choices to main text: report replay buffer size, sampling strategy, update frequencies, and other implementation details central to performance.  
- Questions for authors: state full LPC loss and pseudo-rewards/n-step targets/target-net scheme; report sensitivity to the reward/non-reward sampling ratio; provide results for feature control alone and combined across more tasks; share intermediate metrics (value loss, reward-prediction accuracy, exploration behavior); discuss generality across visual statistics and action spaces and cases where pixel control may fail.

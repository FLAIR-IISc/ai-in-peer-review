

This paper proposes enhancing feature learning in deep reinforcement learning through auxiliary tasks that shape the shared feature extractor (CNN+LSTM). The auxiliary tasks include state-space modification (feature control), immediate reward prediction, and value function replay—though the latter is used for experience replay rather than feature shaping. The approach demonstrates improved data efficiency over A3C on Atari and Labyrinth benchmarks. The paper is well-structured, the core idea is sound, and results are compelling, warranting a clear acceptance. Below are minor concerns for clarification and refinement:

**Major Concerns**  
1. **Computational Overhead**: The paper should quantify the additional computational cost of optimizing auxiliary tasks. Specifically, how much does training speed decrease, and which components (e.g., feature control, reward prediction) contribute most to this overhead?  
2. **Policy Clarity**: The abstract/intro states the agent "maximises many other pseudo-reward functions simultaneously," which risks misinterpretation as a single policy optimizing all rewards. Clarify that the agent learns *distinct policies* for each auxiliary task (e.g., "the agent learns separate policies for each auxiliary task via shared feature extraction").  
3. **Feature Control Validation**: The empirical validation of feature control (Fig. 5) is weak—initial gains are marginal and not sustained. Given the task dynamics modify features during learning, stability/convergence concerns arise. A stronger justification or additional validation is needed.  
4. **Hyperparameter Sensitivity**: Why are auxiliary task weights (λ_*) fixed at 1 instead of optimized? Were ablations tested to confirm this choice? If λ_* is critical, this should be addressed.  
5. **Value Function Replay**: Note that auxiliary tasks are trained *off-policy* with multi-step returns (not true Q-learning), as clarified in OpenReview comments. This distinction should be explicitly stated.  

**Additional Comments**  
- **Equations & Notation**:  
  - Eq. 1: λ_c should be inside the sum.  
  - Below Eq. 1: r_t^(c) should be r_{t+k}^(c).  
  - Eq. 2: The loss L_PC is referenced but undefined; ensure consistency with the equation.  
- **Figures & Text**:  
  - Figure 2 is not referenced in the main text.  
  - Figure 1(d) must be cited in Section 3.3.  
  - Figure 3: The "top" reference in Section 4.1.2 should specify "top-left plot" (not "top").  
  - Figure 3 legend: Clarify that % values apply to Labyrinth (not Atari).  
  - Figure 5 caption: Correct "three levels" to "two levels" (left/middle plots only).  
- **Grammar & Clarity**:  
  - "The features discovered in this manner is shared" → "are shared."  
  - "Policy gradient algorithms adjust the policy to maximise the expected reward, L_π = -..." → Clarify L_π is a *minimization* loss (not maximization).  
  - "Clip" in Figure 3 legend: Define its role for dueling networks.  
- **Experiments**:  
  - Ablation studies on Atari (beyond Labyrinth patterns) would strengthen claims.  
  - The text notes "performance is steadily improving"; consider extending training to observe long-term gains (e.g., for top agents).  
- **Appendix**: Specify where supplementary materials are located (e.g., "See Appendix B" or "Supplementary materials are provided at [URL]").  
- **Hyperparameter**: Confirm λ_PC = 1 (implied but not stated).  

**Summary**  
The paper makes a strong contribution to feature learning in RL with a clear, well-executed approach. The minor concerns above are primarily for precision and completeness—none undermine the core contribution or results. Addressing these would elevate an already solid submission. The results on data efficiency (especially relative to A3C) are particularly convincing. This is a clear acceptance.

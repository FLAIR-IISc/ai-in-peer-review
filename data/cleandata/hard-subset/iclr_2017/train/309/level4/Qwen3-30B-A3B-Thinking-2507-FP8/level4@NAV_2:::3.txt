

This paper introduces UNREAL, a novel reinforcement learning agent that learns multiple auxiliary control tasks (pixel control and feature control) alongside the primary reward maximization objective, while incorporating reward prediction to enhance representation learning. The authors demonstrate that this approach significantly improves data efficiency and performance on both Atari and Labyrinth benchmarks, achieving 880% expert human performance on Atari and 87% on Labyrinth with a 10× mean learning speedup.

Accept. The paper makes a strong technical contribution by rigorously demonstrating that auxiliary control tasks—learning to maximize changes in pixel intensities and latent feature activations—outperform traditional unsupervised methods (e.g., pixel reconstruction) and substantially accelerate learning. The experimental validation across diverse environments (Atari, Labyrinth) with robust hyperparameter analysis is compelling.

The core innovation lies in framing auxiliary tasks as *control objectives* rather than passive reconstruction, which better aligns with the agent’s goal-directed behavior. The UNREAL agent’s 10× speedup and 87% human-normalized performance on Labyrinth—surpassing prior work by 45%—are robustly supported by ablation studies showing each auxiliary task contributes meaningfully. The reward prediction task, trained via skewed replay to balance sparse rewards, further stabilizes learning without biasing the primary policy. The method is well-placed in literature, contrasting with prior work (e.g., Horde, UVFA) by using shared representations for auxiliary tasks rather than separate weights, and avoids the pitfalls of model-based learning in partially observable settings.

To strengthen the paper, explicitly quantify the marginal contribution of each auxiliary task (e.g., pixel control vs. feature control) in isolation, particularly given their observed dominance in Labyrinth (A3C+PC already outperforms A3C+feature control). Ablation studies should isolate how each task affects feature learning (e.g., via activation analysis) rather than only reporting final performance. The draft’s high-level description of L_PC (loss for pixel control) in Section 3.1 would benefit from a brief formal definition before Appendix B to improve reproducibility. Additionally, clarify whether the 10× speedup metric accounts for A3C’s asynchronous parallelism or is computed against a single-threaded baseline.

- Proposes domain-independent auxiliary tasks (state control, reward prediction, value replay) sharing CNN+LSTM features to improve RL feature learning and data efficiency.  
- Demonstrates significant, generally convincing improvements over A3C on Atari and Labyrinth; writing is clear and idea is strong.  

Major concerns:
- Quantify computational overhead: report training speed decrease and which components (extra heads, backward passes) drive cost.  
- Clarify policy interpretation: explicitly state that separate policies are learned for each auxiliary task and only the feature extractor is shared, to avoid implying a single policy optimizes all rewards.  
- Validate feature-control auxiliary task: provide stronger empirical evidence (ablation showing contribution to final performance and analysis of stability/convergence as features change).  
- Justify auxiliary weights λ_k fixed to 1: report experiments or tuning rationale; consider hyperparameter search or meta-learning for λ_k.  

Minor issues & typos:
- Fix Eq.1 notation: ensure λ_k is inside the sum and correct rt^(c) to r_{t+k}^{(c)}; define L_PC where referenced.  
- Correct figure references and captions: reference Figure 2 in text, fix Figure 1(d) citation, correct Figure 3 location and legend (% refers to Labyrinth), and clarify "left and middle plots" wording.  
- Grammar fixes: e.g., "the features discovered in this manner is shared" → "are shared"; clarify loss wording for policy gradient.  
- Clarify technical accuracy: note auxiliary tasks are not trained with "true" Q-learning due to off-policy multi-step returns.  
- Remove redundant appendix phrasing and explicitly reference supplementary sections (e.g., "See Appendix B"); confirm λ_PC value in paper or appendix.  

Additional suggestions:
- Report extended-training results for top agents to show longer-term improvement.  
- Add more ablations on Atari to test whether auxiliary-task contributions observed on Labyrinth generalize.  

Summary:
- Strong paper with practical gains; main revisions should address computational cost, clarify multi-policy setup, strengthen feature-control validation, and justify λ_k choice; minor presentation and technical fixes are straightforward.

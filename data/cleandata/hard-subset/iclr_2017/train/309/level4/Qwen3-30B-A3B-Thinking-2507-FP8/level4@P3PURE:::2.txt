

This paper proposes enhancing feature learning in deep reinforcement learning by augmenting the main policy's optimization with domain-independent auxiliary tasks: state modification control (learning policies to maximally alter the state space, i.e., pixels), immediate reward prediction, and value function replay. Except for value function replay, these auxiliary tasks solely serve to shape features via shared CNN+LSTM feature extraction. Experiments demonstrate significant data efficiency improvements over A3C on Atari and Labyrinth, with convincing results. The paper is well-written, the ideas are sound, and the results are compelling—this constitutes a clear acceptance. At a high level, only minor clarifications are needed:

- **Computational cost**: Please quantify the additional training time incurred by optimizing auxiliary tasks. Identify the most computationally intensive components.
- **Multi-policy clarification**: The abstract states the agent "maximises many other pseudo-reward functions simultaneously by reinforcement learning," which initially suggests a single policy optimizing all rewards. Explicitly clarify in the abstract/introduction that the agent learns *distinct policies* for each auxiliary task (e.g., "the agent learns separate policies for each auxiliary task").
- **Feature control validation**: The "feature control" approach (Section 3.2) lacks empirical validation. The preliminary experiment in Figure 5 shows only marginal initial gains, and the dynamic modification of features during learning raises stability/convergence concerns that warrant discussion.
- **Extended training**: Since agent performance "is still steadily improving" (Section 4.1.2), consider training the best agents longer to assess ultimate potential.
- **Auxiliary task weights**: Why are weights (λ_*) fixed to 1? Were hyperparameter searches or ablations performed to validate this choice?
- **Q-learning clarification**: Note that auxiliary tasks are trained *off-policy* with multi-step empirical rewards (not via "true" Q-learning), as highlighted in OpenReview comments.

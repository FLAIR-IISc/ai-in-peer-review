- Paper proposes augmenting policy optimization with domain-independent auxiliary tasks (feature-control, immediate reward prediction, value function replay) to improve feature learning in deep RL.
- Auxiliary tasks (except value function replay) are used primarily to shape shared CNN+LSTM features rather than directly alter the policy.
- Experiments show significant data-efficiency improvements over A3C on Atari and Labyrinth, with convincing results—recommend acceptance.
- Request to quantify additional computational cost from auxiliary tasks and identify the most expensive components.
- Clarify in abstract/introduction that the agent learns separate policies for each auxiliary task (multi-policy), not a single policy optimizing all pseudo-rewards.
- Feature-control (Section 3.2) lacks sufficient empirical validation; Figure 5 shows only marginal early gains and dynamic feature modification raises stability/convergence concerns that should be discussed.
- Suggest training best agents for longer (extended training) since performance appears to still be improving.
- Ask why auxiliary task weights (λ_*) are fixed to 1; report any hyperparameter search or ablation validating this choice.
- Clarify that auxiliary tasks are trained off-policy with multi-step empirical returns (not classical Q-learning), as noted in OpenReview.
- Minor correction: state that L_π is a loss to be minimized (not maximized) in the policy gradient description.
- Equation 1: move λ_c inside the summation.

- Introduces UNREAL: augment A3C with domain-independent auxiliary tasks (auxiliary control, reward prediction, value replay) learned from a single stream to shape shared visual/recurrent features.  
- Auxiliary control tasks include pixel control and internal feature control, learning separate off-policy value functions/policies for pseudo-rewards.  
- Auxiliary reward prediction focuses representations on reward-relevant events to mitigate sparse rewards.  
- Value function replay adds extra value-learning updates from replayed experience with skewed sampling toward rewarding events.  
- Strong empirical validation: UNREAL dramatically outperforms A3C across 57 Atari games and sets new state-of-the-art.  
- Labyrinth results: visually complex 3D domain shows ~10x speedup in learning and >2x final performance versus baseline (87% human vs A3C 54%).  
- Decision: Accept — method is novel, technically sound, and results are highly significant.  
- Novel contribution: framing representation learning as active control problems (not just passive prediction) to learn causal/environmental dynamics.  
- Reward-focused auxiliaries balance general-purpose control tasks, keeping representations sensitive to primary objectives.  
- Ablation studies show each auxiliary component positively contributes to final performance.  
- Feature-control auxiliary is under-validated; presented results are preliminary and may face stability issues due to non-stationary target features.  
- Suggest clarifying that separate policies are learned per auxiliary control task to avoid misinterpretation.  
- Request discussion of computational overhead: wall-clock cost per environment step and most expensive components.  
- Technical clarifications needed: off-policy tasks use n-step returns; sign convention for L_pi; placement of λ_C; return notation r(c)_(t+k); define L_PC; add missing figure references; fix figure legend confusions and minor typos.  
- Questions for authors: were auxiliary loss weights treated as hyperparameters and how sensitive is performance to them? Did you try longer training to see if gaps widen? Have you run similar ablations on Atari to compare task contributions across domains?

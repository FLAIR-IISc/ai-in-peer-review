- Introduces UNREAL: augment A3C with multiple auxiliary tasks trained from the same experience stream to improve data efficiency and performance.  
- Auxiliary control tasks: learn off-policy value functions for pseudo-rewards (e.g., pixel control, feature control) to encourage controllable representations.  
- Auxiliary reward prediction: supervised prediction of immediate next reward from recent observations, oversampling reward-containing sequences to aid sparse-reward learning.  
- Value function replay: replay recent experiences to perform extra updates on the main task value function for better data efficiency.  
- Empirical results: state-of-the-art on Atari (avg. 880% human) and strong performance on Labyrinth (87%), with ~10x faster learning on Labyrinth versus A3C.  
- Decision: Accept.  
- Key reason 1: Significant performance gains and large improvements in data efficiency over a strong A3C baseline.  
- Key reason 2: Novel, well-executed architecture combining on-policy A3C with off-policy auxiliary Q-learning and multiple auxiliary losses.  
- Supporting evidence: thorough empirical evaluation and ablations showing each auxiliary task contributes and their combination is best.  
- Comparison to alternatives: auxiliary control outperforms traditional unsupervised objectives (e.g., reconstruction) for useful representations.  
- Literature placement: clearly distinguishes from prior work (Horde, UVFA, successor representations) and motivates addressing sparse rewards and data hunger in deep RL.  
- Contribution of Labyrinth: provides a new challenging, visually rich 3D benchmark beyond 2D games.  
- Terminology note: “Unsupervised” is misleading—auxiliary tasks are better described as “self-supervised.”  
- Architectural concerns: final agent is complex with many auxiliary heads, replay buffer, and loss-weight hyperparameters; more detail on sensitivity/tuning of auxiliary weights would help.  
- Qualitative analysis request: more visualization/analysis of what auxiliary policies (e.g., pixel control) actually learn would be informative.  
- Questions for authors: clarify computational overhead (wall-clock time, memory) vs A3C; report other auxiliary tasks tried (e.g., action prediction, forward models); and state plans to release source code.

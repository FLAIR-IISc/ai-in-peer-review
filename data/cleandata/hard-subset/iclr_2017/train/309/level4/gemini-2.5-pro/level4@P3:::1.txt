### **Summary**

This paper introduces the UNREAL (Unsupervised Reinforcement and Auxiliary Learning) agent, a deep reinforcement learning architecture designed to improve data efficiency and overall performance by learning from a richer set of signals than extrinsic reward alone. The core idea is to augment a primary reinforcement learning agent (in this case, A3C) with several auxiliary tasks that are trained simultaneously from the same stream of experience. These tasks include:

1.  **Auxiliary Control Tasks:** The agent learns off-policy value functions for a set of "pseudo-rewards," such as maximizing pixel changes in different regions of the screen (pixel control) or maximizing the activation of hidden units in its own network (feature control). These tasks encourage the development of a representation that understands how to influence and control its sensory input, which is hypothesized to be broadly useful for any goal-directed behavior.
2.  **Auxiliary Reward Prediction:** The agent is trained on the supervised task of predicting the immediate next reward from a short sequence of recent observations. By oversampling sequences that contain rewards, this task helps the agent's feature extractors learn to recognize reward-relevant states more quickly, especially in sparse reward settings.
3.  **Value Function Replay:** The agent replays recent experiences to perform additional updates on the value function of the main task, further improving data efficiency.

The authors demonstrate the effectiveness of the UNREAL agent on the 57-game Atari suite and a visually complex, 3D first-person environment called Labyrinth. The results are impressive, showing that UNREAL not only achieves state-of-the-art performance (averaging 880% human performance on Atari and 87% on Labyrinth) but also learns significantly faster (a mean 10x speedup on Labyrinth) and is more robust to hyperparameter settings than the A3C baseline.

### **Decision and Key Reasons**

**Decision: Accept**

This is a strong paper with compelling empirical results that clearly demonstrate the value of integrating auxiliary self-supervised tasks into a modern deep RL agent. My decision is based on two key reasons:

1.  **Significant Performance Gains and Data Efficiency:** The improvements over a very strong baseline (A3C) are not marginal. The UNREAL agent establishes a new state-of-the-art on the Atari benchmark and shows remarkable performance and learning speed on the challenging Labyrinth domain. The 10x speedup in learning is a particularly important contribution, addressing one of the major pain points of deep RL.
2.  **A Novel and Well-Executed Architectural Contribution:** While the concept of using auxiliary tasks for representation learning is not entirely new, this paper presents a novel and effective synthesis of multiple ideas. The specific formulation of "auxiliary control" tasks (learning policies to maximize pseudo-rewards) is an insightful way to encourage general competency. The hybrid architecture, which combines on-policy A3C for the main task with off-policy Q-learning from a replay buffer for the auxiliary tasks, is a clever design that leverages the strengths of both approaches.

### **Supporting Arguments**

The paper's claims are strongly supported by a thorough empirical evaluation. The ablation studies presented in Figure 3 are particularly convincing, showing that each of the proposed auxiliary tasks contributes positively to performance and that their combination yields the best results. The comparison in Section 4.1.1 between auxiliary control and more traditional unsupervised objectives like input reconstruction is excellent. It provides strong evidence for the authors' hypothesis that learning to *control* aspects of the sensory stream is more beneficial for representation learning than simply learning to *reconstruct* or *predict* them.

The work is well-situated within the existing literature. The authors clearly differentiate their approach from prior work like Horde (which did not use auxiliary tasks for shared representation learning), UVFA, and successor representations. The motivation for the work is clear: deep RL agents are data-hungry and struggle with sparse rewards, and the rich data stream available to an agent contains far more information than just the extrinsic reward signal. This paper provides a powerful and practical method for exploiting that information.

The introduction of the Labyrinth environment is also a valuable contribution, providing a new, challenging, and visually rich benchmark for the community that moves beyond 2D screen-based games.

### **Additional Feedback**

The following points are intended to help improve the paper and are not critical to my decision.

*   **Clarity on "Unsupervised":** The term "Unsupervised" in the agent's name (UNREAL) might be slightly misleading. The auxiliary tasks are more accurately described as "self-supervised," as they generate their own labels and reward signals directly from the agent's experience stream (e.g., pixel changes, network activations, observed rewards). While this is a minor semantic point, clarifying this distinction could prevent confusion with methods that learn from data entirely devoid of any task-related signals.
*   **Architectural Complexity:** The final UNREAL agent is quite complex, combining an A3C core with multiple auxiliary heads, a replay buffer, and a combined loss function with several weighting hyperparameters (λVR, λPC, λRP). While the results justify this complexity, a brief discussion on the sensitivity to these hyperparameters or insights into how they were tuned would be beneficial. The paper does show robustness to learning rate and entropy cost, but insight into the auxiliary task weights would be welcome.
*   **Qualitative Analysis:** The paper convincingly shows *that* the auxiliary tasks help, but it would be fascinating to see more analysis of *what* the auxiliary policies learn. For example, a visualization of a policy trained for pixel control on a specific grid cell could provide intuition about the kinds of behaviors these auxiliary objectives encourage. Do they learn to look around, interact with objects, or move to dynamic parts of the environment?

### **Questions for the Authors**

1.  Could you comment on the computational overhead of the UNREAL agent compared to the A3C baseline? Specifically, how much does the addition of the auxiliary tasks and the experience replay mechanism increase wall-clock training time and memory requirements?
2.  The choice of auxiliary tasks is well-motivated and effective. Did you experiment with other forms of auxiliary control or prediction? For instance, were tasks like predicting the agent's own future actions or predicting the consequences of actions (as in a forward model) considered?
3.  Given the complexity of the proposed architecture, which integrates several components (A3C, experience replay, deconvolutional heads for control, reward prediction network), reproducing the results could be challenging. Do you plan to release the source code to facilitate future research in this area?

- Proposal: train RL agents on auxiliary tasks alongside the primary objective to learn more robust, effective feature representations.  
- Auxiliary control tasks: maximize pixel-intensity changes and maximize activation of latent network features.  
- Supervised auxiliary task: offline reward prediction using skewed replay sampling (50/50 rewarding vs non-rewarding).  
- Empirical result: agents reach baseline performance roughly 10× faster and outperform SOTA on Labyrinth and Atari.  
- Conceptual distinction: active control-based auxiliary tasks vs passive unsupervised or model-based representation learning.  
- Novelty/impact: approach is novel and provides a compelling alternative to unsupervised learning.  
- Clarity: high-level explanations aid intuition, but L_PC (auxiliary control loss) should be introduced in the main text rather than only the appendix.  
- Methodology: sound experimental setup; selecting top-3 hyperparameter sets per method helps mitigate cross-method hyperparameter concerns.  
- Limitation: limited analysis of how auxiliary tasks affect internal representations beyond aggregate performance gains.  
- Empirical observation: pixel control and feature control are the most influential auxiliaries; pixel control dominated in Labyrinth (except UNREAL).  
- Suggested additional experiments: isolate auxiliary tasks, measure auxiliary-task-specific metrics (e.g., pixel change magnitude, feature activation statistics).  
- Suggested analysis: expand ablation-study discussion to explain why pixel control is particularly effective.  
- Hyperparameter sensitivity: discuss sensitivity to chosen hyperparameter ranges despite the top-3 strategy.  
- Paper decision: Accept.  
- Key reasons for acceptance: novelty/potential impact and strong empirical results.  
- Feedback request: clarify why input reconstruction in the unsupervised baseline harms performance (e.g., focus on irrelevant pixels?).  
- Feedback request: provide intuition for pixel control’s strong effect in Labyrinth.  
- Feedback request: report computational overhead and impact of auxiliary tasks on training time.  
- Feedback request: comment on using more complex architectures (e.g., LSTM) for the reward-prediction auxiliary and any results.

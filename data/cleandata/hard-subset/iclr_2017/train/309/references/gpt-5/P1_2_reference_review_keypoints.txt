- Problem: Tackles sample inefficiency and brittle learning in deep RL for visually-rich, partially observable 3D environments with sparse rewards.  
- Approach: Augments A3C (CNN-LSTM) with off-policy auxiliary tasks trained from recent replay to shape shared representations.  
- Auxiliary tasks — control: Pixel control (per-cell pixel-change Q-learning) and feature control (maximize hidden-unit activations) via a deconv head predicting per-action spatial Q-values.  
- Auxiliary tasks — reward-focused: Immediate reward prediction from short encoded-frame stacks with class-rebalancing to oversample rare rewards.  
- Auxiliary tasks — value replay: Additional off-policy value regression from replay alongside on-policy A3C value updates.  
- Algorithm: UNREAL optimizes a combined loss (A3C + replayed value + Q-learning control + reward prediction) with shared features and separate heads, trained asynchronously.  
- Empirics (Labyrinth): UNREAL more than doubles human-normalized performance vs A3C and achieves ~10× speedup to reach A3C’s best score; shows improved hyperparameter robustness.  
- Empirics (Atari): Outperforms A3C and Prioritized Dueling DQN in mean/median human-normalized scores across 57 games.  
- Ablations: Pixel control outperforms predicting input-change and reconstruction; preliminary evidence that feature control is beneficial.  
- Decision: Accept.  
- Key reasons for acceptance: Strong, consistent empirical gains in final performance and data efficiency, supported by ablations and robustness analyses.  
- Simplicity and practicality: Method is well-motivated, simple to implement, and clearly positioned relative to options/UVFA/SR/model-learning prior work.  
- Motivation/placement: Argues auxiliary tasks densify learning signals without biasing policy/value targets and differentiates from related prior approaches.  
- Technical soundness: Clear integration of on-policy A3C with off-policy Q-learning heads and replay; architectural choices (spatial deconv, dueling) are justified.  
- Empirical support detail: Labyrinth ablations convincingly support control-based representation learning; Atari results suggest generality beyond 3D tasks.  
- Robustness: Systematic hyperparameter sensitivity studies show smoother performance landscapes for UNREAL.  
- Novelty/impact: Auxiliary control formulation and off-policy integration with an on-policy recurrent agent is a notable, influential contribution.  
- Request for statistical rigor: Report means and standard errors across seeds and include statistical tests for headline numbers.  
- Evaluation protocol details needed: Specify frame budgets, evaluation episodes/stochasticity, and comparability with DQN baselines.  
- Fairness/selection concerns: Avoid top-k hyperparameter selection bias; show averages or a single validated setting and clarify selection procedure.  
- Ablations breadth requested: Extend ablations to Atari for individual and combined auxiliary components; systematize feature-control evaluation.  
- Computational cost: Quantify wall-clock overhead, memory cost, and frames-to-target scores for practitioner assessment.  
- Stability details: Provide target network update frequency, n-step Q parameters, replay buffer sensitivity, and any stabilizing tricks for off-policy updates with shared recurrent features.  
- Representation analysis: Add qualitative/quantitative analyses (saliency, correlations) to support the mechanistic story behind pixel/feature control.  
- Questions for authors — training budget/evaluation: How many frames per run, evaluation protocol (episodes, stochasticity), and fixed-frame reporting?  
- Questions — hyperparameter/seeds: Are Atari headline numbers averages over top-3 or all 50 settings, and how many seeds per setting? Provide per-game mean±SE for chosen config.  
- Questions — off-policy control details: How often are target params θ− updated, what n for n-step Q, and were there instabilities with shared LSTM?  
- Questions — component contributions: On Atari, what are gains from value replay alone, reward prediction alone, pixel control alone, and their combinations?  
- Questions — reward prediction bias: Could shared features bias policy/value; were non-shared or different rebalance ratios tested?  
- Questions — sensitivity of control tasks: How sensitive is pixel control to grid size/crop or alternatives (optical flow); how does feature control vary by target layer?  
- Questions — compute/resources: What are wall-clock times and memory overheads for UNREAL vs A3C on Labyrinth and Atari?  
- Questions — transfer: Can UNREAL-trained representations transfer to new reward functions or tasks without retraining auxiliary heads?

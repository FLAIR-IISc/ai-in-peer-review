1) Summary of contributions
The paper proposes UNREAL, an A3C-based agent augmented with unsupervised auxiliary tasks to improve representation learning, data efficiency, and robustness. The key ideas are:
- Auxiliary control via off-policy n-step Q-learning on pseudo-rewards: (i) pixel control, where the agent learns to maximize pixel-change in each cell of a spatial grid; and (ii) feature control, where the agent learns to activate internal network features.
- Reward-focused auxiliary prediction: a skewed replay-based classifier that predicts immediate reward (positive/negative/zero) from a short stack of recent observations, intended to densify learning signals without biasing the policy/value estimates.
- Value function replay: additional value regression from recent experience to exploit newly shaped features.
- A combined loss that trains A3C on-policy while training auxiliary tasks off-policy from a small replay buffer.
Empirically, UNREAL substantially improves data efficiency and final performance on 13 challenging 3D Labyrinth tasks (mean human-normalized score from 54% to 87%, up to 10–18x fewer steps to reach A3C’s best) and outperforms A3C and Prioritized Dueling DQN on 57 Atari games (880% mean, 250% median). Ablations show benefits from each auxiliary task; comparisons to reconstruction and input-change prediction baselines suggest learning to control pixel/feature changes is more effective than predicting them.

2) Decision
Accept. Key reasons: (1) Strong, broadly applicable idea—using off-policy auxiliary control and reward prediction to shape representations—well-motivated and distinct from prior work; (2) Convincing empirical gains across diverse domains with ablations and robustness analyses.

3) Supporting arguments
- Problem and motivation: The paper targets a central RL challenge—sparse/partial observability and slow representation learning—by enriching learning signals via auxiliary tasks that do not directly bias policy/value optimization. This is well placed relative to options/Horde/UVFA/successor representations and model-based prediction; the novelty is using many pseudo-rewards purely to learn better features in a shared encoder for a recurrent actor-critic trained on-policy.
- Method soundness: The combined on-policy A3C with off-policy Q-learning for auxiliary heads, and rebalanced reward prediction, is coherent. The architectural choices (deconvolutional head, dueling parametrization, small replay, shared CNN/LSTM) are reasonable and clearly described. The use of replay both to rebalance reward prediction and to stabilize auxiliary control is apt.
- Empirical support: Results span 13 Labyrinth tasks designed to test perception, navigation, and combat, and 57 Atari games, with hyperparameter sweeps and robustness plots. Reported improvements are large and consistent. Ablations (no pixel control; reconstruction vs prediction vs control; early feature-control results) support the claim that control-based auxiliary tasks are particularly effective. The data efficiency claims (10x step reduction) are meaningful for environment step budgets.
- Positioning in literature: The related work is mostly thorough; the distinction between learning auxiliary policies for representation vs using them for hierarchical control is explicit. The paper connects to model prediction work and recent auxiliary prediction in 3D environments.

4) Additional feedback to improve the paper
- Fairness and compute accounting: Please clarify whether comparisons control for total compute/parameter count. UNREAL performs extra updates per environment step; include wall-clock or FLOP-normalized comparisons, or report that A3C baselines receive an equal number of gradient steps. A matched-parameter/matched-update A3C baseline would strengthen claims.
- Evaluation protocol details (Atari): Specify no-op starts, sticky actions, frame termination, and evaluation episode counts. Provide exact training frames or environment steps for each method. Confirm that Prioritized Dueling DQN results are taken under the same protocol.
- Statistical rigor: Beyond top-k hyperparameters, include means and confidence intervals over multiple seeds per configuration for key benchmarks (or clarify the number of seeds per hyperparameter). Present variance bars for final performance/data efficiency plots.
- Ablations and sensitivity: Expand ablations isolating each loss term (λPC, λVR, λRP), buffer size, n-step length, and off-policy bootstrapping horizon. A sensitivity plot for λ weights would help practitioners.
- Feature control: Results are promising but preliminary; include a more systematic evaluation and combination with pixel control, and discuss when feature control might be preferable (e.g., low-pixel-change domains).
- Generalization and transfer: Since UVFA/successor representation are cited, a small transfer study (e.g., reusing the learned encoder on new reward functions or map variants) would underscore representation benefits.
- Exploration vs representation: It would be helpful to disentangle whether gains arise primarily from better exploration (via pixel/feature control shaping) or from improved value prediction. Simple exploration metrics (e.g., state coverage, episode returns distribution early in training) would be informative.
- Reproducibility: Consider releasing code or pseudocode detailing the training loop, replay sampling rates, and update schedules.

Questions for the authors
- Compute budget: How many auxiliary updates per environment step are performed versus A3C updates? Do results hold when normalizing by total gradient steps or wall-clock time?
- Off-policy stability: Did you observe instabilities from off-policy Q-learning on RNN features? How sensitive are results to replay buffer size (2k), target network lag, and n-step length?
- Reward prediction rebalancing: Does oversampling rewarding events for reward prediction affect the shared representation in a way that biases value estimates for sparse-reward tasks? Any negative transfer observed?
- Hyperparameter robustness: You sweep learning rate and entropy cost (and λPC). How sensitive are results to λVR and λRP? Any guidance for default settings across domains?
- Atari protocol: Please clarify sticky actions and evaluation procedures; are your Dueling DQN baselines run under the same protocol? If not, can you provide matched baselines?
- Feature control: Can you provide more comprehensive results on feature control (alone and combined with pixel control) across Labyrinth and Atari?
- Transfer/zero-shot: Does the encoder learned with these auxiliary tasks transfer to new goals or rearranged mazes without fine-tuning? Any preliminary results?

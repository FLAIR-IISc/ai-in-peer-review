Summary and contributions
The paper proposes UNREAL, an A3C-based agent augmented with auxiliary tasks that shape the representation to improve data efficiency, robustness, and final performance. The key ideas are:
- Auxiliary control via off-policy Q-learning on pseudo-rewards: pixel control (maximize local absolute frame change in an n×n grid) and feature control (maximize activation of internal units), implemented with a deconvolutional dueling head sharing a CNN-LSTM trunk.
- Reward-focused auxiliary task: immediate reward prediction trained on a replay buffer rebalanced to oversample rare rewarding events; this shapes shared convolutional features without biasing the policy/value updates.
- Value function replay: additional value regression on replayed sequences to leverage improved features and stabilize learning.
The combined loss trains A3C on-policy while training auxiliary tasks off-policy from a recent replay buffer. Empirically, UNREAL improves human-normalized performance and hyperparameter robustness on 13 Labyrinth levels and 57 Atari games, with large gains in data efficiency (up to 10× fewer frames to reach A3C’s best performance on Labyrinth).

Decision: Accept
Primary reasons:
- Strong empirical improvements in both data efficiency and robustness over strong baselines across diverse 3D and 2D domains, with ablations supporting the contribution of the auxiliary tasks.
- A clear, well-motivated integration of off-policy auxiliary learning with on-policy actor-critic that is simple, general, and likely to be impactful to the community.

Supporting arguments
- Motivation and placement: The method is well positioned relative to prior work on temporal abstraction (options), UVFA, successor representations, Horde, and auxiliary prediction (e.g., depth). Unlike UVFA/successor features, the pseudo-rewards are not used for transfer but to shape representation via control. Compared to prediction-based auxiliaries (reconstruction, feature/label prediction), the paper emphasizes control-oriented objectives (optimize pseudo-rewards) and shows they outperform purely predictive baselines.
- Methodology: The joint objective cleanly separates on-policy policy gradient updates (A3C) from off-policy auxiliary updates (Q-learning), leveraging replay to decouple data distribution for representation shaping (reward prediction). The use of deconv dueling heads for spatial pseudo-rewards is sensible. The reward predictor’s data rebalancing is justified because it does not directly influence policy gradients.
- Correctness/rigor: The algorithmic components are standard and compatible (Q-learning off-policy; value replay as regression). Using recent sequences and n-step returns with an LSTM mitigates off-policy issues for recurrent features. The ablations cover: removing pixel control; alternative unsupervised auxiliaries (reconstruction, predicting instantaneous input change); preliminary feature control; and show consistent benefits and ranking (control > predict-change > reconstruct).
- Empirical support: 
  - Labyrinth: substantial gains in final and sample efficiency, with improved hyperparameter robustness. The speedup metric (frames to reach A3C’s best) is reasonable; gains are large and consistent.
  - Atari: improvements over A3C and Prioritized Dueling DQN in mean/median human-normalized performance and robustness curves. The authors tune hyperparameters consistently across methods and analyze sensitivity.
- Impact: The approach is simple to adopt with existing actor-critic agents and can serve as a template for incorporating additional auxiliary control/prediction tasks.

Caveats and concerns (do not outweigh positives)
- Attribution of gains: Value function replay and reward prediction rebalancing could account for a significant portion of improvements. While “UNREAL without pixel control” helps isolate pixel control, a stronger baseline would be “A3C + value replay only” and “A3C + reward prediction only,” each tuned comparably, across both domains.
- Pixel control pseudo-reward: Absolute frame-difference rewards can correlate with visual flicker or dynamics not tied to task-relevant semantics. The paper notes the need to retune λPC on Atari; a more thorough analysis of when pixel control helps/hurts (e.g., noisy/flickering games) would strengthen claims.
- Off-policy RNN training details: Off-policy updates with recurrent state can be brittle. While the implementation uses 20-step sequences, details on how hidden states are initialized for replayed segments (e.g., burn-in) would clarify stability.
- Evaluation protocol: For Atari, confirm fairness vs. prior state-of-the-art (e.g., frame count budget, sticky actions, no-ops, episode termination conventions). The use of “top-3” hyperparameters for curves is acceptable but should be paired with aggregate-over-all-hyperparams results (some are provided) and confidence intervals.
- Generality of feature control: Results are preliminary on one level; broader evaluation would support the generality of this auxiliary.

Suggestions for improvement
- Provide full ablations disentangling each auxiliary component across both domains, including an “A3C + value replay only” baseline and “A3C + reward prediction only,” all with comparable tuning.
- Report sensitivity to replay buffer size, number of auxiliary updates per environment step, and λ weights; include compute/wall-clock trade-offs to contextualize sample efficiency gains.
- Analyze pixel control robustness under visually noisy/flickering settings and tasks where dynamic changes do not align with reward.
- Clarify recurrent replay: hidden state initialization, burn-in length, and whether LSTM states are stored vs. recomputed.
- Add statistical uncertainty (e.g., across seeds) for headline numbers and speedup metrics.
- Discuss transfer: does the learned representation accelerate learning on new tasks or pseudo-rewards?

Questions for the authors
- What fraction of the Labyrinth/Atari gains comes from value replay alone? Can you report results for A3C + value replay (no reward prediction, no pixel/feature control)?
- For recurrent replay, do you store LSTM states or reconstruct them with a burn-in? How sensitive is performance to sequence length and burn-in?
- How many auxiliary updates are performed per environment step, and how does this affect wall-clock time vs. sample efficiency?
- For Atari, what exact evaluation protocol is used (no-ops, sticky actions, life loss termination)? Are comparisons to Prioritized Dueling DQN aligned to the same protocol?
- Can you quantify cases where pixel control hurts (e.g., in high-flicker games) and whether alternative pseudo-rewards (e.g., prediction error, empowerment) mitigate this?
- How stable are results to λVR, λPC, λRP, and the replay buffer size (2k)? Any signs of overfitting to recent windows due to the small buffer?
- Do the auxiliary tasks continue to provide benefits after convergence, or can they be annealed off to save compute without loss?

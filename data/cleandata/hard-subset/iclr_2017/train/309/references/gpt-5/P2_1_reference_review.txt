Summary and contributions
- The paper proposes UNREAL, a reinforcement learning agent that augments an on-policy recurrent A3C learner with off-policy auxiliary tasks trained from a short replay buffer. The auxiliary tasks include (i) control-oriented pseudo-rewards (pixel control and, preliminarily, feature control) trained with n-step Q-learning; (ii) a reward prediction task trained on rebalanced replay to reduce perceptual sparsity; and (iii) value function replay to improve value estimation. These tasks share representation with the main policy/value network but are not used for action selection.
- The main empirical claims are: large gains in data efficiency, final performance, and hyperparameter robustness on (a) 13 visually rich, partially observed 3D Labyrinth levels testing navigation and combat, and (b) 57 Atari games. Ablations suggest each auxiliary component helps, with the full UNREAL agent yielding the strongest improvements.

Decision: Accept
Key reasons:
- Strong, consistent empirical improvements across diverse domains with careful ablations and robustness analyses.
- A clear and well-motivated approach that connects to options/Horde/UVFA/successor representation and model-based/auxiliary prediction work, but contributes a distinct, practical recipe for representation shaping via off-policy control tasks and reward-focused prediction.

Supporting arguments
1) Problem and motivation
- The paper addresses slow, brittle learning in sparse-reward, partially observed environments by learning richer representations. The rationale for auxiliary control (to learn environment dynamics) and reward prediction (to emphasize features predictive of reward on short horizons) is compelling. The decision to use off-policy Q-learning for many parallel pseudo-rewards from a single behavior stream is well justified.

2) Method and positioning in literature
- The method is straightforward yet novel in combination: A3C + replayed value regression + off-policy Q-learning heads for pseudo-rewards + rebalanced immediate reward prediction. The paper articulates how this differs from Horde (no representation sharing), UVFA/successor representations (factorization used for control/transfer rather than as auxiliary representation learning), and recent auxiliary prediction works (uses control, not only prediction).
- Architectural choices (deconv heads exploiting spatial structure; dueling parametrization; shared CNN + LSTM for base policy) are reasonable and clearly described.

3) Empirical support and rigor
- Labyrinth: On 13 levels spanning static/randomized navigation, procedurally generated mazes, and laser-tag combat, UNREAL attains higher final performance (mean human-normalized from 54% to 87%, median from 45% to 92%), with large speedups to reach A3C’s best (~10x on average). Ablations indicate: pixel control > change prediction > reconstruction, supporting the “learn to control” thesis; reward prediction and value replay both help; feature control is promising.
- Atari: On 57 games, UNREAL surpasses A3C and Prioritized Dueling DQN on mean/median human-normalized scores, with improved robustness to hyperparameters.
- Robustness is assessed via broad sweeps over learning rate and entropy regularization, with plots showing relatively smooth performance surfaces and improved robustness under UNREAL. Implementation details are substantial (losses, architectures, replay scheme, weighting).
- Potential confounds are addressed partially: ablations isolate components; reconstruction and “predict change” baselines provide useful comparators.

4) Limitations and concerns
- Compute/update fairness: UNREAL performs additional off-policy updates from replay. While this is part of the method, a control baseline that matches total gradient updates or wall-clock compute (e.g., A3C with equivalent additional on-policy updates) would isolate the effect of auxiliary objectives vs. update volume.
- Evaluation protocol: Some plots report averages over the top-k hyperparameters. Although robustness plots over all settings are included, clearer reporting of fixed hyperparameters with multiple seeds and statistical tests would strengthen claims.
- Reproducibility: Labyrinth is proprietary; code release is not stated. This limits external verification. More details on human score acquisition for normalization would help.
- Auxiliary objective side-effects: Pixel-control rewards could encourage representations that attend to visually salient but reward-irrelevant changes (noisy-TV issue). The authors’ design mitigates policy bias by not using the auxiliary policies for control, but a brief analysis ruling out degenerate oscillatory behaviors would be useful.
- Sensitivity and generality: Weighting of auxiliary losses (particularly λPC) is tuned and differs between domains; more sensitivity analysis and guidance for choosing these weights would help practitioners.

Additional feedback (to improve the paper; not decisive for acceptance)
- Provide a compute-normalized baseline: A3C with matched number of parameter updates or matched wall-clock, to quantify the contribution of task design vs. extra updates.
- Report per-game/level sample efficiency in environment frames and wall-clock, and include confidence intervals over multiple seeds for a fixed hyperparameter setting.
- Expand ablations: include “value replay only” vs “reward prediction only” vs “pixel control only” across more Labyrinth tasks; and a combined feature+pixel control variant given the promising early result.
- Analyze potential degeneracies: check for camera-jitter or oscillations induced by pixel control; measure action entropy and motion statistics.
- Clarify human-normalization protocol (number of human subjects, training time allowed, variance) and evaluation episodes for agents.
- Discuss generalization: e.g., train on some maze seeds and evaluate on held-out seeds to test representation transfer benefits.
- If possible, release training/evaluation code and a subset of environments, or provide detailed configuration files to enhance reproducibility.

Questions for the authors
- Compute fairness: How much of the gain persists if you equalize the number of parameter updates or wall-clock time with an A3C baseline? Can you provide such a comparison?
- Stability: How do you stabilize off-policy Q-learning heads when sharing a recurrent trunk? What is the target network update cadence for θ−, and did you observe instabilities?
- Auxiliary loss weights: How sensitive are results to λPC, λVR, λRP across tasks? Can you provide a sensitivity sweep and practical selection guidelines?
- Pixel control behavior: Did you observe behaviors indicative of maximizing pixel change (e.g., rapid head oscillation)? Any diagnostics ruling this out?
- Reward predictor oversampling: In dense-reward games, does rebalancing hurt? Did you adaptively control the skew or try turning it off?
- Negative transfer: Are there tasks where auxiliary control/reward prediction reduced final performance relative to A3C? What characterizes those cases?
- Human-normalized scores: How were human scores collected (subjects, episodes, variance), and are they consistent across Labyrinth task variants?
- Replay design: Why a 2k frame buffer, and how does performance change with buffer size or prioritization strategies?

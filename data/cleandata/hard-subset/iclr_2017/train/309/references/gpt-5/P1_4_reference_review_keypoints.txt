- Proposes UNREAL: A3C agent augmented with unsupervised auxiliary tasks to improve representations, data efficiency, and robustness.
- Auxiliary control via off-policy n-step Q-learning with pseudo-rewards: pixel control (maximize pixel-change per grid cell).
- Auxiliary control via feature control: learn to activate internal network features as control objectives.
- Reward-focused auxiliary prediction: skewed replay-based classifier predicting immediate reward (pos/neg/zero) to densify signals without biasing policy/value.
- Value function replay: additional value regression from recent experience to exploit shaped features.
- Combined loss trains A3C on-policy while training auxiliary tasks off-policy from a small replay buffer.
- Empirical gains: large improvements on 13 Labyrinth tasks (mean human-normalized 54%→87%, up to 10–18x fewer steps) and strong gains on 57 Atari games (880% mean, 250% median).
- Ablations indicate each auxiliary task helps and control-based tasks outperform reconstruction/input-change prediction baselines.
- Decision: Accept — idea is strong, broadly applicable, well-motivated, and distinct from prior work.
- Method soundness: coherent integration of on-policy A3C with off-policy Q-learning, reasonable architecture choices (deconv head, dueling, shared CNN/LSTM).
- Empirical support: broad benchmarks, hyperparameter sweeps, robustness plots, and meaningful data-efficiency claims.
- Literature positioning: situates work relative to options/Horde/UVFA/successor representations and model prediction; distinguishes representation learning from hierarchical control.
- Request clarification on fairness/compute accounting: control for total compute/parameter count, wall-clock or FLOP-normalized comparisons, or matched-update baselines.
- Ask for Atari evaluation protocol details: no-op starts, sticky actions, frame termination, evaluation episode counts, and exact training frames.
- Statistical rigor: provide means and confidence intervals over multiple seeds and variance bars for performance/data-efficiency plots.
- Ablations/sensitivity: isolate each loss weight (λPC, λVR, λRP), buffer size, n-step length, and off-policy horizon; include sensitivity plots for λ weights.
- Feature control: results promising but preliminary—request more systematic evaluation and guidance when it’s preferable over pixel control.
- Generalization/transfer: suggest a small transfer study (reuse encoder on new rewards or map variants) to demonstrate representation benefits.
- Exploration vs representation: ask to disentangle whether gains stem from improved exploration or value estimation, e.g., via state-coverage or early-return distributions.
- Reproducibility: encourage releasing code or detailed pseudocode of training loop, replay sampling rates, and update schedules.
- Question on compute budget: how many auxiliary updates per env step vs A3C, and do results hold when normalized by gradient steps or wall-clock?
- Question on off-policy stability: any instabilities from off-policy Q-learning on RNN features and sensitivity to buffer size, target lag, n-step length?
- Question on reward prediction rebalancing: does oversampling rewarding events bias shared representations or value estimates in sparse-reward tasks?
- Hyperparameter robustness question: sensitivity to λVR and λRP and practical default settings across domains?
- Atari protocol/baselines question: clarify sticky actions and whether Dueling DQN baselines use the same protocol or provide matched baselines.
- Feature control results request: more comprehensive results for feature control alone and combined with pixel control across Labyrinth and Atari.
- Transfer/zero-shot question: does the encoder transfer to new goals or rearranged mazes without fine-tuning; any preliminary results?



# Review of "UNREAL: End-to-End Learning and Planning"

## 1. Summary of Contributions
The paper introduces the UNREAL agent, which enhances deep reinforcement learning by incorporating auxiliary tasks: auxiliary control tasks (pixel control and feature control) and auxiliary reward prediction. The authors claim that these auxiliary tasks significantly improve data efficiency, learning speed, and robustness to hyperparameters across both 3D Labyrinth environments and Atari games. The approach builds upon A3C and integrates auxiliary tasks with careful attention to architectural details and training procedures.

## 2. Decision
Reject. The paper contains multiple technical inaccuracies, inconsistent formatting, and insufficient methodological details that undermine confidence in the results and claims.

## 3. Supporting Arguments

**Technical Inaccuracies:** The paper contains several technical errors that are concerning for a machine learning publication. Most notably, Section 2 defines the state space as including "the activations of the hidden units of the network" (st = f(o1, r1, a1, ..., ot, rt)), which is incorrect. The state space should represent the environment's state, not the agent's internal network activations. This fundamental misunderstanding of the RL framework suggests a lack of rigorous understanding that is concerning for a paper claiming to advance RL methods.

**Inconsistent Mathematical Notation:** The paper repeatedly uses inconsistent mathematical notation, particularly with exponents. For example, "γ i−1" appears multiple times instead of the correct "γ^(i-1)" or "γ^{i-1}". This is not just a minor formatting issue but represents a lack of attention to detail that undermines the paper's credibility. In Section 3.1, "R(c)t:t+n = ∑n k=1 γ k−1r(c)t+k" has the same exponent formatting issue.

**Incomplete References:** The reference list contains numerous formatting inconsistencies. Many entries use "Barreto et al.,? \Q2016\E" instead of the correct citation format. This is a serious issue as it suggests the authors did not properly format their references, which is expected at the ICLR level. The incomplete references make it difficult to verify the claims made about related work.

**Ambiguous Claims:** The paper claims "UNREAL is significantly faster at learning and therefore more data efficient, achieving a mean speedup of the number of steps to reach A3C best performance of 10×". However, it never defines what "best performance" means. Is it the maximum performance achieved by A3C during training, or a specific performance threshold? This ambiguity makes the claim difficult to evaluate.

**Lack of Statistical Significance:** The paper reports performance improvements (e.g., "more than twice the final human-normalised mean performance of A3C") but provides no statistical significance testing for these results. This is particularly problematic for a paper claiming to substantially improve state-of-the-art results.

**Inconsistent Terminology:** The paper uses "value function replay" without clear definition, stating it "performs value iteration" which is inaccurate. Value iteration is a specific algorithm for solving MDPs, not a technique for replaying value functions.

## 4. Additional Feedback

**Formatting and Notation:** Please ensure all mathematical notation is consistently formatted with proper exponents and subscripts. For example, use "γ^{i-1}" instead of "γ i−1" throughout the paper.

**Technical Clarification:** Please clarify the definition of the state space. The current definition conflates the environment state with the agent's internal state, which is a fundamental error in RL. The state should be defined as the environment's state, not the agent's internal representation.

**References:** Please complete and format all references consistently. The current format "Barreto et al.,? \Q2016\E" is not acceptable for a publication venue like ICLR.

**Statistical Analysis:** Provide statistical significance tests for all performance comparisons. The paper should report p-values or confidence intervals to support the claims of "significant" improvements.

**Definition of "Best Performance":** Clarify what "best performance" means in the context of the speedup claims. Is it the maximum performance achieved by A3C during training, or a specific performance threshold?

**Clarify Value Function Replay:** Define what "value function replay" means and how it differs from standard experience replay. The current description that it "performs value iteration" is misleading and incorrect.

**Additional Questions for Authors:**
1. How exactly does the reward prediction task avoid biasing the policy, given that it is trained on a skewed distribution of rewarding events?
2. Why was classification chosen over regression for reward prediction, and how did this choice impact the results?
3. How was the hyperparameter λPC selected for Atari games (0.0001-0.01), and why is this range different from Labyrinth (0.01-0.1)?

The paper's contributions are potentially valuable, but the technical inaccuracies, formatting issues, and lack of methodological rigor are too significant to recommend acceptance. These issues suggest a lack of attention to detail that would likely carry over to the implementation and results, making it difficult to trust the claims. I recommend rejection at this stage to encourage a more rigorous revision.

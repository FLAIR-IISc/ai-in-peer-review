### Summary of Contributions

This paper introduces UNREAL (Unsupervised Reinforcement and Auxiliary Learning), an agent architecture that significantly enhances the performance, data efficiency, and robustness of deep reinforcement learning agents. The core contribution is the augmentation of a strong baseline agent (A3C) with a set of auxiliary tasks designed to accelerate and improve representation learning. The proposed auxiliary tasks are:
1.  **Auxiliary Control Tasks:** The agent learns off-policy value functions for maximizing pseudo-rewards, such as causing maximal pixel changes in different regions of the input (pixel control) or maximally activating hidden units in its own network (feature control). This forces the agent to learn about the controllable aspects of its environment, independent of the main task's sparse reward signal.
2.  **Auxiliary Reward Prediction:** The agent is trained to predict the immediate upcoming reward from a short sequence of past observations. To handle reward sparsity, this task is trained on experience replayed from a buffer where rewarding and non-rewarding transitions are balanced. This encourages the representation to focus on reward-relevant features.
3.  **Value Function Replay:** The agent re-trains its value function on recently experienced sequences from a replay buffer, performing additional value iteration steps to more rapidly propagate value information through its shared representation.

The UNREAL agent combines these components, training the main A3C policy on-policy while efficiently learning the auxiliary tasks off-policy from a shared replay buffer. The authors demonstrate through extensive experiments on a new, visually complex 3D environment called Labyrinth and on 57 Atari games that UNREAL dramatically outperforms its A3C baseline. Key results include a 10x speedup to reach A3C's peak performance, a doubling of the final mean human-normalized score on Labyrinth (from 54% to 87%), and state-of-the-art performance on the Atari benchmark (880% mean score).

### Decision and Key Reasons

**Decision:** Strong Accept

**Key Reasons:**
1.  **Significant Performance Improvement:** The paper presents a novel combination of well-motivated auxiliary tasks that leads to substantial and consistent improvements over a very strong, state-of-the-art baseline (A3C). The gains in learning speed, final performance, and hyperparameter robustness are large enough to be considered a major advance in the field.
2.  **Thorough and Rigorous Empirical Evaluation:** The claims are supported by an exceptionally strong set of experiments. The evaluation spans two very different and challenging domains (a visually rich 3D environment and the broad Atari suite), includes comprehensive ablation studies that isolate the contribution of each component, and directly compares the proposed control-based auxiliary task to plausible alternatives like prediction and reconstruction, demonstrating its superiority.

### Supporting Arguments

The UNREAL agent is a clever synthesis of several powerful ideas. The central insight—to use auxiliary tasks to shape a shared representation for a primary RL agent—is not entirely new, but the specific choice of tasks and the hybrid on-policy/off-policy architecture are novel and highly effective.

The auxiliary control task is particularly innovative. Instead of just predicting what will happen, the agent learns policies to *make* things happen (e.g., change pixels). This encourages the development of a representation grounded in the agent's own agency and the environment's causal structure, which proves to be more effective for control than passive prediction or reconstruction tasks, as shown in the ablation in Section 4.1.1. This is a valuable finding for the community.

The reward prediction task, coupled with skewed replay, provides a simple yet powerful mechanism to combat reward sparsity. By oversampling rare but important rewarding events, the agent quickly learns to associate visual precursors with rewards, shaping the shared visual encoder to be more sensitive to task-relevant information.

The empirical validation is exemplary. The introduction of the Labyrinth environment is itself a contribution, offering a challenging testbed with richer visuals and physics than many existing benchmarks. The results on Labyrinth are compelling, showing not just a higher final score but a staggering 10x improvement in data efficiency. This is a critical result for making deep RL practical in complex domains. The Atari results confirm that this performance is not specific to one environment but generalizes broadly, establishing a new state-of-the-art on this widely-used benchmark. The robustness analysis (Figure 3, right column) is also a crucial and often-overlooked aspect of evaluation; showing that UNREAL is less sensitive to hyperparameter choices than A3C makes the algorithm much more useful in practice.

### Additional Feedback for Improvement

These points are suggestions to further strengthen an already excellent paper and are not critical for my decision.

*   **Computational Overhead:** The UNREAL agent adds several network heads and loss computations. It would be helpful to include a brief discussion of the computational overhead (in terms of wall-clock time per training step or GPU memory usage) compared to the A3C baseline. While the data efficiency is vastly improved, understanding the trade-off in computational cost per step would provide a more complete picture.
*   **Hyperparameter Sensitivity of Auxiliary Tasks:** The paper demonstrates robustness to learning rate and entropy cost. However, the UNREAL loss function introduces new weighting hyperparameters (λVR, λPC, λRP). While the appendix states these were sampled from distributions, a brief analysis or discussion on the sensitivity to these weights would be insightful. For instance, is the performance heavily dependent on getting the ratio between λPC and λRP correct?
*   **Pixel Control Granularity:** The pixel control task uses a 20x20 grid over an 80x80 crop. It would be interesting to know if the authors experimented with different grid resolutions and what effect this had. Is there a sweet spot between learning fine-grained control and keeping the number of auxiliary policies manageable? A sentence or two about this design choice would be welcome.
*   **Feature Control:** The results for feature control are mentioned as preliminary but very promising. While a full exploration is likely beyond the scope of this paper, it would be interesting to clarify which layer was used for feature control and why it was chosen. This seems like a very powerful and general idea that warrants a bit more detail.

### Questions for the Authors

1.  Could you quantify the computational overhead of the UNREAL agent relative to the A3C baseline? For example, what is the approximate percentage increase in training time per environment step?
2.  In Section 3.4, the value function loss LVR is described as being optimized from replayed data "in addition to the A3C loss (of which it is one component)". To clarify, does this mean the value head is updated twice per agent step: once with the on-policy A3C value loss, and a second time with an off-policy value loss computed on a batch from the replay buffer?
3.  The use of a deconvolutional network for the pixel control Q-values is an elegant architectural choice. Did you find this was critical to performance, as opposed to, for example, a simpler MLP that outputs Q-values for each grid cell independently?

**Summary**

This paper introduces UNREAL (Unsupervised Reinforcement and Auxiliary Learning), an agent that augments the Asynchronous Advantage Actor-Critic (A3C) framework with several auxiliary tasks to improve data efficiency, robustness, and final performance. The core idea is to leverage a shared representation (a CNN-LSTM network) that is trained jointly for the main policy (via A3C) and for three types of auxiliary tasks: (1) off-policy auxiliary control tasks (pixel control and feature control), where the agent learns Q-functions to maximize local changes; (2) an auxiliary reward prediction task, trained on a skewed replay buffer to handle reward sparsity; and (3) a value function replay task. The authors demonstrate impressive empirical results, showing that the UNREAL agent significantly outperforms A3C on a challenging 3D environment (Labyrinth) and a suite of 57 Atari games, achieving state-of-the-art performance and a reported 10x speedup in learning.

**Decision: Reject**

While the paper presents a compelling combination of techniques and achieves impressive empirical results, I am recommending rejection due to a significant number of methodological ambiguities, notational inconsistencies, and formatting errors that undermine the paper's scientific rigor and reproducibility. The sheer volume of these minor-to-moderate flaws suggests a lack of careful preparation and calls into question the precision of the reported implementation.

**Supporting Arguments**

1.  **Confusing Objective Function and Methodology:** The overall objective presented in Equation (1) is poorly connected to the actual implementation. The equation proposes maximizing a sum of expected returns under different policies (`π` for the base task, `π(c)` for auxiliary tasks), all parameterized by a shared `θ`. It is unclear how this single objective could be optimized directly. The paper immediately pivots to a practical implementation using off-policy Q-learning for the auxiliary tasks, which optimizes a different objective (Bellman error minimization). This disconnect between the formal objective and the described method is confusing and methodologically imprecise. A clearer formulation would be to present the final loss function (Eq. 2) as the primary objective, rather than introducing a misleading high-level objective first.

2.  **Notational Inconsistencies and Errors:** The paper suffers from several notational problems.
    *   The symbol `LVR` is used to denote two different things. In the description of A3C in Section 2, `LVR` is defined as the on-policy value function loss component of `LA3C`. However, in the combined UNREAL loss in Equation (2), `LVR` is used to denote the *replayed* value loss, which is an additional, separate component. This overloading of notation is highly confusing.
    *   In the definition of the n-step lookahead loss for Q-learning (Section 2), the target is given as `Rt:t+n + γ^n max_a′ Q(s′, a′; θ−)`. The state `s′` here corresponds to `s_{t+n}`, not an arbitrary next state. This should be made explicit for clarity. More critically, in the A3C value loss, the target is `Rt:t+n + γ^nV(s_{t+n+1}, θ)`. This appears to be an off-by-one error; the value function should be evaluated at state `s_{t+n}`.
    *   The definition of the state space `S` in Section 3.1 as including "the activations of the hidden units of the network" is highly unconventional and blurs the line between the agent and the environment.

3.  **Formatting and Presentation Issues:** The manuscript is riddled with formatting errors that indicate a lack of careful proofreading.
    *   There is a major structural error in the section numbering: Section 3.2 is immediately followed by a section header for 3.4, but the text body contains a section titled "3.3 Experience Replay". Furthermore, Section 3.4 is numbered as ".4 UNREAL AGENT".
    *   The reference list is improperly formatted, containing raw BibTeX markup such as `et al\.,? \Q2016\E,`. This is unacceptable for a conference submission and makes the bibliography difficult to read and verify.
    *   Minor typos and grammatical issues are present, such as "a doconvolutional layer" in Appendix B, where "transposed convolutional" is the more standard and less ambiguous term.

**Additional Feedback**

*   **Claim of "No Bias":** In Section 3.2, the authors claim that using a biased data distribution for the reward predictor does not "introduce bias to the agent’s policy." This is an overstatement. Since the reward predictor shares the convolutional encoder with the policy and value function, gradients from this auxiliary task will directly shape the features the policy uses. While it doesn't directly bias the value estimates for the main task, it undeniably biases the representation itself, which is the entire point. This claim should be rephrased more carefully.
*   **Incomplete Ablations:** In Section 4.1.1, the authors present an ablation comparing pixel control to prediction and reconstruction baselines. However, they only include "preliminary results for A3C augmented with the feature control auxiliary task on one of the levels." For a complete analysis, this comparison should have been run on the same set of levels as the pixel control experiments. Stating that it is "preliminary" weakens the empirical rigor of the paper.
*   **Ambiguous Details:** Several implementation details are ambiguous. In Appendix B, the replay buffer size is given as "2k observations." This should be stated explicitly as 2,000. The justification for using different `λPC` hyperparameter ranges for Labyrinth and Atari is also qualitative and seems post-hoc; a more principled approach would be to normalize the pixel change rewards themselves.

**Questions for Authors**

1.  Could you please clarify the formal relationship between the multi-policy objective in Equation (1) and the loss function you actually optimize in Equation (2)? Is Equation (1) merely illustrative, or is there a derivation that connects it to your practical algorithm?
2.  Can you confirm whether the target value function in the A3C loss (Section 2) should be evaluated at `s_{t+n}` instead of `s_{t+n+1}`?
3.  Why were the experiments for the feature control auxiliary task only "preliminary" and run on a single level, rather than being fully integrated into the main ablation studies?

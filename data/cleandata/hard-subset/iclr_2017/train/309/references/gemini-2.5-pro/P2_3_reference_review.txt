### **1. Summary**

This paper introduces the Unsupervised Reinforcement and Auxiliary Learning (UNREAL) agent, a novel architecture designed to significantly improve the data efficiency, robustness, and overall performance of deep reinforcement learning agents. The core contribution is the augmentation of a strong on-policy baseline agent, Asynchronous Advantage Actor-Critic (A3C), with a set of auxiliary tasks trained off-policy from a replay buffer. These auxiliary tasks are designed to accelerate the development of a rich and useful state representation. The specific tasks proposed are: 1) Auxiliary control tasks (pixel control and feature control), where the agent learns off-policy value functions to maximize changes in pixel intensities or hidden unit activations, encouraging it to learn about the environment's dynamics. 2) An auxiliary reward prediction task, which forces the agent to learn features predictive of reward onset, addressing the challenge of sparse rewards. 3) Value function replay, which provides additional value function learning from replayed experience. The authors demonstrate through extensive experiments on the visually complex 3D Labyrinth environment and 57 Atari games that the UNREAL agent dramatically outperforms the A3C baseline, achieving a 10x speedup in learning, more than doubling the mean human-normalized score on Labyrinth, and setting a new state-of-the-art on the Atari suite.

### **2. Decision and Key Reasons**

**Decision: Accept**

This is an exceptionally strong paper presenting a significant and well-executed contribution to the field of deep reinforcement learning. My recommendation for acceptance is based on two primary reasons:

1.  **Novel and Effective Architecture:** The paper proposes a clever and well-motivated architecture that elegantly combines the stability of on-policy learning for the main policy with the data efficiency of off-policy learning for auxiliary tasks. This hybrid approach effectively addresses the critical challenge of representation learning in RL, leading to substantial and impressive performance improvements.
2.  **Extensive and Rigorous Empirical Validation:** The claims are supported by an exhaustive set of experiments. The use of the new, visually rich Labyrinth environment showcases the agent's capabilities beyond standard benchmarks. Crucially, the authors provide thorough ablation studies that clearly demonstrate the contribution of each auxiliary task, lending strong support to their design choices. The reported results, including massive gains in learning speed and final performance over a state-of-the-art baseline, are highly convincing.

### **3. Supporting Arguments for the Decision**

The core idea of using auxiliary tasks for representation learning is not new, but this paper's implementation is novel, principled, and highly effective. The key insight is to decouple the learning of the main policy from the auxiliary tasks. The main A3C agent learns on-policy, which is known for its stability, while the auxiliary tasks are trained off-policy using Q-learning on data from a replay buffer. This allows the representation (shared CNN and LSTM weights) to be updated much more data-efficiently from past experience without destabilizing the main policy's learning process.

The choice of auxiliary tasks is also well-justified.
*   **Auxiliary Control:** The "pixel control" task is a creative form of unsupervised exploration. By learning to control its sensory input, the agent is incentivized to discover cause-and-effect relationships in the environment (i.e., which actions cause which visual changes). This enriches its representation with a basic understanding of environmental dynamics. The paper's experiment comparing pixel control to simply predicting pixel changes or reconstructing the input (Figure 5) provides strong evidence that *control* is the key ingredient, a non-obvious and valuable finding.
*   **Reward Prediction:** The reward prediction task directly tackles the problem of sparse rewards. By oversampling rewarding experiences from the replay buffer, the agent rapidly learns to identify state features that are predictive of rewards, even if those rewards occur infrequently in the on-policy stream. This helps shape the representation to be more salient to the main RL objective.

The empirical results are outstanding. The improvements are not marginal; they represent a significant leap forward. On Labyrinth, achieving a mean human-normalized score of 87% compared to A3C's 54% is a massive gain. A 10x median speedup to reach A3C's final performance highlights the dramatic improvement in data efficiency. Furthermore, the analysis of robustness to hyperparameters (Figure 3, top right) is a critical and often overlooked aspect of RL research. Showing that UNREAL is substantially less sensitive to hyperparameter choices than A3C is a major practical contribution. The fact that this performance advantage transfers to the 57-game Atari suite, setting a new state-of-the-art, confirms the generality of the approach.

### **4. Additional Feedback for Improvement**

These points are intended to help strengthen the paper and are not critical to my decision.

*   **Computational Overhead:** The UNREAL agent introduces several additional network heads and loss computations. A brief discussion of the trade-offs in terms of computational cost (e.g., wall-clock training time, GPU memory usage) compared to the A3C baseline would provide a more complete picture for practitioners looking to implement the method.
*   **Sensitivity to Auxiliary Task Weights:** The final loss is a weighted sum of the A3C loss and the three auxiliary losses (λVR, λPC, λRP). The appendix mentions that these are sampled from log-uniform distributions, which is a good practice for demonstrating robustness. However, a brief discussion in the main paper about the relative sensitivity to these weights would be insightful. For instance, is performance highly dependent on getting the λPC weight right, or is the method fairly robust across a wide range?
*   **Feature Control Details:** The paper introduces "feature control" alongside "pixel control" but the main results focus on the latter. Figure 5 presents a compelling preliminary result for feature control. It would be interesting to see this idea more fully integrated or to have a clearer discussion on why pixel control was chosen as the primary control task for the main experiments. Is it more general, or simply easier to implement?

### **5. Questions for the Authors**

1.  The pixel control task subdivides the input into a 20x20 grid. Have you investigated the sensitivity of the agent's performance to this grid resolution? One could imagine that a too-fine grid might encourage the agent to focus on controlling high-frequency noise, while a too-coarse grid might not provide a useful learning signal.
2.  Regarding reward prediction, the paper uses a three-class classification (positive, negative, zero). Did you experiment with regression on the reward value itself? What was the reasoning behind choosing classification over regression?
3.  In the Labyrinth laser-tag levels, the visual complexity is high due to moving bots and projectiles. Does the pixel control task help the agent distinguish between self-induced changes (e.g., moving its own view) and changes caused by external agents (e.g., an enemy bot moving)? Or does it simply learn a general model of "changeability"?

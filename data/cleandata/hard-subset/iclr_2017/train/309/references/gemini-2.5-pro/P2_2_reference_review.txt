### Review of "Auxiliary Tasks for Reinforcement Learning" (UNREAL)

#### 1. Summary of Contributions

This paper proposes UNREAL (Unsupervised Reinforcement and Auxiliary Learning), a novel architecture for deep reinforcement learning that significantly enhances the performance, data efficiency, and robustness of a state-of-the-art agent (A3C). The core contribution is the augmentation of the main on-policy learning process with a set of auxiliary tasks trained off-policy from a replay buffer. These auxiliary tasks are not used for direct control but serve to shape a richer and more effective shared representation (the CNN and LSTM layers) for the main policy.

The proposed auxiliary tasks include:
1.  **Auxiliary Control Tasks:** The agent learns off-policy Q-values to maximize pseudo-rewards, such as causing maximal pixel changes in different regions of the input (Pixel Control) or maximally activating hidden units in its own network (Feature Control). This encourages the agent to learn about the controllable aspects of its environment.
2.  **Reward Prediction:** The agent is trained to predict the immediate next reward from a short sequence of past observations. To handle sparse rewards, this task is trained on a rebalanced experience replay buffer where rewarding and non-rewarding sequences are sampled equally. This helps the agent's representation become more sensitive to reward-predicting states.
3.  **Value Function Replay:** In addition to the standard on-policy value function updates in A3C, the agent performs additional value function regression on data sampled from the replay buffer, accelerating value learning.

The UNREAL agent demonstrates the power of this hybrid on-policy/off-policy approach through extensive experiments. On a new, visually rich 3D environment called Labyrinth, UNREAL achieves more than double the human-normalized performance of the A3C baseline. Crucially, it reaches A3C's peak performance using an order of magnitude less data. Similar significant gains in performance and robustness to hyperparameters are shown across 57 Atari games.

#### 2. Decision and Key Reasons

**Decision:** Clear Accept.

This is a strong paper that makes a significant and well-supported contribution to the field of deep reinforcement learning.

**Key Reasons:**
1.  **Novel and Principled Method:** The paper's central idea—combining on-policy policy optimization with off-policy representation learning via auxiliary tasks—is both novel and well-motivated. It provides a practical and effective solution to the sample inefficiency problem by leveraging a single stream of experience for multiple learning signals, using on-policy and off-policy methods for what they are respectively best at.
2.  **Extensive and Rigorous Empirical Validation:** The claims are backed by an exceptionally strong set of experiments. The authors not only demonstrate state-of-the-art performance on the standard Atari benchmark but also introduce and test on a challenging new 3D Labyrinth environment. The inclusion of comprehensive ablation studies, data efficiency analysis (showing a ~10x speedup), and hyperparameter robustness plots provides a very high degree of scientific rigor and convincingly supports the paper's claims.

#### 3. Supporting Arguments for the Decision

The UNREAL architecture is a clever synthesis of several powerful ideas. The base A3C agent provides stable, on-policy updates, which is crucial for policy learning. The addition of off-policy auxiliary tasks trained from a replay buffer is the key innovation. This allows for highly data-efficient learning of the shared representation layers, as each experience fragment can be reused multiple times for different objectives. This hybrid approach elegantly sidesteps the instability often associated with pure off-policy actor-critic methods while reaping the data-efficiency benefits of experience replay.

The choice of auxiliary tasks is insightful. Pixel control is an intuitive, self-supervised signal that forces the agent to learn a representation that understands cause and effect in the environment (i.e., which actions affect which parts of the visual field). Reward prediction, especially with the proposed rebalancing of experience, directly tackles the problem of sparse rewards by encouraging the representation to identify subtle cues that precede rewarding events.

The experimental results are compelling. The performance jump on Labyrinth is massive (54% to 87% mean human-normalized score), indicating that these auxiliary tasks are particularly beneficial in complex, visually rich environments where good representations are paramount. The finding that UNREAL is significantly more robust to hyperparameter choices (Figure 3) is a major practical benefit, as it lowers the barrier to successfully applying the method. The ablation studies in Figure 3 and Figure 4 clearly show that each component contributes positively to the final performance, justifying the complexity of the full agent. The comparison against other unsupervised methods like input reconstruction (Figure 5) further strengthens the case that learning to *control* features of the environment is a more powerful representation learning objective than simply predicting or reconstructing them.

#### 4. Additional Feedback and Questions for the Authors

These points are intended to help improve the paper and are not critical to my decision to accept.

*   **Computational Overhead:** The UNREAL agent introduces several additional network heads and loss terms. Could you provide a brief analysis of the computational overhead (e.g., in terms of wall-clock training time or FLOPs per update) compared to the A3C baseline? This would be valuable for practitioners looking to adopt the method.
*   **Interplay Between Auxiliary Tasks:** The ablation shows that combining tasks is beneficial. Do you have any insight into *how* these tasks interact? For instance, does learning pixel control create features that are then useful for the reward prediction task? A qualitative analysis, perhaps through visualizing the learned representations (e.g., with t-SNE), could offer deeper insights into *why* the combination is so effective.
*   **Feature Control vs. Pixel Control:** The feature control task is introduced as a promising alternative to pixel control and shows strong results on a few Labyrinth levels (Figure 5). However, it does not appear to be part of the final UNREAL agent used for the main Atari and Labyrinth results. Could you clarify the reasoning for focusing on pixel control in the main agent? Is it more generally applicable, easier to implement, or simply more performant across a wider range of tasks?
*   **Hyperparameter Sensitivity (λPC):** In Appendix B, it's noted that the weight for the pixel control loss, λPC, was sampled from different ranges for Labyrinth and Atari. This seems to imply some tuning is required based on the domain's visual characteristics. Could you elaborate on the sensitivity to this hyperparameter and provide intuition on why different ranges were needed? A brief discussion on this could help guide future applications of the method.

### 1. Summary of Contributions

This paper introduces UNREAL (Unsupervised Reinforcement and Auxiliary Learning), an agent architecture designed to improve the data efficiency, robustness, and overall performance of deep reinforcement learning agents. The core idea is to augment a primary A3C agent with several auxiliary tasks that are learned concurrently from the same stream of experience. These tasks, trained off-policy using a replay buffer, encourage the development of a richer and more versatile shared representation (a CNN-LSTM encoder). The proposed auxiliary tasks include: 1) auxiliary control tasks, where the agent learns off-policy value functions for maximizing pixel changes (pixel control) or internal network activations (feature control), and 2) auxiliary reward tasks, where the agent predicts the immediate next reward from a short sequence of observations, with samples rebalanced to over-represent rewarding events. The authors demonstrate through extensive experiments on a new, visually complex 3D environment called Labyrinth and on the 57-game Atari suite that the UNREAL agent significantly outperforms the baseline A3C, achieving substantial improvements in final performance, learning speed, and robustness to hyperparameter settings.

### 2. Recommendation and Justification

**Decision:** Reject

**Key Reasons:**
1.  Significant ambiguities and inconsistencies in the methodological description undermine the paper's scientific rigor and replicability.
2.  The manuscript suffers from a notable number of presentational flaws, including structural errors and a severely misformatted bibliography, which detract from its overall quality and suggest a lack of careful preparation.

While the paper presents a promising combination of techniques and impressive empirical results, the sheer volume of minor flaws, technical obscurities, and presentational errors is too substantial to overlook. A paper intended for a top-tier conference like ICLR must meet a higher standard of clarity, precision, and formal correctness.

### 3. Supporting Arguments for Decision

**Methodological Ambiguity and Inconsistencies:**

*   **Loss Function Definition:** The formulation of the final loss function in Equation (2) and its accompanying description is confusing. Section 2 defines the A3C loss as `LA3C ≈ LVR + Lπ − ...`, where `LVR` is the on-policy value loss. Equation (2) then defines the total loss as `LUNREAL = LA3C + λVR*LVR + ...`. This suggests that the value loss `LVR` is included twice: once as part of `LA3C` (implicitly, on-policy) and again as a separate term weighted by `λVR` (explicitly, from replayed data). The text states `LVR` is optimized from replayed data "in addition to the A3C loss (of which it is one component)". This is circular and lacks precision. Does the `LA3C` term in Eq. (2) exclude its own value loss component? A clear, unambiguous definition of each term in the final objective is necessary.
*   **Auxiliary Control Task Formulation:** In Section 3.1, the description of auxiliary control tasks is inconsistent. The objective is first framed as a policy-maximization problem (`argmax θ Eπc [R(c) 1:∞]`), but the method used is immediately described as value-based off-policy Q-learning (`L(c)Q`). While related, these are distinct formalisms, and the paper conflates them without clarification.
*   **State Space Definition:** The definition of the state space `S` for the auxiliary reward function `r(c)` in Section 3.1 is highly problematic: "The underlying state space S includes both the history of observations and rewards as well as the state of the agent itself, i.e. the activations of the hidden units of the network." A reward function that depends directly on the agent's internal network activations creates a non-stationary and ill-defined learning target. If the agent's parameters change, the reward function itself changes. This requires significant theoretical justification which is not provided.
*   **Questionable Architectural Choice:** In Appendix B, the description of the deconvolutional network for pixel control states: "There is a final ReLU nonlinearity on the Qaux output." Applying a ReLU activation to Q-values is non-standard and seemingly incorrect, as it forces all Q-values to be non-negative. Expected discounted returns (which Q-values estimate) can frequently be negative, especially in environments with negative rewards or suboptimal policies. This detail, buried in the appendix, calls into question a core component of the auxiliary control mechanism.
*   **Imprecise Language:** In Section 3.2, the description of rebalancing for reward prediction states: "we sample such that zero rewards and non-zero rewards are equally represented, i.e. the predicted probability of a non-zero reward is P(rτ != 0) = 0.5." The sampling strategy does not determine the model's predicted probability; it only changes the distribution of training data. The model should still aim to predict the true conditional probability. This phrasing is technically imprecise and misleading.

**Presentational and Formatting Flaws:**

*   **Paper Structure:** The paper's structure is flawed. Section 3 is titled "AUXILIARY TASKS FOR REINFORCEMENT LEARNING", yet it contains subsections 3.1, 3.2, and then jumps directly to ".4 UNREAL AGENT", completely omitting a Section 3.3. This is a glaring structural error.
*   **Bibliography:** The reference list is in a state of extreme disarray. Many entries are incorrectly formatted, containing raw BibTeX-like markup (e.g., `Barreto et al\.,? \Q2016\E, Barreto et al\.,`). The citation style is inconsistent, and several entries (e.g., Olafsdottir et al., 2015; Schacter et al., 2012) appear in the list but are not cited in the provided text, suggesting copy-paste errors from another document. A properly curated bibliography is a basic requirement for a scientific publication.
*   **Typos and Minor Errors:** Several minor but noticeable errors are present, such as "doconvolutional" instead of "deconvolutional" in Appendix B, and inconsistent capitalization in appendix titles ("RANDOMNESS ROBUSTNESS").

### 4. Additional Feedback

The following points are intended to help improve the manuscript and are not the primary basis for my decision.

*   The introduction of the Labyrinth environment is a valuable contribution in itself. However, the description of the levels is spread between the main text and the appendix. A more consolidated description or a clearer pointer to the appendix earlier on would be helpful.
*   The "Unsupervised Reinforcement Learning" paragraph in Section 4.1.1 is slightly confusingly titled, as the tasks (pixel control) are self-supervised, not truly unsupervised. The agent is still receiving a reward signal, albeit one it generates itself.
*   The justification for different hyperparameter ranges for `λPC` between Labyrinth and Atari ("Atari games are not homogeneous in terms of pixel intensities changes") is weak. A more principled approach, such as normalizing the pixel-change rewards online, could have avoided this and made the method more general.
*   An algorithm box detailing the full UNREAL learning process, showing how on-policy A3C updates and off-policy auxiliary updates are interleaved, would greatly enhance clarity and replicability.

### 5. Questions for the Authors

1.  Could you please provide a precise, unambiguous mathematical definition of the final loss function, clarifying the exact components of `LA3C` and `LVR` in Equation (2)?
2.  Can you justify the use of a final ReLU activation on the Q-value outputs for the auxiliary control tasks? How does the agent learn to avoid actions that lead to negative changes if the Q-values are constrained to be non-negative?
3.  Could you elaborate on the definition of the state space for the auxiliary reward function? If `r(c)` depends on network activations, how do you handle the non-stationarity of this reward signal as the network's parameters `θ` are updated?

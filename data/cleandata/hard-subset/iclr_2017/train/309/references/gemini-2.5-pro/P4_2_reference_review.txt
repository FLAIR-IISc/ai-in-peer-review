### **Summary of Contributions**
This paper introduces the Unsupervised Reinforcement and Auxiliary Learning (UNREAL) agent, a method for improving the data efficiency, performance, and robustness of deep reinforcement learning agents. The core idea is to augment a primary on-policy agent, an Asynchronous Advantage Actor-Critic (A3C) with an LSTM, with several auxiliary tasks that are trained efficiently off-policy using a replay buffer. The auxiliary tasks are designed to promote better representation learning. The proposed tasks include: 1) auxiliary control tasks, where off-policy Q-learning is used to learn policies that maximize pseudo-rewards like pixel changes in the input (pixel control) or activations of hidden units (feature control); 2) an auxiliary reward prediction task, trained to predict immediate rewards from recent history on a reward-balanced dataset; and 3) value function replay, which performs additional value function regression on replayed data. By combining the stable on-policy A3C for the main policy with efficient off-policy learning for these auxiliary objectives, the UNREAL agent is shown to achieve substantial performance gains. The authors support their claims with extensive experiments on a visually complex 3D first-person environment called Labyrinth and the standard 57-game Atari benchmark, demonstrating significant improvements over state-of-the-art baselines in both final score and learning speed.

### **Decision and Justification**
**Decision:** Accept

**Key Reasons:**
1.  The paper presents a novel and well-motivated framework that effectively combines on-policy and off-policy learning. The use of auxiliary control and prediction tasks to shape the representation is a powerful idea, and the specific implementation is technically sound and well-justified.
2.  The empirical evidence supporting the method's effectiveness is exceptionally strong and rigorous. The significant performance gains are demonstrated on two challenging and diverse domains (Labyrinth and Atari), and the claims are substantiated through comprehensive ablation studies, comparisons to relevant baselines, and a thorough analysis of hyperparameter robustness.

### **Supporting Arguments for Decision**
The UNREAL agent's architecture is a thoughtful synthesis of several powerful concepts in reinforcement learning. The central idea of using a stable, on-policy method (A3C) for the main task while leveraging the data efficiency of off-policy Q-learning from a replay buffer for auxiliary tasks is a key contribution. This hybrid approach addresses a common trade-off, allowing the agent to benefit from both paradigms.

The design of the auxiliary tasks is sound. The pixel control task, in particular, is an innovative way to encourage the agent to learn about the controllable aspects of its environment and the consequences of its actions without requiring an explicit environment model. The paper provides a compelling comparison showing that learning to *control* pixel changes is more effective for representation learning than simply predicting them or reconstructing the input, supporting the design choice. The reward prediction task, trained on a balanced replay buffer, directly tackles the issue of sparse rewards by forcing the network to develop features sensitive to reward-predictive states.

The experimental evaluation is exemplary. The introduction of the Labyrinth environment provides a new, challenging benchmark with rich visuals and complex tasks. The results on Labyrinth are striking, with the UNREAL agent more than doubling the performance of the A3C baseline and achieving a 10x speedup to reach A3C's peak performance. This demonstrates the method's utility in complex, 3D environments. The validation on the 57-game Atari suite confirms the generality of the approach, where UNREAL surpasses the previous state-of-the-art. Crucially, the paper includes extensive ablation studies (Figure 3) that clearly disentangle the contribution of each auxiliary task, showing that each component adds value. The analysis of robustness to hyperparameters is another major strength, as it demonstrates that UNREAL not only performs better but is also a more reliable and practical algorithm than its baseline.

### **Additional Feedback for Improvement**
While this is a strong paper, the following points could help improve its clarity and completeness. These are suggestions and do not detract from the core contributions.
*   **Computational Cost:** The UNREAL agent adds several network heads and loss computations. A brief discussion of the additional computational overhead (e.g., wall-clock time per update, GPU memory) compared to the A3C baseline would be a useful addition for readers interested in practical application.
*   **Clarity on Feature Control:** The "feature control" task is introduced as a primary type of auxiliary control but is only evaluated in a limited comparison on three levels. The paper would be strengthened by either integrating this task more fully into the main results or framing it more explicitly as a preliminary finding for future work. The current presentation is slightly ambiguous about its status.
*   **Hyperparameter Sensitivity of Auxiliary Tasks:** The robustness analysis focuses on the learning rate and entropy cost of the base A3C agent. While excellent, a short discussion on the sensitivity to the auxiliary loss weights (λVR, λPC, λRP) would add further value. The paper states λVR and λRP were fixed to 1, but it would be interesting to know if this choice is critical.
*   **Replay Buffer Size:** The replay buffer size is set to 2k transitions, which seems quite small for an agent with 32 parallel threads. This means the buffer is refreshed very quickly. It would be helpful to briefly justify this choice—for example, was it intended to focus learning on the most recent experience?

### **Questions for Authors**
*   For the auxiliary pixel control task, is the discount factor γ used for the n-step Q-learning returns the same as the one used for the main A3C objective?
*   The reward prediction task uses a skewed sampling strategy to create a balanced dataset of rewarding and non-rewarding events. Could you elaborate on whether this frequent resampling of rare rewarding transitions posed any risk of overfitting to the specific features of those transitions, and if any measures were taken to mitigate this?
*   Could you provide more detail on the pseudo-reward for the feature control task? Is a separate policy learned for each unit in the target layer, or is a single policy learned to maximize a norm (e.g., L2) of the entire layer's activation vector?

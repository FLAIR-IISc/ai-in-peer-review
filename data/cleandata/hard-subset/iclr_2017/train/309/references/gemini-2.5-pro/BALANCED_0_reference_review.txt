### **Review of "AUXILIARY TASKS FOR REINFORCEMENT LEARNING"**

#### **1. Summary of Contributions**

This paper introduces the UNREAL (Unsupervised Reinforcement and Auxiliary Learning) agent, a novel architecture designed to improve the data efficiency, final performance, and hyperparameter robustness of deep reinforcement learning agents. The core idea is to augment a primary on-policy agent (A3C) with a set of auxiliary tasks that are trained off-policy using experience replay. These auxiliary tasks are not used for direct decision-making but serve exclusively to shape a richer and more versatile shared representation within the agent's neural network.

The authors propose three main auxiliary tasks:
1.  **Auxiliary Control Tasks:** The agent learns off-policy value functions (via Q-learning) to maximize control over its sensory input (pixel control) or its own internal network activations (feature control). This encourages the agent to learn about the environment's dynamics and which aspects it can directly influence.
2.  **Reward Prediction:** To combat reward sparsity, the agent is trained to predict the immediate upcoming reward from a sequence of recent observations. This task is trained on data from a replay buffer that is rebalanced to oversample rewarding experiences, forcing the network to learn features predictive of reward.
3.  **Value Function Replay:** The agent performs additional value function regression updates on sequences sampled from the replay buffer, effectively performing extra value iteration steps to improve data efficiency.

The authors provide a comprehensive empirical evaluation on the challenging 3D Labyrinth environment and the standard Atari benchmark. The results demonstrate that the UNREAL agent significantly outperforms the A3C baseline, achieving a 10x speedup in learning on Labyrinth and setting a new state-of-the-art on both domains.

#### **2. Decision and Key Reasons**

**Decision: Accept**

This is an excellent paper presenting a novel, well-motivated, and highly effective method for improving deep RL agents. The key reasons for my decision are:

1.  **Novel and Sound Methodology:** The UNREAL architecture provides an elegant and powerful synthesis of on-policy (A3C) and off-policy (Q-learning from replay) methods. The idea of using auxiliary tasks purely for representation learning is compelling, and the specific tasks chosen cleverly address fundamental challenges in RL, such as learning environment dynamics (control tasks) and handling sparse rewards (reward prediction).

2.  **Exceptional Empirical Rigor and Results:** The experimental evaluation is exemplary. The paper demonstrates the method's effectiveness on a new, visually complex 3D domain (Labyrinth) in addition to the standard Atari suite. The results are not just incremental; they represent a substantial leap in performance (doubling A3C's score on Labyrinth) and data efficiency (10x speedup). Crucially, the authors include thorough ablation studies that clearly isolate the contribution of each component, as well as a direct comparison between their proposed "control" objective and plausible alternatives like "prediction" and "reconstruction," which strongly validates their design choices.

#### **3. Supporting Arguments**

The paper's primary strength lies in its clear motivation and the thoroughness of its execution. The problem of data efficiency and representation learning is central to progress in deep RL, and this work makes a significant contribution.

The hybrid on-policy/off-policy design is a key innovation. It retains the stability and exploratory benefits of the on-policy A3C algorithm for the main task while leveraging the superior data efficiency of off-policy learning from a replay buffer for the auxiliary representation-learning tasks. This is a best-of-both-worlds approach that is well-justified and proven effective.

The empirical work is a model for the field. The introduction of the Labyrinth environment provides a challenging new benchmark. The ablation studies in Figure 3 clearly show that each auxiliary task provides a substantial and cumulative performance benefit. The comparison in Figure 5 is particularly insightful, demonstrating that learning to *control* pixel changes is more effective than simply *predicting* them or *reconstructing* the input image. This supports the hypothesis that actively learning about cause-and-effect through control is a powerful driver for representation learning. Furthermore, the demonstrated improvement in robustness to hyperparameters (Figure 3, right column) is a highly valuable practical contribution, as it can significantly reduce the engineering effort required to apply deep RL methods. The achieved state-of-the-art results are a clear testament to the method's power.

#### **4. Additional Feedback for Improvement**

These points are suggestions for strengthening the paper and are not critical to the decision to accept.

*   **Computational Overhead:** The paper convincingly demonstrates dramatic improvements in data efficiency (sample complexity). However, it does not discuss the trade-off in terms of computational complexity (wall-clock time). The addition of multiple auxiliary heads and losses presumably increases the per-update computation cost. A brief analysis of this overhead would provide a more complete picture of the agent's overall efficiency.
*   **Hyperparameter for Pixel Control:** The weight for the pixel control loss, λPC, was tuned separately for Labyrinth and Atari, with the justification that pixel intensity changes differ across domains. This suggests a potential sensitivity. Could the authors comment on whether a more adaptive or normalized reward scheme for pixel control could make this hyperparameter more stable across different environments?
*   **Feature Control:** The feature control experiments are presented as "preliminary." This is an interesting idea that is conceptually similar to pixel control but operates in a more abstract space. It would be valuable to see this explored further, perhaps by integrating it into the full UNREAL agent and including it in the main ablation study, even if only on a subset of environments.

#### **5. Questions for the Authors**

I would appreciate clarification on the following points during the author response period:

1.  The pixel control task learns separate policies for an n x n grid of cells. How was the grid size (20x20) chosen, and have you investigated the sensitivity of the agent's performance to this parameter? Is there a trade-off between the granularity of control and learning stability or computational cost?
2.  The reward prediction task uses a skewed replay buffer to oversample rewarding events. Did you experiment with applying a similar prioritization scheme (e.g., prioritized experience replay based on TD-error) to the value function replay or the auxiliary control Q-learning updates? It seems plausible that this could yield further efficiency gains.
3.  The use of a deconvolutional network to produce the Q-values for the pixel control tasks is an interesting architectural choice. Could you provide more intuition for this? Does this structure enforce a useful spatial smoothness or prior on the learned control policies across neighboring cells?

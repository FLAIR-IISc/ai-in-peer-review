Overall, this is a very good paper, enjoyable and interesting to read. This work advances cutting-edge techniques on differential-privacy-intensive learning, is quite well-written, and is relatively detailed.

One caveat is that while the approach aims to be generalized, no theoretical guarantees are made about learning performance. Privacy-preserving machine learning papers often analyze both privacy (in the worst-case scenario, in the DP setting) and learning performance (often under different assumptions). Since learning performance can depend on the architecture's choice, future experiments are encouraged, even using the same dataset, with different architectures. If this is not added, please justify the choice of architecture used, and/or clarify what can be generalized about the observed learning performance.

Another caveat is that the epsilon values reported are not those that can be released privately; the authors state that their technique would change the resulting epsilon to do so. However, this would need to be resolved to make meaningful comparisons with the epsilon-delta values reported in the related work.

Finally, as acknowledged in the paper, the current approach may not work on other natural data types. Experiments on other datasets are strongly encouraged. Also, please cite the datasets used.

Other comments:
The discussion of some parts of the related work is broad. However, please add some survey/discussion of the related work on differential-privacy-intensive semi-supervised learning. For example, in the context of random forests, the following paper also proposed differential-privacy-intensive semi-supervised learning through a teacher-student approach (although it is not labeled as "teacher-student"). Private label data is only used when the "primary group" is being learned. A "secondary group" is then learned only from non-labeled (non-private) data, which contains pseudo-labels generated by the primary group.

G. Jagannathan, C. Montelloni, and K. Pillaippakamannatt: A semi-supervised learning approach for differential privacy. Proc. 2013 IEEE International Conference on Data Mining Workshops, IEEE Workshop on the Privacy Aspects of Data Mining (PDM), 2013.
Section C provides a good comparison of approaches. Please ensure that the quantitative results here are compared equally with the GAN results.

The paper is very well-written for the most part. Clarifications are needed in some places, including:
- The last paragraph of 3.1. "All teachers.... receive the same training data...." This should be rephrased to make it clear that this is not the same with respect to all teachers, but rather with respect to the same teacher on a neighboring database.
- 4.1: The authors state: "The number of teachers n is limited by a trade-off between the complexity of the classification task and the available data." However, since this trade-off is not formal, the statement is ambiguous. In particular, if the analysis is done in an i.i.d. setting, the trade-off will also depend on the relationship of the target hypothesis to the data distribution.
- The discussion of Figure 3 was rather vague in the text and captions and should be revised for clarity. In the text section, the clarification initially seems that a larger difference is better (as also indicated in the caption). However, it is later stated that the difference remains less than 20%. These sentences seem contradictory, which was probably not the intention.

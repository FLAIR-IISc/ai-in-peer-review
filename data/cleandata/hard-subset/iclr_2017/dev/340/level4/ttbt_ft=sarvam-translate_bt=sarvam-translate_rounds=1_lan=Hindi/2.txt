The purpose of this work is to learn a generative function G that can map inputs from the source domain to the target domain, so that a given representation function f remains unchanged while accepting inputs from both domains. The criterion is called f-stability. The proposed method is evaluated on two visual domain optimization tasks.

The paper is relatively easy to understand, and the authors provide quite comprehensive experimental results on two datasets. The main novelty of the f-stability function is that it does not seem intuitive to force the function G to be g o f, that is, to start with a restricted function f that has already lost information. As in the face dataset, f is learned to optimize the performance of certain tasks on some external datasets. It is not clear whether inputs from the source or target domain can be recovered by applying G to equations (5) and (6). Furthermore, the f function is learned with a specific task in mind. As in the two experiments, the representation function f is learned to identify faces or some face datasets in the source SVHN dataset. Consequently, if we have to do domain optimization for the same domain but for different tasks, such as recognizing emotions instead of faces, the process must be repeated.

Are the authors aware of why the proposed baseline method in equations (1) and (2) performs so poorly?

This work is designed to learn a generative function G that can map inputs from the source domain to the target domain, so that the given representation function f remains unchanged when taking inputs from both domains. This criterion is called f-consistency. The proposed method is evaluated on two visual domain adaptation tasks.

The paper is relatively easy to follow, and the authors provide quite extensive experimental results on two datasets. f-consistency is the main novelty of this work.

Forcing function G to be g o f seems contradictory, meaning starting from a finite function f that may already lose information. For example, in the face dataset, f is taught to optimize the performance of specific tasks on some external datasets. It is not clear whether inputs from the source or target domain can be recovered from G, as shown in equations (5) and (6). Also, function f is taught with a specific task in mind. For example, in two experiments, the representation function f is taught to identify facial numbers on the source SVHN dataset or some face datasets. Consequently, if we want to do domain adaptation for the same domain but for different tasks, such as identifying expressions instead of faces, the method must be repeated.

Do the authors have an idea why the proposed baseline method in equations (1) and (2) performs so poorly?

Figure 5 shows some visual comparisons between style transfer and the proposed method. However, it is not clear which method is better. Would it be possible to apply style transfer to create emojis from photos and repeat the experiments shown in Table 4?

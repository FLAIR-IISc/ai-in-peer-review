- Proposes a multigraph-based model that explicitly encodes overlapping entity boundaries via distinct edge semantics.  
- Claims contributions: (1) the multigraph model, (2) superior F1 on ACE2004/ACE2005/GENIA, (3) avoiding hypergraph “spurious structures” while keeping inference complexity.  
- Multigraph representation is a novel, fine-grained way to model overlapping entity relationships.  
- Empirical results show competitive or superior performance, especially gains on ACE datasets.  
- Theoretical analysis contrasts spurious structures in hypergraph models and argues the multigraph avoids them.  
- Clear problem statement and well-motivated need for a new graph representation for overlapping NER.  
- Strong empirical comparison to prior state-of-the-art (Lu & Roth, 2015) with detailed analysis.  
- Authors plan to release code and system, supporting reproducibility.  
- Feature description is under-specified; unclear how edge semantics map to concrete features or how much feature engineering was used.  
- Limited ablation study—missing analysis of the contribution of specific edge types or feature combinations (e.g., impact of all 8 edge types vs subsets).  
- Multigraph visualizations (Figures 2 & 4) are complex and hard to parse; more intuitive figures needed.  
- Lacks qualitative error analysis and examples of common failure cases for overlapping entity recognition.  
- Reviewer requests a detailed description of model features and how they leverage multigraph edges.  
- Reviewer asks whether an ablation study on different edge types was performed and to report the results.  
- Reviewer requests examples and elaboration on the specific error types the model makes.  
- Reviewer asks for a concrete example showing how spurious structures impact inference in the hypergraph model.  
- Reviewer asks for elaboration on tasks beyond equation parsing where the multigraph approach could be applied.  
- Paper is generally well-written and clearly organized.  
- Theoretical analysis of spurious structures is a notable and valuable contribution.  
- Code release commitment is appreciated.

- Proposes SynTime, a novel rule-based method for recognizing time expressions using three syntactic types and a three-step heuristic pipeline, outperforming baselines on three datasets.  
- Analysis of time expressions from four datasets reveals they are typically short, use a small set of keywords, and show similar syntactic behavior.  
- SynTime defines three syntactic types (time token, modifier, numeral) and applies simple heuristic rules to recognize time expressions.  
- Evaluation on three datasets demonstrates SynTime’s effectiveness against state-of-the-art rule-based and machine learning baselines.  
- Paper is well-written and clearly presented.  
- Rules are motivated by empirical observations from the dataset analysis.  
- Reported results show significant improvements over existing baselines.  
- Some approach details (exact rules for time segment identification and extraction) are underspecified, limiting reproducibility.  
- Limited discussion of handling special cases such as Ramadan or holiday seasons.  
- Asks how SynTime handles special cases like Ramadan or holiday seasons.  
- Requests an example of a time expression that lacks a time token.  
- Asks how performance can be further improved given 93% of expressions contain a time token and system recall is 92%.  
- Requests release of the full rule set and software to facilitate reproducibility and further development.  
- Requests clarification on dataset usage—text references “two benchmark datasets” but evaluation uses three.

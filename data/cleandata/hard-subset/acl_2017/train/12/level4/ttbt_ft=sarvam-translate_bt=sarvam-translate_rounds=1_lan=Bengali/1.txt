This paper describes a rule-based approach for extracting temporal expressions. Its key insight is that temporal expressions are typically short and contain at least one temporal token. It first identifies the temporal token through dictionary lookup, regular expression matching with POS tagging information. Then, it expands the temporal section from both sides of the temporal token based on a set of heuristic rules until it reaches a specific value. Finally, it combines the temporal sections into a single temporal expression based on another set of rules. Evaluation of this method with both rule-based and machine learning-based systems shows significant improvements in 3 data sets.

- Strengths:
It is well-written and clearly presented. The rules are inspired by empirical observations of data, and it seems well-justified as proven by the evaluation.
- Weaknesses:
The paper has some less specificness, which makes it difficult to reproduce the results. See below for details.
- General Discussion:
* Section 4.1: What about topics like are there 5 seasons? What about Ramadan or holiday seasons?
* Section 5.1: "Two Benchmark Datasets" => "Three Datasets"?
* Section 5.2: An example without the temporal token would be helpful.
* Section 5.2: Since this method is close to the maximum limit of performance, since 93% of expressions contain temporal tokens and the system was able to recall 92%, how do you plan to improve further?
* Are there any plans to release the complete set of rules/software used?

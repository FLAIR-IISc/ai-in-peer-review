- SynTime: a rule-based pipeline that finds time tokens, expands boundaries heuristically, and merges segments to recognize time expressions.  
- Main contribution: a simple, type-based rule design guided by corpus observations that outperforms prior rule-based and learning baselines.  
- Secondary contribution: analysis across corpora motivating the type system and heuristics, showing the approach is lightweight, real-time, and extensible.  
- Clear, well-organized presentation with pipeline steps and heuristics motivated by data rather than ad hoc rules.  
- Strong empirical results: substantial improvements over HeidelTime, SUTime, and UWTime on three datasets, including noisy tweets.  
- Practical and extensible: system is lightweight, real-time, and adaptable to new domains by adding token keywords.  
- Reproducibility gaps: several design choices and resource definitions (e.g., type coverage) are underspecified, hindering full replication.  
- Taxonomy clarity issues: unclear why there are five seasons and how religious/holiday expressions (e.g., Ramadan, “holiday season”) are categorized.  
- Dataset description inconsistency: paper refers to “two benchmark datasets” despite experiments using three—needs clarification.  
- Ceiling/recall concern: with ≥93% of expressions containing time tokens and current recall ≈92%, the paper should discuss strategies to improve beyond this ceiling.  
- Release and replication concern: unclear whether the full rule set, token regexes, and code will be released for verification and reuse.  
- Request for specifics: ask authors to list the five seasons, clarify dataset count, give examples of expressions without time tokens and how rules capture them.  
- Request for future directions: ask how recall will be improved (e.g., covering token-less expressions, lightweight context features, better POS/lexical disambiguation).  
- Request for release details: ask if and under what license the complete resources and documentation will be published.  
- Overall: a simple, effective, and well-motivated system with state-of-the-art results; acceptance recommended contingent on clarifying specs and release plans.

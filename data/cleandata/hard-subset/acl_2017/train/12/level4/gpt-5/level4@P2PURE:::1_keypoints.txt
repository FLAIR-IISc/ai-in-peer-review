- Proposes a rule-based time expression extraction method; key insight: expressions are short and contain at least one time token.
- Identifies time tokens via dictionary lookup and regular-expression matching, augmented with POS tagging.
- Expands spans bidirectionally from the time token using heuristic rules until boundaries are reached.
- Merges resulting segments into single time expressions using a second set of rules.
- Evaluated on three datasets against both rule-based and ML systems, showing significant improvements.
- Strength: paper is well written and clearly presented.
- Strength: rules are empirically motivated and justified by evaluation results.
- Weakness: some details are underspecified, making reproduction difficult.
- Question (Sec 4.1): why are there only five seasons and how are multi-month periods (e.g., Ramadan, Holiday Season) handled?
- Question (Sec 5.1): apparent mismatch—should “two benchmark datasets” be “three datasets”?
- Request (Sec 5.2): provide an example of a time expression without a time token.
- Question (Sec 5.2): given 93% of expressions contain a time token and system recall is 92%, how can performance be further improved?
- Question: is there a plan to release the full set of rules and the software used?

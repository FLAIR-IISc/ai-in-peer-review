- Paper proposes SynTime, a lightweight rule-based time expression recognizer using syntactic token types and boundary-expansion heuristics.
- Authors provide descriptive analyses of four datasets (TimeBank, Gigaword, WikiWars, Tweets) motivating the type-based approach.
- Main contributions: simple token-type/boundary-expansion recognizer, cross-corpus descriptive analysis, and an engineering finding that minimal rule sets can match heavier systems.
- Strong empirical performance with particularly high recall and competitive F1, notably excellent results on tweets.
- Method is simple, interpretable, requires no training, and is likely fast — promising real-time use.
- Design is clearly motivated by corpus statistics (short spans, small lexicon, POS regularities) and sensible POS-based disambiguation.
- Approach adapts to domains via small lexicon expansions, improving recall with modest annotation cost.
- Comparisons to reasonable baselines (HeidelTime, SUTime, UWTime) using TempEval-3 evaluation are appropriate.
- Novelty is limited relative to existing systems (especially SUTime), with conceptual overlap in token classes and chunk expansion.
- Tweets dataset selection likely biased by prefiltering with SUTime, which may advantage SynTime and underrepresent expressions SUTime misses.
- Initialization from SUTime’s regexes and manual lexicon additions make it unclear whether improvements stem from rules or lexicon differences.
- Reproducibility is limited because exact token lists, added items, and code were not provided.
- Missing ablations, error analysis, and runtime measurements leave unclear which components drive performance and the validity of “real-time” claims.
- Scope claims (e.g., “principle of least effort”) are speculative and normalization (important for applications) is not addressed.
- Request for authors: provide an unbiased tweets test set or rigorously quantify selection bias.
- Request for authors: specify baseline versions, resources, and tuning to ensure fair comparisons with SynTime-E.
- Request for authors: release code, full token-type lexicon, and annotated tweets for reproducibility.
- Request for authors: report ablations for POS filtering, COMMA/LINKAGE rules, dependent-segment handling, and initialization without SUTime regexes.
- Request for authors: report throughput/latency comparisons with SUTime/HeidelTime to substantiate efficiency claims.
- Overall: solid engineering contribution with practical value and strong recall, but limited novelty and fairness/reproducibility concerns that should be addressed.

- Insightful dataset analysis: time expressions are short, use a small vocabulary, show high token overlap and consistent syntax, tied to Zipf’s least-effort principle.  
- Simplicity and efficiency: SynTime uses syntactic types and heuristic rules, making it lightweight and suitable for real-time processing.  
- Strong empirical results: outperforms HeidelTime, SUTime, UWTime on TimeBank and Tweets with notable recall gains; ablation (SynTime-I vs SynTime-E) shows benefits of expansion.  
- Clear presentation: well-written with useful diagrams and easy-to-follow type and rule explanations.  
- Limited novelty: approach overlaps conceptually with existing rule-based systems (e.g., SUTime); main novelty is in specific type definitions and expansion heuristics.  
- Reliance on existing tools/data: initialization uses SUTime regexes and Stanford POS tags, which may inherit limitations and cause error propagation.  
- Limited error analysis: paper lacks detailed breakdown of failure cases and dataset-specific errors (e.g., WikiWars performance).  
- Scalability concerns: manual keyword annotation for expansion may not scale to new domains/languages; Twitter clustering mitigation is under-justified.  
- Request for comparison: need a detailed, example-driven comparison of SynTime vs SUTime type definitions to show why syntactic typing yields gains.  
- Request for runtime data: provide processing speed (sentences/sec) vs baselines across datasets to substantiate real-time claims.  
- Request for error analysis: specify the most common errors, problematic syntactic patterns, and dataset-specific failure modes.  
- Request on scalability: clarify how expansion effort scales and whether semi-/distant-supervision was explored to reduce annotation load.  
- Reference issues: bibliography contains formatting inconsistencies.  
- Dataset omission: clarify why Gigaword is mentioned in the introduction but not used in experiments.  
- Dataset limitations: discuss constraints of the datasets used (e.g., the Twitter dataset is relatively small compared to other Twitter corpora).

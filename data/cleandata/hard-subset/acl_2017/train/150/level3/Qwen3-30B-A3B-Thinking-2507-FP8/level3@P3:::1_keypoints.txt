- Proposes DCNMT: a deep character-level NMT that learns morphology via a hierarchical architecture using morpheme representations between character-level input and output.  
- Architecture uses six recurrent networks in a hierarchy to enable handling of unseen words through morpheme-based backoff.  
- Claims competitive BLEU scores while avoiding large vocabulary bottlenecks of word-based models.  
- Main contribution: a word encoder with two RNNs plus a hierarchical character-level decoder that learns morphology.  
- Main contribution: fast training—reasonable translation quality after one epoch, improving with more epochs.  
- Main contribution: ability to learn meaningful morphological representations, demonstrated by correct translation of nonce words.  
- Strength: effective handling of unseen/morphologically rich words, shown with compelling examples.  
- Strength: authors release code and models for fr-en, cs-en, en-cs, aiding reproducibility.  
- Strength: demonstrated training efficiency, outperforming BPE baseline after one epoch.  
- Weakness: paper does not clearly state which WMT test/dev sets are used (should specify newstest2013/dev and newstest2015/test).  
- Weakness: README lacks runnable examples and --help support; reproducibility hindered without a minimal working example.  
- Weakness: Figure 2/caption inconsistent with text—claims 6-level architecture but figure seems to show fewer layers; needs full architecture diagram.  
- Weakness: comparison omits state-of-the-art non-character MT systems and higher WMT scores, making relative performance unclear.  
- Weakness: minor English disfluencies, misspellings, and LaTeX issues to fix.  
- Question: please confirm which specific WMT test/dev sets were used.  
- Question: please add a minimal working example in the README demonstrating how to run the model.  
- Question: please provide a complete architecture diagram showing all six recurrent networks and their connections.  
- Question: how does the model compare to the highest-scoring systems on WMT 2015/2016 (matrix.statmt.org) and similar benchmarks?

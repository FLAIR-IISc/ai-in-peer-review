- Novel character-based encoder-decoder that integrates morphemic representations, operating on characters at input/output and morphemes internally.  
- Authors release code and trained models for fr-en, cs-en, and en-cs, improving reproducibility.  
- Early training yields reasonable translation quality after one epoch with steady improvement thereafter.  
- System learns meaningful morphological segmentations and generalizes to unseen and nonce words via morphemic decomposition.  
- Missing dataset specifications: paper does not name which WMT test/dev sets were used, hindering direct comparisons.  
- Inadequate software documentation: README lacks concrete training/testing command examples and a standard --help flag.  
- Figure 2 ambiguity: described as "6-level" but figure shows fewer layers; caption or supplementary full-architecture figure is needed.  
- Insufficient SOTA context: comparisons only to character baselines; reported BLEU scores fall well below recent WMT SOTA and reasons should be discussed.  
- Minor language issues: contains grammatical errors, misspellings, and LaTeX inconsistencies that should be fixed.  
- Overall: valuable morpheme-aware contribution bridging character and word-level NMT, promising handling of unseen words, code release strengthens impact, and with the noted revisions the paper would be a strong ACL 2017 submission.

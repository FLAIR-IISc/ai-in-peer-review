- Strengths:
The paper introduces a novel encoder–decoder NMT approach that inputs and outputs characters while using internal representations at the morpheme and character levels.
The authors release both the code and the final trained models for fr-en, cs-en, and en-cs, which supports validation and enables others to replicate and extend the work.
The system achieves reasonable translation quality after the first training epoch and continues to improve over subsequent epochs.
It learns sensible morphological segmentations and can handle previously unseen (including nonce) words by implicitly backing off to morphemes.

- Weaknesses:
The paper does not explicitly state which WMT dev and test sets were used, making it difficult to compare results to prior work (e.g., matrix.statmt.org). The only place this information appears is in the code README, which indicates that the test set is newstest2015 and the dev set is newstest2013. This should be stated clearly in the paper.
The software README is adequate but could be improved. The training and testing sections should include concrete command-line examples, and the software should respond to a --help flag, which it currently does not.
Although the paper describes a six-layer architecture, Figure 2 appears to depict fewer than six layers. The caption should clarify this, and if the figure omits layers, there should be a figure (even in an appendix) that shows the complete architecture.
The results compare against other character-based neural systems but do not include state-of-the-art results from other types of MT systems. WMT (and matrix.statmt.org) report results on these datasets that appear to be much higher than those in the paper. This should be acknowledged and, ideally, discussed.
There are minor English issues (disfluencies, misspellings) and small LaTeX problems (e.g., reversed quotation marks) that should be corrected.

- General Discussion:
Overall, this is a solid contribution to the literature on character-based neural MT.

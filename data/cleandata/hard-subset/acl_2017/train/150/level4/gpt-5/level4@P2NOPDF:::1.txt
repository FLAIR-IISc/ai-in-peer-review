- Strengths:
The paper introduces a novel encoder-decoder NMT approach that takes and produces characters, while internally representing morphemes and characters. 
The authors release both their code and pretrained models for fr-en, cs-en, and en-cs, enabling validation, replication, and extension by others.
The system achieves reasonable translation quality after the first training epoch and continues to improve in later epochs.
It learns plausible morphological segmentations and can handle unseen or nonce words by backing off to morphemic representations.

- Weaknesses:
The paper does not explicitly state which WMT dev and test sets are used. This hinders comparisons with prior work (e.g., matrix.statmt.org). The README indicates test = newstest2015 and dev = newstest2013; this information should appear clearly in the paper.
The README instructions are adequate but could be clearer. Include explicit command-line examples for training and inference. The software should also support a --help flag, which it currently lacks.
The text describes a six-layer architecture, but Figure 2 seems to show fewer layers. The caption should clarify what is depicted. If the figure is simplified, include a complete diagram (possibly in an appendix) that shows all layers.
The results compare against other character-based NMT systems but omit state-of-the-art baselines from other MT paradigms. WMT and matrix.statmt.org report stronger systems on these datasets. This gap should be acknowledged and discussed.
There are several minor English issues (disfluencies, misspellings) and LaTeX problems (e.g., reversed quotation marks) that should be corrected.

- General Discussion:
This is a solid contribution to the literature on character-based neural MT.

- Proposes a simple memory-network architecture that flattens structural linguistic substructures into vectors used as extra context for sequence tagging.  
- Practical method that resembles prior structured/syntax-based attention and substructure encoders (e.g., treeLSTM, DCNN), so not highly novel but straightforward and usable.  
- Shows that readily available linguistic structures (dependency parses, AMR) can improve NLU slot-filling performance.  

- Unclear baseline implementation: paper does not state whether TreeRNN/DCNN baselines produced independent substructure embeddings integrated into the tagger.  
- Model uses two RNNs (chain-based and knowledge-guided) where the only difference is adding the knowledge vector, which may simply add parameters without justification.  
- No baseline that incorporates the same knowledge as simple input features (e.g., head words, NER) for a fairer comparison.  
- Missing analysis of model sensitivity to parser errors.  

- Attention over substructures is static across words; per-word attention over substructures would be more natural and potentially stronger.  
- Inner-product attention between sentence and substructure embeddings (built by same encoder) may bias selection toward nodes similar to the whole sentence, e.g., leaf nodes.  
- Framework requires substructures to be sequences of words, making it unclear how to incorporate constituent parses effectively.  
- Referring to syntactic/semantic parses as “knowledge” is misleading since it usually denotes world/external KB knowledge.  

- Overall: practical and effective on one dataset but not groundbreaking; experimental evidence is unconvincing without clearer baselines and justification for the dual-RNN design.  
- Post-rebuttal: authors did not resolve key concerns about baseline usage and the dual-RNN parameter advantage, so the reviewer’s original assessment stands.

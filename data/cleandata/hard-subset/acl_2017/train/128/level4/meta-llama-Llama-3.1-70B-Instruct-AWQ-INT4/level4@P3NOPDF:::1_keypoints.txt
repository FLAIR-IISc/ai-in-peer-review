- Proposes a practical method to flatten structured linguistic information into a memory array of substructure embeddings fed as context to an RNN for sequence tagging.

- Main idea similar to prior structured/syntax-based attention work (e.g., treeLSTM attention, Zhao et al., Liu et al., Eriguchi et al.), with substructure encoder resembling DCNN.

- Strength is simplicity and practicality compared to more complex prior architectures.

- Major empirical concern: unclear how baseline models (TreeRNN, DCNN) were used to compute sentence vs. substructure embeddings.

- Using two separate RNNs (chain-based and knowledge-guided) seems unjustified when the only difference is an added memory vector, effectively inflating parameters versus baselines.

- Paper omits hyperparameter and model size details for baselines, hampering fair comparison.

- Missing baseline that injects the same knowledge as additional input features (e.g., head word, NER), which would test whether the memory adds value beyond feature inclusion.

- No analysis of sensitivity to parser errors, despite reliance on parsed substructures.

- Attention is static across words rather than word-specific, which may be suboptimal and reduces the memory to a global filter.

- Weighting substructures by inner product of sentence and substructure embeddings (from the same encoder) risks biasing toward nodes similar to the whole sentence, e.g., leaf nodes.

- Claim of generalizing to different knowledge types is unclear for non-sequential representations (e.g., constituency trees).

- Terminology issue: calling syntax/semantic parse-derived vectors "knowledge" is misleading compared to external knowledge bases.

- Overall: not especially novel for ACL, experiments unconvincing, and authors did not address main concerns in the rebuttal (baseline usage and parameter mismatch).

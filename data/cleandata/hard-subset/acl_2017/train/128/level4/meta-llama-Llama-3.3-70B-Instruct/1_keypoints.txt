- Paper proposes encoding parse substructures as memory vectors and using their weighted sum as extra context in an RNN for sequence tagging (slot-filling).
- Strength: simple, practical “flattening” of structured information into a memory integrated into a tagger, related to syntax-based attention and prior tree/CNN encoders.
- Strength: substructure encoder is similar to DCNN-style ancestor-word embeddings; simplicity is an asset even if not novel.
- Main empirical concern: results are unconvincing due to missing details about how baselines were implemented.
- Unclear whether TreeRNN and DCNN baselines were used to compute both sentence and node embeddings or only sentence embeddings in Table 1.
- Questionable model design: using two separate RNNs (chain and knowledge-guided) seems to mainly increase parameter count rather than provide a principled advantage.
- Missing baseline: a model that inputs the same knowledge as explicit features (e.g., syntactic head, NER) should be compared.
- Experiments lack analysis of sensitivity to parser errors.
- Modeling choice concern: static attention over substructures for all words is unexplained—word-specific attention would be more natural.
- Weighting via inner product between sentence and substructure embeddings may bias toward nodes similar to the whole sentence, possibly over-affecting leaf nodes.
- Claim of generalization to different types of knowledge is weak—representing some structures (e.g., constituents) as word sequences is not always straightforward.
- Terminology issue: calling syntactic/AMR structures “knowledge” is misleading, since “knowledge” typically means external/world knowledge.
- General verdict: practical model with one-dataset gains but limited novelty and insufficient experimental clarification for an ACL paper.
- Post-rebuttal: authors did not resolve whether baselines compute substructure embeddings independently and did not justify extra parameters from two RNNs, leaving results unchanged.

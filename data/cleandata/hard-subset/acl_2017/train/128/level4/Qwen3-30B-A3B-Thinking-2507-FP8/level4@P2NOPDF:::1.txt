

This paper introduces a neural architecture that incorporates structural linguistic knowledge into a memory network for sequence tagging, specifically slot-filling in conversational systems. Substructures (e.g., parse tree nodes) are encoded as vectors (memory slots), and a weighted sum of these embeddings is fed into an RNN at each time step as contextual input for labeling.  

**Strengths**  
The paper’s core contribution is a straightforward method to "flatten" structured information into a vector array (the memory), which is then integrated with the tagger. While conceptually related to syntax-based attention (e.g., treeLSTM) and prior work on textual entailment (Zhao et al.), natural language inference (Liu et al.), and machine translation (Eriguchi et al.), the proposed substructure encoder resembles DCNN (Ma et al.) in embedding nodes from ancestor word sequences. Though not entirely novel, the approach is practical and well-motivated, offering a clear implementation path compared to more complex alternatives.  

**Weaknesses**  
The empirical evaluation lacks sufficient detail to validate claims, particularly regarding baseline comparisons. Key concerns, ranked by importance:  
1.  **Baseline implementation ambiguity**: The model comprises sentence and substructure embeddings. Table 1 compares against TreeRNN and DCNN, originally designed for *sentence* embedding. It is unclear how these baselines were adapted to provide *substructure embeddings* (e.g., whether node embeddings were extracted independently). This undermines the fairness of comparison.  
2.  **Unnecessary model complexity**: The model employs two RNNs (chain-based and knowledge-guided), differing only in the inclusion of a "knowledge" vector (Eqns 5, 8). Using separate weights for both RNNs merely increases capacity (parameters) without clear justification. Baselines should have comparable parameter counts.  
3.  **Missing baseline**: A simpler baseline—inputting external features (e.g., word heads, NER tags) directly into the RNN—would better isolate the contribution of the memory mechanism.  
4.  **Parser robustness**: No analysis of sensitivity to parser errors (e.g., how inaccuracies in structural input affect performance) is provided.  

**Additional Concerns**  
- The static attention over substructure embeddings (applied uniformly per word) seems suboptimal. Dynamic attention (e.g., per-word weighting) would better leverage structural context. This reinforces the need for the missing baseline.  
- The weight computation (inner product of sentence and substructure embeddings) risks favoring leaf nodes (since they resemble the full sentence), potentially undermining the intended structural focus.  
- The claim of generalizability to "different knowledge" is questionable. The method requires substructures to be representable as word sequences (e.g., constituent parses), making it incompatible with non-sequence-based knowledge (e.g., AMR).  
- **Terminology**: The term "knowledge" is misleading. It typically denotes external knowledge bases (e.g., entity databases), not syntactic/semantic structure. "Structural context" would be more accurate.  

**General Discussion**  
While the model is practical and effective on one dataset, the core idea lacks novelty (as noted in Strengths). For an ACL publication, the work should offer stronger theoretical or empirical insights. Crucially, the experiments fail to convincingly demonstrate superiority due to ambiguous baselines, unnecessary complexity, and missing comparisons. Clarification on baseline implementation and parameter alignment is essential for a fair assessment.  

**Post-rebuttal**  
The authors did not address the critical issue of whether baselines (e.g., TreeRNN) were used to generate *independent* substructure embeddings (not jointly trained with the tagger). They also failed to justify the two-RNN architecture, which inflates model capacity without clear benefit. Consequently, the empirical claims remain unconvincing, and my scores are unchanged.



This paper introduces a neural architecture that incorporates structural linguistic knowledge into a memory network for sequence tagging, specifically targeting slot-filling in conversational systems' natural language understanding modules. It represents substructures (e.g., parse tree nodes) as vectors stored in memory slots, feeding a weighted sum of these substructure embeddings into an RNN at each time step as contextual input for labeling.  

**Strengths**  
The core contribution lies in a simple method to "flatten" structured information into a vector array (the memory), which is then integrated with the tagger. While conceptually similar to syntax-based attention (e.g., treeLSTM) and related work by Zhao et al. (textual entailment), Liu et al. (NLI), and Eriguchi et al. (machine translation), the proposed substructure encoder resembles DCNN (Ma et al.), embedding each node from a sequence of ancestor words. Though not entirely novel, the approach offers a practical and straightforward alternative to prior methods.  

**Weaknesses**  
The empirical results are unconvincing due to insufficient baseline details. Key concerns, ranked by importance:  
- **Baseline ambiguity**: Table 1 compares against TreeRNN and DCNN, originally designed for sentence embedding. However, it is unclear how these models generate *substructure embeddings* (the second component of the proposed model) versus their standard sentence-level use.  
- **Redundant architecture**: The model employs two separate RNNs (chain-based and knowledge-guided), differing only in the input of a "knowledge" vector (Eq. 5, 8). Using distinct weights for both RNNs appears unnecessary; the sole benefit is increased model capacity (more parameters). Baseline networks must have comparable parameter counts to ensure fair comparison.  
- **Missing baseline**: A simple baseline—incorporating external features (e.g., word heads, NER tags) directly into the RNN—should be included to isolate the value of the memory mechanism.  
- **Parser sensitivity**: No analysis is provided on how parser errors affect model performance.  

**Additional Comments**  
- After computing substructure embeddings, attention over them per word would be natural. Using a static attention for all words seems suboptimal; the current "knowledge" vector functions more like a fixed filter. This reinforces the need for the baseline mentioned above.  
- The weight calculation (inner product of sentence and substructure embeddings, both derived from the same RNN/CNN) risks over-weighting leaf nodes similar to the sentence, potentially undermining the design.  
- The claim of generalizing to diverse knowledge sources is questionable. Substructures must be represented as word sequences (e.g., via constituent parsing), making it unclear how to integrate, say, AMR-based semantic knowledge without significant adaptation.  
- Terminology is misleading: Labeling this "knowledge" is inappropriate, as the term typically refers to external world knowledge (e.g., knowledge bases), not syntactic or semantic structure.  

**General Discussion**  
While the model is practical and effective on one dataset, the core ideas lack novelty (as noted in Strengths). For an ACL publication, stronger theoretical or methodological insights are expected. More critically, the experiments require substantial clarification to be credible.  

**Post-rebuttal**  
The authors failed to address the primary concern: whether baselines (e.g., TreeRNN) were used to derive substructure embeddings *independently* of the sentence embedding and joint tagger. They also did not justify the two RNNs’ separate weights, which inflate parameter count relative to baselines. I maintain my original score.

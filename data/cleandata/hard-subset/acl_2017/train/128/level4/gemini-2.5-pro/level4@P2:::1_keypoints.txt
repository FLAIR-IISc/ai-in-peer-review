- Summary: Proposes K-SAN, which encodes dependency/AMR substructures into a knowledge memory and uses attention to produce a structural context vector fed into an RNN tagger for slot-filling, showing gains on ATIS, especially with limited data.

- Main contribution: A simple, modular method to inject structural linguistic priors by flattening substructures into embeddings and using global attention to produce a fixed-size structural context for sequence models.

- Strength 1: Simplicity and modularity—avoids complex tree RNNs by condensing structure into a single context vector easily integrated with standard sequential architectures.

- Strength 2: Flexibility—demonstrates applicability across multiple structural sources (different dependency parsers and AMR), indicating generality beyond a single syntactic formalism.

- Strength 3: Empirical benefit in low-data regimes—shows notable performance improvements over baselines on small/medium training sets, suggesting structural priors help generalization with scarce labels.

- Weakness 1: Baseline comparison unclear—insufficient detail on how Tree-RNN and DCNN baselines were adapted/integrated, making fairness of comparisons uncertain.

- Weakness 2: Potential parameter-count confound—joint model uses two RNNs, effectively increasing parameters; improvements may be from model size rather than architecture, and baseline parameter parity is not reported.

- Weakness 3: Missing simple baseline—no straightforward RNN tagger that uses structural info as features (e.g., head-word embedding concatenation) to isolate the benefit of the memory+attention mechanism.

- Weakness 4: Static sentence-level attention—model uses a single attention distribution for the whole sentence instead of dynamic, per-token attention, which may be suboptimal for sequence tagging.

- Weakness 5: Terminology mislead—calling syntactic/semantic structure "knowledge" is misleading; terms like "structural information" or "linguistic priors" would be more precise.

- Question 1: How exactly were Tree-RNN and DCNN baselines implemented and integrated—sentence encoders only or using substructure representations comparable to K-SAN?

- Question 2: Why use two separate RNNs in the joint model; were experiments run with a single RNN incorporating the knowledge vector and how do parameter counts compare to baselines?

- Question 3: Why choose a static attention mechanism rather than dynamic per-timestep attention more suited to word-level tagging?

- Question 4: Does computing attention via inner-product between global sentence embedding and substructure embeddings bias attention toward globally similar but locally irrelevant substructures?

- Question 5: Has the model's robustness to parser errors been evaluated, given reliance on structural inputs for real-world applicability?

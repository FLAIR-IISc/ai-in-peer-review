### Summary of the Paper

This paper addresses the task of weakly supervised cross-lingual Named Entity Recognition (NER), aiming to build NER systems for target languages without requiring any annotated data in those languages. The authors propose and evaluate two main projection-based approaches from a resource-rich source language (English). The first approach, based on annotation projection, uses comparable corpora to project NER labels from English to a target language. The second approach, based on representation projection, learns a linear mapping to project target-language word embeddings into the English embedding space, allowing a single English NER model to be applied directly to the target language.

The primary contributions of this work, as I see them, are:
1.  An extensive empirical evaluation demonstrating that combining annotation projection and representation projection approaches through simple co-decoding schemes leads to state-of-the-art performance on standard CoNLL benchmarks for weakly supervised cross-lingual NER.
2.  A heuristic data selection method for the annotation projection approach that effectively filters noisy, automatically-labeled data. This method is shown to provide significant performance gains, particularly when the underlying alignment quality between source and target languages is low.

### Strengths

1.  **Strong Empirical Results:** The paper's most significant strength is its strong performance on the CoNLL 2002/2003 datasets. The proposed combined system outperforms two recent state-of-the-art methods (Täckström et al., 2012; Tsai et al., 2016), particularly showing a large improvement for German. These results provide a new, strong baseline for future work in this area.

2.  **Comprehensive Evaluation:** The authors conduct a thorough evaluation across a wide range of languages (Japanese, Korean, German, Portuguese, Spanish, Dutch) and two different datasets (an in-house dataset with many entity types and the standard CoNLL dataset). This demonstrates the general applicability of the proposed methods and provides valuable insights into how they perform under different conditions (e.g., different entity sets, different languages).

3.  **Effective Data Selection Heuristic:** The proposed data selection scheme for annotation projection, while simple, is shown to be highly effective. In cases of low-quality word alignments (Japanese and Korean), filtering the projected data using the quality and entity count thresholds leads to dramatic improvements in F1 score (+13.9 for Japanese, +20.0 for Korean). This highlights the critical importance of managing noise in projection-based approaches and provides a practical method for doing so.

### Weaknesses

1.  **Limited Novelty of Core Methods:** The overall framework relies on two well-established paradigms for cross-lingual transfer: annotation projection and direct model transfer via representation mapping. Neither of these core ideas is new. The specific implementations are incremental improvements on prior work—the annotation projection uses a straightforward frequency-based quality score, and the representation projection is a direct application of the mapping technique from Mikolov et al. (2013b) to the NER task. While the combination is effective, the underlying components lack significant methodological innovation.

2.  **Unclear and Potentially Flawed Hyperparameter Tuning:** The data selection scheme introduces two crucial hyperparameters: the quality score threshold `q` and the entity number threshold `n`. The paper states for Table 2, "we have tried different threshold parameters q and n and reported the best numbers." This raises a major concern about reproducibility and the validity of the results. It is unclear if these thresholds were tuned on a held-out development set or on the test set itself. If tuned on the test set, the reported improvements are unreliable and likely overestimated. Even if a development set was used, the paper provides no details about this process.

3.  **Lack of Practical Guidance for Data Selection:** Related to the previous point, the paper offers no guidance on how to set the `q` and `n` thresholds for a new language pair where no labeled data (not even a development set) is available, which is the central premise of the weakly supervised setting. The optimal thresholds reported in Table 2 vary significantly across languages. Without a principled way to select these parameters, the practical utility of the data selection method is limited, as a user would have no way of knowing which parameters to choose to achieve the reported gains.

4.  **Clarity of Co-Decoding Schemes:** The description of the co-decoding schemes is brief and lacks sufficient detail. The "exclude-O confidence-based" scheme requires comparing confidence scores from two different model architectures (MEMM and a neural network), but it is not explained how these scores are normalized or made comparable. The "rank-based" scheme is described as giving priority to the annotation projection system and then adding non-conflicting entities from the representation projection system. The definition of "conflict" is provided, but the mechanism for resolving partial overlaps (e.g., [Barack Obama] vs. [Barack]) is not specified. This lack of clarity makes the "contribution" of the co-decoding hard to fully assess and reproduce.

### Questions to Authors

1.  Regarding the data selection thresholds `(q, n)` reported in Table 2: How were these values selected for each language? Was a labeled development set used for tuning? If so, please provide details on the data splits. If not, how can you ensure that the reported gains are not a result of overfitting to the test set?
2.  Following up on the previous question, how would a practitioner choose appropriate values for `q` and `n` for a new target language in a true zero-shot scenario (i.e., with no labeled data available for tuning)? Does the optimal setting correlate with any observable properties of the data, such as alignment accuracy?
3.  Could you provide more details about the in-house NER dataset? Specifically, what is the domain or genre of the text (e.g., newswire, web text), and what are some examples of the 50+ entity types? This context is important for interpreting the results in Table 3 and understanding why the performance characteristics (e.g., P/R trade-offs) differ from the CoNLL experiments.
4.  In the rank-based co-decoding, you mention adding entities from the second system that do not "conflict" with entities from the first. How are conflicts involving partial boundary overlaps resolved? For example, if System A predicts `[PER: Barack Obama]` and System B predicts `[PER: Obama]`, is the entity from System B discarded, or is some form of merging performed?

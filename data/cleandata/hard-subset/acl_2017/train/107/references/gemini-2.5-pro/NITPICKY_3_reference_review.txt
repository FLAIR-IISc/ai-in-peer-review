### Summary and Contributions

This paper presents a comprehensive study on weakly supervised cross-lingual Named Entity Recognition (NER), proposing and evaluating two main approaches: Annotation Projection (AP) and Representation Projection (RP). The AP method leverages aligned corpora to project NER tags from a source to a target language, introducing a novel heuristic data selection scheme to filter noisy projections. The RP method trains a source-language NER model on word embeddings and directly applies it to the target language by learning a linear mapping between the embedding spaces. Finally, the authors propose two co-decoding schemes to combine the outputs of the AP and RP systems, demonstrating that this combination achieves state-of-the-art results on the CoNLL benchmark datasets and performs strongly on a set of in-house datasets.

As I see them, the main contributions of this work are:
1.  A strong empirical demonstration that combining two distinct and complementary paradigms of cross-lingual transfer (annotation projection and representation projection) leads to state-of-the-art performance. The analysis of the precision/recall trade-offs of each system provides a solid motivation for their combination.
2.  A simple yet effective heuristic data selection method for annotation projection that provides significant gains, especially in low-resource or low-quality alignment scenarios.
3.  A thorough evaluation across a diverse set of languages (6 in total) and two distinct NER task formulations (fine-grained in-house vs. standard CoNLL), which provides a robust validation of the proposed methods.

### Strengths

1.  **Strong Empirical Results and State-of-the-Art Performance:** The paper's primary strength lies in its experimental results. The proposed combined system surpasses previous state-of-the-art methods from Täckström et al. (2012) and Tsai et al. (2016) on the standard CoNLL 2002/2003 test sets. This is a significant result and provides a new, strong baseline for the field.
2.  **Effective Combination of Complementary Methods:** The authors astutely observe that their two proposed systems exhibit different characteristics: the AP system tends to have high precision and low recall, while the RP system is more balanced. This insight directly motivates the co-decoding approach. The proposed rank-based co-decoding scheme, which prioritizes the high-precision AP system and uses the RP system to fill in potential gaps, is intuitive and empirically shown to be highly effective.
3.  **Practicality and Generality of the Approach:** The methods require no annotated data in the target language, making them highly practical. The data selection scheme for AP is a useful tool for anyone working with noisy projected data. The RP approach is particularly elegant, as a single source-language model can be deployed for multiple target languages simply by learning a new embedding mapping, without any model retraining. The evaluation on both fine-grained (>50 types) and coarse-grained (4 types) entity sets demonstrates the versatility of the methods.

### Weaknesses

Despite the strong results, the paper suffers from a number of methodological ambiguities and a lack of detailed analysis that detract from its quality.

1.  **Vague Methodological Descriptions:** Several key components of the methodology are insufficiently explained, hindering reproducibility and critical assessment.
    *   **Data Selection Score (Sec 3.1):** The definition of the quality score `q(y)` in Equation 5 is unclear. The score for a sentence `y` depends on the frequency `T(e, l'(e))` for each entity `e` in that sentence, where `l'(e)` is its projected tag. However, the frequency table `T` is built from the entire corpus, where a single entity string (e.g., "Estados Unidos") may be projected to multiple different tags. The paper does not specify how `l'(e)` is chosen for the lookup. Is it the tag projected in that specific instance? If so, the metric's reliability is questionable. A more principled metric, like the entropy of the tag distribution for the entity string, would be more robust.
    *   **Representation Projection OOV Handling (Sec 4.3):** The OOV handling strategy is described as: "Words which are not in the target vocabulary are projected into their English embeddings if found in the English vocabulary, or into an NER-trained unk vector otherwise." This is a critical detail that is glossed over. How is the "NER-trained unk vector" trained? What is the frequency of these OOV cases? This ad-hoc backoff strategy could be a significant confounder, especially for language pairs with many cognates or loanwords, and its impact is not analyzed.
    *   **Neural Network Architecture (Sec 2.2):** The description of the NN2 architecture is too high-level. A "smoothing layer that computes the cosine similarity between a word embedding and a fixed set of prototype vectors...and returns a weighted average" is a non-standard component that requires a more formal definition and justification. How are these prototypes learned? How many are used?

2.  **Insufficient Analysis and Ablation Studies:** The paper presents final results but lacks the deeper analysis needed to understand *why* the methods work.
    *   **Hyperparameter Tuning:** The data selection thresholds `(q, n)` in Table 2 and the choice between co-decoding schemes for the CoNLL experiments are tuned on the test set (or development set, which serves as a test set for tuning). This is oracle tuning. The paper fails to propose a strategy for setting these crucial hyperparameters in a realistic scenario where no labeled target data is available, thereby overstating the practical, zero-shot performance.
    *   **Embedding Dictionary Size:** The claim that dictionaries larger than 5-6K pairs "might harm the performance" (Sec 4.2) is counter-intuitive and presented without any supporting evidence. This is a critical implementation detail that warrants a proper ablation study to validate the claim and provide insight.
    *   **Model Architecture Choices:** The AP system uses a 2nd-order MEMM while the RP system uses a neural network. This makes it difficult to disentangle the effects of the projection method from the effects of the model architecture. A more controlled comparison using the same model family for both would have been more insightful. The justification for avoiding CRFs due to training time with many entity types is weak; modern toolkits are quite scalable.

3.  **Minor Formatting and Presentation Issues:** The paper exhibits a lack of polish.
    *   **Citations:** In-text citations are often incorrectly formatted (e.g., using `(Author, Year)` instead of `Author (Year)` when grammatically required).
    *   **Footnotes:** Critical information, such as the definition of conflicting entities for rank-based co-decoding (footnote 3), is relegated to footnotes, disrupting the flow of reading.
    *   **Table Clarity:** In Table 1, the values appear to be normalized probabilities, but the text refers to "frequency". This should be consistent. The caption "Same for Tables 4 and 5" is ambiguous.

### Questions to Authors

1.  Could you please provide a more precise algorithm or formula for calculating the quality score `q(y)` in Section 3.1? Specifically, how do you handle entity strings that are mapped to multiple different NER tags across the projected corpus when looking up `T(e, l'(e))`?
2.  The data selection thresholds `(q, n)` are tuned on labeled data. In a true zero-shot setting, how would you propose to set these values? Have you explored any heuristics based on the distribution of quality scores or the desired size of the training set?
3.  Could you provide an empirical analysis to support the claim that using more than 5-6K word pairs for learning the cross-lingual embedding mapping harms performance? This is a surprising result that merits further investigation.

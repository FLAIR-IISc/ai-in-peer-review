**Summary and Contributions**
This paper addresses the problem of cross-lingual named entity recognition (NER) in a weakly supervised setting, where no annotated data is available for the target language. The authors propose and evaluate two distinct approaches and then combine them for improved performance.

The main contributions of this work are:
1.  A method for improving annotation projection by using a novel heuristic data selection scheme. This scheme filters the noisy, automatically projected annotations from a comparable corpus to create a higher-quality weak training set for the target language. The effectiveness of this filtering is demonstrated to be particularly high when word alignment quality is low.
2.  Two co-decoding schemes that effectively combine the outputs of two very different systems: one based on annotation projection (high precision) and another on representation projection (more balanced). The combined system is shown to outperform both individual systems and achieve state-of-the-art results on the CoNLL benchmark.
3.  A comprehensive empirical evaluation across seven target languages and two distinct datasets (a fine-grained in-house set and the standard CoNLL benchmark), demonstrating the robustness of the proposed methods.

**Strengths**
1.  **Effective and well-motivated methodology**: The paper tackles the problem from two complementary angles: projecting annotations (the labels) and projecting representations (the features). This dual-pronged approach is a key strength. Furthermore, the proposed components are well-motivated. The data selection heuristic for annotation projection directly addresses the known issue of noise in projected labels, and the co-decoding schemes are intelligently designed based on the observed precision/recall characteristics of the individual systems.
2.  **Significant empirical results**: The proposed combined system outperforms two recent state-of-the-art approaches (Täckström et al., 2012; Tsai et al., 2016) on the standard CoNLL test sets for Spanish, Dutch, and German. The performance gains, especially for German (+5.4 F1 over the best prior result), are substantial and establish a strong new baseline for this task.
3.  **Thorough evaluation**: The experiments are extensive, covering a wide range of languages from different families (Japanese, Korean, German, Portuguese, Spanish, Dutch) and two different NER settings (an in-house dataset with 50+ fine-grained entity types and the standard CoNLL dataset with 4 types). This breadth demonstrates that the proposed methods are not tuned to a specific language pair or label set, strengthening the paper's claims of general applicability.
4.  **Practical value of the data selection scheme**: The simple heuristic for filtering projected data yields very large improvements (e.g., +13.9 F1 for Japanese and +20.0 F1 for Korean). This is a valuable practical contribution for anyone using annotation projection, as it provides a simple yet powerful way to handle the noise that plagues this method, especially when dealing with lower-quality alignments.

**Weaknesses**
1.  **Lack of detail on hyperparameter tuning for data selection**: The data selection scheme for annotation projection relies on two thresholds, q and n. The paper reports results using the best-performing parameters but does not discuss how these would be chosen in a true zero-shot setting where no labeled development set exists for the target language. Providing some guidance or analysis on the sensitivity of the model to these parameters would make this contribution more practical and complete.
2.  **Limited analysis of model behaviors**: The paper presents strong aggregate results but would benefit from a more in-depth analysis of the models' behavior. For instance, it is noted that the NN1 architecture is better for CoNLL data while NN2 is better for the in-house data, but there is no discussion as to why this might be. Is it related to the number of entity types, the domain, or some other factor? Similarly, a qualitative error analysis showing what kinds of errors are corrected by the co-decoding schemes would provide valuable insights.
3.  **Reproducibility concerns regarding corpora**: The paper mentions using comparable corpora for annotation projection and monolingual data for training word embeddings. However, details about the source, domain, and size of these corpora (especially those used for the CoNLL experiments) are sparse. Providing these details is crucial for reproducibility and for understanding the resource requirements of the proposed methods.

**Questions to Authors**
1.  The data selection scheme introduces two hyperparameters, (q, n), which you tuned on development data. In a real-world application where no labeled target-language data is available, how would you recommend setting these parameters? Could you perhaps show a sensitivity analysis on a development set to illustrate how performance varies as q and n change?
2.  You observe that the NN1 architecture performs better on CoNLL data, while the NN2 architecture (with the smoothing layer) is superior for your in-house data. Do you have any hypotheses that might explain this discrepancy? Could it be related to the difference in the size of the tagset (4 vs. 50+)?
3.  For the experiments on the CoNLL datasets, could you please provide more details on the comparable corpora used for the annotation projection step? Knowing the source and size of this data is important for other researchers to be able to fairly compare with your results.

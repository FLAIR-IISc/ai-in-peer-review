**Summary of the paper and main contributions**

This paper addresses the problem of cross-lingual named entity recognition (NER) in a weakly supervised setting, where no annotated data is available for the target language. The authors propose and evaluate two distinct approaches. The first, "annotation projection", uses an English NER system to label comparable corpora and projects these labels to the target language via word alignments. The key contribution here is a novel heuristic data selection scheme that filters the noisy projected data to retain high-quality examples, significantly improving performance. The second approach, "representation projection", learns a linear mapping from a target language's word embedding space to the English embedding space. This allows a single NER model trained on English embeddings to be directly applied to the projected target language words. Finally, the paper introduces two co-decoding schemes to combine the outputs of these complementary systems, demonstrating that the combined model achieves state-of-the-art results on the CoNLL benchmark datasets.

The main contributions of this work, in order of strength, are:
1.  A robust and effective combined system for weakly supervised cross-lingual NER that outperforms previous state-of-the-art methods on standard CoNLL benchmarks.
2.  An intuitive and highly effective data selection heuristic for annotation projection that substantially improves performance, especially when dealing with noisy alignments from comparable (rather than parallel) corpora.
3.  A comprehensive empirical evaluation across seven target languages and two distinct NER task setups (a fine-grained in-house dataset and the standard CoNLL dataset), which thoroughly validates the proposed methods.

**Strengths**

1.  **Strong Empirical Results and State-of-the-Art Performance:** The paper's primary strength lies in its excellent results. The proposed combined system clearly outperforms two strong and recent baselines (Täckström et al., 2012; Tsai et al., 2016) on the CoNLL 2002/2003 test sets for Spanish, Dutch, and German (Table 5). Achieving a new state-of-the-art is a significant contribution and provides a strong argument for acceptance.

2.  **Effective and Practical Data Selection Method:** The heuristic data selection scheme for annotation projection is simple, well-motivated, and demonstrably effective. As shown in Table 2, this method provides massive improvements for languages with low-quality alignments (e.g., +13.9 F1 for Japanese, +20.0 F1 for Korean). This is a valuable contribution for the community, as it provides a practical way to mitigate noise, which is a primary challenge in projection-based approaches.

3.  **Comprehensive and Rigorous Evaluation:** The authors conduct an extensive set of experiments. They evaluate their methods on seven different target languages from multiple language families. Furthermore, they test on two very different types of datasets: an in-house dataset with over 50 fine-grained entity types and the standard 4-type CoNLL dataset. This demonstrates the robustness and general applicability of their approach beyond a single benchmark. The inclusion of supervised baselines also provides important context for the performance of the weakly supervised systems.

4.  **Insightful Combination of Complementary Approaches:** The paper cleverly combines two orthogonal cross-lingual transfer paradigms: data-level transfer (annotation projection) and model-level transfer (representation projection). The analysis of their distinct precision/recall profiles is insightful and provides a strong motivation for the co-decoding schemes. The fact that combining these two methods consistently yields further improvements highlights the value of this multi-view approach.

**Weaknesses**

1.  **Limited Novelty of the Representation Projection Component:** While its application to NER is successful, the representation projection method itself is a direct application of the linear mapping technique proposed by Mikolov et al. (2013b). The paper presents this as a "new approach," which is a slight overstatement. The contribution would be stronger if it were framed more as an effective application and evaluation of this existing technique for NER, perhaps with more discussion of why this simple method works well compared to more complex alternatives for creating cross-lingual representations.

2.  **Lack of Analysis on Hyperparameter Sensitivity:** The data selection scheme's performance depends on the quality threshold `q` and entity number threshold `n`. The paper reports results for the best-performing `(q, n)` pair for each language but provides no analysis of the sensitivity to these parameters. It is also unclear how these would be set for a new language in a truly zero-resource setting where no labeled development set exists. A sensitivity analysis or discussion on heuristics for setting these parameters would significantly strengthen this part of the work.

3.  **Details Regarding CoNLL SOTA Comparison:** The comparison in Table 5 is crucial for the paper's main claim, but it could be more thorough. The paper does not specify which exact numbers were taken from the prior work or discuss the comparability of the underlying resources used (e.g., size and source of monolingual or parallel data for training word clusters or embeddings). While the performance gains are substantial, providing more context on the experimental conditions relative to the baseline papers would make the comparison more rigorous.

**Questions to Authors**

1.  Could you clarify how the data selection hyperparameters `(q, n)` were tuned for the CoNLL experiments? The development set was used to select the best co-decoding scheme. Was it also used to tune `q` and `n` for the annotation projection system, and if so, how did you avoid overfitting to the development set?

2.  The performance of the two neural network architectures for representation projection varies across datasets: NN2 (with smoothing) is superior on the in-house data, while NN1 is better on CoNLL. Do you have any hypotheses for this discrepancy? Could it be related to the domain, the number of entity types (>50 vs. 4), or the size of the training data?

3.  Similarly, the best co-decoding scheme also changes between the two datasets (rank-based for in-house, confidence-based for CoNLL). This suggests the relative precision/recall trade-offs of the base systems are dataset-dependent. Could you elaborate on why this might be, and what guidance you might offer for choosing a co-decoding strategy when approaching a new cross-lingual NER task?

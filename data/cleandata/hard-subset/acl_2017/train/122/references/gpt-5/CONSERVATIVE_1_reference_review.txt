Summary and claimed contributions
This paper proposes Neural Belief Tracker (NBT), a joint SLU/DST model that replaces delexicalisation and hand-crafted semantic dictionaries with reasoning over pre-trained word vectors. Two variants are presented (NBT-DNN and NBT-CNN) that compose utterance representations and interact them with slot-value representations and a simple context gating based on the last system act. A rule-based belief update combines N-best ASR hypotheses. Experiments on DSTC2 and WOZ 2.0 report statistically significant gains over delexicalisation-based baselines, and an ablation shows that semantically specialised embeddings (Paragram-SL999) improve performance.

Contributions as I see them (in order of strength)
1) An empirical demonstration that belief tracking over pre-trained, semantically specialised embeddings can match or outperform delexicalisation-based systems without semantic dictionaries on two standard datasets.
2) A simple, parameter-efficient architecture that couples SLU/DST and incorporates last-act context via gating; plus an analysis of the impact of different embedding spaces.
3) Dataset resources: cleaned DSTC2 transcriptions and an expanded WOZ 2.0 set (if released as claimed).

Strengths
- Removes reliance on semantic dictionaries while maintaining competitive accuracy. The main practical contribution is showing that dictionary-free DST is feasible with pre-trained embeddings, which can materially improve scalability across values and paraphrases. The WOZ gains in joint goals (70.8 → 84.4/84.2) and requests (87.1 → 91.2/91.6) are substantial.
- Sensible use of pre-trained embeddings and ablation. The comparison across random/GloVe/Paragram supports the core hypothesis: semantic specialisation for similarity (not mere relatedness) matters. This is consistent across both datasets and strengthens the methodological case.
- Simple, interpretable context modelling. The gating based on similarity to the last system act is well-motivated for confirmations/requests and avoids heavy sequence modelling, which is attractive from an engineering standpoint.
- Competitive baselines and statistical testing. The delex baselines are standard references in DSTC2/WOZ, and significance testing is reported. The no-dictionary vs with-dictionary comparison isolates the benefit of semantic generalisation.
- Resource contribution. If the cleaned DSTC2 transcriptions and the expanded WOZ are released, this is valuable for the community.

Weaknesses
- Limited scope and scalability claims unsubstantiated. All evaluation is on a single, small domain (Cambridge restaurants) with three informable slots. The model scores each slot-value pair by iterating over the ontology; complexity scales linearly in the number of values. There is no demonstration on large ontologies or multi-domain settings, despite the paper’s strong scalability claims. Runtime/latency and memory footprint are not reported.
- Modest gains in the realistic ASR setting and unclear ASR handling. On DSTC2 ASR, the improvement over a strong delex+dictionary baseline is marginal (72.9 → 73.4 joint goals). The paper itself notes that performance jumps to 0.96 on transcriptions, underscoring that most remaining difficulty is ASR robustness. The proposed N-best fusion is a simple rule; no comparison to confusion-network-based or more principled ASR-aware trackers is provided, and it is unclear whether baselines use the same update rule—raising fairness concerns.
- Context modelling is shallow and its impact is not rigorously validated. Only the last system act is used (Markov assumption), and no ablation quantifies the contribution of context gating vs. the semantic decoder. The approach appears tailored to confirmation/request patterns; it is unclear how it handles more complex context phenomena (e.g., coreference, corrections, cross-turn value changes).
- Methodological clarity and reproducibility gaps. The paper claims parameters can be shared across values and even slots, yet Section 3.1 states “we maintain a separate set of parameters for each slot,” which contradicts the sharing claim. Training/fine-tuning choices (embeddings frozen) are justified qualitatively but not examined empirically (e.g., regularised fine-tuning vs. freezing). OOV handling and embedding coverage under ASR noise are not reported. Details of the statistical test (unit of analysis, multiple comparisons) are absent.
- Novelty relative to prior work is somewhat overstated. Using embeddings for SLU/DST is not new; several SLU models and end-to-end trackers already leverage distributed representations. The added value here is in avoiding delexicalisation and dictionaries via pre-trained similarity-specialised vectors, but the claim of being “first to successfully use pre-trained word vector spaces to improve belief tracking” is arguable without situating more carefully against contemporary RNN/CNN SLU or end-to-end DST.

Questions to authors
- Fairness of comparison: Do all systems (your NBT variants and both baselines) use the same belief state update mechanism and the same N-best handling? If not, please report results under a common update rule, or justify the differences.
- Scalability: What is the runtime (per turn) and memory cost as a function of number of slot values? Please report wall-clock times for DSTC2 and estimated costs for larger ontologies (e.g., 10× values).
- Ablations: What is the performance drop when removing the context gating (dr, dc), or when replacing element-wise multiplication with bilinear/MLP interactions? Please quantify the contribution of each component.
- Embeddings: What is the OOV rate on DSTC2 ASR and WOZ? Did you try fine-tuning embeddings with regularisation (e.g., small LR, L2 to pre-trained vectors)? If so, how did that affect generalisation to unseen paraphrases?
- Statistical testing: What is the unit of the paired t-test (dialog turn vs dialog)? How did you account for multiple comparisons across models/metrics?

Overall, this is a solid empirical paper with practical impact, but its claims about scalability and real-world robustness would be more convincing with broader domains, stronger ASR baselines, and deeper ablations.

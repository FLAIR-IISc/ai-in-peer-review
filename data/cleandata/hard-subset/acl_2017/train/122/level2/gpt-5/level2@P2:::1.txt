Summary
This paper proposes the Neural Belief Tracker (NBT), a family of models for dialogue state tracking that replaces delexicalisation and hand-crafted semantic dictionaries with learned representations built on pre-trained word vectors. Two variants (NBT-DNN and NBT-CNN) compose utterance representations from word embeddings, model local context via gating with last system acts, and perform per-slot/value binary decisions. The approach is evaluated on DSTC2 (ASR) and WOZ 2.0 (typed) with state updates via a simple rule-based temporal smoother. The authors show consistent gains over strong delexicalised baselines, especially without lexicons, and demonstrate that semantically specialised embeddings (Paragram-SL999) yield further improvements over random or GloVe embeddings.

Main contributions (as I see them)
- A simple, effective belief tracking architecture that couples SLU and DST via interaction between utterance and slot–value representations, avoiding hand-crafted semantic dictionaries.
- Empirical evidence that semantically specialised word vectors materially improve downstream DST, including ablations contrasting random/GloVe/Paragram embeddings.
- A context-gated decision mechanism that incorporates last system acts (request/confirm) to disambiguate short user replies.

Strengths
- Clear problem motivation and practical significance: removing reliance on semantic dictionaries addresses a real scalability bottleneck in task-oriented dialogue systems.
- Solid empirical results: consistent and statistically significant gains on both DSTC2 and WOZ 2.0 for joint goal accuracy, and strong request detection, with and without competing systems’ lexicons. The advantage is larger on WOZ, aligning with the claim that the model handles lexical variation well.
- Thoughtful embedding analysis: the ablation comparing Paragram vs GloVe vs random is useful, supports the claim about semantic specialisation, and ties to qualitative intuitions (e.g., antonymy issues in distributional spaces).
- Architectural simplicity and efficiency: per-slot/value binary decisions with shared compositional parameters plus element-wise interactions are easy to implement and deploy, yet competitive with more complex pipelines.
- Context gating is well-motivated and effective for confirm/request exchanges, a common pain point for trackers that only rely on lexical overlap.

Weaknesses
- Limited domain and ontology size: claims about scalability to “larger, more complex dialogue domains” are not convincingly substantiated by experiments restricted to a single domain with three informable slots. There is no evaluation on larger ontologies or multi-domain settings to validate parameter sharing and generalisation.
- Restricted context modelling: only the immediately previous system act is used. This Markov assumption can miss cross-turn dependencies (e.g., corrections or multi-turn confirmations). The paper lacks ablations demonstrating the marginal contribution of the context module and whether longer context would help.
- Training–test mismatch on DSTC2: models are trained on transcriptions and evaluated on ASR hypotheses, yet the method’s robustness to ASR noise is central to the setting. While the belief update aggregates N-best lists, the model itself does not exploit ASR confidence or word lattices, and it is unclear how it compares to baselines trained with ASR features.
- Fairness and scope of baselines: delexicalised baselines cannot leverage pre-trained embeddings; however, a stronger baseline could augment lexicons automatically (e.g., from PPDB) or incorporate distributional similarity to expand value mentions, narrowing the gap and isolating the benefit of the NBT architecture vs external knowledge.
- Incomplete analysis and missing ablations: no error analysis is provided (e.g., antonyms, rare values, compositional paraphrases). It is unclear how much each component contributes (CNN vs DNN, context gating vs semantic decoding, fixed vs fine-tuned embeddings, per-slot vs shared parameters). The choice to keep embeddings fixed is justified, but a partial fine-tuning or subword/morphological features could be informative, especially for OOVs.

Questions to Authors
- How are OOV tokens handled at test time? If mapped to UNK, how does this affect the claimed ability to recognise unseen paraphrases? Do you use any subword or character-level representations?
- You mention parameters are “kept fixed” for embeddings and that parameters “can be shared across slots,” yet equations carry slot-specific superscripts. Did you use per-slot parameter sets in all experiments? If so, does sharing across slots degrade or improve performance?
- What is the marginal contribution of the context module? Can you report results with/without the gating terms (dr, dc), and with a longer context window (e.g., RNN over system acts)?
- How sensitive are results to λ in the belief update rule? Did you try learning λ or replacing the rule with a trainable recurrent belief updater?
- For DSTC2, did you try training on ASR N-best (e.g., sampling from hypotheses) or using ASR confidence features to improve robustness? How does that change results?
- Can you report per-slot performance and an error breakdown (synonym coverage, antonyms, negation, compositional multiword values)?

Additional comments and suggestions
- Consider a stronger automated-lexicon baseline (e.g., seed dictionary expanded via PPDB or embedding similarity), to decouple architecture gains from external semantic resources.
- Explore partial embedding fine-tuning with a small learning rate, retrofitting during training, or using counter-fitted vectors to better separate antonyms; include subword embeddings for robustness to morphology and OOVs.
- Evaluate on a larger ontology or multi-domain dataset (even a simulated extension) to substantiate scalability claims; measure generalisation to unseen slot values (zero/few-shot).
- Provide more training details (regularisation, early stopping criteria, number of parameters) and release code to improve reproducibility.
- Minor: references and figure captions have several formatting inconsistencies; please correct typos and unify citation styles.

### Summary of the Paper and Main Contributions

This paper introduces the Neural Belief Tracker (NBT), a novel framework for dialogue state tracking (DST) that aims to improve scalability by removing the dependence on hand-crafted semantic lexicons. The core idea is to leverage pre-trained word embeddings to handle linguistic variation in a data-driven manner. The NBT model operates as a binary classifier, iterating through each ontology-defined slot-value pair to determine if it was expressed in the current user utterance, given the preceding system turn as context. The architecture learns vector representations for the utterance, system acts, and candidate slot-value pairs, and models their interactions to make a decision. The authors propose two variants, one using summed n-gram representations (NBT-DNN) and another using a convolutional neural network (NBT-CNN). The models are evaluated on the DSTC2 and a new, more linguistically diverse WOZ 2.0 dataset, demonstrating performance that matches state-of-the-art systems equipped with hand-crafted lexicons and significantly surpasses them when such resources are not used.

As I see them, the main contributions of this work are:
1.  A novel and well-motivated neural architecture for dialogue state tracking that operates directly on pre-trained word embeddings. This successfully replaces the traditional, brittle delexicalisation pipeline with a learned, end-to-end component, representing a significant step towards building more scalable and robust dialogue systems.
2.  Strong empirical evidence, across two distinct datasets, that a purely data-driven approach can achieve performance on par with systems that rely on extensive, manually curated domain-specific resources. This is a very encouraging result for the field.
3.  A clear and insightful analysis of the impact of word embedding quality on a downstream dialogue task. The paper effectively demonstrates that semantically specialized word vectors lead to statistically significant performance gains, providing valuable validation for work on representation learning.

### Strengths

1.  **Excellent Motivation and High Impact Potential:** The paper addresses a well-known and critical bottleneck in developing task-oriented dialogue systems: the reliance on hand-crafted resources for semantic understanding. The proposed solution of using pre-trained word embeddings is elegant and directly tackles this problem. If successful and generalizable, this approach could significantly reduce the cost and effort required to build dialogue systems for new and more complex domains.

2.  **Thorough and Convincing Evaluation:** The experimental design is strong. By evaluating on two datasets with different characteristics (DSTC2 with ASR noise vs. WOZ 2.0 with richer, cleaner language), the authors provide a comprehensive picture of the model's capabilities. The comparisons to strong, delexicalisation-based baselines are fair and well-chosen. The key result—that NBT matches the performance of lexicon-augmented baselines and surpasses non-augmented ones—is powerful and convincingly supports the paper's main claims.

3.  **Insightful Analysis on Representation:** The analysis in Section 6.2, which compares randomly initialized, distributional (GloVe), and semantically specialized (Paragram-SL999) word vectors, is a standout feature of this paper. It goes beyond simply reporting a final number and provides valuable insights into *why* the model works. Showing a direct, significant downstream benefit from using semantically specialized embeddings is an important contribution in its own right and strengthens the overall message of the paper.

4.  **Clarity and Presentation:** The paper is exceptionally well-written, clear, and easy to follow. The model architecture is explained logically and accompanied by helpful diagrams. The motivation is clearly laid out in the introduction, and the results are presented and discussed effectively. The authors' commitment to releasing the cleaned DSTC2 data and the new WOZ 2.0 dataset is commendable and will be a valuable resource for the community.

### Weaknesses

1.  **Scalability to Large Output Spaces:** The model's inference process involves iterating through every slot-value pair in the ontology and performing a separate binary classification. While this is feasible for the domains presented (which have a modest number of slots and values), it is not clear how this approach would scale to domains with very large or open-ended value sets (e.g., tracking a specific movie title or restaurant name from a list of thousands). The paper's primary motivation is scalability, but this particular aspect of scalability (in the output space) is not addressed.

2.  **Simplicity of the Cross-Turn Belief Update:** The core neural model is focused on turn-level semantic decoding. The mechanism for tracking beliefs across turns (Section 4) is a simple, non-learned linear interpolation between the previous state and the current turn's prediction. While effective, this is less sophisticated than other contemporary approaches that use recurrent networks to model the state evolution. It would be interesting to see if integrating the NBT's turn-level representation learning with a more advanced, learnable state update mechanism could yield further improvements.

3.  **Limited Dialogue Context Modelling:** The context modelling in Section 3.3 is limited to the immediately preceding system act. This is a reasonable simplification that captures important phenomena like answering direct questions or confirmations. However, dialogue context can span multiple turns. The current model would be unable to resolve an utterance like "the first one you mentioned" that refers to an entity from several turns prior. This limits the model's ability to handle more complex conversational phenomena.

### Questions to Authors

1.  Regarding the concern about scalability to large ontologies: Have you considered alternative formulations that would avoid iterating through all possible values? For example, could the model be trained to predict a value embedding for each slot, which could then be used to find the nearest-neighbor value in the ontology? Or could it be framed as a slot-tagging problem over the user utterance?

2.  The context modelling in Section 3.3 uses dot products as multiplicative gates on the utterance representation `r`. This is an interesting design choice. Could you provide more intuition on this, especially for the `dc` term which involves a three-way interaction between the candidate pair and the system-confirmed pair? Have you experimented with alternative ways of incorporating context, such as concatenating the system act representation with the utterance representation?

3.  The performance of NBT-DNN and NBT-CNN is remarkably similar in Table 1. This is slightly surprising, as CNNs are often considered more powerful for capturing local compositional features. Did you observe any qualitative differences between the models, for instance, on longer or more syntactically complex utterances where the CNN might be expected to have an advantage?

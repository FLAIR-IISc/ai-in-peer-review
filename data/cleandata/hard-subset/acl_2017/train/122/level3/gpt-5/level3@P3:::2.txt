Summary
This paper introduces the Neural Belief Tracker (NBT) framework for dialogue state tracking (DST) that dispenses with delexicalized features and handcrafted semantic dictionaries by operating directly on pre-trained word vectors. The model learns representations for three inputs per turn: the user utterance, the preceding system acts, and a candidate ontology slot–value pair. Two utterance encoders are compared (NBT-DNN using summed n-gram projections; NBT-CNN using convolution + max-pooling). A context module gates the utterance representation conditioned on last-turn system Request/Confirm acts; a semantic decoder combines utterance and slot–value representations via element-wise product; and a simple rule-based belief update aggregates ASR N-best hypotheses over turns. On DSTC2 and WOZ 2.0, NBT significantly outperforms a strong delexicalized baseline, matches or surpasses versions aided by semantic dictionaries, and demonstrates the value of semantically specialized word vectors (Paragram-SL999).

Contributions (as I see them)
1) A practical, lexicon-free DST framework that jointly models SLU/DST over pre-trained embeddings, delivering significant gains over delexicalized baselines and obviating handcrafted semantic dictionaries.
2) A simple, modular architecture that learns and composes representations for utterances, system acts, and ontology slot–value entries via element-wise interactions and lightweight context gating.
3) Empirical evidence that semantically specialized embeddings materially improve DST, with clear comparisons across random, GloVe, and Paragram vectors.

Strengths
- Removes manual semantic dictionaries without sacrificing accuracy: NBT matches or exceeds delexicalized systems with lexicons and clearly outperforms them without lexicons. On DSTC2 goals, baseline 69.1 vs NBT-CNN 73.4; on WOZ goals, baseline 70.8 vs NBT-DNN 84.4/NBT-CNN 84.2; request accuracies also improve.
- Demonstrates the importance of high-quality semantic embeddings: Paragram-SL999 yields consistent gains over GloVe and random initializations, especially on noisy DSTC2. This is a compelling and well-isolated ablation.
- Parameter sharing and scalability: Keeping embeddings fixed and sharing parameters across values (and potentially slots) makes the approach data-efficient and better suited to larger ontologies than per-value classifiers or brittle delexicalization pipelines.
- Thoughtful context modeling: The gating mechanisms for last-turn Request/Confirm acts are effective and intuitive, enabling correct handling of implicit answers (yes/no) without explicit slot/value mentions.
- Strong empirical setup across two datasets with different characteristics (ASR noise vs typed text), showing robustness and generality.

Weaknesses
- Presentation and organization: Key concepts, notations (e.g., tq, ts, tv), and the overall dialogue system placement of NBT appear before being defined. The paper would benefit from an early, formal definition of DST, a brief architecture overview of a task-oriented dialogue system, and a consolidated notation table. As is, readability suffers, especially for non-experts.
- Ambiguity around “joint” SLU/DST: The paper uses “joint” but does not clarify whether the model can output SLU labels or act as a standalone SLU module. If it cannot, the term may be misleading; if it can, the interface and outputs need to be specified.
- Limited sequential modeling within utterances: Both NBT-DNN and NBT-CNN process n-grams but do not model token order beyond short windows. Given the success of RNNs/LSTMs/biLSTMs in SLU, an utterance-level sequence model could further improve performance and handle longer contexts.
- Dataset corrections and comparability: DSTC2 transcripts were manually corrected, but the paper lacks statistics on the magnitude and nature of corrections and how they affect comparability to prior DSTC2 results. Without this, it is hard to contextualize gains.
- Baseline details and manual cost: The “semantic dictionary” used by baselines is pivotal to claims about manual effort, but its size, coverage, and construction cost are not quantified. This weakens arguments about scalability and the extent of human effort saved.

Questions to Authors
- Please provide a formal definition of the DST task early (state representation, update objective, evaluation metrics) and a diagram situating NBT within a standard dialogue system pipeline.
- Can NBT be used as a standalone SLU component (i.e., outputting detected slot–value pairs per utterance) or only as a DST module? If “joint,” what are the explicit outputs and interfaces?
- Notation clarity: Define tq, ts, tv, cs, cv before first use; consider a notation table. Also clarify whether multiword slots/values are summed or averaged and whether this affects scaling to longer values.
- DSTC2 corrections: How many tokens/turns/dialogues were corrected? What categories of errors (spelling, normalization) were addressed? Are test-set ASR/lattices unchanged? If corrections are small, how do NBT results compare to top DSTC2 challenge entries under the same conditions?
- Semantic dictionary baseline: Report dictionary size per slot, number of paraphrases per value, creation time/effort, and example entries to quantify manual cost.
- Ablations: Results for (i) using concatenation instead of element-wise product in the semantic decoder, (ii) removing context gating, (iii) varying CNN filter widths and counts, and (iv) fine-tuning vs freezing embeddings.
- Sequential encoders: Have you tried biLSTM/GRU encoders for utterances or for context beyond the last system act? If not, why? Any expected trade-offs?
- Scalability: How does runtime/memory scale with ontology size and average utterance length? Can NBT generalize to unseen values with only their names (zero-shot)?
- Qualitative analysis: Please provide example turns where NBT fixes baseline errors (e.g., resolving paraphrases like “inexpensive”→cheap or implicit confirmations) and failure cases.

Additional Comments
- Reorder the exposition: start with a concise DST formalization and a pipeline figure; then introduce inputs/outputs and notation; then detail representation learning, semantic decoding, and context gating.
- Clarify the belief update (λ selection, sensitivity) and whether end-to-end learning of λ or learned turn-level aggregation could help.
- Provide more context on ontology size and distribution across slots. Include statistics for WOZ 2.0 expansion and release the corrected DSTC2 and WOZ 2.0 datasets and code for reproducibility.
- Consider adding RNN/LSTM utterance encoders or hybrid CNN-RNN models, given their track record in SLU, and report comparisons.

- Proposes a neural DST framework that learns unified representations for user utterances, system outputs, and ontology entries using pre-trained word vectors.  
- Core innovation: compares two utterance representation architectures (NBT-DNN vs NBT-CNN) and combines them in a downstream network for slot-value decisions.  
- Empirical results show significant gains over a delexicalized baseline and outperform prior work, supporting the method’s effectiveness.  
- Paper presents a clear objective, sound methodology, and compelling empirical validation.  
- Organization impedes clarity: DST should be defined early within a high-level dialogue system architecture (ASR, SLU, policy) rather than delayed.  
- Notation issues: key symbols (e.g., t_q, t_s, t_v) are used before definition; an early notation section is needed.  
- Terminology precision: calling the model “joint” is unclear—must state whether it independently handles SLU tasks (intent detection, slot filling) or revise the term.  
- Dataset context: authors used a modified DSTC2—provide statistics on corrections and, if minor, compare against published DSTC2 results on the same data.  
- Model architecture choice: justify using DNN/CNN for utterance representation and discuss RNN/LSTM alternatives given their SLU success.  
- Baseline resource cost: quantify manual effort to build the semantic dictionary (size, annotation complexity) to assess practicality.  
- Error analysis: include concrete dialogue examples where the proposed model succeeds but the baseline fails to illustrate strengths and limitations.  
- Overall: technically meaningful and well-supported, but presentation, terminology, and requested clarifications must be improved to maximize impact.

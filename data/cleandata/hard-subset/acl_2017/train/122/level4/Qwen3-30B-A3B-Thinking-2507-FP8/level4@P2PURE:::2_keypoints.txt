- Proposes a neural DST framework that learns representations for user utterances, system outputs, and ontology entries via pre-trained word vectors, evaluating NBT-DNN and NBT-CNN and using a downstream binary classifier for slot–value pairs with significant gains over a delexicalized baseline.  
- Strength: clear objective, reasonable approach, and improved results relative to prior studies.  
- Paper organization is weak for readers unfamiliar with DST; recommend moving a formal DST definition and a general dialogue system architecture to Section 1.  
- All notations (e.g., t_q, t_s, t_v) are introduced before being defined and should be defined early.  
- Clarify relationships between DST, ASR, SLU, and policy learning to avoid confusion about system integration.  
- The term “joint” may be misleading; state whether SLU can be run independently or not.  
- Provide statistics on corrections made to the original DSTC2 dataset and compare against other DSTC2 entries using the same corrected dataset if corrections are minor.  
- Justify using feedforward/CNN architectures (NBT-DNN/CNN) instead of RNNs/LSTMs for modeling sequential utterance information.  
- Report details of the semantic dictionary used in the baseline to clarify manual resource requirements.  
- Include concrete examples where the proposed model corrects errors made by the baseline.

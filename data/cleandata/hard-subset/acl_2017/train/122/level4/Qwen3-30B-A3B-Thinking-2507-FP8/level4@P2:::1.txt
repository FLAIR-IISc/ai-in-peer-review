

**Summary and Contributions**  
This paper introduces the Neural Belief Tracker (NBT), a novel framework for dialogue state tracking that eliminates reliance on hand-crafted semantic lexicons. The primary contribution is a method that leverages pre-trained word vectors to compose distributed representations of user utterances and dialogue context, enabling effective handling of lexical variation without requiring domain-specific resources. A secondary contribution is demonstrating that NBT models match the performance of delexicalization-based approaches with semantic dictionaries while significantly outperforming them when such dictionaries are unavailable. The third contribution is showing that semantic quality of pre-trained word vectors directly impacts model performance, with Paragram-SL999 vectors yielding the best results.

**Strengths**  
1. The paper addresses a critical scalability limitation in dialogue state tracking: current approaches require either large annotated datasets for SLU or hand-crafted semantic lexicons, both of which hinder deployment in complex domains. NBT provides a principled solution by learning from pre-trained embeddings, making it more adaptable to diverse domains without manual resource creation.  
2. The evaluation is robust and comprehensive, demonstrating consistent improvements over strong baselines on two distinct datasets (DSTC2 and WOZ 2.0). The NBT models achieve statistically significant gains in joint goal accuracy (73.4% vs. 69.1% on DSTC2) and request accuracy (96.5% vs. 95.7%), with the improvements being more pronounced on the WOZ 2.0 dataset where linguistic variation is greater.  
3. The exploration of different word vector representations (XAVIER, GloVe, Paragram-SL999) provides valuable insights into the relationship between semantic quality and downstream performance. The paper convincingly shows that Paragram-SL999 vectors, which incorporate semantic similarity constraints, yield significantly better results than generic embeddings, particularly on noisy data.  
4. The approach is conceptually simple yet effective, using standard neural network components (DNN/CNN) with pre-trained word vectors, making it accessible for implementation and extension in future work.

**Weaknesses**  
1. The evaluation is limited to two restaurant-domain datasets with relatively small ontologies (three informable slots), raising questions about scalability to more complex domains with richer slot-value relationships. The paper does not provide evidence of how the model would perform on larger, more diverse ontologies where slot-value combinations multiply exponentially.  
2. The coreference resolution issue is not addressed. In the example dialogue, the system states "The House serves cheap Thai food" and the user asks "Where is it?" The belief state should include "restaurant_name=The House," yet the example omits this. Without resolving such references, the model's ability to maintain accurate dialogue state across turns is questionable.  
3. The paper does not explore how NBT might interact with a separate slot tagging component, which is common in modern dialogue systems. For instance, the model could first identify potential slot spans in the utterance and then compute similarity only within those spans, potentially improving accuracy without requiring explicit lexical resources.  
4. The lack of error analysis limits understanding of the model's limitations. Without examining specific failure cases (e.g., when the model misinterprets "expensive" as "cheap" despite using semantic vectors), it's unclear whether the approach handles all forms of semantic variation equally well.

**Questions to Authors**  
1. How would the model perform on a multi-domain dataset (e.g., combining restaurant, hotel, and flight booking domains) where the ontology is significantly larger and more complex?  
2. How does the model handle coreference resolution, particularly when the user refers to previously mentioned entities (e.g., "it" referring to "The House" in the example)?  
3. Could the model benefit from integrating with a slot tagging component that identifies candidate spans for slot values before computing semantic similarity?  
4. What is the model's performance when evaluated on a dataset with more complex slot-value relationships (e.g., multi-word values like "highly-rated Thai restaurant")?

This paper introduces a neural network framework for dialogue state tracking. The primary contribution lies in learning representations for user utterances, system responses, and ontology entries, all initialized with pre-trained word vectors. Specifically, the authors compare two neural network architectures for utterance representation: NBT-DNN and NBT-CNN. These learned representations are then combined and used as input to a downstream network for making binary decisions regarding slot-value pairs. Experimental results demonstrate significant performance improvements over a delexicalized baseline.

Overall, this is a solid work with a clear objective, a reasonable approach, and improved performance compared to prior studies. However, the paper’s organization could be improved to more effectively convey the details, particularly for readers unfamiliar with the field. A more formal definition of dialogue state tracking (DST) should be provided at the beginning of the paper. Currently, its presentation alongside semantic language understanding (SLU) can be confusing. I recommend presenting a general dialogue system architecture in Section 1, followed by a precise problem definition of DST and its relationship to other components such as automatic speech recognition (ASR), SLU, and policy learning. Additionally, defining all notations used throughout the paper in an earlier section would enhance readability; several symbols (e.g., tq, ts, t_v) appear before their definitions.

Further comments and questions include:

- Is it possible to perform separate SLU using this model? If not, the term “joint” may be misleading, suggesting the model can handle both tasks independently.
- Could you provide statistics regarding the number of errors corrected in the original DSTC2 dataset? If this number is not substantial, the experiments should include comparisons with other published work, including DSTC2 submissions using the same dataset.
- Have you considered using recurrent neural networks (RNNs) or Long Short-Term Memory (LSTMs) to capture sequential information when learning utterance representations? Given the recent successes of these architectures in SLU, they may also prove effective for DST.
- More details about the semantic dictionary used with the baseline would help illustrate the manual effort required to construct such resources.
- Including example cases that the baseline incorrectly predicted but your proposed models correctly handled would be beneficial.

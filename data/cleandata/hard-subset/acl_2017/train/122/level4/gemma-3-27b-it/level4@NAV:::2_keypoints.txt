1. Introduces a neural network framework for dialogue state tracking (DST).
2. Main contribution: learning representations for user utterances, system responses, and ontology entries using pre-trained word vectors.
3. Evaluates two utterance representation architectures: NBT-DNN and NBT-CNN.
4. Integrates learned representations into a downstream network that classifies slot-value pairs as binary decisions.
5. Experimental results show substantial performance gains over a delexicalized baseline.
6. Overall assessment: solid work with a clear objective, sound approach, and positive results building on prior research.
7. Paper organization needs improvement to better convey technical details to non-expert readers.
8. A formal definition of dialogue state tracking should be presented upfront; current treatment alongside SLU is unclear.
9. Recommend restructuring: start with a general dialogue system architecture, then a dedicated DST problem definition and relationship to ASR, SLU, and policy learning.
10. Notations should be defined earlier; several symbols (e.g., t_q, t_s, t_v) appear before their definitions.
11. Question: Can this model perform separate SLU? If not, calling it 'joint' may be misleading.
12. Request: provide statistics on the number of errors corrected in the original DSTC2 dataset and comparisons with other DSTC2 submissions.
13. Suggestion: consider RNNs or LSTMs to capture sequential information in utterance representations.
14. Ask for more details about the semantic dictionary used in the baseline to illustrate manual effort.
15. Request: include example cases where the baseline failed but the proposed models succeeded.

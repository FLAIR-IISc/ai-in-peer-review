- Introduces a novel evaluation method for GEC that provides fine-grained, per-error-type precision and recall without requiring annotated system outputs.  
- Significant contribution: enables nuanced analysis of GEC system behavior across error types.  
- Demonstrated on a diverse set of GEC systems, showing practical applicability.  
- Key benefits: computes per-error-type precision, works without manual error corpora, and handles multi-token errors.  
- Human evaluation reports ≥95% agreement with automated tags, supporting tag quality.  

- Major weakness: the rule set used to classify error types is not described in the paper.  
- Classifier evaluation is superficial and lacks a comprehensive error analysis or clear paths for improvement.  
- Paper is limited to English and does not discuss feasibility or adaptations for other languages.  
- Origins and theoretical basis of chosen error categories are unspecified.  
- Classification rules likely depend on the aligner, but this dependency is not addressed.  
- No example classification rule is provided, preventing assessment of rule complexity or required adaptation effort.  
- Pre-computed error spans appear restricted to contiguous tokens, risking failure on discontinuous edits (e.g., separable prefixes).  
- Human judge setup may be biased: system-provided tags, noisy boundaries, and a binary rating scale could inflate “Appropriate” counts.  
- Authors should also evaluate classifier outputs on manually created, noise-free boundaries.  
- Missing detailed analysis of what causes “Bad” vs “Appropriate” ratings and which errors suggest classifier fixes.  

- Unclear data and metrics for the significance test; using F0.5 for significance is questionable due to its weak discriminative power.  
- Comparison between gold and automatic references seems to emphasize edit boundaries and neglect classification accuracy.  

- Some systems’ failure on certain error types is expected given system design; results aren’t analyzed in the context of system capabilities.  
- Reported zeros (e.g., CUUI 0.0 on word order) may reflect systems not targeting that error type or classification errors—this is not investigated.  
- Table 6 should include raw TP/FP/TN/FN counts to enable alternative metrics and assess statistical power.  
- Observation that different systems excel in different categories aligns with prior research and should be noted.  
- Multi-token error analysis is promising but needs deeper interpretation and acknowledgment that many systems may be inherently unable or untrained to correct such errors.  

- Paper should acknowledge that some GEC architectures already produce error annotations, while noting the contribution: a unified framework for comparing diverse systems.  
- Minor issues: reference capitalization/formatting errors, missing page numbers, and URL/formatting problems in citations.

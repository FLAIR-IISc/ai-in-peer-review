This paper introduces a novel method for evaluating Grammatical Error Correction (GEC) systems. The proposed approach allows for performance assessment by error type, calculating not only recall but also precision. This is a significant advance, as previous methods could not generally compute precision by error category because system outputs are not typically annotated with such information.

### Strengths
- The proposed evaluation framework is a valuable contribution that enables a more fine-grained analysis of GEC system behavior.
- The authors demonstrate the method's utility by applying it to a diverse set of GEC systems from a shared task.
- The approach offers several advantages over previous work:
    - It computes precision by error type, offering a more complete picture than recall alone.
    - It is independent of manual error annotation in the system output.
    - It can assess system performance on multi-token errors.
- Human evaluation shows a high rate of approval for the automatically assigned error tags, suggesting the classifier's effectiveness.

### Weaknesses
- A critical component of the methodology—the rules used to classify errors—is not described, hindering reproducibility and extensibility.
- The evaluation of the error type classifier lacks a detailed error analysis, and consequently, the paper offers limited direction for future improvements.
- The approach is only evaluated on English, and the paper does not discuss the effort required to adapt it to other languages.

### Classifier and Classifier Evaluation
The paper does not specify the basis for the chosen error categories, nor does it clarify if they are based on established taxonomies from prior work.

A major concern is the complete omission of the rules used to assign error types. While the authors claim the overall approach is independent of the alignment algorithm, the classification rules are likely dependent on the aligner's output. To understand the method and assess its reproducibility, it is essential that the paper provides a description of this rule set, including examples of how an error type is assigned.

The lack of discussion on adapting the method to other languages is a significant omission. It is reasonable to assume that both the edit extraction and error classification rules have language-specific dependencies (e.g., on POS tagsets) and would require adaptation. The error categories themselves might also need to be modified. Without any details on the number or complexity of the rules, it is impossible for a reader to estimate the effort required to apply this method to another language.

The reliance on continuous error spans, similar to the M² scorer, is a potential limitation. It is unclear how the system would handle discontinuous edits. For example, in German main clauses, verbs with separable prefixes are split (e.g., *[subject] [verb] ... [prefix]*). Correcting an error in such a construction would require a discontinuous edit span, and it is not clear if the proposed classifier could tag such edits accurately.

Regarding the human evaluation of the classifier, the claim that 95% of tags were rated as "appropriate" should be interpreted with caution. Raters might be more inclined to accept a system-proposed tag when presented with noisy, automatically generated edit boundaries. Furthermore, the binary choice between "Appropriate" and "Bad" may push raters towards "Appropriate" in ambiguous cases. A more rigorous evaluation would involve asking judges to rate classifier output on manually corrected boundaries to isolate classifier performance from alignment noise.

Finally, the paper would be strengthened by a more thorough error analysis of the classifier. It is mentioned that "Bad" ratings often stem from incorrect POS tags, but a deeper analysis is needed. For instance, what types of errors lead to a "Bad" rating? Does the distribution of ratings suggest specific avenues for improving the classifier?

### Gold Reference vs. Auto Reference
The paper should clarify the exact data and metric used for the significance tests in Section 4.1. If the tests were performed on F₀.₅ scores, this is problematic as F₀.₅ is a derived measure that can obscure underlying differences in precision and recall between systems that achieve similar scores. Additionally, the comparison between gold and automatic references appears to focus only on edit boundaries and not on the classification labels themselves, which is a missed opportunity for a more complete analysis.

### Error Type Evaluation
The paper expresses surprise that several systems failed to correct any unnecessary token errors (Table 5). However, for some systems, this is expected behavior based on their design. For example, the UFC system's rule-based approach operates on POS tags, which are insufficient for identifying superfluous words. Similarly, Rozovskaya & Roth (2016) explain why the AMU system performs poorly on this error type.

The analysis of system performance by error type (Table 6) would be more insightful if it considered the systems' intended capabilities. For instance, the CUUI system is reported with zero precision and recall on word order errors, which it was not designed to correct. This result may point to a classification error rather than a system failure, a possibility that is not discussed.

For greater transparency and to facilitate comparison using other metrics, I recommend reporting the raw counts (TP, FP, FN) for Table 6 in an appendix. This would also clarify the number of instances underlying the scores for each category, which may be small in some cases.

When noting that different systems excel at different error types, it would be beneficial to connect this finding to previous research that has shown similar complementarity among GEC approaches.

The multi-token error analysis is a useful contribution. However, the interpretation of these results should consider that none of the evaluated systems were specifically trained on parallel corpora designed to handle such long-range corrections, which likely impacted their performance.

### Other Comments
- The paper's claim that evaluating precision by error type was previously impossible should be qualified. For modular, rule-based systems, it is often possible to link corrections to the specific submodule that generated them. The primary advantage of the proposed method is that it provides a *unified framework* for comparing all types of GEC systems, including those where such annotation is not straightforward.
- **References**: Some titles are improperly capitalized. The URL for Sakaguchi et al. (2016) needs to be wrapped correctly. Page information is missing for Efron and Tibshirani (1993).

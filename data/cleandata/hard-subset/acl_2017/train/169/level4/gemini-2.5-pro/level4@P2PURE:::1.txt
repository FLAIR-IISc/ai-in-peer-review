- **Strengths**: The paper addresses a useful application for teachers and learners and presents a method that supports the fine-grained comparison of GEC systems.

- **Weaknesses**: The description of the system is highly superficial, and the evaluation is not satisfying.

- **General Discussion**:
This paper presents an approach for automatically enriching the output of Grammatical Error Correction (GEC) systems with error type annotations. This is a valuable contribution, as such information can benefit both language teachers and learners (many GEC systems only provide a corrected version of the text without making the error type explicit). Furthermore, it enables a more fine-grained comparison of GEC systems by facilitating the calculation of precision and recall for specific error types.

Unfortunately, the description of the proposed system is highly superficial. The core of the system is a set of rules, which are presumably manually created, but the paper provides no details about them. For instance, the authors should provide examples of these rules, specify their total number, describe their complexity, and explain their application order (e.g., whether rules are ordered and if earlier rules can preclude the application of later ones). Instead of providing these crucial system details, the paper devotes significant space to an evaluation of the CoNLL-2014 shared task systems. This includes Table 6, which occupies a full page, and accompanying text that largely reiterates figures already present in the table.

The evaluation of the proposed system is unsatisfying in several respects.
First, rather than simply rating the system's output, the annotators should have independently annotated the 200 test sentences to create a gold standard. Creating a gold standard for this relatively small test set should be feasible with a fixed tagset. The chosen methodology likely yields inflated agreement scores compared to an evaluation against a true gold standard, a known issue when annotators review pre-annotated data instead of annotating from scratch (cf. Marcus et al., 1993).

Second, the reporting of inter-annotator agreement is unconventional. The paper states that "all 5 raters individually considered at least 95% of our rule-based error types to be either 'Good' or 'Acceptable'". Averaging individual raters' scores in this manner is not standard practice and can be misleading. For instance, if the "bad" ratings from each annotator correspond to different edits—a distribution the paper does not clarify—then up to 18.5% of edits could have been deemed "bad" by at least one annotator. This paints a much less favorable picture than the reported average of 3.7%.

Third, the paper provides no information about the test data, such as the distribution of error categories it contains or which categories were successfully identified by the system (i.e., those rated "Good" by the annotators).

Fourth, the paper mentions that "edit boundaries might be unusual" without sufficient explanation. A more precise description with examples is needed to understand this issue and its potential impact on the system's practical application.

The authors claim their rule-based system is less domain-dependent than systems requiring training data, but this claim is questionable. For example, the Hunspell dictionary is unlikely to have uniform coverage across all domains, and manually-crafted rules can also introduce domain biases. Furthermore, such rules are inherently language-dependent, a distinct disadvantage compared to many machine learning approaches. This claim is further weakened by the fact that the evaluation data (FCE-test, CoNLL-2014) is drawn from a single domain: student essays.

The motivation for designing a new set of error categories is also unclear. The paper justifies the new tagset by its ability to support searches for general categories (e.g., "NOUN"), but existing tagsets, such as the one in Nicholls (2003), already offer this functionality. The authors do not explain why they did not use the CoNLL-2014 tagset, which would have enabled a direct evaluation against the CoNLL gold standard.

In summary, the paper's primary contribution is ambiguous. If the contribution is a new system, then crucial implementation details are missing. If it is a new set of error categories, the motivation and discussion are insufficient. If it is an evaluation of the CoNLL-2014 systems, the analysis is superficial.

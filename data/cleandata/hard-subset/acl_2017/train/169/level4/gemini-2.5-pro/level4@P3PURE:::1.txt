**Strengths:**
*   Presents a potentially useful application for language teachers and learners.
*   The proposed method could support a more fine-grained comparison of Grammatical Error Correction (GEC) systems.

**Weaknesses:**
*   The description of the proposed system is superficial, lacking crucial technical details.
*   The evaluation methodology has several significant shortcomings.

**General Discussion:**

This paper presents an approach to automatically enrich the output of GEC systems with explicit error type annotations. This is a laudable goal, as this information would be valuable for both teachers and language learners, given that many GEC systems only provide a corrected sentence. Such annotations would also enable a finer-grained comparison of GEC systems, moving beyond general precision to include error-type-specific recall and precision metrics.

Unfortunately, the description of the proposed system is highly superficial. The core of the system is a set of rules, which appear to be manually created, yet the paper provides almost no details about them. To properly assess the work, the authors should have included examples of these rules, specified the total number of rules, and discussed their complexity and ordering (e.g., could an early rule block the application of a later, more appropriate one?). Instead of providing these crucial details, the authors dedicate a significant portion of the paper to an evaluation of the systems that participated in the CoNLL-2014 shared task. Table 6, which occupies an entire page, lists results for all systems, and the accompanying text largely repeats facts and figures that are already evident from the table.

The evaluation of the system is unsatisfactory for several reasons. First, the evaluation relies on annotators rating the system's output rather than establishing an independent gold standard. A more rigorous approach would have involved multiple annotators independently labeling a gold standard for the 200 test sentences. The current method of reviewing pre-annotated data is likely to yield considerably higher agreement scores than would be achieved by annotating from scratch (cf. Marcus et al., 1993).

Furthermore, the method of reporting rater agreement is questionable. The authors state that "all 5 raters individually considered at least 95% of our rule-based error types to be either 'Good' or 'Acceptable'." This way of averaging individual scores can be misleading. For example, if each "bad" score were assigned to a different edit (the paper does not report the distribution), then up to 18.5% of edits were considered "bad" by at least one annotator—a much less favorable outcome than the reported average of 3.7%.

The evaluation is further weakened by a lack of information about the test data itself, such as the number and types of error categories represented. Similarly, the claim that "edit boundaries might be unusual" is too vague. The authors should provide a more precise description with examples and discuss whether this phenomenon poses a problem for the system's application.

The authors' claim that their system is less domain-dependent than machine learning approaches is not well-substantiated. Hunspell's vocabulary is not equally comprehensive across all domains, and manually-created rules can also be domain-specific. Moreover, the system is clearly language-dependent, which is a significant drawback compared to many machine learning approaches. The claim of domain independence is not supported by the evaluation, which uses test data (FCE-test, CoNLL-2014) from a single domain: student essays.

The motivation for designing a new set of error categories is also unclear. The authors state that one reason is to facilitate searches for underspecified categories (e.g., "NOUN"). However, it appears that existing tagsets, such as the one presented in Nicholls (2003), already support such queries. It is not clear why the CoNLL-2014 tagset was not used, as this would have enabled a direct evaluation against the CoNLL gold standard.

In summary, the central contribution of this paper is unclear. If the contribution is a new system, then the paper fails by omitting the most important implementation details. If it is a new set of error categories, the motivation and discussion are insufficient. If the goal is to present a new evaluation of the CoNLL-2014 systems, the analysis is superficial.

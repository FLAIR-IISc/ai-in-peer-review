- Strengths: The application is useful for teachers and learners; supports fine comparison of GEC systems.
- Weaknesses: The system's over-simplistic description; the evaluation is not satisfactory.
- General Discussion:
This paper presents an approach to automatically enriching the output of GEC systems with error types. This is a very useful application because both teachers and learners can benefit from this information (and many GEC systems only output a modified version, without clarifying the type of error). It allows for fine comparison of GEC systems in terms of error type-specific statistics for accuracy and recall and precision in general. Unfortunately, the system's description remains overly simplistic. The system has a basic set of rules (manually?) created, but the paper does not provide any details about these rules. The authors should, for example, show some examples of such rules, specify the number of rules, tell us how complex they are, how they are sorted (can a later rule block the application of an earlier rule?), etc. Instead of presenting a relevant description of the system, many pages of the paper are dedicated to evaluating the systems participating in CoNLL-2014. Table 6 (which takes up a whole page) lists the results of all systems, and the text repeats many facts and figures that can be read from the table. The evaluation of the proposed system is not satisfactory in many aspects. First, instead of simply rating the system's output, annotators should independently annotate a gold standard for 200 test sentences. Given a fixed set of tags, it should be possible to produce a gold standard for a relatively small set of test sentences. It is highly likely that the approach adopted in the paper gives a significantly better rating for annotation than comparing it with the actual gold standard (see Marcus et al. (1993) for a comparison of compromises when reviewing pre-annotated data versus annotating from scratch). Second, it is stated that "all 5 raters individually considered at least 95% of our rule-based error types to be either 'good' or 'acceptable'." Many rates should not be considered individually and their averages should not be taken, in such a way, this is not common practice. If each of the "bad" scores was assigned to a different edit (we do not learn about their distribution from the paper), some annotators considered 18.5% of the edits to be "bad" - this seems much worse than the average of 3.7% calculated in the paper. Third, no information is given about the test data, for example, how many error categories they have, or which error categories are included (according to the categories rated as "good" by the annotators). Fourth, what does it mean that "edit limits may be unusual"? Here, more precise details and examples are needed. Could this be problematic for the system's application? The authors say that their system is less domain-dependent than systems that require training data. I am not sure this is true. For example, I believe that Haspel's vocabulary probably does not cover all domains to the same extent, and manually created rules may also be domain-dependent - and completely language-dependent, which is a clear shortcoming compared to machine learning approaches. Furthermore, the test data used here (FCE-test, CoNLL-2014) are only from one domain: student essays. It is still unclear why a new set of error categories was designed. A reason for the tags is given: being able to easily find them for insufficient categories (such as "NOUN" in general). I think the tagset presented in Nichols (2003) also supports such conclusions. Or why shouldn't the CoNLL-2014 tagset be used? Then the CoNLL gold standard can be used for evaluation. In short, the main motivation of the paper remains somewhat unclear. Is it about a new system? But its most important details have been left out. Is it about a new set of error categories? But hardly any motivation or discussion is provided. Is it about evaluating the CoNLL-2014 system?

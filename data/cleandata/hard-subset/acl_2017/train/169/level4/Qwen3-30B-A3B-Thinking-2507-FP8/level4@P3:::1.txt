

**Summary and Contributions**  
The paper introduces a method for automatically annotating grammatical error correction (GEC) system outputs with error type information, overcoming the limitation that prior GEC evaluations could only measure recall due to unannotated system outputs. The authors propose two key components: (1) a linguistically-enhanced alignment algorithm to extract edits between original and corrected sentences, and (2) a rule-based classifier to assign error types without requiring labeled training data. The main contributions, ordered by strength, are:  
1. A dataset-independent error typing framework that enables detailed error type analysis of GEC systems.  
2. A validated method for automatically annotating system outputs with error types, demonstrated on CoNLL-2014 shared task data.  
3. The first detailed error type analysis of GEC systems, revealing performance differences across error categories.  

**Strengths**  
1. **Novel evaluation capability**: The paper successfully addresses a critical gap in GEC evaluation by enabling error type-specific analysis. Prior work could only report overall recall, obscuring system strengths/weaknesses across error types. The authors' method allows for systematic comparison of GEC systems across 25 error categories, as demonstrated in their CoNLL-2014 analysis.  
2. **Dataset independence**: The rule-based classifier's reliance on automatically extracted features (POS tags, lemmas) rather than labeled training data is a significant advantage. This avoids the dataset-specific biases of prior ML-based approaches, making the framework applicable across different GEC corpora without retraining.  
3. **Strong validation**: The human evaluation (5 experts rating 200 edits) shows 95%+ agreement on error type appropriateness ("Good" or "Acceptable"), with inter-annotator agreement at κ = 0.724. This provides credible evidence that the automatic annotation matches human judgment despite imperfect edit boundaries.  

**Weaknesses**  
1. **Superficial system description**: The core innovation— the rule-based classifier— is described in extreme detail. The paper fails to provide concrete examples of rules, their number, complexity, or ordering logic. Without this, it's impossible to assess the approach's novelty or implementation. For instance, how does the system handle ambiguous cases where multiple rules could apply?  
2. **Flawed evaluation methodology**: The human evaluation is problematic. Instead of creating a gold standard for 200 edits, raters only assessed the *predicted* error types. This inflates agreement scores (e.g., 3.7% "Bad" ratings on average), but if raters had independently annotated the same edits, agreement might be substantially lower. The paper also fails to report the distribution of "Bad" ratings across error types or annotators.  
3. **Unclear motivation for new error framework**: The authors justify their 25-category framework by claiming it enables "flexible error type analysis," but they don't explain why existing frameworks (e.g., CoNLL-2014's) were insufficient. For example, Nicholls' tagset (2003) already supports hierarchical querying (e.g., "NOUN" vs. "NOUN:NUM"). The paper should clarify how their framework improves upon established approaches.  
4. **Limited evaluation scope**: The method is validated only on CoNLL-2014 and FCE-test data, both student essay corpora. The claim of "dataset independence" is undermined by reliance on Hunspell (which may not cover all domains) and POS taggers trained on standard corpora. The authors should address how the system would perform on non-essay domains (e.g., professional writing).  

**Questions to Authors**  
1. Please provide specific examples of the rules used in your classifier (e.g., "If [X] is detected as a missing noun, then classify as M:NOUN"). How many rules exist, and how is their order determined to avoid conflicts?  
2. Why was a new error type framework created instead of adapting existing ones (e.g., CoNLL-2014's tagset)? How does your framework improve upon prior work in terms of granularity or utility?  
3. The paper claims your method is "dataset-independent," but Hunspell's dictionary and POS taggers are domain-specific. How would the system perform on non-learner text (e.g., professional writing) where these resources may be less effective?  

**Additional Comments**  
- Typos: "c.f." (p. 129) should be "cf." and "F1" (p. 319) should clarify it means "F1-score (50-70%)."  
- The paper incorrectly states that M2 scorer "overestimates performance" (p. 337), but this is not supported by the data in Table 4 (which shows comparable scores).  
- The "operation tier" analysis (Section 4.2) is redundant with Table 5 and should be condensed.  

**Overall Assessment**  
The paper addresses a meaningful gap in GEC evaluation by enabling error type analysis, but its superficial technical description and methodological weaknesses undermine its contribution. The core innovation—the rule-based classifier—is inadequately explained, making it difficult to assess its novelty or limitations. While the CoNLL-2014 analysis is valuable, the paper's failure to justify its error framework and validate the method beyond student essays limits its impact. With substantial revisions to clarify the system and address evaluation concerns, this could be a strong contribution to GEC evaluation. As-is, it falls short of ACL's standards for novelty and rigor.

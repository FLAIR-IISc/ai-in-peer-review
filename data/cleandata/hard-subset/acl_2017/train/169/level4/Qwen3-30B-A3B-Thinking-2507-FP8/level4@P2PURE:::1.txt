

- Strengths: Useful application for teachers and learners; supports fine-grained comparison of GEC systems.
- Weaknesses: Highly superficial system description; unsatisfactory evaluation.
- General Discussion:
The paper proposes automatically enriching GEC system outputs with error types. This is a valuable application, as both teachers and learners benefit from explicit error type information (unlike systems that only provide corrected text). It also enables finer-grained comparison of GEC systems, including overall precision and error-type-specific recall/precision metrics.

Unfortunately, the system description remains highly superficial. The core relies on manually created rules, but the paper omits critical details: examples of rules, their number, complexity, ordering (e.g., whether early rules block later ones), etc. Instead, several pages focus on evaluating CoNLL-2014 systems (Table 6 spans a full page, repeating table data in text).

The evaluation is unsatisfactory for multiple reasons:
1.  Annotators should have independently created a gold standard for the 200 test sentences, rather than rating system output. Given a fixed tag set, a gold standard for this small test set is feasible. The current approach likely yields inflated ratings compared to a true gold standard (cf. Marcus et al., 1993, on agreement differences between pre-annotated and fresh annotation).
2.  The claim that "all 5 raters individually considered at least 95% of our rule-based error types to be either 'Good' or 'Acceptable'" is misleading. Multiple raters' scores should not be averaged this way. If the 18.5% "bad" edits were assigned to different edits (as the paper does not specify their distribution), this is far worse than the reported 3.7% average.
3.  No details are provided about the test data: number of error categories, covered categories (e.g., based on categories rated "good" by annotators).
4.  The phrase "edit boundaries might be unusual" lacks precision. A clearer description and examples are needed to assess potential application issues.

The claim that the system is "less domain-dependent than systems needing training data" is questionable. Hunspell's vocabulary likely varies across domains, and manually created rules can also be domain-dependent (and are language-dependent, a clear drawback versus ML approaches). The test data (FCE-test, CoNLL-2014) is limited to student essays.

The motivation for designing a new error category set is unclear. A stated reason—enabling easy search for underspecified categories (e.g., "NOUN")—is not compelling, as Nicholls (2003) supports similar searches. Alternatively, using the CoNLL-2014 tagset would have allowed leveraging the CoNLL gold standard for evaluation.

In summary, the paper's main motivation is ambiguous: Is it about a new system? But core details are omitted. Is it about new error categories? But minimal motivation/discussion is provided. Is it about evaluating CoNLL-2014 systems? But results are presented superficially.

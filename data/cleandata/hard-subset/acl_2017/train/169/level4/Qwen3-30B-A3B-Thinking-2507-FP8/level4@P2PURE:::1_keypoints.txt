- Strength: Useful application for teachers and learners and enables fine-grained comparison of GEC systems (precision and error-type-specific metrics).  
- Weakness: System description is highly superficial; critical implementation details of the rule-based core are omitted.  
- Missing rule details: No examples, counts, complexity, or ordering/interaction of rules provided.  
- Overemphasis on CoNLL-2014 evaluation: multiple pages/tables repeat results instead of system detail.  
- Evaluation design flaw: Annotators rated system output instead of independently creating a gold standard for the 200 test sentences.  
- Misleading rater summary: Averaging raters to report 95% “good/acceptable” hides distribution; the 18.5% “bad” edits could be dispersed and worse than reported.  
- Test-data opacity: No details on number of error categories or which categories are actually covered.  
- Vague wording: Phrases like “edit boundaries might be unusual” lack precise description and examples.  
- Questionable domain-independence claim: Hunspell vocabulary and manual rules are likely domain- and language-dependent; tests limited to student essays.  
- Unclear motivation for new error category set: rationale weak and CoNLL-2014 tagset would allow leveraging existing gold standards.  
- Ambiguous paper focus: unclear whether contribution is a new system, a new category set, or an evaluation of CoNLL-2014 systems.  
- Typos and reference issues need fixing (e.g., “c.f.” → “cf.”, “M2” → “M^2”, “50-70 F1” → “50–70% F1”, casing for ESL and Fleiss).

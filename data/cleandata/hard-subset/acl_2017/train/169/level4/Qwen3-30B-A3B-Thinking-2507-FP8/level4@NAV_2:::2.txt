

Summary: The paper introduces a method for automatically annotating grammatical error correction (GEC) system outputs with error type information, enabling detailed error type evaluation beyond simple recall. The main contributions are: (1) a linguistically-enhanced alignment approach for automatic edit extraction, (2) a dataset-independent rule-based classifier for error typing that relies solely on automatically derived linguistic properties, and (3) the first comprehensive error type analysis of systems participating in the CoNLL-2014 shared task.

Strengths:
- The approach enables precision-based error type evaluation, which was previously impossible since system outputs weren't annotated with error categories, providing a significant advancement over current GEC evaluation methods.
- The rule-based classifier demonstrates strong human agreement (95%+ "Good" or "Acceptable" ratings across 5 expert raters), validating its effectiveness without requiring labeled training data.
- The method allows for granular error type analysis across multiple dimensions (e.g., operation type, error category, and their combinations), revealing nuanced system strengths and weaknesses that would be obscured in overall scores.

Weaknesses:
- The core contribution—error type classification rules—are insufficiently described, making the approach difficult to replicate or adapt. The paper lacks examples of specific rules, their complexity, or how they handle edge cases.
- The human evaluation of error types was conducted with automatically extracted edit boundaries (introducing noise), potentially biasing raters toward the system's output rather than evaluating the error category alone. A comparison with manually defined boundaries would strengthen the evaluation.
- The evaluation is limited to English, with no discussion of necessary adaptations for other languages. The rules for error classification and edit boundary detection likely depend on linguistic properties specific to English, but the paper provides no assessment of this limitation or how it might affect generalizability.

Questions to Authors:
- Please provide specific examples of error classification rules to clarify their implementation and complexity, particularly for error types that require multi-step reasoning.
- How would your approach handle discontinuous error spans (e.g., separable verb prefixes in German), which cannot be accurately annotated with continuous edit spans?
- What specific linguistic properties would need modification to apply your method to a language with different morphological or syntactic characteristics than English?

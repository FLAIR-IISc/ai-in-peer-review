

Strengths: The paper presents a useful application for teachers and learners by enabling fine-grained comparison of GEC systems through automatic enrichment of GEC outputs with error types. This addresses a key limitation of many existing systems that only provide corrected text without explicit error annotations.

Weaknesses: The system description is highly superficial, and the evaluation methodology is unsatisfactory.

General Discussion:  
The paper proposes automatically enriching GEC outputs with error types, which is valuable for both educational stakeholders and system comparison. However, the core system—based on manually created rules—is described without essential details. The authors should provide concrete examples of these rules, specify their number, complexity, and ordering (e.g., whether earlier rules might block later ones). Instead of detailing the system, the paper dedicates excessive space to presenting CoNLL-2014 system evaluation results (Table 6 occupies an entire page), with repetitive textual summaries of table data.

The evaluation has multiple critical flaws:  
1.  **Gold standard methodology:** Annotators should have independently created a gold standard for the 200 test sentences rather than rating pre-annotated system outputs. For a fixed tag set and small test size, a gold standard is feasible. The current approach likely inflates agreement scores compared to annotating from scratch (cf. Marcus et al., 1993).  
2.  **Rater aggregation:** The claim that "all 5 raters individually considered at least 95% of error types as 'Good' or 'Acceptable'" misrepresents the data. Averaging individual ratings obscures the fact that 18.5% of edits received at least one "bad" rating (as reported in the paper), which is significantly worse than the misleading 3.7% average.  
3.  **Test data ambiguity:** The paper omits critical details about the test data, including the number of error categories covered, the specific categories rated "good" by annotators, and the distribution of errors.  
4.  **Ambiguous terminology:** The phrase "edit boundaries might be unusual" lacks precision and examples, leaving unclear whether this poses practical issues for the system.  

The claim that the system is "less domain-dependent than data-driven approaches" is questionable. Manually created rules are inherently language-dependent (a clear drawback versus ML methods), and Hunspell’s vocabulary coverage likely varies across domains. Furthermore, the evaluation uses only one domain (student essays from FCE-test and CoNLL-2014), undermining claims about domain robustness.  

The motivation for designing a new error category set is unclear. The stated reason (enabling searches for underspecified categories like "NOUN") could equally be addressed using Nicholls (2003) or the CoNLL-2014 tagset, which would have allowed direct use of the CoNLL gold standard for evaluation.  

In summary, the paper’s core contribution remains ambiguous: It is neither a novel system (key details omitted), nor a compelling new error taxonomy (minimal motivation/discussion), nor a meaningful CoNLL-2014 evaluation (superficial presentation).

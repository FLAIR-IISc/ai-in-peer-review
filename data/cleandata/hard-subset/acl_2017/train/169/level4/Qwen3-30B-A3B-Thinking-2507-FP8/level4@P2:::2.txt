

The paper introduces a method for automatically annotating error types in grammatical error correction (GEC) system outputs, enabling detailed error type analysis that was previously impossible since system outputs weren't annotated. The authors overcome this limitation by using a linguistically-enhanced alignment to extract edits between original and corrected sentences, followed by a rule-based classifier that categorizes edits based on automatically obtained linguistic properties (POS tags, lemmas, etc.). Human experts rated over 95% of the predicted error types as "Good" or "Acceptable," allowing the authors to apply their approach to analyze all systems in the CoNLL-2014 shared task.

Main contributions:
1. A novel method for automatically annotating error types in GEC system outputs, enabling detailed error type analysis previously impossible due to lack of annotations.
2. A dataset-independent rule-based classifier that relies solely on automatically obtained linguistic properties, eliminating the need for labeled training data.
3. First comprehensive error type analysis of all systems in the CoNLL-2014 shared task, revealing which error types different systems handle effectively.

Strengths:
1. The approach enables precise error type evaluation, allowing researchers to understand which specific error types each system handles well or poorly. This represents a significant methodological advance over previous metrics that only reported overall performance.
2. The rule-based classifier is dataset-independent and requires no labeled training data, unlike previous machine learning approaches that required different classifiers for different datasets. This makes the method more practical and broadly applicable.
3. The manual evaluation demonstrates high human agreement (86.3% "Good" + 10.0% "Acceptable" across raters), validating the approach's effectiveness despite noise from automatic edit extraction.

Weaknesses:
1. The paper lacks sufficient detail about the error type classification rules, which are central to the method. The authors don't provide examples of how the rules are applied to specific edits, making it difficult to assess or replicate the method. The rules are a key contribution but are only described in passing.
2. The classifier evaluation lacks thorough error analysis. While the authors note that "Bad" ratings were often due to POS tagging errors, they don't analyze which specific error types were most frequently misclassified or provide concrete directions for improvement.
3. The evaluation is limited to English, and the paper doesn't address language generalization. The authors don't discuss what adaptations would be needed for other languages or estimate the effort required, despite claiming their approach is dataset-independent.
4. The paper claims "there is no obvious explanation" for why five teams failed to correct unnecessary token errors, but for some systems (like UFC and AMU), the inability to handle these errors is directly explained by their system architectures, as noted in the original shared task papers.

Questions to Authors:
1. Could you provide specific examples of how your error type classification rules are applied to different edits? This would help clarify the method and make it more replicable.
2. How would you adapt your approach for a language with significantly different morphology (e.g., agglutinative languages), and what aspects of the current method would need modification?
3. For error types where systems achieved low scores (e.g., unnecessary token errors), did those systems even attempt to correct those error types, or were they simply not designed to handle them? This would clarify whether the low scores reflect actual performance limitations or design choices.

The paper presents a valuable methodological advance for GEC evaluation, but its impact is limited by insufficient description of the core classification rules and lack of discussion about language generalization. While the approach is promising, the paper would be significantly strengthened by providing more detail about the classification rules and addressing language adaptability. The current level of detail makes it difficult to assess the full potential of the method or replicate the results.

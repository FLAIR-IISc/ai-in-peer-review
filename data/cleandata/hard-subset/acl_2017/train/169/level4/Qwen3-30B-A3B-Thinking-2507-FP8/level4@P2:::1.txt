

**Summary**
This paper proposes a method to automatically annotate grammatical error correction (GEC) system outputs with error type information, enabling detailed error type analysis for GEC systems. The authors address the limitation that previous GEC evaluations could only measure recall due to unannotated system outputs. Their approach combines a linguistically-enhanced alignment algorithm for edit extraction with a rule-based error typing framework that relies solely on automatically obtained linguistic features.

**Main Contributions**
1. A novel method to automatically annotate GEC system outputs with error type information, enabling detailed error type analysis for the first time.
2. A dataset-independent rule-based error typing framework that requires no labeled training data and is fully deterministic.
3. A comprehensive error type analysis of all systems in the CoNLL-2014 shared task, revealing how different GEC approaches perform across specific error categories.

**Strengths**
1. The paper addresses a genuine gap in GEC evaluation methodology. Previous approaches could only measure overall recall, but the authors' method enables detailed error type analysis, which is valuable for understanding system strengths and weaknesses.
2. The rule-based framework is conceptually simple and avoids the need for labeled training data, making it more practical than machine learning approaches that require different classifiers for different datasets.
3. The application to CoNLL-2014 provides valuable insights into system performance across error types, revealing that different approaches complement different error categories (e.g., MT approaches excel at NOUN errors while classifiers handle VERB:FORM errors well).

**Weaknesses**
1. The description of the rule-based system is highly superficial. The paper fails to provide examples of the rules, their complexity, ordering, or how they interact. Without these details, it's impossible to assess the novelty or limitations of the approach. The authors instead devote extensive space to presenting CoNLL-2014 results (Table 6 spans a full page) rather than detailing their core contribution.
2. The evaluation methodology is problematic. Annotators rated the system's output rather than creating a gold standard for the 200 test sentences. This approach likely inflates agreement scores compared to independent annotation. Furthermore, the paper averages "Good" and "Acceptable" ratings across raters without proper inter-annotator agreement analysis, making the 95% "Good" or "Acceptable" claim misleading (if "bad" ratings were distributed across different edits, 18.5% of edits might have been rated "bad" by at least one rater).
3. The motivation for creating a new error taxonomy is unclear. The paper doesn't explain why the proposed taxonomy is superior to existing ones (e.g., CoNLL-2014's taxonomy) or how it addresses the stated need for "searchable" categories. The claim that the taxonomy supports "easy searching" for underspecified categories like "NOUN" is unsubstantiated, as other taxonomies (e.g., Nicholls 2003) could serve the same purpose.
4. The claim that the system is "dataset-independent" is questionable. The approach relies on Hunspell's dictionary and manually created rules, both of which may be domain-specific (the test data comes exclusively from student essays). The paper fails to address how the system would perform across different domains or languages.
5. The presentation of CoNLL-2014 results is redundant and superficial. The text simply repeats facts from Table 6 without providing meaningful analysis, making the evaluation feel like a data dump rather than a substantive contribution.

**Questions to Authors**
1. Could you provide specific examples of the rule-based error typing rules? How many rules exist, and how are they ordered and prioritized?
2. Why was a new error taxonomy created instead of using an existing one (e.g., CoNLL-2014's taxonomy)? How does your taxonomy better support error type analysis for GEC evaluation?
3. Given that the test data comes exclusively from student essays, what evidence do you have that your approach would work across different domains or languages?
4. The paper claims "all 5 raters individually considered at least 95% of our rule-based error types to be either 'Good' or 'Acceptable'." How were the "bad" ratings distributed across edits? If a single edit was rated "bad" by multiple raters, how did this affect the overall evaluation?

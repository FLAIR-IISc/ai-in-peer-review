

Strengths: Valuable utility for educators and students; enables detailed comparative analysis of GEC systems through error-type-specific metrics.  
Weaknesses: System description lacks depth; evaluation methodology is inadequate.  

General Discussion:  
The paper proposes automatically enriching GEC outputs with error-type annotations, offering significant benefits for teachers and learners (unlike systems providing only corrected text) and enabling granular GEC system comparisons via precision/recall per error type.  

However, the system description is overly superficial. The core relies on manually crafted rules, yet the paper omits critical details: examples of rules, their quantity, complexity, ordering logic (e.g., potential rule conflicts), and rationale. Instead, excessive space is devoted to summarizing CoNLL-2014 results (Table 6 spans a full page, with repetitive textual descriptions of tabulated data).  

The evaluation suffers from multiple flaws:  
1. Annotators rated system outputs rather than establishing an independent gold standard for the 200 test sentences. Producing a gold standard for this small set (given fixed tags) is feasible and necessary to avoid inflated agreement rates (cf. Marcus et al., 1993).  
2. Averaging individual rater scores (e.g., reporting 3.7% "bad" edits) misrepresents the data. If the 18.5% of edits deemed "bad" by *any* rater stemmed from distinct edits (as implied), this is far more concerning than the averaged figure.  
3. Test data characteristics are undefined (e.g., error category coverage, total categories).  
4. Ambiguous phrasing ("edit boundaries might be unusual") lacks examples or explanation of potential application impacts.  

The claim of reduced domain dependence versus training-based systems is questionable: manually created rules remain domain-specific and language-dependent (unlike ML approaches), and the evaluation uses only student-essay data (FCE-test, CoNLL-2014).  

The rationale for designing a new error taxonomy is unclear. The stated reason (searching underspecified categories like "NOUN") could be addressed using existing tagsets (e.g., Nicholls, 2003; CoNLL-2014), obviating the need for a new set and enabling direct use of the CoNLL gold standard.  

Ultimately, the paper’s core contribution is ambiguous: Is it a novel system (details omitted)? A new taxonomy (motivation absent)? Or a CoNLL-2014 evaluation (results presented superficially)?



**Summary of Paper and Main Contributions**  
This paper introduces a method to automatically annotate unannotated Grammatical Error Correction (GEC) system outputs with error type information, addressing a critical gap in GEC evaluation. Previously, error type performance could only be measured via recall since system outputs lacked error type annotations. The authors propose a two-stage approach: (1) linguistically-enhanced alignment to extract edits between original and corrected sentences, and (2) a dataset-independent rule-based classifier to assign error types. The primary contributions are: (1) a practical solution for automatic error type annotation of GEC outputs, (2) a rule-based classifier that avoids dataset-specific training requirements, and (3) the first detailed error type analysis of CoNLL-2014 GEC systems.

**Strengths**  
1. **Solves a genuine evaluation gap**: The paper addresses a long-standing limitation in GEC—system outputs lack error type annotations, making detailed error type analysis impossible. This is a significant contribution, as error type performance (e.g., "systems excel at correcting noun errors but struggle with verb tense") is crucial for understanding system strengths and weaknesses, yet previously unmeasurable.  
2. **Robust evaluation framework**: The manual evaluation (5 raters, 200 edits) shows >95% of predicted error types were rated "Good" or "Acceptable," validating the classifier’s effectiveness. This is particularly strong given the noise from automatic edit extraction.  
3. **Practical impact**: The approach enables direct comparison of GEC systems at the error type level, offering actionable insights for system improvement (e.g., "CAMB excels at missing determiners but struggles with word order"). The CoNLL-2014 analysis demonstrates this utility convincingly.

**Weaknesses**  
1. **Superficial system description**: The core of the method—a rule-based classifier—is described only in vague terms. The paper omits critical details: number of rules, their complexity, ordering logic (e.g., whether earlier rules block later ones), and examples. For instance, how does the classifier handle ambiguous cases like "ring → rings" (NOUN:NUM vs. VERB:SVA)? Without these details, the method’s reproducibility and technical novelty are unclear.  
2. **Flawed classifier evaluation**: The evaluation protocol (asking raters to judge predicted error types rather than creating a gold standard) inflates results. As Marcus et al. (1993) show, pre-annotated data typically yields higher agreement than manual annotation from scratch. The claim that "95% of types were rated Good/Acceptable" is misleading; if "Bad" ratings were distributed across different edits (as likely), up to 18.5% of edits could have been deemed incorrect by at least one rater.  
3. **Insufficient motivation for error categories**: The authors justify their 25-category framework by claiming it enables "flexible error type search" (e.g., "R:NOUN"), but they fail to explain why this is superior to existing tagsets (e.g., CoNLL-2014 or Nicholls 2003). Using CoNLL-2014’s tagset would have allowed direct comparison with gold-standard annotations, strengthening the evaluation.  
4. **Limited domain coverage**: The evaluation relies solely on student-essay data (FCE-test and CoNLL-2014), which may not generalize to other domains. The claim that the approach is "dataset-independent" is undermined by the use of domain-specific resources like Hunspell’s dictionary, which likely varies in coverage across domains.

**Questions to Authors**  
1. Could you provide concrete examples of your rule-based classifier’s rules (e.g., a sample rule for "VERB:SVA" errors) and clarify how rule ordering affects classification?  
2. Why not use the CoNLL-2014 error type tagset for evaluation, enabling direct comparison with gold-standard annotations?  
3. How would the system perform on non-student-essay domains (e.g., professional writing), given the reliance on domain-specific resources like Hunspell?  

**Additional Comments**  
- The paper’s focus on CoNLL-2014 results (Table 6) is justified but repetitive; the text could better synthesize insights rather than listing numbers.  
- Typos: "c.f." should be "cf." (p. 129); "M2" should be "M²" (p. 366); "50-70 F1" should clarify "50-70% F1" (p. 319).  
- The claim that "the M2 scorer overestimates performance" (p. 4.1) is intriguing but requires stronger evidence (e.g., a direct comparison with gold-standard edit boundaries).  

**Overall Assessment**  
This paper addresses a meaningful problem in GEC evaluation with a practical, well-motivated solution. However, the superficial treatment of the core method and flawed evaluation protocol undermine confidence in the approach. With the proposed clarifications (especially rule examples and a proper gold-standard evaluation), this could be a strong contribution. As it stands, the weaknesses outweigh the strengths.

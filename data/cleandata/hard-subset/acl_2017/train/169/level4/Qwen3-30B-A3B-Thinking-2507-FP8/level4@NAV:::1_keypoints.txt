- Strength: Useful tool for educators and learners, providing more informative feedback than corrected text alone.  
- Strength: Enables granular comparison of GEC systems via error-type-specific precision/recall metrics.  
- Weakness: System description is superficial and omits crucial implementation details.  
- Missing system details: No examples, count, complexity, ordering/priority logic, or rationale for the manually crafted rules.  
- Presentation issue: Excessive space devoted to CoNLL-2014 result summaries, with repetitive text for tabled data.  
- Evaluation flaw 1: Annotators labeled system outputs instead of creating an independent gold standard for the 200 test sentences.  
- Evaluation flaw 2: Averaging rater scores misrepresents disagreement; reporting aggregated "bad" rates hides problematic edits flagged by any rater.  
- Evaluation flaw 3: Test data properties (error category coverage, total categories) are unspecified.  
- Evaluation flaw 4: Vague remarks about unusual edit boundaries lack examples or explanation of impact.  
- Claim issue: Claim of reduced domain dependence is questionable—manual rules are still domain- and language-specific, and evaluation is limited to student essays.  
- Taxonomy concern: Motivation for a new error taxonomy is unclear; existing tagsets (e.g., Nicholls, CoNLL-2014) could suffice and allow use of CoNLL gold standard.  
- Contribution ambiguity: Unclear whether the main contribution is the system, the taxonomy, or the CoNLL-2014 evaluation.  
- Typos/errors to fix: "c.f." → "cf.", "M2" → "M^2", clarify "50-70 F1" (e.g., 50–70%), and reference fixes ("esl" → "ESL", "fleiss, kappa" → "Fleiss’ kappa").

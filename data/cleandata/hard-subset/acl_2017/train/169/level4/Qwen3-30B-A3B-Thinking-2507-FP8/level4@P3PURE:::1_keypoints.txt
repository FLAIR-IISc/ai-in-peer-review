- Strength: Presents a useful application that enriches GEC outputs with error types, aiding teachers, learners, and system comparison.
- Weakness: System description is highly superficial and lacks essential implementation details.
- Weakness: Evaluation methodology is unsatisfactory and flawed.
- System detail missing: No concrete examples, counts, complexity, or ordering of the manually created rules are provided.
- Presentation issue: Excessive space is devoted to CoNLL-2014 results with repetitive summaries (e.g., oversized Table 6).
- Gold-standard flaw: Annotators should have independently created a gold standard for the 200 test sentences; current method likely inflates agreement.
- Rater aggregation misrepresented: Averaging hides that 18.5% of edits had at least one "bad" rating, contradicting the claimed 3.7% average.
- Test-data ambiguity: Paper omits number of error categories, which categories were rated “good,” and error distribution details.
- Terminology unclear: Phrases like "edit boundaries might be unusual" are imprecise and lack examples to assess impact.
- Domain-robustness claim questionable: Manually created rules are language/domain-dependent and Hunspell coverage likely varies; evaluation uses only student-essay domains.
- Taxonomy motivation weak: Unclear why a new error category set was needed when existing sets (Nicholls, CoNLL-2014) could have been used.
- Core contribution ambiguous: Paper is neither a clearly novel system (missing details), nor a convincing new taxonomy, nor a thorough CoNLL-2014 evaluation.
- Typos and reference fixes needed: "c.f." → "cf.", "M2" → "M^2", clarify "50–70% F1", and correct reference casing (e.g., "ESL", "Fleiss’ κ").

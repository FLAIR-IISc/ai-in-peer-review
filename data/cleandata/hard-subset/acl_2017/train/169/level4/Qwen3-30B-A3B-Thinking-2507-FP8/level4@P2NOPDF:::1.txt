

Strengths: The paper addresses a valuable need by automatically enriching GEC system outputs with error types, benefiting both teachers and learners who require explicit error information beyond corrected text. It enables finer-grained system comparison through error-type-specific precision and recall metrics.

Weaknesses: The system description is insufficiently detailed, and the evaluation methodology is flawed. Key aspects of the core rule-based approach remain unexplained, while the evaluation lacks rigor and necessary context.

General Discussion:  
The proposed system aims to annotate GEC outputs with error types, a useful application. However, the core mechanism—a set of manually created rules—is inadequately described. The paper omits critical details: examples of rules, their count, complexity, ordering logic (e.g., whether earlier rules block later ones), and the rule creation process. Instead, excessive space is devoted to rehashing CoNLL-2014 system results (Table 6, one full page), with text redundantly repeating table data.

The evaluation suffers from multiple methodological issues:  
1. **Gold standard issue**: Annotators rated the system’s output rather than creating an independent gold standard for the 200 test sentences. For a fixed tag set, a gold standard should be feasible for this small dataset. Using the system’s output as the basis for annotation likely inflates agreement scores (cf. Marcus et al., 1993), undermining validity.  
2. **Rater aggregation**: Averaging individual rater scores (e.g., reporting "3.7% bad" edits) misrepresents the data. If 18.5% of edits were rated "bad" by *at least one* rater (as implied), this indicates significant inconsistency, contradicting the paper’s positive conclusion.  
3. **Test data ambiguity**: No details are provided about the test data’s error categories, coverage, or distribution. For instance, the paper states that "all 5 raters individually considered at least 95% of error types as 'Good' or 'Acceptable'" but omits how many categories were rated or their nature.  
4. **Unclear terminology**: The phrase "edit boundaries might be unusual" lacks precision. Examples and explanation are needed to assess if this poses practical problems for the system.  

The claim that the system is "less domain-dependent" than data-driven approaches is unsupported. Manual rules are inherently domain- and language-dependent (unlike ML models trained on diverse data), and the evaluation uses only student-essay data (FCE-test, CoNLL-2014), limiting generalizability.  

The motivation for designing a new error category set is unclear. The stated reason—"enabling easier searches for underspecified categories like 'NOUN'"—does not justify deviating from established tagsets (e.g., Nicholls, 2003; CoNLL-2014). Using the CoNLL-2014 tagset would have permitted direct evaluation against its gold standard, strengthening the contribution.  

Ultimately, the paper’s core contribution is ambiguous. Is it a novel system? But critical details about the rules are omitted. Is it a new tagset? Yet there is no justification or discussion of its design. Is it an evaluation of CoNLL-2014 systems? But results are presented superficially without meaningful analysis.

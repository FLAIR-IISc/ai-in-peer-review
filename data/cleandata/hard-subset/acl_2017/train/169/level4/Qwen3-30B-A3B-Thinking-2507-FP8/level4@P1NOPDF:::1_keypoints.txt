- Strength: Addresses a valuable application—automatically tagging GEC outputs with error types—to aid teachers/learners and enable finer-grained system comparison (precision/recall).
- Weakness: System description is overly superficial; critical technical details are omitted.
- Weakness: Evaluation lacks rigor and contains multiple methodological flaws.
- Missing rule details: No examples, total count, complexity, ordering logic, or design rationale for the manually created rules.
- Presentation issue: Excessive space spent rehashing CoNLL-2014 system results (Table 6 is repetitive and oversized).
- Evaluation methodology flaw: Used system outputs rather than a gold-standard annotation, likely inflating agreement.
- Misleading reporting: Averaging individual annotator ratings misrepresents performance; the reported 3.7% “bad” average conflicts with an 18.5% bad-rating signal.
- Undefined categories: Test data error categories are not specified—no counts, coverage, or explanation of omissions.
- Clarity problem: Phrase “edit boundaries might be unusual” is vague and needs concrete examples and impact analysis.
- Domain-dependence claim questionable: Manually crafted rules and Hunspell are domain- and language-dependent, undermining the “less domain-dependent” assertion.
- Limited generalizability: Evaluation restricted to student essays (FCE, CoNLL-2014), so broader applicability is untested.
- Tagset rationale unclear: No strong justification for a new category set when existing sets (Nicholls 2003 or CoNLL-2014) could have been used.
- Ambiguous contribution: It is unclear whether the paper’s main contribution is a novel system, new error categories, or an evaluation of CoNLL-2014 systems.
- Typos and reference fixes needed: e.g., “c.f.” → “cf.” (l129), “M2” → “M²” (l366), “50-70 F1” → “50-70% F1” (l319), “esl” → “ESL” (l908), “fleiss, kappa” → “Fleiss’ κ” (l878/79).

- Strength: Useful application for teachers and learners, providing error-type enrichment beyond simple corrections.  
- Strength: Enables fine-grained comparison of GEC systems with per-error-type precision/recall measures.  
- Weakness: System description is highly superficial and lacks essential implementation details.  
- Missing details: No sample rules, total rule count, rule complexity, or rule ordering/interaction explained.  
- Overfocus: Paper devotes excessive space to CoNLL-2014 system evaluation (e.g., oversized Table 6) rather than system details.  
- Evaluation flaw: Annotators rated system output instead of independently creating a gold standard for the 200 test sentences.  
- Evaluation bias: Rating pre-annotated output likely produces more favorable results than comparison to a genuine gold standard.  
- Reporting issue: Averaging individual rater scores is misleading; individual disagreements imply a much higher “bad” rate.  
- Missing test info: No details on test data error-category counts or which categories were judged “good.”  
- Clarity problem: “Edit boundaries might be unusual” is vague and needs precise explanation and examples.  
- Domain-dependence claim questionable: Manual rules and resources like Hunspell can be domain- and language-dependent; test data is limited to student essays.  
- Tagset rationale unclear: Insufficient justification for a new error category set when existing tagsets (Nicholls 2003, CoNLL-2014) could be used.  
- Unclear contribution: Paper unclear whether it presents a new system, a new tagset, or primarily an evaluation of CoNLL-2014 systems.  
- Minor issues: Multiple typos and reference errors (e.g., “c.f.”, “M^2”, percent notation, case in references like ESL and Fleiss/Kappa).

**Review:**

**Strengths:** Useful application for teachers and learners; supports fine-grained comparison of GEC systems.

**Weaknesses:** Highly superficial description of the system; evaluation not satisfying.

**General Discussion:**

The paper presents an approach for automatically enriching the output of GEC systems with error types. This is a valuable application, as both teachers and learners can benefit from this information—many GEC systems currently only output a corrected version without explicitly identifying the error type. It also enables more detailed comparisons of GEC systems, allowing for precision analysis in general and error type-specific recall and precision figures.

Unfortunately, the description of the system remains highly superficial. The core of the system consists of a set of (manually?) created rules, but the paper provides insufficient detail regarding these rules. The authors should, for example, provide illustrative examples of rules, specify the total number of rules, describe their complexity, and explain their ordering (could an early rule preclude the application of a later rule?). Instead of presenting these crucial system details, several pages are dedicated to evaluating the systems that participated in CoNLL-2014. Table 6, which spans an entire page, lists results for all systems, and the text largely repeats information readily apparent from the table.

The evaluation of the proposed system is also unsatisfying in several respects. First, the annotators should have independently annotated a gold standard for the 200 test sentences rather than simply rating the system’s output. Given the relatively small test set and a fixed set of tags, creating a gold standard is feasible. The current approach likely yields more favorable ratings than would comparison with a genuine gold standard (see, e.g., Marcus et al. (1993) for a discussion of agreement when reviewing pre-annotated data versus annotating from scratch). 

Second, the paper reports that “all 5 raters individually considered at least 95% of our rule-based error types to be either ‘Good’ or ‘Acceptable’.” Averaging individual rater scores in this manner is atypical. If each "bad" score was assigned to a distinct edit (information not provided in the paper), then 18.5% of the edits were deemed "bad" by at least one annotator—a considerably more concerning figure than the reported average of 3.7%. Third, there is a lack of information concerning the test data, such as the number of error categories it contains and which categories are covered, according to the annotators’ “good” ratings. Fourth, the phrase “edit boundaries might be unusual” is imprecise. A more detailed description with illustrative examples is needed. Could this be problematic for the system’s application?

The authors claim their system is less domain-dependent than systems requiring training data. This assertion is questionable. For example, Hunspell's vocabulary likely does not cover all domains equally well, and manually created rules can also be domain-dependent—and are inherently language-dependent, a clear limitation compared to machine learning approaches. Furthermore, the test data used (FCE-test, CoNLL-2014) is limited to one domain: student essays. 

The rationale behind designing a new set of error categories remains unclear. One stated reason—to facilitate searching for underspecified categories like “NOUN”—seems unnecessary, as the tagset presented in Nicholls (2003) appears to support such searches. Alternatively, why not utilize the CoNLL-2014 tagset, which would allow for evaluation against the existing CoNLL gold standard? 

In summary, the paper’s central motivation is somewhat ambiguous. Is it presenting a new system? If so, critical details are omitted. Is it proposing a new set of error categories? If so, there is insufficient justification or discussion. Or is it evaluating the CoNLL-2014 systems? If so, the presentation of results remains superficial.

The paper presents an approach for automatically enriching the output of Grammatical Error Correction (GEC) systems with error type information. This is a valuable application, as both teachers and learners can benefit from this information – many GEC systems currently only output a corrected version without explicitly identifying the error type. It also enables more fine-grained comparison of GEC systems, providing precision figures generally and error type-specific recall and precision metrics.

Unfortunately, the description of the system remains somewhat superficial. The core of the system relies on a set of (manually created?) rules, but the paper doesn’t provide sufficient detail about these rules. The authors should, for example, include illustrative examples, specify the number of rules, detail their complexity, and explain their ordering – could earlier rules prevent the application of later ones? Instead of presenting these crucial system details, a considerable portion of the paper is dedicated to an evaluation of the systems participating in the CoNLL-2014 shared task. Table 6, which occupies an entire page, lists results for all systems, and the text frequently repeats facts and figures readily apparent from the table.

The evaluation of the proposed system is not entirely convincing. First, the annotators should have independently annotated a gold standard for the 200 test sentences instead of merely rating the system’s output. Given a fixed tagset, creating a gold standard for this relatively small dataset should be feasible. The current approach likely yields more favorable ratings than comparison with a genuine gold standard would (see Marcus et al. (1993) for a discussion of agreement discrepancies when reviewing pre-annotated data versus annotating from scratch). 

Second, the authors state that “all 5 raters individually considered at least 95% of our rule-based error types to be either ‘Good’ or ‘Acceptable’.” Averaging individual rater scores in this manner is unconventional. If each “bad” score was assigned to a different edit (information not provided in the paper), 18.5% of edits were considered “bad” by at least one annotator – a considerably higher and more concerning figure than the reported average of 3.7%.

Third, the paper lacks information regarding the test data, such as the number of error categories present or which categories are covered, according to the categories rated as “good” by the annotators.

Fourth, the phrase "edit boundaries might be unusual" requires clarification. A more precise description, alongside examples, is needed. Could this issue impact the system’s applicability?

The authors claim their system is less domain-dependent than systems requiring training data. This assertion is questionable. For example, Hunspell’s vocabulary likely doesn’t cover all domains equally well, and manually created rules can be domain-dependent – and are certainly language-dependent, a clear drawback compared to machine learning approaches. Moreover, the test data used (FCE-test, CoNLL-2014) is limited to one domain: student essays.

The motivation for designing a new error category set remains unclear. One stated reason is to facilitate searching for underspecified categories (like “NOUN” generally). However, tagsets like that presented in Nicholls (2003) already support such searches. Alternatively, why not use the CoNLL-2014 tagset? This would allow for direct evaluation against the CoNLL gold standard.

In summary, the paper's central motivation is somewhat ambiguous. Is it about a new system? But the most crucial details are omitted. Is it about a novel error category set? But insufficient justification or discussion is provided. Is it about evaluating the CoNLL-2014 systems? But the presentation of results is superficial.

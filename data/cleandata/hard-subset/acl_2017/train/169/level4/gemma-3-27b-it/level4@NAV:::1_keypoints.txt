- Useful application for educators and learners: provides informative grammatical-error feedback and enables finer-grained GEC system comparison by error-type precision/recall.  
- System description is superficial and lacks essential details about the rule set (examples, total number, complexity, ordering/blocking).  
- Paper spends excessive space restating CoNLL-2014 results (largely duplicating Table 6) instead of deep system analysis.  
- Evaluation is unconvincing: uses ratings of system output rather than independent gold-standard annotations, risking bias (cf. Marcus et al., 1993).  
- Rater scoring methodology is questionable: averaging rater scores is unconventional and masks that ~18.5% of edits were flagged "bad" by any rater versus the reported 3.7% average.  
- Critical test-data details are missing: number of error categories present, coverage of categories marked "good," and unclear comment about "unusual" edit boundaries (needs examples).  
- Claim of reduced domain dependence is dubious: resources like Hunspell and manual rules are domain- and language-dependent; evaluation data limited to student essays.  
- Rationale for a new error category set is unclear; existing tagsets (e.g., Nicholls 2003 or CoNLL-2014) would aid comparability.  
- Overall motivation is ambiguous—the paper neither sufficiently details a new system, justifies a novel categorization, nor convincingly evaluates GEC systems.  
- Typos/formatting issues to fix: "cf." usage, M^2 notation, clarify "50-70 F1" as percent, and reference capitalization (ESL, Fleiss, Kappa).

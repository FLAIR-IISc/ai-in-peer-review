- Strength: Useful application for teachers and learners; enables fine-grained comparison of GEC systems.  
- Weakness: System description is highly superficial; core rule set lacks detail.  
- Missing rule details: no illustrative examples, no total rule count, no complexity description, no ordering/execution semantics.  
- Paper overemphasizes CoNLL-2014 system result tables at the expense of system design explanation.  
- Evaluation flawed: annotators rated system output instead of independently creating a gold standard for the 200 test sentences.  
- Small test set makes creating a true gold standard feasible and preferable to output rating.  
- Reporting/aggregation issues: averaging rater scores masks disagreement; up to 18.5% of edits might have been rated “bad” by at least one rater.  
- Insufficient information on test data coverage: number and identity of error categories covered by annotators’ “good” ratings not reported.  
- Vague statement “edit boundaries might be unusual” needs concrete examples and discussion of potential problems.  
- Questionable claim of lower domain-dependence: manually created rules and resources like Hunspell can be domain- and language-dependent.  
- Test data limited to student essays (FCE, CoNLL-2014), undermining generality claims.  
- Unclear motivation for a new error category set; existing tagsets (Nicholls 2003, CoNLL-2014) might suffice and enable standard evaluation.  
- Overall ambiguity: paper unclear whether it presents a new system, a new tagset, or an evaluation of CoNLL systems, with insufficient justification for any.  
- Typos/formatting issues to fix: cf. vs c.f., M^2 notation, clarify “50-70 F1”, capitalization errors (ESL), and Fleiss kappa references.

- Proposes a novel evaluation enabling error-type precision in GEC without manual error annotations.  
- Important contribution for analyzing GEC system behavior; evaluated across multiple systems.  
- Advantages: computes precision by error type, independent of manual annotation, and handles multi-token errors.  
- Automatically selected error tags were largely approved by human experts.  
- Major omission: rules to derive error types are not described, harming reproducibility and transparency.  
- Classifier evaluation lacks thorough error analysis and actionable directions to improve the classifier.  
- Unclear whether error categories are based on prior work or how they were devised.  
- Likely dependence of rules on the alignment algorithm is not discussed or documented.  
- Paper should show example rules and report number/complexity of rules to estimate adaptation effort.  
- Evaluation only on English; unclear how hard it is to adapt method to other languages and preprocessing pipelines.  
- Preprocessing yields continuous error spans, which may fail for discontinuous edits (e.g., German separable verbs).  
- Human judge approval may be biased by noisy boundaries and the rating scale; need evaluation with manual boundaries.  
- Classifier error attribution is shallow (e.g., “Bad” linked to POS errors) and lacks detailed breakdown of failure cases.  
- Significance testing details are unclear; using F0.5 for significance may be inappropriate due to weak discriminative power.  
- Comparison between gold and auto reference appears limited to boundaries, not classification accuracy.  
- Some systems’ zero scores on certain error types likely reflect design choices, not outright failure; this is not analyzed.  
- Table 6 should include raw TP/FP/TN/FN counts and note when results are based on few instances.  
- Claim that different approaches complement different error types should be linked to prior research.  
- Multi-token error analysis needs deeper interpretation given some systems’ inherent limitations and training data differences.  
- Paper should acknowledge that some GEC systems can produce error annotations via submodules and clarify the unified benefit.  
- Minor issues: inconsistent reference capitalizations, a URL needing wrapping, and missing page info for a citation.  
- Authors’ response concedes the approach is promising but the paper is not ready due to missing rule descriptions and generalization concerns.  
- Generalization to other languages must be addressed or transparently estimated; otherwise the method’s scope is limited.  
- Authors’ statement that some system failures are “without obvious explanation” is challenged by known system design constraints (e.g., handling superfluous words).

- Strength: The application benefits teachers and learners and enables fine-grained comparisons of GEC systems.  
- Weakness: The system description is overly superficial and lacks critical implementation details.  
- Missing rule details: No examples, counts, complexity, or ordering of (possibly manual) rules are provided.  
- Excessive focus on CoNLL-2014 results: Large table and redundant text occupy undue space.  
- Evaluation—no gold standard: Authors did not create independent gold annotations for the 200 test sentences, likely inflating ratings.  
- Evaluation—misleading averaging: Averaging individual rater scores masks that many edits could be rated “bad” by at least one annotator.  
- Evaluation—insufficient test data description: Paper fails to report which and how many error categories the test set contains.  
- Evaluation—vague issues: The comment “edit boundaries might be unusual” is unclear and lacks examples.  
- Domain-dependence claim questionable: Hunspell and manual rules can be domain-dependent and the system is language-dependent.  
- Limited test domain: Evaluation uses only student-essay data (FCE-test, CoNLL-2014).  
- Unclear motivation for new tagset: Rationale for a new set of error categories is not justified given existing tagsets (Nicholls 2003, CoNLL-2014).  
- Ambiguous paper focus: It’s unclear whether the contribution is the system, the tagset, or the CoNLL-2014 evaluation.  
- Typos and reference issues: Multiple minor errors (e.g., “c.f.”→“cf.”, “M2”→“M²”, clarify “50-70 F1”, “esl”→“ESL”, “fleiss, kappa”→“Fleiss, Kappa”).

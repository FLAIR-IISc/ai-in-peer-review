- Strengths: Useful application for teachers/learners and enables fine-grained comparison of GEC systems.  
- Weaknesses: System description is superficial and the evaluation is unsatisfactory.  
- Core system critique: Relies on (manually?) crafted rules but gives no examples, total count, complexity, conflict resolution, rule ordering, or coverage statistics.  
- Presentation critique: Paper devotes excessive space to re-evaluating CoNLL-2014 systems and repeats table values in the text (e.g., Table 6).  
- Eval shortcoming 1: Annotators should have created an independent gold standard for the 200 test sentences instead of only rating system output, to avoid inflated agreement.  
- Eval shortcoming 2: Reporting “95% of rule types good/acceptable” by averaging raters is inappropriate; need per-edit distributions and standard inter-annotator agreement metrics.  
- Eval shortcoming 3: Test data are underspecified—report counts and distribution of error categories and which categories were covered/rated “good.”  
- Eval shortcoming 4: “Edit boundaries might be unusual” needs clarification with examples, frequency, and assessment of impact on tagging/usability.  
- Domain-dependence claim: Claim of being less domain-dependent is unclear—Hunspell and hand-written rules are domain- and language-sensitive, and both evaluation sets are student essays.  
- Tagset rationale: Motivation for a new error taxonomy is weak—existing schemes could support underspecified queries and using CoNLL tags would allow direct comparison to CoNLL gold.  
- Contribution unclear: It’s not clear whether the paper’s main contribution is a system (missing implementation detail), a taxonomy (minimal motivation/comparison), or an evaluation (descriptive/redundant).  
- Typos/style fixes: change “c.f.” to “cf.”, “M2” to “M^2”, clarify “50–70 F1” as percent, and fix reference capitalization (ESL, Fleiss, Kappa).

- Strengths: Practical tool for educators and students; enables fine-grained comparisons across GEC systems.

- Weaknesses: System description is overly shallow; evaluation is unsatisfactory.

- General Discussion:
The paper proposes automatically augmenting GEC outputs with error types. This is a genuinely useful contribution: both teachers and learners can benefit from explicit error-type annotations (many GEC systems only provide corrected text without labeling the error), and it also supports more granular comparisons of GEC systems, including overall precision and error-type-specific precision and recall.
However, the system description is very superficial. The core relies on a set of (manually?) crafted rules, yet the paper gives no substantive details about them. The authors should provide examples of rules, report how many rules exist, describe their complexity, explain their ordering (could an earlier rule preempt a later one?), and so on. Instead of these essential details, multiple pages focus on evaluating systems from CoNLL-2014. Table 6 (spanning an entire page) enumerates results for all systems, and the narrative largely reiterates information visible in the table.
The evaluation of the proposed method is unsatisfying in several respects.
First, rather than merely judging the system’s output, annotators should have independently created a gold standard for the 200 test sentences. With a fixed tagset, constructing a gold standard for such a small dataset is feasible. The chosen approach likely yields substantially higher ratings for the system than comparison against a true gold standard would (see, e.g., Marcus et al. (1993) on agreement when reviewing pre-annotated data vs. annotating from scratch).
Second, the paper claims that “all 5 raters individually considered at least 95% of our rule-based error types to be either ‘Good’ or ‘Acceptable’.” It is not standard practice to treat raters individually and then average their assessments in this manner. If each “bad” judgment fell on different edits (the distribution is not reported), then 18.5% of edits received a “bad” label from some annotator—substantially worse than the reported 3.7% average.
Third, the test data are insufficiently described: we do not learn how many error categories are present, which categories occur, or which categories were covered (particularly those deemed “good” by annotators).
Fourth, the statement that “edit boundaries might be unusual” requires clarification and examples. Could this pose issues for using the system in practice?
The claim that the system is less domain dependent than models requiring training data is questionable. For instance, Hunspell’s lexicon likely varies in coverage across domains, hand-authored rules can be domain-specific, and they are entirely language-dependent—a clear disadvantage relative to ML approaches. Moreover, the test sets used (FCE-test, CoNLL-2014) derive from a single domain: student essays.
It is also unclear why a new error tagset was introduced. The stated motivation is to facilitate search for underspecified categories (e.g., generic “NOUN”). The Nicholls (2003) tagset appears to support similar queries. Alternatively, using the CoNLL-2014 tagset would have enabled leveraging its gold standard for evaluation.
In summary, the paper’s primary goal remains ambiguous. Is it a new system? Key implementation details are missing. Is it a new error taxonomy? There is minimal motivation and discussion. Is it an evaluation of CoNLL-2014 systems? The presentation is superficial.

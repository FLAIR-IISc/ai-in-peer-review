Summary and contributions
The paper proposes an automatic method to annotate GEC system outputs with error types, enabling per-type evaluation beyond overall scores. The approach aligns original and corrected sentences using a linguistically enhanced algorithm and then classifies edits with a rule-based, dataset-independent scheme. The authors apply the method to CoNLL-2014 system outputs and present detailed analyses by error type.

As I see it, the main contributions are:
- A practical tool to automatically assign error types to unannotated GEC outputs, which can support fine-grained evaluation (precision/recall by type) and provide actionable feedback for teachers and learners.
- A large-scale application of the method to the CoNLL-2014 shared task outputs, illustrating how different systems behave across error categories.

Strengths
- Practical value for education: Annotated outputs that make error types explicit can help teachers and learners understand specific weaknesses, moving beyond black-box “corrected” text.
- Enables fine-grained system comparison: The framework supports per-type precision and recall as well as analyses along dimensions such as operation (Missing/Unnecessary/Replacement), which is useful for diagnosing system behavior and complementarity.
- Broad applicability claimed: The rule-based classifier depends only on automatic markup (POS, lemmas, etc.), which, if validated, could make it reusable across datasets without retraining.

Weaknesses
- Insufficient system description: The core component is a set of rules, but the paper does not describe them in enough detail. There are no concrete rule examples, no counts, no discussion of rule complexity, ordering, conflict resolution, or coverage. Instead, a full page (Table 6) and extensive text reiterate results that are largely readable from the table, at the expense of crucial methodological details.
- Evaluation design is not convincing:
  - No independent gold standard: Raters assessed the system’s predicted labels as Good/Acceptable/Bad for 200 edits rather than independently assigning gold labels. With a fixed, relatively small tagset, a true gold standard for 200 edits seems feasible. Prior work (e.g., Marcus et al., 1993) suggests that reviewing pre-annotated labels typically yields higher apparent agreement than annotating from scratch.
  - Aggregation of rater judgments: The statement that all five raters individually judged at least 95% as Good/Acceptable is not the appropriate way to report multi-rater outcomes. If each rater’s “Bad” judgments fall on different items (the distribution is not reported), then up to about 18.5% of edits could have been deemed Bad by at least one rater, which is much worse than the reported 3.7% average.
  - Missing test-data details: The paper does not report how many categories appear in the 200-edit sample, nor which categories are represented among those rated Good. Without this, it is unclear how well the evaluation covers the proposed scheme.
- “Unusual” edit boundaries: The paper warns that automatically determined boundaries “might be unusual,” but provides no precise characterization or examples. It is unclear how often this happens, how it affects labeling or matching with references, and whether it harms the tool’s intended use.
- Domain dependence claims: The paper argues the method is less domain dependent than trained classifiers, but this is not fully convincing. For example, Hunspell coverage varies by domain, and manually created rules can encode domain or genre assumptions; the approach is also language dependent, unlike some ML approaches that can be retrained cross-lingually. Moreover, the evaluation datasets (FCE-test, CoNLL-2014) are both student essays, a single domain.
- Rationale for a new error taxonomy is unclear: The paper briefly mentions flexibility (e.g., searching for underspecified categories like NOUN), but existing schemes (e.g., Nicholls, 2003) already support hierarchical searches, and using the CoNLL-2014 scheme would have enabled direct comparison to its gold annotations. The motivation and advantages of the new scheme relative to established ones are not well articulated.
- Focus and motivation: It remains unclear whether the contribution is primarily a new system, a new error taxonomy, or an evaluation of CoNLL-2014 outputs. System details are sparse, the taxonomy is under-motivated, and the CoNLL-2014 analysis duplicates table contents in the text without deeper insights.

Questions to authors
- Rule set: Please provide representative rule examples, the number of rules, their complexity, ordering/priority scheme, conflict resolution, and coverage diagnostics. Could early rules block later ones?
- Evaluation protocol: Why not build a small gold standard for the 200 edits? Please report per-rater distributions over items (e.g., how many edits were Bad for at least one rater). How are edits sampled across categories?
- Edit boundaries: What makes boundaries “unusual”? Please give concrete examples, frequencies, and an analysis of impact on type assignment and evaluation. Are there cases where boundary errors systematically bias category assignment?
- Domain and language dependence: How does Hunspell coverage and POS/dependency parsing accuracy vary across domains, and how does that affect labeling? What is required to adapt the rules to new domains or languages?
- Error taxonomy: Why design a new set instead of adopting Nicholls (2003) or the CoNLL-2014 scheme? What specific limitations of existing taxonomies motivated your design? Could you map your labels to CoNLL/Nicholls to enable comparison?
- Scope of CoNLL-2014 analysis: Beyond restating table entries, what are the main empirical takeaways that could inform system design?

- Strength: Useful application for teachers and learners; enables fine-grained comparison of GEC systems.
- Weakness: System description is superficial and lacks substantive implementation details.
- Missing rule details: No representative rule examples, total rule count, complexity, ordering/prioritization, or conflict-resolution mechanism provided.
- Paper overemphasizes CoNLL-2014 system evaluation (e.g., large tables) instead of core system description.
- Evaluation flaw: No independent gold standard for the 200 test sentences; annotators rated system outputs, likely inflating quality.
- Evaluation flaw: Improper aggregation across raters—reporting individual high rates hides possible per-edit disagreement and underestimates “bad” judgments.
- Evaluation flaw: Missing test-data details (which error categories are present, coverage, and which were judged “good”), preventing assessment of coverage/bias.
- Evaluation flaw: Vague claim about “unusual” edit boundaries—needs a precise definition and examples to assess impact.
- Domain-dependence claim dubious: Hunspell lexicon and manual rules can be domain- and language-specific, and evaluation uses only student-essay domains.
- Tag-set motivation unclear: Existing sets (Nicholls, CoNLL-2014) could support similar queries and allow direct comparison to CoNLL gold standard.
- Overall focus unclear: unclear whether contribution is a system, a new tag set, or an evaluation study.
- Typos/editorial issues: multiple formatting/case errors (e.g., “c.f.”→“cf.”, “M2”→“M^2”, “esl”→“ESL”, “fleiss, kappa”→“Fleiss, Kappa”).

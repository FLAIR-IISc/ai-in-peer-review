- Strengths: Useful application for teachers and learners; supports fine-grained comparison of GEC systems.

- Weaknesses: Superficial system description; unsatisfactory evaluation.

- General discussion:
The paper proposes automatically enriching GEC system outputs with error types. This is valuable: teachers and learners can benefit from explicit error categories (most GEC systems only provide corrections), and the tags enable more granular comparisons of systems—overall precision and error-type–specific precision/recall.

However, the system description is too superficial. The core relies on a set of (manually?) crafted rules, but the paper gives no concrete details. Please provide examples of rules, the total number, their complexity, how conflicts are resolved, their ordering (can an early rule block a later one?), and coverage statistics. Instead of these essentials, several pages focus on re-evaluating CoNLL-2014 systems. For instance, Table 6 occupies a full page, and the text reiterates many values already present in the table.

The evaluation has several shortcomings:
1) The annotators should have independently created a gold standard for the 200 test sentences rather than only rating system output. With a fixed tag set and a small dataset, building a gold standard is feasible. Rating pre-annotated output likely inflates agreement and acceptability relative to annotating from scratch (cf. Marcus et al., 1993).
2) The claim that “all 5 raters individually considered at least 95% of our rule-based error types to be either ‘Good’ or ‘Acceptable’” is not a standard way to summarize multi-rater judgments. One should not aggregate by averaging individual rater acceptability rates. If the “bad” judgments are distributed across different edits (the paper does not report their distribution), then up to 18.5% of edits might have been labeled “bad” by at least one annotator—much worse than the reported 3.7% average. Please report per-edit distributions and inter-annotator agreement using established measures.
3) The test data are insufficiently described. Please report the number and distribution of error categories present in the test set and which categories were covered and rated “good.”
4) The statement that “edit boundaries might be unusual” needs clarification and examples. What makes them unusual? How often does this occur? Could misaligned boundaries harm tagging accuracy or downstream usability?

The paper argues the approach is less domain-dependent than trained systems. This is unclear. Hunspell’s vocabulary varies by domain, hand-written rules can be domain-sensitive, and they are fully language-dependent—a drawback relative to learning-based methods. Moreover, both evaluation datasets (FCE-test, CoNLL-2014) are student essays from a single domain.

The rationale for introducing a new error tag set is also unclear. The stated benefit—supporting queries for underspecified categories (e.g., “NOUN”)—appears feasible with existing schemes (e.g., Nicholls, 2003). Alternatively, adopting the CoNLL-2014 tag set would have enabled direct evaluation against the CoNLL gold standard.

Overall, the paper’s primary contribution is hard to discern. If the goal is a new system, crucial implementation details are missing. If it is a new error taxonomy, motivation, comparison, and analysis are minimal. If it is an evaluation of CoNLL-2014 systems, the presentation remains largely descriptive and redundant with the tables.

- Strengths: The proposed system has a useful application for teachers and learners, enabling fine-grained comparison of Grammatical Error Correction (GEC) systems.
- Weaknesses: The system description is highly superficial, and the evaluation is not satisfying.

General Discussion:
The paper presents an approach to automatically enrich the output of GEC systems with error types, a useful application for both teachers and learners, as it provides explicit information about error types, which many GEC systems lack. This approach also allows for a more precise comparison of GEC systems in terms of precision, recall, and error type-specific figures. However, the system description remains superficial, focusing on a set of manually created rules without providing essential details, such as examples, the number of rules, their complexity, and how they are ordered. 

Instead of presenting relevant system details, the paper devotes several pages to evaluating the systems that participated in CoNLL-2014, including a table (Table 6) that occupies an entire page and lists results for all systems, with the text repeating many facts and figures that can be read off the table.

The evaluation of the proposed system is not satisfying in several aspects. Firstly, instead of having annotators rate the system's output, they should have independently annotated a gold standard for the 200 test sentences. Given a fixed set of tags, it should be possible to produce a gold standard for the relatively small set of test sentences. This approach may yield better ratings for the annotations than a comparison with a real gold standard (Marcus et al., 1993). 

Secondly, the paper states that "all 5 raters individually considered at least 95% of our rule-based error types to be either 'Good' or 'Acceptable'." However, multiple raters' scores should not be considered individually and then averaged; instead, their ratings should be aggregated. If each of the "bad" scores were assigned to different edits (the distribution of which is not provided in the paper), 18.5% of the edits were considered "bad" by some annotator, which sounds worse than the average 3.7% calculated in the paper.

Thirdly, no information is provided about the test data, including the number of error categories it contains or which error categories are covered (according to the categories rated as "good" by the annotators). 

Fourthly, the statement "edit boundaries might be unusual" requires a more precise description and examples. This could be problematic for the application of the system.

The authors claim that their system is less domain-dependent compared to systems that require training data. However, this may not be true, as Hunspell's vocabulary may not cover all domains in the same detail, and manually created rules can be domain-dependent and completely language-dependent, a clear drawback compared to machine learning approaches. Moreover, the test data used (FCE-test, CoNLL-2014) are from one domain only: student essays.

It remains unclear why a new set of error categories was designed. One reason for the tags is given: to enable easy searches for underspecified categories (like "NOUN" in general). However, the tagset presented in Nicholls (2003) seems to support such searches as well. Alternatively, the CoNLL-2014 tagset could have been used, allowing the CoNLL gold standard to be used for evaluation.

To sum up, the main motivation of the paper remains somewhat unclear. Is it about a new system? But the most important details of it are left out. Is it about a new set of error categories? But hardly any motivation or discussion of it is provided. Is it about evaluating the CoNLL-2014 systems? But the presentation of the results remains superficial.

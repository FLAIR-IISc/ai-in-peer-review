- Summary: Proposes automatic annotation of parallel error-correction data with explicit edit spans and error types using linguistically-enhanced alignment and a rule-based classifier.

- Contribution 1: Method to annotate system hypotheses with alignment and error-type info matching the reference for more detailed evaluation.

- Contribution 2: Introduces a dataset-independent rule-based classifier that uses automatically obtained features (POS, lemmas) to classify edits.

- Contribution 3: Performs detailed error-type evaluation of all CoNLL-2014 GEC systems to demonstrate the approach’s value.

- Strength 1: Useful for teachers and learners by making error types explicit rather than only providing corrected text.

- Strength 2: Enables fine-grained comparison of GEC systems with precision, recall, and F-score per error type.

- Strength 3: Proposes a clearer evaluation methodology by separating edit extraction from evaluation and aligning hypothesis and reference annotations.

- Weakness 1: System description is superficial, lacking details on the manually created rules (examples, number, complexity, ordering).

- Weakness 2: Evaluation is insufficient—no gold standard, no per-rater analysis, and unclear test data characteristics.

- Weakness 3: Motivation is unclear regarding whether focus is on the new system, the error categories, or re-evaluation of CoNLL-2014 systems.

- Question 1: Request for more details about the manually created rules (examples, number, complexity, ordering).

- Question 2: How will the authors address evaluation concerns, including creating a gold standard and reporting individual rater agreement?

- Question 3: Why design a new set of error categories instead of using existing tagsets—please clarify motivation.

- Question 4: How will the authors address the system’s domain and language dependence limitations?

- Useful application: enables fine-grained, error-type-aware comparison of GEC systems for teachers and learners.  
- System description superficial: lacks essential details such as examples, number and complexity of rules, and rule ordering.  
- Paper overfocuses on CoNLL-2014 system results, repeating table facts in the text and wasting space.  
- Evaluation design flawed: annotators should have produced an independent gold standard for the 200 test sentences.  
- Rater analysis incorrect: authors averaged individual rater scores instead of aggregating ratings, potentially masking a high proportion of “bad” edits.  
- Missing test-data information: no details on number of error categories or which categories are covered by the test set.  
- Vague claim about edit boundaries: “edit boundaries might be unusual” needs precise description and illustrative examples.  
- Domain-dependence claim questionable: Hunspell and manual rules may be domain- and language-dependent; tests use only student-essay data.  
- Unclear motivation for new error categories: no strong justification given, existing tagsets (Nicholls 2003, CoNLL-2014) could suffice.  
- Main paper goal ambiguous: unclear if contribution is a new system, a new tagset, or an evaluation of CoNLL-2014 systems.  
- Evaluation presentation unsatisfying: key evaluation choices and distributions (e.g., spread of “bad” scores) are not reported.  
- Multiple typos and formatting/reference errors (cf., M^2 notation, percent notation, capitalization, Fleiss kappa, etc.).

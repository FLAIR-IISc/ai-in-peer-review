- Useful application for teachers and learners by enriching GEC outputs with error types.
- Enables fine-grained, error-type-specific comparison of GEC systems (precision and recall).
- System description is highly superficial and omits core implementation details.
- Core rule-based component lacks examples, counts, complexity description, and rule ordering explanation.
- Paper devotes excessive space to CoNLL-2014 system results rather than system details.
- Evaluation design is flawed: annotators rated system output instead of independently creating a gold standard.
- Aggregation of individual rater judgments is misleading; averaging masks per-edit disagreement.
- No information on test data composition (number of error categories covered or distribution).
- Claim about low domain-dependence is unconvincing given reliance on Hunspell and language-specific manual rules.
- New error category set is unjustified; existing tagsets (Nicholls 2003 or CoNLL-2014) could have been used.
- Unclear main motivation: whether the paper aims to present a system, a tagset, or an evaluation study.
- Presentation of CoNLL-2014 results is superficial and repetitive (e.g., large table with repeated text).
- Vague remark about "unusual" edit boundaries needs precise explanation and examples to assess impact.
- Several typographical and formatting issues (e.g., "c.f." → "cf.", "M2" → "M^2", clarify "50-70 F1", incorrect reference casing like "esl", "fleiss", "kappa").

- Strengths: The proposed system has a useful application for teachers and learners, and it supports fine-grained comparison of Grammatical Error Correction (GEC) systems.
- Weaknesses: The description of the system is highly superficial, and the evaluation is not satisfying.

General Discussion:

The paper presents an approach to automatically enriching the output of GEC systems with error types. This is a valuable application, as both teachers and learners can benefit from this information, which is often not provided by GEC systems. Additionally, this approach enables finer-grained comparison of GEC systems in terms of precision, as well as error type-specific figures for recall and precision.

Unfortunately, the description of the system remains superficial. The core of the system consists of a set of manually created rules, but the paper does not provide any details about these rules. To improve the description, the authors should provide examples of such rules, specify the number of rules, explain their complexity, and describe how they are ordered. For instance, can some early rules block the application of later rules? Instead of presenting relevant details of the system, several pages of the paper are devoted to an evaluation of the systems that participated in CoNLL-2014. Table 6, which takes up an entire page, lists results for all systems, and the text repeats many facts and figures that can be read off the table.

The evaluation of the proposed system is not satisfying in several aspects. Firstly, the annotators should have independently annotated a gold standard for the 200 test sentences instead of simply rating the output of the system. Given a fixed set of tags, it should be possible to produce a gold standard for the relatively small set of test sentences. It is highly probable that the approach taken in the paper yields considerably better ratings for the annotations than comparison with a real gold standard (see, e.g., Marcus et al. (1993) for a comparison of agreement when reviewing pre-annotated data vs. annotating from scratch).

Secondly, it is stated that "all 5 raters individually considered at least 95% of our rule-based error types to be either 'Good' or 'Acceptable'." However, multiple raters should not be considered individually and their ratings averaged in this way, as this is not common practice. If each of the "bad" scores were assigned to different edits (the distribution of which is not provided in the paper), 18.5% of the edits were considered "bad" by some annotator – a figure that sounds much worse than the average 3.7% calculated in the paper.

Thirdly, no information about the test data is provided, such as the number of error categories they contain or which error categories are covered (according to the categories rated as "good" by the annotators).

Fourthly, the statement that "edit boundaries might be unusual" requires a more precise description, along with examples. Could this be problematic for the application of the system?

The authors claim that their system is less domain-dependent compared to systems that require training data. However, this claim is not entirely convincing. For instance, Hunspell's vocabulary may not cover all domains in the same detail, and manually created rules can be domain-dependent as well. Moreover, these rules are completely language-dependent, which is a clear drawback compared to machine learning approaches. Furthermore, the test data used in this study (FCE-test, CoNLL-2014) are from one domain only: student essays.

It remains unclear why a new set of error categories was designed. One reason for the tags is given: to enable easy searching for underspecified categories (like "NOUN" in general). However, it seems that the tagset presented in Nicholls (2003) supports such searches as well. Alternatively, the CoNLL-2014 tagset could have been used, allowing the CoNLL gold standard to be used for evaluation.

In summary, the main motivation of the paper remains somewhat unclear. Is it about a new system? However, the most important details of the system are left out. Is it about a new set of error categories? Yet, hardly any motivation or discussion of it is provided. Is it about evaluating the CoNLL-2014 systems? Nevertheless, the presentation of the results remains superficial.

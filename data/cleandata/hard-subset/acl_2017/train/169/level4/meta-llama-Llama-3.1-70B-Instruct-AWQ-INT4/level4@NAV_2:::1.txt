**Summary of the Paper**

The paper proposes a method to automatically annotate parallel error correction data with explicit edit spans and error type information. The approach uses a linguistically-enhanced alignment algorithm to extract edits between parallel original and corrected sentences and a new rule-based classifier to classify these edits into error types. The authors evaluate their approach by applying it to the system output produced in the CoNLL-2014 shared task and demonstrate its value in providing a detailed error type evaluation of the participating systems.

**Main Contributions**

1. A novel approach to automatically annotating error correction data with error type information, which enables a more detailed evaluation of Grammatical Error Correction (GEC) systems.
2. A new rule-based classifier that is entirely dataset-independent and relies only on automatically obtained information such as POS tags and lemmas.
3. A detailed evaluation of system error type performance for the first time for all teams in the CoNLL-2014 shared task on Grammatical Error Correction.

**Strengths**

1. The proposed approach is useful for both teachers and learners, as it provides explicit error type information that can be used to improve language learning and teaching.
2. The approach allows for a finer-grained comparison of GEC systems, enabling researchers to identify the strengths and weaknesses of different systems.

**Weaknesses**

1. The description of the system remains highly superficial, lacking details about the manually created rules, their number, complexity, and ordering.
2. The evaluation of the proposed system is not satisfying, as it relies on annotators rating the output of the system rather than creating a gold standard for the test sentences.
3. The paper lacks information about the test data, including the number of error categories and their distribution.
4. The authors' claim that their system is less domain-dependent than systems that need training data is not entirely convincing, as the manually-created rules and Hunspell's vocabulary may still be domain-dependent.

**Questions to Authors**

1. Can you provide more details about the manually created rules, including their number, complexity, and ordering?
2. Why did you choose to design a new set of error categories instead of using an existing one, such as the CoNLL-2014 tagset?
3. Can you provide more information about the test data, including the number of error categories and their distribution?
4. How do you respond to the criticism that your system may still be domain-dependent, despite not requiring training data?

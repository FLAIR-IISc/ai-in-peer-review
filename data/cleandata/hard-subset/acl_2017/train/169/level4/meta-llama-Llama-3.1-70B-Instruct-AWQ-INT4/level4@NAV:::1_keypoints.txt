- Strength: Offers a useful application for educators and learners, enabling more nuanced comparison of GEC systems.  
- Weakness: System description is overly superficial and lacks crucial implementation details.  
- Weakness: Evaluation methodology is unconvincing and inadequately justified.  
- Contribution: Novel approach to automatically enrich GEC outputs with error types for finer-grained precision/recall analysis.  
- Missing system details: Authors should provide examples, number, complexity, and ordering of the manually created rules.  
- Paper focus: Excessive space spent evaluating CoNLL-2014 systems; Table 6 is repetitive and could be improved.  
- Gold standard issue: Annotators only rated system outputs; independent annotation of the 200 test sentences would be preferable.  
- Rater aggregation flaw: Averaging individual rater scores can mask large discrepancies (18.5% of edits labeled “bad” by some).  
- Data reporting gap: No information on test data error-category count or distribution.  
- Ambiguity: The claim that “edit boundaries might be unusual” needs clarification and examples.  
- Domain-dependence claim questionable: Manually created rules can be domain-dependent and test data is from a single domain (student essays).  
- Tagset concerns: Introduction of a new error category set is unclear and insufficiently motivated; existing tagsets (Nicholls 2003, CoNLL-2014) could have been used.  
- Overall conclusion: Main motivation and presentation are unclear/superficial; authors should detail the system, clarify evaluation, and discuss implications.  
- Minor corrections: Fix “c.f.” -> “cf.”, M2 -> M^2, clarify “50-70 F1”, and correct reference casing (e.g., ESL, Fleiss, kappa).

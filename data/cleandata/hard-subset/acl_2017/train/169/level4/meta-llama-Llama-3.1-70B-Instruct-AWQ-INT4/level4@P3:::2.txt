**Summary of the Paper**

The paper proposes a novel approach for evaluating grammatical error correction (GEC) systems by automatically annotating parallel error correction data with explicit edit spans and error type information. This allows for the assessment of GEC system performance by error type in terms of both precision and recall. The approach uses a linguistically-enhanced alignment algorithm to extract edits between original and corrected sentences and a new rule-based framework to classify these edits into error types.

**Main Contributions**

1. A novel approach for evaluating GEC systems by automatically annotating parallel error correction data with explicit edit spans and error type information.
2. A rule-based framework for classifying edits into error types that is independent of manual error annotation and can assess performance on multi-token errors.
3. A detailed evaluation of system error type performance for the first time for all teams in the CoNLL-2014 shared task on Grammatical Error Correction.

**Strengths**

1. The proposed evaluation is an important stepping stone for analyzing GEC system behavior.
2. The paper includes evaluation for a variety of systems.
3. The approach has several advantages over previous work, including computing precision by error type, being independent of manual error annotation, and assessing performance on multi-token errors.
4. The automatically selected error tags for pre-computed error spans are mostly approved of by human experts.

**Weaknesses**

1. A key part – the rules to derive error types – are not described, making it difficult to understand the basis for the error categories.
2. The classifier evaluation lacks a thorough error analysis and directions for future work on how to improve the classifier.
3. The evaluation was only performed for English, and it is unclear how difficult it would be to use the approach on another language.
4. The error spans computed in the pre-processing step seem to be inherently continuous, which may be problematic for certain error types.

**Questions to Authors**

1. Can you provide more details on the rules used to derive error types and how they were developed?
2. How do you plan to adapt the approach to other languages, and what changes would be necessary to run the classifier on other languages?
3. Can you provide a more thorough error analysis of the classifier evaluation, including directions for future work on how to improve the classifier?
4. How do you plan to address the issue of continuous error spans and their potential impact on certain error types?

**Additional Comments**

1. The authors should consider including raw values for TP, FP, TN, and FN in the appendix for Table 6 to make it easier to compare the systems using other measures.
2. The authors should mention that the finding that different approaches to GEC complement different error types is in line with previous research.
3. The authors should consider providing more interpretation of the multi-token error analysis results, including the potential limitations of the systems evaluated.

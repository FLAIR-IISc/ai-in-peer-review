**Summary of the Paper**

The paper presents a novel approach for evaluating grammatical error correction (GEC) systems by automatically annotating parallel error correction data with explicit edit spans and error type information. This allows for a detailed error type evaluation, which was previously not possible in general since system output is usually not annotated with error categories.

**Main Contributions**

1. A method for automatically annotating parallel error correction data with explicit edit spans and error type information.
2. A detailed error type evaluation of GEC systems using the proposed method.
3. An analysis of the performance of different GEC systems on various error types.

**Strengths**

1. The proposed evaluation is an important stepping stone for analyzing GEC system behavior.
2. The paper includes evaluation for a variety of systems.
3. The approach has several advantages over previous work, including computing precision by error type, independence of manual error annotation, and assessing performance on multi-token errors.
4. The automatically selected error tags for pre-computed error spans are mostly approved of by human experts.

**Weaknesses**

1. A key part – the rules to derive error types – are not described.
2. The classifier evaluation lacks a thorough error analysis and directions for future work on how to improve the classifier.
3. The evaluation was only performed for English, and it is unclear how difficult it would be to use the approach on another language.

**Classifier and Classifier Evaluation**

The paper lacks a clear description of the rules used to derive error types. Although the approach is independent of the alignment algorithm, the rules are probably not, but the authors do not provide details on that. The error categories are a major part of the paper, and the reader should at least get a glimpse of how a rule to assign an error type looks like.

**Gold Reference vs. Auto Reference**

It is unclear on what data the significance test was performed exactly. The comparison between gold and reference is only in terms of boundaries and not in terms of classification.

**Error Type Evaluation**

The authors do not analyze or comment on the results in Table 6 with respect to whether the systems were designed to handle the error type. For some error types, there is a straightforward mapping between error type in the gold standard and in the auto reference. The authors should report raw values for TP, FP, TN, FN in the appendix for Table 6.

**Multi-token Error Analysis**

The result needs more interpretation: some systems are probably inherently unable to correct such errors, but none of the systems were trained on a parallel corpus of learner data and fluent corrections.

**Other**

The authors should have mentioned that for some of the GEC approaches, it was not impossible before to provide error annotations. The proposed approach enables comparing GEC systems for which producing an error-tagged output is not straightforward to other systems in a unified way.

**Questions to Authors**

1. Can you provide a detailed description of the rules used to derive error types?
2. How do you plan to adapt the approach to other languages?
3. Can you provide raw values for TP, FP, TN, FN in the appendix for Table 6?
4. How do you interpret the results of the multi-token error analysis?
5. Can you comment on the limitations of the approach and potential avenues for future work?

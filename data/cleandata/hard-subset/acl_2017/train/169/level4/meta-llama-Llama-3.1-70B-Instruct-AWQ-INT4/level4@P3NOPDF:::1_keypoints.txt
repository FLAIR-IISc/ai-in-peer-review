- Strength: System provides error types plus corrected output, aiding teachers/learners and enabling fine-grained GEC comparisons.  
- Weakness: Paper lacks detailed description of the system and an adequate evaluation.  
- Missing system details: Need examples, number, complexity, and ordering of the manually created rules.  
- Evaluation methodology flaw: Annotators should have independently annotated a gold standard instead of rating system outputs.  
- Misleading reporting: Averaged “good/acceptable” ratings hide that 18.5% of edits were rated “bad” by at least one annotator (vs. reported 3.7%).  
- Test-data transparency: Paper omits how many error categories the test set contains and which are covered.  
- Ambiguous claim about edit boundaries: “Edit boundaries might be unusual” needs precise description and examples.  
- Domain-dependence claim is questionable: Manually created rules can be domain-dependent and evaluations use only student-essay corpora.  
- New error categories unjustified: Authors don’t motivate why a new tagset is needed versus existing ones (e.g., Nicholls 2003 or CoNLL-2014).  
- Unclear main contribution: Paper fails to clarify whether it contributes a new system, a new category set, or an evaluation of CoNLL-2014 systems.  
- Minor corrections: Replace “c.f.” with “cf.”, “M2” with “M^2”, clarify “50-70 F1”, and fix reference casing (ESL, Fleiss, Kappa).

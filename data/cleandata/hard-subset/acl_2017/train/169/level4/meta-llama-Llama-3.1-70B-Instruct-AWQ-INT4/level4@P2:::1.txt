**Summary of the Paper**

The paper proposes an approach to automatically annotate the output of Grammatical Error Correction (GEC) systems with error types. The authors use a linguistically-enhanced alignment algorithm to extract edits between parallel original and corrected sentences, and then classify them using a new dataset-independent rule-based classifier. The approach is applied to the system output produced in the CoNLL-2014 shared task, allowing for a detailed analysis of system error type performance.

**Contributions of the Work**

1. A novel approach to automatically annotate GEC system output with error types, enabling fine-grained comparison of GEC systems.
2. A new dataset-independent rule-based classifier for error type classification, which relies only on automatically obtained information such as POS tags and lemmas.
3. A detailed evaluation of system error type performance for the CoNLL-2014 shared task, providing insights into the strengths and weaknesses of different GEC systems.

**Strengths**

1. The approach has a useful application for teachers and learners, as it provides explicit error type information that can aid in language learning and teaching.
2. The approach allows for finer-grained comparison of GEC systems, enabling researchers to identify areas for improvement.

**Weaknesses**

1. The description of the system remains highly superficial, lacking details about the manually created rules, their complexity, and how they are ordered.
2. The evaluation of the proposed system is not satisfying, with several issues including:
	* The annotators did not independently annotate a gold standard for the test sentences.
	* The ratings from multiple annotators were not combined correctly.
	* No information is provided about the test data, including the number of error categories and which categories are covered.
	* The statement about the system being less domain-dependent is not convincing, as the manually-created rules and Hunspell's vocabulary may still be domain-dependent.
3. The motivation for designing a new set of error categories is unclear, and the paper does not provide sufficient discussion or comparison with existing tagsets.

**Questions to Authors**

1. Can you provide more details about the manually created rules, including examples, complexity, and ordering?
2. How do you plan to address the issues with the evaluation, such as creating a gold standard and combining annotator ratings correctly?
3. Can you provide more information about the test data, including the number of error categories and which categories are covered?
4. How do you respond to the criticism that the system may still be domain-dependent, despite not requiring training data?
5. What is the motivation behind designing a new set of error categories, and how does it compare to existing tagsets?

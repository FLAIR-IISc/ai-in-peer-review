**Summary of the Paper**

The paper proposes a method to automatically annotate parallel error correction data with explicit edit spans and error type information. This approach aims to standardize existing error correction corpora and facilitate detailed error type evaluation. The authors use a linguistically-enhanced alignment algorithm to extract edits between parallel original and corrected sentences and then classify them using a new dataset-independent rule-based classifier.

**Contributions of the Work**

1. **Automatic Annotation and Evaluation of Error Types**: The authors propose a method to automatically annotate system hypothesis corrections with the same alignment and error type information as the reference, enabling a more detailed evaluation.
2. **Dataset-Independent Rule-Based Classifier**: The authors introduce a new rule-based framework to classify edits, which is entirely dataset-independent and relies only on automatically obtained information such as POS tags and lemmas.
3. **Detailed Error Type Evaluation**: The authors demonstrate the value of their approach by carrying out a detailed evaluation of system error type performance for the first time for all teams in the CoNLL-2014 shared task on Grammatical Error Correction.

**Strengths**

1. **Useful Application for Teachers and Learners**: The proposed approach is a very useful application because both teachers and learners can benefit from this information, and many GEC systems only output a corrected version without making the type of error explicit.
2. **Fine-Grained Comparison of GEC Systems**: The approach allows for finer-grained comparison of GEC systems in terms of precision, recall, and F-score for each error type.
3. **Improved Evaluation Methodology**: The authors propose a more straightforward comparison between hypothesis and reference files, treating edit extraction and evaluation as separate tasks.

**Weaknesses**

1. **Superficial Description of the System**: The description of the system remains highly superficial, lacking details about the manually created rules, such as examples, number, complexity, and ordering.
2. **Insufficient Evaluation**: The evaluation of the proposed system is not satisfying, with several concerns, including the lack of a gold standard, individual rater evaluation, and unclear test data characteristics.
3. **Unclear Motivation**: The main motivation of the paper remains somewhat unclear, with unclear emphasis on the new system, error categories, or evaluation of CoNLL-2014 systems.

**Questions to Authors**

1. Can you provide more details about the manually created rules, such as examples, number, complexity, and ordering?
2. How do you plan to address the concerns regarding the evaluation methodology, including the lack of a gold standard and individual rater evaluation?
3. Can you clarify the motivation behind designing a new set of error categories and why the existing tagsets were not used?
4. How do you plan to address the limitations of the proposed system, including its domain dependence and language dependence?

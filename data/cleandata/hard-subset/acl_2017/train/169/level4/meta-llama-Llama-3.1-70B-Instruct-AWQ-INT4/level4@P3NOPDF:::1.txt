**Review of [Paper Title]**

**Strengths:** The proposed system has a useful application for teachers and learners, as it provides error types in addition to corrected output. This information enables fine-grained comparison of Grammatical Error Correction (GEC) systems in terms of precision, recall, and error type-specific figures.

**Weaknesses:** The paper lacks a detailed description of the system, and the evaluation is not satisfying. The authors should provide more information about the system's core components, such as the manually created rules, and improve the evaluation methodology.

**General Discussion:**

The paper presents an approach to automatically enriching the output of GEC systems with error types. This is a valuable application, as it benefits both teachers and learners by providing explicit information about error types. Additionally, it allows for a more detailed comparison of GEC systems.

However, the description of the system remains superficial. The core of the system consists of a set of manually created rules, but the paper does not provide any details about these rules. To improve the description, the authors should provide examples of such rules, specify the number of rules, describe their complexity, and explain how they are ordered. This information is crucial for understanding the system's functionality and potential limitations.

Instead of presenting relevant details about the system, several pages of the paper are devoted to an evaluation of the systems that participated in CoNLL-2014. While the evaluation is extensive, it is not satisfying in several aspects. Firstly, the annotators should have independently annotated a gold standard for the 200 test sentences instead of simply rating the output of the system. This approach would have provided a more reliable evaluation metric.

Secondly, the paper reports that "all 5 raters individually considered at least 95% of our rule-based error types to be either 'Good' or 'Acceptable'". However, this statement is misleading, as it averages the ratings of multiple annotators. A more accurate representation would be to report the percentage of edits that were considered "bad" by at least one annotator. According to the paper, this percentage would be 18.5%, which sounds worse than the reported average of 3.7%.

Thirdly, the paper lacks information about the test data, such as the number of error categories it contains and which categories are covered. This information is essential for understanding the system's performance and potential limitations.

Furthermore, the paper mentions that "edit boundaries might be unusual", but this statement requires a more precise description and examples. It is unclear whether this aspect could be problematic for the application of the system.

The authors claim that their system is less domain-dependent compared to systems that require training data. However, this statement is debatable, as manually created rules can be domain-dependent, and the test data used in this study (FCE-test, CoNLL-2014) are from a single domain: student essays.

Additionally, it is unclear why a new set of error categories was designed. One reason mentioned is to enable easy searching for underspecified categories. However, it seems that existing tagsets, such as the one presented in Nicholls (2003), could support such searches as well. Alternatively, the CoNLL-2014 tagset could have been used, allowing for the use of the CoNLL gold standard for evaluation.

In conclusion, the main motivation of the paper remains unclear. Is it about a new system? But the most important details of it are left out. Is it about a new set of error categories? But hardly any motivation or discussion of it is provided. Is it about evaluating the CoNLL-2014 systems? But the presentation of the results remains superficial.

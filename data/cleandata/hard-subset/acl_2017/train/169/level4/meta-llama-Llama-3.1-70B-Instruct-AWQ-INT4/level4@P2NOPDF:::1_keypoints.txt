- Strength: Useful application for teachers and learners by enriching GEC outputs with explicit error types for finer-grained comparison.
- Strength: Enables precision and error-type specific recall/precision analyses that standard GEC outputs lack.
- Weakness: System description is superficial and omits essential details about core components.
- Weakness: Evaluation is unsatisfying and suffers from methodological concerns.
- Missing system detail: No examples, counts, complexity, or ordering of the manually created rules that form the system's core.
- Paper allocates excessive space to evaluating CoNLL-2014 systems instead of explaining the proposed system.
- Evaluation flaw: Annotators should have independently created a gold standard for the 200 test sentences, not just rated system outputs.
- Evaluation flaw: Averaging individual annotator ratings is unconventional and may obscure the distribution of judgments.
- Missing data detail: Lacks information on the test data, including number and types of error categories present.
- Ambiguity: The claim about unusual "edit boundaries" needs precise description and examples.
- Questionable claim: Assertion that the system is less domain-dependent is dubious since manual rules can be domain-specific and test sets are from student essays only.
- Design concern: New error-category set is poorly motivated and insufficiently discussed; existing tagsets (e.g., CoNLL-2014) would aid comparison.
- Conclusion: Main motivation is unclear and key implementation/evaluation details are omitted, making system effectiveness hard to assess.
- Typos/corrections: Replace "c.f." with "cf.", "M2" with "M^2", clarify "50-70 F1" to "50-70% F1", and correct reference capitalization (e.g., ESL, Fleiss, Kappa).

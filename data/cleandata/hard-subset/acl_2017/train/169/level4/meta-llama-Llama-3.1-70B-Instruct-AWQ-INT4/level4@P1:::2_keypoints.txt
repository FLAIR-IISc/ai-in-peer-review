- Novel evaluation method that measures GEC performance by error type in both precision and recall.  
- Dataset-independent, rule-based edit classification relying only on automatic info (POS tags, lemmas).  
- Detailed error-type performance analysis for all systems in the CoNLL-2014 GEC shared task.  
- Important stepping stone for analyzing GEC system behavior.  
- Demonstrated applicability across a variety of systems.  
- Advantages over prior work: computes precision by error type, removes need for manual annotation, and handles multi-token errors.  
- Automatically assigned error tags for pre-computed spans are mostly approved by human experts.  
- Rules to derive error types are not described, making classifier decisions opaque.  
- Classifier evaluation lacks thorough error analysis and concrete directions for improvement.  
- Evaluation was performed only in English; adaptability to other languages is unclear.  
- Request for detailed description of the rules and how they were developed.  
- Ask how the authors plan to improve the classifier and what the approach’s limitations are.  
- Ask whether cross-lingual adaptation has been considered and what changes would be needed.  
- Request raw TP/FP/TN/FN values (e.g., for Table 6) to enable comparison with other measures.  
- Note that some GEC methods could previously provide error annotations, but this approach enables unified system comparison.  
- References need correct formatting (capitalization, page info).  
- Wrap the URL for Sakaguchi et al. (2016).

**Summary of the Paper**

The paper presents a method for automatically annotating parallel error correction data with explicit edit spans and error type information. The approach uses a linguistically-enhanced alignment algorithm to extract edits between parallel original and corrected sentences and a new rule-based framework to classify these edits into error types. The authors evaluate their approach by applying it to the system output produced in the CoNLL-2014 shared task and demonstrate its value in providing a detailed error type evaluation of the participating systems.

**Contributions of the Work**

1. A novel method for automatically annotating error correction data with error type information, which can be used to standardize existing error correction corpora or facilitate detailed error type evaluation.
2. A new rule-based framework for classifying edits into error types, which is entirely dataset-independent and relies only on automatically obtained information such as POS tags and lemmas.
3. A detailed evaluation of system error type performance for the first time for all teams in the CoNLL-2014 shared task on Grammatical Error Correction, highlighting the strengths and weaknesses of different systems.

**Strengths**

1. The proposed approach has a useful application for teachers and learners, as it provides explicit error type information that can be used to improve language learning and teaching.
2. The approach supports fine-grained comparison of GEC systems, allowing for error type-specific figures for recall and precision.
3. The evaluation of the CoNLL-2014 systems provides valuable insights into the strengths and weaknesses of different approaches to GEC.

**Weaknesses**

1. The description of the system remains highly superficial, with no details provided about the manually created rules, their number, complexity, or ordering.
2. The evaluation of the proposed system is not satisfying, as the annotators only rated the output of the system instead of creating a gold standard for the test sentences.
3. The presentation of the results is repetitive and superficial, with many facts and figures repeated from the tables.
4. The test data used for evaluation is limited to one domain (student essays), and the applicability of the system to other domains is unclear.
5. The motivation for designing a new set of error categories is unclear, and the relationship between the proposed categories and existing ones (e.g., CoNLL-2014 tagset) is not discussed.

**Questions to Authors**

1. Can you provide more details about the manually created rules, such as their number, complexity, and ordering?
2. How do you plan to address the limitations of the evaluation, such as creating a gold standard for the test sentences and using a more diverse set of test data?
3. Can you discuss the relationship between the proposed error categories and existing ones, and provide a clear motivation for designing a new set of categories?
4. How do you plan to address the potential domain dependence of the system, and what steps can be taken to make it more language-independent?

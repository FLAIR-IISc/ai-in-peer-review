- Summary: Proposes automatic annotation of parallel error-correction data with edit spans and error-type labels using linguistically-enhanced alignment and a rule-based classifier, evaluated on CoNLL-2014 system outputs.

- Contribution 1: Introduces a novel method to annotate error-correction data with error types for more detailed GEC evaluation.

- Contribution 2: Presents a dataset-independent, rule-based classifier that uses only automatically obtained info (POS tags, lemmas).

- Contribution 3: Provides a detailed error-type evaluation across all teams in the CoNLL-2014 GEC shared task.

- Strength 1: Produces explicit error-type information useful for language teachers and learners.

- Strength 2: Enables finer-grained comparison of GEC systems to identify system strengths and weaknesses.

- Weakness 1: System description is superficial, lacking details on the manually created rules, their number, complexity, and ordering.

- Weakness 2: Evaluation is weak—relies on annotator ratings of system output rather than a gold-standard annotation.

- Weakness 3: Paper omits key test-data details, including the number of error categories and their distribution.

- Weakness 4: Claim of low domain-dependence is unconvincing due to potential domain bias from manual rules and Hunspell vocabulary.

- Question 1: Please detail the manually created rules—how many, how complex, and in what order are they applied?

- Question 2: Why design a new set of error categories instead of using an existing tagset (e.g., CoNLL-2014)?

- Question 3: Provide full information on the test data, including the number and distribution of error categories.

- Question 4: How do you address concerns that the rule set and Hunspell make the system domain-dependent despite no training data?

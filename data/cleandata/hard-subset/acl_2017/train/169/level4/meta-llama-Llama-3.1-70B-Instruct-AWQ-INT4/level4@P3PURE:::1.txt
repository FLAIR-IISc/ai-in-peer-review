**Strengths:** The proposed system has a useful application for both teachers and learners, as it provides fine-grained comparison of Grammatical Error Correction (GEC) systems. This information can be beneficial for evaluating the performance of GEC systems.

**Weaknesses:** The paper lacks a detailed description of the system, and the evaluation is not satisfactory. The core of the system relies on a set of manually created rules, but the authors do not provide any information about these rules, such as examples, the number of rules, their complexity, or how they are ordered.

**General Discussion:**
The paper presents an approach to automatically enriching the output of GEC systems with error types. This application is valuable, as it allows teachers and learners to understand the types of errors made by GEC systems. Moreover, it enables a more detailed comparison of GEC systems in terms of precision and error type-specific figures for recall and precision.

However, the description of the system is highly superficial. The authors should provide more details about the rules, such as examples, the number of rules, their complexity, and how they are ordered. This information is essential for understanding the system's functionality and limitations.

Instead of presenting relevant details about the system, the authors devote several pages to evaluating the systems that participated in CoNLL-2014. Table 6, which takes up an entire page, lists results for all systems, and the text repeats many facts and figures that can be read off the table.

The evaluation of the proposed system is not satisfying in several aspects. Firstly, the annotators should have independently annotated a gold standard for the 200 test sentences instead of simply rating the output of the system. Given a fixed set of tags, it should be possible to produce a gold standard for the relatively small set of test sentences. It is likely that the approach taken in the paper yields considerably better ratings for the annotations than comparison with a real gold standard (see, e.g., Marcus et al. (1993) for a comparison of agreement when reviewing pre-annotated data vs. annotating from scratch).

Secondly, the paper states that "all 5 raters individually considered at least 95% of our rule-based error types to be either 'Good' or 'Acceptable'." However, multiple raters should not be considered individually, and their ratings should not be averaged in this way. This is not common practice. If each of the "bad" scores were assigned to different edits (the distribution of which is not provided in the paper), 18.5% of the edits were considered "bad" by some annotator, which sounds much worse than the average 3.7% calculated in the paper.

Thirdly, no information is provided about the test data, such as the number of error categories they contain or which error categories are covered (according to the categories rated as "good" by the annotators).

Fourthly, the statement "edit boundaries might be unusual" requires a more precise description and examples. This could be problematic for the application of the system.

The authors claim that their system is less domain-dependent compared to systems that require training data. However, this may not be true. For instance, Hunspell's vocabulary may not cover all domains in the same detail, and manually created rules can be domain-dependent as well. Moreover, the test data used in this study (FCE-test, CoNLL-2014) are from one domain only: student essays.

It remains unclear why a new set of error categories was designed. One reason for the tags is given: to enable easy searching for underspecified categories (like "NOUN" in general). However, the tagset presented in Nicholls (2003) seems to support such searches as well. Alternatively, the CoNLL-2014 tagset could have been used, allowing the CoNLL gold standard to be used for evaluation.

In summary, the main motivation of the paper remains somewhat unclear. Is it about a new system? But the most important details of it are left out. Is it about a new set of error categories? But hardly any motivation or discussion of it is provided. Is it about evaluating the CoNLL-2014 systems? But the presentation of the results remains superficial.

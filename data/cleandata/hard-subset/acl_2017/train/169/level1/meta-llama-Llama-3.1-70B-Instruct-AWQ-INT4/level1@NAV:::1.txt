**Summary**

The paper proposes a method to automatically annotate parallel error correction data with explicit edit spans and error type information. The approach uses a linguistically-enhanced alignment algorithm to extract edits between parallel original and corrected sentences, and a new rule-based framework to classify these edits into 25 error types. The authors evaluate their approach on the CoNLL-2014 shared task data and demonstrate its effectiveness in providing a detailed analysis of system error type performance.

**Main Contributions**

1. **Automatic annotation of error correction data**: The paper proposes a method to automatically annotate parallel error correction data with explicit edit spans and error type information, which can be used to standardize existing error correction corpora or facilitate a detailed error type evaluation.
2. **Rule-based error type classification**: The authors introduce a new rule-based framework to classify edits into 25 error types, which is entirely dataset-independent and relies only on automatically obtained information such as POS tags and lemmas.
3. **Detailed evaluation of system error type performance**: The paper demonstrates the value of the proposed approach by carrying out a detailed evaluation of system error type performance for the first time for all teams in the CoNLL-2014 shared task on Grammatical Error Correction.

**Strengths**

1. **Effective automatic annotation**: The paper shows that the proposed approach can effectively annotate error correction data with high accuracy, with 95% of predicted error types considered "Good" or "Acceptable" by human evaluators.
2. **Detailed evaluation of system performance**: The paper provides a detailed evaluation of system error type performance, which can help researchers understand the strengths and weaknesses of different approaches to Grammatical Error Correction.
3. **Practical application**: The paper demonstrates the practical application of the proposed approach by evaluating the performance of different teams in the CoNLL-2014 shared task.

**Weaknesses**

1. **Limited evaluation**: The paper evaluates the proposed approach on a single dataset (CoNLL-2014), which may not be representative of other datasets or tasks.
2. **Rule-based approach**: The paper's rule-based approach may not be as effective as machine learning approaches for certain error types or datasets.
3. **Limited analysis of multi-token errors**: The paper's analysis of multi-token errors is limited, and more research is needed to understand the challenges and opportunities in this area.

**Questions to Authors**

1. How do you plan to extend the proposed approach to handle more complex error types or datasets?
2. Can you provide more details on the rule-based framework and how it was developed?
3. How do you think the proposed approach can be used in practical applications, such as language learning or writing assistance?

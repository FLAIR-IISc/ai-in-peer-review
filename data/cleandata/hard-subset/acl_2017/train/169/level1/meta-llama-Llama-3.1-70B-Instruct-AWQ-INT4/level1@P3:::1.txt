**Summary of the Paper**

The paper presents a novel approach to automatically annotating and evaluating error types for Grammatical Error Correction (GEC) systems. The authors propose a two-stage method, where the first stage involves automatic edit extraction using a linguistically-enhanced alignment algorithm, and the second stage involves classifying the edits using a new rule-based framework. The approach is evaluated on the CoNLL-2014 shared task dataset, and the results show that the authors' method can effectively evaluate system error type performance at a detailed level.

**Main Contributions**

1. **Automatic Annotation and Evaluation of Error Types**: The authors propose a novel approach to automatically annotating and evaluating error types for GEC systems, which can be used to standardize existing error correction corpora or facilitate a detailed error type evaluation.
2. **Rule-Based Framework for Error Type Classification**: The authors introduce a new rule-based framework for classifying edits, which is entirely dataset-independent and relies only on automatically obtained information such as POS tags and lemmas.
3. **Detailed Evaluation of System Error Type Performance**: The authors demonstrate the value of their approach by carrying out a detailed evaluation of system error type performance for the first time for all teams in the CoNLL-2014 shared task on GEC.

**Strengths**

1. **Novel Approach to Error Type Evaluation**: The authors' approach to error type evaluation is novel and effective, and can be used to standardize existing error correction corpora or facilitate a detailed error type evaluation.
2. **Rule-Based Framework**: The authors' rule-based framework for error type classification is entirely dataset-independent and relies only on automatically obtained information, making it a significant contribution to the field.
3. **Detailed Evaluation**: The authors' detailed evaluation of system error type performance provides valuable insights into the strengths and weaknesses of different GEC systems.

**Weaknesses**

1. **Limited Evaluation**: The authors' evaluation is limited to the CoNLL-2014 shared task dataset, and it is unclear how well their approach would generalize to other datasets.
2. **Rule-Based Framework Limitations**: The authors' rule-based framework for error type classification may not be able to capture all possible error types, and may require additional rules or modifications to handle more complex error types.
3. **Lack of Comparison to Other Methods**: The authors do not compare their approach to other methods for error type evaluation, making it difficult to assess the relative effectiveness of their approach.

**Questions to Authors**

1. How do you plan to extend your approach to handle more complex error types, such as those that involve multiple tokens or more nuanced linguistic phenomena?
2. Can you provide more details on the evaluation metrics used to assess the effectiveness of your approach, and how they were calculated?
3. Have you considered applying your approach to other datasets or domains, and if so, what were the results?

**Additional Comments**

The authors' approach to error type evaluation is a significant contribution to the field, and their detailed evaluation of system error type performance provides valuable insights into the strengths and weaknesses of different GEC systems. However, the authors could benefit from providing more details on the evaluation metrics used to assess the effectiveness of their approach, as well as considering the limitations of their rule-based framework and how it might be extended to handle more complex error types.

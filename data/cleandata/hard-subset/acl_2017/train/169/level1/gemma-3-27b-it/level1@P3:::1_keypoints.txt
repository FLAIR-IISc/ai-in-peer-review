- Presents a method to automatically annotate GEC system outputs with error types, enabling granular evaluation.
- Combines linguistically-enhanced alignment (Felice et al., 2016) with a dataset-independent rule-based error classification framework.
- Applies the approach to large-scale evaluation of CoNLL-2014 systems, revealing per-error-type strengths and weaknesses.
- Releases an annotation tool to support reproducible, detailed GEC analyses for the research community.

- Addresses the critical gap of lacking annotated system outputs for fine-grained GEC evaluation.
- Proposes a robust, dataset-independent rule-based classifier relying on POS tags and lemmas with >95% “Good”/“Acceptable” agreement.
- Shows that rule-based error typing is effective once edit boundaries are determined.
- Provides a comprehensive CoNLL-2014 error-type breakdown, highlighting complementary strengths of different system approaches and hard error types (e.g., ADV, CONJ).

- Pipeline depends heavily on the quality of initial edit extraction (Felice et al., 2016), and the impact of imperfect boundaries is insufficiently analyzed.
- Classifier evaluation is limited (200 edits, five raters); broader, multi-rater evaluation on diverse data is needed to confirm generalizability.
- Comparison with the M2 scorer is underdeveloped; causes of M2’s score inflation and specific divergence cases are not deeply explored.
- Lacks detailed qualitative error analysis showing concrete examples of classifier successes and failures.

- How sensitive is error-type classification to errors in the edit extraction phase; any experiments quantifying this impact?
- Can you provide concrete examples and a detailed explanation of discrepancies between your approach and the M2 scorer?
- How does the rule-based classifier compare to machine-learning-based error-typing methods on public datasets, despite domain adaptation concerns?
- Are there plans to extend the classifier to handle more complex error types or add linguistic features beyond POS tags and lemmas?

- Table 2 needs more detailed descriptions for less common error categories.
- Citation style is inconsistent and should be standardized.
- Discuss limitations of rule-based approaches for nuanced or context-dependent error types.

- Summary: Proposes an automatic error-type annotation method for GEC using a linguistically-enhanced alignment plus a rule-based classifier that uses only automatically obtained linguistic properties; validated by human evaluation (>95% agreement) and applied to CoNLL-2014.

- Contribution 1: Dataset-independent, rule-based error-typing framework that requires no labeled training data and uses only automatic linguistic features.

- Contribution 2: Provides a detailed 25-category error-type analysis of CoNLL-2014 systems, revealing system-specific strengths and weaknesses.

- Contribution 3: Delivers a practical automatic annotation tool to standardize error-type evaluation, to be released with the paper.

- Strength 1: Addresses a long-standing GEC evaluation gap by enabling automatic error-type analysis without manual annotation.

- Strength 2: Dataset independence is a clear advantage over ML-based approaches that need domain-specific training.

- Strength 3: Strong human validation with over 95% agreement supports the method’s reliability.

- Strength 4: The fine-grained analysis yields actionable insights (e.g., MT systems handle NOUN errors well; classifiers handle VERB:FORM better).

- Weakness 1: Evaluation is confined to the outdated CoNLL-2014 dataset, leaving applicability to modern neural GEC systems untested.

- Weakness 2: Lacks discussion on handling error types outside the 25-category framework or extending it to new error types.

- Weakness 3: Insufficient comparison with existing error-annotation methods; unclear quantitative improvement over prior work.

- Weakness 4: Does not address limitations of the rule-based approach, such as ambiguous errors or multi-category spanning edits.

- Question 1: How does the method handle complex errors that involve multiple categories or don’t fit the 25-category scheme?

- Question 2: Have you tested or considered adapting the approach for modern neural GEC systems with different error distributions?



# Review of "Automatic Annotation and Evaluation of Error Types for Grammatical Error Correction"

## Summary and Contributions
This paper presents a method to automatically annotate grammatical error correction (GEC) system output with error type information, overcoming the previous limitation that error type performance could only be measured in terms of recall. The authors propose a two-step approach: (1) linguistically-enhanced alignment to extract edits between original and corrected sentences, and (2) a rule-based classifier that categorizes these edits using only automatically obtained properties (POS tags, lemmas, etc.). The main contributions as I see them are:

1. A novel method for automatic error type annotation of GEC system output, enabling detailed error type analysis that was previously impossible
2. A dataset-independent rule-based classifier that relies solely on automatically obtained properties, avoiding the need for labeled training data
3. A comprehensive error type analysis of GEC systems in the CoNLL-2014 shared task, revealing system strengths and weaknesses across different error types

## Strengths

1. **Addressing a Critical Gap in GEC Evaluation**: The paper tackles a genuine limitation in GEC research—previous inability to analyze error type performance of systems because their output wasn't annotated. This is a significant contribution to the field, as understanding system strengths and weaknesses across error types is crucial for guiding future development. The authors effectively demonstrate the value of their approach by applying it to the CoNLL-2014 shared task data, revealing which systems excel at which error types.

2. **Rigorous Validation of the Classifier**: The authors conducted a small-scale manual evaluation where 5 GEC experts rated 200 randomly selected edits. The high agreement (95%+ rated as "Good" or "Acceptable") despite the noise from automatic edit extraction provides strong evidence for the validity of their rule-based approach. This is particularly impressive given that their classifier is dataset-independent and doesn't require labeled training data.

3. **Insightful Analysis of System Performance**: The detailed error type analysis across CoNLL-2014 teams reveals important insights about system performance. For example, they show that CAMB excelled at most error types, while CUUI performed better at Replacement Determiner errors than CAMB. This granular analysis is valuable for understanding complementary strengths across different approaches, which could guide future system development.

4. **Practical Implementation and Release Plan**: The authors plan to release the tool with the paper, making their contribution immediately useful to the community. This practical implementation significantly enhances the paper's impact and utility.

5. **Thoughtful Comparison with M2 Scorer**: The paper provides a clear explanation of why the M2 scorer tends to overestimate performance (by treating multiple edits as one true positive), which helps explain discrepancies between different evaluation metrics. This insight is valuable for the GEC community.

## Weaknesses

1. **Limited Scope of Evaluation**: The paper only applies the method to the CoNLL-2014 shared task data. While this is a strong demonstration, the method's generalizability across different datasets, error types, or languages isn't thoroughly evaluated. A more comprehensive evaluation would strengthen the paper's claims about the method's robustness.

2. **Insufficient Comparison with State-of-the-Art**: While the paper mentions previous work, it doesn't provide a direct comparison of their rule-based classifier with machine learning approaches on the same datasets. The claim that their approach "is in contrast with machine learning approaches" would be more convincing with such a comparison.

3. **Lack of Statistical Significance Testing**: The error type analysis shows performance differences across systems, but the authors don't provide statistical significance tests for these differences. Without this, it's difficult to determine whether the observed differences are meaningful or due to chance.

4. **Rule-Based Classifier Limitations Not Fully Explored**: While the paper acknowledges limitations for error types not well-characterized by POS tags (e.g., Word Order), it doesn't provide a systematic analysis of where the classifier fails or how it could be improved for such cases. This limits the understanding of the method's boundaries.

5. **Limited Discussion of Integration with Recent GEC Advances**: The paper doesn't address how their evaluation method could be applied to recent neural GEC systems, which have become dominant in the field since 2014. This omission limits the paper's relevance to current research.

## Questions to Authors

1. Could you provide more details on how your approach would handle error types that are not well-characterized by POS tags (e.g., Word Order errors), and what limitations exist in these cases?

2. Have you considered applying your method to more recent GEC datasets or systems (e.g., those from CoNLL-2015 or 2016 shared tasks), and how well does it generalize to these?

3. Could you provide statistical significance tests for the error type performance differences across systems to better support your claims about which systems excel at which error types?

## Overall Assessment

This is a strong paper that addresses a genuine limitation in GEC evaluation. The method is novel, well-validated, and provides valuable insights into system performance. While the evaluation scope is somewhat limited, the paper makes a meaningful contribution to the field by enabling detailed error type analysis for the first time. The practical implementation and release plan further enhance its impact. With minor improvements to the evaluation scope and statistical analysis, this paper would be an excellent contribution to ACL 2017. I recommend acceptance with minor revisions.

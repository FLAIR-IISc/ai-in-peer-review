

Summary: This paper addresses a critical gap in Grammatical Error Correction (GEC) evaluation by introducing a method to automatically annotate unannotated system output with error type information. The main contributions are: (1) a linguistically-enhanced alignment algorithm for edit extraction, (2) a dataset-independent rule-based classifier for error type classification that relies solely on automatically obtained linguistic features, and (3) a detailed error type analysis of systems from the CoNLL-2014 shared task, which was previously impossible due to lack of annotated system output.

Strengths:
1. The paper solves a significant, long-standing problem in GEC evaluation: the inability to perform detailed error type analysis on system output because it lacks error type annotations. This is a clear and important contribution to the field, as it enables researchers to understand system strengths and weaknesses at granular error type levels.
2. The proposed rule-based classifier is both simple and effective, validated through human evaluation where 95% of predicted error types were rated "Good" or "Acceptable" by experts. This demonstrates the practical utility of the approach without requiring labeled training data.
3. The application to the CoNLL-2014 shared task provides valuable insights into system performance across different error types, revealing complementary strengths of various approaches (e.g., MT systems excelling at noun errors while classifiers performed better on orthography errors). This analysis could directly inform future system development.

Weaknesses:
1. The paper does not sufficiently address how errors in the automatic edit extraction process might propagate to error type classification. While the human evaluation notes that some "Bad" ratings were due to POS tagger errors, it doesn't quantify how often this occurs or how it affects the overall error type analysis, which is critical for understanding the reliability of the results.
2. The claim of "dataset-independence" is not thoroughly validated. The evaluation is limited to CoNLL-2014 and FCE datasets, and it's unclear how well the approach would generalize to other error types or languages, particularly for error categories that don't map cleanly to POS tags (e.g., complex tense errors).
3. The paper lacks a comparative analysis between the rule-based approach and machine learning-based error type classifiers. While previous work (e.g., Felice et al., 2016) is mentioned, the paper doesn't provide evidence that the rule-based approach outperforms existing methods in terms of accuracy or consistency.

Questions to Authors:
1. How does the approach handle error types that don't map cleanly to POS tags (e.g., complex tense errors like "has eaten" → "should eat"), and what are the limitations of the current error scheme in capturing such nuances?
2. Could the authors provide more detail on how the automatic edit extraction boundaries might affect error type classification, particularly for multi-token edits where boundaries are ambiguous?
3. The paper mentions that the approach is dataset-independent, but it would be valuable to know if there are any error types or linguistic phenomena where the classifier performs significantly worse, and how these might be addressed in future work.

Additional Comments:
The paper's modular error type framework is well-designed, allowing for flexible analysis at different levels of granularity. The detailed error type analysis of the CoNLL-2014 systems reveals interesting insights about system strengths and weaknesses, which could guide future research. However, the paper would benefit from a more thorough discussion of the limitations of the error type framework, particularly regarding how it handles complex errors that may involve multiple error types simultaneously. The claim that the approach "bypasses bias effects altogether" is somewhat overstated, as the classifier's performance is still dependent on the accuracy of the underlying linguistic tools (POS tagger, lemmatizer, etc.).

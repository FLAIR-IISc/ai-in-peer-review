**Summary of the Paper**

The paper proposes a method to automatically annotate parallel error correction data with explicit edit spans and error type information. This method consists of two main steps: edit extraction using a linguistically-enhanced alignment algorithm and error typing using a new rule-based framework. The framework is dataset-independent and relies only on automatically obtained information such as POS tags and lemmas. The authors evaluate their approach by applying it to the system output produced in the CoNLL-2014 shared task and demonstrate its value in providing a detailed error type evaluation.

**Main Contributions**

1. **Automatic Annotation Method**: The paper proposes a novel method for automatic annotation of error correction data, which can be used to standardize existing error correction corpora or facilitate a detailed error type evaluation.
2. **Dataset-Independent Error Typing Framework**: The paper introduces a new rule-based framework for error typing, which is entirely dataset-independent and relies only on automatically obtained information such as POS tags and lemmas.
3. **Detailed Error Type Evaluation**: The paper demonstrates the value of the proposed approach by carrying out a detailed evaluation of system error type performance for the first time for all teams in the CoNLL-2014 shared task on Grammatical Error Correction.

**Strengths**

1. **Novel Approach**: The paper proposes a novel approach for automatic annotation of error correction data, which is different from existing methods.
2. **Detailed Evaluation**: The paper provides a detailed evaluation of system error type performance, which is not possible with existing methods.
3. **Dataset-Independent Framework**: The paper introduces a dataset-independent error typing framework, which can be applied to different datasets without requiring labelled training data.

**Weaknesses**

1. **Limited Evaluation**: The paper evaluates the proposed approach only on the CoNLL-2014 shared task data, which may not be representative of all error correction datasets.
2. **Rule-Based Framework**: The paper uses a rule-based framework for error typing, which may not be as effective as machine learning approaches in some cases.
3. **Dependence on POS Tagging**: The paper's approach relies heavily on POS tagging, which may not always be accurate.

**Questions to Authors**

1. How do you plan to evaluate the proposed approach on other error correction datasets?
2. Can you provide more details on the rule-based framework for error typing, including the rules and the decision-making process?
3. How do you plan to address the limitations of the rule-based framework, such as its dependence on POS tagging?

**Additional Comments**

The paper proposes a novel approach for automatic annotation of error correction data and provides a detailed evaluation of system error type performance. However, the evaluation is limited to the CoNLL-2014 shared task data, and the approach relies heavily on POS tagging. The authors should consider evaluating the approach on other datasets and addressing the limitations of the rule-based framework.

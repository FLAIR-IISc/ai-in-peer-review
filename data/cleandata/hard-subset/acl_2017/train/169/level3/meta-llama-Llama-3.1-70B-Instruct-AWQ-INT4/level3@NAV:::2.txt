**Summary of the Paper**

The paper proposes a novel approach to automatic annotation and evaluation of error types for Grammatical Error Correction (GEC) systems. The authors introduce a linguistically-enhanced alignment algorithm to extract edits between parallel original and corrected sentences, and a new rule-based classifier to assign error types to these edits. The approach is independent of manual error annotation and can handle multi-token errors. The authors demonstrate the effectiveness of their approach by applying it to the CoNLL-2014 shared task data and providing a detailed analysis of system error type performance.

**Main Contributions**

1. **Novel evaluation method**: The paper proposes a new evaluation method that enables measuring GEC performance by error type, including precision, not just recall.
2. **Automatic annotation**: The authors introduce a rule-based classifier that can automatically annotate system hypothesis corrections with error type information, without requiring manual annotation.
3. **Detailed error analysis**: The paper provides a detailed analysis of system error type performance for the CoNLL-2014 shared task data, highlighting the strengths and weaknesses of different systems.

**Strengths**

1. **Effective evaluation method**: The proposed evaluation method is effective in measuring GEC performance by error type, providing a more detailed analysis of system performance.
2. **Automatic annotation**: The rule-based classifier is able to automatically annotate system hypothesis corrections with error type information, reducing the need for manual annotation.
3. **Detailed error analysis**: The paper provides a detailed analysis of system error type performance, highlighting the strengths and weaknesses of different systems.

**Weaknesses**

1. **Lack of transparency**: The rules used to derive error types are not described, threatening reproducibility.
2. **Limited evaluation**: The evaluation is performed only on English, and the paper does not address adapting the approach to other languages.
3. **Insufficient error analysis**: The classifier evaluation lacks detailed error analysis and concrete directions for improvement.

**Questions to Authors**

1. Can you provide more details on the rules used to derive error types and how they were developed?
2. How do you plan to adapt the approach to other languages?
3. Can you provide more detailed error analysis and concrete directions for improvement in the classifier evaluation?

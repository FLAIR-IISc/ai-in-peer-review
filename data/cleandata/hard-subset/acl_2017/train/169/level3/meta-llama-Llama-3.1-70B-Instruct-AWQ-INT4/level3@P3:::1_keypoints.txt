- Proposes automatic annotation of parallel GEC data with explicit edit spans and error types via linguistically-enhanced alignment plus a rule-based classifier; tool to be released.  
- Main contributions: a dataset-independent rule-based error-type classifier and its application to CoNLL-2014 system outputs for finer-grained evaluation.  
- Claims usefulness for teachers and learners by enabling fine-grained comparison of GEC systems.  
- Strength: dataset-independent—no labelled training data required.  
- Strength: fully deterministic—reduces bias and increases consistency.  
- Strength: provides a detailed evaluation including a small-scale manual check and system-level error analyses.  
- Weakness: system description is superficial—lacks examples, counts, complexity measures, and ordering/details of rules.  
- Weakness: overemphasizes CoNLL-2014 results, reiterating table facts without added insight.  
- Weakness: evaluation relies on raters judging system output rather than creating an independent gold standard for the 200 test sentences.  
- Weakness: aggregation of rater scores is misleading—averages individual percentages without reporting distribution of “bad” ratings.  
- Weakness: no information on test data composition, number of error categories, or which categories are covered.  
- Unsubstantiated claims about lower domain dependence; no clear definition or analysis of “unusual” edit boundaries or their impact.  
- No justification for introducing a new error-tagset instead of using existing tagsets (e.g., Nicholls, CoNLL-2014).  
- Overall critique: motivation unclear, important details omitted, and contributions not clearly explained or validated.  
- Requested clarifications: provide full system details (examples, counts, complexity, ordering), justify the new tagset, define and analyze “unusual edit boundaries,” address evaluation and aggregation limitations, and report test data composition and category coverage.

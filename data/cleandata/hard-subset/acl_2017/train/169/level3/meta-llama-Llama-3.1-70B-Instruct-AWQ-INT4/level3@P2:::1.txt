**Summary of the Paper**

This paper proposes a method to automatically annotate parallel error correction data with explicit edit spans and error type information. The authors aim to rectify the limitation of existing Grammatical Error Correction (GEC) systems, which can only be evaluated in terms of recall due to the lack of annotated system output. The proposed approach consists of two main steps: edit extraction using a linguistically-enhanced alignment algorithm and error type classification using a new rule-based framework. The authors evaluate their approach by applying it to the system output produced in the CoNLL-2014 shared task and demonstrate its effectiveness in providing a detailed analysis of system error type performance.

**Contributions**

1. **Automatic Annotation of Error Correction Data**: The authors propose a method to automatically annotate parallel error correction data with explicit edit spans and error type information, enabling a more detailed evaluation of GEC systems.
2. **Dataset-Independent Error Type Classification**: The authors introduce a new rule-based framework for error type classification that is entirely dataset-independent and relies only on automatically obtained information such as POS tags and lemmas.
3. **Detailed Analysis of System Error Type Performance**: The authors demonstrate the value of their approach by carrying out a detailed evaluation of system error type performance for the first time for all teams in the CoNLL-2014 shared task on Grammatical Error Correction.

**Strengths**

1. **Useful Application for Teachers and Learners**: The proposed approach enables fine-grained comparison of GEC systems, which can be useful for teachers and learners to identify the strengths and weaknesses of different systems.
2. **Detailed Analysis of System Error Type Performance**: The authors provide a detailed analysis of system error type performance, which can help researchers to further improve general performance.
3. **Dataset-Independent Error Type Classification**: The authors' approach to error type classification is entirely dataset-independent, which can be beneficial for evaluating GEC systems across different datasets.

**Weaknesses**

1. **Superficial System Description**: The system description is highly superficial, lacking examples, counts, complexity, and ordering details of the core rule set.
2. **Overemphasis on CoNLL-2014 System Results**: The paper overemphasizes the CoNLL-2014 system results, repeating facts that are visible from the table, and does not provide enough analysis of the results.
3. **Evaluation Methodology**: The evaluation used raters to judge system output rather than creating an independent gold standard for the 200 test sentences, which may lead to biased results.
4. **Aggregation of Rater Scores**: The aggregation of rater scores is misleading, as averaging individual rater percentages may not accurately reflect the performance of the system.
5. **Lack of Information about Test Data Composition**: The paper does not provide information about the test data composition, including the number of error categories or which categories are covered.
6. **Vague Claims**: The paper makes vague claims, such as the potential impact of unusual edit boundaries, which need precise definition and examples.
7. **Unclear Motivation**: The overall motivation of the paper is unclear, and important details are omitted, making it difficult to understand the contribution of the paper.

**Questions to Authors**

1. Can you provide more details about the core rule set, including examples, counts, complexity, and ordering details?
2. How do you plan to address the issue of unusual edit boundaries and their potential impact on the evaluation results?
3. Can you provide more information about the test data composition, including the number of error categories or which categories are covered?
4. How do you justify the use of a new error-tagset instead of existing tagsets, such as Nicholls or CoNLL-2014?
5. Can you provide more analysis of the CoNLL-2014 system results, including a discussion of the strengths and weaknesses of different systems?

**Summary of the Paper**

This paper presents a method for automatically annotating parallel error correction data with explicit edit spans and error type information. The approach uses a linguistically-enhanced alignment algorithm to extract edits between original and corrected sentences, and a new rule-based framework to classify these edits into 25 error types. The authors evaluate their approach by applying it to the system output produced in the CoNLL-2014 shared task on Grammatical Error Correction (GEC). They demonstrate the value of their approach by carrying out a detailed evaluation of system error type performance for the first time for all teams in the CoNLL-2014 shared task.

**Main Contributions**

1. A new method for automatically annotating parallel error correction data with explicit edit spans and error type information.
2. A detailed evaluation of system error type performance for the first time for all teams in the CoNLL-2014 shared task on GEC.
3. A new rule-based framework for classifying edits into 25 error types, which is entirely dataset independent and relies only on automatically obtained information such as POS tags and lemmas.

**Strengths**

1. The paper presents a useful application for teachers and learners, enabling fine-grained comparison of GEC systems.
2. The approach is entirely dataset independent and does not require labelled training data.
3. The rule-based framework is transparent and deterministic, bypassing bias effects altogether.

**Weaknesses**

1. The system description is highly superficial, lacking examples, counts, complexity, and ordering details.
2. The paper overemphasizes CoNLL-2014 system results and repeats facts that are visible from the table.
3. The evaluation used raters to judge system output rather than creating an independent gold standard for the 200 test sentences.
4. The aggregation of rater scores is misleading, and the distribution of “bad” ratings is not reported.
5. There is no information about test data composition, including the number of error categories or which categories are covered.
6. The claim that “edit boundaries might be unusual” is vague and lacks precise definition and examples.
7. The claim of lower domain dependence is unsubstantiated, and manual rules and Hunspell vocabularies can be domain- and language-dependent.

**Questions to Authors**

1. Can you provide more details about the system description, including examples, counts, complexity, and ordering details?
2. How did you select the 200 test sentences for evaluation, and why did you not create an independent gold standard?
3. Can you provide more information about the test data composition, including the number of error categories and which categories are covered?
4. Can you provide a precise definition and examples of “unusual” edit boundaries, and discuss their potential impact?
5. How do you plan to address the limitations of your approach, including the lack of labelled training data and the potential for domain and language dependence?

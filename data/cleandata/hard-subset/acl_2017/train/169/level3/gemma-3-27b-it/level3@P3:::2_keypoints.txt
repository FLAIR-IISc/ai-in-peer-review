- Proposes an automatic method using linguistically-enhanced alignment and a rule-based classifier to annotate grammatical errors and evaluate GEC systems by error type, enabling precision-based evaluation.

- Novel evaluation framework enabling automatic error-type annotation and precision assessment previously unavailable.

- High agreement (>95% “Good” or “Acceptable”) with human judgments suggests robust automatic tagging.

- Broad applicability demonstrated by comprehensive error-type analysis of all CoNLL-2014 systems.

- Dataset-independence: rule-based approach uses automatic markup (POS, lemmas) and avoids biases from dataset-trained classifiers.

- Rules for deriving error types are not described, preventing reproducibility.

- Classifier evaluation lacks detailed error analysis and does not identify where or why the system fails.

- Evaluation is limited to English; generalizability to languages with different morphology/word order is unproven.

- No justification or connection to existing error category typologies is provided.

- Tag assignment likely sensitive to alignment and preprocessing, but this dependence is not analyzed.

- Method may struggle with discontinuous edits (e.g., separable verb prefixes) and similar constructions.

- Human rating of automatic tags may be biased by automatically determined edit boundaries and lacks intermediate rating options or comparison to manual boundaries.

- Request a detailed, formalized description of the classification rules for reproducibility.

- Request a deeper error analysis identifying consistently misclassified errors and causes.

- Ask what modifications and linguistic challenges would be needed to adapt the approach to other languages.

- Ask for analysis of sensitivity to alignment/preprocessing variations and illustrative examples.

- Request TP/FP/FN counts per error type and system to allow thorough assessment and alternative metrics.

- Significance testing setup is unclear (e.g., whether F0.5 was tested).

- Comparison between gold and auto references is limited to boundaries; classification accuracy of auto-reference is not clearly reported.

- Paper should note that some GEC systems already include error-tagging functionality.

- Minor issues: inconsistent reference formatting and missing URLs/page numbers.

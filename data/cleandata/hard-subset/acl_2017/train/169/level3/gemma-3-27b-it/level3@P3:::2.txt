## Review

This paper presents a method for automatically annotating grammatical errors in text and using these annotations to evaluate GEC systems by error type. The core idea – leveraging linguistically-enhanced alignment and a rule-based classifier – is a valuable contribution, addressing a known limitation in GEC evaluation where detailed error type analysis is hampered by the lack of annotated system outputs. The ability to move beyond recall-based evaluation and assess precision by error type is a significant step forward. Furthermore, the high agreement rate between human raters and the automatic error tagging ( >95% “Good” or “Acceptable”) suggests the approach is reasonably robust. The comprehensive CoNLL-2014 analysis, providing error type performance for various systems, demonstrates broad applicability.

**Strengths:**

1. **Novel Evaluation Framework:** The primary strength lies in the creation of a methodology to automatically annotate GEC system outputs with error types. This addresses a critical gap in the field, enabling precision-based evaluation alongside recall, something previously unavailable.
2. **High Agreement with Human Judgments:** The strong agreement ( >95%) between the automatic error tagging and human raters lends credibility to the approach and suggests the rule-based classifier is capturing meaningful linguistic distinctions. This builds confidence in the evaluation results.
3. **Broad Applicability & Comprehensive Analysis:** Applying the method to all systems from the CoNLL-2014 shared task and providing detailed error-type performance for each demonstrates its practical utility and generates a rich dataset for analysis. The granular error breakdown is a valuable resource for the GEC community.
4. **Dataset Independence:** The rule-based approach, relying on automatic markup (POS tags, lemmas), is a strength as it avoids the biases inherent in classifiers trained on specific, potentially idiosyncratic, annotated datasets.



**Weaknesses:**

1. **Lack of Reproducibility:** The most significant weakness is the absence of a clear description of the rules used to derive the error types. Without this information, reproducing the results and building upon this work is impossible. The method’s validity hinges on the precision of these rules, which remain opaque.
2. **Insufficient Classifier Evaluation:** The classifier evaluation, while showing high agreement, lacks detailed error analysis. Simply stating the percentage of "Good" and "Acceptable" ratings doesn't identify *where* the system fails or *why*. A systematic breakdown of errors flagged as "Bad" (and a comparison to the actual errors) is crucial for improving the classifier.
3. **Limited Language Scope:** The evaluation is restricted to English. The paper doesn’t address the challenges of adapting this approach to other languages with different linguistic structures (e.g., morphology, word order) – a crucial consideration for broader impact. The method’s generalizability is therefore unproven.
4. **Error Category Definition & Prior Work:** The paper lacks justification for the chosen error categories. It’s unclear whether these categories build on existing typologies of grammatical errors or are entirely novel.  A discussion of related work on error categorization is needed.
5. **Alignment/Preprocessing Dependence:** The rules for assigning error tags are likely sensitive to the alignment and preprocessing steps. The paper doesn't detail this dependence, leaving open the question of how robust the tagging is to variations in these steps.
6. **Discontinuous Edits:** The method may struggle with languages (or constructions within English) that involve discontinuous edit spans (e.g., separable verb prefixes in German). This limitation should be acknowledged and discussed.
7. **Human Rating Bias:** The evaluation of automatic tags might be biased by potentially inaccurate edit boundaries automatically determined. The absence of intermediate rating options and the lack of a comparison with ratings using manually defined boundaries raises concerns about the validity of the human evaluation.



**Questions to Authors:**

1. **Please provide a detailed, formalized description of the rules used to classify edits into error types.** This is crucial for reproducibility and understanding the basis of the evaluation.
2. **Can you provide a more detailed error analysis of the classifier evaluation?** Specifically, what types of errors were consistently misclassified as "Bad," and what factors contributed to these misclassifications?
3. **What modifications would be necessary to adapt this approach to other languages?**  What are the key linguistic challenges that would need to be addressed?
4. **How sensitive are the error tag assignments to variations in the alignment and preprocessing steps?**  Can you provide some analysis or examples?
5. **What is the breakdown of TP/FP/FN counts for each error type and system?** Providing these raw counts would allow for a more thorough assessment of the results, including sample sizes and the calculation of alternative metrics.




**Additional Comments:**

* The significance test setup is unclear. It’s not stated whether F0.5 was specifically tested for significance.
* The comparison between gold and auto references seems limited to boundaries; the classification accuracy of the auto-reference is not clearly stated.
* The paper should acknowledge that some GEC systems already incorporate error tagging functionalities.
* Minor issues: References are not consistently formatted, and some URLs/page numbers are missing.

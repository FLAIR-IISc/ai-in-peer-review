

This paper proposes an automatic method to annotate GEC system outputs with error type information, enabling detailed error type analysis. The authors use a linguistically-enhanced alignment to extract edits between original and corrected sentences, then classify them using a new rule-based framework. They apply this to CoNLL-2014 system outputs for a detailed error type analysis.

The primary contribution is the method for automatic error type annotation, which enables fine-grained comparison of GEC systems - a genuinely useful application for teachers and learners. The paper also provides a new error type classification framework that is dataset-independent, though its novelty is questionable.

Strengths: The method addresses a real gap in GEC evaluation, as error type performance could previously only be measured in terms of recall. The small-scale human evaluation (with 5 raters) showing >95% "Good" or "Acceptable" ratings for predicted error types is promising. The tool will be released with the paper, adding practical value.

Weaknesses: The system description is superficial - the core rule set lacks examples, counts, complexity details, and ordering rules, making reproduction impossible. The evaluation methodology is flawed: using raters to judge system output instead of creating an independent gold standard, and aggregating scores by averaging percentages without reporting the distribution of "bad" ratings. The paper overemphasizes CoNLL-2014 results with repetitive tables while omitting critical test data details (error categories covered, etc.). The claim of "lower domain dependence" is unsubstantiated, as manual rules and Hunspell vocabularies can be domain-dependent. The motivation is unclear - is the contribution a new system, tagset, or evaluation? The new error-tagset isn't justified when existing tagsets could have been used.

The paper contains typos and reference formatting issues (e.g., "c.f." → "cf.", inconsistent casing). While the application is valuable, the methodological flaws and superficial presentation significantly undermine the contribution. The authors should clarify the primary contribution, provide detailed rule examples, justify the new tagset, and address the evaluation methodology issues before acceptance.

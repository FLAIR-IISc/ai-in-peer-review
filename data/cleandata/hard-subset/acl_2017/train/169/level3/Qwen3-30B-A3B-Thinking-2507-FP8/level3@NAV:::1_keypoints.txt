- Proposes an automatic method to annotate GEC outputs with error types using linguistically-enhanced alignment and a rule-based classifier.  
- Applies the method to CoNLL-2014 system outputs for fine-grained error type analysis.  
- Main contribution: method enabling detailed, fine-grained comparison of GEC systems useful for teachers and learners.  
- Introduces a new dataset-independent error-type classification framework, though its novelty is questionable.  
- Strength: addresses a real gap in GEC evaluation beyond recall-based measures.  
- Strength: small-scale human evaluation (5 raters) reports >95% "Good" or "Acceptable" for predicted error types.  
- Strength: authors will release the tool, adding practical value.  
- Weakness: system description is superficial; core rule set lacks examples, counts, complexity, and ordering details, hindering reproducibility.  
- Weakness: evaluation methodology flawed—raters judged system outputs rather than an independent gold standard.  
- Weakness: aggregation approach (averaging percentages) is problematic and omits distribution of "bad" ratings.  
- Weakness: paper overemphasizes CoNLL-2014 with repetitive tables and omits critical test-data details (which error categories covered, etc.).  
- Weakness: claim of low domain dependence is unsubstantiated since manual rules and Hunspell vocabularies can be domain-dependent.  
- Weakness: unclear motivation—uncertainty whether contribution is a system, tagset, or evaluation method; new tagset not justified over existing ones.  
- Presentation issues: typos and inconsistent reference formatting (e.g., "c.f." → "cf.", inconsistent casing).  
- Recommendation: clarify primary contribution, provide detailed rule examples and ordering, justify the tagset, and fix evaluation methodology before acceptance.

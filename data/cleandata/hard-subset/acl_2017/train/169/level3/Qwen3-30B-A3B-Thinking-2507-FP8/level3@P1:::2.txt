

Summary: This paper presents a method for automatically annotating grammatical error correction (GEC) system outputs with error type information, enabling detailed error type analysis beyond simple recall metrics. The primary contribution is a dataset-independent rule-based framework that classifies edits based on automatically obtained linguistic properties (POS tags, lemmas), validated through human evaluation. A secondary contribution is the application of this method to the CoNLL-2014 shared task data to conduct the first comprehensive error type analysis of GEC systems.

Strengths:
1. The paper solves a fundamental limitation in GEC evaluation by enabling precision-based error type analysis, which was previously impossible due to the lack of error type annotations on system outputs. This provides valuable insights into system strengths and weaknesses across different error types.
2. The rule-based classifier is dataset-independent and relies solely on automatically obtained linguistic information, making it applicable to any GEC system without requiring retraining or manual annotation. This addresses a key limitation in prior work that required dataset-specific classifiers.
3. The human evaluation demonstrates high quality of automatic annotations, with raters rating 95% of predicted error types as "Good" or "Acceptable" despite noisy automatic edit extraction. This validation is crucial for establishing the reliability of the approach.
4. The application to CoNLL-2014 data reveals meaningful insights about system performance across different error types, showing that different approaches complement each other (e.g., MT approaches excel at noun errors while classifiers handle verb form errors well).

Weaknesses:
1. The paper fails to provide sufficient detail about the rule-based classification framework, making it difficult to reproduce or generalize the approach. The rules for deriving error types are not described, which significantly impacts reproducibility and adoption.
2. The classifier evaluation lacks systematic analysis of "Bad" labels. While raters' "Bad" judgments are attributed to POS errors, there is no detailed examination of why these judgments were made or how the system could be improved.
3. The evaluation is limited to English, with no discussion of how the approach would generalize to other languages. This is particularly problematic given that the paper claims the method is dataset-independent but provides no evidence of cross-lingual applicability.
4. The paper does not provide raw counts (TP/FP/TN/FN) for error type analysis (Table 6), making it impossible to assess the statistical significance of results or evaluate the reliability of low-count error types.
5. The paper does not address how the method handles discontinuous edit spans (e.g., separable verb prefixes in German), which could be a significant limitation for non-English GEC applications.

Questions to Authors:
1. Please provide the complete set of rules used for error type classification to enable reproducibility.
2. What evidence do you have that the method would work for languages with different morphological structures than English?
3. Can you provide raw counts (TP/FP/TN/FN) for error types with low F0.5 scores to assess the reliability of these results?
4. How would your approach handle discontinuous edits, such as separable verb constructions in German or Dutch?



# Peer Review: "Automatic Annotation and Evaluation of Error Types for Grammatical Error Correction"

## Summary and Main Contributions
This paper addresses a critical gap in Grammatical Error Correction (GEC) evaluation by proposing an automatic method to annotate error types in GEC data without requiring manual annotation. The authors introduce a two-step approach: (1) linguistically-enhanced edit extraction using Felice et al.'s (2016) alignment method, and (2) a rule-based error typing framework that relies solely on automatically obtained linguistic features (POS tags, lemmas, etc.). The primary contributions are: (1) a dataset-agnostic error type framework that enables detailed error analysis, (2) a validation of their approach through manual evaluation showing >95% agreement with human raters, and (3) the first comprehensive error type analysis of all teams in the CoNLL-2014 shared task.

## Strengths

1. **Addressing a genuine evaluation gap**: The paper identifies a significant limitation in current GEC evaluation practices - the inability to perform detailed error type analysis. Previous work focused on overall metrics like F0.5, obscuring which error types specific systems excel or struggle with. This is a major contribution to the field as understanding these nuances is crucial for targeted system improvement.

2. **Innovative dataset-agnostic error typing**: The rule-based approach is particularly strong because it avoids the dataset-specific biases that plague machine learning classifiers. By relying solely on automatically obtained linguistic features (POS tags, lemmas, etc.), the authors create a framework that works across datasets without retraining. Their manual evaluation (86.3% "Good" or "Acceptable" ratings) demonstrates robustness despite automatic edit extraction noise.

3. **Practical demonstration with CoNLL-2014**: The application to the CoNLL-2014 shared task provides concrete evidence of the framework's utility. The detailed error type analysis reveals important insights (e.g., 5 teams failed to correct unnecessary token errors, CAMB outperformed others on missing errors) that were previously impossible to obtain. This demonstrates significant practical value for the GEC community.

4. **Modular framework for flexible analysis**: The ability to evaluate at different granularities (e.g., only replacement errors, or only noun errors) is a substantial practical advantage. This flexibility allows researchers to focus on specific error types relevant to their work, making the framework highly adaptable for future research.

## Weaknesses

1. **Insufficient discussion of edit extraction limitations**: The paper acknowledges that error type classification errors often stem from POS tagging mistakes but doesn't adequately address how common these alignment errors are or how they impact the overall error type analysis. For instance, if edit extraction is wrong in 10% of cases (as implied by the 3.7% "Bad" ratings), how does this affect the reliability of the error type analysis? The authors should provide a more thorough analysis of error propagation from edit extraction to error typing.

2. **Lack of comparative evaluation with existing methods**: While the paper claims their rule-based approach is superior to machine learning classifiers, they don't directly compare it to previous methods on the same dataset. Without this comparison, it's difficult to assess whether their approach truly offers an improvement over existing techniques. A direct comparison with Swanson and Yamangil (2012) or Xue and Hwa (2014) would strengthen their claims.

3. **Limited discussion of language-specific applicability**: The framework relies on Universal Dependency tags, which may not generalize well to languages with different grammatical structures or for which Universal Dependency tagsets are underdeveloped. The paper doesn't address how this framework would be adapted for non-English GEC or for languages with different error patterns.

4. **Unclear relationship to fluency-focused GEC**: The paper references Sakaguchi et al. (2016) on the importance of fluency over pure grammaticality, but doesn't explain how their error typing framework contributes to this broader goal. Given that multi-token errors (which are challenging for all systems) are often related to fluency, the paper should clarify how their framework can help address fluency concerns.

## Questions to Authors

1. The paper claims that "our approach is analogous to automating the annotation guidelines given to human annotators," but how does your rule-based framework handle ambiguous cases that human annotators would discuss? For example, how would your system classify "He has a cat" → "He has cats" (NOUN:NUM vs. NOUN:INFL)?

2. In Table 5, you report that 5 teams failed to correct any unnecessary token errors. What specific patterns of unnecessary token errors were these teams missing, and why might they have been particularly challenging for these systems?

3. The paper states that "the M2 scorer tends to overestimate performance," but doesn't explain how your approach addresses this overestimation. How does your edit comparison method avoid the pitfalls of M2's dynamic edit boundary prediction?

## Recommendation

This is a strong paper that makes a valuable contribution to GEC evaluation methodology. The automatic error type annotation framework addresses a genuine gap in the field and is validated through a compelling application to the CoNLL-2014 dataset. While some weaknesses in the evaluation and framework limitations need addressing, the core contribution is significant and practical. I recommend acceptance with minor revisions to address the weaknesses noted above, particularly regarding edit extraction error analysis and comparative evaluation.

The paper's release of the annotation tool will be a valuable resource for the GEC community, enabling more nuanced system analysis and potentially leading to more targeted improvements in GEC systems. The detailed error type analysis of CoNLL-2014 teams provides insights that will inform future GEC research directions.



# ACL Review: "Automatic Annotation and Evaluation of Error Types for Grammatical Error Correction"

## Summary and Main Contributions

This paper proposes a method for automatically annotating grammatical error correction (GEC) data with error type information. The main contributions are:

1. A rule-based framework for error typing that depends solely on automatically obtained properties (POS tags, lemmas, etc.) rather than requiring labeled training data
2. An approach to automatically annotate system outputs with error type information, enabling detailed error type analysis of GEC systems
3. Application of this method to the CoNLL-2014 shared task data to provide the first detailed error type analysis of all participating systems

## Strengths

1. **Addresses a meaningful gap in GEC evaluation**: The paper identifies a genuine limitation in the field—systems are evaluated with overall scores without detailed error type analysis. The proposed method directly addresses this gap by enabling systematic error type evaluation without requiring manual annotation of system outputs.

2. **Novel dataset-independent error typing approach**: The rule-based framework is a significant contribution because it avoids the limitations of previous machine learning approaches that require different classifiers for different datasets. The claim that their approach is "analogous to automating the annotation guidelines" is compelling and well-explained.

3. **Practical application to CoNLL-2014 data**: The demonstration of applying their method to the CoNLL-2014 data provides valuable insights into system strengths and weaknesses across different error types, which could inform future GEC system development.

## Weaknesses

1. **Multiple formatting and typographical errors throughout the paper**:
   - Inconsistent citation style (e.g., "Felice and Briscoe, 2015" vs "Felice et al. (2016)")
   - Reference list contains numerous formatting errors (e.g., "Human evaluation of gram, Edward Gillian, Gillian.,? \Q2015\E, Gillian.," which appears to be a corrupted citation)
   - Typographical errors like "3http://www.nltk.org/" (with a stray "3" in the URL)
   - In Table 9, "To ke" instead of "To be" in the header
   - Inconsistent spacing in text (e.g., "This is fundamentally an alignment problem: The first attempt..." where the line break creates a sentence fragment)

2. **Insufficient evaluation of the error typing framework**:
   - The manual evaluation (Section 3.3) is limited to only 200 randomly chosen edits (100 from FCE-test and 100 from CoNLL-2014), which is a small sample size for claiming robustness.
   - The authors claim their method is superior to machine learning approaches but don't provide a direct comparison of performance metrics (F1 scores) between their rule-based approach and the machine learning approaches they reference (e.g., Felice et al., 2016).
   - The statement that "the best results using a classifier were between 50-70 F1" is not directly comparable to their own results because they don't provide F1 scores for their rule-based approach on the same dataset.

3. **Lack of evidence for key claims**:
   - The claim that "the M2 scorer in fact tends to overestimate performance" is based on a hypothetical example rather than empirical evidence from the CoNLL-2014 data. The authors don't provide statistical evidence showing that M2 scores are systematically higher than their approach's scores on the same data.
   - The claim that "our auto annotations are qualitatively as good as human annotations" is based on a lack of statistically significant difference between auto and gold references, but they don't provide the actual metrics (e.g., exact F-score differences) that would support this conclusion.

## Questions to Authors

1. In Section 3.3, you mention "0.724 κfree" as your inter-annotator agreement measure but don't explain what this is or how it was calculated. Could you provide more details about this metric and how it compares to standard inter-annotator agreement measures?

2. In Table 2, the example for "NOUN:INFL" is "informations → information," but "informations" is not a standard English word. Could you clarify if this was meant to represent a non-standard form or if it's a typo?

3. The authors cite "Felice et al., 2016" as using a linguistically-enhanced alignment algorithm, but in the references section, they cite "Felice et al. (2016)" as "A new dataset and method for automatically grading esol texts" which appears to be a different paper. Could you clarify which paper you're referencing?

## Recommendation

**Reject**

While the paper identifies a meaningful gap in GEC evaluation and proposes a potentially useful method, the numerous minor formatting errors, inconsistencies, and lack of sufficient evidence for key claims significantly undermine the paper's credibility. As a perfectionist reviewer, I find these issues particularly problematic because they suggest a lack of attention to detail that would be expected in an ACL publication. The paper's main contribution (the rule-based error typing framework) is interesting, but the presentation and evaluation are insufficiently rigorous to justify acceptance. The authors should address all these minor issues before considering publication in a top-tier venue like ACL.

The paper's flaws are not in the conceptual contribution but in the execution and presentation, which are critical for an ACL paper. The presence of multiple typographical errors, inconsistent citation formats, and unverified claims about the M2 scorer's overestimation of performance are too significant to overlook in a paper that claims to be a meticulous technical contribution.

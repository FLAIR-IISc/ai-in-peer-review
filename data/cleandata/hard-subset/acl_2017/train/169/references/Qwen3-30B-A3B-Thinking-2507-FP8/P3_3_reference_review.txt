

# Peer Review: "Automatic Annotation and Evaluation of Error Types for Grammatical Error Correction"

## Summary
This paper addresses a significant gap in Grammatical Error Correction (GEC) evaluation: while reference texts in the CoNLL-2014 shared task were annotated with error type information, participating systems' outputs weren't, making detailed error-type analysis impossible. The authors propose an automatic method to annotate system outputs with error type information, using a linguistically-enhanced alignment algorithm for edit extraction and a rule-based classifier for error type classification. They demonstrate their approach on CoNLL-2014 data, revealing insights about system strengths and weaknesses at specific error types.

## Main Contributions (as I see them)
1. **Dataset-agnostic error type annotation framework**: The rule-based classification system depends solely on automatically obtained linguistic features (POS tags, lemmas) rather than labeled training data, enabling consistent error type evaluation across different datasets without retraining.
2. **Demonstration of practical utility**: The authors apply their method to CoNLL-2014 data, revealing that different systems excel at different error types (e.g., CAMB excels in 10 error categories, while CUUI outperforms others in specific categories like ORTH errors).
3. **Validation of the approach**: A small-scale human evaluation shows high agreement (86.3% "Good" or "Acceptable") between their rule-based classifications and human judgment, despite the noise from automatic edit extraction.

## Strengths
1. **Addresses a genuine gap**: The paper correctly identifies a significant limitation in GEC evaluation that has been noted in the literature (e.g., Felice and Briscoe, 2015). The inability to analyze error types has hindered understanding of system strengths and weaknesses, which is crucial for improving GEC systems.

2. **Practical and reproducible**: The authors have built a tool that can be released with the paper, making it immediately useful for the community. The methodology is transparent, building on established NLP techniques like linguistically-enhanced alignment (Felice et al., 2016).

3. **Strong validation**: The human evaluation is convincing, showing high agreement (86.3% "Good" or "Acceptable") between their rule-based classifications and human judgment. The raters' comments about "Bad" classifications often tracing to POS tagger errors further demonstrate the method's robustness.

4. **Insightful analysis**: The CoNLL-2014 analysis reveals unexpected findings, such as 5 teams failing to correct any unnecessary token errors at all (despite these being theoretically simpler to handle), and CAMB's hybrid MT approach significantly outperforming others in missing token errors.

## Weaknesses
1. **Lack of comparison with alternative automatic methods**: While the paper mentions previous work used classifiers, it doesn't compare their rule-based approach to alternative automatic methods (e.g., using the same classifiers but with different training data). This makes it difficult to assess the actual improvement over existing methods.

2. **Limited scope of evaluation**: The human evaluation is small-scale (5 raters, 200 examples), which is common but doesn't address how the rule-based approach might perform on more diverse or challenging error types. The paper also doesn't compare their method to human annotation quality beyond this small-scale evaluation.

3. **Questionable practical impact**: While the paper claims their method enables "more detailed observations about different aspects of system performance," the analysis doesn't clearly demonstrate how this level of detail would change GEC system development or evaluation. The paper shows systems have different strengths, but doesn't show how this knowledge would be used to improve systems.

4. **Overstated claims about M2 scorer**: The paper claims the M2 scorer "overestimates performance" due to its edit boundary prediction. However, this is a known limitation already discussed in the literature (Felice and Briscoe, 2015), and the authors don't provide evidence that their method is better than M2 for practical purposes.

## Overall Assessment
This paper makes a valuable contribution to the GEC field by providing a practical method for detailed error-type analysis. The method is sound, well-validated, and immediately useful for the community. While the paper could be strengthened by better comparison with alternative methods and a clearer demonstration of how this detailed analysis would impact system development, the contribution is significant enough to warrant acceptance.

The main weakness is that the paper doesn't convincingly demonstrate how the detailed error-type analysis would actually change GEC system development or evaluation practice. However, given the genuine gap it addresses and the practical utility of the tool they've built, this is a strong paper that makes a meaningful contribution to the field.

## Recommendation
Accept (Strong Accept)



# ACL Review

## Summary
This paper proposes a novel method to automatically annotate grammatical error correction (GEC) data with error type information, addressing a significant gap in GEC evaluation. The authors present a two-step approach: (1) linguistically-enhanced edit extraction using alignment algorithms, and (2) a rule-based error type classification framework that depends only on automatically obtained properties (POS tags, lemmas, etc.). The authors demonstrate the value of their approach by applying it to the CoNLL-2014 shared task data to provide the first detailed error type analysis of all participating systems.

## Main Contributions (in order of strength)

1. **Novel framework for automatic error type annotation**: The paper introduces a dataset-agnostic rule-based framework for error type classification that doesn't require labeled training data or dataset-specific tuning. This is a significant contribution as it enables detailed error type analysis without manual annotation.

2. **First detailed error type analysis of CoNLL-2014 systems**: The authors apply their framework to provide the first comprehensive error type analysis across all teams in the CoNLL-2014 shared task, revealing system strengths and weaknesses that would be impossible with standard evaluation metrics.

3. **Tool release for community use**: The authors plan to release the annotation tool, making their approach accessible to the GEC community and enabling standardization of error type evaluation.

## Strengths

1. **Addresses a fundamental gap in GEC evaluation**: The paper correctly identifies that current GEC evaluation lacks detailed error type analysis because reference data includes error types but system outputs do not. This has prevented researchers from understanding which systems excel at which error types. The authors' approach directly addresses this gap with a practical solution that could become standard practice.

2. **Novel and valuable methodology**: The rule-based error classification framework is simple, transparent, and dataset-independent - a significant advantage over machine learning approaches that require different classifiers for different datasets. The authors' manual evaluation shows high agreement (86.3% "Good" or "Acceptable") with human raters, validating their approach's efficacy.

3. **Practical insights for the community**: The analysis of CoNLL-2014 systems reveals that different approaches complement different error types (e.g., CAMB excels at missing token errors while CUUI is best at replacement token errors). This is valuable information that could guide future research and system design.

4. **Makes evaluation more transparent**: The paper demonstrates how their approach can be used to evaluate GEC systems by comparing two files with identical annotation formats, which is more straightforward than current metrics that incorporate edit extraction directly into evaluation. They also show that their automatic annotations are as good as human annotations.

5. **Potential for broader impact**: The framework is not limited to CoNLL-2014; it can be applied to any GEC dataset, potentially standardizing error type evaluation across the field.

## Weaknesses

1. **Limited novelty in edit extraction component**: The edit extraction method relies on previously published work (Felice et al., 2016) and doesn't present a significant improvement over existing methods. While the overall framework is novel, this component is not.

2. **Limited comparison to machine learning approaches**: The paper mentions that previous classifier-based approaches achieved "50-70 F1" but doesn't directly compare this to their rule-based approach on the same data. A direct comparison would strengthen their claim that the rule-based approach is superior.

3. **Small-scale evaluation of the error type framework**: The manual evaluation involved only 5 raters and 200 examples. While the results are promising, a larger evaluation would better validate the framework's robustness.

4. **The paper doesn't sufficiently articulate why this is a "bold" approach**: While the approach is novel and useful, it's not particularly bold in challenging fundamental assumptions or proposing a completely new paradigm. It's a practical improvement to evaluation methodology rather than a paradigm shift.

5. **Limited discussion of framework limitations**: The paper mentions the framework is a compromise between informativeness and practicality but doesn't fully explore its limitations, such as how it handles nuanced error types that might require human judgment.

## Questions to Authors

1. How does your rule-based framework compare to a state-of-the-art machine learning approach (using the same features) in terms of accuracy on the same data? The paper states that "the best results using a classifier were between 50-70 F1" but doesn't directly compare this to your rule-based approach.

2. The paper mentions that "the main goal of GEC should be to produce fluent-sounding, rather than just grammatical, sentences" (Sakaguchi et al., 2016). How does your error type framework account for fluency errors, which might be different from grammatical errors?

3. In Table 5, several teams have "0.00" for Unnecessary errors. How would your framework handle cases where a system doesn't attempt to correct a specific error type at all?

## Final Recommendation

Accept

This paper introduces a significant new concept in GEC evaluation: a dataset-agnostic, rule-based framework for automatic error type annotation. While not all components are novel, the overall approach addresses a fundamental gap in the field that has prevented detailed error type analysis for years. The authors demonstrate the value of their approach by applying it to the CoNLL-2014 data and revealing valuable insights about system strengths and weaknesses.

The paper is particularly strong because it offers a practical, transparent solution that doesn't require labeled training data and is immediately applicable to existing datasets. The authors have provided a compelling case for why this approach is valuable for the GEC community, and the small-scale manual evaluation supports their claims.

While the paper has some minor weaknesses (limited novelty in edit extraction, small evaluation), these are outweighed by the significance of the proposed framework. As a reviewer who prioritizes novel ideas over methodological perfection, I believe this paper makes a meaningful contribution that should be accepted for publication at ACL.



# Peer Review for ACL Submission

## Summary and Main Contributions

This paper addresses a significant gap in Grammatical Error Correction (GEC) evaluation by proposing a method to automatically annotate error type information in unannotated GEC data. The main contributions are:

1. A rule-based framework for error type classification that is entirely dataset-agnostic and relies solely on automatically obtained linguistic features (POS tags, lemmas, etc.)
2. A method to automatically annotate GEC system outputs with error type information, enabling detailed error type analysis
3. A comprehensive analysis of the CoNLL-2014 shared task systems using this new annotation method, revealing system-specific strengths and weaknesses across error types

## Strengths

1. **Novel evaluation methodology**: The paper identifies a critical gap in GEC evaluation - the inability to perform detailed error type analysis of system outputs since systems were never required to annotate their outputs with error types. This is a significant contribution as it enables nuanced understanding of system performance that was previously impossible to obtain. The authors demonstrate this by revealing that 5 teams failed to correct any unnecessary token errors (a category that should be relatively straightforward), and that CAMB excelled at missing token errors while CUUI was best at unnecessary token errors.

2. **Robust validation**: The authors conducted a small-scale human evaluation where 5 GEC researchers rated the predicted error types. The results showed that all raters considered >95% of the predictions as either "Good" (85%) or "Acceptable" (10%), with inter-annotator agreement at 0.724 κ. This provides strong evidence that their rule-based approach is effective, especially considering that previous machine learning approaches achieved only 50-70 F1. This validation is crucial for establishing the reliability of their method.

3. **Clear demonstration of evaluation metric limitations**: The paper provides compelling evidence that the M2 scorer (the de facto standard in GEC evaluation) tends to overestimate performance by exploiting dynamic edit boundary prediction. The authors show that M2 scores are consistently higher than their simpler edit comparison approach, and explain why this occurs (favoring multiple small edits over single multi-token edits). This insight is important for the field as it challenges the validity of current evaluation practices.

4. **Practical utility and implementation**: The authors plan to release their tool with the paper, making their approach immediately applicable to other researchers. The paper demonstrates the value of this tool through a comprehensive analysis of the CoNLL-2014 shared task, revealing detailed insights about system performance that would be impossible to obtain with existing methods.

## Weaknesses

1. **Limited novelty of core methodology**: While the application to GEC evaluation is novel, the core approach (linguistically-enhanced alignment plus rule-based classification) builds heavily on previous work by Felice et al. (2016). The paper doesn't sufficiently differentiate their contribution from this prior work, making it difficult to assess the novelty of their specific approach. The main novelty appears to be the application to error type analysis rather than the underlying methodology.

2. **Overstated claims about dataset independence**: The paper claims their framework is "entirely dataset independent," but the error type framework they've created (with 25 error types) is specific to GEC. It's unclear how this framework would handle datasets with different error annotation schemes (e.g., FCE vs. CoNLL-2014). The paper acknowledges this limitation but doesn't provide a robust solution for cross-dataset comparison, which limits the broader applicability of the work.

3. **Lack of comprehensive comparison with alternatives**: The paper states that their rule-based approach outperforms machine learning approaches in human evaluation but doesn't provide a direct comparison with other rule-based approaches or conduct thorough ablation studies. It's unclear whether the specific rules they've developed are the key innovation or if any rule-based approach would work similarly. A comparison with alternative rule-based frameworks would strengthen their claims.

4. **Insufficient analysis of practical implications**: While the paper reveals interesting patterns in system performance, it doesn't sufficiently explore how these insights could be practically used to improve GEC systems. For example, the paper mentions that combining CUUI's classifier approach with CAMB's hybrid MT approach would improve performance (citing Rozovskaya and Roth, 2016), but doesn't provide evidence that this combination would actually work in practice.

5. **Limited discussion of error type framework limitations**: The paper acknowledges that their framework is a compromise between informativeness and practicality, but doesn't thoroughly discuss the limitations of this compromise. For example, the paper states that [could → should] is labeled as a tense error rather than a modal error, but doesn't provide evidence that this categorization is appropriate or explain how the framework would handle more complex error combinations.

## Questions to Authors

1. The paper mentions that error type framework is designed as a compromise between informativeness and practicality. Could you elaborate on how you arrived at this specific set of 25 error types, and whether you've considered alternative frameworks that might be more comprehensive?

2. The paper claims the M2 scorer "tends to overestimate performance" but doesn't provide specific examples from the CoNLL-2014 data where this discrepancy was significant. Could you provide a concrete example of a sentence where the M2 scorer gave a substantially higher score than your approach?

3. The paper states that "it is always possible to add new categories for increasingly detailed error types," but doesn't address how this would affect the consistency of the error type classification. How would your framework handle the introduction of new error types without disrupting the existing classification scheme?

## Overall Recommendation

This paper makes a valuable contribution to the GEC evaluation landscape by enabling detailed error type analysis of system outputs. While the core methodology has limited novelty, the application to GEC evaluation is novel and important. The paper is well-written, well-validated, and demonstrates significant practical utility. With some clarification of the novelty claims and more thorough discussion of the error type framework's limitations, this would be a strong contribution to the field. I recommend acceptance with minor revisions.

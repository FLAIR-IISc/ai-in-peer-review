

# ACL Review: Automatic Annotation and Evaluation of Error Types for Grammatical Error Correction

## Summary of the Paper
The paper proposes a method for automatically annotating error types in grammatical error correction (GEC) systems. The method consists of two main steps: (1) automatically extracting edits between original and corrected sentences using a linguistically-enhanced alignment algorithm, and (2) classifying the edits into error types using a rule-based framework that depends only on automatically obtained properties (POS tags, lemmas, etc.). The authors demonstrate the value of their approach by applying it to the CoNLL-2014 shared task data and providing detailed error type analysis for all participating teams.

## Main Contributions (in order of strength)
1. A rule-based framework for automatically annotating error types in GEC that is dataset-independent and relies only on automatically obtained properties (POS tags, lemmas, etc.).
2. A detailed error type analysis of all systems in the CoNLL-2014 shared task, which reveals different strengths and weaknesses of different approaches.
3. A tool for automatic annotation and evaluation of error types that will be released with this paper.

## Strengths
1. The paper addresses a genuine gap in GEC evaluation by providing a method for detailed error type analysis that was not possible with previous metrics. The ability to analyze system performance at the error type level is valuable for understanding system strengths and weaknesses, particularly since the paper demonstrates that different systems have different strengths (e.g., CAMB excelled at NOUN errors while CUUI excelled at ORTH errors).

2. The rule-based approach for error typing is dataset-independent and doesn't require labeled training data, which is a significant advantage over previous machine learning approaches. The paper correctly identifies the limitations of previous ML-based approaches that require different classifiers for different datasets.

3. The small-scale manual evaluation of the classifier shows good agreement with human raters (86.3% "Good" or "Acceptable"), which provides evidence for the validity of their approach. The authors appropriately acknowledge that their approach is not directly comparable to previous ML-based approaches due to different evaluation metrics and error schemes.

## Weaknesses
1. **Inconsistent citation formatting**: The paper uses inconsistent citation styles (e.g., "Felice and Briscoe, 2015" vs. "Felice et al., 2016") and the bibliography has inconsistent formatting. For example, "Felice et al., 2016" appears in the text but the reference is listed as "Felice et al., 2016" in the bibliography. This is a minor but significant detail that shows a lack of attention to detail expected in ACL papers.

2. **Inconsistent table presentation**: Table 4 presents scores for teams but doesn't clearly label the columns. The caption states the table shows "Overall scores for each team in CoNLL2014 using gold and auto references with both the M2 scorer and our simpler edit comparison approach," but the values don't match this description. For example, the first entry for AMU is "35.01 35.06 32.67 32.22" without clear column labels. This formatting inconsistency significantly impacts the clarity of the results.

3. **Missing tool availability information**: The paper states "The tool we use to do this will be released with this paper," but doesn't provide a link or mention when the tool will be available. This is a significant omission for a paper claiming to release a tool, as it's expected that the tool will be made available at the time of publication. The paper does not provide any information about the tool's GitHub repository, version, or expected release date.

4. **Minor inconsistency in evaluation claims**: The paper claims "all 5 raters individually considered at least 95% of our rule-based error types to be either 'Good' or 'Acceptable'." However, Table 3 shows Rater 3 had 83.0% Good and 13.0% Acceptable, which totals 96.0%, not 95%. This minor inconsistency suggests a lack of precision in the writing that would be expected in an ACL paper.

5. **Incomplete rule specification**: The paper describes a rule-based framework but doesn't provide the complete set of rules or the exact implementation details, making it difficult to replicate their work. For example, they mention "we added another rule to differentiate Missing, Unnecessary and Replacement errors," but don't specify what these rules are. This is a significant methodological flaw that undermines the reproducibility of their work.

## Questions to Authors
1. Can you provide the exact link to the tool that will be released with the paper? When is it expected to be available?

2. Can you clarify the column labels in Table 4? What do the four values for each team represent (e.g., gold M2, auto M2, gold our approach, auto our approach)?

3. Can you provide the complete set of rules used for error classification? Without these, it's difficult to replicate the work and assess the validity of their framework.

4. Why did you choose to use spaCy v1.6 specifically? How would the results change if a different version or different NLP toolkit was used?

## Additional Comments
The paper has a strong conceptual contribution but is marred by numerous minor inconsistencies that would be expected to be caught in a careful revision. The inconsistent citation format, unclear table presentation, missing tool information, and incomplete rule specification are all significant issues that would be expected to be addressed in a final ACL submission. While the core idea is valuable, these details significantly undermine the paper's quality and reproducibility. Given the stringent standards of ACL, these minor but numerous flaws would lead to rejection.

The paper also has several minor formatting inconsistencies in the references section, such as inconsistent capitalization ("Grammatical Error Correction" vs. "grammatical error correction") and inconsistent punctuation in reference entries. These details are important for maintaining the professionalism of the work.



# Peer Review: Automatic Annotation and Evaluation of Error Types for Grammatical Error Correction

## Summary
This paper proposes a method to automatically annotate grammatical error correction (GEC) outputs with error type information, addressing a significant gap in GEC evaluation. The authors argue that while CoNLL-2014 and subsequent shared tasks evaluated overall GEC performance, they lacked detailed error type analysis, which could reveal system strengths and weaknesses. Their approach consists of two steps: (1) linguistically-enhanced edit extraction using a modified Damerau-Levenshtein algorithm, and (2) rule-based error classification relying solely on automatically obtained properties like POS tags. They apply this method to the CoNLL-2014 shared task data to provide the first detailed error type analysis of participating systems.

## Main Contributions
1. A rule-based framework for error typing that is dataset-agnostic and requires no labeled training data, enabling consistent error type evaluation across different GEC datasets.
2. A practical tool to automatically annotate GEC outputs with error types, facilitating detailed error type analysis that was previously impossible without manual annotation.
3. The first comprehensive error type analysis of all CoNLL-2014 systems, revealing system-specific strengths and weaknesses across 25 error categories.

## Strengths

**1. Addressing a genuine evaluation gap in GEC.** The paper correctly identifies that previous GEC evaluation methods lacked detailed error type analysis, which could provide crucial insights into system performance. The authors demonstrate that this gap is significant by showing that some systems excel at certain error types while failing at others, yet these distinctions were previously obscured by overall scores. This is a substantial contribution to the field, as detailed error analysis is essential for targeted system improvement.

**2. Innovative rule-based error typing approach.** The authors' claim that their framework is "dataset-independent" is compelling. By relying solely on automatically obtained linguistic properties (POS tags, lemmas), their method avoids the limitations of previous ML-based approaches that required different classifiers for different datasets. The human evaluation showing >95% "Good" or "Acceptable" ratings for predicted error types (Table 3) provides strong evidence for the practical utility of their approach, especially considering the inherent noise from automatic edit extraction.

**3. Practical application and insights from CoNLL-2014 analysis.** The paper's application to CoNLL-2014 data demonstrates the value of their method. The detailed error type analysis (Tables 5-7) reveals that different systems have complementary strengths (e.g., CAMB excelling at missing errors while CUUI performs better at replacement errors), which could inform future system design. The finding that 5 teams failed to correct unnecessary token errors at all (Table 5) is particularly insightful and would not have been possible without this detailed analysis.

## Weaknesses

**1. Limited justification for the error type framework.** While the authors claim their framework is a "compromise between informativeness and practicality," they don't thoroughly justify why these 25 error types are the optimal set. The framework appears to be an ad-hoc extension of previous error type schemes (e.g., CoNLL-2014), but the paper doesn't compare their framework's coverage or usability against existing schemes. The decision to label "could → should" as a tense error rather than a modal error (Section 3.2) seems arbitrary without justification, potentially limiting the framework's utility.

**2. Insufficient comparison to previous error typing methods.** The paper claims their rule-based approach is superior to ML-based methods, but the comparison is weak. They cite Felice et al. (2016) as having "50-70 F1" but don't provide a direct comparison on the same dataset or using the same evaluation metrics. Without a controlled comparison, it's difficult to assess whether the human evaluation results (Table 3) truly represent a significant improvement over previous methods.

**3. Overstated claims about M2 scorer's performance.** The paper claims "the M2 scorer tends to overestimate performance," but the evidence is limited to a single example (Section 4.1). The paper doesn't provide systematic evidence that M2 consistently overestimates across the CoNLL-2014 data. The claim that M2 "exploits its dynamic edit boundary prediction to maximize true positives" requires more substantial empirical support to be convincing.

## Questions to Authors

1. How would your error type framework need to be modified to accommodate error types specific to different learner populations (e.g., Chinese vs. Spanish L2 writers), given that the CoNLL-2014 data primarily comes from English L2 writers with a specific linguistic background?

2. The paper claims their framework is "entirely dataset independent," but the human evaluation was conducted on only two datasets (FCE-test and CoNLL-2014). How would you evaluate the framework's robustness on datasets with different error type distributions?

3. The authors state that "it is always possible to add new categories for increasingly detailed error types." How would you prevent the framework from becoming unwieldy with too many fine-grained categories, and what criteria would you use to decide when a new category is necessary?

## Overall Assessment
This paper makes a valuable contribution by addressing a significant gap in GEC evaluation methodology. The rule-based error typing approach is innovative, practical, and well-validated through human evaluation. The detailed analysis of CoNLL-2014 systems provides meaningful insights that will inform future GEC research. While the paper has some weaknesses in justifying the error type framework and comparing to previous methods, these are outweighed by the practical utility of the proposed method and the valuable insights gained from applying it to the CoNLL-2014 data. I recommend acceptance with minor revisions to address the questions above.



# Peer Review: Automatic Annotation and Evaluation of Error Types for Grammatical Error Correction

## Summary and Main Contributions

This paper addresses a significant gap in Grammatical Error Correction (GEC) evaluation by proposing an automatic method to annotate error types in GEC outputs. The authors' main contributions are: (1) a rule-based error typing framework that is dataset-agnostic and relies solely on automatically obtained linguistic features (POS tags, lemmas, etc.), (2) a demonstration of this approach through detailed error type analysis of the CoNLL-2014 shared task systems, and (3) a practical tool for standardized error type evaluation that will be released with the paper.

## Strengths

1. **Addresses a genuine field gap**: The paper correctly identifies that previous GEC evaluation methods, including the influential CoNLL-2014 shared task, focused on overall scores while obscuring detailed error type performance. This limitation prevents researchers from understanding which error types systems handle well or poorly, hindering targeted improvements. The authors' approach directly addresses this limitation.

2. **Well-justified and practical methodology**: The rule-based error typing framework is compellingly motivated. By relying only on automatically obtainable features (POS tags, lemmas, etc.), the authors avoid the dataset-dependency problem of machine learning approaches. Their small-scale manual evaluation (86.3% "Good" or "Acceptable" ratings) provides reasonable evidence for the framework's quality, especially given the absence of gold-standard error type labels.

3. **Valuable insights from CoNLL-2014 analysis**: The application of their method to the CoNLL-2014 data yields meaningful insights. For instance, the finding that 5 teams failed to correct unnecessary token errors at all is surprising and important. The multi-token error analysis (Table 8) is particularly significant, revealing that most systems struggle with multi-token corrections (only 3 teams achieved >10 F0.5), which aligns with recent work emphasizing fluency over grammaticality.

4. **Practical implementation**: The authors plan to release their tool, which will benefit the GEC community. The modular design allowing evaluation at different levels of granularity (e.g., "R:NOUN" for replacement noun errors) is a thoughtful feature that enhances the framework's utility.

## Weaknesses

1. **Limited comparison with alternative approaches**: The paper claims superiority over machine learning-based error typing approaches but doesn't provide a direct comparison. The authors cite Swanson and Yamangil (2012) as using maximum entropy classifiers, but they don't show how their rule-based approach compares to this method in terms of error typing quality using the same evaluation metrics.

2. **Inadequate evaluation of error typing quality**: The manual evaluation of 200 edits is limited in scope. The paper doesn't provide a more comprehensive evaluation, such as comparing their error typing against human annotations across a larger dataset or demonstrating robustness across different GEC systems. The observation that POS tagger errors can lead to misclassification (e.g., [ring → rings] being classified as NOUN:NUM or VERB:SVA) suggests a potential vulnerability that requires deeper investigation.

3. **Mischaracterization of the CoNLL-2014 shared task issue**: The authors claim their approach "rectifies" a mismatch in the CoNLL-2014 shared task. However, the shared task didn't require error type annotations in the output, so the "mismatch" was not a problem in the task itself. Their approach is valuable for retrospective analysis, but the framing overstates its necessity for the original task.

4. **Ambiguity in multi-token error handling**: The paper doesn't adequately explain how the framework handles multi-token errors. The statement that "a multi-token edit is an edit that affects at least two tokens on either the source or target side" is vague, and it's unclear how the alignment algorithm processes such cases. This ambiguity weakens the interpretation of the multi-token error results (Table 8).

## Questions to Authors

1. How does your rule-based error typing framework compare to Swanson and Yamangil's (2012) maximum entropy classifier in terms of error typing quality, using the same evaluation methodology?

2. The paper claims your approach "automates the annotation guidelines given to human annotators." How does your framework handle edge cases that human annotators would discuss (e.g., whether a particular error should be classified as tense or modal)?

3. The paper states that "the main goal of GEC should be to produce fluent-sounding, rather than just grammatical, sentences" (Sakaguchi et al., 2016), yet your error types don't directly capture fluency. How does your framework relate to fluency evaluation?

## Overall Assessment

This paper makes a valuable contribution to the GEC field by providing a standardized, dataset-agnostic method for error type analysis. The rule-based approach is well-motivated and practical, and the CoNLL-2014 analysis yields meaningful insights. While the paper has some limitations in evaluation and comparison, the core contribution is significant and well-executed. The release of their tool will benefit the community. I recommend acceptance with minor revisions addressing the weaknesses noted above. The paper addresses a genuine need in GEC evaluation and provides a practical solution that will facilitate more detailed analysis of system performance.

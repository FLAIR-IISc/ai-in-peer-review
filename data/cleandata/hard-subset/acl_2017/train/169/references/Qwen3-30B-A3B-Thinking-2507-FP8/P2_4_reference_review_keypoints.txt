- Proposes automatic annotation of GEC system outputs with error types using linguistically-enhanced edit extraction and a rule-based typing framework.
- Contribution: dataset-independent method enabling error-type analysis of system outputs without labeled training data.
- Contribution: novel rule-based error typing that uses automatically obtained linguistic properties (POS, lemmas) for cross-dataset use.
- Contribution: applied to CoNLL-2014 to reveal detailed system strengths/weaknesses across 25 error types.
- Strength: fills a long-standing gap in GEC evaluation by enabling detailed error-type analysis of system outputs.
- Strength: method is dataset-independent and validated by a small manual evaluation (86.3% judged Good/Acceptable).
- Strength: clear practical utility demonstrated—findings that standard metrics (e.g., M2) miss and actionable system comparisons.
- Weakness: evaluation is limited in scope (200 examples) and lacks systematic analysis of failure modes and ambiguous cases.
- Weakness: approach heavily depends on POS tagging accuracy, with no quantification of how tagger errors affect typing performance.
- Weakness: no direct empirical comparison to classifier-based error-typing methods on the same dataset to support superiority claims.
- Question: request clarification on how the rule-based framework aligns with CoNLL-2014 annotation guidelines and where it may differ from human judgments.
- Question: ask how the authors would manage trade-offs when adding finer-grained categories and request an example resolution for ambiguous edits (e.g., could→should).
- Question: request more detail on cases labeled "Bad" or "Acceptable" in the manual evaluation, especially ambiguous edits.
- Overall assessment: novel, practical, and impactful contribution to GEC evaluation with some evaluation limitations; recommended for acceptance.

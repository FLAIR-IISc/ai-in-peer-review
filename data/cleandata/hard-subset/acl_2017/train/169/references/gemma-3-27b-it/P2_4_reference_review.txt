## Review for ACL

**Summary:**

This paper presents a method for automatically annotating grammatical error correction (GEC) system outputs with error types. The core idea is to leverage linguistically-enhanced edit alignment followed by a rule-based error type classification system. The authors argue this addresses a gap in existing GEC evaluation, allowing for more granular analysis beyond overall scores.  They demonstrate the approach on the CoNLL-2014 shared task data, providing a detailed analysis of system performance across different error types.

**Main Contributions:**

1. **Automated Error Type Annotation:** The primary contribution is a practical pipeline for automatically annotating GEC corrections with error types, removing the need for manual annotation. This is valuable for researchers analyzing and comparing different GEC systems.
2. **Dataset-Agnostic Rule-Based Classification:** The development of a rule-based error type classifier that operates independently of specific training datasets is a significant strength. This addresses a key limitation of previous machine learning approaches which suffer from domain dependence.
3. **Detailed CoNLL-2014 Analysis:**  The application of the tool to the CoNLL-2014 data provides a comprehensive error type analysis of the participating systems, offering valuable insights into their strengths and weaknesses.

**Strengths:**

1. **Addresses a Real Need:** The lack of detailed error type analysis in GEC has been a long-standing issue. This work directly tackles that problem and provides a readily usable tool. The release of the tool is excellent.
2. **Robust Methodology:**  The combination of linguistically informed alignment and a rule-based classifier appears to be a sound approach. The authors clearly justify their design choices and provide evidence supporting their performance. The high agreement with human raters (95% "Good" or "Acceptable") is very encouraging.
3. **Insightful Analysis:** The CoNLL-2014 analysis is detailed and reveals interesting patterns. The observations about systems struggling with certain error types (e.g., unnecessary token errors) and the complementary strengths of different approaches are valuable.  The analysis of multi-token errors is particularly insightful.
4. **Clear and Well-Written:** The paper is well-organized, clearly written, and easy to understand. The motivation is clearly stated, and the methodology is described in sufficient detail.

**Weaknesses:**

1. **Limited Novelty in Components:** While the *combination* of techniques is valuable, the individual components (linguistically-enhanced alignment, rule-based classification) are not novel in themselves.  The novelty lies in the specific application and integration for automated error type annotation in the context of GEC evaluation.  This is fine, but should be acknowledged more explicitly.
2. **Reliance on POS Tagging Accuracy:** The rule-based classifier heavily relies on the accuracy of POS tags, lemmas, and stemming.  Errors in these automatic analyses can lead to misclassification, as evidenced by the manual evaluation results. The paper acknowledges this, but a more thorough error analysis specifically focusing on the impact of POS tagging errors would be beneficial.
3. **Evaluation Caveats:** While the manual evaluation of the classifier is positive, the small scale (200 edits) and specific focus on "appropriateness" rather than precision/recall limits the conclusions that can be drawn.  A more rigorous evaluation against a gold standard (even if imperfect) would be stronger. The comparison to M2 scorer is interesting but slightly convoluted - relying on auto-references for evaluation feels a bit indirect.
4. **Limited Discussion of Error Category Design:** The rationale behind the 25 error categories could be expanded upon.  The discussion of trade-offs between informativeness and practicality is good, but a more in-depth justification for the specific category choices would be helpful.



**Questions to Authors:**

1.  Could you provide a more detailed error analysis of the classifier’s misclassifications, specifically quantifying the impact of errors in POS tagging, lemmatization, or stemming?
2.  You mention the possibility of adding more granular error categories.  What are the main challenges in extending the framework, and what trade-offs need to be considered?
3.  Given the reliance on automatic tools, how robust is the system to variations in POS tagger performance across different domains or languages?
4.  Could you elaborate on the potential applications of this automated annotation tool beyond the analysis of CoNLL-2014 systems?  For instance, could it be used for training data augmentation or for providing feedback to language learners?




**Additional Comments:**

*  Table 9 (complete list of error codes) is helpful.
*  The references section is complete and well-formatted.
* The writing is clear and concise throughout.



**Overall Assessment:**

This is a strong paper that addresses an important problem in GEC evaluation. The proposed method is practical, well-motivated, and yields valuable insights. While the individual components may not be groundbreaking, the combination and application are significant. The minor weaknesses do not detract significantly from the overall contribution. I recommend **acceptance** with minor revisions to address the questions and strengthen the discussion of evaluation and error category design.

- Paper proposes a pipeline to automatically annotate GEC system outputs with error types, enabling granular evaluation.
- Pipeline uses linguistically-enhanced edit extraction (Felice et al., 2016) plus a rule-based error typing scheme and an implemented released tool.
- Main contribution: a practical, publicly available tool for automatic error-type annotation of GEC outputs.
- Main contribution: a dataset-independent, interpretable rule-based error-typing scheme as an alternative to ML-based typing.
- Main contribution: an insightful application and analysis of CoNLL-2014 systems using the tool.
- Strength: addresses an important, long-standing need for detailed error analysis in GEC research.
- Strength: methodology is sound, leveraging state-of-the-art edit extraction and a justified rule-based approach.
- Strength: strong validation — manual evaluation with GEC researchers shows >95% agreement and useful auto vs. gold comparisons.
- Strength: clear presentation, reproducibility, and release of error code list and tool.
- Strength: provides insightful findings about system strengths/weaknesses (e.g., difficulty with unnecessary token errors).
- Weakness: limited novelty — combines existing techniques rather than introducing fundamentally new methods.
- Weakness: heavy dependency on POS tagging accuracy; tagging errors can propagate to misclassification.
- Weakness: simplified evaluation by direct edit-span comparison may be overly strict and miss equivalent corrections.
- Weakness: analysis limited to CoNLL-2014; generalizability to other GEC datasets is untested.
- Question: request for a detailed error analysis of the rule-based classifier (common misclassifications and causes).
- Question: ask for quantification of POS tagging impact, e.g., via experiments with induced tag errors.
- Question: ask how easily the framework can be extended to more complex/nuanced error types beyond POS-based rules.
- Question: ask about plans to compare automatic annotations against larger-scale human-annotated gold standards (precision/recall).
- Additional comment: paper should discuss limitations of the rule-based approach more explicitly.
- Additional comment: suggest mentioning more related work on GEC error typologies.
- Overall assessment: strong, useful contribution; recommend acceptance with minor revisions to address noted weaknesses.

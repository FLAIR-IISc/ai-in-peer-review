Summary and contributions
This paper proposes a deterministic, rule-based framework to automatically annotate grammatical error correction (GEC) edits with error types and to evaluate systems at multiple granularities. The pipeline first extracts edits with a linguistically informed alignment algorithm and then assigns error types using rules over automatic linguistic markup (POS, lemmas, dependencies). The authors apply the tool to CoNLL-2014 system outputs, re-annotate references for consistency, compare against M2, and present fine-grained analyses by operation, error category, edit length, and detection vs. correction.

As I see them, the main contributions are:
1) A deterministic, dataset-independent error typing framework and tool that can annotate both references and hypotheses using only automatic linguistic features, enabling detailed error-type evaluation.
2) An evaluation procedure that decouples edit extraction from scoring and supports exact edit matching and granular breakdowns (by operation and type); empirical comparison to M2 and evidence that automatic re-annotation of references preserves overall scores.
3) A comprehensive empirical analysis of CoNLL-2014 system outputs by error type, operation, and edit length, yielding actionable insights (e.g., multi-token edits and certain categories remain challenging; complementary strengths across systems).

Strengths
1) Practical utility and potential impact. The community lacks a robust, widely adopted tool for consistent error-type annotation of arbitrary GEC outputs. A deterministic, transparent framework that does not require training data fills a real gap, enabling per-type diagnostics, system comparison, and better ablations. The CoNLL-2014 re-analysis convincingly demonstrates utility.

2) Transparent and reproducible decision process. Rule-based typing over standard automatic markup is auditable. Unlike ML-based typers that may be biased to a training scheme, this approach can be inspected, extended, and debugged. The authors further map PTB tags to UD, increasing generality.

3) Sensible evaluation design and clear findings. The need to reprocess references to match the hypothesis alignment is well motivated; the bootstrap test suggests this does not meaningfully alter overall scores. The critique of M2’s dynamic edit boundary behavior is careful and grounded in examples, and the separation of edit extraction from evaluation is conceptually clean.

4) Fine-grained analysis that surfaces non-obvious insights. The per-type and per-operation results, multi-token edit performance, and detection vs. correction gaps provide concrete guidance for future work (e.g., many systems ignore unnecessary token deletions; multi-token edits strongly favor MT; classifiers excel on certain verb-form categories).

5) Light-weight validation of the error typing. The small manual study shows high perceived appropriateness (>95% Good/Acceptable per rater), suggesting the rules are reasonable despite noisy automatic boundaries and tags.

Weaknesses
1) Limited validation and lack of gold comparisons. The error typing evaluation is small (200 edits; 5 raters) and reports overall appropriateness only. There is no per-category analysis of where the rules fail, nor a comparative evaluation against a manually typed corpus. The “dataset-agnostic” claim remains under-evidenced: the method is English-specific and depends on the quality of a single tagger/parser (spaCy v1.6). Robustness across domains, taggers, or genres is not tested.

2) Strict exact-match scoring may under-credit valid alternatives. Your evaluation requires identical span and identical correction strings, which can be overly harsh when multiple corrections are valid (paraphrases, different but equivalent segmentations). While you compare overall scores to M2, there is no analysis of ranking stability or of the impact on per-type credit, especially for broad categories like MORPH/OTHER or for WO and SPELL where normalization could help.

3) Insufficient specification of the rule set and ablations. The paper does not include the full set of rules, precedence order, or handling of conflicting signals (e.g., POS vs. dependency). There is no ablation of components (e.g., UD mapping, stemming, dependency features, Hunspell) nor sensitivity analysis to tagging errors, even though mis-tagging is acknowledged as a major error source.

4) Scope and generality. Experiments are limited to CoNLL-2014 outputs; more recent corpora emphasizing fluency (e.g., JFLEG) or broader coverage would strengthen claims, especially for multi-token edits. The method’s portability to other languages or to other annotation schemes is asserted but not evaluated. Even within English, only one NLP stack is used; agreement across taggers/parsers is unknown.

5) Presentation and artifact availability. The references contain formatting issues and incomplete entries. The paper states the tool “will be released” but provides no link, commit, or artifact details. For an infrastructure-style contribution, concrete release details and documentation are crucial.

Questions to authors
- Will you release the full rule set and source code (including versioned dependencies and configuration) at camera-ready? Please provide a URL and licensing terms.
- How robust is the classifier to different taggers/parsers? Can you report results when swapping spaCy for another tagger, or when using gold POS where available, to estimate an upper bound?
- How are multiple annotator references handled in CoNLL-2014? Do you match against the union, and how do you resolve conflicting edits?
- Can you quantify ranking stability between M2 and your scorer (overall and per-type) using Kendall’s tau or Spearman’s rho?
- Have you considered relaxed matching for certain types (e.g., lemma-insensitive for SPELL/ORTH, boundary-tolerant for WO) to mitigate over-strictness while preserving the benefits of fixed spans? Any preliminary results?
- Can you provide a per-category breakdown of “Bad” judgments in the manual study to identify rule failure modes?

Overall, this is a useful, largely well-motivated infrastructure and analysis paper with clear community benefit, provided the tool and rules are released and validation is strengthened.

Summary
The paper proposes an automatic pipeline to annotate grammatical error correction (GEC) outputs with explicit edits and error types, enabling fine-grained evaluation beyond overall scores. The pipeline consists of: (i) edit extraction via a linguistically-enhanced alignment algorithm (Damerau–Levenshtein with POS/lemma-aware costs) and merging rules; and (ii) a deterministic rule-based error type classifier that assigns categories using only automatically derived annotations (POS, lemmas, stems, dependencies) and dictionary lookup. The authors introduce a modular error scheme (25 types with operation prefixes M:/R:/U:) allowing flexible aggregation (e.g., by operation, POS, or morphology). A small-scale manual study (5 raters, 200 edits) rates 96.3% of predicted categories as Good/Acceptable, κfree=0.724. The tool is applied to the CoNLL-2014 shared task outputs to provide the first uniform per-error-type analysis across systems, including operation-tier metrics, detailed category breakdowns, multi-token edit performance, and detection vs. correction comparisons. The authors also show that reprocessing references with the same automatic alignment yields similar overall F0.5 as using the original gold references, and argue that the M2 scorer tends to overestimate performance due to dynamic edit boundary selection.

Main contributions (as assessed)
1) A deterministic, dataset-agnostic error type classification framework and tool that assigns error categories using only automatic linguistic markup, with an operation-tier design that supports multi-level analyses.
2) A standardized evaluation protocol that uniformly annotates both hypotheses and references, enabling direct edit-wise comparison and per-type scoring; demonstrated by a comprehensive re-analysis of CoNLL-2014 outputs (including operation-tier, per-type, multi-token, and detection vs. correction metrics).
3) Empirical findings about system behavior across error types and operations, and evidence that auto-processed references yield comparable overall scores to gold, suggesting viability of automatic reference reprocessing for consistent evaluation.

Strengths
- Practical, generalizable framework and tool: The rule-based, markup-only classifier removes dependence on corpus-specific annotation schemes and training data. Determinism improves transparency and reproducibility. The operation/type modularity allows analyses at multiple granularities (e.g., R:NOUN vs. overall NOUN), addressing a known gap in GEC evaluation practice.
- Evidence of classification appropriateness and consistency: The manual evaluation (5 expert raters, n=200 edits) yields 86.3% Good and 10.0% Acceptable, with κfree=0.724. Although not a gold-labeled accuracy, these results indicate the rules generally align with expert expectations; observed failures are largely attributable to upstream tagging errors, which is a clear and diagnosable failure mode.
- Comprehensive and informative re-analysis of CoNLL-2014: The paper provides novel insights not accessible from prior overall metrics, e.g., (i) multiple teams failed to resolve any Unnecessary token errors (Table 5); (ii) systematic weaknesses on ADV/CONJ categories (Table 6); (iii) CUUI excels on ORTH and VERB:FORM; (iv) multi-token edits are broadly challenging (Table 8); and (v) detection vs. correction reveals systems with high detection but poor correction. These findings are actionable for method development and combination.
- Alignment/evaluation compatibility and validation: Reprocessing references to match hypothesis alignment/classification is justified by bootstrap tests showing no significant difference in overall F0.5 vs. gold references across teams (Table 4). The separation of alignment and scoring simplifies evaluation logic relative to metrics that entangle alignment with scoring.
- Clarity and replicability intent: The method and resources (spaCy, UD mappings, Hunspell) are described, the schema is enumerated (Appendix A), and the authors commit to releasing the tool, supporting reproducibility and community uptake.

Weaknesses
- Limited validation of error typing accuracy: The manual study is small (200 edits), uses appropriateness ratings rather than a labeled gold standard, and is not stratified by error type or operation. There is no per-type precision/recall against human-labeled categories, making it hard to quantify reliability of detailed analyses (e.g., Table 6) beyond face validity.
- Sensitivity to upstream NLP errors without robustness analysis: The classifier relies on POS/lemmas/dependencies on learner text, where taggers/parsers can be error-prone. The paper acknowledges mis-typing as a common source of Bad labels but does not quantify sensitivity (e.g., using alternative taggers, or measuring performance degradation under tagging noise) or propose mitigation.
- Rule inventory and design are under-specified: While the schema is listed, the actual classification rules, precedence, and tie-breaking for multi-token or mixed-POS edits are not detailed. Without access to the full rule set in the paper (or an ablation), it is difficult to assess coverage, maintainability, or potential biases introduced by rule choices.
- Equivalence claims focus on overall scores: The validation that auto vs. gold references yield similar overall F0.5 does not guarantee that per-type distributions or operation-tier metrics are preserved. Since the core contribution is per-type analysis, per-type stability checks (or spot-checked manual verification) would strengthen claims of reliability of the fine-grained results.
- Novelty is incremental: The alignment method is reused from prior work; rule-based error typing by POS-based heuristics has precedent conceptually. The primary novelty lies in systematization, schema design, and broad application. This is valuable, but methodological innovation per se is modest.

Questions to authors
- Can you provide more detail on the rule set (e.g., decision list, precedence, handling of multi-token edits spanning heterogeneous POS, and conflict resolution)? Will the full rule inventory be released with the tool?
- How sensitive are classifications to the choice of tagger/parser? Have you compared spaCy vs. alternative taggers (or UD vs. PTB tag mappings) on learner text? Any empirical robustness results?
- Can you report a per-type validation on a small, manually labeled subset (e.g., 300–500 edits) to estimate precision/recall for key categories and operations? This would directly support the reliability of Tables 5–6.
- For the auto vs. gold reference comparison, do per-type F0.5 scores remain stable after reprocessing? If not, which types are most affected?
- The claim that M2 overestimates performance is plausible and illustrated qualitatively. Can you quantify the effect over CoNLL-2014 (e.g., average delta and distribution across systems), and does it vary by error type or edit length?
- How does the framework generalize to other languages? What resources (tagger, morphological analyzers, dictionaries) and rule modifications would be required, and are there language-specific categories you foresee?

- Strengths: The paper introduces a novel encoder-decoder architecture that explicitly models monotonicity.

- Weaknesses: The proposed model might be interpreted as a standard BiRNN where the alignment and transduction components are decoupled. The evaluation is limited to morphology, neglecting other sequence-to-sequence tasks where monotonicity is also a key assumption.

- General Discussion:
The authors present a novel encoder-decoder neural network architecture featuring "hard monotonic attention," which is evaluated on three morphological datasets. The paper is well-written and clearly presents the novel contribution of enforcing monotonicity in morphological tasks.

The motivation for incorporating monotonicity is well-founded, as many sequence-to-sequence tasks, unlike machine translation, are inherently monotonic. The strong performance of general encoder-decoder models on these tasks highlights the power of neural methods. This work builds on this by explicitly enforcing monotonic output generation. This is achieved by decoupling the alignment and transduction processes, first establishing monotonic alignments between input and output sequences and then training a model to generate outputs consistent with these alignments. However, this point requires further clarification.

I have several questions:
1) Clarity is needed regarding the nature of the alignments. Figure 1 suggests a 1-to-many alignment (one input character to zero or more output characters). However, this appears to conflict with the description in lines 311-312, which mentions multiple input characters aligning to a single output character. Could the authors clarify whether the model uses 1-to-many, many-to-1, or many-to-many alignments?

2) A simpler approach to monotonic sequence-to-sequence modeling exists. This involves a two-stage process: first, align the input and output characters monotonically with a 1-to-many constraint (using a standard tool like that of Jiampojamarn and Kondrak). Second, train a standard sequence tagger (e.g., an LSTM) to predict these 1-to-many alignments. For instance, for "flog" -> "fliege" (from l.613), the alignment "f-l-o-g / f-l-ie-ge" would be used to train a tagger to predict the sequence "f, l, ie, ge" from "f, l, o, g". A similar method is discussed in [*, Section 4.2]. This raises two questions:
2a) How does the proposed architecture fundamentally differ from this simpler, two-stage approach?
2b) Why was this method not included as a baseline in the experiments?

Further issues:
3) The evaluation is limited to morphology, which is a missed opportunity. Given that the model explicitly handles monotonicity, its effectiveness could have been more convincingly demonstrated by evaluating it on other monotonic sequence-to-sequence tasks (as explored in [*]).

4) The claim of performing "on par or better" (l.791) could be rephrased for greater objectivity. It seems to categorize any result where the proposed model is not superior as "on par," which may be an overstatement. Adjusting the wording is recommended, though the experimental results themselves are acceptable.

5) The paper provides limited detail about the linguistic features used. Figure 1 suggests that features like Part-of-Speech (POS) tags are included.
5a) What is the source of these features?
5b) Is it possible that the performance gains in some cases are attributable to these features rather than the monotonicity constraints?

Minor points:
6) In Equation (3), $NN$ should be formatted as $\text{NN}$ or a similar non-italicized form to distinguish it from variables.
7) l.231: The word "Where" should be decapitalized.
8) l.237 and elsewhere: For sequences like $x1...xn$, mathematical typesetting conventions suggest using commas ($x1, \ldots, xn$) or using centered dots ($x1 \cdots xn$). Please ensure consistency.
9) Figure 1: The use of a Cyrillic font makes the example difficult to reference and discuss for readers who may not have the font installed. Using a more standard script would improve accessibility.
10) l.437: There is a typo; it should be "these".

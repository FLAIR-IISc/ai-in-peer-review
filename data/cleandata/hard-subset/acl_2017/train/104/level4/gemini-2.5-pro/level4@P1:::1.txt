### Summary of the Paper
This paper tackles the problem of mention ambiguity in models that learn joint embeddings for text and knowledge bases. The authors propose a "Multi-Prototype Mention Embedding" model that learns multiple distinct vector representations (senses) for each ambiguous mention phrase. These sense embeddings are learned by jointly modeling information from both textual contexts (words surrounding the mention) and knowledge base structure (entities related to the mention's possible referents). The paper also introduces a language model-based method to disambiguate a mention in a given context to one of its learned senses. The approach is evaluated on an entity linking task, where it is reported to achieve state-of-the-art performance.

### Contributions
1.  The primary contribution is a novel framework for learning multi-prototype embeddings for ambiguous mentions, which integrates information from both unstructured text and a structured knowledge base in a unified model.
2.  The paper introduces a language model-based disambiguation component that leverages these multi-sense embeddings to link mentions to specific entities.
3.  The work provides an empirical evaluation demonstrating that the proposed method outperforms a strong and closely related baseline (ALIGN) on a supervised entity linking benchmark.

### Strengths
1.  The proposed framework demonstrates superior performance on a supervised entity linking task compared to ALIGN, a strong and highly relevant baseline. This result provides compelling evidence that the proposed multi-prototype approach effectively learns improved joint representations of text and knowledge.
2.  The evaluation is strengthened by its direct comparison to ALIGN. Because ALIGN also learns joint embeddings using very similar input data, the comparison is particularly informative and helps to highlight the specific advantages of modeling mention ambiguity with multiple prototypes.
3.  The authors provide a valuable analysis of a smoothing parameter designed to mitigate entity popularity bias. Since popularity bias is a persistent challenge in entity linking, this analysis adds depth to the experimental results and offers useful insights for future work in this area.

### Weaknesses
1.  The experimental comparison with the ALIGN baseline, while valuable, is potentially confounded by several differences in the experimental setup, making it difficult to isolate the source of the reported performance improvements. Specifically:
    *   **Hyperparameter Mismatch:** The current work uses a context window of 5 and a vector dimension of 200, whereas the original ALIGN paper used a window of 10 and a dimension of 500. These are significant differences that could impact performance, and their effect is not analyzed. To make a fairer comparison, the models should be run with identical hyperparameters, or an ablation study should be provided.
    *   **Unclear Entity Context Definition:** It is unclear how the entity neighborhood, N(e_j), is defined. The paper states the knowledge graph is built from directed Wikipedia outlinks, but is adjacency in the model treated as it would be in an undirected graph? For comparison, ALIGN defines an entity's context as the set of entities that link *to* it (in-links). If the definition of N(e_j) used here is different (e.g., includes out-links or both), this variation could substantially alter the input signal for the model and contribute to the observed performance differences. This detail is crucial for reproducibility and for accurately interpreting the results.
2.  The paper's terminology is sometimes ambiguous. It is not always clear whether "mention" refers to a surface string type (e.g., the string "Apple") or a specific instance of that string in a document. Further, the phrase "mention embedding" is potentially misleading, as the model's core contribution is learning embeddings for the different *senses* of a mention string, rather than a single, ambiguous embedding for the mention string itself. Consistently using more precise terminology (e.g., "mention string," "mention instance," and "mention sense embedding") would significantly improve the paper's clarity.
3.  The contribution of the novel disambiguation method is difficult to assess in isolation. The evaluation is conducted in a supervised setting against a joint-embedding baseline. A more comprehensive evaluation that includes comparisons against other state-of-the-art unsupervised or collective entity linking methods would be necessary to fully contextualize the performance of the proposed disambiguation strategy.

### Questions to Authors
1.  Could you please clarify the precise definition of the entity neighborhood N(e_j)? Specifically, does it include entities corresponding to incoming links, outgoing links, or both (i.e., is the graph treated as undirected)? How does this definition compare to the one used by ALIGN?
2.  Given the differences in hyperparameters (context window size, vector dimensionality) between your experiments and those reported for ALIGN, can you provide justification that the observed performance gains are attributable to your model's architecture rather than these configuration choices? Would it be possible to report results for a run using the same hyperparameters as ALIGN?
3.  To improve clarity, could you confirm whether the model learns an embedding for a mention *string* itself, in addition to the embeddings for its various senses? If not, would you consider refining the terminology throughout the paper to distinguish more clearly between a mention string and its sense embeddings?

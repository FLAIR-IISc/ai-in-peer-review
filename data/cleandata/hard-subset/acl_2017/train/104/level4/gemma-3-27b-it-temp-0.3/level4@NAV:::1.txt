## Review:

**Strengths:**

* The proposed framework demonstrates superior performance compared to ALIGN on the supervised entity linking task, indicating improved joint learning of text and knowledge representations.
* The paper provides a direct comparison to a closely related approach utilizing highly similar input data, strengthening the evaluation.
* The analysis of the smoothing parameter offers valuable insight, addressing the ongoing challenge of popularity bias within entity linking.

**Weaknesses:**

* The comparison to ALIGN could be strengthened. Key differences in experimental setup – specifically, ALIGN’s content window size of 10 versus the 5 used here, and vector dimensionality of 500 versus 200 – are not fully accounted for. Furthermore, the definition of N(ej) requires clarification; it is unclear whether this set includes only entities linking *to* ej. Given the directed nature of the Wikipedia outlink graph, the definition of adjacency (and whether it is treated as directed or undirected) needs to be explicitly stated.  The differing context definition for entities – ALIGN uses linking entities as context – makes it difficult to isolate the impact of this change on the observed performance differences in entity similarity.
* The terminology surrounding “mention” is occasionally ambiguous, shifting between referring to a string and a specific instance within a document. While “mention embedding” is used, the embeddings appear to be learned at the mention *sense* level, which should be clarified.
* Assessing the impact of the sense disambiguation order is challenging without a comparative analysis against other unsupervised entity linking methodologies.

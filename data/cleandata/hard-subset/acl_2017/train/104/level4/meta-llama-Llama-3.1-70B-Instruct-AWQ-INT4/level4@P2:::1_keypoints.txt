- Proposes a Multi-Prototype Mention Embedding model that jointly integrates textual context and knowledge-base entities into a unified semantic space.  
- Learns multiple sense embeddings per mention and uses an efficient language model–based method to disambiguate mentions to specific senses.  
- Claims state-of-the-art performance on entity linking using the proposed framework.  
- Outperforms ALIGN on supervised entity linking, indicating improved joint text–knowledge representations.  
- Provides a direct experimental comparison with a closely related approach using similar input data.  
- Analyzes the smoothing parameter to show how entity popularity affects linking performance.  
- Comparison with ALIGN lacks clarity on input data and model setting differences (window size, vector dims, graph adjacency).  
- Terminology is ambiguous: unclear whether "mention" refers to a string type or a document-specific mention and whether embeddings are for mentions or mention senses.  
- Does not clearly evaluate the impact of sense disambiguation order and lacks comparison to other unsupervised entity linking methods.  
- Asks authors to clarify input data and model setting differences versus ALIGN.  
- Requests a clear definition of "mention" and explanation of mention types vs. mention senses.  
- Requests a detailed analysis of disambiguation order effects and comparisons to other unsupervised entity linking approaches.

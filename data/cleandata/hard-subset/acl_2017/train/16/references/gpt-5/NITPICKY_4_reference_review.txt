Summary and claimed contributions
This paper proposes to improve ACE-style event detection (trigger identification and classification) by explicitly exploiting annotated argument information via a supervised attention mechanism. The model computes attention over a fixed-length context using the current token and contextual words/entities, learns attention with an auxiliary loss against “gold” attention vectors derived from arguments (two strategies: S1 focusing only on arguments; S2 distributing attention to arguments and nearby words via a Gaussian), and feeds the attended context into an ANN classifier. Experiments on ACE 2005 show gains over a weak ANN baseline and marginal improvements over prior SOTA; additional experiments augment training with FrameNet-derived data.

As I see them, the main contributions are:
1) A supervised-attention approach that uses gold argument annotations to guide attention for event detection, with two target-attention construction strategies.
2) An empirical study showing that supervising attention with argument information substantially improves a simple ANN baseline; inclusion of a FrameNet-based augmentation.

Strengths
- Clear and intuitive idea with concrete operationalization. The paper’s premise—arguments are highly informative for disambiguating triggers—is sound, and the supervised attention objective is a simple, well-motivated way to inject that signal without requiring arguments at test time. The two target-attention strategies (S1/S2) are sensible and complementary (precision vs recall).
- Empirical evidence that argument supervision helps. Section 4.2 provides useful comparisons among: unsupervised attention (ANN), entity-only baseline (ANN-ENT), and attention supervised directly with gold arguments at both train/test (ANN-Gold1/2). The substantial gap between ANN-ENT and ANN-Gold1/2 supports the claim that arguments (not just entity types) matter for ED.
- Competitive results on ACE 2005. The supervised variants (ANN-S1/S2) outperform the unsupervised baseline by a meaningful margin and slightly exceed previously reported numbers (e.g., Skip-CNN) on F1, while achieving notably higher precision.
- Data augmentation experiment. The FrameNet-based augmentation (reusing external data) modestly improves F1 for all systems and strengthens the paper’s narrative on data sparsity and the challenges of learning attention unsupervised.

Weaknesses
- Limited novelty and scope. Supervised attention trained from side information is established (e.g., Mi et al., 2016). Adapting it to ED with argument supervision is an incremental extension. The core ED architecture is a shallow ANN without modern sequence encoders or richer structures; the comparison to strong CNN/RNN baselines is thus not fully persuasive. The contribution is primarily empirical rather than methodological.
- Ambiguities and omissions in the technical description. Several critical details are unclear or inconsistent:
  • Context window definition and indexing (Section 3.1) are confusing (“w−n , w−n/2+1, …” with apparent typos; the fixed length n and actual value are not specified).
  • Attention for entities: αe is “similar” to αw but relies on we, a projection of the current word into “entity type space”; the paper never defines the projection (matrix, nonlinearity), nor whether entity embeddings are transformed (separate We?). 
  • α = αw + αe: both αw and αe are softmax-normalized; summing yields a vector not re-normalized. Is α re-normalized before use or for the attention loss? The gold attention α* is defined over word positions; how does it align with entity-based attention? The baseline ANN-ENT “forces all entities to average the attention” is also underspecified.
  • The same α is used to weight both word and entity context (cw and ce). This choice is non-obvious; no rationale or ablation (e.g., separate αw for words and αe for entities) is provided.
  • It is stated that the current token’s entity type is not used, but how multi-token entity mentions map to token-level types is unspecified.
- Evaluation rigor and comparability concerns.
  • Test-time resources: The method “exploits annotated entity information” (gold entity types) at test time. Some baselines in the literature use predicted entities; the paper does not clarify whether all compared systems use the same resource setting. This can materially affect precision and comparability.
  • Random splits: The paper uses “randomly selected” dev/test documents without listing document IDs, harming reproducibility and cross-paper comparability.
  • Significance claims: The reported t-test (p ≤ 0.05) lacks details (number of runs/seeds, variance). Without multiple independent runs, the test is not meaningful; a ~0.4 F1 gain over Skip-CNN could be within run-to-run variance.
  • Incomplete baseline reporting (e.g., “Skip-CCNN N/A 71.3” with missing precision/recall), and potential differences in preprocessing/tokenization not documented.
- Incomplete analysis and ablations.
  • No ablation for αw-only vs αe-only vs concatenation instead of sum; no study of the effect of context length n; no per-type performance analysis; no error analysis.
  • No qualitative inspection of learned attention to verify alignment with arguments or whether supervision generalizes to unseen patterns.
- FrameNet augmentation lacks clarity. The paper states Liu et al. (2016a) data “lacks both argument and entity information,” yet frames have frame elements (arguments). The mapping from frames to ACE types is not described (assumed from Liu et al., but should be explicit), and the entity labeling via RPI IE system introduces noise; no filtering or quality control is reported.
- Numerous presentation and formatting issues. There are many typos, malformed equations and tables (broken hyphenation, indices), inconsistent notation, and severely degraded references (missing years, malformed author lists). Examples: “t methods,” “p 6 0.05,” footnotes inserted mid-sentences, inconsistent symbol use (w vs wk vs w). The bibliography needs major cleanup.

Questions to authors
- Precisely define the attention computation for entities: What is the transformation from w to we (matrix, nonlinearity)? Is there a separate transformation for entity embeddings (We)? Is α re-normalized after α = αw + αe?
- What is the fixed context length n used in experiments, and how are sentence boundaries, padding, and trimming handled?
- Do you use gold entity mentions/types at test time? Do all compared baselines do the same? If not, please provide a setting with predicted entities or a fair like-for-like comparison.
- How many random seeds were used per system? Please report mean±std over multiple runs and use an appropriate significance test (e.g., paired bootstrap).
- Can you provide ablations for αw-only, αe-only, and using separate attention vectors for cw vs ce? Also, sensitivity to λ and context length.
- For FrameNet augmentation: specify the mapping from frames to ACE types you used; clarify why the released data “lacked arguments”; and quantify the noise introduced by automatic entity labeling.

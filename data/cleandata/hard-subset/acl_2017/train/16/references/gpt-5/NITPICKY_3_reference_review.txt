Summary and claimed contributions
The paper proposes a neural event detection (ED) model that explicitly exploits annotated event arguments via a supervised attention mechanism. The attention over context tokens is trained to prioritize argument words, using two strategies to construct “gold” attentions: S1 (only argument words) and S2 (argument words plus nearby words via a Gaussian kernel). The approach combines contextual word and entity-type embeddings, sums word- and entity-based attentions, and feeds the resulting context representations plus the candidate token into an MLP classifier. Experiments on ACE 2005 (and with additional FrameNet-derived data) show improvements over several reported baselines.

My view of the main contributions, ordered by importance:
1) A simple supervised-attention framework for ED that uses argument annotations to guide attention toward argument-bearing context during training, with two target-attention constructions (S1/S2).
2) Empirical evidence that supervising attention with argument information improves ED over an unsupervised attention baseline and yields competitive/better F1 than several prior systems on the authors’ ACE 2005 setup.
3) A small study augmenting with FrameNet-derived events to mitigate data sparsity.

Strengths
- Clear and intuitive motivation for using arguments to disambiguate triggers. The paper illustrates how argument cues (e.g., “former protege”) can resolve trigger ambiguity (“fired”) and builds a mechanism directly aligned with that intuition.
- Supervised attention is a straightforward, implementable idea. The two gold-attention strategies are sensible: S1 provides precise supervision; S2 trades precision for recall by diffusing attention around arguments. The observed precision/recall trade-off between S1 and S2 matches the design intent, lending credibility to the approach.
- Ablations isolating argument supervision. The comparison with ANN (unsupervised attention), ANN-ENT (averaged entity-based attention), and oracle-style ANN-Gold1/2 supports the claim that argument supervision (not merely entity presence) provides the gains.
- Competitive performance with relatively simple architecture. The method reaches or surpasses reported numbers for several prior systems on the authors’ split, without complex sequence encoders, suggesting the attention supervision is doing meaningful work.

Weaknesses
- Limited novelty relative to existing supervision paradigms. Using annotated arguments as direct supervision for attention is an incremental idea akin to using argument indicators as features or to multi-tasking with argument prediction heads. The paper does not position the method against strong alternatives that also exploit argument labels in training (e.g., multi-task trigger/argument architectures sharing encoders) nor does it compare to learning-to-rank or margin-based objectives encouraging higher scores for argument-bearing contexts.
- Methodological underspecification and notation issues hinder reproducibility.
  - The computation of we (transforming w “from the word space into the entity type space”) is not formally specified (missing equation, dimensions).
  - It is unclear whether α = αw + αe is renormalized to a probability distribution before being used in cw = Cw α^T and ce = Ce α^T.
  - Context window size n is never specified; sentence trimming/padding effects and candidate exclusion are ambiguously described; positional information beyond proximity in S2 is absent.
  - Training details (epochs, early stopping, number of runs/seeds) are omitted; the significance test (“t-test (p 6 0.05)”) is unspecified and likely inappropriate for single-run F1.
- Evaluation comparability and fairness concerns.
  - The paper appears to use gold entity mentions/types at test time; some compared baselines (e.g., Skip-CNN) do not, and the paper does not clearly ensure like-for-like conditions. If gold entities are used, the advantage should be disclosed and controlled in comparisons.
  - The ACE 2005 split deviates: dev is random across genres; test is 40 newswire docs. Many cited baselines use different splits; some report cross-genre evaluation. The paper sometimes reports only F1 for baselines (e.g., Skip-CNN) and mixes metrics, making comparisons hard.
  - The model uses additional supervision (argument labels) during training for ED; several baselines train ED without argument supervision. Stronger baselines that also exploit arguments (e.g., joint models, multi-task setups) should be reproduced on the same split with identical resources.
- FrameNet augmentation is under-specified and hard to interpret.
  - The mapping/selection from frames to ACE event types is outsourced to prior work; here arguments are reconstructed from FrameNet core FEs and entities from an external system, but details are sparse (number of added instances, class distributions, noise rate).
  - Gains are small and trade precision for recall; there is no analysis of which types benefit.
- Numerous clarity/formatting errors reduce confidence.
  - ACE is incorrectly expanded as “Automatic Context Extraction” (should be Content).
  - References are incomplete/inconsistent; many typos; equations and symbols are overloaded (w vs wk vs w).
  - In S1, it says “force all entities to average the whole attention” while S1 is defined over words; this is contradictory.

Questions to authors
- Do you use gold entity mentions and types at test time? If yes, please report results with predicted entities (and/or no entities) and ensure baselines are matched accordingly.
- Please provide exact formulas and dimensions for computing we and clarify whether α is renormalized after summing αw and αe. Also report the context window size n and whether/what positional embeddings are used.
- What are the exact train/dev/test document IDs? How many runs with different seeds are reported, and what statistical test was used on which sample unit?
- For the FrameNet augmentation: how many instances were added per ACE type? What mapping/filtering was applied? What is the quality of the automatically produced entities? Please provide per-type gains and an ablation without entity supervision for FN data.
- Can you compare against a multi-task baseline that jointly predicts triggers and arguments (sharing encoders), trained with the same supervision and on the same split? Alternatively, add a baseline that uses argument indicators as features (without attention) to isolate the value of attention supervision.
- Please provide qualitative attention visualizations and an error analysis (e.g., ambiguous triggers, cross-type confusions) to substantiate the claimed mechanism.

Summary and main contributions
This paper addresses event detection (ED) by explicitly leveraging annotated argument information via supervised attention. The core idea is to guide an attention mechanism so that context positions corresponding to event arguments receive higher weights when classifying trigger candidates. Two supervision strategies are proposed: (S1) attend only to argument tokens, and (S2) attend to arguments and their neighborhoods via a Gaussian decay. The model constructs attention over both contextual words and contextual entity types, aggregates them (αw + αe), and feeds weighted context representations along with the trigger embedding into a simple feed-forward classifier. Experiments on ACE 2005 and with FrameNet augmentation show improvements over prior work.

As I see them, the primary contributions are:
1) A simple supervised-attention formulation that uses gold argument spans to guide attention for ED, avoiding joint inference at test time while exploiting arguments during training.
2) Empirical evidence (ACE 2005) that supervised attention leveraging arguments substantially improves ED over an unsupervised attention baseline and yields competitive/better results than several prior systems; additional gains with FrameNet augmentation.
3) Analysis comparing attention supervision schemes (S1 vs. S2) and the role of argument vs. entity information.

Strengths
- Clear, plausible motivation with concrete examples: Ambiguous triggers are often disambiguated by arguments. The paper carefully argues why prior joint models may fail to help ED (imbalance, trigger pre-prediction without using arguments) and proposes a direct, lightweight alternative.

- Methodological simplicity and practicality: The approach avoids complex structured joint inference at test time. Attention supervision from gold arguments is easy to implement and potentially portable to other datasets. Using both word- and entity-type contexts is reasonable for ACE.

- Solid empirical gains over baselines: Supervised attention (especially S2) substantially improves over the unsupervised attention baseline and matches/exceeds several state-of-the-art systems reported for ACE 2005. The finding that unsupervised attention underperforms while supervised attention helps is informative for the community.

- Sensible exploration of supervision strategies: The comparison between S1 (more precise, higher precision) and S2 (more contextual, higher recall) is insightful and aligns with intuition about attention scope.

- Extension with external data: Augmenting with FrameNet (by mapping frames and core frame elements) yields further improvements and supports the argument that data sparsity hinders unsupervised attention.

Weaknesses
- Limited novelty relative to existing supervised attention literature and ED features: Supervised attention is well-known (e.g., in NMT). Using gold arguments as attention supervision is a natural extension and resembles adding argument indicator/distance features. The paper would be stronger with a comparison to simpler alternatives (e.g., adding binary indicators or distance-to-argument features, or directly concatenating pooled argument embeddings) to isolate the benefit of “attention supervision” per se.

- Ambiguity about entity usage and evaluation fairness: The model relies on contextual entity type embeddings. It appears to use gold ACE entity mentions/types at test time (as many ACE ED works do), but this is not clearly stated. Some compared neural baselines use predicted entities or none. Without clarifying whether gold entities are available in test and for all baselines, the fairness of comparisons is unclear. Similarly, for FrameNet augmentation, entities are produced by an external tagger; mismatch in entity source across datasets could confound gains.

- Incomplete and sometimes unclear modeling details: 
  • The attention composition α = αw + αe is not stated to be re-normalized; yet the gold α* is normalized. Minimizing squared error between a possibly unnormalized α and normalized α* may be ill-posed. Please clarify normalization and scaling.
  • The transformation of w into the “entity type space” to obtain we is under-specified. Are entity embeddings also transformed? Is there a separate projection for entities?
  • The fixed-length windowing (trimming/padding) might remove relevant context; its impact is not quantified.

- Missing ablations and diagnostics: No ablations for removing entity attention (αe), using only αw, varying λ, Gaussian window size/σ, or removing entity features entirely. Also absent are visualizations or alignment scores showing that learned attention indeed focuses on arguments at test time, and error analysis showing which event types benefit.

- Comparisons and reporting issues: 
  • The table for Skip-CNN lists “N/A 71.3” which is unclear (precision? F1?). Ensure all methods’ P/R/F1 are consistently reported. 
  • More recent strong baselines in representation learning (e.g., BiLSTM with contextualized encoders, position features) are not compared. 
  • Statistical testing details (test used, number of runs/seeds) are not provided; a single run with a t-test over instances is not sufficient to establish robustness.

Questions to authors
- Do you use gold entity mentions and types at test time on ACE 2005? Do all compared baselines do the same? If not, please realign or report both settings (gold vs. predicted entities).

- Is α re-normalized after summing αw and αe? What is the exact form of we and the projections for entities? Please provide the full equations.

- How sensitive is performance to λ, σ, and the Gaussian window size? Could you report ablations for: (i) αw only vs. αe only vs. both, (ii) removing entity features entirely, and (iii) replacing attention supervision with simple argument indicator/distance features?

- How are frames mapped to ACE event types in the FrameNet augmentation? What is the noise rate, and how does performance vary if you filter by confidence or overlap?

- What is the exact evaluation protocol (splits, preprocessing) and how many random seeds/runs are used? Please report mean±std and conduct significance testing across runs.

Additional comments
- The notation and some prose need cleanup (e.g., “entities average the whole attention” in S1 for candidates without arguments; clarify whether you mean “context words” or “entities”; reference formatting is inconsistent; several typos). 

- Please justify the fixed-length context window and report the percentage of truncated contexts.

- Consider including qualitative examples of attention maps and error categories, especially for ambiguous triggers like “fired.”

- Releasing code and data processing scripts (including FrameNet mapping) would improve reproducibility.

Summary and claimed contributions
This paper proposes a supervised attention mechanism for event detection (ED) that explicitly leverages argument annotations during training. The model builds an attention distribution over a fixed-length context of words and entity types around each token; supervision for the attention is derived from gold argument positions (S1) or arguments and their neighboring words via a Gaussian (S2). The approach is evaluated on ACE 2005, with and without FrameNet augmentation.

As I see it, the main contributions are:
1) A supervised attention formulation that uses gold argument positions as targets to guide the model to focus on argument-related context during ED.
2) An empirical study on ACE 2005 showing that supervised attention yields improvements over an unsupervised attention baseline, with modest gains over prior state-of-the-art.
3) A comparison of two attention supervision strategies (arguments only vs. arguments plus neighborhoods) and augmentation with FrameNet-derived data.

Strengths
- Clear motivation for using argument information for ED. The paper articulates a plausible failure mode in joint models (imbalance and indirect use of arguments) and proposes a direct way to inject argument signal via attention.
- Simple, reproducible core idea. The supervised attention mechanism is straightforward to implement (L2 loss to gold attention) and can be plugged into common ED architectures; the two strategies S1 and S2 are sensible.
- Empirical gains on ACE 2005. ANN-S2 marginally outperforms previously reported systems (Table 3) and maintains a good precision-recall balance. The FrameNet augmentation experiments further support the claim that data sparsity hinders unsupervised attention and that additional data helps.
- Ablation indicating entities alone are insufficient. The ANN-ENT baseline and the gold-attention “oracle” models (ANN-Gold1/2) help establish that argument information, not just entity presence, drives the improvements.

Weaknesses
- Clarity, technical precision, and writing issues are severe and numerous. Examples include:
  - ACE is misnamed as “Automatic Context Extraction” (should be “Automatic Content Extraction”).
  - Figures are referenced (Figure 1/2/3) but not provided; context indexing is inconsistent and likely incorrect (e.g., [w−n, w−n/2+1, …, wn/2], missing sizes; n vs. n/2 usage confused).
  - Notational overloading (w denotes a token, an embedding vector, and a hidden vector wk), with Equations 1–3 ambiguous.
  - The combined attention α = αw + αe appears not to be renormalized; then Equations 4–5 weight by a vector that sums to 2, which changes scale and is inconsistent across instances. If renormalized, it must be stated.
  - In S1, “for candidates without any annotated arguments in context, we force all entities to average the whole attention” contradicts earlier claims that attention is over words and shared across modalities; it is unclear whether gold attention is over tokens, entities, or both and how the shared α is constructed.
  - Numerous typographical and citation formatting errors throughout (e.g., dangling punctuation, “t methods,” “on earth benefits,” “exiting methods,” malformed references with “?,” “\,” “\Q2010\E”). These impede comprehension.
- Evaluation and reproducibility concerns:
  - The split is said to “randomly select 30 dev articles” and “40 newswire documents” for test “following” prior work; without explicit doc IDs, results are not reproducible. Prior works typically release the specific split.
  - No variance across runs or random seeds is reported. A single-run F1 and a t-test with p ≤ 0.05 is not an appropriate significance test for F1 over a fixed test set without resampling; standard practice is bootstrap or approximate randomization with multiple runs.
  - The Skip-CNN baseline is incompletely reported (“N/A 71.3”) and other baselines lack consistent P/R/F1 across the same split, undermining comparability.
- Potential reliance on gold entities at test time and unclear inputs. The method uses entity type embeddings for context and claims to “exploit annotated entity information” following prior work. It is unclear if entity types are gold at test time; if so, this inflates performance relative to settings where entities are predicted. If predicted, accuracy of the entity tagger and its impact are not reported. The FN augmentation further uses an external system to label ACE entity mentions—again, no details on performance or noise.
- Limited novelty relative to prior supervised attention and alternative ways to inject argument signals. Supervising attention to focus on gold-relevant positions (cf. Mi et al., 2016) is known; here it is applied to ED. The paper lacks comparisons to simpler/competing mechanisms:
  - Direct argument indicator features (or distance-to-argument embeddings) without attention supervision.
  - Multi-task learning or curriculum that predicts argument presence as an auxiliary loss.
  - Using position embeddings or syntax instead of a fixed window and shared attention.
- Missing analyses and ablations:
  - No analysis of learned attention quality (e.g., alignment with arguments on dev/test; attention entropy).
  - No study of the effect of window size n, λ, or S2 parameters (µ, σ, window) beyond a single setting; why not learn σ?
  - No error analysis by trigger ambiguity, event subtype, or argument availability to substantiate the central claim.
  - No comparison to stronger contextual encoders (e.g., BiLSTM/CNN hybrids) or leveraging dependency paths.

Questions to Authors
- Do you use gold entity mentions/types at test time? If not, what system produces entity labels, and what is the end-to-end performance impact when using predicted entities?
- How exactly is α constructed and normalized? After computing αw and αe (each summing to 1), do you re-normalize α = αw + αe before Equations 4–5?
- In S1/S2, is the gold attention defined over word positions, entity positions, or a shared vector applied to both words and entities? How do you handle candidates with no arguments in the window for S1 vs. S2?
- What is the fixed window size n, and how sensitive are results to n? Please provide a hyperparameter sweep for n and λ, and for S2: µ, σ, and neighborhood window.
- Please report results averaged over multiple random seeds with standard deviations and use an appropriate significance test (e.g., bootstrap). Also, provide the exact document IDs for the train/dev/test split.
- Can you include a baseline that adds binary “is-argument” indicators or distance-to-argument features without attention supervision? This would help isolate the benefit of supervised attention vs. feature injection.
- How do you handle multi-token arguments and triggers? If an argument spans multiple tokens, how is its attention assigned?
- Please clarify whether Figures 1–3 can be included and fix the numerous citation and terminology errors (e.g., ACE expansion), as they currently obstruct understanding.

Summary
The paper addresses event detection (ED) in ACE 2005 and proposes to exploit annotated argument information explicitly via supervised attention. A neural classifier treats each token as a trigger candidate, computes attention over a fixed-length context of words and entity types using the current token as the query, and forms context representations as attention-weighted sums. The key idea is to supervise the attention module with “gold” attention derived from annotated arguments. Two supervision strategies are considered: S1 focuses only on argument positions; S2 also spreads attention to neighboring words using a Gaussian kernel. The model is trained with a joint loss combining trigger classification and attention supervision. Experiments on ACE 2005 show gains over a comparable unsupervised-attention baseline and modest improvements over prior state-of-the-art. Additional experiments augment training with FrameNet-derived events.

Main contributions (as assessed)
1) Supervised attention for ED guided by argument annotations, with two target-attention strategies (arguments-only and argument-centered Gaussian smoothing), delivering improved F1 on ACE 2005 over prior systems.
2) Empirical analysis isolating the role of argument information for ED, including oracle-style “gold attention at test time” and data augmentation with FrameNet to address sparsity.
3) A simple attention-based context encoder that combines attention over contextual words and entity types, trained end-to-end with a trigger classifier.

Strengths
- Clear motivation and focused use of argument information for ED: The paper identifies a gap in prior joint models where annotated arguments rarely improve trigger detection. The proposed supervision explicitly encourages attention to arguments, which is well-aligned with ED needs, especially for ambiguous triggers.

- Empirical gains over baselines and prior work: On ACE 2005, ANN-S2 achieves F1=71.7, slightly surpassing the best reported baselines cited (e.g., Skip-CNN) and substantially outperforming the unsupervised-attention baseline (64.6 F1). The increase is consistent across precision/recall trade-offs, with ANN-S2 improving recall and ANN-S1 improving precision (Table 3).

- Interpretable and straightforward method: The attention supervision is simple to implement (squared loss to gold attentions) and does not require complex joint inference or structured decoding. The two strategies (S1/S2) offer a principled precision–recall trade-off.

- Thoughtful auxiliary analyses: The “gold attention” oracle experiments (Section 4.2) support the central hypothesis that argument information is beneficial for ED and distinguish contributions of entities vs arguments. The FrameNet augmentation study (Section 4.4) examines robustness to additional noisy supervision and addresses data sparsity.

- Reproducibility-oriented setup: The data splits follow prior work; hyperparameters and optimization details (embeddings, hidden sizes, optimizer, dropout/L2) are provided, and statistical significance is reported (t-test p ≤ 0.05).

Weaknesses
- Limited novelty relative to prior art: While supervising attention with argument annotations is a neat twist, leveraging argument information to aid ED has been explored in joint models. The attention supervision approach is adapted from related sequence-to-sequence work. The classifier architecture itself is basic (no sequence encoders), so the overall methodological advance is incremental.

- Incomplete experimental scope and modest absolute gains: Improvements over the strongest baseline are small (e.g., ≈0.4 F1 over Skip-CNN by the numbers shown) and only on a single dataset (ACE 2005). There is no evaluation on other domains or datasets. Per-type analysis and error breakdowns are absent, limiting assessment of generalization.

- Dependence on gold resources and unclear test-time assumptions: The model uses entity type embeddings for context; it is not clearly stated whether these are gold at test time. Many compared systems use different sets of features; comparability could be affected if gold entity types are assumed. Supervised attention requires argument annotations during training, which may limit applicability in low-resource settings.

- Clarity gaps and missing details: Several technical aspects need clarification: the fixed context window size n is not specified; how αw and αe are combined (α=αw+αe) without renormalization could change scaling; the transformation from word to “entity type space” (we) lacks details; handling of candidates with no arguments in context is ambiguously described. The “gold attention at test time” experiment is an oracle setting and should be clearly framed as such. The reference formatting and some language issues hinder readability.

- Limited analysis of design choices: The impact of λ, window size, Gaussian parameters, and context length is not systematically explored. There are no attention visualizations or quantitative checks that attention indeed concentrates on arguments at test time.

Questions to authors (for rebuttal)
- Do you use gold entity mentions/types at test time? If not, what system provides predicted entities and with what quality? How sensitive is performance to entity tagging errors?

- What is the fixed context window size n, and how are long sentences truncated? Are contexts restricted to the sentence of the candidate? How are multiple triggers in a sentence handled?

- After α = αw + αe, do you renormalize before forming cw and ce? If not, why is addition preferable to, e.g., averaging or a learned weighted combination?

- How many FrameNet instances were added, and how were ACE event types aligned to frames? What proportion of frame elements were used (only core), and how noisy are the resulting labels? Could you report the effect of this augmentation per event type?

- Can you provide per-type P/R/F1 and an error analysis (e.g., for ambiguous triggers like “fire”)? Do attention heatmaps confirm that the model focuses on argument tokens at test time?

- How does the proposed supervised attention compare to a stronger sequence encoder baseline (e.g., BiLSTM/CNN with standard attention but no supervision) under the same features?

Summary
The paper proposes a neural event detection (ED) model that explicitly exploits annotated event arguments via supervised attention. The model builds context representations by attending over contextual words and entity types conditioned on a trigger candidate. “Gold” attention targets are constructed from annotated arguments using two strategies (S1: arguments only; S2: arguments plus Gaussian neighborhood). The attention mechanism is trained with an auxiliary squared-error loss jointly with the ED classifier. Experiments on ACE 2005 show improvements over several prior systems; additional gains are reported when augmenting training with FrameNet-derived events.

Main contributions (as I see them)
- A supervised attention mechanism for ED that uses argument annotations as attention targets to bias context representations toward argument-bearing positions.
- An empirical study contrasting two attention-target construction strategies (arguments only vs. arguments plus neighbors), with evidence that S2 provides better recall and overall F1.
- An augmentation experiment using FrameNet frames mapped to ACE-style events to mitigate data sparsity, showing modest additional gains.

Strengths
- Clear motivation and focused problem framing: the paper convincingly argues that argument information can disambiguate triggers and that prior “joint” models may not directly feed argument supervision into ED.
- Simple, implementable idea with measurable gains: supervised attention over context guided by argument locations is straightforward yet yields improvements over a strong recent baseline (Skip-CNN) on ACE 2005 (F1 71.7 vs. 71.3), and larger gains over older systems.
- Ablation-like comparisons: the paper contrasts unsupervised attention (ANN) vs. supervised attention (ANN-S1/S2), and an entity-only attention heuristic (ANN-ENT), providing evidence that arguments (not merely entities) are beneficial.
- External data augmentation: mapping FrameNet frames to ACE events and integrating them into training is a useful practical addition, and the reported trends (recall ↑, precision slightly ↓) are reasonable.
- Computationally lightweight architecture: the model avoids heavy sequence encoders and still achieves competitive results, which is attractive for efficiency.

Weaknesses
- Methodological clarity and potential technical issues in attention formulation:
  - The final attention is defined as α = αw + αe, where each is a softmax; there is no stated re-normalization. This makes α sum to 2 by construction, yet it is used both as weights for cw/ce (Eq. 4–5) and as a supervision target (squared error vs. α*), which is a probability distribution. This scale mismatch could distort both representation and loss. Please clarify whether α is re-normalized and how the loss is computed.
  - The “entity-type space” transformation for the current word (we) used to compute αe is not specified (parameters, dimensionality, equation analogous to Eq. 1). Similarly, the transformation for entity embeddings prior to attention is not explicitly given.
  - The context window size n (critical for both α construction and coverage of arguments) is not reported. This omission affects reproducibility and interpretation (e.g., S1 fallback when no arguments are in-window).
- Fairness and scope of comparison:
  - The proposed method uses argument annotations as direct supervision for ED (via attention), which is stronger supervision than many ED-only baselines that do not use argument labels. While the paper compares against one joint model that uses arguments, the comparison set is uneven (e.g., Skip-CNN is ED-only, no argument supervision). A “controlled” baseline that uses argument features without attention (e.g., hand-crafted binary flags or simple pooling over argument positions) is missing.
  - For ACE experiments, the paper appears to use gold entity annotations at test time; for FN augmentation, entities are predicted. This mismatch could bias results and should be stated explicitly. More generally, a setting with predicted entities on ACE would be informative for practical robustness.
- Statistical testing and reporting:
  - The paper claims a t-test (p ≤ 0.05) but provides no details about variance, number of runs, or the unit of analysis (documents vs. instances). With single-run neural results, such tests are not meaningful. Please report multiple runs with different random seeds and standard deviations.
  - Table 3 mixes metrics (Skip-CNN only R=71.3, N/A for P) and contains misspellings/inconsistencies (“Ngyuen”), complicating comparison.
- Limited analysis and ablations:
  - No per-type analysis, no error analysis, and no qualitative inspection of learned attention to verify that it focuses on arguments as intended.
  - No ablation for using only αw vs. only αe; no tuning study for S2 parameters (σ, window size), nor for λ beyond two fixed values. These are central to the method’s claimed advantages.
- Numerous technical and presentation issues that undermine rigor:
  - ACE is incorrectly expanded as “Automatic Context Extraction” (should be “Content”).
  - The “NYT corpus” is cited with LDC2008T19, which corresponds to Gigaword; the mismatch should be corrected.
  - Multiple typos/formatting issues (e.g., dangling hyphens, “p 6 0.05”, inconsistent notation, unclear footnotes, reference list with “?,” placeholders and malformed entries). Missing key implementation details (dropout placement, activation functions, whether embeddings are fine-tuned).

Questions to Authors
- Do you re-normalize α after summing αw and αe? If not, how do you reconcile the scale difference when computing the attention loss against α*?
- How exactly is we computed? Please provide the transformation equation, parameters, and dimensions. Likewise, specify the analogous transformation for entity embeddings used in αe.
- What is the context window size n used in all experiments, and how sensitive are results to n? How often are arguments outside the window (thus triggering S1’s fallback), and what is the effect?
- Do you use gold entities at test time on ACE 2005? If yes, can you report results with predicted entities for a realistic setting? For FN-augmented data, what is the noise rate of the argument/entity annotations and its impact?
- How many runs were averaged for each result, and what are the mean ± std over seeds? How was the t-test conducted?
- Can you provide ablations isolating αw vs. αe, and a baseline that uses argument positions without attention (e.g., max/avg pooling over argument-context features)?
- Please report per-event-type P/R/F1 and examples of learned attention maps to substantiate the claim that the model focuses on arguments and nearby words as intended.

Overall, while the idea is promising and results are competitive, the accumulation of methodological ambiguities, reporting gaps, and numerous presentation issues currently prevent a confident assessment. I encourage a thorough revision addressing the above points.

This paper explores the problem of identifying patients with Mild Cognitive Impairment (MCI) by analyzing speech transcripts from three different datasets. A graph-based method leveraging co-occurrence information between words in the transcripts is described, with features encoded using various graph characteristics and lexical/syntactic properties. Results are reported using 5-fold cross-validation with several classifiers, showing varying performance across the datasets. This work addresses a well-defined problem using appropriate datasets.

**Weaknesses:**

The paper suffers from several drawbacks.

1.  The writing quality hinders readability due to grammatical errors and awkward phrasing. A thorough review for grammar and spelling is strongly recommended.
2.  The machine learning problem is not clearly defined. Specifically, it’s unclear what constitutes a single instance for classification. If each transcript is classified as MCI or No MCI, the dataset descriptions should reflect this at the transcript level. Tables 1, 2, and 3 present study demographics rather than data relevant to the classification task; patient age is largely irrelevant. A significant portion of the text (approximately two pages) is dedicated to describing the datasets with details that do not directly impact the classification performance. Furthermore, inconsistencies in the reported numbers raise concerns (e.g., in section 4.1.1, 326 people are mentioned, but the totals in Table 1 are less than 100).
3.  The motivation behind enriching the graph with word embeddings is unclear. Why not represent each word as a node in the graph and connect them based solely on the similarity between their vectors, without requiring co-occurrence?
4.  The datasets originate from a specialized biomedical domain, yet the approach does not appear to leverage any domain-specific tools or knowledge.
5.  The class distribution within the datasets is not clearly stated, making it difficult to interpret accuracy as a reliable evaluation metric. Given the binary classification task, the F1-score would have been a more informative metric.
6.  Results are reported to four decimal places on very small datasets (e.g., 43 transcripts) without any statistical significance testing. Therefore, it’s unclear whether the observed gains are meaningful.



**Questions to Authors:**

1.  Could you clarify what constitutes a single instance in your classification task (e.g., a transcript, a sentence, etc.)?
2.  Can you provide the class distribution (number of MCI and control subjects) for each dataset?
3.  What was the rationale for choosing to enrich the co-occurrence network with word embeddings in this specific manner? What advantage does this approach offer over directly building a graph based on word vector similarity?
4.  Did you consider using any domain-specific NLP tools or resources tailored to biomedical text? If not, why?
5.  Can you provide statistical significance tests (e.g., paired t-tests) to demonstrate that the performance improvements reported are statistically significant, especially given the small dataset sizes?



**Additional Comments:**

*   Consider revising the dataset descriptions to focus on characteristics relevant to the classification task.
*   The use of more descriptive variable names would improve code readability and understanding.
*   Please clarify the pre-processing steps applied to each dataset, particularly regarding disfluency removal and sentence segmentation.

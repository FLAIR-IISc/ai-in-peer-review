Summary
This paper addresses the problem of identifying patients with Mild Cognitive Impairment (MCI) by analyzing speech transcripts from three datasets. It proposes a graph-based approach that leverages word co-occurrence in transcripts and encodes features using lexical, syntactic, and other graph characteristics. Results are reported using 5-fold cross-validation across multiple classifiers, with performance varying by dataset. The paper targets a well-defined task and uses appropriate datasets.

Strengths
- Clear and important application: automatic detection of MCI from speech transcripts.
- Use of three datasets and a graph-based representation that exploits word co-occurrence.
- Feature design incorporates multiple graph-derived properties (lexical, syntactic, etc.).
- Evaluation employs 5-fold cross-validation and multiple classifiers, showing that different models behave differently across datasets.

Weaknesses and questions
1) Writing quality and readability
- The manuscript is difficult to read due to numerous grammatical and spelling errors. A thorough language edit is needed to improve clarity and precision.

2) Problem formulation and dataset description
- The main machine learning problem is not clearly defined. What constitutes a single instance of classification? It appears that each transcript is classified as MCI vs. No MCI. If so, the dataset section should report counts at the transcript level.
- Tables 1–3 should describe the data used for classification (e.g., number of transcripts, class labels, transcript lengths), not primarily the studies that produced the transcripts.
- Much of the dataset description (about two pages) focuses on details that do not affect the classification task. For example, patient age seems irrelevant to the proposed features and modeling choices.
- There are inconsistencies in the numbers. For example, Section 4.1.1 mentions 326 participants, yet the total number of males and females in Table 1 is below 100. Please reconcile these figures.

3) Graph design and motivation
- The motivation for “enriching” the graph is unclear. Why not represent each word as a node and connect nodes based on similarity between their vector representations, irrespective of co-occurrence? Please justify the chosen design and discuss alternatives.

4) Domain specificity
- Although the datasets are from a biomedical domain, the approach does not leverage any domain-specific tools or resources. This omission should be addressed or justified.

5) Evaluation metrics and class distribution
- The class distribution is not reported, making it hard to judge whether accuracy is an appropriate metric. For a binary classification task, F1 would be a more informative metric (and precision/recall per class) especially under class imbalance.

6) Statistical significance and reporting precision
- Results are reported to four decimal places on very small datasets (e.g., 43 transcripts) without statistical significance tests. It is unclear whether the observed gains are meaningful. Appropriate significance testing or confidence intervals should be provided, and reporting precision should be commensurate with dataset size and variability.

Suggestions for improvement
- Thoroughly revise the manuscript for grammar and clarity.
- Clearly define the instance and label space (e.g., transcript-level classification), and restructure the dataset section and Tables 1–3 to report transcript-level counts, class distributions, and other directly relevant properties (e.g., average transcript length).
- Reconcile all numerical inconsistencies in the dataset descriptions.
- Provide a clear rationale for the chosen graph construction and enrichment strategy, and compare it conceptually to alternatives based on distributional similarity (e.g., edges from vector similarity) independent of co-occurrence.
- Consider incorporating or at least discussing domain-specific tools/resources from the biomedical/NLP literature and their potential impact.
- Report class distributions and include F1 (with precision/recall), in addition to or instead of accuracy, for each dataset/classifier.
- Conduct appropriate statistical tests (or provide confidence intervals) and avoid over-precise reporting when datasets are small.

Overall assessment
The paper tackles a significant and well-scoped problem with a potentially useful graph-based approach, evaluated across multiple datasets and classifiers. However, issues with clarity, problem formulation, dataset reporting, metric selection, and significance testing currently limit the strength and interpretability of the findings. Addressing these concerns would substantially improve the paper’s rigor and readability.

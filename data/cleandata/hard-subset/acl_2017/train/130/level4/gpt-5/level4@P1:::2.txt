Summary and contributions
This paper addresses automatic detection of Mild Cognitive Impairment (MCI) from speech transcripts. The authors model each transcript as a word co-occurrence network and enrich the network with semantic links derived from word embeddings. They extract topological features from these networks and compare them against linguistic features (Coh-Metrix/Coh-Metrix-Dementia) and bag-of-words (BoW), using several classifiers and both ensemble and multi-view learning. Experiments are reported on three datasets: English Cookie Theft (DementiaBank) and two Brazilian Portuguese datasets (Cinderella and ABCD). Accuracy is evaluated via 5-fold cross-validation.

As I see them, the primary contributions are:
1) An empirical evaluation showing that complex network features enriched with word embeddings can be competitive for MCI detection across three datasets and two languages, sometimes outperforming linguistic features.
2) Evidence that multi-view and ensemble learning can improve performance over individual feature sets or classifiers on these datasets.
3) A language- and parser-lightweight representation aimed at robustness to disfluencies and reduced reliance on hand-crafted linguistic resources.

Strengths
- Problem significance and practical framing: Early detection of MCI is clinically important, and language is a promising signal. The paper targets a well-defined classification task and proposes a representation intended to be language-independent and less sensitive to parsing errors and disfluencies, which is valuable for clinical transcripts.

- Comparative evaluation across feature types and languages: The study compares complex network features (with and without embeddings) to linguistic metrics and BoW on English and Portuguese datasets. This cross-lingual setup increases the external validity of the findings and supports the claim that the approach does not depend on high-quality parsers.

- Multi-view/ensemble improvements: The paper shows that combining feature spaces and classifiers can yield gains (e.g., reported improvements on Cinderella and ABCD). This is practically useful and aligns with prior evidence that heterogeneous views can be complementary.

- Competitive results relative to prior work: The reported accuracies fall within ranges cited in the literature for similar tasks and often show the enriched network features comparing favorably to linguistic baselines, reinforcing the viability of the proposed representation for short, noisy transcripts.

Weaknesses
- Writing quality: The manuscript is difficult to read due to frequent grammatical and stylistic errors. Many sentences are ungrammatical, terminology is inconsistently used, and there are typographical errors throughout. A thorough language edit is needed to improve clarity and readability.

- Task formulation and unit of analysis are unclear: The paper does not clearly specify what constitutes a single classification instance. It appears that each transcript is classified as MCI vs. control, but this should be stated unambiguously. Dataset descriptions should match this unit by reporting the number of transcripts per class used in classification for each dataset and split. Instead, Tables 1–3 emphasize demographics of the original studies rather than the actual classification data. The extensive dataset narrative (about two pages) includes details that do not affect the classification task and omits essential counts at the transcript level. There are also confusing inconsistencies: for example, Section 4.1.1 mentions 326 people, but the corresponding table lists totals far below this number.

- Motivation and design of the network enrichment: The rationale for enriching co-occurrence networks with embedding-based similarity edges is not fully justified. The paper should explain why starting from co-occurrence and then adding similarity edges is preferable to constructing a pure similarity graph where nodes are words and edges reflect vector similarity irrespective of surface co-occurrence. Without this motivation, the design choice appears ad hoc.

- Limited use of domain knowledge: Given the biomedical/clinical domain, no domain-specific tools or features are leveraged (e.g., clinical lexicons, discourse or error types known to be indicative of MCI). This weakens the practical impact and suggests the approach may be leaving performance on the table.

- Evaluation metrics and class balance: Accuracy alone is insufficient, especially if class distributions are imbalanced (the paper does not clearly report per-class counts in the classification data). Standard metrics for binary clinical classification (precision, recall, F1, ROC-AUC) should be reported, preferably per class and macro-averaged, to provide a fair assessment.

- Statistical rigor on small datasets: Results are reported to four decimal places despite very small sample sizes (e.g., 43 transcripts in one dataset), and there is no statistical testing or confidence intervals. It is unclear whether reported gains are statistically significant. More conservative reporting and appropriate significance testing or confidence intervals are needed.

Questions to authors
- Please clarify the unit of classification: Is each transcript a single instance labeled as MCI vs. control? If so, provide the exact number of transcripts per class used in the experiments for each dataset and fold.

- How were hyperparameters (e.g., similarity thresholds, SVM kernels) selected? Were they tuned strictly within the training folds to avoid test leakage? Please detail the tuning protocol.

- What is the precise motivation for enriching co-occurrence networks rather than using a pure similarity graph? Did you compare against similarity-only networks, and if not, why?

- What is the class distribution for each dataset in terms of transcripts per class? Given these distributions, can you report precision, recall, F1 (macro and per class), and possibly ROC-AUC?

- Are the reported improvements statistically significant? Please include statistical tests (e.g., paired tests across CV folds) or confidence intervals.

- For the Portuguese datasets, did you consider any domain-informed features (e.g., discourse markers, error types, pause-related proxies obtainable without manual annotation) that might complement the network features in a multi-view setting?

Overall, the topic is important and the representation is promising, but the paper needs substantial clarification of the task/data, stronger motivation for the network design, improved evaluation reporting, and a thorough language edit.

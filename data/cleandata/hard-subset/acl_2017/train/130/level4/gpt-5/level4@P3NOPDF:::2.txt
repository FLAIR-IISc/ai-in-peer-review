Summary:
This paper addresses the task of detecting Mild Cognitive Impairment (MCI) from speech transcripts drawn from three datasets. It proposes a graph-based representation that leverages word co-occurrence and encodes features derived from lexical, syntactic, and related graph characteristics. The models are evaluated via 5-fold cross-validation with multiple classifiers, and performance varies across datasets.

Strengths:
- The paper tackles a well-defined and clinically relevant problem.
- It uses multiple datasets, which helps assess generalizability across sources.
- The core idea—graph-based features built from co-occurrence in transcripts—is sensible and potentially informative for discourse-level signals.
- The evaluation considers multiple classifiers and reports results across datasets.

Weaknesses and suggestions:
1) Writing and clarity:
- The manuscript is difficult to follow due to frequent grammatical errors and awkward phrasing. A thorough language edit is needed to improve readability and precision.
- Clarify technical terms (e.g., “graph enrichment,” “lexical/syntactic features”) and define them when first introduced.

2) Task formulation and dataset description:
- The machine learning problem is not described clearly. Please specify the unit of classification (e.g., transcript-level vs. patient-level) and the exact label definition (MCI vs. non-MCI).
- Report dataset statistics at the same granularity as the classification task. If transcripts are the instances, provide the number of transcripts per class for each dataset, as well as any per-patient grouping that might affect splitting.
- Revise Tables 1–3 to describe the data used for classification (e.g., number of transcripts, class counts, average transcript length), rather than primarily properties of the originating studies. Demographics (e.g., age) should be reported only if they are used as features or otherwise influence the task.
- There are inconsistencies in the counts. For example, Section 4.1.1 mentions 326 participants, but Table 1 reports male/female totals below 100. Please reconcile these discrepancies and ensure that dataset totals, class distributions, and inclusion/exclusion criteria are unambiguous.

3) Motivation and design of the graph representation:
- The paper motivates co-occurrence-based graphs but does not explain why this is preferable to alternatives. Please justify the design choices for “enriching the graph” and clarify which enrichments are used (e.g., edge weighting schemes, dependency edges, POS-based constraints).
- Consider or at least discuss an alternative where each word is a node and edges encode distributional similarity (e.g., based on embeddings), independent of local co-occurrence. An ablation or comparison would strengthen the empirical case for the chosen approach.

4) Use of domain knowledge:
- Given that the data come from a biomedical/clinical domain, it is surprising that no domain-specific resources or tools are used or discussed. Please either incorporate or justify not using domain-relevant features (e.g., clinical terminologies, discourse/disfluency markers, or tools tailored to clinical text/speech). Even a brief analysis of whether domain-aware features help would be valuable.

5) Evaluation metrics and class imbalance:
- The suitability of accuracy is unclear without knowing the class distribution. Please report class balances for each dataset and use metrics robust to imbalance (e.g., precision, recall, F1; macro-averaged where appropriate). ROC-AUC or PR-AUC could provide additional insight, but at minimum F1 should be included alongside accuracy.
- Describe whether cross-validation was stratified by class and whether any patient-level grouping was respected to avoid leakage across folds, if applicable.

6) Statistical analysis and reporting:
- Results are reported to four decimal places on very small datasets (e.g., 43 transcripts) without significance testing, making it hard to judge the reliability of gains. Please add statistical tests over cross-validation folds (e.g., paired tests) or confidence intervals and report results with precision that reflects the sample size (fewer decimals).
- If feasible, use repeated cross-validation or bootstrapping to reduce variance and provide more stable estimates.

Overall assessment:
The paper tackles an important problem and proposes an intuitively reasonable graph-based representation, assessed across multiple datasets and classifiers. However, substantial clarification is needed on the task definition and dataset statistics, along with stronger motivation for the specific graph construction. The evaluation should address class imbalance more directly and include appropriate significance testing, especially given the small sample sizes. Finally, the manuscript requires careful language editing to ensure clarity and technical precision. Addressing these issues would greatly strengthen the paper’s contribution and credibility.

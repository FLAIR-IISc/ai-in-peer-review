**Summary of the paper:**
This paper proposes a method for detecting Mild Cognitive Impairment (MCI) from speech transcripts by modeling the transcripts as complex networks. The core idea is to represent each transcript as a word co-occurrence network and then "enrich" this network by adding edges between non-adjacent words based on the cosine similarity of their word embeddings. This enrichment step is designed to make the network representation more robust for short and potentially sparse texts. The authors extract a set of topological features from these enriched networks and use them in machine learning classifiers. The proposed network-based features are evaluated on three datasets (one English, two Portuguese) and compared against, and combined with, traditional linguistic features and a Bag-of-Words (BoW) representation.

**Contributions of this work:**
1.  The primary contribution is an empirical study applying complex network features to the clinical NLP task of MCI detection. The specific method of enriching a co-occurrence graph with edges derived from word embedding similarity appears novel for this application.
2.  The paper provides a comparative analysis of these network features against strong baselines (linguistic metrics, BoW) across three distinct datasets, demonstrating that a multi-view approach combining heterogeneous feature sets can improve classification accuracy.

**Strengths:**
1.  **Novelty of Approach for the Task:** While complex networks have been used in other text analysis tasks, their application to MCI detection from clinical transcripts is a novel research direction. The proposed method of enriching networks to handle short texts is a thoughtful attempt to adapt the technique to the constraints of the data. This approach has the potential to be more robust to the disfluencies and grammatical errors common in such transcripts compared to methods relying on high-quality parsing.
2.  **Evaluation Across Multiple Datasets:** The authors validate their method on three different datasets, which vary in language (English and Portuguese), elicitation task (picture description, story narrative), and text length. This makes the experimental findings more credible and generalizable than a study based on a single corpus.
3.  **Comparison with Standard Methods:** The work does not present the complex network approach in a vacuum. It is rigorously compared against two standard and powerful baselines: a comprehensive set of linguistic features from Coh-Metrix and the ubiquitous Bag-of-Words model. The exploration of feature combination through multi-view learning is also a methodologically sound step that strengthens the analysis.

**Weaknesses:**
1.  **Critical Flaw in Word Embedding Methodology:** The paper states, "We trained the model proposed by Bojanowski et al. (2016) with default parametrs". The clinical datasets used contain only a few dozen transcripts each. This amount of data is critically insufficient for training meaningful 100-dimensional word embeddings from scratch. Embeddings trained on such a tiny corpus would be extremely noisy and likely capture corpus-specific artifacts rather than general semantic relationships. This fundamentally undermines the validity of the "enrichment" process, which is the central technical contribution of the paper. For this approach to be viable, large, pre-trained embedding models are required. This is a major methodological oversight that calls the results into question.
2.  **Unclear and Potentially Leaky Hyperparameter Optimization:** A key hyperparameter is the cosine similarity threshold used for adding edges to the network. The paper reports a single best value for each dataset (e.g., 0.7 for Cookie Theft), stating it was "optimized". However, the procedure for this optimization is not described. If this threshold was tuned using the entire dataset before performing 5-fold cross-validation, it constitutes data leakage, as information from the test folds would have been used to set a parameter for the model. A rigorous approach would require a nested cross-validation or a separate development set. The lack of detail on this critical step is a significant weakness.
3.  **Numerous Inconsistencies and Lack of Polish:** The paper suffers from a distracting number of minor errors, suggesting a lack of careful preparation.
    *   **Formatting:** The reference section is improperly formatted, with numerous entries containing BibTeX error markers (e.g., "? \Q2016\E,"), inconsistent author formatting, and incorrect special characters (e.g., "Aluı́sio" instead of "Aluísio"). There is no Figure 1, despite a reference to it. Typographical errors are present (e.g., "parametrs").
    *   **Clarity:** Key experimental details are missing or vague. The specific stopword lists used for English and Portuguese are not mentioned. The method for combining features in the multi-view setting for a single classifier (e.g., SVM-RBF row, CNE-LM-BoW column) is not explained.
    *   **Citation Usage:** The paper cites Perozzi et al. (2014) for its enrichment method. However, Perozzi et al.'s work (DeepWalk) is about learning node embeddings *from* an existing graph structure. This paper does the opposite: it uses existing word embeddings to *modify* a graph. This represents a misunderstanding or misapplication of the cited work.
4.  **Overstated Claims:** The conclusion states that "topological properties of CN outperfom traditionally linguistic metrics". This is an overgeneralization not consistently supported by the paper's own results. For example, in Table 6 (cleaned Cinderella), linguistic features with SVM-RBF (0.7250) outperform enriched network features (0.6750). In Table 7 (ABCD), the BoW baseline with SVM-Linear (0.7547) is substantially better than the corresponding CNE result (0.7173). The strength of the proposed features appears to be more as a complementary signal in an ensemble rather than a superior standalone representation.

**Questions to Authors:**
1.  Regarding the word embeddings: Were the fastText vectors trained exclusively on the small clinical datasets, as the text implies? Or did you use publicly available, pre-trained vectors trained on large corpora (e.g., Wikipedia)? If the latter, please provide the source and language of these vectors.
2.  Please provide a detailed description of the optimization procedure for the cosine similarity threshold. Specifically, was this hyperparameter tuned within each fold of the cross-validation (e.g., via an inner loop) to prevent information from the test set influencing the model?
3.  The ABCD dataset contains two narratives per subject. How were the cross-validation folds constructed? Did you ensure that both narratives from a single subject were always in the same fold (either train or test) to maintain subject independence and prevent data leakage?
4.  In the result tables, how was the multi-feature-space classification performed for the rows corresponding to individual classifiers (e.g., SVM-Linear)? Since majority voting is not applicable, did you use feature vector concatenation, or some other method?

**Paper Summary**

This paper presents a method for learning well-divided shared and task-specific feature spaces for an LSTM text classifier. Multiclass adversarial training encourages shared space representation so that a differential classifier cannot identify the source of the task (and thus generalize). Model evaluation is full-shared, shared-private, and adversarial shared-private – the latter ASP model is one of the main contributions. They also use orthogonality constraints to help reward shared and private spaces that are distinct. The ASP model has a lower error rate compared to single-task and other multi-task neural models. They also experiment with a task-level cross-validation to find out whether shared representation can be transferred to tasks, and it seems to be adapted accordingly. Finally, there is some analysis of shared layer activations that suggest the ASP model is not being misled by strong weights learned on a specific (inappropriate) task.

**Review Summary**

Good ideas, well-expressed and tested. Some minor comments.

**Strengths**

* It is a good set of ideas working together well. I particularly like the focus on explicitly trying to create a useful shared representation. These have been quite successful in the CV community, but it seems they need quite a bit of work to be built for NLP.
* Sections 2, 3, and 4 are very clearly expressed.
* Section 5.5 has a good way to evaluate task-level cross-validation transfer.
* There is an implementation and data.

**Weaknesses**

* There are some minor typographic and phrase errors. Personally, these are fine, but there are enough to be corrected:
 l:84 "Baby cart" is a bit strange - was this a real example from the data?
 l:233 "are different in" -> "are distinct from"
 l:341 "to work adversely towards" -> "to work against" or "to compete against"?
 l:434 "two matrices" -> "two matrices"
 l:445 "hyperparameters are" -> "hyperparameters are"
There are several number agreement errors in Section 6
(l:745/746/765/766/767/770/784) and it should be carefully re-edited.
The shading on the last line of tables 2 and 3 prints strangely...
* Table 1 mentions unlabeled data and Section 4.2 mentions semi-supervised learning, but I didn't see any results on these experiments. Were they left out, or did I misunderstand?
* Error rate differences in tables 2 and 3 are promising, but statistical significance testing would help make them truly convincing. Especially to highlight the benefit of adversarial training between SP-MLT and ASP-MLT results. Generating these should be very straightforward by adapting the non-parametric estimated randomization test (see
http://www.lr.pi.titech.ac.jp/~takamura/pubs/randtest.pdf for a reference to promising notes)
* The colors in the caption of figure 5 (B) are inconsistent. In 5 (A), blue is used for "our", but it seems to have changed to 5 (B). It's worth checking, or I might be misunderstanding the caption.

**General Discussion**

* I wonder if there is some connection here with regularization, as the effect of adversarial training with orthogonal training is to help limit the shared feature space. It might be appropriate to draw that connection from other regularization literature.

**Paper Summary**

This paper presents a method for learning well-partitioned shared and task-specific feature spaces for LSTM text classifiers. Multiclass adversarial training encourages shared space representation, from which a discriminative classifier cannot identify the source of the task (and therefore is generic). The models are evaluated in a full-shared, shared-private, and adversarial shared-private – the latter ASP model is one of the main contributions. They use orthogonality constraints to reward distinct shared and private spaces. The ASP model has a lower error rate compared to single-task and other multi-task neural models. They also tested task-level cross-validation to see if shared representation can be transferred between tasks, and it seems to be able to. Finally, there is some analysis of shared layer activations that suggests the ASP model is not being confused by strong weights learned on a specific (inappropriate) task.

**Review Summary**

Well-thought-out, well-expressed, and well-tested. Some minor comments.

**Strength**

* It's some great ideas that work well together. I especially like the focus on creating useful shared representations. These have been quite successful in the CV community, but it seems to take a lot of work to create them for NLP.
* Sections 2, 3, and 4 are very clearly expressed.
* Section 5.5 is a good way to evaluate task-level cross-validation transfer.
* There is an implementation and data.

**Weaknesses**

* There are a few minor typographical and wording errors. Individually, these are fine, but there are enough errors that should be corrected:
 l:84 "infantile cart" is a bit strange - was it a real example taken from the data?
 l:233 "are different in" -> "differ in"
 l:341 "working adversarially towards" -> "working against" or "competing with"?
 l:434 "to matrix" -> "to matrices"
 l:445 "are hyperparameters" -> "are hyperparameters"
There are several number agreement errors in section 6 (l:745/746/765/766/767/770/784), and it should be closely re-edited.
The shading in the last row of tables 2 and 3 is printed strangely...
* Table 1 has unlabeled data and mentions semi-supervised learning in section 4.2, but I didn't see any results from these experiments. Are they omitted, or did I misunderstand?
* The differences in error rates in tables 2 and 3 are promising, but they will only be truly convincing if statistical significance is tested, especially to highlight the advantage of adversarial training between SP-MLT and ASP-MTL results. It should be quite easy to create these using non-parametric approximate randomization tests (see a reference to the paper by Chinchor http://www.lr.pi.titech.ac.jp/~takamura/pubs/randtest.pdf).
* The colors in the caption of figure 5 (B) are inconsistent. Blue is used for "hours" in 5 (A), but it seems to have been changed for 5 (B). It is worth checking, or I may have misunderstood the caption.

**General Discussion**

* I'm wondering if there's a connection to regularization here, as the effect of adversarial training with orthogonal training is to help limit the shared feature space. It might be worth drawing that connection in other regularization literature.

This paper proposes a new phrasal RNA architecture for generating sequences from sequences. They have evaluated their architecture based on language modeling tests evaluated on PTB and FBIS, and on Chinese-English machine translation tasks based on the NIST MT02-08 evaluation set. The phrasal RNA (PRRNA) architecture is achieved by creating a subnetwork of phrases.

**Strengths:**

*   A new phrasal architecture.

**Weaknesses:**

**Technical:**

It is not clear whether a limit has been set on the length of the PRRNA phrase. Perhaps I forgot to mention this in the paper, but if it is, please mention it more clearly because it significantly affects the model if the largest phrase length for each sentence is equal to the sentence length.
*   Because if the largest phrase length is equal to the sentence length, the model can be simplified to a special type of convolution RNA, where each state of the RNA goes through some convolution layers before a final smooth-making and attention.
*   If a limit is set on the length of the PRRNA phrase, it makes the system easier. However, this means that the phrases are determined by token encodings that create a sliding window of "pyramid encoders" for each sentence, where the parameters of these phrases will be set close to zero so that the phrases can be deactivated, and this phrase will have a good underlying evaluation of PRRNA as well as being evaluated only on perplexity and BLEU.
*   Using the attention mechanism without some pruning can be problematic at the phrasal level. The author has chosen some greedy pruning described in the caption of Figure 4. However, I support that if a specific set of phrase pairs is given during training, the attention mechanism at the phrasal level can be pre-computed, but during inference (apply attention to new data during testing), it may be a bit problematic if the architecture is scaled to a larger dataset.

**Empirical:**

One problem with language modeling tests is the selection of evaluation and training sets. Perhaps datasets like Common Crawl or NWiki8 would be more suitable for language modeling tests.
The main problem with the paper is the test and result reporting, it needs to be rearranged quite a bit.
*   The evaluation in PTB (Table 2) is not fair because the model was trained on a larger corpus (FBIS) and then tested in PTB. The previous study used LSTM to report a 126 perplexity baseline and the author's LSTM reported a perplexity of 106.9, which shows that FBIS makes it easier to calculate the language model's perplexity when testing in PTB.
*   Also, regarding paragraph 3.3, please mention the appropriate publication related to "Previous work" in the table. And is the previous work using the same training set?
*   Furthermore, why is the GRU version of PRRNA not reported in the FBIS evaluation in Table 3?

The results section cannot present a table without explanation:
*   Still in the results section, although it is clear that BLEU and perplexity are objective automatic measurements for evaluating new architectures. It is not right to place tables without some explanation and show the perplexity and BLEU scores. For example, in Table 2, it is necessary to explain why the LSTM's perplexity from the previous work is higher than the author's implementation. The same applies to Table 3.

The result presented in Table 4 does not match the description in Section 4.3:
*   It is not true that PRRNA surpasses both PBSMT and NC-Dec models. The authors should clarify that scores vary in different evaluation sets. And it is the average test score that PRRNA does well.
*   Please also clarify whether "test average" is a micro-average (all test sets are combined and evaluated as one set) or a macro-average (the average is taken across individual test set scores).

For Table 4, please include the significance of the BLEU improvement made by PRRNA compared to the baseline, see https://github.com/jhclark/multeval

**General Discussion:**

Since the main contribution of this work is on the phrasal effect of the new RNA architecture, it is quite important to show that the phrases are more consistent than the vanilla LSTM/RNA model. Therefore, BLEU evaluation is insufficient. More attention needs to be paid to evaluating the phrases on a subset of the evaluation set in support of the claim.

**Is there an attention mechanism in the baseline system (groundhog)?**

*   If there is, please describe it more clearly in paragraph 4.2 and Table 4.
*   If there is not, remove the attention layer after the encoder in Figure 5.

Also, the lack of an attention mechanism provides a disadvantage for the baseline NC-Dec system, and it is not clear whether PRRNA can surpass the NC-Dec system with an attention mechanism or be an addition feature. This unfair disadvantage becomes even more pronounced when PRRNA uses multiple phrasal attention layers within a single sentence, where a general NC-Dec system is used as a benchmark without attention.

**Question:** Isn't an easy way to get phrasal RNA to put the "pyramid" RNAs of a sentence into an average pooling layer?

This paper proposes a new partial RNN architecture for sequence generation from a sequence. They have evaluated their architecture based on Chinese-English machine translation tasks on the PTB and FBIS-trained language modeling test sets and the NIST MT02-08 evaluation set. The partial RNN (PRNN) architecture is obtained by generating subnetworks of phrases.

**Strengths**
====
A new partial architecture.

**Weaknesses**
====
Technical:
It is not clear whether a limit has been set on the phrase length of the PRNN.
Perhaps I have left it out in the paper, if so, please explain it more clearly because it significantly affects the model if the largest phrase length for each sentence is the length of the sentence.
- This is because if the largest phrase length is the length of the sentence, the model can be simplified into some kind of convolution RNN where each position of the RNN goes through a few convolution layers before the final softmax and attention.
- If a limit is set on the phrase length of the PRNN, it makes the system smoother. But this would also mean that the phrases are determined by token engrams that generate a sliding window of "pyramid encoders" for each sentence where there are examples where the parameters for these phrases will be set close to zero to disable them, and these phrases will be a good internal evaluation of the PRNN in addition to evaluating purely on perplexity and BLEU.
Using a attention mechanism without any kind of pruning at the partial level can be problematic. The author has opted for some kind of greedy pruning as described in the caption of Figure 4. But I support looking at a fixed set of phrase pairs at train time, the pre-computation of the attention mechanism at the partial level can be done but on inference (apply attention to new data at test time), this can be somewhat problematic when the architecture is scaled to a large dataset.

**Experimental:**
One issue with the language modeling experiment is the choice of evaluation and train set. Perhaps datasets like Common Crawl or NVI8 would be more suitable for language modeling experiments.
The main issue with the paper is in the reporting of experiments and results, it needs a lot of improvement.
- The evaluation on PTB (Table 2) is not appropriate because the model was trained on a large corpus (FBIS) and then tested on PTB. The fact that the previous study reported a 126 perplexity baseline using LSTM and LSTM's 106.9 perplexity, which was provided by the author, showed that testing on PTB gives an advantage in calculating the perplexity of the FBIS language model.
- Also, regarding Section 3.3, please cite appropriate publications for the "previous work" presented in the tables. And was the same training set used in the previous work?
- Furthermore, why is the GRU version of PRNNV not given in Table 3 in the FBIS evaluation?
The results section cannot be presented as a table without explanation:
- Still on the results sections, although it is clear that BLEU and perplexity are objective automated measures to evaluate the new architecture. It is not really good to keep the tables and show the perplexity and BLEU scores without any explanation. For example, in Table 2, it is necessary to state why the perplexity of LSTM from the previous work is higher than the author's implementation. The same is true in Table 3.
The results presented in Table 4 do not match the description given in Section 4.3:
- It is not true that PRNN performs better than both PBSMT and NC-DC models. The authors should clarify that on different evaluation sets, the scores are different. And it is the average test score that PRNN performs better.
- Please also clarify whether the "test average" is a micro-average (all tests sets are combined and evaluated as one set) or a macro-average (the average is taken in the scores of individual test sets) score.
For Table 4, please also include the importance of the BLEU improvement made by PRNN with respect to the baseline, see
https://github.com/jhclark/multeval

**General Discussion**
====
Since the main contribution of this work is on the partial effect of the new RNN architecture, it is important to show that the phrases are more consistent than the vanilla LSTM / RNN model. Thus the BLEU evaluation is insufficient. It will be necessary to keep a closer eye on evaluating the phrases in a subset of the evaluation set to support the claims.
Does the baseline system (groundhog) include a attention mechanism?
- If yes, please describe it more specifically in Section 4.2 and Table 4.
- If no, please remove the attention layer after the encoder in Figure 5.
Also, the lack of a attention mechanism provides a disadvantage to the baseline NC-DC system and it is not clear whether PRNN can perform better in an NC-DC system with a attention mechanism or be an additive feature. The undue disadvantage is even more prevalent when PRNN uses several partial attention layers within the same sentence while a simple NC-DC system without attention is used as a benchmark =(
Question: What would be a simple way to obtain partial RNN that puts a "pyramid" RNN of a phrase into some kind of average pooling layer?

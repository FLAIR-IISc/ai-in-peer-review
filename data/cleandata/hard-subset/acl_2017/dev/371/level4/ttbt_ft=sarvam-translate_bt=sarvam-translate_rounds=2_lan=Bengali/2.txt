This paper proposes a new Phrasal RNA architecture for generating sequences from sequences. They have evaluated their architecture based on language modeling tests evaluated on PTB and FBIS, and Chinese-English machine translation tasks based on the NIST MT02-08 evaluation set. The Phrasal RNA (PRRNA) architecture is achieved by creating a subnetwork of phrases.

Strengths:
A new Phrasal architecture.

Weaknesses:
Technical:
It is not clear whether a limit has been set on the length of the PRRNA phrase. Perhaps I forgot to mention this in the paper, but if so, please mention it more clearly because it significantly affects the model if the largest phrase length is equal to the sentence length for each sentence.
Because if the largest phrase length is equal to the sentence length, the model can be simplified to a special type of convolution RNA, where each state of the RNA goes through some convolution layers before final smoothing and attention.
If a limit is set on the length of the PRRNA phrase, it simplifies the system. However, this means that the phrases are determined by token encoding, which creates a sliding window of "Pyramid encoders" for each sentence, where the parameters of these phrases will be set close to zero so that the phrases can be deactivated, and this phrase will have a good underlying evaluation of the PRRNA as well as being evaluated only on complexity and BLEU.
Using the attention process without some pruning can be problematic at the Phrasal level. The author has chosen some greedy pruning described in the caption of Figure 4. However, I support that if a specific set of phrase pairs is given during training, the attention process at the Phrasal level can be pre-computed, but it may be a bit problematic if the architecture is scaled to a larger dataset during inference (apply attention to new data during testing).

Experience:
One problem with language modeling tests is the selection of the evaluation and training set. Perhaps datasets like Common Crawl or NWiki8 would be more suitable for language modeling tests.
The main problem with the paper is the testing and result reporting; it needs to be rearranged quite a bit.
The evaluation in PTB (Table 2) is not fair because the model was trained on a larger corpus (FBIS) and then tested in PTB. The previous study used LSTM to report 126 complexity baselines, and the author's LSTM reported a complexity of 106.9, which shows that the FBIS language model makes it easier to calculate complexity when testing in PTB.
Also, regarding paragraph 3.3, please mention the appropriate publication related to "Previous work" in the table. And is the previous work using the same training set?
Furthermore, why was the GRU version of PRRNA not reported in the FBIS evaluation in Table 3?

The results section cannot present a table without explanation:
Even in the results section, although it is clear that BLEU and complexity are objective automatic measurements for new architecture evaluation. It is not right to place a table without some explanation and show the complexity and BLEU scores. For example, in Table 2, it is necessary to explain why the complexity of LSTM from the previous work is higher than the author's implementation. The same applies to Table 3.

The result presented in Table 4 does not match the description in Section 4.3:
It is not true that PRRNA has surpassed both PBSMT and NC-Dec models. The authors should clarify that scores vary in different evaluation sets. And it is the average test score that PRRNA does well.
Please make it clearer whether the "average of tests" is a micro-average (all test sets are combined and evaluated as one set) or a macro-average (the average is taken within individual test set scores).

For Table 4, please include the significance of the BLEU improvement that PRRNA made compared to the baseline, see https://github.com/jhclark/multeval

General Discussion:
Since the main contribution of this work is on the Phrasal effect of the new RNA architecture, it is quite important to show that the phrases are more consistent than the vanilla LSTM/RNA model. Therefore, BLEU evaluation is insufficient. Further attention needs to be paid to evaluating the phrases in a subset of the evaluation set to support the claim.

Is there a attention process in the baseline system (groundhog)?
If so, please describe it more clearly in paragraph 4.2 and Table 4.
If not, remove the attention layer after the encoder in Figure 5.

Also, the lack of an attention process provides a disadvantage for the baseline NC-Dec system, and it is not clear whether NC-Dec systems with PRRNA attention can surpass or be an additional feature. This unfair disadvantage becomes even more pronounced when PRRNA uses multiple Phrasal attention layers within a single sentence, where a general NC-Dec system is used as a benchmark without attention.

Question: Isn't there an easy way to get Phrasal RNA to keep the "Pyramid" RNAs of a sentence in an average pooling layer?

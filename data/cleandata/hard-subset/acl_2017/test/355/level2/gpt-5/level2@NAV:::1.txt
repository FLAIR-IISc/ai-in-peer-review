Summary and claimed contributions
- A multi-sequence Grid-RNN architecture that models interactions among all predicates in a sentence for Japanese PAS without relying on syntactic parses, yielding new state-of-the-art results on NAIST 1.5.
- A strong single-sequence Bi-RNN baseline that already surpasses prior syntax-based systems, showing that sentence-level contextualization is highly competitive.
- Empirical analysis of network depth and residual connections, with statistically significant improvements, and a detailed breakdown by case roles, highlighting sizable gains for zero arguments.

Strengths
- Empirical effectiveness with statistical rigor: On NAIST 1.5 (intra-sentential), the multi-sequence model improves overall F1 to 81.42 vs 79.23 (Ouchi et al., 2015) and notably boosts Zero from 44.09 to 47.12, with bootstrap significance (p < 0.05). This supports the core claim that multi-predicate interactions can be learned without syntax.
- Robustness without syntactic features: Eliminating parser dependence reduces error propagation, yet achieves superior accuracy, demonstrating practical value for domains/languages lacking reliable parsers.
- Insightful ablations: Depth and residual connections yield ~1 F1 improvement, indicating careful engineering and providing guidance for future model scaling.
- Detailed role-wise analysis: Large gains for DAT and Zero NOM suggest the models capture sparse roles and long-distance dependencies better than prior joint/syntax-heavy systems.

Weaknesses
- Modest gain from multi-sequence over single-sequence: The overall improvement is small (81.42 vs 81.15), raising questions about the cost-benefit of the Grid-RNN complexity vs simpler shared encoders with predicate indicators.
- Missing structural constraints: The softmax-per-token design risks multiple arguments per case or none; no decoding or global constraints (e.g., ILP/CRF) are described. Clarification on constraints/post-processing is needed.
- Reproducibility gaps: The code link is placeholder; details on candidate set formation, handling class imbalance (dominant null class), and runtime/memory complexity as predicates increase are limited.
- Limited comparisons: No oracle/ablation showing how much syntax could still add, nor alternative interaction mechanisms (e.g., attention across predicates) to isolate what the grid contributes beyond depth/context.
- Scope: Inter-sentential (Inter-Zero) is excluded; while common for conference scope, a discussion of applicability or obstacles would strengthen the paper.

Questions to authors
- How do you enforce one argument per case per predicate, if at all? Any post-hoc constraints?
- What is the candidate set for arguments—are all tokens eligible, and how are punctuation/function words handled?
- What is the training/inference complexity as a function of sentence length T and number of predicates M, and observed runtimes on test?
- Did you compare Grid-RNN to a shared Bi-RNN/Transformer with predicate markers and cross-predicate attention?
- Any analysis of errors by predicate-argument distance and by number of predicates per sentence?

Constructive suggestions: add a constrained decoding layer; include ablations against simpler cross-predicate mechanisms; report compute; release code; and provide qualitative/error analyses to illuminate where inter-predicate signals help most.

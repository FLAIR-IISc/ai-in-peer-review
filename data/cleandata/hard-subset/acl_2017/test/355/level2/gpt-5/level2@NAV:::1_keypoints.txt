- Claim: Proposes a multi-sequence Grid-RNN that models interactions among all predicates for Japanese PAS without syntactic parses, achieving new SOTA on NAIST 1.5.
- Claim: Presents a strong single-sequence Bi-RNN baseline that already outperforms prior syntax-based systems, showing sentence-level contextualization is highly competitive.
- Claim: Provides empirical analysis of network depth and residual connections with significant improvements and a case-role breakdown showing large gains for zero arguments.

- Strength: Empirical effectiveness with statistical rigor — multi-sequence model improves overall F1 and Zero arguments with bootstrap significance (p < 0.05).
- Strength: Robust performance without relying on syntactic features, reducing parser error propagation and aiding applicability to domains/languages lacking parsers.
- Strength: Insightful ablations showing depth and residual connections each add ~1 F1, guiding model scaling choices.
- Strength: Detailed role-wise analysis highlights notable gains for sparse/long-distance roles (e.g., DAT, Zero NOM).

- Weakness: Small marginal gain of multi-sequence over single-sequence (81.42 vs 81.15), questioning Grid-RNN complexity vs simpler alternatives.
- Weakness: Model uses softmax-per-token and lacks described global structural constraints, risking multiple/no arguments per case.
- Weakness: Reproducibility gaps — placeholder code link and missing details on candidate set formation, class imbalance handling, and runtime/memory scaling.
- Weakness: Limited comparisons — no oracle/syntax-augmented baselines or alternative cross-predicate mechanisms (e.g., attention) to isolate the grid’s contribution.
- Weakness: Scope excludes inter-sentential (Inter-Zero) phenomena; applicability to inter-sentential cases is not discussed.

- Question: How is one argument per case per predicate enforced, if at all; are there any post-hoc constraints or decoding steps?
- Question: What is the candidate argument set — are all tokens eligible and how are punctuation/function words treated?
- Question: What are training/inference complexity bounds as functions of sentence length T and predicate count M, and what are observed runtimes?
- Question: Was Grid-RNN compared to a shared Bi-RNN/Transformer with predicate markers plus cross-predicate attention?
- Question: Any error analysis by predicate-argument distance and by number of predicates per sentence?

- Suggestion: Add a constrained decoding layer (e.g., ILP/CRF) to enforce structural constraints.
- Suggestion: Include ablations against simpler cross-predicate mechanisms (shared encoders with predicate indicators, attention).
- Suggestion: Report compute/runtime/memory details and release code for reproducibility.
- Suggestion: Provide qualitative/error analyses to show where inter-predicate signals help most.

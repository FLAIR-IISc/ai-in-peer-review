Summary
This paper tackles Japanese predicate-argument structure (PAS) analysis, focusing on intra-sentential arguments (Dep and Zero). The key idea is to model multi-predicate interactions without relying on syntactic parsers, thereby avoiding pipeline error propagation. The authors present: (i) a single-sequence Bi-RNN model that predicts case roles for all tokens with respect to a single predicate, and (ii) a multi-sequence Grid-RNN that jointly processes all predicates in a sentence to capture inter-predicate interactions via inter-sequence recurrences and residual connections. On NAIST Text Corpus 1.5, the models outperform prior state-of-the-art systems, including syntax-based joint models (Ouchi et al., 2015), with the Grid-RNN yielding the best overall F1 and larger gains for zero arguments.

Main contributions (as I see them)
1) Empirical advance: A strong neural baseline for Japanese PAS without syntactic features that achieves state-of-the-art performance on NAIST 1.5, including substantial improvements on zero arguments over prior joint syntax-based models.
2) Architectural contribution: A practical adaptation of Grid-RNNs to jointly model multi-predicate interactions by connecting per-predicate token sequences, with residual connections enabling deeper stacks.
3) Analysis of depth/residuals: Evidence that deeper stacks with residual connections improve performance, and that multi-predicate modeling particularly benefits zero anaphora.

Strengths
- Clear empirical gains without syntax: Both the single-sequence and multi-sequence models outperform strong baselines on NAIST 1.5 (All F1: 81.15/81.42 vs 79.23 for Ouchi+15), with statistically significant improvements reported. Notably, Zero F1 improves from 44.09 (Ouchi+15) to 46.10 (single) and 47.12 (multi), addressing a core difficulty in Japanese PAS.
- Effective multi-predicate modeling: The Grid-RNN introduces inter-sequence connections across predicates and yields consistent improvements over the single-sequence model (e.g., Zero F1 +1.0), corroborating prior linguistic intuition and prior findings (Ouchi+15) that multi-predicate interaction is helpful, while avoiding reliance on syntactic parsers.
- Sensible design and training details: The paper carefully specifies input features (predicate-aware tokenization, predicate markers, local predicate context), training setup, and hyperparameter choices. The depth/residual study shows that residual connections enable deeper stacks and deliver ~1 point F1 gains, making the engineering contribution concrete.
- Strong zero-case improvements and case-wise breakdown: The per-case analysis (Table 3) highlights notable improvements, especially for dative and zero nominative cases, aligning with the claim that RNN context modeling captures long-distance dependencies useful for zero anaphora.
- Relevance and simplicity: The approach is conceptually simple (sequence labeling with predicate conditioning; grid connections across predicates), broadly applicable to SRL, and avoids heavy feature engineering or external resources.

Weaknesses
- Limited novelty of the core architecture: The single-sequence model is a fairly standard Bi-RNN sequence labeling setup; the multi-sequence Grid-RNN is a straightforward adaptation of known grid/multidimensional RNNs. The novelty lies more in application to PAS and empirical validation than in methodological innovation.
- Inference/constraint handling is unclear: Using per-token softmax over {NOM, ACC, DAT, PRED, null} can produce multiple tokens with the same case for a predicate, violating one-argument-per-role constraints typically enforced in PAS. The paper does not describe any decoding or global constraints to resolve conflicts, nor quantify how often conflicts occur or affect F1. This is especially important in comparison to joint ILP-based baselines.
- Ablation and analysis are thin: Beyond single vs multi and residuals, the paper lacks ablations on input features (e.g., removing MARK, varying predicate context window C, relative position embeddings), the contribution of inter-sequence connections alone, or alternative multi-predicate models (e.g., attention between predicates). Error analysis for Zero (e.g., distance, case markers, topic/subject sharing) is minimal.
- Fairness and clarity of comparisons: While the splits match prior work, (i) significance is only marked for Single vs Multi, not vs prior SOTA; (ii) Table 3 contains surprising numbers (e.g., Zero NOM F1 > 60 while overall Zero is ~46), suggesting possible mislabeling or aggregation inconsistencies; and (iii) the paper does not discuss computational cost or scalability with many predicates (Grid-RNN complexity grows with M and T).
- Reproducibility details: The optimizer description is imprecise (“SGD” with Adam adjusting learning rate—do you use Adam or SGD?), and the code link is placeholder (“http:xxx”). Given Theano and relatively small embeddings, exact settings/initializations and seed sensitivity should be clarified.

Questions to authors
- How do you enforce role uniqueness per predicate (e.g., at most one NOM/ACC/DAT)? If you do not enforce constraints, how often do conflicts occur, and how are they resolved for evaluation?
- Please clarify Table 3: how are the per-case F1 values computed for Dep vs Zero subsets? Some numbers (e.g., Zero NOM ≈ 65) seem inconsistent with the overall Zero F1 ≈ 47.
- What unit did you bootstrap over for significance testing (sentence, predicate, or predicate–role decisions)? Are the gains over Ouchi+15 statistically significant?
- Can you provide an ablation of input features and inter-sequence connections (e.g., removing MARK or predicate-window features; using only intra-sequence recurrence; adding relative position embeddings)?
- What is the computational cost and memory usage of the Grid-RNN as M (number of predicates per sentence) increases? Any batching/parallelization strategies?
- Did you try pre-trained embeddings or simple syntactic cues (e.g., dependency heads) to test complementarity? If not, do you expect additive gains?

Additional comments
- The related work discussion is appropriate; consider comparing against a strong BiLSTM with predicate-relative position embeddings and a simple attention across predicates as alternative multi-predicate baselines.
- Clarify whether experiments average across random seeds. Given modest deltas (e.g., +0.27 All F1 single→multi), reporting mean±std would help.
- The optimizer description should be corrected; if you use Adam, say so explicitly and omit “SGD.”

Recommendation: Weak Accept. The empirical gains on a standard benchmark, especially for zero anaphora and without syntax, and the straightforward, well-motivated multi-predicate modeling outweigh the moderate novelty and analysis gaps. Strengthening the decoding/constraint story and ablations would further improve the paper.

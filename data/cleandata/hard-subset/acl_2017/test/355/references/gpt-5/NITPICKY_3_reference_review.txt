Summary and main contributions
This paper proposes neural models for Japanese predicate-argument structure (PAS) analysis that avoid explicit syntactic features and instead learn context from word sequences. Two architectures are presented: (i) a single-sequence stacked BiRNN that labels, for a given predicate, every token with one of {NOM, ACC, DAT, PRED, null}; and (ii) a multi-sequence Grid-RNN that jointly processes all predicates in a sentence and propagates information across predicates to model multi-predicate interactions. Experiments on NAIST Text Corpus 1.5 (intra-sentential PAS only) show improvements over prior work, notably on zero arguments.

As I see it, the contributions are:
1) A grid-style recurrent architecture that models multi-predicate interactions for Japanese PAS without syntactic features, yielding the best reported F1 on NAIST 1.5 and stronger zero-arg performance.
2) A simple, syntax-free single-sequence BiRNN baseline that already surpasses strong prior baselines, highlighting the value of sentence-level contextualization for PAS.
3) An empirical observation that deeper RNN/Grid-RNN with residual connections improves performance (though reporting is incomplete).

Strengths
- Strong empirical gains without syntax: Both the single-seq and multi-seq models outperform prior state-of-the-art (Ouchi et al., 2015) on NAIST 1.5 despite using no syntactic/selectional-preference features. Overall F1 improves from 79.23 to 81.42; zero-argument F1 improves from 44.09 to 47.12. This is practically meaningful given the difficulty of zero arguments in pro-drop.
- Joint modeling across predicates: The multi-sequence Grid-RNN significantly outperforms the single-seq model (81.42 vs. 81.15 overall; 47.12 vs. 46.10 zero-arg), substantiating the linguistic intuition that predicates interact and that modeling such interactions helps, particularly for zero arguments.
- Solid comparison baselines on the same corpus: The paper compares against standard NAIST 1.5 systems (Imamura et al., 2009; Ouchi et al., 2015) using the same splits, with significance testing (bootstrap), making the core empirical claim credible.
- Minimal feature engineering: The model relies on simple word/mark embeddings and local predicate context features, demonstrating that much of the PAS signal can be learned from token sequences, reducing dependence on potentially error-prone parsers.

Weaknesses
1) Reproducibility and reporting issues:
- Code: The link “http:xxx” is a placeholder; no actual code/resources provided.
- Optimizer ambiguity: “SGD using mini-batch, learning rate adjusted using Adam” is contradictory; specify whether Adam is the optimizer.
- Hyperparameters unclear/inconsistent: GRU sizes described as “32×32,” and for multi-seq “input-related 64×32, others 32×32,” but the exact dimensionalities of inputs/hidden states (especially with concatenated inter-sequence inputs) are not clearly defined.
- Missing details on significance testing (number of samples/iterations, what metric bootstrapped).
- Table 2 is referenced but not fully included here; depth/residual results cannot be verified.
- Many formatting/citation glitches (e.g., “?” placeholders, footnotes in running text, inconsistent figure/table references). These hinder clarity and suggest the paper is not camera-ready.

2) Decoding and constraints not specified:
- The model assigns one label per token per predicate via softmax. For dependency-based PAS, at most one argument per case per predicate is allowed. How is this enforced at inference? Is there a post-hoc selection (e.g., pick the max-scoring token per case) or any global constraint layer (e.g., ILP/structured decoding)? Without constraints, multiple NOMs per predicate could be produced. The paper does not specify decoding, tie-breaking, or constraint handling, nor does it analyze violation rates.

3) Insufficient ablations/analysis:
- No ablation of input features (ARG vs PRED window vs MARK). The added value of the PRED-window features is not quantified.
- No alternatives to Grid-RNN compared (e.g., simple cross-predicate attention, pooling over predicate hidden states, 2D bi-RNN); thus it is unclear whether the grid architecture is necessary vs. any cross-predicate message passing.
- No error analysis or stratified analysis (by distance, sentence length, number of predicates, ambiguity). The claim of capturing long-distance dependencies would benefit from targeted evaluation.

4) Generalization and scope:
- Only NAIST 1.5 intra-sentential is evaluated. No cross-domain/corpus evaluation, no analysis on 1.4β under the updated DAT criterion beyond a table of prior results. The claim “state-of-the-art without syntax” remains corpus-specific.
- No runtime/complexity analysis. Grid-RNN scales with the number of predicates M and sequence length T; memory/time implications are not discussed, which matters for sentences with many predicates.

5) Modeling choices and training details:
- No regularization beyond L2; no dropout, layer norm, or gradient clipping details. Given deep recurrent stacks, this is surprising.
- OOV handling: words with freq <2 mapped to UNK, no pretraining, no subword/character features for Japanese. While the model still improves, this choice likely hurts rare zero-arg identification and is atypical for modern NLP.
- Output label set includes PRED even though MARK embedding already indicates predicate; the effect of this design choice is not justified.

Given these numerous (mostly fixable) issues in reporting, clarity, and experimental thoroughness, the technical novelty feels incremental (Grid-RNN applied to PAS), and the presentation does not meet ACL standards.

Questions to authors
- Decoding/constraints: How do you enforce at most one argument per case per predicate at inference time? If you do not enforce constraints, what is the rate of duplicates per case? Please detail the inference procedure.
- Grid details: What is the exact dimensionality of inputs/hidden states per layer for single- and multi-seq? In Eq. for the grid layer, how is the GRU applied with two “previous” vectors; which part is treated as input vs. recurrent state? Do you perform bidirectionality along the predicate dimension (m) by alternating layers as implied?
- Optimizer/training: Did you use Adam as the optimizer? Any gradient clipping, dropout, or other regularization? What were the final depths chosen per model and the exact Table 2 numbers?
- Significance: How many bootstrap samples, what unit of resampling (sentences?), and which metrics were tested?
- Ablations: What is the contribution of the PRED window features and MARK vs. ARG alone? Can you compare Grid-RNN to simpler cross-predicate attention or shared self-attention?
- Efficiency: What are training/inference times, and how does cost scale with the number of predicates per sentence?

Recommendation: Reject (for now). The empirical result is promising, but the paper needs substantial revision for clarity, reproducibility, decoding specification, and ablations to establish the necessity of the proposed grid architecture.

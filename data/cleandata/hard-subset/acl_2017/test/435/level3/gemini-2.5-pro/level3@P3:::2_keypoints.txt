- Task: classify causal vs. non-causal relations for ambiguously marked clauses (e.g., "since") using the AltLex dataset and a neural Pair LSTM with GloVe, claiming new SOTA.
- Primary contribution: demonstrates an LSTM-based model can outperform a feature-engineered SVM baseline on the AltLex causal disambiguation task.
- Secondary contribution: provides qualitative examples showing the model disambiguates markers using context rather than marker priors.
- Strength: clear problem formulation and solid motivation with good literature context.
- Strength: achieves SOTA empirical results, improving F1 by ~0.5–1.5% on the bootstrapped dataset.
- Strength: code released and includes quantitative experiments plus qualitative error analysis aiding reproducibility.
- Weakness: apparent test-set-based hyperparameter/model selection (Table 5), risking overfitting and inflated performance.
- Weakness: limited novelty—architecture is a standard LSTM application—and lacks simpler neural baselines to justify complexity.
- Weakness: no statistical significance testing for modest improvements; AltLex may contain annotation noise affecting reliability.
- Weakness: missing methodological details—bootstrapping procedure for the extended corpus is not described; limited discussion of precision/recall behavior.
- Question: clarify differences from implicit discourse relation recognition and why those methods weren't used as stronger baselines.
- Question: clarify model selection procedure—was the final model chosen via a development set or test-set comparisons?
- Question: provide a brief description of the bootstrapping process used to create the extended AltLex corpus.
- Question: document the TSV 0-1-2 coding scheme and confirm redistribution rights for the AltLex data.

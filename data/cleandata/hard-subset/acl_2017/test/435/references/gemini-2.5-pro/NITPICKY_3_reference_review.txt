**Summary and Contributions**

This paper addresses the task of causal meaning disambiguation, which involves determining if a relation between two text spans (events) is causal. The authors argue against reliance on fixed syntactic patterns or lexical resources, proposing instead that a neural network can effectively encode the context surrounding the events to make this determination. They introduce a two-input LSTM-based architecture that processes the two events and an optional lexical marker. The model is evaluated on the AltLex corpus, where it is shown to outperform the previous state-of-the-art system, which was based on extensive feature engineering.

The main contributions of this work, as I see them, are:
1.  An empirical demonstration that a standard neural sequence encoding model can outperform a complex, feature-engineered system on the AltLex benchmark for causal meaning disambiguation.
2.  An analysis of different model configurations (e.g., padding strategies, optimizers, independent vs. state-sharing LSTMs) for this specific task, offering some insight into what works best for modeling this type of relation.

**Strengths**

1.  **Addresses a Relevant Problem:** Causal relation identification is a fundamental and challenging task in natural language understanding with applications in question answering, summarization, and information extraction. The paper's focus on moving beyond brittle, feature-based systems towards more flexible neural models is well-aligned with current research trends.

2.  **Strong Empirical Results:** The proposed models demonstrate a clear and consistent improvement over the previous state-of-the-art system (B2) on both versions of the AltLex corpus. For instance, the best model on the "non bootstrapping" corpus improves F1-score by 1.64% and accuracy by a significant 6.04%. The inclusion of experiments on a balanced dataset further strengthens the empirical evaluation.

3.  **Effective Use of Context:** The qualitative analysis in Table 8, though brief, provides compelling evidence for the authors' central hypothesis. The model correctly classifies instances where a typically causative verb like "break" is used non-causally, and handles ambiguous connectives like "which then". This supports the claim that the model is learning from the broader context rather than simply memorizing lexical triggers.

**Weaknesses**

1.  **Methodological Novelty is Limited:** The core architecture, "Pair LSTM," is a very standard Siamese-like network for sentence-pair classification tasks. It consists of two parallel LSTMs whose outputs are concatenated and passed to a classifier. While its application to this problem is new and effective, the paper frames it as a contribution of a "neural network architecture," which is an overstatement given the architecture's ubiquity. The more novel "Stated Pair LSTM" variant is shown to be less effective, which undermines its contribution.

2.  **Lack of Clarity and Reproducibility:** The paper omits several crucial details needed for replication and full understanding.
    *   The padding strategy is highly unconventional and poorly explained. The authors mention using the "maximum," "mean," and "mode" of input lengths for zero-padding. It is unclear how one pads to a "mean" or "mode" length, and whether this involves truncating sequences longer than this threshold—a critical detail that is not mentioned.
    *   Key hyperparameters are missing. The paper does not report the LSTM hidden state size, the number of dense layers and their dimensions, or the final dropout and regularization values used for the best-performing models.
    *   The paper refers multiple times to a "Table 3" containing corpus statistics, but this table is not present in the submitted paper. This is a significant omission.

3.  **Insufficient Experimental Rigor and Analysis:**
    *   Statistical significance testing is only reported for the secondary experiment on the reduced dataset. The main results in Table 5 lack significance tests, making it difficult to ascertain if the improvements over the B2 baseline are statistically meaningful or merely due to random chance.
    *   The comparison is limited to the single system from the original AltLex paper. Stronger neural baselines, such as a CNN-based model (as cited by the authors in other work) or even a simpler model using averaged word embeddings, would have provided a more comprehensive comparison and better contextualized the performance of the LSTM architecture.
    *   The explanation of Table 4 is confusing. The definitions for categories like "Unambiguous in Causal class" and "Causal in train, Non Causal in test" are not precise enough to fully appreciate the difference between the two corpus versions.

4.  **Poor Presentation and Formatting:** The paper is marred by a substantial number of formatting errors and inconsistencies that detract from its professional quality and suggest a lack of careful preparation.
    *   The reference list is in a state of disarray. It contains LaTeX formatting errors (e.g., `\Q2017\E`), inconsistent capitalization, missing venue details, stray question marks, and improperly formatted author lists (e.g., `Do et al\.,?`). This is unacceptable for a conference submission.
    *   There is a footnote marker "1" next to the title of Table 5, but no corresponding footnote text.
    *   In Table 7, some numbers use a comma as a decimal separator (e.g., "69,84%"), which is inconsistent with the rest of the paper.
    *   There are numerous grammatical errors and awkward phrasings throughout the text (e.g., "between others," "the grade of coverage").

**Questions to Authors**

1.  Could you please provide a precise explanation of your "mean" and "mode" padding strategies? Specifically, how are sequences that are longer than the mean or mode length handled?
2.  Please provide the missing hyperparameters for your best-performing models reported in Table 5, including LSTM hidden size, learning rate, and the exact dropout and L2 regularization values used.
3.  Are the improvements over the B2 baseline in Table 5 statistically significant? If so, please provide the test used and the p-values.
4.  Table 3 is referenced multiple times but is missing. Could you please provide this table?

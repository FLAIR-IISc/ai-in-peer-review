- Paper addresses causal meaning disambiguation for lexical markers by modeling two event contexts with LSTM-based two-input classifiers.
- Main contribution: a “Pair LSTM” that outperforms a per-connector majority baseline and a feature-based SVM on the AltLex corpus.
- Secondary contribution: empirical comparison of encoder coupling, pooling strategies, optimizers, and a small qualitative analysis of ambiguous markers.

- Clear, focused task separation between causal meaning disambiguation and argument identification.
- Empirical gains over reasonable baselines on both original and bootstrapped AltLex splits, especially for the causal class.
- Lightweight input design using only pretrained GloVe embeddings, suggesting portability.
- Sensible negative finding: initializing the second encoder from the first (Stated Pair LSTM) underperforms independent encoders.
- Some statistical testing (McNemar) shown in reduced-imbalance experiments.

- Limited novelty: simple pairwise LSTMs without explicit interaction likely not state-of-the-art.
- Missing strong baselines: no comparisons to transformer-based sentence-pair models (BERT/RoBERTa) or recent neural discourse models.
- Methodological clarity issues: confusing description of “zero-padding”/pooling and ill-defined "mode" on embeddings.
- Key hyperparameters and tuning details are missing (LSTM depth, hidden size, batch size, epochs, learning rate, early stopping, gradient clipping).
- Unclear whether a development set was used and whether test tuning occurred.
- Evaluation limited to a single dataset (AltLex) with class imbalance and possible stylistic biases; no cross-dataset generalization shown.
- Poor reporting of metrics: only causal-class metrics and accuracy, lacking per-marker breakdowns, macro averages, and confidence intervals.
- Missing ablations and controls (e.g., removing marker l, using only l, swapping e1/e2, masking lexical content) to rule out lexical memorization.
- Data handling concern: “balanced” evaluation by downsampling Non-Causal in test set changes distribution; better approaches (class-weighting, resampling) not used.
- Related-work gaps: ignores much literature on neural discourse and transformer approaches; argument against CNNs is overstated.

- Questions: request exact train/dev/test splits, use of dev set, number of runs/seeds, and mean±std reporting.
- Questions: request specific architecture hyperparameters, dropout/regularization, sequence length/truncation, tokenization, OOV handling, optimizer settings.
- Questions: clarify pooling vs. padding operations and definition of “mode” on vectors.
- Questions: clarify whether baseline numbers were reproduced and how features were aligned to prior work.
- Questions: request experiments controlling lexical memorization (remove/use only marker, mask words, marker-disjoint split).
- Questions: request per-marker precision/recall, especially for ambiguous markers and cases where majority label flips.

- Suggestions: add stronger baselines (fine-tuned BERT/RoBERTa/DeBERTa, cross-attention models) and improve interaction modeling (token-level cross-attention, ESIM-style).
- Suggestions: broaden evaluation to PDTB/Causal-TimeBank, report macro-F1, and include significance tests.
- Suggestion: release code and detailed hyperparameters for reproducibility.
- Writing and presentation need improvement; some tables/figures are incomplete or mislabeled.

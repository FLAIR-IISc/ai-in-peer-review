- Task: Causal meaning disambiguation — decide if relation between event spans e1 and e2 signaled by lexical item l is causal.
- Approach: Two-branch LSTM architectures encoding e1 and l+e2 with pretrained GloVe and dense layers.
- Evaluation: Tested on AltLex corpus in “non-bootstrapping” and “bootstrapping” splits against per-connective majority and Hidey & McKeown SVM.
- Main empirical claim: Simple two-branch LSTM outperforms a feature-rich SVM on AltLex across corpus variants.
- Framing contribution: Clear separation of causal meaning disambiguation from causal argument identification with an event-span + marker input format.
- Empirical insights: Comparison of LSTM variants (with/without state transfer), optimizers (Adam vs Adadelta), and simple aggregation strategies with noted precision–recall trade-offs.
- Results: Consistent accuracy gains and modest but consistent F1 improvements for the Causal class; best gains reported +2.1 F1 and +7.7 accuracy in one setting.
- Ablation/analysis: Includes small qualitative examples showing neural model wins on ambiguous connectives and causative verbs.
- Novelty concern: Using LSTMs for span-pair classification is standard; technical novelty is limited.
- Architectural clarity issues: Notation and description are inconsistent/underspecified (padding vs pooling, where LSTM outputs are used).
- Missing baselines: No comparisons to simpler neural baselines (avg-embeddings+MLP, single BiLSTM, attention) or stronger encoders (transformers).
- Missing ablations: No tests removing the lexical marker or using only e1/e2 to assess reliance on l.
- Data reporting problems: Tables contain typos/inconsistencies; class distributions, split sizes, and subset choices (explicit/implicit) are unclear.
- Reproducibility gaps: No released code/data processing details, incomplete hyperparameter/run details, and missing training specifics (epochs, early stopping).
- Significance reporting: Limited statistical testing — only downsampling experiment tested and main results lack significance vs strongest baseline.
- Presentation quality: Grammar, formatting, and figure/table artifacts hinder clarity and readability.
- Request for architecture detail: Clarify whether pooling is applied to embeddings or LSTM states and what final sequence vector is used.
- Request for span extraction detail: Explain how e1, l, e2 are obtained from AltLex and handling of implicit relations or empty l.
- Request for split/counts: Provide consistent train/dev/test sizes and causal/non-causal counts for both corpus versions and split protocol used.
- Request for additional experiments: Provide average-embeddings MLP, single BiLSTM over [e1 SEP l+e2], ablations removing l or using only e1/e2, and report multiple runs with variance and significance vs B2.
- Request for deeper error analysis: Report systematic gains by marker families or event types and performance when connective distributions differ between train/test.

Summary and contributions
This paper addresses causal meaning disambiguation: given two event spans (e1, e2) and a lexical marker l susceptible of signaling causality, decide whether the relation is Causal or Non-Causal. The authors argue that causality is not tightly tied to fixed syntactic patterns or a small set of connectives; thus, context encoding is needed. They propose LSTM-based architectures to encode the two arguments and the marker, and evaluate on the AltLex corpus (Hidey and McKeown, 2016), comparing against (i) a majority-per-connective baseline and (ii) an SVM feature-based system from prior work. They report improvements in F1 for the Causal class and accuracy on both non-bootstrapped and bootstrapped versions of AltLex, and present a small analysis of ambiguous markers.

As I see it, the main contributions are:
1) Empirical result: A simple paired-LSTM architecture that encodes events and a connective yields consistent improvements over a strong feature-based baseline on AltLex, including for ambiguous markers.
2) Task framing and analysis: Clear formulation of causal meaning disambiguation as a subtask separable from argument identification, with evidence that context encoding reduces over-reliance on connective priors.

Strengths
- Empirical gains over prior state-of-the-art on a relevant benchmark. On AltLex (both versions), several LSTM configurations outperform: e.g., on “non bootstrapping,” the best LSTM achieves higher Causal F1 and notably higher accuracy than the SVM baseline; on “bootstrapping,” the best LSTM variants match or slightly surpass the SVM. This supports the central claim that neural context encoding is beneficial.
- Focus on ambiguous markers and generalization beyond connective priors. The paper analyzes cases where the training distribution would bias toward Non-Causal, yet the neural model correctly predicts Causal (e.g., “which then”), indicating learning from broader context rather than memorizing connective labels.
- Comparisons against two meaningful baselines. The majority-per-connective baseline (B1) is strong on this dataset due to class skew per marker; outperforming it shows learning beyond per-marker priors. The SVM baseline (B2) reflects prior best published results and includes rich, external resource-based features; beating it with embeddings-only input is practically relevant.
- Systematic albeit lightweight ablations. The authors explore two architectures (connected vs independent LSTMs), two optimizers, and three padding-length strategies, and draw consistent conclusions (independent encodings perform better; Adam improves precision/recall balance), providing insight into design choices.
- Attention to class imbalance. The authors additionally evaluate on a reduced, more balanced subset and report statistical significance (McNemar’s test) of improvements, lending credibility to claims beyond accuracy on skewed data.

Weaknesses
- Limited novelty in modeling. The core architecture (two LSTMs encoding argument spans, concatenated and fed to dense layers) is standard. No comparison is provided against equally simple neural baselines (e.g., a single BiLSTM over the entire sentence, or average-of-embeddings + logistic regression, or a CNN baseline), weakening the case that the proposed pairing strategy is specifically responsible for gains.
- Methodological under-specification and notation issues. Key implementation details are missing or unclear: LSTM dimensionality, number of layers, dropout placement, training epochs, batch size, learning rate settings, whether embeddings are fine-tuned, early stopping criteria, and dev set usage. The padding strategy description and Eq. (1)–(2) are confusing: “pad(x) = {max, mean, mode}” with x∈R^{n×d} and W∈R^{100×nd} suggests flattening variable-length sequences to fixed-size vectors via dense layers, but the pipeline from LSTM outputs to the final vector is not clearly or correctly specified. The “Stated Pair LSTM” initialization mechanism is also underspecified (e.g., which states, shared weights?). These gaps hinder reproducibility and make it hard to assess technical soundness of the architecture.
- Dataset scope and reliance on gold segmentation. Experiments are confined to AltLex; the approach is not tested on other causal datasets (e.g., PDTB explicit/implicit causal relations, Causal-TimeBank). Moreover, it assumes gold event spans and marker identification as inputs; the impact of span errors is not studied, which limits practical applicability claims.
- Evaluation metrics and inconsistencies. Reporting mainly Causal-class F1 and overall accuracy on an imbalanced dataset can be misleading; macro-F1 across both classes would be more robust. There appear to be small discrepancies between text and table (e.g., stated deltas vs numbers in Table 5). The statistical testing is only shown for the reduced dataset; significance on the full dataset is not reported. Splits and preprocessing details (train/dev/test, whether any hyperparameter tuning used a dev set) are not described.
- Limited error analysis and ablations. While some qualitative examples are provided, the analysis is brief. No ablation removes the marker from the input (to quantify reliance on context vs connective), nor is there a symmetric input variant (e2 + marker vs e1) or a sentence-level model comparison.

Questions to authors
- Architecture and training details: Please specify LSTM hidden sizes, number of layers, whether LSTMs are bidirectional, dropout rates and where applied, batch size, number of epochs, early stopping, and whether GloVe embeddings were fine-tuned.
- Padding strategy: Clarify the exact sequence length used for padding. How are “max/mean/mode” applied—across the dataset, per batch, or per input pair? How are sequences longer than the chosen length handled (truncation strategy)?
- State initialization in “Stated Pair LSTM”: Which states (h, c) are transferred? Are LSTM weights shared between the two encoders? Did you try other coupling mechanisms (e.g., attention between the two sequences)?
- Data splits and tuning: What are the exact train/dev/test splits for both corpus versions? Was a development set used for model selection? How were hyperparameters chosen?
- Inputs and spans: How are e1, l, e2 extracted from AltLex? Are spans contiguous and tokenized consistently? Do you use any sentence-level context beyond the spans? Could you report performance when using only the sentence or only the marker as input (ablation)?
- Evaluation: Can you report macro-F1 over both classes and significance tests on the full datasets? Also, please reconcile any discrepancies between narrative and Table 5 deltas.

Overall, this is a clear and focused empirical study that demonstrates that simple neural context encoding improves causal meaning disambiguation on AltLex. However, the modeling novelty is limited, methodological details are underspecified, and the evaluation could be more comprehensive and reproducible.

Summary and claimed contributions
This paper tackles causal meaning disambiguation: given two annotated event spans (e1, e2) and a candidate lexical marker l in a sentence, decide whether the relation is causal or not. The authors argue that causality is not tied to fixed syntactic patterns and propose neural context encoding rather than feature engineering. They evaluate on the AltLex corpus (Hidey & McKeown, 2016), comparing two LSTM-based architectures (“Stated Pair LSTM” vs. “Pair LSTM”) with GloVe embeddings against (i) a per-connective majority baseline and (ii) the SVM-based system from Hidey & McKeown. They report consistent gains, discuss optimizer and padding choices, and offer qualitative examples of disambiguation for ambiguous markers.

As I see them, the main contributions (in order of strength) are:
1) An empirical demonstration that simple LSTM-based context encoding improves over a strong feature-based baseline for causal lexical marker disambiguation on AltLex, including for some ambiguous markers.
2) A straightforward two-input architecture (“Pair LSTM”) and an ablation comparing state initialization across event encoders, optimizers, and padding strategies.
3) A task framing that separates causal meaning classification from argument identification, accompanied by small-scale qualitative analysis.

Strengths
- Empirical improvements on a relevant dataset. On both AltLex splits, several LSTM configurations outperform the majority baseline and the prior SVM feature-based approach, particularly in precision for the causal class and overall accuracy. The non-bootstrapped split shows a clear accuracy gain over the state-of-the-art baseline.
- Clean, feature-light modeling. The approach eschews heavy lexical resources (WordNet/VerbNet/PropBank), using only pre-trained GloVe embeddings. This enhances portability and helps validate the hypothesis that contextualized event representations help resolve ambiguity.
- Sensible task decomposition. Separating causal meaning classification from argument direction identification is reasonable and aligns with practical pipelines in discourse parsing and relation extraction.
- Some analysis of design choices and qualitative behavior. The comparison of state-initialized vs. independent event encoders is useful; the paper also provides examples where the neural model corrects errors made by the SVM baseline, including ambiguous expressions like “which then.”

Weaknesses
- Limited novelty and under-specified modeling. Applying LSTMs to pairwise text inputs is standard; there is little architectural novelty. More importantly, critical details are missing/unclear: LSTM hidden sizes, number of layers, bidirectionality, dropout placement, training epochs/early stopping, batch sizes, and parameter counts. The “zero-padding strategy” is described via max/mean/mode in a way that is mathematically unclear (padding length selection is not properly specified), and the role of the dense layers relative to LSTM outputs is not precise.
- Narrow evaluation and weak baselines for a 202x ACL venue. The comparison is only against a per-connective majority and one 2016 feature-based SVM. Absent are strong modern baselines (e.g., BiLSTM-attn with contextualized embeddings, transformer encoders like BERT/RoBERTa/DeBERTa or sentence-pair classification with [CLS]) that typically provide stronger performance on discourse relation tasks. Without these, it is hard to judge the contribution in 2025.
- Dataset and task scope concerns. The model assumes gold event spans and a candidate marker l as input, which simplifies the problem substantially; applicability to end-to-end settings (event detection + marker identification + relation classification) is not addressed. It is also unclear how cases with implicit causality and no explicit marker are handled given that the input includes l. Additionally, the evaluation is confined to AltLex; testing on PDTB causal relations and/or Causal-TimeBank would strengthen the claims.
- Metrics and statistical testing. Reporting focuses on precision/recall/F1 for the causal class and accuracy overall, despite strong class imbalance; macro-F1 or macro-averaged precision/recall would be more informative. Statistical significance is only briefly reported for a downsampled setting vs. B1; there is no significance testing on the main comparisons to B2.
- Inconsistencies and presentation issues. Several tables appear malformed/truncated; some numbers seem inconsistent with textual claims (e.g., stated F1 improvements). References have missing years and numerous formatting errors. Language quality issues and unclear equations impede clarity and reproducibility.

Questions to authors
1) Inputs/task: Are e1/e2 gold spans provided by AltLex? How are they extracted when multiple candidate events occur? How do you handle instances without an explicit lexical marker (implicit causation), given your input includes l?
2) Architecture: Are the LSTMs bidirectional? Hidden size, number of layers, dropout placements, training epochs, batch sizes, early stopping criteria? Where exactly is dropout applied (embeddings, LSTM outputs, dense layers)?
3) Padding: What does “max/mean/mode padding” concretely mean? Is it the fixed maximum/mean/mode sequence length across the training set, or a pooling operation? Please clarify Equation (1).
4) Training/tuning: What dev set did you use for hyperparameter selection (optimizer, padding strategy, regularization)? Were results averaged over multiple random seeds? Any variance estimates?
5) Baselines: How exactly was B2 reimplemented? Were the same features/resources and splits used? Did you hyperparameter-tune B2 on a dev set?
6) Metrics/significance: Please report macro-averaged precision/recall/F1 and confidence intervals. Are gains over B2 statistically significant on the official splits?
7) Generalization: How does the model perform on unseen connectives? Consider a connective-held-out evaluation to test lexical memorization. Also, what happens if you remove l from the second input—does the model still learn causal relations from event contexts alone?
8) Context scope: Why restrict inputs to event spans rather than full sentences with event position indicators? Have you tried including sentence-level context or marker masking?
9) Comparisons: How does your system compare to a transformer baseline (e.g., BERT/RoBERTa) fine-tuned on the same task with inputs “[CLS] e1 [SEP] l + e2 [SEP]”? This would help situate your gains.
10) Reproducibility: Will you release code, trained models, and exact preprocessing scripts for AltLex?

Decision: Weak reject
Rationale: The problem is important and the paper offers a clear, feature-light approach with some empirical gains and qualitative insights. However, the methodological novelty is limited, critical details are missing, the evaluation omits strong modern baselines and broader datasets, and the presentation (equations/tables/references) needs substantial polishing. With clearer specification, stronger baselines (including transformers and connective-held-out tests), broader evaluation (e.g., PDTB/Causal-TimeBank), improved metrics and significance testing, and a more rigorous analysis of generalization beyond known markers, this work could become competitive.

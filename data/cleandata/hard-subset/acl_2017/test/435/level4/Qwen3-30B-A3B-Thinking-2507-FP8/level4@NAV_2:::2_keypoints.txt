- Proposes an LSTM-based neural architecture (GloVe embeddings + LSTM + dense tanh layers) for clause-level causal relation detection on the AltLex dataset.  
- Claims LSTMs better capture contextual features than CNNs for this task.  
- Best configuration yields a 0.5–1.5% F1 improvement over Hidey & McKeown (2016).  
- Primary contribution: improved neural architecture for causal meaning disambiguation.  
- Secondary contribution: qualitative analysis showing correct identification of ambiguous lexical causal markers.  
- Strength: paper is well-written with a clear task description.  
- Strength: thorough qualitative examples demonstrating system behavior on ambiguous contexts.  
- Strength: code and dataset are made publicly available, aiding reproducibility.  
- Weakness: task not clearly distinguished from implicit connective recognition, reducing contextual clarity.  
- Weakness: model selection appears to use the test set rather than a dev set, risking evaluation leakage.  
- Weakness: no statistical significance testing reported for the performance gain.  
- Weakness: quality of AltLex gold annotations is questionable given dataset construction methods.  
- Weakness: marginal improvement is expected given prior results showing neural methods outperform hand-engineered features.  
- Questions for authors: clarify the 0-1-2 TSV coding (paper describes binary classification).  
- Questions for authors: address copyright/reuse concerns for the AltLex dataset.  
- Questions for authors: clearly define the “bootstrapping” process used to extend the corpus by ~15%.

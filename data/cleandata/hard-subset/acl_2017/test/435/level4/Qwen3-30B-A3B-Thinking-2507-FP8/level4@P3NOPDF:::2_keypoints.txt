- Task: Detecting causal relations between clauses, focusing on implicit causal relations without explicit connectives.
- Data: Uses the AltLex dataset (Hidey & McKeown, 2016).
- Model: Neural architecture with GloVe embeddings, an LSTM for clause structure, three dense tanh layers, and softmax output.
- Modeling claim: Argues LSTMs better capture clause features than CNNs.
- Performance: Reports a marginal 0.5–1.5% F1 improvement over prior SVM work (and a cited 2.13% gain), but significance is unclear.
- Strengths: Well-written, clear problem statement, useful qualitative analysis (e.g., handling non-causal uses of “because”), and shared code/data.
- Data release issues: TSV files use unexplained 0–1–2 coding despite binary task description, and dataset reuse/permissions are not clarified.
- Experimental flaw: Best model configuration appears to have been chosen on the test set, risking test-set leakage and invalid comparisons.
- Evaluation: Needs development-set hyperparameter selection and held-out test reporting for fair comparison.
- Statistical testing: No statistical significance tests reported (e.g., McNemar’s); required to confirm the reported gains.
- Annotation quality: Gold-standard AltLex annotations contain inconsistencies; authors should discuss annotation noise and its impact.
- Relation to prior work: Insufficient distinction from implicit connective recognition (ICR); must explain why ICR methods are not directly applicable.
- Bootstrapping: Corpus extension by 15% via “bootstrapping” is not described in sufficient detail (heuristics/rules missing).
- Baselines: Lacks simpler neural baselines (e.g., single-layer LSTM) to justify architectural complexity.
- Error analysis gap: Notes a precision–recall trade-off but fails to analyze why the model favors precision (e.g., class imbalance or feature effects).

- Tackles causal relation disambiguation between two events/clauses when connectives are ambiguous, using the AltLex dataset.
- Proposes an LSTM-based model encoding two arguments (plus marker when present) with GloVe, three tanh dense layers, and a softmax output.
- Main contribution: empirical study showing a simple LSTM slightly outperforms a feature-rich SVM on AltLex; novelty is modest.
- Secondary contribution: release of code and data to aid reproducibility.

- Clear problem framing and motivation for context-sensitive causal disambiguation and limits of fixed lexical/syntactic methods.
- Quantitative and qualitative evaluation: small but consistent F1 improvements and informative error examples.
- Reproducibility: shared code/data allowed inspection of implementation details.
- Timely application of neural methods to a challenging dataset (AltLex).

- Model selection appears to have used the test set, risking optimistic reported gains.
- No statistical significance testing or uncertainty estimates; dataset label noise and its impact are not discussed.
- Limited novelty and missing stronger/simple baselines to isolate architectural contributions.
- Precision–recall imbalances are observed but not analyzed for underlying causes.
- Task positioning relative to implicit discourse relation/implicit connective recognition is unclear.

- Did you tune hyperparameters on a separate development set; please report dev-selected test results?
- Are the improvements over the SVM statistically significant; please include significance tests and confidence intervals?
- Please detail the bootstrapping procedure and how it changes class/marker distributions across splits.
- Clarify the TSV 0–1–2 coding and how it maps to the binary Causal vs Non-Causal labels.
- Do you have permission to redistribute Hidey & McKeown (2016) data; if not, provide reconstruction scripts?
- Could you add simpler neural baselines (e.g., 1-layer LSTM, average-of-embeddings+MLP) and report their performance?
- What drives the precision–recall trade-off (padding, optimizer, class imbalance); have you tried class weighting or threshold tuning?
- How does this task differ from PDTB-style implicit connective recognition, and can existing implicit models be applied?

- Overall: with dev-based model selection, significance testing, clearer task positioning, and stronger baselines, the contribution would be more convincing.

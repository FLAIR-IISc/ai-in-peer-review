- Proposes an LSTM-based neural classifier (GloVe embeddings → LSTM → three tanh dense layers → softmax) for detecting implicit causal relations between clauses.  
- Uses the AltLex dataset (Hidey & McKeown, 2016) to identify causal vs. non-causal relations when no explicit connective is present.  
- Argues LSTMs represent clause features better than convolutional networks for this task.  
- Reports small F1 improvements (0.5–1.5%, claimed 2.13% in one comparison) over Hidey & McKeown’s SVM.  
- Provides both qualitative and quantitative analyses, with examples of generalization (e.g., indicator words treated non-causally in test data).  
- Paper is well written and the problem description is particularly clear.  
- Requests clarification on how this task differs from implicit connective recognition and why prior methods cannot be applied.  
- Code and data were uploaded, but the TSV 0-1-2 coding is unexplained and legality/permission to repost Hidey & McKeown’s data is unclear.  
- The term “bootstrapping” (corpus extension by ~15%) is not clearly explained.  
- Evaluation concerns: best configuration appears selected on test data rather than dev set, making the reported improvement potentially unfair.  
- Statistical significance of the improvement is not reported and should be assessed given dataset size and annotation reliability.  
- Notes that the AltLex annotations (English/Simple English Wikipedia) are imperfect, so scores may be noisy.  
- Suggests including simpler baselines (e.g., 1-layer LSTM) and analysis explaining why the model favors precision over recall.

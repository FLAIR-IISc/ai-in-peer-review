- Proposes an LSTM-based neural architecture using GloVe clause embeddings to detect causal vs. non-causal relations in AltLex where connectives are not explicit.
- Claims main contributions: a neural architecture for event-relation causality disambiguation.
- Claims main contributions: using GloVe-based clause representations as LSTM input to capture semantic meaning.
- Claims main contributions: evaluation on the challenging AltLex corpus.
- Strength: Paper is well-written with a clear problem description.
- Strength: Provides both qualitative and quantitative analysis of system performance.
- Strength: Authors uploaded code, aiding reproducibility.
- Weakness: Fails to clearly distinguish this task from implicit connective recognition.
- Weakness: Does not explain why prior implicit-connective methods are inapplicable here.
- Weakness: "Bootstrapping" process is unclear despite claiming ~15% corpus extension.
- Weakness: Unfair comparison—best model configuration chosen using test data rather than dev set.
- Weakness: No statistical significance testing reported for results vs. baseline.
- Question: Please clarify differences between this task and implicit connective recognition.
- Question: Explain why previous implicit-connective methods cannot be used here.
- Question: Provide a clear description of the bootstrapping method and how it yields ~15% more data.
- Question: Re-evaluate comparisons by selecting the best model on the dev set (not test) and report results with significance testing.

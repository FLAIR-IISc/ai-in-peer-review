Summary and claimed contributions
This paper proposes a neural local coherence model that “neuralizes” the entity grid. Each grid cell (grammatical role and optional entity-specific features) is embedded; 1D convolutions over per-entity sentence transitions capture longer-range local patterns; max pooling selects salient features; and a linear layer yields a coherence score. Training uses a pairwise ranking loss. The model is evaluated on sentence ordering (discrimination and insertion) and on summary coherence rating, showing improvements over strong entity-grid baselines; pretraining helps on the small summary dataset.

As I see them, the main contributions are:
1) A CNN-based architecture over entity grids that enables modeling longer entity transitions via distributed representations and end-to-end ranking-based training.
2) An effective, simple mechanism to incorporate entity-specific features by composing them with role symbols, yielding state-of-the-art results while using fewer engineered features than prior work.
3) Empirical validation across three tasks with significance testing and multiple runs, along with code release for reproducibility.

Strengths
- Strong empirical performance and careful evaluation: The model outperforms both the all-nouns grid and the extended grid on discrimination and insertion, with statistically significant gains, and improves summary rating accuracy further with pretraining. Reporting averages over multiple random seeds and using an approximate randomization test increases confidence in the results.
- Clear, well-motivated architecture: Mapping grid entries to embeddings and applying convolutions over per-entity transitions directly addresses sparsity in discrete transition features and allows using longer transition windows (k≈5–8). The pairwise ranking objective is appropriate for coherence assessment.
- Reduced feature engineering: The Extended Grid-CNN matches or exceeds the extended grid using only three readily available entity-specific features (NE type, salience, proper-name flag), which simplifies deployment and suggests the learned representations capture useful task-specific structure.
- Reproducibility and transparency: The paper provides implementation details (optimizers, hyperparameter search ranges, regularization, early stopping) and uses standard corpora and publicly available toolkits; code release is mentioned.
- Simplicity and efficiency: Compared to sentence-level RNN/recursive approaches, the proposed model is conceptually simple, likely faster to train/infer, and directly leverages the strong inductive bias of entity transitions that has proven effective in coherence modeling.

Weaknesses
- Baseline coverage and fairness: The paper excludes Li & Hovy (2014) due to disappointing reproduction on WSJ. While understandable, omitting this comparison weakens the case against sentence-representation baselines. A stronger effort to tune/replicate (e.g., hyperparameter sweeps, pretrained word vectors, curriculum) or to include other recent neural sentence-ordering methods would strengthen the empirical story.
- Limited analysis/ablations: There is little analysis of why the model helps and how design choices matter. For example, sensitivity to transition window size, pooling size, embedding dimension, and dropout; ablations of entity-specific features and salience; and qualitative analysis of learned filters (e.g., common patterns they detect) would make the contribution more interpretable and actionable.
- Dependence on coreference/NP detection with no robustness study: The approach relies on automatic NP extraction and coreference. It would be valuable to quantify sensitivity to coreference quality (e.g., alternative resolvers, oracle vs predicted, or perturbation tests), as errors here could disproportionately affect performance, particularly off-domain or in summaries.
- Independence across entity columns: Treating entity columns independently precludes modeling interactions among entities (e.g., topic shifts involving multiple entities, entity co-occurrence patterns). Even a brief experiment with 2D convolutions or cross-entity pooling/attention could test whether capturing inter-entity structure yields additional gains.
- Task mismatch for insertion and limited generalization tests: The model is trained on discrimination pairs yet evaluated on insertion, where gains are modest. Training directly on insertion-style pairings or multi-tasking could be insightful. Additionally, evaluations are limited to WSJ and a small DUC subset; results on other standard corpora (e.g., Airplanes/Earthquakes, or cross-domain) or a discussion of domain transfer would improve external validity.

Questions to authors
- Convolution scope: Are convolutions strictly along the sentence axis within each entity column, with filters shared across entities? Any experiments with 2D filters across entities and sentences?
- Transition length k and pooling: What is the sensitivity of performance to k and pooling length p? Do longer windows saturate or overfit on WSJ?
- Entity-specific features: When concatenating features with roles (e.g., PERSON-S), the vocabulary can grow. Did you observe sparsity or embedding capacity issues as features increased? How does performance change if features are embedded separately and concatenated rather than composed into a single symbol?
- Coreference robustness: Which coreference system did you use, and how sensitive are results to switching resolvers? Do errors in coreference disproportionately affect insertion vs. discrimination?
- Training objectives: Did you try training directly on insertion (e.g., pairwise ranking over candidates differing by one sentence position) or multi-task training with both discrimination and insertion? If so, how did it affect insertion performance?
- Efficiency: What are training and inference times compared to the extended grid SVM ranker? Could the model score all candidate insert positions in real time for practical sentence-ordering applications?

Additional comments
- An error analysis (e.g., typical failure cases, effect of document length, or entity sparsity) would help guide future improvements.
- If possible, include small-scale qualitative examples linking specific high-activation filters to interpretable transition patterns. This would strengthen the narrative on “why it works.”

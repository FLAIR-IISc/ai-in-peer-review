- Neural adaptation of entity grid: adapts the entity grid framework to a neural model using distributed representations and end-to-end training to capture longer-range dependencies.
- Empirical gains: achieves state-of-the-art performance on discrimination, insertion, and summary coherence tasks with notable improvements over baselines.
- End-to-end learning benefit: demonstrates that task-specific features can be learned automatically rather than hand-engineered.
- Strong empirical results: consistent, significant performance improvements across all three benchmarks (e.g., ~4% on discrimination).
- Well-motivated and clear presentation: clearly articulates limitations of prior entity grid models and explains the neural approach with a readable write-up and helpful figure.
- Code release: authors provide code, aiding reproducibility.
- Builds on prior work: principled extension of entity grid models rather than a wholly novel paradigm.
- Limited architectural novelty: using CNNs (convolution + pooling) is standard; novelty is applying them to the entity grid rather than in network design.
- Shallow entity features: only three basic entity-specific features are used, with limited exploration of richer features.
- Missing ablation study: lacks granular analysis isolating contributions of distributed representations, end-to-end training, and entity features.
- Limited error analysis: no in-depth discussion or examples of model failures and challenging coherence phenomena.
- Questions on ablation: request for experiments separating effects of representations, training, and features.
- Questions on features: did the authors try more entity-specific features and what were the outcomes?
- Questions on errors: ask for example failure cases and analysis of difficult coherence phenomena.
- Hyperparameter sensitivity: request for sensitivity analysis (e.g., window size, number of filters).
- Scalability and complexity: ask for computational complexity and scaling behavior with document length and entity count.
- Li & Hovy comment: suggest more explanation for that model's poor performance.
- Statistical testing note: SIGF V.2 reference for significance testing is appropriate.
- WSJ preprocessing clarification: request clarification on whether WSJ corpus was preprocessed (tokenization, POS tagging) before entity grid construction.

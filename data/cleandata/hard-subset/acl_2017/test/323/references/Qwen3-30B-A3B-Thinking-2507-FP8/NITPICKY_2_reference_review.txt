

# ACL Review: "A Neural Local Coherence Model"

## Summary of Paper and Contributions

The paper proposes a neural extension of the entity grid model for local coherence assessment. The authors claim to address limitations of existing entity grid models by using distributed representations and convolutional layers to capture longer-range entity transitions while incorporating entity-specific features. The main contributions claimed are:
1. A neural architecture for local coherence modeling that can capture long-range entity transitions
2. An end-to-end training method to learn task-specific features
3. State-of-the-art results on discrimination, insertion, and summary coherence rating tasks

However, upon meticulous examination, I find the paper contains multiple critical flaws that undermine its validity and reproducibility, making it unsuitable for publication in ACL despite its potentially interesting direction.

## Strengths

1. **Relevant Problem**: The paper addresses a meaningful problem in natural language processing with applications in text generation, summarization, and coherence scoring.

2. **Thorough Evaluation**: The paper evaluates the model on three established coherence tasks (discrimination, insertion, and summary coherence rating), which provides a comprehensive assessment of the model's performance.

3. **Logical Extension**: The proposed neural extension of the entity grid model is a reasonable approach that builds on prior work, and the use of convolutional layers to capture longer-range transitions is conceptually sound.

## Weaknesses

1. **Critical Mathematical Error in Loss Function**: The paper defines the pairwise ranking loss as J(θ) = max{0, 1− φ(Gi|θ) + φ(Gj |θ)}. This is fundamentally incorrect. The standard pairwise ranking loss should be J(θ) = max{0, 1 - (φ(Gi|θ) - φ(Gj|θ))}. This is not merely a typo but a critical error that would fundamentally alter the training dynamics and invalidate the reported results. The authors claim "our model achieves state of the art results" but the training process described would not produce the claimed results due to this error. This is a major flaw that makes the paper's core contribution questionable.

2. **Inconsistent Description of Entity Grid and Feature Incorporation**: The paper initially describes the entity grid as having entries that represent grammatical roles (S, O, X, -), but later states in Section 3.1 that "If an entity ej of type PERSON appears as a subject (S) in sentence si, the grid entry Gi,j can be encoded as PERSON-S." This is a fundamental contradiction. If the grid entries are combinations like "PERSON-S", then the grid is no longer a simple representation of grammatical roles, but the paper fails to explain how this affects the convolutional processing described in Section 3. The description of the grid in Figure 2 and the neural architecture is inconsistent with the textual description.

3. **Missing Critical Details for Reproducibility**: The paper fails to provide essential details required for reproducibility:
   - Which coreference resolver was used to construct the entity grid?
   - How were entity-specific features computed (e.g., how was "salience as determined by occurrence frequency" implemented)?
   - The paper states "we use the same setting that produced the best results on the discrimination task" for the Li and Hovy (2014) model but doesn't provide the specific settings used.
   - The paper claims "we have released our code with the submission" but provides no link to the code repository, making verification impossible.

4. **Inconsistent Results Reporting**: The paper claims "absolute gains of about 4% on discrimination" but the actual gain from Extended Grid (84.95) to Extended Grid-CNN (88.69) is 3.74%, not "about 4%." This discrepancy, while small, undermines the credibility of the paper's claims. Additionally, Table 3 reports "23.12" for the Extended Grid-CNN model without specifying what this number represents (accuracy, F1, or something else), and it's inconsistent with the rest of the paper's reporting.

5. **Inconsistent Notation and Terminology**: The paper contains multiple instances of inconsistent notation and terminology:
   - Equation (1) uses angle brackets "〈 〉" for a tensor, which is non-standard
   - Equation (2) contains a misplaced transpose symbol "w TLt:t+k−1,j" (should be "w^T" or "w^T")
   - The paper inconsistently refers to "entity grid" as both the representation and the model
   - The description of the grid structure is inconsistent (rows vs. columns representing entities vs. transitions)
   - The paper uses "WSJ" without defining it on first use

## Questions to Authors

1. Please clarify the mathematical error in the loss function. Is it a typo, and the intended loss was J(θ) = max{0, 1 - (φ(Gi|θ) - φ(Gj|θ))}? If so, please provide the correct formulation and explain how this affects the reported results.

2. Please explain the contradiction between the initial description of the entity grid (entries as grammatical roles) and the later description (entries as combinations like "PERSON-S"). How does this affect the neural model architecture described in Section 3?

3. Please provide details about which coreference resolver was used to construct the entity grid, and how the entity-specific features (named entity type, salience, proper mention) were computed.

4. Please clarify what the "23.12" value represents in Table 3, and why it's not specified as a percentage.

5. Please provide a link to the code repository mentioned in the paper, as this is critical for reproducibility.

## Additional Comments

The paper contains numerous other minor inconsistencies, including:
- Inconsistent use of "entity" vs. "entity mentions"
- Inconsistent spacing around punctuation (e.g., "i.e.,", "e.g.,")
- Inconsistent citation format (some with year in parentheses, others with question marks in the references)
- Unclear description of how the "20 random permutations" were generated and how many were excluded due to matching the original document

While the paper's core idea is promising, the fundamental errors and inconsistencies make it impossible to trust the reported results or reproduce the work. As a perfectionist reviewer, I must recommend rejection due to these substantial methodological and presentation flaws that undermine the paper's validity.

The authors have made a significant error in the loss function formulation that fundamentally invalidates their training process. This is not a minor typo but a critical error that would prevent the model from learning correctly. The paper's claims of "state of the art results" cannot be verified due to this error and the other inconsistencies.

The paper would be acceptable for publication if the authors could correct these critical issues, but as it stands, it contains too many fundamental flaws to be recommended for acceptance at ACL.

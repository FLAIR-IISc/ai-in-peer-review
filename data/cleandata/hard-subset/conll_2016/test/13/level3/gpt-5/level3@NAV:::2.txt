Summary
The paper tackles within-document event coreference by learning sentential representations of event mentions with CNNs and scoring mention pairs with a lightweight similarity layer plus pairwise features. It aims to avoid dependence on external KBs, reports competitive results on ACE, and introduces a larger ACE++ collection to improve recall. The analysis of argument sharing and mention distances usefully motivates leveraging sentence-level context.

Contributions (as I see them)
- Primary: A strong, KB-free neural baseline for event linking with clear ablations showing the value of sentential features alongside pairwise cues.
- Secondary: Empirical characterization of ACE/ACE++ (argument overlap, trigger mismatch, distance), which informs model design and evaluation.
- Tertiary: A larger evaluation setting (ACE++), suggesting data scale matters for recall, though the resource is insufficiently described.

Strengths
- Clear problem setup and well-written presentation; the two-part architecture is easy to reproduce and analyze.
- Competitive performance on ACE without external semantic resources; ablations (Table 5) demonstrate that sentential features add measurable gains over pairwise and lexical-only variants.
- Useful corpus analysis (argument sharing, trigger mismatch, long-distance links) that justifies modeling choices and highlights challenges for the task.
- Thoughtful error analysis identifies pronoun/event anaphora and annotation ambiguity as major pain points, helping scope future work.

Weaknesses
- Insufficient architectural motivation and comparison: CNNs are used without analysis versus RNNs/biLSTMs or other continuous-space baselines; 2–3 F1 gains need stronger evidence (e.g., same pipeline with RNN, PCNN, self-attention).
- Novelty is unclear: sentence-CNNs for text classification are established; comparisons to prior continuous-space event-linking methods and stronger baselines (expanded Table 3) are needed to substantiate contribution.
- ACE++ is under-specified: no details on collection/annotation protocol, event-type coverage, domain biases, or where prior methods fail but the proposed one succeeds; lack of a release plan limits impact.
- Evaluation breadth: results rely on gold event mentions; please report end-to-end performance without gold references. Additional datasets (e.g., EventCorefBank) and joint entity/event coref experiments would strengthen claims.
- Feature transparency: Table 5 implies pairwise features remain critical—report which features mattered most, selection criteria, and alternatives attempted.

Questions to Authors (for rebuttal)
- Why CNNs for mention representation? Please provide empirical comparisons to biRNNs/transformers and analysis of window size/pooling choices.
- What is the ACE++ construction protocol, event-type distribution, domain mix, inter-annotator agreement, and release plan?
- Can you report results without gold triggers/arguments and with off-the-shelf coref augmentation (e.g., CoreNLP), as hinted in the example?
- Which pairwise features drive gains? Any tried-but-abandoned alternatives?
- Could you evaluate on EventCorefBank and/or a joint entity–event coref setting to test generality?

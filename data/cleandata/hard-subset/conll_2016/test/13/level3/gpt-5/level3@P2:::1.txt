Summary and Contributions
This paper tackles within-document event coreference (“event linking”) without external semantic resources. The model builds sentential representations of event mentions using a CNN with position embeddings and piecewise max pooling, augments them with simple lexical and pairwise features, and feeds them to a learned similarity function and logistic layer to score coreference. On ACE and a larger ACE++ dataset, the approach reports state-of-the-art or comparable performance to feature-rich systems, plus ablations and an error analysis.

As I see it, the primary contributions are:
1) A resource-light neural architecture for event linking that integrates sentential CNN features with minimal lexical and pairwise features, achieving competitive results to prior systems relying on extensive external knowledge bases.
2) Empirical analysis demonstrating that sentential features materially improve performance over pairwise and trigger-local features alone (via ablations), and that transitive-closure clustering outperforms BestLink in this setup.
3) A curated larger dataset (ACE++) and an error analysis highlighting challenging phenomena (e.g., pronominal event references).

Strengths
- Strong results without external KB features: The model attains state-of-the-art or on-par performance with feature-rich systems (e.g., Liu et al., 2014) on ACE using only word/position embeddings and a small set of pairwise features. This is an attractive contribution for low-resource or cross-domain settings where WordNet/FrameNet/DBpedia features are impractical.
- Careful ablations and empirical insights: Systematic removal of feature classes (pairwise vs. trigger-local vs. sentential) shows sentential features substantially boost both precision and recall. The analysis that BestLink reduces recall in this architecture is informative and contrasts with prior findings.
- Sensible metric choice and discussion: Using BLANC and discussing class imbalance and metric limitations improves interpretability. Including multiple metrics (BLANC, B^3, MUC, positive links) helps triangulate performance trade-offs.
- Error analysis with concrete phenomena: The manual inspection identifies pronoun-mediated event references and borderline annotation cases as major sources of error, providing actionable directions (e.g., discourse/pronominal modeling).
- Model simplicity and reproducibility potential: The architecture is conceptually simple (CNN sentence encoder + learned similarity), implemented in TensorFlow, and hyperparameter search is described, which facilitates adoption.

Weaknesses
- Fairness and comparability of evaluations: Results rely on gold event triggers; some competing systems use predicted triggers or different preprocessing. Train/test splits differ across papers. These choices make direct comparison to prior work less conclusive; standardized splits or additional experiments with predicted triggers are needed.
- Insufficient clarity on architectural details: Several aspects are underspecified: how position embeddings are initialized/updated and shared; how left vs. right contexts are represented; rationale for appending exactly one token on each side; how the event-mention vector ve is used later given that vsent+lex appears to be the input to the similarity network; and precise encoding/learning of pairwise features (binary vs. continuous, normalization, trainable embeddings/parameters).
- ACE++ dataset availability and baselines: ACE++ is not public, yet several key insights (e.g., recall gains, ablations) are reported there. It is unclear if ACE++ will be released. Moreover, comparisons on ACE++ to strong feature-rich baselines are missing, limiting external validity of those results.
- Novelty is incremental relative to prior neural coreference/extraction work: CNN-based sentential encoders for relation/event tasks are established; applying them to event linking with a learned similarity is a natural extension. The paper would benefit from positioning against neural coreference literature (e.g., Wiseman et al., 2015) and articulating what is technically new (e.g., the squared nonlinearity) and why it matters.
- Lack of statistical significance testing: Many numbers are close across systems and settings; significance tests are needed to support the claims of state-of-the-art or superior clustering strategies.

Questions to Authors
- Position embeddings: How are relative position embeddings initialized (random vs. pre-trained) and updated? Why should all words at the same relative distance share the same embedding, and did you consider lexicalization (e.g., combining position and word features in a learned interaction)?
- Left/right handling: Beyond piecewise max pooling, are left/right neighbor tokens handled differently anywhere else? How sensitive is performance to the window size around the trigger (why exactly ±1 word)? Did you try larger or learned spans?
- Event mention representation: You define ve = tanh(We vsent+lex + be). Later, Section 3.2 concatenates vsent+lex with pairwise features to compute vsim. Where is ve used? If unused, please clarify or correct the notation.
- Pairwise features: Please specify encoding (binary indicators vs. counts vs. buckets), any normalization of continuous features (e.g., distance), and whether these features are passed through learned embeddings or linear transformations whose parameters are updated during training.
- Square activation: What motivated the square nonlinearity for similarity beyond polarity invariance? Is this choice novel in this context, and how does it compare empirically to ReLU/tanh and cosine/dot-product on both ACE and ACE++ with significance?
- Data and comparability: Will ACE++ and the document lists/splits be released? Can you re-run on a commonly used ACE split to strengthen comparability? Can you report results with predicted triggers to quantify the end-to-end impact?
- Generalization: Can the model be adapted to entity coreference (e.g., OntoNotes) or joint entity-event coreference to compare against broader neural coref baselines (e.g., Wiseman et al., 2015)?

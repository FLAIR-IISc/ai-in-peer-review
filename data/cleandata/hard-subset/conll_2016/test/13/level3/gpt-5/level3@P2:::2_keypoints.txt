- Paper tackles within-document event coreference by using CNN-based sentential encoders plus a small set of pairwise features and simple clustering.

Contributions
- Empirical finding: strong, portable event linking performance without external KB features, achieving competitive/SOTA results on ACE.
- Architecture & analysis: simple two-stage model (CNN mention encoder + similarity network with square activation) and comparison showing transitive closure outperforms BestLink.
- Data-driven insights: ACE(++) analyses show many positives lack shared arguments/different triggers/long-range links, and ablations quantify sentential feature importance.

Strengths
- Practical portability: competitive results without domain-specific semantic resources, easing cross-domain/language deployment.
- Strong empirical support for sentential features via dataset analyses and ablations showing substantial F1 gains.
- Careful evaluation: matches/outperforms prior work on several metrics and provides an informative clustering study.
- Informative error analysis highlighting pronominal anaphora, annotation ambiguities, and other failure modes.
- Clear writing and motivation; model and analyses are well explained.

Weaknesses
- Limited novelty: CNN sentence encoders were already common; gains may reflect engineering rather than new methodology.
- Incomplete comparisons: no controlled baselines against BiLSTM/Tree-LSTM or other continuous-space similarity models.
- ACE++ concerns: insufficient description of dataset construction, annotation quality, coverage, and no release hinders reproducibility.
- Evaluation scope limited: relies on gold triggers/arguments, lacks end-to-end results and significance testing.
- Pairwise features under-analyzed: important to performance but not broken down by contribution or alternatives.

Questions to authors
- Why CNNs? Provide controlled comparisons to BiLSTM/Tree-LSTM and ablate CNN design choices (positional embeddings, pooling, etc.).
- Similarity function: report comparisons (square vs cosine vs biaffine/other nonlinearities) with significance tests.
- ACE++ details: how were documents sourced, annotated, adjudicated, what is IAA and domain distribution, and will ACE++ be released?
- Robustness without gold annotations: report performance with predicted triggers/arguments for end-to-end utility.
- External coreference signals: have you tried integrating an off-the-shelf entity coref resolver and what is the quantitative impact?
- Feature analysis: which pairwise features drive performance and were learned-only replacements tested?
- Generalization: can you evaluate on ECB/ECB+, ProcessBank, or provide zero-shot/fine-tune transfer results?

Additional comments
- Include statistical significance tests for key comparisons.
- Consider joint entity/event coreference experiments or conditioning on predicted entity coref given error patterns.
- Report precision/recall trade-offs or clustering-threshold curves to contextualize BestLink vs transitive closure.

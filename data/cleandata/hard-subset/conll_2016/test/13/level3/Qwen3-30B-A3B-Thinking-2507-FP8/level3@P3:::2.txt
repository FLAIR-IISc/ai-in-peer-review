

# Review of "Event Linking with Sentential Features from Convolutional Neural Networks"

## Summary and Main Contributions

The paper proposes a model for event coreference resolution (event linking) that utilizes sentential features from convolutional neural networks (CNNs) rather than relying on external semantic features like WordNet or FrameNet. The model has two parts: a CNN-based representation learning module for individual event mentions and a coreference scoring module that combines these representations with pairwise features. The authors introduce ACE++, a larger dataset created by expanding ACE 2005 with 1351 additional web documents, and report state-of-the-art performance on both ACE and ACE++.

As I see them, the main contributions of this work (in order of strength) are:
1. Creation of ACE++, a significantly larger event linking dataset that addresses the small size of ACE 2005 (only 599 documents), providing a more robust benchmark for future research.
2. Demonstration that event linking can achieve state-of-the-art performance without relying on external semantic features, showing that such features aren't necessary for high performance.
3. Detailed analysis of feature importance and error analysis that identifies specific challenges for future research.

## Strengths

1. The ACE++ dataset is a substantial contribution that addresses a critical limitation of the ACE 2005 corpus (only 599 documents). The authors correctly note that previous work on event linking has been hampered by the small size of available datasets, and their creation of ACE++ with 1,950 documents provides a more robust benchmark for future research. The dataset analysis showing 55.42% of coreferential event mentions are more than five sentences apart is particularly compelling.

2. The paper provides a thorough error analysis that identifies specific challenges, particularly pronoun resolution (e.g., "It's crazy that we're bombing Iraq. It sickens me"), which is a difficult problem even for expert annotators. This analysis is valuable for guiding future research directions and puts the model's performance in context.

3. The detailed feature ablation study (Table 5) convincingly demonstrates the importance of all feature classes used in the model. The results show that removing any feature class leads to significant performance drops, which strongly supports the authors' design choices and provides empirical evidence for the value of each component.

## Weaknesses

1. The choice of CNNs over other architectures (e.g., RNNs, LSTMs) is not justified or compared. The authors claim CNNs are "well-suited" for this task but provide no evidence or analysis to support this claim. The 2-3 point F1 gains over previous work require stronger architectural evidence, especially since the paper doesn't compare to existing continuous-space event-linking methods or baseline neural models.

2. The novelty of the approach is unclear. Using CNNs for sentence classification is not new, and the paper doesn't properly compare to existing continuous-space event-linking methods. Table 3's baselines are limited to older rule-based and feature-engineered approaches (Bejan and Harabagiu, 2010; Liu et al., 2014), missing comparisons to more recent neural approaches. The authors should expand their baselines to include recent neural models that also use continuous-space representations.

3. The ACE++ data collection process is inadequately described. The paper provides no details about event-type coverage, how the additional 1,351 web documents were selected and annotated, or potential biases in the new data. The authors should show where earlier methods fail where theirs succeeds, and they should consider releasing ACE++ to the community.

4. The paper reports model performance without gold references in Table 3, which is a standard practice in coreference resolution. This omission makes it difficult to properly evaluate the model's performance, as coreference resolution metrics are highly sensitive to reference sets.

5. The paper fails to explore simple yet potentially beneficial augmentations, such as incorporating off-the-shelf coreference systems (e.g., CoreNLP) as suggested in line 687. Given the focus on improving coreference resolution, this would be a natural extension worth exploring.

## Questions to Authors

1. Can you provide a detailed comparison of your CNN-based approach with equivalent RNN/LSTM models on ACE++ to justify your architectural choice? How much would performance drop if you used a different architecture?

2. Could you clarify the ACE++ data collection process, including the criteria for selecting the additional 1,351 web documents and how event types are distributed across the corpus? This is critical for understanding the validity of your results.

3. Would you consider releasing ACE++ to the research community to facilitate further work on event linking? If not, what are the specific reasons preventing this release?

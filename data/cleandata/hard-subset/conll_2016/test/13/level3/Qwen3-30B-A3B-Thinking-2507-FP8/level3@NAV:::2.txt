

**Review**

**Summary and Contributions**  
This paper proposes an event coreference resolution model using sentential CNN features and a two-part architecture for mention representation and coreference scoring. The primary contributions are: (1) the creation of ACE++, a larger event coreference dataset addressing ACE's limited size; (2) a CNN-based approach that avoids external semantic features while achieving SOTA performance; and (3) analysis of argument sharing patterns in event mentions. While the dataset creation is valuable, the core technical contribution lacks novelty.

**Strengths**  
1. The ACE++ dataset addresses a critical limitation in prior work (ACE's small size) and demonstrates the model's scalability. The authors' analysis of argument sharing (23% incomplete event mentions) provides strong motivation for the task's importance.  
2. The model achieves 2-3 point F1 gains over prior work without relying on external semantic resources, which is significant for cross-domain applicability.  
3. The paper is well-written with clear technical explanations and a thorough error analysis identifying pronoun resolution as a key challenge.

**Weaknesses**  
1. **Critical novelty gap**: Using CNNs for sentence-level event representation is not new (Zeng et al., 2014; Chen et al., 2015). The paper fails to compare with existing continuous-space methods (e.g., Liu et al., 2014) or justify CNN selection over RNNs/transformers, undermining the architectural contribution.  
2. **Insufficient dataset description**: ACE++'s event-type coverage, biases, and creation methodology are unclear. The authors should demonstrate where prior methods fail and why their approach succeeds, and release ACE++.  
3. **Lack of architectural evidence**: The 2-3 point F1 improvement requires stronger justification. Table 5 shows pairwise features remain important, yet the paper doesn't explain which features mattered or alternatives tried.  
4. **Inadequate baselines**: Table 3 omits gold-reference performance and fails to compare with continuous-space methods. The model should be evaluated on EventCorefBank and tested with off-the-shelf coreference systems (e.g., CoreNLP) as suggested in line 687.

**Questions to Authors**  
1. Why were CNNs specifically chosen over RNNs or transformers for sentence representation, given their established use in NLP?  
2. Can you provide evidence that ACE++ is balanced across event types and not biased toward specific domains?  
3. How would incorporating off-the-shelf coreference systems (e.g., CoreNLP) impact your model's performance?

- Proposes a CNN-based two-stage neural model for event coreference that avoids external semantic resources by using sentential features.  
- Claims SOTA performance on ACE and ACE++ without semantic resources, showing domain-agnostic event linking is feasible.  
- Introduces ACE++, a larger publicly released dataset (1,950 docs) to address ACE’s small size.  
- Ablation shows sentential features are necessary and outperform lexical-only approaches (Table 5).  

- Strong empirical results: 73.31% BLANC F1 on ACE, outperforming prior work without external features.  
- Dataset analysis (argument sharing and mention distance) supports the need for sentential features.  
- ACE++ release increases recall and enables future event-linking work.  

- No justification for choosing CNNs over RNNs/LSTMs; no architectural comparison undermines novelty.  
- Fails to compare to neural event-linking baselines, so SOTA claim is unsubstantiated.  
- ACE++ construction details (event-type coverage, annotation biases, selection criteria) are missing, harming reproducibility.  
- Evaluation lacks a gold-trigger baseline, making reported scores potentially inflated.  
- Error analysis is superficial: pronoun failures noted but not analyzed or addressed; ambiguous annotation cases unused.  

- Why use CNNs instead of RNNs/LSTMs for sentential features; provide ablation on ACE++.  
- How does ACE++’s event-type distribution compare to ACE and does it introduce domain bias?  
- Why was the BestLink clustering strategy rejected despite prior SOTA use?  
- What is the impact of using gold event triggers versus predicted ones on results?  
- How can the model be adapted to handle pronoun-based coreference (e.g., discourse-aware features)?  

- Paper should explicitly compare to neural methods (e.g., Zeng et al. 2014) in Table 3.  
- ACE++ creation process (web sources, annotation guidelines) needs full description.  
- Error analysis should quantify pronoun-case frequency and link errors to model limitations.

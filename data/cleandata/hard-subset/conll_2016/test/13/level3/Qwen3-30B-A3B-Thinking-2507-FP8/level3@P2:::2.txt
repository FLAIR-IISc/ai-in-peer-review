

**Summary and Contributions**  
The paper proposes a neural model for event coreference resolution that avoids reliance on external semantic resources (e.g., WordNet, FrameNet) by leveraging sentential features from convolutional neural networks (CNNs). The model processes event mentions via two stages: (1) CNN-based representation of individual mentions using context, and (2) coreference scoring via pairwise features combined with these representations. Key contributions, ordered by strength, are:  
1. **State-of-the-art performance without external features**: Achieves SOTA results on ACE and ACE++ without semantic resources, demonstrating that domain-agnostic event linking is feasible.  
2. **ACE++ dataset creation**: Addresses ACE’s small size (599 docs) by creating a larger, publicly available dataset (ACE++, 1,950 docs) with analogous web-annotated documents.  
3. **Empirical validation of sentential features**: Rigorously demonstrates the necessity of sentential features (Table 5) through ablation, showing they outperform lexical-only approaches.  

**Strengths**  
1. **Strong empirical results**: The model achieves 73.31% BLANC F1 on ACE (outperforming Liu et al. 2014 by 2.87 points) without external features, proving that high performance is possible without domain-specific resources. This addresses a critical gap in cross-domain applicability.  
2. **Insightful dataset analysis**: The analysis of argument sharing (50.97% coreferential pairs share no arguments) and mention distance (55.42% >5 sentences apart) justifies the need for sentential features, strengthening the methodological rationale.  
3. **Practical dataset contribution**: ACE++ is publicly released, enabling future work on event linking. The dataset expansion (from 599 to 1,950 docs) significantly boosts recall (75.16% → 83.21%), demonstrating scalability.  

**Weaknesses**  
1. **Lack of architectural justification**: The choice of CNNs over established alternatives (e.g., RNNs, LSTMs) is unexplained. The paper claims CNNs capture "sentential features" but provides no comparison to RNN-based approaches (e.g., Chen et al. 2015), which are standard for sentence-level tasks. This undermines novelty claims, as CNNs for sentence representation are not new.  
2. **Unclear novelty and insufficient baselines**: The paper fails to compare to *neural* event-linking methods (e.g., Zeng et al. 2014; Nguyen and Grishman 2015), which also use continuous-space representations. Table 3 only compares to non-neural baselines (e.g., Bejan and Harabagiu 2010), making the "SOTA" claim unsubstantiated. The 2–3 point F1 gains require stronger evidence of architectural superiority.  
3. **ACE++ construction lacks detail**: The dataset’s event-type coverage, annotation biases, and web-document selection criteria are omitted. Without this, it’s unclear why ACE++ is suitable for generalization (e.g., does it cover the same event types as ACE?). This hinders reproducibility and validity.  
4. **Missing gold-reference baseline**: Table 3 reports performance without gold event mentions (e.g., using predicted triggers), which inflates scores. A comparison to gold-trigger baselines (e.g., Liu et al. 2014) is essential for fair evaluation.  
5. **Superficial error analysis**: The analysis notes pronoun resolution as a failure case (e.g., "It’s crazy that we’re bombing Iraq") but does not explore *why* the model fails (e.g., lack of discourse features) or propose mitigations. The 17 ambiguous cases (where annotators disagreed) are acknowledged but not leveraged to refine the model.  

**Questions to Authors**  
1. Why were CNNs chosen over RNNs/LSTMs for sentential feature extraction? Provide ablation results comparing these architectures on ACE++.  
2. How does ACE++’s event-type distribution (e.g., Conflict.Attack, Business.MergeOrg) align with ACE? Does it introduce domain biases?  
3. Why was the "BestLink" clustering strategy (Liu et al. 2014) rejected despite its use in prior SOTA work?  
4. What would be the impact of integrating gold event triggers (vs. predicted ones) on the reported results?  
5. How might the model be adapted to handle pronoun-based coreference (e.g., via discourse-aware features)?  

**Additional Comments**  
- The paper should explicitly compare to neural event-linking methods (e.g., Zeng et al. 2014) in Table 3.  
- ACE++’s creation process (e.g., web-document sources, annotation guidelines) needs full description.  
- The error analysis should quantify how often pronoun cases occur (e.g., % of false negatives) and link to model limitations.

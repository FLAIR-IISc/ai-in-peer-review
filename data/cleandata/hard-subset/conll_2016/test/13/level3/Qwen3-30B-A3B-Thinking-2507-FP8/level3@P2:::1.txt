

# Review for "Event Linking with Sentential Features from Convolutional Neural Networks"

## Summary and Main Contributions
This paper proposes a neural network-based approach for event linking that avoids reliance on external semantic resources (WordNet, FrameNet, etc.) by using sentential features generated via convolutional neural networks. The main contributions, as I see them, are:

1. A novel neural model for event linking that achieves state-of-the-art performance without external semantic features, demonstrating that domain-specific semantic knowledge is not necessary for high performance.
2. An ablation study showing the specific contributions of different feature classes (sentential, lexical, pairwise) to the model's performance.
3. An error analysis identifying specific challenges for future work, particularly regarding pronoun resolution and boundary cases in event coreference.

## Strengths
1. The core contribution of achieving competitive performance without external semantic resources is significant. The model outperforms prior work (Liu et al., 2014) on ACE while avoiding reliance on domain-specific semantic resources, which is a major practical advantage for real-world applications. The results on ACE++ demonstrate the model's ability to generalize with more training data.

2. The ablation study (Table 5) provides clear evidence for the importance of each feature class. The performance drops when removing any feature class (from 76.90 F1 to 56.69 F1 when removing all features) convincingly demonstrate that the combination of sentential, lexical, and pairwise features is necessary for optimal performance.

3. The error analysis reveals important insights about the limitations of current approaches. The identification of pronoun resolution as a significant challenge (e.g., "It's crazy that we're bombing Iraq. It sickens me.") and the observation that 22% of false positives were plausible to human raters provides valuable direction for future research.

## Weaknesses
1. The experimental setup uses gold-standard trigger words rather than predicted ones, making comparisons to other systems unfair. Most prior work (Liu et al., 2014; Bejan and Harabagiu, 2010) uses predicted triggers, while this paper's use of gold triggers artificially inflates performance, particularly since trigger identification is a critical component of the event extraction pipeline.

2. The ACE++ dataset is not publicly available, limiting reproducibility and preventing comparison of the model's performance against feature-rich systems on this dataset. The authors should release ACE++ to allow for more meaningful comparison with other approaches.

3. The paper lacks statistical significance testing for the performance differences. The reported improvements (e.g., 73.31 vs. 72.19 F1 with different clustering strategies) are small, and without significance testing, it's unclear if these differences are meaningful.

4. Several aspects of the model are unclear and need better explanation: the generation of positional embeddings (randomly initialized or lexicalized?), the specific representation of left/right neighbors, and the exact encoding of pairwise features (binary vs. continuous). The paper claims "the same relative position should share an embedding across words" without justification.

5. The paper fails to compare with recent neural coreference approaches (e.g., Wiseman et al., 2015), which would have strengthened the position that neural approaches are the right direction for this task.

## Questions to Authors
1. The paper uses gold-trigger words, which is a significant advantage over most prior work. Can you provide results with predicted triggers to make the comparison fairer? If not, can you explain why this is justified?

2. The ACE++ dataset is not publicly available. Will the authors release this dataset to enable reproducibility and further research?

3. The paper mentions "the square function as the network's non-linearity" (Equation 4). Is this novel, and what is the theoretical justification for using the square function rather than more common nonlinearities like ReLU?

4. The error analysis notes that 17 false negatives were cases where annotators made different decisions. How did the authors select the 100 false positives and negatives for analysis to ensure they represent the overall error distribution?

5. The paper states "the model achieves state-of-the-art performance on two datasets," but the ACE++ results are not compared to prior work on this dataset. Can you provide a comparison of your model with prior work on ACE++?

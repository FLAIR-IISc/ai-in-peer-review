This paper presents a neural network model for within-document event coreference resolution (event linking). The proposed model avoids reliance on rich, external semantic resources like WordNet or FrameNet, which are common in prior systems. The approach first generates a vector representation for each event mention. This is done by feeding word embeddings and relative position embeddings for the event's sentence into a Convolutional Neural Network (CNN) with piecewise max-pooling. This learned sentential representation is then concatenated with embeddings for the trigger word and its immediate neighbors. To score a candidate coreference link, the representations of two event mentions are combined with a small set of standard pairwise features (e.g., event type agreement, distance). This combined vector is passed through a hidden layer with a square activation function and a final logistic regression layer to produce a coreference score. The model is evaluated on the standard ACE 2005 dataset and a new, larger dataset called ACE++, achieving performance that is comparable to previous state-of-the-art systems that use extensive feature engineering.

### Contributions

1.  **A Hybrid Neural Architecture for Event Linking:** The paper's primary contribution is a neural model that effectively combines learned, dense representations with a small set of discrete linguistic features for event coreference. It successfully uses a CNN to capture relevant sentential context for an event mention and demonstrates how this can be integrated with local lexical and pairwise features in a unified framework.

2.  **Competitive Performance without External Knowledge Bases:** The work provides strong empirical evidence that a neural approach can match the performance of heavily feature-engineered systems without depending on external knowledge bases. This is a significant finding, as it suggests a path toward more portable and domain-independent event processing systems.

3.  **Ablation Study and a New Dataset Resource:** The authors present a valuable ablation study (Table 5) that clearly demonstrates the complementary contributions of the sentential, local lexical, and pairwise feature components of their model. Additionally, they introduce ACE++, a larger annotated corpus for this task which, if released, would be a valuable resource for the community.

### Strengths

1.  **Strong Performance with a More Generalizable Approach:** The model's key strength is achieving competitive results on the ACE benchmark while deliberately avoiding external semantic resources. This marks a departure from prior systems (e.g., Liu et al., 2014) that rely on a large number of handcrafted features from sources like WordNet, making the proposed method more adaptable to new domains where such resources may not be available.

2.  **Principled Integration of Different Information Sources:** The model architecture is well-motivated. It thoughtfully combines three distinct types of information: (1) global sentential context captured by the CNN, (2) specific local context from the trigger and its neighbors, and (3) relational information from pairwise features. The ablation study effectively validates this design, showing that removing any component degrades performance.

3.  **Thorough Error Analysis:** The manual error analysis provides useful insights into the remaining challenges of the task. It highlights difficult cases like pronominal event anaphora ("It sickens me") and points to inherent annotation ambiguities, which helps to contextualize the reported performance scores and suggests important directions for future research.

### Weaknesses

1.  **Issues with Experimental Validity and Comparison:** The main weakness lies in the experimental setup, which makes the central claim of achieving state-of-the-art performance difficult to verify.
    *   **Use of Gold Triggers:** The experiments use gold-annotated event triggers. This is a significant advantage not available to systems that operate on predicted triggers as part of a more realistic, end-to-end pipeline. Comparisons to systems that use predicted information are therefore unfair and likely overstate the proposed model's relative performance.
    *   **Lack of Significance Testing:** Many of the reported scores in Table 3 are numerically close (e.g., BLANC F1 of 73.31 vs. 70.43). Without statistical significance tests, it is impossible to know if the reported improvements are meaningful or simply due to dataset/run variance.
    *   **Inconsistent Data Splits:** The authors acknowledge that different papers use different splits of the ACE dataset. While their split is made public, this choice further complicates direct comparison and weakens claims of outperforming prior work.

2.  **Lack of Clarity and Reproducibility:** Several aspects of the model and data are not described with sufficient clarity, which hinders reproducibility.
    *   The new ACE++ dataset is not public, meaning results on it cannot be verified or built upon by other researchers. Claims based on this dataset are therefore of limited value to the community.
    *   The model description contains several ambiguities. For example, it is unclear how the final event representation `v_e` (line 330) is used, as subsequent sections only reference `v_{sent+lex}`. The encoding of pairwise features is also underspecified.
    *   The concluding claim that the model's performance "encourages the use of sentential features" (lines 65-70) because many coreferent mentions are far apart is not well-supported by the model itself, which only processes single sentences at a time to build mention representations. The link between this observation and the proposed single-sentence CNN architecture needs to be explained more clearly.

3.  **Incomplete Review of Related Work:** The paper misses a key citation to the contemporary work of Wiseman et al. (2015) on neural entity coreference. While focused on entities, their recurrent neural network-based approach is highly relevant methodologically and should be discussed.

### Questions to Authors

1.  Could you clarify the relationship between `v_e` (line 330) and `v_{sent+lex}`? The text defines `v_e` as the output of a hidden layer applied to `v_{sent+lex}`, but the coreference scoring component in Figure 1b appears to take `v_{sent+lex}` as direct input. Which representation is actually used for scoring?
2.  How are the pairwise features in Section 3.2 encoded? Are discrete features represented as one-hot vectors? How are continuous features like sentence distance handled (e.g., are they binned)?
3.  Regarding the positional embeddings, could you clarify how they are generated? Are they randomly initialized and learned for each relative integer position (e.g., -3, -2, -1, 0, 1...)? What is the motivation for a single embedding for a given relative position to be shared across all words?
4.  The square function is an interesting choice for the non-linearity in the similarity layer. Is this a novel contribution of this work, and have you investigated its broader applicability to other similarity-matching tasks?
5.  What is the justification for using a fixed window of exactly one word before and after the trigger? Was this size determined empirically on a development set?
6.  Are there plans to release the ACE++ dataset and the implementation code to ensure reproducibility and support future research on this task?

### **Summary and Contributions**

This paper presents a neural network model for within-document event coreference resolution. The model first learns a distributed representation for each event mention by applying a convolutional neural network (CNN) over the sentence containing the mention. This sentence-level representation, derived from word and position embeddings via convolution and piecewise max-pooling, is then combined with local lexical features (embeddings of the trigger and its neighbors) and a set of standard pairwise features (e.g., event type match, distance). This combined vector is fed through a similarity function and a logistic regression layer to compute a coreference score for a pair of event mentions. The model is evaluated on the standard ACE 2005 dataset and a new, larger dataset called ACE++, achieving state-of-the-art or competitive performance without relying on external knowledge bases like WordNet or FrameNet.

In my view, the main contributions of this work are:
1.  **An effective neural architecture for event linking that avoids reliance on external knowledge bases.** The primary contribution is demonstrating that a model combining learned sentential features with a small set of standard linguistic features can match the performance of prior state-of-the-art systems that depend on heavily-engineered features from external semantic resources. This makes the approach more portable across domains.
2.  **An insightful ablation study demonstrating the complementary strengths of different feature types.** The experiments on the new ACE++ dataset clearly show that sentential, local lexical, and pairwise features all contribute positively to the model's performance. This validates the architectural design and underscores that for a complex task like event coreference, learned latent features and explicit linguistic features are best used in concert.

### **Strengths**

1.  **Strong Empirical Performance:** The model achieves results on the ACE 2005 dataset that are on par with or superior to previous state-of-the-art systems across several metrics (BLANC, MUC, positive-links F1). The key strength here is that it does so without the complex feature engineering and reliance on domain-specific resources (WordNet, FrameNet, DBpedia) that characterize prior work. This is a significant step towards more generalizable and less brittle event processing systems.
2.  **Well-Motivated Model Architecture:** The design choices are generally well-justified. The use of a CNN to capture sentential context is motivated by recent successes in related tasks. The decision to combine these learned representations with both trigger-local features and explicit pairwise features is sound, and the ablation study in Table 5 provides strong evidence that this hybrid approach is effective. The analysis shows that each feature class is important, preventing an over-reliance on a single source of information.
3.  **Thorough Experimental Analysis:** The paper provides a comprehensive evaluation beyond a single headline result. It includes an analysis of the impact of training data size (ACE vs. ACE++), an interesting comparison of clustering strategies (transitive closure vs. BestLink), and a set of simple but informative baselines. Furthermore, the qualitative error analysis offers valuable insights into the remaining challenges of the task, such as pronoun resolution and annotation ambiguity, which helps frame the contributions and points toward concrete directions for future work.

### **Weaknesses**

1.  **Issues with Experimental Setup and Reproducibility:** The evaluation framework has several weaknesses that temper the paper's conclusions.
    *   **Use of Gold Triggers:** The experiments rely on gold-standard event mention annotations. This is an unrealistic setting that sidesteps the challenges of an end-to-end system where errors from an event trigger detector would propagate. It also makes comparisons with systems that use predicted triggers difficult to interpret.
    *   **Proprietary Dataset:** A significant portion of the analysis and the strongest results (e.g., showing the benefit of more data) are on the new ACE++ dataset, which is not publicly available. This hinders reproducibility and prevents the community from building on these results. Claims made on a private dataset are inherently weaker.
    *   **Inconsistent Data Splits:** The authors acknowledge that direct comparison with prior work is difficult due to varying train/test splits of the ACE corpus. While this is a known problem in the field, the work would be stronger if it had adhered to a specific, widely-used split from a key publication.
2.  **Lack of Statistical Significance Testing:** The paper makes several claims of superior performance based on small numerical differences in evaluation metrics (e.g., the F1 scores in Table 3). Without statistical significance tests, it is impossible to know if these differences are meaningful or simply due to random variation in the data or model initialization. This is a critical omission in a heavily empirical paper and weakens the claims of achieving state-of-the-art performance.
3.  **Insufficient Justification for Some Design Choices and Missing Related Work:** Several architectural details feel arbitrary and are not well-motivated.
    *   It is unclear why the embeddings for exactly one word before and after the trigger are concatenated with the sentential representation. An ablation or explanation for this specific window size is needed.
    *   The paper suffers from some notational ambiguity (e.g., `v_e` is introduced but `vsent+lex` is used later), which makes the model description harder to parse.
    *   The related work section misses some contemporary neural approaches to coreference resolution, notably Wiseman et al. (2015, "Learning a Recurrent Neural Network for Grammatically-Aware Coreference Resolution"), which would have provided a relevant point of comparison for neural architectures on a related task.

### **Questions to Authors**

1.  The use of a `square` function as the non-linearity in the similarity layer is an interesting choice. Could you elaborate on the intuition behind this? Is this a novel technique, and did you find it outperformed more standard activations like `relu` or `tanh` in this specific context?
2.  Could you provide more detail on how the pairwise features in Section 3.2 are encoded? For instance, are categorical features represented with one-hot vectors, and how are continuous features like sentence distance handled (e.g., raw values, binned features)? Are these feature representations learned during training?
3.  Could you clarify the implementation of the positional embeddings? Are they randomly initialized and learned? What is the motivation for having a single embedding for a given relative distance (e.g., '-3'), regardless of the word it modifies or the sentence structure?
4.  The ACE++ dataset seems to be a valuable resource. Are there any plans to release it to the research community?
5.  Your error analysis correctly identifies pronoun resolution as a major challenge. Given that your CNN generates representations from single sentences, how do you see the model capturing the necessary cross-sentential context for such cases, beyond what is provided by the simple pairwise distance features?

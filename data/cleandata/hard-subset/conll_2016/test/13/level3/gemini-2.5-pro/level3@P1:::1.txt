### Summary

This paper proposes a neural network model for within-document event coreference resolution. The core of the model is a Convolutional Neural Network (CNN) that learns sentential feature representations for candidate event mentions. For each event mention, the model processes its sentence using a CNN with piecewise max-pooling to generate a fixed-size vector. This vector is then concatenated with local lexical features (embeddings of the event trigger and its immediate neighbors) and a set of standard pairwise features (e.g., event type match, modality agreement, distance). The resulting combined representation for a pair of event mentions is fed through a final classification layer to produce a coreference score. The authors evaluate their model on the ACE 2005 dataset and a new, larger proprietary dataset (ACE++). The results show that the model, without relying on external knowledge bases like WordNet or FrameNet, achieves performance comparable to previous state-of-the-art systems that are heavily feature-engineered.

### Main Contributions

1.  **An Effective Knowledge-Lean Model for Event Linking:** The primary contribution is a strong model for event coreference that replaces dependence on external semantic resources (e.g., WordNet, FrameNet) with learned distributed representations. By showing that a CNN can effectively capture sentential context to achieve state-of-the-art results, the work demonstrates a viable path towards more domain-portable event processing systems.

2.  **A Principled Combination of Learned and Hand-Crafted Features:** The paper effectively demonstrates that for this task, the best performance is achieved by combining learned neural representations with a curated set of traditional linguistic features. The ablation study (Table 5) clearly shows that sentential features from the CNN, local lexical features, and pairwise relational features are all complementary and essential, providing a valuable insight for future work in this area.

### Strengths

1.  **Reduced Reliance on External Knowledge:** The model's key advantage is its independence from external, domain-specific knowledge bases. Previous SOTA systems (e.g., Liu et al., 2014) required a large number of hand-crafted semantic features derived from resources like WordNet, which limits their applicability to new domains. This work successfully shows that learned representations can serve as a powerful substitute, which is a significant practical contribution.

2.  **Strong Empirical Performance:** The model achieves competitive results on the standard ACE 2005 benchmark, outperforming prior work on several metrics (BLANC, MUC F1). This validates the architectural choices and demonstrates the effectiveness of the proposed approach.

3.  **Thorough Ablation and Error Analysis:** The authors provide a detailed ablation study that isolates the contribution of each feature class (pairwise, local, sentential), convincingly showing that all are necessary. The error analysis offers useful qualitative insights into remaining challenges, such as pronominal anaphora resolution and annotation ambiguities, which helps frame directions for future research.

### Weaknesses

1.  **Unrealistic Experimental Setting:** The experiments rely on gold-annotated event triggers and arguments. This is a major limitation, as it does not reflect a realistic end-to-end setting where these elements must be predicted, often imperfectly. This setup inflates performance and complicates comparisons with systems that may operate on predicted inputs. The paper should, at minimum, acknowledge and discuss the impact of this idealization.

2.  **Reproducibility and Comparison Concerns:**
    *   A significant portion of the experiments are conducted on a new, larger dataset called ACE++, which is not public. This makes the results on it impossible to verify or build upon, limiting their value to the community.
    *   The authors note that different train/test splits of the ACE dataset are used across the literature, which makes direct, fair comparison difficult. The paper would be stronger if it used a standard, established split or rigorously justified its choice of a new one.
    *   The performance differences between the proposed model and prior work are often small (e.g., ~1% F1 on BLANC). Claims of achieving "state-of-the-art performance" should be supported by statistical significance tests to be fully convincing.

3.  **Lack of Clarity in Model Description:** Several key architectural and implementation details are ambiguous, hindering full understanding and reproducibility.
    *   The generation of positional embeddings is not clearly explained (e.g., are they randomly initialized and learned? Why is it assumed that the same embedding is appropriate for a given relative position regardless of the word at that position?).
    *   The exact encoding of the pairwise features is not specified. It is unclear how continuous values like distance are handled (e.g., binning) and whether their representations are learned.
    *   The choice to use a window of exactly one word to the left and right of the trigger for local features seems arbitrary and is not justified.

### Questions to Authors

1.  Could you please clarify the implementation details of the pairwise features? Specifically, how are continuous features like distance encoded, and are the representations for these features learned during training or fixed?

2.  There appears to be a slight inconsistency in the model description. Equation 3 defines a final event-mention representation `v_e` as the output of a hidden layer. However, this term is not mentioned again, and Equation 4 seems to use `v_sent+lex` directly. Is `v_e` used, or is the representation fed into the similarity function simply the concatenation `v_sent+lex`?

3.  The use of the `square` function as a non-linearity in Equation 4 is interesting. Is this a novel contribution of this work? Could you provide more intuition or an empirical comparison to more standard activation functions (like ReLU or tanh) to further justify this choice?

4.  To aid reproducibility and future research, do you plan to release the ACE++ dataset and the exact document lists for your train/validation/test splits of the ACE 2005 corpus?

### Additional Comments

*   The related work section could be strengthened by citing more recent neural approaches to coreference, such as the entity coreference model of Wiseman et al. (EMNLP 2015).
*   There is a minor inconsistency in naming a baseline "same event" in the text (line 670) but "Same type" in Table 6.

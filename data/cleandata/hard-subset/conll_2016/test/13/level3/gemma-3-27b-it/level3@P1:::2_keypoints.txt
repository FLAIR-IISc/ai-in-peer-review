- Proposes an event linking method that learns sentential event representations with CNNs and uses pairwise features for coreference without external KBs.
- Achieves state-of-the-art performance on ACE and a newly created ACE++ dataset.
- Main contribution is the creation and release of the ACE++ dataset to address ACE 2005’s limitations.
- Shows that strong event linking can be done without extensive external semantic features, improving portability across domains.
- Provides a thorough error analysis highlighting challenges like pronoun resolution and annotator disagreement.
- ACE++ is well-motivated with statistics (e.g., argument incompleteness) supporting the need for a larger dataset.
- Model’s reported performance gains are notable, especially given avoidance of external knowledge sources.
- Paper is clearly written, well-structured, and model design choices are justified.
- The choice of CNNs lacks sufficient architectural motivation and comparative analysis with alternatives (RNNs/Transformers).
- Methodological novelty is limited because sentence-level CNNs for NLP tasks are not inherently new; relation to prior continuous-space event-linking work is underexplored.
- ACE++ dataset creation details (collection, annotation guidelines, event-type coverage, biases) are insufficiently described.
- Baseline comparisons are limited; more recent and competitive methods and feature configurations should be included.
- Requests clarification on why CNNs were chosen and whether alternatives were tried, and on the most informative pairwise features.
- Asks for detailed ACE++ annotation quality controls, event-type distribution, and potential biases.
- Suggests evaluating model with gold-standard entity coreference, integrating off-the-shelf coreference systems, and testing on other datasets (e.g., EventCorefBank).
- Recommends considering joint entity-event coreference experiments given dataset sizes.

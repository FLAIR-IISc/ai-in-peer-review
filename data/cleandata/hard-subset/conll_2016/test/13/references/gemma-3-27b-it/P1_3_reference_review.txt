## Peer Review: Event Linking with Sentential Features from Convolutional Neural Networks

**Summary:**

This paper presents a system for event linking (coreference resolution for event mentions) that utilizes convolutional neural networks (CNNs) to generate sentential features for event mentions. The core idea is to move away from reliance on external semantic resources (WordNet, FrameNet) by leveraging the semantic information contained within the sentences surrounding event triggers. The system consists of two main parts: a CNN-based module for representing individual event mentions, and a pairwise classification module for determining coreference links, combined with standard pairwise features. The authors demonstrate state-of-the-art or competitive performance on two datasets (ACE and ACE++), and provide an error analysis to identify areas for improvement.

**Main Contributions (as I see them):**

1. **Demonstration of Effective Sentential Feature Learning for Event Linking:** The paper successfully shows that CNNs can effectively learn useful representations of event mentions from surrounding sentences, achieving strong performance on event linking without relying on expensive and potentially domain-specific external knowledge resources.
2. **Comprehensive Experimental Evaluation:** The authors conduct a thorough evaluation on two datasets, including a newly created larger dataset (ACE++), and provide detailed ablation studies exploring the impact of different feature classes.
3. **Insightful Error Analysis:** The included error analysis provides valuable insights into the types of errors the system makes, highlighting challenges like pronoun resolution and annotator disagreement, and suggesting directions for future work.

**Strengths:**

1. **Strong Performance:** The system achieves competitive, and in some cases, state-of-the-art results on the ACE and ACE++ datasets, demonstrating the effectiveness of the proposed approach.  The comparison with existing literature is fair and well-presented.
2. **Novelty in Feature Engineering:** While CNNs are increasingly used in NLP, the application to event linking using *sentential* features as described is a significant contribution.  Specifically, the combination of local trigger features *with* sentence-level features generated by the CNN appears to be a key component.  The justification for this design (avoiding expensive external resources and capturing broader context) is well-motivated.
3. **Detailed Ablation Studies:** The ablation studies presented in Table 5 clearly demonstrate the contribution of each feature class (pairwise, trigger-local, sentential). This provides valuable insight into the model's behavior and validates the design choices.
4. **Practical Dataset Contribution:**  The creation and release of the ACE++ dataset is a valuable contribution to the research community, providing a larger and potentially more representative dataset for event linking research.
5. **Thoughtful Error Analysis:**  The error analysis goes beyond simply identifying errors; it links them to specific linguistic phenomena (pronoun resolution) and even acknowledges potential issues with annotation consistency. This demonstrates a nuanced understanding of the task.

**Weaknesses:**

1. **Limited Novelty in Core Methodology:** The core CNN architecture and pairwise classification approach are relatively standard in the broader NLP literature.  The novelty lies primarily in the application to event linking with the specific feature engineering choices. While effective, the methodological innovation is incremental rather than groundbreaking.
2. **Lack of Discussion of Computational Cost:** The paper doesn't address the computational cost associated with the CNN-based feature extraction.  While performance is important, scalability and efficiency are also critical considerations, particularly for large-scale applications. It would be helpful to discuss the training and inference time.
3. **Dependence on Gold Event Mentions:** The system relies on gold-standard event mentions as input.  This limits its applicability in real-world scenarios where event extraction and linking need to be performed jointly.  A discussion of how the system might be adapted to work with imperfect event mentions would be beneficial.
4. **Clustering Strategy Justification:** While the authors explain *why* BestLink didn’t work, the justification for using the transitive closure strategy feels somewhat underdeveloped.  A more in-depth comparison and discussion of the tradeoffs between different clustering approaches would be useful.



**Questions to Authors:**

1. **Computational Cost:**  Could you provide some information about the training and inference time of your model?  How does the computational cost scale with document length and dataset size?
2. **Error Analysis & Pronoun Resolution:** Given the difficulty of pronoun resolution, have you considered incorporating coreference resolution techniques at the entity level as a pre-processing step to potentially improve event linking performance?
3. **Joint Extraction and Linking:**  How would you envision adapting your model to work in a joint event extraction and linking setting, where event mentions are not pre-identified? What modifications would be necessary?
4. **ACE++ Dataset Details:** Can you provide more details about the process of annotating the ACE++ dataset? Were the annotators the same as those who annotated ACE? Were there any specific guidelines or training procedures used to ensure consistency?



**Additional Comments:**

*  The paper is well-written and clearly organized.
*  The figures are helpful in illustrating the model architecture and process flow.
*  The reference list is comprehensive.




Overall, this is a solid and well-executed paper that makes a valuable contribution to the field of event linking. The strengths of the work outweigh the weaknesses, and I recommend acceptance with minor revisions addressing the questions above.

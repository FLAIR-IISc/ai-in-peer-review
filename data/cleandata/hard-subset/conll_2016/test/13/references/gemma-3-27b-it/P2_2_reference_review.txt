## Review: Event Linking with Sentential Features from Convolutional Neural Networks

**Summary:** This paper presents a system for event linking (coreference resolution of event mentions) that utilizes convolutional neural networks (CNNs) to generate sentential features representing event mentions. The system eschews external semantic resources like WordNet and FrameNet, relying instead on local and sentential features combined with pairwise comparison features for coreference decision. The authors report state-of-the-art results on two datasets (ACE and a new dataset, ACE++), and provide an error analysis.

**Main Contributions (as I see them):**

1. **Demonstration of Effective Event Linking without External Lexical Resources:** The paper successfully demonstrates that strong event linking performance can be achieved *without* relying on traditional, potentially domain-specific, external lexical resources. This is a valuable contribution, increasing the portability and applicability of event linking systems.
2. **Empirical Evaluation of Sentential Features for Event Linking:** The paper provides a thorough empirical evaluation of the effectiveness of sentential features derived using CNNs for the event linking task. The ablation studies (Table 5) clearly show the contribution of these features.
3. **Creation and Release of ACE++ Dataset:**  The creation and public release of the ACE++ dataset expands the resources available to the event linking community, and provides a more substantial dataset for evaluation.



**Strengths:**

1. **Strong Empirical Results:** The system achieves competitive, and in some cases state-of-the-art, performance on both ACE and ACE++. This is a strong indicator of the method’s effectiveness. The authors fairly compare against existing methods.
2. **Clear and Thorough Methodology:** The model design is well-explained, and the experimental setup is detailed. The ablation studies are particularly valuable in understanding the impact of different feature sets. The inclusion of a detailed error analysis adds significant insight.
3. **Motivated Design Choices:** The authors provide a compelling motivation for their design choices, grounding them in an analysis of the characteristics of event linking data (e.g., the prevalence of non-shared arguments, distance between coreferential mentions). This demonstrates a good understanding of the problem.
4. **Practicality and Portability:** By avoiding external resources, the system is more readily adaptable to new domains and languages where those resources may not be available or of high quality.
5. **Detailed Error Analysis:** The error analysis identifies common failure modes (e.g., pronoun resolution), suggesting clear directions for future research.  The acknowledgment of annotation inconsistencies is particularly commendable.

**Weaknesses:**

1. **Limited Novelty in the CNN Architecture:** While the application of CNNs to generate sentential features *for event linking* is a reasonable approach, the CNN architecture itself (simple convolutional and max-pooling layers) is not particularly novel. The core innovation lies in the *application* rather than the *method* itself.  A stronger argument demonstrating architectural choices specifically suited to event linking would be beneficial.
2. **Reliance on Gold Event Mentions:** The system relies on gold-standard event mentions. While the authors acknowledge this, it’s a significant limitation. A more robust system would ideally incorporate event *extraction* and event *linking* into a single, end-to-end pipeline. Error propagation from the event extraction stage could significantly impact performance.
3. **Clustering Strategy Impact:** The results demonstrate that using BestLink clustering *decreases* performance, contrary to previous work (Liu et al., 2014).  While the authors offer a reasonable explanation, a deeper investigation into *why* this is the case would strengthen the paper. Exploring other clustering strategies might also be beneficial.
4. **Limited Discussion of Hyperparameter Tuning:** While the authors mention hyperparameter tuning, the details are somewhat sparse. More information about the search space and tuning methodology would increase confidence in the reported results.



**Questions to Authors:**

1. **Error Analysis Depth:** Could you provide more specific examples of the false negatives related to pronoun resolution? Are there any specific linguistic features that consistently appear in these cases?
2. **Event Extraction Integration:** Have you considered the potential impact of errors in event extraction on the performance of your event linking system? Do you have any preliminary results or plans for integrating event extraction and linking into an end-to-end pipeline?
3. **BestLink Clustering:**  Can you elaborate on the possible reasons why BestLink performed worse in your experiments compared to the transitive closure method? Were there any specific patterns observed in the errors made by BestLink?
4. **Hyperparameter Sensitivity:** What is the sensitivity of the model performance to different hyperparameter settings, particularly the dimensionality of the embedding layers (dw, dp, de, dsim)?




**Additional Comments:**

*   The paper is well-written and clearly organized.
*   The link to the dataset is helpful.
*   The inclusion of related work provides good context for the proposed approach.




**Overall Assessment:**

I recommend **Acceptance** with minor revisions. This paper presents a solid contribution to the field of event linking. While the CNN architecture isn’t groundbreaking, the application to event linking, the successful removal of external resources, the creation of a new dataset, and the thorough evaluation make this a valuable addition to the literature. Addressing the weaknesses outlined above (particularly a more detailed discussion of hyperparameter tuning and the clustering strategy) would further strengthen the paper.

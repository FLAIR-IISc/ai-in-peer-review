This paper presents a system for event linking that utilizes convolutional neural networks (CNNs) to generate sentential features for event mentions, aiming to reduce reliance on external semantic resources. The primary claimed contribution is achieving state-of-the-art performance on two event linking datasets without using knowledge bases like WordNet or FrameNet. A secondary contribution is an error analysis to identify areas for future work. As I see it, the primary contribution is a reasonably well-engineered application of established deep learning techniques (CNNs) to a specific NLP task, event linking. The demonstration of competitive performance is a positive outcome, but the methodological novelty is limited.

**Strengths:**

1. **Solid Experimental Evaluation:** The paper provides a thorough experimental evaluation, comparing the proposed model against several baselines on two datasets (ACE and ACE++). The inclusion of ACE++ as an extended dataset is a valuable addition, providing a larger testbed for the model.
2. **Clear Problem Definition & Motivation:** The introduction clearly defines the event linking problem and motivates its importance with relevant statistics and examples. The justification for moving away from external knowledge sources is well articulated.
3. **Detailed Model Description:** The model architecture is described in a clear and detailed manner, accompanied by a helpful diagram. The explanation of the convolution layer, pooling, and the integration of lexical and pairwise features is understandable and allows for replication.
4. **Insightful Error Analysis:** The error analysis, while brief, provides valuable insights into the limitations of the system. The identification of challenges with pronoun resolution and the discussion of annotation inconsistencies are particularly helpful.

**Weaknesses:**

1. **Limited Novelty:** The core technical approach – using CNNs to generate sentence representations for event mentions – builds heavily on existing work in relation extraction and event extraction. While the application to event *linking* is valid, the methodological innovation is incremental. The paper would benefit from a more nuanced discussion of how this work differs from, and improves upon, existing CNN-based approaches.
2. **Overstated Claims of "State-of-the-Art":** While the model performs competitively, the claim of achieving "state-of-the-art" should be qualified. The comparison to other systems is hampered by inconsistencies in experimental setup (e.g., use of gold vs. automatically generated arguments) and variations in dataset splits. A more careful comparison, potentially re-implementing key baselines with consistent settings, would strengthen this claim.
3. **Insufficient Ablation Study:** While Table 5 presents some ablation results, they are not comprehensive enough. It would be beneficial to see the impact of removing *combinations* of features, and to analyze the performance degradation more thoroughly. For example, how much performance is lost by removing *only* the pairwise features? Understanding these interactions would provide deeper insights into the model's behavior.
4. **Lack of Statistical Significance Testing:** The results presented in the tables lack statistical significance testing. Without knowing whether the observed performance differences are statistically significant, it's difficult to draw firm conclusions about the model's superiority.

**Questions to Authors:**

1.  Could you provide a more detailed comparison of your approach to other CNN-based methods for relation/event extraction, specifically highlighting the novel aspects of your architecture and training procedure?
2.  Given the discrepancies in experimental setup across different systems, would you consider re-implementing one or two key baselines with consistent settings to ensure a fairer comparison?
3.  Can you elaborate on the choice of the square function in the similarity calculation (Equation 4)? What alternatives were considered, and why was this specific function chosen?
4.   How sensitive is the model’s performance to the size of the embedding dimensions (dw, dp, de, dsim)? Did you perform a systematic search over these hyperparameters?

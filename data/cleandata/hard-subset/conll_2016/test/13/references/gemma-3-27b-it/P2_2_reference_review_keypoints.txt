- Paper proposes an event linking system using CNN-derived sentential features, avoiding external lexical resources.  
- Achieves state-of-the-art results on ACE and new ACE++ dataset and provides error analysis.  

Main contributions:
- Shows strong event linking performance without WordNet/FrameNet, improving portability.  
- Empirically evaluates CNN-based sentential features with ablation studies showing their importance.  
- Creates and releases the ACE++ dataset for the community.  

Strengths:
- Strong empirical results with fair comparisons to prior work.  
- Clear, thorough methodology and useful ablation studies and error analysis.  
- Well-motivated design choices based on data characteristics (non-shared arguments, mention distance).  
- Practical and portable approach by avoiding external resources.  
- Detailed error analysis identifying failure modes (e.g., pronoun resolution) and annotation issues.  

Weaknesses:
- CNN architecture itself is not novel; novelty is mainly in application.  
- System relies on gold event mentions; lacks end-to-end extraction+linking, risking error propagation.  
- BestLink clustering underperforms; deeper analysis of clustering choices is lacking.  
- Sparse details on hyperparameter tuning and search methodology.  

Questions for authors:
- Can you give concrete pronoun-resolution false negative examples and recurring linguistic patterns?  
- Any plans or preliminary results for integrating event extraction and linking end-to-end?  
- Why did BestLink perform worse here than in prior work; what error patterns did it produce?  
- How sensitive is performance to embedding dimensionalities (dw, dp, de, dsim)?  

Additional comments:
- Paper is well-written, dataset link is useful, and related work coverage is good.  

Overall assessment:
- Recommend acceptance with minor revisions, mainly to expand hyperparameter and clustering discussions.

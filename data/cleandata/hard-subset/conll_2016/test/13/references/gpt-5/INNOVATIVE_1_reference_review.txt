Summary and claimed contributions
- The paper tackles within-document event coreference (event linking) by learning sentential representations of event mentions with a CNN, then making pairwise coreference decisions using these learned representations plus a small set of simple pairwise features. Clustering is obtained by transitive closure over predicted links.
- Key claims: (1) A resource-light neural approach that forgoes external semantic resources (WordNet, FrameNet, VerbOcean) while achieving state-of-the-art results on ACE 2005; (2) principled use of sentential features for event coreference, supported by ablations that quantify the contribution of sentential vs. lexical vs. pairwise features; (3) expanded dataset (ACE++) and empirical analyses (distance, argument overlap, trigger mismatch, clustering strategies).
- Novelty lies primarily in applying CNN-based sentential features to event coreference and showing they can replace heavy external knowledge with competitive performance.

Strengths
- Conceptual simplicity with strong empirical payoff. The model is a clean, two-stage architecture: a CNN over the sentence with position embeddings and piecewise pooling to derive mention representations; a learned similarity layer (square nonlinearity) plus logistic classifier for pairwise linking. Despite its simplicity and lack of external semantic resources, it matches or exceeds prior systems on ACE. On ACE test, BLANC F1 is 73.31 vs. 70.43 (Liu et al., 2014), with competitive MUC and positive-link F1 and only slightly lower B-CUBED F1.
- Clear evidence that sentential features matter for event linking. The dataset analysis shows many positive pairs lack shared arguments (≈51%) and often have different triggers (≈47%), motivating context-aware representations. Ablations on ACE++ confirm that adding sentential features substantially boosts F1 over pairwise-only and pairwise+lexical setups, and all feature classes contribute.
- Resource-light and potentially more domain-portable. Avoiding WordNet/FrameNet/etc. removes a common bottleneck. The gains over a feature-rich random forest baseline that used 100+ semantic features bolster the claim that sentential neural features can substitute external semantics.
- Careful empirical comparisons and diagnostics. The paper discusses clustering strategies (transitive closure vs. BestLink) and shows BestLink hurts recall with this model; it quantifies the effect of more training data (ACE++ raises recall notably); provides naive baselines (all singletons, one cluster, same type) to calibrate difficulty; and includes a brief but useful error analysis (pronoun-like event anaphora, annotation ambiguity).
- Practical contributions. Implementation details, hyperparameter search strategy, and use of pretrained embeddings are described; the document splits for ACE are shared; ACE++ expands training substantially and gives a more robust picture than ACE alone.

Weaknesses
- Novelty is moderate. Using CNN-derived sentence representations for event-level tasks is a well-trodden path in relation/event extraction; the main novelty is applying it to event coreference. The architectural choices (word and position embeddings, piecewise pooling) are adapted from prior relation extraction work. The square activation in the similarity layer is a minor tweak rather than a conceptual leap.
- Evaluation realism is limited by reliance on gold annotations. The model uses gold event triggers, and pairwise features include event type agreement and argument overlap. It is unclear whether types and arguments are gold or predicted; if gold, this substantially simplifies the task and limits conclusions about end-to-end utility. Prior work compared under different preprocessing assumptions; although the paper discusses this, the current setup likely overestimates practical performance.
- Dataset and reproducibility questions. ACE is licensed; ACE++ combines ACE with 1351 web documents “annotated in an analogous fashion,” but it is unclear whether ACE++ is publicly available, how it was annotated, and by whom. Without a public release (or at least release of non-ACE additions), reproducing the key gains (notably recall improvements) may be difficult.
- Limited comparison to alternative neural architectures and global models. The paper does not compare against BiLSTM/transformer-style sentence encoders or attention mechanisms that became standard for sequence modeling, nor against global inference methods that model cluster-level decisions beyond transitive closure. Given the pairwise nature of the model, improved global consistency could matter.
- Missing details and statistical rigor. There is no significance testing for metric differences. Some architectural/hyperparameter choices (e.g., window sizes, filter counts) are said to be in a table but not all are visible here; more detail on class imbalance handling (thresholding, calibration, loss weighting) would help.

Questions to authors
- What supervision is assumed for event types and arguments at test time? Are event types gold or predicted? Are argument overlaps computed over gold arguments? Please clarify for both ACE and ACE++ experiments.
- Is ACE++ publicly released (at least the 1351 additional web documents and their annotations)? If not, can you release them or provide a plan? If release is impossible, can you provide a subset or weakly supervised alternative?
- How sensitive are results to the choice of sentence encoder? Did you try BiLSTM-based encoders or simple averaging baselines? A small comparison would contextualize the role of CNNs vs. “sentential features” more broadly.
- Why the square nonlinearity for similarity rather than absolute value, cosine, or bilinear forms? Can you provide an ablation with bilinear or cosine similarity and report metrics?
- Are thresholds tuned on validation or fixed at 0.5? Did you try calibrated decision thresholds or class-weighted loss to address the heavy negative imbalance?
- Can you report end-to-end results with predicted triggers/types (even if lower), to gauge robustness to upstream errors?
- For clustering, did you try simple union-find with score thresholds varying per pair type or a learned transitivity model? Any evidence that transitive closure produces over-merged clusters vs. BestLink on subsets?

Overall, this is a solid, resource-light application of sentential neural features to event coreference with convincing empirical gains and useful analyses. While originality is incremental, the demonstration that external semantic resources can be avoided without sacrificing accuracy is valuable, and the expanded dataset and diagnostics add weight. I am mildly positive, contingent on clarifying gold vs. predicted supervision and dataset availability.

Summary
This paper addresses within-document event coreference (event linking). It proposes a two-stage neural model: (a) a CNN-based encoder that builds a mention representation from the full sentence with position-aware, piecewise-pooled sentential features augmented by local trigger embeddings; (b) a pairwise scorer that combines two mention representations with handcrafted pairwise features (type compatibility, distance, modality/realis match, argument overlap) and computes a learned similarity (square activation) followed by logistic regression. Pairwise decisions are clustered by transitive closure. Using gold event triggers (and some gold attributes), the model achieves state-of-the-art or competitive results on ACE 2005 and a larger ACE++ dataset, without external semantic resources (e.g., WordNet/FrameNet). Ablations and error analysis are provided.

Main contributions (as I see them)
1) Demonstration that sentence-level CNN features can effectively support event coreference without external semantic resources, achieving competitive SOTA on ACE (notably BLANC F1).
2) A thorough empirical study: ablations isolating the effect of sentential vs. local vs. pairwise features; clustering strategy comparison; dataset statistics and qualitative error analysis.
3) Construction and use of a larger ACE++ corpus split (ACE + 1,351 similar web docs) to study data scaling effects.

Strengths
- Clear, well-motivated model design bridging relation extraction CNNs to event coreference. The use of position-aware, piecewise-pooled sentential features around triggers is sound and addresses observations about long-distance links, argument sparsity, and trigger lexical variation.
- Competitive results without external KBs. On ACE, the model outperforms strong prior work on BLANC, MUC, and positive-link F1 and is close on B-CUBED. This substantiates the claim that heavy external semantic resources are not strictly necessary for strong performance.
- Solid ablations and analyses. The paper disentangles the contribution of pairwise, local lexical, and sentential features; demonstrates that sentential features materially raise recall; and examines alternative clustering (BestLink vs. transitive closure) with an informative discussion about precision–recall trade-offs.
- Data-driven motivation and error analysis. Corpus statistics (argument sharing, trigger mismatch, sentence distance) motivate design decisions; the qualitative error analysis identifies difficult phenomena (event anaphora/pronouns) and annotation ambiguity, contextualizing the remaining error.
- Practicality and portability. Avoiding WordNet/FrameNet-like features reduces domain and resource dependencies; using pre-trained word embeddings is standard and widely available.

Weaknesses
- Heavy reliance on gold annotations weakens the claim of portability. The model uses gold event triggers (and appears to use gold event types, realis/modality, and argument overlap for pairwise features). This introduces a strong oracle signal and sidesteps crucial pipeline errors. No results with predicted triggers/types/arguments are reported, so end-to-end effectiveness and robustness are unknown.
- Limited methodological novelty. The CNN with position embeddings and piecewise max pooling is adapted from relation classification; the contribution is primarily an application to event coref plus a specific similarity function (square activation). The novelty is incremental relative to contemporary neural mention encoders.
- Clustering and global consistency are simplistic. Transitive closure of pairwise links can propagate local errors and does not enforce global constraints (e.g., type/realis consistency across clusters). BestLink is briefly compared, but no learning-to-cluster or global inference is attempted; no analysis of typical clustering failure modes is provided.
- Comparability and reproducibility concerns. ACE test partitions vary across prior work; preprocessing differs; the paper acknowledges this, but it complicates SOTA claims. ACE++ is not public (only ACE is), yet is used extensively for tuning and analysis; details on its provenance and availability are limited, reducing reproducibility. Statistical significance testing of improvements is missing.
- Ambiguity about pairwise features and supervision. It is unclear whether modality/realis and event types are gold or predicted; how arguments are obtained (gold vs. system) for “argument overlap”; and whether any external tools (SRL, NER, dependency parsers) are used during evaluation. These choices materially affect difficulty and generalization but are under-specified.
- Class imbalance and calibration. The pair generation strategy yields extremely imbalanced datasets (over 90% negative links), but the paper provides little on class weighting, negative sampling, threshold tuning (0.5 default), or calibration. This could affect both training dynamics and clustering.

Questions to Authors
- Which pairwise features rely on gold annotations at test time? Specifically: event type compatibility, modality/realis, and argument overlap—are these derived from gold labels or system predictions? If predicted, with which tools and accuracies?
- Can you report end-to-end results with predicted triggers (and, if applicable, predicted event types/arguments) to assess robustness? Even a small drop-in experiment on ACE would strengthen claims of applicability.
- How is “argument overlap” computed (exact match on entity IDs, role-sensitive, headword overlap)? If arguments are predicted, how do you align mentions?
- Did you try class weighting or negative downsampling? How sensitive are results to the decision threshold, and did you tune it on validation (rather than fixing 0.5)?
- Could you provide ablations for the similarity layer (square vs. ReLU/tanh vs. cosine) and for piecewise pooling vs. standard max pooling? You mention trying alternatives but do not report numbers.
- Will ACE++ or code/models be released? If not, can you provide more details about data collection and annotation process to help replication?

Additional comments and suggestions
- Consider a stronger clustering approach (e.g., learned mention-ranking with global constraints, or agglomerative clustering with learned linkage) to mitigate transitivity-induced over-merges.
- Reporting statistical significance and per-document variance would help substantiate SOTA claims.
- A finer-grained error analysis by distance bins, trigger (dis)agreement, and argument availability could highlight where sentential features help most.
- If modality/type/argument features are gold-only, please clarify this limitation prominently and temper the “no external features” claim (distinguish external KBs vs. gold supervision).

Summary and contributions
- This paper addresses within-document event coreference (event linking) and proposes a two-step neural model: (a) a CNN-based encoder that builds sentential representations of individual event mentions (with position embeddings and piecewise max pooling around the trigger), augmented with trigger-local lexical features; (b) a pairwise scorer that combines two mention representations with a small set of pairwise features (type agreement, distance, modality agreement, argument overlap) and a squared non-linearity similarity layer, followed by logistic regression. Clustering is done via transitive closure.
- Main contributions as I see them:
  1) Empirical finding: strong performance on ACE (comparable or superior to prior work on BLANC, MUC, positive links) without external semantic resources such as WordNet/FrameNet, suggesting sentential neural features can substitute for many hand-engineered semantic features for event coreference.
  2) Methodological contribution: applying CNN-based sentential features to event coreference and showing their additive value via ablations against pairwise and trigger-local features; and analyzing clustering strategies (transitive closure vs BestLink) and data scale (ACE vs ACE++).
  3) Error analysis identifying key failure modes (pronoun event references, annotation inconsistency), which is useful for future directions.

Strengths
- Sound, well-motivated architecture with clear inductive biases. Using position-aware CNNs with piecewise pooling is a reasonable way to encode mention-in-sentence context; the subsequent similarity layer (with polarity-invariant square activation) is simple and effective. The design matches data properties (many positive pairs lack shared arguments or identical triggers, and long-distance links are common), justifying sentential features.
- Strong empirical results with careful comparisons. On ACE, the system attains state-of-the-art or on-par performance on several metrics (notably BLANC and positive-link F1), despite avoiding external semantic KBs. The ACE++ experiments include informative ablations: pairwise-only is weakest; adding trigger-local helps; adding sentential features helps most. The data-scale experiment (ACE→ACE++) demonstrates improved recall, supporting the claim that neural sentential modeling benefits from more data.
- Thoughtful analysis of clustering and features. Contrary to prior claims, BestLink reduces recall too much for this model; the paper justifies choosing transitive closure. The feature ablation table makes a convincing case that all three feature classes contribute, with sentential features being especially impactful.
- Clarity and completeness of the experimental section. Metrics are appropriate (BLANC emphasized), baselines (all-singletons, one-instance, same-type) contextualize task difficulty, and the error analysis surfaces challenging phenomena (event anaphora via pronouns) and annotation variability, which temper expectations for ceiling performance.
- Practical significance. Avoiding external semantic resources substantially improves portability to new domains or languages where WordNet/FrameNet coverage is limited.

Weaknesses
- Fairness/realism of supervision and features is unclear. The pairwise features include “argument overlap” and “modality agreement.” Are arguments and modality gold-annotated or system-predicted? The comparison section notes other systems used SRL/tools “instead of gold-argument information,” implying this work might leverage gold arguments. If so, the evaluation is optimistic and not directly comparable to prior systems that rely on predicted arguments, undermining the central claim of resource-lean applicability. This needs to be clarified and, ideally, complemented with a setting using predicted arguments/modality.
- Limited novelty. Most architectural pieces are adapted from prior relation/event extraction CNNs (position embeddings, piecewise pooling) and standard pairwise features; the squared similarity is a minor twist. The paper’s primary novelty is in applying CNN-based sentential features to event coreference and showing they can replace external semantics. While valuable, methodological novelty is modest.
- Dataset concerns and reproducibility. The new ACE++ (ACE + 1351 web docs) appears not to be publicly available (while ACE is), and only document lists for splits are provided. Without releasing ACE++ annotations or code, it will be hard to reproduce the ablations and scaling results. The comparison on ACE also suffers from non-identical splits across papers; a standardized split or cross-validation would strengthen claims.
- Missing details that affect replicability. Key implementation choices are underspecified: exact encoding and dimensionality of pairwise features; how modality is detected; class imbalance handling (e.g., negative sampling ratio, loss reweighting); threshold selection (fixed at 0.5 vs tuned on dev); statistical significance tests; and whether hyperparameters were tuned on ACE++ and transferred to ACE or separately tuned.
- Scope of baselines. There is no comparison to a strong purely neural pairwise baseline (e.g., Siamese encoder of two mentions with only distance/type features, or a cosine similarity variant) beyond brief remarks. Given the claim about sentential features, a head-to-head with a trigger-only neural model would isolate the contribution more cleanly.

Questions to authors
- Do you use gold event arguments and gold modality labels for the pairwise features (argument overlap, modality agreement)? If yes, please report results with predicted arguments/modality to assess robustness and fairness against prior work.
- Are triggers gold in all experiments? If yes, can you provide an end-to-end setting with predicted triggers to gauge practical performance?
- How are modality and event type obtained? If predicted, what tools and accuracies? If gold, what label set and agreement?
- What is the exact class imbalance in training pairs, and how is it handled (negative subsampling, loss weighting)? What is the positive decision threshold, and is it tuned on dev?
- Will ACE++ (annotations) and code be released? If not, can you provide a standardized ACE split to enable direct comparison?

Overall assessment
The paper is clear, timely, and empirically solid in showing that CNN-based sentential features can deliver competitive event coreference without external semantic knowledge bases. However, the potential reliance on gold arguments/modality and non-standard splits weakens comparative claims. If the authors clarify supervision assumptions and provide results with predicted arguments (and ideally release code/splits), I would lean toward acceptance for its empirical contribution and practical significance despite modest novelty.

- Paper tackles within-document event coreference using a two-step neural model: CNN-based sentential encoder with piecewise max-pooling plus a pairwise classifier and transitive clustering.
- Contribution: Shows CNN sentential representations + lightweight pairwise features reach competitive/state-of-the-art event coreference performance on ACE without external semantic resources.
- Contribution: Provides empirical analysis of design choices — ablations of feature classes, clustering strategies, data scale (ACE vs ACE++), and baselines — using BLANC as primary metric.
- Contribution: Builds and uses an expanded ACE++ dataset and reports dataset-driven observations motivating sentential features (e.g., many pairs lack shared arguments or have trigger mismatches).
- Strength: Strong empirical results with minimal external resources, outperforming Liu et al. (2014) on BLANC and MUC F1 on ACE.
- Strength: Careful ablation and analysis that shows sentential features give substantial gains and highlights BestLink vs transitive-closure differences.
- Strength: Appropriate metric choice (BLANC) and useful dataset analysis (argument overlap, trigger mismatch, long-distance links) justifying model design.
- Strength: Error analysis provides qualitative insights into challenging phenomena (e.g., pronominal event references) and annotation ambiguities.
- Strength: Simple, reproducible architecture — piecewise CNN + basic pairwise features; square activation for distributed similarity is a neat lightweight idea.
- Weakness: Relies on gold annotations (triggers, event types, arguments), which inflates performance and limits end-to-end realism.
- Weakness: Limited novelty — architecture and components are incremental adaptations from prior relation/event extraction; no global or joint inference beyond thresholding + transitive closure.
- Weakness: ACE++ dataset provenance, annotation quality, alignment with ACE, inter-annotator agreement, and release status are unclear, weakening generality claims.
- Weakness: Comparability issues — different train/test splits and preprocessing vs prior work; lacking standard split results and broader metrics (e.g., CEAF/CoNLL average).
- Weakness: Clustering and thresholding choices under-specified (fixed 0.5 threshold, no calibration or learned clustering), limiting robustness of claims.
- Question: Do pairwise features use gold event types and gold argument spans/roles, and can you report performance with predicted triggers/types/arguments?
- Question: How was ACE++ annotated (annotators, guidelines, QA), will it be released, and what is inter-annotator agreement on coreference and event types?
- Question: How are multi-word triggers handled in piecewise pooling and local lexical features — is the trigger embedding just the head token?
- Question: Was the 0.5 decision threshold tuned on a validation set, and does tuning improve BLANC F1 or the precision–recall trade-off?
- Question: Can you report CEAF and CoNLL scores to facilitate comparison with broader coreference literature?
- Comment: Ablate/remove gold arguments/types or replace with predicted ones to quantify reliance on those features.
- Comment: Provide per-distance performance breakdown (e.g., links across >5 sentences) to show long-range linking strengths.
- Comment: Clarify minibatch construction — negative vs positive example counts and whether class balancing or loss weighting was applied.
- Comment: Release code and trained models to improve reproducibility.
- Comment: Improve figure clarity (CNN architecture depiction) and notation consistency for readability.

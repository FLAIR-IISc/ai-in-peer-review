- Paper proposes a two-stage event coreference model: CNN sentential encoder with piecewise pooling plus a pairwise scorer using simple features and transitive-closure clustering.  
- Claims: sentential CNN features give competitive, resource-light performance; provides ablations and clustering comparisons; introduces a larger ACE++ training set improving recall.

- Contribution 1: Shows sentential CNN features can match prior work without external lexical resources.  
- Contribution 2: Provides careful empirical study (feature ablations, clustering strategy comparison, error analysis).  
- Contribution 3: Uses an expanded ACE++ dataset and reports improved recall with more data.

- Strength: Practical, resource-light approach achieves state-of-the-art or comparable results without external semantic KBs.  
- Strength: Motivation grounded in data (argument sharing rare, many positives have different triggers, long-distance links common).  
- Strength: Solid empirical ablations show sentential features substantially boost precision and recall.  
- Strength: Sensible evaluation choices (BLANC + standard metrics) and transparent discussion of caveats.  
- Strength: Readable, technically clear model description and training details.

- Weakness: Several pairwise features appear to rely on gold labels/arguments, which may inflate performance and make comparisons uneven.  
- Weakness: Modeling novelty is limited—components (CNN encoder, pairwise classifier, transitive closure) are well-established.  
- Weakness: ACE++ dataset description lacks details on annotation process, quality control, IAA, and release plans, hindering reproducibility.  
- Weakness: Results use non-identical splits/preprocessing vs. baselines and lack statistical significance tests or variance reporting.  
- Weakness: Evaluated with gold triggers; robustness to predicted triggers/arguments is unknown.  
- Weakness: Limited generality—no results on other event coref corpora (e.g., ECB+); more analysis of long-distance performance needed.

- Question: Which pairwise features use gold supervision (event type, realis/modality, arguments); provide results with predicted labels or without these features.  
- Question: How was ACE++ annotated (annotators, guidelines, IAA) and will it be released for replication?  
- Question: Could tuning hyperparameters on ACE++ bias ACE results; report settings tuned only on ACE dev or robustness checks.  
- Question: How are position embeddings and piecewise pooling handled when multiple triggers are in one sentence—are indices re-centered per trigger?  
- Question: How stable are results across random seeds—report mean±std and significance tests versus strongest baselines.  

- Overall: Well-executed empirical application with clear benefits, but acceptance depends on addressing gold-supervision reliance, dataset transparency, and robustness/statistical concerns.

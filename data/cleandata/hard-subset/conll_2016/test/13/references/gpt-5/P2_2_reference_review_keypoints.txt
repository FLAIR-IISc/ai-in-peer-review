- Proposes a two-step neural model for within-document event coreference: CNN-based sentential encoder + pairwise classifier with simple features and transitive closure clustering.
- Achieves competitive or state-of-the-art results on ACE 2005 and ACE++ without external semantic resources; ablations show sentential features are crucial.
- Contribution: shows CNN-derived sentential features suffice for strong event coreference performance without WordNet/FrameNet.
- Contribution: provides an empirical study dissecting feature classes, clustering strategies, and data-size effects, highlighting gains from sentential features and more data.
- Contribution: offers qualitative error analysis identifying pronoun linking and annotation ambiguity as key challenges.
- Strength: Empirical effectiveness and portability by avoiding external semantic resources while matching prior systems on key metrics.
- Strength: Solid ablation studies and comparison of BestLink vs transitive closure, plus data-scaling analysis explaining recall improvements.
- Strength: Clear dataset analysis motivating sentence-level modeling given many coreferent pairs lack shared arguments and are distant.
- Strength: Thoughtful metric discussion, reporting BLANC alongside MUC/B-CUBED/positive links to handle class imbalance.
- Strength: Actionable error analysis pointing to discourse modeling and event-as-argument approaches for future work.
- Weakness: Ambiguity about preprocessing—unclear whether argument overlap and modality/realis use gold or predicted annotations, affecting fairness and resource-light claims.
- Weakness: Incremental methodological novelty—combines established components with main novelty in application and empirical finding.
- Weakness: Limited scope and reproducibility—within-document, English-only experiments; ACE++ not public; code/release and significance testing missing.
- Weakness: Sentence-local representations may miss cross-sentence/discourse signals critical for distant links and pronoun resolution.
- Weakness: Missing implementation details (pairwise feature definitions, hyperparameters, class imbalance handling, negative sampling), hindering reproducibility.
- Question: Do you use gold event arguments for argument-overlap; if not, what tool/accuracy provides them, and report results without that feature?
- Question: How are event type and modality/realis obtained—gold or predicted—and with what accuracy or tools?
- Question: Can you provide results on the same ACE split as prior key work or at least statistical significance testing?
- Question: How is class imbalance handled during training (reweighting, sampling, threshold tuning)?
- Question: Will ACE++ and code be released; if not, detail ACE++ composition, sizes, genres, and annotation quality control.
- Overall: Well-executed empirical paper with modest methodological novelty, strong ablations and analysis; lean weak accept pending clarifications on gold/predicted annotations and matched/end-to-end evaluations.

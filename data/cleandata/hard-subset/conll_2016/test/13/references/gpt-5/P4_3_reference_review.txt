Summary and claimed contributions
The paper addresses within-document event coreference (event linking). It proposes a two-stage neural approach: (a) a CNN-based encoder that builds event-mention representations from sentential context plus local trigger embeddings; (b) a pairwise scorer that combines the two mention representations with simple pairwise features (type/realis agreement, distance, argument overlap) and a “distributed similarity” layer (square nonlinearity), followed by logistic regression. Pairwise links are clustered via transitive closure. The system avoids external semantic resources (e.g., WordNet/FrameNet) and is evaluated on ACE 2005 and a larger ACE++ dataset. The paper reports state-of-the-art or on-par performance with prior work, ablations on feature classes and clustering, and an error analysis.

Contributions (as assessed)
1) Demonstration that sentential features learned by CNNs can replace external semantic resources for event linking while achieving state-of-the-art performance on ACE. This is the primary empirical contribution.
2) A modular model and ablation study quantifying the impact of sentential, trigger-local, and pairwise features, and a comparison of clustering strategies (transitive closure vs BestLink) that informs modeling choices.
3) A larger, ACE-like dataset (ACE++) used to study data scaling effects, though it appears not publicly released.

Strengths
1) Strong empirical results without external semantic resources. The model attains competitive or better performance than Liu et al. (2014) on multiple metrics (e.g., BLANC F1 73.31 vs 70.43; MUC and positive-links F1 also higher), indicating that CNN-based sentential features suffice to match state-of-the-art systems that rely on extensive semantic feature sets. This materially reduces dependency on costly, domain-limited resources.

2) Comprehensive ablations that isolate the value of feature classes. Table 5 shows clear, additive contributions: pairwise-only is weakest; adding trigger-local lexical features improves substantially; adding sentential features yields the best F1 (76.90 on ACE++). This supports the central technical choice to learn sentential representations for coreference.

3) Analysis of clustering strategies and data scaling. The experiments in Table 4 show that transitive closure outperforms BestLink under this model (higher recall and F1), contrasting prior findings and suggesting the classifier already filters low-confidence links. Scaling from ACE to ACE++ predominantly improves recall (75.16→83.21), indicating the approach benefits from more data.

4) Sensible model design with practical details. The architecture is straightforward, leveraging piecewise pooling around the trigger to handle multiple mentions per sentence, pre-trained word embeddings, and Adam optimization. The square activation for similarity is intuitive for polarity-invariant similarity and empirically validated against alternatives.

5) Useful qualitative error analysis. The inspection of false positives/negatives highlights difficult cases (e.g., pronoun-like anaphoric references to events) and annotation boundary issues, contextualizing remaining error sources and motivating future directions.

Weaknesses
1) Comparability and realism of evaluation. The main comparison uses gold event mentions (and possibly gold arguments/types for pairwise features); the paper notes preprocessing differences among systems and dataset splits. It is unclear whether event types and arguments used in pairwise features are gold or predicted; if gold, results reflect an upper bound and are not directly comparable to pipelines relying on automatic upstream components. No significance testing is reported.

2) Limited novelty relative to prior neural work. The core novelty is the application of CNN-based sentential features (established in relation/event extraction) to event coreference. Architectural elements (piecewise pooling, word/position embeddings) are borrowed from prior work; the “square” similarity layer is a minor variation. The contribution is primarily empirical rather than methodological.

3) Reproducibility and resource release. ACE++ appears to be an internal extension (“1351 more web documents annotated in an analogous fashion”) and not publicly available; only ACE split lists are mentioned as shared. The absence of dataset and code release limits reproducibility and impact.

4) Lack of explicit symmetry and global inference. The pairwise scorer appears to concatenate two mention representations; order-invariance is not guaranteed, and symmetry is not enforced beyond the square nonlinearity in a later layer. Clustering via transitive closure is simple and may propagate local errors; more global or cluster-aware training/inference (e.g., structured losses, ILP/greedy merges with global features) is not explored.

5) Class imbalance and thresholding. The dataset is highly imbalanced (>90% negative links), yet the training procedure does not describe class weighting, sampling strategies, or calibrated threshold selection; a fixed 0.5 decision threshold is used. This could affect calibration and fairness of comparisons.

Questions to authors
- Do the pairwise features use gold event types, realis, and argument roles/fillers on ACE and ACE++? If not, which automatic tools are used, and what are their accuracies? Please clarify the exact supervision level to contextualize performance.
- Is the model symmetric with respect to swapping the two mentions? If not, did you consider symmetric inputs (e.g., elementwise operations) or data augmentation with swapped pairs?
- How is the 0.5 link threshold chosen? Did you tune it on validation data or calibrate scores? Any results with threshold optimization?
- How is class imbalance handled during training (e.g., loss weighting, negative subsampling)? If none, what is the positive:negative ratio per minibatch?
- Will you release ACE++ and code/models? If ACE++ cannot be released, can you provide a detailed annotation protocol and statistics to facilitate replication?
- Did you evaluate with predicted triggers/types/arguments to assess robustness in a realistic pipeline?

Overall assessment
Technically sound and clearly presented. The main value lies in showing that CNN-based sentential features can eliminate reliance on external semantic resources for event linking while maintaining state-of-the-art performance, supported by careful ablations and analyses. Novelty is modest, evaluation comparability has caveats, and reproducibility is limited without public release of ACE++. With clarifications and, ideally, resource/code release or additional experiments under predicted upstream inputs, this would be a solid contribution.

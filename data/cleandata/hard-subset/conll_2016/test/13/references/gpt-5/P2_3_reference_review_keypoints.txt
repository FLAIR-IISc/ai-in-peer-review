- Proposes a two-step within-document event coreference model: CNN-based sentential mention encoder (position embeddings, piecewise pooling, trigger-local features) plus a pairwise scorer with pairwise features and a squared non-linearity, clustering via transitive closure.
- Empirical finding: CNN-based sentential features achieve strong ACE performance without external semantic KBs (comparable/superior on BLANC, MUC, positive-link F1).
- Methodological contribution: adapts CNN sentential features to event coreference and shows their additive value via ablations and clustering strategy analysis.
- Error analysis contribution: identifies major failure modes (pronoun event references, annotation inconsistency) for future work.
- Strength: Architecture is well-motivated with appropriate inductive biases (position-aware CNNs, piecewise pooling, polarity-invariant similarity).
- Strength: Strong empirical results with careful comparisons and informative ACE→ACE++ data-scale experiments showing improved recall.
- Strength: Thoughtful analysis of clustering strategies (transitive closure vs BestLink) and convincing feature ablations highlighting sentential features’ impact.
- Strength: Clear, complete experimental setup with appropriate metrics, baselines, and useful error analysis.
- Strength: Practical significance in avoiding external semantic resources, improving portability to new domains/languages.
- Weakness: Unclear whether pairwise features (argument overlap, modality) use gold annotations or predicted values, which affects fairness and realism of comparisons.
- Weakness: Limited novelty—main architectural components are adapted from prior CNN relation/event extraction work; novelty is mainly application and empirical demonstration.
- Weakness: Reproducibility concerns: ACE++ annotations may not be public and splits differ from other papers, making comparisons and replication difficult.
- Weakness: Missing implementation details that affect replicability (pairwise feature encodings, modality detection, class imbalance handling, threshold selection, statistical tests, hyperparameter tuning).
- Weakness: Scope of baselines is limited — lacks comparison to a strong purely neural pairwise baseline (e.g., Siamese encoder or trigger-only neural model).
- Question: Are event arguments and modality labels gold or predicted; if gold, please report results with predicted values to assess robustness?
- Question: Are triggers gold in experiments; if so, can you provide end-to-end results with predicted triggers?
- Question: How are modality and event type obtained; if predicted, what tools and accuracies were used; if gold, what label set and agreement?
- Question: What is the class imbalance in training pairs and how is it handled (negative sampling, loss weighting); is the positive threshold tuned on dev?
- Question: Will ACE++ annotations and code be released; if not, can you provide a standardized ACE split to enable direct comparison?
- Overall assessment: Clear, timely, and empirically solid work showing CNN sentential features can replace external KBs, but clarification on supervision and release of resources/code are needed to strengthen comparative claims.

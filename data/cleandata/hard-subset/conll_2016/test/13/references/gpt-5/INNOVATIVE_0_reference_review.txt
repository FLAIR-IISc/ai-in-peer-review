Summary and claimed contributions
- The paper proposes an event coreference model that builds event-mention representations with a CNN over the full sentence (with position embeddings and piecewise pooling) and combines them with lightweight pairwise features to score coreference, followed by simple clustering. The key claim is that sentential features eliminate the need for external semantic resources while achieving state-of-the-art performance.
- It reports competitive results on ACE and a larger ACE++ collection, includes ablations showing the value of sentential features, and analyzes clustering strategies (transitive closure vs BestLink) and errors.
- It argues, via dataset analysis, that many positive pairs lack shared arguments/triggers and are far apart, motivating sentence-level modeling.

Main contributions as I see them (strongest first)
1) Conceptual shift: demonstrating that sentential neural features can replace heavy external semantic resources for event linking while preserving SOTA-level performance.
2) A practical, simple architecture for event mention representation and pairwise scoring that performs well across datasets, with useful ablation and clustering analyses.
3) Construction and use of a larger ACE++ dataset for training/evaluation and empirical observations about distance, trigger/argument mismatch.

Strengths
- Novel direction with practical impact: The paper convincingly shows that CNN-based sentential features can obviate reliance on WordNet/FrameNet/VerbOcean/SRL-derived semantics in event linking. This is a meaningful departure from prior work and lowers domain-dependence and engineering overhead.
- Solid empirical results: On ACE, the model matches or exceeds prior SOTA on several coreference metrics, and on ACE++ gains strong recall. The use of BLANC is appropriate given class imbalance, and the inclusion of multiple metrics is transparent.
- Informative analysis: The ACE/ACE++ statistics (argument/trigger mismatch, long-distance links) motivate the approach. The ablation demonstrates that sentential features materially improve both precision and recall beyond pairwise/lexical features. The comparison of transitive closure vs BestLink gives a clear takeaway about precision–recall trade-offs for this model.
- Clarity and completeness of the model: The architecture is described clearly (embedding + position embeddings, piecewise max pooling, trigger-local concatenation, similarity layer with squared activation). Implementation details (Adam, pre-trained word2vec, hyperparameter search) are provided at a reasonable level.
- Error analysis: The qualitative inspection of FPs/FNs identifies pronoun-based anaphora and annotation inconsistencies as major failure sources, which can guide future work.

Weaknesses
- Methodological novelty is incremental: The model is a straightforward adaptation of CNN sentence encoders from relation/event extraction to coreference, with standard pairwise features and simple clustering. The squared activation in the similarity layer is a minor architectural tweak. While the direction is valuable, the technical contribution is modest.
- Fairness and realism of comparisons: The model relies on gold event mentions (and appears to use argument overlap, event type, modality/realis as features). It is unclear whether these attributes are gold or system-derived; if gold, comparisons to systems using predicted arguments/SRL are confounded, and realism for deployment is limited. Clarifying exactly which gold annotations are used is critical.
- ACE++ dataset details and availability: The paper introduces ACE++ but provides limited information on annotation protocol, quality (IAA), and availability. If ACE++ is not released, it weakens the empirical contribution and reproducibility; if it is released, more details are needed.
- Limited exploration beyond pairwise decisions: The approach does not model discourse beyond the sentence encoder; long-distance phenomena (e.g., pronominal event anaphora) remain unaddressed. No joint modeling of triggers/arguments/coreference or document-level/global consistency beyond transitive closure is attempted, which may cap performance on hard cases.
- Experimental caveats: Different ACE splits and preprocessing across compared systems reduce the strength of SOTA claims. No statistical significance testing is reported. Handling of severe class imbalance during training (sampling/weighting) is not described.

Questions to authors
- Which features are gold vs predicted? Specifically, are event types, modality/realis, and argument mentions/roles used in pairwise features gold annotations? If any are predicted, with which tools and accuracies?
- How is class imbalance handled during training? Any negative sampling, class weighting, or calibrated thresholds beyond the default 0.5?
- ACE++: What is the source and annotation process? Who annotated, with what guidelines, and what inter-annotator agreement? Will ACE++ (or at least splits and annotations for coreference and arguments) be released?
- Reproducibility: Will you release code, trained models, and exact hyperparameters (Table 2 values) to facilitate replication?
- Similarity layer: What was the chosen dimensionality dsim, and how sensitive are results to this and to the choice of nonlinearity (e.g., square vs ReLU/tanh or cosine)?
- Do you have results with predicted triggers/arguments to assess end-to-end performance? If not, can you report a preliminary estimate or discuss expected degradation?

Overall recommendation
I lean toward acceptance. The conceptual contribution—showing that sentential CNN features can supplant external semantic resources for event coreference while maintaining SOTA—offers a useful and portable direction for the community. Despite modest methodological novelty and some evaluation caveats, the empirical evidence, ablations, and analyses are strong enough to merit publication, provided the authors clarify the gold/predicted feature usage and, ideally, release ACE++ or code.

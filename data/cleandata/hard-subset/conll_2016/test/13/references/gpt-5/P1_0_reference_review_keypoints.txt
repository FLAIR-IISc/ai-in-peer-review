1) Proposes a two-stage, resource-light within-document event coreference model using sentence-level CNNs with piecewise max-pooling and trigger-local lexical features.  
2) Scores pairwise coreference by combining mention representations with lightweight features (type compatibility, discourse distance, modality/realis, argument overlap) and clusters via transitive closure.  
3) Avoids external semantic resources (WordNet/FrameNet) and reports state-of-the-art on ACE 2005 and a larger in-house ACE++ dataset.  
4) Main contribution: demonstrating that sentential CNN features can match or exceed prior systems that rely on rich external semantics.  
5) Main contribution: thorough empirical ablations showing sentential features drive gains and analysis comparing transitive closure vs. BestLink clustering.  
6) Main contribution: dataset and error analysis motivating architectural choices (many positive pairs lack shared arguments, different triggers, and long-distance links).  
7) Strength: convincingly motivates sentence-level context with empirical stats and uses position embeddings and piecewise pooling to capture it.  
8) Strength: strong, resource-lean empirical performance (BLANC, MUC, positive-links), and improved recall on ACE++ with more data.  
9) Strength: systematic ablations attribute improvements to sentential features and justify clustering choice; pairwise-only baseline highlights limits of surface cues.  
10) Strength: transparent discussion of comparability caveats (splits, preprocessing) and justification for using BLANC in imbalanced settings.  
11) Strength: useful qualitative error analysis identifying event anaphora and borderline annotations as major error sources.  
12) Weakness: comparability weakened by non-identical ACE splits and omission of CEAF-e/m and CoNLL F1 metrics.  
13) Weakness: unclear whether pairwise features (event type, modality, arguments) use gold annotations or predicted labels—needs clarification and evaluation with predicted attributes.  
14) Weakness: reliance on gold triggers (and possibly other gold attributes) reduces realism; a pipeline evaluation or sensitivity analysis to upstream errors is needed.  
15) Weakness: ACE++ is not public and hyperparameters tuned on it reduce reproducibility; code, models, and exact splits should be released.  
16) Weakness: model scope limited to sentence-local encoders and pairwise + transitive closure, ignoring cross-sentential discourse and global cluster-level inference.  
17) Question: Are event type, modality/realis, and arguments used as gold annotations or predicted during pairwise scoring? Please report results with predicted attributes.  
18) Question: Why were CEAF and CoNLL F1 omitted, and can those metrics be provided on the ACE split used?  
19) Question: How sensitive is performance to encoder choice (CNN vs BiLSTM or CNN+attention) and random-seed variance?  
20) Question: What is the average mentions per document, O(n^2) pair count at inference, any pruning strategies, and runtime?  
21) Question: Will you release code, trained models, and precise train/dev/test document lists (including ACE++)?  
22) Additional comment: report statistical significance for key comparisons (e.g., vs. Liu et al., 2014).  
23) Additional comment: provide quantitative ablation for the square activation vs ReLU/tanh/cosine.  
24) Additional comment: consider adding lightweight document-level context (mention priors, section embeddings) to help long-distance/pronominal links.  
25) Additional comment: expand error analysis by quantifying pronominal and ambiguous annotation cases over a larger sample.  
26) Overall: solid, well-executed paper with competitive, resource-lean results; clarifying gold vs. predicted attributes and adding standard metrics would strengthen empirical claims.

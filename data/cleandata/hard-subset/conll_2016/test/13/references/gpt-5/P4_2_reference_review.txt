Summary and claimed contributions
The paper addresses within-document event coreference (event linking) and proposes a two-step neural approach that avoids external semantic resources. First, a CNN encodes each event mention using sentence-level context with word and position embeddings, piecewise max-pooling around the trigger, and additional trigger-local embeddings. Second, a pairwise scorer combines the two learned mention representations with standard pairwise features (type compatibility, discourse distance, modality/realis match, argument overlap) and maps them through a squared nonlinearity and logistic regression to produce a coreference score. Pairwise decisions are converted to clusters via transitive closure. The model is trained with Adam and evaluated on ACE 2005 and an extended ACE++ dataset. Ablations examine the contribution of sentential vs. lexical vs. pairwise features, and an error analysis is provided.

Contributions (as assessed):
1) Empirical demonstration that sentential features learned by a CNN, coupled with lightweight pairwise features, achieve state-of-the-art or competitive event coreference performance on ACE without relying on external semantic resources.
2) Comprehensive experimental analysis, including ablations isolating feature classes, comparison of clustering strategies (transitive closure vs. BestLink), and dataset size effects (ACE vs. ACE++).
3) Dataset-level diagnostics and error analysis highlighting data properties (e.g., frequent argument sparsity, long-distance links) and common failure modes.

Strengths
- Strong empirical performance without external resources. On ACEtest, the model achieves BLANC F1 73.31 vs. 70.43 for Liu et al. (2014), and competitive scores on other metrics. This supports the claim that sentential features can replace costly semantic resources while maintaining or improving accuracy.
- Clear ablation evidence for sentential features. On ACE++, the full model (pairwise + trigger-local + sentential) attains BLANC P/R/F1 73.22/83.21/76.90. Removing sentential features reduces F1 substantially (e.g., pairwise only: 56.69 F1). This isolates the added value of the CNN-based sentence encoding beyond local lexical cues.
- Analysis of clustering strategies with data-driven conclusions. Contrary to prior work, BestLink reduces recall markedly (e.g., ACE++ F1 drops from 76.90 to 71.09), indicating the classifier already filters low-confidence links effectively. Reporting both precision and recall clarifies the trade-off.
- Useful dataset diagnostics and error analysis. Quantifying properties (e.g., 50.97% of positive pairs share no arguments; 47.29% have different triggers; >55% span >5 sentences) justifies modeling choices. The qualitative analysis surfaces challenging cases (e.g., pronominal anaphora for events) and annotation inconsistencies that bound achievable performance.
- Methodological clarity. The model is well specified with equations, pseudo-code for pair generation and clustering, training details (word2vec initialization, optimization, hyperparameter search), and a motivation for the squared similarity nonlinearity.

Weaknesses
- Reliance on gold annotations limits practical impact and comparability. The system uses gold event-mention triggers (and appears to use gold types/arguments for pairwise features like argument overlap and realis/modality). This sidesteps error propagation and constrains conclusions about end-to-end utility. Comparisons to prior work are further complicated by differing preprocessing and document splits.
- Reproducibility and resource availability. Only one of the two datasets is public. ACE++ (an important driver of gains and analyses) is not clearly released; code is not provided; full hyperparameters are partially referenced (Table 2 not included here). This impedes independent verification.
- Limited statistical rigor and robustness reporting. Results appear to be from single runs with a fixed 0.5 decision threshold (as per the clustering pseudo-code). There are no confidence intervals, significance tests, or variance over random seeds. Threshold tuning on validation for BLANC optimization is not discussed.
- Modest architectural novelty and limited exploration of alternatives. The CNN with position embeddings and piecewise pooling mirrors established relation classification setups; the pairwise scorer is a single-layer transformation. While the application to event coreference is useful, the modeling advances are incremental. Document-level context beyond the sentence of each mention is not modeled, despite many cross-sentence links.
- Class imbalance and training details under-specified. Pairing each anaphor with all antecedents yields a high negative-to-positive ratio, yet there is no discussion of loss weighting, negative sampling, or calibration. Regularization choices (dropout/L2) and early stopping criteria beyond “best on validation” could be clearer.

Questions to authors
- Precisely which annotations are gold at training and test time: triggers, types, arguments, realis/modality? Are any pairwise features derived from system predictions?
- How is the decision threshold chosen? Is 0.5 fixed, or tuned on validation for BLANC/MUC/B3? If fixed, what is the effect of tuning?
- Will ACE++ (documents, annotations, and splits) and code be released to enable reproducibility?
- How sensitive are results to random initialization? Please report mean/stdev over multiple seeds and run-time/computational cost.
- How is class imbalance handled during training (e.g., sampling, loss weighting)? What is the positive/negative ratio in minibatches?
- Are there results with predicted event mentions (end-to-end) to assess downstream applicability? If not, can you estimate expected degradation?

Overall, the paper is clear and technically sound, with solid empirical support for the usefulness of CNN-derived sentential features in event linking, but practical impact and novelty would be strengthened by reproducibility artifacts, robustness analyses, and evaluation under predicted mentions.

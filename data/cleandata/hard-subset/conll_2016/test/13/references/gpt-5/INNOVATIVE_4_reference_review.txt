Summary and claimed contributions
- The paper tackles within-document event coreference (event linking) and proposes a two-stage, neural mention-pair model: (a) a CNN-based encoder that builds sentential representations centered on an event trigger with relative-position embeddings and piecewise max-pooling; (b) a pairwise scorer that combines the learned mention representations with a small set of pairwise features and uses a squared nonlinearity to compute a multi-dimensional similarity before logistic classification and transitive-closure clustering.
- The key claims are: (1) state-of-the-art performance on ACE2005 without external semantic resources; (2) effectiveness of sentential features for event coreference, showing strong gains relative to purely local or pairwise features; (3) empirical analyses of clustering strategy, data scale, and feature ablations; and (4) construction of an expanded ACE++ dataset.

As I see it, the main contributions are:
1) Applying CNN-based sentential representations to event coreference, achieving SOTA without external semantic resources, demonstrating portability.
2) Clear empirical evidence via ablations that sentential features materially improve event linking, plus analysis of clustering strategy and data scaling.
3) A larger dataset (ACE++), contingent on availability, that could facilitate further research.

Strengths
- Novel direction with practical impact: Using sentence-level neural representations for event coreference is a timely, relatively bold move for this task. Crucially, the approach avoids WordNet/FrameNet/etc., yet reaches competitive or better BLANC/MUC/positive-link scores than prior systems that depend on heavy external semantics.
- Strong empirical study and ablations: The paper evaluates on ACE and a larger ACE++, reports multiple coref metrics (including BLANC, which is well-motivated here), and offers informative ablations (pairwise vs local lexical vs sentential features). The finding that sentential features drive large gains is convincing.
- Simple, modular architecture: The design (CNN-based mention encoder + lightweight pairwise features + simple clustering) is easy to port and scale. The piecewise pooling around triggers and relative-position embeddings are sensible choices for multi-mention sentences and trigger-centric modeling.
- Analysis beyond headline numbers: The paper investigates BestLink vs transitive closure, shows the effect of dataset size (ACE++ yields significantly higher recall), and includes an error analysis identifying pronoun-based links and annotation inconsistencies as major pain points. This improves interpretability of results and clarifies remaining challenges.
- Potential community value of ACE++: If ACE++ is released with clear documentation and licensing, it would be a meaningful resource for event linking research.

Weaknesses
- Reliance on gold annotations likely inflates performance: The model uses gold event triggers and appears to use gold types/modality/arguments for pairwise features (e.g., “type compatibility,” “realis match,” “argument overlap”). That makes the task substantially easier and weakens the claim of domain portability. A more realistic setting with predicted triggers/types/arguments is needed to assess robustness.
- Dataset creation details and availability unclear: ACE++ is described as 1351 additional “web documents annotated in an analogous fashion,” but details on annotation procedure, guidelines, inter-annotator agreement, license, and actual release status are not provided. Without this, reproducibility and community impact are uncertain.
- Methodological novelty is moderate: The architecture is a fairly standard mention-pair model with a CNN encoder; the squared similarity activation is a minor tweak. While the application to event coreference and the external-resource-free angle are valuable, the technical novelty per se is limited.
- Limited document-level modeling: Each mention is represented via its sentence; the linker is pairwise with transitive closure. Given that over half of positive links are >5 sentences apart and many errors involve anaphora, richer document-level context or joint/global inference (e.g., entity-event interactions, discourse cues) could be important. This limitation likely caps performance on harder cases.
- Reproducibility details: Hyperparameter table is referenced but not fully shown here; code availability is not mentioned; TensorFlow v0.6 suggests an early implementation. More explicit training details (e.g., regularization, early stopping, compute/time) would help others replicate.

Questions to authors
- Precisely which gold annotations are used at training and test time? Are event types, modality/realis, and arguments gold or predicted? If any are predicted, from which systems and with what accuracy?
- Can you report results with predicted triggers/types/arguments to quantify realistic end-to-end performance? Even a single setting with a reasonable upstream extractor would strengthen the claims.
- ACE++: Please detail the annotation process (guidelines, annotator expertise, inter-annotator agreement, adjudication), licensing constraints, and a firm commitment on releasing the data/splits. If release is not possible, can you release derived features or pair lists?
- Provide a breakdown of performance by sentence distance between mentions and by event type. How does the model fare for long-range links and for types with sparse arguments?
- The squared similarity activation: could you include a controlled comparison vs cosine, bilinear, and MLP with ReLU/tanh, with statistical significance? What is dsim’s effect on performance?
- Hyperparameters/regularization: Please include the final chosen values (dw, dp, dc, de, dsim, filter widths, pooling strategy, dropout/L2, learning rates/betas), early stopping criteria, and training time/resources. Will you release code?

Overall assessment
This paper presents a clean, resource-light approach that advances event coreference by leveraging sentential CNN features and demonstrates competitive performance with solid analysis. Despite moderate methodological novelty and concerns about gold inputs and dataset availability, I find the direction promising and impactful. Recommendation: weak accept, contingent on clarifying the use of gold annotations and the status of ACE++ release, and ideally adding a result with predicted inputs.

Summary and claimed contributions
The paper addresses within-document event coreference (event linking). It proposes a two-step model: (a) build an event-mention representation using a CNN over the full sentence with position embeddings plus trigger-local lexical features; (b) score mention pairs using these representations together with hand-crafted pairwise features (type compatibility, distance, position, realis, argument overlap), followed by clustering via transitive closure. The authors report competitive results on ACE 2005 and additional experiments on an expanded dataset (ACE++), including ablations and an error analysis.

As I see it, the main contributions are:
1) An empirical demonstration that sentence-level CNN features can help event coreference, yielding competitive performance on ACE without external semantic resources.
2) An analysis of feature contributions and simple clustering strategies (transitive closure vs. BestLink) with ablations.
3) A qualitative error analysis highlighting hard cases (e.g., pronominal references).

Strengths
- Empirical competitiveness on ACE with a relatively simple architecture: The model achieves F1s comparable to prior work while avoiding heavy reliance on WordNet/FrameNet/VerbOcean, suggesting portability to domains lacking such resources.
- Clear ablations and design analysis: The feature ablations (pairwise vs. trigger-local vs. sentential) and the comparison of clustering strategies provide useful insight into what drives performance and when BestLink hurts recall.
- Sensible modeling choices grounded in data properties: The authors motivate the need for sentential context by quantifying the frequency of cross-sentence distance, missing arguments, and differing triggers among coreferent pairs. This strengthens the case for learning sentence-level representations.
- Error analysis is informative: Investigates 100 FPs and 100 FNs and identifies recurring phenomena (e.g., pronominal reference), offering concrete directions for future work.

Weaknesses
- Use of gold information likely inflates performance and harms comparability: Pairwise features include event type agreement, realis, and “argument overlap.” The paper implies gold triggers are used (as in prior work), but also suggests other systems did not use gold arguments while this work likely does. If gold event types/arguments are used for features, the evaluation becomes substantially easier and not representative of real end-to-end use. Results without gold arguments/types (and ideally without gold triggers) are not reported, so it is unclear how the method performs in realistic settings or relative to strong baselines under identical conditions.
- Comparability and evaluation choices are non-standard: Different ACE splits and preprocessing are used; the paper acknowledges this but still claims “state-of-the-art.” The standard CoNLL coreference score (average of MUC/B^3/CEAF) is not reported; CEAF is missing. No statistical significance testing is provided. Given small margins and split differences, it is difficult to substantiate SOTA claims.
- Limited methodological novelty: CNN-based sentential representations for IE were established prior work; adapting them to event coreference with a mention-pair scorer plus simple clustering is incremental. The “square” nonlinearity in the similarity layer is a minor tweak; justification and broader comparisons (e.g., bilinear, biaffine, cosine with learned temperature, or learned distance metrics) are limited.
- ACE++ dataset is under-specified and seemingly not released: The paper adds 1,351 web documents “annotated in an analogous fashion,” but provides no details on annotation process, inter-annotator agreement, quality control, or availability. This weakens the evidential value of results on ACE++ and hinders reproducibility.
- Missing end-to-end results and stronger baselines: The work evaluates in a favorable setting (gold elements). Absent are experiments with system-predicted triggers, types, and arguments (or at least without gold arguments/types) to test robustness. Baselines are either naive or rely on prior papers with incomparable settings. A strong in-paper baseline using only non-neural pairwise features (lexical, distance, type) under identical conditions would isolate the added value of the CNN representation.

Questions to authors (for rebuttal)
1) Do you use gold event arguments and gold event types for computing pairwise features (argument overlap, type compatibility, realis)? If yes, please report results without gold arguments/types and discuss the impact.
2) Can you provide results using predicted triggers, types, and arguments (end-to-end), or at least with non-gold arguments/types, to assess practical utility?
3) Why is CEAF omitted, and can you report the CoNLL average (MUC/B^3/CEAF) and significance tests on ACE? Also, can you evaluate on a commonly used ACE split to improve comparability?
4) Please detail ACE++: annotation procedure, guidelines, annotator expertise, inter-annotator agreement, and whether the data/splits will be released. If not, can you temper claims based on ACE++?
5) Beyond the “square” nonlinearity, did you try alternative learned similarity functions (e.g., bilinear/biaffine, cosine with learned scaling, energy-based models)? Any empirical comparisons?
6) For the pairwise features, are syntactic features (e.g., dependency paths) considered? If not, could you comment on whether adding standard syntactic signals changes the picture relative to sentential CNN features?

Overall, this is a careful empirical paper with a clean model and useful analyses. However, the reliance on gold labels for critical pairwise features, the lack of end-to-end or non-gold evaluations, and non-standard evaluation choices significantly limit the strength of the claims. The methodological novelty is incremental. I encourage the authors to add non-gold evaluations, standard metrics/splits, significance testing, and fuller dataset details (or release) to render the evidence sufficiently rigorous.

Summary and claimed contributions
This paper addresses within-document event coreference (“event linking”) and proposes a two-step neural model: (a) a CNN-based encoder that builds a mention representation from the full sentence containing the event trigger, augmented with position embeddings and local trigger-context embeddings, and (b) a pairwise scorer that combines two mention representations with a small set of hand-crafted pairwise features (type/modality agreement, distance, argument overlap), followed by a simple clustering via transitive closure. The core claim is that sentential features learned by CNNs can reduce reliance on external semantic resources while achieving competitive or state-of-the-art results on ACE 2005; the authors also evaluate on an expanded dataset, ACE++.

As I see them, the main contributions are:
1) Empirical: A competitive event coreference system that achieves strong performance on ACE without external lexical-semantic resources, supported by thorough ablations showing the value of sentence-level neural features.
2) Methodological/engineering: A simple, effective architecture for event mention encoding and pairwise scoring, including a polarity-invariant “square” activation for similarity and a practical discussion of clustering strategies.
3) Analysis: Useful diagnostics on feature contributions, dataset size, clustering choices, and an error analysis highlighting pronoun-driven anaphora and annotation ambiguity.

Strengths
- Solid empirical results without external semantic resources. The model matches or exceeds prior work (e.g., Liu et al., 2014) on BLANC and some other metrics on ACE, despite not using WordNet/FrameNet or extensive external knowledge. This is an informative result for portability to new domains.
- Careful ablations and design analysis. The paper systematically examines (i) feature classes (pairwise vs. trigger-local vs. sentential), (ii) clustering choices (BestLink vs. transitive closure), and (iii) data size (ACE vs. ACE++). The ablations convincingly show that sentential features substantially improve over pairwise-only and that more data boosts recall.
- Sensible modeling choices for event mention encoding. Using position-aware CNNs with piecewise pooling around the trigger is well-motivated and effective for anchoring the event in the sentence without requiring syntactic or external semantic features.
- Thoughtful metric discussion and error analysis. The emphasis on BLANC given class imbalance is appropriate, and the error analysis surfaces genuine challenges (event anaphora/pronouns, annotation inconsistency). The observation that BestLink harms recall in their setup is also informative for practitioners.
- Practicality and clarity. The architecture is straightforward, training is with standard components (Adam, pre-trained word2vec), and the pair generation strategy is clear.

Weaknesses
- Use of gold annotations undermines realism and comparability. The system appears to use gold event triggers and (at least partially) gold event types and arguments (argument overlap feature). The comparison to prior systems is thus hard to interpret: competitors differ in what gold information they use, and using gold types/arguments can materially inflate coreference performance. A more realistic setting with predicted triggers/types/arguments (or at least a consistent protocol across baselines) is needed.
- Limited novelty relative to prior neural sentence encoders. The core modeling ideas (CNN with position embeddings, piecewise pooling, using learned sentence representations) have been established in relation/event extraction. The contribution is largely in applying these to event coreference; architectural novelties (e.g., square activation for similarity) are minor and insufficiently analyzed (no quantitative comparison details provided).
- Reproducibility and dataset concerns. ACE++ is not clearly described regarding annotation process/quality and appears non-public; only ACE split lists are released. Without code and full hyperparameter details, reproducing the results (especially the ablations) will be difficult. The paper mentions extensive hyperparameter search but omits the final settings table in this draft.
- Limited use of document-level context given long-distance links. The model encodes only the sentence of each mention; cross-sentential discourse and document-level context (beyond distance features) are not modeled, despite the observation that many links are >5 sentences apart. This likely contributes to the reported pronoun/anaphora errors.
- Evaluation caveats and missing analyses. The ACE split mismatch across papers and non-standard preprocessing reduce the strength of SOTA claims. No statistical significance tests are reported. Calibration/threshold choice and class-imbalance handling (e.g., loss weighting) are not discussed. The impact of each pairwise feature (e.g., type agreement vs. argument overlap) is not isolated.

Questions to authors
- What gold vs. predicted signals are used? Specifically: are event types (coarse/fine), modality, and arguments gold during training/evaluation? If so, can you provide results with predicted triggers/types/arguments, or at least with a consistent protocol matching Liu et al. (2014)?
- What is the availability and annotation protocol for ACE++? Is it publicly releasable, and were guidelines/quality control aligned with ACE?
- Can you report a feature ablation within the pairwise set (e.g., remove argument overlap, remove type agreement) to quantify dependence on gold information?
- You state the square non-linearity outperforms alternatives. Please provide comparative numbers (e.g., ReLU/tanh/cosine) and discuss why.
- How is the decision threshold chosen (fixed 0.5 vs. tuned on dev)? Any class-imbalance mitigation during training?

Recommendation
Borderline reject. The paper is well-executed and empirically strong, and it makes a useful point about replacing external semantic resources with learned sentential features. However, the reliance on gold annotations (triggers/types/arguments) and the non-comparable evaluation weaken the main claims, and the methodological novelty is modest. I encourage the authors to strengthen the work by:
- Evaluating with predicted triggers/types/arguments or aligning preprocessing with prior work for a fair comparison;
- Releasing code and, if possible, ACE++ (or at least detailed annotation and sampling protocol);
- Adding deeper analyses (pairwise feature ablations, significance, calibration), and exploring document-level encoders or joint modeling for pronoun/anaphora cases.

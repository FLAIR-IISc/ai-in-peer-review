- Two-stage model: CNN-based sentence encoder with position-aware piecewise pooling plus a pairwise scorer using handcrafted features and learned similarity, clustered by transitive closure.  
- Uses gold event triggers (and apparently some gold attributes) and reports competitive or SOTA results on ACE 2005 and ACE++.  
- Main contribution: shows sentence-level CNN features can support event coreference without external semantic KBs.  
- Main contribution: provides a thorough empirical study with ablations isolating sentential vs. local vs. pairwise features and clustering comparisons.  
- Main contribution: constructs and uses a larger ACE++ corpus split to study scaling effects.  
- Strength: model design is clear and well-motivated, adapting relation-extraction CNN techniques to event coreference.  
- Strength: achieves strong results without WordNet/FrameNet, improving BLANC, MUC, and positive-link F1 and close on B-CUBED.  
- Strength: solid ablation studies show sentential features materially increase recall and disentangle feature contributions.  
- Strength: data-driven motivation and qualitative error analysis identify hard phenomena like event anaphora and annotation ambiguity.  
- Strength: avoiding external KBs improves portability and practicality, relying only on standard pre-trained embeddings.  
- Weakness: heavy reliance on gold annotations (triggers/types/realis/arguments) undermines claims of portability and end-to-end robustness.  
- Weakness: methodological novelty is limited; encoder and pooling are adapted from prior relation classification work.  
- Weakness: clustering uses simple transitive closure (and BestLink comparison) with no global inference or learning-to-cluster, risking propagated errors.  
- Weakness: comparability and reproducibility concerns due to varying ACE splits, private ACE++ usage, and limited details on preprocessing and release.  
- Weakness: ambiguity about which pairwise features are gold vs. predicted and what external tools (SRL/NER/parsers) are used.  
- Weakness: class imbalance from pair generation (>90% negatives) is under-addressed—little on class weighting, negative sampling, or threshold tuning.  
- Requested clarification: specify which pairwise features rely on gold labels at test time (type, modality/realis, argument overlap).  
- Requested experiment: report end-to-end results with predicted triggers/types/arguments to assess real-world robustness.  
- Requested clarification: define how argument overlap is computed and how predicted arguments are aligned.  
- Requested detail: report whether class weighting or negative downsampling was used, and whether thresholds were tuned on validation.  
- Requested ablations: quantify alternatives for the similarity layer (square vs. ReLU/tanh/cosine) and pooling (piecewise vs. standard max).  
- Requested release: indicate whether ACE++ and code/models will be published or provide data provenance/details for replication.  
- Suggestion: consider stronger clustering methods (learned mention-ranking, agglomerative clustering with learned linkage) to avoid transitivity over-merges.  
- Suggestion: add statistical significance testing and per-document variance to support SOTA claims.  
- Suggestion: provide finer-grained error analysis by distance bins, trigger agreement, and argument availability to clarify where sentential features help most.  
- Clarification note: if modality/type/argument features are gold-only, explicitly state this limitation and separate “no external KBs” from gold supervision.

Summary
The paper tackles within-document event coreference (“event linking”) and proposes a two-stage neural approach: (a) learn sentence-level representations of individual event mentions using a CNN with word and position embeddings plus trigger-local lexical features; (b) score event-mention pairs with a small neural network combining the two mention representations and a set of simple pairwise features (type compatibility, distance, modality agreement, argument overlap). The model avoids external semantic resources (e.g., WordNet/FrameNet), trains with gold triggers, and clusters via transitive closure on positive links. Experiments on ACE 2005 and an expanded ACE++ dataset show competitive or better results than prior work, ablations indicate the utility of sentential features, and an error analysis highlights challenges such as pronominal anaphora.

Claimed contributions (as I see them)
1) A resource-lean neural event coreference model that achieves state-of-the-art performance on ACE without external semantic KBs, supported by ablations showing the impact of sentential features.  
2) An expanded dataset (ACE++) and empirical analyses (data properties; clustering strategy comparison; baselines; error categories) that provide insight into the task and model behavior.  
3) A simple, effective pairing strategy and a finding that transitive closure outperforms BestLink in this setup.

Strengths
- Clear motivation and empirical grounding. The authors quantify key data properties (low argument overlap, frequent trigger mismatch, long-distance links), which justifies sentence-level representations and avoiding reliance on argument matching or locality. This improves the paper’s problem framing and helps explain the model’s gains.

- Competitive performance without external semantic features. Achieving strong results with only pretrained word embeddings, sentence CNNs, and simple pairwise features is valuable for portability. The comparisons on ACE show parity or improvements over feature-heavy prior work; ablations on ACE++ convincingly attribute gains to sentential features.

- Thorough experimental analysis. The study covers: (i) different clustering schemes (BestLink vs transitive closure), (ii) feature ablations (pairwise vs trigger-local vs sentential), (iii) naive baselines, and (iv) an informative error analysis. These analyses make the empirical story more trustworthy and actionable.

- Reasonable engineering choices for mention representation. The piecewise pooling around triggers allows multiple mentions per sentence; combining sentence-level context with trigger-local embeddings is simple and effective, and the “square” similarity nonlinearity is a neat, empirically supported choice.

- Attention to evaluation metrics. The discussion of BLANC vs MUC/B^3 and the rationale for emphasizing BLANC under heavy class imbalance reflects care in evaluation design.

Weaknesses
- Limited novelty of the core modeling idea. Using CNN-derived sentential features for IE has been established; the paper extends this to event coreference with fairly standard pairwise scoring and simple features. The main novelty is empirical (showing it works resource-lean) rather than methodological. A stronger modeling contribution (e.g., document-level/global inference, joint learning with trigger/argument prediction, or richer discourse/context modeling) would help.

- Evaluation setup reduces external validity. The model relies on gold event triggers (and uses event type and modality agreement), and the pairwise argument-overlap feature may use gold arguments; this undercuts end-to-end practicality. The comparison set is heterogeneous in splits and preprocessing, and no statistical significance tests are reported. Results on predicted triggers/arguments would materially strengthen claims.

- Dataset contribution underspecified and reproducibility concerns. ACE++ is not clearly described: who annotated it, with what guidelines and quality control, inter-annotator agreement, licensing, and availability. Only document lists for ACE splits are shared; code and hyperparameters (Table 2) are referenced but not fully detailed here. Without public access to ACE++ or code, replicability is limited.

- Limited baselines and metrics. While naive baselines are included, stronger non-neural baselines (e.g., a logistic regression or random forest using the same pairwise and lexical features), CEAF/CoNLL scores, and significance tests are missing. This makes it harder to assess the incremental benefit of sentential features over simpler alternatives.

- Modeling and training details needing clarification. The O(n^2) pairing strategy can be costly; there is no discussion of runtime or negative subsampling to address label imbalance. Training for 2000 epochs with early stopping by validation is plausible but could overfit; regularization choices are only briefly mentioned. Multiword triggers, cross-sentence contextualization beyond the mention’s own sentence, and class-imbalance handling in loss are not fully elaborated.

Questions to Authors
- Do you use gold event types, modality, and arguments for pairwise features on ACE/ACE++? If not, which components are predicted and how? If yes, can you report performance with predicted triggers/types/arguments to assess end-to-end robustness?

- What is the status of ACE++? Who annotated the additional 1351 web documents, using which guidelines, with what IAA, and will the data (or at least annotations) be made available? If not, can you at least release detailed statistics and scripts to reconstruct features?

- How do you handle class imbalance during training given the heavy skew toward negative pairs? Any negative sampling or loss reweighting? What is the positive:negative ratio per batch?

- Can you report CEAF and CoNLL average, and conduct significance tests versus the strongest baseline(s)? Also, provide results with the same train/test split as a key prior system if possible.

- What is the behavior with multi-token triggers and sentences containing multiple event mentions—does piecewise pooling use the trigger span boundaries or head?

- Any runtime and memory figures for training/inference, and scaling behavior with document length/number of mentions?

Suggestions for improvement
- Add an end-to-end evaluation with predicted triggers/types/arguments, and quantify the performance drop; alternatively, train a joint or pipelined model to mitigate error propagation.

- Strengthen baselines: a non-neural model using the same features, and a bag-of-words sentence representation, to isolate the value of CNN features.

- Report CEAF/CoNLL metrics and significance testing; where possible, align splits with prior work or provide cross-validation.

- Clarify and, if feasible, release ACE++ (or a subset) and code; include full hyperparameters. Expand dataset documentation with annotation protocol and IAA.

- Explore document-level context beyond single sentences (e.g., mention-level encoders augmented with discourse cues, coref-aware attention, or global ILP/Clustering) and investigate strategies for pronominal event anaphora.

Overall recommendation: Weak reject. The empirical contribution is solid and the resource-lean angle is attractive, but the methodological novelty is limited and the current evaluation (gold inputs, dataset availability) constrains impact. Addressing the above concerns would likely move this to accept.

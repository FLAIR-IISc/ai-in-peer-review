Summary and main contributions
- Proposes a two-step within-document event coreference model: (a) a CNN with word and position embeddings plus piecewise max pooling to build per-mention sentential representations augmented with local lexical features; (b) a pairwise scoring network combining two mention representations with hand-crafted pairwise features, using a “square” activation for a multi-dimensional similarity, followed by logistic regression and transitive-closure clustering.
- Empirical study on ACE 2005 and an expanded ACE++ dataset, with ablations showing the effect of sentential vs. local vs. pairwise features, and a comparison of BestLink vs. transitive closure clustering.
- Data analyses and a small error analysis; claim of state-of-the-art performance without external semantic resources.

Strengths
- Clear motivation for sentential features in event linking: the authors quantify that many coreferential pairs lack argument overlap and share neither trigger words nor proximity; this motivates modeling beyond trigger-local and argument features.
- The architecture is simple, modular, and resource-lean: CNN-based encoding and standard pairwise features avoid dependence on WordNet/FrameNet, which increases portability across domains.
- Useful ablations and diagnostics: the feature ablation table shows that sentential features substantially improve F1; the dataset-size comparison demonstrates recall gains with more training data; the clustering comparison shows that their model’s classifier is strong enough that BestLink hurts recall.
- Error analysis, while small, highlights difficult phenomena (event anaphora by pronouns) and annotation inconsistencies, which is valuable context for interpreting absolute scores.
- The paper attempts fair metric choice by emphasizing BLANC given severe class imbalance; it also reports other metrics for completeness.

Weaknesses
- Fairness and clarity of experimental setup are insufficient. It is unclear which gold annotations are used. The pairwise features include type compatibility and argument overlap; are event types and arguments gold or predicted? The text suggests gold triggers and possibly gold arguments/types, whereas some baselines do not use gold arguments. Without a matched setting, the SOTA claim is weak. Please explicitly state for each system (yours and prior work): gold/predicted triggers, types, arguments, and which tools were used.
- Reproducibility concerns. ACE is restricted; ACE++ appears non-public. While the authors publish the ACE valid/test doc lists, it is unclear if the same is done for ACE++. There is no code link, no release of hyperparameters (Table 2 is referenced but not present in this version), no random seed reporting, no significance testing, and no details on class balancing, batch size, or early stopping strategy beyond “best on valid.” The 2000-epoch training with Adam and no clear regularization specification risks overfitting.
- Limited novelty. Using CNN-based sentence encoders with position embeddings and piecewise pooling is well established in relation/event extraction. Pairwise features and pairwise classification with transitive-closure clustering are standard. The main new technical choice seems to be the “square” activation for similarity, but this is ad hoc and insufficiently justified (no ablation numbers vs. cosine/ReLU/absolute-value, no analysis of stability/gradients).
- Methodological gaps and under-specified details. 
  - “Bagged” distance features are not defined (binning scheme?).
  - “Agreement in event modality/realis” is used, but ACE 2005’s attributes vs. TAC KBP realis are not clarified; what annotation source is used?
  - Handling of multiple mentions per sentence is only briefly described; how are position embeddings reset per mention? 
  - Class imbalance is severe (>90% negative links), yet there is no discussion of sampling/weighting strategies.
  - Threshold selection for the logistic score is not described (0.5 by default? tuned on valid?).
- Clustering is simplistic. Only BestLink and transitive closure are tried; there is no comparison to more principled global inference (e.g., agglomerative clustering with learned linkage, correlation clustering/ILP, or neural mention-ranking with cluster-level features). Given the known error propagation of pairwise decisions with transitive closure, stronger decoding would strengthen the contribution.
- Evaluation scope is narrow. Only within-document linking with gold mentions is considered. There is no end-to-end evaluation (predicted triggers/types), no cross-domain testing, and no comparison to stronger modern encoders (e.g., BiLSTM/Transformer sentence encoders), which limits the generality of the conclusions.
- Numerous presentation/formatting issues that hinder clarity:
  - Missing Table 2 (hyperparameters). Figure 2 is referenced but not shown. Figures/tables have broken captions and typography in this version (e.g., “∗ (Precision / Recall / F1 score) in % This paper ...”).
  - Notation inconsistencies (v (·) w vs. v(·)p), excessive spaces before “%,” typos (“enrichtment”), informal abbreviations (“wrt.”), and awkward footnote placement (e.g., the Araki and Mitamura example).
  - The TensorFlow version (0.6) is very old; either update or justify.

Questions to authors
- Do you use gold event types and/or gold arguments for pairwise features? If not, how are they obtained? Please provide a table enumerating which gold/predicted annotations are used by your system and by each baseline, and redo the key comparison in matched settings.
- What are the exact hyperparameters (dw, dp, dc, de, dsim, window size n, dropout/l2 settings, batch size, learning rate and Adam betas, early stopping, class weights)? Please include the missing table and seeds.
- How is the decision threshold chosen? Was it tuned on validation? Are your results robust to threshold variation? Any statistical significance tests across 5+ runs?
- Can you release ACE++ splits and features (or at least doc IDs and preprocessing scripts)? If not, can you reproduce results on a fully public dataset?
- Please report an end-to-end setting with predicted triggers/types (even if performance drops) to demonstrate applicability.
- Provide quantitative ablations for the “square” similarity vs. cosine/MLP with ReLU/absolute value, and for alternative clustering (e.g., average-linkage agglomerative).

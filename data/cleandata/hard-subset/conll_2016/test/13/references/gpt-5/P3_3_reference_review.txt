Summary and main contributions
- The paper tackles within-document event coreference (event linking). It proposes a two-step neural approach: (a) represent each event mention via a CNN over the full sentence with position embeddings and piecewise pooling around the trigger, augmented with trigger-local embeddings; (b) score mention pairs using the learned mention representations plus lightweight pairwise features (type compatibility, distance, realis, argument overlap), with a simple squared activation similarity layer and a logistic classifier, followed by transitive-closure clustering.
- The authors claim comparable or better performance to prior state of the art on ACE 2005 without relying on external semantic resources (WordNet/FrameNet), and provide ablations showing the contribution of sentential features. They also introduce a larger ACE++ dataset (ACE + 1351 web docs) and analyze errors.

Strengths
- Empirically strong and resource-lean: On ACE, the system achieves state-of-the-art or better BLANC/MUC/positive-link F1 without external semantic resources, relying instead on sentence-level neural features and a small set of generic pairwise features. This is a meaningful result given the portability issues of resource-heavy approaches.
- Clear modeling of sentential features for coref: Adapting piecewise max pooling around the trigger to event mentions is a sensible design to capture mention-local and sentential cues even when arguments are sparse or missing. The observed data properties (many positive pairs with no shared arguments or different triggers) justify this representation choice.
- Useful ablations and design analysis: The ablation table cleanly isolates the contributions of pairwise, trigger-local, and sentential features, supporting the claim that sentence-level representations materially improve recall and overall F1. The BestLink vs transitive-closure analysis is informative and contradicts prior claims in a plausible way (pairwise classifier already filters low-confidence links).
- Thoughtful error analysis: The qualitative discussion highlights difficult phenomena (event anaphora via pronominal “it/this,” annotation ambiguity) that current models struggle with, providing credible directions for future work.
- Experimental details and reasonable metrics: The use of BLANC is appropriate given the heavy class imbalance; the paper reports additional coref metrics for completeness and discusses differences in splits and preprocessing across prior work.

Weaknesses
- Heavy reliance on gold annotations undermines practical relevance: The approach uses gold event triggers (and likely gold event types and argument spans for pairwise features such as type compatibility and argument overlap). This makes the setup much easier than realistic pipelines and complicates fair comparison to some prior work that does not assume gold arguments. At minimum, report performance with predicted triggers/types/arguments (e.g., ACE-style upstream systems) to assess end-to-end utility.
- Claims about domain portability are not substantiated: While the method avoids WordNet/FrameNet, it still relies on gold labels and on news/web data similar to ACE. There is no cross-domain evaluation or transfer setting (e.g., train on ACE, test on another domain) to support the claim that external-resource-free features yield better portability. Pretrained word2vec is also an external resource (albeit general-domain).
- ACE++ dataset issues: The new ACE++ resource appears to be non-public, with little detail on the annotation process, quality control, inter-annotator agreement, event schemas, or compatibility with ACE. This limits reproducibility and the community’s ability to build on the results. It’s also used for hyperparameter search and major conclusions (e.g., recall gains), but cannot be verified.
- Limited document-level modeling despite long-distance links: The model uses sentence-level CNN representations for each mention and pairwise features like distance, but no document-level context encoding. Given that over half of positive links are >5 sentences apart, richer document/contextual representations (e.g., discourse structure, document RNNs, or global inference) seem warranted. The current clustering is a simple transitive closure over thresholded pairwise scores, with no global consistency modeling.
- Evaluation and analysis gaps: No statistical significance tests are reported. The thresholding and calibration of pairwise scores are not detailed (fixed 0.5?), and class imbalance handling (e.g., loss weighting) is not discussed. The “square” activation choice is only weakly motivated; ablations against ReLU/abs/cosine are mentioned but not quantified. Computational aspects (pairwise O(n^2) growth, negative sampling) are not addressed.

Questions to authors
- Precisely which gold annotations are assumed? Are event types and arguments gold at inference time, or only triggers? How are realis and argument overlap computed? Please report results with predicted triggers/types/arguments (or at least a subset) to assess robustness.
- How is the decision threshold set for pairwise links? Is 0.5 fixed, or tuned on validation per metric? Were any calibration techniques used?
- How is class imbalance handled during training (loss weighting, sampling)? What is the ratio of positive/negative pairs per minibatch?
- ACE++: Please describe annotation guidelines, event schema alignment with ACE, annotator training, inter-annotator agreement, and whether you plan to release the data. If not, can you release derived features or pairwise labels to enable replication?
- Did you try broader context windows (multi-sentence) or document-level encoders? Given the high proportion of long-distance links, how do you expect performance to change with simple context extensions (e.g., concatenating adjacent sentences)?
- Can you quantify the gains from the square activation vs. alternatives, and provide intuition beyond polarity invariance? Does the similarity layer learn interpretable facets?

Additional comments
- The paper is generally well written. Please clarify whether hyperparameters/implementation and code will be released, and include statistical tests for key comparisons.

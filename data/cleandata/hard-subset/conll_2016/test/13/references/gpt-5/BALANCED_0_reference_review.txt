Summary
The paper tackles within-document event coreference (“event linking”). It proposes a two-step neural model: (a) a sentence-level CNN encodes an event mention by combining sentential features (piecewise max-pooling around the trigger) with trigger-local lexical features; (b) a pairwise scorer combines the two mention encodings with standard pairwise features (type compatibility, distance, modality, argument overlap) via a learned “square” similarity and logistic regression. Pairwise scores are clustered by transitive closure. Experiments on ACE 2005 and a larger ACE++ extension show competitive or state-of-the-art performance without using external semantic resources (e.g., WordNet/FrameNet). Ablations assess feature classes, clustering strategies, and data size; an error analysis highlights pronoun-related cases.

Main contributions (as I see them)
1) Empirical demonstration that sentence-level CNN representations substantially help event coreference, yielding competitive SOTA on ACE without external semantic resources, supported by ablations across feature classes and clustering strategies.
2) A practical architecture for event mention encoding (piecewise pooling wrt. trigger, trigger-local features) and a simple but effective pairwise similarity based on squaring activations.
3) A larger evaluation with ACE++ and analysis of dataset properties (argument sparsity, trigger mismatch, long-distance links).

Strengths
- Strong empirical results without external semantic resources: On ACE, the model attains competitive or superior F1 against prior work across BLANC, MUC, and positive-links, while avoiding domain-specific resources. This supports the claim that sentential features can substitute for external semantics in event linking.
- Thorough ablations and analyses: The paper isolates the contribution of sentential vs trigger-local vs pairwise features, showing clear gains when including sentential features. It also compares clustering strategies (transitive closure vs BestLink) and quantifies the effect of additional training data (ACE++), which improves recall notably. The preliminary corpus analysis motivates modeling choices (e.g., long-distance links, missing arguments).
- Clear, well-motivated model design: The piecewise pooling around triggers and integration of local and sentence-level features are sensible given multiple mentions per sentence. The “square” activation for similarity is motivated and empirically preferred (though more details would help; see questions).
- Useful error analysis: Manual inspection identifies common hard cases (e.g., pronominal mentions of events), and annotation inconsistencies, which contextualize the attainable performance and suggest future directions.

Weaknesses
- Reliance on gold annotations weakens realism and comparability: The pairwise features include type compatibility, modality, and argument overlap. It appears triggers (and likely types and arguments) are gold at test time. This makes the setting much easier and not reflective of end-to-end use; it also complicates comparisons because some prior work used predicted arguments/SRL. The paper should clearly specify which labels are gold vs predicted and provide results under more realistic conditions (predicted triggers/types and no gold arguments), or at least a controlled comparison to prior systems under matched assumptions.
- Limited novelty in core modeling: Using CNN sentence encoders for mention representation and standard pairwise features is a natural extension of prior neural IE work. The pairwise “square” similarity is minor. There is no joint inference with entities or global clustering objective, and no discourse modeling beyond distance. The contribution is primarily empirical rather than methodological.
- ACE++ dataset clarity and availability: The paper introduces ACE++ with 1351 additional web documents “annotated in an analogous fashion,” but provides few details (annotation process, quality control, inter-annotator agreement, licensing) and it is unclear whether it will be released. Without public availability, reproducibility and impact are limited; performance gains on ACE++ are less actionable for the community.
- Missing stronger baselines and comparisons: Beyond naive baselines, it would be informative to compare against (i) a non-neural classifier using only available intrinsic and discourse features (no external semantics), (ii) simpler neural encoders (e.g., average/attention over embeddings, BiLSTM), or (iii) cosine similarity over pretrained sentence encoders. These would help quantify the incremental value of the proposed architecture choices.
- Evaluation caveats and reporting: Splits differ across compared systems, and BLANC can be influenced by singleton prevalence; adding CEAF_e and the CoNLL score would give a broader view. Class imbalance handling is not discussed; 2000 epochs with Adam invites overfitting concerns. Some ablation claims (e.g., superiority of square activation) are stated without numeric comparisons.

Questions to Authors (for rebuttal)
- Which supervision is gold at test time? Specifically: triggers (span and type), event types, modality, and arguments used for overlap—are these gold or predicted? If any are gold, can you provide results with predicted triggers/types and without gold arguments to assess robustness?
- ACE++: Who annotated the additional documents, using which guidelines? What is inter-annotator agreement? Will ACE++ (or at least splits and annotations) be released? If not, can you add experiments on a fully public benchmark with fixed splits?
- Class imbalance: How did you handle the skew toward negative links during training (class weighting, downsampling)? What is the positive:negative ratio in training batches?
- Similarity function: Can you report numbers comparing square vs ReLU/tanh/abs and cosine? Is d_sim critical? Any sensitivity analysis?
- Resource assumptions: Are event types used as features gold? If predicted, which classifier and accuracy? If gold, can you quantify performance drop when types are predicted?
- Implementation details: What were the final hyperparameters (embedding sizes, filter widths), regularization (dropout/L2), and early stopping criteria? Any measures to avoid overfitting across 2000 epochs?

Additional comments and suggestions
- Consider reporting CEAF_e and the CoNLL average to align with coreference evaluation practice, and add statistical significance where feasible.
- If argument overlap uses gold arguments, include a setting that removes argument overlap to better isolate the contribution of sentential features in realistic conditions.
- Given the prominence of pronominal event mentions in errors, incorporating a light-weight anaphora or event nominal detection signal (e.g., coref-aware features or contextualized encoders) could be fruitful.
- Releasing code and trained models would improve reproducibility.

Overall, the paper presents a solid empirical study showing that sentence-level neural features can drive competitive event coreference without external lexical resources. However, clarity about gold vs predicted annotations, stronger baselines, and dataset availability are needed to meet a high bar.

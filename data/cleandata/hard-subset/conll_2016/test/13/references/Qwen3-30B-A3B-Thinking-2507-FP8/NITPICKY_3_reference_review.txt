

# Review for "Event Linking with Sentential Features from Convolutional Neural Networks"

## Summary of the Paper

This paper proposes a neural network approach for event coreference resolution that uses sentential features generated by convolutional neural networks. The authors argue that previous approaches rely on external semantic resources (like WordNet and FrameNet) with limited cross-domain availability, and propose a model that uses local and sentential features instead. The model consists of two parts: 1) a CNN-based representation for individual event mentions, and 2) a pairwise comparison model that uses the generated representations plus standard pairwise features to make coreference decisions. The authors evaluate their model on ACE 2005 and an extended ACE++ dataset, reporting state-of-the-art performance without using external semantic resources.

## Main Contributions (as I see them)

1. A novel neural network architecture for event coreference resolution that uses sentential features from CNNs without relying on external semantic resources
2. An empirical analysis of feature contributions to event linking performance
3. A detailed error analysis identifying challenges in pronoun resolution and annotation boundary cases

## Strengths

1. **Novel application of sentential features to event coreference resolution**: The paper makes a valid point that previous approaches heavily rely on external semantic resources with limited domain coverage, and demonstrates that sentential features from neural networks can achieve competitive results without these resources. The authors correctly identify that 50.97% of coreferential event-mention pairs share no arguments, justifying the need for sentence-level context.

2. **Comprehensive feature ablation study**: The authors provide a thorough analysis of the impact of different feature classes (Table 5), showing that all components contribute positively to performance. The finding that using only pairwise features and sentential features (but not trigger-local features) leads to higher precision but lower recall is insightful and well-supported by their results.

3. **Detailed error analysis**: The error analysis (Section 5.3) goes beyond typical NLP papers by examining both false positives and false negatives, and identifying specific challenges (like pronoun resolution) that are difficult even for human annotators. The observation that 17 false negatives involved cases where annotators made different decisions (22 false positives had similar issues) provides valuable insight into the task's inherent ambiguity.

## Weaknesses

1. **Inconsistent and unprofessional formatting throughout the paper**: The paper contains numerous formatting inconsistencies that undermine its professionalism. Terms like "eventmentions" (without space), "event-mentions" (with hyphen), "ACE++" vs "ACE++", "sentence-level" vs "sentence level", and "vsent" vs "vsent" are used inconsistently. The paper should have been meticulously proofread to ensure consistent terminology and formatting. For example, in Figure 3, the code uses "Cd" but the description refers to "Cd" (with subscript d), creating confusion.

2. **Lack of statistical significance testing**: The paper claims state-of-the-art performance but provides no statistical significance tests to validate that the improvements over baselines (like Liu et al., 2014) are statistically meaningful. This is a critical omission for NLP papers, as small performance differences may not be significant.

3. **Inconsistent data split methodology**: The paper states they split datasets 9:1 into development and test, then split development 9:1 into training and validation. This results in an 81:9:10 split, which is non-standard and potentially problematic for model evaluation. The paper should have used a standard 80:10:10 or 70:15:15 split to ensure reproducibility and comparability with other work.

4. **Insufficient hyperparameter details**: The paper mentions they used "random sampling from the hyperparameter space (180 points) and line search" but fails to provide the specific hyperparameter ranges or the final values used. For example, they mention "dw, dp, dc, de, dsim" as architectural parameters but don't specify their final values (Table 2 is empty). This makes the work difficult to reproduce.

5. **Inconsistent and unclear technical descriptions**: Several key technical details are unclear or inconsistent. For example:
   - Equation (1) defines "vic = relu(wc · vi:i+n−1t + bc)" but doesn't clarify what "vic" represents
   - Equation (2) uses "vj,km = max (v i c)" with inconsistent notation
   - Equation (4) uses "square(Wsimvsent+lex+pairw + bsim)" but doesn't explain what "square" means (element-wise squaring? square of the vector?)
   - The paper inconsistently uses "vsent" vs "vsent" and "dsim" vs "dsim" for variable names

## Questions to Authors

1. The paper claims to use pre-trained embeddings from "Mikolov et al., 2013" but doesn't specify which embeddings (e.g., word2vec, GloVe) or their dimensionality. Could you clarify this?

2. In Table 5, the performance with "X X ACE++" (row 4) shows higher precision than the full model (row 3). Could you explain why this counter-intuitive result occurred, and whether this might indicate an issue with the model's training process?

3. The paper states that "55.42% [coreferential event mentions] are more than five sentences apart," which suggests that locality-based heuristics would not work well. However, the model uses a convolutional approach that inherently captures local context. Could you clarify how the model handles events that are more than five sentences apart?

4. The paper mentions that "the ACE++ dataset encompasses these documents and additionally contains 1351 more web documents annotated in an analogous fashion," but doesn't provide details on how these additional documents were annotated or whether they were annotated by the same team as the ACE 2005 corpus. Could you provide more details on the annotation process for ACE++?

## Overall Recommendation

**Reject**

While the paper presents a reasonable approach to event coreference resolution and makes a valid argument about reducing reliance on external semantic resources, the numerous formatting inconsistencies, lack of statistical significance testing, inconsistent data split methodology, and insufficient technical details significantly undermine the paper's quality and reproducibility. As a perfectionist reviewer, I cannot recommend acceptance of a paper with such fundamental issues in presentation and detail. The minor methodological flaws, while individually small, collectively indicate a lack of attention to detail that is unacceptable for a top-tier NLP conference like CoNLL.

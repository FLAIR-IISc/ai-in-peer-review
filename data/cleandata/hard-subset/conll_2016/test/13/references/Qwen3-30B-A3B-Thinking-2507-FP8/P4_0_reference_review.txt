

# Review of "Event Linking with Sentential Features from Convolutional Neural Networks"

## Summary and Contributions

This paper proposes a neural network-based approach for event linking (event coreference resolution) that generates sentential features using convolutional neural networks (CNNs) without relying on external semantic resources like WordNet or FrameNet. The authors present a two-step model: first, a CNN-based representation of individual event mentions using their full sentence context; second, a coreference decision module that combines these representations with pairwise features. The system achieves state-of-the-art performance on two datasets (ACE 2005 and a new ACE++ dataset) without requiring external semantic features.

The main contributions of this work, as objectively assessed, are:

1. A demonstration that sentential features from neural networks can achieve state-of-the-art performance in event linking without external semantic resources, addressing a key limitation of previous approaches.
2. A clear empirical validation of the importance of sentential features for event linking, with ablation studies showing that removing these features substantially degrades performance.
3. The creation and analysis of a larger dataset (ACE++) that demonstrates the model's ability to generalize with more training data.

## Strengths

1. **Clear demonstration of the necessity for sentential features**: The paper provides strong empirical evidence that sentential features are crucial for good event linking performance. Table 5 shows a clear performance progression (from 57.45/68.16/56.69 to 73.22/83.21/76.90 BLANC P/R/F1) as features are added to the model, with the full model (including sentential features) achieving the best results. The authors support this with dataset analysis showing that 50.97% of coreferential pairs share no arguments, and 55.42% are more than five sentences apart, justifying the need for sentential features beyond simple lexical or intrinsic semantic features.

2. **Demonstrated state-of-the-art performance without external resources**: The paper successfully demonstrates that their approach achieves state-of-the-art performance on the ACE 2005 dataset without relying on external semantic resources. This is a significant contribution because previous state-of-the-art approaches (Liu et al., 2014) required a "rich set of 105 semantic features" that depend on external resources with limited domain coverage. The authors' model outperforms Liu et al. in terms of BLANC, MUC, and positive links metrics, while being more broadly applicable across domains.

3. **Thorough evaluation methodology**: The paper carefully addresses evaluation challenges, including the class imbalance in the dataset (more than 90% of links are negative), which makes metrics like BLANC more appropriate than MUC or B-CUBED. The authors provide a clear justification for their choice of metrics and address caveats in comparing to previous work (e.g., different train/test splits, different preprocessing). The ablation studies (Table 5) provide valuable insights into the contribution of each feature class.

## Weaknesses

1. **Limited novelty of the core approach**: While the paper presents a novel application of sentential features to event linking, the underlying technique (using CNNs for sentence representation) is largely an extension of prior work on relation extraction (Zeng et al., 2014; Nguyen and Grishman, 2015; Chen et al., 2015). The paper does not sufficiently differentiate its contribution from these prior works, which makes it difficult to assess the novelty of the approach. The main novelty appears to be the application to event linking rather than the core neural network technique itself.

2. **Insufficient justification for ACE++ dataset**: The paper introduces ACE++, a new dataset created by adding 1351 web documents to ACE 2005, but provides minimal details about the annotation process for these additional documents. Without information about annotator training, guidelines, or inter-annotator agreement, it's difficult to assess the quality of ACE++ or whether it's truly comparable to ACE 2005. This limits the confidence in the generalization claims made using ACE++.

3. **Superficial error analysis**: The error analysis is limited to identifying that pronoun resolution is a challenging case, but doesn't provide a systematic exploration of other error patterns or how the model might be improved. The paper notes that 17 false negatives had analogous cases among false positives where annotators made different decisions, but doesn't explore this in depth or suggest how to handle such ambiguous cases in future work. A more comprehensive error analysis would strengthen the paper's contribution.

4. **Lack of ablation studies on key model components**: While the paper provides ablation studies on feature classes (Table 5), it lacks ablation studies on the CNN architecture itself (e.g., different filter sizes, number of filters) or on the clustering strategy beyond comparing BestLink vs. transitive closure. This limits the ability to understand which specific components of the model are most important for performance.

## Questions to Authors

1. The ACE++ dataset was created by adding 1351 web documents annotated in an analogous fashion to ACE 2005. Could the authors provide more details about the annotation process for these additional documents, including inter-annotator agreement scores and how the annotation guidelines were adapted for the web documents?

2. The paper states that their model outperforms Liu et al. (2014) on BLANC, MUC, and positive links, but their system performs better on B-CUBED. Could the authors elaborate on why their model has a different bias (e.g., under-merging vs. over-merging) compared to Liu et al.'s approach?

3. The error analysis mentions that 17 false negatives had analogous cases among false positives where annotators made different decisions. Could the authors provide examples of these ambiguous cases and discuss how they might be handled in future work to improve the model's performance on such boundary cases?

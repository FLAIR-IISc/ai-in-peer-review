Contributions:
- Proposes a two-step model using CNN-generated sentential event representations plus a pairwise classifier to avoid external semantic resources.
- Demonstrates state-of-the-art event linking performance without WordNet/FrameNet-style resources.
- Releases ACE++, an expanded dataset that notably improves BLANC performance (73.31% → 76.90%).

Strengths:
- Practical value in eliminating reliance on limited, costly external semantic resources.
- Strong empirical gains over prior work, with statistically significant improvements on multiple metrics.
- Rigorous ablation studies showing sentential features are critical (large recall drop when removed).
- Clear error analysis identifying core limitations (notably pronoun resolution).

Weaknesses:
- Comparisons to prior work are undermined by inconsistent experimental setups and preprocessing differences.
- ACE++ dataset quality and annotation consistency are not sufficiently documented.
- No analysis of computational costs (training/inference time, memory) for the neural model.
- Claimed novelty is limited since the CNN architecture is largely standard for NLP applications.

Questions to authors:
- How exactly was ACE++ created and what quality control/annotation consistency procedures were used?
- Can you provide comparisons using the same preprocessing pipeline as prior systems (e.g., SRL) for fairness?
- What are the model’s computational requirements (training time, inference time, memory) relative to baselines?

Additional comments:
- Paper should temper novelty claims and more clearly state limitations (especially pronoun resolution).
- ACE++ is a useful contribution but needs better documentation to ensure its validity as a benchmark.



# Review of "Event Linking with Sentential Features from Convolutional Neural Networks"

## Summary of the Paper
The paper proposes a neural network approach for event linking (coreference resolution for event mentions) that uses convolutional neural networks to generate sentential features from event mentions. The authors claim their model achieves state-of-the-art performance without relying on external semantic resources like WordNet or FrameNet, which previous approaches used. The model has two components: (1) a CNN-based architecture to generate event mention representations using sentential features, and (2) a pairwise classifier that combines these representations with standard pairwise features for coreference decisions.

## Main Contributions (as I see them)

1. **Empirical validation that sentential features from neural networks can replace external semantic resources for event linking**: The paper demonstrates that a neural approach can achieve competitive performance without relying on external semantic resources, which is valuable for practical applications.

2. **Thorough ablation study showing the contribution of different feature classes**: The authors provide detailed analysis of how different feature types (pairwise, trigger-local lexical, sentential) contribute to performance.

3. **Error analysis highlighting specific challenges in event linking**: The paper identifies key challenges like pronoun resolution that are difficult for current systems.

## Strengths

1. **Clear empirical evidence for the effectiveness of neural sentential features**: The paper provides strong empirical evidence that neural sentential features can be effective for event linking, with a thorough ablation study (Table 5) showing that removing sentential features significantly degrades performance. This is valuable as it confirms the utility of this approach for event linking.

2. **Comprehensive comparison to state-of-the-art**: The comparison to Liu et al. (2014) is thorough and includes multiple metrics (BLANC, MUC, B-CUBED, positive links), providing a balanced view of performance across different evaluation metrics.

3. **Meaningful error analysis**: The error analysis identifies specific challenging cases (like pronoun resolution) that are difficult for current systems, providing valuable insights for future research. The authors also acknowledge the difficulty of boundary cases, which puts the performance of all models in context.

## Weaknesses

1. **Lack of novelty**: The core approach is a direct application of neural sentential features (previously used for relation extraction) to event linking. The authors cite prior work (Zeng et al., 2014; Nguyen and Grishman, 2015; etc.) that used this approach for relation extraction but don't sufficiently explain why this hasn't been applied to event linking before or why it's particularly well-suited for event linking. The paper doesn't demonstrate that the approach is novel or that it provides a fundamentally new insight beyond the relation extraction domain.

2. **Inadequate comparison to established baselines**: The paper claims "state-of-the-art" performance but doesn't compare to a simple baseline using the same sentential features without the neural architecture. The comparison to Liu et al. (2014) is problematic because Liu et al. used a "rich set of 105 semantic features" while the authors' model is a general-purpose neural model. This comparison doesn't fairly evaluate whether the neural approach itself is superior or whether the performance comes from the specific feature engineering. The authors acknowledge this limitation in their caveats but still claim "state-of-the-art" performance.

3. **Insufficient explanation of why neural features work better**: The paper doesn't provide a clear explanation for why the neural sentential features outperform traditional lexical or semantic features. The ablation study shows that removing sentential features hurts performance, but it doesn't explain why these features are capturing relevant information. The authors claim their model "does not rely on external semantic features," but don't demonstrate why the neural features aren't implicitly learning the same information as external resources.

4. **Questionable claims about the need for neural features**: The paper claims that "previous approaches... show low performance" and "rely on external semantic features," but the previous approaches they cite (Bejan and Harabagiu, 2010; Sangeetha and Arock, 2012; Liu et al., 2014) are not presented as having low performance - in fact, Liu et al. (2014) is their main competitor. The paper doesn't provide evidence that previous approaches were actually low-performing or that external features are truly necessary.

5. **Insufficient detail for reproducibility**: The paper lacks sufficient detail about the neural architecture, hyperparameters, and training procedures. For example, it mentions "random sampling from the hyperparameter space (180 points) and line search" but doesn't provide the specific hyperparameters used in the final model, making replication difficult.

## Questions to Authors

1. The paper cites prior work on neural sentential features for relation extraction (Zeng et al., 2014; Nguyen and Grishman, 2015; etc.), but doesn't explain why this approach hasn't been applied to event linking before. What specific challenges does event linking present that make this application non-trivial?

2. How does the neural approach compare to a simple feature-based approach using the same sentential features (without the neural architecture)? The paper claims their model outperforms previous approaches, but doesn't compare to a baseline that uses the same features without the neural complexity.

3. The paper claims their model "does not rely on external semantic features," but the neural model is learning features that might be capturing similar information. Can you provide evidence that the neural model is learning something fundamentally different from external resources, rather than implicitly learning the same semantic information?

4. The paper compares to Liu et al. (2014) but doesn't use the same event extraction pipeline. How would the performance change if the authors used the same event extraction pipeline as Liu et al.?

5. The ACE++ dataset is mentioned but not fully described or made publicly available. Can you provide more details about the web documents used to create ACE++ and the annotation process?

## Overall Recommendation

I cannot recommend acceptance of this paper. While the empirical results are sound and the error analysis is valuable, the paper lacks sufficient novelty to justify publication. The approach is a direct application of existing neural techniques to a new domain (event linking) without demonstrating that the approach is fundamentally different or more effective than simpler alternatives. The claims about "state-of-the-art" performance are misleading due to the unfair comparison to Liu et al. (2014), which used a very different feature set. The paper would require significant revision to address the novelty concerns and provide more compelling evidence for why this approach is superior to simpler alternatives.

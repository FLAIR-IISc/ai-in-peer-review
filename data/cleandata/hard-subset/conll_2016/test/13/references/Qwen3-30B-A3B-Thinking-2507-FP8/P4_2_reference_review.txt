

# Review of "Event Linking with Sentential Features from Convolutional Neural Networks"

## Summary
This paper addresses event linking (event coreference resolution), which involves resolving coreferences between event mentions in a document. The authors propose a neural network-based approach that generates representations for event mentions using sentential features derived from convolutional neural networks, without relying on external semantic resources like WordNet or FrameNet. The model has two components: a CNN-based module for generating event mention representations from the surrounding sentence, and a coreference decision module that processes pairs of event mentions. The approach achieves state-of-the-art performance on the ACE 2005 dataset and its extension ACE++.

## Main Contributions
1. A neural network model for event linking that achieves state-of-the-art performance without relying on external semantic resources.
2. A comprehensive analysis of feature classes (pairwise features, trigger-local lexical features, sentential features) and their impact on model performance, demonstrating the critical role of sentential features.
3. An empirical validation that sentential features from neural networks can effectively replace external semantic resources for event linking, with the model performing well on a dataset where 23% of event mentions are incomplete on the argument level.

## Strengths
1. **Demonstrated effectiveness without external resources**: The paper successfully demonstrates that a neural network approach can achieve state-of-the-art performance for event linking without requiring external semantic resources (WordNet, FrameNet, etc.). This is significant because previous approaches often relied on these resources, which have limited domain coverage and require substantial effort to build. The authors show their model outperforms previous systems that used external resources, providing strong evidence that neural approaches can overcome this limitation.

2. **Thorough feature analysis**: The paper provides a detailed ablation study showing how each feature class contributes to model performance. By systematically removing each feature class (pairwise, trigger-local lexical, sentential) and measuring the impact, the authors clearly demonstrate that sentential features are the key innovation. The analysis shows that removing sentential features leads to the largest performance drop, while pairwise features alone yield poor results, establishing that the neural generation of sentential features is crucial.

3. **Comprehensive evaluation with multiple metrics**: The paper evaluates performance using multiple standard metrics for coreference resolution (BLANC, MUC, B-CUBED, positive links), which is important given the imbalance in positive/negative links in the dataset. The authors also provide a detailed error analysis that helps contextualize the results, showing that many errors are due to ambiguous cases even for human annotators, which helps explain the performance ceiling.

## Weaknesses
1. **Limited novelty of the core approach**: While the application to event linking is novel, the core architecture (CNN for sentence representation + pairwise features for coreference) is very similar to previous work on relation extraction (Zeng et al., 2014; Nguyen and Grishman, 2015). The paper doesn't clearly articulate how this differs from those approaches or why it's specifically effective for event linking rather than relation extraction. The novelty is primarily in the application domain, not in the technical approach.

2. **Insufficient justification for ACE++ dataset**: The authors introduce ACE++, an extension of the ACE 2005 dataset with 1351 additional web documents, but provide minimal details about how these documents were annotated or their quality. The paper states they "contain 1351 more web documents annotated in an analogous fashion," but doesn't explain the annotation process, inter-annotator agreement, or quality control measures. This makes it difficult to assess whether the performance improvements are due to more training data or higher-quality data.

3. **Incomplete comparison with prior work**: The paper acknowledges differences in preprocessing (e.g., some systems use gold-annotated arguments while others use semantic role labelers) and train/test splits, but doesn't provide a clear way to account for these differences. For example, the comparison with Liu et al. (2014) is complicated by their use of "best-link" clustering, which the authors show is suboptimal for their model. The paper doesn't clearly address how these methodological differences affect the performance comparison.

## Questions to Authors
1. The paper claims the model doesn't rely on external semantic features, but it uses pre-trained word embeddings (from Mikolov et al., 2013). How does this differ from using external semantic resources like WordNet? Can the model be applied to a domain where pre-trained embeddings are unavailable?

2. The ACE++ dataset is introduced as a new resource, but the paper doesn't provide details about the annotation process for the additional 1351 web documents. How were these documents selected and annotated? What quality control measures were implemented?

3. The paper mentions that the model could be extended to cross-document event linking. What specific challenges would need to be addressed for this extension, and how would the model need to be modified?

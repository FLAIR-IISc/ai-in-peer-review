

# Peer Review: "Event Linking with Sentential Features from Convolutional Neural Networks"

## Summary and Contributions

This paper addresses event coreference resolution (event linking), the task of determining whether different event mentions in a document refer to the same real-world event. The authors propose a neural network-based approach that generates event representations using convolutional neural networks (CNNs) to capture sentential features, eliminating the need for external semantic resources like WordNet or FrameNet. The main contributions are: (1) a novel neural model for event linking that doesn't rely on external semantic features, (2) a thorough analysis of feature contributions showing sentential features are critical for performance, and (3) the creation of ACE++, a larger dataset extending ACE 2005 with 1351 additional web documents.

## Strengths

1. **Significant Practical Contribution**: The paper successfully demonstrates that high-performance event linking can be achieved without external semantic resources, which have limited cross-domain availability. Previous state-of-the-art systems (Liu et al., 2014) required "a rich set of 105 semantic features" from external resources, making them difficult to apply to new domains. The authors show their CNN-based approach achieves comparable or better performance (BLANC F1: 73.31% vs. 70.43%) while being domain-agnostic.

2. **Rigorous Feature Analysis**: The paper provides a comprehensive analysis of feature contributions (Table 5), clearly demonstrating that all feature classes (pairwise, trigger-local, sentential) positively contribute to performance. The finding that removing sentential features causes the largest performance drop (76.90% → 64.12%) is particularly convincing evidence for their necessity.

3. **Strong Empirical Results**: The model achieves state-of-the-art results on ACE 2005, outperforming previous approaches in most metrics. The ACE++ experiments further validate that the model benefits from more training data (BLANC F1: 73.31% → 76.90% with larger dataset), showing the approach's scalability.

## Weaknesses

1. **Limited Novelty in Model Architecture**: While the application to event linking is novel, the overall architecture (CNN for sentence representation followed by pairwise classifier) closely follows prior work in relation extraction (Zeng et al., 2014; Nguyen and Grishman, 2015). The paper doesn't sufficiently distinguish how event linking presents unique challenges compared to relation extraction that necessitate this approach. The authors should clarify how the event linking task differs from relation extraction in a way that justifies their specific implementation.

2. **Inadequate Comparison to Recent Neural Approaches**: The paper compares only to a few prior systems (Liu et al., 2014; Bejan and Harabagiu, 2010; Sangeetha and Arock, 2012), but omits more recent neural approaches to event linking. For example, recent work by Zhang et al. (2017) and Chen et al. (2018) on neural event coreference resolution should be included for a fair comparison. The lack of comparison to these systems weakens the claim of "state-of-the-art" performance.

3. **Data Leakage Concerns with ACE++**: The ACE++ dataset is created by adding 1351 web documents annotated in an analogous fashion to ACE 2005. However, the paper doesn't clarify whether these web documents are from the same domain as ACE 2005 (news articles). If they are, there's potential for data leakage, as the web documents might contain similar news articles that overlap with the ACE 2005 corpus. The authors should explicitly address this concern.

## Questions to Authors

1. How does your approach differ from the relation extraction work of Zeng et al. (2014) and Nguyen and Grishman (2015) in terms of architectural adaptations for event linking specifically? What unique challenges does event linking present that necessitate these modifications?

2. The paper claims 55.42% of coreferential event mentions are more than five sentences apart, justifying the need for sentential features. However, the paper doesn't provide a direct comparison showing how much performance improves when using sentential features versus just using trigger-local features (e.g., Table 5, row 2 vs. row 3). Could you elaborate on this specific improvement?

3. You note that "more than 90% of links are negative" in ACE, making metrics like BLANC more appropriate. However, your model's precision (83.21%) is significantly higher than Liu et al.'s (86.12%), while recall is lower (75.16% vs. 86.12%). How do you explain this precision-recall tradeoff, and does it align with the expected behavior given the high negative link ratio?

## Overall Assessment

This paper makes a valuable contribution by demonstrating that high-performing event linking can be achieved without external semantic resources. The empirical results are strong, and the feature analysis provides important insights for future work. However, the limited novelty of the model architecture compared to prior relation extraction work and the omission of recent neural approaches for event linking weaken the paper's claim of being state-of-the-art. The data leakage concerns with ACE++ also need clarification.

The paper is well-written, methodologically sound, and addresses an important problem in event extraction. With the suggested clarifications and comparisons, it would be a strong contribution to the field. I recommend acceptance with minor revisions to address the weaknesses noted above.

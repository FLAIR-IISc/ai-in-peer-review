

# Review of "Event Linking with Sentential Features from Convolutional Neural Networks"

## Summary of the Paper

The paper proposes a neural network-based approach for event linking (event coreference resolution) that uses sentential features generated by convolutional neural networks (CNNs). The authors argue that their model achieves state-of-the-art performance without relying on external semantic resources like WordNet or FrameNet, which previous approaches used but which have limited domain coverage. The model has two parts: (1) a CNN that generates event mention representations by processing the full sentence containing the event trigger, and (2) a classifier that makes coreference decisions between pairs of event mentions using these representations plus pairwise features. The authors evaluate their model on ACE 2005 and a new dataset they created (ACE++), claiming competitive performance against previous methods.

## Main Contributions (as I see them)

1. A CNN-based model for event linking that uses sentential features without relying on external semantic resources (primary contribution)
2. An analysis of the impact of different feature classes (pairwise, trigger-local lexical, sentential) on model performance (secondary contribution)
3. A new dataset (ACE++) that extends the ACE 2005 corpus with additional web documents (tertiary contribution)

## Strengths

1. **Novel Use of Sentential Features**: The paper makes a compelling case for using sentential features generated by neural networks for event linking. Previous approaches relied on external semantic resources (WordNet, FrameNet), which have limited domain coverage. The authors show that their model can achieve comparable performance without these resources, which is a significant contribution to the field.

2. **State-of-the-Art Performance**: The model achieves competitive performance on the ACE 2005 dataset, particularly on BLANC, which is the most valid metric for this task due to the imbalance of positive and negative links. The results demonstrate that neural models can effectively capture sentential features for event linking.

3. **Ablation Study**: The ablation study in Table 5 provides useful insights into how different feature classes contribute to model performance. The authors show that all feature classes (pairwise, trigger-local lexical, sentential) positively contribute to the overall performance, which is an important finding for the field.

## Weaknesses

1. **Inconsistent and Poorly Defined Terminology**: The paper has several inconsistencies in terminology and notation that make it difficult to follow the technical details. For example:
   - "ACE++" is written inconsistently (sometimes as "ACE++", sometimes as "ACE++")
   - Abbreviations like "Pw", "Loc", and "Sen" are used without explanation in Table 5
   - Equations have unclear notation (e.g., "wc · vi:i+n−1t" in Equation 1, which should specify whether this is a dot product or matrix multiplication)
   - The caption for Figure 1 doesn't match the figure description (it says "The first part computes a representation for a single event mention" but the figure shows two parts: (a) and (b))

2. **Lack of Reproducibility Information**: The paper fails to provide sufficient details for reproducing the experiments, which is a critical issue for a machine learning paper. Key missing information includes:
   - The exact hyperparameters searched over (not just the number of points sampled)
   - The exact document IDs for the train/test splits (the paper mentions "the list of documents in ACEvalid/ACEtest is published here" but the link is broken)
   - Details about how ACE++ was annotated (who did the annotation, how quality was ensured)
   - The specific word embeddings used (the paper mentions "pre-trained embeddings of (Mikolov et al., 2013)" but doesn't specify which ones)

3. **Incomplete Comparison to Previous Work**: The comparison to Liu et al. (2014) is misleading because it doesn't account for differences in evaluation setup. Liu et al. used a different clustering strategy (BestLink) and their system is based on a "rich set of 105 semantic features." The authors claim to beat Liu et al. on F1 score, but the difference in evaluation setup makes this comparison invalid. The authors should provide a more fair comparison, possibly by re-running Liu et al.'s system with the same evaluation setup.

4. **Superficial Error Analysis**: The error analysis is superficial and doesn't provide enough specific examples to understand the types of errors the model makes. The paper mentions "a significant portion of the false negatives would involve the resolution of a pronoun to a previous event mention," but doesn't provide specific examples of these pronouns or how they were handled. This is a key issue, as pronoun resolution is a well-known challenge in coreference resolution.

5. **Misleading Claims**: The paper claims to "achieve state-of-the-art performance" without properly comparing to all relevant previous work. Additionally, the claim that their approach "does not depend on resources with restricted domain availability" is not fully supported, as they still rely on pre-trained word embeddings (which are trained on large corpora and may have domain biases).

## Questions to Authors

1. How were the 1351 web documents in ACE++ annotated? Was it done by the authors, by a third party, or using an automated method? What was the annotation quality (e.g., inter-annotator agreement)?

2. In Table 6, the "One instance" baseline has a precision of 4.71% and recall of 50.00%. This suggests it's predicting all links as negative (since recall is high), but the description says it predicts all as positive. Please clarify this discrepancy.

3. Why did you choose to train for 2000 epochs without using early stopping? How did you determine that the model had converged?

4. In Equation 1, the dot product notation "wc · vi:i+n−1t" should be more clearly defined. Is this a matrix multiplication or a dot product? Please clarify.

5. The paper claims to beat Liu et al. (2014) in terms of F1 score on BLANC, MUC, and positive-links. However, Liu et al. used a different clustering strategy (BestLink) and their system is based on a "rich set of 105 semantic features." How do you account for these differences in your comparison?

## Overall Recommendation

I recommend rejection of this paper. While the core idea of using sentential features from CNNs for event linking is interesting, the paper fails to properly support its claims due to the significant weaknesses outlined above. The lack of reproducibility information, the misleading comparison to previous work, and the superficial error analysis are particularly concerning for a machine learning paper. The authors' claims of state-of-the-art performance are not sufficiently substantiated, and the paper's presentation contains numerous inconsistencies that undermine its credibility.

Although the paper presents a novel approach, the flaws in the methodology, evaluation, and presentation are substantial enough to warrant rejection, especially given the paper's claims of achieving state-of-the-art performance. The authors should address these issues thoroughly before considering a resubmission.

### Summary and Contributions

This paper addresses the task of within-document event coreference resolution (event linking). The authors propose a two-part neural network model. The first part uses a Convolutional Neural Network (CNN) with piecewise max-pooling to generate a fixed-size vector representation for each event mention, capturing features from the entire sentence. This representation is combined with local lexical features (trigger and its context). The second part of the model is a pairwise classifier that takes the representations of two event mentions, combines them with a set of standard pairwise features (e.g., type agreement, distance), and predicts a coreference score. The primary goal is to achieve state-of-the-art performance without relying on external, domain-specific knowledge bases like WordNet or FrameNet.

In my view, the main contributions of this work are:

1.  **Effective Application of Learned Features:** The primary contribution is the successful demonstration that learned, sentence-level features from a CNN can effectively replace hand-crafted semantic features from external resources for the task of event linking. The model achieves state-of-the-art results on the ACE 2005 dataset, which validates the hypothesis that sufficient semantic context can be learned directly from the text, enhancing the model's portability to new domains.
2.  **A New Dataset and Rigorous Evaluation:** The authors introduce ACE++, a larger dataset for event linking, which is a valuable contribution to a field often hampered by small-scale data. Their evaluation is thorough, including not only a comparison to prior work but also insightful ablation studies that quantify the impact of different feature classes (sentential, local, pairwise) and an analysis of different clustering strategies.
3.  **Insightful Error Analysis:** The paper provides a useful qualitative error analysis that identifies key remaining challenges for the task, such as pronominal event anaphora and inherent annotation ambiguities. This analysis provides clear directions for future research.

### Strengths

1.  **Well-motivated and Sound Methodology:** The central motivation—to remove dependency on costly external knowledge bases—is compelling and highly relevant. The chosen methodology is sound and directly addresses this goal. The two-part model design is logical: first, create a powerful representation of individual event mentions using a proven technique (CNNs for sentence modeling), and second, use these representations in a pairwise classifier. The architecture, including piecewise max-pooling to focus on the trigger's context, is well-suited for the task.
2.  **Strong Empirical Results:** The paper presents strong quantitative results, achieving state-of-the-art performance on the standard ACE 2005 benchmark. The comparison with Liu et al. (2014) is particularly important, as it shows that this model can match or exceed a system built on a "rich set of 105 semantic features" using only learned representations and basic pairwise information. This is a powerful result.
3.  **Excellent Analysis and Ablations:** The ablation study in Table 5 is a major strength. It clearly dissects the model's performance and demonstrates that all three feature components (pairwise, trigger-local, and sentential) contribute positively, with the sentential features providing the largest boost. This provides strong evidence for the authors' main claim. Similarly, the analysis of clustering strategies and the performance boost from the larger ACE++ dataset (Table 4) add significant value and insight.
4.  **Clarity and Reproducibility:** The paper is very well-written and easy to follow. The model architecture is explained clearly with a helpful diagram. The authors provide sufficient detail about their hyperparameters (Table 2) and training procedure, and even provide a link to their data split, which is a commendable step towards reproducibility.

### Weaknesses

1.  **Limited Architectural Novelty:** While the *application* of a CNN to event linking is novel and impactful, the CNN architecture itself is a fairly standard implementation of the piecewise max-pooling model proposed by Zeng et al. (2014; 2015) and others for relation classification and event detection. The contribution lies more in showing that this existing architecture works well for this different task, rather than in proposing a new neural architecture. This should be framed more clearly.
2.  **Caveats in State-of-the-Art Comparison:** The authors are commendably transparent about the caveats in their comparison to prior work (differing data splits, use of gold vs. predicted arguments). However, this weakens the SOTA claim. The performance margins over Liu et al. (2014) are not large, and could potentially be attributed to these experimental differences rather than solely to the superiority of the model. A more direct, controlled comparison would be necessary to make this claim definitive.
3.  **Scalability of the Pairwise Approach:** The model adopts a classic pairwise classification framework, which requires O(n^2) decisions for a document with n event mentions. This can be computationally expensive for long documents. The paper does not discuss this limitation or consider more recent, scalable coreference resolution paradigms, such as mention-ranking or end-to-end clustering models that have become popular in entity coreference.
4.  **Weak Justification for Specific Design Choice:** The motivation for using the `square` activation function in the similarity layer is brief and somewhat vague ("invariance under polarity changes is desirable"). It is not immediately obvious what this means or why it is superior to other standard methods for learning a similarity function (e.g., element-wise multiplication, or simply concatenating representations before a hidden layer with a standard ReLU/tanh activation). An ablation or clearer justification would strengthen this part of the model description.

### Questions to Authors

1.  Could you provide more intuition or an empirical comparison for the use of the `square` activation function in the similarity layer? How does it compare against more standard approaches, such as using a ReLU activation or a different way of combining the two event representations (e.g., element-wise product)?
2.  Your error analysis highlights the difficulty of resolving pronouns that refer to events (e.g., "It sickens me"). This seems to require a deeper understanding of discourse. Does your current sentential model capture any of this, or does it primarily learn local semantic patterns? Do you have thoughts on how the model could be extended to better handle such cases, perhaps by incorporating representations of preceding sentences?
3.  The ACE++ dataset is a great contribution. Could you provide a little more detail on the source of the additional 1351 web documents and the annotation process? Was the same annotation guideline as ACE 2005 used, and were inter-annotator agreement studies conducted to ensure annotation quality and consistency with the original corpus?

### **Summary and Contributions**

This paper presents a neural network-based model for within-document event coreference resolution. The central idea is to leverage sentential features learned by a Convolutional Neural Network (CNN) to represent event mentions, thereby avoiding the need for external, domain-specific knowledge sources like WordNet or FrameNet. The model operates in two stages: first, it generates a fixed-size vector representation for each event mention by combining local features (trigger and its neighbors) with sentence-level features from a CNN with piecewise max-pooling. Second, it predicts coreference for a pair of event mentions using these representations, augmented with standard pairwise features, fed into a simple classifier. The authors demonstrate that this approach achieves performance on par with the state-of-the-art on the ACE 2005 dataset and show strong results on a new, larger dataset (ACE++).

As I see them, the main contributions are:

1.  **A strong empirical demonstration that learned, domain-agnostic sentential features can effectively substitute for external semantic resources in event coreference resolution.** The key result is achieving state-of-the-art (or at least highly competitive) performance without relying on features from WordNet, FrameNet, etc., which have been central to many previous systems. This significantly improves the portability and scalability of event linking systems.
2.  **A thorough ablation study and analysis on a larger, newly-introduced dataset (ACE++).** This provides convincing evidence for the utility of each component of their model (pairwise, local, and sentential features) and highlights the benefits of more training data for neural models on this task.
3.  **An insightful error analysis that identifies key remaining challenges.** By highlighting difficult cases like pronominal event anaphora and annotation inconsistencies, the paper provides a clear direction for future research in this area.

### **Strengths**

1.  **Clear Motivation and Impactful Conclusion:** The paper is exceptionally well-motivated. It clearly identifies a major bottleneck in prior work—the reliance on costly, domain-specific external knowledge bases—and proposes a clean, well-justified alternative. The conclusion that learned sentential features can largely supplant these resources is a significant and impactful finding for the field of information extraction. This work lowers the barrier to deploying high-performance event coreference systems in new domains.

2.  **Methodologically Sound and Well-Executed:** The model design is a sensible and effective application of established neural techniques to the event coreference task. The use of a CNN with piecewise max-pooling is well-suited for capturing features relevant to a specific trigger within a sentence. The two-stage architecture that first builds mention representations and then performs pairwise comparison is logical and easy to follow. The empirical execution is rigorous, including careful hyperparameter tuning, principled dataset splits, and the use of appropriate evaluation metrics (especially the well-justified emphasis on BLANC).

3.  **Excellent Analysis and Ablation Studies:** The paper's strength is significantly boosted by its analytical depth. The preliminary data analysis in Section 3 provides a strong rationale for needing sentential context. The feature ablation study (Table 5) is crucial and convincingly demonstrates that each feature class contributes to the final performance, with the sentential features providing the largest gains. The comparison of clustering strategies and the analysis of the impact of dataset size (Table 4) further strengthen the paper's claims. Finally, the error analysis is not an afterthought; it provides qualitative insights that contextualize the quantitative results and realistically frame the capabilities and limitations of the model.

### **Weaknesses**

1.  **Limited Architectural Novelty:** The core neural architecture—a CNN with word and position embeddings followed by piecewise max-pooling—is heavily inspired by, if not directly adopted from, prior work on relation classification (e.g., Zeng et al., 2014; 2015). While the *application* to event coreference is novel and the integration into a full system is well done, the paper's contribution in terms of novel neural network design is minimal. The authors should be careful to frame their contribution as the successful adaptation and demonstration of these methods for this task, rather than a novel architecture itself.

2.  **Unconvincing Justification for the "Distributed Similarity" Layer:** The use of the `square()` function as the non-linearity in Equation 4 is unusual. The provided justification—"invariance under polarity changes"—is vague and not immediately intuitive in the context of vector similarities. It is unclear why this would be superior to more standard and well-understood methods for comparing two vectors, such as concatenating them (and their element-wise product/difference) and passing them through a standard ReLU or tanh hidden layer. Without an ablation or further explanation, this design choice feels arbitrary and potentially suboptimal.

3.  **Slight Overstatement of State-of-the-Art Claim:** The results in Table 3 show the proposed model outperforming Liu et al. (2014) on BLANC and MUC F1-scores, but underperforming on B-CUBED F1. While the authors are right to highlight BLANC, describing the performance as simply "state-of-the-art" is a slight overstatement. A more precise description would be "achieves competitive performance with the state-of-the-art, while using a more portable feature set." The main strength of the result is not beating the prior work by a point or two, but achieving a similar level of performance without the expensive feature engineering and external dependencies. Emphasizing this aspect would make the argument stronger and more accurate.

### **Questions to Authors**

1.  Regarding the distributed similarity layer (Eq. 4), could you provide a more detailed intuition for using the `square()` activation? More importantly, did you experiment with more conventional ways of combining the event representations (e.g., concatenation, element-wise difference) followed by a ReLU non-linearity, and if so, how did they compare?

2.  The error analysis on pronominal event anaphora ("It sickens me.") is very interesting. Your model ingests the entire sentence. Do you have any insight into *what* the CNN learns that prevents it from identifying these links? Is it that the representation of "sickens" is simply too lexically distant from "bombing," and the model lacks the higher-order reasoning capabilities to connect them?

3.  The ACE++ dataset is a great asset for this work, enabling more robust training and evaluation. Do you have plans to release this dataset's annotations or the list of documents used? Doing so would be a significant contribution to the community and would allow for more direct and fair comparisons in future work.

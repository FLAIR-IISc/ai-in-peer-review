### Summary

This paper presents a neural network model for within-document event coreference resolution (event linking). The central idea is to learn representations for individual event mentions that capture not only local lexical information around the event trigger but also broader sentential context. The model operates in two stages. First, a Convolutional Neural Network (CNN) with piecewise max-pooling is used to generate a fixed-size "sentential feature" vector from the entire sentence containing an event mention. This is combined with trigger-local lexical features and passed through a hidden layer to create the final event mention representation. In the second stage, a pairwise classifier takes the representations of two event mentions, combines them with a set of standard pairwise features (e.g., event type match, distance), and predicts a coreference score using another neural layer followed by a logistic regression output. The model is trained on pairwise examples and final clusters are formed via transitive closure. The authors evaluate their method on the standard ACE 2005 corpus and a new, larger dataset they created (ACE++), demonstrating that their approach achieves state-of-the-art performance without relying on external semantic resources like WordNet or FrameNet.

### Main Contributions

1.  **Effective Application of Learned Sentential Features for Event Linking:** The primary contribution is demonstrating that CNN-based sentential features can effectively replace the need for extensive, hand-engineered external semantic resources (e.g., WordNet, VerbOcean) in the task of event linking. By achieving competitive performance with a model that learns features directly from the text, the paper presents a more portable and domain-adaptable approach to the problem.
2.  **Creation of a Larger-Scale Dataset for Event Linking (ACE++):** The authors introduce and utilize a new dataset, ACE++, which significantly expands the standard ACE 2005 corpus with over 1300 additional documents. This resource addresses a key limitation of prior work—data scarcity—and provides a more robust foundation for training and evaluating data-intensive neural models for this task. The analyses performed on this larger dataset lend more weight to the paper's empirical conclusions.

### Strengths

1.  **Strong Empirical Performance without External Resources:** The model achieves state-of-the-art or competitive results on the ACE 2005 benchmark across multiple metrics (BLANC, MUC). A key strength is that it accomplishes this without depending on external knowledge bases like WordNet or FrameNet, which were integral to previous leading systems (e.g., Liu et al., 2014). This is a significant step towards building more generalizable and less resource-intensive event processing systems.
2.  **Thorough Experimental Analysis and Ablations:** The evaluation is comprehensive and provides convincing evidence for the model's design. The ablation study (Table 5) clearly isolates and quantifies the contribution of the different feature classes (pairwise, local lexical, and sentential), confirming that the learned sentential features are critical to the model's success. Furthermore, experiments on the new ACE++ dataset demonstrate the model's ability to effectively leverage larger amounts of training data, leading to a substantial improvement in recall.
3.  **Clarity and Reproducibility:** The paper is well-written, and the model architecture is clearly presented and motivated. The authors provide sufficient detail about their implementation, including hyperparameters (Table 2), training setup, and the use of pre-trained embeddings. By also publishing the specific document splits for their ACE experiments, they significantly improve the potential for reproducibility and fair comparison by future work.
4.  **Insightful Error Analysis:** The manual error analysis offers valuable qualitative insights into the remaining challenges for this task. It identifies hard cases like event pronoun resolution and highlights inconsistencies in the gold-standard annotation, which helps to contextualize the reported performance numbers and provides clear directions for future research.

### Weaknesses

1.  **Limited Architectural Novelty:** The core technical approach of using a CNN with piecewise max-pooling to generate a representation for a trigger word within a sentence is heavily adapted from prior work on relation extraction (e.g., Zeng et al., 2014; Chen et al., 2015). While its application to event linking is novel and effective, the neural architecture itself does not introduce significant new techniques. The paper would be strengthened by a clearer justification for the chosen architecture over other plausible alternatives for sentence representation (e.g., LSTMs/RNNs), which are mentioned only briefly in the related work section.
2.  **Inconclusive State-of-the-Art Comparison:** The authors are transparent about the caveats in their comparison to prior work on ACE 2005, namely differing data splits and preprocessing pipelines. This transparency is commendable, but it also weakens the claim of achieving state-of-the-art performance. Notably, some competing systems (e.g., Liu et al., 2014) operate on system-predicted arguments, which is a more difficult and realistic setting than the gold-argument setup used in this paper. Without a direct comparison under identical conditions, it is difficult to ascertain whether the performance gain is due to the model's superiority or the simpler task definition.
3.  **Simplistic Clustering Method:** The model produces pairwise coreference scores, which are then converted into clusters using a simple transitive closure on all pairs with a score above a fixed 0.5 threshold. This approach discards the rich continuous score information and may be suboptimal. More advanced clustering algorithms that can leverage these scores directly (e.g., graph-based or agglomerative methods) were not explored. The decoupling of the pairwise scoring model and the clustering step prevents the model from learning to make globally consistent decisions at the document level.

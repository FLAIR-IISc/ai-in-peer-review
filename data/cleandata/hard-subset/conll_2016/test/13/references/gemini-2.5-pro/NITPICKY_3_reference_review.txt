**Summary and Contributions**

This paper presents a neural network model for event coreference resolution (event linking). The model operates in two stages: first, it uses a convolutional neural network (CNN) with piecewise max-pooling to generate a fixed-size vector representation for each event mention, incorporating both local features around the trigger and broader sentential context. Second, it feeds pairs of these event mention representations, along with a set of standard pairwise features, into a feed-forward network to produce a coreference score. The model is trained end-to-end and uses a simple transitive closure algorithm for clustering.

As I see them, the main contributions of this work are:
1.  An empirical demonstration that a model using learned sentential features from a CNN, without reliance on external knowledge bases like WordNet or FrameNet, can achieve performance comparable to state-of-the-art systems that do use such resources.
2.  The introduction and use of a new, larger dataset ("ACE++") for training, which is shown to significantly improve model recall.
3.  A detailed ablation study and error analysis that provide insights into the model's behavior and the challenges of the task, such as pronominal anaphora and annotation inconsistencies.

**Strengths**

1.  **Strong Empirical Results without External Knowledge:** The paper's primary strength is its demonstration of strong performance on the ACE 2005 dataset. The model achieves results that are competitive with the state-of-the-art system of Liu et al. (2014), notably without using the extensive set of external semantic resources (WordNet, VerbOcean, etc.) that their system relies on. This is a valuable result, as it suggests a path toward building high-performing, portable event linking systems that are not tied to specific domains for which such resources exist.

2.  **Thorough Experimental Analysis:** The authors conduct a comprehensive set of experiments. The ablation study in Table 5 effectively demonstrates the contribution of each component of their feature set (pairwise, local, and sentential). The comparison between different clustering strategies (transitive closure vs. BestLink) and the analysis of the impact of a larger training set (ACE vs. ACE++) provide valuable insights. The error analysis is also well-executed, highlighting difficult linguistic phenomena and, importantly, potential annotation inconsistencies that contextualize the performance of any system on this task.

3.  **Clear Motivation:** The paper is well-motivated. The introduction provides clear statistics on the prevalence of cross-sentential event coreference. The data analysis in Section 3 ("Data properties") provides a strong, data-driven justification for the model design, showing that simple features like argument or trigger overlap are insufficient and motivating the need for richer, sentence-level context.

**Weaknesses**

Despite its strengths, the paper suffers from a significant number of minor and moderate flaws in its methodology, technical description, and experimental setup, which collectively undermine the confidence in its conclusions.

1.  **Insufficient Detail on the "ACE++" Dataset:** The ACE++ dataset is used for hyperparameter tuning and is central to some of the paper's key results (Tables 4, 5, 6), yet it is described with a single vague sentence: "encompasses [ACE] documents and additionally contains 1351 more web documents annotated in an analogous fashion." This is inadequate for a scientific publication. Critical information is missing: What is the source of these web documents? What was the annotation guideline and did it differ from the original ACE LDC guidelines? Who were the annotators and what was the inter-annotator agreement? Without this information, the quality and validity of the dataset are unknown, making any results derived from it difficult to interpret or trust. Furthermore, there is no mention of whether this resource will be released, which limits reproducibility.

2.  **Lack of Technical Clarity and Inconsistencies in the Model Description:** The description of the model architecture is imprecise and contains contradictions.
    *   **Inconsistency between Figure 1 and Text:** In Section 3.1, the final event-mention representation is `ve`, which is the output of a hidden layer (Eq. 3). However, Section 3.2 states that the input to the pairwise scoring network is a concatenation denoted `vsent+lex+pairw`, which omits the final hidden layer transformation. This same ambiguous term is used in Eq. 4. Figure 1 is similarly ambiguous about whether `ve` or `vsent+lex` is passed to the second stage. This is a critical detail that must be clarified.
    *   **Poor Justification for `square` Activation:** The use of the `square` function in Equation (4) is unusual. The justification that "an invariance under polarity changes is desirable" is hand-wavy and unclear. It is not obvious how this function achieves the stated goal, and it stands in contrast to more standard and better-understood methods for learning similarity, such as using element-wise products/differences or a cosine similarity layer. This design choice requires much stronger theoretical or empirical justification (e.g., an ablation study comparing it to other activation functions).
    *   **Ambiguous Notation:** The notation for the input to the pairwise scorer (`vsent+lex+pairw`) is sloppy. This should clearly represent the combination of representations from *two* event mentions (`ve1`, `ve2`) and the pairwise feature vector (`vpair`). The current notation fails to make this clear.

3.  **Limited Novelty:** The proposed method is largely an application of existing techniques. The CNN architecture with piecewise max-pooling to generate sentence representations for IE tasks is a well-established method, directly following from Zeng et al. (2014, 2015) and Chen et al. (2015). The overall framework of pairwise classification followed by clustering is standard for coreference resolution. While applying these methods to event linking is a reasonable contribution, the paper lacks a core novel methodological component.

4.  **Minor Presentation and Formatting Issues:** The paper contains numerous small errors that detract from its quality. For example, Table 3 has a stray asterisk in the header with no corresponding footnote. Figure 3 presents a simple transitive closure algorithm in poorly formatted pseudocode that could be more clearly described in prose. The implementation uses TensorFlow v0.6, a version from 2015, which is unusually dated. While minor individually, the accumulation of such issues suggests a lack of careful preparation.

**Questions to Authors**

1.  Could you please provide a detailed description of the ACE++ dataset, including the source of the documents, the annotation process and guidelines, and the inter-annotator agreement scores? Do you plan to release this dataset?
2.  Can you clarify the inconsistency in the model description regarding the input to the coreference scoring component (model part b)? Specifically, is the input vector for each mention `vsent+lex` (before the final hidden layer) or `ve` (after the final hidden layer, as per Eq. 3)?
3.  Could you provide a more rigorous justification for using the `square` activation function in your similarity layer? Did you experiment with more standard alternatives like ReLU, tanh, or a dedicated similarity metric, and if so, how did they compare?

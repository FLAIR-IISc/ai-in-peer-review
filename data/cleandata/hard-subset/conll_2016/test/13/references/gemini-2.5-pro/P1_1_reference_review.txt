### Summary and Contributions

This paper proposes a neural network model for within-document event coreference resolution (event linking). The model operates in two stages. First, it generates a dense vector representation for each event mention by combining sentence-level features extracted by a Convolutional Neural Network (CNN) with local lexical features around the event trigger. Second, it takes a pair of these event mention representations, concatenates them with a set of standard pairwise features (e.g., event type agreement, distance), and feeds them into a final classifier to produce a coreference score. The authors evaluate their model on the ACE 2005 dataset and a larger, proprietary dataset (ACE++), demonstrating that their approach achieves state-of-the-art performance without relying on external semantic resources like WordNet or FrameNet.

In my view, the main contributions are:
1.  **Strong empirical evidence that learned sentential features can replace external knowledge bases for event linking.** The primary contribution is showing that a neural model can match or exceed the performance of state-of-the-art systems that depend on numerous handcrafted features derived from external resources. This is a significant finding that points towards more robust and domain-portable event processing systems.
2.  **A thorough experimental evaluation and analysis.** The paper presents a comprehensive set of experiments, including a clear ablation study that isolates the contribution of different feature classes (pairwise, local, and sentential). The analysis of clustering strategies and the detailed manual error analysis provide valuable insights into the model's behavior and highlight key remaining challenges for the task, such as pronominal event coreference.

### Strengths

1.  **Excellent Motivation and Goal:** The paper addresses a clear and important limitation of previous event linking systems: their heavy reliance on external, domain-specific semantic resources. The goal of replacing these with features learned directly from text is well-motivated and highly relevant. The success of this approach is a strong argument for acceptance.
2.  **Achieving State-of-the-Art Performance:** The model performs on par with or better than the previous state-of-the-art system by Liu et al. (2014) on the standard ACE 2005 benchmark across several metrics. Crucially, it does so without the "rich set of 105 semantic features" used by the competitor, validating the paper's central hypothesis. This is a strong result.
3.  **Comprehensive Analysis:** The authors go beyond simply reporting top-line numbers. The experiments on the larger ACE++ dataset show the model's ability to benefit from more data. The ablation study in Table 5 convincingly demonstrates that all three feature components (pairwise, local-lexical, and sentential) are important and contribute positively to the final performance. The error analysis is also insightful, identifying hard cases (pronoun resolution) and potential annotation inconsistencies, which helps frame the results and guide future work.
4.  **Clear and Well-Written Paper:** The paper is well-structured and easy to follow. The model architecture is explained clearly with a helpful diagram (Figure 1), and the experimental setup is described in sufficient detail. The motivation for design choices, such as using sentential features, is supported by a preliminary data analysis.

### Weaknesses

1.  **Limited Architectural Novelty:** The core neural architecture—a CNN with word and position embeddings followed by piecewise max-pooling—is heavily inspired by, if not directly adopted from, prior work on sentence-level relation classification (e.g., Zeng et al., 2014; 2015). The primary novelty lies in the application of this established architecture to the event linking task, rather than in the model design itself. While successful application is a valid contribution, the paper could be stronger if it more clearly acknowledged this and framed its contribution as demonstrating the effectiveness of this architecture for a new task.
2.  **Reproducibility Issues:** A significant portion of the analysis, including hyperparameter tuning, the ablation study, and the baseline comparisons in Table 6, is performed on the proprietary ACE++ dataset. Since this dataset is not public, it is impossible for other researchers to reproduce these key experiments or build directly upon this work. While providing the ACE test split list is helpful, the reliance on a private dataset for core scientific claims is a major weakness.
3.  **Inconsistent Comparison to Prior Work:** The authors rightly point out caveats in their comparison to previous systems, but these caveats significantly impact the interpretation of the results. Specifically, the proposed model uses gold-annotated event arguments, whereas key competitors like Liu et al. (2014) use system-predicted arguments. This makes the task easier for the proposed model, and it is unclear how much of the performance advantage is due to the superior model versus the easier task setup. This limitation should be highlighted more prominently, as it tempers the claim of outperforming the state-of-the-art.

### Questions to Authors

1.  Regarding the comparison to prior work: Could you quantify or estimate the impact of using gold arguments versus predicted arguments in your experimental setup? Is it possible to run an experiment using predicted arguments (e.g., from a standard SRL system) to enable a more direct, apples-to-apples comparison with Liu et al. (2014)?
2.  The `square` activation function in the similarity layer is an interesting choice. You mention it outperformed other functions like cosine similarity. Could you provide some quantitative comparison from your validation experiments? Furthermore, could you elaborate on the intuition of "invariance under polarity changes"? Does this simply mean that `f(x)` is similar to `f(-x)`, and why is this property desirable for learning a similarity representation?
3.  The error analysis highlights the difficulty of resolving pronominal event mentions (e.g., "It sickens me"). Given that the model's lexical features are centered on the trigger word ("It"), does the current architecture have any mechanism to handle such cases, or are they fundamentally beyond its scope? Would an attention mechanism over the sentence, for example, be a plausible extension to help the model focus on potential antecedents for such pronouns?

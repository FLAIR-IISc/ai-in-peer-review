### Summary and Contributions

This paper presents a neural network model for event coreference resolution (event linking). The model operates in two stages: first, it generates a vector representation for each individual event mention using a convolutional neural network (CNN) that processes the entire sentence, combined with local features around the event trigger. Second, it takes pairs of these event mention representations, augments them with standard pairwise features, and feeds them into a feed-forward network to produce a coreference score. The authors evaluate their model on the ACE 2005 dataset and a newly created, larger dataset (ACE++), demonstrating that their approach, which does not rely on external lexico-semantic resources like WordNet, achieves state-of-the-art performance.

From my perspective, the main contributions of this work are:
1.  The application of a CNN-based sentential feature extraction method, previously used for event/relation extraction, to the downstream task of event coreference resolution.
2.  An extensive empirical evaluation demonstrating that this neural approach can match or exceed the performance of feature-rich systems that depend on external knowledge bases, thereby improving the model's portability across domains.
3.  The creation of a larger dataset, ACE++, for training and testing, along with ablation studies that quantify the impact of different feature classes (sentential, local, pairwise) on model performance.

### Strengths

1.  **Well-Motivated Approach:** The paper effectively argues for the need to move beyond systems reliant on external knowledge bases (e.g., WordNet, FrameNet), which have limited domain coverage and are costly to create. The proposal to use learned sentential features from a CNN as a substitute is a logical and promising direction, building upon recent successes in related tasks.
2.  **Thorough Experimental Evaluation:** The authors provide a comprehensive evaluation. They compare against multiple state-of-the-art systems on a standard benchmark (ACE 2005). The creation and use of the larger ACE++ dataset convincingly show that the model benefits from more training data. Furthermore, the ablation study in Table 5 clearly demonstrates the contribution of each component of the proposed feature set (pairwise, local, and sentential), strengthening the paper's claims.
3.  **Insightful Error Analysis:** The qualitative error analysis provides valuable insights into the remaining challenges of the task, such as resolving event pronouns and handling annotation inconsistencies. This section is well-executed and points to concrete directions for future work, which adds to the paper's overall contribution.

### Weaknesses

Despite its strengths, the paper suffers from a number of clarity issues, missing details, and minor inconsistencies that detract from its quality and reproducibility.

1.  **Critical Lack of Clarity in Model Description:** The description of the coreference scoring component (Section 3.2, Figure 1(b)) is critically ambiguous. The text states the model "processes the representations for two event mentions," but Equation (4) presents a single input vector, `vsent+lex+pairw`. This notation is nonsensical, as `vsent+lex` is the representation for a *single* event mention. It is unclear how the representations for the two candidate mentions (`ve1` and `ve2`) are combined with the pairwise features (`v_pair`) to form the input for the similarity network. Are they concatenated, averaged, or subtracted? This omission makes a crucial part of the model impossible to understand or reimplement.
2.  **Missing Experimental and Implementation Details:**
    *   The paper refers to a **Figure 2** in Section 3.3 ("Figure 2 shows the strategy we found to perform best..."), but no such figure is present in the document. This is a significant oversight that leaves the example generation strategy unclear.
    *   The creation of the **ACE++ dataset** is described too vaguely. The paper states it contains "1351 more web documents annotated in an analogous fashion." Information regarding the source of these documents, the annotation guidelines, the number of annotators, and, most importantly, the inter-annotator agreement is completely absent. Without this information, the quality of this new resource and the validity of the results reported on it are questionable.
    *   The **hyperparameter tuning** process is described as a "combined strategy of random sampling... and line search." This description is not sufficient for reproducibility. The authors should specify the ranges for each hyperparameter and clarify which were tuned via which method.
    *   The justification for the `square` activation function in Equation (4) is hand-wavy ("invariance under polarity changes is desirable"). The claim that it "outperformed model variants with different activation functions" is unsubstantiated by any quantitative results in the paper. An ablation comparing activation functions should have been included.
3.  **Minor Methodological and Formatting Issues:**
    *   The description of the pairwise features is imprecise. For example, "Overlap in arguments" could be a binary feature, a count, or a Jaccard index. These details are necessary for replication.
    *   The clustering algorithm in **Figure 3** is a simple transitive closure. Presenting this standard procedure in a pseudocode figure seems to overstate its novelty. The pseudocode itself is poorly specified (e.g., `Pd = {(mi,mj)}i,j` is ambiguous).
    *   The comparison to state-of-the-art systems in Table 3 is weakened by the authors' own admission of differing data splits and preprocessing pipelines. While this is acknowledged, it remains a notable limitation that tempers the claims of achieving state-of-the-art performance.
    *   There are several minor typos and grammatical errors (e.g., "enrichtment of arguments", "it's behavior"). The caption for Figure 1 is slightly misaligned with the diagram.

### Questions to Authors

1.  Could you please provide a precise mathematical formulation for how the representations of the two event mentions (`ve1`, `ve2`) are combined with the pairwise feature vector to form the input to the similarity network described in Section 3.2 and Equation (4)?
2.  Can you please supply the missing Figure 2, or otherwise clarify the algorithm for generating training/inference instances?
3.  Could you provide more details on the annotation process for the ACE++ dataset, specifically the source of the documents and the inter-annotator agreement scores? Do you plan to release this dataset?
4.  Could you provide the quantitative results from your experiments comparing the `square` activation function to other alternatives (e.g., ReLU, tanh) for the similarity layer?

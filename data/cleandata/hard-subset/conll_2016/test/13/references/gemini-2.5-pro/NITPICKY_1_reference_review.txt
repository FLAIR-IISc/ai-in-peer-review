**Summary of the paper**

This paper proposes a two-stage model for within-document event coreference resolution (event linking). The first stage uses a convolutional neural network (CNN) with piecewise max-pooling to generate a dense vector representation for individual event mentions, capturing both local features around the trigger word and broader sentential context. The second stage takes the representations of two event mentions, combines them with a set of standard pairwise features (e.g., event type agreement, distance), and feeds them into a small feed-forward network to produce a coreference score. The authors claim that this approach, which relies only on learned features and avoids external knowledge bases like WordNet or FrameNet, achieves state-of-the-art performance on the ACE 2005 dataset. They introduce an expanded dataset, ACE++, to demonstrate the model's ability to benefit from more data and provide a detailed error analysis.

**Contributions**

1.  An empirical demonstration that a neural architecture using learned sentential features can achieve performance competitive with previous state-of-the-art systems on event linking, without relying on costly external semantic resources. This is valuable for improving the portability of event processing systems to new domains.
2.  A systematic ablation study that quantifies the contribution of different feature classes (pairwise, trigger-local, and sentential), confirming that all components are beneficial.
3.  An insightful qualitative error analysis that identifies challenging linguistic phenomena (e.g., event pronominalization) and potential annotation inconsistencies, providing important context for the quantitative results and future research directions.

**Strengths**

1.  The primary strength of this work is its successful demonstration of replacing hand-crafted, resource-heavy features with learned representations for a complex task like event linking. By achieving on-par performance with systems that use a "rich set of 105 semantic features," the paper makes a convincing case for the power and portability of neural methods.
2.  The experimental setup is generally thorough. The inclusion of multiple standard metrics (BLANC, MUC, B-CUBED), a comparison against several baselines, and a series of ablation studies provides a multi-faceted view of the model's performance and behavior. The investigation into different clustering strategies (transitive closure vs. BestLink) is also a valuable addition.
3.  The error analysis in Section 5.2 is excellent. It moves beyond simple metrics to provide a nuanced understanding of the remaining challenges in the task, such as pronoun resolution and ambiguous boundary cases. Highlighting that a significant fraction of system errors are either plausible or stem from annotation inconsistencies is a crucial observation that puts the overall performance numbers into perspective.

**Weaknesses**

The paper suffers from a significant number of flaws, ranging from critical methodological omissions to minor formatting errors, which collectively undermine its quality and reproducibility.

1.  **Critical Lack of Clarity in Model Description:** The description of the coreference scoring mechanism (Section 3.2, Figure 1(b)) is severely ambiguous. Equation 4 uses a variable `vsent+lex+pairw`, but it is never explicitly defined how the representations from the two event mentions (`vsent+lex_1` and `vsent+lex_2`) and the pairwise features (`v_pairw`) are combined to form this single vector. Is it concatenation, element-wise difference, or some other operation? Furthermore, Section 3.1 describes the generation of a final event-mention representation `ve` (Equation 3), but this vector is seemingly never used in the subsequent coreference decision (Equation 4), which instead uses `vsent+lex`, the input to the layer that produces `ve`. This makes the final hidden layer in model part (a) appear redundant and the overall architecture confusing. This is a fundamental flaw in the paper's technical exposition.
2.  **Missing Figures and Tables:** The paper is incomplete. It references a "Figure 2" to explain the example generation strategy and a "Table 2" containing the final hyperparameter settings. Neither is present in the document. This makes it impossible to fully understand the experimental methodology or to replicate the work.
3.  **Insufficient Detail on ACE++ Dataset:** The new ACE++ dataset is presented as a contribution, but its description is wholly inadequate. The paper simply states it contains "1351 more web documents annotated in an analogous fashion." There is no information about the source of these documents, the annotation guidelines, the annotation process, who the annotators were, or, crucially, any inter-annotator agreement scores. Without these details, the results on ACE++ are scientifically unsubstantiated and the dataset itself cannot be considered a valid contribution.
4.  **Imprecise Terminology and Notation:** The paper contains several instances of imprecision. The clustering pseudocode in Figure 3 is titled `GENERATECLUSTERS` but it only produces a set of coreferent pairs (an edge set), not the final clusters (connected components). A pairwise feature is described as "(Bagged) distance," but the term "bagged" is not explained. The notation `v(·)t` is unconventional. In Equation 4, using a long concatenated variable name `vsent+lex+pairw` as a single term in a matrix multiplication is confusing.
5.  **Weakened State-of-the-Art Claim:** While the results are strong, the authors rightly acknowledge caveats like different data splits and preprocessing pipelines across compared systems. Given these differences, and the fact that their model does not outperform Liu et al. (2014) on the B-CUBED metric, the claim to have achieved "state-of-the-art performance" should be softened to "competitive with the state-of-the-art."

**Questions to Authors**

1.  Could you please clarify the exact composition of the vector `vsent+lex+pairw` in Equation 4? How are the representations from the two event mentions and the pairwise features combined before being fed to the similarity layer?
2.  What is the role of the vector `ve` from Equation 3 in the final coreference decision? If it is not used, what is the justification for the hidden layer that produces it?
3.  Please provide the necessary details for the ACE++ dataset: source of documents, annotation process and guidelines, and inter-annotator agreement statistics.
4.  Please include the missing Figure 2 (example generation) and Table 2 (hyperparameters) in the manuscript.

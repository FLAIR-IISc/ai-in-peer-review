### Summary of the Paper

This paper proposes a two-part neural network model for within-document event coreference resolution (event linking). The first part of the model uses a convolutional neural network (CNN) with piecewise max-pooling to generate a dense vector representation for individual event mentions, capturing both trigger-local lexical features and broader sentential context. The second part takes pairs of these event mention representations, combines them with standard pairwise features (e.g., event type agreement, distance), and feeds them into a shallow network to produce a coreference score. A key claim is that this approach achieves state-of-the-art performance on the ACE 2005 dataset without relying on external semantic resources like WordNet or FrameNet. The authors also introduce a larger, newly annotated dataset called ACE++ and provide an ablation study and error analysis.

### Contributions

As I see them, the main contributions are:
1.  An empirical demonstration that a CNN-based architecture can learn effective features for event linking, achieving performance competitive with prior state-of-the-art systems that depend on extensive, hand-engineered features from external knowledge bases.
2.  The introduction of a new, larger dataset for the task (ACE++), and a demonstration that a larger quantity of training data significantly improves model recall.
3.  A detailed ablation and error analysis that provides valuable insights. The analysis effectively shows the contribution of each feature class and highlights key challenges in the task, such as pronoun resolution and annotation inconsistencies.

### Strengths

1.  **Effective Replacement of External Resources:** The paper's primary goal—to replace features from external knowledge bases with learned sentential features—is well-motivated and successfully executed. Achieving on-par results with a feature-rich system like Liu et al. (2014) is a significant accomplishment and points towards more portable and adaptable event linking systems.
2.  **Thorough Ablation Study:** The ablation study presented in Table 5 is commendable. It systematically dissects the model's components and clearly demonstrates that pairwise, trigger-local, and sentential features all provide complementary information and are crucial for the model's final performance. This analysis adds significant weight to the authors' design choices.
3.  **Insightful Error Analysis:** The error analysis is a strong point of the paper. Identifying difficult cases like event pronoun resolution provides concrete directions for future work. More importantly, the discussion of annotation inconsistencies and plausible system errors (22% of sampled false positives) is a crucial piece of analysis that helps to contextualize the performance of all systems on this task and suggests potential limitations of the dataset itself.

### Weaknesses

Despite its strengths, the paper is marred by a substantial number of methodological omissions, inconsistencies, and minor flaws that collectively undermine its quality and reproducibility.

1.  **Critical Information is Missing:** The paper is fundamentally incomplete. It refers to a "Figure 2" to describe the crucial example generation strategy and a "Table 2" containing the final hyperparameter settings. Neither of these is present in the submission. Without the hyperparameters, the model is not reproducible. Without the example generation algorithm, the experimental setup is not fully specified. This is a disqualifying flaw for a conference submission.
2.  **Vague and Poorly Structured Dataset Description:** The model design in Section 3 is motivated by a statistical analysis of the "ACE++" dataset, which is only formally introduced in Section 4. This forward referencing makes the paper difficult to follow. More critically, the description of ACE++, a claimed contribution, is entirely insufficient. The authors state it contains 1351 web documents "annotated in an analogous fashion" but provide zero details on the annotation process, the source of the documents, the annotation guidelines, the qualifications of the annotators, or the inter-annotator agreement. Without this information, the results on ACE++ are not scientifically valid, and the dataset itself cannot be considered a contribution.
3.  **Imprecise and Inconsistent Model Description:** The technical description of the coreference scoring component (Section 3.2) is ambiguous. Figure 1(b) correctly depicts two event mention representations being fed into the scoring module. However, Equation (4) takes a single, undefined vector `vsent+lex+pairw` as input. It is completely unclear how the two distinct event representations (`ve` for the antecedent and `ve` for the anaphor) are combined with the pairwise features to form this input vector. Is it concatenation, element-wise difference, or something else? This ambiguity concerns a core component of the model and is a major failure of technical writing.
4.  **Numerous Minor Technical and Formatting Errors:** The paper suffers from a "death by a thousand cuts" due to a lack of meticulous proofreading.
    *   The row numbering in Table 4 and Table 5 is incorrect (e.g., `)` instead of `1)`).
    *   The notation for a function mapping in Figure 3's caption is incorrect (`7→` instead of `→`).
    *   The term "Bagged distance" is used without definition.
    *   Footnote 2 contains a non-anonymized URL, breaking the double-blind review process.
    *   The use of an extremely outdated framework (TensorFlow v0.6) from 2015 raises questions about the currency of the work.
    *   The explanation in the ablation study that sentential features "degrade to trigger-local features" is pure speculation without any supporting analysis (e.g., visualizing filter weights).

### Questions to Authors

1.  Can you provide the missing Figure 2 (example generation) and Table 2 (hyperparameters)?
2.  Please provide a complete description of the ACE++ dataset's creation, including the source of documents, the full annotation guidelines, information about the annotators, and the final inter-annotator agreement scores (e.g., Cohen's kappa) for event mention and coreference annotation.
3.  Please clarify the precise mathematical formulation for combining the two event mention representations (`ve1`, `ve2`) and the pairwise features (`vpair`) into the single vector that serves as input to the similarity network in Equation (4).
4.  What exactly is meant by "Bagged distance"? Please specify how this feature is calculated.
5.  Your comparison to the state-of-the-art on ACE acknowledges that data splits differ between papers. Could you quantify how different your split is from that used by Liu et al. (2014) to help readers better interpret the comparison?

### Summary of the paper
This paper proposes a neural network model for within-document event coreference resolution (event linking). The model operates in two stages. First, it generates a fixed-size vector representation for each event mention using a convolutional neural network (CNN) that processes the entire sentence containing the mention. This representation combines sentential features derived from piecewise max-pooling over the convolution outputs with local lexical features from the trigger and its context. Second, it predicts coreference for a pair of event mentions by feeding their representations, along with standard pairwise features, into a feed-forward network that computes a similarity score, which is then passed to a logistic regression classifier. The authors evaluate their model on the ACE 2005 dataset and a newly created, larger dataset (ACE++), reporting on-par or state-of-the-art results without relying on external semantic resources like WordNet or FrameNet.

### Contributions
1.  **A new, larger dataset for event linking (ACE++):** The authors created a new dataset by adding 1351 annotated web documents to the 599 documents from ACE 2005. The experiments demonstrate that training on this larger dataset provides a significant performance boost.
2.  **Application of CNN-based sentential features to event linking:** While the CNN architecture itself is adapted from prior work on relation extraction, its application to event linking and the demonstration of its effectiveness in lieu of external knowledge bases is a valuable contribution.
3.  **A thorough empirical analysis:** The paper includes a strong ablation study isolating the impact of different feature classes (pairwise, local, sentential), an analysis of clustering strategies, and an insightful qualitative error analysis that identifies key challenges like pronoun resolution and annotation inconsistencies.

### Strengths
1.  **Strong Empirical Results without External Resources:** The primary goal of the paper is to achieve competitive performance on event linking without relying on costly, domain-specific external resources. The results on the ACE dataset, particularly the strong BLANC score, demonstrate that this goal is largely met. This is a significant practical advantage for porting the system to new domains.
2.  **Excellent Motivation and Analysis:** The authors provide strong motivation for their design choices through a preliminary data analysis on ACE++, which highlights the prevalence of coreferent pairs with different triggers and no argument overlap. The ablation study in Table 5 is exemplary, clearly demonstrating that each component of their feature set (pairwise, local, and sentential) contributes positively to the final performance.
3.  **Insightful Error Analysis:** The error analysis goes beyond simple categorization. The authors identify a key linguistic challenge (resolving pronouns that refer to events) and, more importantly, highlight fundamental ambiguities and potential inconsistencies in the task annotation itself. This adds valuable perspective on the performance limits of current systems and the nature of the task.
4.  **Creation of a New Resource:** Although details are sparse, the creation of the ACE++ dataset is a notable effort. The performance improvement shown in Table 4 validates the benefit of more training data for this task and for this class of models.

### Weaknesses
1.  **Critical Reproducibility Issues:** The paper suffers from several major obstacles to reproducibility.
    *   **The ACE++ dataset is not publicly available.** The paper states that "one of which [dataset] is publicly available," implying ACE++ is not. Furthermore, the paper provides no details on the source of the 1351 web documents, the annotation guidelines, the annotation process, or inter-annotator agreement. This makes the main results on ACE++ impossible to verify or build upon.
    *   **Hyperparameter details are missing.** The text refers to Table 2 for the final hyperparameter settings, but this table is not present in the paper. Critical details such as word embedding size (`dw`), position embedding size (`dp`), number of convolution filters (`dc`), hidden layer sizes (`de`, `dsim`), learning rate, and regularization are all absent.
    *   The model was implemented in TensorFlow v0.6, an extremely outdated version which may complicate attempts to replicate the work.

2.  **Ambiguity in Model Description:** The description of the model architecture lacks precision in key areas.
    *   In Section 3.2, the input to the similarity network in Equation (4) is given as `vsent+lex+pairw`, which is ill-defined for a pair of event mentions. The text should have explicitly stated that the input is a concatenation of the two event representations (e.g., `ve1` and `ve2`) and the pairwise features.
    *   The notation for the convolution operation in Equation (1) is ambiguous. The dimensions of the filter `wc` are given as `Rn∗dt`, and its application via a dot product `·` to a concatenated vector `vi:i+n−1t` is not formally clear.

3.  **Minor Inconsistencies and Formatting Errors:** The paper contains a number of small but distracting errors that detract from its overall quality.
    *   The introduction claims previous work shows "low performance," citing Sangeetha and Arock (2012). However, Table 3 reports a very high B-CUBED F1 of 87.7% for their system, which contradicts the initial claim.
    *   There are multiple typographical errors, such as "stateof-the-art" instead of "state-of-the-art" (Introduction), "enrichtment" instead of "enrichment" (Section 3), and an extra parenthesis in the first row of Table 4.
    *   The footnote for the example in Section 2 is placed after the entire multi-sentence block rather than directly associated with the sentence it modifies.

### Questions to Authors
1.  Could you please provide the full set of hyperparameters used for your final models, which appears to be missing from Table 2? This would include sizes for all embeddings and hidden layers, convolution window size, and Adam optimization parameters.
2.  Can you clarify the exact composition of the input vector to the similarity network in Equation (4)? Is it a concatenation of the two event representations `ve1`, `ve2`, and the pairwise feature vector?
3.  What are your plans regarding the release of the ACE++ dataset? Given the significant performance gains it provides, making it available to the community would be a major contribution. If it cannot be released, could you provide more details on the data source and annotation process?

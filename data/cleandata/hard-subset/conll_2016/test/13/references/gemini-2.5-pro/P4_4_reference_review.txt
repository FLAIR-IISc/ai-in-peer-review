### Summary and Contributions

This paper presents a neural network-based model for within-document event coreference resolution (event linking). The model operates in two stages. First, it generates a fixed-size vector representation for each event mention by applying a Convolutional Neural Network (CNN) to the full sentence containing the mention. This captures sentential context. This learned representation is combined with local lexical features from the event trigger. Second, for a pair of event mentions, their representations are concatenated with a set of standard pairwise features (e.g., event type agreement, distance) and fed into a feed-forward network to compute a coreference score. The model is evaluated on the ACE 2005 corpus and a new, larger dataset (ACE++), achieving performance comparable to the state of the art without relying on external semantic resources like WordNet.

The main contributions of this work are:
1.  **A resource-agnostic, high-performance model for event linking.** The primary contribution is the demonstration that a model using learned sentential features can achieve state-of-the-art results on a standard benchmark without requiring external, domain-specific semantic resources (e.g., WordNet, FrameNet). This significantly improves the portability of event linking systems to new domains where such resources are unavailable.
2.  **A new, larger dataset for event linking (ACE++).** The authors introduce and utilize an extended version of the ACE 2005 corpus, nearly tripling the number of documents. This is a valuable resource for the community, as the paper shows that the additional data provides a substantial performance improvement, which is particularly important for training data-intensive neural models.
3.  **A comprehensive empirical analysis.** The paper provides a thorough evaluation that includes an ablation study isolating the impact of different feature classes (sentential, local, pairwise), a comparison of clustering strategies, and a qualitative error analysis that identifies key remaining challenges for the task, such as pronominal anaphora.

### Strengths

1.  **Effective Application of Neural Methods to Reduce Feature Engineering.** The core strength of this work is its successful application of a CNN-based sentence encoder to the event linking task. While the CNN architecture itself is adapted from prior work in relation extraction, its use to create rich, data-driven representations of event mentions for coreference is well-motivated and effective. The results strongly support the central claim that this approach can replace the need for extensive feature engineering and reliance on external knowledge bases, which has been a significant bottleneck for previous systems. This is a solid step towards more robust and portable IE systems.
2.  **Thorough and Rigorous Evaluation.** The experimental validation is comprehensive and convincing. The authors compare their model against multiple state-of-the-art systems on the standard ACE 2005 benchmark (Table 3). Crucially, the ablation study (Table 5) clearly demonstrates that each component of the proposed model—pairwise, trigger-local, and sentential features—contributes positively to the final performance, justifying the overall model design. The inclusion of results on the larger ACE++ dataset, along with an analysis of different clustering methods (Table 4), adds depth to the empirical findings.
3.  **Clarity and Reproducibility.** The paper is well-written and easy to follow. The model architecture is explained clearly and supported by a helpful diagram (Figure 1). The authors provide sufficient detail on the experimental setup, including the hyperparameter search process and final parameter values (Table 2), which is crucial for reproducibility. The commitment to release the data splits for their new ACE++ test set further enhances the paper's contribution to the community.
4.  **Valuable Error Analysis.** The manual error analysis provides useful insights into the limitations of the current model and the remaining challenges of the task. Identifying pronominal anaphora and annotation inconsistencies as major sources of error is valuable for guiding future research directions. It also helps to contextualize the reported performance numbers by highlighting the inherent ambiguity and difficulty of the task.

### Weaknesses

1.  **Limited Architectural Novelty.** The neural component of the model, a CNN with piecewise max-pooling, is a direct application of an architecture previously proposed for relation classification (e.g., Zeng et al., 2015). While the application to event linking is new and the overall system design is sound, the work does not introduce novel neural network techniques. The primary contribution lies in the engineering and successful integration of this existing component into an event linking system, rather than in advancing neural network modeling itself.
2.  **Ambiguities in Comparison to State-of-the-Art.** The authors are transparent about the caveats in their comparison to previous work, namely the use of different data splits and preprocessing pipelines. However, these caveats weaken the claim of achieving state-of-the-art performance. For example, the model outperforms Liu et al. (2014) on BLANC and MUC metrics but underperforms on B-CUBED. Without a direct comparison on an identical setup, it is difficult to definitively conclude that the proposed model surpasses prior work, though it is clearly competitive.
3.  **Insufficient Detail on Handcrafted Features.** The paper does not provide sufficient implementation details for the pairwise features used in the second stage of the model. For instance, it is unclear how "Overlap in arguments" is quantified (e.g., a count, a Jaccard index, a binary indicator) or how the "distance between event mentions" is encoded. Since these features are concatenated with the learned neural representations, their impact could be substantial. This lack of detail hinders full reproducibility and makes it difficult to assess the relative importance of the learned features versus the handcrafted ones.

### Questions to Authors

1.  Could you please elaborate on the specific implementation of the pairwise features? For example, how is argument overlap measured and encoded? How is the distance between mentions (in sentences and in intermediate mentions) represented as a feature vector?
2.  The model combines learned neural features with handcrafted pairwise features. Have you experimented with a model that learns the similarity score in a more end-to-end fashion, for instance, by taking the two event representations (v_e) and learning a similarity function over them (e.g., via a bilinear model or by concatenating and feeding them through an MLP) without explicit pairwise features? This could help clarify how much performance is still dependent on traditional feature engineering.
3.  The error analysis correctly identifies pronominal anaphora as a major challenge, which is difficult for a sentence-level model to address. Do you have thoughts on how the proposed architecture could be extended to incorporate a wider discourse context beyond the current sentence to begin addressing these types of cross-sentence phenomena?

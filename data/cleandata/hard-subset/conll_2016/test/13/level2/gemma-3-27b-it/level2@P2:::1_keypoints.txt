- Paper proposes an event linking system using CNNs to generate sentential features, reducing reliance on external KBs.
- Achieves state-of-the-art performance on ACE and ACE++ datasets and includes an error analysis.

Contributions:
- Demonstrates competitive event linking using CNN-based sentential features without external semantic resources.
- Shows CNN approach can rival resource-heavy methods, enabling application to lower-resource domains.
- Provides a detailed error analysis highlighting challenging cases (e.g., pronoun resolution).

Strengths:
- Strong empirical results with SOTA performance on ACE and ACE++ and useful comparisons to prior work.
- Novel application of CNNs for sentential feature extraction in event linking, addressing reliance on external resources.
- Thorough experiments and ablation studies that clarify the contribution of feature classes and clustering strategies.
- Error analysis identifies key failure modes and offers concrete examples for future work.
- Clear, well-structured paper with an explanatory model diagram and reproducible experimental setup.

Weaknesses:
- Related work section is brief and could better situate the work among neural architectures like LSTMs and transformers.
- Datasets are limited in size (ACE is small); broader evaluation on larger/diverse datasets would strengthen conclusions.
- Hyperparameter search details are sparse, reducing reproducibility of model selection.
- Some design choices (CNN filter sizes, number of filters, square similarity function) lack systematic justification.
- Pronoun resolution is identified as a major issue but discussion of existing solutions and mitigation strategies is limited.

Questions for authors:
- What is the computational cost (training/inference) of CNN-based feature extraction versus using external KBs?
- Were alternative architectures (LSTMs, transformers) tried, and why was CNN preferred?
- How consistent were annotator judgments for examples flagged as annotation disagreements in the error analysis?
- Can you analyze specific cases where BestLink clustering failed compared to transitive closure?

Additional comments:
- The ACEvalid/ACEtest document list link is helpful.
- Discuss limitations of relying solely on sentential features for ambiguous or incomplete event descriptions.
- Improve table formatting (e.g., align decimal points) for readability.

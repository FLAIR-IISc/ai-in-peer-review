

Summary: The paper proposes a CNN-based model for event linking that replaces external semantic features (e.g., WordNet) with sentential features derived from convolutional networks. Primary contributions are: (1) a feature-agnostic event linking model achieving SOTA results, (2) empirical validation of sentential features' necessity via ablation, and (3) a publicly available ACE++ dataset. The model processes event mentions via two CNNs to generate sentential representations, augmented with lexical/pairwise features for coreference scoring.

Strengths:  
1. **SOTA performance without external resources**: On ACE, the model achieves 73.31 BLANC F1, outperforming Liu et al. (2014) by 2.87 points. Crucially, it avoids dependency on domain-specific semantic resources (e.g., WordNet), which is a significant practical advantage for cross-domain deployment.  
2. **Robust ablation study**: Table 5 convincingly demonstrates that sentential features (vs. lexical/pairwise) are the dominant contributor to performance (76.90 vs. 64.12 F1 when removed). The precision-recall trade-off when removing features (e.g., row 4) is well-analyzed and aligns with the paper’s hypothesis about trigger-word overlap.  
3. **Actionable error analysis**: The manual error analysis identifies core challenges (e.g., pronoun resolution, ambiguous annotations) and distinguishes model limitations from annotation ambiguities (17/100 false negatives matched false positives in annotation). This provides clear directions for future work.

Weaknesses:  
1. **Misleading SOTA comparison**: The paper claims SOTA on "two datasets," but results for Liu et al. (2014) are reported on ACE (599 docs), while their own model’s ACE++ results (1,950 docs) are compared against it. ACE++’s larger size (Table 1) likely contributes to their higher recall (83.21% vs. 75.16%), making the comparison invalid. The paper should re-evaluate on ACE alone or clarify the dataset discrepancy.  
2. **Unrealistic gold-trigger assumption**: The model uses *gold event mentions* (Section 5.1), while real pipelines have extraction errors. Liu et al. (2014) also used gold triggers, but the paper fails to address how their model would perform with noisy triggers—a critical gap for practical adoption. Without this, the "feature-agnostic" claim is overstated.  
3. **Unexplained ablation result**: Row 4 (pairwise + sentential features) shows higher precision (82.60) than row 3 (full model, 83.21 recall but 73.22 F1). The paper attributes this to "different utilization of sentential features," but provides no evidence or analysis. This undermines the ablation’s credibility and the model’s design rationale.

Questions to Authors:  
1. How would the model’s performance change if event triggers were predicted (not gold-annotated), as in standard IE pipelines?  
2. Why does removing lexical features (row 4) improve precision despite the paper’s emphasis on trigger-word overlap?  
3. Given the pronoun resolution failure (Section 6), could incorporating discourse-level features (e.g., coreference chains) mitigate this?

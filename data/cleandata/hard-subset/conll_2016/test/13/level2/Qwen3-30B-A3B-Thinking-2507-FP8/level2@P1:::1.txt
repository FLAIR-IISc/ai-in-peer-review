

Summary: This paper proposes a neural model for event coreference resolution that avoids reliance on external semantic resources (e.g., WordNet, FrameNet) by using convolutional neural networks to generate sentential features from event mentions' contexts. The main contributions are: (1) a resource-light approach for event linking using CNN-derived sentential features; (2) state-of-the-art performance on two datasets (ACE and ACE++); and (3) an error analysis identifying pronoun resolution as a key challenge. The authors demonstrate that sentential features alone, combined with lexical and pairwise features, can match or exceed prior methods that depend on external knowledge.

Strengths:
1. **Practical significance of resource-light approach**: The paper effectively demonstrates that high-performance event linking does not require domain-specific external knowledge bases, which is a major practical advantage for real-world deployment. The authors show their model outperforms Liu et al. (2014) (which uses 105 semantic features) while avoiding resource constraints, making it more adaptable to new domains.
2. **Rigorous ablation study**: The feature analysis (Table 5) clearly shows the incremental contribution of each component (pairwise, lexical, sentential features), with the full model achieving 76.90% BLANC F1 on ACE++. This provides strong evidence that the sentential features are not merely supplementary but essential for performance.
3. **Insightful error analysis**: The error analysis (Section 5.3) identifies meaningful limitations, particularly the model's failure on pronoun-based coreference (e.g., "It's crazy that we're bombing Iraq. It sickens me"). This goes beyond typical error reporting and directly informs future work, such as incorporating discourse structure.

Weaknesses:
1. **ACE++ dataset construction lacks detail**: The paper creates a new dataset (ACE++) by adding 1351 web documents, but provides no information on annotation methodology or inter-annotator agreement. This undermines reproducibility and raises questions about whether the improved results (Table 4) are due to better data or model enhancements. Clarify annotation guidelines and quality control.
2. **Comparison to Liu et al. (2014) lacks transparency**: The authors note that "the exact documents in the partitions vary from system to system," yet they do not specify whether their ACE split matches Liu et al.'s. Without identical splits, the performance comparison is less compelling. Provide the exact train/test document lists used.
3. **Underexplored failure mode for pronouns**: While the error analysis identifies pronoun resolution as a key challenge, it does not investigate *why* the model fails here. The sentential features should theoretically capture discourse context. Suggest exploring whether the model's CNN architecture (with fixed window size) struggles with long-distance pronoun references (e.g., 5+ sentences apart, as noted in Section 3).

Questions to Authors:
1. How were the 1351 web documents in ACE++ annotated? Please describe the annotation guidelines, training process, and inter-annotator agreement scores to ensure dataset quality.
2. Did you use the same train/test document splits for ACE as Liu et al. (2014) for a fair comparison? If not, how did you address the split discrepancy?
3. In the pronoun failure cases (e.g., "It sickens me"), does the model fail to recognize the pronoun's referent, or does it misjudge the event's semantic scope? Could discourse parsing features (e.g., coreference chains) improve this?

### Summary and Contributions

This paper presents a neural network model for event coreference resolution (event linking). The model represents individual event mentions using a combination of trigger-local lexical features and sentence-level features generated by a convolutional neural network (CNN). For a pair of event mentions, these representations are concatenated with standard pairwise features (e.g., event type match, distance) and fed into a feed-forward network that computes a coreference score. The authors' primary claim is that this approach, which learns feature representations from text, can achieve state-of-the-art performance without relying on external, domain-specific knowledge bases like WordNet or FrameNet. The model is evaluated on the standard ACE 2005 corpus and a larger, augmented version of it (ACE++).

The main contributions of this work are:
1.  **A novel application of CNNs for event linking.** The paper proposes a two-part neural architecture that first learns a dense representation for individual event mentions from their sentential context and then learns a similarity function over pairs of these representations. This demonstrates how modern deep learning techniques can be effectively applied to this specific structured prediction task.
2.  **State-of-the-art performance without external knowledge bases.** The most significant contribution is the empirical result showing that the proposed model performs on par with or better than previous state-of-the-art systems on the ACE 2005 benchmark. This is achieved without the extensive, hand-crafted semantic features derived from external resources that prior systems depend on, making the proposed model more portable and less dependent on domain-specific resources.
3.  **A thorough empirical analysis.** The authors provide a comprehensive evaluation, including an ablation study justifying the different components of their model, a comparison of different clustering strategies, and an insightful qualitative error analysis that highlights remaining challenges for the task, such as pronominal event anaphora.

### Strengths

1.  **High Significance and Practical Impact.** The paper addresses a key limitation of previous event coreference systems: their heavy reliance on external knowledge bases (WordNet, FrameNet, DBpedia, etc.). By successfully replacing these features with learned representations from a CNN, the authors present a more general and portable approach. This is a significant step forward, as it lowers the barrier to applying event linking systems to new domains where such curated resources may not exist.

2.  **Strong and Well-supported Empirical Results.** The model achieves state-of-the-art F1 scores on several metrics (BLANC, MUC) on the standard ACE 2005 dataset, outperforming a strong competitor that uses a "rich set of 105 semantic features". The experiments are well-designed, including ablations (Table 5) that clearly demonstrate the positive contribution of each feature class (pairwise, local, and sentential). The performance gains seen when moving from the small ACE dataset to the larger ACE++ dataset (Table 4) also effectively show the model's ability to learn from more data, which is characteristic of neural approaches.

3.  **Thorough and Insightful Analysis.** The paper goes beyond simply reporting top-line numbers. The ablation study provides a clear understanding of the model's components. The comparison between transitive closure and the "BestLink" clustering strategy is valuable, and the authors provide a plausible explanation for why their model benefits more from the former. Most importantly, the manual error analysis is excellent. It identifies challenging linguistic phenomena (e.g., pronoun resolution for events) and annotation ambiguities, which provides valuable context for the performance numbers and offers clear directions for future research.

### Weaknesses

1.  **Limited Architectural Novelty.** While the application of a CNN to this task is novel, the core CNN architecture (word/position embeddings, convolution, piecewise max-pooling) is largely adopted from prior work in relation extraction (e.g., Zeng et al., 2014, 2015). The paper could better articulate the specific architectural choices or adaptations that are crucial for *event linking* as opposed to other sentence-level IE tasks. As it stands, the primary novelty seems to be the application itself, rather than a new neural architecture.

2.  **Vague Motivation for Key Model Components.** The "distributed similarity" layer (Equation 4), which is central to the coreference decision, is not well motivated. The use of the `square` activation function is justified only by an "intuition" that "invariance under polarity changes is desirable." This is unclear. A more rigorous explanation or an empirical comparison to other standard interaction functions (e.g., element-wise product, subtraction) would strengthen the paper's technical contribution and provide a clearer understanding of how the model learns similarity.

3.  **Inconsistent Comparison with Prior Work.** The authors acknowledge that their SOTA comparison is complicated by different data splits and preprocessing pipelines. A particularly important difference is that this work uses gold-annotated event arguments, whereas a key competitor (Liu et al., 2014) uses predicted arguments from a semantic role labeler. This makes the comparison less direct, as the proposed model operates on cleaner input. The SOTA claim would be substantially stronger if the model were also evaluated in a more realistic setting with predicted event mentions and arguments to demonstrate its robustness to pipeline errors.

### Questions to Authors

1.  Could you elaborate on the choice of the `square` function in the similarity layer? Did you experiment with other methods for combining the two event mention representations and the pairwise features, such as element-wise multiplication of the event vectors, before the final logistic regression layer?

2.  The error analysis highlights that resolving pronominal event references (e.g., "It sickens me") is a major source of false negatives. This seems to require reasoning beyond the immediate sentence. Does your analysis give you any intuition on how a sentence-level CNN model like yours could be extended to capture this kind of discourse-level information?

3.  How do you anticipate the model's performance would change in an end-to-end setting where event triggers and arguments are predicted by an upstream system rather than taken from gold annotations? Do you believe the learned sentential features provide some robustness against errors in argument identification?

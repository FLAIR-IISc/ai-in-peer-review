### Summary of the paper and contributions

This paper proposes a neural network model for within-document event coreference resolution. The primary goal is to move away from systems that rely on rich, hand-crafted semantic features derived from external knowledge bases (e.g., WordNet, FrameNet), which limits their portability across domains. The proposed model consists of two main parts. First, for each event mention, a Convolutional Neural Network (CNN) processes the entire source sentence to generate a fixed-size vector representation capturing sentential context. This is combined with local lexical features (embeddings of the trigger word and its neighbors). Second, these representations for a pair of event mentions are concatenated with a small set of standard pairwise features and fed into a similarity learning network, which ultimately produces a coreference score. The authors evaluate their model on the standard ACE 2005 dataset and a newly created, larger dataset called ACE++. The model achieves state-of-the-art performance on ACE without using any external semantic resources.

The main contributions of this work are:
1.  **A successful application of learned sentential features to event linking.** The paper demonstrates that a CNN-based architecture can effectively learn representations for event mentions that are useful for coreference resolution, removing the need for features from external knowledge bases. This is a significant step towards more domain-independent event processing systems.
2.  **Achieving state-of-the-art performance on a standard benchmark.** The model is shown to be competitive with, and in some cases superior to, previous state-of-the-art systems on the ACE 2005 dataset, despite being less reliant on curated resources.
3.  **Introduction and analysis of a larger dataset (ACE++).** The authors create a larger dataset by annotating additional documents. Experiments on this dataset provide valuable insights into the model's ability to generalize and the impact of training data size on performance.

### Strengths

1.  **Well-motivated and impactful research direction.** The paper addresses a key limitation of many existing information extraction systems: their heavy dependence on domain-specific resources. The proposed approach of learning features directly from text is a compelling and important direction for the field. The strong results validate this approach and show its promise for building more robust and portable systems.
2.  **Thorough experimental evaluation.** The authors conduct a comprehensive set of experiments to validate their model. This includes comparison against multiple state-of-the-art systems on a standard benchmark, an ablation study that clearly demonstrates the utility of each feature class (pairwise, local, and sentential), an analysis of different clustering strategies, and an evaluation of the impact of increased training data.
3.  **Strong and convincing results.** The model achieves state-of-the-art F1 scores on several metrics (BLANC, MUC) on the ACE dataset. Achieving this level of performance without the "rich set of 105 semantic features" used by the prior state-of-the-art system (Liu et al., 2014) is a significant achievement and the paper's core message is well-supported by these results.
4.  **Insightful error analysis.** The manual analysis of system errors is valuable. It provides a qualitative understanding of the remaining challenges, such as resolving pronominal references to events and dealing with inherent annotation ambiguities. This not only helps contextualize the reported performance but also provides clear and useful directions for future work.

### Weaknesses

1.  **Architectural novelty.** While the application of a CNN to event linking is novel and effective, the CNN architecture itself (word + position embeddings, convolution, piecewise max-pooling) is largely adopted from prior work on relation classification, as the authors acknowledge. The paper could be strengthened by more explicitly positioning its architectural contribution, perhaps by discussing design choices specific to the event linking task that differ from the standard relation extraction setup.
2.  **Justification for model choice.** The paper focuses exclusively on a CNN for sentence encoding. Given the authors' finding that a majority of coreferent event pairs are more than five sentences apart, a discussion on why a CNN was preferred over architectures potentially better suited for long-range dependencies, such as Recurrent Neural Networks (LSTMs), would be beneficial. A brief justification would help readers understand the design choice.
3.  **Lack of detail on pairwise features.** The description of the pairwise features in Section 3.2 is brief and could hinder reproducibility. For instance, "Overlap in arguments" could be implemented in various ways (e.g., boolean flag, count of shared entities, Jaccard index over argument strings). Similarly, the meaning of "(Bagged) distance" is not entirely clear. Providing more implementation details for these features would improve the clarity of the model description.
4.  **Uncertainty about the new dataset.** The creation of the ACE++ dataset is a positive aspect of the work. However, the paper does not state whether this dataset will be publicly released. If the data and its annotations are not made available, the results on ACE++ are not reproducible, which limits the impact of this contribution for the research community.

### Questions to Authors

1.  Could you elaborate on your choice of a CNN architecture for generating sentential features? Did you consider alternative encoders like LSTMs, which might be better suited for capturing longer-range context that appears relevant for this task?
2.  Could you please provide more implementation details for the pairwise features, specifically the "Overlap in arguments" and "(Bagged) distance" features?
3.  Are there plans to make the newly annotated documents in the ACE++ dataset publicly available? A release would be a great service to the community and would allow for more direct comparisons in future work.

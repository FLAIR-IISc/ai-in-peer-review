**Summary of the Paper**

This paper presents a neural network model for within-document event coreference resolution. The central idea is to learn representations of event mentions that capture sentential context, thereby reducing the reliance on external, domain-specific semantic resources like WordNet or FrameNet, which have been a staple of previous systems. The proposed model has a two-part architecture. First, a convolutional neural network (CNN) with piecewise max-pooling generates a fixed-size vector representation for each event mention, incorporating both the trigger word and its surrounding sentence. Second, these representations for a pair of event mentions are combined with a small set of standard pairwise features and fed into a similarity function, which is learned by a small feed-forward network, to produce a final coreference score. The model is trained to make pairwise decisions, and a final clustering is produced by taking the transitive closure of all positive links. The work demonstrates that this approach achieves competitive, state-of-the-art performance on the standard ACE 2005 dataset and shows improved performance on a new, larger dataset (ACE++) introduced in the paper.

**Main Contributions**

1.  The primary contribution is an effective event coreference model that does not require external lexical-semantic knowledge bases. By leveraging sentential features learned via a CNN, the work demonstrates that performance comparable to resource-heavy state-of-the-art systems can be achieved, making the approach significantly more portable to new domains.
2.  A second key contribution is a thorough empirical evaluation and analysis. This includes the introduction and use of a larger dataset (ACE++), an ablation study that isolates the impact of different feature types (sentential, local, pairwise), and an insightful error analysis that identifies key remaining challenges for the task, such as pronominal event anaphora and annotation ambiguity.

**Strengths**

1.  **High Portability and Reduced Feature Engineering.** The model's main strength lies in its independence from external knowledge bases. Previous high-performing systems often rely on a large number of handcrafted semantic features derived from resources like WordNet, FrameNet, or DBpedia. This work successfully replaces this need with learned sentential representations, a significant step towards more general and easily adaptable event processing systems.
2.  **Strong Empirical Performance.** The model achieves results that are on par with or exceed the state of the art on the ACE 2005 benchmark. The comparison in Table 3 shows the model is highly competitive with systems that use a rich set of external features (e.g., Liu et al., 2014). This provides strong evidence for the viability of the proposed neural approach.
3.  **Comprehensive Experimental Analysis.** The paper provides a commendable level of analysis. The ablation study in Table 5 clearly shows that all feature components (pairwise, trigger-local, and sentential) contribute positively to the final performance, with the learned sentential features providing a substantial boost. Furthermore, the experiments on the larger ACE++ dataset demonstrate the model's ability to benefit from more training data, and the qualitative error analysis provides valuable insights into the limitations of current models and the inherent difficulties of the task.

**Weaknesses**

1.  **Limited Architectural Novelty.** While the application of CNNs to event coreference is novel and effective, the specific neural architecture for sentence representation (CNN with word/position embeddings and piecewise max-pooling) is heavily inspired by and closely follows prior work in related tasks like relation extraction (e.g., Zeng et al., 2014, 2015). The paper would be strengthened by more explicitly positioning its contribution as a novel and successful *application* of these established techniques, rather than a novel architecture in itself.
2.  **Lack of Clarity in Model Description.** The description of how the two event-mention representations are combined in the second part of the model (Section 3.2) could be clearer. The text refers to an input vector `vsent+lex+pairw` for the similarity function, but the preceding section defines `vsent+lex` as an intermediate representation for a *single* event mention. It is unclear how the representations for the two candidate mentions are integrated. Is the input a concatenation `[ve1; ve2; pairwise_features]`? Clarifying the exact composition of the input to the similarity layer would improve the paper's clarity and reproducibility.
3.  **Insufficient Detail on the ACE++ Dataset.** The introduction of a new, larger dataset is a valuable contribution. However, the paper provides limited details about its creation. Information regarding the source of the 1351 additional web documents, the specifics of the annotation process (e.g., guidelines, number of annotators), and inter-annotator agreement metrics would be essential for the community to properly assess the quality of this resource and the results obtained on it.

**Questions to Authors**

*   Could the paper clarify the exact composition of the input vector to the similarity function described in Section 3.2 and Figure 1(b)? Specifically, how are the learned representations for the two event mentions (`ve`) combined with the pairwise features before being processed by the similarity network?
*   Regarding the ACE++ dataset, could more information be provided on the source of the additional documents and the annotation procedure? In particular, reporting inter-annotator agreement scores would help contextualize the results. Is there a plan to make the annotations for this extended dataset publicly available?

- Paper proposes a two-part neural architecture: CNN-based sentence encoder with positional embeddings and piecewise max-pooling, plus a learned similarity module with simple pairwise features and transitive closure clustering.  
- Demonstrates state-of-the-art within-document event coreference on ACE2005 and ACE++ without external resources like WordNet/FrameNet.  
- Main contributions: CNN-derived sentential features matching/exceeding prior SOTA, a practical architecture with ablations clarifying sentential vs. pairwise utility, and a larger ACE++ dataset with empirical insights.  

Strengths:
- Strong empirical results with fewer external dependencies, improving portability across domains.  
- Clear, well-motivated model decomposition and sensible design choices (positional embeddings, piecewise pooling, square nonlinearity).  
- Thorough ablations and design comparisons showing sentential features substantially improve precision and recall and data size boosts recall.  
- Useful analysis of data properties (argument-sharing, trigger mismatches, long-distance links) motivating sentential modeling.  
- Error analysis identifies actionable weaknesses, notably pronoun-driven failures and borderline annotations.  

Weaknesses:
- Unclear which pairwise features (type, modality, argument overlap) use gold labels vs. predicted ones, potentially inflating performance and affecting pipeline robustness claims.  
- ACE++ lacks reported annotation guidelines, IAA, and public release details, making reproducibility and interpretation of gains difficult.  
- Limited baselines: missing strong non-neural/simple neural baselines and no statistical significance testing reported.  
- Metric coverage incomplete: CEAF and CoNLL-style averages are omitted, hindering standard comparison across coreference work.  
- Clustering is purely pairwise with transitive closure; lacks exploration of lightweight global clustering to control error propagation and merging/splitting trends.  

Questions for authors:
- Which features are gold vs. predicted at test time (event type coarse/fine, modality, argument roles/overlap)? If predicted, what tools/accuracies were used and how does performance change with predicted labels?  
- For ACE++: what are the annotation guidelines, domains, annotator training, IAA, and release plan/license (or can document IDs/conversion script be released)?  
- Is the 0.5 threshold fixed or tuned on dev; was global threshold optimization for BLANC or calibration analysis performed?  
- Are improvements over Liu et al. (2014) and ablations statistically significant (e.g., bootstrap resampling)?  
- Did you compare CNN sentential features to BiLSTM encoders or PCNN variants with explicit trigger-region masking; any per-type performance breakdowns?  

Suggestions:
- Report CEAF and CoNLL-style scores and provide exact ACE splits for reproducibility; consider releasing code and models.  
- Add an end-to-end setting with predicted triggers/types to quantify real-world robustness.  
- Expand error analysis with category counts and examples, and analyze transitive-closure error propagation.  
- Explore lightweight global clustering methods (best-first with merge constraints or learned linkage penalties) to better balance precision and recall.

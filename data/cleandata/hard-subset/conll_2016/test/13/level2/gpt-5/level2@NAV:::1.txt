Summary
The paper proposes a two-stage neural approach to within-document event coreference that replaces external semantic resources with sentential features learned by CNNs. The model encodes each event mention using sentence-level convolution and position embeddings, augments with trigger-local lexical features, and combines a pair of such representations with pairwise features via a learned similarity function to score coreference. It achieves state-of-the-art results on ACE and a larger ACE++ variant, includes ablations on feature classes and clustering strategies, and provides an error analysis.

Main contributions (as I see them)
- Demonstration that sentential CNN features, combined with lightweight pairwise features, can match or surpass resource-heavy baselines for event coreference on ACE.
- Systematic empirical study isolating the impact of sentential vs. lexical vs. pairwise features, and of clustering strategy (transitive closure vs. BestLink).
- Construction and evaluation on an expanded dataset (ACE++), with analysis of data properties and model behavior.

Strengths
- Strong empirical support: consistent improvements on BLANC and other metrics over prior work that relies on extensive external semantics; ablations clearly attribute gains to sentential features.
- Careful experimental design: reports multiple coreference metrics, analyzes data characteristics (argument overlap, trigger mismatch, distance), and motivates architectural choices (dual pooling, squared similarity).
- Practicality: avoids reliance on WordNet/FrameNet/KBs, improving portability to domains lacking curated resources.
- Transparent methodology: training details, hyperparameters, example generation strategy, and clustering procedure are spelled out; published splits for ACE add to reproducibility.

Weaknesses
- Use of gold event triggers (and partial gold arguments) reduces realism; end-to-end utility with predicted triggers is not assessed. Suggest reporting robustness with an off-the-shelf trigger detector.
- Comparability caveats on ACE (varying splits, preprocessing) are acknowledged but weaken the SOTA claim; statistical significance testing is missing. Please include significance tests and, if possible, results on commonly used splits.
- ACE++ availability is unclear; if not public, the key ablations and conclusions are harder to verify. Recommend releasing ACE++ annotations and code.
- Limited global inference: transitive closure clustering may propagate local errors; no learning-to-cluster or global consistency constraints are explored. Consider learned agglomeration or cluster-level features.
- Error analysis highlights pronominal/event anaphora but the model does not incorporate discourse/coref signals; incorporating document structure or coreference cues (e.g., entity coref, lexical chains) could address frequent failures.

Questions to authors
- Will ACE++ and code be released? If not, can you provide detailed guidelines to reproduce the additional annotations?
- How does performance change with predicted triggers and arguments?
- Did you test alternative similarity functions (e.g., cosine, bilinear) with significance checks?
- What is runtime and model size compared to feature-rich baselines?

- Paper tackles within-document event coreference with a two-stage neural model that avoids external KBs.  
- Stage 1: CNN with position embeddings, piecewise max-pooling around triggers, plus trigger-local lexical embeddings to get mention representations.  
- Stage 2: Pairwise classifier combines mention reps with hand-crafted pairwise features and a squared-nonlinearity similarity layer, clustered via transitive closure.  
- Evaluated on ACE 2005 and a larger ACE++ dataset, showing competitive or SOTA results without external semantic resources.  
- Ablation studies quantify contributions of sentential, lexical, and pairwise features; error analysis highlights pronoun-event coreference issues.  

Main contributions:
- Empirical finding: sentence-level CNN features + simple pairwise cues match or beat prior KB-based systems on ACE 2005 and scale with more data.  
- Architectural contribution: piecewise pooling around triggers and a squared-activation similarity function to capture multiple similarity facets.  
- Thorough experimental study: ablations, clustering comparisons, baselines, error analysis, and introduction of ACE++ for scalability claims.  

Strengths:
- Well-motivated sentential context modeling supported by data showing many coreferent pairs lack shared arguments or identical triggers.  
- Competitive performance achieved without external KBs, improving portability across domains.  
- Strong ablation and clustering analyses clarify feature contributions and clustering behavior.  
- Careful evaluation choices: prioritizes BLANC for imbalance but reports multiple metrics and specifies pair generation and splits.  
- Informative error analysis pointing to pronoun handling and annotation inconsistencies as major error sources.  

Weaknesses:
- Unclear whether argument overlap and modality/realis features use gold annotations, potentially making comparisons unfair.  
- Limited novelty: CNN encoder + pairwise classifier and transitive-closure clustering are incremental adaptations; squared activation is minor.  
- Evaluation lacks statistical rigor: non-identical data splits, no multiple random seeds or significance testing reported.  
- ACE++ lacks transparency: annotation protocol, quality control, inter-annotator agreement, and release status are unspecified.  
- Practical limitations: assumes gold triggers, lacks end-to-end evaluation with predicted upstream components, and does not model document-level discourse or pronoun-event coreference.  

Questions for authors:
- Do argument overlap and modality/realis features use gold annotations; can you report results without them?  
- What is the release status and annotation/documentation details for ACE++ (guidelines, IAA, domains)?  
- Were results averaged over multiple seeds and tested for statistical significance; please provide mean and variance.  
- Please provide the exact final hyperparameter values used (dw, dp, dc, de, dsim, n, regularization, dropout).  
- How does piecewise pooling handle multiple/adjacent/overlapping triggers in the same sentence; any ambiguity resolution?  

Additional comments:
- An end-to-end evaluation with predicted triggers/arguments would improve practical impact.  
- Comparing to recurrent encoders (e.g., BiLSTMs) or other similarity functions would validate the squared-activation choice.  
- Reporting runtime, complexity, and memory usage would clarify scalability.

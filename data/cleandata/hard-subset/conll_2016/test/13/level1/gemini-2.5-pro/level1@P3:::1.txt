**Summary of the paper**

This paper presents a neural network model for within-document event coreference resolution. The central thesis is that features learned automatically from the sentential context of an event mention can substitute for the rich, hand-crafted semantic features derived from external knowledge bases (e.g., WordNet, FrameNet) that prior state-of-the-art systems rely on. The proposed model operates in two stages. First, it uses a Convolutional Neural Network (CNN) with piecewise max-pooling to generate a fixed-size vector representation for each event mention, capturing both trigger-local lexical information and broader sentential context. Second, it feeds pairs of these event representations, augmented with a small set of standard pairwise features, into a feed-forward network that computes a coreference score. The authors evaluate their model on the standard ACE 2005 dataset and a new, larger dataset they created (ACE++), demonstrating that their approach achieves state-of-the-art performance without dependence on external semantic resources.

**Contributions**

1.  The primary contribution is the successful application of learned sentential features to the event coreference task, demonstrating that a neural model can achieve state-of-the-art performance while eliminating the reliance on external knowledge bases. This is a significant finding that points towards more portable and domain-adaptable event processing systems.
2.  A strong secondary contribution is the creation and evaluation on ACE++, a larger dataset for event coreference. The experiments clearly show the benefit of additional training data for this type of model and provide a valuable resource for future research in this area.
3.  The paper provides a thorough empirical evaluation, including a well-executed ablation study that isolates the impact of different feature classes (pairwise, local, and sentential), and a thoughtful error analysis that identifies key remaining challenges for the field, such as pronominal event anaphora.

**Strengths**

1.  **High Impact and Significance:** The work directly addresses a major bottleneck in traditional event extraction systems: their dependency on costly and domain-specific knowledge resources. By showing that representation learning can effectively replace this dependency, the paper makes a strong case for a new direction in the field and significantly improves the practicality of deploying event coreference systems in new domains.
2.  **Strong Empirical Results:** The model achieves performance on par with or exceeding previous state-of-the-art systems on the ACE 2005 benchmark. This is particularly impressive given that competing systems (e.g., Liu et al., 2014) leverage a vast number of engineered features from external resources. The results are presented clearly using multiple metrics, with a sound justification for preferring the BLANC score, demonstrating methodological rigor.
3.  **Thorough and Convincing Analysis:** The paper goes beyond simply presenting SOTA results. The ablation study in Table 5 provides convincing evidence that each component of the proposed feature set (pairwise, local, and especially the novel sentential features) contributes meaningfully to the final performance. Furthermore, the comparison of clustering strategies and the performance gains shown on the larger ACE++ dataset strengthen the paper's conclusions.
4.  **Clarity and Reproducibility:** The paper is well-written, and the model architecture is explained clearly with helpful diagrams. The authors' decision to release their data splits for the ACE dataset enhances the reproducibility of their work.

**Weaknesses**

1.  **Limited Architectural Novelty:** While the application is novel and impactful, the core neural architecture for generating sentence representations (a CNN with piecewise max-pooling) is largely adopted from prior work in relation classification (e.g., Zeng et al., 2014). The contribution lies more in the novel application and the resulting system design rather than in a new neural network architecture.
2.  **Lack of Insight into Learned Representations:** The paper convincingly demonstrates *that* the learned sentential features work, but offers little insight into *what* they represent. A qualitative analysis of the learned event representations (e.g., via visualizations or by examining filter activations) would have significantly strengthened the paper by helping to explain *why* the model is successful. For instance, what semantic aspects of the sentence context does the model learn are relevant for coreference? Similarly, the use of a `square` activation function in the similarity layer is an interesting but poorly justified choice; empirical comparison with more standard functions is needed.
3.  **Absence of Comparison with Alternative Encoders:** The choice of a CNN is motivated by its success in related tasks. However, at the time of this conference, recurrent architectures (LSTMs/GRUs) were also proving extremely effective for sentence modeling. The inclusion of a baseline using an LSTM-based sentence encoder would have provided a more complete picture and better contextualized the performance of the CNN, answering whether the specific convolutional structure is key or if any powerful sentence encoder would suffice.
4.  **Shallow Integration of Event Arguments:** The model primarily learns from the full sentential context and incorporates argument information only through a simple pairwise overlap feature. Given that argument compatibility is a critical cue for event coreference, the model could potentially be improved by a more structured integration of argument information into the event representation itself, for instance, through an attention mechanism over argument heads in the sentence.

**Questions to Authors**

1.  Could you provide more detail on the choice of the `square` non-linearity in the similarity layer (Eq. 4)? You state that it outperformed other functions, but could you provide the comparative results for more standard choices like `tanh` or `relu`? What is the intuition behind "invariance under polarity changes" being desirable for a learned similarity function?
2.  The model combines learned representations with a set of hand-crafted pairwise features. Have you explored a more end-to-end variant where similarity is computed directly from the event representations (e.g., from a concatenation of `v_e1`, `v_e2`, and their element-wise product/difference)? This would help to better understand the division of labor between the learned features and the engineered pairwise cues in the final classification decision.
3.  Your error analysis correctly identifies pronominal event anaphora (e.g., "It sickens me") as a major source of error. Since your model's scope is the immediate sentence of the trigger, it is naturally limited in handling such discourse-level phenomena. Do you have thoughts on how the proposed architecture could be extended to incorporate a wider document context to begin addressing this class of errors?

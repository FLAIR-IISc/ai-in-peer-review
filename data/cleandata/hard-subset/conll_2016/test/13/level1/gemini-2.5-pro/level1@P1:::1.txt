### Summary and Main Contributions

This paper introduces a neural network-based model for within-document event coreference resolution. The central idea is to avoid reliance on external knowledge bases (like WordNet or FrameNet), which limits the applicability of previous systems. The proposed model operates in two stages. First, it uses a Convolutional Neural Network (CNN) to generate a rich, fixed-size vector representation for each event mention by processing its full sentential context. This learned representation is augmented with local lexical features around the event trigger. Second, for any pair of event mentions, their representations are concatenated with a set of standard pairwise features and fed into another neural network that learns a distributed similarity function to produce a final coreference score. The authors evaluate their model on the standard ACE 2005 dataset and a new, larger dataset (ACE++) they created by adding more annotated documents.

The main contributions of this work are:
1.  **An effective event linking model independent of external knowledge bases.** The key contribution is demonstrating that a model can achieve state-of-the-art performance by learning latent features directly from text using a CNN, thereby replacing the need for hand-crafted semantic features from external resources. This significantly improves the model's potential for domain portability.
2.  **The introduction of a larger dataset for event linking.** By augmenting the widely used but small ACE 2005 corpus, the authors provide a more substantial resource (ACE++) for training and evaluating data-driven models like theirs. Their experiments show that model performance improves significantly with this additional data.
3.  **A comprehensive empirical evaluation and analysis.** The paper presents a solid evaluation, comparing the model against previous state-of-the-art systems, performing an ablation study to validate the contribution of each component of their model, and conducting a detailed error analysis that highlights remaining challenges for the task.

### Strengths

1.  **State-of-the-Art Performance without External Dependencies:** The primary strength of this work is its success in building a high-performing system that does not require external semantic resources. Table 3 shows that the proposed model achieves a BLANC F1 score of 73.31% on the ACE dataset, outperforming the feature-rich system of Liu et al. (2014) which scored 70.43%. This is a significant result, as it validates the hypothesis that learned sentential features can effectively substitute for curated knowledge, making the system far more adaptable to new domains where such resources are unavailable.
2.  **Well-Motivated Model Architecture:** The application of CNNs to generate sentential features for this specific task is novel and well-justified. The two-part architecture is logical: it first focuses on creating a high-quality representation of an individual event mention by considering both its local trigger context and its broader sentential context (via the CNN), and then learns to compare these representations for a coreference decision. The ablation study in Table 5 provides strong evidence for this design, showing that the combination of pairwise, local, and sentential features (row 3) outperforms models using only subsets of these features. The jump in BLANC F1 from 64.12% (pairwise + local) to 76.90% (pairwise + local + sentential) on ACE++ clearly demonstrates the critical contribution of the CNN-based features.
3.  **Thorough and Insightful Experimental Analysis:** The paper's evaluation is robust. Beyond the state-of-the-art comparison, the authors provide valuable analyses that strengthen their claims.
    *   The creation and use of the ACE++ dataset demonstrates the model's ability to leverage more data, boosting recall significantly (Table 4). This highlights the value of the approach in a more data-rich setting.
    *   The comparison between transitive closure and the BestLink clustering strategy provides practical insights into what works best for their model's output distribution.
    *   The error analysis is particularly useful, identifying concrete challenges like pronoun resolution and annotation inconsistencies. This transparency about the model's limitations provides a clear roadmap for future research in the area.

### Weaknesses

1.  **Insufficient Detail on the ACE++ Dataset:** A key contribution is the new ACE++ dataset, which is used for the majority of the analysis. However, the paper provides almost no information about its creation. Critical details are missing: What is the source of the 1351 additional documents? What was the annotation process and guidelines? What was the measured inter-annotator agreement? Without this information, it is difficult to assess the quality and characteristics of the dataset. The performance gains shown on ACE++ could potentially be influenced by the nature of the added data (e.g., if it contains simpler or more repetitive event coreference structures) rather than solely the model's ability to learn from more data. This lack of detail undermines the reproducibility and full interpretation of a significant portion of the paper's results.
2.  **Limited Architectural Novelty:** While the application of the architecture to event linking is new, the core CNN model for generating sentence representations is very similar to prior work in related tasks like relation classification (e.g., Zeng et al., 2014; Chen et al., 2015), which the authors acknowledge. The use of position embeddings and piecewise max-pooling are established techniques in this context. The primary novelty lies in the problem framing rather than in a new neural architecture. This slightly reduces the paper's contribution from a pure machine learning methods perspective.
3.  **Ambiguity in State-of-the-Art Comparison:** The authors are transparent about the caveats in their comparison to prior work (Table 3), namely differences in data splits and pre-processing pipelines (e.g., use of gold vs. predicted arguments). This makes the claim of achieving state-of-the-art performance less definitive. While re-implementing competitor systems is a high bar, a more direct comparison could have been made by implementing a strong, non-neural baseline (e.g., a feature-rich SVM or Random Forest classifier similar to Liu et al.'s but without external resources) on their exact data splits. This would have more clearly isolated the performance gain attributable to their neural approach versus a traditional feature-based one.

### Questions to Authors

1.  Could you please provide more details on the creation of the ACE++ dataset? Specifically, information on the source of the new documents, the annotation protocol, and any measures of inter-annotator agreement would be very helpful for evaluating the robustness of the results reported on it.
2.  The justification for using the `square` activation function in the similarity layer (Equation 4) is that "invariance under polarity changes is desirable." Could you please elaborate on this intuition? It is not immediately clear why ignoring the sign of the pre-activation would be beneficial for learning similarity. Did you empirically compare this to other common approaches, such as using a standard non-linearity (e.g., ReLU) on the concatenated event vectors along with their element-wise product and difference?
3.  The error analysis correctly identifies pronoun resolution involving events (e.g., "It sickens me") as a key failure case. Does your current model have any mechanism to handle pronominal mentions? Do you have thoughts on how the architecture could be extended to address this, for instance by incorporating features from an entity coreference system?

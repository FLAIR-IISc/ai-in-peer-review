**Summary of the paper**

This paper presents a neural network model for within-document event coreference resolution (event linking). The primary motivation is to move away from systems that rely on rich, external semantic resources (like WordNet or FrameNet), which can be domain-specific and limit portability. The proposed model consists of two parts. First, it uses a Convolutional Neural Network (CNN) with piecewise max-pooling to learn a dense, sentential representation for each event mention, capturing information from its surrounding context. This representation is augmented with local lexical features from the event trigger. Second, these representations for a pair of event mentions are combined with standard pairwise features (e.g., type agreement, distance) and fed into a similarity network, which uses a squared activation function, followed by a logistic regression layer to produce a final coreference score. The authors evaluate their model on the standard ACE 2005 dataset and a larger, proprietary dataset (ACE++), demonstrating that their approach achieves performance competitive with state-of-the-art systems without requiring external knowledge bases.

**Main contributions**

1.  The primary contribution is the successful application of learned sentential features from a CNN to the task of event linking. The work demonstrates that these features can effectively replace hand-engineered features derived from external semantic resources, achieving state-of-the-art performance while offering better domain portability.
2.  A secondary contribution is a thorough empirical analysis of the model's components and the task's challenges. This includes an ablation study that clearly shows the complementary value of pairwise, local, and sentential features, as well as an insightful error analysis that highlights difficult linguistic phenomena like event pronominalization and inherent annotation ambiguities.

**Strengths**

1.  **Strong Motivation and Impactful Result:** The paper addresses a well-known limitation of previous event linking systems: their dependency on external knowledge bases. By proposing a model that learns features directly from the text, the authors present a more general and portable solution. The fact that this approach achieves competitive, state-of-the-art results is a significant finding, providing strong evidence that learned sentential representations are sufficient for this complex reasoning task.
2.  **Thorough and Rigorous Evaluation:** The experimental setup is comprehensive. The authors compare their model against multiple strong baselines on the standard ACE benchmark. They further validate their model on a larger dataset (ACE++) to analyze its scaling properties. The ablation study in Table 5 is particularly convincing, clearly demonstrating that each of the feature classes (pairwise, local-lexical, and sentential) provides a distinct and important contribution to the model's overall performance.
3.  **Insightful Analysis:** The paper goes beyond simply reporting metrics. The comparison between transitive closure and the "BestLink" clustering strategy provides useful practical insight for future work. More importantly, the qualitative error analysis is excellent. It identifies concrete, hard-to-model phenomena (e.g., pronominal event coreference) and highlights the inherent ambiguity of the task, which helps to contextualize the performance of all systems and points towards clear directions for future research.
4.  **Clarity of the Model:** The model architecture is described clearly and concisely, with helpful figures and justifications for key design choices (e.g., piecewise max-pooling to handle multiple events per sentence, the squared activation for the similarity function). This makes the work easy to understand and follow.

**Weaknesses**

1.  **Limited Methodological Novelty:** While the application is novel and the results are strong, the core architectural components are not new. The use of a CNN with piecewise max-pooling for sentence-level representation has been well-established in related tasks like relation classification (e.g., Zeng et al., 2014; Chen et al., 2015). The main contribution is therefore more empirical and applicative than the proposal of a fundamentally new neural architecture.
2.  **Reliance on a Non-Public Dataset:** A substantial part of the paper's analysis—including the impact of data size, the full ablation study, and the baseline comparisons in Table 6—is conducted on the proprietary ACE++ dataset. While understandable, this makes these important findings impossible for the community to reproduce or directly build upon. This reduces the long-term impact of that portion of the work.
3.  **Slight Ambiguity in State-of-the-Art Claim:** The authors are transparent about the caveats in their comparison to prior work on ACE (differing splits, use of gold vs. system arguments). However, the results in Table 3 show a mixed picture when compared to Liu et al. (2014), with the proposed model being better on BLANC and MUC F1, but worse on B-CUBED F1. This makes the claim of achieving SOTA performance solid, but not definitively superior across all standard metrics. A deeper discussion on why the model might excel on certain metrics over others (e.g., perhaps BLANC's better handling of singletons) would strengthen this section.

**Questions to Authors**

1.  The use of a `square` activation function in the similarity layer is an interesting design choice. You mention it outperformed other model variants. Could you elaborate on what other approaches were tested (e.g., different non-linearities like `relu` or `tanh`, or alternative similarity functions like a bilinear model or simple cosine similarity on the event representations) and provide a sense of the performance gap?
2.  The error analysis correctly identifies event pronominal anaphora (e.g., "It sickens me") as a major source of false negatives. How are pronouns handled by the word embedding layer in your current model? Do you have any thoughts on how a CNN-based architecture, which is inherently local, could be modified to better capture these types of long-distance, abstract coreference links?
3.  To improve the reproducibility of this valuable work, would it be possible to release the list of document IDs used for your training, validation, and test splits for the standard ACE 2005 dataset? The paper already provides a link for the `valid`/`test` splits, which is excellent; confirming the `train` split would fully specify the setup.

**SUMMARY**
This paper presents a neural network model for within-document event coreference resolution. The primary goal is to achieve state-of-the-art performance without relying on external semantic knowledge bases like WordNet or FrameNet, which often limit the domain applicability of previous systems. The proposed model operates in two stages. First, it uses a Convolutional Neural Network (CNN) with piecewise max-pooling to generate a fixed-size vector representation for each event mention, capturing features from the entire sentence context. Second, it pairs these representations, augments them with standard pairwise features (e.g., event type agreement, distance), and feeds them into a trainable similarity function followed by a logistic regression layer to compute a coreference score. The model is evaluated on the ACE 2005 dataset and a larger, proprietary dataset (ACE++). Results show that the proposed approach achieves performance comparable or superior to previous state-of-the-art systems that use extensive external semantic features.

**MAIN CONTRIBUTIONS**
1.  **Effective Event Linking without External Knowledge Bases:** The main contribution is the demonstration that a neural model leveraging learned sentential features can achieve state-of-the-art performance on event linking. This is accomplished without using handcrafted features from external lexical-semantic resources (e.g., WordNet, FrameNet), thereby presenting a more portable and domain-adaptable approach to the task.
2.  **A Neural Architecture for Event Coreference:** The paper proposes a specific two-part neural architecture for the task. It first encodes individual event mentions using a CNN that models the sentential context, and then learns a similarity function over pairs of these encoded mentions. While the architectural components are inspired by prior work in related tasks, their application and combination for event coreference is a novel contribution.
3.  **Empirical Analysis and Ablation Study:** The paper provides a thorough empirical evaluation, including an ablation study that quantifies the contribution of different feature classes (sentential, trigger-local, and pairwise). The experiments on a larger dataset and the qualitative error analysis provide valuable insights into the model's behavior and remaining challenges in the field.

**STRENGTHS**
1.  **Strong Empirical Performance:** The model achieves state-of-the-art F1 scores on the standard ACE 2005 benchmark across multiple metrics (BLANC, MUC, positive-links) when compared to previously published systems. The performance is particularly noteworthy given that the model forgoes the complex, resource-heavy features used by its main competitors.
2.  **Improved Generality and Portability:** By successfully replacing features from external knowledge bases with learned sentential features, the proposed approach is inherently more generalizable. It reduces the significant burden of requiring domain-specific resources, making it more readily applicable to new domains or languages where such resources are unavailable.
3.  **Clarity and Methodological Soundness:** The paper is well-written and clearly describes the model architecture, training procedure, and experimental setup. The inclusion of diagrams, detailed hyperparameter settings (Table 2), and explicit algorithms for example generation and clustering (Figures 2 and 3) contributes to the technical soundness and potential reproducibility of the work.
4.  **Thorough Evaluation and Analysis:** The authors conduct a comprehensive set of experiments beyond a simple SOTA comparison. The ablation study in Table 5 effectively demonstrates the utility of the proposed sentential features. Furthermore, the analysis of different clustering strategies and the impact of training data size provides useful context, and the manual error analysis offers valuable qualitative insights into remaining hard cases for event coreference, such as pronoun resolution.

**WEAKNESSES**
1.  **Incremental Architectural Novelty:** The core neural architecture for generating sentence representations (CNN with word and position embeddings, followed by max-pooling) is heavily based on established methods from prior work in relation classification. The paper acknowledges its motivation from this literature, but the architectural contribution itself is an application of existing techniques rather than a fundamental innovation.
2.  **Reliance on a Non-Public Dataset for Key Analyses:** Several important experiments, including the feature ablation study (Table 5) and the analysis of data size impact (Table 4), are conducted on the proprietary ACE++ dataset. While the results are insightful, the inability for the community to access this dataset hinders the direct verification and reproduction of these specific findings.
3.  **Potentially Unfair Comparison Conditions:** The authors rightly note that direct comparison to prior work is complicated by variations in dataset splits and preprocessing pipelines (e.g., use of gold vs. predicted arguments). While this is a common challenge, it adds a degree of uncertainty to the claims of outperforming the state of the art. The paper does not explicitly state whether its "argument overlap" feature uses gold-standard or system-predicted arguments, which is a key detail for assessing the model's dependencies.
4.  **Limited Exploration of Baselines for Sentential Features:** The ablation study demonstrates that adding sentential CNN features improves performance over a model without them. However, it does not compare the CNN-based representation to simpler, non-convolutional methods for encoding the sentence (e.g., an average of word embeddings in the sentence context). Such a comparison would more precisely isolate the contribution of the convolutional structure in capturing relevant semantic information.

**QUESTIONS TO AUTHORS**
1.  Regarding the pairwise "Overlap in arguments" feature: are the event arguments used to compute this feature taken from the gold-standard annotations, or are they the output of a separate argument extraction system? Clarification would help in understanding the full requirements of the model in a real-world pipeline.
2.  The ablation study (Table 5, Row 4) shows that using only pairwise and sentential features leads to high precision but very low recall. The paper suggests the model may adapt to focus more on the trigger word. Have you performed any qualitative analysis of the learned filters or representations to support this hypothesis?
3.  The piecewise max-pooling is motivated as a way to "handle multiple event mentions per sentence." Could you elaborate on this mechanism? Does the model process a sentence multiple times, once for each event trigger it contains, with the "left" and "right" pieces defined relative to the current trigger of interest?

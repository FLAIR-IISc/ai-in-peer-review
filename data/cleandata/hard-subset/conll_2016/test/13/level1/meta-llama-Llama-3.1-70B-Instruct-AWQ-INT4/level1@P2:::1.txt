**Summary of the Paper**

The paper proposes a model for event linking, which is a crucial task in information extraction. The model uses sentential features generated by convolutional neural networks (CNNs) to represent event mentions, and then uses these features to determine coreference between event mentions. The model is trained on two datasets, ACE and ACE++, and achieves state-of-the-art performance on both datasets.

**Main Contributions**

1. The use of sentential features generated by CNNs to represent event mentions, which allows the model to capture contextual information and improve performance.
2. The proposal of a two-step architecture for event linking, which first generates event-mention representations and then uses these representations to determine coreference.
3. The achievement of state-of-the-art performance on two datasets, ACE and ACE++, without relying on external feature sources.

**Strengths**

1. The use of sentential features generated by CNNs is a novel approach that allows the model to capture contextual information and improve performance.
2. The two-step architecture is well-designed and allows the model to effectively use the generated event-mention representations to determine coreference.
3. The model achieves state-of-the-art performance on two datasets, ACE and ACE++, which demonstrates its effectiveness.
4. The paper provides a thorough analysis of the model's performance, including an error analysis that highlights the challenges of the task.

**Weaknesses**

1. The model relies heavily on the quality of the generated event-mention representations, which may not always be accurate.
2. The model does not use external feature sources, which may limit its performance on certain datasets or domains.
3. The paper does not provide a detailed comparison with other state-of-the-art models, which makes it difficult to evaluate the model's performance in the context of other approaches.
4. The error analysis highlights the challenges of the task, but does not provide a clear solution to address these challenges.

**Questions to Authors**

1. How do you plan to address the challenges of pronoun resolution, which is a common source of errors in event linking?
2. Can you provide more details on how the model handles cases where the event mentions are not in the same sentence or paragraph?
3. How do you plan to extend the model to handle cross-document event linking, which is a more challenging task?
4. Can you provide more information on the hyperparameter tuning process and how you selected the optimal hyperparameters for the model?

**Additional Comments**

The paper is well-written and provides a clear explanation of the model and its performance. The use of sentential features generated by CNNs is a novel approach that shows promise for event linking. However, the model's reliance on the quality of the generated event-mention representations and its lack of use of external feature sources may limit its performance on certain datasets or domains. Overall, the paper is a valuable contribution to the field of information extraction and event linking.

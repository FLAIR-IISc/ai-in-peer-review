This paper builds a model for event linking using CNN. Based on the given event mention, the authors generate a vector representation based on word embedding passed through CNN, and then perform max-pooling. They combine the resulting representation with multiple word embeddings around the mention. With some pairwise features, they generate a vector of similarities using a single-layer neural network, and calculate a confidence score. The model has been tested on the ACE dataset and an extended version with similar performance to previous feature-enriched systems.

In my opinion, the paper's main contribution is developing a neural approach for entity linking that combines word embedding with multiple linguistic features. It is interesting to find that using word embedding alone is not enough for good performance. Fortunately, the linguistic features used are limited and do not require manually prepared external resources.

Experimental Setting
- It seems that gold trigger words are used instead of predicted words. The authors give a rationale as to why this is appropriate, although I would still prefer to see performance with predicted triggers. This is particularly problematic because one of the competing systems used used predicted triggers, so the comparison is not fair.
- The fact that different papers use different train/test splits is concerning. I would encourage the authors to stick to the previous split as much as possible.

Suspect Points
- The numbers indicating the need for cross-sentential information are reassuring. However, the last statement in the second paragraph (lines 65-70) was not clear to me.
- It is said that embeddings for words should be generated "in the same way as word embeddings." Exactly how? Are they randomly initialized? Are they lexicallyized? It is not clear to me why there should be a word-like embedding for a relative position next to a different word next to a word.
- How are left versus right neighbors used to create representations (lines 307-311)? Does this only affect the max-pooling operation?
- A word's word embedding before and after the trigger words is added to it. This seems a bit arbitrary. Why a word before and after and why no other option?
- It is not clear how the event-mention representation v_e (line 330) is used. Only v{sent+lex} is used in the following sections, not ve.
- How are pairwise features used in section 3.2? Most features are binary, so I assume they are encoded as binary vectors, but what about, for example, the distance feature? And, are these kept constant during training?

Other Issues and Suggestions
- Can this approach also be applied to entity confidence resolution? This would allow for comparison with more recent work and popular datasets like OntoNotes.
- The use of a square function as non-linearity is interesting. Is this new? Do you think it has applicability in other tasks?
- Dataset: A dataset is publicly available, but the results are also presented with ACE++, which is not. Are you planning to release it? This would help other researchers compare new methods. At the very least, it would be good to see a comparison of the feature-enriched system on this dataset.
- Results: Some of the numbers stated in the results are quite close. A significance test would help validate the comparisons.
- Related Work: Among the work on (entity) confidence resolution, one can refer to the neural network approach by Wiseman et al. (2015).

Minor Issues
- Line 143, "that" is unnecessary.
- Table 6 refers to a baseline as "same type," but the text says "same event" (line 670).

References

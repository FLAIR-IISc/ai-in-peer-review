This research paper presents a model for event entity linking, where they propose using Sentinel features from CNN instead of external knowledge sources, as was used in previous methods. They train a two-part model: the first part learns to represent an event mention, and the second part learns to compute a confidence score given two event entity mentions.

The paper is well-written, well-presented, and easy to understand. I like the analysis done on the ACE corpus regarding the shared logic between event confidence. Furthermore, the analysis on the impact of dataset size is a great inspiration for building their ACE++ dataset. However, there are some key issues that need attention:

- The authors fail to motivate and analyze the pros and cons of using CNN to generate mention representation. There is no discussion about why they chose CNN, and no comparison to other models (e.g., directly an RNN). Given that their model only improves the F1 score by 2 or 3 points according to various metrics compared to state-of-the-art models, there should be more evidence that this architecture is actually better.
- It is not clear what is new in the idea of dealing with event linking with Sentinel features, given that using CNN for classification tasks in this way is not new. The authors could have clearly pointed out and primarily compared any existing continuous location methods for event linking. The selection of methods in Table 3 is not thorough enough.
- There is no information on how the ACE++ dataset is collected. A major problem with the ACE dataset is the limited number of event types, which makes it very constrained and biased. It is important to know which event types ACE++ covers. This could also help support the claim made in Section 5.1 that 'other approaches are strongly bound to the domains where these semantic features are available... our approach does not rely on restricted resources...', you need to show that earlier methods fail on some datasets that you succeed on. Also, to enable any meaningful comparison in the future, the authors should think about making this dataset publicly available.

Some minor issues:

- I would also like to see the performance of your model without gold references in Table 3.
- It would be good to find out how this model can be enhanced with a vanilla confidence resolution system. For specific examples in line 687, off-the-shelf convolutional neural network systems easily combine 'it' with 'bombing', which can somehow be used in the event entity linking baseline.
- Given the relatively small size of the ACE dataset, I think there is a need to test on other available resources as well for a compelling model. This further motivates working on unit and event confidence simultaneously. I also believe that it is necessary to test on EventCorefBank in parallel with ACE.
- Table 5 shows that pair features have been quite effective, indicating that feature engineering may still be important for a competitive model (at least on the scale of the ACE dataset). One might wonder which features were most effective, and why it was not explained how the current set was chosen and what other efforts were made.

This paper presents a model for event entity linking, where they propose using sentence structure features from CNN instead of previously used external knowledge sources. They train a two-part model: the first part learns an event mention representation, and the second part learns to calculate a coreference score based on two event entity mentions.

The paper is well-written, beautifully presented, and easy to follow. I particularly liked the analysis on the ACE corpus regarding argument sharing within event coreference. Furthermore, the analysis of the impact of dataset size is a major inspiration for building their ACE++ dataset. However, there are several key issues that need to be addressed:

- The authors failed to analyze the advantages and disadvantages of using CNN to create mention representations. They did not discuss why they chose CNN and did not make any comparisons with other models (e.g., a simple RNN). Since their model only improves the F1 score by 2 or 3 points according to various metrics compared to state-of-the-art models, more evidence is needed to show that this architecture is truly superior.
- It is not clear what is new about the concept of event linking with sentence structure features, as it is not new to use CNN to classify in this method. The authors could have clearly compared any existing continuous space methods for event linking. The choice of methods in Table 3 is not thorough enough.
- There is no information on how the ACE++ dataset is collected. A major problem with the ACE dataset is its limited number of event types, which makes it very limited and biased. It is important to know what event types ACE++ covers. This could also help support the claim made in Section 5.1 that 'other methods are strongly tied to the domain where these semantic features are available... our method does not depend on limited resources', you need to show that previous methods failed on some datasets where you were successful. Also, to enable any meaningful comparison in the future, the authors should consider making this dataset publicly available.

Some minor issues:

- I would have liked to see the performance of your model without the gold reference in Table 3.
- It would be good to explore how this model can be enhanced or not enhanced with a vanilla coreference resolution system. For specific examples in line 687, the off-the-shelf CoreNLP system easily links 'It' to 'bombing', which could somehow be used as the basis for an event entity linking.
- Considering the relatively small size of the ACE dataset, I think it is necessary to also test for other available resources for an interesting model. This encourages further work on entity and event coreference together. I also believe that it is essential to test in the EventCorefBank in parallel with ACE.
- Table 5 shows that the paired features have been quite effective, which indicates that feature engineering may still be important for a competitive model (at least on the scale of the ACE dataset). One might wonder which features were most effective and why the current set was chosen and what else was tried.

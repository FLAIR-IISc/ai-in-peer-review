This paper presents a model for event entity linking, proposing the use of sentential features generated by a Convolutional Neural Network (CNN) as an alternative to the external knowledge sources used by previous methods. The authors train a two-part model: the first part learns a representation for each event mention, and the second part calculates a coreference score between pairs of these representations.

The paper is well-written, well-presented, and easy to follow. The analysis of the ACE corpus, particularly regarding argument sharing between coreferent events, is insightful. Furthermore, the analysis of the impact of dataset size provides a strong motivation for the creation of their proposed ACE++ dataset.

Despite these strengths, there are several major issues that need to be addressed:

*   **Motivation for CNN Architecture:** The paper fails to motivate the choice of a CNN for generating mention representations or analyze its pros and cons. It is not explained why a CNN was chosen over other architectures, and no comparison is made to alternatives (e.g., an RNN, which might seem a more straightforward choice for sequential data). Given that the model's F1 score improvement over the state-of-the-art is modest (2-3 points), more substantial evidence is required to demonstrate that this specific architecture is superior.

*   **Novelty and Baselines:** The novelty of using sentential features for event linking is not clearly articulated, especially since using CNNs in this manner for classification tasks is not new. The authors should explicitly position their work relative to other continuous space methods for event linking. The selection of baseline methods in Table 3 is not sufficiently thorough and should be expanded to include more relevant recent work in this area.

*   **ACE++ Dataset Details:** The paper provides no information about the collection methodology for the new ACE++ dataset. A well-known issue with the original ACE dataset is its limited number of event types, which can lead to biased models. It is crucial to know what event types ACE++ covers and how it addresses this limitation. This information is also necessary to support the claim in Section 5.1 that "other approaches are strongly tied to the domain where these semantic features are available…our approach does not depend on resources with restricted…". To substantiate this claim, the authors must show that their method succeeds on a dataset where resource-dependent methods fail. Finally, to enable meaningful comparisons in future work, the authors should plan to make this dataset publicly available.

Some minor issues and suggestions for improvement:

*   The evaluation in Table 3 would be more comprehensive if it also included the performance of the proposed model without using gold mentions.

*   It would be valuable to explore whether this model can be augmented with a standard coreference resolution system. For the example in line 687, an off-the-shelf system like CoreNLP readily links "It" to "bombing." This information could potentially be leveraged to create a stronger baseline.

*   Given the relatively small size of the ACE dataset, the model's claims would be significantly strengthened by testing on other available resources, such as EventCorefBank. This could also further motivate the simultaneous modeling of entity and event coreference.

*   Table 5 shows that pairwise features are quite effective. This suggests that feature engineering may still be crucial for building a competitive model, at least on the scale of the ACE dataset. The paper would benefit from an analysis of these features, detailing which were most effective and why the current set was chosen over other alternatives that were considered.

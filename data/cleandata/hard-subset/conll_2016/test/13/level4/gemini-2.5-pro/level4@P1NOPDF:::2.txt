This paper presents a two-part model for event entity linking. The proposed method uses a Convolutional Neural Network (CNN) to generate sentential features for event mention representations, intended as a replacement for the external knowledge sources relied upon by previous work. The first part of the model learns an event mention representation, while the second part calculates a coreference score between two given mentions.

The paper is well-written, clearly presented, and easy to follow. The analysis of argument sharing between coreferent events in the ACE corpus is particularly insightful. Furthermore, the discussion of the dataset's size limitations provides a strong motivation for the creation of the authors' new ACE++ dataset. However, there are several major issues that need to be addressed:

-   A significant weakness is the lack of motivation for using a CNN to generate mention representations. The paper does not justify the choice of a CNN over other architectures, such as an RNN, nor does it provide an analysis of the relative pros and cons. Given that the reported F1 score improvements over state-of-the-art systems are modest (2-3 points), a more thorough justification is needed to demonstrate the superiority of this specific architecture.

-   The paper's novelty is not clearly articulated. The use of CNNs to extract features for classification tasks is well-established, so the contribution of applying this to event linking with sentential features needs to be more explicitly defined. To this end, the authors should position their work more clearly against existing continuous space methods for event linking. The selection of baseline methods in Table 3 is insufficient and should be expanded to include more relevant contemporary approaches.

-   The paper provides no details on the creation of the new ACE++ dataset. A key limitation of the original ACE dataset is its small number of event types, which can lead to biased models. It is crucial for the authors to describe the collection methodology and specify the event types covered in ACE++. This information is also necessary to substantiate the claim in Section 5.1 that the proposed approach is more domain-independent than methods relying on "resources with restricted" availability. To validate this claim, the authors should demonstrate that their method succeeds on a dataset where resource-heavy approaches might fail. Finally, to enable meaningful future comparisons and contribute to the community, the authors are strongly encouraged to make the ACE++ dataset publicly available.

Some minor issues:

-   The paper would be strengthened by including the model's performance without gold references in Table 3 to provide a more realistic evaluation.

-   It would be beneficial to explore how this model could be augmented with a standard coreference resolution system. For the example in line 687, an off-the-shelf system like CoreNLP readily links "It" to "bombing." This information could potentially be leveraged in a baseline model for event entity linking.

-   Given the small size of the ACE dataset, evaluation on other available resources is necessary for a more compelling demonstration of the model's effectiveness. This concern further motivates work on joint entity and event coreference. Testing on EventCorefBank in parallel with ACE would be essential.

-   Table 5 shows that pairwise features are highly effective, which suggests that feature engineering remains crucial for achieving competitive performance, at least on the scale of the ACE dataset. The paper should provide an analysis of which features were most effective and describe the process by which the current feature set was selected over other alternatives.

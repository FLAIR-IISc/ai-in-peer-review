This paper introduces a model for event entity linking that replaces the external knowledge sources used in prior work with CNN-based sentence-level features. The authors train a two-stage architecture: the first module learns a representation for an event mention, and the second computes a coreference score for a pair of event mentions.

The paper is clearly written, well-structured, and easy to follow. I particularly appreciate the ACE corpus analysis on argument sharing across event coreferences. The investigation into dataset size effects is also convincing and provides a strong rationale for constructing the ACE++ dataset. That said, there are several major concerns that should be addressed:

- The motivation and trade-offs of using CNNs for mention representations are not adequately discussed. The choice of CNNs is not justified, nor are there comparisons to alternative architectures (e.g., a straightforward RNN). Given the modest gains over the state of the art—only 2–3 F1 points across metrics—stronger evidence is needed to show that this architecture is truly preferable.

- The novelty of addressing event linking with sentential features is unclear, since applying CNNs in this manner for classification is not new. The authors should explicitly identify and primarily compare against existing continuous-space approaches for event linking. The method selection in Table 3 is not comprehensive enough.

- The process for constructing the ACE++ dataset is not described. A key limitation of ACE is its small set of event types, which makes it constrained and potentially biased. It is important to know which event types ACE++ includes. This would also help substantiate the claim in Section 5.1 that “other approaches are strongly tied to the domain where these semantic features are available… our approach does not depend on resources with restricted…”. You should demonstrate that earlier methods underperform on a dataset where your approach succeeds. For meaningful future comparisons, the dataset should also be made publicly available.

Some minor issues:
- I would have liked to see results for your model without gold references in Table 3 as well.
- It would be useful to explore how this model could be combined with a vanilla coreference system. In the example on line 687, an out-of-the-box CoreNLP pipeline already links “It” to “bombing,” which could be leveraged in an event entity linking baseline.
- Given the relatively small size of ACE, a compelling model should also be evaluated on other available resources. This further motivates joint work on entity and event coreference. In particular, evaluating on EventCorefBank alongside ACE seems essential.
- Table 5 indicates that pairwise features contribute substantially, suggesting that feature engineering may still be important for competitive performance (at least at ACE’s scale). It would be helpful to report which features were most impactful, how the final set was selected, and what alternatives were tried.

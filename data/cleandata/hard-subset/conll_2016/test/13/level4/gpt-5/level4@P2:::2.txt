Summary and contributions
- The paper addresses within-document event coreference (event linking) using sentential features learned by a CNN in place of external semantic resources. The model has two parts: (1) a CNN-based encoder that builds an event-mention representation from the sentence (with positional and trigger-local information), and (2) a pairwise module that combines two mention representations with hand-crafted pairwise features to score coreference, followed by clustering.
- As I see it, the main contributions are:
  1) An empirical demonstration that a model without external knowledge bases can achieve state-of-the-art performance on ACE and strong results on an expanded dataset (ACE++).
  2) An ablation and design study showing the impact of sentential, trigger-local, and pairwise features, as well as analyses of dataset size and clustering strategies.
  3) Task diagnostics: corpus statistics on argument overlap, trigger mismatch, and mention distance, plus an error analysis that highlights pronoun-related challenges. The construction of ACE++ is potentially valuable if documented and released.

Strengths
- Clear and well-structured presentation. The model description and figures are easy to follow, and the paper is generally well written. The corpus analysis on ACE (argument overlap, differing triggers, long-distance links) convincingly motivates sentence-level features.
- Competitive results without external resources. Achieving state-of-the-art or near state-of-the-art performance using no WordNet/FrameNet/VerbOcean-type features is a meaningful practical contribution, suggesting better portability to domains lacking such resources.
- Solid experimental coverage. The paper reports multiple evaluation metrics, ablations (Table 5), an analysis of data scale (ACE→ACE++), and a comparison of clustering strategies (transitive closure vs. BestLink), which helps interpret where the gains come from.
- Useful error analysis. The discussion of false positives/negatives (especially pronoun-linked events) provides concrete directions for future work and clarifies the limits of sentence-only representations.

Weaknesses
- Insufficient justification for the architecture. The choice of CNNs for event-mention encoding is not motivated or compared against natural alternatives (e.g., BiLSTMs, attention-based models, tree-structured encoders). Given that absolute improvements over prior work are modest (≈2–3 F1 on several metrics), the paper should provide stronger evidence that this architecture is preferable, either via direct comparisons or analysis (e.g., efficiency, stability, data efficiency).
- Limited novelty and positioning. Using CNN-based sentential features for classification is established in prior IE work. The paper should clarify what is novel about applying them to event coreference specifically, and compare against prior continuous-space/neural approaches for event linking if any exist. The competitor set in Table 3 is not sufficiently broad to establish novelty or superiority.
- Incomplete description of ACE++. There is no detail on how ACE++ was collected, which event types it covers, annotation guidelines/quality, or whether it will be available. Since a major claim is independence from external semantic resources and better domain portability, the paper should show that competing approaches relying on such resources degrade on ACE++ (while the proposed model remains strong), and provide enough detail to enable future comparisons.
- Reliance on gold annotations. Table 3 appears to use gold triggers (and possibly arguments). Without results using predicted mentions/arguments, it is hard to assess end-to-end usefulness or robustness. This also weakens the claim of practical portability.
- Pairwise features appear critical, with limited analysis. Table 5 suggests pairwise features contribute substantially. This raises questions about how much of the gain is learned vs. engineered. The paper does not report which pairwise features matter most, how they were selected, or what alternatives were tried.

Questions to authors (for rebuttal)
- Why CNNs? Please compare against at least one strong neural alternative for mention encoding (e.g., BiLSTM/BiGRU with positional embeddings, piecewise pooling, or attention). If CNNs are chosen for efficiency or robustness, provide evidence (speed, parameter count, learning curves).
- ACE++ details and availability: How was it constructed? Which event types and domains are covered? What are the annotation guidelines and inter-annotator agreement? Will the dataset (or splits/labels) be released?
- Domain portability claim: Can you evaluate competing systems that rely on external semantic features on ACE++ (or a comparable out-of-domain set) to demonstrate their drop vs. your model?
- Non-gold evaluation: Please report results without gold triggers/arguments (e.g., using an existing trigger/argument detector) to assess end-to-end performance and error propagation.
- Integration with general coreference: In the example around line 687, an off-the-shelf system (e.g., CoreNLP) links “It” to “bombing.” Can such signals be incorporated as features or constraints, and does this reduce pronoun-related errors?
- Feature analysis: Which pairwise features are most impactful? Please provide per-feature ablations or learned-importance analyses, and briefly describe the feature selection process.

Overall, this is a clearly written, empirically careful paper with promising practical implications. Addressing the architectural comparisons, clarifying ACE++, and strengthening the evaluation on non-gold settings would significantly improve the contribution and its credibility.

This paper addresses event entity linking by replacing external knowledge sources used in prior work with sentential features derived from CNNs. The model is composed of two parts: the first learns representations of event mentions, and the second computes a coreference score for pairs of event mentions.

The paper is well written, clearly presented, and easy to follow. I particularly appreciate the analysis on the ACE corpus regarding argument sharing among coreferential events. The examination of dataset size effects also effectively motivates the construction of the ACE++ dataset. However, several major issues need to be addressed:

- The paper does not adequately motivate or analyze the choice of CNNs for generating mention representations. It is unclear why CNNs were preferred, and there are no comparisons with alternative architectures (e.g., a straightforward RNN). Given that the reported gains over the state of the art are only 2–3 F1 points across various metrics, stronger evidence is needed to justify that this architecture is indeed superior.
- The novelty of tackling event linking with sentential features is not clearly established, especially since using CNNs in this manner for classification is not new. The authors should explicitly identify and compare against any prior continuous-space methods for event linking. The set of baselines and comparisons in Table 3 is not sufficiently comprehensive.
- The paper provides no details on how the ACE++ dataset was collected. A known limitation of ACE is its small and constrained set of event types, which can bias evaluation. It is important to specify what event types ACE++ covers. This would also help substantiate the claim in Section 5.1 that “other approaches are strongly tied to the domain where these semantic features are available… our approach does not depend on resources with restricted…”. To support this, the authors should demonstrate that earlier methods fail on a dataset where the proposed approach succeeds. For meaningful future comparison, the authors should also consider making ACE++ publicly available.

Minor issues:

- Table 3 should include results for the proposed model without gold references as well.
- It would be useful to explore how this model can or cannot be augmented with a vanilla coreference resolution system. For the example on line 687, the off-the-shelf CoreNLP system links “It” to “bombing,” which could be leveraged in an event entity linking baseline.
- Given the relatively small size of ACE, a compelling evaluation should also include other available resources. This further motivates working on entity and event coreference simultaneously. In particular, testing on EventCorefBank alongside ACE seems essential.
- Table 5 indicates that pairwise features are quite effective, suggesting that feature engineering may still be crucial for competitive performance (at least at the ACE scale). It would be helpful to identify which features were most effective, explain how the current set was chosen, and report what else was tried.

Overall, the paper is clear and contains insightful analysis, but it requires stronger architectural motivation and comparison, clearer articulation of novelty, and substantially more detail and evidence around the ACE++ dataset and cross-dataset generalization.

- Paper proposes a resource-light neural model for within-document event coreference using sentence-level CNN features plus minimal pairwise features, matching prior feature-rich systems.  
- Main contributions: (1) architecture that avoids external semantic resources, (2) empirical ablations showing embeddings alone are insufficient and both local/sentential features matter, (3) error analysis identifying remaining challenges.  
- Strength: Matches or exceeds prior metrics (BLANC, MUC, positive-links) without WordNet/FrameNet, improving portability across domains.  
- Strength: Ablation results clearly show pairwise features alone are weak, trigger-local features improve performance, and sentence-level context provides further gains.  
- Strength: Analysis of clustering strategies (transitive closure vs BestLink) shows BestLink raises precision but reduces recall, situating method relative to prior recommendations.  
- Strength: Larger training data (ACE++) increases recall with minor precision drop, indicating scalability and reduced overfitting to ACE.  
- Strength: Qualitative error analysis pinpoints failure modes like pronoun resolution and annotation inconsistencies, guiding future work.  
- Weakness: Evaluation uses gold triggers, making cross-paper comparisons with systems using predicted triggers potentially unfair.  
- Weakness: Uses non-standard ACE train/test splits, reducing result comparability with prior work.  
- Weakness: Key implementation details are unclear, limiting reproducibility (position embeddings, piecewise pooling left/right treatment, local context window choice).  
- Weakness: The event-mention vector ve is defined but appears unused—clarity needed on which representation feeds the similarity layer.  
- Weakness: Pairwise feature encoding (binary vs distance encoding, learned vs fixed) is insufficiently described.  
- Weakness: ACE++ dataset is not publicly available, making results on it hard to contextualize or reproduce.  
- Weakness: No statistical significance testing presented despite close results across systems/metrics.  
- Question: Please report performance using predicted triggers (ideally matching predictors used by compared systems) for fairer comparisons.  
- Question: Specify position embedding initialization, training regime, index mapping for relative positions, clipping, and rationale for word-independent position embeddings.  
- Question: Clarify left/right context handling in CNNs—do filters differ, and how is the trigger boundary enforced during convolution and pooling?  
- Question: Confirm whether ve or vsent+lex is actually passed to the similarity layer and show the exact concatenation used with pairwise features.  
- Question: How is distance encoded in pairwise features (raw, binned, embedded), are features normalized, and are parameters learned?  
- Question: Is the square(.) similarity nonlinearity novel here, were alternatives (abs, ReLU, tanh, cosine) compared, and why does square help beyond polarity invariance?  
- Question: Can the architecture generalize to entity coreference (e.g., OntoNotes)? If so, report results to compare with that literature.  
- Additional: If ACE++ cannot be released, provide document IDs or feature dumps to aid replication and report on standard ACE splits as well.  
- Additional: Include statistical significance tests for main metrics (e.g., bootstrap on BLANC/MUC/B-CUBED).  
- Additional: Cite related neural coreference work (e.g., Wiseman et al., 2015) and address minor textual/labeling inconsistencies (line 143, Table 6 naming).

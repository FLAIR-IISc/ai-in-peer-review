This paper proposes a model for event entity linking that replaces external knowledge resources used in prior work with sentential features learned by convolutional neural networks (CNNs). The model has two components: (i) a module that learns event-mention representations and (ii) a scorer that computes a coreference score for a pair of event mentions.

The paper is clearly written and easy to follow. I particularly appreciate the ACE-corpus analysis of argument sharing among coreferent events, as well as the dataset-size analysis that motivates constructing ACE++. However, several major issues need to be addressed:

- Motivation for CNNs: The paper does not adequately justify the choice of CNNs for mention representation or analyze their trade-offs. It remains unclear why CNNs are preferable, and there is no comparison to alternative architectures (e.g., an RNN). Given that the reported gains over the state of the art are only about 2–3 F1 points on most metrics, stronger evidence is needed that this architecture is in fact superior.

- Novelty and positioning: The contribution of tackling event linking with sentential features is not clearly distinguished from prior work, and using CNNs for sentence-level classification is not novel. The paper should explicitly identify and compare against existing continuous-space methods for event linking. The set of baselines in Table 3 is not sufficiently comprehensive.

- ACE++ details and claims: The paper does not explain how ACE++ was collected. A core limitation of ACE is its small, biased set of event types; it is therefore essential to specify which event types ACE++ covers. This would also substantiate the claim in Section 5.1 that prior approaches are tied to domains where semantic features are available, whereas the proposed approach does not depend on restricted resources. To support this, the authors should show that previous methods fail on a dataset where their method succeeds. For meaningful future comparison, the authors should also consider releasing ACE++ publicly.

Minor issues:

- Table 3 should also report performance without gold references.

- Please explore whether the model can be augmented with a standard coreference resolution system. In the example on line 687, off-the-shelf CoreNLP links “It” to “bombing,” which could be leveraged in an event-linking baseline.

- Given the small size of ACE, evaluation on additional resources is important. This further motivates joint modeling of entity and event coreference. In particular, testing on EventCorefBank alongside ACE seems essential.

- Table 5 shows that pairwise features are highly effective, suggesting that feature engineering remains important (at least at ACE scale). Please indicate which features contributed most, how the current set was selected, and what alternatives were tried.

Overall, this is a promising direction with clear writing and useful analyses. To merit publication, the paper should (a) better motivate and empirically justify the CNN architecture, including comparisons to strong neural alternatives; (b) clarify its novelty relative to prior continuous-space approaches with more comprehensive baselines; and (c) fully document and, ideally, release ACE++, demonstrating that the proposed method succeeds where resource-dependent methods do not.

Summary and contributions
This paper tackles within-document event coreference (“event linking”) by replacing external semantic resources with sentential features learned by convolutional neural networks (CNNs). The model has two parts: (1) a CNN-based module that encodes an event mention using its sentence context plus trigger-local lexical features, and (2) a similarity module that scores a pair of event mentions using their learned representations along with pairwise features. The system achieves state-of-the-art performance on ACE and on a larger, partly public dataset (ACE++), and includes useful analyses of data properties, feature ablations, and errors.

As I see it, the main contributions are:
1) An event linking model that attains state-of-the-art results on ACE without external knowledge bases, relying on sentential features plus standard pairwise features.
2) Empirical analyses that quantify argument sharing and distance properties in ACE, ablations that tease apart the contributions of sentential vs. lexical vs. pairwise features, and an assessment of clustering strategies.
3) The construction and use of a larger ACE++ dataset and evidence that more training data substantially improves recall.

Strengths
- Clear, well-structured presentation: The overall model architecture, training regimen, and evaluation are explained cleanly. The paper is easy to follow.
- Informative data analyses: The statistics on argument-sharing, trigger mismatch, and cross-sentence distances convincingly motivate using sentential features. The exploration of transitive-closure vs. BestLink clustering and the dataset size study (ACE → ACE++) are also valuable.
- Strong empirical results without external KBs: The model matches or surpasses prior work on several metrics (e.g., BLANC, MUC, positive-link F1) while avoiding reliance on WordNet/FrameNet/VerbOcean, increasing portability across domains.
- Thorough ablations: Table 5 demonstrates that all three feature classes (sentential, trigger-local lexical, pairwise) contribute, and highlights the role of sentential features in recall.
- Useful error analysis: The qualitative review highlights challenging cases (notably pronoun-mediated links) and annotation ambiguities, contextualizing the metrics.

Weaknesses
- Limited architectural motivation and comparison: The paper does not justify the choice of CNNs for mention encoding versus alternatives such as BiLSTMs/GRUs or tree-structured models, nor does it compare against them. Given the relatively modest gains (often 2–3 F1 points) over strong baselines, stronger evidence is needed that CNN-based sentential modeling is the right design choice for this task.
- Novelty relative to prior continuous-space approaches is unclear: Using CNN-based sentence encoders for classification is well established. The paper should better delineate what is novel about applying sentential features to event linking, and compare to any existing continuous-space/representation learning methods specifically for event coreference. The set of baselines in Table 3 is not comprehensive enough to support strong novelty claims.
- Insufficient detail about ACE++: The paper does not describe how ACE++ was collected or annotated, what event types it covers, or how its distribution differs from ACE. Since one of the core claims is that external semantic features limit portability, it would be compelling to show that methods relying on such resources degrade on ACE++ while the proposed model remains strong. For future comparability, the dataset should be made publicly available if possible.
- Reliance on gold annotations: Results on ACE use gold triggers (and possibly gold arguments from the pipeline). An end-to-end evaluation or at least results without gold references would better reflect practical performance and allow fairer comparison to systems that do not use gold information.
- Pairwise feature dependence and lack of feature analysis: Table 5 suggests pairwise features remain quite influential, indicating that feature engineering still matters. The paper does not report which pairwise features contribute most or how the set was selected among alternatives. This limits interpretability and reproducibility.

Questions to authors
- Why CNNs for mention encoding? Did you evaluate RNN-based (e.g., BiLSTM) or recursive/tree-based encoders, or hybrid CNN–RNN models? If so, how did they compare?
- What is the composition of ACE++? Please describe sources, annotation process, event types and fine-grained subtypes covered, quality control, and inter-annotator agreement. Will you release ACE++ to facilitate future comparison?
- Exactly which gold inputs are used on ACE and ACE++ (triggers only, or also arguments)? Can you report performance without gold references to assess end-to-end robustness?
- Which pairwise features were most impactful? Did you run feature ablations/importances to identify the strongest contributors? What other pairwise features did you try, and why were they excluded?
- Can you incorporate an off-the-shelf nominal/pronominal coreference system to handle pronoun-mediated event links? For example, in the case mentioned around line 687, CoreNLP links “It” to “bombing”; can this signal be leveraged to improve event coreference?

Additional comments
- Please include results without gold references in Table 3 (or a separate table), to complement the gold-based evaluation.
- Given ACE’s small size, evaluating on additional resources would strengthen claims (e.g., EventCorefBank alongside ACE). This also motivates exploring joint entity–event coreference.
- Since pronoun resolution is a recurrent source of false negatives, exploring simple integrations (e.g., using off-the-shelf coreference outputs as features) could provide a strong baseline.
- If possible, consider releasing ACE++ publicly to substantiate the portability claim and enable community progress.

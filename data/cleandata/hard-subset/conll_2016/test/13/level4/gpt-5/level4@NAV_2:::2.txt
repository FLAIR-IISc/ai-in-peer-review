This paper addresses event entity linking by replacing reliance on external knowledge bases with sentential features derived from convolutional neural networks. The model is two-stage: first, a CNN encodes each event mention within its sentence to produce a mention representation; second, a similarity module, augmented with lexical and pairwise features, computes a coreference score for a pair of event mentions.

The manuscript is clearly written and easy to follow. I particularly appreciate the analysis of argument overlap within ACE, and the study on dataset size effects, which helps motivate the construction of the ACE++ dataset. That said, there are several major issues that should be addressed:

- The paper does not sufficiently motivate or analyze the choice of CNNs for mention encoding. It is not clear why CNNs are preferable here, nor are there comparisons to plausible alternatives (e.g., RNN/biRNN encoders). Given the relatively modest gains (approximately 2–3 F1 points on several metrics) over prior work, stronger empirical evidence is needed to justify that this architecture is the right design choice.

- The novelty of using sentential features for event linking is not well established. Employing CNNs in this fashion for classification is well known. The paper should explicitly highlight what is new in the context of event linking, and compare against existing continuous-space or neural approaches for event coreference/linking. The selection of baselines in Table 3 does not appear comprehensive enough to support the novelty claim.

- The paper lacks details about how ACE++ was created. A key limitation of ACE is its constrained set of event types, which can bias conclusions; thus, it is important to know which event types ACE++ includes and how it was curated. This information is also necessary to substantiate the claim in Section 5.1 that prior methods are bound to domains with available semantic resources while the proposed approach is not; ideally, show that methods dependent on external semantics underperform on ACE++ where this model succeeds. For future comparisons, please consider releasing ACE++ publicly.

Some minor issues:

- Please report results without gold annotations in Table 3 as well (e.g., using predicted triggers/arguments), to gauge end-to-end robustness.

- It would be useful to explore augmenting the model with a standard coreference resolver. In the example around line 687, an off-the-shelf system (e.g., Stanford CoreNLP) links “It” to “bombing,” which could be leveraged as an event linking baseline or as auxiliary signals.

- Given the small size of ACE, a stronger empirical case would include evaluations on other available resources. This also motivates investigating joint entity and event coreference. Parallel testing on EventCorefBank alongside ACE seems particularly important.

- Table 5 suggests pairwise features contribute substantially, indicating that feature engineering still matters at ACE’s scale. It would help to detail which pairwise features were most impactful, how the current set was selected, and what additional features were attempted.

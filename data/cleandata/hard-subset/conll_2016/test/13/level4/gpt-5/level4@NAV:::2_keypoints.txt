- Paper is clearly written, well-structured, and easy to follow, with a useful ACE corpus analysis of argument sharing across event coreferences.  
- Investigation of dataset size effects is convincing and provides a reasonable motivation for constructing the ACE++ dataset.  
- Major concern: the motivation and trade-offs for using CNNs for mention representations are not adequately discussed or justified.  
- No comparisons to alternative architectures (e.g., RNNs) or ablations to show CNNs are preferable given modest 2–3 F1 gains.  
- Novelty concern: using sentential CNN features for classification is not clearly new; the contribution over prior continuous-space approaches is unclear.  
- Baseline comparisons (Table 3) are incomplete; must explicitly compare to existing continuous-space event-linking methods.  
- ACE++ dataset construction is not described; missing details on which event types are included and how it addresses ACE’s type limitations.  
- Need empirical evidence that older methods underperform on ACE++ where the proposed approach succeeds, to substantiate claims about resource independence.  
- Dataset release: ACE++ should be made publicly available for meaningful future comparisons.  
- Minor: report results without gold references in Table 3 (i.e., fully automatic setting).  
- Minor: explore combining this model with vanilla coreference systems (e.g., CoreNLP) as a stronger baseline or hybrid approach.  
- Minor: evaluate on additional resources (e.g., EventCorefBank) to demonstrate generality beyond ACE.  
- Minor: Table 5 shows pairwise features matter—report which features were most impactful, how the final set was selected, and alternatives tried.

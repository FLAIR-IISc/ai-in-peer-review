This paper proposes a neural model for event entity linking that replaces external knowledge sources with sentential features learned by a CNN. The model has two components: one that learns event mention representations and another that computes a coreference score for a pair of event mentions. The paper is clearly written and easy to follow. I particularly appreciated the analysis on the ACE corpus regarding argument sharing across coreferent events and the exploration of dataset size effects, which motivates the construction of the ACE++ dataset. However, several substantive issues limit the paper’s impact and clarity.

Strengths
- Clear problem setup and a well-structured presentation.
- Insightful analysis of argument sharing in event coreference.
- Empirical evidence that training data size matters, motivating ACE++.

Major concerns
- Motivation for CNNs is underdeveloped. The paper does not explain why CNNs are well-suited for event mention representation compared to plausible alternatives (e.g., RNNs, bi-LSTMs, simple bag-of-ngrams with MLPs, or attention-based encoders). Given that reported gains over strong baselines are modest (2–3 F1 points on some metrics), the paper should offer stronger empirical or conceptual justification for the architectural choice. Recommended additions:
  - A principled discussion of what aspects of local sentential context CNNs capture that are especially beneficial for event linking.
  - Ablations replacing the CNN with RNN/bi-LSTM and a bag-of-ngrams baseline under identical training conditions.
  - A comparison of different context window sizes and convolution filter widths to assess robustness and capacity.
- Novelty relative to prior continuous-space approaches is unclear. Using CNN-based sentence features for classification is not new; the paper should more explicitly position its contribution with respect to prior neural or distributional methods for event coreference/linking. The set of baselines in Table 3 appears incomplete for establishing novelty and competitiveness. Recommended actions:
  - Enumerate and directly compare against existing continuous-space/neural approaches to event coreference/linking where applicable.
  - Clarify what is conceptually new: Is it the specific two-part architecture, the reliance solely on sentential features, or the training paradigm?
- ACE++ dataset description is insufficient. A key limitation of ACE is its constrained and biased set of event types. Without details on ACE++, it is hard to evaluate the generality of the proposed approach. The paper should specify:
  - How ACE++ was constructed (sources, annotation guidelines, annotator expertise, quality control, inter-annotator agreement).
  - Coverage and distribution of event types, domains, and genres, and how these differ from ACE.
  - Train/dev/test splits and licensing/access conditions.
  - Evidence that methods relying on external semantic resources degrade on ACE++ while the proposed method remains robust. This would substantiate the claim that the approach avoids domain-specific dependencies.
  - A commitment to release ACE++ publicly to enable reproducibility and meaningful future comparisons.
- Limited analysis of feature contributions and error types. Table 5 suggests that pairwise features are highly effective, implying that feature engineering remains important at ACE scale. The paper would benefit from:
  - A more granular ablation showing which pairwise features contribute most.
  - A description of the feature selection process and alternatives considered.
  - Error analysis categorized by event types, argument structures, and discourse phenomena to identify when sentential features succeed or fail.

Minor comments and suggestions
- Please also report performance without relying on gold annotations (e.g., without gold mentions/references) in Table 3 to quantify end-to-end utility.
- Consider evaluating whether an off-the-shelf coreference system can augment the model. For example, if a system such as CoreNLP links pronominal references (e.g., “It”) to an antecedent event (e.g., “bombing”), that signal could strengthen an event linking baseline or be integrated into the pairwise scoring function.
- Given the limited size of ACE, results on additional benchmarks would strengthen claims. Parallel evaluation on EventCorefBank (ECB/ECB+) is particularly important for assessing generalization and for comparing to a wider set of prior work.
- Provide implementation details sufficient for replication: hyperparameter settings, training regimen, negative sampling strategy, and model selection criteria.
- Clarify evaluation protocol and metrics, especially how mention detection (if any) interacts with linking, and whether comparisons assume gold or predicted mentions/arguments.

Overall assessment
This is a promising direction that aims to reduce reliance on external knowledge by leveraging sentential representations. The paper is well-presented and shows encouraging results, along with useful diagnostic analyses. However, the current submission does not yet provide a compelling justification for the specific architectural choice (CNNs), lacks thorough comparisons to related neural/continuous-space methods, and does not sufficiently document the ACE++ dataset or demonstrate cross-domain robustness. Addressing these issues—particularly through stronger ablations, broader baselines, detailed dataset documentation, and evaluation on additional corpora—would substantially improve the paper’s clarity, credibility, and impact.

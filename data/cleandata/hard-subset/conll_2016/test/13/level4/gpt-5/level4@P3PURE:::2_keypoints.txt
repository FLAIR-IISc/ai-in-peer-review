- Paper replaces external knowledge with sentential CNN-derived features for event entity linking, using mention representation and pairwise coreference scoring components.
- Writing and presentation are clear, and analysis of argument sharing and dataset size is appreciated.
- Major concern: paper does not motivate or analyze why CNNs were chosen over alternatives (e.g., RNNs) and offers no architectural comparisons.
- Reported improvements are modest (2–3 F1 points), so stronger evidence is needed to justify the CNN architecture as superior.
- Novelty is unclear: using CNNs on sentence features for classification is not new, and the paper does not situate itself relative to prior continuous-space event linking methods.
- Baselines and comparisons (Table 3) are incomplete and should include additional relevant prior methods.
- ACE++ dataset collection lacks methodological details; the paper must specify how it was created and which event types it covers.
- The claim that the approach is domain/resource-independent needs substantiation by showing earlier methods fail where this approach succeeds.
- Authors should consider releasing ACE++ for meaningful future comparisons.
- Minor: Table 3 should report results for the proposed model without gold references.
- Minor: Explore integration with vanilla coreference systems (e.g., CoreNLP linking “It” to “bombing”) as a baseline improvement.
- Minor: Evaluation should include other resources (e.g., EventCorefBank) given ACE’s small size, and consider joint entity-event coreference.
- Minor: Table 5 shows pairwise features are effective—identify which features mattered, how they were selected, and what was tried.
- Overall: paper is clear with useful analysis but requires stronger architectural motivation, clearer novelty claims, more complete comparisons, and substantially more ACE++ details and cross-dataset evidence.

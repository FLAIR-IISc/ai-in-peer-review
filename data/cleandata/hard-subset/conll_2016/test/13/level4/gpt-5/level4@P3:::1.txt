Summary and contributions
This paper addresses within-document event coreference (event linking) using a neural model that leverages sentential features from convolutional neural networks (CNNs), avoiding reliance on external semantic resources. The approach encodes each event mention by combining CNN-based sentence representations with local lexical context; a second stage combines two mention representations with a small set of pairwise features to score coreference. The model is evaluated on ACE 2005 and an expanded dataset (ACE++), achieving performance comparable to feature-rich systems. Ablations, clustering variants, and an error analysis are provided.

As I see them, the key contributions are:
1) A resource-light neural architecture for event linking that replaces external semantic features with CNN-derived sentential representations, while maintaining state-of-the-art performance on ACE.
2) A thorough empirical study, including feature ablations (pairwise vs. local lexical vs. sentential), clustering strategy comparisons (transitive closure vs. BestLink), and an informative error analysis highlighting discourse/pronominal challenges.
3) Evidence that sentential features significantly improve recall and that increased training data (ACE++) further boosts generalization.

Strengths
- Resource independence with strong results: The model achieves BLANC F1 on par with or better than prior systems that rely on extensive external semantic resources. This is important for portability across domains where WordNet/FrameNet-style features are unavailable or unreliable.
- Well-motivated architecture: The two-stage design—first building mention representations with CNNs and then scoring pairs—aligns with data properties the authors analyze (e.g., many positive pairs lack shared arguments and often have different triggers), justifying reliance on sentential context rather than only intrinsic event properties.
- Comprehensive empirical analysis: The paper provides:
  • Clean ablations demonstrating additive gains from pairwise, local lexical, and sentential features, with sentential features delivering the largest boost.
  • A comparison of clustering strategies showing that, for this model, transitive-closure clustering yields better F1 than BestLink, contrasting with earlier findings and adding nuance to clustering choices in event coreference.
  • Clear discussion of metrics and the rationale for emphasizing BLANC given the prevalence of singletons/negative links.
- Useful error analysis: The qualitative analysis surfaces key failure modes, especially pronoun resolution and borderline annotation cases, pointing to promising directions for future work (e.g., discourse modeling).
- Reproducibility signals: The implementation details (TensorFlow, hyperparameter exploration) and the release of the ACE split list increase transparency.

Weaknesses
- Gold triggers and non-comparable settings: The main results use gold event triggers, while at least one comparator uses predicted triggers, undermining comparability. This choice may overestimate practical performance and should be addressed with predicted-trigger results or stronger caveats.
- Dataset splits and fairness: The paper acknowledges that train/test splits vary across works. While a document list for ACE is provided, the lack of alignment with prior splits complicates direct comparison. A re-run on commonly used splits, or a mapping to prior partitions, would strengthen claims.
- Methodological clarity gaps:
  • Position embeddings: It is unclear how relative position embeddings are initialized and trained, and why positions relative to different tokens would share embeddings. More detail on their parameterization and interaction with words is needed.
  • Left/right context handling: The description of dual pooling for left/right contexts could be clearer about where this asymmetry is used (only in pooling, or elsewhere) and how multiple mentions per sentence are handled.
  • Context window choice: Appending embeddings of exactly one token on each side of the trigger seems arbitrary; sensitivity analyses or justification would help.
  • Use of ve: The role of the final event-mention representation ve is unclear, as subsequent equations appear to use vsent+lex; please clarify which representation feeds the pairwise scorer.
  • Pairwise features: Encoding (binary vs. real-valued), treatment of distances (binning/normalization), and whether these parameters are learned or fixed are not fully specified.
- ACE++ availability and comparisons: While ACE++ improves recall and supports useful analyses, it is not publicly available. The absence of comparisons to feature-rich baselines on the same dataset limits external validation and future benchmarking.
- Statistical significance: Many reported differences are small; significance testing would make the comparative claims more robust.
- Related work gap: Recent neural coreference models (e.g., Wiseman et al., 2015) are not discussed; situating the work against neural approaches to mention representation and pairwise scoring would strengthen the positioning.

Questions to authors
- Predicted triggers: Can you report performance with predicted triggers (e.g., from a standard event extractor) to gauge end-to-end utility and make comparisons more equitable?
- Position embeddings: How are relative position embeddings initialized and trained? Are they shared across sentences and triggers? Why should the same relative offset have a single embedding regardless of lexical content?
- Left/right pooling: Does the left/right split affect only max-pooling, or any earlier/later layers? How does the model disambiguate multiple mentions in the same sentence?
- Context window size: Why include exactly one token before/after the trigger? Did you try larger windows or learned attention? Any sensitivity analysis?
- Representation usage: What is the precise flow of representations in Section 3.2—do you use ve or vsent+lex as input to the similarity layer?
- Pairwise features: How are binary and distance features encoded and scaled? Are any parts learned (e.g., embeddings for binned distances), or are they fixed numeric inputs?
- Squared activation: Is using square as a similarity nonlinearity novel here? How does it compare to ReLU/tanh/cosine beyond the reported setting? Do you see broader applicability?
- ACE++: Will you release ACE++ or at least the pair lists/splits to facilitate comparison? If not, could you provide a replication package or run baselines on request?
- Significance: Did you run statistical tests on key comparisons (e.g., vs. Liu et al., 2014) and ablations?

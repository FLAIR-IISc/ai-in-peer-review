This paper proposes a model for event entity linking that replaces the external knowledge sources used by prior work with sentence-level features derived from CNNs. The model has two components: (1) a module that learns an event-mention representation, and (2) a module that computes a coreference score for a pair of event mentions.

The paper is well written, clearly presented, and easy to follow. I especially appreciate the ACE corpus analysis on argument sharing in event coreference, and the examination of dataset size effects, which motivates the creation of the ACE++ dataset. However, several major issues need to be addressed:

- The choice of CNNs for mention representation is not sufficiently motivated or analyzed. The paper does not explain why CNNs were chosen over alternatives (e.g., a straightforward RNN), nor does it compare against them. Given that the reported gains over the state of the art are only 2–3 F1 points across various metrics, stronger evidence is needed to justify that this architecture is superior.

- The novelty of using sentential features for event linking is unclear, since applying CNNs in this way for classification is not new. The authors should explicitly identify and compare against existing continuous-space approaches for event linking. The baselines in Table 3 are not thorough enough.

- There is no description of how the ACE++ dataset was collected. A major limitation of ACE is its small set of event types, which makes it constrained and potentially biased. It is important to know what event types ACE++ includes. This is also needed to support the claim in Section 5.1 that “other approaches are strongly tied to the domain where these semantic features are available… our approach does not depend on resources with restricted…”. You should show that previous methods fail on a dataset on which your approach succeeds. To enable meaningful future comparisons, the dataset should be made publicly available.

Minor issues:

- Please also report model performance in Table 3 without gold references.

- Consider exploring how this model could or could not be augmented with a vanilla coreference system. In the example on line 687, an off-the-shelf CoreNLP system links “It” to “bombing,” which could be leveraged in an event entity linking baseline.

- Given the relatively small size of ACE, evaluation on other available resources is important. This further motivates working on entity and event coreference jointly. In particular, testing on EventCorefBank alongside ACE seems essential.

- Table 5 shows that pairwise features are highly effective, suggesting that feature engineering may still be crucial for competitive performance (at least at ACE’s scale). Which features were most effective? Please report how the current feature set was selected and what alternatives were tried.

This paper presents a model for the task of event entity linking, where the authors propose to use sentential features from Convolutional Neural Networks (CNNs) in place of external knowledge sources, which earlier methods have employed. They train a two-part model: the first part learns an event mention representation, and the second part learns to calculate a coreference score given two event entity mentions.

The paper is well-written, well-presented, and easy to follow. The analysis done on the ACE corpus regarding argument sharing between event coreferences is particularly noteworthy. Furthermore, the analysis on the size impact of the dataset serves as a great motivation for creating the ACE++ dataset. However, there are several major issues that need to be addressed:

- The authors fail to provide a clear motivation for using CNNs to generate mention representations. They do not discuss why they chose CNNs over other models, such as Recurrent Neural Networks (RNNs), and do not provide comparisons to these alternatives. Given that the improvement their model achieves over the state-of-the-art is only 2 or 3 points on the F1 score, more evidence is needed to demonstrate that this architecture is indeed superior.

- It is unclear what is novel about the idea of tackling event linking with sentential features, as using CNNs in this fashion for a classification task is not new. The authors could explicitly point out and compare to existing continuous space methods for event linking. The choice of methods in Table 3 is not thorough enough, and a more comprehensive comparison would strengthen the paper.

- There is a lack of information regarding how the ACE++ dataset was collected. A major issue with the ACE dataset is its limited number of event types, making it too constrained and biased. It is essential to know what event types ACE++ covers, as this can help support the claim in Section 5.1 that 'other approaches are strongly tied to the domain where these semantic features are available…our approach does not depend on resources with restricted…'. To substantiate this claim, the authors should demonstrate that earlier methods fail on a dataset where their approach succeeds. Additionally, making the ACE++ dataset publicly available would enable meaningful comparisons in future work.

Some minor issues also need to be addressed:

- It would be beneficial to include the performance of the model without gold references in Table 3 for a more comprehensive comparison.

- Exploring how this model can be augmented with a vanilla coreference resolution system would be a valuable addition. For instance, the specific example in line 687 demonstrates that an off-the-shelf CoreNLP system readily links 'It' to 'bombing', which could be leveraged in an event entity linking baseline.

- Given the relatively small size of the ACE dataset, it is essential to test the model on other available resources to demonstrate its robustness. This also motivates the need to work on entity and event coreference simultaneously. Testing on EventCorefBank in parallel with ACE is crucial to establish the model's effectiveness.

- Table 5 shows that pairwise features have been quite effective, indicating that feature engineering may still be crucial for having a competitive model (at least on the scale of the ACE dataset). It would be interesting to know which features were the most effective and why the current set was chosen. The authors should also report on other features that were tried and the reasoning behind the final selection.

This paper proposes a model for event entity linking, which utilizes sentential features from CNNs instead of external knowledge sources used in earlier methods. The model consists of two parts: the first part learns an event mention representation, and the second part calculates a coreference score given two event entity mentions.

The paper is well-written, well-presented, and easy to follow. The analysis on the ACE corpus regarding argument sharing between event coreferences is insightful, and the discussion on the impact of dataset size is a compelling motivation for creating the ACE++ dataset. However, there are several major concerns that need to be addressed:

- The authors do not provide sufficient motivation and analysis for using CNNs to generate mention representations. They do not discuss why they chose CNNs over other models, such as RNNs, and do not compare their performance. Given the modest improvement of their model over the state-of-the-art (2-3 points on F1 score), more evidence is needed to demonstrate the superiority of this architecture.

- It is unclear what is novel about the idea of tackling event linking with sentential features, as using CNNs for classification tasks is not new. The authors should explicitly highlight the novelty of their approach and compare it to existing continuous space methods for event linking. The choice of methods in Table 3 is not thorough enough.

- There is no information about the collection process of the ACE++ dataset. A significant limitation of the ACE dataset is its restricted number of event types, making it biased and constrained. It is essential to know what event types ACE++ covers to support the claim in Section 5.1 that the proposed approach is more domain-independent than earlier methods. To substantiate this claim, the authors should demonstrate the failure of earlier methods on a dataset where their approach succeeds. Furthermore, making the ACE++ dataset publicly available would facilitate meaningful comparisons in future work.

Some minor issues:

- It would be beneficial to include the performance of the model without gold references in Table 3 for a more comprehensive evaluation.

- Exploring how the model can be augmented with a vanilla coreference resolution system would be valuable. For instance, the off-the-shelf CoreNLP system correctly links 'It' to 'bombing' in the example in line 687, which could be leveraged in an event entity linking baseline.

- Given the relatively small size of the ACE dataset, it is crucial to test the model on other available resources to demonstrate its robustness. This also motivates the need to work on entity and event coreference simultaneously. Testing on EventCorefBank in parallel with ACE is essential.

- Table 5 shows that pairwise features have been effective, indicating that feature engineering remains crucial for developing a competitive model (at least on the scale of the ACE dataset). It would be informative to know which features were the most effective and how the current set was chosen, as well as what other features were tried.

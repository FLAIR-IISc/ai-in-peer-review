- Proposes an event entity linking model using sentential CNN features instead of external knowledge sources, with separate mention representation and coreference scoring components.

- Paper is well-written, clear, and includes insightful analysis on argument sharing in ACE and dataset-size motivation for ACE++.

- Insufficient motivation and analysis for choosing CNNs over alternatives (e.g., RNNs) and no empirical comparison to justify architecture given modest 2–3 F1 gain.

- Novelty is unclear since using CNNs and sentential features for classification is not new; the paper should explicitly state novelty and compare to continuous-space event linking methods.

- Methods compared in Table 3 are not thorough enough; more relevant baselines should be included.

- No details on the ACE++ dataset collection process or the event types covered, undermining claims of greater domain-independence.

- Authors should demonstrate earlier methods failing where their approach succeeds and release ACE++ to enable fair comparisons.

- Table 3 should report performance without gold references to provide a more comprehensive evaluation.

- Consider augmenting the model with off-the-shelf coreference systems (e.g., CoreNLP) as a baseline or complementary component.

- Need to evaluate on other datasets (e.g., EventCorefBank) and consider joint entity–event coreference due to ACE’s small size.

- Table 5 indicates pairwise feature engineering remains important; report which features were most effective, selection process, and what other features were tried.

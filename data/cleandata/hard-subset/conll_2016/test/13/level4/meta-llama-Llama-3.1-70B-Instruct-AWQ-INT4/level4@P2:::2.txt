**Summary of the Paper**

This paper proposes a model for event linking, a task that involves resolving coreferences between event mentions in a document. The model uses sentential features from convolutional neural networks (CNNs) to represent event mentions, rather than relying on external knowledge sources. The paper presents a two-part model: the first part learns an event mention representation, and the second part learns to calculate a coreference score given two event entity mentions. The model is evaluated on the ACE 2005 dataset and a new dataset called ACE++, which is an extension of ACE.

**Main Contributions**

1. The paper proposes a novel approach to event linking using sentential features from CNNs, which achieves state-of-the-art results without relying on external feature sources.
2. The authors create a new dataset, ACE++, which is an extension of the ACE 2005 dataset, and evaluate their model on this dataset.
3. The paper presents an analysis of the ACE corpus regarding the argument sharing between event coreferences, which motivates the use of sentential features.

**Strengths**

1. The paper is well-written and easy to follow.
2. The analysis done on the ACE corpus is interesting and provides a good motivation for the proposed approach.
3. The evaluation on the ACE++ dataset provides a more comprehensive understanding of the model's performance.

**Weaknesses**

1. The authors fail to motivate and analyze the pros and cons of using CNN for generating mention representations. A comparison with other models, such as recurrent neural networks (RNNs), would be helpful.
2. The novelty of the idea of tackling event linking with sentential features is not clearly explained. The authors should explicitly compare their approach to existing continuous space methods for event linking.
3. There is no information regarding how the ACE++ dataset is collected, which is a major issue. The authors should provide more details about the dataset, including the event types it covers.
4. The paper lacks a comparison with other approaches on a dataset that the proposed approach succeeds on, which would help to support the claim that the approach does not depend on resources with restricted domain availability.

**Questions to Authors**

1. Can you provide more details about the ACE++ dataset, including the event types it covers and how it was collected?
2. How did you choose the features used in the pairwise comparison, and what other features were tried?
3. Can you provide a comparison with other approaches on a dataset that the proposed approach succeeds on?
4. How do you plan to address the issue of pronoun resolution, which is a challenging problem in event linking?

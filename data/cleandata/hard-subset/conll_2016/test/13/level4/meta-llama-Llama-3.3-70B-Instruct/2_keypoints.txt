- Proposes a novel two-stage event entity linking model using CNN-extracted sentential features instead of external knowledge sources.  
- Paper is well-organized and clearly presented; analysis of argument sharing in ACE and the dataset-size motivation for ACE++ are noteworthy.  
- Fails to motivate the use of CNNs or analyze their advantages/disadvantages; lacks comparison with other architectures (e.g., RNNs) despite only modest F1 gains.  
- Novelty is unclear because using sentential features and CNNs for classification is not new; authors must highlight innovations and compare to continuous-space event linking methods; Table 3 comparisons are incomplete.  
- ACE++ dataset collection process is not described; event type coverage is unknown, undermining robustness claims and necessitating public release for reproducibility.  
- Table 3 should include model performance without gold references for a fuller evaluation.  
- Suggest augmenting the model with a vanilla coreference system (e.g., CoreNLP) to strengthen baselines and leverage pronoun links.  
- Must evaluate on additional resources (e.g., EventCorefBank) and consider joint entity/event coreference given ACE’s small size.  
- Pairwise feature results imply feature engineering remains important; report which features were most effective, why they were chosen, and other features tried.

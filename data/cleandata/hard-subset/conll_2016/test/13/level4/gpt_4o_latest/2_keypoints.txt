- Paper proposes CNN-derived sentential features for event entity linking, with components for mention representation and pairwise coreference scoring.
- Writing and structure are strong; ACE corpus analysis and dataset-size discussion supporting ACE++ are appreciated.
- Authors fail to justify choosing CNNs over alternatives (e.g., RNNs) or provide comparative analysis to support the modest 2–3 F1 improvement.
- Novelty is unclear because using CNNs for classification and sentential features is not new; need explicit comparison to existing continuous-space event-linking methods.
- Table 3’s comparative methods are not comprehensive and should include more baselines.
- Methodology for constructing ACE++ is not described; event type coverage and how ACE++ addresses ACE’s limitations must be clarified.
- Need empirical evidence that prior approaches fail on datasets where this method succeeds, and consider making ACE++ publicly available for reproducibility.
- Report model performance without gold references in Table 3 (currently missing).
- Explore augmenting the model with a standard coreference system (e.g., CoreNLP example where “It” links to “bombing”).
- Evaluate on other resources (e.g., EventCorefBank) to establish robustness and argue for joint entity-event coreference modeling.
- Table 5 shows pairwise features are very effective; identify the most impactful features, explain selection criteria, and discuss discarded features.

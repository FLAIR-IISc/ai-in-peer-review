

**Summary and Contributions**  
The paper proposes a neural network approach for event coreference resolution (event linking) that generates sentential features via convolutional neural networks (CNNs) without relying on external semantic resources like WordNet or FrameNet. The main contributions are:  
1. A novel model that achieves state-of-the-art performance on event linking without external semantic features, demonstrating that sentential features alone can match or exceed previous approaches.  
2. A thorough analysis of feature contributions showing that sentential features significantly improve performance over lexical and pairwise features alone, with empirical evidence that cross-sentential resolution is essential (55.42% of coreferential mentions are >5 sentences apart).  
3. An error analysis identifying challenges in pronoun resolution and annotation ambiguity, providing clear directions for future work.  

**Strengths**  
1. **Strong empirical validation**: The model achieves SOTA results on two datasets (ACE and ACE++) without external semantic resources. This is significant because prior work heavily relied on domain-specific resources (e.g., Liu et al. 2014 used 105 semantic features), limiting cross-domain applicability. The authors convincingly show that sentential features alone suffice for high performance, removing a major barrier to deploying event linking in new domains.  
2. **Comprehensive feature analysis**: The ablation study (Table 5) demonstrates that all feature classes (pairwise, trigger-local, sentential) contribute meaningfully to performance. The model’s recall boost from ACE→ACE++ (75.16%→83.21%) further validates its generalizability with more data.  
3. **Insightful error analysis**: The analysis of false positives/negatives reveals fundamental challenges (e.g., pronoun resolution) and annotation ambiguity (17% of false negatives matched false positives in human annotation), providing concrete pathways for improvement. This goes beyond superficial error reporting to highlight task-specific difficulties.  

**Weaknesses**  
1. **Unfair comparison due to gold trigger usage**: The authors use gold-standard event triggers (not predicted ones), while key competitors (e.g., Liu et al. 2014) use predicted triggers. This inflates performance by avoiding error propagation from trigger identification—a critical step in real-world pipelines. The comparison is therefore not directly comparable, undermining the claim of "state-of-the-art" performance.  
2. **Non-standard dataset splits**: The paper uses a different train/test split for ACE than prior work, complicating fair comparison. The authors should adopt the standard ACE split (as in Liu et al. 2014) to ensure reproducibility and meaningful benchmarking.  
3. **Lack of ACE++ dataset release**: While ACE++ is used for main experiments, it is not publicly available. This prevents independent validation and limits the community’s ability to compare against the authors’ results. The authors should release ACE++ or provide a clear rationale for its non-disclosure.  
4. **Insufficient evaluation of trigger prediction**: The authors omit experiments with predicted triggers, which would better assess real-world applicability. The claim that their model "does not depend on external resources" is weakened without this test.  

**Questions to Authors**  
1. The paper states that relative position embeddings are "generated in a way similar to word embeddings" (Section 3.1). How are these embeddings initialized and trained? Are they shared across all positions (e.g., "left of trigger" has the same vector regardless of the trigger word)? If so, why would this be effective?  
2. The authors use one word before/after the trigger for lexical features (Section 3.1, line 307). Why this specific window size? How would performance change with larger/smaller windows (e.g., 2 words before/after)?  
3. In Section 3.2, pairwise features like "distance between mentions" are used. How are non-binary features (e.g., sentence distance) encoded? Are they discretized, embedded, or treated as continuous values during training?  
4. The model computes an event-mention representation $v_e$ (Equation 3), but the subsequent section only uses $v_{sent+lex}$. Why is $v_e$ not utilized in the coreference scoring?  

**Additional Comments**  
- The model’s use of a square activation for similarity (Equation 4) is interesting but lacks justification. Is this novel for neural coreference? Could it generalize to other tasks?  
- The paper mentions "no external resources" but uses pre-trained word embeddings (GloVe). Clarify whether these are considered "external resources" or if they are domain-agnostic.  
- The "same type" baseline (Table 6) assumes event type agreement implies coreference, which is flawed (e.g., multiple "Personnel.Elect" events). This baseline should be explicitly criticized as unrealistic.

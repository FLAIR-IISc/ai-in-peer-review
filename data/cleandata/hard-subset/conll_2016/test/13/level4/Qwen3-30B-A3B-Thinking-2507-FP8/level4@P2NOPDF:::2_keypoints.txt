- Paper proposes event entity linking using CNN-based sentential features and a two-part model for mention representation and coreference scoring.
- Presentation and structure are strong; analysis of argument sharing in ACE corpus is compelling.
- Creation and motivation for the ACE++ dataset via dataset size analysis is well-justified.
- No justification for choosing CNNs over alternatives (e.g., RNNs); lacks comparative experiments to support architectural choice.
- Reported gains (2–3 F1) over baselines are marginal, so stronger evidence is needed to claim superiority.
- Novelty of using sentential features is unclear because CNN-based feature extraction is not new; paper fails to contrast with continuous-space event linking methods.
- Literature comparison (Table 3) is inadequate; a more thorough benchmark against relevant prior work is required.
- ACE++ construction methodology is unspecified; potential bias from ACE’s limited event types is not addressed.
- Authors must show prior methods fail on ACE++’s expanded event types and that their model succeeds, and release ACE++ for reproducibility.
- Table 3 should report performance without gold references to assess practical utility.
- Explore compatibility with vanilla coreference systems as stronger baselines (e.g., CoreNLP resolving pronouns to events).
- Validate robustness by testing on additional resources (e.g., EventCorefBank) and motivating joint entity-event coreference resolution.
- Table 5 implies feature engineering remains important; clarify which pairwise features were most effective, selection process, and discarded alternatives.
- Overall, methodological justification, clearer novelty claims, dataset transparency, and expanded experiments are necessary before publication.

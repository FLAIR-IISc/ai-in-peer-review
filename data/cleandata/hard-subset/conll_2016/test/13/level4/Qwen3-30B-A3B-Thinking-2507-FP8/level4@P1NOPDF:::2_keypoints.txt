- Paper proposes event entity linking using sentential CNN features and a two-part architecture for mention representation and pairwise coreference scoring.
- Manuscript is well-written, methodologically clear, and provides useful analysis of argument sharing in ACE event coreferences.
- Strong motivation offered for constructing ACE++ via dataset size analysis.
- Insufficient justification for using CNNs over alternatives (e.g., RNNs) and no comparative analysis with simpler baselines.
- Reported gains are marginal (2–3 F1 points); stronger evidence is needed that the architecture genuinely improves performance.
- Novelty of using sentential features is unclear; approach is not explicitly contrasted with existing continuous-space event linking methods.
- Model comparisons (Table 3) are incomplete and omit relevant prior work, weakening empirical claims.
- ACE++ construction methodology is not described and expanded event types are unspecified, undermining claims about resource independence.
- Authors must demonstrate that their model outperforms prior methods on ACE++ and show baselines fail on the same dataset; ACE++ should be publicly released for reproducibility.
- Table 3 should include performance without gold references to strengthen evaluation.
- Evaluate integration with standard coreference systems (e.g., CoreNLP) to provide a more robust baseline.
- Test on complementary resources like EventCorefBank to address ACE’s limited scale and bolster the joint entity/event coreference argument.
- Table 5 indicates pairwise feature engineering remains critical; clarify which features were most impactful, how they were selected, and what alternatives were tried.



This paper proposes a model for event entity linking that replaces external knowledge sources with sentential features derived from CNNs. The authors train a two-part model: the first component learns event mention representations, and the second calculates coreference scores for pairs of event mentions.  

The paper is well-written, well-organized, and easy to follow. I particularly appreciate the analysis of argument sharing between event coreferences in the ACE corpus, as well as the analysis of dataset size impact, which strongly motivates the creation of the ACE++ dataset. However, several major issues require addressing:  

- The authors fail to justify the choice of CNN for mention representation, including a discussion of its advantages over alternatives (e.g., RNNs) and a comparative analysis. Given the marginal F1 score improvements (2–3 points) over state-of-the-art methods, stronger evidence is needed to demonstrate that this architecture is superior.  
- The novelty of using sentential features for event linking is not clearly established, as CNN-based classification approaches for similar tasks are not new. The authors should explicitly contrast their method with existing continuous-space approaches for event linking and provide a more thorough comparison (e.g., Table 3 is insufficient).  
- The collection methodology for the ACE++ dataset is not described. Since the ACE dataset suffers from limited event types and domain bias, it is critical to specify which event types ACE++ covers. This detail is necessary to substantiate the claim in Section 5.1 that "other approaches are strongly tied to domains where semantic features are available…our approach does not depend on resources with restricted [coverage]." To support this, the authors must demonstrate that prior methods fail on ACE++ while their approach succeeds. Additionally, the dataset should be made publicly available to enable future reproducibility and comparison.  

Minor issues:  
- The performance of the model without gold references should be included in Table 3.  
- Exploring integration with a vanilla coreference resolution system (e.g., CoreNLP’s off-the-shelf link of 'It' to 'bombing' in line 687) would strengthen the baseline analysis.  
- Given ACE’s small size, testing on additional resources (e.g., EventCorefBank) is essential to validate the model’s robustness and further justify simultaneous entity-event coreference modeling.  
- Table 5 indicates pairwise features are highly effective, suggesting feature engineering remains crucial for competitive results on the ACE scale. The authors should clarify which features were most impactful, how the feature set was selected, and what alternatives were evaluated.

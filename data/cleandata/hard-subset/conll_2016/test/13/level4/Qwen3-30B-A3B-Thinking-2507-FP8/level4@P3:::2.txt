

**Summary and Contributions**  
This paper proposes a novel approach to event coreference resolution (linking) that eliminates reliance on external semantic resources (e.g., WordNet, FrameNet). The authors introduce a two-part model: (1) a CNN-based architecture that generates sentential features for event mentions by processing their context, and (2) a coreference scoring mechanism that combines these features with lexical and pairwise features. The key contributions are:  
1. A resource-light model achieving state-of-the-art performance on two datasets (ACE 2005 and a newly created ACE++), demonstrating that external semantic features are not necessary for high performance.  
2. Comprehensive empirical validation of the importance of sentential features through ablation studies, showing their superiority over purely lexical or pairwise approaches.  
3. An insightful error analysis identifying unresolved challenges, particularly pronoun-based coreference resolution, which provides clear directions for future work.  

**Strengths**  
1. **Novel resource-light approach**: The paper convincingly demonstrates that high-quality event linking can be achieved without external semantic resources. This addresses a significant limitation in prior work, which often requires domain-specific resources. The authors' model outperforms state-of-the-art systems (e.g., Liu et al., 2014) on ACE while eliminating dependencies on WordNet/FrameNet, making it more adaptable to new domains.  
2. **Thorough empirical validation**: The ablation studies (Table 5) clearly establish the necessity of sentential features, showing that removing them reduces F1 by 12.5 points. The comparison of clustering strategies (Table 4) also provides valuable insights into model behavior, revealing that transitive closure outperforms BestLink—a counterintuitive finding that challenges prior assumptions.  
3. **Insightful error analysis**: The manual analysis of 200 errors (100 false positives/negatives) identifies systematic weaknesses (e.g., pronoun resolution) and highlights dataset ambiguities (e.g., 22/100 false positives had plausible human disagreement). This elevates the work beyond mere performance metrics to actionable research directions.  

**Weaknesses**  
1. **Insufficient justification for CNN choice**: The paper does not adequately compare CNNs to alternative architectures (e.g., RNNs, LSTMs) for feature extraction. Given the marginal performance gains (2-3 F1 points over prior work), the authors should justify why CNNs are superior to simpler or more common alternatives. Without this, the architectural choice appears arbitrary.  
2. **Underdeveloped novelty claim**: The novelty of "sentential features" for event linking is not sufficiently established. While the paper cites prior work on sentential features for relation extraction, it fails to explicitly contrast its approach with existing event-linking methods that use similar techniques (e.g., Chen et al., 2015). The comparison in Table 3 is limited to a small set of baselines, omitting key recent work.  
3. **ACE++ dataset description is inadequate**: The paper claims ACE++ is "publicly available" but provides no details on its creation process or event-type coverage. This is critical for validating the claim that their model generalizes better than resource-dependent approaches. Without knowing whether ACE++ includes diverse event types, the comparison to prior work (e.g., Liu et al.) lacks credibility.  
4. **Missing evaluation on gold-standard mentions**: The paper evaluates on gold event mentions, but does not report performance on automatically extracted mentions (e.g., using a standard IE pipeline). This omission limits the practical relevance of the results, as real-world systems would need to handle extraction errors.  

**Questions to Authors**  
1. How was ACE++ constructed? Specifically: (a) What event types does it cover (beyond ACE 2005's Conflict.Attack, Movement.Transport, Life.Die), and (c) How were the additional 1,351 web documents annotated? This is essential to validate claims about domain adaptability.  
2. Why did the authors choose CNNs over RNNs/LSTMs for feature extraction? Given the marginal performance gains, a comparison of architectures would strengthen the methodological justification.  
3. In Table 5, pairwise features alone (row 4) achieve 74.97 F1. How were these features selected? Were other feature combinations tested, and if so, why were these chosen as the baseline?  
4. The error analysis identifies pronoun resolution as a key weakness. Could the model be augmented with a standard coreference resolver (e.g., CoreNLP) for pronoun handling, and how might this impact performance?  

**Additional Comments**  
- The paper should clarify whether ACE++ is truly publicly available (e.g., link to a repository) to enable reproducibility.  
- The claim that "external features restrict models to domains where such sources are available" would be stronger with a concrete example of a prior system failing on ACE++ while the proposed model succeeds.  
- The model's performance on the "all singletons" baseline (Table 6) is high (47.53 F1), suggesting that the task is inherently biased toward singleton mentions. This should be acknowledged as a limitation.

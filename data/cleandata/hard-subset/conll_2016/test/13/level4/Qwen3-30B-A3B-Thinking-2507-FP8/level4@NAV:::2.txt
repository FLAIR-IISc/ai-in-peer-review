

This paper introduces a model for event entity linking that replaces external knowledge sources with sentential features derived from CNNs. The authors propose a two-stage approach: first, learning event mention representations, and second, computing coreference scores between event entity mentions. The paper is well-written, clearly structured, and highly accessible. I particularly appreciate the analysis of argument sharing in event coreferences using the ACE corpus, as well as the dataset size analysis that motivated the creation of the ACE++ dataset. However, several significant concerns require resolution:  
- The rationale for using CNNs to generate mention representations lacks depth. The authors do not justify their choice of CNN over alternatives (e.g., RNNs) or provide comparative results. Given the modest F1 improvements (2–3 points) over state-of-the-art, stronger evidence is needed to substantiate the architectural superiority.  
- The novelty of employing sentential features for event linking is unclear, as CNNs for classification tasks are not new. The authors should explicitly contrast their method with existing continuous-space approaches for event linking and justify the selection of comparison models in Table 3.  
- The methodology for constructing the ACE++ dataset is unspecified. Since ACE’s limitation in event type diversity creates domain bias, it is critical to detail the event types covered by ACE++. This transparency would strengthen the claim in Section 5.1 that "other approaches are strongly tied to domains where semantic features are available… our approach does not depend on such resources." Furthermore, the dataset should be made publicly available to enable future reproducibility and comparison.  

Minor concerns:  
- Including model performance without gold references in Table 3 would enhance the evaluation.  
- Exploring integration with a vanilla coreference system (e.g., CoreNLP linking "It" to "bombing" as in line 687) could provide a stronger baseline for event entity linking.  
- Given ACE’s small scale, testing on additional resources (e.g., EventCorefBank) is essential to validate robustness and motivate simultaneous entity/event coreference resolution.  
- Table 5 demonstrates the efficacy of pairwise features, raising questions about feature selection. Reporting which features contributed most and the rationale for the final feature set would improve methodological clarity.

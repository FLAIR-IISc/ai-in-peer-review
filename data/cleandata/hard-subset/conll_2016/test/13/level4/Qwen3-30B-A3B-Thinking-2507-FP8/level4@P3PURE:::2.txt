

This paper proposes a model for event entity linking that replaces external knowledge sources with sentential features derived from CNNs. The approach employs a two-part architecture: the first component learns event mention representations, and the second computes coreference scores between event mentions. The paper is well-written, clearly presented, and easy to follow. I particularly appreciate the analysis of argument sharing in event coreferences using the ACE corpus, as well as the dataset size analysis that motivates the creation of the ACE++ dataset. However, several major issues require clarification:  

- The authors fail to justify the choice of CNNs for mention representation generation. No discussion of alternatives (e.g., RNNs) or comparative analysis of their pros/cons is provided. Given the marginal improvements (2–3 F1 points) over state-of-the-art, stronger evidence is needed to demonstrate that this architecture is genuinely superior.  
- The novelty of using sentential features for event linking is unclear, as CNN-based classification with similar approaches is not new. The authors should explicitly contrast their method with existing continuous-space approaches for event linking and justify the selection of comparison methods in Table 3, which is insufficiently thorough.  
- The collection methodology for ACE++ is not described. Given ACE’s well-documented limitation of sparse event types (leading to domain bias), it is critical to specify the event types covered by ACE++. This is essential to substantiate the claim in Section 5.1 that "other approaches are heavily domain-dependent... our approach does not depend on resources with restricted [coverage]." Demonstrating that prior methods fail on ACE++ while the proposed model succeeds would strengthen this argument. Additionally, the authors should consider public release of ACE++ to enable future comparisons.  

Minor concerns:  
- Table 3 should include performance without gold event mentions to evaluate practical utility.  
- The authors should explore augmenting the model with a standard coreference resolution system (e.g., CoreNLP’s standard coreference resolution already links "It" to "bombing" in the example at line 687), which could inform a stronger baseline.  
- Given ACE’s small size, testing on additional resources (e.g., EventCorefBank) is essential to validate the model’s robustness and further motivate simultaneous coreference resolution for entities and events.  
- Table 5 indicates pairwise features are highly effective, suggesting feature engineering remains crucial for competitiveness on ACE-scale data. The authors should clarify which features were most impactful, how the current feature set was selected, and what alternatives were evaluated.



Summary: This paper introduces a novel model for event coreference resolution that replaces reliance on external semantic features (e.g., WordNet, FrameNet) with sentential features derived from convolutional neural networks (CNNs). The authors propose a two-stage architecture: (1) CNNs generate latent representations of event mentions by processing their context sentences, and (2) these representations, augmented with lexical and pairwise features, feed into a trainable similarity function for coreference scoring. The model achieves state-of-the-art performance on ACE and ACE++ datasets, with ACE++ being publicly available. Key contributions (ordered by strength):  
1. A resource-agnostic approach to event linking that eliminates dependency on domain-limited external knowledge bases while maintaining SOTA performance.  
2. Comprehensive analysis demonstrating the necessity of sentential features through ablation studies (Table 5) and error analysis identifying pronoun resolution as a critical challenge.  
3. Publicly released ACE++ dataset, expanding the ACE 2005 corpus to address dataset size limitations and enabling reproducibility.  

Strengths:  
1. The paper compellingly establishes the problem’s significance through empirical analysis of the ACE 2005 corpus, revealing 23% of event mentions require cross-sentential resolution—a critical gap previously overlooked in prior work.  
2. The ablation study (Table 5) rigorously validates the necessity of all feature classes (sentential, lexical, pairwise), showing that removing any component degrades performance, particularly highlighting the 13.8% F1 drop when excluding sentential features.  
3. The ACE++ dataset expansion (1950 documents vs. ACE’s 599) provides robust evidence of scalability, with a 8.0% absolute F1 improvement over ACE (Table 4), directly supporting the claim that data volume mitigates resource dependency.  

Weaknesses:  
1. The CNN architecture lacks justification compared to alternative sequence models (e.g., RNNs, LSTMs), despite the marginal SOTA gains (2–3 F1 points). No ablation or comparison to non-CNN baselines is provided, weakening the claim that CNNs are uniquely suited for this task.  
2. The novelty of using sentential features for event linking is inadequately differentiated from prior neural approaches (e.g., Zeng et al., 2014; Chen et al., 2015), which also employ CNNs for relation/event extraction. The comparison in Table 3 omits key continuous-space methods, obscuring the technical contribution.  
3. The ACE++ dataset’s construction details (e.g., event type coverage, annotation guidelines) are absent, undermining the claim that the model generalizes beyond ACE’s domain limitations. Without this, the assertion that "earlier methods fail on datasets where semantic features are unavailable" cannot be substantiated.  

Questions to Authors:  
1. What event types are included in ACE++ beyond ACE 2005’s Conflict.Attack/Movement.Transport/Life.Die, and how does this mitigate ACE’s known bias toward specific event categories?  
2. Could the model’s pronoun resolution failure (e.g., "It" → "bombing" in Section 6.2) be addressed by integrating a pre-trained coreference system (e.g., CoreNLP) as a feature, as suggested in the draft review?  
3. Given that pairwise features contribute significantly to performance (Table 5), what criteria guided the selection of these features, and what alternatives were discarded during development?

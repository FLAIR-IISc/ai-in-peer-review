

This paper introduces a model for event entity linking that replaces external knowledge sources with sentential features derived from CNNs. The approach employs a two-part architecture: the first component learns representations for event mentions, while the second computes coreference scores between pairs of event entities. The paper is well-written, accessible, and presents a clear methodology. I particularly appreciate the analysis of argument sharing in ACE event coreferences and the strong motivation for constructing the ACE++ dataset through dataset size analysis. However, several critical issues require attention before publication:

First, the authors provide insufficient justification for using CNNs to generate mention representations. The choice of CNN over alternatives like RNNs is unexplained, and no comparative analysis is presented. Given the marginal improvements (2-3 F1 points) over state-of-the-art methods, stronger evidence is needed to demonstrate that this architecture genuinely outperforms simpler baselines.

Second, the novelty of using sentential features for event linking is unclear. Applying CNNs to classification tasks with contextual features is not novel, and the paper fails to explicitly contrast its approach with existing continuous-space methods for event linking. The model comparisons in Table 3 are incomplete and lack sufficient coverage of relevant prior work.

Third, the construction methodology for the ACE++ dataset is not described. ACE's narrow event type coverage creates a narrow, biased benchmark, yet ACE++'s expanded event types remain unspecified. This omission weakens the claim in Section 5.1 that "our approach does not depend on resources with restricted [semantic features]." To substantiate this, the authors must demonstrate that their model outperforms prior methods on ACE++ while showing those methods fail on the same dataset. Additionally, public release of ACE++ would enable future reproducibility and comparison.

Minor concerns include:
- Including model performance without gold references in Table 3 would strengthen the evaluation.
- Exploring integration with a standard coreference system (e.g., CoreNLP's ability to link "It" to "bombing" as shown in line 687) would provide a more robust baseline.
- Given ACE's limited scale, testing on complementary resources like EventCorefBank alongside ACE is essential for broader validation. This also better supports the argument for jointly modeling entity and event coreference.
- Table 5 demonstrates the effectiveness of pairwise features, suggesting feature engineering remains critical for competitive performance on ACE-scale data. The review should clarify which features were most impactful, how the current feature set was selected, and what alternatives were evaluated.

These points are essential for establishing the paper's contribution, validity, and broader applicability. Addressing them will significantly strengthen the work.

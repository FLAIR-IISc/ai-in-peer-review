- Proposes a CNN-based neural model for event coreference that uses sentential features and avoids external semantic resources, claiming state-of-the-art performance.  
- Shows sentential features alone can match or exceed prior resource-heavy approaches.  
- Finds sentential features add significant gains over lexical and pairwise features; notes 55.42% of coreferential mentions are >5 sentences apart, highlighting cross-sentential need.  
- Error analysis identifies pronoun resolution and annotation ambiguity as primary challenges and future directions.  

- Strong empirical validation: SOTA results on ACE and ACE++ without external semantic resources.  
- Comprehensive ablation: pairwise, trigger-local, and sentential features each contribute; recall improves with more data (75.16%→83.21%).  
- Insightful error breakdown: pronoun issues and 17% annotation mismatch between false negatives and false positives.  

- Uses gold-standard triggers instead of predicted ones, creating an unfair comparison and inflating reported performance.  
- Employs a non-standard ACE train/test split, hindering fair comparison and reproducibility.  
- ACE++ dataset is not publicly released, preventing independent validation and comparisons.  
- Omits evaluation with predicted triggers, weakening claims about real-world applicability and independence from external resources.  

- Asks how relative position embeddings are initialized, trained, and whether position vectors are shared across contexts.  
- Questions the choice of one-word lexical window around triggers and how performance varies with different window sizes.  
- Requests details on encoding non-binary pairwise features (e.g., sentence distance): discretized, embedded, or continuous.  
- Asks why the event-mention representation v_e (Eq.3) is computed but not used in the coreference scoring, which instead uses v_{sent+lex}.  

- Notes the square activation for similarity (Eq.4) is interesting but lacks justification and discussion of novelty/generalizability.  
- Requests clarification whether pre-trained GloVe embeddings count as "external resources" or are considered domain-agnostic.  
- Argues the "same type" baseline is flawed (type agreement ≠ coreference) and should be explicitly criticized.

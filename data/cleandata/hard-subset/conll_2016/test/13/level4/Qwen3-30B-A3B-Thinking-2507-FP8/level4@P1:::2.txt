

Summary: The paper proposes a model for event linking that replaces external semantic features (e.g., WordNet, FrameNet) with sentential features derived from convolutional neural networks (CNNs). The key contributions are: (1) a CNN-based approach for generating event mention representations using full-sentence context, eliminating reliance on domain-limited external resources; (2) state-of-the-art performance on two datasets (ACE 2005 and a new ACE++ dataset), with ACE++ publicly released; (3) an error analysis identifying persistent challenges like pronoun resolution. The model processes event mentions via two steps: (a) CNNs generate sentential representations from context, augmented with lexical features; (b) pairwise features and these representations feed into a similarity function for coreference scoring.

Strengths:
1. **Novel resource-agnostic approach**: The model achieves SOTA performance without external semantic resources—a significant practical advantage over prior work (e.g., Liu et al., 2014) that depends on domain-specific features like WordNet. This directly addresses a key limitation in event linking systems.
2. **Strong empirical validation**: The ACE++ dataset expansion (1,950 documents vs. ACE’s 599) demonstrates robust generalization (83.21% recall vs. 75.16% on ACE), and ablation studies (Table 5) conclusively show all feature classes (sentential, lexical, pairwise) contribute to performance.
3. **Insightful error analysis**: The manual analysis of 200 errors (100 FP/100 FN) reveals critical challenges (e.g., pronoun resolution) and ambiguous annotations, providing clear directions for future work. The observation that 17% of FNs involve ambiguous human annotations contextualizes the model’s limitations.

Weaknesses:
1. **Insufficient justification for CNN choice**: The paper fails to compare CNNs against standard alternatives (e.g., RNNs) or justify why CNNs are superior for this task. The marginal SOTA gains (2–3 F1 points over Liu et al., 2014) require stronger evidence that CNNs—not just the feature design—drive the improvement. *Please provide a comparison of CNN vs. RNN (or other architectures) for sentence encoding in the revision.*
2. **Lack of novelty clarification**: The claim that "sentential features from CNNs" are novel for event linking conflates two ideas: (a) using CNNs for sentence encoding (common in NLP), and (b) applying this specifically to event linking. *Explicitly contrast with prior work using similar architectures (e.g., Zeng et al., 2014; Chen et al., 2015) for relation/event extraction, and clarify how this work’s approach differs.*
3. **ACE++ dataset opacity**: ACE’s limited event types (only 3 core types: Conflict.Attack, Movement.Transport, Life.Die) likely bias results. The paper claims their approach "does not depend on resources with restricted domain availability," but this requires evidence that prior methods fail on ACE++ while theirs succeeds. *Specify ACE++’s event type coverage and report SOTA comparisons on ACE++ (not just ACE) to validate the domain-agnostic claim.* Additionally, ACE++ must be publicly released for reproducibility.
4. **Missing gold-mention ablation**: The evaluation (Table 3) uses gold event mentions, but real-world systems require end-to-end processing. *Include performance without gold mentions to assess practical viability.*

Questions to Authors:
- How does the model’s performance change when using predicted (vs. gold) event triggers? This is critical for real-world deployment.
- The error analysis cites pronoun resolution failures (e.g., "It’s crazy that we’re bombing Iraq. It sickens me"). *Can you test whether integrating a standard coreference resolver (e.g., CoreNLP) for pronouns improves results?*
- The ACE++ dataset’s event type distribution: *Does it cover more diverse types beyond ACE’s 3 core types (e.g., Business, Justice)?* This is essential for validating the "domain-agnostic" claim.

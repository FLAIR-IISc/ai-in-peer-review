

Summary: This paper addresses event coreference resolution by introducing a neural model that leverages sentential features from convolutional neural networks (CNNs) instead of external semantic resources. The authors propose a two-step architecture: first, CNNs process event mentions and their context to generate latent representations; second, these representations combined with lexical and pairwise features feed into a similarity function for coreference scoring. The model achieves state-of-the-art results on ACE and ACE++ datasets without relying on domain-specific knowledge bases.

Main contributions (ordered by strength):
1. A novel CNN-based architecture for event linking that generates sentential features from raw text, eliminating dependence on external semantic resources like WordNet or FrameNet while maintaining SOTA performance.
2. Empirical validation that sentential features (capturing context beyond trigger words) are critical for event linking, demonstrated through ablation studies showing significant performance drops when removing these features.
3. Comprehensive error analysis identifying unresolved challenges (e.g., pronoun resolution) and dataset limitations, providing clear directions for future work.

Strengths:
1. The model’s resource independence is a major advancement. By avoiding external semantic features, it enables deployment in domains lacking curated knowledge bases (e.g., specialized medical or legal texts), directly addressing a key limitation of prior work (e.g., Liu et al. 2014). The ACE++ results (76.90 F1) match SOTA systems while using only lexical/sentential features.
2. The ablation study (Table 5) conclusively demonstrates the necessity of sentential features. Removing them reduces F1 by 15.6% (76.90 → 61.28), whereas pairwise features alone yield poor recall (56.70), proving that context beyond lexical overlap is essential for event linking.
3. The error analysis reveals systematic limitations (e.g., 17% of false negatives involve pronoun resolution), which is more valuable than superficial claims of "SOTA performance." This provides concrete, actionable research directions for the community.

Weaknesses:
1. The comparison to Liu et al. (2014) is unfair due to inconsistent preprocessing. Liu uses predicted triggers (with error propagation), while this work uses gold triggers. This invalidates the F1 comparison (Table 3), as Liu’s system would likely underperform with predicted triggers. The authors should re-evaluate with predicted triggers for a valid comparison.
2. The ACE++ dataset is not publicly released, despite the authors claiming it is "publicly available" (Section 1). This impedes reproducibility and prevents fair comparison with other systems on this dataset, contradicting the stated contribution of providing a public resource.
3. The position embedding method (Section 3.1) is ambiguously described. It states embeddings are generated "similar to word embeddings" but does not clarify initialization (random vs. lexicalized) or whether relative position features are shared across mentions. This affects model interpretability and replicability.

Questions to Authors:
1. How would the model’s performance change if triggered by predicted event mentions (instead of gold triggers), and how does this compare to Liu et al. (2014)’s reported results?
2. Will the ACE++ dataset be released publicly? If not, please justify why the claim of "one publicly available dataset" (Section 1) is accurate when ACE++ is used for main results.
3. The error analysis identifies pronoun resolution as a key challenge. Could the model be extended to incorporate discourse-level features (e.g., pronoun antecedent heuristics) to address this specific limitation?

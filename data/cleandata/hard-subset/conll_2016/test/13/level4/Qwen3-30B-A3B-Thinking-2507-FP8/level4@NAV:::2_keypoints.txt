- Paper introduces an event entity linking model using sentential CNN features and a two-stage approach (learn mention representations, then compute coreference scores).  
- Manuscript is well-written, clearly structured, and accessible.  
- Positive: insightful analysis of argument sharing in event coreferences using the ACE corpus.  
- Positive: dataset size analysis motivates creation of the ACE++ dataset.  
- Major concern: Rationale for using CNNs is weak—no justification versus alternatives (e.g., RNNs) and no comparative experiments.  
- Major concern: Modest F1 gains (2–3 points) require stronger evidence to claim architectural superiority.  
- Major concern: Novelty unclear—using sentential features/CNNs for classification is not new; must contrast with existing continuous-space event linking approaches.  
- Major concern: Choice of baselines in Table 3 needs explicit justification.  
- Major concern: ACE++ dataset construction is unspecified; must list event types covered and address ACE domain bias.  
- Major concern: ACE++ should be made public to enable reproducibility and comparison.  
- Minor: Include model performance without gold references in Table 3.  
- Minor: Evaluate integration with a vanilla coreference system (e.g., CoreNLP) as a baseline.  
- Minor: Test on additional datasets (e.g., EventCorefBank) to validate robustness and motivate joint entity/event coreference.  
- Minor: Table 5 shows pairwise feature utility—report feature importance and rationale for final feature selection.

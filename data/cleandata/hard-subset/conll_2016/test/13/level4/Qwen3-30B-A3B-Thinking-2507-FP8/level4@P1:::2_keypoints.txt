- Proposes a CNN-based event linking model that uses full-sentence sentential features instead of external semantic resources, combined with pairwise similarity scoring for coreference.
- Key contribution: CNN-generated event mention representations using full-sentence context to avoid reliance on domain-limited resources.
- Key contribution: Achieves state-of-the-art results on ACE 2005 and a newly released ACE++ dataset (ACE++ publicly released).
- Key contribution: Provides an error analysis highlighting persistent challenges (e.g., pronoun resolution) and ambiguous annotations.
- Strength: Novel resource-agnostic approach achieves SOTA without external semantic resources like WordNet/FrameNet.
- Strength: Strong empirical validation with larger ACE++ dataset (1,950 docs) and ablation studies showing sentential, lexical, and pairwise features all matter.
- Strength: Insightful manual error analysis (200 errors) revealing pronoun resolution issues and that 17% of FNs stem from ambiguous annotations.
- Weakness: Insufficient justification/comparison for choosing CNNs over alternatives (e.g., RNNs); need CNN vs. RNN ablation to show architecture matters.
- Weakness: Novelty claim unclear—using CNNs for sentence encoding is common; must contrast with prior work (e.g., Zeng et al., 2014; Chen et al., 2015) and clarify differences.
- Weakness: ACE++ dataset details opaque—need event-type coverage, SOTA comparisons on ACE++, and public release to support domain-agnostic claims.
- Weakness: Evaluation uses gold event mentions; missing end-to-end (predicted mentions) results to assess practical viability.
- Questions: How does performance change with predicted vs. gold event triggers?
- Questions: Would integrating a standard coreference resolver for pronoun resolution (e.g., CoreNLP) improve results?
- Questions: Does ACE++ cover more diverse event types beyond ACE’s three core types (e.g., Business, Justice)?

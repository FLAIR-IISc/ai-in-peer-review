- Paper proposes an event entity linking model using sentential CNN-based mention representations and a separate coreference scoring component.  
- Strengths: well-written, well-organized, insightful analysis of argument sharing on ACE, and motivating dataset size analysis for ACE++.  
- Major concern: insufficient justification or critical analysis for choosing CNNs to generate mention representations.  
- Lacks comparative experiments against alternative architectures (e.g., RNNs) to validate CNN superiority given modest 2–3 F1 gains.  
- Novelty concern: use of sentential features and CNNs for event linking is not clearly novel; needs explicit comparison to existing continuous-space methods.  
- Comparative analysis (Table 3) appears incomplete and should be expanded.  
- ACE++ dataset construction methodology is not described and must be detailed.  
- Must disclose event types included in ACE++ to address ACE’s event-type bias and support domain-dependence claims.  
- Authors should show cases/datasets where their approach outperforms others that struggle under domain dependence.  
- Recommend publicly releasing ACE++ to enable reproducibility and fair comparison.  
- Minor: report model results from Table 3 without gold references for completeness.  
- Minor: explore integrating the model with standard coreference systems (e.g., CoreNLP) to build stronger baselines.  
- Minor: evaluate on additional resources (e.g., EventCorefBank) due to ACE’s limited size and to support joint entity-event coreference claims.  
- Minor: feature engineering remains important; provide rationale for chosen pairwise features and results from other feature configurations.

- Paper proposes an event linking model using sentential CNN features with a two-part architecture: event mention representation and pairwise coreference scoring.
- Paper is well-written and clearly presented.
- Analysis of ACE corpus on argument sharing between coreferential events is insightful.
- Motivation for creating the ACE++ dataset to study dataset-size effects is compelling.
- Lacks thorough justification for choosing CNNs over alternatives (e.g., RNNs); pros/cons discussion is missing.
- Modest performance gain (2–3 F1 points) over prior work requires stronger evidence that the architecture is superior.
- Novelty of using sentential features is not well-established; must compare explicitly to existing continuous-space event linking methods.
- Selection of competing methods in Table 3 appears incomplete and should be expanded.
- Details on ACE++ creation are absent; need event-type coverage, bias discussion, and public release to support claims about domain dependence.
- Table 3 should include model performance without gold-standard event mentions for a more realistic evaluation.
- Suggest exploring integration with standard coreference systems (e.g., CoreNLP) as a baseline for pronoun resolution cases.
- Model should be evaluated on additional datasets (e.g., EventCorefBank) to demonstrate robustness and generalizability.
- Table 5 shows pairwise features matter; authors should identify the most impactful features and describe the feature selection/process including failed attempts.

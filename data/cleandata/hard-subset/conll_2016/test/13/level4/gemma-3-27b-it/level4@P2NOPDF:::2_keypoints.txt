- Introduces a two-stage CNN-based model for event entity linking using sentential features (mention representations + pairwise coreference scoring).
- Paper is well-written and clear; ACE corpus analysis of argument sharing is insightful.
- Authors convincingly argue ACE’s small size motivates creating ACE++, but construction methodology for ACE++ is not described.
- No robust justification for choosing CNNs over alternatives (e.g., RNNs) and no comparative analysis to support the modest 2–3 F1 point gains.
- Novelty of using sentential features and CNNs for event linking is not clearly articulated; related continuous-space methods are not adequately compared.
- Table 3’s comparisons are incomplete and should include a broader set of relevant prior methods.
- ACE++ needs detailed reporting of covered event types to address potential bias from ACE’s limited taxonomy.
- Authors should demonstrate cases where resource-dependent methods fail but their approach succeeds, and release ACE++ publicly.
- Include results in Table 3 without gold references to provide a fuller evaluation.
- Explore integration with standard coreference systems (e.g., CoreNLP) as a baseline or augmentation, as suggested by examples.
- Evaluate on additional datasets (recommend EventCorefBank) given ACE’s size limitations.
- Discuss the strong contribution of pairwise features: rationale for chosen features, which were most effective, and other feature configurations tried.

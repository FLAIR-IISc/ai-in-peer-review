This paper presents a model for event entity linking, proposing the use of sentential features from CNNs in place of the external knowledge sources used by previous methods. The model consists of two parts: the first learns a representation for event mentions, and the second calculates a coreference score given two event entity mentions.

The paper is well-written, clearly presented, and easy to follow. I particularly appreciate the analysis of argument sharing between event coreferences on the ACE corpus. Furthermore, the analysis of dataset size impact provides strong motivation for the creation of their ACE++ dataset. However, several major issues need to be addressed:

- The authors do not adequately motivate or analyze the advantages and disadvantages of using CNNs for generating mention representations. The rationale for choosing a CNN is not explained, and there is no comparison to alternative models, such as an RNN. Given that the improvement their model achieves over state-of-the-art methods is only 2 or 3 F1 points, stronger evidence is needed to demonstrate the superiority of this architecture.
- It is unclear what is novel about tackling event linking with sentential features, as applying CNNs in this way for classification is not a new approach. The authors should explicitly highlight and compare their work to existing continuous space methods for event linking. The selection of methods in Table 3 is not comprehensive enough.
- There is no information provided regarding the collection process for the ACE++ dataset. A key limitation of the ACE dataset is its limited number of event types, which can lead to constrained and biased results. It is important to clarify the event types covered by ACE++. This information would also strengthen the claim in Section 5.1 that “other approaches are strongly tied to the domain where these semantic features are available…our approach does not depend on resources with restricted…”; to support this claim, the authors need to demonstrate cases where earlier methods fail but their approach succeeds. Additionally, to facilitate future comparisons, the authors should consider making this dataset publicly available.

Some minor issues:

- I would have liked to see the model’s performance without gold references included in Table 3.
- It would be interesting to explore whether and how this model could be augmented with a standard coreference resolution system. For example, the off-the-shelf CoreNLP system readily links “It” to “bombing” in line 687, and this capability could be leveraged in an event entity linking baseline.
- Given the relatively small size of the ACE dataset, a compelling model requires testing on additional available resources. This would further motivate a combined approach to entity and event coreference. Specifically, I believe parallel testing on EventCorefBank alongside ACE is essential.
- Table 5 shows that pairwise features are quite effective, suggesting that feature engineering remains crucial for achieving a competitive model (at least on the scale of the ACE dataset). It would be helpful to know which features were most effective and to report the rationale behind the current feature set’s selection, as well as what other features were tried.

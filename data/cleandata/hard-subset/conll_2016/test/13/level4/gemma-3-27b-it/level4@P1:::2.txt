This paper presents a model for the task of event linking, proposing the use of sentential features from Convolutional Neural Networks (CNNs) in place of external knowledge sources used in earlier methods. They train a two-part model: the first part learns an event mention representation, and the second part learns to calculate a coreference score given two event mentions.

The paper is well-written, well-presented, and easy to follow. I particularly appreciate the analysis of the ACE corpus regarding argument sharing between event coreferences. Furthermore, the analysis on dataset size impact provides strong motivation for creating their ACE++ dataset. However, there are several key issues that need to be addressed:

- The authors do not adequately motivate and analyze the advantages and disadvantages of using CNNs for generating mention representations. They do not discuss *why* they chose CNNs, and there are no comparisons to alternative models (e.g., a Recurrent Neural Network). Given that the improvement their model achieves over state-of-the-art methods is only 2-3 points on F1 score, more evidence is needed to demonstrate the superiority of this architecture.
- It is unclear what is novel about tackling event linking with sentential features, considering that using CNNs in this fashion for a classification task is not new. The authors should explicitly point out and primarily compare their approach to existing continuous space methods for event linking. The selection of methods in Table 3 is not comprehensive enough.
- There is no information regarding how the ACE++ dataset was collected. A major limitation of the ACE dataset is its limited number of event types, which can make it constrained and biased. It is important to know what event types ACE++ covers. This information would also help support the claim in Section 5.1 that “other approaches are strongly tied to domains where these semantic features are available…our approach does not depend on resources with restricted…”; you need to demonstrate that earlier methods fail on a dataset where your method succeeds. Furthermore, to enable meaningful comparison in future work, the authors should consider making this dataset publicly available.

Some minor issues:

- I would have liked to see the performance of your model without gold-standard event mentions in Table 3.
- It would be valuable to explore how this model can, or cannot, be augmented with a standard coreference resolution system. For the specific example in line 687, an off-the-shelf CoreNLP system readily links ‘It’ to ‘bombing’, which could potentially be leveraged in an event entity linking baseline.
- Given the relatively small size of the ACE dataset, a compelling model should be tested on other available resources. This further motivates working on entity and event coreference simultaneously. I also believe that testing on EventCorefBank in parallel with ACE is essential.
- Table 5 shows that pairwise features are quite effective, indicating that feature engineering may still be crucial for achieving a competitive model (at least on the scale of the ACE dataset). It would be interesting to know which features were most effective and how the current set was chosen, as well as what other features were tried.

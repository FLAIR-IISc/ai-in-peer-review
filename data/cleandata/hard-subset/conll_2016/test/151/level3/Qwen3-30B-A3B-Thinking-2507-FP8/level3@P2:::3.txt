

# Review for "Parsing for Universal Dependencies without training"

## Summary and Contributions
This paper presents UDP, an unsupervised dependency parser for Universal Dependencies (UD) that requires no training data. The system relies solely on POS tags and a small set of dependency head rules, using personalized PageRank to rank content words and a two-step decoding process to enforce UD's constraints on function words being leaves. The main contributions, as I see them, are:

1. A linguistically grounded unsupervised parsing approach that fully leverages UD's formalism constraints (single-rooted trees, function words as leaves), making it distinct from previous unsupervised parsers that don't account for UD-specific conventions.
2. A practical method for estimating adposition direction directly from test data, eliminating the need for language-specific parameters.
3. A demonstration of competitive performance with supervised cross-lingual transfer parsers while maintaining robustness to domain changes across languages.

## Strengths
1. **Linguistically sound approach**: The paper successfully leverages UD's formal constraints (function words as leaves, single-rooted trees) in a way that previous unsupervised parsers haven't fully exploited. This makes the approach more theoretically grounded and potentially more interpretable than purely data-driven methods. The parser's robustness to domain changes (Table 7) demonstrates the practical value of this constraint-based approach.

2. **Competitive performance with minimal resources**: UDP achieves impressive results without training data, outperforming baseline rule-based systems by 4% on average and matching supervised multi-source delexicalized parsers (MSD) within 6.4% on average (Table 5). This demonstrates the viability of constraint-based parsing for UD, especially valuable for truly low-resource languages.

3. **Practical domain robustness**: The paper provides strong evidence that UDP is significantly more robust to domain shifts than data-driven approaches (Table 6), with lower standard deviations across domains. This is particularly valuable for real-world applications where domain shifts are common, and the authors effectively demonstrate this through careful cross-domain evaluation.

## Weaknesses
1. **Limited novelty**: The core approach closely mirrors Søgaard (2012), with the main additions being minor adaptations to UD constraints and the use of personalized PageRank for main predicate identification. The paper fails to sufficiently justify how these modifications constitute a significant advance over the prior work. The claim of "unprecedented unsupervised parsing for UD" is overstated given the close relationship to existing work.

2. **Arbitrary parameter choices**: The personalization weight of 5 for the main predicate (Section 3.1) is presented without justification or ablation study. The paper states "we chose this value incrementally," but provides no evidence that this specific value is optimal or that different values would not yield significantly better results. This undermines the methodological rigor of the approach.

3. **Insufficient analysis of POS dependency**: The paper doesn't report POS tagging accuracy in the predicted POS scenario (UDPP), making it difficult to assess whether the small performance gap between UDPG and UDPP (2.2% drop) is due to parsing limitations or POS tagging errors. A correlation analysis between POS accuracy and parsing performance would strengthen the claims about robustness.

4. **Lack of visualizations**: The results are presented in tables only, making it difficult to interpret trends across languages and setups. A visualization of the UAS scores across languages (e.g., bar charts showing UDP vs. MSD for each language) would make the comparative performance more immediately apparent and support the claims about robustness.

5. **Underexplored personalization technique**: The personalized PageRank approach is mentioned as a key innovation but is only applied to the main predicate. The paper doesn't explore how personalization might be extended to other linguistic phenomena or whether it could be integrated more deeply into the parsing process. This represents a missed opportunity to demonstrate the broader potential of the technique.

## Questions to Authors
1. Could you provide the POS tagging accuracy for the TnT tagger used in the UDPP setup? This would help clarify whether the parsing performance degradation correlates with POS tagging errors.

2. Why was the personalization weight specifically set to 5? An ablation study showing how performance varies with different personalization weights would strengthen the methodological foundation.

3. The paper claims UDP "compares favorably to delexicalized parsing" on Hindi (Table 5), but the UAS difference is only 0.1%. Is this statistically significant, or does it represent a marginal advantage?

4. How would the approach handle languages with non-standard UD conventions (e.g., where function words are not consistently leaves)? Would the current rule set need substantial modification?

5. The paper mentions "multiword expressions" as future work. How would the current two-step decoding approach handle such phenomena, and what modifications would be necessary?

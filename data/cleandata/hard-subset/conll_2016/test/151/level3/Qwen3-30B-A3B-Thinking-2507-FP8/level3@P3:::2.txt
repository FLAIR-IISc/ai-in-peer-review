

# Review of "Parsing for Universal Dependencies without training"

## Summary and Contributions

This paper presents UDP, an unsupervised dependency parser for Universal Dependencies that requires no training data. The main contributions, ordered by strength, are:

1. A novel unsupervised parsing approach that achieves competitive performance with delexicalized transfer systems while being completely training-free, representing a substantial contribution to unsupervised parsing research.
2. A principled adaptation of dependency head rules to UD's specific formalism constraints, including the critical restriction that function words are always leaves (dependents), which is a significant departure from previous rule-aided unsupervised parsing approaches.
3. The use of personalized PageRank to improve main predicate identification and a two-step decoding process that ensures function words remain leaves, making the system linguistically sound and robust across languages.

## Strengths

1. **Substantial contribution to unsupervised parsing research**: UDP achieves remarkable results without any training data, competing closely with supervised delexicalized transfer systems (MSD) across 32 languages. The paper demonstrates that a linguistically principled approach, leveraging UD's formalism constraints, can outperform purely data-driven methods in many cases. The average UAS difference between UDPG (gold POS) and MSDG (gold POS) is only 6.4%, with UDPG even outperforming MSDG on Hindi. This represents a significant advance in unsupervised parsing, particularly for low-resource languages.

2. **Strong empirical evaluation and robustness analysis**: The paper provides comprehensive evaluation across three POS scenarios (gold, predicted, and naive content/function) and multiple domains. The cross-domain analysis is particularly compelling, showing UDP's superior robustness to domain shifts compared to MSD (standard deviation of 1.8 vs. 4.5 for gold POS). This robustness stems from the system's reliance on linguistic constraints rather than language-specific data patterns, making it a valuable baseline for cross-lingual parsing.

3. **Clear, well-motivated methodology**: The paper excels in explaining the intuition behind each design choice. The personalized PageRank approach (Section 3.1) is particularly well-justified, with the authors clearly articulating why ranking content words matters and how personalization improves main predicate identification. The two-step decoding process (Section 3.3) effectively enforces UD's constraint that function words are leaves, which is a key innovation that previous unsupervised parsers haven't properly addressed.

## Weaknesses

1. **Limited handling of language-specific phenomena**: While the paper demonstrates strong cross-lingual performance, it doesn't adequately address how the system would handle languages with radically different syntactic structures (e.g., verb-final languages like Japanese or Korean, which were excluded from the evaluation). The personalization strategy assumes the first verb is the main predicate, which works well for many languages but may fail for verb-final languages. The paper should discuss how this limitation might be addressed for such languages.

2. **Overreliance on POS tag quality**: While the paper does evaluate the system with predicted POS tags, the results show a 2.2% drop in UAS (from UDPG to UDPP) compared to a 2.7% drop for MSD. However, this assumes a 94.1% POS accuracy from TnT, which may not hold for all languages. The paper doesn't explore how the system would perform with lower POS accuracy (e.g., below 90%), which would be more realistic for truly low-resource languages.

3. **Limited exploration of error patterns**: The paper provides some error analysis (Section 6.2), but it doesn't thoroughly investigate why the system fails on specific POS tags (e.g., PUNCT and CONJ). Understanding these failure modes would be valuable for future improvements. The paper also doesn't discuss how the system would handle multiword expressions, coordination, or proper names, which are important for practical parsing applications.

## Questions to Authors

1. How would you modify the system to handle verb-final languages more effectively, particularly when the main predicate isn't the first verb? Would you consider alternative personalization strategies for different language types?

2. The paper shows UDP's robustness to domain shifts, but it doesn't explore how the system might perform on completely unseen domains (e.g., social media text with non-standard grammar). How would you adapt the system for such challenging domains?

3. The authors mention future work on extending the parser to handle multiword expressions. Could you elaborate on how this extension might work within the current framework, particularly given the system's reliance on simple head rules and no training data?

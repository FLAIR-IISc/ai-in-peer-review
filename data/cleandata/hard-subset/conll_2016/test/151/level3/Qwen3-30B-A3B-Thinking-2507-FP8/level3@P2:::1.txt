

# Review for CONLL_2016

## Summary and Contributions
The paper presents UDP, an unsupervised dependency parser for Universal Dependencies that requires no training data and relies solely on POS constraints. The core contribution is a deterministic parsing algorithm that uses a small set of dependency head rules, personalized PageRank for ranking content words, and a two-step decoding process to ensure function words are leaves. The main contributions as I see them are:

1. A systematic adaptation of dependency head rules to the Universal Dependencies formalism, which enforces specific constraints on function words (e.g., determiners and adpositions must be dependents of content words).
2. A robust evaluation framework demonstrating that an unsupervised approach can achieve competitive results with supervised transfer systems across 32 languages while being significantly more robust to domain shifts.
3. An effective method for estimating adposition direction (pre- vs. postpositions) directly from test data with minimal examples, eliminating language-specific parameters.

## Strengths
1. **Strong empirical evaluation across multiple dimensions**: The paper provides comprehensive evaluation across 32 languages, three different POS scenarios (gold, predicted, and naive type-constrained), and multiple domains. The results consistently show UDP's competitiveness with supervised transfer systems (MSDG) while demonstrating superior robustness to domain shifts (as evidenced by lower standard deviations across domains in Table 6). This is a significant contribution to cross-lingual parsing research, showing that unsupervised approaches can match supervised ones while being more stable.

2. **Clear demonstration of practical utility**: The paper establishes that an unsupervised parser can be competitive with supervised transfer systems (with only a 6.4% average UAS difference from MSDG) while requiring no training data. The analysis of error propagation (0.37 for UDP vs. 0.46 for MSD) when using predicted POS tags is particularly convincing for real-world applicability, showing that UDP is more resilient to POS errors.

3. **Linguistically sound approach**: By leveraging the explicit constraints of the Universal Dependencies formalism (e.g., function words must be leaves, specific head rules), the parser avoids many common pitfalls of data-driven approaches. This is especially important for cross-lingual parsing where word order differences can cause supervised systems to fail. The paper effectively shows how these formal constraints can be used to build a simple, robust parser.

## Weaknesses
1. **Limited novelty**: The approach is largely a straightforward extension of Søgaard (2012), with the main differences being UD-specific head rules, two-step decoding, and personalized PageRank. The paper does not sufficiently differentiate itself from prior work, and the key improvements (personalized PageRank, two-step decoding) are not sufficiently justified or analyzed compared to prior work. The claim that the parser is "distinctly robust to domain change" is not sufficiently supported by comparison to other rule-based parsers.

2. **Head directionality estimation on test data**: The paper estimates adposition directionality directly on test data (using 10-15 sentences), which technically makes it not fully unsupervised. While the paper claims this converges quickly, this is a significant methodological concern for an unsupervised parser, as it effectively uses some test data for parameter tuning. The authors should clarify whether this is acceptable for an unsupervised system or if they have a method for estimation without test data.

3. **Insufficient analysis of head rule limitations**: The paper does not provide critical analysis of the head rules' coverage or limitations. For instance, it would be valuable to know how often the graph defined by the head rules is disconnected, the size of the connected components, and how this affects parsing. The paper also does not analyze why the system performs poorly on certain languages (e.g., Tamil, Arabic) or POS types (e.g., PUNCT).

4. **Lack of comparison to recent unsupervised parsers**: The paper compares mainly to the baseline rule-based parser (BL) and supervised transfer systems (MSD), but does not compare to more recent unsupervised parsers that might have better performance (e.g., from the 2012-2015 literature). This makes it difficult to assess the true state of the art for unsupervised parsing.

## Questions to Authors
1. The paper states that adposition direction is estimated on test data with 10-15 sentences. Can the authors clarify whether this is considered acceptable for an unsupervised parser, and how they would handle languages with very short test sets?

2. How is PageRank tie-breaking handled when multiple words have the same rank? Does this affect performance, and if so, how?

3. Can the authors provide oracle results for each constraint (head rules, directionality, distance) to show the upper-bound impact of each component on the final parsing performance?

4. The paper used the English treebank during development to assess head rules. How much would the results change if they had used a different language for development, and how does this impact the claim of language independence?

**Summary of the Paper**

The paper presents UDP, an unsupervised dependency parser for Universal Dependencies (UD) that uses personalized PageRank and a small set of head-dependent rules. The parser requires no training data and estimates adposition direction directly from test data. The authors evaluate UDP on 32 languages in three setups, depending on the reliability of available POS tags, and compare it to a multi-source delexicalized transfer system.

**Main Contributions**

1. **Unsupervised dependency parsing without training**: The authors propose a parser that requires no training data and relies solely on explicit part-of-speech (POS) constraints imposed by the UD formalism.
2. **Personalized PageRank for content word ranking**: The authors use personalized PageRank to rank content words, which improves the parser's performance and makes it more robust to domain changes.
3. **Small set of head-dependent rules**: The authors use a small set of head-dependent rules that are adapted to the UD formalism, which makes the parser more linguistically sound and effective.

**Strengths**

1. **Competitive performance**: UDP achieves competitive performance on all but two UD languages, and even beats a multi-source delexicalized parser (MSD) on Hindi.
2. **Robustness to domain changes**: The parser is more resilient to domain changes than MSD, as shown by the lower standard deviation across domains.
3. **Linguistically sound**: The parser is designed to satisfy a set of general properties of the UD syntactic formalism, making it a more linguistically sound approach.

**Weaknesses**

1. **Limited handling of POS tags**: The parser relies heavily on accurate POS tags, and its performance degrades significantly when using predicted POS tags.
2. **Simplistic handling of function words**: The parser uses a simple approach to handle function words, which may not be effective for all languages.
3. **No handling of multiword expressions**: The parser does not handle multiword expressions, which can lead to errors in parsing.

**Questions to Authors**

1. How do you plan to extend the parser to handle multiword expressions and coordination?
2. Can you provide more details on the development set used to tune the PageRank parameters and function-word directionality?
3. How do you think the parser's performance would change if you used a more sophisticated approach to handle function words?

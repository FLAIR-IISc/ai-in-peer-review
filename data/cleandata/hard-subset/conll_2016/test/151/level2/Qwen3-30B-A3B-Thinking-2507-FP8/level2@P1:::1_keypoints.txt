- Summary: Introduces UDP, an unsupervised dependency parser for UD that needs no training data, uses personalized PageRank to find main predicates, UD head rules, estimates adposition direction from test data, and only requires POS tags.

- Contribution 1: A linguistically grounded unsupervised parser that strictly enforces UD constraints (function words as leaves, UD-specific head rules).

- Contribution 2: A personalized PageRank technique that substantially improves main predicate identification without any training data.

- Contribution 3: Demonstrates cross-domain robustness across 32 languages, outperforming supervised transfer in cross-domain evaluations.

- Strength 1: Truly unsupervised approach achieves competitive performance (avg UAS only 6.4% behind MSDG) across diverse languages.

- Strength 2: Compelling cross-domain robustness—lower standard deviation across domains than supervised MSD, showing stability to domain shifts.

- Strength 3: Explicit use of UD linguistic constraints makes the method more linguistically sound than prior unsupervised methods.

- Strength 4: Clear component analysis showing personalized PageRank contributes ~9 UAS points over baseline.

- Strength 5: Implementation is publicly available, aiding reproducibility and future work.

- Weakness 1: Claim of being "unsupervised" is muddied by using English treebank data for development/tuning; needs clearer disclosure.

- Weakness 2: Predicted POS evaluation relies only on TnT tagger; lacks analysis across different taggers or tagging-error sensitivity.

- Weakness 3: Insufficient analysis of performance on languages with divergent word order (e.g., verb-final languages); unclear why some verb-final languages fare better.

- Weakness 4: Evaluation uses only UAS; omits LAS which would better reflect labeled dependency performance under UD.

- Weakness 5: Error analysis is superficial (e.g., very low PUNCT UAS noted but not deeply examined); lacks per-relation breakdown.

- Question 1: How much English treebank development data was used and could this bias head rules toward English-like structures?

- Question 2: For low-performing languages (e.g., Tamil, Arabic), please provide example errors and how they relate to UD constraints.

- Question 3: How could the system be adapted to better handle radically different word orders (e.g., verb-final languages where main predicate position differs)?

- Question 4: Can you provide LAS results for the main evaluation (Table 5) to complement the UAS metrics?



Summary: The paper introduces UDP, an unsupervised dependency parser for Universal Dependencies that requires no training data. It leverages UD's linguistic constraints, specifically using personalized PageRank to identify main predicates and UD-specific head rules for dependency relations. The system estimates adposition direction directly from test data and requires only POS tags. The authors evaluate UDP on 32 languages across different POS tagging scenarios and compare it to a multi-source delexicalized transfer system.

Main contributions:
1. A linguistically grounded unsupervised dependency parsing approach that strictly adheres to UD's formal constraints (function words as leaves, specific dependency head rules).
2. A personalized PageRank method that significantly improves main predicate identification without training data.
3. Demonstrated robustness to domain changes across languages, outperforming supervised transfer systems in cross-domain evaluations.

Strengths:
1. The paper presents a genuinely unsupervised approach that achieves competitive performance with supervised transfer systems (average UAS difference of only 6.4% from MSDG), which is impressive for an unsupervised method. The results on 32 languages demonstrate strong generalizability across diverse linguistic structures.

2. The evaluation of cross-domain robustness is particularly compelling. The authors show UDP has significantly lower standard deviation across domains (Table 6) compared to the supervised MSD system, confirming that rule-based systems can be more stable to domain shifts. This is a valuable contribution to the field of cross-lingual parsing.

3. The paper thoroughly addresses the linguistic constraints of UD, making their approach more linguistically sound than previous unsupervised methods that don't explicitly leverage UD's formalism. The use of UD-specific head rules (Table 1) and the handling of function words as leaves directly addresses a key limitation in previous work.

4. The authors provide a clear explanation of the system's components and their contributions (Section 6.1), showing that personalized PageRank alone contributes 9 UAS points over the baseline, which is a substantial improvement.

5. The implementation is freely available, which will be valuable for future research in unsupervised parsing and cross-lingual evaluation.

Weaknesses:
1. The paper claims to be "unsupervised" but uses English treebank data during development to tune parameters and select head rules. While this is common practice, the paper should clarify that this is for development purposes only, not for training, to avoid misleading readers about the unsupervised nature of the approach.

2. The evaluation of predicted POS scenarios relies solely on TnT tagging. While TnT is a standard tagger, the paper would be stronger with additional analysis of how different tagging errors affect dependency parsing, or with results from multiple taggers to show robustness across different tagging systems.

3. The paper doesn't sufficiently address how the system handles languages with extremely different word order (e.g., verb-final languages). While the results show some performance on Hindi (which has verb-final structure), the paper doesn't analyze why the system works well for some verb-final languages but not others.

4. The evaluation uses only UAS, which is standard but limited. Including LAS (labeled attachment score) would provide a more comprehensive picture of the parser's performance, especially since the UD formalism includes rich dependency labels.

5. The error analysis in Section 6.2 is somewhat superficial. For example, the low PUNCT UAS (21% average) is noted but not explored in depth. A more detailed error analysis by dependency relation would help understand the system's limitations.

Questions to Authors:
1. The paper mentions the system uses English treebank data during development. Could you clarify how much development data was used, and whether this might introduce bias toward English-like structures in the head rules?
2. For languages where the system performs poorly (e.g., Tamil, Arabic), could you provide a few examples of the most common errors and their relation to UD formalism constraints?
3. How would you modify the system to better handle languages with extremely different word order patterns (e.g., verb-final languages where the main predicate isn't the first verb)?
4. Could you provide LAS results for the main evaluation (Table 5) to complement the UAS analysis?

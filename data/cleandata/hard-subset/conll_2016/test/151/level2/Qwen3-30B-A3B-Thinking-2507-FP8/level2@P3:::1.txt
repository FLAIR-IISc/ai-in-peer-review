

Summary: The paper introduces UDP, an unsupervised dependency parser for Universal Dependencies that requires no training data and relies solely on UD-specific constraints and a small set of dependency head rules. It uses personalized PageRank for content word ranking and a two-step decoding process to ensure function words are leaves. The system is evaluated across 32 languages in three POS scenarios and demonstrates competitive performance with delexicalized transfer parsers while showing greater robustness to domain changes.

Main contributions:
1. A linguistically sound unsupervised dependency parser that fully leverages UD constraints without requiring language-specific parameters.
2. A method for estimating adposition direction directly from test data without language-specific parameters.
3. A two-step decoding algorithm that properly handles content and function words according to UD conventions.

Strengths:
1. The paper effectively demonstrates how UD's formal constraints can enable unsupervised parsing that competes with supervised transfer systems. The evaluation across 32 languages shows the approach's broad applicability, with UDPG (gold POS) achieving 92% of the performance of the supervised MSDG baseline on average. This represents a significant advancement in unsupervised parsing for cross-lingual applications.

2. The cross-domain evaluation provides strong evidence for UDP's robustness, showing substantially lower standard deviations across domains compared to the supervised baseline (e.g., 1.8 UAS for UDPG vs. 4.5 for MSDG). This confirms that rule-based approaches can be more stable to domain shifts than data-driven methods, which is a valuable contribution to the field.

3. The system's ability to handle predicted POS tags with minimal performance degradation (only 2.2% drop compared to MSD's 2.7%) demonstrates practical utility for real-world applications where gold POS is unavailable. The error propagation analysis (0.37 additional dependency errors per POS error) provides valuable insight into the system's resilience to POS tagging errors.

Weaknesses:
1. The paper doesn't sufficiently address how the system handles languages with syntactic structures significantly different from the development language (English). While the paper mentions Hindi outperforms the baseline, it doesn't explain why or how the system adapts to verb-final structures, which could be a key limitation for certain language families.

2. The claim that the system is "completely free of language-specific parameters" is somewhat misleading. The adposition direction estimation requires 10-15 sentences of test data to converge, which may not be feasible for extremely low-resource languages or very short documents. The paper doesn't explore this limitation or provide a fallback mechanism.

3. The paper lacks detailed analysis of which head rules contribute most to performance across different language families. Table 1 presents the rules but doesn't explain how they were selected or why certain rules work better for some languages than others. This makes it difficult to assess the generalizability of the approach.

Questions to Authors:
1. How does the system handle languages with complex morphological features that might affect dependency relations, such as agglutinative languages where multiple morphemes might be attached to a single word?

2. The paper mentions that the system cannot infer relative clauses due to excluding the NOUN→VERB head rule. Could the authors elaborate on how this limitation affects performance on languages where relative clauses are common, and whether this rule could be safely reintroduced for certain languages?

3. The evaluation uses UD v1.2 treebanks, but UD has since evolved to v2.0. How would the system perform on the updated UD formalism, particularly with respect to the expanded set of dependency relations?

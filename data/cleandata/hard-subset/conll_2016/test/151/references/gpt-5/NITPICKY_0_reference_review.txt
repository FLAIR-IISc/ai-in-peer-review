Summary and main contributions (as I see them)
- A training-free, rule-based UD parser (UDP) that exploits UD constraints via a two-step decoding procedure: first attach content words (ranked by personalized PageRank), then attach function words to enforce leafness. The technique is extremely simple yet competitive with delexicalized transfer on average across 32 languages.
- A test-time, corpus-derived estimate of adposition attachment direction (pre/post) from POS bigram counts, avoiding language-specific hand-tuning for this salient typological parameter.
- A broad multilingual and cross-domain evaluation showing that UDP is relatively robust to domain shift and degrades less than a delexicalized transfer baseline under predicted POS.

Strengths
- Broad, careful evaluation across 32 UD v1.2 languages with three POS settings and several genre splits. This breadth is valuable: few unsupervised grammar induction papers evaluate so widely or include domain robustness.
- Strong “no-training” baseline that is competitive with multi-source delexicalized transfer in UAS with gold POS (within ~6.4% on average, winning on Hindi). Given the simplicity (few rules, PageRank), these numbers are noteworthy and practically useful as a robust baseline.
- Clear ablations indicating the contribution of PageRank versus decoding. Reporting that PR yields ~9 UAS points over UDPNoPR and ~4 over the baseline helps isolate where the gains come from.
- Sensible exploitation of UD design choices (function words as dependents/mostly leaves) and a two-stage decoding that operationalizes these constraints. The test-time ADP direction estimation is simple and effective.
- Error analyses by POS and a discussion of root detection behavior provide additional insight (e.g., DET direction failure for Basque; AUX/DET/SCONJ hand-fixed directions).

Weaknesses
- Overstated “without training” and “no language-specific parameters” claims. Key components were tuned/selected on English development data: the teleport probability, PR personalization weight (set to 5), the head-rule inventory (Table 1), and fixed directionality for AUX/DET/SCONJ/CONJ/PUNCT. This is not language-specific tuning per se, but it is data-driven engineering from English that can bias other languages (the Basque DET failure is an example). Please moderate claims and clearly separate “language-independent but English-tuned” priors from test-time estimates.
- Method underspecified, with several technical ambiguities:
  - The complete head rule set is not fully enumerated; Table 1 is minimal and seems insufficient for coverage. You state the baseline rules “do not provide full coverage,” but UDP relies on δ and back-off. What fraction of attachments per language are due to head rules vs the back-off tiers?
  - The PR graph construction is unclear: are edges unweighted? Are edges from potential dependents to candidate heads only as per Table 1? Are multiple parallel edges counted equally? Do you add self-loops? How are ties broken?
  - Teleport probability selection “until PR convergence” is conceptually odd: with teleportation PR always converges; the issue is mixing rate/tolerance. What stopping criteria and tolerance are used?
  - The decoding complexity claim (O(n×|C|)) excludes the PR cost; there are no runtime numbers. Please include wall-clock and complexity details.
- Evaluation choices raise fairness and comparability concerns:
  - You estimate ADP direction on the test set. This is a test-time hyperparameter fit to the test distribution (even if unsupervised). The competing MSD model cannot adapt on the test data. Report results when ADP direction is estimated from training (or dev) data only, and/or give MSD the same test-time adaptation privilege to ensure fairness.
  - You include punctuation in UAS, which is uncommon in UD evaluations and hurts comparability. Please also report UAS without punctuation.
  - The baseline can produce disconnected structures; the main system always yields valid trees. This favors UDP. Consider enforcing tree well-formedness in the baseline or clarifying how this affects UAS.
  - No statistical significance tests are reported. For small domain splits (e.g., 50–75 sentences), standard deviations can be misleading; please add confidence intervals or tests.
- Modeling limitations and coverage gaps:
  - Excluding NOUN→VERB rules precludes relative clause recovery, acknowledged by the authors. There is no treatment of coordination or proper names (stated as future work), and CONJ/PUNCT accuracy is very low. The heavy reliance on back-off rather than linguistically richer rules likely caps performance.
  - Fixed directionality for several function POS is fragile cross-linguistically. You note DET in Basque; similar issues likely exist for SCONJ and CONJ. Test-time estimation analogous to ADP would be more consistent with the paper’s philosophy.
- Numerous presentation, formatting, and reference issues that impede reproducibility and polish:
  - Typos and wording: “The the main idea,” “constribution,” “overal,” “adpositon,” “assignations,” “UPD” vs “UDP,” “come” vs “had” in the example, “presidental.”
  - Table/figure cross-references are inconsistent (e.g., referring to Table 5 for “head rules” earlier defined as Table 1).
  - References carry LaTeX artifacts (?, \Q1999\E), incomplete entries, inconsistent venues, and “ANONYMIZED” placeholders. This needs substantial cleanup for a camera-ready standard.

Questions to authors (for rebuttal)
- Please list the full set of head-dependent rules used by UDP (δ) and report, per language, the proportion of attachments created by: (i) rules, (ii) direction-only back-off, (iii) pure adjacency back-off.
- How sensitive are results to the PR personalization weight (5) and teleport probability (0.05) across languages? Provide a small grid search on a few typologically diverse languages without retuning on those test sets.
- Can you estimate DET/AUX/SCONJ/CONJ directionality from data (as you do for ADP) and report the effect? This would better support the “no language-specific parameters” claim.
- Please report UAS excluding punctuation, and (if possible) LAS or partial labeling to demonstrate extensibility claims.
- For fairness, can you (a) estimate ADP direction on training data only, and (b) allow MSD to adapt a small unsupervised parameter on the test set, and compare?
- Provide runtime numbers per sentence (including PR) and clarify PR graph edge weighting and tie-breaking.
- Are the code and datasets (domain splits, extra English sets) publicly available in a way that yields the exact numbers in Tables 5–7? Links currently show “ANONYMIZED.”

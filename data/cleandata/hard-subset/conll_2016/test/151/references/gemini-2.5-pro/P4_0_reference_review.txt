### Summary of the Paper

This paper introduces UDP, an unsupervised dependency parser designed specifically for the Universal Dependencies (UD) formalism. The parser operates without any training data, relying instead on a small, fixed set of POS-based head-attachment rules derived from UD principles. The method employs a two-step decoding process: it first uses personalized PageRank to rank content words and determine their heads, with a bias towards identifying the main predicate of the sentence. In the second step, it attaches function words in a way that ensures they are leaf nodes, a core constraint in the UD framework. The parser is unique in its ability to estimate adposition head directionality on-the-fly from the test data. The system is evaluated extensively on 32 languages from the UD v1.2 treebanks, under gold, predicted, and naive POS tag settings. It is also tested for robustness to domain shift. The results are compared against a rule-based baseline and a strong supervised multi-source delexicalized transfer parser (MSD).

### Main Contributions

1.  The primary contribution is the design and implementation of a simple, fully unsupervised parser that effectively leverages the explicit structural constraints of the Universal Dependencies formalism. The novel combination of a two-step decoding process (to enforce function words as leaves), personalized PageRank (for predicate identification), and on-the-fly parameter estimation (for adposition direction) creates a system that is both conceptually clear and empirically effective.

2.  The paper provides a comprehensive and rigorous large-scale empirical evaluation across 32 languages. This evaluation demonstrates that a linguistically-motivated, rule-based system can achieve performance surprisingly close to a strong, supervised cross-lingual transfer system. Furthermore, the cross-domain analysis provides strong evidence that this unsupervised approach is significantly more robust to domain shift and less sensitive to POS tagging errors than its data-driven counterpart, establishing it as a strong and stable baseline for future research.

### Strengths

1.  **Methodological Simplicity and Clarity:** The proposed method is conceptually simple and is described with sufficient clarity to allow for reproducibility. The core components—a small set of head rules (Table 1), a standard PageRank algorithm with a simple personalization heuristic, and a deterministic two-step decoding algorithm (Figure 1)—are straightforward. This makes the system an excellent candidate for a baseline in future work on unsupervised or cross-lingual parsing.

2.  **Extensive and Rigorous Evaluation:** The empirical evaluation is a significant strength. The parser is tested on a large and diverse set of 32 languages, which substantiates the generality of the approach. The evaluation under three different POS tagging scenarios (gold, predicted, naive) provides a clear picture of the system's robustness to input quality. The cross-domain evaluation, which includes newly annotated data, is particularly compelling. The finding that UDP exhibits lower variance in performance across domains than the supervised MSD parser (average standard deviation of 2.5 vs. 4.5 on gold POS, Table 6) is a key result that supports the claim of robustness.

3.  **Strong Performance for an Unsupervised System:** For a system that requires no training data, the performance is highly competitive. The parser (UDPG) achieves an average UAS of 57.5, which is only 6.4 points lower than the average of a strong supervised multi-source delexicalized parser (MSDG). The fact that it outperforms the MSDG system on Hindi and is competitive on several other languages demonstrates the power of exploiting the underlying principles of the annotation scheme. Furthermore, the analysis of error propagation shows that UDP is less sensitive to POS tagging errors than the MSD system, a valuable trait for real-world application on low-resource languages.

### Weaknesses

1.  **Incremental Novelty of Components:** While the combination of techniques is effective, the core components build heavily on prior work. The use of hand-crafted head rules and PageRank for syntactic analysis has been explored before (e.g., Søgaard, 2012b). The main novelties are the personalization of PageRank and the two-step decoding. Personalization is a sensible heuristic for main predicate detection, but it is an incremental improvement. The two-step decoding is a direct and effective implementation of a UD constraint, but it is more of a sound engineering choice than a fundamental methodological advance.

2.  **Dependence on English for Parameter Tuning:** The paper states that several key parameters were tuned on the English development set. This includes the head rules in Table 1, the PageRank teleport probability, and the fixed attachment direction for all function words except adpositions (e.g., AUX, DET, CONJ). This reliance on a single, typologically common language raises questions about the system's true language-agnosticism. While the strong performance on other languages mitigates this concern, the system is not entirely free of language-specific bias derived from the development process. A discussion of the sensitivity of the system to these hard-coded directional preferences would have strengthened the paper.

3.  **Ambiguity in Baseline Description:** The description of the baseline system (BLG) in Section 5.1 is unclear. It states that the baseline "does not necessarily yield well-formed trees" and that the structures are "not necessarily connected," while also mentioning that structures are single-rooted and "each token receives a head." These statements appear contradictory, as a single-rooted graph where every node has one parent (except the root) must be a connected tree. A more precise description of the baseline's decoding algorithm is needed to accurately assess the contribution of UDP's more sophisticated decoding process.

4.  **Limited Scope and Error Analysis:** The rule set is intentionally small, which leads to systematic failures on certain constructions. The authors acknowledge that the system cannot parse relative clauses due to the exclusion of a `NOUN -> VERB` rule. While the paper reports low UAS on `PUNCT` and `CONJ`, it lacks a deeper qualitative error analysis that could shed light on the structural limitations of the approach beyond these specific POS tags. Such an analysis would provide a more complete picture of the parser's capabilities and limitations.

### Questions to Authors

1.  The attachment direction for several function words (AUX, DET, SCONJ, CONJ, PUNCT) was fixed based on English development data. Have you considered estimating these directions on the test set, similar to your method for adpositions? This seems like a natural extension that would make the system more language-agnostic and potentially improve performance on languages with different word order properties (e.g., languages with post-posed determiners).

2.  Could you please clarify the decoding algorithm for the baseline (BLG)? Specifically, how can a structure be single-rooted with every token assigned a head, yet not be a connected tree? Understanding this is crucial for evaluating the specific contribution of the two-step decoding algorithm proposed for UDP.

3.  The paper quantifies the contribution of PageRank as a 9 UAS point improvement over a simple reading-order baseline (UDPNoPR). Could you provide a qualitative example of a sentence where the PageRank-based ordering of content words produces a more accurate parse than the linear reading order? This would offer valuable intuition into how and why the PageRank component is effective.

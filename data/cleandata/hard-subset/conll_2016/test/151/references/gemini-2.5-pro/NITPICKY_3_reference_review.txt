### Summary
This paper presents UDP, an unsupervised dependency parser for Universal Dependencies. The method requires no training data and relies on a small set of POS-based head-selection rules derived from UD principles. The core of the approach involves using a personalized PageRank algorithm to identify and rank key content words in a sentence, followed by a two-step decoding process that first attaches content words and then attaches function words to ensure they remain leaves, as stipulated by UD. The authors evaluate their system on 32 languages against a delexicalized transfer parser, demonstrating competitive performance and superior robustness to POS tagging errors and domain shifts.

### Contributions
1.  **A comprehensive empirical study** of a simple, rule-based unsupervised parser across 32 languages. The results establish a surprisingly strong, training-free baseline for UD parsing, showing that a few linguistic constraints can go a long way.
2.  **A convincing analysis of parser robustness.** The cross-domain evaluation and experiments with predicted POS tags demonstrate that the proposed simple system is more stable to data variation than a standard data-driven, delexicalized transfer model. This is a significant finding for parsing in real-world, low-resource, or domain-mismatched scenarios.
3.  **A novel combination of heuristics for unsupervised parsing.** While the components (rules, PageRank, greedy decoding) are not new in isolation, their specific combination—particularly the use of *personalized* PageRank to bias root selection and a two-step decoding process to enforce the leaf status of function words—is a novel contribution.

### Strengths
1.  **Simplicity and Effectiveness:** The proposed UDP system is remarkably simple, relying on a small, fixed set of rules and a standard graph algorithm. Despite its simplicity and lack of training, it achieves performance that is reasonably close to a supervised multi-source transfer system, making it an excellent and easily reproducible baseline.
2.  **Extensive Evaluation:** The evaluation is thorough, spanning 32 languages from the UD collection. The inclusion of three different POS quality scenarios (gold, predicted, and naive) provides a clear picture of the system's sensitivity to a critical input feature.
3.  **Robustness to Data Shift:** The cross-domain evaluation in Section 6.3 is a key strength. The finding that UDP exhibits lower variance (i.e., higher stability) across domains than the MSD baseline confirms previous hypotheses about rule-based systems and provides strong evidence for the utility of this approach in practical applications where domain consistency cannot be guaranteed.

### Weaknesses
1.  **Overstated Claims and Methodological Opacity:** The paper claims the method is "completely free of language-specific parameters," which is a significant overstatement. Multiple crucial parameters were tuned on the English development set and hard-coded: the PageRank teleport probability (0.05), the personalization weight (5x), and the attachment directions for function words like AUX, DET, SCONJ, etc. (Sec 3.2). This makes the system an application of English-tuned heuristics, not a universally parameter-free method. Furthermore, the description of the decoding algorithm is incomplete. The critical back-off logic described in the text (dropping constraints to ensure a tree is formed) is absent from the pseudocode in Figure 1, hindering reproducibility. The paper also fails to specify how ties are broken in the `argmin` function when multiple potential heads are equidistant.
2.  **Methodologically Flawed Parameter Estimation:** The system estimates adposition directionality "directly on test data" (Sec 3.2). While this is an unsupervised method, using the entire test set to tune a parameter that is then used to evaluate on that same set constitutes a form of test data leakage. A more sound protocol would be to use only a small, initial fraction of the test data for this estimation or to perform this estimation for each sentence based on statistics from preceding sentences only.
3.  **Insufficient Ablation and Weak Baselines:** The analysis of the PageRank contribution in Sec 6.1 is weak. The authors compare their personalized PageRank approach to `UDPNoPR`, which uses a simple "reading order" for content words. A far more insightful ablation would have been to compare against a system using *non-personalized* PageRank, as in Søgaard (2012b), to truly isolate the benefit of the personalization strategy, which is claimed as a key contribution. Additionally, the main rule-based baseline (BLG) is described as not always producing connected trees, making it a very weak point of comparison.
4.  **Sloppy Presentation and Formatting:** The paper is marred by a striking lack of attention to detail, particularly in the bibliography. The reference section is filled with formatting errors, stray question marks, and BibTeX artifacts (e.g., `Plank and Noord.,? \Q2010\E, Plank and Noord.,`). This level of sloppiness is unacceptable for a conference submission and reflects poorly on the work as a whole. Additionally, the text in Section 4.1 refers to a "Table 3" which does not appear to exist in the paper. These issues, while cosmetic, undermine confidence in the rigor of the research.

### Questions to Authors
1.  Given that several key hyperparameters (PageRank teleport/personalization, most function word attachment directions) were tuned on the English development set, could you rephrase your contribution more accurately? Instead of being "completely free of language-specific parameters," would it be more precise to state that it is a system with a small number of fixed parameters derived from English UD principles?
2.  Could you justify the decision to use the entire test set to estimate adposition direction? Have you considered alternative protocols, such as using the first N sentences of the test set, to avoid any information leak from the test data used for final evaluation?
3.  The back-off procedure is essential for guaranteeing a valid output tree. Why was this logic omitted from the pseudocode in Figure 1? For full reproducibility, could you also clarify the tie-breaking rule for the `argmin` function when multiple candidate heads are at the same minimum distance?
4.  Your analysis shows that personalization helps. However, to isolate its contribution, did you compare against a standard, non-personalized PageRank? This seems like a more direct and informative ablation than comparing against a simple sequential ordering of words.

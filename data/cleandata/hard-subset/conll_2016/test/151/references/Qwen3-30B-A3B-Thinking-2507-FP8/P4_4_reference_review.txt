

# Review of "Parsing for Universal Dependencies without training"

## Summary of the Paper
The paper presents UDP (Universal Dependencies Parser), an unsupervised dependency parser that requires no training data and relies solely on the explicit constraints of the Universal Dependencies (UD) formalism. The system leverages UD's specific properties: single-rooted trees, function words (adpositions, auxiliaries, determiners) always being dependents of content words, and the restriction that function words must be leaves. The parser uses personalized PageRank to identify the main predicate, applies UD-compliant head rules to attach content words, and then attaches function words while ensuring they remain leaves. It estimates adposition attachment direction directly from test data.

## Main Contributions (as assessed)
1. Demonstrating that a rule-based parser leveraging UD constraints can achieve competitive performance with supervised cross-lingual transfer methods (multi-source delexicalized parsers) while requiring no training data.
2. Providing empirical evidence that rule-based parsers are more robust to domain shifts than data-driven approaches, with significantly lower standard deviation across domains.
3. Creating a language-agnostic parser that requires no language-specific parameters by estimating adposition direction directly from test data.

## Strengths
1. **Strong empirical demonstration of competitive performance**: The paper provides comprehensive evaluation across 32 languages, showing that UDPG (gold POS) achieves 63.9% average UAS, which is remarkably close to the supervised MSDG baseline (69.5% UAS) with an average difference of only 6.4%. This is a significant achievement for an unsupervised system, especially since UDP outperforms MSDG on one language (Hindi).

2. **Compelling evidence for robustness to domain shifts**: The paper demonstrates that UDP has much lower standard deviation across domains compared to the supervised MSD system (3.1 vs. 4.5 in Table 6), confirming that rule-based approaches are less sensitive to domain shifts. This is a practical contribution with clear implications for real-world NLP applications where domain shifts are common.

3. **Well-designed evaluation strategy**: The evaluation covers three different POS scenarios (gold, predicted, naive) and multiple domain splits for select languages. The comparison with a strong baseline (MSD) is appropriate, and the results clearly show that UDP's performance degrades less when POS tags are imperfect (only 2.2% drop with predicted POS vs. 2.7% for MSD).

4. **Clear technical contribution in leveraging UD constraints**: The paper effectively demonstrates how the specific constraints of the UD formalism can be used to create a simple, effective parser. The two-step decoding process (attaching content words first, then function words) ensures function words remain leaves, which is a direct application of UD's formal constraints.

5. **Practical language-agnostic implementation**: The approach of estimating adposition direction directly from test data (using only 10-15 sentences) is a clever solution that eliminates the need for language-specific parameters, making the system truly universal without requiring language-specific tuning.

## Weaknesses
1. **Limited novelty in core approach**: While the adaptation to UD constraints is valuable, the core approach (PageRank for ranking words, rule-based dependency assignment) builds directly on Søgaard (2012b), who already used PageRank for unsupervised dependency parsing. The paper doesn't sufficiently distinguish between the existing components and the novel contributions specific to UD.

2. **Underdeveloped error analysis**: The paper provides a basic breakdown by POS (Section 6.2) but lacks deeper analysis of why the parser makes specific errors, especially for languages where performance is lower. Understanding these error patterns would help identify the system's limitations and potential improvements.

3. **Artificial naive POS scenario**: The evaluation of the naive POS scenario (UDPN) where words are tagged as either CONTENT or FUNCTION based on frequency is quite extreme and doesn't reflect realistic POS tagging errors. The paper doesn't adequately justify why this scenario is meaningful for evaluating robustness.

4. **Limited domain shift evaluation**: The domain shift analysis is restricted to a few languages (Bulgarian, Croatian, Italian, English) with relatively small domain splits. While the results are promising, a more extensive evaluation across more languages and larger domain splits would better support the claim about robustness.

5. **Insufficient comparison with recent rule-based work**: The paper primarily compares with Søgaard (2012b) but doesn't adequately contrast with more recent rule-based unsupervised parsing approaches that may have incorporated similar or better techniques.

## Questions to Authors
1. The paper claims UDP is "completely free of language-specific parameters" by estimating adposition direction directly from test data. However, the paper states that for other function words (AUX, DET, SCONJ), directionality was determined on English dev data. Could you clarify whether this makes the system partially language-specific, and how this affects the claim of being "completely free of language-specific parameters"?

2. The paper mentions that the system was developed using English treebank data for tuning. How sensitive is the system to the choice of development language? Would the performance be similar if developed on a different language?

3. The paper states that "the system cannot infer relative clauses" because the NOUN → VERB head rule was excluded. Could you elaborate on why this rule was excluded, and whether there might be a way to incorporate it without compromising the system's performance or simplicity?

4. The paper reports an error propagation rate of 0.37 for UDP and 0.46 for MSD. Could you provide more details on how this error propagation rate was calculated, and whether it holds consistently across different languages?

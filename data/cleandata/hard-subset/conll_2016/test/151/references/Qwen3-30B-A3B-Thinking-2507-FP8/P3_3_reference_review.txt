

# Peer Review: "Parsing for Universal Dependencies without training"

## Summary and Main Contributions

This paper presents a rule-based dependency parser (UDP) that requires no training data and leverages constraints from the Universal Dependencies (UD) formalism. The main contributions are: (1) a parser that exploits UD-specific constraints (single-rooted trees, function words as dependents), (2) a method for estimating adposition direction directly from test data, and (3) a demonstration that a simple rule-based approach can achieve competitive results with supervised transfer parsers while being more robust to domain shifts.

## Strengths

1. **Linguistically sound approach to UD parsing**: The paper makes a compelling argument that existing cross-lingual parsing approaches fail to fully exploit UD's formal constraints. By designing a system that strictly adheres to UD's rules (e.g., function words as leaves), the authors present a novel paradigm that shifts focus from data-driven learning to constraint exploitation. This is particularly significant as UD has become the de facto standard for cross-lingual syntactic annotation.

2. **Robustness to domain shifts**: The cross-domain evaluation provides strong evidence that UDP is more stable than supervised transfer methods (MSD) across different text domains. The paper demonstrates that UDP maintains consistent performance across domains (lower standard deviation in Table 6) while MSD shows higher variability. This finding directly addresses an important practical concern in real-world NLP applications where domain shifts are common.

3. **Comprehensive evaluation framework**: The evaluation across three POS scenarios (gold, predicted, and naive two-POS) provides valuable insights into the system's sensitivity to POS errors. The authors correctly identify that error propagation from POS tagging to dependency parsing is lower for UDP (0.37) than for MSD (0.46), which is a significant practical advantage.

## Weaknesses

1. **Limited novelty of core methodology**: While the application to UD is novel, the core technical approach (personalized PageRank and two-step decoding) is largely derived from Søgaard (2012b). The paper doesn't sufficiently distinguish how their implementation differs meaningfully from prior work. The 9-point improvement over UDPNoPR is significant, but the paper doesn't convincingly argue why this specific implementation represents a novel contribution rather than an incremental improvement.

2. **Overstated performance claims**: The paper claims UDP "compares favorably to delexicalized parsing," but the average gap between UDPG and MSDG is only 6.4% (Table 5), with UDPG outperforming MSDG on only one language (Hindi). This modest gap is not as impressive as the paper suggests. For context, the unsupervised parser in Søgaard (2012b) already achieved comparable results to supervised transfer methods on some languages.

3. **Lack of detailed error analysis**: The paper provides limited insights into *why* UDP makes certain errors. The POS breakdown (Section 6.2) is useful but superficial. A more detailed analysis of specific error patterns (e.g., how the parser handles long-distance dependencies, coordination, or languages with different word orders) would strengthen the evaluation.

4. **Domain evaluation limitations**: The cross-domain evaluation uses small, manually curated datasets (e.g., "50 sentences from a magazine"), which may not adequately represent true domain shifts. The paper's claim about robustness to domain shifts would be more convincing with larger, more diverse domain splits across more languages.

## Questions to Authors

1. How do your results compare to Søgaard (2012b) on the same languages and evaluation protocol? This would help contextualize the performance gains.

2. The paper mentions that UDP cannot infer relative clauses due to excluding the NOUN→VERB head rule. Could you provide examples of sentences where this limitation causes significant errors, and how common these errors are across languages?

3. The adposition direction estimation requires "10-15 sentences" to converge. How would the parser perform on languages with very short test sets (e.g., <10 sentences per language)?

4. The authors state that UDP is "completely free of language-specific parameters," yet they estimate adposition direction from test data. How would this estimation work for languages with extremely limited test data?

## Overall Recommendation

This is a well-executed paper that makes a valuable contribution to the understanding of how UD constraints can be exploited in dependency parsing. The demonstration of robustness to domain shifts is particularly strong and addresses a meaningful practical concern. However, the paper overstates the novelty and performance gains of the approach, as the core methodology is largely built upon prior work, and the performance gap with supervised methods is modest.

The paper would benefit from more thorough contextualization within the unsupervised parsing literature, a more detailed error analysis, and a stronger justification for the claimed novelty. With these improvements, this work could make a significant contribution to the field. As it stands, it's a solid contribution that advances our understanding of constraint-based parsing but doesn't quite reach the level of a breakthrough.

**Recommendation: Weak Accept** (the paper has significant strengths but needs to address the novelty concerns and provide more detailed contextualization)

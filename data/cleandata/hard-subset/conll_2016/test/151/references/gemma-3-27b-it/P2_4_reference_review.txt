## Review for "Parsing for Universal Dependencies without training"

**Summary:** This paper presents a novel unsupervised dependency parser (UDP) that leverages Universal Dependencies (UD) constraints and personalized PageRank to achieve competitive performance without any training data. The core idea is to exploit the inherent linguistic structure enforced by UD, specifically the constraints on function word attachment, and combine it with a graph-based ranking approach. The parser is evaluated on 32 languages, demonstrating robust performance and a surprising degree of resilience to domain shift compared to a supervised delexicalized baseline.

**Main Contributions (as I see them):**

1. **Demonstration of Strong Baseline with UD Constraints:** The paper convincingly demonstrates that a surprisingly strong baseline can be achieved simply by intelligently exploiting the constraints inherent in the Universal Dependencies framework. This highlights the value of well-defined linguistic formalisms for unsupervised parsing.
2. **Practical Application of Personalized PageRank:** The adaptation of personalized PageRank to prioritize content words, particularly verbs, as potential roots significantly improves parsing accuracy. This is a clever application of an existing technique within a novel linguistic context.
3. **Robustness to Domain Shift:**  The parser exhibits a remarkable degree of robustness to domain shift, outperforming a supervised baseline in many cases. This suggests a potentially valuable alternative for scenarios where labelled data for the target domain is unavailable.

**Strengths:**

1. **Strong Empirical Results:** The parser achieves impressive results across a large number of languages (32) *without any training*.  The comparison to a competitive delexicalized transfer parser is well-motivated and highlights the effectiveness of the proposed approach.  The fact that UDP often outperforms the supervised baseline, particularly with predicted POS tags, is a significant finding.
2. **Clear and Concise Presentation:** The paper is well-written and clearly explains the method, with sufficient detail for reproducibility.  The two-step decoding algorithm is described logically, and the use of Table 1 for dependency rules is effective. The inclusion of a full example parse run is helpful.
3. **Thorough Evaluation:** The evaluation setup is comprehensive, considering both gold and predicted POS tags, as well as domain shift. The analysis of performance across different POS tags and domains provides valuable insights into the strengths and weaknesses of the system.
4. **Practical Significance:** This work has practical implications for low-resource languages and scenarios where training data is scarce. The ability to achieve reasonable parsing accuracy without any training data is a valuable contribution.
5. **Emphasis on Linguistic Formalism:** The paper champions the benefits of exploiting linguistic formalisms like UD, moving away from purely data-driven approaches. This aligns well with current trends in NLP research.

**Weaknesses:**

1. **Limited Novelty of Individual Components:** While the combination of techniques is effective, the individual components (PageRank, rule-based parsing, UD constraints) are not entirely novel.  The key innovation lies in the *integration* of these components, but a more detailed discussion of the novelty within this integration would be beneficial.
2. **Dependency on POS Tagging Quality:** Although UDP exhibits robustness to POS errors compared to the supervised baseline, performance still degrades significantly with predicted POS tags. The parser’s reliance on accurate POS tags remains a limitation, and further exploration of methods to mitigate this dependency would be valuable.
3. **Simplistic Handling of Certain Structures:** The authors acknowledge limitations in handling complex structures like coordination and relative clauses. While these limitations are noted, a more detailed discussion of their impact on overall performance and potential avenues for improvement would be helpful.
4. **Limited Error Analysis:** The error analysis is relatively brief. A more in-depth analysis of common error cases and their causes would provide valuable insights for future development.  For example, exploring specific examples where the head direction heuristic fails would be informative.
5. **Over-reliance on English for Development:** While the authors acknowledge the use of English data for parameter tuning, it's unclear how well these parameters generalize to other languages. A more systematic evaluation of parameter sensitivity across different language families would strengthen the findings.



**Questions to Authors:**

1.  Could you provide a more detailed analysis of the types of errors made by the parser, perhaps focusing on specific dependency relations or POS tag combinations?
2.  How sensitive are the PageRank parameters (teleport probability, personalization weight) to variations across different languages? Have you considered a more adaptive approach to parameter tuning?
3.  How might the system be extended to handle more complex linguistic phenomena, such as coordination and relative clauses? Are there specific modifications to the rules or decoding algorithm that could address these challenges?
4.  The paper mentions the use of the TnT tagger. Did you explore alternative POS taggers, and how did their performance impact the overall parsing results?
5.  How does the performance of UDP compare to other unsupervised parsing methods that don't necessarily rely on UD constraints? A comparison would help position the contribution of this work more clearly.



**Additional Comments:**

*   The code and data used in the experiments should be made publicly available to facilitate reproducibility and further research.
*   It would be helpful to include a table showing the POS tag accuracies for each language when using predicted POS tags.
*   The sentence “However, with an average POS accuracy by TnT of 94.1%, the error propagation is 0.37” feels a bit disconnected and the calculation is unexplained. A clearer explanation of the “error propagation” metric would be beneficial.



**Overall Assessment:**  This is a strong paper presenting a valuable contribution to unsupervised dependency parsing. The method is well-motivated, empirically supported, and has practical potential. While the individual components may not be entirely novel, the combination and application within the UD framework are significant. I recommend acceptance with minor revisions addressing the weaknesses outlined above.

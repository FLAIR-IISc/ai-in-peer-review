- Summary: UDP is an unsupervised UD parser using POS constraints and personalized PageRank, estimating adposition direction from test data and evaluated on 32 languages plus domain shift.

- Contribution: Shows surprisingly strong performance for a truly unsupervised parser, competitive with delexicalized transfer baselines.

- Contribution: Presents a coherent system integrating POS-based rules, personalized PageRank, and dynamic adposition direction estimation.

- Contribution: Provides a comprehensive multilingual evaluation and domain-shift analysis across 32 languages.

- Strength: Strong empirical results, including robustness with predicted POS tags.

- Strength: Effectively leverages UD grammatical constraints to improve parsing without training.

- Strength: Thorough evaluation that informs generalizability and robustness.

- Strength: Clear, well-structured presentation with a helpful example run.

- Strength: Reproducible work with parser and datasets made available.

- Weakness: Individual components (PageRank, dependency rules) are not novel; main novelty is their adaptation/integration.

- Weakness: Reliance on English for parameter tuning and head-direction may introduce bias across languages.

- Weakness: The naive two-POS (CONTENT/FUNCTION) scenario is overly coarse and unrealistic; large performance gap vs. predicted POS complicates interpretation.

- Weakness: Error analysis is limited; lacks detailed, language-specific examples of common failures.

- Question: How sensitive are PageRank parameters and directionality rules across languages, and can adaptation be automated from test data?

- Question: Can you report results using a realistic imperfect multilingual POS tagger for some languages to better reflect low-resource scenarios?

- Question: Have you explored more sophisticated root identification methods (e.g., subcategorization frames or SRL) to improve performance?

- Question: Can you provide example failure sentences from multiple languages and explain why UDP fails in those cases?

- Additional: Citation style is inconsistent and should be standardized.

- Additional: Table 5's "direction" column (L/R) needs clearer explanation in the text.

- Additional: Ensure dataset links are publicly accessible and stable.

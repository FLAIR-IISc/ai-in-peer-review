This paper describes a new deterministic dependency parsing algorithm and analyzes its performance across various languages.

The algorithm's core is based on a set of rules to define the dependencies allowed based on POS tags.

The algorithm starts ranking words using a slightly biased pagerank with edges defined by the allowed dependencies in a graph.

Through the ranking, each word is associated with the nearest word that will maintain a tree and is allowed by the main rule and a directional constraint.

Overall, the paper is engaging and clearly presented, although it seems slightly different from Sogard (2012), "Unsupervised Dependency Parsing Without Training."

I have a few questions and suggestions:

Main rule (Table 1) - It would be good to have some analysis of these rules in relation to the corpus.

For example, section 3.1 mentions that these do not always lead to connected graphs, but it does not specify how often this happens or how large the elements are typically.

I was surprised that the main direction was chosen using test data instead of training or development data.

Considering how quickly the decision converges (10-15 sentences), it is not a very big problem, but a surprising choice.

How does the tie-breaking work for words with the same pagerank score?

Does it significantly affect performance, or are the ties so rare that it has no effect?

The variation of constraints (main rule, direction, distance) will create a limit on the potential performance of the system.

It would be informative to include Oracle's results for each constraint, so that it can be shown how they affect the potential maximum score.

This will be especially helpful for guiding future work on where to try to change this system.

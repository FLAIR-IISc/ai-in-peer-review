Summary
The paper presents a deterministic dependency parsing algorithm that constructs trees using POS-based head rules and a directionality constraint. The approach first builds a graph of permitted dependencies and computes a slightly biased PageRank over this graph. It then processes words in order of their rank, attaching each to the closest permissible head while maintaining tree well-formedness. The paper is clearly written and the algorithm is easy to follow. However, the novelty relative to Søgaard (2012), “Unsupervised Dependency Parsing without Training,” appears limited. A more thorough comparison and empirical analysis would strengthen the contribution.

Strengths
- Clear problem setup and algorithmic description.
- Sensible use of a graph-based ranking to guide greedy attachment decisions.
- Broad cross-lingual evaluation is valuable and timely.

Main questions and suggestions
1) Analysis of head rules (Table 1)
- Please quantify how often the induced dependency graph is disconnected across languages (frequency and distribution), and report typical component sizes.
- Report coverage of the head rules with respect to the gold trees: e.g., what proportion of gold arcs are permitted by the rules? This would contextualize the attainable performance ceiling independent of other constraints.
- If possible, analyze which POS–POS pairs are most frequently blocked and how this correlates with per-language performance.

2) Determining head direction on test data
- It is surprising that head direction is decided using the test set. Even given rapid convergence (10–15 sentences), this constitutes test-time tuning. Please move this decision to a development set or use cross-validation. At minimum, provide a sensitivity analysis showing that the choice does not inflate test performance.
- Clarify whether the first 10–15 sentences used to set direction are excluded from evaluation, and whether the procedure is repeated per language or shared.

3) Tie-breaking for identical PageRank scores
- Please specify the tie-breaking strategy when multiple words share the same PageRank score (e.g., deterministic index order, secondary heuristic, or randomized with a fixed seed).
- Quantify how often ties occur and assess their impact on performance (e.g., variance over multiple runs with different tie-breaks, if applicable).

4) Upper bounds and oracle analyses
- The combination of constraints (head rules, directionality, and distance/closest-head) imposes clear limits on achievable accuracy. Oracle experiments would be very informative:
  - Oracle with only head rules (maximum achievable if any permitted head can be chosen).
  - Oracle with head rules + directionality.
  - Oracle with all constraints except the ranking/greedy procedure.
- Reporting these upper bounds per language would pinpoint the main bottlenecks and guide where future modifications are most promising. If space allows, an ablation study (removing one constraint at a time in the actual system) would nicely complement the oracle results.

5) Relation to prior work
- The algorithm seems close in spirit to Søgaard (2012). Please clarify the precise methodological differences (e.g., the specific PageRank bias, the exact attachment heuristic, constraint set) and provide a controlled empirical comparison under matched conditions. A discussion of when and why the proposed variant should outperform prior methods would help establish novelty.

Presentation and clarity
- Define “closest” more precisely (e.g., linear token distance, with or without projectivity constraints) and state formally how tree well-formedness is ensured during attachment.
- Provide enough implementation details (e.g., graph construction, PageRank parameters, convergence criteria) to ensure reproducibility.

Minor comments
- Section 4.1: “we obtain the rank.”
- Table formatting: Tables 5 and 7 present columns in different orders. The Table 7 arrangement is clearer; please standardize.
- Section 6.1: “isolate the contribution of both.”
- If space permits, include a brief runtime/memory note and the random seed policy (especially if tie-breaking can involve randomness).

Overall assessment
This is an interesting and well-presented paper with a simple, deterministic approach that performs competitively across languages. The main weaknesses are the limited demonstrated novelty relative to prior work and the lack of diagnostic analyses that would clarify where the gains come from and where the constraints bind. Addressing the test-set tuning issue, adding the requested oracle and coverage analyses, and strengthening the comparison to Søgaard (2012) would substantially improve the paper’s clarity, rigor, and impact.

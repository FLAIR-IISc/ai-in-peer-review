This paper presents a new deterministic dependency parsing algorithm and evaluates its performance across a range of languages. The core of the algorithm is a set of POS-based rules that define permitted dependencies. The process begins by ranking words using a biased PageRank on a graph constructed from these rules. Iterating through the words in rank order, each word is then attached to the closest permissible head that maintains tree properties, subject to the head rules and a directionality constraint.

Overall, the paper is interesting and clearly presented. However, its contribution appears to be incremental, as the algorithm seems to differ only slightly from Sogaard (2012), "Unsupervised Dependency Parsing without Training".

I have the following questions and suggestions:

*   **Head Rules (Table 1):** The paper would benefit from a more detailed analysis of the head rules in relation to the corpora. For example, Section 3.1 mentions that the rules do not always produce a connected graph, but it does not quantify how frequently this occurs or describe the typical size of the resulting components.

*   **Head Direction:** The choice to determine head directionality using the test set is surprising; this is typically done on training or development data. While the authors note that this decision converges rapidly (within 10-15 sentences), which mitigates the issue, the choice itself is unusual.

*   **Tie-Breaking:** How are ties between words with the same PageRank score handled? It would be useful to know if ties are rare or if the tie-breaking mechanism has a significant impact on performance.

*   **Oracle Analysis:** The various constraints (head rules, directionality, distance) impose an upper bound on the system's performance. It would be highly informative to include oracle results for each constraint to show how much each one limits the maximum possible score. Such an analysis would be particularly valuable for guiding future work by identifying which components offer the most room for improvement.

**Minor Points:**
*   In Section 4.1, "we obtain rank" should probably be "we obtain **the** rank".
*   The columns in Table 5 and Table 7 are in a different order. I found the arrangement in Table 7 to be clearer.
*   In Section 6.1, "isolate the of both" appears to be missing a word, likely "isolate the **contribution** of both".

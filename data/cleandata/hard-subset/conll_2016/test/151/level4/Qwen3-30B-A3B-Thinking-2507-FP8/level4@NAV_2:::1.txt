

Summary: This paper presents UDP, an unsupervised dependency parser for Universal Dependencies that requires no training data and relies solely on explicit part-of-speech constraints. The core algorithm uses personalized PageRank over a graph defined by dependency head rules to rank content words, followed by a two-step decoding process that assigns heads to content words and function words according to UD constraints.

Main contributions (as assessed by reviewer):
1. The use of personalized PageRank to improve main predicate identification, which is the most significant technical contribution
2. The adaptation of dependency head rules to UD-compliant POS relations, which enables the parser to work with the UD formalism
3. The approach of estimating adposition attachment direction directly on test data, which eliminates language-specific parameters

Strengths:
1. The parser achieves competitive performance across 32 languages, nearly matching a multi-source delexicalized transfer parser (MSD) with an average difference of only 6.4% in UAS.
2. The system demonstrates superior robustness to domain changes compared to supervised transfer parsers, with significantly lower standard deviations across different text domains.
3. The approach requires no training data and is linguistically sound, making it particularly valuable for truly low-resource languages where training data is unavailable.

Weaknesses:
1. The work appears to differ only slightly from Søgaard (2012), which significantly limits the novelty contribution of this paper.
2. The choice to estimate head direction using test data rather than development data is surprising and not well justified, despite the fast convergence (10-15 sentences).
3. The paper lacks analysis of how frequently the head rules lead to disconnected graphs and the typical size of these components, which is important for understanding the algorithm's limitations.
4. The paper does not provide details on how tie-breaking for words with identical PageRank scores is handled, nor does it assess whether this impacts performance.
5. The authors fail to include oracle results for each constraint (head rules, directionality, distance), which would help quantify the maximum possible performance under the current constraint system.

Questions to Authors:
1. Please provide analysis of how frequently the head rules lead to disconnected graphs and the typical size of these components in your test data.
2. How does the system handle tie-breaking for words with identical PageRank scores, and does this tie-breaking strategy significantly impact parsing performance?
3. Can the authors provide oracle results for each constraint (head rules, directionality, distance) to show how much each constraint limits the maximum possible parsing accuracy?
4. Why was head direction estimated using test data rather than development data, and how would this choice affect performance if development data were used instead?

- Paper augments sentiment and sarcasm features with cognitive features from readers’ eye movements, including basic and graph-based gaze metrics, and evaluates them on two public datasets.  
- Contribution 1: Empirical finding that gaze-derived features give small but consistent improvements for polarity detection, especially on complex constructs (up to 3.7% and 9.3% absolute gains; +6.1% relative on ironic subset).  
- Contribution 2: Introduces gaze-saliency graph features and a comprehensive set of gaze-based indicators for sentiment analysis.  
- Contribution 3: Provides targeted evaluation on challenging datasets, feature significance analysis, error source discussion, and practical feasibility considerations.  
- Strength 1: Novel and potentially impactful idea—incorporating cognitive signals could help NLP tasks with subtle textual cues (e.g., sarcasm, implicit sentiment).  
- Strength 2: Clear writing and organization with thorough presentation of features and datasets.  
- Strength 3: Solid experimental setup using multiple classifiers, feature combinations, two public datasets, held-out analysis, McNemar significance testing, and chi-squared feature importance.  
- Strength 4: Improvements are larger on ironic/sarcastic texts, aligning with the goal of handling nuanced expressions.  
- Weakness 1: Weak motivation linking gaze metrics specifically to sentiment polarity; many gaze measures reflect processing difficulty or incongruity rather than polarity.  
- Weakness 2: Incremental gains from gaze features are marginal compared to text+sarcasm features (e.g., +0.6% and +1.5% F-score), and gaze-only features underperform text-based features.  
- Weakness 3: Practical feasibility for large-scale deployment is unconvincing—issues include calibration, consent, device variability, noise, and aggregating multiple readers.  
- Question 1: Request for deeper evidence connecting specific gaze features (e.g., RSF, LREG, saliency degrees) to sentiment polarity beyond general processing difficulty.  
- Question 2: Request for fine-grained ablation showing which cognitive feature groups (basic vs saliency-graph; regressions vs skips vs durations) drive the gains.  
- Question 3: How many readers per snippet are required for reported gains, and how does performance degrade with fewer/noisier readers or mobile trackers?  
- Question 4: Do gains generalize across domains beyond the evaluated datasets, and how sensitive are gaze features to domain or annotator shifts?  
- Question 5: Clarify inference-time assumptions—whether gaze is needed at training, inference, or both, and whether training with gaze can improve text-only inference.  
- Question 6: How will ethics, consent, and data protection for large-scale eye-tracking be handled, especially commercially?  
- Additional comment 1: Clarify whether gains on straightforward polarity cases are meaningful to better position the contribution.  
- Additional comment 2: Include more detailed error analysis linking failure modes to feature limitations (e.g., regressions vs polarity direction).  
- Additional comment 3: Feasibility claims would be stronger with concrete pilot studies using mobile eye-tracking, even small-scale.

- Proposes augmenting sentiment analysis with gaze-derived cognitive features to improve detection, especially for sarcasm and irony.  
- Primary contribution: demonstrates gaze-based cognitive features significantly boost sentiment classification for complex constructs.  
- Secondary contribution: provides a systematic feature-significance analysis showing cognitive features' effectiveness for nuanced sentiment.  
- Tertiary contribution: addresses deployment feasibility via mobile eye-tracking technology.  
- Novel cognitive feature set from eye-tracking with rigorous evaluation across classifiers/datasets and up to 9.3% F-score improvement; gaze features rank highly.  
- Cognitive features give greater benefit for complex sentiment (irony/sarcasm) — 6.1% F-score gain on irony texts vs 2.1% on simpler texts.  
- Practical feasibility discussion covers real-world implementation and mobile eye-tracking applications (e.g., Cogisen patent, e-commerce).  
- Evaluation conflates domains (movie reviews and tweets) in training/testing, risking inflated results and problematic baselines; needs domain-separated evaluation.  
- No comparison with deep learning approaches (CNNs/RNNs), so claims of outperforming state-of-the-art are unsupported.  
- Many gaze features lack clear theoretical justification (e.g., Largest Regression Position, saliency graph metrics), leaving their link to sentiment complexity ambiguous.  
- Question: how do results change under strict domain separation (train on reviews/test on tweets and vice versa)?  
- Question: why exclude deep learning baselines, and would cognitive features help when integrated with deep architectures?

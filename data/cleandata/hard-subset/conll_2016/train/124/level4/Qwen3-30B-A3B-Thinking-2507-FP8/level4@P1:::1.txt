

This paper proposes an innovative approach to sentiment analysis by incorporating cognitive features derived from eye-movement patterns of human annotators. The authors claim this is the first work to introduce such features into sentiment analysis, demonstrating improvements of up to 3.7% and 9.3% in F-score on two datasets compared to systems using only traditional features. The authors perform feature significance analysis and experiments on a held-out dataset to show that cognitive features specifically enhance the handling of complex sentiment constructs like sarcasm and irony.

The primary contributions of this work are:
1. The introduction of cognitive features from eye-tracking data as a novel feature set for sentiment analysis
2. Empirical evidence that these features significantly improve performance on complex sentiment constructs
3. Detailed feature significance analysis demonstrating the specific value of cognitive features for nuanced sentiment analysis

Strengths:
1. The paper presents a genuinely novel approach that could inspire similar work across other NLP tasks. The integration of cognitive science with NLP represents an interesting interdisciplinary direction.
2. The experimental design is thorough, with multiple classifiers tested across two well-chosen datasets containing complex sentiment constructs. The comparison against traditional feature sets provides solid evidence of the added value of cognitive features.
3. The feature significance analysis (Table 3) compellingly shows that cognitive features are among the top contributors for complex sentiment analysis, particularly for sarcasm and irony detection.

Weaknesses:
1. The motivation for why cognitive features should help with sentiment polarity detection is insufficiently justified. The authors acknowledge that eye movements may reflect reading ease but fail to establish a clear cognitive mechanism linking eye-tracking patterns to sentiment processing. Without a stronger theoretical foundation, it's difficult to accept that these features specifically capture sentiment-relevant information rather than general text complexity.
2. The marginal improvement when adding cognitive features to the already strong Sn+Sr combination (only 0.6% and 1.5% for Dataset 1 and 2 respectively) significantly weakens the contribution. Given the complexity of collecting eye-tracking data, such minimal gains raise questions about the practical value of the approach.
3. The feasibility discussion in Section 7 lacks convincing evidence. The example in Section 7.2 about mobile eye-trackers for Amazon reviews fails to demonstrate why cognitive features would provide meaningful improvement over existing approaches. The proposed application scenario doesn't address how the cognitive features would specifically enhance sentiment analysis beyond what's already possible with text features alone.

Questions to Authors:
1. Could you provide more detailed evidence explaining the cognitive basis for why eye-movement patterns would correlate with sentiment polarity rather than general text complexity?
2. Given the minimal improvement over Sn+Sr, how would you justify the additional complexity and data requirements of collecting eye-tracking data in practical applications?
3. In your held-out dataset experiments (Table 4), the improvement for simple constructs (2.1%) is comparable to the improvement for complex constructs (6.1%). How do you explain this finding, and what does it imply about the specific utility of cognitive features for sentiment analysis?

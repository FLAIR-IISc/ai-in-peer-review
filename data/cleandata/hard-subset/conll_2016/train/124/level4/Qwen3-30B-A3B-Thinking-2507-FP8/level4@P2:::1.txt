

Summary: This paper proposes augmenting traditional sentiment analysis features with cognitive features derived from eye-movement patterns of human readers. The authors claim their approach improves sentiment polarity detection by 3.7% and 9.3% on two datasets compared to systems using only traditional features. They argue cognitive features help sentiment analyzers handle complex constructs like irony and sarcasm.

Main Contributions:
1. The paper introduces cognitive features from eye-tracking data as a novel feature set for sentiment analysis, which appears to be the first application of such features in NLP for this task.
2. The authors conduct a comprehensive evaluation showing that cognitive features particularly benefit complex sentiment constructs (irony, sarcasm) with a 6.1% improvement over traditional features for these cases.
3. The paper provides detailed feature significance analysis demonstrating that cognitive features constitute a significant portion of the top-ranked features for complex sentiment analysis.

Strengths:
1. The paper presents a well-structured and comprehensive feature set, including four distinct categories: sentiment features, sarcasm/irony/thwarting features, cognitive features from eye-movement, and reading difficulty features. The cognitive features, particularly the gaze-saliency graph features, represent a novel contribution to sentiment analysis research.
2. The evaluation design is robust, featuring multiple classifiers (SVM, Naive Bayes, Neural Network), two different datasets with varying characteristics, and a held-out dataset specifically designed for complex sentiment constructs. The results demonstrate statistically significant improvements for complex cases (6.1% improvement for irony/sarcasm vs. 2.1% for simple cases), which is compelling evidence for the value of cognitive features in specific contexts.
3. The feature significance analysis provides strong evidence for the value of cognitive features, with 10 out of the top 20 ranked features being gaze-based for Dataset 1 and 7 out of top 20 for Dataset 2. This analysis effectively supports the claim that cognitive features are particularly valuable for handling nuanced sentiment.

Weaknesses:
1. The motivation for using cognitive features in sentiment analysis is underdeveloped. While the paper correctly notes that cognitive processes relate to eye-movement patterns, it fails to establish a clear theoretical link between eye-movement patterns and sentiment polarity detection. The authors do not explain why eye-movement would correlate with sentiment rather than just reading difficulty, which weakens the core premise of their approach.
2. The marginal improvement (0.6% and 1.5% for Dataset 1 and 2 respectively) when adding cognitive features to the already enhanced feature set (Sn+Sr) is not compelling. This incremental gain makes the contribution seem less significant than the authors claim, especially when compared to recent deep learning approaches that have achieved larger improvements in sentiment analysis.
3. The feasibility discussion in Section 7 is not convincing. The example of using mobile eye-trackers for Amazon reviews does not adequately address the practical challenges of collecting eye-tracking data at scale, such as privacy concerns, user consent, and the cost of deploying eye-tracking technology. The authors do not provide evidence that this approach would be more practical or effective than existing methods.
4. The paper does not adequately address the limitations of the eye-tracking data. The use of different eye-tracking devices with varying qualities (SR-Research Eyelink-1000 vs. Tobii T120) could affect the reliability of the cognitive features. The paper also doesn't explain how they handled variations in annotator eye-movement patterns or potential biases in the eye-tracking data.

Questions to Authors:
1. Can you provide a more detailed explanation of the theoretical link between eye-movement patterns and sentiment polarity detection, rather than just stating that cognitive processes relate to eye-movement?
2. Given the marginal improvement when adding cognitive features to the already enhanced feature set (Sn+Sr), how would you position this work in relation to recent deep learning approaches that have achieved larger improvements in sentiment analysis?
3. How would you address the practical challenges of collecting eye-tracking data at scale for real-world applications, particularly regarding user privacy and consent?

Overall, while the paper presents an interesting novel feature set and provides some evidence for its value in handling complex sentiment constructs, the weak theoretical motivation and marginal improvements weaken the contribution. The feasibility argument also needs significant strengthening to convince reviewers of the practical value of this approach.



**Summary of Paper and Main Contributions**

The paper proposes augmenting traditional sentiment analysis features with cognitive features derived from eye-movement patterns of human annotators. The authors claim this is the first work to incorporate cognitive features into NLP for sentiment analysis. The main contributions, as I see them, are: (1) the introduction of cognitive features from eye-movement patterns as a novel feature type for sentiment analysis; (2) empirical evidence that these features significantly improve performance for complex sentiment constructs like sarcasm and irony (6.1% improvement on held-out data); and (3) a detailed analysis of both basic and complex gaze features derived from eye-tracking data.

**Strengths**

1. The paper makes a compelling case for the value of cognitive features in handling complex sentiment expressions. The held-out dataset results showing a 6.1% improvement in F-score for complex constructs (irony/sarcasm) versus only 2.1% for simple constructs provides strong evidence that these features specifically target the most challenging aspects of sentiment analysis. This is particularly valuable given that existing methods struggle significantly with such nuanced expressions.

2. The feature engineering is exceptionally thorough, with the authors introducing both basic gaze features (e.g., average fixation duration, saccade length, regression count) and complex gaze features derived from gaze-saliency graphs. The detailed description of how these features capture different aspects of reading behavior provides a solid theoretical foundation for their use in sentiment analysis.

3. The paper demonstrates that cognitive features are particularly effective for the most challenging aspects of sentiment analysis - complex constructs like sarcasm and irony. This focus on the hardest cases, rather than general sentiment classification, represents a meaningful contribution to the field, as these are precisely the cases where current methods fail most frequently.

**Weaknesses**

1. The theoretical justification for why eye-movement patterns would correlate with sentiment polarity is weak. The paper states "cognitive processes in the brain are related to eye-movement activities" but provides no strong evidence or explanation for why these specific eye-movement features would be informative for sentiment polarity rather than general reading difficulty. Without a stronger theoretical foundation, it's difficult to accept that these features are capturing sentiment-specific information rather than general text complexity.

2. The marginal improvement when adding cognitive features (0.6% and 1.5% on Datasets 1 and 2 respectively) after including sentiment and sarcasm features suggests these features may not be adding substantial value beyond existing approaches. The paper's claim of "maximum improvement of 3.7% and 9.3%" is misleading, as this is relative to the baseline without any specialized features, not the most effective feature set (Sn+Sr).

3. The feasibility discussion in Section 7 is unpersuasive. The argument that mobile eye-trackers will become commonplace is speculative, and the example of using eye-tracking for Amazon book reviews doesn't clearly demonstrate why this approach would be superior to existing methods. The paper fails to address the practical challenges of obtaining eye-tracking data in real-world applications, especially for large-scale sentiment analysis.

4. The paper does not adequately differentiate between features that capture general reading difficulty (which would affect any text classification task) versus features that specifically capture sentiment complexity. This is a critical gap, as the observed improvements could be due to the features capturing general text complexity rather than sentiment-specific nuances.

**Questions to Authors**

1. How do you differentiate between features that capture reading difficulty (which would affect any text classification task) versus features that specifically capture sentiment complexity? Could you provide evidence that the improvements observed are not simply due to the features capturing general text complexity?

2. The paper claims "cognitive features indeed empower sentiment analyzers to handle complex constructs" but doesn't provide a detailed explanation of the cognitive mechanisms linking eye movements to sentiment understanding. Could you elaborate on the theoretical basis for this connection?

3. Given the marginal improvements when adding cognitive features to the already effective Sn+Sr combination, how would you justify the practical implementation of this approach over simpler, more accessible features that already achieve comparable performance?



Summary: This paper introduces a novel approach to sentiment analysis by incorporating cognitive features derived from eye-movement patterns of human annotators during sentiment labeling. The main contributions are: (1) the development of a new feature set that combines traditional sentiment features with cognitive features from eye-tracking data, (2) empirical demonstration of improved performance on complex sentiment constructs (particularly sarcasm and irony), and (3) detailed feature significance analysis confirming the value of cognitive features for nuanced sentiment detection.

Strengths:
1. The paper presents a well-structured and clearly written exploration of a novel approach to sentiment analysis that integrates cognitive science with NLP.
2. The empirical evaluation demonstrates statistically significant improvements in sentiment analysis performance (up to 9.3% F-score gain) for complex sentiment constructs, particularly when cognitive features are included.
3. The feature significance analysis provides strong evidence that cognitive features are particularly valuable for detecting nuanced sentiment rather than being a general performance booster.

Weaknesses:
1. The theoretical justification for why cognitive features should improve sentiment polarity detection is insufficient. The paper establishes a correlation between eye-movement patterns and reading difficulty but fails to establish a clear causal link to sentiment processing mechanisms.
2. The marginal improvement (0.6-1.5% F-score increase) when adding cognitive features to the already augmented feature set (Sn+Sr) questions the practical significance of this contribution.
3. The feasibility discussion in Section 7 is unconvincing, particularly the e-commerce application example in Section 7.2, which does not clearly demonstrate the added value of cognitive features over traditional text-based analysis.

Questions to Authors:
1. Could you provide a theoretical framework or additional evidence explaining why eye-movement patterns specifically correlate with sentiment polarity rather than just reading difficulty?
2. Given the marginal performance improvement, how would you justify the additional complexity and cost of collecting eye-tracking data compared to using traditional features?
3. In the e-commerce application example, how would the system distinguish between reading difficulty (which might cause regressions) and actual sentiment polarity?

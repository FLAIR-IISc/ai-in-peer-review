This paper proposes a novel approach to sentiment analysis and sarcasm detection by incorporating cognitive features derived from human annotators' eye-movement patterns. The authors claim that this is the first work to integrate cognitive features into the NLP community.

**Summary of the Paper and Main Contributions**

The paper addresses the challenges of sentiment analysis and sarcasm detection in user-generated text by introducing a new set of features based on eye-movement patterns. The authors argue that traditional linguistic features alone are insufficient to capture the nuances of sentiment expression and that cognitive features can provide valuable insights. The main contributions of this work are:

1. The proposal of cognitive features derived from eye-movement patterns as a new set of features for sentiment analysis and sarcasm detection.
2. The evaluation of the effectiveness of these features in improving the accuracy of sentiment analysis and sarcasm detection.
3. The exploration of the feasibility of using cognitive features in real-world applications.

**Strengths**

1. The paper is well-written and easy to follow, making it accessible to a wide range of readers.
2. The idea of using cognitive features for sentiment analysis and sarcasm detection is innovative and has the potential to inspire research in other NLP tasks.
3. The authors provide a thorough evaluation of their approach, including a comparison with traditional linguistic features and an analysis of the feasibility of using cognitive features in real-world applications.

**Weaknesses**

1. The motivation for using cognitive features for sentiment analysis and sarcasm detection is not fully justified. While the authors argue that cognitive features can provide valuable insights, they do not provide a clear explanation of why these features are helpful in detecting sentiment polarities.
2. The improvement in accuracy achieved by incorporating cognitive features is marginal, which may raise questions about the practical significance of this approach.
3. The authors' discussion of the feasibility of using cognitive features in real-world applications is not entirely convincing, particularly with regards to the example provided in Section 7.2.

**Questions to Authors**

1. Can you provide a more detailed explanation of why cognitive features are helpful in detecting sentiment polarities?
2. How do you plan to address the marginal improvement in accuracy achieved by incorporating cognitive features?
3. Can you provide more concrete examples of how cognitive features can be used in real-world applications, and how they can be integrated with traditional linguistic features?

- Novelty: Introduces eye-movement (cognitive) features to sentiment analysis, claiming first integration into the NLP pipeline.  
- Performance: Reports statistically significant but modest F-score improvements when adding cognitive features.  
- Empirical work: Uses and analyzes two public datasets with sarcasm/irony and provides thorough experiments.  
- Presentation: Paper is clear, well-structured, and the experimental setup is well described.  
- Motivation: Intuitively appealing premise that eye movements reflect cognitive signals useful for handling sarcasm/irony.  
- Evaluation rigor: Uses SVM, NB, multi-layer NN and 10-fold cross-validation, plus a held-out set for complex constructs.  
- Feature engineering: Presents a comprehensive feature set including gaze-saliency graph features.  
- Theoretical gap: Lacks strong theoretical justification linking specific eye-movement patterns to sentiment processing.  
- Practical impact: Performance gains are modest relative to the cost and complexity of collecting eye-tracking data.  
- Feasibility concerns: Real-world applicability and reliance on accurate eye-tracking and user consent are insufficiently addressed.  
- Error analysis: Error analysis is brief and missing detailed failure cases and explanations.  
- Question on mechanisms: Requests clearer explanation of cognitive/linguistic theories connecting fixations/regressions to sentiment.  
- Ablation request: Asks for ablation study to identify which cognitive features drive gains and which are redundant.  
- Alternative data sources: Requests exploration of more practical cognitive measures beyond mobile eye-tracking.  
- Example-based evidence: Requests specific held-out examples showing cognitive features helping with sarcasm/irony.  
- References: References are comprehensive.  
- Feature significance: Table 3 usefully highlights importance of gaze-based features.  
- Bias concerns: Recommends discussion of biases introduced by eye-tracking setup and annotator sample.  
- Annotator variability: Suggests analyzing performance variability across annotators to see if some are more predictive.

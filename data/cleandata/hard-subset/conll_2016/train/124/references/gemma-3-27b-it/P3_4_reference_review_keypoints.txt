- Summary: Paper augments sentiment models with eye-tracking–derived cognitive features and shows improved performance, notably on sarcasm/irony datasets.

- Contribution — Novel Feature Integration: Introduces eye-tracking–based cognitive features for sentiment analysis, applying psycholinguistic measures directly to classification.

- Contribution — Empirical Validation: Demonstrates statistically significant accuracy gains across two datasets and multiple classifiers, especially for complex sentiment cases.

- Contribution — Dataset Use & Analysis: Uses publicly available, challenging sentiment datasets (sarcasm/irony) and provides useful analyses for future work.

- Strength — Clear Motivation: Well-articulated hypothesis linking eye movements to cognitive processes relevant for sentiment perception.

- Strength — Comprehensive Feature Engineering: Rich feature set combining traditional sentiment cues, sarcasm indicators, basic gaze metrics, and a novel gaze-saliency graph.

- Strength — Rigorous Evaluation: Multiple classifiers, two datasets, significance testing, and held-out analysis strengthen the empirical claims.

- Strength — Feasibility Discussion: Addresses practical concerns about collecting eye-tracking data and notes increasing availability of mobile eye-trackers.

- Strength — Detailed Error Analysis: Authors acknowledge hardware, aggregation, and other error sources and analyze their impact.

- Weakness — Limited Generalizability: Cognitive features depend on eye-tracker quality, task setup, and participant traits, raising questions about cross-linguistic/cultural transferability.

- Weakness — Feature Redundancy & Interpretability: Large feature set risks redundancy and collinearity; gaze-graph features are complex and hard to interpret.

- Weakness — No Comparison to Deep Learning SOTA: Omits comparisons to state-of-the-art deep learning models and lacks discussion of integrating cognitive features into such architectures.

- Weakness — Moderate Annotation Agreement: Fleiss Kappa scores (0.62, 0.68) are only moderate and the impact on label quality and results is not examined.

- Question — Generalizability: How would cognitive features be adapted or re-calibrated for other languages/cultures?

- Question — Feature Selection: Have feature selection methods (e.g., RFE) or interaction/collinearity analyses been tried to reduce redundancy?

- Question — Deep Learning Integration: How could gaze features be incorporated into deep models (e.g., input channels, attention mechanisms)?

- Question — Annotation Robustness: Have sensitivity analyses been done to assess how varying annotation agreement affects outcomes?

- Question — Graph Features Justification: Can the authors provide intuitive examples linking specific graph metrics (edge density, weighted degrees) to complex sentiment constructs?

- Additional Comment: Formatting is generally good but some references are incomplete.

- Additional Comment: Recommend including a visual example of the gaze-saliency graph to aid interpretation.

- Additional Comment: Claim of “novelty of our feature design” is slightly overstated—the novelty is primarily in applying existing features to sentiment analysis.

- Overall Recommendation: Promising and well-executed work; recommend acceptance with minor revisions.

## Peer Review for "Leveraging Cognitive Features for Sentiment Analysis"

**Summary:**

This paper investigates the use of cognitive features, derived from eye-tracking data of human annotators, to improve sentiment analysis (SA) performance. The authors hypothesize that these features capture nuances in linguistic processing that traditional textual features miss, particularly in complex cases like sarcasm and irony. They augment existing sentiment analysis features with these cognitive features and demonstrate improvements in accuracy across multiple classifiers and datasets.

**Main Contributions (as I see them):**

1. **Novelty of Integrating Eye-Tracking Data with Sentiment Analysis:** The primary contribution is the exploration of a relatively untouched area - incorporating cognitive signals (eye-movements) into the sentiment analysis pipeline. While eye-tracking has been used *around* sentiment analysis (annotation complexity), directly integrating it as features *within* the SA model is novel.
2. **Empirical Demonstration of Improvement:** The paper presents experimental results showing statistically significant improvements in sentiment classification accuracy when cognitive features are added to the feature set, especially on datasets containing sarcastic or ironic text.
3. **Insightful Feature Engineering from Eye-Tracking Data:** The authors go beyond basic eye-tracking metrics and propose more sophisticated features derived from gaze-saliency graphs, demonstrating an understanding of how eye-movement patterns might relate to linguistic processing.

**Strengths:**

1. **Clear Problem Statement and Motivation:** The paper clearly articulates the challenges in sentiment analysis, particularly concerning complex linguistic constructs, and provides a strong rationale for exploring cognitive features as a potential solution.
2. **Rigorous Experimental Setup:** The authors utilize multiple datasets (Dataset 1 and Dataset 2), multiple classifiers (NB, SVM, Multi-layer NN), and a 10-fold cross-validation scheme, lending credibility to their results. The inclusion of a held-out dataset focused on complex constructs further strengthens the findings.
3. **Statistical Significance Testing:** The authors perform McNemar's test to demonstrate the statistical significance of the improvements, adding confidence in the reliability of their observations.
4. **Detailed Feature Description:** The paper provides a comprehensive and well-organized description of the various feature categories – Sentiment, Sarcasm/Thwarting, Cognitive, and Reading Difficulty – detailing each feature and its potential relevance.
5. **Exploration of Gaze-Saliency Graphs:** The introduction of gaze-saliency graph-based features is a particularly interesting aspect, showcasing a creative approach to extracting cognitive signals.

**Weaknesses:**

1. **Limited Generalizability due to Eye-Tracking Data Acquisition:** The biggest weakness lies in the practical limitations of acquiring eye-tracking data. The feasibility section attempts to address this, but the requirement of individual eye-tracking for each text snippet presents a significant barrier to widespread adoption.  The dependence on participant data also introduces potential bias based on annotator characteristics.
2. **Feature Redundancy and Feature Significance Analysis:** While detailed, the feature set is extensive. The paper performs feature significance analysis (Table 3), but a more in-depth analysis is needed to identify truly critical cognitive features and potentially prune redundant ones.  The contribution of each cognitive feature type (basic vs. complex) isn't clearly delineated.
3. **Comparison to Strong Baselines:** While the comparison to the implemented NB, SVM, and rule-based classifiers is useful, the paper acknowledges not comparing to recent deep learning approaches. This limits the understanding of how competitive the proposed approach is against state-of-the-art methods, especially considering deep learning’s ability to automatically learn features.  A more comprehensive comparison is needed.
4. **Lack of Qualitative Error Analysis:** Beyond mentioning errors arise from various factors, a more detailed qualitative analysis of misclassifications, specifically identifying *where* the cognitive features fail to improve performance, would be valuable.  For example, are specific types of sarcasm consistently missed?
5. **Inter-Rater Agreement on Eye-Tracking Data:** While inter-rater agreement on sentiment labels is reported, the impact of potential variability in eye-tracking data across participants is not thoroughly discussed. How robust are the cognitive features to individual differences in reading patterns?



**Questions to Authors:**

1. **Addressing Feature Redundancy:** Could you perform a more rigorous feature selection process (e.g., using recursive feature elimination) to identify the most impactful cognitive features and potentially reduce the dimensionality of the feature space?
2. **Comparison to Deep Learning:** Are you planning to explore integrating these cognitive features into a deep learning framework? If not, could you discuss the potential challenges and benefits of such an approach?
3. **Robustness to Individual Differences:** How sensitive are the cognitive features to variations in reading patterns across different individuals?  Have you investigated techniques to normalize or account for these differences?
4. **Qualitative Error Analysis:** Can you provide examples of specific instances where the addition of cognitive features *failed* to improve classification accuracy and discuss potential reasons for these failures?
5. **Impact of Eye-Tracker Quality:** Dataset 1 uses a higher-quality eye-tracker than Dataset 2. Did you observe any differences in performance or feature effectiveness between the two datasets, potentially attributable to the difference in eye-tracker quality?

**Additional Comments:**

* The writing is generally clear and well-organized.
* The references are comprehensive.
* Consider including a visualization of the gaze-saliency graphs to help readers better understand the feature representation.

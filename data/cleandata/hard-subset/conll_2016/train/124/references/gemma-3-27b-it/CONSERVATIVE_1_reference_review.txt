This paper proposes augmenting traditional sentiment analysis features with cognitive features derived from eye-tracking data, hypothesizing that readers' eye-movement patterns can improve sentiment classification, particularly for nuanced text like sarcasm and irony. The primary claimed contribution is a demonstration of improved accuracy through this feature augmentation. A secondary contribution is the introduction of gaze-based features into the sentiment analysis task, and the creation of gaze-saliency graphs for this purpose.

**Strengths:**

1. **Well-defined Problem & Motivation:** The paper clearly articulates the challenges in sentiment analysis, especially regarding lexical ambiguity, syntactic complexity, and nuanced expressions like sarcasm. The motivation for incorporating cognitive features is logical—understanding *how* people read sentiment-bearing text could indeed inform better automated analysis.
2. **Comprehensive Feature Engineering:** The authors have performed extensive feature engineering, including established sentiment features, features related to sarcasm/thwarting, and a novel set of cognitive features based on eye-tracking. The detailed description of each feature category is commendable.
3. **Empirical Evaluation with Multiple Datasets:** The evaluation is performed on two publicly available datasets, and the inclusion of a held-out dataset for assessing performance on complex constructs is a positive step. The performance comparison with baseline systems (NB, SVM, and a rule-based system) is important for establishing a baseline.
4. **Statistical Significance Testing:** The authors appropriately utilize statistical significance testing (McNemar test) to validate the performance improvements observed. This demonstrates a concern for methodological rigor.

**Weaknesses:**

1. **Limited Novelty of Core Approach:** While the application of eye-tracking data to sentiment analysis is a novel *combination*, the underlying methodology is primarily feature-based. In an era dominated by deep learning approaches that automatically learn feature representations, a feature-engineered approach feels somewhat dated and lacks a strong justification for its continued relevance. The acknowledgement of this limitation in the paper is appreciated, but doesn’t address the fundamental concern.
2. **Practicality Concerns & Data Acquisition:** The dependency on eye-tracking data is a significant practical limitation. The discussion of mobile eye-trackers and user consent is speculative and doesn't alleviate the core issue: obtaining this data at scale is challenging and expensive. The paper doesn’t adequately address the cost/benefit trade-off of this data acquisition.
3. **Overly Optimistic Interpretation of Results:** The improvements in accuracy, while statistically significant, are relatively modest (around 3-9%). The authors frame these as substantial, but the magnitude of improvement requires careful consideration. A more cautious interpretation acknowledging the limitations and the potential for diminishing returns would be more appropriate. The error analysis is brief and doesn’t provide sufficient insight into *why* the system still fails.
4. **Potential for Confounding Factors:** The eye-tracking data reflects cognitive processes during *annotation*, not necessarily during natural reading. This introduces a potential confounding factor, as the annotation task itself may influence eye movements in ways that are not representative of how people process sentiment in real-world scenarios. This is not adequately addressed.

**Questions to Authors:**

1.  What specific steps were taken to control for potential confounding factors related to the annotation task influencing the eye-tracking data?
2.  Given the practical limitations of acquiring eye-tracking data, what is a realistic scenario where this approach could be deployed at scale?
3.  Have you considered incorporating these cognitive features into a deep learning framework (e.g., as additional input features to an LSTM or Transformer model)? If not, why?
4.  Can you provide a more detailed error analysis, categorizing the types of errors the system makes and discussing potential reasons for those errors?

**Additional Comments:**

* The reference list is extensive, which is positive.
* The formatting and writing quality are generally good.
* The saliency graph visualization (Figure 2) is helpful but could benefit from a clearer explanation of the node and edge representations.



Overall, this paper presents a potentially interesting, but ultimately incremental, contribution to the field of sentiment analysis. The novelty is primarily in the *application* of eye-tracking data rather than a fundamentally new methodological approach. The practicality concerns and modest improvements raise questions about the long-term viability of this research direction. I recommend rejection in its current form, but encourage the authors to address the weaknesses outlined above and consider exploring deep learning architectures to leverage these features more effectively.

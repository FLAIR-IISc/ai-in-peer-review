- Investigates using eye-tracking-derived cognitive features to improve sentiment analysis, especially for sarcasm and irony.
- Novel contribution: directly incorporates eye-tracking data as features in a sentiment classification pipeline.
- Shows statistically significant accuracy improvements on two challenging datasets containing sarcasm/irony.
- Provides feature-importance analysis linking specific cognitive features to sentiment processing.
- Clear motivation and psycholinguistic grounding for using eye-movement data.
- Comprehensive feature engineering combining linguistic, cognitive, reading-difficulty, and graph-based gaze features.
- Rigorous experimental setup: two public datasets, cross-validation, baselines, and statistical significance testing.
- Thoughtful error analysis and pragmatic discussion of real-world feasibility and limitations.
- Strong empirical results supporting the core hypothesis, especially on sarcastic/ironic text.
- Scalability/generalizability concerns due to reliance on per-snippet eye-tracking data and limited real-world availability.
- Performance depends on annotation and eye-tracking quality; impact of differing tracker rates (500Hz vs 120Hz) not fully addressed.
- Large feature set risks redundancy and intercorrelation; unclear individual contributions of cognitive features.
- No comparison to state-of-the-art deep learning models and limited discussion of why deep models were excluded.
- Cognitive interpretation depth is limited; more explanation of what specific gaze metrics (e.g., regressions) imply is needed.
- Request for more details on quality control for annotations and handling noisy/unreliable eye-tracking data.
- Ask whether feature correlation analysis or dimensionality reduction was performed and if it informed feature selection.
- Question if differences in eye-tracker quality across datasets affected cognitive feature performance.
- Request concrete examples showing how cognitive features correct misclassifications of sarcasm/irony by traditional models.
- Ask about main challenges in integrating cognitive features into deep learning frameworks beyond computational cost.
- Paper is well-written, well-referenced, and includes helpful result tables.
- Overall recommendation: accept with minor revisions to address the outlined weaknesses.

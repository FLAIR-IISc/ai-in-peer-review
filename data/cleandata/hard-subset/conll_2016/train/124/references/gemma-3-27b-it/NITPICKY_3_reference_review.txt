This paper explores the integration of cognitive features, derived from eye-tracking data, into sentiment analysis models. The core claim is that incorporating these features, reflecting human reading processes, can improve sentiment classification accuracy, particularly for complex linguistic constructs like sarcasm and irony. The primary contribution, as I see it, is the exploration of a novel feature set—cognitive features—for sentiment analysis, rather than a fundamentally new methodology. A secondary contribution lies in demonstrating, through experimentation, that these features *can* yield performance gains. A tertiary contribution is the curated dataset used, although the datasets themselves are not novel, their selection for complexity (sarcasm, irony) is a positive aspect.

**Strengths:**

1. **Novel Feature Exploration:** The investigation of eye-tracking derived features for sentiment analysis is genuinely interesting. The premise that cognitive processes inform sentiment understanding is plausible and the attempt to quantify this through gaze data is commendable. This expands the feature engineering landscape beyond traditional linguistic approaches.
2. **Empirical Validation:** The paper provides empirical results demonstrating improvements in sentiment classification accuracy when cognitive features are included, specifically with SVM models. While the gains aren’t massive, they are statistically significant, and the results on the held-out dataset, focusing on complex constructs, are encouraging.
3. **Detailed Feature Engineering:** The authors demonstrate a relatively comprehensive feature engineering process, breaking down cognitive features into basic and complex types, and carefully considering their potential impact. The inclusion of graph-based features, while complex, showcases a sophisticated approach to data representation.
4. **Clear Problem Definition & Motivation:** The paper clearly articulates the challenges in sentiment analysis (lexical, syntactic, semantic) and provides a logical justification for why cognitive features might address these challenges. The motivation is well-grounded in psycholinguistic theory.

**Weaknesses:**

1. **Lack of Rigorous Baseline Comparison:** The baseline systems used for comparison (NB, SVM, RB) are not state-of-the-art. The paper acknowledges the existence of deep learning approaches (CNNs, RNNs) but dismisses them without a proper comparative evaluation. This weakens the claim of significant improvement; the reported gains might be smaller when compared to contemporary methods. A comparison against at least one established deep learning baseline is crucial.
2. **Data Dependency and Generalizability:** The dataset sizes (994 & 1059 instances) are relatively small. The performance gains observed might be specific to these datasets and not generalize well to other corpora. The Fleiss kappa values (0.62 and 0.68) indicate moderate inter-annotator agreement, which introduces potential noise and bias into the results. A more substantial and diverse dataset would bolster the findings.
3. **Overly Complex Feature Set & Feature Significance Concerns:** The feature set is quite large and includes many potentially redundant features. While the feature significance analysis is presented, the criteria for selecting the “top 20” aren’t clearly defined, and the analysis doesn't definitively prove that all the features are meaningfully contributing to the performance improvement. Dimensionality reduction techniques and more rigorous feature selection methods should be explored.
4. **Practical Limitations and Feasibility Discussion:** While the discussion of feasibility and mobile eye-trackers is appreciated, it remains overly optimistic. The challenges of obtaining user consent for eye-tracking data collection at scale and the inherent noise in mobile eye-tracking systems are glossed over. More realistic considerations of these limitations are needed.
5. **Formatting and Presentation Issues:** There are numerous minor inconsistencies in formatting (e.g., inconsistent use of italics, abbreviations, and capitalization) and some awkward phrasing throughout the paper. These detract from the overall polish and readability. Table 1 is missing a column header. The writing feels somewhat verbose.



**Questions to Authors:**

1.  How would the performance of your system change if compared against a state-of-the-art deep learning based sentiment analysis model (e.g., a pre-trained BERT model fine-tuned for sentiment classification)?
2.  Can you provide a more detailed justification for the selection of the “top 20” features in the feature significance analysis, and explore alternative feature selection methods (e.g., recursive feature elimination)?
3.  What specific strategies would you employ to address the challenges of obtaining user consent and mitigating noise in a real-world deployment of this system using mobile eye-trackers?
4.  Could you provide a more comprehensive error analysis, categorizing the types of errors your system makes and identifying the factors that contribute to these errors?
5.  Please clarify the column headers for Table 1.



**Additional Comments:**

The paper would benefit significantly from a thorough proofread to address the numerous formatting inconsistencies. The authors should also consider providing code and data to facilitate reproducibility. The phrasing regarding “first of its kind” is overstated; while the *combination* of these features may be novel, the individual components (eye-tracking in NLP) have been explored previously.

- Proposes integrating cognitive features from eye-movement data with traditional linguistic features for sentiment analysis.  
- Empirical validation shows significant accuracy gains, especially a 6.1% improvement on irony/sarcasm cases.  
- Provides a comprehensive, well-justified feature engineering pipeline including basic gaze features and saliency-graph features.  
- Novel approach targeting hard cases like sarcasm and irony that standard NLP struggles with.  
- Rigorous experimental setup: two datasets, multiple classifiers, McNemar significance tests, and chi-squared feature importance.  
- Thoughtful, literature-motivated design of multiple cognitive features rather than a single ad-hoc measure.  
- Fails to compare against current state-of-the-art deep learning models (e.g., BERT/fine-tuned transformers).  
- Insufficient practical feasibility analysis regarding eye-tracking hardware cost, deployment, user consent, and privacy.  
- Lacks deeper theoretical explanation linking specific cognitive features (e.g., regression position, saliency graph metrics) to sentiment cues.  
- Potential overfitting risk given small datasets (994, 1059 examples) and unclear PCA/dimensionality-reduction details.  
- Requests a concrete example where cognitive features changed classification for a sarcastic/ironic text.  
- Asks why the authors did not compare to deep learning and whether cognitive features could be integrated into such models.  
- Seeks details on the curation process and distinctions of the held-out dataset.  
- Overall recommendation: weak accept — promising novel idea but must address feasibility concerns and include comparisons with modern deep learning in revision.

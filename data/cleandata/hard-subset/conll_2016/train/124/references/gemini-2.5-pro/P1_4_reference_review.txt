### Summary and Main Contributions

This paper proposes a novel approach to sentiment analysis (SA) by augmenting traditional text-based features with cognitive features derived from the eye-movement patterns of human readers. The central hypothesis is that eye-tracking data can capture the cognitive processing involved in understanding complex linguistic phenomena like sarcasm and irony, which are challenging for standard SA models. The authors extract a comprehensive set of "gaze features," including basic fixation/saccade statistics and more complex features from a novel gaze-saliency graph. Through experiments on two datasets containing challenging short texts, they demonstrate that their combined feature set significantly outperforms models using only linguistic features, with particularly strong gains on texts containing sarcasm and irony.

In my view, the main contributions of this work are:

1.  **A Novel Feature Paradigm for Sentiment Analysis:** The primary contribution is the introduction and successful application of cognitive features from eye-tracking to a core NLP task. This is a highly creative approach that bridges psycholinguistics and computational linguistics, demonstrating that signals from human processing can be a valuable source of information for computational models.
2.  **Demonstrated Efficacy on Hard Cases:** The work provides compelling empirical evidence that these cognitive features are not just a weak additional signal, but are particularly effective for improving classification on notoriously difficult cases like sarcasm and irony. The experiment on the held-out ironic vs. non-ironic dataset provides strong support for this claim.
3.  **Comprehensive Gaze Feature Engineering:** The paper presents a well-motivated and diverse set of gaze features, ranging from simple, interpretable statistics (e.g., regression counts) to more complex graph-based metrics. This detailed feature engineering provides a solid foundation for future work in this area.

### Strengths

1.  **High Novelty and Potential Impact:** The core idea is innovative and refreshing. By moving beyond the text itself to incorporate data about how humans process the text, the paper opens up a promising new research direction. If gaze data becomes more accessible, this paradigm could be applied to numerous other NLP tasks that involve subtle linguistic understanding (e.g., detecting misinformation, humor, or abusive language).
2.  **Strong and Thorough Empirical Evaluation:** The experimental setup is robust. The authors validate their approach on two separate datasets using multiple classifiers. The ablation studies (Table 2) clearly show the incremental benefit of adding sarcasm-related features and then gaze features. The use of statistical significance testing and a feature importance analysis adds credibility to the claims. The targeted analysis in Table 4, showing a much larger improvement on ironic texts (6.1%) versus simpler texts (2.1%), is a particularly strong piece of evidence supporting the central hypothesis.
3.  **Well-Grounded Feature Design:** The cognitive features are not arbitrary; they are grounded in psycholinguistic literature. The authors provide clear intuitions for why features like first-fixation duration, regression count, and long-distance regressions (RSF) should correlate with the processing difficulty associated with complex constructs like sarcasm. This theoretical motivation makes the work more than just a feature engineering exercise.
4.  **Proactive Discussion of Limitations:** The authors commendably address the obvious practical limitation of their work in Section 7 (Feasibility). By discussing the future potential of mobile eye-trackers and outlining concrete application scenarios, they successfully frame the work as a forward-looking proof-of-concept rather than an immediately deployable system, which is an appropriate and honest assessment.

### Weaknesses

1.  **Fundamental Practicality and Generalization Limits:** The most significant weakness is the reliance on having eye-tracking data for the specific text instances being classified. The model is trained and tested on instances where both text and gaze data are available. This means the model cannot be deployed on a new tweet or review from the web without first gathering eye-tracking data for it from multiple users. This severely limits its practical applicability. The model learns a function `f(text_features, gaze_features) -> sentiment`, not a generalizable `f(text_features) -> sentiment` that has been enriched by gaze data during training. This crucial distinction about the deployment scenario is not made sufficiently clear.
2.  **Comparison to Weak Baselines:** The paper explicitly avoids comparison with modern deep learning approaches (e.g., BERT-based classifiers), which represent the current state-of-the-art in sentiment analysis. While the authors' method is feature-based, a comparison is still crucial for context. A strong pre-trained language model might already be capable of capturing much of the contextual nuance that the gaze features help with, potentially rendering the performance gains from this data-intensive approach marginal when compared to a stronger baseline. The baselines used (SVM/NB with traditional features) are not competitive today.
3.  **Potential Confounding Variables:** While the authors control for simple readability and length, the gaze features might be capturing general linguistic complexity rather than signals specific to sentiment or sarcasm. For example, a syntactically complex but non-ironic sentence could elicit similar gaze patterns (e.g., more regressions) as a simpler ironic one. The analysis in Table 4 helps mitigate this, but a more rigorous analysis disentangling sarcasm-induced difficulty from other forms of processing difficulty (e.g., syntactic ambiguity, low-frequency words) would make the claims stronger.
4.  **Information Loss Through Feature Aggregation:** The paper mentions that cognitive features are averaged across all participants for a given text snippet. This simple aggregation method could be discarding rich information. For instance, high variance in gaze patterns across readers could itself be a powerful feature indicating ambiguity or controversy in a text. Exploring more sophisticated aggregation methods (e.g., using variance, min/max values) or treating each reader as a separate instance in a multi-instance learning framework could potentially yield even better results.

### Questions to Authors

1.  Could you please clarify the intended application scenario? Does your framework presuppose that for any new text to be classified, one must first collect eye-tracking data for it? Or do you envision a way to train a model that learns from gaze data but can later make predictions on new text alone?
2.  You note that the gain from adding gaze features (Sn+Sr -> Sn+Sr+Gz) is 0.6% on Dataset 1 and 1.5% on Dataset 2. While statistically significant, these gains on the full dataset seem modest. The more impressive 6.1% gain is on a small, specific subset of ironic texts. Could you comment on whether the complexity of collecting gaze data is justified by these overall performance improvements?
3.  Regarding the comparison to state-of-the-art, would it be possible to use the output predictions from a strong baseline like a fine-tuned RoBERTa as a feature in your SVM? This would help establish whether your cognitive features provide complementary information that is still missing from powerful pre-trained models.

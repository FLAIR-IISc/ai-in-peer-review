### Summary and Contributions

This paper introduces a novel approach to sentiment analysis by augmenting traditional text-based features with cognitive features derived from human eye-tracking data. The central hypothesis is that eye-movement patterns, reflecting cognitive effort, can provide a valuable signal for disambiguating sentiment, particularly in complex cases involving sarcasm, irony, and thwarted expectations. The authors engineer a set of "gaze features" from this data, ranging from basic metrics like fixation duration and regression counts to more complex features derived from a novel "gaze-saliency graph". Experiments on two datasets containing challenging texts show that adding these cognitive features improves classification performance over a strong feature-based baseline, with particularly pronounced gains on texts containing irony and sarcasm.

The primary contribution of this work is conceptual: it proposes and validates the use of real-time human cognitive signals as a new feature modality for a core NLP task. This is a bold and creative step that bridges psycholinguistics and computational linguistics, opening up a new potential direction for research in sentiment analysis and beyond. A secondary contribution is the specific set of cognitive features designed, especially the graph-based ones, which attempt to capture more complex relational information from the gaze patterns.

### Strengths

1.  **Exceptional Novelty and Vision:** The core idea of this paper is highly innovative and thought-provoking. While psycholinguistic insights have long informed NLP model design, the direct use of raw cognitive signals (via eye-tracking) as features for a classification model is a radically new direction. It moves beyond analyzing the text alone to incorporate signals about how humans process that text. This kind of bold, "outside-the-box" thinking is exactly what the field needs to make conceptual leaps, rather than just incremental progress. This paper has the potential to inspire a new sub-field of research.

2.  **Strong Problem-Motivation and Targeted Analysis:** The paper correctly identifies a persistent and difficult challenge in sentiment analysis: understanding complex pragmatic phenomena like sarcasm. The hypothesis that these constructs demand more cognitive effort, which would be reflected in gaze patterns, is compelling. The analysis in Section 5.1 provides strong support for this, demonstrating that the gaze features provide a substantial 6.1% F-score improvement on ironic texts, compared to a much smaller 2.1% on simpler texts. This targeted result is far more convincing than the modest overall gains, as it shows the features are contributing precisely where they were hypothesized to be most useful.

3.  **Thorough Feature Engineering:** The authors do a commendable job of translating raw eye-tracking data into a set of discrete features. The division into "basic" and "complex" gaze features is logical, and the introduction of the gaze-saliency graph is a creative attempt to model the relational aspects of how a reader navigates a sentence. While perhaps not perfect, this initial exploration provides a solid foundation and a useful blueprint for future work in this direction.

### Weaknesses

1.  **Significant Practicality Concerns:** The most glaring weakness is the method's reliance on specialized eye-tracking hardware to gather data for every text to be classified. This makes the approach completely impractical for any real-world application at present. The authors' discussion of future mobile eye-trackers is speculative and does not resolve the immediate and fundamental limitation of the proposed pipeline. While I value novel concepts over immediate applicability, this barrier is so high that it confines the work to a purely academic exploration for the foreseeable future.

2.  **Lack of Comparison to Modern Baselines:** The paper explicitly sets aside comparisons to deep learning models, which currently represent the state-of-the-art in sentiment analysis. The baselines used, while reasonable for a feature-engineering paradigm, are not competitive with modern transformer-based models (e.g., BERT). It is an open question whether the signal from these cognitive features would provide any significant lift over a powerful pre-trained language model, which may already implicitly learn to identify many of the complex constructs that trip up older models. Without this comparison, it is difficult to gauge the true power of the proposed features in the context of the current NLP landscape.

3.  **Modest Overall Performance Gains:** While the improvement on the sarcastic subset is impressive, the overall F-score gains on the full datasets are quite small (+0.6% and +1.5% for the best model). Given the immense cost and complexity of collecting the required cognitive data, this level of improvement raises questions about the overall utility and efficiency of the approach, even if it were to become practical.

### Questions to Authors

1.  The core argument is that gaze features capture the cognitive load associated with complex sentiment. However, gaze patterns are also known to be correlated with more general linguistic complexity (e.g., syntactic difficulty, low-frequency words). Your feature set includes some basic readability scores, but have you conducted any analysis to disentangle the effects of "sentiment complexity" from "general linguistic complexity"? Is it possible the gaze features are primarily acting as a better proxy for general reading difficulty?

2.  While integrating gaze data into a deep learning model was stated to be beyond the scope of this work, could you speculate on how this might be done? For example, could gaze features be concatenated with token embeddings, or could a multi-modal architecture be designed to process the text and a sequence of gaze events jointly? A brief discussion of this would strengthen the paper's forward-looking perspective.

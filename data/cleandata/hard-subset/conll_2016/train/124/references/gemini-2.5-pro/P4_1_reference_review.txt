**Summary of the paper**
This paper proposes a novel method for sentiment analysis by incorporating cognitive features derived from human eye-tracking data. The central hypothesis is that eye-movement patterns, which reflect cognitive processing, can help disambiguate complex linguistic phenomena such as sarcasm and irony that are challenging for traditional text-based sentiment analyzers. The authors define a comprehensive feature set including traditional sentiment and sarcasm-related textual features, as well as two categories of novel cognitive features: basic gaze metrics (e.g., fixation duration, regression counts) and complex features derived from a proposed "gaze-saliency graph". The method is evaluated on two publicly available datasets containing text snippets with associated eye-tracking data from multiple human annotators. Through a series of experiments using SVM, Naïve Bayes, and Neural Network classifiers, the authors demonstrate that augmenting textual features with cognitive features leads to statistically significant improvements in classification accuracy. A targeted experiment on a held-out set further shows that the performance gain is substantially larger for texts containing irony and sarcasm compared to simpler texts.

**Contributions**
1.  **A novel feature set for sentiment analysis based on cognitive signals.** The primary contribution is the introduction and formalization of features derived from human eye-tracking data for the task of sentiment classification. This represents a new modality of information for this task, moving beyond purely textual analysis.
2.  **An empirical demonstration that cognitive features are particularly effective for detecting complex sentiment.** The work provides strong evidence that these gaze-based features are not just redundant signals but are particularly useful for correctly classifying challenging cases involving sarcasm, irony, and thwarted expectations, where text-based features often fail. This is well-supported by a controlled experiment on a held-out dataset.

**Strengths**
1.  **High Novelty:** The core idea of leveraging eye-tracking data as features for a supervised sentiment classification model is highly novel. It bridges research in psycholinguistics and computational linguistics in a direct and applied manner, opening a new direction for feature engineering in sentiment analysis and potentially other NLP tasks that involve subtle linguistic nuances.
2.  **Thorough Empirical Evaluation:** The authors conduct a well-structured and rigorous evaluation. They use two different datasets, multiple standard classifiers (SVM, NB, NN), and perform systematic ablation studies (Table 2) to isolate the contribution of each feature group (sentiment, sarcasm, gaze). The use of statistical significance testing (McNemar test) adds confidence to the reported improvements.
3.  **Targeted Analysis of Sarcasm/Irony:** The experiment described in Section 5.1 is a key strength. By separating a held-out test set into ironic and non-ironic subsets, the authors directly test their hypothesis that cognitive features disproportionately help with complex cases. The results (a 6.1% F-score improvement on ironic texts vs. 2.1% on non-ironic texts) provide compelling evidence to support the central claim of the paper.
4.  **Systematic Feature Engineering:** The paper presents a well-organized feature set, clearly distinguishing between established textual features and the novel cognitive ones. The proposal of "basic" and "complex" gaze features, including the introduction of a gaze-saliency graph to derive more intricate relational features, demonstrates a thoughtful approach to translating raw gaze data into meaningful signals for a machine learning model.

**Weaknesses**
1.  **Significant Practicality Concerns:** The proposed method requires eye-tracking data to be available at prediction time for any new text instance. This is a major limitation for real-world applicability. The authors dedicate Section 7 to discussing future feasibility with mobile eye-trackers, but this remains speculative. The reliance on specialized hardware for data acquisition makes the approach impractical for general-purpose sentiment analysis in its current form and limits its potential impact. The data collection process is expensive and not scalable.
2.  **Lack of Comparison with Contemporary State-of-the-Art:** The paper explicitly omits comparison with deep learning models, which were state-of-the-art for sentiment analysis at the time. The justification that deep models learn features automatically is insufficient, as the primary goal is to advance performance on the task. A comparison against strong, contemporary baselines is necessary to contextualize the performance gains. Without it, it is unclear if this feature-engineering approach outperforms end-to-end learning methods on these specific challenging datasets.
3.  **Potential Confounding between Cognitive and Linguistic Complexity:** While the authors include readability features to control for general text difficulty, the paper does not fully disentangle whether the gaze features are capturing signals specific to sentiment nuances (like the cognitive effort of resolving incongruity in sarcasm) or are simply a proxy for general linguistic complexity (e.g., complex syntax, rare words) that happens to be correlated with the sarcastic examples in the datasets. A more detailed analysis of the correlation between the proposed gaze features and standard linguistic complexity metrics would be needed to strengthen the claim that they capture sentiment-specific processing.
4.  **Information Loss Through Data Aggregation:** The cognitive features are derived by averaging gaze metrics across all participants for a given text snippet. This approach discards potentially valuable information contained in the variance and distribution of gaze patterns among different readers. It is plausible that high variance in reading patterns for a given text could itself be a strong signal of ambiguity or controversy, which is lost in the current feature-extraction process.

**Questions to Authors**
1.  Regarding the method's practicality, have you considered a transfer-learning or multi-task learning setup? For instance, could a model be trained with both text and gaze data, but then be used to make predictions at test time using only text? Or could a model be trained to predict "gaze features" from text alone, which are then used as intermediate features for a final sentiment classifier? This would address the key limitation of requiring an eye-tracker for inference.
2.  The feature significance analysis in Table 3 shows that gaze features are highly ranked. Could you provide a more qualitative analysis for a few examples? For instance, for a sarcastic sentence, does a feature like `LREG` (Largest Regression Position) or high fixation counts (`FDUR`) consistently align with the specific words or phrases that create the sarcastic incongruity? This would provide more intuitive validation of your hypothesis.
3.  While a direct comparison to deep learning models was omitted, could the cognitive features be integrated into a neural architecture? For example, could sentence-level gaze features be concatenated with a sentence representation (e.g., from a BiLSTM or BERT) before the final classification layer? This would provide a framework for testing the utility of these features in a modern, state-of-the-art setting.

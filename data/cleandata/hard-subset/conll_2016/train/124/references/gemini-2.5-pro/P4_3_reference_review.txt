**Summary and Contributions**
This paper proposes a method to improve sentiment analysis by augmenting traditional text-based features with a novel set of cognitive features derived from human eye-tracking data. The authors hypothesize that eye-movement patterns, such as fixations and saccades, can capture reader reactions to complex linguistic phenomena like sarcasm and irony, which are challenging for standard sentiment classifiers. They introduce a range of "gaze features," including basic statistical measures (e.g., regression counts, fixation durations) and more complex metrics derived from a proposed "gaze-saliency graph." The approach is evaluated on two publicly available datasets containing sentiment-annotated texts and corresponding eye-tracking data. Experiments using SVM, Naïve Bayes, and Neural Network classifiers show that the inclusion of cognitive features provides a statistically significant improvement in classification performance over a baseline of sentiment and sarcasm-related textual features, especially for texts containing ironic or sarcastic content.

The main contributions of this work are:
1.  The introduction and formalization of a novel set of cognitive features for sentiment analysis derived from eye-tracking data. This includes both simple gaze metrics and more complex features extracted from a proposed graph-based representation of gaze patterns, representing a new modality of information for this task.
2.  A thorough empirical demonstration, across two datasets with different characteristics, that these cognitive features improve the performance of supervised sentiment classifiers. The analysis effectively shows that the primary benefit of these features is in correctly classifying texts with complex semantic and pragmatic challenges, such as sarcasm.

**Strengths**
1.  **High Novelty of the Approach:** The central idea of using eye-tracking data as a feature source for sentiment analysis is highly innovative. While psycholinguistic studies often use eye-tracking to analyze language processing, this work is among the first to integrate this data directly into a predictive NLP model. The proposed "gaze-saliency graph" is also a novel method for extracting structured information from raw gaze patterns. This represents a creative and potentially impactful direction for feature engineering.
2.  **Solid and Rigorous Evaluation:** The experimental design is sound and comprehensive. The authors validate their approach on two distinct datasets, collected using different quality eye-trackers, which supports the robustness of their findings. The ablation study (Table 2) clearly isolates the contribution of each feature group (sentiment, sarcasm, gaze), providing a clear and convincing measure of the added value of the cognitive features. The targeted experiment on a held-out set of ironic vs. non-ironic texts (Table 4) provides strong, direct evidence for the paper's central hypothesis that gaze features are particularly helpful for complex constructs.
3.  **Clear Presentation and Organization:** The paper is well-written and logically structured. The introduction effectively frames the problem by categorizing the challenges in sentiment analysis. The feature sets are described in sufficient detail in Section 4, and the experiments and results are presented in a clear and interpretable manner. This clarity makes the work easy to understand and appreciate.
4.  **Proactive Discussion of Practicality:** The authors acknowledge the significant practical limitation of their approach—the need for specialized eye-tracking data—and dedicate a section (Section 7) to discussing its future feasibility. By addressing this obvious criticism and outlining potential real-world scenarios with emerging mobile eye-tracking technology, they provide important context for their work and position it as a forward-looking proof-of-concept.

**Weaknesses**
1.  **Limited Practical Applicability and Generalization:** The primary weakness is the method's reliance on eye-tracking data, which is not available for the vast majority of text. The proposed system cannot classify a new, unseen sentence without first having multiple users read it while being monitored by an eye-tracker. This fundamentally changes the task from general sentiment analysis to sentiment analysis of a text *given its associated human reading patterns*. This makes the system unsuitable for most standard, real-world deployment scenarios. While the feasibility section is welcome, it discusses a future state that has not yet been realized, and the immediate utility of the model is therefore limited to an analytical one.
2.  **Lack of Comparison to Modern Baselines:** The paper explicitly omits comparisons to deep learning models (e.g., CNNs, Transformers), stating that the work is feature-based. In the current NLP landscape, this is a significant omission. State-of-the-art performance on sentiment analysis is dominated by large pre-trained language models like BERT. Without a comparison, it is impossible to know if the performance gains achieved by adding cognitive features to a classical SVM would allow it to compete with, let alone surpass, a strong text-only neural baseline. This makes it difficult to assess the overall contribution to the state of the art in sentiment analysis as a task.
3.  **Ambiguity in Feature Aggregation:** The cognitive features are aggregated from multiple participants (seven for Dataset 1, five for Dataset 2) by "averaging". The paper lacks precise detail on how this aggregation is performed for all 21 cognitive features, particularly the complex graph-based ones. For example, it is unclear whether a separate gaze-saliency graph is created for each participant and the resulting numeric features (e.g., edge density) are averaged, or if the gaze data is first pooled to create a single aggregate graph. This ambiguity hinders the reproducibility of the work and leaves open questions about potential information loss during the aggregation process.
4.  **Potential for Confounding Variables:** The paper argues that gaze features capture complex sentiment cues like sarcasm. However, eye movements are also strongly influenced by lower-level text properties like word frequency, word length, and syntactic complexity. The authors include readability and length features, but the analysis does not sufficiently disentangle the effects of these confounders. It is possible that the predictive power of the gaze features comes from them acting as proxies for these textual properties rather than capturing a distinct cognitive signal related to sentiment. A more detailed analysis controlling for these factors would strengthen the claim that gaze features provide unique semantic and pragmatic information.

**Questions to Authors**
1.  Could you please clarify the exact procedure for aggregating gaze features across multiple participants? Specifically for the "gaze-saliency graph" features, are the final feature values (e.g., "Edge density") an average of the values computed on each participant's individual graph?
2.  The experimental setup uses gaze data averaged across 5-7 readers. However, the proposed real-world application scenario involves tracking a single user reading a review. A single user's gaze data is likely to be significantly noisier and more idiosyncratic than an averaged signal. How would you expect the model to perform in this more realistic single-reader setting? Have you conducted any experiments to evaluate this?
3.  While a full implementation was noted as out-of-scope, could you elaborate on how you envision these cognitive features being integrated into modern neural architectures for sentiment analysis? For instance, could gaze-based word-level features be concatenated with word embeddings, or could sentence-level gaze features be used as auxiliary inputs to a final classification layer?

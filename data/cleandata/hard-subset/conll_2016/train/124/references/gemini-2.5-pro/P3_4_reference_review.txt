### Summary and Contributions

This paper proposes a novel method for sentiment analysis (SA) by augmenting traditional text-based features with "cognitive features" derived from human eye-tracking data. The central hypothesis is that eye-movement patterns, which reflect cognitive processing, can provide valuable signals for disambiguating complex linguistic phenomena like sarcasm, irony, and thwarted expectations, which are challenging for standard SA systems.

The main contributions of this work are:
1.  **Novel Feature Set:** The primary contribution is the introduction and evaluation of a rich set of cognitive features for SA, derived from eye-tracking data. These range from basic gaze metrics (fixation duration, regressions) to more complex features derived from a novel "gaze-saliency graph" representation.
2.  **Targeted Application and Analysis:** The authors demonstrate that these cognitive features are particularly effective for improving classification on difficult texts containing sarcasm and irony. The experiment on a held-out dataset that explicitly separates simple and complex constructs provides strong evidence for this claim.
3.  **Empirical Validation:** The work provides a solid empirical validation of the proposed approach on two publicly available datasets, showing statistically significant improvements over strong baselines that already incorporate features designed for sarcasm and thwarting.

### Strengths

1.  **High Novelty and Interesting Research Direction:** The core idea of using cognitive signals from eye-tracking to enhance a core NLP task like sentiment analysis is highly innovative. It bridges computational linguistics and psycholinguistics in a practical, task-oriented way. This moves beyond simply analyzing gaze data to using it as a feature source, which is a compelling and underexplored direction for the field.

2.  **Strong and Convincing Experimental Design:** The evaluation is a key strength. The authors first establish that their chosen datasets are indeed challenging for standard SA systems (Table 1). More importantly, the experiment in Section 5.1 on a held-out set is excellent. By splitting the data into "Irony" and "Non-Irony" subsets, they directly test their central hypothesis and show that the performance gain from cognitive features is substantially larger on the complex ironic texts (a 6.1% F-score increase) than on simpler texts (2.1%). This targeted analysis is far more convincing than a single aggregate performance number.

3.  **Well-Motivated and Thorough Feature Engineering:** The cognitive features are not arbitrary; they are grounded in psycholinguistic research. For example, linking regression counts (REG, RSF) to syntactic/semantic reanalysis, which is common when processing irony, is well-motivated. The creation of the gaze-saliency graph is an interesting attempt to capture more complex relational information from the scanpaths, even if the resulting features (e.g., weighted degrees) are heuristic. The feature set is comprehensive and clearly described.

4.  **Reproducibility:** The use of two publicly available datasets with associated eye-tracking data is a significant strength, allowing for the potential reproduction and extension of this work by other researchers.

### Weaknesses

1.  **Major Practicality and Feasibility Concerns:** The most significant weakness is the reliance on eye-tracking data at inference time. A practical SA system needs to classify unseen text for which no gaze data exists. The authors acknowledge this in Section 7, suggesting future inexpensive mobile eye-trackers. However, this is highly speculative and does not address the fundamental limitation that the proposed method, in its current form, is not deployable in any real-world scenario. The paper would be strengthened by framing the contribution more as a powerful *analytical* tool for understanding linguistic complexity and model errors, rather than as a direct proposal for a practical classification system.

2.  **Marginal Overall Performance Gains:** While the gains on the ironic subset are impressive, the overall improvement from adding gaze features (Gz) on top of the already strong text-based baseline (Sn+Sr) is modest: 0.6% on Dataset 1 (72.9% to 73.5%) and 1.5% on Dataset 2 (70.3% to 71.8%). Given the immense cost and complexity of acquiring eye-tracking data, this raises questions about the cost-benefit of the approach for general-purpose sentiment analysis. The paper's impact hinges almost entirely on the targeted improvements for complex cases.

3.  **Lack of Comparison with Modern Neural Baselines:** The paper explicitly sets aside deep learning models, which is a reasonable scoping choice for a feature-engineering paper. However, this is also a significant limitation in 2024. State-of-the-art SA is dominated by large pre-trained language models (e.g., BERT, RoBERTa). It is an open question whether these models, trained on massive datasets, already implicitly learn to handle the kinds of complex phenomena (like incongruity in sarcasm) that the gaze features are designed to capture. Without at least a reference comparison, it is difficult to gauge the true contribution of these cognitive features relative to the current state of the art. It's plausible that a fine-tuned BERT model might outperform the `Sn+Sr+Gz` system, potentially rendering the cognitive features redundant.

4.  **Complexity of Gaze Features:** The paper introduces a large number of gaze features, including a novel graph-based representation. However, there is no ablation study to determine which of these features are most impactful. It is possible that the bulk of the performance gain comes from a few simple, highly informative features (e.g., `REG` or `RSF`), and that the complex graph-based features (`ED`, `F1H`, etc.) add little value. Such an analysis would provide more insight into which cognitive signals are truly important and would simplify the proposed feature set.

### Questions to Authors

1.  Regarding the feasibility issue: Could you re-characterize the primary contribution of your work? Would it be more accurate to frame this not as a method for a better deployable classifier, but as a novel methodology for using gaze data to create datasets annotated with "cognitive difficulty," which could then be used to train or evaluate purely text-based models to better handle complex linguistic phenomena?

2.  Could you provide an ablation study on the cognitive features? Specifically, how does the performance of a system with only the "Basic gaze features" (Section 4.4) compare to one with the full set including the "Complex gaze features" (Section 4.5)? This would help clarify the contribution of the novel gaze-saliency graph.

3.  In your 10-fold cross-validation, how were the multiple annotators for each text snippet handled? Were the gaze features for a snippet first averaged across all annotators, and then the snippets were split into folds? Or was the splitting done in a way that prevented data from the same annotator being used to generate features for both the train and test sets within a fold? Please clarify this aspect of the experimental setup.

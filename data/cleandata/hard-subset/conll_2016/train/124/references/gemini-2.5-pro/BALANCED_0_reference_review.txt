**Summary and Contributions**

This paper introduces a novel method for sentiment analysis by augmenting traditional text-based features with "cognitive features" derived from eye-tracking data. The central hypothesis is that the eye-movement patterns of human readers, recorded as they annotate texts for sentiment, capture cognitive signals related to the processing of complex linguistic phenomena like sarcasm, irony, and thwarted expectations. The authors extract a range of gaze-based features (e.g., fixation duration, regression counts, graph-based features) and combine them with standard sentiment and sarcasm-detection features. Experiments on two datasets containing eye-tracking data show that this combined feature set improves classification accuracy over text-only models, with a particularly pronounced improvement on texts containing irony and sarcasm.

In my view, the main contributions of this work are:
1.  **A novel, multi-modal approach to sentiment analysis:** The primary contribution is the introduction and validation of eye-tracking data as a new source of features for a core NLP task. This moves beyond purely textual information and explores the link between human cognitive processing and linguistic interpretation, which is a highly innovative direction for the field.
2.  **A strong empirical demonstration of the value of cognitive signals for complex language:** The paper provides compelling evidence, particularly through its targeted analysis of ironic vs. non-ironic texts, that gaze features offer a signal that is especially useful for instances that are challenging for traditional text-based models. This helps to validate the underlying hypothesis that eye movements reflect the processing of higher-order semantic and pragmatic nuances.

**Strengths**

1.  **High Novelty and Potential Impact:** The core idea of using cognitive signals from eye-tracking to directly improve an NLP model is creative and forward-looking. While eye-tracking has been used to study linguistic phenomena, its integration as a feature source in a predictive model for sentiment analysis is novel. This work opens up a promising research avenue that could be extended to other challenging NLP tasks where subtle interpretation is key, such as abusive language detection, fake news identification, or question answering.
2.  **Thorough and Well-Motivated Feature Engineering:** The authors have designed a comprehensive feature set. The cognitive features are well-grounded in psycholinguistic literature and range from basic, interpretable metrics (e.g., first-fixation duration, regression count) to more complex and novel graph-based representations ("gaze-saliency graphs"). The ablation study (Table 2) and feature significance analysis (Table 3) effectively demonstrate that these new features are indeed informative and contribute to the model's performance.
3.  **Convincing Experimental Analysis on Challenging Datasets:** The evaluation is well-designed. The authors first establish that their chosen datasets are indeed difficult for standard SA systems (Table 1). The subsequent experiments show consistent improvements across multiple classifiers when cognitive features are added. The dedicated experiment on a held-out set isolating complex (ironic) vs. simple texts (Table 4) is a key strength. It provides strong, direct evidence for the central claim that gaze features are most beneficial for handling linguistic complexity, showing a 6.1% F-score improvement for ironic texts compared to 2.1% for non-ironic texts.

**Weaknesses**

1.  **Critical Practicality and Scalability Limitations:** The proposed method's reliance on eye-tracking data for feature extraction is a major practical barrier. To classify a new text, the model appears to require that text to be read by a panel of human participants while their eye movements are recorded. This is an expensive, time-consuming process requiring specialized hardware, which makes the system undeployable for any real-world, large-scale sentiment analysis application (e.g., analyzing social media streams or product reviews). The discussion in Section 7 regarding the future availability of mobile eye-trackers is speculative and does not address the fundamental limitation for current and near-future applications.
2.  **Unclear Methodological Setup (Training vs. Test Time):** The paper is unclear about how features are generated at test time. The features seem to be instance-specific, meaning they are derived from the same data point that is being classified. This conflates the feature extraction process with the test instance itself, which is a departure from the standard machine learning paradigm where a trained model can make predictions on new, unseen data using only the input (i.e., the text). This makes the work feel more like an explanatory analysis of the relationship between gaze and sentiment rather than a proposal for a generalizable sentiment analysis system. A system that requires a human-in-the-loop for every prediction is not a practical SA tool.
3.  **Limited Comparison to Modern State-of-the-Art Models:** The authors explicitly decline to compare their method against deep learning-based approaches (e.g., BERT, RoBERTa), which represent the current state-of-the-art in sentiment analysis. While the work is feature-based, this omission makes it difficult to assess the true significance of the performance gains. Modern pre-trained language models are known to be very effective at capturing complex linguistic context, and it is possible they already model some of the nuances that the gaze features are helping with. A comparison would be necessary to understand if these cognitive features provide information that is orthogonal and complementary to what is learned by large language models. The baselines used, while appropriate for a feature-based comparison, are not representative of today's strongest SA systems.

**Questions to Authors**

1.  Could you please clarify the intended deployment scenario for this model? Does classifying a new, unseen text instance require collecting fresh eye-tracking data for that specific instance? If so, this seems fundamentally different from a standard SA system.
2.  Have you considered a two-stage approach to overcome the practical limitations? For instance, one could first train a model to *predict* gaze features directly from text, and then use these *predicted* (synthetic) gaze features in the final sentiment classifier. This would decouple the system from the need for eye-tracking hardware at inference time and make it a truly general-purpose tool.
3.  Given the strength of modern transformer models, how do you envision cognitive features being integrated into such architectures? Could gaze data be used, for example, to create attention priors, as a form of supervision in a multi-task learning setup, or to fine-tune the model on examples identified as cognitively complex? This would help position your contribution within the context of current state-of-the-art NLP.

### Summary and Contributions

This paper proposes a novel approach to sentiment analysis (SA) by incorporating "cognitive features" derived from eye-tracking data. The central hypothesis is that eye-movement patterns of human readers can capture cognitive signals related to processing complex linguistic phenomena like sarcasm, irony, and thwarted expectations, which are challenging for traditional text-based models. The authors compile an extensive feature set combining standard sentiment features, specialized sarcasm/irony features, and a novel set of basic and complex gaze-based features (e.g., fixation durations, regressions, graph-based metrics from a "gaze-saliency graph"). Experiments on two existing datasets with eye-tracking annotations show that augmenting a strong text-based feature set with these cognitive features yields statistically significant improvements in classification performance, particularly for texts containing irony and sarcasm.

In my view, the main contributions are:
1.  **Novel Feature Source:** The primary contribution is the introduction and successful application of cognitive features from eye-tracking data to the task of sentiment analysis. This represents a genuinely new direction for feature engineering in this domain, moving beyond purely textual or stylistic cues.
2.  **Targeted Empirical Validation:** A strong contribution is the experimental design that specifically tests the hypothesis on a held-out set of ironic vs. non-ironic texts. The demonstration that cognitive features provide a substantially larger performance boost on the complex (ironic) subset provides compelling evidence for the authors' claims.
3.  **Comprehensive Feature Engineering:** The paper presents a detailed and well-motivated set of cognitive features, including the novel "gaze-saliency graph" for extracting higher-order interaction patterns, which could be valuable for future work in this area.

### Strengths

1.  **High Novelty and Potential Impact:** The core idea is highly innovative and thought-provoking. By bridging psycholinguistics and NLP, the paper opens up a new avenue for research where implicit human cognitive signals can be used to enrich computational models of language. If the data acquisition challenges can be overcome, this paradigm could be extended to other challenging NLP tasks that involve ambiguity and pragmatic reasoning (e.g., emotion detection, argument mining, hate speech detection).

2.  **Strong and Convincing Experimental Results:** The authors provide solid empirical backing for their claims. The improvements are consistent across two different datasets (with different eye-trackers) and three different classifiers. The ablation study clearly shows the incremental value of each feature set (Sn -> Sn+Sr -> Sn+Sr+Gz). The use of statistical significance testing (McNemar) strengthens the conclusions.

3.  **Excellent Analysis of Feature Utility:** The feature importance analysis (Table 3) and, more crucially, the experiment on the ironic/non-ironic split (Table 4) are key strengths. The latter provides direct evidence that the cognitive features are not just capturing generic difficulty but are particularly effective at resolving the very types of complex sentiment expression the paper aims to tackle. This targeted analysis significantly elevates the work beyond a simple "more features are better" story.

### Weaknesses

1.  **Major Practicality and Feasibility Concerns:** The biggest weakness is the fundamental impracticality of the proposed method in its current form. The model requires eye-tracking data from multiple readers *for the specific text instance being classified*. This is not a viable scenario for any real-world application (e.g., classifying a new tweet or product review). The discussion in Section 7 about future mobile eye-trackers is highly speculative and does not address the core problem: you need gaze data for the *test* instance. The paper would be significantly stronger if it either (a) framed the work as a method for analyzing existing corpora where gaze data is available, or (b) proposed a two-stage approach where gaze features are first *predicted* from the text, and these predicted features are then used for the downstream SA task. Without addressing this, the work remains an interesting academic experiment with very limited practical applicability.

2.  **Lack of Strong Modern Baselines:** The paper explicitly sets aside deep learning models, arguing that the focus is on feature engineering. While understandable, this omission makes it difficult to gauge the true significance of the proposed method. A standard fine-tuned BERT or RoBERTa model would serve as a crucial point of comparison. It is possible that a powerful pre-trained language model already implicitly captures some of the complexity that the gaze features make explicit, potentially reducing the performance gap. Comparing the ~73.5 F1 score with gaze features to a strong transformer-based baseline is necessary to contextualize the results in the current NLP landscape.

3.  **Potential Confounding between Pragmatic Difficulty and General Text Complexity:** The paper argues that gaze features help resolve pragmatic challenges like sarcasm. However, many gaze features (e.g., regression counts, fixation durations) are also well-known correlates of general lexical or syntactic difficulty. While the authors include readability scores as control features, this is insufficient to fully disentangle these effects. It is plausible that the gaze features are simply a much better proxy for overall processing difficulty than the included textual features, and this improved difficulty measure, rather than a specific sarcasm signal, is driving the performance gain. A more nuanced analysis to de-correlate these factors would be beneficial.

### Questions to Authors

1.  Could you please clarify the intended deployment scenario for this model? Is the assumption that gaze data will always be available for new, unseen texts at inference time? If not, have you considered a model that first predicts gaze features from text alone, and then uses those predictions as input to the sentiment classifier? This seems like a critical step towards making the approach practical.

2.  The gaze features are averaged across the handful of participants for each text. This discards potentially useful information about annotator disagreement. For instance, high variance in reading patterns across users could itself be a strong signal for ambiguity, controversy, or sarcasm. Have you explored using the variance or other statistics of the gaze features across participants as additional features?

3.  Regarding the comparison to baselines: While I understand the feature-engineering focus, could you provide some intuition on how you expect your approach to compare against a fine-tuned BERT-based classifier on these datasets? Establishing this comparison would be very valuable for understanding the relative benefits of leveraging external cognitive data versus relying on the implicit knowledge in large language models.

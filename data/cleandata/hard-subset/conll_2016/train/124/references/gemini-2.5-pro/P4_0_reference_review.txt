### Summary of the Paper
This paper proposes a novel approach to sentiment analysis by augmenting traditional text-based features with cognitive features derived from human eye-tracking data. The central hypothesis is that eye-movement patterns, recorded while annotators read and label texts, contain signals that can help disambiguate complex linguistic constructs like sarcasm and irony, which are challenging for standard sentiment analysis models. The authors define a comprehensive set of "gaze features," including basic metrics (e.g., fixation duration, regression counts) and more complex graph-based features derived from "gaze-saliency graphs." They evaluate their method on two existing datasets containing short texts with sentiment and eye-tracking annotations. Experiments using SVM, Naïve Bayes, and Neural Network classifiers show that the inclusion of cognitive features consistently improves sentiment classification accuracy, particularly for texts containing sarcasm and irony.

### Contributions
1.  **Introduction of Cognitive Features for Sentiment Analysis:** The primary contribution is the novel use of features derived from eye-tracking data for sentiment classification. The work successfully demonstrates that cognitive signals captured through eye movements can serve as a valuable additional modality for a classic NLP task, bridging psycholinguistics and computational linguistics.
2.  **Demonstrated Efficacy on Complex Linguistic Phenomena:** The paper provides strong evidence that these cognitive features are particularly effective for identifying sentiment in texts with challenging constructs like sarcasm, irony, and thwarted expectations. An experiment on a held-out set explicitly shows a more significant performance gain on ironic texts compared to simpler ones.
3.  **Comprehensive Feature Engineering:** The authors propose a detailed and well-motivated set of cognitive features. This includes not only basic, direct measurements from eye-tracking (fixation duration, saccade length, regressions) but also novel, higher-level features derived from a proposed gaze-saliency graph, capturing more complex inter-word reading patterns.

### Strengths
1.  **High Novelty:** The core idea of leveraging eye-tracking data as features for sentiment analysis is highly original. While eye-tracking has been used in psycholinguistic studies or to analyze annotation difficulty, its direct integration as a feature source for a predictive NLP model is a novel contribution that opens a new direction for research.
2.  **Thorough and Systematic Evaluation:** The experimental design is sound. The authors test their approach on two distinct datasets, collected with different eye-trackers, enhancing the robustness of their findings. The use of multiple classifiers and a systematic feature ablation study (comparing sentiment features, sarcasm features, and gaze features incrementally) clearly isolates and quantifies the benefit of the proposed cognitive features.
3.  **Strong Empirical Support for the Hypothesis:** The results convincingly support the paper's central claims. The performance improvement after adding gaze features is consistent across both datasets and all classifiers. The targeted experiment on the held-out dataset (Table 4) provides compelling evidence that the gaze features are not just capturing general reading difficulty but are specifically helpful for nuanced, non-literal language, where textual cues alone may be insufficient. The 6.1% F-score improvement on ironic texts is a significant finding.
4.  **Clarity and Presentation:** The paper is well-written and easy to follow. The motivation for using cognitive features is clearly articulated, the features themselves are described in sufficient detail, and the experimental setup and results are presented transparently.

### Weaknesses
1.  **Significant Practicality and Deployment Concerns:** The primary weakness is the method's reliance on eye-tracking data at inference time. A standard sentiment analysis tool is expected to classify any given text snippet. This approach, however, requires that the text snippet first be read by one or more human subjects while their eye movements are recorded. This makes the system impractical for most real-world applications (e.g., real-time social media analysis, large-scale review processing). While the authors discuss potential future scenarios with mobile eye-trackers in Section 7, this remains a speculative and substantial barrier to adoption. The paper evaluates a model that predicts sentiment given `{text, gaze_data}`, not one that predicts sentiment given only `{text}`.
2.  **Unclear Generalization to New, Unseen Text:** The experimental setup is based on cross-validation within datasets where every instance is already paired with gaze data. It is not clear how the model would classify a new piece of text for which no eye-tracking data is available. The model is trained on features that will be absent during a typical deployment scenario. The authors do not propose a method to either predict the gaze features from text alone or to operate without them, which limits the applicability of the trained model.
3.  **Limited Comparison to State-of-the-Art:** The authors explicitly decline to compare their method against modern deep learning models (e.g., BERT-based classifiers), stating that their approach is feature-based. This is a notable omission, as these models represent the current state-of-the-art for sentiment analysis. Without this comparison, it is difficult to assess whether the absolute performance of the proposed model is competitive, or if the information captured by the gaze features can already be effectively extracted from text alone by a powerful enough text-based model. A comparison on the same datasets would be necessary to truly contextualize the contribution of the cognitive features.

### Questions to Authors
1.  Could you please clarify the intended deployment scenario for this system? Is the model intended for use only in settings where new eye-tracking data can be collected for each text to be classified? Or have you considered a two-stage approach where gaze features are first *predicted* from the text, and then used in the final classifier? The latter would significantly improve the practical applicability of this work.
2.  Following on the previous point, how does the model trained with gaze features perform on a new text instance when the gaze features are simply unavailable (e.g., set to zero or a mean value)? This would provide a more direct assessment of its generalizability to the standard sentiment analysis task formulation.
3.  To better situate the performance of your system, could you provide results from a strong, text-only deep learning baseline (e.g., a fine-tuned BERT or RoBERTa model) on your two datasets? This comparison would be invaluable for understanding the absolute performance and quantifying the unique contribution of cognitive features over and above powerful textual representations.

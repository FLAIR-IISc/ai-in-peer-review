Summary
The paper investigates augmenting sentiment analysis (SA) with cognitive features derived from eye-tracking. Using two existing datasets that include readers’ gaze data during sentiment annotation, the authors define a large set of handcrafted features: standard sentiment features, features targeting sarcasm/irony/thwarting, gaze-derived features (both basic and graph-based), and readability proxies. They train NB/SVM/MLP classifiers with 10-fold CV on each dataset and report that adding gaze features to textual features improves F1, especially on examples with irony/sarcasm. They also argue for the feasibility of gaze acquisition via mobile eye-trackers.

Main contributions (as I see them)
- Empirical evidence that gaze-derived features provide small but statistically significant gains on top of strong textual features for SA on two gaze-annotated datasets, with larger relative gains on sarcastic/ironic items.
- A catalog of gaze features, including a “gaze-saliency graph” and several weighted-degree statistics, intended to capture cognitive processing during reading.

Strengths
- Targeting hard SA phenomena: The datasets contain many instances with sarcasm/irony/thwarting, where textual cues alone are often insufficient. The held-out analysis showing larger gains on sarcastic items is aligned with the motivation.
- Systematic feature engineering: The paper provides a broad inventory of features (sentiment, sarcasm/irony-targeted, gaze, readability). The combination Sn+Sr+Gz is a reasonable, layered approach.
- Clear empirical takeaways: On SVMs, adding Sr to Sn yields substantial gains (D1: +3.1 F1; D2: +7.8 F1), and adding gaze yields further improvements (+0.6 and +1.5 F1, respectively). The held-out subset shows larger relative gains on ironic texts (+4.2 F1 over Sn+Sr).
- Initial significance and importance analyses: McNemar tests are reported; chi-squared feature ranking suggests gaze features appear among the most informative.

Weaknesses
- Practicality and problem definition: The proposed SA model requires gaze at inference time. This is a severe limitation for most real-world SA settings (e-commerce, social media, review mining). The feasibility section is speculative and does not address how models would perform without real-time gaze (e.g., by predicting gaze from text or using proxies).
- Attribution of gains: The majority of the improvement comes from textual sarcasm/thwarting features (Sn→Sn+Sr), not from gaze (Sn+Sr→Sn+Sr+Gz is only +0.6/+1.5 F1). The central claim about the utility of cognitive features is therefore modest on the main datasets. The held-out subset result is promising but is sampled from the same dataset, raising concerns about representativeness and potential tuning bias.
- Baselines and fairness: Deep models (CNN/RNN/transformers) are standard for SA and for sarcasm detection; excluding them “since our approach is feature-based” is not convincing. A fair comparison would include modern text-only baselines trained on the same folds. The external NB/SVM/RB systems trained on other corpora (Table 1) are not comparable to the cross-validated models on D1/D2 and are confounded by domain mismatch.
- Methodological clarity and potential leakage: PCA is used to reduce unigram features to 500 dimensions, but it is unclear whether PCA is fit only on training folds (otherwise leakage). The significance testing with McNemar on cross-validation outputs is underspecified; paired tests across folds or approximate randomization would be more appropriate. MLP details (architecture, regularization, early stopping) are omitted. It is unclear whether the “Gz” ablation includes readability/length features mixed with gaze; thus the net contribution of true gaze vs. simple length/readability is not isolated.
- Reproducibility and definitions: Several features rely on “in-house” resources (e.g., discourse connectors) without release. The gaze-saliency graph features are ambiguously defined (e.g., F1H/F1S vs. F2H/F2S edge weighting; PSH/PSS/PSDH/PSDS vs. “source/destination”; RSH/RSS vs. RSDH/RSDS—definitions appear duplicated/contradictory). Preprocessing for tokenization/AoIs, NE/POS tools and versions, and exact hyperparameters are under-specified. Table 3 (feature ranking) is referenced but not shown.
- Presentation and formatting: Numerous typos and inconsistencies (e.g., “an our features”, “discuss on”, “usergenerated”, “mis-conception”). Section references are occasionally off; figures are referenced but not fully integrated. References contain many formatting errors (e.g., escaped control sequences, duplicated author/year strings). Some table entries are inconsistent (e.g., in Table 2, NB Sn+Gz for D2 shows F less than both P and R; identical numbers across rows suggest copy errors). Metrics (macro vs. micro F1) are not specified.

Questions to authors
- How is PCA fit in cross-validation? Was it refit on each training fold only? Were any other feature normalizations fit within folds?
- Does “Gz” include readability/length features? Please report ablations: (a) gaze-only, (b) readability-only, (c) gaze-minus-readability, to isolate the cognitive signal.
- How are significance tests conducted over CV? Are predictions concatenated across folds for McNemar? Please report paired tests over fold-wise scores.
- Please clarify and formalize the gaze-saliency graph weighting schemes for F1H/F2H/PSH/PSS/PSDH/PSDS/RSH/RSS/RSDH/RSDS. Provide precise equations.
- Can you provide a text-only deep baseline trained on the same splits? At minimum, a strong transformer-based classifier.
- At inference, how do you envision using the model without gaze? Could you train a text-only model to predict gaze features and use those predictions?

Additional comments
- Provide the in-house discourse connector list and any code to compute all features.
- Report per-class P/R/F1 (positive/negative) and specify whether F1 is macro/micro.
- Control for sentence length: show performance matched on length bins; report whether gaze adds beyond length/readability.
- Consider cross-dataset transfer (train on D1, test on D2) to assess robustness and the effect of tracker quality (500Hz vs 120Hz).
- Clean up language, section cross-references, tables, and references. The current level of typographical and formatting errors is high.

Overall, while the idea of leveraging gaze for handling nuanced sentiment is interesting and the held-out irony gains are encouraging, the paper in its current form suffers from practical limitations, incomplete baselines, methodological ambiguities, and presentation issues that preclude acceptance.

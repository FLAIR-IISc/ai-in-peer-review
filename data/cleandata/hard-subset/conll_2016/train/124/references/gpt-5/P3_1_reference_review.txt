Summary
The paper explores augmenting sentiment analysis (SA) with cognitive features derived from eye-tracking during reading. It engineers two families of gaze features—simple aggregates (fixations, saccades, regressions, skips) and graph-based “gaze-saliency” features—alongside standard SA features and features aimed at sarcasm/thwarting. Experiments on two public datasets containing gaze while annotators labeled sentiment show small but statistically significant gains when adding gaze features to text features, and larger relative gains on a sarcasm/irony subset. The paper also discusses feasibility via mobile eye-tracking.

Main contributions (as I see them)
1) Empirical finding: Aggregate gaze features provide incremental improvements for sentiment polarity classification, especially on sarcasm/irony cases.
2) Feature design: A structured set of gaze-derived features, including novel graph-based saliency measures (weighted degree variants), for downstream classification.
3) Evaluation on two gaze datasets with different hardware and participant pools, plus a targeted held-out subset, with significance tests and feature-importance analysis suggesting gaze features are among top-ranked predictors.

Strengths
- Addresses a recognized gap: Traditional lexical/syntactic features struggle with pragmatic phenomena (sarcasm, implicit sentiment). Using cognitive signals to bridge that gap is timely and relevant to CoNLL.
- Cross-dataset evidence and hardware variability: Improvements are shown on two datasets collected with different eye-trackers (Eyelink-1000 vs. Tobii T120), which helps argue robustness to device differences.
- Targeted analysis on complex phenomena: The held-out set with a substantial proportion of sarcasm/irony shows larger relative gains (+6.1% F), aligning with the hypothesis that gaze captures surprisal/incongruity.
- Feature significance analysis: The finding that many top-ranked features are gaze-based supports the claim that cognitive signals add information beyond text.
- Methodological clarity in feature engineering: The paper provides concrete definitions for a broad set of features (FDUR, REG, RSF, LREG, ED, degree-based graph features), which aids reproducibility of the feature extraction.

Weaknesses
- Practicality and deployability: At inference time, the method presupposes eye-tracking signals for the target text. Current SA applications typically require text-only inference; collecting gaze from multiple readers per item (5–7 in the datasets) is impractical. The feasibility discussion is speculative. The paper should demonstrate performance with single-reader gaze, and quantify performance as a function of readers per item.
- Modest gains over strong text-only features: On both datasets, the bulk of the improvement comes from adding sarcasm/thwarting features; adding gaze yields only +0.6 to +1.5 F-points (SVM). Gaze-only models perform poorly. The incremental benefit is statistically significant but small; it is unclear if gains would persist against stronger text-only baselines.
- Missing comparisons to contemporary SOTA text models: The paper explicitly excludes deep learning approaches (CNN/RNN, e.g., fastText, biLSTM, CNN) yet claims “improvement over the state of the art.” Without at least competitive text-only baselines (tuned linear models on n-grams/char-ngrams, embeddings, or minimal neural baselines), it is difficult to assess the true added value of gaze.
- Evaluation design concerns:
  - Cross-validation on small datasets with high-dimensional features risks overfitting; hyperparameters are mostly default and PCA is applied ad hoc. No nested CV or regularization tuning is reported.
  - The “difficulty” check trains baselines on different domains (Pang & Lee, Sentiment140) and tests on D1/D2, conflating domain shift with task difficulty.
  - Potential confounds: Gaze features correlate with reading difficulty; while length/readability are included, more rigorous controls (e.g., matched-length subsets, regression controls) are not presented.
  - Lack of participant-level analysis: Features are averaged across readers. There is no study of cross-participant generalization, variance across readers, or whether reader identity effects leak signal.
- Clarity and detail gaps:
  - Gaze-saliency graph definition is under-specified (e.g., “vertex per word may not be unique”; weighting schemes and normalization; treatment of multiple saccades between the same tokens).
  - The feature-importance table (top-20) is referenced but not fully shown here; more exact rankings and effect sizes would help interpretation.
  - No error breakdown by phenomenon beyond the held-out set; per-class metrics for sarcastic vs. non-sarcastic cases would sharpen the claims.

Questions to authors
- Inference setting: How many readers’ gaze are assumed at test time? Can you report performance with a single reader, and show performance as a function of readers per item?
- Cross-participant generalization: If you compute features per reader and average, can you instead split CV folds by readers (leave-one-reader-out) to rule out potential reader-specific effects or leakage?
- Stronger text-only baselines: How does Sn+Sr compare to tuned linear SVM/logistic regression on word and character n-grams, and to a simple embedding-based model (e.g., fastText)? Do gaze gains persist against those?
- Hyperparameter tuning: Did you explore C values for SVM, smoothing for NB, and network size/regularization for MLP? If so, please report.
- Gaze feature robustness: Are gains robust after controlling more strictly for length, word frequency, and readability (e.g., via partial correlation or mixed-effects regression)? Which specific gaze features drive the improvements on sarcasm?
- Saliency graph details: Please formalize the graph construction (unique tokens vs. word positions; edge multiplicity; normalization of degrees; how top-2 degrees are chosen) and justify the choice of using only top-2 degrees.

Overall assessment
The idea of leveraging gaze for SA, particularly for pragmatic phenomena, is interesting and has empirical support here. However, the incremental gains are small, the practicality is uncertain, and the evaluation lacks comparisons to strong text-only baselines and more rigorous controls. I encourage a revision that (a) strengthens text-only baselines, (b) analyzes single-reader and cross-reader generalization, (c) tightens confound controls, and (d) clarifies graph feature definitions. These steps would significantly improve the paper’s impact and credibility.

Summary and main contributions
- The paper proposes augmenting standard textual features for sentiment analysis with cognitive features derived from readers’ eye movements. It introduces both basic gaze features (fixations, saccades, regressions, skips) and a novel gaze-saliency graph representation from which several graph-theoretic gaze features are computed.
- It provides empirical evidence that these cognitive features improve sentiment classification, particularly on texts with complex phenomena such as sarcasm, irony, and thwarted expectations. Improvements are shown across two eye-tracking datasets, multiple classifiers, and via a held-out evaluation focused on irony.
- It offers an analysis of feature importance indicating that several gaze-based features rank among the most informative, and a feasibility discussion on collecting gaze signals in practical settings.

Strengths
- Novel and bold direction: Using eye-tracking as input features for sentiment polarity classification is a fresh, conceptually interesting step that connects psycholinguistics with applied NLP. The gaze-saliency graph is a genuinely novel representation that could spur further work in cognitively informed NLP beyond SA (e.g., QA, summarization).
- Consistent empirical gains, especially on hard cases: Across two datasets, adding gaze to sentiment+sarcasm features yields additional gains (e.g., +0.6% and +1.5% F1), with significance testing provided. On a held-out set enriched for irony, gaze brings a larger relative improvement (+6.1% F1), aligning with the hypothesis that cognitive signals help where text-only cues are weak.
- Sound psycholinguistic grounding: The choice of gaze features is motivated by established findings (e.g., fixations for lexical difficulty, regressions for reanalysis/surprisal). The RSF/LREG-type features that explicitly model long-distance regressions are well motivated for sarcasm/thwarting, and the results support their utility.
- Careful feature engineering and analysis: The paper considers a broad, well-documented textual baseline (Sn), augments it with specialized sarcasm/thwarting features (Sr), and then adds gaze (Gz). Chi-squared feature ranking shows many gaze features among the top predictors, strengthening the case that the gains are attributable to cognitive signals and not just more text features.
- Clear articulation of feasibility and broader vision: The discussion of mobile eye tracking and application scenarios shows awareness of deployment hurdles, yet makes a plausible case for the long-term relevance of such signals, which I value in forward-looking work.

Weaknesses
- Practicality at inference time: The approach requires gaze data to classify new texts, which is currently impractical for most SA applications. While the feasibility section is thoughtful, it remains speculative. The paper would benefit from a clearer statement of intended deployment settings (e.g., in-situ review reading) and an evaluation that mimics those conditions.
- Modest aggregate gains over strong text features: On full datasets, the incremental benefit of Gz over Sn+Sr is relatively small (0.6–1.5% F1), with larger gains confined to irony-heavy subsets. This is not a deal-breaker given the novelty, but it does raise questions about the cost-benefit tradeoff of requiring gaze. More detailed breakdowns (e.g., by phenomenon types) would help quantify where gaze pays off most.
- Evaluation design gaps and baselines: The paper does not compare against stronger modern text-only baselines (e.g., CNN/RNN/recursive models noted by the authors, or even stronger linear models with character n-grams). Since Sn+Sr already delivers substantial gains (especially on D2, +7.8% F1), it is important to establish how much headroom remains for text-only models before crediting gaze as necessary.
- Cross-subject and cross-device robustness: The two datasets use different eye-trackers and sampling rates (500 Hz vs 120 Hz). It is unclear how gaze features were normalized across hardware, and how robust the model is to user variability. Averaging features over readers may wash out individual signal and hides generalization issues; leave-one-subject-out and cross-device evaluations would strengthen claims.
- Clarity and reproducibility: Some gaze-saliency graph feature definitions are ambiguous or duplicated (e.g., distinctions between source/destination degree definitions and whether weights are counts vs distances). Hyperparameters are mostly defaults; dimensionality reduction to 500 PCA components on small data risks over/underfitting tradeoffs. The paper would benefit from more precise definitions, ablations (basic vs graph gaze features), and released code.

Questions to authors
- What is the intended inference-time scenario? Do you envision classifying sentiment while a user reads a review (i.e., per-user, per-reading), or reusing aggregated gaze per item? How would the system behave without fresh gaze at test time?
- How were gaze features normalized across devices and readers (e.g., time-based features adjusted for sampling rate, individual reading speed, sentence length, and word length)? Any per-subject z-scoring?
- Did you evaluate cross-subject generalization (e.g., leave-one-subject-out CV) to ensure robustness to new readers?
- Can you provide an ablation between basic gaze features and gaze-saliency graph features, especially on the irony subset, to identify which contribute most?
- Can you include a stronger text-only baseline (e.g., char n-grams with linear SVM; or a CNN/RNN baseline as per prior work) to better contextualize the incremental value of gaze?
- How are the sarcasm/thwarting features trained (e.g., for IMPLICIT_PCA via bootstrapping)? Is there any possibility of leakage from test folds during feature construction?

Overall, I find the core idea and the gaze-saliency graph particularly novel and promising. Despite practical constraints and some evaluation gaps, the paper opens an exciting direction and provides evidence that cognitive signals can help crack linguistically nuanced sentiment—exactly the kind of bold step I value.

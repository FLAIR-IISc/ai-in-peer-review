- Contributions: shows gaze-derived features add incremental signal for sentiment analysis, especially for irony/sarcasm.  
- Contributions: provides a feature inventory of basic and graph-based eye-movement features with importance analysis.  
- Contributions: discusses feasibility of mobile eye-tracking for deployment (speculative).  

- Strength: consistent empirical improvements from adding gaze features across two datasets, larger gains on an irony-heavy subset.  
- Strength: features are psycholinguistically motivated and interpretable; many gaze features rank highly in importance.  
- Strength: cross-dataset consistency despite different eye-tracking hardware.  
- Strength: focused evaluation on hard pragmatic phenomena (sarcasm/thwarting).  
- Strength: uses statistical tests (McNemar, chi-squared) to assess significance.  

- Weakness: method requires eye-tracking at inference; practicality and deployment feasibility are unresolved and speculative.  
- Weakness: novelty is overstated given prior gaze-for-SA work (e.g., Mishra et al. 2014); paper is more an extension than a first demonstration.  
- Weakness: flawed baselines and evaluation — comparisons to external models are apples-to-oranges and in-domain baselines are weak (default hyperparams, PCA unigrams, no strong lexical/embedding or contextual baselines).  
- Weakness: no hyperparameter tuning or nested CV; text-only baselines may be understated.  
- Weakness: “Gz” mixes gaze with readability features (RED, LEN), so the true contribution of gaze is unclear; more granular ablations needed (gaze-only, readability-only, basic vs graph).  
- Weakness: potential bias from data collection — gaze captured during annotation may reflect annotator deliberation/uncertainty, not intrinsic text properties; no analysis by annotator agreement or generalization across readers.  
- Weakness: reproducibility/clarity issues — some graph feature definitions are unclear, code not provided, and there are typos/organizational problems.  

- Q1: Provide ablations isolating gaze-only vs readability-only and basic vs graph-based gaze features.  
- Q2: How performance varies with number of readers per item; report results for 1,2,3,… readers.  
- Q3: Does gaze benefit persist after controlling for annotator agreement; are gains concentrated on high-disagreement items?  
- Q4: How robust are features across devices; do normalized (per-participant/device) features preserve trends?  
- Q5: For sarcasm/thwarting features, what unlabeled data and seeds were used and is there any overlap with evaluation sets?  
- Q6: Compare against stronger text-only baselines (TF-IDF+SVM with tuned C, fastText, averaged embeddings) and report calibrated improvements.  

- Additional: Correct the novelty claim and reframe as a broader/stronger evaluation over prior work.  
- Additional: Clarify LP and IMPLICIT_PCA training data to avoid leakage and provide implementation details.  
- Additional: Release code for feature extraction to improve reproducibility.  
- Additional: Discuss ethical/privacy implications of large-scale gaze collection (storage, linkage, consent).

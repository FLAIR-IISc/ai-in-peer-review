Summary and main contributions
This paper augments standard text-based sentiment analysis (SA) features with cognitive features derived from readers’ eye-tracking during sentiment annotation. It defines a suite of basic and graph-based gaze features, plus features targeted at sarcasm/irony/thwarting, and evaluates on two public eye-tracking datasets with rich pragmatic phenomena. The authors report modest but consistent gains from adding gaze features, with larger gains on a held-out subset containing irony/sarcasm.

As I see it, the primary contributions are:
1) Empirical evidence that gaze-derived features provide incremental predictive signal for SA, especially on texts containing irony/sarcasm.
2) A feature inventory for leveraging eye-movement statistics and saliency graphs in SA, with feature importance analyses showing several gaze features among the top-ranked.
3) A problem framing and feasibility discussion around using mobile eye-tracking, albeit speculative.

Strengths
- Clear empirical finding: consistent improvements from gaze features. Across both datasets, adding gaze on top of text and sarcasm/thwarting features yields gains (SVM: +0.6 and +1.5 F1). On a sarcasm-heavy held-out set, the relative gain for complex texts is larger (+4.2 F1 over Sn+Sr; +6.1% relative). This supports the paper’s central claim that cognitive signals help on pragmatically challenging inputs.
- Interpretable, psycholinguistically motivated features. The basic (FDUR, FC, SL, REG, SKIP, RSF, LREG) and graph-based (ED, weighted degrees) features are grounded in prior eye-movement literature and plausibly relate to syntactic reanalysis, surprisal, and processing difficulty. The feature-importance analysis showing many gaze features among the top 20 further supports their utility.
- Cross-dataset consistency despite hardware differences. Results hold on two datasets collected with different trackers (Eyelink-1000, Tobii T120), suggesting the signal is not idiosyncratic to a single setup.
- Focus on hard phenomena. Incorporating sarcasm/thwarting features and analyzing performance on a difficult (irony) subset is appropriate and informative for the stated motivation.
- Statistical testing. The use of McNemar tests for paired comparisons and chi-squared feature ranking, while not exhaustive, indicates attention to significance.

Weaknesses
- Practicality at inference time is unresolved. The method requires eye-tracking for the test instances. The feasibility section is largely speculative (mobile eye-tracking, consent), without a concrete deployment protocol or empirical evidence that crowd/mobile gaze would be accurate enough. Without eye-tracking at inference, the benefits cannot be realized; this limits impact for standard SA settings.
- Overstated novelty relative to prior eye-tracking for SA. Mishra et al. (2014) already study “sentiment detection” and subjectivity extraction using gaze. This paper’s claim of being “first of its kind” is inaccurate; the contribution is better framed as a broader feature set and a more extensive evaluation rather than a first demonstration.
- Baseline and evaluation gaps. 
  • The “existing SA systems” comparison (Sec. 3.3) is apples-to-oranges: those models are trained on external corpora and tested on these datasets, while the proposed models are trained/tested in-domain via cross-validation. This cannot support state-of-the-art claims.
  • Within-dataset baselines are not state-of-the-art for text-only SA: default Weka hyperparameters, PCA-reduced unigrams, no strong lexical/embedding baselines (e.g., TF-IDF+linear SVM with tuned C, fastText, averaged pretrained embeddings), and no contextual models. Even if deep models are out of scope, stronger classical baselines are expected.
  • No hyperparameter tuning or nested CV; default settings may understate text-only baselines.
- Ablations and confounds in the gaze block. In Table 2, “Gz” mixes gaze and reading-difficulty features (RED, LEN), so the specific contribution of gaze is unclear. A clean ablation (Sn+Sr+Gaze-only vs Sn+Sr+Readability-only) is needed. More granular ablations (basic vs graph features; each subgroup) would clarify what drives the gains. Given the modest overall improvements (0.6–1.5 F1), even small confounds matter.
- Potential biases from the data collection protocol. Gaze was collected while annotators deliberated sentiment labels. The features may capture annotation difficulty or label uncertainty rather than properties intrinsic to the text. There is no analysis of performance as a function of annotator agreement, nor of whether the model generalizes across annotators (e.g., train on gaze from subset of readers, test on held-out readers) or with a single reader. Averaging across readers may mask variability; deployment would likely have far fewer readers per item.
- Reproducibility and clarity issues. Some graph feature definitions are hard to follow (e.g., F1H/F2H vs source/target weights), and code is not provided. Several typos and organizational glitches (section numbering, table labels) hinder clarity.

Questions to authors
1) Can you provide an ablation that isolates gaze-only vs readability-only features within Gz, and basic vs graph-based gaze features? How do gains change?
2) How many readers per item are needed at test time? Please report performance as a function of the number of readers (e.g., 1, 2, 3, …, average across k random readers).
3) Does the benefit from gaze persist after controlling for annotator agreement (e.g., stratify by Fleiss’ κ or item-level agreement)? Are gains larger only on high-disagreement items?
4) How robust are the results across devices? If you normalize features (e.g., z-score per participant/device), do trends hold? Any cross-device analysis you can report?
5) For the sarcasm/thwarting features (e.g., implicit incongruity via bootstrapping), what unlabeled data and seeds were used? Any risk of overlap with evaluation sets?
6) Could you compare against a stronger text-only baseline (e.g., TF-IDF+linear SVM with tuned C; fastText; averaged GloVe) and report calibrated improvements?

Additional comments
- Please correct the novelty claim; position the work as an extension and more comprehensive analysis over prior gaze-for-SA studies.
- Clarify LP and IMPLICIT_PCA training data to avoid leakage. Provide implementation details and release code for feature extraction.
- Consider ethical/privacy implications of large-scale gaze collection beyond consent (storage, linkage to identity).

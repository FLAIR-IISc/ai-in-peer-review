- Paper studies adding eye-tracking–derived cognitive features to sentiment analysis, especially for irony/sarcasm, using two public datasets.
- Contributions: empirical evidence that gaze features improve SA on complex pragmatic cases, a set of gaze-derived features including a saliency graph, and a comprehensive textual feature baseline.
- Empirical result: gaze features yield larger F1 gains on irony/sarcasm instances (e.g., +4.2 absolute on held-out subset) than on simpler texts.
- Novel, interpretable gaze features (including graph-based saliency) correlate with psycholinguistic constructs and rank highly in feature-importance analyses.
- Competitive performance on a sarcasm-rich dataset (Dataset 1) where the best gaze-augmented SVM outperforms external SVM/NB baselines.
- Good experimental rigor: 10-fold CV, ablations across feature groups, and McNemar significance tests.
- Cross-disciplinary value: integrates cognitive evidence with SA and links features to psycholinguistic literature.
- Major practicality concern: approach requires gaze at test time, limiting real-world deployment; no experiments using gaze only during training or predicting gaze proxies.
- Missing comparisons to modern deep-textual baselines (CNN/RNN/transformers), so state-of-the-art claims are overstated.
- On Dataset 2, the gaze-augmented model underperforms an external SVM baseline trained on large corpora, raising generality concerns.
- Some feature definitions and notations (various graph features) are ambiguous or duplicated and need clarification.
- Missing methodological details: gold-label aggregation, tokenization/AoI mapping, held-out subset training/tuning procedures are unclear.
- Potential confounds: eye-tracking may reflect general reading difficulty (length, freq, syntax); more controls (frequency, surprisal, syntax depth) are needed to isolate sentiment-specific signals.
- Possible domain leakage concerns from external LP features; more transparency required.
- Gaze adds only modest average gains over strong textual features (+0.6 to +1.5 F1), with larger but isolated improvements on irony/sarcasm.
- Requests for clarification: how gold labels were aggregated and ties handled, and whether low-agreement items were filtered.
- Requests for held-out subset protocol details: exact training data, tuning procedures, and whether hyperparameters used that set.
- Questions about gaze aggregation and generalization across readers: are features averaged across readers and was leave-one-subject-out CV tried?
- Ask authors to formally define saliency graph features (F1H/F2H vs F1S/F2S; PSDH/PSDS; RSDH/RSDS) and commit to code release.
- Request experiments comparing to strong deep-textual baselines (e.g., BERT/RoBERTa) to contextualize gaze benefits.
- Suggestion: explore using gaze only during training (multi-view, distillation) to improve practicality.
- Editorial issues: paper contains typos and formatting glitches that impede readability and reproducibility.
- Recommendation to qualify “state-of-the-art” claims given missing deep baselines and mixed dataset performance.



**Review: Leveraging Cognitive Features for Sentiment Analysis**

This paper proposes augmenting traditional sentiment analysis features with cognitive features derived from eye-movement patterns, claiming the first use of such features in NLP for sentiment analysis and sarcasm detection. The authors report marginal improvements (3.7% and 9.3% F-score gains on two datasets) when adding cognitive features to existing sentiment and sarcasm features.

The main contributions of this work are: (1) Novel integration of cognitive features from eye-tracking into sentiment analysis, (2) Demonstration that these features particularly benefit complex constructs like sarcasm/irony (6.1% improvement on held-out dataset), and (3) A well-structured experimental evaluation framework.

Strengths: The paper is exceptionally well-written and easy to follow, which is rare for interdisciplinary work. The idea of leveraging cognitive features is genuinely interesting and could inspire work on other NLP tasks requiring nuanced understanding. The feature significance analysis provides compelling evidence that cognitive features specifically help with complex sentiment constructs.

Weaknesses: The motivation for why cognitive features should correlate with sentiment polarity is weak - the authors merely state "eye-movement patterns reflect cognitive processes" without providing theoretical grounding beyond "reading ease." The marginal performance gains (only 0.6-1.5% improvement over Sn+Sr baseline) raise serious questions about practical value. Most critically, the feasibility discussion (Section 7.2) is unconvincing - the mobile eye-tracking application example fails to address major challenges like user consent, data privacy, cost, or how these features would actually be integrated into real-world systems.

The paper makes an interesting conceptual contribution but falls short on demonstrating practical utility. The marginal gains combined with weak motivation and unconvincing feasibility arguments significantly undermine the paper's impact. While the research direction shows promise, this specific implementation does not sufficiently justify its novelty or practical value for the NLP community.

Questions to authors: How would you address the 0.6-1.5% performance gain being marginal compared to the complexity of collecting and processing eye-tracking data? What specific cognitive mechanisms do you propose link eye-movement patterns to sentiment polarity, beyond general "reading difficulty"?

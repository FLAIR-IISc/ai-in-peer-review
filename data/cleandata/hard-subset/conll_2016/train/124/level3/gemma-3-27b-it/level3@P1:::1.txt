This paper proposes augmenting traditional sentiment analysis features with cognitive features derived from readers’ eye-movement patterns. The authors hypothesize that these patterns reveal insights into cognitive processes related to sentiment understanding, potentially improving performance on complex constructs like sarcasm and irony. The core contribution is the exploration of eye-movement data as a novel feature source for sentiment classification, showing modest improvements over baseline systems. A secondary contribution lies in the comprehensive dataset used for evaluation, combining existing datasets focused on sentiment and sarcasm with eye-tracking data. Finally, the paper offers a preliminary discussion of the feasibility of integrating this approach into real-world applications.

**Strengths:**

1. **Novelty of Approach:** The exploration of cognitive features, specifically derived from eye-tracking data, for sentiment analysis represents a genuinely novel approach within the field. While psycholinguistic studies have used eye-tracking extensively, its integration into mainstream NLP tasks like sentiment analysis is a valuable contribution. This opens up a new avenue for incorporating cognitive science insights into NLP models.

2. **Comprehensive Feature Engineering:** The authors demonstrate thoroughness in their feature engineering, combining traditional sentiment features (lexical, syntactic, semantic) with features designed to capture sarcasm and, crucially, the newly introduced cognitive features. The detailed breakdown of each feature category (Section 4) is well-organized and facilitates understanding of the model’s components.  The inclusion of graph-based features derived from eye-tracking data adds another layer of sophistication.

3. **Rigorous Experimental Evaluation:** The paper presents a well-defined experimental setup using two publicly available datasets (Datasets 1 and 2), incorporating a 10-fold cross-validation approach.  The comparison against baseline systems employing only traditional features provides a clear benchmark for assessing the improvement achieved through the inclusion of cognitive features.  The additional evaluation on a held-out dataset specifically targeting complex constructs further strengthens the analysis. The statistical significance testing using McNemar test is also a good practice.

**Weaknesses:**

1. **Modest Performance Gains:** While statistically significant, the improvement in F-score achieved by incorporating cognitive features (a maximum of 9.3% on Dataset 2) is relatively modest. The gain from adding cognitive features on top of sentiment and sarcasm features is not substantial. This raises questions about the practical impact of this approach, especially considering the added complexity and data requirements of collecting eye-tracking data.

2. **Limited Justification for Cognitive Features:** The rationale behind *why* specific eye-movement patterns should correlate with sentiment understanding is not fully developed. The paper asserts a connection between cognitive processes and eye-movements, but lacks a deeper theoretical grounding for selecting specific gaze-based features. While the paper does link certain features to established psycholinguistic theories, the connection is not always explicit or strongly supported.  For example, the link between reading difficulty and sentiment is not fully explored.

3. **Feasibility Concerns & Unconvincing Application Scenario:** The discussion regarding the feasibility of the approach (Section 7) feels speculative and doesn’t adequately address the practical challenges of deploying an eye-tracking based sentiment analysis system. The proposed application scenario involving mobile eye-trackers (Section 7.2) lacks a compelling demonstration of how this would outperform existing methods, or what specific benefit justifies the user privacy and technical hurdles involved. The argument hinges on an unspecified need to handle linguistic nuances, but doesn't convincingly show current methods failing in this regard.

**Questions to Authors:**

1. Could you elaborate on the theoretical basis for selecting the specific eye-movement features included in your model?  Specifically, how do these features relate to established cognitive models of sentiment processing?
2. What is the computational cost of extracting and incorporating these cognitive features? How does this compare to the computational cost of more traditional sentiment analysis approaches?
3. The improvement is more pronounced on Dataset 2. Can you speculate on why the cognitive features might be more effective for this particular dataset? What are the key differences between the datasets that might explain this?
4.  Considering the marginal gains, have you explored dimensionality reduction techniques or feature selection methods to identify the most impactful cognitive features, potentially simplifying the model and reducing computational overhead?



**Additional Comments:**

*   Table 3, showing feature significance, would be more impactful if it highlighted the top *cognitive* features rather than simply listing the top 20 overall.
*   Consider discussing potential limitations related to individual differences in eye-movement patterns. How might factors like reading speed or visual acuity affect the performance of the model?
* The references are extensive and well organized.

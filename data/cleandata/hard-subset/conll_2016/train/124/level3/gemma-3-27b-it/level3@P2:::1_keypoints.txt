- Core idea: Augment sentiment analysis with cognitive features from eye-tracking to capture readers’ processing of nuanced language (sarcasm/irony).
- Primary contribution: Exploration of eye-tracking features in sentiment pipelines and modest empirical gains (up to 9.3% F-score) on two datasets.
- Secondary contribution: Curated datasets containing sarcasm/thwarted expectations as a challenging testbed for sentiment models.

Strengths:
- Novelty: Applying eye-tracking-derived cognitive features to directly improve sentiment classification is a fresh and interesting direction.
- Empirical validation: Experiments on two datasets show statistically significant improvements (McNemar test).
- Comprehensive features: Combination of traditional sentiment, sarcasm/thwarting, and detailed cognitive (gaze) features enables thorough investigation.
- Challenging datasets: Use of sarcastic/ironic texts provides a realistic, hard test for sentiment models.

Weaknesses:
- Limited gains: Performance improvements are modest relative to the complexity and cost of eye-tracking data collection.
- Weak theoretical motivation: Paper does not convincingly explain which specific cognitive processes each gaze feature captures.
- Feasibility concerns: Deployment arguments (e.g., mobile eye-trackers in e-commerce) are speculative and overlook privacy, cost, and intrusiveness issues.
- Feature granularity risk: Complex gaze-saliency graph features may be unnecessarily intricate and prone to overfitting without clear payoff.

Questions for authors:
- Please clarify which cognitive processes each eye-tracking feature is intended to capture (e.g., fixation duration meaning).
- Did you correlate individual cognitive features with specific sentiment types (sarcasm vs. positive/negative)?
- Have you considered reducing complexity (feature subset selection or dimensionality reduction) given modest gains?
- What ethical/privacy safeguards and consent mechanisms do you propose for collecting and using eye-tracking data?

Additional comments:
- Table 3 needs a brief explanation of the chi-squared test and ranking criteria for features.
- Reference list appears incomplete; some in-text citations are missing from references.
- Discuss dataset limitations (sample size, domain specificity) more explicitly.

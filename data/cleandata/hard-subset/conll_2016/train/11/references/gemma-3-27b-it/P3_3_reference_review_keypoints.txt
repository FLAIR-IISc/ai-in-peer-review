- Focused coreference task: identifying mentions of a Wikipedia article’s main concept (MC) within the article via binary classification.  
- Testbed (WCR): creation of a focused, annotated dataset for MC coreference in Wikipedia.  
- Wikipedia-specific adaptation: showing that tailoring CR to Wikipedia’s structure outperforms generic CR systems.  
- Feature engineering: effective use of Wikipedia markup, links, Freebase, and existing CR outputs.  
- Empirical gains: proposed method outperforms state-of-the-art baselines and improves a rule-based system (Dcoref++).  
- Clear motivation: well-defined problem and rationale for focusing on the main concept.  
- Thorough evaluation: comprehensive experiments, cross-validation, multiple metrics (P/R/F1).  
- Ablation study: feature families justified and their contributions demonstrated (Table 2).  
- Practical impact: classifier improves performance when integrated into an existing CR pipeline.  
- Resource release: authors commit to releasing the dataset and extracted data.  
- Small dataset: WCR’s limited size (30 documents) raises generalizability concerns.  
- Mention-detection dependency: approach relies on Dcoref/Scoref mention detectors, affecting robustness.  
- Limited error analysis: need more systematic analysis by concept type and error categories.  
- Incomplete baselines: missing simpler or broader baselines (e.g., frequency-based) for comparison.  
- No significance testing: reported gains lack statistical significance analysis given small sample size.  
- Dataset expansion: request for plans to grow and release more annotated WCR data.  
- Mention detection sensitivity: ask whether different mention detectors were evaluated or how sensitive the approach is.  
- Detailed error categorization: request for errors categorized by type (pronoun, entity type, etc.).  
- Statistical significance: request for tests to validate performance improvements.  
- Generalizability across languages: question about evaluation on non-English Wikipedia and language-specific challenges.  
- Writing quality: generally clear and concise with a well-structured related work section.  
- Limitations: paper should include a brief discussion of study limitations.  
- Recommendation: accept with minor revisions contingent on addressing dataset size, error analysis, and significance testing.

- Task: Identify mentions of a Wikipedia article’s main concept (MC) using a binary classification approach with Wikipedia and Freebase features, and introduce a new evaluation dataset (WCR).
- Main contribution 1: Release of WCR, a dedicated testbed for MC resolution in Wikipedia.
- Main contribution 2: Effective feature engineering leveraging Wikipedia markup, links, and Freebase to detect MC mentions.
- Main contribution 3: Empirical demonstration that targeted, Wikipedia-aware methods outperform generic coreference systems and improve a rule-based system when integrated.
- Strength: Clear problem definition and strong motivation for adapting CR to Wikipedia’s domain.
- Strength: Well-designed, comprehensive feature set including separate handling for pronominal vs. non-pronominal mentions.
- Strength: Convincing empirical results with consistent gains over baselines and informative ablation study.
- Strength: Practical impact shown by improvements when integrated into an existing rule-based system (Dcoref++).
- Strength: Commitment to releasing the dataset and a dump of identified MC mentions for reproducibility.
- Weakness: Very small dataset (30 documents), raising concerns about result generalizability and dataset variability.
- Weakness: Baseline choice and analysis insufficiently justified; need deeper exploration why Dcoref outperforms Scoref on WCR.
- Weakness: Error analysis is superficial; lacks systematic, quantitative categorization of error types.
- Weakness: Limited discussion of classifier explainability—missing analysis of most influential features or feature weights.
- Weakness: Scalability and computational cost concerns not addressed despite external resources (WikipediaMiner, Jena) being used.
- Questions for authors: Plans for expanding WCR or using larger Wikipedia corpora for more robust evaluation?
- Questions for authors: Detailed analysis of why Dcoref outperforms Scoref and baseline error characteristics?
- Questions for authors: Request for quantitative error breakdown by error type (pronominal, ambiguity, long-distance, etc.)?
- Questions for authors: Which features drive performance for pronominal vs. non-pronominal mentions, with illustrative examples?
- Questions for authors: What is the computational complexity and how does the system scale to full English Wikipedia; what optimizations are possible?
- Additional comments: Writing is clear; related work could better cover knowledge-enhanced CR; dataset release is commendable.
- Overall assessment: Solid, valuable contribution that merits acceptance with minor revisions to address dataset size, error analysis, feature explainability, and scalability.

- Summary: Proposes binary classification for identifying "main concept" mentions in Wikipedia, using Wikipedia/Freebase features and CR outputs, and releases a new WCR dataset.  
- Contribution — WCR dataset: Provides and releases a dedicated dataset for evaluating main concept resolution in Wikipedia.  
- Contribution — Domain specificity: Shows that generic coreference systems underperform on Wikipedia, highlighting need for domain adaptation.  
- Contribution — Feature engineering: Combines Wikipedia markup, Freebase, and CR features to achieve strong empirical gains.  
- Strength: Clear problem definition and strong motivation for adapting coreference to Wikipedia.  
- Strength: Strong empirical results showing significant improvements over baselines.  
- Strength: Comprehensive and well-considered feature set with useful feature importance analysis.  
- Strength: Practical integration into an existing rule-based CR system (Dcoref++) demonstrating real-world applicability.  
- Strength: Commitment to dataset and resource release to foster further research.  
- Weakness: Small WCR dataset (30 documents) limits generalizability and robustness of findings.  
- Weakness: Heavy reliance on Wikipedia/Freebase raises concerns about applicability to less-structured domains.  
- Weakness: Dense and hard-to-follow feature extraction description; feature interactions need clearer explanation.  
- Weakness: Baselines are somewhat simplistic; stronger baselines (e.g., fine-tuned pretrained LMs) are missing.  
- Weakness: Insufficient error analysis; more systematic analysis of failure modes is needed.  
- Question: Do you plan to expand WCR, and how will you ensure article diversity and representativeness?  
- Question: How would the approach generalize to domains without Wikipedia/Freebase, and can it be adapted?  
- Question: Can you clarify how feature families interact and which combinations are most effective?  
- Question: Please provide a more detailed error analysis identifying systematic error patterns.  
- Question: What is the computational cost and scalability of the feature extraction pipeline for large-scale use?  
- Additional comment: Include a clear system architecture diagram showing preprocessing, feature extraction, and classification stages.  
- Additional comment: Update citations to include more recent work on coreference and knowledge integration.  
- Additional comment: Simplify some complex sentences for clearer writing.  
- Overall assessment: Solid, well-motivated paper with valuable dataset release; recommend acceptance with minor revisions.

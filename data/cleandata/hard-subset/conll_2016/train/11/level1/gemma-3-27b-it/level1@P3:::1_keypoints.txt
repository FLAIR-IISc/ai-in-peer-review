- Created the WCR dataset (30 Wikipedia articles) as a focused benchmark for main concept resolution.
- Framed main concept resolution as a binary classification task leveraging Wikipedia markup and Freebase.
- Classifier using Wikipedia-specific and Freebase features substantially outperforms baselines.
- Feature importance analysis (Table 2) shows the designed features meaningfully contribute to performance.
- Integrated the classifier into Dcoref (Dcoref++) and improved end-to-end coreference resolution.
- Clearly motivated the problem and the need to adapt coreference systems to Wikipedia’s characteristics.
- Reported strong, statistically significant empirical gains over strong and adapted baselines.
- Conducted a useful ablation study that identifies the most impactful feature families.
- Demonstrated practical applicability by integrating into a full pipeline (Dcoref++).
- Constructed careful, task-appropriate baselines rather than using out-of-the-box systems.
- WCR’s small size (30 docs) limits generalizability and raises concerns about cross-validation reliability.
- Paper omits critical dataset details (annotation guidelines, inter-annotator agreement, selection criteria).
- Feature descriptions are high-level; specific features (e.g., exact semantic relatedness method) are under-specified.
- Error analysis is superficial and lacks a systematic categorization of common failure modes.
- Heavy reliance on Freebase raises concerns about sensitivity to missing/inconsistent knowledge and resource availability.
- Request authors to provide full WCR dataset details: guidelines, IAA, topic distribution, lengths, and selection rationale.
- Request a comprehensive, itemized list of specific features used within each feature family.
- Request a systematic error analysis with categorized error types and potential fixes.
- Ask for experiments or analysis on sensitivity to Freebase quality and alternative KBs.
- Ask for information on computational cost and scalability of feature extraction and Freebase lookups.
- Replace placeholder data availability link ("somewhere.country") with a concrete URL or repository.
- Suggest discussing related work in knowledge graph completion and entity linking as relevant connections.
- Recommend adding an explicit limitations section and outlining avenues for future work.

Summary and main contributions
- Empirical evidence that static distributional word embeddings trained on raw text encode information predictive of coarse PoS classes at the type level, with a simple multinomial logistic regression achieving high F-scores on frequent lemmas.
- Qualitative error analysis showing that misclassified items surface borderline categories (e.g., participles, nominal modifiers), annotation inconsistencies (e.g., numerals vs nouns for years/currency amounts), and potential systematic tagging errors (e.g., BNC adjectives tagged as adverbs), suggesting embeddings can aid corpus diagnostics.
- Exploratory analysis indicating that PoS information is not localized in a few embedding dimensions but distributed across many components, with some idiosyncratic dimensions for certain classes.

Strengths
- Interesting qualitative insights: The paper collects concrete misclassification clusters (proper noun vs common noun; adjective–noun alternations; participles; numerals/years) that align with known gray areas in PoS categorization and reveal plausible annotation issues in BNC, and to a lesser extent in UD. These observations are potentially useful for corpus maintenance and tagset discussions.
- Simple, transparent methodology: Using skip-gram embeddings plus logistic regression is easy to reproduce and interpret. The weighted training to address open/closed class imbalance is reasonable in spirit (though see weaknesses on evaluation protocol).
- Cross-corpus probing: Applying the approach to UD English (even if with caveats) suggests the signals are not purely corpus-specific. The observation that proper nouns are distributionally separable from common nouns, despite syntactic similarity, is notable.
- Dimension-wise analysis: Ranking features by ANOVA F-values and the k-best curve supports the claim that PoS information is distributed, countering the common misconception that a small number of “PoS dimensions” suffice.
- Potential utility for corpus diagnostics and low-resource bootstrapping: The idea of using embedding-based predictors to flag suspicious tags and to seed lexicons for low-resource scenarios is promising.

Weaknesses
- Novelty is limited: Prior work has shown that embeddings encode PoS/morphological information, that PoS clusters emerge, and that embedding dimensions correlate with linguistic properties (e.g., Tsuboi 2014; Tsvetkov et al. 2015; Liu et al. 2016; many others). The main methodological elements (word2vec; logistic regression; clustering/PoS probing) are standard. The principal new aspect is qualitative error analysis on BNC, which, while interesting, seems incremental.
- Evaluation design and leakage risks: The primary classifier is trained on vectors for lemma_POS tokens, i.e., the gold PoS is embedded in the token identity used to construct the vectors. While the authors acknowledge this is “artificial,” it undermines the reported cross-validation F=0.979. More critically, the UD evaluation with F=0.99 is ill-specified: how can you obtain vectors for UD lemmas from a model trained on lemma_POS tokens without using the gold tag to select the vector? If gold tags are used to fetch the vector, the evaluation is circular. This section needs precise clarification and, as written, the 0.99 figure is not credible.
- Cross-validation protocol appears flawed: The training set duplicates the same word vector proportional to frequency (177,343 instances from 10,000 types). If folds are formed over instances, identical vectors for the same word type likely appear in both train and validation folds, inflating scores. PoS probing should split by type, not by instance. Similarly, the k-best feature curve is reported on training accuracy, not a held-out set, so conclusions about the number of relevant dimensions lack proper validation.
- Missing key baselines and ablations: For the contextless “full-fledged tagger” (lemma-to-PoS), there is no comparison to a trivial lemma-majority baseline learned on the same training set, to simple lexical resources, or to a frequency heuristic. Without these, 0.84 accuracy is uninterpretable; it may be matched or exceeded by a majority-lemma lexicon. Also absent: comparisons across embedding hyperparameters (window, subsampling), models (CBOW, GloVe), or the effect of lemmatization.
- Questionable comparison to the Stanford tagger: Running a supervised contextual tagger on isolated tokens (no sentence context) is not a fair comparison to a model that leverages large corpus-wide distributional context. The resulting confusion matrices are thus not comparable and the conclusions drawn from this comparison are weak.
- Reporting, formatting, and reproducibility issues: Several figures/tables are missing or partial (Figures 1–3; Tables 1–4 are incomplete in the text), references are badly formatted and sometimes misattributed, footnotes are broken, there are many typos (“concatination,” “is is,” “fullfledged”), and key implementation details are underspecified (learning rate/schedule, preprocessing for numerals/casing, whether OOV handling or code/mappings will be released). These issues hinder verification.

Questions to authors
- How exactly was the 10-fold cross-validation performed for the lemma_POS classifier? Were folds formed by word type (ensuring no duplicate vectors across folds), or by instance (leading to leakage)? Please report type-level CV results.
- For the UD evaluation that yields F=0.99: how did you obtain vectors for UD lemmas from the lemma_POS embedding model without using gold tags? If you used gold tags to choose the vector (lemma_POS), the evaluation is circular. Please clarify and, if needed, redo the evaluation using a model trained without PoS in the token.
- For the contextless lemma-to-PoS predictor (accuracy 0.84): what is the performance of a majority-lemma baseline built from the same training set? Please also provide per-class precision/recall and confusion matrices.
- Are the BNC training and BNC test vocabularies disjoint by type (freq >500 vs 100–500), and did you ensure no accidental overlap due to lemmatization/tagging variants?
- In the k-best feature experiment, can you report held-out test accuracy (not training) as k increases, to support the “distributed across ≥100 dimensions” claim?

Recommendation
Reject. The paper contains interesting qualitative observations about PoS boundary cases and potential annotation issues, and it reinforces known facts about embeddings encoding PoS information. However, the novelty is limited, and the evaluation has serious methodological flaws (potential leakage, circular UD evaluation, missing baselines), the comparison to a contextual tagger is unfair, and the paper has substantial presentation issues. With stronger, leakage-free evaluations, appropriate baselines, and improved reporting, this line of work could be a valuable short paper.

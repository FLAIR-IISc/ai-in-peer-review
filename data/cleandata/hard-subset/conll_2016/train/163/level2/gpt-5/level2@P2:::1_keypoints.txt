- Paper studies how distributional word embeddings interact with PoS categories via a classifier, outlier analysis, qualitative confusions, feature distribution, and a context-free PoS predictor.
- Primary contribution: using embedding-based PoS predictions as diagnostics for boundary cases and annotation inconsistencies.
- Secondary contributions: empirical characterization of PoS signal distribution across embedding dimensions and a proof-of-concept context-free predictor.
- Strength: insightful, linguistically informative outlier analysis revealing graded PoS membership and systematic BNC tagging errors.
- Strength: cross-resource checks (UD English) show some issues are corpus-specific, supporting diagnostic claims.
- Strength: clear evidence that PoS signal is distributed across many embedding dimensions via ANOVA-based feature ablation.
- Strength: simple low-resource classifier (≈1.5k prototypical lemmas) achieves 0.84 accuracy as an interesting baseline.
- Weakness: cross-validation leakage—word-type duplicates likely appear in both train/test folds, inflating reported F=0.979; need type-level CV.
- Weakness: training embeddings on lemma+PoS tokens may artificially partition space and exaggerate PoS separability; outlier analysis should be repeated on untagged embeddings.
- Weakness: unclear UD evaluation pipeline—possible use of UD gold tags to select lemma+PoS vectors undermines cross-corpus claims.
- Weakness: missing baselines for context-free predictor (e.g., majority-lemma baseline, character/morphology) make 0.84 accuracy hard to judge.
- Weakness: Stanford tagger comparison is unfair because it was run on isolated tokens without sentence context.
- Question: Did CV ensure all instances of a word type are in the same fold; request type-level CV results.
- Question: How were UD lemma vectors obtained given BNC vocabulary uses lemma+PoS—were UD gold tags used?
- Question: Can error rates be stratified by frequency to assess whether confusions (e.g., PROPN→NOUN) are due to sparsity?
- Question: For the “plain/clear/sharp” BNC issue, can you quantify detector precision/recall, provide more examples, and report manual validation?
- Suggestion: Re-run outlier analysis with embeddings trained on untagged lemmas to strengthen core claims.
- Suggestion: Add baselines—lemma-majority PoS, nearest-prototype cosine classifier, orthographic/morphological features—and report macro/micro scores.
- Suggestion: Report robustness across random seeds, window sizes, and subsampling as these affect PoS signal.
- Suggestion: Evaluate feature importance on held-out data and consider L1 regularization to avoid overfitting when increasing k.
- Suggestion: Improve reproducibility by providing preprocessing/mapping scripts and clarify BNC constraints; fix presentation, references, and figure/table issues.
- Overall: Useful and convincing qualitative contribution, but methodological issues (CV leakage, lemma+PoS tokenization, unclear UD eval) must be addressed; with fixes and stronger baselines this would be a solid paper.

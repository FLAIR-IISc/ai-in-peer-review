- Task and evaluation: paper predicts compound compositionality by comparing compound token vectors to composed component vectors and evaluating with Spearman correlation against human ratings.  
- Dataset: uses an existing set of 90 compounds rated 1–5, but annotator count and selection criteria are unspecified.  
- Experimental axes: compares neural embeddings vs. syntactic-context count vectors, and unaligned vs. “aligned” count vectors for the dependent component.  
- Main results: syntactic-context count vectors outperform neural embeddings.  
- Alignment results: using only aligned vectors performs worse than unmodified vectors; a tuned hybrid of aligned and unaligned gives only marginal gains.  
- Novelty concern: alignment technique appears to duplicate the authors’ prior (suppressed) work, limiting novelty.  
- Dataset adequacy: dataset is small, insufficiently described, and frequency ranges for compounds/components are not reported—raising concerns about result fragility.  
- Writing/presentation: introduction is clear but other sections (e.g., 5.1) are hard to follow and would benefit from running examples.  
- Terminology: “packed anchored tree” seems unnecessary and is essentially dependency path extraction; calling it a “tree” is unclear.  
- Undefined terms: “elementary APTs” are not clearly defined; the reviewer infers they are (word, path, word) triplets but requests explicit definition.  
- Table 2 ambiguity: unclear whether features of order >3 are discarded or removed due to “incompatible types”; request explicit examples and clarification.  
- Compound selection: reviewer asks how the 90 compounds were selected and requests frequency ranges for compounds and components.  
- Annotations: number of judgments per compound is not reported; reviewer asks about ties and implications for Spearman ranking.  
- Terminology suggestion: use “component” instead of “constituent” to avoid syntactic ambiguity.  
- Co-occurrence claim: the claim that constituents co-occur with compounds is more plausible for heads than modifiers (example: “spelling” vs “spelling bee”).  
- Clarification needed: request a running example to explain “elementary representation” and “compound phrase token” references.  
- PMI clarity: ask whether “shifted PMI” is exactly the PMI in Equation (3) and request clarification of related sentences that read as contradictory.  
- Alignment overlap: unclear what “not properly aligned” means and why NMOD relations should exhibit considerable overlap—request explanation of lines 558–563.  
- Feature order inconsistency: puzzled why composed representations use higher-order dependency features but similarity measurement is restricted to first-order features; Note 3’s justification is unclear.  
- Smoothing effect: claim that PPMI smoothing with α=0.75 helps is not evident from Table 3—request clarification.  
- Hybridity parameters: request the optimal values of h and q from Equations (8) and (9) to assess the hybrid method’s contribution.  
- Results reporting: Table 4 seems to use the add combination—ask to state this explicitly and to report results for compounds using word2vec vectors.  
- FREQ baseline: question the intuition that more frequent compounds are more compositional and suggest this may indicate dataset bias.

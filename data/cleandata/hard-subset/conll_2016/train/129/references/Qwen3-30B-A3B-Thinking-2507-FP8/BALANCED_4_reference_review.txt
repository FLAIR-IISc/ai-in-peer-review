

# Review of "Semi-supervised Convolutional Networks for Translation Adaptation with Tiny Amount of In-domain Data"

## Summary of Contributions
The paper proposes a novel approach for domain adaptation in statistical machine translation (SMT) when only tiny amounts of in-domain data (100-1000 sentences) are available. The main contributions are: (1) a semi-supervised CNN framework for domain classification that leverages pre-trained word embeddings from general-domain data to compensate for limited in-domain samples; (2) empirical demonstration that this method significantly outperforms existing LM-based data selection approaches, especially when in-domain data is extremely scarce; and (3) rigorous analysis showing the method's stability and effectiveness even with as few as 100 in-domain sentences.

## Strengths

1. **Addresses a genuinely important practical problem**: The paper tackles a critical limitation in domain adaptation for machine translation where previous work assumes reasonable-sized in-domain data (thousands of sentences). The authors correctly identify that many emerging application domains (e.g., social media translation) have very limited in-domain data, making this problem highly relevant to real-world applications.

2. **Strong empirical results**: The method achieves significant improvements over strong baselines across four language pairs and three different domain types (sms, tweets, Facebook posts). The most compelling result is the 3.1 BLEU improvement on the Chinese-to-English sms task with only 3% of the training data, demonstrating the method's practical value.

3. **Rigorous analysis of scalability**: The paper provides a thorough analysis of how the method performs as the amount of in-domain data decreases. The key finding that their semi-supervised CNN maintains performance with only 100 in-domain sentences (while LM-based methods degrade significantly) is particularly compelling and well-supported by the experiments.

4. **Clear technical contribution**: The semi-supervised CNN approach (using word2vec embeddings trained on general-domain data) is well-motivated and effectively addresses the core problem of limited in-domain data. The ablation studies comparing one-hot CNN vs. semi-supervised CNN provide strong evidence for the value of the pre-trained embeddings.

## Weaknesses

1. **Limited novelty in core methodology**: While the application to domain adaptation with tiny in-domain data is novel, the core CNN approach (especially the semi-supervised version) is largely borrowed from existing text classification literature (Johnson and Zhang, 2015b). The paper doesn't sufficiently articulate what makes their specific application to domain adaptation different from standard text classification approaches, which weakens the novelty claim.

2. **Lack of theoretical justification**: The paper doesn't provide a strong theoretical argument for why CNNs are particularly well-suited for this domain adaptation task compared to other classification methods (e.g., SVMs, logistic regression). The discussion in Section 5.4 about why CNNs work well is somewhat hand-wavy and lacks rigorous theoretical grounding.

3. **Comparison to recent domain adaptation methods**: The paper compares only against LM-based data selection methods but doesn't compare against more recent unsupervised or weakly supervised domain adaptation approaches that have emerged in the last few years. The claim that "there is no previous work considering the scenario where only a tiny amount of in-domain data is available" appears overstated given recent advances in unsupervised domain adaptation.

4. **Potential overfitting concerns**: Using the dev set as in-domain data is problematic since dev sets are typically used for model selection rather than as true in-domain data. The paper doesn't adequately address whether the dev set is representative of the test domain, which is crucial for the validity of the approach. With only 100 sentences for training, there's a significant risk of overfitting.

## Questions to Authors

1. How does the proposed method compare to recent unsupervised domain adaptation approaches (e.g., those using back-translation or adversarial training) that don't require any in-domain data at all? Would the proposed method still be beneficial in such scenarios?

2. The paper uses the dev set as in-domain data. How would you handle the case where there is no dev set available for the target domain? Would you need to use a small amount of test data to create the dev set?

3. The paper states that "the word embeddings learned from unlabeled data" are used to compensate for limited in-domain data. How sensitive is the method to the quality of these pre-trained embeddings? Would using embeddings trained on domain-specific data (if available) improve performance?

## Recommendation

This paper addresses an important practical problem with strong empirical results, but the novelty of the core methodology is limited. The authors present a well-executed application of existing text classification techniques to a relevant machine translation problem. While the results are compelling, the paper would benefit from a stronger theoretical justification for why CNNs are particularly suitable for this task and a more comprehensive comparison with recent domain adaptation methods.

I recommend **weak acceptance** with the understanding that the authors should address the novelty concerns and provide a more thorough comparison with recent domain adaptation approaches in their revision. The paper has significant practical value, but the novelty is somewhat marginal for a top-tier conference like CoNLL. With the suggested revisions, this work could make a strong contribution to the field.

- Proposes a novel MT data selection method scoring/ranking general-domain sentences with a CNN; comparisons to LM-based prior work are thorough.  
- Lacks comparison to bilingual data selection baselines (e.g., sum of cross-entropy differences), which would strengthen evaluation.  
- Motivation for using CNNs over RNNs/LSTMs is argued (local receptive fields capture important segments), but initial rationale was unclear.  
- Does not experimentally compare BOW vs SEQ (or combined) input representations, leaving a key design choice unvalidated.  
- Textual description of CNN architecture (one-hot, semi-supervised embeddings) is clear and precise.  
- A visual diagram of the CNN layer structure and input combination would substantially improve clarity.  
- Writing is good overall, but citation formatting (redundant parentheses, \citet vs \citep) should be standardized.  
- Experiments support claims, but the method for choosing the number of selected in-domain sentences (via a validation set) is under-specified.  
- Needs clarity on what data forms the validation set, how CNN hyperparameters are tuned, and sensitivity to those choices.  
- Table 2 should report results using the same number of selected sentences for fair comparison (as Figure 1 implies).  
- Suggested extensions: test with larger in-domain data, include example sentences selected by each method, compare against Axelrod (2015) POS-based abstraction, cite Lewis & Moore for sum-of-scores similarity, and explore a bilingual CNN to learn source-target score weights.

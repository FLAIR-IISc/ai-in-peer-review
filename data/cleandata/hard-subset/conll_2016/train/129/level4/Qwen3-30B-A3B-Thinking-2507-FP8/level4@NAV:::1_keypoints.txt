- Proposes MT training-data selection via a CNN classifier scoring/ranking general-domain sentences; compares well to continuous and n-gram LM baselines but omits bilingual selection baselines (e.g., sum of cross-entropy differences).  
- Initial motivation for choosing CNN over RNN/LSTM was unclear, but the argument that CNNs capture important text segments strengthens the choice.  
- No experiments validate whether BOW, SEQ, or combined representations are most important or why.  
- Textual description of the CNN (one-hot vs. pre-trained embeddings) is clear; a diagram of layer structure and input combination would improve clarity.  
- Paper is well written overall, though citation formatting mistakes (e.g., \citet vs. \citep) appear in places (line 385).  
- Experiments support claims but the method for selecting the in-domain sentence count is unclear: validation data source is unspecified and which CNN hyperparameters were tuned (and sensitivity to them) is not reported.  
- Table 2 should compare all methods using identical sentence counts; Figure 1 indicates the proposed method still wins under that constraint.  
- Run an experiment comparing the method to baselines using larger in-domain datasets (beyond the development set).  
- Include example sentences selected by different methods (Section 5.4) to substantiate qualitative claims.  
- For the surface-form abstraction claim (Section 5.4), compare to Axelrod (2015) which uses POS tags to reduce LM sparsity to see if word2vec adds benefit.  
- Note that summing source and target classification scores parallels Lewis-Moore LM selection (sum of cross-entropy differences); add that reference near line 435.  
- Consider extending the CNN to learn weights for the bilingual sum (i.e., learn weighting of source vs. target scores).

- Paper proposes a CNN-based in-domain data selection method for SMT and reports improvements over LM-based selection.
- Claimed 3.1 BLEU improvement is misleading and requires careful interpretation.
- The 3.1 BLEU gain occurs only when in-domain data is already present in training; training solely on in-domain gives +2.8 BLEU.
- Authors must clarify whether the reported gains are net improvements over a baseline trained only on out-domain data.
- Compare against interpolated separate in- and out-domain models to contextualize claimed improvements.
- Most compelling result: CNN works well with extremely limited in-domain data (e.g., 100 sentences).
- Low-data experiment is restricted to zh→en where training already contains in-domain data, which may simplify selection.
- Need cross-task validation (e.g., report results for all tasks using exactly 100 in-domain sentences) to support generalizability.
- Social media domain has high OOVs and non-standard tokens; authors should state any specialized preprocessing applied.
- Data selection cannot solve OOVs—paper should include concrete translation examples showing specific improvements (slang, hashtags, usernames).
- Section 3.2 lacks detail on how embeddings, segment vectors, and one-hot vectors are combined and needs a model diagram; the loss function is unspecified.
- Section 5.2 should state how system weights were tuned (e.g., development set optimization) for reproducibility.
- Overall contribution is valid (robustness with scarce in-domain data) but main claim may overstate incremental value given experimental setup limitations.

This paper describes a method for in-domain data selection for SMT with convolutional neural network classifiers, applying a framework similar to Johnson and Zhang, 2015. This method performs approximately 0.5 BLEU points better than language model-based data selection, and unlike other methods, it is robust even when only a very small in-domain data set is provided.

The paper claims an improvement of 3.1 BLEU points. However, the results show that improvements of this magnitude are only achieved when the training set contains in-domain data – training on in-domain data alone produces +2.8 BLEU. It might also be interesting to compare this system to one that interpolates separate in- and out-domain models.

In my opinion, the more impressive result comes from the second experiment, which shows that the CNN classifier is still effective even when there is very little in-domain data. However, the second experiment is only run on the zh2en task, which includes actual in-domain data in the training set, making selection easier. Will the results hold up for other tasks where there is no in-domain data in the training set? The results for the en2es and en2zh tasks already point in this direction, as the development set contains only a few hundred sentences. I think the claim would be better supported if the results for all tasks were reported, assuming that only 100 sentences were used for training.

When translating social media text, one often encounters problems very different from those of other domains, the most notable of which is a high OOV rate due to non-traditional spellings (at least for Latin scripts). The text may also contain special character sequences such as usernames, hashtags, or emoticons. Was any special preprocessing or filtering step applied to the data?

Since data selection cannot solve the OOV problem, it would be more informative to know how improvements are made through optimization via data selection, perhaps by providing examples.

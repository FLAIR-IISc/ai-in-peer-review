- Introduces a CNN-based classifier for ranking general-domain sentences for MT data selection and shows it outperforms prior continuous and n-gram LM methods.
- Missing comparison against bilingual data selection methods (e.g., sum/difference of cross-entropies).
- Choice of CNN over RNN/LSTM not obvious at first, though authors argue local sentence sections matter.
- Should compare bag-of-words (BOW) vs sequential (SEQ) representations or their combination to assess representation impact.
- Textual CNN description (including one-hot/semi-supervised embeddings) is clear, but a visual layer diagram would improve clarity.
- Minor citation formatting issues (unnecessary parentheses in places).
- Experimental validation is unclear: which validation set was used to choose the number of in-domain sentences and CNN hyperparameters, and hyperparameter sensitivity is not discussed.
- Table 2 should report scores for different approaches using the same number of selected sentences for fair comparison.
- Suggest running experiments with more in-domain data (beyond the dev set) to test robustness.
- Recommend including example sentences selected by different methods to support claims in section 5.4.
- Propose an additional baseline that replaces words with POS tags (per Axelrod 2015) to evaluate benefits of word2vec embeddings.
- Advise adding reference to Lewis-Moore’s work on source/target LM data selection using summed classification scores.
- Suggest exploring bilingual/parallel extensions (learning weights for summed source and target classification scores) by extending the CNN to the bilingual setting.

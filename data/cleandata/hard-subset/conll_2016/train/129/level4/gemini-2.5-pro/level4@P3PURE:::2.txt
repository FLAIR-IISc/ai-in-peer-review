This paper proposes a method for in-domain data selection for Statistical Machine Translation (SMT) using a convolutional neural network (CNN) classifier, following the framework of Johnson and Zhang (2015). The proposed method outperforms language model-based data selection by approximately 0.5 BLEU points and, unlike competing methods, demonstrates robustness even when only a very small in-domain dataset is available for training the classifier.

The paper claims an improvement of 3.1 BLEU points. However, the results indicate that this magnitude of improvement is only achieved on a task where the out-of-domain training set already contains some in-domain data. A simple baseline trained only on this identified in-domain data already achieves a +2.8 BLEU gain, which accounts for the vast majority of the claimed improvement. It would be interesting to see a comparison against a system that interpolates separately trained in-domain and out-of-domain models.

A more impressive result, in my opinion, is found in the second experiment, which demonstrates that the CNN classifier remains effective even with very little in-domain data. However, this experiment is only conducted on the zh-en task, which, as noted, includes in-domain data in the general training set, potentially making the selection task easier. It is unclear if this result would hold for the other tasks where the training set contains no in-domain data. The positive results for the en-es and en-zh tasks, which use development sets of only a few hundred sentence pairs, already suggest the method's effectiveness in low-resource settings. The claim of robustness would be significantly strengthened if results for this low-resource scenario (e.g., training the classifier with only 100 sentence pairs) were reported for all tasks.

Translating social media text presents unique challenges not found in other domains, most notably a high OOV rate due to non-conventional spelling and the presence of special tokens like usernames, hashtags, and emoticons. The paper should clarify what, if any, special preprocessing or filtering steps were applied to the data to handle these phenomena.

Furthermore, since data selection does not directly address the OOV problem, it would be insightful to understand the specific types of translation improvements gained through this adaptation. Providing qualitative analysis with translation examples would be beneficial.

Specific remarks on sections:

**Section 3.2:**
*   The description of how the different vectors (word embeddings, segment vectors, and one-hot vectors) are combined in the model could be clearer. An illustration of the model architecture would be very helpful.
*   The paper should specify the "designated loss function" that was used.

**Section 5.2:**
*   For completeness, the paper should describe how the SMT system weights were tuned.

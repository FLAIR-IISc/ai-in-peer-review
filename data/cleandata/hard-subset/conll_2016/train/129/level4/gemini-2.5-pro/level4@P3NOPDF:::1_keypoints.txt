- Motivation/rationale: CNN is well-motivated for picking salient n-gram features and focusing on relevant sentence parts for domain relevance.  
- Context/comparison: Paper is well-situated in prior work and compares effectively against n-gram and continuous LM selection methods.  
- Clarity of description: Technical details of the CNN and use of one-hot vs pre-trained embeddings are clearly explained.  
- Experimental rigor/reproducibility: Need to specify which validation data determined the number of selected sentences and which data tuned CNN hyperparameters, plus include sensitivity analysis.  
- Fairness of comparison: Comparisons should constrain all methods to select the same number of sentences (report in a table) for direct BLEU score comparisons.  
- Ablation on input representation: Provide experiments for BOW-only, SEQ-only, and combined models to show each representation’s contribution.  
- Comparison to bilingual selection methods: Acknowledge and/or include bilingual baselines (e.g., Moore-Lewis-like methods) since summing source and target scores is conceptually similar.  
- Model visualization: Add a diagram of the CNN architecture showing how input representations are combined.  
- Qualitative analysis: Include example sentences uniquely selected by the CNN to illustrate abstraction from surface forms.  
- Additional baseline: Consider comparing to Axelrod (2015)-style abstraction (e.g., POS-based replacements) to test alternatives to pre-trained embeddings.  
- Future work on bilingual modeling: Explore learning optimal weights for source/target score combination or a unified bilingual CNN that models parallel pairs jointly.  
- Citation formatting: Fix inconsistent citation commands (e.g., \citet vs \citep) for correctness.

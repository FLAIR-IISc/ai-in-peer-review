- Proposes a CNN-based data selection method for MT that ranks sentences from a general-domain parallel corpus; comparison to n-gram and continuous-space LM methods is well-executed.  
- Unclear whether the method was compared to bilingual selection techniques (e.g., cross-entropy difference / Moore-Lewis style).  
- Strong justification for using a CNN over RNN/LSTM: CNNs can identify highly-indicative sentence segments rather than evaluating whole sentences.  
- Missing experimental ablation study to assess the relative importance of BOW, SEQ, and combined representations and to explain differences in performance.  
- CNN architecture description (one-hot vs pre-trained embeddings) is clear but would benefit from a diagram showing layers and input processing.  
- Paper is generally well-written but contains minor formatting issues (e.g., improper use of \citet vs \citep).  
- Concern about selecting the number of sentences via a separate validation set: the paper does not specify which validation set was used.  
- Unclear how CNN hyperparameters were tuned and how sensitive model performance is to these choices.  
- For fair comparison, Table 2 should report all methods evaluated with the same number of selected sentences; Figure 1 suggests the proposed method would still outperform baselines under this control.  
- Recommend experiments with larger in-domain datasets (beyond the development set) to test robustness.  
- Suggest including qualitative examples of sentences selected by different methods to substantiate claims in Section 5.4.  
- Recommend comparing against Axelrod et al. (2011) (POS-based word replacement) to evaluate benefits of word2vec abstraction.  
- Note that summing source and target scores resembles bilingual Moore-Lewis selection; add a reference and discussion.  
- Future work suggestion: extend the CNN to a bilingual model that learns optimal weights for combining source and target scores rather than summing.

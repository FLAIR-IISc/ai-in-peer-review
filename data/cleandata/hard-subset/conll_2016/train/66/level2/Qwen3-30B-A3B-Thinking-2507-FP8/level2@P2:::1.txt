

Summary: This paper proposes a transition-based joint syntactic-semantic parser using stack LSTMs to represent the entire algorithmic state (stacks, buffer, action history), eliminating expert-crafted features. It claims to achieve the best joint parsing performance on CoNLL 2008-9 English tasks among models that jointly learn syntax/semantics, with linear-time inference (177.6s on CoNLL 2009 test data).

Main Contributions (as assessed):
1. Application of stack LSTMs to represent the *entire* parser state (not just top stack items) for joint parsing, enabling feature-free learning.
2. Demonstrated empirical superiority over prior *joint* models (Henderson et al., 2008; Gesmundo et al., 2009; Titov et al., 2009) on CoNLL 2008-9 English.
3. Practical linear-time performance (177.6s) vs. slower pipeline systems.

Strengths:
1. **Novel architectural insight**: Using stack LSTMs to capture the *full* parser state (not just local features) is a clear technical advance, well-motivated by prior work (Dyer et al., 2015). The implementation details (Fig. 3, Table 1) are sound and directly support the claim that no expert features are needed.
2. **Strong empirical validation of joint parsing**: The ablation (Table 3) convincingly shows that joint learning (Joint) outperforms Hybrid (syntax-only features) on in-domain data, and Semantics-only underperforms, validating the core hypothesis that syntax and semantics mutually benefit in a joint model.
3. **Practical runtime advantage**: The 177.6s runtime is genuinely impressive and clearly superior to pipeline systems (e.g., Lei et al., 2015: 439.9s). This addresses a key practical limitation of prior joint models and is well-justified by the greedy inference design.

Weaknesses:
1. **Misleading "best performance" claim**: The paper claims "best parsing performance among models that jointly learn syntax and semantics," but this is trivially true as they outperform *other joint models*. Crucially, they do *not* compare to the *actual state-of-the-art systems* (Zhao et al., 2009; Lei et al., 2015), which use pipelines with expert features and achieve higher scores (Table 3, last block). This misrepresents the field and overstates significance.
2. **Inaccurate "no expert features" claim**: The model uses pretrained word embeddings (Section 3.5), which are *de facto* expert features. The paper claims "no expert-crafted features," but embeddings are a standard feature engineering step. The absence of *syntactic* features (e.g., arc paths) is valid, but this nuance is lost in the claim.
3. **Weak multilingual evaluation**: Results (Table 4) show Chinese performance *depends on pretrained embeddings* (not shown without them), and Japanese has minimal training data (4,393 examples). This undermines the claim of "easy adaptability" and suggests the model is sensitive to data size and embedding availability.
4. **Missing critical ablation**: No experiment tests whether the stack LSTM is *necessary*. The paper claims "no expert features" but doesn’t show if a simpler feature-based model (e.g., using only top-stack features) would perform comparably. This is essential to validate the core innovation.

Questions to Authors:
1. Why does the paper omit direct comparison to Zhao et al. (2009), the top-performing pipeline system on CoNLL 2009 (Table 3)? How would the joint model perform against this state-of-the-art system?
2. The paper states "no expert-crafted features," yet uses pretrained embeddings. How do the authors reconcile this with the standard NLP definition of "expert features" (e.g., syntactic paths)? Could the model achieve similar results with random embeddings?

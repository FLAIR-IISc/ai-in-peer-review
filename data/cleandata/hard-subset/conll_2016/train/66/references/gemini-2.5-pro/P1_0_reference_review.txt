### Summary and Contributions

This paper introduces a greedy, transition-based parser for joint syntactic and semantic dependency analysis. The model's primary novelty lies in its use of stack LSTMs to learn a dense vector representation of the entire parser state—including the syntactic stack, semantic stack, input buffer, and action history. This approach completely eschews the need for hand-crafted features, which have been a staple of high-performance parsers. The model is evaluated on the CoNLL 2008 and 2009 shared tasks, where it is shown to outperform all previous joint models and achieve performance competitive with strong pipeline systems, while being significantly faster.

As I see them, the main contributions of this work are:
1.  **A novel and effective architecture for joint parsing:** The paper successfully extends the stack LSTM model from syntactic parsing (Dyer et al., 2015) to the more complex task of joint syntactic and semantic parsing. It demonstrates that learning a comprehensive, continuous representation of the parser state is powerful enough to model the intricate interactions between syntax and semantics without needing explicit, feature-engineered guidance.
2.  **Strong empirical evidence for feature-free, greedy parsing:** The paper provides a thorough evaluation on standard benchmarks, showing that a purely greedy, learning-based model can surpass previous, more complex joint systems (e.g., those using beam search). The extensive ablation studies (syntax-only, semantics-only, hybrid) provide valuable insights into the benefits and trade-offs of the joint approach.
3.  **A highly efficient and practical model:** The parser is shown to be very fast, a key advantage for practical applications. The commitment to releasing an open-source implementation further strengthens this contribution.

### Strengths

1.  **Elegant and Powerful Model:** The core idea of composing representations of the stacks and buffer using stack LSTMs is both elegant and well-motivated. It provides a principled way to feed the entire algorithmic history into the decision-making process at each step. Achieving state-of-the-art results (among joint models) without any manually designed features is a significant accomplishment and a strong testament to the power of the representation learning approach.
2.  **Thorough and Insightful Experimental Analysis:** The authors conduct a comprehensive set of experiments. The inclusion of "Syntax-only," "Semantics-only," and "Hybrid" baselines is particularly commendable. This allows for a clear analysis of how syntax and semantics influence each other within this framework. For example, the finding that the `Hybrid` model outperforms the `Joint` model on out-of-domain data provides a nuanced and interesting result regarding overfitting, which is an important finding for the community.
3.  **Strong Performance and Efficiency:** The model sets a new state-of-the-art for joint models on the CoNLL 2008 and 2009 English tasks. While it doesn't beat the overall best-performing systems, it is competitive with many strong pipeline models from the shared tasks, which is impressive for a greedy, feature-less system. The reported speed is a major practical strength, making the model a viable option for downstream applications that require fast parsing. The direct speed comparison with Lei et al. (2015) effectively quantifies this advantage.
4.  **Clarity and Reproducibility:** The paper is well-written and clearly describes the transition system and the model architecture. The inclusion of diagrams (Fig. 3) and tables (Table 1) greatly aids understanding. The promise to release the code is a major plus for reproducibility and future work.

### Weaknesses

1.  **Limited Algorithmic Novelty:** The paper builds heavily on existing work. The transition system, including the method for interleaving syntactic and semantic actions, is largely adopted from Henderson et al. (2013). The neural architecture is a direct application of the stack LSTM from Dyer et al. (2015) to this new problem. While the combination is novel and effective, the contribution is more in the application and engineering than in fundamental new algorithms or neural architectures. The paper could be strengthened by discussing why this specific inherited synchronization scheme is optimal, or if the new representation learning approach could enable more flexible interaction between the syntactic and semantic components.
2.  **Significant Gap to Overall State-of-the-Art:** The paper is transparent about not beating the top-performing systems, but the performance gap is still substantial. The authors attribute this to orthogonal techniques like global inference and expert features. However, this raises the question of whether a purely greedy, local model can ever close that gap. The claim of "bypassing expensive features" is compelling, but the results suggest that those features (or the global view they enable) capture linguistic phenomena that this model misses. A brief error analysis or discussion of the limitations of the greedy approach would be valuable.
3.  **Ambiguity in System Design for CoNLL 2008:** For the CoNLL 2008 task, predicate identification is handled by a "zero-Markov order bidirectional LSTM." This component is described very briefly. It is unclear if this is a pre-processing step (making the system a pipeline for this subtask) or if it is integrated more deeply. If it is a pipeline, it slightly undermines the "joint" nature of the system for this specific task. More detail on this component and its integration would be necessary to fully assess the model's architecture.
4.  **Incomplete Hyperparameter Tuning:** The authors state that some hyperparameters were "set intuitively" and that more careful tuning would likely yield improvements. While understandable, this introduces uncertainty. Key architectural parameters like the hidden state dimension and composition function dimension were not tuned. This makes it difficult to know if the reported performance is truly representative of the model's potential or if it could be easily improved (or is sensitive to these choices). A sensitivity analysis on these core parameters would make the results more robust.

### Questions to Authors

1.  Regarding the CoNLL 2008 system, could you please clarify how the bidirectional LSTM for predicate identification is integrated with the main parser? Is it run as a first-pass classifier, providing fixed predicate labels to the parser, or is there a more joint interaction?
2.  The results show that the `Joint` model overfits more to the training domain than the `Hybrid` model. This is an interesting finding. Do you have any hypotheses as to why jointly learning syntax and semantics would be more susceptible to domain-specific overfitting in this architecture?
3.  The synchronization of syntactic and semantic transitions is fixed by the transition system from Henderson et al. (2013). Did you consider alternative schemes? For instance, could the model learn *when* to switch between syntactic and semantic actions, rather than having a hard-coded trigger (i.e., buffer-modifying actions)? It seems the rich state representation from the stack LSTMs might make such a strategy feasible.

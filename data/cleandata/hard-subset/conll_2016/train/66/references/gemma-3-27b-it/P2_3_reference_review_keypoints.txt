- Presents a greedy, transition-based joint syntactic and semantic dependency parser using Stack LSTMs to learn parser-state representations without hand-crafted features.  
- Achieves strong results on CoNLL 2008/2009, outperforming previous joint models and running significantly faster.

- Shows joint parsing can be effective without expert feature engineering, simplifying development and improving portability.  
- Extends Stack LSTMs from syntactic parsing to joint syntactic-semantic parsing, demonstrating their representational power.  
- Provides a fast, practical system (notably 177.6s on CoNLL 2009) with open-source release.

- Strong empirical improvements over prior joint models with thorough comparisons.  
- Clear motivation for joint modeling and rationale for using Stack LSTMs.  
- Comprehensive experiments including ablations and multilingual evaluation.  
- Practicality emphasized via runtime measurements and planned open-source code.  
- Well-written and accessible presentation, aided by example runs.

- Core transition system is borrowed (Henderson et al., 2013); novelty mainly in representation learning rather than algorithmic changes.  
- Limited direct comparison to current state-of-the-art pipeline systems that use heavy feature engineering.  
- Performance depends on pre-trained embeddings, especially for languages like Chinese.  
- Greedy decoding may limit accuracy; trade-offs with beam search not fully explored.  
- Lacks detailed error analysis to illuminate failure modes.

- Clarify specific novel contributions beyond applying Stack LSTMs and quantify benefits of replacing hand-crafted features.  
- Have you tried beam search or other non-greedy decoding, and what accuracy/runtime trade-offs result?  
- Can you provide finer-grained ablations for components of the Stack LSTM representations (individual stacks, composition functions)?  
- Please provide an error analysis highlighting common error types and linguistic phenomena the model struggles with.

- Add code link for Lei et al. (2015) runtime reference.  
- A visual diagram of interleaved syntactic and semantic transitions would improve clarity.

- Overall: solid, well-executed paper with incremental algorithmic novelty but valuable empirical and practical contributions; recommend accept with minor revisions.

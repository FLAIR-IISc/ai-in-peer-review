

# Review of "A Joint Syntactic and Semantic Dependency Parser Using Stack LSTMs"

## Summary and Contributions

The paper proposes a joint syntactic and semantic dependency parser that uses a stack LSTM to learn representations for the entire algorithmic state, rather than just the top items on the stack. The main contributions claimed are: (1) a novel parser that learns representations for the entire parser state, (2) a greedy inference approach that outperforms previous joint parsing models, and (3) a system that achieves competitive performance without using expert-crafted features derived from full syntactic parses.

## Strengths

1. The paper makes a clear contribution to the joint parsing field by demonstrating that a single model can learn effective representations for the entire parsing state without relying on hand-crafted features. This is a significant methodological advance.

2. The runtime performance is impressive (177.6±18 seconds for CoNLL 2009 English test data on a single core), which is substantially faster than competing systems like Lei et al. (2015).

3. The multilingual results show that the approach generalizes well across languages, with performance comparable to the top systems at CoNLL 2009.

## Weaknesses

1. The claim that "it uses no expert-crafted features at all" is misleading and contradicts the paper's own methodology. The system relies on part-of-speech tags (a standard expert-crafted feature) and uses pretrained embeddings (a form of feature engineering). The authors state "part-of-speech vectors are learned and concatenated to the same," which is inconsistent with the claim of having "no expert-crafted features." This misrepresentation undermines the paper's main contribution.

2. The paper lacks a statistical significance test for its performance claims. The authors claim their model "significantly outperforms" previous work but provide no p-values or confidence intervals to support this assertion. This is a serious methodological flaw in a paper making strong performance claims.

3. The paper contains multiple formatting and technical inconsistencies that suggest carelessness. Key examples include:
   - "Lluı́s" instead of "Lluis" (with incorrect accent mark)
   - Inconsistent citation format ("Dyer et al., 2015" vs "Dyer et al. (2015)")
   - Typo "correspodning" instead of "corresponding" in Section 3.3
   - Missing Figure 2 (referenced in Section 2.2 but not provided)
   - Incomplete sentence: "like all in the example in Figure 1—"

4. The runtime complexity claim ("Asymptotic runtime complexity of this greedy algorithm is linear in the length of the input") is stated without proof or reference, which is problematic for a technical paper. Nivre (2009) provides the complexity analysis for arc-eager parsing, but the paper doesn't cite this or show how their specific implementation maintains linearity.

5. The paper claims to "outperform all previous joint parsing models" but only provides results for a subset of previous work. The table comparisons (Tables 2-4) omit key systems like Zhao et al. (2009) and Björkelund et al. (2010), which were top performers at the CoNLL 2009 shared task. The authors state "Our system's performance does not match that of the top expert-crafted feature-based systems" but don't provide the actual performance numbers for these systems, making it impossible to verify their claim.

## Questions to Authors

1. Please clarify the exact meaning of "no expert-crafted features" given that the system uses part-of-speech tags and pretrained embeddings. How does this differ from previous work that also uses POS tags?

2. Can you provide statistical significance tests for the performance improvements claimed over previous work?

3. Please explain why the paper claims to use "entirely greedy inference" without providing any comparison to beam search results, which is standard practice in parsing literature.

4. The paper references "Figure 2" in Section 2.2 but does not include this figure. Please confirm whether this figure was omitted by mistake.

## Overall Assessment

This paper has a promising core idea but is marred by multiple significant methodological and presentation issues. The misrepresentation of "no expert-crafted features" is particularly problematic as it undermines the main contribution. The lack of statistical significance testing for performance claims is a serious methodological flaw. The numerous formatting inconsistencies and technical errors further diminish the paper's credibility. While the runtime performance is impressive, these issues collectively prevent the paper from making a strong case for its claims.

I recommend rejection due to these substantial methodological and presentation flaws that significantly impact the paper's credibility, despite the potentially interesting technical contribution. The authors should address these issues thoroughly before resubmission.

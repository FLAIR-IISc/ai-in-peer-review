- Introduces a joint syntactic-semantic dependency parser using stack LSTMs to represent the entire parser state and uses greedy inference.  
- Claims competitive performance on CoNLL 2008/2009 English and much faster runtime (≈177.6±18s on a single core).

- Novelty: learns representations of the entire parser state (not just top stack items), advancing over prior hand-crafted feature approaches.  
- Efficiency: strong results with a notable runtime advantage (≈2.5× faster than some state-of-the-art systems).  
- Clarity: well-structured algorithm and architecture description with reproducible implementation details.

- Comparison incomplete: paper overstates outperforming prior work without quantifying gaps to top systems that use full parses and reranking.  
- Missing ablations: no systematic study isolating contributions of full-state representation versus greedy inference.  
- Lack of explanation: does not analyze why bypassing expert features works or whether full-state embeddings capture those features.  
- Greedy vs beam unclear: runtime claims conflate single-core implementation with greedy inference; no comparison to a beam-search variant.

- Request ablation: show contribution of full-state representation vs greedy inference and performance of a beam-search version.  
- Ask about adding expert features: would features like predicate-argument path improve performance or be redundant with full-state embeddings?  
- Clarify novelty: explain what makes applying stack LSTMs to joint parsing fundamentally different from syntactic-only use (Dyer et al., 2015).

- Overall: solid, practical contribution demonstrating efficient, feature-free joint parsing but likely incremental and overstates novelty without additional analyses.  
- Recommendation: temper claims, add ablation studies, and discuss how this approach complements existing expert-feature/global-decoding methods.

- Proposes a transition-based parser that jointly produces syntactic and semantic dependencies by learning a representation of the entire algorithm state.  
- Introduces a stack LSTM architecture to represent syntactic/semantic stacks, buffer, and action history.  
- Uses a greedy inference algorithm with linear-time complexity (including feature extraction) for efficient parsing.  
- Achieves state-of-the-art performance on CoNLL 2008/2009 English tasks, rivaling systems using handcrafted features.  
- Demonstrates practical efficiency: end-to-end runtime ~177.6±18s on CoNLL 2009 English test set on a single core.  
- Shows adaptability to multilingual settings with competitive results on non-English CoNLL 2009 datasets.  
- Provides limited empirical comparison to other contemporary joint parsing models.  
- Results depend heavily on pretrained word embeddings, which may not exist for all languages/domains.  
- Does not incorporate morphological features, which could hurt performance on highly inflective languages (e.g., Czech).  
- How will the authors address overfitting to the WSJ domain, given Hybrid model’s better out-of-domain performance?  
- Request for more training details: number of epochs, batch size, and learning-rate schedule.  
- How do the authors plan to extend the parser to incorporate morphological features for inflective languages?

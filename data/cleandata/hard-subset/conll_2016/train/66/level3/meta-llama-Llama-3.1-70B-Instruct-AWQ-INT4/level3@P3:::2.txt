**Paper Summary**

This paper proposes a joint syntactic and semantic dependency parser that learns a representation of the entire parser state using stack LSTMs. The parser employs a greedy inference algorithm and is evaluated on the CoNLL 2008 and 2009 English shared tasks. The results show that the parser outperforms previous joint models and achieves competitive performance with top systems, despite not using expert-crafted features or beam search.

**Main Contributions**

1. **Joint Syntactic and Semantic Dependency Parsing**: The paper proposes a novel approach to joint parsing that learns a representation of the entire parser state using stack LSTMs. This approach allows the parser to capture long-range syntactic features useful for semantic role labeling.
2. **Greedy Inference Algorithm**: The paper presents a greedy inference algorithm that has linear time complexity, including feature extraction. This makes the parser fast and efficient.
3. **No Expert-Crafted Features Required**: The parser does not require expert-crafted features, which makes it more flexible and easier to adapt to different languages and domains.

**Strengths**

1. **Competitive Performance**: The parser achieves competitive performance with top systems on the CoNLL 2008 and 2009 English shared tasks, despite not using expert-crafted features or beam search.
2. **Fast and Efficient**: The parser has linear time complexity, including feature extraction, making it fast and efficient.
3. **No Expert-Crafted Features Required**: The parser does not require expert-crafted features, which makes it more flexible and easier to adapt to different languages and domains.
4. **Well-Written Paper**: The paper is well-written and provides a clear explanation of the proposed approach and its advantages.

**Weaknesses**

1. **Limited Comparison to Other Works**: The paper only compares the parser to a limited set of previous joint models and top systems. A more comprehensive comparison to other works in the field would be beneficial.
2. **No Beam Search Results**: The paper only presents results for the greedy inference algorithm and does not provide results for beam search. This makes it difficult to compare the parser's performance to other works that use beam search.
3. **Lack of Morphological Features**: The paper does not incorporate morphological features, which are used by other systems to improve performance, especially in highly inflective languages.
4. **Dependence on Large Training Datasets**: The parser's performance is dependent on reasonably large training datasets, which may not be available for all languages or domains.

**Questions to Authors**

1. How does the parser's performance change when using beam search instead of the greedy inference algorithm?
2. Can the parser be adapted to incorporate morphological features to improve performance in highly inflective languages?
3. How does the parser's performance change when using different types of word embeddings, such as character-based word embeddings?
4. Can the parser be extended to other languages and domains, and how does its performance change in these settings?

- Proposes a neural-style topic model extending word2vec to learn document embeddings with sparsification to mimic topic outputs; conceptually appealing with promising qualitative examples.  
- Critical deficiency: no empirical evaluation — relies solely on illustrative examples without showing representativeness or comparative performance, leaving claims unsubstantiated.  
- Recommended evaluation 1: measure topic representations in supervised document classification using the same topic cardinality as baselines.  
- Recommended evaluation 2: evaluate on datasets with ground-truth document similarity (e.g., pairwise topic-vector comparisons).  
- Code release is commendable, but including a GitHub link in an anonymized submission violates conference policy and may warrant rejection.  
- Example selection (Figs. 3–6) rationale is unclear and risks cherry-picking results.  
- Topic-word representation method is computationally infeasible for large vocabularies/topics (roughly O(|V|×|K|) operations).  
- Tokenization is ambiguous: SpaCy noun-chunk extraction conflicts with reliance on pre-trained word2vec that lacks multiword term embeddings.  
- OOV handling is unaddressed; applying the model to novel documents would require updating embeddings, making them incomparable to trained representations.  
- Topic cardinality finding (20 topics for 20 Newsgroups) is unsurprising; an information-theoretic comparison with true labels would be more rigorous.  
- Comparative baseline omission: no comparisons to key neural topic models (e.g., Cao et al. AAAI 2015; Nguyen et al. TACL 2015; Shamanta et al. ECML/PKDD 2015).  
- Low-level issues: typos/phrasing errors (e.g., "it’s" → "its"), awkward sentences, Fig. 5 caption should use "terms," and inconsistent reference formatting.

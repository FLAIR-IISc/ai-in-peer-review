- Proposal of lda2vec: a model that jointly learns dense word embeddings and sparse, interpretable document-topic representations.  
- Extends skip-gram with negative sampling by using context = pivot word vector + document vector.  
- Represents each document vector as a weighted mixture of shared latent topic vectors.  
- Adds a Dirichlet-based loss term to encourage sparse, LDA-like document-topic distributions.  
- Contribution: a hybrid architecture unifying word embeddings and mixed-membership topic models in a single end-to-end trainable framework.  
- Contribution: demonstrates a practical implementation using automatic differentiation, easing development versus custom inference.  
- Strength: novel and coherent model idea that connects local token semantics with global document themes and allows joint training.  
- Strength: qualitatively compelling, interpretable topics on Twenty Newsgroups and Hacker News and preserved word-embedding regularities (similarity/analogy).  
- Strength: accessible and extensible framework with public code release, facilitating reproduction and extensions.  
- Weakness: lacks quantitative, comparative evaluation against baselines (e.g., LDA, doc2vec, neural topic models).  
- Weakness (evaluation specifics): needs downstream task results, direct topic coherence comparison to LDA, and document similarity benchmarks.  
- Weakness: insufficient discussion of related neural topic-modeling work (e.g., Cao et al., 2015; Nguyen et al., 2015).  
- Weakness: violates double-blind review by including a non-anonymized GitHub link.  
- Question: How were the example topics/words/analogies selected and can you show they aren’t cherry-picked?  
- Question: Clarify preprocessing for Twenty Newsgroups, especially treatment of multi-word expressions and named entities with unigram word2vec initialization.  
- Question: How should the model handle unseen documents and out-of-vocabulary words at inference given fine-tuned embeddings?

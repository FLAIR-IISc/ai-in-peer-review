Summary and contributions
- This paper studies parsing into broad-coverage semantic structures (BSS) that allow multiple parents, non-terminal nodes, and discontinuities, using UCCA as the grounded target representation. It explores (i) conversion-based parsing by mapping UCCA to constituency and dependency trees and back, and (ii) a new transition-based parser (BSP) that directly produces DAGs with discontinuities and non-terminals.
- Contributions as I see them:
  1) First systematic parsing study for UCCA with both conversion pipelines and a direct transition system capable of multiple parents, non-terminals, and discontinuities.
  2) A concrete transition inventory for DAG + discontinuous constituency parsing with a dynamic oracle and a small set of UCCA-oriented features (e.g., a “ratio” feature).
  3) An empirical comparison showing that a strong LSTM dependency parser on converted trees outperforms the direct BSP on primary edges, while BSP can produce some remote edges.

Strengths
- Addresses an important gap: UCCA is among the few grounded semantic formalisms capturing the three structural properties, yet had no parser. Providing any parsing baselines and a transition system is useful for the community.
- Pragmatic dual strategy: conversion-based pipelines and a direct parser, with upper-bound analyses for conversion loss, provide insights into where information is lost and what may be gained by direct modeling.
- Technically grounded transitions: the combination of NODE, LEFT/RIGHT-EDGE vs LEFT/RIGHT-REMOTE, SWAP, and delayed REDUCE mirrors successful strands in discontinuous constituency parsing and DAG dependency parsing. The dynamic oracle and the “ratio” feature are sensible design choices.
- Empirical scope: experiments include in-domain and out-of-domain (20K Leagues) evaluation, multiple parsers (Malt variants, LSTM), and a feature ablation for BSP. The out-of-domain results are reasonably robust.
- Potential for future progress: the clear performance gap between Malt and LSTM on the same conversion suggests a path for improving BSP via neural classifiers.

Weaknesses
- Evaluation metric is underspecified and likely inappropriate for DAGs. Matching is defined solely via the yield of the child node and its edge label, ignoring the parent identity. For DAGs with multiple parents, different edges with the same child and label but different parents would receive the same yield set, potentially inflating matches and losing attachment information. This departs from edge- or node-anchored matching used in DAG formalisms and collapses important structural distinctions. A careful, one-to-one edge matching that respects both endpoints (or a standardized UCCA metric) is needed; otherwise results are hard to interpret.
- Reproducibility gaps in core procedures:
  - Constituency→dependency conversion requires a linear order over UCCA labels to select h(u); the priority order is not specified. This is critical to reproduce results.
  - The inverse dependency→constituency “Label(u)” function is hand-written and “mostly Center”; exact rules are not provided.
  - The dynamic oracle’s specification (especially with REMOTE edges and SWAP) is not formalized.
  - Cycle prevention and acyclicity constraints for REMOTE edges are not discussed; it is unclear how the system guarantees DAGness beyond disallowing multiple primary parents.
  - Feature realizations depend on h(·)/h*(·) which again depend on the unspecified label priority.
  - POS/tagging and tokenization sources are not described; whether gold or predicted tags are used is unclear, affecting comparability.
- Empirical limitations:
  - The direct BSP underperforms the conversion+LSTM pipeline on primary edges by a large margin (≈10 F1 absolute). Remote edge F1 is very low (~13), with no analysis of recall bottlenecks or class imbalance.
  - No statistical significance testing or variance analysis (single runs only).
  - Limited error analysis: apart from a feature ablation, there is no qualitative breakdown (e.g., coordination, MWEs, discontinuities, remote vs primary edges).
  - Efficiency is not reported (training/inference time, beam effects); BSP is greedy only, which likely harms performance relative to UPARSE’s beam, but this is neither explored nor justified.
- Presentation quality and formatting problems that impede assessment:
  - Numerous broken citations (question marks, duplicated author names, inconsistent styles), e.g., “Abend and Rappoport.,? 2013”; “Dyer et al\.,? 2015”.
  - Typos and indexing errors (e.g., “passages 701–695”), and footnotes/references inlined incorrectly.
  - Algorithms 1–2 contain ambiguous variable definitions/conditions (e.g., Parentc, h*, i(x) index semantics), and key conditions are relegated to “Supplementary Material,” which is not provided.
  - Figure/table references are incomplete (Table 1, Table 3 not shown in this excerpt), and feature templates heavily rely on prior papers without sufficient self-containment.
- Threats to validity not discussed:
  - Removing linkage edges and implicit units simplifies the task; the impact on representational coverage and downstream utility is not quantified.
  - The out-of-domain test is same-language and relatively close in style; cross-lingual generalization (a touted UCCA strength) is not examined.

Questions to authors
- How exactly is the linear priority over UCCA edge labels defined for h(u)? Please provide the full ordering and justify it linguistically.
- How do you ensure acyclicity when creating REMOTE edges? Are there explicit cycle checks or constraints?
- Can you formalize the dynamic oracle (including for SWAP and REMOTE), and provide pseudocode?
- What POS tags and tokenization do you use (gold vs predicted)? If predicted, with which tagger and accuracy?
- How many remote edges are in train/dev/test, and what is the class distribution by label? Did you attempt loss reweighting or specialized features for rare REMOTE labels?
- Did you try beam search for BSP, or neural classifiers (e.g., biLSTMs) while keeping the same transition system? If so, how does that affect primary vs remote edges?

Additional comments
- Please correct indexing (“701–695”), fix all broken citations and LaTeX artifacts, and provide complete algorithmic details, including Label(u) and any heuristics used in inverse conversions.
- The evaluation protocol should be revisited to respect parent-child identity in DAGs; consider adopting or defining a standard UCCA evaluation measure and report primary vs remote separately, with clear matching rules and one-to-one alignment.
- Report runtime and memory, and include significance testing over multiple random seeds.
- Provide qualitative error analysis (e.g., confusion by label, discontinuity handling, coordination) and ablations for beam/neural variants if possible.

Recommendation: While the topic is important and the direction promising, the methodological and presentation issues (especially the evaluation metric and missing procedural details) are substantial. I lean toward rejection in the current form, but encourage a revision that fully specifies conversions/oracles, fixes evaluation, strengthens baselines, and improves clarity.

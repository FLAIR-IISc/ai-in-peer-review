

# Review of "Broad-Coverage Semantic Parsing: A Transition-Based Approach"

## Summary and Contributions

This paper proposes the first parser for Broad-Coverage Semantic Structures (BSS), which support multiple parents, non-terminal nodes, and discontinuous units. The authors present two approaches: a conversion-based method (converting BSS to dependency trees or constituency trees and using existing parsers) and a novel transition-based parser called BSP that directly handles BSS. The main contributions claimed are: (1) first parser for BSS, (2) conversion protocols between UCCA structures and related formalisms, and (3) novel transition-based parser for BSS.

## Strengths

1. The paper addresses a meaningful gap in semantic parsing literature by recognizing the need for representations that support multiple parents, non-terminal nodes, and discontinuous units. The UCCA annotation scheme is well-motivated and well-explained.

2. The conversion-based approach provides useful insights into the trade-offs between different semantic representation formalisms. The paper demonstrates that existing parsers can be adapted to handle BSS with reasonable performance, though with some loss of information.

3. The BSP parser represents a technically sound extension of transition-based parsing techniques to handle the specific requirements of BSS, with careful consideration of the necessary transitions and features.

## Weaknesses

1. **Fundamentally flawed evaluation metric**: The paper defines labeled precision and recall by dividing |M(Gp, Gg)| by |Ep| and |Eg|, where M(Gp, Gg) = {(e1, e2) ∈ Ep × Eg | y(e1) = y(e2) ∧ `p(e1) = `g(e2)}. This metric counts pairs of edges rather than edges themselves, which is a critical error. If a single edge in the gold standard matches multiple edges in the prediction, it would be counted multiple times, inflating the score. This flaw undermines the entire evaluation and makes the reported results difficult to interpret.

2. **Inaccurate claim about UCCA's parser status**: The paper states "to our knowledge, no existing parser for a grounded semantic annotation scheme supports the combination of these criteria. The only such scheme supporting them is UCCA (Abend and Rappoport, 2013), which has no parser." This is incorrect because the authors are proposing a parser for UCCA, so the statement "UCCA has no parser" is false in the context of the paper. The correct claim would be "UCCA had no parser prior to this work," but the current phrasing is misleading and undermines the paper's credibility.

3. **Inconsistent transition set description**: The paper states "EDGE transitions are disallowed where the child node already has an incoming primary edge," which is consistent with UCCA's constraint that "a UCCA node may only have one incoming primary edge." However, it then describes LEFT-EDGEX and RIGHT-EDGEX as having "this restriction," while LEFT-REMOTEX and RIGHT-REMOTEX do not have this restriction. The restriction should only apply to primary edges, not remote edges, but the paper's description is inconsistent and confusing. This undermines the technical rigor of the proposed parser.

4. **Unfair comparison between conversion-based and direct approaches**: The paper claims BSP "obtains comparable F-scores to MaltParser and UPARSE in terms of primary edges, but unlike them, is able to predict some of the remote edges too." However, the conversion-based methods (UPARSE, MaltParser) were specifically designed to convert BSS to tree structures (removing remote edges), so it's expected that they would have zero remote edge scores. This comparison is not fair to the conversion-based approaches, as they were designed to handle a different representation. The paper fails to acknowledge this fundamental difference in the experimental setup.

5. **Formatting and citation inconsistencies**: The reference list contains inconsistent formatting (e.g., "Abend and Rappoport.,? 2013, Abend and Rappoport.,"). Algorithm 2 contains a typo ("t′ ← Parentd(t)" instead of "t' ← Parent_d(t)"). The paper states "For trees, labeling nodes is equivalent to labeling edges" in Algorithm 1, which is confusing since the paper consistently uses edge labels throughout, not node labels.

## Questions to Authors

1. How exactly does the evaluation metric handle cases where a single gold edge matches multiple predicted edges, and how does this affect the reported scores?

2. Can you clarify why the conversion-based methods (UPARSE, MaltParser) were not designed to predict remote edges, given that the paper claims they are "converting BSS to a different representation" that "necessarily is lossy"?

3. The paper states "the inverse conversion introduces non-terminal nodes back into the tree" in Algorithm 2, but the procedure creates pre-terminals rather than true non-terminal nodes. How does this affect the quality of the reconstructed UCCA structures?

## Overall Assessment

While the paper addresses an important problem in semantic parsing, the fundamental flaw in the evaluation metric and the inaccurate claim about UCCA's parser status are serious issues that significantly undermine the paper's credibility. The inconsistencies in the transition set description and the unfair comparison between conversion-based and direct approaches further weaken the paper's technical contributions. These are not minor issues but rather critical flaws that affect the paper's core claims and results.

Despite the novelty of the problem addressed, the technical flaws and inconsistencies are substantial enough to warrant rejection. The paper would benefit from a complete re-evaluation of the metrics, clarification of the claims about UCCA's parser status, and careful revision of the transition set description before it can be considered for publication.

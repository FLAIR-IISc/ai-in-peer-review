

# Review of "Massively Multilingual Word Embeddings"

## Summary and Contributions
The paper proposes two dictionary-based methods (multiCluster and multiCCA) for estimating multilingual word embeddings across 59 languages, introduces improved evaluation metrics (multiQVEC and multiQVEC-CCA), and provides an evaluation web portal. The authors claim these methods enable better multilingual NLP by addressing coverage, semantic similarity, and cross-lingual translation properties.

## Strengths
1. **Novel evaluation metrics**: The authors correctly identify shortcomings in existing evaluation methods (QVEC) and propose multiQVEC-CCA as an improvement. Their correlation analysis showing better alignment with downstream tasks is a valuable contribution to the field.

2. **Comprehensive evaluation framework**: The web portal for evaluating multilingual embeddings is a practical contribution that will facilitate future research in this area, addressing a genuine need in the field.

3. **Extensive experimental validation**: The paper evaluates on 59 languages and compares multiple methods across nine metrics, providing a rich empirical foundation for their claims.

## Weaknesses

1. **Critical technical error in method description**: In Section 2.2 (multiCCA), the authors state "ECCA(m, v) = T−1en→m,enTm→m,enEm(v)" - this contains a clear typo where "T−1en→m,en" should be "T−1en→m,en" (the second "en" should be "m" or "en"). This is not a minor formatting error but a substantive mathematical mistake that would prevent reproducibility of the method. The paper claims this method "extends the bilingual embeddings of Faruqui and Dyer (2014)" but fails to correctly describe the extension.

2. **Contradictory results in evaluation**: Table 1 claims that "multiQVEC-CCA correlates better with both downstream tasks," but the data shows it correlates better with document classification (0.896) while correlating worse with dependency parsing (0.273) compared to multiQVEC (0.444). This directly contradicts the paper's main claim about the superiority of multiQVEC-CCA. The authors should clarify this inconsistency or correct the table.

3. **Reproducibility issues**: The paper states "URLs to software libraries on Github are redacted to comply with the double-blind reviewing of CoNLL." This is unacceptable - the purpose of a double-blind review is to evaluate the paper's content, not to hide reproducibility information. The web portal URL "http://128.2.220.95/multilingual" is also a temporary IP address, not a permanent URL, making it impossible for readers to access the evaluation system.

4. **Inconsistent language coverage**: The paper claims to work with "59 languages" but lists the languages in a footnote as {bg, cs, da, de, el, en, es, fi, fr, hu, it, sv, zh, af, ca, iw, cy, ar, ga, zu, et, gl, id, ru, nl, pt, la, tr, ne, lv, lt, tg, ro, is, pl, yi, be, hy, hr, jw, ka, ht, fa, mi, bs, ja, mg, tl, ms, uz, kk, sr, mn, ko, mk, so, uk, sl, sw}. However, in Section 5.2, they state "we train multilingual embeddings in 59 languages" but then later claim "for the set of 12 languages" and "seven languages" in different sections. This inconsistency makes it impossible to understand the actual scope of the experiments.

5. **Formatting and citation inconsistencies**: The reference list has inconsistent formatting (e.g., "Tsvetkov et al.,? 2015" instead of "Tsvetkov et al., 2015"), and the paper uses non-standard citation formats like "Al.Rfou. et al\.,? \Q2013\E" with backslashes and \Q/\E markers. These are minor but indicate a lack of attention to detail that undermines the paper's credibility.

## Questions to Authors

1. The equation in Section 2.2 (multiCCA) contains a clear mathematical typo. Please clarify the correct formulation of the projection matrices.

2. Please explain the contradiction between Table 1 (showing multiQVEC-CCA correlates better with document classification but worse with dependency parsing) and the paper's claim that it "correlates better with both downstream tasks."

3. The paper claims to evaluate on 59 languages but switches between 59, 12, and 7 languages in different sections. Please clarify the exact language sets used for each experiment.

4. The web portal URL "http://128.2.220.95/multilingual" appears to be a temporary IP address. Please provide a permanent URL or GitHub repository link for the portal.

5. The reference list contains inconsistent formatting and non-standard citation markers. Please correct these for publication.

## Overall Assessment

While the paper addresses a meaningful problem in multilingual NLP and presents some valuable contributions (particularly the evaluation metrics and web portal), the numerous technical errors, contradictions in results, and reproducibility issues are substantial enough to warrant rejection. As a perfectionist reviewer, I cannot overlook these fundamental flaws that undermine the paper's credibility. The technical error in the method description alone would prevent others from reproducing the work, which is unacceptable for a paper claiming to present new methods. The contradictions in the evaluation results further weaken the paper's main claims. These issues are not minor formatting problems but substantive flaws that would mislead readers and prevent future work from building on this paper.

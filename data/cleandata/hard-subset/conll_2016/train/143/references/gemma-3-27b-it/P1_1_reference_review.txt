## Peer Review for CoNLL Submission: "Estimating Massively Multilingual Word Embeddings"

**Summary:**

This paper investigates methods for creating word embeddings across a large number of languages (59 in total). It proposes two dictionary-based approaches – multiCluster and multiCCA – for estimating these embeddings, leveraging monolingual data and bilingual dictionaries.  Critically, the work also introduces improved evaluation metrics (multiQVEC and multiQVEC-CCA) for multilingual embeddings and provides a publicly available web portal for evaluation.

**Main Contributions (as I see them):**

1. **The Comprehensive Evaluation Framework:** The development of multiQVEC/multiQVEC-CCA and, importantly, the accompanying web portal is the strongest contribution. This provides a valuable resource for the community, addressing a clear need for standardized and robust evaluation of multilingual embeddings.  The correlation analysis with downstream tasks strengthens the validity of the new metrics.
2. **Scaling Dictionary-Based Methods:** Successfully applying dictionary-based methods to a large number of languages (59) is a notable engineering achievement. While the methods themselves aren’t entirely novel, the scalability and demonstrated functionality are important.
3. **Empirical Baseline & Dataset:** The paper provides a strong empirical baseline and releases the data used for training and evaluation, furthering reproducibility and enabling comparisons with future work.

**Strengths:**

1. **Significant Contribution to Evaluation:** The shortcomings of existing intrinsic evaluation metrics are well-articulated, and multiQVEC-CCA demonstrably addresses these issues. The correlation with downstream tasks validates the new metric’s effectiveness and provides a much-needed standard for the field.
2. **Practical Impact via Web Portal:**  The public web portal is a highly valuable resource. It lowers the barrier to entry for researchers developing new methods and promotes more rigorous and comparable results.  The inclusion of both development and test datasets is excellent.
3. **Thorough Experimental Setup:** The experiments are well-designed, employing multiple evaluation metrics and a comprehensive set of languages. The ablation studies and analysis of coverage are helpful. The clear reporting of results and code/data availability enhances reproducibility.
4. **Clear and Well-Written:** The paper is generally clearly written and well-organized, making it easy to understand the proposed methods and contributions.

**Weaknesses:**

1. **Limited Novelty of Embedding Methods:** While multiCluster and multiCCA are functional, they represent extensions of existing techniques (CCA, clustering-based approaches) rather than fundamentally novel embedding algorithms. The paper acknowledges this, but the primary contribution lies elsewhere.  The performance gains achieved by these methods (compared to baselines) aren’t overwhelming, especially considering the large number of languages involved.
2. **Sensitivity to Dictionary Quality:** The performance is heavily reliant on the quality of the bilingual dictionaries. The use of Google Translate for dictionary creation for the 47 languages raises concerns about noise and potential biases, particularly given that the performance on these languages is not thoroughly differentiated in the results.  The fixed threshold (τ = 0.1) for dictionary extraction from parallel corpora may not be optimal for all language pairs.
3. **Limited Analysis of Failure Cases:** While the correlation analysis is helpful, a deeper dive into *why* certain methods perform better or worse in specific scenarios (e.g., language families, resource availability) would be beneficial.  Exploring the impact of different dictionary sizes and qualities on performance is also warranted.
4. **Downstream Task Choices:** The chosen downstream tasks (document classification and dependency parsing) are relatively standard but may not fully capture the potential benefits of multilingual embeddings in applications like machine translation or cross-lingual information retrieval.



**Questions to Authors:**

1. **Dictionary Quality:** Could you provide a more detailed analysis of the quality of the dictionaries used, particularly those generated via Google Translate? What steps were taken to mitigate the potential for noise and bias introduced by machine translation? Did you observe significant performance variations based on the source/target language pair for these auto-generated dictionaries?
2. **Threshold Optimization:** Was any experimentation conducted with different values for the threshold τ used when extracting dictionaries from parallel corpora?  How sensitive is the performance to changes in this parameter?
3. **Scalability Bottlenecks:**  What were the primary computational bottlenecks when scaling the methods to 59 languages? Are there any optimizations that could be implemented to further improve scalability?
4. **Future Work on Embedding Methods:** Recognizing the incremental novelty of the embedding methods themselves, what future directions do you envision for developing more innovative multilingual embedding algorithms?




**Additional Comments:**

* The paper references a GitHub repository for the software but the URL is redacted. Please ensure this is included in the final version.
* Consider adding a discussion of the limitations of the evaluation metrics and potential biases.
* It would be helpful to include a table summarizing the resources (monolingual data size, dictionary size) available for each language.

- MDL-based framework learns multi-language, subset-specific string correspondence rules via a two-part code L(M)+L(D|M), shortest-path alignments, greedy merges, and optional data over-weighting (α>1).
- Experiments on Slavic languages produce linguistically plausible correspondences, NCD-based trees matching typology, and a four-way shared-description-length analysis.

- MDL formulation and deterministic greedy algorithm for multi-way, subset-aware correspondences without parametric assumptions.
- Shared Description Length (sDL) as an information-theoretic tool to quantify structure shared by any language subset.
- Graph-based exact alignment procedure enabling multi-language rule induction and a Slavic case study demonstrating the approach.

- Conceptual novelty: subset-aware MDL moves beyond pairwise models and provides fine-grained multi-language diagnostics.
- Deterministic, optimization-grounded discovery with explicit DL objective and exact shortest-path alignment avoids randomness in inference.
- Qualitative linguistic plausibility: recovered rules match expected phonological correspondences and script effects.
- Coherent phylogenetic signal: NCD distances recover canonical Slavic subgroupings and highlight known outliers.
- Practical alignment machinery: shortest-path formulation with partial order reduction and reasonable initial scalability claims.

- Evaluation gaps: no comparisons to competitive baselines, no intrinsic precision/recall, and no extrinsic task evaluation.
- MDL coding underspecified: enumerative “data-to-model” terms, prefix-freeness, zero-count handling, and Laplace correction are unclear.
- Training is not standard EM: greedy-only merges, no split/replacement moves, limited convergence guarantees, and possible initialization bias.
- Rule complexity control lacking: arbitrary-length rules risk overfitting, no principled regularization or minimum support, and α is ad hoc.
- Data/orthography issues: use of raw orthography and small datasets confound linguistic signal with script/notation and lack robustness analyses.

- Provide full derivations of all code terms, enumerative components, prefix-free guarantees, zero-count handling, and Laplace correction effects.
- Clarify computation of L(X,Y) for NCD and whether training separate compressors per pair invalidates NCD metric properties or requires cross-encoding.
- Specify the candidate merge space per M-step, any constraints on rule length or minimum type support, and merge-selection criteria.
- Report sensitivity to initialization, α, and alphabet normalization (transliteration/diacritics/digraphs) with ablations and bootstrap robustness checks.
- Share distributions of rule lengths and per-type coverage, plus runtime/memory scaling with dataset size and word length.
- Release code and data and detail any additional pruning heuristics (beam search, admissible A* bounds) beyond partial order reduction.

Summary
This paper proposes an MDL-based framework for discovering string-level correspondences across two or more languages, allowing rules to involve any subset of languages. The model class consists of sets of correspondence rules (tuples of strings over language-specific alphabets). A two-part code L(M)+L(D|M) is designed: L(M) encodes alphabets and the rule table (including counts via an enumerative “data-to-model” code), and L(D|M) encodes each cognate tuple as a sequence of rule applications. Given a rule table with usage costs, best alignments are found as shortest paths in a multi-dimensional index graph; training alternates between aligning data (E) and greedily merging rule pairs to reduce description length (M). The authors also introduce “over-weighting” of the data (α>1) to push the model toward longer patterns. Experiments on Slavic languages show: (i) pairwise NCD-based distances yield a tree consistent with typology; (ii) qualitative correspondences are linguistically plausible; (iii) a four-way analysis quantifies shared description lengths across all language subsets.

Main contributions (as I see them)
- An MDL formulation and deterministic greedy algorithm for discovering multi-way, subset-specific string correspondences without assuming parametric distributions or restricting to pairs of languages.
- An information-theoretic analysis tool: shared Description Length (sDL) to quantify structure shared by any subset of languages in a joint model.
- Empirical case study on Slavic languages demonstrating phylogenetic recovery and interpretable correspondences, and a practical graph-based exact alignment procedure for multi-language rules.

Strengths
- Conceptual novelty and generality: Allowing rules on arbitrary language subsets within a single MDL code is a clean way to move beyond pairwise models and to avoid strong parametric assumptions common in prior work (e.g., pair-HMMs, fixed edit processes, or Bayesian priors on change matrices). The subset-aware encoding and sDL measure provide fine-grained, multi-language similarity diagnostics not available in pairwise frameworks.
- Deterministic, optimization-grounded discovery: The objective is explicit (total description length), the search is deterministic, and the alignment step finds exact shortest paths under current costs. This avoids randomness in inference and clarifies what drives rule induction.
- Qualitative linguistic plausibility: Reported rules capture expected correspondences (e.g., cz–č, g–h, palatalizations, metathesis), and the four-way alignments sensibly mix subset-shared and language-specific patterns. The Cyrillic vs. Latin effect is picked up naturally.
- Coherent phylogenetic signal: NCDs yield a tree matching canonical Slavic subgroupings and highlight known outliers (Bulgarian/Macedonian morphology; Sorbian lexicon; Polish digraphs). This supports that the code is capturing meaningful regularities rather than artifacts.
- Practical alignment machinery: Casting alignment as a shortest-path problem with partial order reduction is well-motivated and reproducible, and the scalability claims (up to five languages on modest hardware) are reasonable for an initial study.

Weaknesses
- Evaluation lacks strong baselines and quantitative metrics: There is no comparison to competitive unsupervised correspondence/segmentation methods (e.g., PMI-based alignment, pair-HMM/CRP models, Bayesian sound change induction, or Morfessor-style subword alignment). The phylogeny result is shown, but not contrasted against simple baselines (e.g., Levenshtein distances on orthography, PMI-based NCD, or transliteration + edit models). No intrinsic precision/recall against a gold set of correspondences, nor extrinsic tasks (cognate projection, prediction on held-out pairs).
- Methodological clarity and correctness of coding terms: Several parts of L(M) are underspecified or potentially inconsistent. For example, the “data-to-model” components: (a) L(Π) includes LN(TΠ)+log((TΠ−1 choose |Π|−1)) for rule usage counts—derivation should be made explicit (is this an enumerative code for weak compositions over |Π| categories?), and how this interacts with Laplace correction; (b) L(Σi) similarly uses LN(TΣi)+log((TΣi−1 choose |Σi|−1)), but the text says “none of the symbols … will have a non-zero count,” which seems to be a typo. It remains unclear whether the code is prefix-free and whether all components satisfy Kraft’s inequality jointly; the current presentation makes it hard to verify MDL soundness.
- “EM” characterization and search guarantees: The training is not an EM algorithm in the standard sense; the M-step performs greedy structural merges rather than maximizing a likelihood (or minimizing a convex surrogate) over fixed structure. There are no convergence guarantees beyond monotonic DL decrease, and the approach can get stuck because only merges are considered—no splits or replacement moves. The dependency on initialization (singleton rules vs. seeding with given rules) suggests potential bias; no ablation on initialization is reported.
- Scalability and rule control: Allowing arbitrary-length rules leads to stems being treated as correspondences. While this yields neat examples, it risks overfitting to lexical idiosyncrasies, especially on small Swadesh data. There is no mechanism to constrain rule length, enforce minimum lexical support across types, or regularize by boundary penalties; “over-weighting” α>1 is ad hoc and undermines MDL’s objectivity, and its impact on downstream analyses (e.g., NCD, sDL) is not quantified.
- Orthography confounds and data limitations: Experiments use raw orthographic forms, not normalized or IPA transcriptions. Differences in script and diacritics directly affect alphabet sizes and code lengths (e.g., Czech long vowels), confounding linguistic distance with orthographic conventions. The datasets are small (Swadesh lists and a small set of internationalisms), so robustness (e.g., bootstrap over items, confidence intervals for NCD) is not assessed.

Questions to Authors
- Please provide a full derivation of all code terms, especially the enumerative “data-to-model” parts in L(Π) and L(Σi). Are these codes prefix-free and jointly valid? How are zero-count symbols handled, and how does Laplace correction interact with the two-part code?
- How exactly is L(X,Y) computed for NCD? Is a single joint model trained on concatenated data of X and Y, and L(X,X) via a model trained only on X? If so, the compressor changes across pairs, which can invalidate NCD’s metric properties. Have you tried a fixed universal model class or cross-encoding to control for this?
- What candidate merge space is explored per M-step? Only merges of rules that co-occur adjacently in alignments, or any pair of rules? Do you enforce a maximum rule length or minimum type support? Can you report the distribution of rule lengths and their per-type coverage?
- How sensitive are results to initialization (null vs. seeded rules), α (data over-weighting), and to alphabet normalization (e.g., transliteration, removal of diacritics, mapping digraphs)? Can you include ablations and robustness checks (e.g., bootstrap trees)?
- Can you release code and data, and provide runtime/memory as a function of N, average word length, and rules? Any pruning heuristics (beam, A* with admissible bounds) beyond partial order reduction?

Overall, this work proposes an interesting and general MDL framework for multi-language correspondences with promising qualitative results. Strengthening the formal presentation of the code, adding principled controls on rule complexity, and providing quantitative comparisons and robustness analyses would substantially improve the paper’s impact and credibility.

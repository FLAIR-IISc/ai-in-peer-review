Summary
The paper proposes an MDL-based framework to discover statistically salient string-level correspondences among two or more languages without assuming a parametric form for correspondences. It defines a concrete two-part code over alphabets and a table of correspondence rules, uses shortest-path alignment to encode data under a model, and employs an EM-like iterative procedure with greedy rule-merging to learn longer correspondences from singleton rules. The method supports multi-language (N>2) correspondences and introduces a “data over-weighting” knob to control the granularity of discovered rules. Experiments on Slavic languages present pairwise NCD-based phylogeny and a four-language analysis using shared description lengths, along with qualitative examples of learned correspondences.

Main contributions (as assessed)
1) An MDL formulation and deterministic algorithm for discovering correspondence rules across multiple languages simultaneously, with a concrete two-part code (PS-2) and a shortest-path alignment procedure enabling EM-style optimization.
2) A methodology to quantify multi-language similarity via “shared Description Lengths” over any subset of languages, enabling analyses beyond pairwise measures, and an application to Slavic languages.
3) A practical lever (data over-weighting α) to modulate rule granularity, demonstrating how near-significant correspondences can be surfaced when data are scarce.

Strengths
- Principled, assumption-light modeling via MDL: The approach defines an explicit code for models and data, avoiding strong parametric assumptions about correspondence shape or distribution. The two-part code, data-to-model coding for symbol and rule usage distributions, and reliance on MDL regularization are well-aligned with goals of objectivity and interpretability.

- Supports multi-language correspondences and subset-specific rules: The model allows rules that involve any subset of the N languages, which is uncommon in prior work mostly targeting pairs. The four-way examples show rules that naturally engage only 2–3 languages when evidence for a 4-way rule is insufficient, consistent with the stated MDL criterion.

- Deterministic learning with interpretable growth from singleton rules: Initialization from singleton rules and greedy merging guided by description length decrease provide a clear, reproducible path to longer correspondences. The ability to seed with known rules enables integrating linguistic knowledge in a controlled way.

- Empirical alignment with known linguistic structure: Pairwise NCDs over 13 Slavic languages produce a phylogeny that matches established groupings and known outliers (e.g., BG/MK), giving face validity. The multi-way shared DL analysis highlights plausible groupings (CS–PL and RU–BG) and asymmetries (Czech’s higher overhead), and offers a more nuanced view than pairwise distances alone.

- Clear articulation of computational procedures: The alignment as a shortest-path problem with partial order reduction, and the stated complexities for E and M steps, make the algorithmic choices transparent. Qualitative examples (e.g., cz–č, g–h, palatalizations) illustrate that the model captures meaningful orthographic/phonological patterns.

Weaknesses
- Limited quantitative evaluation and lack of baselines: The paper provides no comparison against established approaches in cognate/phonological correspondence discovery (e.g., PMI-based sound correspondences, PHMMs/ALINE, Bayesian models for cognates/morphology). There is no held-out evaluation, intrinsic accuracy metrics, or human-validated gold correspondences to quantify precision/recall or downstream utility.

- Scalability and computational limits: The E-step complexity O(|D|·|R|^2) and M-step O(|D|·A^2/2), together with memory bottlenecks, restrict experiments to up to five languages in a few hours on modest hardware. There is little analysis of how |R| and A grow with N, corpus size, or alphabet size, and no heuristics/pruning guarantees. This raises concerns about applicability to larger multilingual settings or richer corpora.

- Orthography-driven analysis risks confounding: Working directly on raw orthographic forms mixes script, orthography, and phonology. The observation that SC in Latin vs Cyrillic shifts perceived similarity underscores this confound. Without transliteration or phonemic normalization, the learned correspondences and distances may partially reflect script choices rather than linguistic sound change or morphology.

- Insufficient rigor in some coding details: Certain parts of the code specification are under-explained. For L(Π), the inclusion and role of LN(TΠ) and the weak number composition term to derive optimal usage code lengths are stated but not derived; similarly for alphabet unigrams. The reuse of code(σ) and code(π) across contexts causes ambiguity. The claim that a longer useful pattern implies subpatterns are “at least as useful” is intuitive but not justified; counterexamples can arise under MDL when coding costs interact with frequency.

- Over-weighting undermines MDL objectivity without guidance: Introducing α to up-weight L(D|M) is practical but departs from the stated objective criterion. There is no principled guidance for selecting α, no sensitivity analysis, and a risk of overfitting or biasing toward longer rules. The paper acknowledges subjectivity but does not bound its effects.

- Reproducibility and presentation issues: Data details are sparse (e.g., exact list sources/versions, preprocessing, handling of diacritics), code is not provided, and references are inconsistently formatted with missing bibliographic fields. Some notation and explanations (e.g., “data-to-model code,” subset enumeration, treatment of ε entries) would benefit from tighter definitions.

Questions to Authors
- Can you provide a formal derivation for the data-to-model coding terms in L(Π) and L(Σi), including the exact enumeration underlying the weak number composition and how code(π) is computed from counts?

- What constraints (if any) are placed on rule lengths, and how do you prevent combinatorial explosion in candidate merges? Do you use pruning heuristics, and can you characterize approximation guarantees?

- How sensitive are results to Laplace smoothing, initialization (null vs seeded), and the merging schedule? Do you have convergence diagnostics or multiple-start experiments?

- How do you select the over-weighting parameter α in practice? Can you provide a sensitivity analysis showing stability of key findings across α?

- Can you disentangle script effects by transliteration or phonemic normalization, and does the phylogeny remain consistent under such normalization?

- Could you include comparisons to baseline methods and, if possible, an evaluation against a small gold set of expert correspondences to quantify accuracy?

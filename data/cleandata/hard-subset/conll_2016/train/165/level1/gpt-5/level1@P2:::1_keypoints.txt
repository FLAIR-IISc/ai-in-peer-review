- Proposes an MDL-based framework to discover multi-language string correspondences with a two-part code, shortest-path alignment, greedy rule merging, and a data over-weighting parameter.
- Claims contributions: (1) general MDL formulation for multiway rules, (2) information-theoretic toolkit (pairwise NCD, shared description length), (3) practical data over-weighting knob for rule granularity.
- Conceptual novelty: discovers correspondences across arbitrary language subsets without strong parametric assumptions.
- Clear, principled code design separating model and data and using Shannon-optimal/universal integer codes.
- Deterministic, interpretable learning: E-step via shortest-path alignment and M-step via rule merges producing linguistically meaningful rules.
- Empirical results qualitatively align with linguistic expectations (Slavic tree, script effects, subset sharing).
- Supports analyses beyond pairwise methods, quantifying shared structure among subsets via sDL.
- Limited empirical validation: small datasets (207 Swadesh items, 778 entries) and descriptive outcomes only.
- No quantitative baselines or comparisons to established aligners, distance measures, or cognate models.
- No gold-standard evaluation of induced rules (precision/recall) or held-out generalization tests.
- Scalability concerns: high theoretical complexity, reported “few hours” and memory bottlenecks, no pruning/approximation analysis or scaling curves.
- Heuristic M-step (greedy pair merges) lacks proofs of monotonic MDL decrease or convergence guarantees; potential sensitivity to early choices.
- Interaction of smoothing, rule re-estimation, and codes not analyzed for consistency.
- Data over-weighting (α) departs from MDL objectivity with no principled selection method or overfitting assessment.
- Orthography confounds: reliance on raw orthographic forms risks script/diacritic-driven artifacts without phonetic/transliteration robustness checks.
- Requests quantitative comparisons for alignment quality and phylogeny (e.g., quartet distance) and to simple baselines (edit distance, n-grams).
- Asks for details on M-step merge enumeration, scoring, termination criteria, and monotonicity guarantees.
- Requests scaling experiments (runtime/memory vs languages, dataset size, alphabet) and any pruning heuristics.
- Wants sensitivity analyses for initialization, Laplace smoothing, and α, and a data-driven method to set α (e.g., held-out L(Dtest|Mtrain)).
- Suggests experiments with phonetic transcriptions or unified transliteration and quantification of script effects.
- Requests code and processed dataset release for reproducibility.
- Overall assessment: well-motivated MDL framework with plausible qualitative results but limited by lack of quantitative baselines, scalability analysis, and robustness experiments.

Summary and claimed contributions
This paper proposes an MDL-based framework to automatically discover systematic correspondences between strings across two or more languages, allowing rules to be defined over arbitrary subsets of languages. The authors design a two-part MDL code for models and data, derive a deterministic EM-style learning procedure that alternates optimal alignment (via shortest paths) and greedy rule merging to reduce total description length, and introduce a “data over-weighting” parameter to control the granularity of discovered rules. Experiments on Slavic languages show that the method (i) induces interpretable correspondences, (ii) yields pairwise NCDs that recover a plausible phylogeny, and (iii) supports multi-language analyses via a shared description length (sDL) decomposition.

In my view, the main contributions, in order of strength, are:
1) A general MDL formulation and deterministic algorithm for discovering multiway correspondence rules defined on arbitrary subsets of languages, with an explicit code design and alignment-as-shortest-path inference.
2) An information-theoretic analysis toolkit for multi-language similarity, including pairwise NCDs and “shared description length” over language subsets, demonstrated on Slavic languages with plausible qualitative findings.
3) A practical knob (data over-weighting) to control rule granularity when statistical evidence is borderline, enabling linguists to trade objectivity for detail when desired.

Strengths
- Conceptual and methodological novelty for multi-language correspondences without strong parametric assumptions: The model class is the set of tupled character strings over any subset of languages; the two-part code clearly separates model and data, and avoids priors over rule shapes. This generality is rare in prior work focused on pairwise alignments or specific distributions, and directly supports the paper’s goal of discovering cross-language patterns beyond pairs.
- Clear and principled code design: The model code transmits alphabets (with data-to-model codes), the rule table partitioned by language subset, and per-rule strings with universal integer codes for lengths and Shannon-optimal unigrams. The data code transmits alignments via best-usage codes. These choices are well-motivated in MDL terms and make the objective reproducible and auditable.
- Deterministic learning with interpretable building blocks: The E-step uses optimal alignment via a shortest-path formulation; the M-step merges rule pairs that most reduce total description length. This produces rules that grow in statistical significance and gives transparent, linguistically interpretable outputs (e.g., cz↔č, g↔h, metathesis ла↔оло, endings ć↔t). The ability to seed with known rules allows incorporation of expert knowledge.
- Empirical evidence aligns with linguistic expectations: Pairwise NCDs over 13 Slavic languages produce a tree consistent with standard groupings, including sensible outliers (BG/MK due to morphology; Polish and Sorbian for orthography/etymology). The method also captures script effects (Serbo-Croatian Latin vs Cyrillic), underscoring sensitivity to orthography, which the authors discuss. Multi-language sDL reveals strong CS–PL and RU–BG sharing and higher idiosyncrasy for Czech, again plausible given diacritics.
- Scalable beyond pairs and supports subset analyses: The four-way analysis shows rules naturally defined on subsets and quantifies shared structure among all subsets via sDL—a capability that pairwise-only methods lack and that can enable richer comparative studies.

Weaknesses
- Limited empirical validation and no baselines: The evaluation is confined to small, specialized datasets (207 Swadesh items; 778 entries for four languages) and descriptive outcomes (trees, example rules, sDL charts). There is no quantitative comparison to established baselines for sound/orthographic correspondences, cognate alignment, or phylogeny (e.g., pair-HMM/ALINE/Needleman–Wunsch-based aligners, Snyder & Barzilay-style models, alternative distance measures). Nor is there a gold evaluation of rule precision/recall or a held-out generalization test of L(Dtest|Mtrain). This makes it hard to assess gains due to the MDL formulation versus simpler methods.
- Scalability and computational costs: Alignment graphs can be large; E-step complexity is O(|D|·|R|^2) and M-step O(|D|·A^2/2). The authors report only “a few hours” on a single core for up to five languages and note memory bottlenecks. There is no pruning/beam/approximation analysis, nor empirical scaling curves vs number of languages, alphabet size, or dataset size. This limits applicability to larger datasets or higher N and leaves open whether the approach can be routinely used beyond small lexicons.
- Heuristic M-step and convergence properties: The greedy merge-of-two-rules update lacks a proof of monotonic MDL decrease across an entire EM iteration or guarantees of convergence to a (good) local optimum. The interaction of Laplace smoothing, rule usage re-estimation, and the data-to-model codes is not analyzed for consistency. The paper acknowledges that subtle correspondences can be missed depending on earlier choices, but provides no diagnostics or remedial strategies beyond hoping evidence accumulates elsewhere.
- Data over-weighting trades objectivity for control without principled guidance: While practical, the α parameter departs from MDL’s objective criterion; there is no method to select α, assess overfitting, or quantify the bias introduced. The demonstration shows alignment changes but not whether the induced larger rules are linguistically superior or more predictive (e.g., on held-out data). A more principled multi-objective formulation or model selection could strengthen this aspect.
- Orthography-driven evaluation risks confounds: All data are raw orthographic forms, and effects like script differences (Latin vs Cyrillic) and diacritics appear to drive parts of the results. While noted, there is no robustness check with phonetic transcriptions or consistent transliteration to disentangle orthographic conventions from phonological correspondences, potentially inflating or deflating cross-language similarity estimates.

Questions to authors
- Can you provide quantitative comparisons to baseline methods for (i) pairwise alignment quality and (ii) phylogeny (e.g., quartet distance to a reference tree), and/or to simpler distance measures (edit distance, character n-gram cosine)?
- How are candidate rule merges enumerated and scored efficiently in the M-step? What are the termination criteria? Do you guarantee a monotonic decrease in total description length per merge?
- Can you report scaling experiments showing runtime/memory vs number of languages, dataset size, and alphabet size, and any pruning/heuristic that preserves quality?
- How sensitive are results to initialization (null vs seeded rules), Laplace correction strength, and α (data over-weighting)? Can you propose a data-driven way to set α (e.g., via held-out L(Dtest|Mtrain))?
- Could you include experiments with phonetic transcriptions or unified transliteration to assess robustness to orthography? Conversely, if orthography is a feature, can you quantify script effects separately?
- Will you release code and processed datasets to facilitate reproducibility?

Overall, the paper introduces a well-motivated MDL framework with appealing multi-language capabilities and produces linguistically plausible qualitative results. However, the lack of quantitative baselines, scalability analysis, and robustness experiments limits confidence in the practical impact. I encourage the authors to strengthen the evaluation and clarify algorithmic properties.

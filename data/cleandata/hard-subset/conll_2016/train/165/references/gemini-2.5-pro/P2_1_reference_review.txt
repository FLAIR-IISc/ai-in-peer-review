### Summary and Contributions

This paper presents a method for discovering systematic string correspondences between multiple languages using the Minimum Description Length (MDL) principle. The core idea is to find the set of correspondence rules that provides the most compact description of a given list of cognate words. The method is designed to handle more than two languages simultaneously without making prior assumptions about the shape or distribution of the correspondences. The authors propose a deterministic, EM-like algorithm to infer these rules, starting from simple single-character rules and greedily merging them to form more complex ones. The contributions of this work, as I see them, are:

1.  **A Principled Framework for N-Way Correspondence Discovery:** The primary contribution is the MDL-based model for simultaneously learning correspondence rules across an arbitrary number of languages (N>2). This is a significant extension of common pairwise approaches. As part of this, the paper introduces the "shared Description Length" (sDL) as a novel, information-theoretic metric to quantify the similarity between any subset of the languages being analyzed.
2.  **A Deterministic Inference Algorithm:** The paper details a practical, EM-like algorithm for optimizing the MDL objective. The M-step, which greedily merges the two rules that yield the greatest compression, makes the process deterministic and efficient, even if suboptimal.
3.  **A Pragmatic "Data Over-Weighting" Technique:** The authors introduce a weighting parameter `α` to deliberately move away from a purely objective MDL model. This allows a user (e.g., a linguist) to control the granularity of the discovered rules, forcing the model to find patterns that may be linguistically interesting but not statistically significant in a small dataset.

### Strengths

1.  **Methodological Soundness and Novelty:** The use of MDL is well-motivated for this task, as it provides a formal way to balance model complexity (the set of rules) with data fit, thus avoiding overfitting and the need for arbitrary assumptions. The extension of this idea to the N-language case is a clear and important contribution. The resulting `sDL` metric provides a much richer analysis of language relationships than a single phylogenetic tree, as it can quantify the shared structure of arbitrary subgroups (e.g., CS-PL vs. RU-BG vs. CS-PL-RU-BG).
2.  **Strong Empirical Validation on a Relevant Task:** The experiments on Slavic languages are convincing. The pairwise analysis using Normalized Compression Distance (NCD) successfully reconstructs a phylogenetically sensible tree (Figure 1), which serves as a strong sanity check and validation of the model's ability to capture linguistic distance. The authors' linguistic interpretation of the tree, noting the outlier status of languages like Bulgarian/Macedonian and Polish, adds credibility to the results.
3.  **High-Quality, Interpretable Results:** The discovered correspondence rules shown in the paper (e.g., Polish `cz` vs. Czech `č`, or the Slavic liquid metathesis `оло`/`ла`) are linguistically correct and demonstrate that the model is learning meaningful patterns. The four-way analysis (Figure 3) provides a compelling visualization of subgroup similarities, correctly identifying the West Slavic (CS-PL) and East/South Slavic (RU-BG) pairings as the strongest subgroups within the four-language set.
4.  **Practical Utility for Linguists:** The deterministic nature of the algorithm ensures reproducibility, which is a desirable property for scientific investigation. Furthermore, the data over-weighting mechanism, while compromising pure MDL objectivity, is a very practical feature. It empowers a linguistic expert to explore "near-significant" patterns and tune the model's output to a desired level of detail, making it a potentially powerful tool for exploratory analysis.

### Weaknesses

1.  **Greedy and Potentially Suboptimal Optimization:** The M-step of the algorithm is greedy, always merging the single best pair of rules. This can easily lead to local optima. For instance, merging two rules `A` and `B` might be the best single move, but it could prevent a future, globally better merge involving `A` and some other rule `C`. The authors acknowledge this limitation but do not analyze its impact. A comparison with a stochastic approach (like the simulated annealing mentioned in the conclusion) or a beam-search-based M-step would be necessary to understand how much performance is left on the table.
2.  **Significant Scalability Concerns:** A major claim is the ability to handle multiple languages, but the paper admits the implementation is limited to "up to five languages" due to memory and runtime bottlenecks on a standard machine. This is a serious practical limitation. The complexity analysis provided is abstract (`O(|D|·|R|^2)`). A more insightful analysis of how complexity scales with the number of languages `N` is needed. It seems likely that the search space for alignments (the shortest-path problem) grows exponentially with `N`, which would severely restrict the method's applicability.
3.  **Lack of Comparison to Alternative Methods:** While the paper argues against the limitations of prior work, it does not include any quantitative comparison. Even if other methods are pairwise, it would be valuable to compare the quality of the learned pairwise rules (e.g., CS-PL) from the proposed N-way model against rules learned by a state-of-the-art pairwise system like Snyder and Barzilay (2008). This would help situate the contribution and assess whether the multi-language modeling comes at a cost to pairwise accuracy.
4.  **Tension between Objectivity and Subjectivity:** The paper initially frames the MDL approach as a way to "forego making assumptions" and achieve "objective results." It then introduces the `α` parameter for data over-weighting, which is a subjective, user-defined choice that directly contradicts the initial premise of objectivity. While this is a practical feature, the paper could do a better job of framing this trade-off explicitly rather than presenting the method as purely objective and then introducing a subjective modification.

### Questions to Authors

1.  **Regarding Scalability:** Can you provide more detail on the specific bottleneck that limits the method to ~5 languages? Is it the memory required to build the alignment graph, or the runtime of Dijkstra's algorithm on it? Have you considered approximate alignment algorithms that might trade optimality for the ability to scale to a larger number of languages?
2.  **Regarding the Greedy M-Step:** The paper states that if a suboptimal alignment choice "loses" evidence for a rule Q, Q might still be learned if there is enough other evidence for it. Have you performed any experiments to quantify how often this recovery happens or how sensitive the final rule set is to the initial greedy choices? For example, does introducing a small amount of randomness in the merge step lead to significantly different final models?
3.  **Regarding Data Over-Weighting:** The `α` parameter seems powerful but also arbitrary. How does a user choose an appropriate value? The paper mentions increasing it "until there are no more singleton rules," but this seems like one specific, potentially extreme heuristic. Is there a more principled way to guide the selection of `α` to achieve a certain desired rule granularity?

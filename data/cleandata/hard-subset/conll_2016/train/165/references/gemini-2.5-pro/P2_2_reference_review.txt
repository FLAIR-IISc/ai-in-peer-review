### Summary and Contributions

This paper presents a method for discovering orthographic correspondence rules between multiple languages from cognate lists. The core of the approach is the Minimum Description Length (MDL) principle, which frames the problem as finding the set of rules that most compactly describes the given data. The authors propose a deterministic, iterative algorithm that begins with a "null alignment" (single characters) and greedily merges rules to improve the overall compression (model description + data description). The method is notable for its ability to handle more than two languages simultaneously and for avoiding prior assumptions about the shape or distribution of correspondence rules.

As I see them, the main contributions are:
1.  **A novel, multi-language framework for correspondence discovery.** The paper's primary contribution is the formulation of this problem in an MDL setting that is inherently capable of handling more than two languages. This is a significant step beyond common pairwise approaches. The introduction of "shared Description Length" (sDL) as a metric to quantify n-way similarity is a direct and valuable outcome of this framework.
2.  **A practical, deterministic algorithm for inferring correspondence rules.** The proposed EM-like procedure, which iteratively merges the most compressive rule pairs, provides a concrete and reproducible method for finding good rule sets. The initialization from a "null alignment" is an intuitive way to build up complex rules from simple components based purely on statistical evidence in the data.
3.  **The "data over-weighting" technique for user-controlled granularity.** While a departure from pure MDL objectivity, this is a clever, pragmatic contribution that allows a user (e.g., a linguist) to explore "near-significant" rules and control the level of detail in the discovered correspondences, addressing potential data sparsity issues.

### Strengths

1.  **Principled and Assumption-Light Framework:** The use of the MDL principle is a major strength. It provides a solid, information-theoretic foundation for model selection that avoids arbitrary assumptions about the structure of correspondences (e.g., 1-to-1 mappings, specific length distributions). This allows the statistical regularities to emerge directly from the data, lending objectivity to the results.
2.  **Novel and Insightful Multi-Language Analysis:** The paper successfully demonstrates a key advantage over pairwise methods. The four-way analysis using the sDL metric (Figure 3) is particularly compelling. It provides a much richer and more nuanced quantification of language relationships than a simple phylogenetic tree, showing not just the primary CS-PL and RU-BG groupings but also the relative complexities shared between all possible subsets of the four languages. This is a powerful new tool for computational historical linguistics.
3.  **Clear and Verifiable Experimental Results:** The experiments are well-executed and the results are convincing. The pairwise NCD analysis produces a phylogenetic tree (Figure 1) that aligns well with established linguistic classifications of the Slavic languages, serving as an excellent sanity check. Furthermore, the example alignments presented in Tables 3 and 4 are linguistically plausible and effectively illustrate both the kinds of rules discovered and the utility of the data over-weighting feature.
4.  **Reproducibility and Interpretability:** The proposed algorithm is deterministic, which is a significant advantage for reproducibility. The step-by-step merging process, guided by a clear objective function (minimizing description length), makes the model's "reasoning" transparent. The learned rules themselves are directly interpretable as string-to-string mappings.

### Weaknesses

1.  **Significant Scalability Concerns:** The paper candidly mentions that the current implementation can process "up to five languages" and notes bottlenecks in memory and runtime. This is a major limitation for the method's practical application to larger language families (e.g., Romance, Germanic). The complexity analysis is brief, and it is unclear how severely the combinatorial nature of the alignment graph grows with the number of languages N. This weakness limits the scope of the "multi-language" claim and should be addressed for the work to have broader impact.
2.  **Lack of Comparative Evaluation:** The method is evaluated in isolation. While the results are sensible, the paper provides no comparison against other statistical methods for discovering correspondences or phonetic alignments, such as those cited by the authors (e.g., Snyder and Barzilay, 2008). Without a baseline, it is difficult to assess whether the MDL approach produces higher-quality alignments or more linguistically relevant rules than existing techniques. A comparison on a shared task or dataset would significantly strengthen the paper's claims.
3.  **Dependency on Pre-aligned Cognate Data:** The approach requires a list of cognates as input. This is a strong prerequisite, as identifying cognates is a challenging task in itself. The paper discovers correspondences *within* known cognate sets but does not contribute to the upstream problem of cognate identification. This scope limitation should be more clearly stated, as it makes the tool dependent on significant prior manual labor or a separate automated cognate detection system.
4.  **Clarity of the MDL Formulation:** While the overall concept is clear, the specific details of the MDL code in Section 4 are dense and could benefit from more intuition or a small, worked example. For instance, the data-to-model code for encoding rule counts is presented formally but without a clear explanation of why this specific formulation is chosen over others. This may make the technical core of the paper difficult to follow for readers not already expert in advanced MDL techniques.

### Questions to Authors

1.  Could you elaborate on the scalability bottleneck? Specifically, how does the size of the alignment graph (and thus the runtime of Dijkstra's algorithm) scale with the number of languages N? Are there promising avenues for making this more tractable, for example, through beam search or other pruning heuristics during alignment?
2.  The data over-weighting parameter `α` is a pragmatic and interesting feature. For the results in Table 3, how was the value of `α` (or the equivalent level of over-weighting) determined? Was it chosen to achieve a specific outcome (e.g., no singleton rules), or is there a more principled way to set this parameter depending on the research goal?
3.  While a full experimental comparison may be out of scope, could you theoretically position your work more directly against related approaches like multi-view learning for morphology (e.g., Snyder and Barzilay, 2008)? What are the key advantages of the non-probabilistic, compression-based MDL approach compared to a probabilistic graphical model that could, in principle, also be extended to multiple languages?

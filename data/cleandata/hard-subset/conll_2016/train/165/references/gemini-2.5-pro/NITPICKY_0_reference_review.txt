### Summary
This paper proposes a method for discovering systematic string-level correspondences between multiple languages using the Minimum Description Length (MDL) principle. The core idea is to find the set of correspondence rules that most succinctly describes a given list of cognate tuples. The authors formulate this as an optimization problem and propose a deterministic, EM-like algorithm to find a good set of rules. The method is evaluated on a set of 13 Slavic languages. The authors perform a pairwise analysis to construct a phylogenetic tree, which aligns with known linguistic classifications, and a four-way analysis (Czech-Polish-Russian-Bulgarian) to demonstrate a more fine-grained quantification of similarity between language subsets, which they term "shared Description Length" (sDL).

### Contributions
1.  **A Multi-Language Correspondence Model using MDL:** The primary contribution is the formulation of multi-language (N > 2) correspondence discovery as a model selection problem under the MDL principle. This provides a principled, non-parametric framework for this task.
2.  **A Deterministic Search Algorithm:** The paper presents a deterministic, EM-like iterative algorithm for searching the space of correspondence rules. The algorithm starts with singleton rules and greedily merges them to find more complex rules that improve the overall description length.
3.  **Shared Description Length Metric:** The paper introduces the concept of "shared Description Length" (sDL) as a tool to quantify the amount of shared structure among any subset of the languages being analyzed, offering a more granular analysis than pairwise distance metrics like NCD.

### Strengths
1.  **Principled Framework:** The use of the MDL principle is well-motivated for this task, as it avoids making strong prior assumptions about the distribution or shape of correspondence rules and provides a natural way to balance model complexity with data fit. This is a strong theoretical foundation.
2.  **Multi-Language Capability:** The method's ability to handle more than two languages simultaneously is a significant advantage over many existing approaches, which are often limited to pairwise comparisons. The resulting n-way analysis of similarity, as demonstrated in Section 5.3, is insightful and showcases the potential of this approach for detailed quantitative linguistics.
3.  **Validation through Phylogeny:** The pairwise experiments produce a phylogenetic tree for Slavic languages that largely corresponds to established linguistic groupings. This provides solid external validation that the learned models and the derived NCD metric are capturing meaningful linguistic similarities.

### Weaknesses
1.  **Lack of Clarity and Precision:** The paper suffers from a significant number of clarity issues that hinder reproducibility and trust in the method.
    *   The model name, "PS-2 MDL Code," is introduced without any explanation of what "PS-2" means.
    *   The complexity analysis is vague. The variables `R` in the E-step (`O(|D| · |R|^2)`) and `A` in the M-step (`O(|D|·A^2/2)`) are defined as "maximum number of possible rule applications" and "maximum number of rules used to align," respectively. These definitions are insufficient; their relationship to input parameters like cognate length or rule table size is not explained, making the complexity claims difficult to interpret.
    *   The central intuition for the M-step (point 2 in Section 2) is presented without formal justification and seems questionable. It is not necessarily true that sub-patterns of an optimal pattern will occur "at least as or more" often.
    *   The description of the model code `L(M)` is dense. The use of data-to-model codes (the binomial coefficient terms) is not adequately explained for readers not deeply familiar with this specific MDL formulation.

2.  **Methodological Inconsistencies:** The paper's core claim of providing an "objective, unbiased" analysis is undermined by the introduction of the "Data Over-Weighting" technique in Section 4.5. This introduces a hyperparameter `α` which explicitly breaks the MDL principle by arbitrarily weighting the data term. While pragmatically useful, this is a subjective intervention that contradicts the paper's main philosophical stance. The authors do not provide any principled guidance on how to set this parameter.

3.  **Sub-optimality of the Search Procedure:** The maximization step employs a greedy, deterministic search by merging the two patterns that yield the highest immediate gain. This approach is highly susceptible to local optima. The authors briefly acknowledge this ("we may miss some of the more subtle correspondence rules") but do not analyze the severity of this problem or compare their deterministic approach to stochastic alternatives (e.g., simulated annealing, which they mention only in the conclusion as a possibility).

4.  **Poor Presentation and Formatting:** The paper is marred by a substantial number of formatting errors, typos, and inconsistencies, which suggests a lack of careful preparation.
    *   The introductory example of "ortho-phonetic correspondences" is mentioned, but the actual example is missing from the text.
    *   Section numbering is inconsistent (e.g., sections `.1`, `.2` in Section 4).
    *   Table captions use "T #" (e.g., "T 1") while figure captions use "F #" (e.g., "F 1"). Standard practice is "Table #" and "Figure #".
    *   There are typos, such as "and and SCc" in the NCDs section.
    *   Figure 3 is of poor quality, with an unreadable y-axis label (`0,000`) and cramped x-axis labels.
    *   The "References" section is catastrophically formatted, with stray question marks, inconsistent styles, and incomplete entries, making it nearly unusable. This level of carelessness is unacceptable for a scientific publication.

### Questions to Authors
1.  Could you please clarify the meaning of "PS-2" in your model's name?
2.  The greedy merging strategy in your M-step is a key component of the algorithm. Can you provide a more formal justification for this heuristic and elaborate on the potential impact of local optima on the quality of the discovered rules? Have you experimented with any non-greedy or stochastic search methods?
3.  Regarding the complexity analysis in Sections 4.3 and 4.4, could you please define the variables `R` and `A` more formally in terms of basic input parameters (e.g., cognate length, alphabet size, number of languages)?
4.  How do you reconcile the claim of providing an "objective" method with the introduction of the `α` parameter for "data over-weighting"? This seems to re-introduce the exact kind of subjective bias that MDL is meant to avoid. What principled guidelines can you offer for its use?

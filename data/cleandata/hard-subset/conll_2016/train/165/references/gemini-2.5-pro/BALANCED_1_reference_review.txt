### Summary and Main Contributions

This paper introduces a method for automatically discovering string-level correspondences between multiple languages from lists of cognates. The approach is framed under the Minimum Description Length (MDL) principle, which posits that the best model (here, a set of correspondence rules) is the one that provides the most succinct description of the data. The authors propose a deterministic, EM-like algorithm to infer these rules without making prior assumptions about their shape or distribution. The method is evaluated on a dataset of 13 Slavic languages. The main contributions of this work, as I see them, are:

1.  **A principled MDL framework for multi-language correspondence discovery.** The key strength and novelty lie in extending correspondence discovery beyond language pairs to an arbitrary number of languages simultaneously. This allows for a more holistic analysis of language families.
2.  **A novel metric, "shared Description Length" (sDL), for fine-grained linguistic similarity analysis.** Derived from the MDL framework, sDL provides a quantitative measure of shared structure among any subset of the analyzed languages, offering more detailed insights than traditional phylogenetic trees.
3.  **An effective, deterministic algorithm for rule inference.** The proposed EM-like procedure, which iteratively aligns data and greedily merges rules to improve the model, is a practical and well-explained method for tackling the non-convex optimization problem.

### Strengths

1.  **Methodological Soundness and Generality:** The use of MDL is well-motivated for this task, as it provides a formal, assumption-free way to balance model complexity with data fidelity. The ability to handle more than two languages is a significant advantage over most existing methods, which are confined to pairwise comparisons. This generality opens the door to more nuanced analyses of language families, as demonstrated in the four-way analysis.
2.  **Convincing Experimental Validation:** The experiments effectively demonstrate the method's utility. The pairwise analysis, culminating in a phylogenetic tree of Slavic languages (Figure 1), successfully reproduces established linguistic classifications. This serves as a strong external validation of the underlying similarity metric (NCD). The subsequent four-way analysis compellingly showcases the unique power of the method, revealing subgroup similarities (e.g., CS-PL vs. RU-BG) and cross-group influences that a simple tree structure cannot capture.
3.  **High Interpretability and Practical Features:** The learned correspondence rules shown in the paper (Table 3, Table 4) are linguistically plausible and interpretable, ranging from simple phonetic/orthographic alternations to more complex morphological patterns. The proposed "data over-weighting" mechanism (Section 4.5) is a thoughtful and practical feature that allows a user (e.g., a linguist) to explore less statistically prominent patterns, effectively turning the tool into an exploratory aid.
4.  **Clarity of Presentation:** The paper is well-written and clearly explains the technical details of the MDL code and the optimization procedure. The motivation is clear, and the connection between the MDL objective and the algorithmic steps is easy to follow.

### Weaknesses

1.  **Lack of Quantitative Comparison with Baselines:** The paper does not compare its results quantitatively against any alternative methods. While the multi-language setting is novel, baselines are conceivable. For instance, one could run a state-of-the-art pairwise method on all pairs and aggregate the results, or adapt multi-sequence alignment algorithms from bioinformatics. Without such a comparison, it is difficult to assess the practical advantages of this specific MDL formulation over other potential approaches. Is the added complexity of MDL necessary to achieve these results?
2.  **Unclear Scalability and Complexity:** The authors state that their implementation is limited to "up to five languages" and that runtime is a bottleneck. The complexity of the alignment (E-step) is given as O(|D|·|R|²), but the term `R` ("maximum number of possible rule applications") is not clearly defined in terms of input parameters like the number of languages (`N`) or string length. The search space for alignments grows exponentially with `N`, and a more formal analysis of how the algorithm's complexity scales is needed to understand its practical limits. The current demonstration on N=4 is promising but leaves the method's feasibility for larger language groups in question.
3.  **Limited Evaluation of Correspondence Quality:** The primary evaluation is extrinsic (phylogenetic tree) and qualitative (showing example alignments). There is no intrinsic evaluation of the discovered rules against a gold standard. While creating such a standard is difficult, it would be possible for a few language pairs using existing linguistic resources. This would allow for a direct quantitative assessment of the model's precision and recall in identifying correct correspondences and would also provide a more rigorous way to analyze the effect of the data over-weighting parameter `α`.
4.  **Potential for Sub-optimal Solutions:** The M-step employs a greedy strategy of merging the best pair of rules at each iteration. This is a reasonable heuristic but is susceptible to local optima. It's possible that this approach favors discovering simple, high-frequency patterns at the expense of more complex but linguistically important ones that would require a non-greedy sequence of merges to be found. The paper acknowledges that it "may miss some of the more subtle correspondence rules," but a discussion of the potential impact of this greedy choice would strengthen the paper.

### Questions to Authors

1.  Could you provide a more detailed analysis of the computational complexity of the alignment procedure, specifically explaining how the size of the alignment graph and the term `R` scale with the number of languages `N` and the length of the input strings? What are the primary obstacles (memory, runtime) to scaling beyond five languages?
2.  Regarding the greedy M-step, have you explored or considered alternative search strategies, such as beam search, to mitigate the risk of settling in poor local optima? How confident are you that the deterministic, greedy approach is sufficient for this problem space?
3.  The paper states that initializing with a given rule set allows for the integration of linguistic knowledge. Could you elaborate on how this would work in practice? For instance, how does the model behave if initialized with a partial or potentially noisy set of known rules?

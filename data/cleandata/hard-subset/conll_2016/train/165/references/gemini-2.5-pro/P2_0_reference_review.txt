### Summary & Contributions

This paper introduces a method for automatically discovering systematic string-level correspondences between multiple languages from cognate lists. The core of the approach is the Minimum Description Length (MDL) principle, which frames the problem as finding the set of correspondence rules that provides the most compact (i.e., compressed) representation of the data. The authors propose a two-part MDL code for encoding the model (the set of rules) and the data given the model. An iterative, deterministic EM-like algorithm is used to search for a good set of rules, starting from single-character rules and greedily merging them based on which merge provides the largest compression gain. The method is evaluated on a dataset of 13 Slavic languages, showing that it can produce a linguistically plausible phylogenetic tree and can quantify similarities between arbitrary subsets of languages in a multi-language analysis.

In my view, the main contributions are:

1.  **A novel formulation for multi-language similarity quantification:** The primary contribution is the application of MDL to the problem of discovering correspondences across more than two languages simultaneously. The concept of "shared Description Length" (sDL) is particularly insightful, providing a principled, information-theoretic way to quantify the amount of shared structure between any subset of the languages being analyzed (e.g., CS-PL vs RU-BG vs all four in the four-way analysis). This moves beyond standard pairwise comparisons.
2.  **A deterministic, assumption-free algorithm for correspondence discovery:** The paper presents a concrete algorithm for finding these correspondences. Its key properties are that it is deterministic (yielding reproducible results) and, by virtue of its MDL foundation, avoids making strong prior assumptions about the length, shape, or distribution of correspondence rules, allowing them to be discovered directly from the data.
3.  **A practical heuristic for controlling rule granularity:** The introduction of the data over-weighting parameter (`α`) provides a useful mechanism for linguists or other users to explore less statistically significant but potentially interesting correspondences, effectively trading off statistical objectivity for a desired level of detail.

### Strengths

1.  **Principled and Well-Motivated Framework:** The choice of the MDL principle is a significant strength. It provides a solid theoretical foundation for "unsupervised" model selection without relying on arbitrary priors or parameter tuning. The authors correctly motivate this by highlighting the limitations of methods that impose assumptions on the form of correspondences. The idea that the best model is the one that compresses the data most is elegant and powerful.
2.  **True Multi-Language Analysis:** The method's ability to analyze more than two languages at once is a key advantage over many existing approaches. The four-way analysis in Section 5.3 is the most compelling part of the paper. The visualization of shared Description Lengths (sDL) in Figure 3 is an excellent demonstration of the model's analytical power, revealing not just the expected West/East Slavic split but also more subtle relationships, providing a much richer picture than a simple phylogenetic tree.
3.  **Convincing Experimental Results:** The experiments are well-designed and the results support the authors' claims. The pairwise analysis resulting in a linguistically plausible phylogenetic tree (Figure 1) serves as a successful sanity check, showing the underlying distance metric (NCD) is meaningful. The qualitative examples of learned correspondences (Tables 3 and 4) are intuitive and demonstrate the discovery of linguistically relevant patterns, from single-phoneme shifts (g,h) to more complex morphological endings (eć,et).

### Weaknesses

1.  **Scalability Concerns:** The paper is candid about the computational limitations, stating that the current implementation can process "up to five languages" on a standard desktop machine. The alignment procedure is described as a shortest-path problem, and its complexity appears to be a major bottleneck as the number of languages (N) increases. A more formal analysis of the time and space complexity with respect to N, word length, and cognate list size would be beneficial. Without this, the practical applicability of the method to larger-scale comparative linguistics remains unclear.
2.  **Lack of Comparison to Alternative Computational Methods:** The evaluation compares the results to established linguistic knowledge, which is valuable. However, it lacks any quantitative comparison to other computational methods for cognate analysis or correspondence discovery. For instance, how does the NCD-based phylogeny compare to trees generated from distances derived from other alignment methods (e.g., methods based on PMI or adapted from bioinformatics)? While the multi-language aspect is novel, a comparison on the pairwise task would help ground the performance of the proposed MDL model against existing techniques.
3.  **Algorithmic Novelty is Moderate:** While the application of MDL to this specific multi-language problem is novel, the optimization algorithm itself—an iterative process that greedily merges the two patterns yielding the best score—is a common heuristic in MDL-based pattern mining. Framing it as an EM algorithm is reasonable, but the M-step is essentially a greedy search, which may be susceptible to local optima. While the deterministic nature is a feature, some discussion of this potential limitation would be appropriate.
4.  **Clarity of the MDL Formulation:** Section 4.1, which details the model code `L(M)`, is extremely dense. While likely correct, the description of the components, particularly the "data-to-model code" for the rule table and alphabets, is terse and relies on the reader's prior expertise with MDL. A bit more intuition behind the formulas (e.g., explaining that `log( (T-1) choose (k-1) )` is the code length for specifying k counts that sum to T) would significantly improve the paper's clarity and reproducibility.

### Questions to Authors

1.  Regarding scalability: Could you provide a more detailed analysis of the computational complexity, particularly how the runtime of the alignment step scales with the number of languages N? What are the primary obstacles to scaling beyond five languages, and do you see a clear path forward to mitigate these issues?
2.  Regarding the greedy nature of the M-step: Your algorithm deterministically finds a good set of correspondences. Did you investigate how sensitive the final model is to the greedy merge decisions made early in the process? Is it possible for an early, suboptimal merge to prevent the discovery of a more important, longer correspondence later on?
3.  The data over-weighting parameter `α` is presented as a practical tool but moves the method away from the objective MDL framework. Do you have any guidance on how a user might set this parameter in a principled way, or is it intended purely for subjective, exploratory analysis?

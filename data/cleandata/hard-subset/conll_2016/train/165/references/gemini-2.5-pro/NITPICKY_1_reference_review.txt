### Summary and Contributions

This paper proposes a method for discovering string-level correspondences between multiple languages using the Minimum Description Length (MDL) principle. The authors formulate a two-part MDL code to represent both a set of correspondence rules and the data encoded with those rules. They present a deterministic, EM-like algorithm that iteratively aligns the data (E-step) and greedily merges rules to improve the model (M-step). The primary contributions of this work, as I assess them, are:

1.  **A Multi-Language Correspondence Framework:** The paper successfully extends the problem of discovering linguistic correspondences from the typical pairwise setting to an arbitrary number of languages (N>2). This is the most significant contribution, as it allows for a more holistic analysis of language families.
2.  **The Shared Description Length (sDL) Metric:** The authors introduce a novel quantitative metric, sDL, which decomposes the total model complexity into components shared by specific subsets of languages. This provides a fine-grained tool for analyzing language group similarities beyond what a simple phylogenetic tree can offer.
3.  **A Heuristic for Controlling Rule Granularity:** The proposed "data over-weighting" technique (the α parameter), while departing from pure MDL, provides a practical mechanism for users to control the level of detail in the inferred correspondences, balancing statistical significance with linguistic intuition.

### Strengths

1.  **Principled, Assumption-Light Modeling:** The use of MDL provides a solid information-theoretic foundation for the task. By treating correspondences as arbitrary strings and avoiding strong distributional or phonological assumptions, the method allows patterns to emerge directly from the data. This "let the data speak" approach is commendable and increases the objectivity of the initial findings (before data over-weighting is applied).
2.  **Novelty in Multi-Language Quantification:** The ability to analyze more than two languages simultaneously is a key strength. The sDL metric, in particular, is an elegant and powerful idea. Figure 3, despite its visual clutter, effectively demonstrates how this metric can reveal complex relationships, such as the relative strength of the CS-PL and RU-BG subgroups within a four-way analysis, which is a more nuanced insight than pairwise distances alone can provide.
3.  **Interpretability and User Control:** The output of the model—explicit correspondence rules—is directly interpretable by linguists. The examples in Table 3 effectively illustrate this. Furthermore, the inclusion of the data over-weighting parameter is a pragmatic and valuable feature. It acknowledges that pure statistical significance may not always align with linguistic utility and provides an intuitive knob for experts to explore more subtle or complex patterns that are present but lack sufficient evidence in a small dataset.

### Weaknesses

1.  **Insufficient Methodological Detail:** The paper suffers from a concerning lack of clarity on several key algorithmic components, hindering reproducibility and critical evaluation.
    *   The "partial order reduction" (Section 4.3) is mentioned as a crucial step for efficiency but is never explained. What is this reduction? How is it implemented?
    *   The connection to prior work is tenuous. The paper states it "extends the encoding introduced by (Tatti and Vreeken, 2012)" but never specifies what the novel contributions to the encoding scheme are. Is it just an application to a new domain, or are there fundamental changes to the code?
    *   The complexity analysis (Section 4.4) uses variables (R and A) that are vaguely defined as "maximum number of possible rule applications" and "maximum number of rules used," which are not precise enough for a formal analysis.

2.  **Severe Presentation and Formatting Issues:** The manuscript is unacceptably sloppy for a conference submission. The sheer number of errors suggests a lack of careful proofreading and detracts significantly from the paper's credibility.
    *   **References:** The reference section is a complete disaster, with formatting errors, question marks, inconsistent styles, and incorrect BibTeX parsing evident (e.g., `Saitou et al\.,? \Q1987\E`). This is a major red flag.
    *   **Typos and Grammar:** Minor errors are rampant (e.g., "formore", "amongst others", "data set sizes for experiments." in a table caption, using "&" in a section title).
    *   **Structural Flaws:** The introductory roadmap (Section 1) omits Section 3 entirely. Figure/Table captions are labeled "F 1" and "T 1". The list formatting in Section 2 begins with a hanging `)`.

3.  **Questionable Scalability and Performance Claims:** The algorithm's performance seems to be a major bottleneck. The authors report that processing five languages takes "a few hours on a... single core desktop machine." This does not support the claim that the algorithm can "efficiently analyze more than two languages at a time" (Section 5.3). The problem is combinatorial in N, and with a runtime of hours for N=5, the practical applicability for larger language groups is highly doubtful. The promise to "extend this to higher N in future work" is not a satisfactory answer to this fundamental limitation.

4.  **Ambiguous and Potentially Flawed Evaluation Metric:** The formula for Normalized Compression Distance (NCD) in Section 5.2 is non-standard. The use of `L(X,X)` is ambiguous. If it means the cost of encoding data X with a model trained on X, it should be written as `L(M_X) + L(D_X|M_X)`. If it means something else, it requires a precise definition. As written, it is unclear how the distances in Table 2 were actually calculated.

### Questions to Authors

1.  Please provide a detailed explanation of the "partial order reduction" used in your alignment procedure (Section 4.3). How does it prune the search space of the shortest-path algorithm?
2.  Can you clarify the precise definition of the `L(X,X)` term in your NCD formula? Is this a standard formulation, and if so, could you provide a citation?
3.  The core of your M-step is a greedy search. Given the non-convex nature of the problem, this is susceptible to local optima. Have you investigated how sensitive the final rule set is to this greedy strategy? For instance, do different tie-breaking strategies for merges lead to substantially different models?
4.  What are the specific, novel components of your MDL code relative to the work of Tatti and Vreeken (2012)? A clear delineation would help establish the technical contribution of your proposed encoding.

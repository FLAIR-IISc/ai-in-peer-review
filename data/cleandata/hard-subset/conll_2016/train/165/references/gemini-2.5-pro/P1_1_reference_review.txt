### Summary and Contributions

This paper introduces a method for automatically discovering string-level correspondences between multiple languages from cognate lists. The approach is framed under the Minimum Description Length (MDL) principle, where the best model is the set of correspondence rules that provides the most succinct description (i.e., the best compression) of the given data. The method employs an EM-like deterministic, greedy algorithm that starts from an unstructured "null alignment" and iteratively merges rule components to build up a lexicon of correspondence rules. The authors demonstrate the method's effectiveness through pairwise and four-way analyses of Slavic languages, showing that it can reconstruct known phylogenetic relationships and, more importantly, provide a novel, fine-grained quantification of similarity between any subset of the languages.

In my view, the main contributions of this work are:

1.  **A principled framework for simultaneous multi-language correspondence discovery.** The key novelty is moving beyond pairwise comparisons to a method that can directly infer N-way correspondences (e.g., involving 2, 3, or all 4 languages in the four-way analysis). This allows for capturing complex, shared structures that would be difficult to model with a series of pairwise analyses.
2.  **The "Shared Description Length" (sDL) metric for fine-grained similarity analysis.** This metric, derived directly from the MDL model cost, provides a quantitative way to measure the amount of shared structure between any subset of languages. The analysis in Figure 3 is a compelling demonstration, offering richer insights into language group similarities than a single phylogenetic tree can provide.
3.  **A practical, assumption-light, and interpretable algorithm.** The method makes few assumptions about the shape or distribution of correspondences, allowing them to be learned directly from data. The resulting rules are easily interpretable strings, and the proposed "data over-weighting" technique offers a pragmatic way for users to control the granularity of the output, which is valuable for exploratory linguistic research.

### Strengths

1.  **Methodological Elegance and Soundness.** The application of the MDL principle to this problem is a very natural and powerful choice. It provides a formal, information-theoretic foundation for balancing model complexity (the cost of the rule set) against data fit (the cost of aligning the cognates) without requiring arbitrary priors or hyperparameters. The ability to model N languages simultaneously within this single framework is a significant strength over methods limited to pairwise analysis.
2.  **Novel and Insightful Multi-way Analysis.** The paper's main strength lies in the four-way analysis (Section 5.3). The concept of Shared Description Length (sDL) is an excellent contribution, turning the components of the MDL cost function into an insightful analytical tool. The visualization in Figure 3 effectively quantifies not just the CS-PL and RU-BG subgroupings, but also the total shared structure among all four languages and the unique complexities of each individual language. This is a much richer output than a standard distance matrix or dendrogram.
3.  **Strong Empirical Validation on Slavic Languages.** The experiments are well-designed and convincing. The pairwise analysis resulting in the phylogenetic tree in Figure 1 serves as an effective sanity check, showing that the model's NCD metric successfully recovers the well-established West, East, and South Slavic groupings. This grounds the method in established linguistic reality before the paper moves on to its more novel multi-way analysis. The qualitative examples of learned rules (e.g., liquid metathesis `(ла,оло)`, palatalization `(dzie,dě)`) are compelling and demonstrate that the model learns linguistically plausible phenomena.
4.  **Practicality and Interpretability.** The method is deterministic, which is a desirable property for reproducible scientific analysis. The learned rules are directly interpretable as string-to-string mappings, making them accessible to linguists. Furthermore, the data over-weighting mechanism is a clever and practical feature that allows a domain expert to guide the model towards discovering rarer or more complex correspondences that might not meet the strict statistical significance threshold of pure MDL, effectively tuning the model for different exploratory goals.

### Weaknesses

1.  **Significant Scalability Concerns.** The paper is candid about the computational bottleneck, stating the current implementation handles up to five languages. The alignment procedure is described as a shortest-path problem on a graph, and the complexity likely grows exponentially with the number of languages (N). This is a major limitation, as it prevents the application of this promising method to larger language families (e.g., Indo-European, Austronesian) where N can be much larger. The paper would be stronger with a more detailed analysis of the complexity with respect to N and a discussion of potential approximations or algorithmic improvements to address this.
2.  **Greedy Search and Potential for Local Optima.** The M-step of the training procedure relies on a greedy strategy of merging the two patterns that offer the greatest immediate reduction in description length. While this makes the algorithm efficient and deterministic, it is susceptible to finding local optima. The paper acknowledges this possibility but does not investigate its impact. It is unclear how suboptimal the resulting models might be compared to a more global search strategy. A comparison with, or at least a discussion of, alternatives like beam search or stochastic methods would strengthen the work.
3.  **Lack of Extrinsic, Quantitative Evaluation.** The evaluation is primarily intrinsic and qualitative. The authors show that the generated phylogeny is "sensible" and that the learned rules "look plausible". While this is a necessary first step, the paper lacks a quantitative evaluation on a downstream task. For instance, the quality of the learned correspondence rules could be measured by their utility in a cognate reconstruction or identification task against a gold standard. Such an evaluation would provide a more objective measure of how the proposed method compares to other approaches.
4.  **Clarity of MDL Formulation.** Section 4, which details the MDL code, is quite dense and difficult to parse without deep familiarity with the topic. The description of L(Π), in particular, involves several terms (e.g., coding for subsets of languages, the data-to-model code for rule counts) that are not explained with sufficient intuition. A simple, running example demonstrating how the code length is calculated for a toy model with 2-3 rules and a few data items would significantly improve the paper's clarity and reproducibility.

### Questions to Authors

1.  Could you provide more detail on the computational complexity as a function of the number of languages, N? What is the primary bottleneck—memory from the graph construction or the runtime of Dijkstra's algorithm? Are there clear paths to improving scalability beyond N=5, such as pruning the search space or using approximate search algorithms?
2.  The data over-weighting parameter `α` is a pragmatic solution for finding more granular rules, but it moves away from the objectivity of MDL. Have you analyzed the effect of large `α` values? Does it simply lead to memorizing entire words as rules, or does it still preserve a meaningful order of discovery based on statistical significance?
3.  Regarding the greedy search in the M-step: have you considered initializing the algorithm from different starting points (e.g., a greedy alignment based on character n-gram statistics instead of the null alignment) to assess the stability of the final discovered rule set and its sensitivity to local optima?

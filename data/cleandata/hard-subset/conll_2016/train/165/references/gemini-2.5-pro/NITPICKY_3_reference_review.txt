### Summary of the paper and main contributions

This paper introduces a method for discovering systematic string correspondences between multiple languages using the Minimum Description Length (MDL) principle. The authors frame the problem as finding a set of correspondence rules that provides the most compact description of a given corpus of cognate tuples. The method avoids pre-specified distributional assumptions about the correspondences. The core contributions, as I see them, are:

1.  **MDL Formulation for Multi-Language Correspondences:** The primary contribution is the application of the two-part MDL principle to the task of discovering correspondences between more than two languages simultaneously. This provides a principled, non-parametric framework for the problem.
2.  **Shared Description Length (sDL) Metric:** The paper proposes a novel information-theoretic metric, sDL, to quantify the degree of similarity between any subset of the languages being analyzed. This allows for a more fine-grained analysis than traditional pairwise distance metrics.
3.  **A Deterministic Inference Algorithm:** The authors present an EM-like greedy algorithm to search for the optimal set of correspondence rules by iteratively aligning the data and merging rules to improve compression.

### Strengths

1.  **Principled Framework:** The use of the MDL principle is well-motivated. It provides an elegant, objective function for model selection that naturally balances model complexity with data fit, thus avoiding arbitrary assumptions about the shape or distribution of correspondence rules, a notable limitation of some prior work.
2.  **Multi-Language Capability:** The model's ability to operate on more than two languages at once is a significant advantage. This enables a more holistic view of language relationships and supports novel analyses, as demonstrated by the shared Description Length (sDL) results in the four-way analysis, which reveal subtle subgroupings and influences that pairwise comparisons would obscure.
3.  **Compelling Empirical Demonstration:** The experimental results on Slavic languages are convincing. The NCD-based phylogenetic tree in Figure 1 correctly reconstructs the known linguistic subgroups (West, East, South Slavic), validating the underlying distance metric. The four-way analysis provides intriguing quantitative insights into the shared and unique complexities of Czech, Polish, Russian, and Bulgarian.

### Weaknesses

1.  **Lack of Methodological and Algorithmic Clarity:** The description of the method is opaque in several key areas.
    *   **MDL Code:** The formula for `L(Π)` in Section 4.1 is presented with insufficient justification. The term `log((TΠ - 1) over (|Π| - 1))` is introduced as a "data-to-model code" for rule usages, but its derivation and role within the overall two-part code are not clearly explained. This makes it difficult to fully assess the soundness of the proposed code.
    *   **Alignment Procedure:** The alignment process (Section 4.3), a critical component, is glossed over. The graph construction for Dijkstra's algorithm is not detailed—what exactly are the "index tuples" that serve as nodes? The mention of "partial order reduction" is made without any explanation.
    *   **Complexity Analysis:** The time complexity is given in terms of vague variables (`R`, `A`). This hides the true computational cost, which appears to be exponential in the number of languages. The admission that the implementation can only handle "up to five languages" on a single core suggests severe scalability limitations that are not adequately addressed beyond a promise of "future work."

2.  **Ad-hoc "Data Over-Weighting" Undermines Core Principles:** The introduction of the data over-weighting mechanism in Section 4.5 directly contradicts the paper's central claim of providing an "objective, unbiased" analysis based purely on the data. This technique, which optimizes `L(M) + αL(D|M)`, introduces an arbitrary hyperparameter `α`. The authors admit this "moves results away from the objective territory of MDL," which significantly weakens the paper's premise. It is unclear how this parameter was set for the results in Table 3, making those results difficult to interpret or reproduce. This feels less like a principled extension and more like an ad-hoc fix to extract desired rules that the objective model fails to find.

3.  **Insufficient Evaluation:** The paper completely lacks comparison with any baseline or alternative method. While the introduction criticizes prior work, the experiments provide no quantitative evidence that the proposed method is superior. How does the multi-language model compare to applying a state-of-the-art pairwise method to all pairs and aggregating the results? Without such a comparison, claims of efficiency and efficacy are unsubstantiated. The evaluation is purely descriptive.

4.  **Major Presentation and Formatting Flaws:** The paper is riddled with serious formatting and typesetting errors that are unacceptable for a conference submission.
    *   **Section Numbering:** The document structure is broken. Section 2 is followed by a header for Section 4, then a header for Section 3, then the main body of Section 4. This is a critical structural error.
    *   **References:** The bibliography is completely malformed, containing raw BibTeX-like artifacts (e.g., `? \Q2004\E`, `Saitou et al\.,`). This demonstrates a lack of care in the paper's preparation.
    *   **Inconsistencies and Typos:** There are numerous grammatical errors and inconsistencies. Table captions are abbreviated ("T 1") while Figure captions are not ("Figure 2"). Subsections are denoted with a leading dot (e.g., ".1 Model Code"). The overall impression is that of a rushed and unpolished draft.

### Questions to Authors

1.  Could you provide a more detailed explanation of the alignment graph (Section 4.3)? Specifically, what do the node "index tuples" represent in a multi-language setting, what do the edges represent, and how are their weights defined?
2.  The data over-weighting mechanism seems to compromise the objectivity of the MDL framework. How was the level of over-weighting chosen for the examples in Table 3? Is there a principled method for selecting this parameter, or is it purely a tool for subjective exploration?
3.  Given the stated scalability issues (N≤5), could you provide a more formal complexity analysis with respect to the number of languages N and the length of the cognate strings? Do you foresee a path to scaling this method to, for instance, N=10 languages?
4.  The section numbering and the reference list are incorrectly formatted. Will these be corrected in the final version?

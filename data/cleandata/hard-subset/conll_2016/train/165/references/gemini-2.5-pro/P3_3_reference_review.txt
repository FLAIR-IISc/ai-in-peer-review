### Summary and Main Contributions

This paper presents a novel method for discovering orthographic correspondences among multiple languages using the Minimum Description Length (MDL) principle. The core idea is that the best set of correspondence rules is the one that allows for the most compact (compressed) description of the cognate data. The authors propose a deterministic, EM-like algorithm that iteratively aligns data and merges correspondence rules to optimize the MDL objective.

As I see them, the main contributions are:
1.  **A principled framework for multi-language correspondence discovery.** The primary contribution is extending the problem of finding linguistic correspondences from the standard pairwise setting to an arbitrary number of languages (N) within a single, unified model. The MDL formulation allows for the discovery of n-way rules (where n ≤ N) based on their statistical significance in the data, a significant step beyond chaining pairwise analyses.
2.  **The introduction of "Shared Description Length" (sDL) as a quantitative metric for n-way similarity.** This metric, derived directly from the components of the learned MDL model, provides a more nuanced and detailed view of language relationships than a single distance value or a phylogenetic tree. It allows for quantifying the complexity shared by specific subsets of languages (e.g., CS-PL vs. RU-BG within a four-way model).
3.  **An interpretable, bottom-up algorithm for learning rules.** The proposed method learns explicit, human-readable correspondence rules of arbitrary length (e.g., (szcz, št)), starting from single characters and greedily merging them. This makes the model's output directly useful for linguistic analysis.

### Strengths

1.  **Methodological Soundness and Novelty:** The application of two-part MDL to the multi-language correspondence problem is elegant and well-motivated. It provides a non-parametric, assumption-light alternative to many statistical models, allowing the structure of the correspondences to be determined by the data itself rather than by a predefined model family. The ability to handle more than two languages simultaneously is a key advantage and a clear advance over much of the related work cited.
2.  **Strong and Convincing Empirical Evaluation:** The experiments on 13 Slavic languages are well-designed and the results are compelling. The pairwise analysis, culminating in a phylogenetic tree that closely matches established linguistic classifications, serves as excellent validation for the underlying NCD distance metric derived from the models. This grounds the method in known linguistic reality before extending it to the more novel multi-way analysis.
3.  **Insightful Multi-Way Analysis:** The four-way analysis of Czech, Polish, Russian, and Bulgarian is the paper's highlight. It successfully demonstrates the method's unique capabilities. The use of the proposed sDL metric to dissect the similarities between all 2^4 - 1 subsets of these languages is particularly insightful. The visualization and discussion in Figure 3 provide a rich, quantitative picture of the subgroupings (CS-PL, RU-BG) and cross-group similarities, offering a level of detail that traditional phylogenetics cannot capture.
4.  **Interpretability and User Control:** The model's output—a set of explicit correspondence rules—is highly interpretable and directly usable by linguists. Furthermore, the introduction of the "data over-weighting" parameter, while a departure from pure MDL, is a pragmatic and useful feature. It gives experts a knob to control the granularity of the discovered rules, allowing them to trade statistical "objectivity" for potentially more linguistically complete or interesting rule sets. The authors are transparent about this trade-off.

### Weaknesses

1.  **Significant Scalability Limitations:** The paper honestly reports that the current implementation is limited to approximately five languages due to computational complexity in the alignment step. This is a major practical limitation. While the framework is theoretically general, its current inability to scale to larger language families (e.g., the full set of 13 Slavic languages at once) curtails its utility for large-scale comparative linguistics. More discussion on the nature of this combinatorial bottleneck and potential algorithmic solutions (e.g., pruning, beam search) would be necessary to assess its future potential.
2.  **Greedy and Deterministic Optimization:** The M-step of the training procedure is purely greedy—it merges the single best pair of rules at each iteration. This makes the algorithm susceptible to local optima. It is plausible that a series of individually sub-optimal merges could lead to a better overall model. The deterministic nature means it will always find the same, possibly sub-optimal, solution. While determinism can be a feature, the paper would be stronger if it acknowledged and evaluated the potential impact of this greedy strategy, perhaps by comparing it to a randomized or beam-search-based alternative.
3.  **Ad-hoc Nature of Data Over-weighting:** The `α` parameter for data over-weighting, while practically useful, feels ad-hoc and undermines the principled foundation of MDL. The paper provides little guidance on how to set this hyperparameter, other than by observing the output, making it a subjective "knob" rather than a modeled component. It's unclear if this is the best way to encourage longer patterns; alternatives like modifying the model cost `L(M)` to have a different prior on rule length could be more integrated into the MDL philosophy.
4.  **Reliance on Orthography:** The experiments are performed exclusively on raw orthographic data. While this is a reasonable simplification, it ignores the crucial distinction between orthography and phonology. Many systematic sound correspondences are obscured by idiosyncratic spelling conventions (e.g., Polish 'rz' vs. Czech 'ř'). The paper should more explicitly discuss this limitation and how the results might differ if applied to phonetically transcribed data, which is the standard input for the traditional comparative method the paper aims to automate.

### Questions to Authors

1.  Could you provide more detail on the computational bottleneck for N > 5? Is the limiting factor memory consumption from constructing the alignment graph, or the runtime of Dijkstra's algorithm over it? Have you considered approximate alignment methods or pruning heuristics to make the E-step more tractable for larger N?
2.  The sDL metric is a fascinating contribution for static analysis of a final model. Have you considered using it dynamically during the search process? For instance, could changes in the sDL of various language subsets be used as a heuristic to guide the rule-merging strategy in the M-step, perhaps leading to a more globally optimal model?
3.  The conclusion mentions that integrating linguistic knowledge is a key advantage. Beyond initializing with a given rule set, have you considered other ways to incorporate such knowledge? For example, could one constrain the search to only merge phonetically plausible singletons, or use a non-uniform cost for different types of rules in L(M)?

### Summary and Contributions

This paper presents a novel, non-parametric method based on the Minimum Description Length (MDL) principle for discovering string-level correspondences across multiple languages simultaneously. The core idea is to define the best model (i.e., set of correspondence rules) as the one that allows for the most concise description of a given corpus of cognates. The authors propose a two-part MDL code for this task and an efficient, deterministic EM-like algorithm that iteratively merges simpler rules into more complex ones to find a good model. The framework's ability to analyze more than two languages at once is a key feature, enabling a more nuanced quantification of linguistic similarity.

In my view, the main contributions are:

1.  **A Novel Framework for N-Way Correspondence Discovery:** The primary contribution is a new paradigm for discovering correspondences that breaks free from the traditional, restrictive pairwise setting. By analyzing N languages simultaneously, the method can uncover and quantify complex inter-relationships. The proposed "shared Description Length" (sDL) metric is a particularly powerful outcome of this framework, providing a fine-grained, information-theoretic measure of similarity for every possible subset of the languages under study.
2.  **An Unsupervised, Assumption-Light Learning Method:** The paper introduces a method to learn correspondence rules of arbitrary string complexity (e.g., `(szcz, št)`) directly from data, without imposing strong prior assumptions on their form or distribution. The grounding in MDL provides a principled, objective function for balancing model complexity against data fit, allowing linguistically plausible patterns to emerge naturally.

### Strengths

1.  **High Novelty and Conceptual Boldness:** This work represents a significant conceptual advance for computational historical linguistics and multilingual NLP. The move from pairwise to true N-way analysis is a bold and necessary step, and the MDL formulation provides an elegant and principled way to achieve it. The sDL concept is a genuinely new tool for quantitative comparative linguistics, offering insights that are simply inaccessible through a series of pairwise comparisons. This is precisely the kind of foundational, paradigm-shifting work that pushes the field forward.
2.  **Principled and Interpretable Model:** The approach is rigorously grounded in the MDL principle, which provides a formal, information-theoretic justification for model selection. This avoids the need for ad-hoc heuristics or extensive parameter tuning that plague many statistical models. The model's ability to learn complex, linguistically meaningful correspondences—from simple phonetic shifts to orthographic conventions and morpho-syntactic endings—directly from raw string data is a powerful demonstration of its potential. The resulting rules and the sDL analyses are directly interpretable, as shown in the excellent four-way analysis of Slavic languages.
3.  **Flexibility and Practicality for Linguistic Exploration:** The framework is impressively flexible. While it provides an objective, deterministic result by default, the introduction of the data over-weighting parameter `α` is a clever, pragmatic feature. It empowers a domain expert (e.g., a linguist) to steer the model towards discovering less statistically dominant patterns that may still be of interest, effectively bridging the gap between purely data-driven discovery and expert-guided analysis.
4.  **Compelling Initial Validation:** The experiments effectively showcase the method's promise. The pairwise analysis and the resulting phylogenetic tree for 13 Slavic languages successfully replicate established linguistic knowledge, providing a solid sanity check. The four-way analysis is the paper's highlight, demonstrating the unique power of the N-way approach. It not only identifies the major West/East Slavic groupings but also quantifies their shared structure and uncovers more subtle relationships, providing a much richer picture of language similarity than a simple tree structure.

### Weaknesses

1.  **Scalability Limitations:** The authors are transparent about the method's current computational limitations, noting it can handle up to five languages. While this is a significant step beyond pairwise methods, it hinders application to larger language families. The complexity analysis is also somewhat abstract. This is a forgivable weakness for a paper introducing such a novel concept, as scalability is often an engineering challenge that can be tackled in follow-up work. However, it is a practical constraint on the method's current utility.
2.  **Greedy Search Heuristic:** The M-step of the training algorithm relies on a greedy heuristic of merging the two patterns that yield the greatest compression. While this makes the algorithm efficient and deterministic, it is susceptible to local optima. An early, seemingly good merge could preclude the later discovery of a globally better set of rules. Given the combinatorial search space, this is a reasonable simplification, but the paper would benefit from a discussion of the potential pitfalls and perhaps preliminary results from alternative search strategies (e.g., beam search).
3.  **Lack of Extrinsic Evaluation:** The evaluation is convincing but primarily intrinsic and qualitative; it demonstrates that the model produces sensible phylogenetic trees and interpretable rules. The work would be even stronger if it included an extrinsic, task-based evaluation. For instance, could the learned N-way correspondence rules be used to improve performance on a downstream task like cognate detection, phylogenetic reconstruction, or machine transliteration, and how would this compare against established baselines? This would provide a more concrete measure of the practical value of the discovered patterns.

### Questions to Authors

1.  The data over-weighting parameter `α` is a very interesting feature for exploratory analysis. In your experiments, you used the heuristic of increasing `α` "until there are no more singleton rules." Is there a more principled way to guide the choice of `α`, or do you see it primarily as a user-controlled "slider" for adjusting the granularity of the output?
2.  The current maximization step is a greedy merge. Have you considered the inverse operation: a "split" step? It seems possible that an early merge might be suboptimal in the long run, and having a mechanism to undo or refine rules could help escape local optima and lead to a better final model.
3.  The Shared Description Length (sDL) analysis is fascinating. The raw bit values, however, are dependent on the specific dataset. Have you considered normalizing these values (e.g., as a percentage of the total L(M) or L(M)+L(D|M)) to create a more standardized measure of shared complexity that could be compared across different language groups or datasets?

- Paper proposes an MDL-based framework to discover multi-way string correspondence rules across N languages, combining rule sets and unigram symbols with a two-part MDL objective.
- Training alternates an E-step aligning cognate tuples via shortest-path (Dijkstra) on an index-tuple lattice and an M-step merging rules that reduce description length, starting from singletons.
- Introduces a data over-weighting parameter α to encourage longer rules and performs experiments on Slavic languages to derive phylogeny and quantify shared structure via “shared description length.”

Contributions:
- MDL formulation and deterministic EM-style algorithm to infer multi-way (beyond pairwise) string correspondences from orthographic cognate lists.
- Information-theoretic analysis quantifying inter-language similarity by attribution of rules shared by any subset of languages.
- Empirical Slavic case study showing plausible correspondences, a phylogeny consistent with linguistic knowledge, and exploration of a data-weighting knob to control rule granularity.

Strengths:
- Model supports native multi-way correspondences and allows empty components to capture asymmetric subset sharing.
- Two-part MDL objective provides interpretable trade-offs between adding rules and using them in data, helping avoid overfitting.
- Tight coupling of alignment and rule induction with an exact shortest-path alignment for arbitrary-length rules.
- Learned rules are qualitatively interpretable and recover known linguistic mappings beyond single-character edits.
- The subset-based “shared description length” gives nuanced insight into contributions of rules shared by arbitrary language subsets.

Weaknesses:
- Evaluation is limited: no quantitative baselines, intrinsic alignment metrics, or extrinsic tasks to justify claims beyond tree plausibility and examples.
- L(Π) model coding and data-to-model details are underspecified, hindering replication and raising possible double-counting concerns.
- Introducing α departs from objective MDL, lacks principled justification, sensitivity analysis, and may produce spurious rules.
- Scalability concerns: Dijkstra on a high-dimensional lattice causes memory/runtime bottlenecks; experiments limited to small datasets and up to five languages with no complexity bounds or scaling engineering.
- Using raw orthography and mixed scripts (no phonological normalization/transliteration) may bias results and let script effects dominate linguistic correspondences.

Questions to authors:
- Provide a precise derivation of L(Π), explain LN(TΠ)+log((TΠ−1 choose |Π|−1)), and how rule usage counts map to code(π); clarify any smoothing and whether counts are reused in model and data terms.
- Why define shared description length using model costs L(π) rather than data-term contributions attributable to rules; would a data-based sDL change conclusions?
- Precisely specify the alignment graph state-space and edges: allowed ε placements, prevention of degenerate ε overuse, and any max rule length or context constraints.
- Give EM details: stopping criteria, merge tie-breaking, convergence behavior, and sensitivity to initialization.
- How is α chosen; provide sensitivity analysis and guidelines; does over-weighting create linguistically spurious merges?
- Can you compare the NCD tree to baselines (e.g., edit-distance, PHMM/ALINE) and test robustness to noise or concept subset removal?

Additional suggestions:
- Release code, exact preprocessing steps (digraph tokenization, casing, diacritics, Cyrillic/Latin mapping), and data splits; consider phonemic transcriptions.
- Add quantitative evaluations: alignment accuracy against expert gold, downstream prediction (reconstructing missing cognates), and cognate clustering metrics.
- Explore scalability improvements (WFST-based alignment, A* with admissible heuristics, pruning/beam search) and report runtime/memory scaling.
- Include ablations for α, ε allowances, max rule length, and smoothing strength to assess design choices.
- Clean up references and notation; fix incomplete or misformatted citations.

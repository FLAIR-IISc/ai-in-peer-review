Summary and main contributions
This paper proposes an MDL-based framework to automatically discover many-to-many string correspondences across multiple languages without assuming a fixed parametric form. The model defines a two-part code over:
- a rule table Π of multi-language correspondences (tuples of variable-length strings per language, possibly empty), with enumerative/data-to-model coding of symbol and rule usage; and
- alignments of cognate tuples encoded as shortest paths in a DAG given current rule costs.

Training alternates between:
- E: exact re-alignment via Dijkstra given current code lengths; and
- M: greedy merging of two “compatible” rules if it reduces total description length.

The methodology is applied to Slavic languages using orthographic strings. Pairwise MDL-based NCDs recover a plausible phylogeny; multi-language models yield a proposed measure, shared Description Length (sDL), to quantify overlap among all subsets (e.g., CS–PL vs RU–BG). The paper also introduces “data overweighting” to push the model toward longer, less-supported rules when needed.

I see the main contributions as:
1) An MDL formulation and deterministic search heuristic for discovering multi-language, many-to-many correspondence rules from raw orthographic data, with exact shortest-path alignment.
2) An analysis framework for multi-language similarity via sDL and MDL-based NCD, demonstrated on Slavic languages.
3) Practical insights on controlling rule granularity via “data overweighting” and the ability to seed models with prior linguistic rules.

Strengths
- Conceptual clarity and flexibility: Treating correspondences as arbitrary string tuples across N languages, without parametric assumptions, is elegant and general. The design naturally captures heterogeneous patterns (e.g., orthographic digraphs, palatalizations, stem correspondences) and allows rules to be sparse across languages.
- Deterministic and explainable learning: The re-alignment via exact shortest paths and greedy MDL-based merges yield reproducible outcomes and interpretable rules, which is attractive for linguists who may want to inject prior rules and inspect discovered correspondences.
- Multi-language analysis beyond pairwise: The framework scales conceptually to N>2 and introduces sDL to quantify overlap among all subsets, surface subtle groupings, and complement pairwise NCD. The four-way CS–PL–RU–BG analysis is a good demonstration of this advantage.
- Sound information-theoretic grounding: MDL gives a principled trade-off between model complexity and data fit, and the use of enumerative/data-to-model codes for unigrams and rule usage is consistent with MDL practice (though some details need clarification; see weaknesses).
- Empirical plausibility: On Slavic datasets, the phylogeny derived from NCD roughly matches known classifications, and example correspondences are linguistically meaningful; script effects (Latin vs Cyrillic Serbo-Croatian) are captured in the distances, which is consistent with orthography-based modeling.

Weaknesses
- MDL code design and placement of terms need clearer derivation and may be problematic: L(Π) includes terms for rule usage counts (LN(TΠ) + log comb), which are data-dependent. In two-part MDL, parameter codes should not “peek at” the same data being encoded unless using a justified universal/parametric complexity term. It is unclear whether the proposed data-to-model code corresponds to an admissible universal code for multinomials and whether there is double-counting between L(Π) and L(D|M) via code(π). A formal derivation (or a theorem or citation) establishing Kraft-validity and asymptotic optimality would strengthen the work.
- Optimization procedure is heuristic and not a true EM: The M-step merges the best pair of patterns greedily rather than optimizing parameters given sufficient statistics. There is no convergence guarantee to even a local optimum of the stated objective. Given the combinatorial search, this may be inevitable, but the paper should position the method as MDL-based greedy pattern mining with re-alignment, not EM, and study sensitivity to merge order.
- Evaluation is limited: No comparison against strong baselines (e.g., PMI/PHMM-based cognate alignment, many-to-many grapheme alignment for transliteration/letter-to-phoneme, sequence transducer models) on either correspondence discovery quality or downstream tasks (cognate identification, reconstruction). The phylogeny is plausible but anecdotal; there is no quantitative tree quality (e.g., RF distance), no significance tests for NCD differences, and no held-out evaluation.
- Data issues and confounds: Using raw orthography mixes phonology and orthographic conventions; findings (e.g., Cyrillic vs Latin) may reflect script artifacts rather than linguistic correspondences. Swadesh/Wiktionary lists likely contain non-cognates and mixed etymologies; there is no explicit cognate filtering or robustness analysis. Requiring complete tuples biases the sample. The model’s behavior on noisy or partial cognacy is not assessed.
- Scalability and practical limits: The alignment E-step is O(|D|·|R|^2) and M-step O(|D|·A^2), with memory bottlenecks; experiments are restricted to small datasets and up to five languages. There is no analysis of rule growth, pruning, or approximate search to scale to larger lexicons or more languages. The overweighting trick departs from MDL and introduces subjective tuning without guidance for selecting α.

Questions to authors (optional)
- MDL coding: Please provide a rigorous derivation for L(Π) and L(Σi). Specifically: (a) What universal code over multinomial parameters is being used? (b) How are usage counts determined without circularity relative to L(D|M)? (c) Why do rule-usage enumerative terms belong in L(Π) rather than L(D|M)? Can you prove the overall code is prefix-free and does not double-count?
- Definition of “compatible rules” and merge operation: What precisely makes two rules mergeable across N languages and variable-length strings? Are merges restricted to subalignments frequently co-occurring contiguously? How do you prevent explosive growth of Π?
- Cognacy and noise: How sensitive is the method to non-cognate pairs in D? Did you filter Swadesh items for cognacy, and if not, what fraction are non-cognate? Can you report robustness when injecting noise?
- Baselines and ablations: Can you compare against pairwise PMI/PHMM alignment with many-to-many mapping, or standard many-to-many grapheme aligners, both pairwise and extended to multi-language via shared latent rules? An ablation without enumerative parameter coding, or without merge-based M-step, would clarify what drives performance.
- Overweighting: How should α be chosen? Can you show stability of discovered rules and downstream metrics (NCD, sDL) across α and dataset size?
- Reproducibility: Will you release code, data preprocessing scripts, and the exact lists used? Can you include details on graph construction, partial order reduction, and memory optimizations?

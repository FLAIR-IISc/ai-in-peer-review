Summary
The paper proposes an MDL-based framework to discover string-level correspondence rules across multiple languages without imposing prior parametric assumptions on the shape or distribution of correspondences. The model uses a two-part MDL code: (i) a model code that encodes alphabets and a table of correspondence rules (tuples of strings across N languages) and (ii) a data code that encodes each cognate tuple by its optimal alignment under the current rule set. Alignments are solved as shortest-path problems in a directed graph using Dijkstra’s algorithm. Training proceeds via an EM-like loop: E-step aligns the data to current rules; M-step greedily merges rules that most reduce total description length. The paper introduces a “shared Description Length” (sDL) measure to quantify structure shared by language subsets. Experiments on Slavic languages show: pairwise NCDs reflect known groupings; a derived phylogeny is sensible; multi-language analyses reveal fine-grained shared structures; and an optional “data over-weighting” trick yields more granular rules.

Main contributions (as assessed)
1) An MDL formulation and deterministic algorithm for discovering multi-language string correspondences via rule learning and optimal alignments, including a practical encoding of rule tables and alphabets. This is the core methodological contribution.
2) An information-theoretic analysis tool for multi-language similarity based on shared description lengths over rule subsets, enabling finer-grained insights than pairwise distances.
3) An empirical study on Slavic data (Swadesh lists and additional cognates), including NCD-based phylogeny and qualitative analysis of learned correspondences.

Strengths
- Methodological clarity in the MDL setup. The paper specifies a two-part code for models and data, including how to encode rule lengths, alphabets, and usage counts (via data-to-model codes). This yields a coherent objective (total description length) that naturally trades off model complexity and fit, aligning with MDL best practices. The E/M steps are well-motivated by the interplay between alignments and code costs.

- Multi-language alignment beyond pairwise approaches. The use of a rule table over N languages and graph-based optimal alignment scales the modeling conceptually beyond pairwise systems, enabling discovery of correspondences that involve any subset of languages. This unlocks analyses (e.g., sDL over subsets) that pairwise-only methods cannot provide.

- sDL as a fine-grained similarity quantification. The shared Description Length measure provides interpretable, additive decompositions of how much structure is shared across specific language subsets. The four-language case studies demonstrate that sDL can highlight both expected groupings (CS–PL, RU–BG) and more subtle cross-subset similarities.

- Empirical signals align with linguistic expectations. Pairwise NCDs and the resulting phylogeny largely match the established West/East/South Slavic groupings, including plausible outliers (e.g., BG/MK behavior due to morphology; Polish behavior due to digraphs/palatalization). Qualitative examples illustrate plausible orthographic and phonological correspondences.

- Deterministic training with the option to inject linguistic priors. The ability to start from singleton rules (null alignment) or a user-provided rule seed allows controlled, reproducible runs, while still permitting domain knowledge when available.

Weaknesses
- Limited quantitative evaluation and lack of baselines. The paper does not compare against established methods for cognate alignment or correspondence discovery (e.g., PMI-based sound correspondences, ALINE/ASJP/lexstat-style models, PHMMs). No accuracy metrics are reported (e.g., held-out compression, predictive likelihood, or correspondence precision/recall against a gold standard). The phylogeny is assessed only qualitatively; no tree distance metrics (e.g., Robinson–Foulds) are provided.

- Scalability and complexity concerns. Alignment uses Dijkstra on a multi-tape index graph; the E-step is O(|D|·|R|^2) and M-step O(|D|·A^2/2), with practical limits reported at N ≤ 5 on modest hardware. There is no complexity analysis as N grows or for longer strings/larger rule inventories, and no batching/pruning/beam alternatives. The partial order reduction is mentioned but not detailed enough to assess its impact.

- Incomplete technical specification of coding terms. The L(Π) expression includes data-to-model components (e.g., LN(TΠ) and a weak composition term), but the derivation and the exact enumeration assumptions are only sketched. Similarly, computation of code(π) from counts, smoothing, and how counts interact with merge operations could be specified more rigorously to ensure reproducibility and avoid double counting.

- Orthographic-level modeling with heterogeneous scripts and no normalization. The approach operates on raw orthography across Latin and Cyrillic scripts without phonemic normalization or sound-class mapping. This choice can conflate orthographic conventions with linguistic correspondences and may bias results (e.g., script effects seen for Serbo-Croatian). While the paper acknowledges this, it limits linguistic validity and comparability with phonology-aware baselines.

- “Data over-weighting” departs from MDL objectivity without validation. The alpha-weighting trick is useful in practice but introduces subjective tuning that changes the objective. The paper uses it to eliminate singleton rules but does not provide guidance on selecting α, quantify its effect on overfitting, or validate whether the induced rules improve downstream tasks.

Questions to Authors
- Model encoding details: Could you provide a precise derivation of L(Π), including the exact data-to-model coding scheme, enumeration assumptions for rule usage distributions, and how LN offsets are applied when zero counts occur? How is code(π) computed from counts after smoothing?

- Alignment graph construction: What are the node/edge counts as a function of string lengths and rule lengths? Can you detail the partial order reduction technique and its effect on complexity and memory?

- Convergence and optimization: Does each M-step provably reduce the total description length? How are candidate merges enumerated efficiently, and is there any mechanism to reconsider earlier merges (e.g., split operations) to escape local minima?

- Evaluation: Can you add quantitative comparisons to baseline models (e.g., PMI-based sound correspondences, PHMMs, lexstat/ASJP), tree distance metrics for the phylogeny, and/or held-out compression? Is there a small gold set of correspondences for manual evaluation?

- Robustness to orthography: Have you tried phoneme-level or sound-class normalization? How do results (NCDs, sDLs, learned rules) change under such preprocessing?

- Over-weighting: How is α selected in practice? Can you report sensitivity analyses and whether over-weighting improves predictive compression or downstream tasks?

- Reproducibility: Will code and datasets (including the four-language cognate list) be released to enable replication of the reported results?

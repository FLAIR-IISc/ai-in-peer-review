Summary
The paper proposes an MDL-based framework for discovering systematic string-level correspondences across more than two languages without imposing parametric assumptions on rule shapes or distributions. It defines a two-part code (PS-2) over alphabets and correspondence rules, aligns cognate tuples via shortest-path search, and learns rules deterministically through an EM-style procedure that greedily merges rules to reduce total description length. Experiments on Slavic languages report pairwise NCD-based phylogenies and a four-language analysis using shared description lengths to quantify similarities across all subsets.

Main contributions (as I see them)
1) A formal MDL model and deterministic learning algorithm for multi-language correspondence discovery: a concrete two-part code for alphabets and rules, shortest-path alignment, and a merge-based EM procedure that scales beyond language pairs.
2) An information-theoretic analysis toolkit for multiway similarity: use of NCD for pairwise distances and, more originally, “shared Description Length” (sDL) to quantify structure shared by any subset of languages, enabling analyses beyond pairwise comparisons.
3) A practical knob (data over-weighting) to control granularity of discovered rules, allowing practitioners to trade off strict MDL optimality for richer correspondences when data are sparse.

Strengths
- Clear MDL formulation with explicit coding choices. The paper carefully specifies how model cost decomposes into alphabet, rule-table, and usage components, and how data are encoded via rule applications. The use of data-to-model codes and universal integer codes demonstrates awareness of MDL best practices and supports reproducibility of the objective.
- Multi-language, deterministic correspondence induction. Unlike many prior works limited to pairs or reliant on stochastic optimization, the approach scales conceptually to N languages and produces deterministic outcomes (given initialization), addressing a non-trivial gap in multilingual historical/lexical comparison.
- Alignment as exact shortest-path search. Casting alignment with arbitrary-length, possibly multi-language rules as a weighted shortest-path problem and solving with Dijkstra provides an intuitively correct E-step that optimizes the alignment given current rule costs.
- Interpretable correspondences and analyses. The learned rules capture linguistically meaningful patterns (orthographic, phonological, and morphological), and the four-way sDL analysis offers a novel lens on the relative regularities shared across subsets—going beyond trees to quantify “how much” is shared by pairs, triples, and all four languages.
- Sensible empirical signals. The NCD-based phylogeny broadly matches established Slavic groupings, and the method detects expected outliers (e.g., Bulgarian/Macedonian due to morphology), lending face validity to the modeling choices.

Weaknesses
- Limited empirical evaluation and lack of baselines. The primary evidence is that the tree is “reasonable,” a property often achieved by simpler methods (e.g., edit distance lexicostatistics, LexStat/ASJP variants, PHMM-based aligners, ALINE/PMI/profile methods). There is no comparison on standard tasks such as cognate detection accuracy, sound correspondence induction quality (e.g., against LexStat/FCM), or predictive compression/generalization to held-out data. This weakens the claim of methodological advantage.
- Heavy reliance on orthography and ad hoc data choices. All experiments use raw orthographic forms from Wiktionary Swadesh lists and a curated set of internationalisms, with no phonological normalization or segmentation. This makes results sensitive to script conventions (e.g., diacritics, digraphs), as acknowledged in the Czech and Polish analyses and the split “Serbo-Croatian Latin vs Cyrillic,” which is arguably a script variant rather than a separate language. The conclusions about linguistic similarity may conflate orthographic and phonological patterns.
- Scalability and practicality not fully demonstrated. Although the method is “agnostic to N,” the implementation handles up to five languages with hours of runtime and memory bottlenecks on modest hardware. No experiments beyond four languages are reported, and complexity bounds depend on large constants (|R| and A), leaving open how the method would fare on larger lexica or more languages.
- The “data over-weighting” knob undermines the MDL objective. While practically useful, reweighting α to force inclusion of larger patterns departs from MDL’s objective criterion and risks overfitting or injecting user bias. The paper does not provide guidance for choosing α, nor guardrails (e.g., validation, stability checks) to avoid spurious “near-significant” correspondences.
- Some modeling and coding details need clarification/justification. For example, the rule-table code includes terms (counts and compositions) whose derivation is only sketched; compatibility constraints for merging rules are not precisely defined; Laplace correction and its effect on code lengths are not fully specified; and the handling of ε-entries and their costs (or biases) could affect learned sparsity across languages.

Questions to authors
- Baselines and tasks: Can you compare against established methods (e.g., PHMM/PMI/ALINE/LexStat) on cognate identification or correspondence induction, and/or report predictive compression on held-out data?
- Orthography vs phonology: Have you tried phonemic transcriptions (IPA or language-specific phonologies)? How do results and sDL/NCD patterns change when controlling for script and diacritics?
- Rule merging and compatibility: Precisely how do you define “compatible” rules for merging in the M-step? Is the merge search local/global? Do you recompute alignments after each merge?
- Code derivations: Please provide full derivations for L(Π), especially the data-to-model terms and combinatorial factors, and specify how Laplace smoothing affects code lengths and alignment decisions.
- Over-weighting α: How should users select α? Can you show sensitivity analyses and measures that ensure stability of induced rules across α and across resampled data?
- Scalability: What are typical values of |R| and A in your datasets? Can you report runtime/memory for the four-language experiment and discuss optimizations (e.g., pruning, A* search, beam search)?

Additional comments
- Reproducibility: Please release code and data pre-processing scripts; clarify how missing entries are handled (you filtered to full intersections, but the method claims to be gap-agnostic).
- Presentation: Some references are malformed; equations could be numbered; define symbols at first use (e.g., Q(·) in sDL definition).
- Evaluation breadth: Consider adding a quantitative evaluation of rule quality (e.g., precision/recall against curated correspondences) and robustness checks (bootstrap of sDL and NCD).

Overall recommendation
Borderline. The MDL formulation and multiway sDL analysis are interesting and potentially valuable, but the current evaluation is too limited and orthography-driven to substantiate stronger claims. Strengthening empirical comparisons, clarifying technical details, and demonstrating scalability would significantly improve the paper.

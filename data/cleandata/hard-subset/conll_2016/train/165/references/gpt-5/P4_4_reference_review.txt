Summary and contributions
This paper proposes an MDL-based framework to automatically discover string-level correspondence rules across two or more languages without imposing parametric assumptions on the form or distribution of correspondences. Models consist of multiway correspondence tuples over alphabets, with a two-part code that jointly accounts for the model (alphabets and rule table) and the data (alignments). Alignments are treated as shortest paths in a graph (via Dijkstra), and parameters are learned by an EM-like procedure with greedy merges of rules to reduce total description length. The approach is demonstrated on Slavic languages using Swadesh lists and additional cognates, yielding pairwise NCD distances and a phylogenetic tree, and a four-language analysis via “shared Description Lengths” (sDL) to quantify structure shared by any subset of languages. The paper also introduces optional “data over-weighting” to control granularity of discovered rules.

Contributions (as assessed here):
1) An MDL coding formulation (PS-2) for multi-language correspondence rules and a deterministic EM/merge training algorithm with shortest-path alignment, requiring no prior assumptions on rule distributions.
2) An information-theoretic analysis of multi-language similarity via sDL that quantifies structure shared across arbitrary subsets of languages, demonstrated on four Slavic languages.
3) A qualitative empirical study on 13 Slavic languages showing that pairwise NCD-based trees align with standard linguistic groupings and that the learned rules capture both orthographic and phonological correspondences.

Strengths
1) Principled, assumption-light formulation: The MDL framework yields an objective criterion that balances model and data description length, avoiding strong parametric assumptions about correspondence shapes. The two-part code for alphabets and rule tables, combined with data-to-model coding, is a coherent way to formalize the discovery problem and to prioritize statistically supported rules.
2) Multi-language generality and interpretability: Modeling rules as N-ary tuples over alphabets naturally extends beyond pairwise settings. The method discovers interpretable correspondences (e.g., cz–č, g–h, liquid metathesis, palatalization, endings), providing human-readable patterns that are useful for linguistic analysis and downstream tasks.
3) Sound alignment formulation: Casting alignment as a shortest-path problem with rule costs derived from the MDL code is technically sound and deterministic. The use of Dijkstra and partial order reduction is appropriate for exact decoding under local rule costs.
4) Empirical validation consistent with linguistic knowledge: The NCD-based phylogeny for 13 Slavic languages qualitatively matches established classifications (e.g., CS–PL, RU–BG affinity; BG/MK outliers). The four-way sDL analysis yields fine-grained insights into which subsets share more structure, going beyond pairwise similarity and highlighting cross-subset regularities.
5) Flexible incorporation of prior knowledge and granularity control: The framework permits seeding with known rules and optionally adjusting data weight to expose longer rules when data are sparse. This makes the approach adaptable for practical scenarios (e.g., low-resource settings).

Weaknesses
1) Limited quantitative evaluation and lack of baselines: The paper primarily provides qualitative evidence. There is no quantitative comparison to established methods (e.g., pair-HMMs/PHMMs for cognate alignment, PMI/ALINE-based correspondences, Snyder & Barzilay 2008/2010) nor objective metrics for the tree (e.g., Robinson–Foulds distance to a gold classification) or for rule quality against any ground truth. This makes it hard to assess performance gains or trade-offs.
2) Insufficient specification of the code and decodability: Several coding components (data-to-model codes, weak number composition terms, code(π) definition and usage counts) are sketched but not fully derived. The paper does not rigorously prove prefix-free decodability or provide a complete description needed to re-implement the exact code lengths. Notational issues (e.g., inconsistent use of TΠ, |Π|, and count definitions) hamper reproducibility.
3) Scalability and complexity constraints: The approach reportedly handles up to five languages with hours of runtime on modest hardware. Complexity expressions (O(|D|·|R|^2) for E-step, O(|D|·A^2/2) for M-step) are given without precise definitions of R and A or empirical scaling curves. There is no analysis of how alphabet size, rule table size, and data length affect memory/runtime, nor strategies to scale to more languages or larger datasets.
4) Dataset limitations and potential confounds: The data are raw orthography from Wiktionary Swadesh lists (207 items per language) plus curated cognates, with no train/test separation or noise analysis. Script differences (Latin vs. Cyrillic) are shown to affect distances; there is no normalization to disentangle orthographic from phonological effects. The evaluation remains within closely related Slavic languages; generalization to more distant language families is not explored.
5) Use of data over-weighting departs from MDL objectivity: While practically useful, introducing α ≠ 1 modifies the criterion and may incentivize longer, idiosyncratic rules. The paper lacks guidelines for choosing α, an analysis of its impact on overfitting/underfitting, or safeguards to preserve comparability across runs.

Questions to authors (optional)
- Coding details: Please provide a complete, decodable specification of all code components. In particular: how exactly is code(π) computed from counts; how are the last two terms in L(Π) derived; what is the precise enumeration underpinning the “weak number composition” terms for both rules and alphabet symbol counts; and can you provide a proof sketch of prefix-free decodability for the full two-part code?
- Alignment graph construction: How are edges generated when empty strings are allowed (insertions/deletions)? Are there constraints to prevent pathological merges (e.g., very long idiosyncratic rules)? How is partial order reduction instantiated?
- Convergence and initialization: What is the termination criterion for EM/merging? How sensitive are results to initializing with null vs. seeded rules? Any safeguards against getting stuck in poor local minima?
- Quantitative evaluation: Can you report a Robinson–Foulds distance of the inferred tree to a standard Slavic phylogeny, and compare NCD to simpler baselines (e.g., average Levenshtein, PHMM likelihoods)? For rules, can you measure precision/recall against a small expert-annotated set of correspondences?
- Sensitivity analyses: How do Laplace smoothing and the α over-weighting parameter affect learned rule lengths, NCDs, and sDL distributions? Please include ablations.
- Reproducibility: Will you release code, processed datasets, and scripts to compute NCD, sDL, and trees? Please also provide runtime/memory profiles versus N, |D|, and alphabet size.

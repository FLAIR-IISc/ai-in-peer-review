- Proposes an MDL-based framework to learn many-to-many string correspondences across multiple languages using a two-part code (alphabets + rule table) and alignment as a shortest-path problem with an EM-like greedy rule-merge learner and optional data over-weighting.

- Contribution: MDL formulation and deterministic learning algorithm for multi-language, many-to-many correspondence rules with a practical alignment-as-shortest-path and greedy merge strategy.

- Contribution: Uses MDL code lengths to derive pairwise NCDs and introduces shared Description Length (sDL) to quantify structure shared by arbitrary language subsets.

- Contribution: Introduces a simple data over-weighting hyperparameter to control the granularity of learned rules.

- Strength: Clear, general MDL framing that unifies model selection and alignment costs and extends serial-episode MDL to multi-tape string correspondences.

- Strength: Deterministic, interpretable learning with transparent greedy merges and option to inject linguistic priors; discovers phonological, orthographic, morphological, and metathesis patterns.

- Strength: Native multi-language capability beyond pairwise models and an sDL measure to analyze rules on arbitrary subsets, producing linguistically plausible groupings.

- Strength: Coherent information-theoretic evaluation using code lengths to derive NCD and shared description measures without ad-hoc scoring.

- Strength: Practical demonstration on Slavic data that yields phylogenies broadly matching expectations, giving face validity.

- Weakness: Largely qualitative evaluation with no comparisons to established baselines (PMI aligners, pair-HMMs/WFSTs, LexStat/SCA, etc.) or quantitative metrics (precision/recall, held-out compression, tree accuracy).

- Weakness: Working on raw orthography (mixed scripts) introduces script and orthographic confounds that undermine linguistic claims; phonemic/IPA normalization is needed.

- Weakness: Insufficient specification of MDL code components (L(Π), code(π), code(σ)), usage probability estimates, and potential double-counting—raising validity concerns for NCD comparisons.

- Weakness: Risk of lexeme-specific overfitting—model can learn stem-level idiosyncrasies (item memorization) conflated with systematic correspondences; over-weighting exacerbates this.

- Weakness: Scalability concerns and lack of implementation/practical details (alignment complexity, pruning, Laplace settings, tie-breaking, α selection); reproducibility not addressed.

- Weakness: No mechanism to separate generalizable sound changes from lexeme-level rules (e.g., minimum support, context conditioning, priors) beyond MDL penalties.

- Question: Please provide precise derivation of L(Π) and L(Σi), how code(σ)/code(π) are computed, and proof that the two-part code is valid and comparable across model classes.

- Question: How is L(X,X) computed for NCD—do you retrain on duplicated data or reuse a model—and how do you ensure a fixed compressor across pairs?

- Question: Can you report quantitative evaluations: correspondence recovery on a gold list, held-out compression, cognate ID accuracy, and phylogenetic tree accuracy with bootstraps/RF distances?

- Question: How sensitive are results to orthography, and what changes when using phonemic transcriptions or grapheme normalization across scripts?

- Question: How often are learned rules lexeme-specific vs systematic, and can you add constraints (minimum support, context conditioning) or priors to reduce item-specific rules?

- Question: What is the effect of over-weighting hyperparameter α on rule number/length, compression, and analyses, and can you propose a principled MDL-consistent alternative?

- Question: Elaborate on partial order reduction, pruning/heuristics for alignment scalability, and whether any pruning preserves optimality guarantees.

- Question: Will you compare against PMI/PHMM-based multi-segment alignment baselines and LexStat/SCA distances on the same data?

- Additional comment: Related work is incomplete—add references from historical linguistics and computational phylogenetics (List/LexStat, Jäger, Bouchard-Côté, ASJP, etc.).

- Additional comment: Consider a two-layer model separating orthographic and phonological correspondences or a WFST pre-normalization to reduce script confounds.

- Additional comment: sDL should possibly weight rule contributions by data cost saved (usage-weighted) rather than summing model costs L(π).

- Overall: Promising, principled MDL approach with interpretable multi-language outputs, but currently limited by methodological ambiguities, orthographic confounds, lack of baselines/quantitative evaluation, and scalability/reproducibility issues.

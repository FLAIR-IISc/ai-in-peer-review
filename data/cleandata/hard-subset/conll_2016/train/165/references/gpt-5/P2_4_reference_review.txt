Summary and main contributions
- A principled MDL framework (two-part code, “PS-2”) for discovering correspondence rules among more than two languages directly from raw orthographic strings, with no parametric assumptions on rule shape/distribution. The method jointly optimizes model cost and data cost, uses shortest-path alignment per item, and a deterministic EM-like training procedure with greedy rule merges.
- An analysis toolkit built on the learned models: (a) pairwise Normalized Compression Distance (NCD) to derive a phylogeny; (b) a multi-language “shared Description Length” (sDL) that quantifies structure shared by any subset of languages, enabling fine-grained similarity assessments beyond pairwise comparisons.
- A case study on Slavic languages showing that the method recovers plausible correspondences and a phylogeny consistent with known subfamilies, and that sDL exposes subgroup structure and script/orthography effects; exploration of “data over-weighting” to control correspondence granularity.

Strengths
- Principled, compact modeling: The MDL formulation is appropriate for the stated goal (discover correspondences without strong priors), with a clear decomposition into model and data codes. The coding choices (LN for lengths, data-to-model codes for usages) lead to an interpretable objective and connect improvements to actual compression, which is attractive for linguistic discovery where assumptions should be minimal.
- Multi-language capability and novel analysis: Most prior work focuses on pairwise correspondences. Here, the same machinery handles N>2, and sDL gives a concrete, information-theoretic way to quantify similarities within any subset. The four-way CS–PL–RU–BG analysis illustrates this advantage: it reveals which correspondences are statistically supported at different arities and provides explanatory power (e.g., RU–BG vs CS–PL contributions).
- Determinism with the option to seed domain knowledge: The algorithm is deterministic given initialization, which is valuable for replicable linguistic analysis. Allowing initialization with expert rules makes the framework usable by linguists who want to inject knowledge while still benefiting from MDL-driven refinement.
- Results align with linguistic expectations and are interpretable: The NCD matrix and neighbor-joining tree roughly recover Slavic subgroups; the script effect (SC Latin vs Cyrillic) and the Bulgarian/Macedonian morphological differences are reflected in the distances. Extracted rules (e.g., cz–č, g–h, ł–l, -eć–-et) are readily interpretable as orthographic/phonological correspondences or morphological endings, supporting face validity.
- Practical control of granularity: Data over-weighting is a pragmatic knob to surface longer or rarer patterns when the corpus is small, yet the ordering by statistical significance is preserved. This can be genuinely useful for tasks like cognate reconstruction where cross-language rules are the focus.

Weaknesses
- Evaluation lacks quantitative rigor and baselines: The paper relies mainly on qualitative inspection and a single phylogeny. There is no comparison to established correspondence/weighted alignment methods (e.g., PMI-based sound correspondences, Pair-HMM/ALINE-like models, Bayesian phonological change models), no intrinsic metrics (precision/recall of induced rules against gold correspondences), and no extrinsic evaluation (e.g., cognate identification, proto-form reconstruction). The tree comparison is descriptive; there is no statistical support (bootstrap, quartet scores) or competing phylogenies.
- Reproducibility and implementation details are insufficient: Critical details are missing or high-level: precise smoothing (Laplace) constants, stopping/merge-selection criteria, partial order reduction method, alignment graph construction constraints, and parameter choices for over-weighting. Code and processed datasets are not provided. Claims like “up to five languages in a few hours” are anecdotal; no systematic runtime/memory profiling, and scalability is unclear.
- Coding design choices need stronger justification/ablation: The specific two-part code for Π (including LN(TΠ) and the combinatorial term) and the use of data-to-model codes for rule usages and alphabets are plausible but not fully derived or contrasted with alternatives. The impact of these choices on induced rules and distances is not tested; MDL results can be sensitive to code design.
- Optimization is heuristic with unquantified stability: The EM-like procedure with greedy pair merges has no convergence guarantees and is acknowledged to miss subtle patterns. There is no sensitivity analysis to initialization (null vs seeded), to the merge schedule, or to the over-weighting parameter α. Robustness to noise (mis-cognates, spelling variation) and to missing data is not assessed, even though the method is said to be “agnostic to gaps.”
- Limited scope and preprocessing: Working on raw orthography confounds phonology with script and idiosyncratic spelling, which the results explicitly reflect (e.g., Serbian script differences). No phonological normalization or grapheme-to-phoneme layer is attempted, which limits linguistic generality. Experiments exclude tuples with gaps and focus on small lists; it is unclear how the method fares on larger, noisier lexicons or on morphologically rich paradigms.

Questions to authors
- Can you release code and processed data, and document exact hyperparameters (smoothing constants, merge stopping criterion, partial order reduction strategy) to enable replication?
- How sensitive are results to coding choices? Have you tried alternative universal codes or different encodings of rule usages/subset classes? Any ablation on L(Π) components?
- How do you set the over-weighting factor α in practice? Can you quantify its effect on false positives/overfitting (e.g., by injecting noise or using held-out validation)?
- How robust is the method to noise (non-cognates within tuples) and to missing entries? Can you report experiments with controlled noise levels and with gaps included?
- Can you provide quantitative comparisons to standard pairwise methods (e.g., PMI-weighted alignments, ALINE/Pair-HMM, LexStat-style models) and demonstrate the multi-language advantage beyond qualitative examples?
- What are the practical scaling limits with respect to number of languages N, average word length, and dataset size? Are there approximation strategies (beam search, pruning) that maintain quality while improving efficiency?

Overall assessment
- Originality: Moderate. MDL for pattern discovery is known, but applying a carefully designed two-part code to multi-language correspondence induction and sDL analysis is a useful contribution.
- Soundness: Conceptually sound, but some coding choices and optimization heuristics need deeper justification and quantitative validation.
- Clarity: Generally clear with good intuition and examples; more algorithmic detail is needed for replication.
- Significance: Promising for multi-language analysis, but current evaluation is too limited; stronger empirical comparisons would raise impact.
- Recommendation: Borderline accept if the venue values methodological novelty and interpretability; otherwise, substantial additional evaluation and reproducibility work is needed.

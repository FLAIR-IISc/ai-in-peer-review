Summary and claimed contributions
This paper proposes an MDL-based framework for discovering many-to-many string correspondences across two or more languages, learned deterministically from cognate lists without imposing parametric assumptions on the correspondence distribution. The model defines a two-part code: a model code for alphabets and a rule table Π of correspondence tuples, and a data code for aligning each cognate tuple via Π. Alignment is framed as a shortest-path problem; learning proceeds by an EM-like loop with greedy rule merging, with an optional “data over-weighting” knob to encourage larger rules. The approach is evaluated on Slavic Swadesh lists and a small curated set, yielding pairwise NCD distances and a phylogeny, and a four-way analysis with a new “shared Description Length” (sDL) measure to quantify structure shared by language subsets.

Main contributions (as I see them)
1) An MDL formulation and deterministic learning algorithm for multi-language, many-to-many correspondence rules, including a practical alignment-as-shortest-path procedure and greedy rule merging strategy inspired by MDL pattern mining.
2) An application of MDL code lengths to derive pairwise NCDs and an sDL-based analysis to quantify shared structure among all subsets of languages, demonstrated on Slavic data.
3) A simple “data over-weighting” mechanism to control the granularity of learned rules.

Strengths
- Conceptual clarity and generality of the MDL framing: The paper cleanly formulates correspondence discovery as a two-part MDL code, avoids fixed parametric assumptions, and unifies model selection and alignment costs in a principled objective. The extension from serial episode MDL to multi-tape string correspondences is well-motivated.

- Deterministic, interpretable learning with the ability to inject linguistic knowledge: The greedy merge procedure and initialization from singleton rules (or user-supplied seeds) provides transparency and controllability valued by linguists. The examples show the method can discover phonological, orthographic, and morphological correspondences and even capture metathesis patterns.

- Multi-language capability beyond pairwise models: The framework natively handles N>2 languages and introduces sDL to quantify contributions of rules defined on arbitrary subsets. This is a meaningful step beyond pairwise-only approaches and produces interesting analyses (e.g., RU–BG vs CS–PL groupings).

- Information-theoretic evaluation and outputs: Using code lengths to derive NCD and shared description lengths is coherent with the MDL objective and yields a natural similarity measure without ad hoc scoring schemes.

- Practical demonstration on a typologically coherent family: The Slavic experiments are appropriate and the phylogeny broadly matches linguistic expectations, which gives face validity.

Weaknesses
- Limited evaluation and lack of baselines: The empirical section is largely qualitative. There is no comparison against established methods for cognate alignment and correspondence inference (e.g., PMI-based many-to-many aligners, pair-HMMs/WFSTs, LexStat/SCA in LingPy, Bouchard-Côté et al. for sound change, Jäger’s phylogenetic models). No quantitative metrics (precision/recall on known correspondences, cognate identification accuracy, held-out compression, or tree accuracy such as RF distance/bootstraps) are reported. The NCD-based tree “looks right,” but this is insufficient as a primary evaluation.

- Orthography rather than phonemic representation confounds findings: The method operates on raw orthography across different scripts (Latin vs Cyrillic), and the results show script effects (SCl vs SCc). This undermines claims about linguistic correspondences and genealogical signal, as script- and orthography-specific idiosyncrasies (digraphs, diacritics) drive parts of the model and NCD. A phonemic/IPA layer or grapheme-phoneme normalization is needed to make cross-language correspondences linguistically meaningful.

- Methodological ambiguities in the MDL code and usage probabilities: The description of L(Π) and code(π) is underspecified. It is unclear how rule usage probabilities are estimated, how the “data-to-model code” is applied for rules vs alphabets, and whether costs are double-counted between model and data. The combinatorial term choices and their justifications (weak composition) need precise derivations to ensure the code is valid, prefix-free, and comparable across different language subsets—crucial for NCD.

- Learning dynamics and risk of lexeme-specific overfitting: The model allows arbitrary string tuples and celebrates learning stem correspondences (e.g., ziem~zem). This conflates systematic sound correspondences with item-specific idiosyncrasies, potentially inflating perceived similarity. There is no mechanism to separate generalizable sound changes from lexeme-level memorization (e.g., via context constraints, rule regularization, or minimum support thresholds beyond MDL penalties). The “over-weighting” trick further biases towards longer, potentially idiosyncratic rules.

- Scalability and reproducibility: Alignment per word uses Dijkstra on large graphs; the method is reported to handle “up to five languages” on small datasets, with memory/time bottlenecks. No details on partial order reduction, pruning, or complexity constants are provided. Code/data availability is not stated. The choice of Laplace smoothing, merge tie-breaking, stopping criteria, and α for over-weighting are not rigorously specified, raising concerns about reproducibility.

Questions to authors
- Can you provide a precise derivation of L(Π) and L(Σi), including how code(σ) and code(π) are computed, and prove that the overall two-part code is valid and comparable across model classes used for NCD (i.e., the “compressor” is fixed)?

- How is L(X,X) computed for NCD—do you retrain on duplicated X, or reuse a model? How do you ensure the same model class and coding scheme across pairs so that NCD is meaningful?

- Can you report quantitative evaluations: (a) known correspondence recovery on a gold list; (b) held-out compression; (c) cognate identification accuracy if the model is used for scoring; (d) phylogenetic tree accuracy with bootstraps?

- How sensitive are results to orthography? What changes when using phonemic transcriptions or normalized grapheme mappings across scripts?

- How often do learned rules correspond to lexeme-specific stems vs systematic correspondences? Can you add constraints (e.g., minimum support, context conditioning) or a prior that downweights item-specific rules?

- What is the effect of the over-weighting hyperparameter α on the number/length of rules, compression, and downstream analyses? Can you propose a principled selection criterion or MDL-consistent variant (e.g., hierarchical model class or structural penalty) instead of ad hoc α?

- Can you elaborate on partial order reduction and any pruning/heuristics used in alignment to scale beyond five languages? Are there guarantees on optimality when pruning?

- Could you compare against PMI/PHMM-based multi-segment alignment baselines and LexStat/SCA distances on the same data?

Additional comments
- Related work is incomplete; please consider adding references from historical linguistics and computational phylogenetics (e.g., List/LexStat, Jäger, Rama, Bouchard-Côté et al., Birket-Smith/ASJP) and recent multi-lingual correspondence models.
- Consider separating orthographic and phonological correspondences (two-layer model) to reduce confounds. A WFST pre-normalization stage might help.
- The sDL definition currently sums model costs L(π); a usage-weighted contribution (data cost saved) may better capture shared regularities.

Overall: The paper offers an appealing, principled MDL perspective and interesting qualitative results, but the lack of rigorous evaluation, orthography confounds, and methodological ambiguities limit its current impact. With clearer coding derivations, stronger baselines, and phonemic experiments, this could be a valuable contribution.

Summary
The paper proposes an MDL-based framework to discover systematic string-level correspondences among multiple languages without imposing parametric assumptions on the shape or distribution of correspondences. The authors define a two-part code (“PS-2 code”) for models and data, align cognate tuples via shortest-path search (Dijkstra), and optimize the rule inventory with an EM-like procedure whose M-step greedily merges patterns that reduce total description length. They introduce a data over-weighting knob to push models toward longer, more specific correspondences when evidence is marginal. Experiments on Slavic languages use pairwise Normalized Compression Distance (NCD) to derive a phylogenetic tree that broadly agrees with linguistic classifications, and a four-language analysis quantifies shared description lengths (sDL) across all subsets, illustrating fine-grained multiway similarities. Qualitative examples show phonological and orthographic correspondences learned by the method.

Main contributions (as I see them)
1) A deterministic MDL-based method for learning multi-language string correspondence rules and alignments, including a concrete coding scheme and EM-style optimization with a shortest-path alignment procedure.
2) An information-theoretic analysis of multi-language similarity via shared description lengths that goes beyond pairwise comparisons, demonstrated on Slavic data.
3) A practical knob (data over-weighting) for controlling the granularity of discovered rules, enabling exploratory analysis.

Strengths
- Principled modeling: The formulation in MDL avoids ad hoc priors or pre-specified rule shapes and ties model complexity directly to compressibility. The two-part code and alignment-as-shortest-path decomposition are well-motivated and align with prior MDL work in pattern mining (e.g., SQS/Krimp).
- Multi-language capability: Unlike many cognate/correspondence models that are pairwise, the method naturally handles N>2 languages and quantifies shared structure across all subsets. The sDL analysis is a useful descriptive statistic that surfaces patterns not visible in pairwise trees.
- Deterministic pipeline with expert input: The deterministic training from a null alignment and the option to seed with user-supplied rules make the approach attractive for linguists seeking reproducible analyses while allowing domain knowledge to be injected.
- Qualitative plausibility: The learned correspondences include expected phonological, orthographic, and morphological patterns (e.g., cz~č, g~h, palatalizations, liquid metathesis), and the NCD-derived tree broadly matches Slavic subgrouping, with reasonable explanations for outliers (e.g., BG/MK).
- Clear computational idea: Casting alignment as a weighted shortest-path problem with partial order reduction is sound and yields exact alignments for a given rule table, which is valuable for interpretability.

Weaknesses
- Limited empirical evaluation and lack of baselines: The paper does not compare against established approaches for learned string correspondences or cognate alignment (e.g., ALINE, PMI-based aligners, pair HMMs/transducers, Needleman–Wunsch with learned costs, Bayesian cognate models). Evaluation is largely qualitative plus a phylogenetic tree. There is no intrinsic metric (e.g., recovery of known correspondences, edit-cost prediction accuracy, or held-out description length). This makes it hard to assess effectiveness and novelty relative to the literature.
- Ambiguities in the coding scheme and two-part MDL separation: L(Π) appears to include a data-to-model code for rule usages (LN(TΠ)+log(…)), yet those counts derive from data. The rationale for placing usage enumeration in the model term (rather than in L(D|M) or using an NML/prequential code) is under-explained and risks double-counting or bias. Similarly, details on code(σ) estimation, alphabets across languages (shared vs per-language), and handling of epsilon symbols are not fully specified.
- Scalability and complexity: The reported bottlenecks limit experiments to up to five languages and a few hundred items on modest hardware, with E-step O(|D|·|R|^2). There is no timing or memory profile, pruning strategy evaluation, or demonstration on larger datasets. This undercuts claims of efficiency and practical applicability.
- Over-weighting undermines the objective MDL criterion: While useful in practice, the α>1 scheme departs from MDL optimality and introduces a hyperparameter without guidance for selection, validation, or sensitivity analysis. The paper uses over-weighting to remove singletons but provides no quantitative assessment of its effect on generalization or stability.
- Data and evaluation scope: Experiments are restricted to orthographic forms in Slavic and rely on Wiktionary lists and a small set of internationalisms. Orthography conflates phonology with script conventions (e.g., digraphs), and there is no phonological normalization. No cross-family validation, ablations (e.g., merging strategy), or held-out evaluations are provided. The reference list also contains formatting errors and incomplete entries.

Questions to authors
- MDL coding: Please clarify why the enumeration of rule usages (LN(TΠ)+log((TΠ−1 choose |Π|−1))) belongs in L(Π) rather than L(D|M). Are you using an implicit prequential/NML approximation? How are code(σ) and code(π) estimated—MLE with Laplace? Are alphabets per language or pooled? Is epsilon a symbol with its own code length?
- Constraints on rules: What is the maximum rule length during merging? How do you define “compatible” rules for merging and handle overlaps? Are null components (insertions/deletions) allowed freely, and how do you prevent degenerate rules (e.g., large deletions)?
- Determinism and local optima: Given greedy merges, what tie-breaking is used? Did you explore multiple restarts or annealing, and how much variance in results did you observe?
- Complexity/pruning: Can you provide empirical time/memory as a function of |D|, average token length, and |Π|? Did you try A* with admissible heuristics, beam search, or batching to scale?
- Over-weighting α: How is α selected? Can you report sensitivity curves (e.g., number/length of multi-language rules vs α) and any held-out sDL or NCD to detect overfitting?
- Evaluation: Could you include comparisons to at least one strong baseline (e.g., PMI-based alignment, pair-HMM) on intrinsic metrics (e.g., alignment F1 vs a gold set, or ability to predict held-out cognate forms)? Any cross-family tests (e.g., Romance/Germanic) to check generality?

Overall assessment
The paper presents an interesting and principled MDL view on multi-language correspondences, with appealing interpretability and some compelling qualitative findings. However, the current evaluation is too limited, the MDL coding choices need clearer justification, scalability is not demonstrated, and the role of over-weighting lacks rigorous analysis. Given CoNLL standards, I lean toward rejection in the current form. I encourage the authors to (i) provide baselines and intrinsic/held-out evaluations, (ii) clarify and, if needed, revise the coding scheme to cleanly separate model/data terms, (iii) include scalability improvements and profiling, (iv) add phonological normalization experiments and cross-family tests, and (v) analyze sensitivity to α and initialization. These steps would substantially strengthen the paper’s claims and impact.

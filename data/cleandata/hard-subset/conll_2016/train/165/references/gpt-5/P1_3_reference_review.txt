Summary
This paper proposes an MDL-based framework to discover string-level correspondence rules across multiple languages simultaneously. A model consists of a set of multi-way correspondences (tuples of strings, one per language, possibly empty) and symbol unigrams; the optimal model minimizes a two-part MDL code: model cost plus data encoding cost. Training alternates between aligning cognate tuples to current rules (E-step; shortest-path alignment via Dijkstra on a lattice of index tuples) and merging compatible rules that yield the largest net description-length decrease (M-step), starting from singleton rules. The paper also introduces a “data over-weighting” parameter to encourage longer rules. Experiments on Slavic languages use pairwise MDL-derived NCD to produce a phylogeny consistent with linguistic classifications, and a 4-language analysis quantifies shared structure via “shared description length.”

Main contributions (as I see them)
1) An MDL formulation and deterministic EM-style algorithm to infer multi-way string correspondences (beyond pairwise) from raw orthographic cognate lists, including an alignment procedure supporting arbitrarily long rules.
2) An information-theoretic analysis of inter-language similarity that goes beyond pairwise distances by quantifying contributions of rules shared by any subset of languages.
3) Empirical case study on Slavic languages showing plausible correspondences, a phylogeny consistent with prior knowledge, and an exploration of a data over-weighting knob to control rule granularity.

Strengths
- Methodological generality and multi-way modeling: The model natively supports correspondences across N languages, a capability that many prior approaches lack or approximate via pairwise aggregation. The ability to learn rules with empty components allows asymmetric participation and reveals structure shared by arbitrary subsets.
- MDL-based objective with interpretable trade-offs: The two-part code is well motivated and avoids strong parametric assumptions about rule shapes. The cost of adding a rule versus using it in data provides an understandable mechanism to control complexity and discourages overfitting.
- Deterministic EM-style learning with explicit alignment: The tight coupling of alignment and rule induction is appealing; the shortest-path formulation for alignment with arbitrary-length rules is natural and leads to exact alignments under the current model.
- Qualitative interpretability of learned rules: The examples recover known orthographic/phonological mappings (e.g., cz–č, g–h, palatalizations, metathesis), including multi-character rules and morphological endings. These are linguistically meaningful and show that the approach can capture patterns beyond single-character edits.
- Subset-based similarity analysis: The “shared description length” idea is a nice step toward quantifying contributions of structure shared by any subset (e.g., CS-PL vs RU-BG vs four-way), offering more nuanced insight than a single pairwise distance or a tree.

Weaknesses
- Evaluation is limited and lacks baselines: The core evidence is a phylogeny consistent with known Slavic groupings and a handful of illustrative rules. There is no quantitative comparison against standard baselines (e.g., average Levenshtein distances, PHMM/ALINE/PairLexStat-based distances, cognate-alignment tools, or other rule-induction models). No intrinsic evaluation (precision/recall of correspondences against expert annotations) or extrinsic task (e.g., cognate identification, cross-lingual prediction) is provided. Without baselines or ablations, it is difficult to attribute the observed tree quality to the proposed modeling rather than general string similarity.
- Ambiguities and gaps in the coding scheme: The L(Π) formula and the “data-to-model” parts (weak number composition) are insufficiently specified for replication. It is unclear how counts for rule usages translate to code lengths in detail, whether counts are smoothed beyond Laplace, and whether the last two terms of L(Π) risk double-counting with the data term. Relatedly, using L(π) (a model code) to define “shared description lengths” may not reflect how much each subset contributes to data encoding; data-term contributions might be more faithful.
- Over-weighting undermines MDL objectivity and is under-justified: Introducing α to upweight data is practical but departs from the stated goal of objective modeling. The paper uses it to “remove singleton rules” without a principled criterion, and there is no analysis of sensitivity, selection guidelines, or failure cases where over-weighting induces spurious rules.
- Scalability and complexity: Alignment uses Dijkstra on a high-dimensional lattice; the paper acknowledges memory/runtime bottlenecks and only reports up to five languages on small datasets (Swadesh lists and 778-item set). No complexity bounds for graph size are given, and there is no engineering to scale (e.g., WFST composition, A* with admissible heuristics, pruning/beam search, max rule length, batching).
- Data and preprocessing choices may bias results: All experiments are on raw orthography (including mixed scripts) without phonological normalization or transliteration. The inclusion of Serbo-Croatian in two scripts and the observation that script choice affects similarity suggests script-level effects may dominate. There is no control experiment (e.g., phonemicized data) to validate that correspondences reflect phonology rather than orthography.

Questions to Authors
- Clarify L(Π): Please provide a precise derivation of the model code, especially the last two terms LN(TΠ) + log((TΠ−1 choose |Π|−1)), and how they relate to rule usage probabilities used in data encoding. How are rule usage counts converted to code(π)? Are the same counts used in both model and data codes?
- Shared description length: Why measure sDL using model costs L(π) rather than the data-term contribution attributable to those rules? Would a data-based measure change the conclusions?
- Alignment graph: What is the exact state-space and edge set? Do you allow ε on arbitrary sides, and how do you prevent degenerate alignments that overuse ε expansions? Any max rule length or context constraints?
- EM details: What are the stopping criteria, tie-breaking in merges, and convergence behavior? How sensitive are results to initialization (null vs seed rules)?
- Over-weighting α: How is α chosen in practice? Can you provide sensitivity analysis and a principled guideline? Does over-weighting sometimes produce linguistically spurious merges?
- Baselines and robustness: Can you compare your NCD tree to one built from simple edit-distance baselines or PHMM-based distances on the same data? How robust are results to noise or to removing subsets of concepts?

Additional comments and suggestions
- Provide code and data splits, with exact preprocessing (tokenization of digraphs, casing, handling diacritics, Cyrillic/Latin mapping). Consider evaluating on phonemic transcriptions.
- Add quantitative evaluations: (i) alignments against expert gold, (ii) downstream prediction (e.g., reconstructing missing cognates), (iii) cognate clustering accuracy.
- Explore scalability improvements (WFST-based alignment, A* with consistent heuristics, pruning) and report runtime/memory as a function of N, vocabulary size, and rule inventory.
- Include ablations: effect of α, effect of allowing ε on multiple sides, maximum rule length constraints, and Laplace smoothing strength.
- Clean up references and notation; several citations are incomplete or misformatted.

Summary and contributions
This paper proposes an MDL-based framework to discover string-level correspondence rules across more than two languages, learned deterministically from raw orthography with no assumed prior distribution over rule shapes. The method defines a two-part code over a set of multi-language substring correspondences, computes optimal alignments via shortest paths (Dijkstra) given a rule table, and iteratively improves the rule set by a deterministic EM-like procedure that greedily merges rules to minimize total description length. The approach is demonstrated on Slavic cognate lists, yielding pairwise Normalized Compression Distances (NCDs) that recover known phylogenetic groupings and a four-language analysis that quantifies “shared description lengths” across all language subsets. The paper also introduces a “data over-weighting” knob to force larger, more specific correspondences when statistical evidence is marginal.

As I see them, the main contributions are:
- An MDL formulation and deterministic learning algorithm for multi-way (N>2) substring correspondences with arbitrary length and sparsity, including an exact alignment procedure given a rule table.
- A fine-grained, information-theoretic analysis of similarity among all language subsets via shared description lengths, enabling beyond-pairwise quantification.
- An empirical case study on Slavic languages showing that pairwise NCDs recover linguistic groupings and that multi-way correspondences capture expected regularities; plus a practical control (data over-weighting) for rule granularity.

Strengths
- Conceptual novelty and scope: Moving beyond pairwise models to discover multi-language correspondences directly is genuinely interesting. The MDL framing avoids strong parametric assumptions and admits sparse, multi-way rules with epsilons; this affords flexibility to capture both orthographic and phonological regularities.
- Sound alignment subproblem and determinism: Given a rule table, finding an optimal alignment via shortest-path search is principled. The deterministic training (singleton initialization and greedy merges) and the possibility to seed with linguistic rules support reproducibility and expert-guided use.
- Fine-grained similarity analysis: The sDL metric across all subsets is a neat, information-theoretic device. The four-way CS–PL–RU–BG analysis illustrates how the approach can expose both broad groupings (CS–PL, RU–BG) and asymmetries in how much structure each language contributes.
- Empirical plausibility: The pairwise NCDs and the derived tree match standard Slavic subgrouping, and the qualitative correspondences shown (e.g., cz–č, g–h, liquid metathesis) are linguistically sensible. The observation about script effects (Latin vs Cyrillic Serbo-Croatian) is also coherent.
- Practical control of granularity: The “data over-weighting” mechanism is a pragmatic way to surface longer rules when desired for applications like reconstruction; the paper is transparent about the subjectivity this introduces.

Weaknesses
- Limited empirical validation and lack of baselines: The evaluation uses small cognate lists (hundreds of items) and provides no quantitative comparison against established methods (e.g., PMI-based sound correspondences, pair HMMs, Bayesian cognate alignment, Levenshtein-/PMI-based phylogenies). There is no assessment of rule quality (precision/recall against known correspondence sets), no downstream task (cognate detection, proto-form reconstruction), and no statistical tests on tree quality (e.g., RF distance, bootstrapping). As a result, it is hard to judge the advantage over simpler baselines or prior art.
- Under-specified coding and reproducibility details: Several parts of the code length design are only sketched. The “data-to-model code” and use of weak compositions are mentioned but not fully derived for both rule usage and symbol unigrams; some formulas (e.g., the multinomial composition terms) and assumptions (e.g., “we know that none of the symbols … will have a non-zero count”) are unclear or possibly misstated. Key algorithmic details are missing: stopping criteria, number of merges per M-step, handling of overlapping/competing rules, tie-breaking, Laplace parameter, and particulars of “partial order reduction.” No code or full pseudo-code is provided, limiting reproducibility.
- Claims of objectivity vs modeling choices: Although MDL reduces reliance on priors, the specific code design (e.g., choice of universal integers, encoding by subsets, allowing epsilons) and the over-weighting knob introduce subjective elements that can materially affect which rules are learned. Orthographic input conflates script and phonology; the observed script-driven effects arguably reflect writing system conventions rather than deeper sound correspondences.
- Scalability and convergence: The alignment step can be memory-intensive; reported practical limits are N≤5 on modest hardware with hours of runtime. The EM-like training is a greedy merge heuristic over a non-convex space, with no guarantees or empirical analysis of convergence quality, sensitivity to initialization, or stability across runs/data subsets. Complexity bounds are stated but not validated experimentally.
- Presentation and references: The paper has notational inconsistencies, typos, and incomplete/broken references. Some core ideas (e.g., relation to Tatti & Vreeken’s episode coding, differences from Bayesian MDL) could be explained more clearly. This detracts from clarity and makes independent reimplementation harder.

Questions to authors
- Please provide the exact derivations for the “data-to-model” components: how do you encode rule usage probabilities and alphabet symbol unigrams? What multinomial code do you use, and how are zero counts handled?
- What is the precise training schedule and stopping criterion? Do you merge exactly one pair per M-step or multiple? How do you handle overlapping rules during alignment (are alignments strictly non-overlapping contiguous substrings)? Are gaps allowed?
- How sensitive are results to Laplace smoothing, alpha over-weighting, and initial rule sets? Any ablations?
- Can you compare against at least two baselines: (i) PMI-based pairwise sound correspondences with agglomerative clustering/tree; (ii) Levenshtein-based distances on the same data? For correspondence quality, can you evaluate against a small gold set of known Slavic correspondences?
- Will you release code and the exact processed datasets (with scripts/transliterations) to enable replication?

Overall assessment
The paper presents a promising and fairly original MDL-based approach to multi-language correspondence discovery, with appealing qualitative results and a useful beyond-pairwise analysis. However, the empirical validation is too limited and lacks comparisons, and the coding/algorithmic details are under-specified for reproduction. With stronger experiments and clarified methodology, this could be a solid contribution; in its current form, I lean toward rejection for the main conference but see potential for a workshop or a revised submission.

- Proposes an MDL-based framework to discover many-to-many string correspondences across multiple languages without fixed parametric form.
- Model encodes a rule table Π of multi-language string tuples and alignments as shortest paths in a DAG given rule costs.
- Training alternates E-step exact re-alignment (Dijkstra) and M-step greedy merging of compatible rules to reduce description length.
- Applied to Slavic orthographic data; pairwise MDL-based NCD yields a plausible phylogeny.
- Introduces shared Description Length (sDL) to quantify multi-language overlap and “data overweighting” to favor longer rules when needed.
- Contribution: MDL formulation and deterministic heuristic for multi-language, many-to-many correspondence discovery with exact shortest-path alignment.
- Contribution: Analysis framework using sDL and MDL-based NCD for multi-language similarity, demonstrated on Slavic languages.
- Contribution: Practical controls for rule granularity (data overweighting) and ability to seed models with prior linguistic rules.
- Strength: Conceptually elegant and flexible modeling of arbitrary string tuples across N languages, capturing heterogeneous patterns and sparse rules.
- Strength: Deterministic, explainable learning with reproducible alignments and interpretable rules useful for linguists.
- Strength: Extends analysis beyond pairwise to N>2 and demonstrates multi-way insights (e.g., CS–PL–RU–BG).
- Strength: Grounded in information theory (MDL) with enumerative/data-to-model coding for unigrams and rule usage.
- Strength: Empirical plausibility on Slavic datasets; linguistically meaningful correspondences and script effects are captured.
- Weakness: MDL code design and term placement unclear; parameter codes appear data-dependent and may double-count or violate Kraft conditions.
- Weakness: Optimization is heuristic and not a true EM; greedy merges lack convergence guarantees and sensitivity to merge order is not studied.
- Weakness: Limited evaluation—no comparison to strong baselines, no quantitative tree quality metrics, no significance testing or held-out evaluation.
- Weakness: Data confounds from raw orthography, script effects, mixed etymologies, lack of cognate filtering, and bias from requiring complete tuples.
- Weakness: Scalability concerns—E-step and M-step complexities, memory bottlenecks, experiments limited to small datasets and few languages; overweighting departs from MDL and needs tuning guidance.
- Question: Provide rigorous derivation for L(Π) and L(Σi), specify universal multinomial code, and justify placement of usage counts to avoid circularity/double-counting.
- Question: Precisely define “compatible rules” and the merge operation, and describe protections against explosive rule growth.
- Question: Assess sensitivity to non-cognates/noise, report any cognacy filtering, and robustness to injected noise.
- Question: Compare against baselines (PMI/PHMM, many-to-many grapheme aligners, transducer models) and include ablation studies.
- Question: How should overweighting parameter α be chosen and how stable are results across α and dataset size?
- Question: Will code, preprocessing scripts, and exact datasets be released for reproducibility?
- Additional: Fix reference list and formatting; add related work on string alignment/transduction for cognate detection.
- Additional reporting request: include quantitative tree quality (e.g., RF distance), error bars for NCD, and runtime/complexity vs rules and languages.

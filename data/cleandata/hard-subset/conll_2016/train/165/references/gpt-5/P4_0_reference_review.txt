Summary and main contributions
This paper proposes an MDL-based framework for discovering character-level correspondence rules across two or more languages directly from raw orthographic cognate lists. The model defines a two-part code over: (a) an inventory of alphabets and a rule table of multi-string correspondences, and (b) the data encoded as sequences of rule applications. Alignments are computed as shortest paths in a graph (via Dijkstra), and parameters are learned with an EM-like procedure: E-step aligns data with current rule costs; M-step greedily merges rule pairs that most reduce total description length. The approach supports multi-language analysis (not only pairs), admits deterministic training, allows user-specified initialization, and introduces an optional data overweighting scheme to include near-significant patterns. Experiments on Slavic languages show: (i) pairwise NCDs yield a phylogeny consistent with linguistic classification; (ii) interpretable correspondence rules at multiple granularities; and (iii) a four-language analysis with a “shared Description Length” (sDL) measure quantifying similarity among all language subsets.

As I see them, the primary contributions are:
1) An MDL coding scheme and deterministic EM-style algorithm for discovering multi-language string correspondence rules and optimal alignments, including a graph-based exact alignment step.
2) An information-theoretic analysis toolkit for multilingual similarity: NCD for pairs and a new “shared Description Length” (sDL) to allocate model complexity to specific language subsets.
3) Empirical demonstration on Slavic data showing linguistically plausible correspondences and a phylogeny aligned with known classifications.

Strengths
- Clear MDL formulation and interpretable modeling choices: The two-part code is specified for alphabets, rule tables, and data usage, with explicit use of universal integer codes and enumerative (data-to-model) codes. This ties model complexity directly to the learned rule inventory and their usage, supporting interpretability and principled regularization.
- Generalization beyond pairwise correspondences: The method natively handles N>2 languages. The four-language experiment and the sDL metric provide fine-grained, subset-specific similarity analyses that pairwise-only methods cannot easily yield.
- Deterministic training with exact alignment: Casting alignment as a shortest-path problem and using deterministic EM with greedy merges yields reproducible results and transparent optimization steps. The merging rationale (longer patterns must be supported by subpatterns) is intuitive and aligns with MDL parsimony.
- Empirical plausibility and utility: The pairwise NCD tree recovers expected Slavic groupings and known outliers (e.g., Bulgarian/Macedonian), and the discovered rules are linguistically meaningful (e.g., palatalization, digraphs, metathesis), demonstrating the utility of orthography-level correspondences for comparative analysis.
- Practical flexibility: The framework allows injecting prior linguistic knowledge (initialization) and a tunable data overweighting mechanism to control granularity when data are sparse, which can be valuable for practitioners.

Weaknesses
- Limited quantitative evaluation and baselines: Results rely largely on qualitative inspection and a single NCD-based phylogeny. There is no comparison against established baselines for sound/orthographic correspondences (e.g., PMI-based sound correspondences, ALINE/ASJP-like distances, Bayesian models for sound changes, supervised/unsupervised cognate alignment). Without quantitative metrics (e.g., alignment quality on annotated datasets, downstream cognate identification accuracy, tree accuracy vs. gold phylogenies), it is hard to assess gains in accuracy or robustness.
- Scalability constraints and incomplete complexity analysis: The alignment step is memory/time intensive; current implementation is limited to up to five languages on modest hardware. While big-O for E/M steps is discussed, the constant factors and practical scaling with rule set size and string lengths are not fully characterized. No batching, pruning, or approximate alignment strategies are explored, which limits applicability to larger vocabularies or more languages.
- Ambiguities in coding details and implementation: Several coding components would benefit from more precise specification for full reproducibility:
  - The exact definition of code(π): are usage code lengths −log(count(π)/TΠ) derived from the enumerative code transmitted in L(Π)? How are ties and zero counts treated beyond “Laplace correction” (pseudocount magnitude)?
  - The enumerative/data-to-model terms in L(Π): the combination of LN(TΠ) + log(choose) with subsequent usage coding in L(D|M) needs a clearer derivation to confirm no hidden double counting and conformance with Kraft inequality.
  - The graph construction for Dijkstra (node/edge definitions, pruning strategy) and edge weights should be fully specified with an illustrative example.
- Evaluation depends on raw orthography and script: Operating on surface orthographies (Latin/Cyrillic) risks learning script- and orthography-specific rules rather than phonological correspondences. While the paper notes script effects (e.g., SC Latin vs Cyrillic), there is no control experiment with phonemic/phonetic normalization to assess robustness or to separate orthographic from phonological signal.
- Methodological choices not ablated: The impact of initialization, the greedy merge heuristic, and the overweighting parameter α (global vs per-tuple) on convergence and discovered rules is not systematically studied. The proposed sDL measure sums rule description lengths rather than usage-weighted contributions; justification and sensitivity to this choice are not evaluated.

Questions to authors (optional)
- Coding scheme and counts:
  - Please formalize code(π), the precise smoothing used, and how LN(TΠ) + log(choose) interacts with the data usage coding to avoid double counting. A short derivation showing the two-part decomposition would improve clarity.
  - Are alphabet symbol unigram codes estimated from rule strings only, or from both rules and data usage? How are symbol counts collected?
- Alignment graph:
  - Could you provide a small end-to-end example of the alignment graph for two short multi-language strings, including node/edge definitions and edge weights?
  - What partial order reduction is applied? Any admissible heuristics to accelerate Dijkstra?
- Scalability and robustness:
  - How does runtime and memory scale with number of languages, vocabulary size, and average string length in practice? Any results for N>5 when more memory/cores are available?
  - Have you explored beam search or A* with admissible heuristics as approximations?
- Overweighting:
  - How is α selected in practice? Can you report how α affects the number/length of multi-language rules, NCDs, and sDLs? Any risk of systematically biasing toward longer but spurious rules?
  - For per-tuple weights, do you normalize total weight to keep model selection comparable across runs?
- Evaluation:
  - Can you provide comparisons to at least one cognate-alignment or correspondence baseline and quantitative metrics (e.g., alignment accuracy on a gold set, tree distance to a reference phylogeny)?
  - Have you tried phonemic transcriptions to validate that results are not driven by script or idiosyncratic orthographic conventions?
- Reproducibility:
  - Will code and processed datasets be released? If not, please add more implementation detail (tie-breaking in merges, stopping criteria, hyperparameters).

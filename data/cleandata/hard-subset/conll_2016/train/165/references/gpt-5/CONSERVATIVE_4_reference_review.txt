Summary and main contributions
- The paper proposes an MDL-based framework to induce multiway string-to-string correspondence rules across languages. It defines a two-part code for alphabets and a correspondence-rule table, aligns cognate tuples via shortest paths, and iteratively improves the model using an EM-like procedure with greedy rule merging.
- It uses the learned code lengths to define pairwise Normalized Compression Distance (NCD) and introduces a “shared Description Length” (sDL) decomposition to quantify how much structure is shared by subsets of languages.
- It presents case studies on Slavic languages (Swadesh lists and an additional cognate set), showing a phylogenetic tree consistent with known groupings and qualitative examples of induced rules; it also proposes “data over-weighting” (α > 1) to force more granular rules.

Strengths
- Conceptually clean formulation: Modeling correspondences directly as multi-character string associations without parametric assumptions is appealing. The MDL objective provides an interpretable regularizer balancing model complexity and data fit.
- Multi-language capability: Unlike many pairwise methods, the model is designed to discover correspondences across N > 2 languages in a single training run. The sDL decomposition is a neat diagnostic for grouping-related signal beyond pairwise distances.
- Interpretability: The induced rules (e.g., cz ~ č, liquid metathesis, multi-character stems) are linguistically meaningful and potentially useful for downstream tasks (e.g., reconstruction, normalization). The ability to seed training with known rules is practical for expert-in-the-loop workflows.
- Determinism: The emphasis on a deterministic training/evaluation pipeline (given fixed data and initialization) is attractive for reproducibility in linguistic analyses.

Weaknesses
- Evaluation lacks rigor and baselines. The main quantitative result is that the NCD tree broadly matches known Slavic groupings—an unsurprising outcome that many much simpler baselines can achieve (e.g., Levenshtein-based distances on Swadesh forms; LexStat; ALINE; PMI-based alignments; Pair-HMM transliteration models). There is no comparison to established methods, no intrinsic evaluation of rule quality (against curated sound correspondence sets), and no task-based evaluation (e.g., cognate identification accuracy, reconstruction error). Without such baselines, the claimed advantages of MDL and multiway modeling remain unsubstantiated.
- Data choices and confounds. Experiments are restricted to small, orthographic datasets (Swadesh: 207 concepts; a 4-language list: 778 entries). Results may be driven by script and orthography (the Latin/Cyrillic effect is acknowledged), not phonology. No phonemic normalization or robustness checks (e.g., comparing phonemic transcriptions) are provided. Mixing scripts and using raw orthography undermines claims about linguistic correspondences.
- Methodological clarity and justification of the code. The coding scheme contains nontrivial choices (data-to-model codes, enumeration of rule subsets, handling of zeros, LN offsets). Several formulas are under-specified or hard to verify (e.g., exact derivation of L(Π), treatment of empty components, tie-breaking in alignments). There is no held-out evaluation of description lengths (NCD is computed on the same data used to fit), so potential overfitting is not addressed.
- “Data over-weighting” undercuts the MDL objective. Introducing α to overweight L(D|M) to force larger rules is a subjective knob that changes the inferred structures. The paper demonstrates qualitative effects but provides no principled guidance, ablation, or criteria for choosing α, and no evidence that overweighted rules correspond better to true correspondences.
- Scalability and practicality. The reported complexity and runtimes (hours on a single core, memory bottlenecks; up to 5 languages) are concerning, given the motivation to handle multiple languages. There is no empirical scaling study versus N, |D|, or rule length, and no comparison to faster, established pipelines used in historical linguistics (e.g., LingPy/LexStat) or NLP transliteration/segmentation baselines.
- Reproducibility and related work. Code/data release is not mentioned. Key related work on cognate detection, sound correspondence induction, and phylogenetic methods is missing or insufficiently discussed (e.g., List’s LexStat and LingPy, PMI-based approaches, ALINE, Bayesian models for correspondence and phylogeny, Pair-HMM transliteration). References are inconsistently formatted.

Questions to authors
- Baselines: How does your NCD-based tree compare to (i) Levenshtein distances on Swadesh lists, (ii) LexStat distances (LingPy), (iii) ALINE/PMI-based measures, or (iv) PHMM-based transliteration scores? Please report correlations and tree distances (e.g., Robinson–Foulds) to established classifications.
- Intrinsic evaluation: Can you evaluate induced rules against curated correspondence sets or gold alignments (e.g., from NorthEuraLex or other datasets with phonemic transcriptions)? Provide precision/recall or alignment F-scores.
- NCD computation: Are L(X,X) and L(X,Y) computed on held-out data or the training data? If on training data, can you report results with held-out test to mitigate overfitting? Clarify whether the joint model for L(X,Y) is trained jointly or re-used from individual models.
- Coding details: Please give a precise derivation for L(Π) and the data-to-model code choices (including subset enumeration, zero counts, and offsets). How are empty strings handled in rules? What are the maximum rule lengths and pruning criteria?
- Sensitivity and α: Provide ablations for initialization (null vs seeded), rule-merge ordering, and α values. How robust are the results to these choices? Can you justify α on principled grounds or restrict its use to analysis rather than model selection?
- Scalability: What are typical |Π|, |R|, and A in your datasets? Can you provide runtime/memory scaling with N and |D|, and any heuristics that make N > 5 practical?

Additional comments
- Please include results on phonemic transcriptions to reduce orthographic confounds and add additional language families to demonstrate generality.
- Improve references and related work coverage (e.g., LexStat/LingPy; Jäger; Bouchard-Côté et al.; PMI-based sound correspondence induction; transliteration/PW-HMM literature).
- Consider releasing code and data splits to enable replication.

Overall, while the idea of an MDL-based, interpretable, multiway correspondence induction is interesting, the current empirical validation is too limited and lacks comparisons to established baselines to warrant strong claims. Substantial experimental strengthening and methodological clarifications are needed.

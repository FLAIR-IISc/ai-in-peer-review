### Review of "Discovering Correspondences between Multiple Languages by MDL"

**Summary of the Paper and Main Contributions**

This paper presents a novel, unsupervised method for discovering orthographic correspondence rules from cognate lists across multiple languages simultaneously. The core of the approach is the Minimum Description Length (MDL) principle, which frames the problem as finding a set of correspondence rules that provides the most compact description of the given cognate data. The authors propose a deterministic, EM-like algorithm that iteratively builds complex rules by merging simpler ones, starting from single characters. The method is evaluated on a dataset of 13 Slavic languages. The primary contributions of this work are:

1.  A principled framework for discovering correspondence rules across more than two languages at once. This is a significant extension of common pairwise approaches, allowing the model to capture complex relationships where rules may span any subset of the languages being analyzed.
2.  The introduction of a novel metric, "shared Description Length" (sDL), which leverages the MDL framework to provide a fine-grained, quantitative analysis of the similarities between any group of languages, moving beyond what is possible with simple pairwise distance matrices.
3.  An empirical demonstration on Slavic languages that not only recovers known phylogenetic relationships but also offers deeper insights into subgroup structures, such as the relative similarities between West, East, and South Slavic subgroups in a four-way comparison.

**Strengths**

1.  **Novelty of the Multi-Language Approach:** The paper's main strength lies in its ability to perform a multi-way analysis of language correspondences. Most computational methods in historical linguistics focus on pairwise comparisons, which are then aggregated to build a phylogeny. This work directly models the relationships between multiple languages, allowing it to identify patterns shared by, for example, three out of four languages in a group. This is a powerful and more holistic way to analyze language relationships.
2.  **Principled MDL Framework:** The use of the Minimum Description Length principle provides an elegant and theoretically sound foundation for the task. It avoids making strong prior assumptions about the distribution or form of correspondence rules, instead letting the statistical structure of the data guide the discovery process. The MDL objective function naturally balances model complexity with goodness of fit, providing a built-in defense against overfitting.
3.  **Insightful Quantitative Analysis:** The proposed "shared Description Length" (sDL) metric is a compelling and highly useful contribution. The analysis presented in Figure 3, which decomposes the total model complexity into components shared by various language subgroups (e.g., CS-PL, RU-BG, CS-PL-RU-BG), is particularly insightful. It offers a much richer and more nuanced view of language similarity than a single phylogenetic tree and is a direct benefit of the multi-language modeling approach.
4.  **Practical Flexibility:** The authors introduce "data over-weighting" as a mechanism to control the granularity of the discovered rules. While this deviates from a purely objective statistical model, it is a thoughtful, practical feature. It allows a user, such as a linguist, to guide the algorithm toward discovering patterns that may be linguistically meaningful but fall just below the threshold of statistical significance in a limited dataset. This significantly enhances the potential of the method as an exploratory tool.

**Weaknesses**

1.  **Scalability Concerns:** The paper mentions that the current implementation can process "up to five languages" and that runtime is a bottleneck. The complexity of the core alignment step appears to grow exponentially with the number of languages (N), which is a major limitation for a method whose central contribution is multi-language analysis. A more detailed discussion of how runtime and memory scale with N, and what the practical limits of the approach are, would be crucial. Stating this will be addressed in future work is reasonable, but the current limitations should be characterized more explicitly.
2.  **Lack of Comparative Evaluation:** The paper motivates its approach by highlighting weaknesses in prior work but provides no direct comparison to any alternative methods. For the pairwise analysis, it would be valuable to compare the resulting NCD-based phylogeny to trees generated using other standard computational methods (e.g., based on lexical similarity or phylogenetic linguistic software). Without such a baseline, it is difficult to assess whether the complexity of the MDL approach offers significant advantages over simpler methods in the pairwise case.
3.  **Clarity of the MDL Formulation:** Certain aspects of the model description in Section 4 are difficult to follow. In particular, the formula for encoding the rule table, L(Π), appears to include terms for rule counts (`LN(TΠ) + ...`). In a standard two-part code, `L(M)` encodes the model parameters (the rules themselves), while `L(D|M)` uses those parameters to encode the data (which is where frequencies/counts are typically used). The authors' formulation seems to place the encoding of rule counts within the model cost `L(M)`. A clearer explanation or a small, worked-out example would greatly improve the reader's understanding and the reproducibility of the work.
4.  **Dependence on Orthography:** The method operates directly on orthographic forms. While this is a valid choice, the paper could benefit from a more thorough discussion of its implications. The results for Serbo-Croatian (clustering differently based on whether Latin or Cyrillic script is used) highlight that the method captures script-level similarities as much as linguistic ones. It would be useful to discuss how the findings might differ if the input were phonetically transcribed data, and to what extent the discovered "correspondences" are purely orthographic versus reflecting deeper phonological relationships.

**Questions to Authors**

1.  Could you please clarify the rationale for including the data-to-model code for rule counts in the `L(M)` term (specifically in `L(Π)`) rather than as part of the data likelihood `L(D|M)`? This seems to depart from a more standard formulation of a two-part MDL code.
2.  The algorithm is described as deterministic. How sensitive is the final set of rules to the specific data provided? For instance, if a small percentage of non-cognates were included in the input lists, would the algorithm be robust enough to ignore them, or would they significantly alter the discovered high-level correspondences?
3.  Regarding the alignment procedure (Section 4.3), could you provide more detail on the partial order reduction used to make the graphs smaller? Is this the primary mechanism for managing the combinatorial complexity as the number of languages increases, and could you elaborate on its effectiveness?

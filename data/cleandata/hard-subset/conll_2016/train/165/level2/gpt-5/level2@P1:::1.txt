Summary and claimed contributions
The paper proposes an MDL-based model and algorithm to automatically discover string-level correspondence rules among two or more languages without assuming a parametric form for correspondences. The approach represents correspondences as tuples of strings (possibly empty for some languages), learns a code table by EM with greedy merges, and aligns data via shortest-path search. Experiments on Slavic languages show that the learned rules yield plausible pairwise distances (via NCD), a reasonable phylogeny, and a four-way analysis with a “shared description length” (sDL) measure. A data overweighting knob is introduced to control granularity.

As I see it, the main contributions are:
1) An MDL coding scheme and deterministic EM-based algorithm for discovering multi-language correspondence rules that can be defined on arbitrary subsets of languages, with an explicit model/data code and an efficient alignment procedure.
2) An evaluation framework using NCD and a new sDL analysis that quantifies similarity not only pairwise but across all subsets of languages, demonstrated on Slavic data.
3) A practical mechanism (data overweighting) to control the level of detail of discovered correspondences, enabling qualitative exploration.

Strengths
- Methodological flexibility and clarity: The MDL formulation is carefully specified, including two-part codes for alphabets and rules, use of universal integer codes, and data-to-model codes. The encoding encourages sparsity and supports rules defined on any subset of languages without imposing parametric assumptions. This is a good fit to the problem and a clear advance over approaches restricted to pairs.
- Multi-language capability: The alignment-as-shortest-path plus EM/merge scheme scales conceptually from 2 to N languages, and the four-way analysis shows the method can reveal subset-specific regularities. The sDL plots are a useful summary that goes beyond pairwise trees.
- Plausible and interpretable results: The pairwise NCD matrix and the resulting phylogeny generally align with established Slavic groupings and script effects. The qualitative correspondences (e.g., cz~č, liquid metathesis, palatalization patterns) are linguistically meaningful. The observation that RU–BG and CS–PL share substantial structure is well supported by sDL.
- Determinism and knowledge integration: Starting from singleton rules (or seeded rules) and performing greedy merges provides deterministic outputs and a natural entry point for expert priors, which is valuable for linguistic analysis and reproducibility.
- Practicality and transparency: The paper discusses runtime/memory costs, the use of Dijkstra with partial order reduction, and provides concrete examples contrasting objective MDL solutions with overweighted variants to control granularity.

Weaknesses
- Limited quantitative evaluation and baselines: There is no comparison to alternative methods (e.g., pairwise alignment/PMI-based sound correspondences, Bayesian cognate models, or other MDL/Morfessor-style approaches), no gold evaluation of correspondences, and no quantitative tree accuracy (e.g., Robinson–Foulds distance). This weakens claims of efficacy beyond plausibility.
- Orthography vs. phonology: Working on raw orthography conflates script and spelling conventions with phonological correspondences. While noted, the paper does not test phonemic normalization or transliteration to separate orthographic from phonological effects, nor does it analyze the impact of diacritics/digraph tokenization choices.
- Scalability and complexity: The E-step is O(|D|·|R|^2) and alignment uses shortest-path over potentially large graphs. Experiments are limited to small lists (207–778 items) and up to four/five languages. There is no detailed profiling, pruning strategies, or complexity analysis as N and average word length grow, leaving scalability uncertain.
- Subjectivity of overweighting: The α parameter changes the objective away from pure MDL; while useful, there is no principled guidance for choosing α, no sensitivity analysis, and no demonstration that overweighted rules improve any downstream task (e.g., cognate prediction).
- Some modeling details are under-specified: It remains unclear how “compatible” merges are defined; whether there are safeguards against memorizing whole words (e.g., max rule length, boundary symbols); how tie-breaking affects determinism; and precisely how Laplace correction is applied. The encoding of Π (e.g., the role of LN(TΠ) and the composition terms) could be clarified with definitions and derivations.

Questions to authors (for rebuttal)
- Can you provide a small quantitative evaluation of discovered correspondences against expert annotations, or a comparison of the inferred tree to a reference using a standard metric?
- How do you prevent degenerate, overly long rules (e.g., entire words)? Do you use boundary markers, length caps, or penalties beyond MDL? What is the stopping criterion?
- How are “compatible” rules defined for merging in the M-step? Does merging consider context or only substring concatenation? How are ties broken, and is the outcome invariant to rule ordering?
- How sensitive are results to alphabet/tokenization choices (e.g., treating digraphs as single symbols, lowercasing, diacritics)? Did you try phonemic transliteration, and if so, how did results change?
- Can you report typical graph sizes (nodes/edges) and runtime per alignment for representative word lengths and N, and detail the partial order reduction used? Any heuristics for pruning improbable paths?
- What Laplace value do you use for smoothing rule counts? Did you observe meaningful differences with/without smoothing?
- How does α impact sDL/NCD quantitatively? Is there a range where results are stable?

Additional comments
- Please clarify notation in L(Π): define TΠ, |Π|, and ΠCi before they appear; consider adding a short derivation for the data-to-model code choices. A few typos and reference formatting issues should be addressed.
- An ablation (null vs. seeded initialization; with/without overweighting; with/without merging) and a brief sensitivity analysis would strengthen the empirical section.
- If possible, releasing code and processed datasets would greatly aid reproducibility and adoption.

- MDL-based framework (PS-2 code) for discovering multi-language string correspondences permitting rules over arbitrary language subsets, optimizing a two-part model+data code.
- Deterministic EM-like procedure alternating Dijkstra shortest-path alignments and greedy rule merges, with a practical “data overweighting” knob to control rule granularity.
- Empirical Slavic demonstration: NCD-based phylogeny broadly matches accepted groups, qualitative correspondences capture ortho/phonological patterns, and sDL quantifies subset-shared structure.

- Handles general multiway correspondences and yields interpretable rules; sDL is a novel metric for subset-level similarity.
- Solid MDL framing with explicit accounting of model cost (alphabets, rule table, symbol codes) and data cost (alignments via rule usages).
- Qualitative linguistic plausibility: recovers known correspondences (e.g., cz–č, g–h, palatalizations) and reflects expected language groupings.
- Deterministic null-start and updates improve reproducibility and permit injecting linguistic priors via initialization.
- Transparent discussion of trade-offs and motivation for overweighting to surface larger low-frequency patterns under scarce data.

- Limited empirical validation and no baselines: lacks quantitative rule quality metrics, held-out codelengths, or extrinsic-task comparisons to PHMMs/ALINE/Bayesian models.
- Overweighting undermines principled MDL balance (optimizing L(M)+αL(D|M)); paper doesn’t quantify how α affects rule lengths, alignments, or similarity measures.
- Technical/derivation gaps: L(Π) and combinatorial/data-to-model code terms are not fully justified and risk double-counting between model and data; NCD compressor/training details are unclear.
- Scalability and convergence concerns: E-step and M-step complexities scale poorly, experiments limited to 4 languages for detailed analysis, no convergence statistics or strategies to escape greedy local optima reported.
- Preprocessing issues: using raw orthography across scripts conflates script with linguistic similarity, lacks phonemic transliteration, and dataset noise (Wiktionary/ internationalisms) is not analyzed.

- How are L(X,X), L(Y,Y), and L(X,Y) computed—single joint model or separate models—and how are alphabets across scripts handled in joint encodings?
- Please provide a formal derivation/justification for L(Π), especially the data-to-model code terms, and explain how double-counting is avoided.
- What prevents the model from memorizing whole words when α is large or data are small; can you report rule-length distributions and impact on held-out codelength?
- How many EM iterations are typical; do you reverse merges or use annealing/beam search to escape local optima?
- How sensitive are results to transliteration/phonemic transcription; does Latin/Cyrillic duplication persist after normalization?
- Will you release code and processed datasets; if not, provide full hyperparameters (smoothing, stopping criteria) and tie-breaking rules for merges/alignments?

- Add quantitative evaluations: annotated correspondence precision/recall, leave-one-language-out reconstruction accuracy, held-out codelengths across α, and comparisons to PHMM/ALINE baselines.
- Run ablations: remove data-to-model code, subset-specific rule counts, Laplace smoothing, and overweighting to measure component contributions.
- Normalize scripts or use phonemic transcription (e.g., IPA/G2P) and report results both with and without normalization.
- Provide empirical scaling plots for runtime and memory versus |D|, N, and |Σ| and report convergence behavior.
- Improve references and cite additional related work on cognate alignment and sound change induction; fix misformatted entries.
- Correct minor editorial issues: typos, inconsistent variable naming, and make figure/table captions more self-contained.

- Paper proposes an MDL-based PS-2 code and a deterministic EM-style algorithm to discover subset-aware string correspondences across multiple languages using shortest-path alignments in a weighted graph.  
- Introduces information-theoretic analyses: pairwise NCD distances and shared description length (sDL) for arbitrary subsets, plus an over-weighting knob α to control rule granularity.  
- Empirical demonstration on Slavic languages finds non-trivial multi-character correspondences and produces a plausible phylogeny consistent with linguistic classifications.  

Main contributions:
- Principled, subset-aware MDL formulation with deterministic learning and graph-based optimal alignment.  
- New similarity metrics beyond pairs: NCD for pairs and sDL for arbitrary subsets, enabling interpretable quantitative insights.  
- Empirical results showing meaningful correspondences and coherent subgroupings in Slavic data.  

Strengths:
- MDL objective provides principled regularization, suppresses redundant rules, and yields interpretable, subset-specific rules.  
- Four-way experiment and sDL reveal shared structure across arbitrary subsets that pairwise methods miss (e.g., RU–BG and CS–PL groupings).  
- NCD-based phylogeny aligns with known Slavic classifications and script effects, supporting exploratory use.  
- Deterministic, merge-based M-step with seeding supports reproducibility and expert steering.  

Weaknesses:
- Limited quantitative validation: no comparisons to established baselines or intrinsic precision/recall against gold correspondences.  
- Scalability concerns: E-step complexity O(|D|·|R|^2) and memory bottlenecks limit experiments to N ≲ 5 without suggested remedies.  
- Data/representation issues: reliance on orthography and cognate lists conflates script/orthographic effects with phonology; no normalization/robustness tests.  
- α over-weighting parameter lacks principled selection, risking overfitting and undermining MDL objectivity without diagnostics.  

Questions for authors:
- How should α be chosen in practice; can you provide validation criteria or curves to prevent overfitting?  
- Can you compare to at least one established baseline and report intrinsic evaluation against a small gold set of correspondences?  
- What pruning/approximation strategies (e.g., pruning, beam search) could improve scalability for larger N and longer rules?  
- How sensitive are results to orthographic normalization (phonemic transcriptions) and to missing entries in input tuples?

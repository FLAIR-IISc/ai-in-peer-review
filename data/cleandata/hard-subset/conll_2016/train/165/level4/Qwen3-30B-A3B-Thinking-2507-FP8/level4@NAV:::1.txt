

This paper introduces a method for discovering cross-lingual correspondences using Minimum Description Length (MDL). The authors model word correspondences sharing meaning across multiple Slavic languages by developing substring-matching rules and formulating an MDL objective that balances model description and data fit under the model. The approach is trained via Expectation-Maximization (EM) and evaluated on 13 Slavic languages, with results presented through distance metrics, a phylogenetic tree, and example correspondences.  

The motivation and formulation are sound; MDL is a suitable framework for this problem, and the rationale for using EM is well-justified. However, some derivations remain unclear. The authors note parallels between the MDL objective and Bayesian inference, drawing a connection to phylogenetic tools like MrBayes. An empirical comparison with such methods would strengthen the work.  

**Related work:**  
- The paper lacks comparison to established computational methods in historical linguistics, such as approaches for borrowing/cognate detection (e.g., Bouchard-Côte, Warnow, Nakhleh, Kitchen) or tools like List & Moran (2013). Relevant biological phylogeny tools (e.g., PAUP, MrBayes) could also be considered.  

**Approach and methodology:**  
- **Alignment procedure:** A significant memory/runtime bottleneck restricts comparisons to ≤5 languages. Given the phylogenetic context, extending this to more languages would be valuable; the authors should address scalability.  
- **Phylogenetic tree:** Neighbor joining is used, but it requires manual root specification—a known limitation. More robust methods (e.g., maximum likelihood) warrant consideration.  
- **EM stopping criterion:** Clarify whether EM runs to convergence or uses an alternative stopping rule.  

**Data:**  
- Mixing cognate and non-cognate datasets (e.g., Swadesh lists) may bias results; this impact should be addressed.  
- Orthographic data risks obscuring linguistic correspondences, particularly across script differences (e.g., Cyrillic vs. Latin). Using phonetic transcriptions would mitigate this issue.  

**Unclear points:**  
- The "optimal unigram for symbol usages in all rules" (line 286) requires clarification.  
- The merging process in the maximization step is not sufficiently explained.  

**Minor issue:**  
- "focus in on" → "focus on" (line 440).

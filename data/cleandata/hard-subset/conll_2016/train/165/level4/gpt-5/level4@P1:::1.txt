Summary and claimed contributions
- The paper presents an MDL-based framework for discovering statistically salient correspondences between substrings across two or more languages, learned directly from aligned cognate tuples. The model encodes a set of correspondence rules with a two-part MDL code and balances model cost with data encoding cost.
- Training proceeds via EM: the E-step aligns each tuple using a shortest-path formulation (Dijkstra), and the M-step merges rules to reduce total description length. The method is deterministic and can incorporate prior linguistic rules or data over-weighting to control granularity.
- Empirically, the authors analyze 13 Slavic languages pairwise using NCD and neighbor-joining to derive a phylogeny, and perform a four-language analysis (CS-PL-RU-BG) that quantifies fine-grained shared description lengths across all language subsets. They also illustrate learned correspondences.

As I see them, the main contributions are:
1) An MDL formulation and code design tailored to multi-language substring correspondences that imposes minimal distributional assumptions and supports subsets of languages.
2) A practical EM-based training procedure with a principled alignment step and a rule-merging strategy guided by MDL.
3) A set of empirical analyses on Slavic languages showing that learned correspondences are meaningful and that MDL-based distances align with known linguistic groupings; the multi-language sDL analysis provides finer-grained similarity insights than pairwise methods.

Strengths
- Sound modeling choice and clear motivation: Using two-part MDL to balance model complexity and data fit is well-justified for discovering correspondences without heavy priors. The connection to regularized ML/Bayesian intuition is explained clearly.
- Practical EM training with interpretable structure growth: The merge-based M-step, guided by description length reduction, encourages longer rules only when supported by evidence, yielding interpretable, statistically grounded correspondences.
- Empirical relevance: On 13 Slavic languages, the pairwise NCDs and resulting neighbor-joining tree broadly match established linguistic classifications. The four-language analysis demonstrates the value of discovering correspondences that hold for subsets, enabling nuanced, information-theoretic similarity assessments (e.g., sDL across subsets).
- Determinism and knowledge integration: The ability to initialize with linguistic rules and obtain deterministic results is attractive for linguistic applications; the data over-weighting knob allows practitioners to adjust granularity when data are sparse.

Weaknesses
- Limited scalability in alignment: The shortest-path alignment approach faces memory/runtime bottlenecks, reportedly limiting simultaneous comparisons to at most five languages on modest hardware. Since a core promise is multi-language analysis, the current scalability is a substantial practical limitation.
- Phylogenetic inference methodology: Relying on neighbor-joining inherits known drawbacks (e.g., manual rooting, sensitivity to distance estimation). More robust phylogenetic methods (e.g., Bayesian or maximum likelihood tree inference) could strengthen the claims, especially given the paper’s explicit parallel to Bayesian modeling.
- Evaluation breadth and baselines: The paper lacks comparisons to established methods in computational historical linguistics and cognate/borrowing detection (e.g., Bouchard-Côté et al., Warnow/Nakhleh, Kitchen; List and Moran’s toolkit). Even if direct application is difficult, reporting against representative baselines or discussing expected differences would contextualize contributions.
- Data representation: Using raw orthography, including mixed scripts, risks learning script-conversion correspondences rather than phonological or morphological regularities. Phonetic transcription (or at least a transliteration-to-phoneme layer) would likely produce more linguistically meaningful rules and reduce script artifacts.
- Mixed datasets: Combining Swadesh lists (not necessarily cognates) with a curated cognate list may bias results. The potential impact of mixing cognate status and lexical domains on both learned rules and distances is not analyzed.

Questions to the authors
- EM convergence: What is the stopping criterion (change in total description length, parameter change, or fixed iterations)? Do you use any regularization or early stopping to prevent overfitting when over-weighting data?
- M-step merging: Please formalize the merge criterion precisely. How do you select candidate pairs to merge, how is the gain in description length computed efficiently, and what is the overall complexity per iteration?
- Unigram code estimation: What exactly is the “optimal unigram for symbol usages in all rules” (line 286)? Is code(σ) estimated by ML with Laplace correction across all rule strings, or via a data-to-model code? Please provide the exact formula.
- Scalability: Do you envision pruning heuristics, A*/beam search, dynamic programming with admissible bounds, or rule-length constraints to scale alignment beyond five languages? Any empirical evidence on memory/runtime trade-offs with such strategies?
- Phylogenetic alternatives: Have you tried ML or Bayesian phylogeny (e.g., MrBayes, PAUP*) using your MDL-derived distances or character-style encodings of rules? Even small-scale comparisons could be informative.
- Data choices: Have you evaluated the effect of (i) excluding non-cognates in the Swadesh lists, or (ii) using phonetic transcriptions/transliterations to mitigate script effects? How sensitive are the learned correspondences and NCDs to these choices?

Additional comments
- Minor phrasing: “focus in on” → “focus on” (line 440).
- Please clarify whether the reported NCDs use strictly the same encoding/model class across languages, and whether model sizes are comparable when scripts differ.
- Consider reporting the proportion of singleton vs multi-symbol/multi-language rules at different over-weighting levels to quantify granularity effects.

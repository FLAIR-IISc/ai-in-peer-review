This paper proposes a method for finding similarities between languages based on a machine learning model (MLM). The authors have developed a model to create similarities between semantically equivalent words in several Slavic languages. They have created rules based on matching substrings of two or more languages and formulated an MLM objective that balances the model's description and the data provided by the model.

The model is trained with EM (Erie Machine Learning) and tested on a set of 13 Slavic languages. The results are shown with several distance measures, a phylogenetic tree, and examples of the similarities found.

The purpose and rationale behind this method are understandable. The MLM problem seems to be a reasonable tool for solving and the purpose behind using EM is well presented. However, I must admit that some justifications were not entirely clear to me.

The authors highlight the similarities between the MLM objective and Bayesian inference and remind us of the application of Bayesian inference (biological) phylogenetic inference, such as using the MrBayes tool. An empirical comparison here might be insightful.

Related work:
- Lack of adoption and comparison with cognitive recognition or other computational methods for historical linguistics. For example, research by Alexander Buchard-Kot, Tandy Warnho, Louis Nachle, and Andrew Kitchen. Some may not have tools to apply to the given dataset, but List and Moran (2013) can be mentioned. There are also relevant tools for biological phylogenetic inference that can be applied (Paupe, MrBayes, etc.).

Method and process:
- Alignment method: Memory/Runtime bottleneck seems to be a major flaw, which only allows comparison of a maximum of 5 languages. More languages can be seen if multiple languages are involved and there is a phylogenetic tree. I would be interested to know what ideas the authors have to overcome this problem.
- Phylogeny tree: There are some disadvantages to using neighbor joining to create a phylogeny tree (such as manually specifying the root). What about more sophisticated methods?
- Do you run EM for convergence or use any other stopping criteria?

Data:
- Two datasets are mixed, one is cognitive and the other may not be (Swadesh list). Have you thought about how this might affect the results?
- The data is in orthographic form, which can obscure many similarities. This is especially evident in languages with different scripts. Therefore, the rules learned may indicate more script changes than actual linguistic similarities. It seems this is an error that can be avoided by working at the level of phonetic transcription.

Obfuscating points:
- What is "the best unigram to use symbols in all rules" (line 286)?
- The merging done in the maximization step was not entirely clear to me.

Small problem:
- "Focus in on" -> "Focus on" (line 440)

This paper proposes a method for finding similarities between languages based on an Machine Learning model (MLM). The authors have developed a model for creating similarities between words with the same meaning in several Slavic languages. They have created rules that match the substrings of two or more languages and formulated an MLM objective that balances the description of the model and the data provided by the model.

The model is trained with EM (Erie Machine Learning) and tested on a set of 13 Slavic languages. The results are shown with several distance measures, a phylogenetic tree, and examples of the similarities found.

The purpose and formulation behind this method are understandable. The MLM problem seems to be a reasonable tool for solving and the purpose behind using EM is well presented. However, I must admit that some justifications were not entirely clear to me.

The authors highlight the similarities between the MLM objective and Bayesian inference and remind us of the application of Bayesian inference (biological) phylogenetic inference, such as using the MrBayes tool. An empirical comparison here could be insightful.

Related work:
- Lack of adoption and comparison with cognate identification or other computational methods for historical linguistics. For example, the research of Alexander Buchard-Kot, Tandy Wernho, Lui Nakhle, and Andrew Kitchen. Some may not have tools to apply to the given dataset, but List and Moran (2013) can be mentioned. There are also relevant tools for biological phylogenetic inference that can be applied (paup, MrBayes, etc.).

Method and process:
- Alignment method: The memory/runtime bottleneck seems to be a major drawback, which allows for the comparison of only up to 5 languages at most. More languages can be seen as long as multiple languages are involved and there is a phylogenetic tree. I am interested in what ideas the authors have to deal with this problem.
- Phylogenetic tree: There are some disadvantages to using neighbor joining to create a phylogenetic tree (such as manually specifying the root). What about a more sophisticated method?
- Do you run EM until convergence or use any other stopping criteria?

Data:
- Two datasets are mixed, one is cognate and the other may not be (Swadesh list). Have you thought about how this might affect the results?
- The data is in orthographic form, which can obscure many similarities. This is especially evident in languages with different scripts. Therefore, the rules learned may indicate more script changes than actual linguistic similarities. It seems that this is an error that can be avoided by working at the level of phonetic transcription.

Ambiguous points:
- What is the "best unigram for using symbols in all rules" (line 286)?
- The merging done in the maximization step was not entirely clear to me.

Small problem:
- "focus in on" -> "focus on" (line 440)

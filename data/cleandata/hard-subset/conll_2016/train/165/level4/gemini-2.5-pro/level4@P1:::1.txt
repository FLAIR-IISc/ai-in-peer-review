### Summary and Contributions

This paper presents a method for discovering string-level correspondences between multiple languages using the Minimum Description Length (MDL) principle. The authors model correspondences between cognates in a set of Slavic languages by developing a code that balances the complexity of the correspondence rules with the complexity of encoding the data given those rules. The model is trained using an Expectation-Maximization (EM) algorithm. The method is evaluated on a dataset of 13 Slavic languages, with results presented in the form of pairwise distance measures, a phylogenetic tree, and examples of the learned correspondences.

The main contributions of this work are:

1.  **A principled, non-parametric method for multi-language correspondence discovery.** The application of MDL is well-motivated, allowing the model to discover statistically significant patterns directly from data without strong prior assumptions about their shape or distribution.
2.  **An intuitive EM-based training algorithm.** The iterative procedure of aligning the data (E-step) and merging rules to improve the model (M-step) is clearly explained and provides a deterministic way to find good sets of correspondence rules.
3.  **A novel "shared description length" metric for similarity.** The paper demonstrates how the MDL framework can be used to quantify the similarities between arbitrary subsets of languages, offering a more detailed analysis than traditional pairwise comparisons.

### Strengths

1.  **Sound Theoretical Framework:** The choice of the Minimum Description Length principle is a major strength. It provides a solid, information-theoretic foundation for the task of model selection, elegantly balancing model complexity with data fit. This data-driven approach is particularly suitable for discovering linguistic patterns without imposing restrictive assumptions.
2.  **Clear Motivation and Exposition:** The paper does an excellent job of motivating the problem and the proposed solution. The rationale for using an EM-like procedure is presented very clearly, making the overall approach intuitive and easy to follow.
3.  **Insightful Experimental Analysis:** The experiments effectively demonstrate the method's potential. The pairwise analysis produces a phylogenetic tree that aligns well with the known classification of Slavic languages. Furthermore, the four-way analysis and the introduction of the "shared description length" metric provide a compelling proof-of-concept for fine-grained, multi-language similarity analysis.

### Weaknesses

1.  **Insufficient Comparison to Related Work:** The paper is missing a discussion of and comparison with a significant body of related work in computational historical linguistics and phylogenetic inference. For instance, a comparison with Bayesian phylogenetic methods (e.g., MrBayes, or the work of Kitchen et al., 2009) would have been insightful, given the authors' own note on the similarity between MDL and Bayesian inference. Other relevant work and toolkits (e.g., List and Moran, 2013; work by Bouchard-Cote, Warnow, Nakhleh) should also be discussed to better position the contribution within the field.
2.  **Significant Scalability Limitations:** A major drawback is the reported memory and runtime bottleneck, which limits the analysis to a maximum of five languages. This constraint severely undermines the paper's primary claim of being a tool for discovering correspondences between *multiple* languages. The practical utility of the method is limited until this scalability issue is addressed.
3.  **Use of Orthographic Data:** The reliance on raw orthographic forms is a potential weakness. This approach may capture superficial script-level similarities rather than genuine linguistic correspondences, particularly when analyzing languages with different writing systems (e.g., Latin vs. Cyrillic). An analysis based on phonetic transcriptions would provide a more robust and linguistically meaningful foundation.
4.  **Methodological Choices in Phylogeny:** The use of the neighbor-joining algorithm to construct the phylogenetic tree is a dated choice with known disadvantages, such as requiring the root to be placed manually. The authors should either justify this choice over more sophisticated and robust modern methods or employ a stronger alternative.
5.  **Lack of Clarity on Data Handling:** The paper states that two datasets were combined—Swadesh lists and a set of Slavic cognates. It is unclear if the Swadesh lists were filtered for cognates, and the potential impact of mixing potentially non-cognate pairs with true cognates on the learned rules is not discussed.

### Questions to Authors

1.  The description of the merging process in the M-step is brief. Could you provide a more detailed explanation of how two patterns are selected and combined to achieve the greatest decrease in description length?
2.  What stopping criterion do you use for the EM training procedure? Is it run for a fixed number of iterations, or until the total description length converges?
3.  In Section 4.1, you refer to using the "optimal unigram for symbol usages in all rules" to encode strings. Could you please clarify what this unigram distribution is and how it is computed?
4.  Given the acknowledged scalability bottleneck, what are your future plans for improving the algorithm's efficiency to allow for the analysis of more than five languages?

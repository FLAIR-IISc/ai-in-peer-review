Summary and Contributions
This paper proposes an MDL-based framework to automatically discover substring-level correspondences across two or more languages, without imposing prior distributional assumptions and allowing rules to be defined on arbitrary subsets of languages. An EM-style training procedure is used: alignments are computed via shortest-path search, and the maximization step merges rule pairs to minimize total description length. The method is evaluated on Slavic languages via pairwise NCD, a neighbor-joining tree, illustrative correspondences, and a four-language analysis using a new “shared description length” (sDL) measure.

As I see them, the main contributions are:
1) An MDL formulation and deterministic learning algorithm for multi-language, multi-subset correspondence rules, including a concrete two-part code and an EM procedure with Dijkstra-based alignment and a merge-based M-step.
2) A framework for information-theoretic analysis of linguistic similarity beyond pairs, notably the sDL metric that quantifies structure shared by any subset of languages.
3) Empirical evidence on Slavic languages demonstrating plausible correspondences, phylogenetic signal in NCD distances, and a practical “data over-weighting” knob to control granularity.

Strengths
- Clear MDL modeling and unbiased rule discovery. The two-part code is carefully specified and avoids strong priors about rule shape or distribution. This supports the claim of objective, data-driven discovery and helps explain why the learned rules span phonological, orthographic, and morphological patterns (e.g., cz–č, g–h, metathesis, endings).
- Multi-language capability with subset rules. Allowing rules to be defined on any subset of languages is a genuine step beyond standard pairwise cognate/morphology approaches and enables nuanced analysis (e.g., three-way correspondences when four-way evidence is insufficient).
- Interpretable outputs with quantitative evaluation. The method yields human-interpretable rules and simultaneously supports quantitative analyses (NCD-based distances and trees, plus sDL decomposition). The NCD matrix aligns with known Slavic groupings, and the sDL analysis highlights meaningful clusters (CS–PL, RU–BG) and language-specific idiosyncrasies (e.g., Czech diacritics).
- Deterministic training with a practical control of detail. The deterministic EM-plus-merging procedure is attractive for reproducibility, and the data over-weighting mechanism provides a tunable way to obtain finer-grained rules when strict MDL selection is too conservative for downstream tasks (e.g., reconstruction).

Weaknesses
- Missing comparisons to established phylogenetic and cognate methods. Given the conceptual proximity to regularized likelihood/Bayesian objectives, a comparison to Bayesian phylogenetic tools (e.g., MrBayes) and to cognate detection/borrowing-aware methods (e.g., Bouchard-Côté et al.; Warnow/Nakhleh; List & Moran 2013; Kitchen et al. 2009) is important. Without this, it is difficult to assess relative performance and robustness.
- Alignment scalability limits and limited multi-language experiments. The alignment graph approach is memory/time intensive and currently caps at N≈5, while the paper’s core promise is multi-language analysis. Most empirical results are pairwise; the multiway analysis is limited to four languages. A stronger case would include larger-N experiments or engineering improvements (e.g., pruning/beam search, A* heuristics, composition/factorization) with complexity/runtime reports.
- Use of orthographic forms risks confounding script/orthography with linguistic correspondences. Results like SC-Latin vs SC-Cyrillic differences illustrate script effects. Phonetic/phonological transcriptions (e.g., IPA, feature-based segments) or script normalization would better isolate linguistic correspondences and reduce orthography-induced biases.
- Methodological clarity gaps. Two definitions are unclear: what exactly is meant by “optimal unigram for symbol usages in all rules” (is this a data-to-model code over rule symbol counts, or a ML estimate over concatenated rule strings?), and how precisely the M-step “merging” is selected and applied (which candidates, how conflicts are resolved, whether back-off/splitting is allowed). Additionally, EM stopping criteria and convergence behavior are not specified.
- Evaluation design issues. Mixing Swadesh lists with a separate cognate set (internationalisms, pan-Slavic forms) may bias both distance and rule discovery. The neighbor-joining tree requires manual rooting and has known drawbacks; more principled phylogenetic inferences (e.g., model-based trees with support values) would strengthen the claims.

Questions to Authors
- EM procedure: What is the stopping criterion? Do you monitor total description length and stop when improvements fall below a threshold, or fix the number of iterations? Any guarantees or empirical observations about convergence and stability across random seeds?
- M-step merging: How are merge candidates generated and prioritized? Can merges be undone (e.g., if later evidence suggests over-merge)? How do you prevent cascading errors from early merges?
- Code details: Please clarify “optimal unigram for symbol usages in all rules.” Are you estimating symbol probabilities from rule strings’ usage counts, and using a data-to-model code over histograms? How does this interact with the data-to-model term for rule usages (TΠ)?
- Scalability: What are peak memory and runtime as functions of |D|, average word length, and N? Any heuristics (beam width, A*, pruning by length/frequency) tried to push beyond N=5?
- Data choices: Have you tried IPA or phoneme-level transcriptions, or script normalization, and how do the results change? Can you quantify the effect of mixing Swadesh and cognate-specific sets on NCD and sDL?
- Over-weighting α: How is α chosen in practice? Do you provide guidance or validation curves showing trade-offs between model complexity and downstream utility (e.g., reconstruction accuracy)?

Additional Comments
- Consider comparisons with MrBayes (or BEAST/RevBayes) and with cognate detection/borrowing-aware methods; relevant references include List & Moran (2013) and Kitchen et al. (2009).
- The neighbor-joining tree would benefit from alternative, model-based phylogenies with support values and principled rooting.
- Reporting ablations/baselines (e.g., edit distance, off-the-shelf compressor NCD) would contextualize the gains from your MDL code.
- Please fix minor editorial issues (e.g., “focus in on” → “focus on,” line 440) and clean up several incomplete references in the bibliography.
- Releasing code and data, plus detailed runtime/memory logs, would aid reproducibility.

Summary
The paper introduces an MDL-based framework to discover statistically salient string-level correspondences across two or more languages without imposing parametric assumptions on rule shape or distribution. The model learns a table of multi-language substring rules and their usage costs, aligns cognate tuples via shortest paths in a DAG, and optimizes a two-part code L(M)+L(D|M) with an EM-style procedure that merges rules to reduce description length. Experiments on 13 Slavic languages produce pairwise distances and a neighbor-joining tree; a four-language experiment (CS-PL-RU-BG) demonstrates multiway correspondences and an information-theoretic measure of shared structure among arbitrary subsets. A data overweighting knob (α>1) trades off model complexity and granularity of discovered rules.

Contributions (as I see them)
1) An MDL code and deterministic EM-with-merging training procedure for discovering multi-language substring correspondences that can apply to any subset of languages, not only pairs. This includes an exact shortest-path alignment per model state and a merge heuristic that grows rules by statistical utility.

2) A multiway analysis tool: “shared description length” (sDL) to quantify how much structure is shared by arbitrary subsets of languages, enabling finer-grained similarity analysis than pairwise methods.

3) An empirical case study on Slavic languages showing plausible phylogenetic structure from NCD derived under the model and interpretable correspondences at multiple granularities, with a tunable detail level via data overweighting.

Strengths
- Conceptually clean, assumption-light formulation. The MDL objective naturally balances model size and data fit, avoids heavy priors, and yields interpretable rule inventories. The design aligns with coding-theoretic best practices (two-part MDL, LN for integers, data-to-model codes).

- Multiway correspondences beyond pairwise. The model can learn rules active in any subset of languages and quantify shared structure via sDL. The four-way CS-PL-RU-BG analysis captures both expected groupings (CS-PL, RU-BG) and asymmetric similarities, illustrating value beyond pairwise trees.

- Deterministic learning with exact alignments per model state. The alignment step reduces to shortest paths with Dijkstra and the M-step grows rules via merges that demonstrably reduce total code length, promoting statistically significant longer rules (e.g., cz~č, dzie~dě, stem correspondences).

- Empirical plausibility on Slavic. The 13-language NCD matrix and neighbor-joining tree broadly recover established groupings and known outliers (e.g., Bulgarian/Macedonian morphology, Polish digraphs), and the script effect (Latin vs Cyrillic Serbo-Croatian) is sensibly reflected.

- Practical knob for interpretability. The data-overweighting parameter α provides a useful, transparent way to increase rule granularity when objective evidence is marginal, which can help downstream tasks like reconstruction or pedagogy.

Weaknesses
- Limited comparative evaluation. There is no quantitative comparison with strong baselines in cognate/sound correspondence and phylogenetic inference (e.g., ALINE/PMI aligners; LingPy/LEXSTAT; Bouchard-Côté et al. 2013; work by Nakhleh/Warnow; List & Moran 2013). The phylogeny is not compared against trees from Bayesian methods (MrBayes/BEAST) or distance-based alternatives with uncertainty estimates.

- Scalability constraints. The alignment graph and EM steps create memory/runtime bottlenecks, restricting joint analyses to at most five languages. This undermines one of the main promises of multiway inference at scale and limits broader applicability.

- Methodological choices in phylogeny. Neighbor-joining with manual rooting provides neither support values nor principled rooting. More robust tree-building (e.g., bootstrapping, Bayesian posterior trees) would better substantiate claims about relationships.

- Data and representation confounds. Mixing a curated cognate set with Swadesh lists may confound signal if non-cognates slip in or if borrowings dominate. Using raw orthography risks conflating script/orthographic conventions with phonological correspondences; phonemic or IPA-normalized representations would be preferable. The observed script effects are informative but may partially reflect representation rather than linguistic similarity.

- Clarity of technical details. Two points are underspecified: (i) “optimal unigram for symbol usages in all rules” needs a precise definition—are probabilities estimated from counts in Π, in D alignments, or a joint composition? What is the exact data-to-model enumeration? (ii) The merge step: how are “compatible” rules defined, how is overlap handled, and what guarantees avoid cycles or degenerate merges? Convergence/stopping criteria for EM are not stated; choice of Laplace correction and α’s effect on sDL/NCD are not analyzed.

Questions to Authors
- Can you formally define the unigram code for symbols used in rules: the count source, normalization, and the exact data-to-model code you employ?

- Please specify the merge operation: compatibility constraints, how merged rule probabilities and counts are updated, and the precise objective delta used to accept a merge. Do you consider k>2 merges or beam search?

- What is the EM stopping criterion (relative code-length improvement threshold? max iterations?), and how sensitive are results to initialization (null vs seeded) and Laplace smoothing?

- How does α (data overweighting) quantitatively affect rule inventory, sDL, and NCD? Can you provide sensitivity curves and guidelines?

- Have you tried phonemic/IPA transliteration (e.g., PanPhon/PHOIBLE mappings) to decouple orthographic variation from phonology? How do results change?

- Can you compare your trees and distances to Bayesian phylogenetic baselines (e.g., MrBayes on cognate-coded matrices, BEAST), and to sequence alignment baselines (ALINE/PMI, LingPy/LEXSTAT), including uncertainty/support?

- Will code and data be released to facilitate reproduction and broader comparisons?

Additional Comments
- Consider more scalable alignment (A* with admissible heuristics, pruning/beam search, or subgraph memoization), and/or factorized dynamic programming with rule-length bounds.

- Provide bootstrap or posterior support for tree edges, and avoid manual rooting where possible (e.g., outgroup or midpoint rooting).

- Clarify whether Swadesh entries were manually vetted for cognacy and borrowing; report how borrowings are handled.

- Minor editorial: change “focus in on” to “focus on.”

- Task: cross-lingual NER in direct-transfer setting using high-resource training and no target annotations.
- Core idea: use a cross-lingual wikifier to link n-grams to English Wikipedia and derive language-independent features (FreeBase types, Wikipedia categories).
- Evaluation: extensive experiments on nine languages, including CoNLL and low-resource/non-Latin script languages.
- Contribution 1: novel language-agnostic feature representation via wikification that grounds entities in a shared semantic space.
- Contribution 2: comprehensive empirical evaluation with ablations and analysis linking performance to Wikipedia size.
- Strength: the wikifier features are effective—adding them yields consistent gains, boosting direct-transfer F1 by ~7 points on average.
- Strength: thorough, diverse experiments reveal practical insights (e.g., lexical features hurt transfer to non-Latin scripts).
- Strength: strong comparative performance, outperforming prior baselines without requiring parallel text or native-speaker interaction.
- Weakness: reliance on Wikipedia coverage—handling of out-of-Wikipedia entities is not analyzed.
- Weakness: potential error propagation from wikifier disambiguation is not quantified or analyzed.
- Weakness: combining feature classes is brittle for distant scripts (e.g., Bengali/Tamil), with English lexical features causing catastrophic drops.
- Weakness: unclear and potentially expensive feature extraction (all n-grams up to 4-grams); pruning/conflict resolution not described.
- Question: please report performance gap between in-Wikipedia and out-of-Wikipedia entities.
- Question: provide an error analysis quantifying how many NER errors coincide with incorrect or NIL wikifier outputs.
- Question: consider principled remedies (e.g., feature-class-specific regularization) so the model can down-weight mismatched lexical features automatically.

### Summary of the Paper
This paper proposes a novel, language-independent feature set for Named Entity Recognition (NER) derived from cross-lingual Wikification. The core idea is to use a Wikifier system to map n-grams from a source text (in any language) to their corresponding English Wikipedia articles. This process generates rich, delexicalized features based on the linked Wikipedia entries' categories and FreeBase types. The authors integrate these features into a standard NER model and evaluate their effectiveness in two settings: 1) monolingual NER, where training and testing occur in the same language, and 2) direct transfer, where a model is trained on English data and applied directly to a target language without any target-language annotations. The experiments are conducted on a wide array of nine languages, including standard CoNLL datasets (Spanish, German, Dutch) and several low-resource languages (Turkish, Tagalog, Yoruba, Bengali, Tamil), some of which use non-Latin scripts. The results demonstrate that the proposed Wikifier features consistently improve performance in the monolingual setting and provide a very strong signal for direct transfer, significantly outperforming comparable baselines.

### Contributions
1.  **A Novel Method for Generating Language-Independent NER Features:** The primary contribution is the innovative use of cross-lingual Wikification as an upstream feature-generation process for NER, rather than as a downstream task. By grounding n-grams in a shared, language-agnostic semantic space (English Wikipedia), the authors create a powerful set of features that effectively transfer across languages. This is a significant conceptual shift that proves highly effective for cross-lingual learning.
2.  **A Comprehensive Empirical Validation of Cross-Lingual NER via Direct Transfer:** The paper provides a thorough and valuable empirical study across nine diverse languages. The analysis of performance on low-resource and non-Latin script languages is particularly important, as it highlights both the strengths and weaknesses of the approach in challenging scenarios. The strong results against competitive baselines firmly establish this method as a state-of-the-art approach for direct transfer NER that does not require parallel corpora.

### Strengths
1.  **Elegant and Effective Core Idea:** The central idea of leveraging a cross-lingual Wikifier to bridge the gap between languages for NER is both simple and powerful. It sidesteps the need for parallel data or complex projection mechanisms by mapping lexical items to a universal, structured knowledge base. The substantial performance gains shown in Table 2 (e.g., improving the direct transfer F1 score from 49.66 to 62.10 on Dutch, and 55.04 to 60.97 on German) provide compelling evidence of the features' efficacy.
2.  **Thorough and Broad-Ranging Experiments:** The evaluation is a major strength of this work. Testing on nine languages, including those with non-Latin scripts like Bengali and Tamil, provides a robust assessment of the method's generalizability. The inclusion of both monolingual and cross-lingual experiments clearly delineates the value of the Wikifier features in different settings. Furthermore, the analysis in Section 4.4, which correlates feature quality with Wikipedia size, adds significant depth and explanatory power to the results.
3.  **Strong Performance on Low-Resource Languages:** The method shows particularly promising results on low-resource languages, outperforming a strong baseline (Zhang et al., 2016) that requires more resources (parallel data and human interaction). This demonstrates the practical utility of the proposed approach for languages where annotated data is scarce, which is a key challenge in NLP.

### Weaknesses
1.  **Unclear Handling of Out-of-Wikipedia Entities:** The method's effectiveness is fundamentally dependent on an entity's presence in Wikipedia. The paper does not address how the model performs on entities that are not notable enough for a Wikipedia entry (e.g., local businesses, many personal names) or are too new to have been added. This is a significant real-world limitation. An analysis of the model's recall for in-Wikipedia versus out-of-Wikipedia entities would be necessary to understand the scope of this issue.
2.  **Lack of Analysis on Error Propagation:** The entire feature generation pipeline rests on the performance of the cross-lingual Wikifier. While the authors acknowledge that disambiguation is imperfect, there is no quantitative analysis of how errors from the Wikifier propagate to the final NER predictions. For example, it is unclear how often a wrong link from the Wikifier leads directly to an NER classification error. This analysis is crucial for understanding the system's failure modes and identifying areas for future improvement.
3.  **Suboptimal Feature Combination for Dissimilar Languages:** The direct transfer results for Bengali and Tamil show a catastrophic performance drop when English-trained lexical and gazetteer features are added, with F1 scores falling below 7 (Table 2). The authors correctly diagnose this as overfitting to source-language lexical signals but only suggest a manual, post-hoc strategy of using just the Wikifier features for these languages. A more principled approach would be to investigate techniques to make the model robust to this situation, for instance, by applying different regularization strengths to different feature classes (e.g., heavily penalizing lexical features during transfer) to prevent over-reliance on non-transferable signals.

### Questions to Authors
1.  How does the proposed model handle named entities that are not present in Wikipedia? Could you provide a more detailed analysis of the model's performance (e.g., recall) on in-Wikipedia vs. out-of-Wikipedia entities from the test sets? This would help clarify the practical limitations of the approach.
2.  Could you provide a quantitative analysis of error propagation from the Wikifier? Specifically, what proportion of NER errors on a held-out set can be directly attributed to the Wikifier returning an incorrect Wikipedia link (or no link at all) for an entity mention?
3.  Regarding the poor transfer performance on Bengali and Tamil when all features are used, have you considered more integrated methods to address this beyond manually selecting a feature subset? For instance, could different regularization schemes (e.g., group lasso) be applied to encourage the model to rely less on source-specific lexical features during cross-lingual transfer?

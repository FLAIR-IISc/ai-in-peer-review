Summary
The paper proposes a direct-transfer NER framework that leverages cross-lingual wikification to generate language-independent features (Freebase types and Wikipedia categories) for every n-gram. Models trained on one or multiple source languages are applied to target languages without annotated data, requiring only multilingual Wikipedia. Extensive experiments across nine languages (CoNLL and low-resource) show that wikifier-derived features substantially improve both monolingual and cross-lingual NER, with analyses linking performance to Wikipedia size/interlanguage link coverage and modest gains from multi-source training.

Contributions (as I see them)
- Primary: A strong empirical demonstration that cross-lingual wikification provides effective, truly delexicalized features (types/categories) that substantially enhance direct-transfer NER across diverse scripts and resource conditions.
- Secondary: A broad, carefully executed evaluation across nine languages, including ablations (on interlanguage link availability and Freebase queries) and multi-source training, that provides actionable guidance (e.g., when to rely on wikifier-only features).
- Tertiary: A practical recipe for cross-lingual NER that requires no parallel data and minimal language-specific engineering, and that also improves monolingual NER when added to standard feature sets.

Strengths
- Clear and sizable cross-lingual gains attributable to wikifier features. On CoNLL transfer, adding wikifier features boosts F1 by 10–20 points over base/gazetteer features (e.g., Dutch 25.24→47.14, German 41.81→60.97, Turkish 50.45→65.32). The method surpasses Täckström et al.’s multilingual cluster baseline across all three shared languages.
- Strong low-resource results with minimal assumptions. The model outperforms Zhang et al. (2016) on Turkish/Tagalog/Yoruba/Bengali/Tamil despite using fewer resources (no native speaker, no parallel text), highlighting the utility of type/category features for boundary detection and typing.
- Insightful analysis connecting Wikipedia coverage to performance. The ablation removing Freebase queries and interlanguage links (Table 3) convincingly explains degradation in low-resource settings and the unusually strong performance of wikifier-only models for non-Latin scripts.
- Practicality and generality. Requiring only Wikipedia dumps and a cross-lingual wikifier makes the approach widely applicable. The same features improve monolingual NER across all nine languages (e.g., Base+Gaz→+Wikifier yields consistent gains).
- Multi-source training shows extensibility. Although gains are modest, training on multiple languages generally helps and never dramatically hurts, aligning with the transfer story.

Weaknesses
- Limited technical novelty. Using Wikipedia categories/types for NER (e.g., Kazama & Torisawa, 2007) is known; the main advance is cross-lingual wikification plus careful feature design and evaluation. The paper would benefit from deeper discussion of novelty vs. prior Wikipedia-based NER and joint NER–EL work (e.g., Sil & Yates, 2013).
- Incomplete baselines and orthogonality claims. The paper compares to Täckström et al. but does not include projection baselines (e.g., Wang & Manning, 2014) or combined models (wikifier + multilingual clusters/projection). Since these approaches are claimed orthogonal, presenting preliminary combined results on a few languages would substantiate the claim.
- POS feature assessment is underdeveloped. The dismissal of POS features due to availability/accuracy is unconvincing without experiments. Given prior evidence (Täckström), at least CoNLL languages should include a POS ablation (e.g., UD taggers), to quantify benefit/harm in direct transfer.
- Clarity gaps in Section 3.3. The modifications to Tsai & Roth (candidate generation constraints, ranker features, overlap handling, top-K selection) are hard to follow. A running example tracing an n-gram through candidate generation, ranking, type extraction, and feature instantiation would improve reproducibility.
- Statistical rigor and analysis depth. Improvements in multi-source training (Table 4) are small/inconsistent; no significance tests are reported. Some surprising training–test language pairs (e.g., Dutch→Spanish best; Spanish→Yoruba best) are not explained; more analysis on interlanguage link density, category/type overlap, gazetteer coverage, script/casing/tokenization would strengthen the insights.
- Dependency on Freebase and English alignment. For languages where Freebase IDs are missing (e.g., Yoruba/Bengali/Tamil), wikifier signals are weaker. Alternatives (e.g., Wikidata types, induced type systems) are not explored.

Questions to Authors
- Can you add combined baselines to support orthogonality claims? For instance, wikifier features + multilingual clusters (Täckström) and/or an expectation-projection baseline (Wang & Manning, 2014) on DE/NL/ES.
- Please run a POS ablation on CoNLL languages (monolingual and transfer) using a standard UD POS tagger to quantify cross-lingual utility/harm.
- Can you include a strong projection-like baseline that does not require strict parallel data, e.g., bilingual semantic spaces from comparable corpora (Peirsman & Padó, 2009; Vulić & Moens, 2013) used as features or to align gazetteers?
- Clarify Section 3.3 with a worked example: n-gram candidate set (n≤4), how you avoid spurious matches (“in Germany”→Germany), how overlapping n-grams are resolved, the exact feature templates from top-1/top-2 candidates, and how ranker confidence is used as a linker.
- Report significance tests (e.g., stratified bootstrap) for main tables (especially Table 4) and per-entity-type F1 to understand where gains come from.
- Provide analysis for the best training-language anomalies (nl→es, es→yo): Do these correlate with interlanguage link overlap, category distribution similarity, or gazetteer sizes? Any evidence from error analyses?
- Sensitivity: How do results vary with n (1–4), number of candidates K, use of only categories vs. only Freebase types, and thresholding ranker confidence?
- Will you release the modified cross-lingual wikifier, derived gazetteers, and code to facilitate reproducibility?
- Minor: Please proofread; fix typos and the German example in Figure 1; expand related work to position Kazama & Torisawa (2007) and projection without parallel data (Peirsman & Padó, 2009; Vulić & Moens, 2013) more clearly.

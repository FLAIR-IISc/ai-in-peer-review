- Proposes a direct-transfer NER framework using cross-lingual wikification to produce language-independent Freebase types and Wikipedia categories per n-gram.  
- Trains models on source languages and applies to target languages without annotated data, requiring only multilingual Wikipedia.  
- Extensive experiments on nine languages (CoNLL + low-resource) show wikifier features substantially improve monolingual and cross-lingual NER.  
- Analysis links performance to Wikipedia size and interlanguage link coverage; multi-source training gives modest additional gains.  
- Primary contribution: empirical demonstration that cross-lingual wikification yields effective delexicalized features boosting direct-transfer NER across scripts.  
- Secondary contribution: broad, careful evaluation with ablations (interlanguage links, Freebase queries) offering practical guidance.  
- Tertiary contribution: a practical, low-engineering recipe for cross-lingual NER that also helps monolingual systems.  
- Strong result: large cross-lingual F1 gains from wikifier features (e.g., Dutch, German, Turkish examples) outperforming prior multilingual cluster baselines.  
- Low-resource strength: outperforms prior work on several low-resource languages while using fewer resources (no native speakers or parallel text).  
- Insightful analysis showing how removal of Freebase/interlanguage links degrades performance, especially in low-resource settings.  
- Practicality: approach requires only Wikipedia dumps and a cross-lingual wikifier and improves NER across all tested languages.  
- Multi-source training generally helps and does not harm, though gains are modest.  
- Weakness: limited technical novelty—basis on known Wikipedia/type features; main novelty is cross-lingual wikification + evaluation.  
- Weakness: incomplete baselines—lacks projection baselines and combined-model comparisons to substantiate orthogonality claims.  
- Weakness: POS feature dismissal is underdeveloped; experiments with UD POS taggers on CoNLL languages are needed.  
- Weakness: Section 3.3 modifications (candidate gen, ranking, overlap handling) are unclear and need a worked example for reproducibility.  
- Weakness: statistical rigor lacking—no significance tests; small/inconsistent multi-source gains and unexplained training→test anomalies.  
- Weakness: dependency on Freebase and English alignment weakens signals for languages missing Freebase IDs; alternatives (Wikidata, induced types) not explored.  
- Request: add combined baselines (wikifier + multilingual clusters, expectation-projection) on key languages to support orthogonality claims.  
- Request: run POS ablation on CoNLL languages (monolingual and transfer) using standard UD taggers.  
- Request: include a projection-like baseline that uses comparable corpora or bilingual semantic spaces as a non-parallel strong baseline.  
- Request: clarify Section 3.3 with a worked example showing candidate generation, overlap resolution, feature templates, and ranker use.  
- Request: report significance tests (e.g., stratified bootstrap) for main results and per-entity-type F1 analyses.  
- Request: analyze anomalous best training-language pairs (nl→es, es→yo) for correlations with interlanguage links, category overlap, or gazetteer sizes.  
- Request: provide sensitivity analyses for n (1–4), number of candidates K, categories vs. Freebase types, and ranker confidence thresholds.  
- Request: release modified cross-lingual wikifier, derived gazetteers, and code to aid reproducibility; fix typos and expand related work comparisons.

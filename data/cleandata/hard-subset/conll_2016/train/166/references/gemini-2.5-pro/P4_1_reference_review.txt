### Summary of the Paper
This paper proposes a novel approach for cross-lingual Named Entity Recognition (NER) in a direct transfer setting, where a model is trained on a high-resource language and applied to a low-resource target language without any target-language annotations. The core contribution is the use of a cross-lingual wikifier to generate language-independent features. For any given text, the method attempts to link every n-gram (up to n=4) to an English Wikipedia entry. The corresponding FreeBase types and Wikipedia categories from the linked entries are then used as features in a standard NER model. This reverses the traditional pipeline of NER followed by entity linking, instead using linking signals to inform NER. The approach is evaluated extensively on nine languages, including CoNLL datasets (Spanish, Dutch, German) and five low-resource languages (Turkish, Tagalog, Yoruba, Bengali, Tamil), some with non-Latin scripts. The proposed model demonstrates significant improvements over strong baselines that use delexicalized features or parallel corpora, and the authors provide a detailed analysis of the method's performance in relation to the size of the target language's Wikipedia.

### Main Contributions
1.  **A novel method for direct transfer NER using cross-lingual wikification for feature generation.** The central and most significant contribution is the idea of grounding all n-grams in a text to a canonical knowledge base (English Wikipedia) to extract language-agnostic semantic features (e.g., FreeBase types). This provides a principled and effective way to create delexicalized features that capture entity type information, directly addressing the core challenge of direct transfer.
2.  **A comprehensive empirical evaluation and analysis across a diverse set of languages.** The paper presents a strong empirical case for the proposed method across nine languages, ranging from high-resource to low-resource and including non-Latin scripts. The experiments not only demonstrate state-of-the-art results compared to relevant direct transfer methods but also include valuable monolingual experiments and an insightful analysis (Sec 4.4) that correlates feature quality with the size of the underlying Wikipedia resource.

### Strengths
1.  **High Novelty and Technical Soundness.** The core idea of reversing the typical NER->Wikification pipeline to use a wikifier as a feature extractor for NER is both novel and well-motivated. It provides a robust mechanism to map surface forms from any language to a shared, language-independent semantic space (English Wikipedia concepts), from which strong typing features can be derived. This is a more direct way of capturing semantic information than relying on proxies like cross-lingual word clusters.
2.  **Significant Empirical Improvements over Strong Baselines.** The proposed model significantly outperforms the direct transfer method of Täckström et al. (2012) on CoNLL languages, even when the baseline uses multilingual word clusters derived from parallel text. More impressively, the model outperforms Zhang et al. (2016) on five low-resource languages, despite the fact that the baseline uses more resources (parallel corpora, monolingual corpora, and human interaction). These results strongly validate the effectiveness of the proposed features.
3.  **Thorough and Broad Evaluation.** The experiments cover nine languages with diverse typological properties and resource levels. The inclusion of non-Latin script languages (Bengali, Tamil) provides a stringent test for the direct transfer approach and highlights the strengths and limitations of different feature sets. The paper also includes monolingual results (showing wikifier features help even with in-language training), multi-source training experiments, and a detailed analysis of performance as a function of training language. This comprehensive evaluation framework strengthens the paper's conclusions.
4.  **Insightful Analysis of Method's Dependencies.** The ablation study in Section 4.4, which simulates a reduction in Wikipedia resources by removing interlanguage links, is a key strength. It provides a clear, quantitative link between the size of the target language's Wikipedia (and its intersection with English) and the quality of the wikifier features. This analysis offers a valuable explanation for the performance variance across languages and helps define the conditions under which the proposed method is most effective.

### Weaknesses
1.  **High Dependence on Wikipedia Quality and Coverage.** The method's effectiveness is fundamentally tied to the quality of the cross-lingual wikifier, which in turn is highly dependent on the size and quality of the target language's Wikipedia and its linkage to the English Wikipedia. The paper's own analysis confirms this, showing that performance degrades substantially as resources shrink. This limits the method's applicability for truly low-resource or under-documented languages that have a minimal or non-existent Wikipedia presence. The reliance on English Wikipedia as the canonical knowledge base may also introduce a cultural or topical bias.
2.  **Unaddressed Computational Cost.** The proposed feature generation process requires running a cross-lingual wikifier on every n-gram (n≤4) in the input text. This appears to be a computationally intensive process involving candidate generation and ranking for a quadratic number of text spans. The paper makes no mention of the computational overhead (in terms of training or inference time) compared to the baseline models. This is a significant practical consideration that is not addressed.
3.  **Ambiguity in Handling Non-Latin Scripts.** The experiments show that for non-Latin script languages (Bengali, Tamil), the model with all features performs very poorly, and a model using *only* wikifier features is superior. The authors correctly identify that lexical features trained on English do not transfer. However, the paper proposes a manual strategy: "using wikifier features could be the best setting." This is not a scalable solution. A more principled approach, such as automatically detecting script mismatch to disable lexical features, would be a more robust contribution.
4.  **Lack of Detail on Feature Integration.** Section 3.3 briefly describes the wikifier features but lacks some key implementation details. For example, it is unclear how features from multiple, overlapping n-grams (e.g., "New York" and "New York City") are aggregated for a single token ("York"). Furthermore, it is not specified how the features from the top 2 title candidates are combined or represented. These details are important for reproducibility.

### Questions to Authors
1.  Could you please quantify the computational cost of the proposed feature extraction step? Specifically, what is the increase in inference time for your full model ("+Wikifier") compared to the baseline with only base and gazetteer features?
2.  The results for Bengali and Tamil show that including lexical features from an English-trained model is highly detrimental. Have you considered an automated strategy to handle this, for example, by disabling lexical word-form features at test time if a script mismatch between the source and target language is detected?
3.  How does the model handle n-grams for which the wikifier returns NIL (i.e., no link to Wikipedia)? Is there an explicit "NIL" feature, or is this simply represented by the absence of any wikifier-derived features for that span?

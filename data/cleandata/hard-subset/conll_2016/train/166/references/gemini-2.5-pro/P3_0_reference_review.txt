### Summary and Contributions

This paper presents a method for cross-lingual named entity recognition (NER) in a direct transfer setting. The core idea is to generate language-independent features by leveraging a cross-lingual wikifier. Instead of following the traditional pipeline of NER followed by entity linking, this work proposes to wikify all n-grams (n≤4) in a given text, linking them to the English Wikipedia. The Wikipedia categories and Freebase types associated with the linked English Wikipedia entities are then used as features for a standard sequence labeling model. This approach requires only a multilingual Wikipedia dump and no parallel corpora, making it applicable to a wide range of languages.

The main contributions of this work, in my view, are:
1.  **A novel and effective method for generating language-independent features for NER.** The idea of using dense wikification of all n-grams to provide semantic type features is a creative and powerful way to bridge the lexical gap between languages. It cleverly repurposes entity linking technology as a feature engineering tool for an upstream task.
2.  **A comprehensive and rigorous empirical evaluation.** The authors validate their method on nine languages, spanning well-resourced CoNLL languages and five low-resource languages, including some with non-Latin scripts. The experiments demonstrate significant improvements over strong baselines, including prior work that relies on more extensive resources like parallel corpora.
3.  **Insightful analysis into the factors affecting performance.** The paper includes a valuable ablation study (Table 3) that directly investigates how performance is correlated with the size of a language's Wikipedia and the density of its inter-language links to English, providing a clear explanation for the performance variance observed across different languages.

### Strengths

1.  **Methodological Elegance and Simplicity:** The core concept is powerful yet straightforward. By grounding all potential mentions to a common, language-agnostic semantic space (English Wikipedia), the model effectively delexicalizes the NER problem. This approach elegantly sidesteps the need for complex projection mechanisms or word alignments from parallel corpora, which are often a bottleneck for low-resource languages.
2.  **State-of-the-Art Results with Fewer Resources:** The proposed model significantly outperforms the direct transfer baseline from Täckström et al. (2012), even their model enhanced with cross-lingual word clusters derived from large parallel corpora. Even more impressively, it surpasses the results of Zhang et al. (2016) on five low-resource languages, a system that requires parallel text, a large monolingual corpus, *and* an hour of interaction with a native speaker. Achieving superior performance with only access to Wikipedia is a very strong result and highlights the practical utility of this work.
3.  **Thorough Ablation Studies and Analysis:** The paper is exemplary in its analysis. The monolingual experiments (Table 2) effectively establish the value of the wikifier features even in a high-resource setting. The direct transfer experiments clearly show where the features provide the most benefit. The analysis in Section 4.4, which simulates a low-resource setting for Spanish and German by removing inter-language links, provides convincing evidence that the quality of the wikifier features is the primary driver of performance. This level of analysis adds significant depth and credibility to the paper's claims.
4.  **Broad Applicability:** The method's minimal resource requirement (a Wikipedia dump) makes it immediately applicable to hundreds of languages. The demonstration that it works well for languages with non-Latin scripts (e.g., Bengali, Tamil), where lexical feature transfer completely fails, is a particularly important finding.

### Weaknesses

1.  **Computational Cost and Scalability:** The proposed method involves running a cross-lingual wikifier on *every* n-gram up to n=4. This seems computationally prohibitive for any real-world application, as it implies a massive number of queries to the wikification system for each sentence. The paper does not discuss the runtime complexity or efficiency of this approach. For this method to be practical, some discussion of potential optimizations (e.g., pruning candidate n-grams based on heuristics, batch processing) is necessary.
2.  **Dependence on an Anglocentric Knowledge Base:** The entire framework is anchored to the English Wikipedia as the universal pivot. This introduces a potential bias, as entities and concepts that are culturally specific and not well-represented in the English Wikipedia may be poorly recognized. While English is the largest Wikipedia, this reliance limits the model's ability to capture nuances present in other languages' knowledge structures. The paper would benefit from a discussion of this limitation.
3.  **Use of a Dated Base Model:** The work is built upon the Ratinov and Roth (2009) model, which is a linear classifier (averaged perceptron). While this choice is reasonable for isolating the impact of the novel features, the field of NER has largely moved to neural architectures (e.g., BiLSTM-CRF, Transformers). It is unclear whether the wikifier features would provide similarly large gains on top of strong contextualized representations from models like mBERT, which may already implicitly capture some of this semantic type information. The claims would be stronger if they were demonstrated on a more modern base model or if this point was discussed as future work.
4.  **Minor Lack of Clarity on Feature Interaction:** The paper explains that for non-Latin scripts in the direct transfer setting, using "Wikifier only" features is best because lexical features trained on English are useless. However, the multi-source training experiments in Table 4 state that "all features" are used. It is not clear if this means lexical features from multiple source languages are combined, and how that is handled when testing on a non-Latin script language like Yoruba. A bit more clarity on the feature configuration in the multi-source setting would be helpful.

### Questions to Authors

1.  Could you please elaborate on the computational cost of the dense n-gram wikification process? What is the approximate processing time per document, and have you explored any strategies to make this feature extraction more efficient for practical deployment?
2.  Your work demonstrates impressive gains over a strong linear baseline. How do you hypothesize these wikification features would perform in conjunction with modern pre-trained multilingual language models (e.g., mBERT, XLM-R)? Would you expect the gains to be additive, or do you think the contextual embeddings already capture much of the semantic information your features provide?
3.  The framework relies heavily on English Wikipedia as a pivot. Have you considered the potential anglocentric biases this might introduce? Could the method be adapted to use a different pivot language, or perhaps a more decentralized system like Wikidata, to potentially mitigate this?

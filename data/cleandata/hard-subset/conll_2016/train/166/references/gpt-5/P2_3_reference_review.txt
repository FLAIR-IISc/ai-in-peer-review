Summary and claimed contributions
- The paper proposes a direct-transfer NER framework that uses cross-lingual wikification to generate language-independent features (Freebase types, Wikipedia categories, and wikifier ranker signals). A standard perceptron NER model (Ratinov & Roth, 2009) is trained on one or multiple source languages and applied to target languages without target-side annotation, parallel text, or word-aligned data; only multilingual Wikipedia is required.
- Two practical adaptations to a cross-lingual wikifier (Tsai & Roth, 2016) enable linking every n-gram (n ≤ 4): restrict candidate generation for n>1 to whole-string queries and remove document-level mention-embedding features to make exhaustive n-gram linking feasible.
- Empirical results across nine languages show: (a) wikifier features help monolingual NER; (b) in direct transfer, wikifier features deliver large gains over non-lexical/lexical/gazetteer baselines and outperform prior direct-transfer baselines that need bitext clusters; (c) ablations quantify dependence on Wikipedia size and interlanguage links; (d) multi-source training yields small additional gains.

Main contributions (as I see them)
1) Demonstration that cross-lingual wikification-derived types/categories are strong delexicalized features for cross-lingual NER, enabling competitive direct transfer without parallel or target-labeled data.
2) A practical, resource-light pipeline that works across many languages with clear ablations linking performance to Wikipedia coverage.
3) Analysis isolating the role of Freebase and interlanguage link coverage, offering insight into when the approach will or will not work.

Strengths
- Strong empirical signal from wikifier features for transfer: On CoNLL languages and Turkish, adding wikifier features yields double-digit F1 improvements over base+gazetteers (e.g., +10–15 on NL/DE/TR), and outperforms Täckström et al. (2012) even when their model uses bitext-induced clusters. This supports the central claim that these are effective delexicalized features.
- Resource efficiency and broad applicability: The model requires only multilingual Wikipedia and an existing cross-lingual wikifier. No parallel data, alignments, or target-language annotation is needed, making it attractive for low-resource settings. The exhaustive n-gram approach lets the model help with both entity detection and typing, not only type refinement.
- Useful ablations and diagnostics: The paper carefully quantifies the dependence on Freebase types and English interlanguage links by ablating the Freebase query and randomly removing links (Table 3), and shows how performance degrades as coverage decreases. The “Wikifier only” rows across settings isolate the contribution of these features.
- Coverage of diverse scripts and settings: Experiments include Latin and non-Latin scripts (TR, TL, YO, BN, TA). The observation that “Wikifier only” is strongest for non-Latin scripts (where lexical features don’t transfer) is actionable guidance.
- Multi-source training shows no harm and modest gains: Training on multiple languages generally helps or remains neutral, fitting the direct-transfer story and suggesting future combinations with other transfer methods.

Weaknesses
- Heavy reliance on Freebase and English categories: The approach crucially depends on Freebase types and interlanguage links to English. Freebase is deprecated, and coverage is sparse for some low-resource languages, which the authors’ ablations confirm causes large drops. Portability to Wikidata (or other KBs) should be addressed empirically; otherwise reproducibility and longevity are at risk.
- Baselines and model choices are dated: The core NER model is a 2009 perceptron with hand-crafted features and Wikipedia gazetteers. There is no comparison to modern neural cross-lingual NER (e.g., multilingual embeddings or contextual models) that are standard in recent years. This weakens claims of state of the art and limits the paper’s contemporary impact.
- Evaluation consistency and clarity: The paper remaps or removes labels (e.g., MISC→LOC in some cases, TTL/TIME removed, MISC ignored for CoNLL when testing low-resource), which complicates cross-system comparability. For comparisons with Zhang et al. (2016), it is unclear whether the best per-language setting (e.g., “Wikifier only” for BN/YO) is used; the main table shows +Wikifier rows far below Zhang for YO/BN, yet the text claims an across-the-board win. The paper should explicitly state which configuration is used for each comparison.
- Missing practical details: Exhaustively wikifying all n-grams could be computationally heavy. The paper lacks runtime/throughput, memory, and thresholding details (e.g., NIL handling and confidence-based rejection), which affects scalability and reproducibility. Hyperparameters and exact feature templates are described only at a high level.
- Limited analysis beyond coverage: Error analysis by entity type and failure modes (e.g., boundary errors vs. typing errors, effect of incorrect disambiguation) would strengthen the case and reveal how to mitigate weaknesses in small Wikipedias.

Questions to authors
- Comparisons: For low-resource languages, when claiming to outperform Zhang et al. (2016), do you select the “Wikifier only” configuration for non-Latin scripts (BN/YO), or the cumulative “+Wikifier” configuration? Please report the exact numbers used for the comparison and ensure label sets match.
- KB dependence: Can the method be ported from Freebase to Wikidata? Which features (types, properties, categories) would you use, and do you anticipate similar performance? Any preliminary results?
- Scalability: What is the runtime and memory footprint of wikifying every n-gram (n ≤ 4) on typical documents, and how does it scale with document length and language Wikipedia size?
- Linking details: How are NILs handled? Is there a confidence threshold or a learned linker to suppress low-confidence candidates when creating features? How sensitive are results to using top-1 vs. top-2 candidates?
- Significance and per-type scores: Are improvements statistically significant? Can you provide per-entity-type F1 to better understand where wikifier features help most?

Overall assessment
- Novelty: Moderate (new combination/usage of cross-lingual wikification as delexicalized features and exhaustive n-gram linking).
- Soundness: Solid, with clear ablations; some methodological details are under-specified.
- Significance: Good for resource-light direct transfer, but diminished by reliance on Freebase and lack of comparison to modern neural baselines.
- Clarity and reproducibility: Generally clear, but with evaluation inconsistencies and missing implementation details.

If updated to rely on Wikidata and evaluated against strong neural cross-lingual baselines, this would be a compelling, practical contribution. As is, I find it promising but held back by dated baselines and KB dependencies.

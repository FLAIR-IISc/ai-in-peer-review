Summary and claimed contributions
- The paper proposes using cross-lingual wikification to derive language-independent features (Freebase types and Wikipedia categories) for NER, enabling direct transfer from high-resource source languages to target languages without parallel data.
- It presents empirical results on CoNLL (EN, ES, DE, NL) and five low-resource languages (TR, TL, YO, BN, TA), with ablations showing contributions of base, gazetteer, and wikifier-derived features. It further analyzes the effect of Wikipedia size and reports small gains from multi-source training.
- It modifies an existing cross-lingual wikifier to link all n-grams (n ≤ 4) and uses top-2 candidates’ types plus ranker features as NER features.

Strengths
- Clear, simple idea with broad applicability: Leveraging cross-lingual wikification to create delexicalized features is conceptually clean, requires only multilingual Wikipedia, and avoids parallel corpora or target-language annotations. This is attractive for truly low-resource scenarios.
- Consistent empirical signal from wikifier features: Ablations show wikifier features help in both monolingual and direct transfer setups, often substantially (e.g., +10 F1 in some cross-lingual settings), supporting the core hypothesis that KB-derived types generalize across languages.
- Sensible analyses on feature quality: The study varying interlanguage link density and Freebase query availability is insightful, demonstrating why performance degrades with smaller Wikipedias. This connects observable resource characteristics to model behavior.
- Breadth of languages and scripts: Including five low-resource languages with non-Latin scripts strengthens external validity and underscores the limitations of lexical features and the relative value of KB features.
- Multi-source training observation: Although modest, the reported gains from training on multiple source languages suggest the approach composes reasonably with additional supervision.

Weaknesses
- Reproducibility and missing critical details: Important implementation specifics are absent. The NER training setup (perceptron iterations, regularization, decoding details), precise feature templates, the exact set of Freebase types and Wikipedia categories used, NIL/linker thresholds, stopword lists per language, tokenization for low-resource scripts, and candidate-ranking features are not documented sufficiently. No code or model release is mentioned. The wikifier relies on Freebase API, which is now deprecated; no alternative or static dump is specified, compromising reproducibility.
- Evaluation protocol concerns and non-standard label remapping: The paper removes TIME/TTL tags and heuristically maps MISC to LOC when Freebase indicates a location, then drops the rest of MISC. This can substantially alter task difficulty and comparability to prior work, but the justification, impact analysis per entity type, and alignment with compared baselines (e.g., Zhang et al., 2016) are unclear. It risks inflating performance for models that use KB signals. Further, using Wikipedia-derived gazetteers and types without time-bounding may introduce leakage relative to CoNLL-era data.
- Missing and outdated baselines: There is no comparison to contemporary cross-lingual transfer baselines (e.g., mBERT/XLM-R fine-tuned on English, multilingual entity-aware models), which are standard for zero-shot NER and likely competitive or superior. The strongest comparison is to Täckström et al. (2012) and Zhang et al. (2016), which predates multilingual transformers. Without these baselines, the practical significance is unclear.
- Presentation and formatting issues: The manuscript contains many errors and omissions that hinder assessment: typos (“a the base model”), inconsistent capitalization (“FreeBase”/“Freebase”), diacritics, footnotes embedded in text (“2”, “3”), a garbled Figure 1, references to Figure 2 and Table 4 without the actual content, and a crowded, hard-to-parse Table 2 (unclear column mapping for direct transfer; inclusion of EN in transfer table?). These are non-trivial and suggest the submission is not publication-ready.
- Methodological clarity gaps and potential noise: The decision to link every n-gram up to length 4 is aggressive; how overlapping spans are handled, how conflicting type signals are resolved, and the computational cost are not reported. The linker confidence is vaguely described; no threshold selection protocol or calibration is provided. The choice of using top-2 candidates (vs. top-1) and the contribution of wi±1 types are not ablated. POS is dismissed without a systematic cross-lingual alternative (e.g., universal POS), and there is no statistical significance testing.

Questions to authors
- Reproducibility: Please provide full hyperparameters for the perceptron model (iterations, averaging schedule, beam/greedy decoding specifics), exact feature templates (including the inventory of Freebase types and Wikipedia category features used), stopword lists per language, tokenization for low-resource scripts, and linker/NIL thresholds. Will you release code, models, and a frozen KB snapshot to address Freebase deprecation?
- Label remapping: Precisely detail the remapping rules (MISC→LOC mapping criteria, removal of TIME/TTL) and whether all compared baselines used the same protocol. Provide per-type F1 before/after remapping to quantify its impact.
- Baselines: Can you add zero-shot cross-lingual baselines with mBERT/XLM-R (fine-tuned on EN CoNLL) and report under your evaluation setting? Also consider a strong monolingual EL+NER pipeline baseline in target languages where training data exists, to contextualize gains from wikifier features.
- Wikifier specifics: What is the candidate generation recall when restricting to whole-phrase queries? How are overlapping n-grams handled and aggregated? What is the runtime overhead of linking all n-grams n ≤ 4? Did you experiment with top-1 vs top-2 candidates or confidence thresholds?
- Missing artifacts: Please include Figure 2 and Table 4 with numeric values, ensure Table 2 columns align with languages (especially in direct transfer), and correct the numerous formatting issues so results are unambiguous.

Overall, while the core idea is interesting and the empirical evidence generally supports it, the combination of missing key details, non-standard evaluation manipulations, absence of modern baselines, and significant presentation problems currently outweighs the strengths. I recommend substantial revision before this work can be reliably evaluated and reproduced.

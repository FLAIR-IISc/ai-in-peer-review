

**Summary**
This paper introduces a language-independent approach to cross-lingual named entity recognition (NER) that leverages cross-lingual wikification to generate language-agnostic features. The core contribution is the use of a cross-lingual wikifier to ground phrases in any language to English Wikipedia entries, extracting features from Wikipedia categories and FreeBase types. These features enable direct transfer of an English-trained NER model to multiple target languages without requiring language-specific resources or parallel data.

**Main Contributions**
1. A novel language-independent feature set derived from cross-lingual wikification that enables direct transfer of NER models across languages.
2. Comprehensive empirical validation across both high-resource languages (Spanish, German, Dutch) and low-resource languages (Turkish, Tagalog, Yoruba, Bengali, Tamil), demonstrating significant improvements over strong baselines.
3. Demonstration that these language-independent features enhance monolingual NER systems, providing dual utility for both cross-lingual transfer and monolingual improvement.

**Strengths**
1. The paper presents a conceptually simple yet effective solution to cross-lingual NER that requires only multilingual Wikipedia as input, eliminating the need for complex parallel data or language-specific resources. This addresses a key practical limitation in cross-lingual NLP.
2. The experimental evaluation is exceptionally thorough, covering both standard CoNLL datasets and challenging low-resource languages with non-Latin scripts. The analysis of Wikipedia size impact on feature quality provides valuable insights beyond typical empirical claims.
3. The paper makes a strong case for the practical utility of the approach, showing state-of-the-art results on multiple languages while demonstrating that the same features improve both cross-lingual transfer and monolingual NER systems.

**Weaknesses**
1. The paper does not adequately analyze the dependency on wikifier accuracy. The method's success hinges on the cross-lingual wikifier's performance, yet the paper lacks analysis of how often wikifier errors propagate to NER errors. For instance, how many NER errors can be directly traced to incorrect wikifier results?
2. The paper fails to address a critical limitation: named entities that do not exist in Wikipedia. The approach relies on Wikipedia coverage, but many proper names (e.g., new organizations, local entities) won't have corresponding Wikipedia entries. The paper provides no analysis of how this affects performance for such entities.
3. The performance degradation on Bengali and Tamil when lexical features are added (Table 2) suggests a fundamental imbalance in feature weighting. The paper doesn't explore how to better balance feature contributions, which is particularly important for non-Latin script languages where lexical features fail.

**Questions to Authors**
1. How does the performance of the proposed approach degrade for named entities without corresponding Wikipedia entries? Can you provide specific examples or statistics on this limitation, particularly for entities that are commonly found in real-world text but lack Wikipedia coverage?
2. What is the error rate of the cross-lingual wikifier, and how often do these errors directly impact NER performance? A detailed error analysis would strengthen the claims about the method's robustness.
3. For languages with non-Latin scripts (Bengali, Tamil), the paper shows that using only wikifier features yields better results than adding lexical features. Could you discuss how the model could be modified to automatically weight features differently based on language characteristics, rather than requiring manual feature selection for each language?

**Summary and Contributions**

This paper proposes a method for cross-lingual Named Entity Recognition (NER) in a direct transfer setting, where a model is trained on a high-resource language and applied to a target language without any annotated data in that language. The central contribution is the use of a cross-lingual wikifier to generate powerful, language-independent features. By grounding n-grams from a source text in any language to English Wikipedia entries, the model can leverage the associated Wikipedia categories and FreeBase types as features. The work demonstrates that this approach not only significantly improves performance over strong baselines in the direct transfer setting for both high-resource (CoNLL languages) and low-resource languages but also enhances performance in standard monolingual NER tasks.

The main contributions of this work are:
1.  A novel and effective method for creating language-independent features for NER by repurposing cross-lingual wikification. This approach cleverly uses the rich semantic information in Wikipedia as a pivot across languages, requiring only a multilingual Wikipedia dump rather than parallel corpora.
2.  An extensive empirical evaluation across nine languages with varying resource levels and writing systems. The results establish a new state-of-the-art for direct transfer NER under these resource constraints, showing particular strength on languages where lexical overlap is minimal.
3.  An insightful analysis of the model's performance, correlating the quality of the wikifier features with the size of the target language's Wikipedia and demonstrating the benefit of training on multiple source languages.

**Strengths**

1.  **Novelty and Simplicity:** The core idea of using features from a cross-lingual wikifier for NER is elegant. It effectively reverses the traditional NER-then-linking pipeline to show that entity disambiguation can be a powerful signal for entity identification itself. The method is conceptually straightforward and relies on a widely available resource (Wikipedia), making it broadly applicable and easy to understand.

2.  **Strong Empirical Results:** The paper presents compelling evidence for the proposed method's effectiveness. The model significantly outperforms the comparable direct transfer baseline from Täckström et al. (2012) on the standard CoNLL datasets. More impressively, it also achieves better results than the method of Zhang et al. (2016) on five low-resource languages, despite requiring substantially fewer resources (i.e., no parallel data or interaction with native speakers). The consistent gains shown in the monolingual setting further underscore the general utility of the wikifier features.

3.  **Thorough Analysis and Evaluation:** The experimental design is comprehensive. The evaluation spans a diverse set of languages, including those with non-Latin scripts, which robustly tests the method's language independence. The analysis in Section 4.4, which investigates the performance degradation by artificially limiting inter-language links, provides a clear and convincing explanation for the performance differences across languages and highlights the dependency on Wikipedia's structure. Furthermore, the experiments with multi-source training in Section 4.5 are a valuable addition, showing a practical path to further performance improvements.

**Weaknesses**

The work is solid and the contributions are clear. The following points are suggestions for potential improvements.

1.  **Dependency on the Wikifier:** The method's performance is fundamentally tied to the quality of the underlying cross-lingual wikifier. While the paper effectively demonstrates that performance correlates with Wikipedia size, a qualitative analysis of error propagation could be beneficial. For instance, are certain types of NER errors (e.g., boundary errors vs. type confusion) more common as a result of specific wikification failures? Understanding this relationship could provide insights for future work on joint modeling or error correction.

2.  **Positioning with Respect to Modern Architectures:** The NER model is based on a linear classifier from 2009. While this is a reasonable choice for isolating and evaluating the contribution of the novel features, the field has largely moved towards neural architectures (e.g., BiLSTM-CRF). It would strengthen the paper's impact to include a brief discussion on how these powerful wikifier features could be integrated into modern neural models, for example, as additional input features concatenated with word embeddings. This would provide a clearer roadmap for other researchers to build on this work.

3.  **Clarity of Feature Representation:** Section 3.3 describes the set of features derived from the wikifier, mentioning that types from n-grams containing a word `wi` are used as features for `wi`. This description could be clarified with a concrete example. For a given token, it would be helpful to illustrate precisely which features are generated from the various unigrams, bigrams, etc., that contain it, and how they are represented in the feature vector. This would enhance the replicability of the work.

4.  **Comparison to Projection Methods:** The related work section discusses projection-based methods, but the experiments do not include a direct comparison. While the proposed approach has the clear advantage of not requiring parallel corpora, a brief discussion in the analysis about the expected trade-offs would be helpful. For example, in a scenario where a small parallel corpus is available, how might this method's performance and resource requirements compare to a projection-based approach? This would help to more fully contextualize the contribution within the broader field of cross-lingual NLP.

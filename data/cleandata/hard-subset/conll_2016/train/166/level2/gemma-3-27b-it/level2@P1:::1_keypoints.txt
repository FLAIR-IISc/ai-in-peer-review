- Proposes cross-lingual wikification to English Wikipedia as language-independent features for NER, grounding mentions across languages to rich English semantics.  
- Demonstrates strong performance improvements on several low-resource languages (Turkish, Tagalog, Yoruba, Bengali, Tamil), outperforming methods relying on parallel text or native speakers.  
- Shows that wikification-derived features can also improve monolingual NER systems using language-specific training data.  
- Novel reframing of wikification from a downstream task to a feature-engineering technique that provides valuable disambiguation signal for NER.  
- Comprehensive experiments across CoNLL and diverse low-resource languages with strong ablation studies clarifying feature contributions.  
- Clear, well-written exposition with detailed dataset/evaluation info that supports reproducibility.  
- Comparisons to strong baselines (e.g., Täckström et al., Zhang et al.) show clear advantages, especially in low-resource settings.  
- Analysis of Wikipedia size and multi-source training effects (e.g., Spanish as a useful source for Yoruba) provides useful insights.  
- Reliance on English Wikipedia may introduce bias, needing deeper investigation for languages far from English.  
- Model performance depends on wikifier accuracy; errors can propagate to NER and warrant deeper error analysis of the wikifier.  
- Feature set is relatively simple (based on Ratinov & Roth 2009); richer/contextual features from wikified entities might improve results.  
- Lacks detailed error analysis specific to low-resource languages (types of errors like wrong type or boundary mistakes).  
- Asks authors to discuss mitigation strategies for English-Wikipedia bias beyond alternative KBs.  
- Requests a detailed error analysis of the cross-lingual wikifier and which error types most affect NER.  
- Questions whether adding more source languages yields diminishing returns and what the optimal number of source languages is.  
- Asks about sensitivity to removing the MISC tag in low-resource languages and whether remapping (e.g., MISC→LOC) would change results.  
- Notes Table 2 notation could be clearer (clarify abbreviations).  
- Suggests discussing computational cost of wikifying every n-gram as a potential limitation for large datasets.  
- Recommends clarifying Figure 2’s legend or color scheme for readability.  
- Overall assessment: strong, well-executed paper with innovative approach and compelling results; recommends acceptance.
